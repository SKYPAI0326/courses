Make 定時抓網頁入門

1 小時學會建立每日 09:00 自動執行的 Scenario，串接 RSS Module 抓 BBC 新聞並寫入 Google Sheets，在 Sheet 中親眼看到新增的一列。

學習成果 · OUTCOMES
能在紙上畫出 Scenario / Module / Trigger / Connection 四概念的層級關係圖
能獨立建立「每日 09:00 自動觸發」Scenario，並在 Make 畫面看到下次預定執行時間
能串 RSS Module 抓網頁資料，從 Output Bundle 指出至少 3 個可用欄位
能把抓到的資料寫入 Google Sheets，在 Sheet 中親眼看到新增的一列
(01) · 場景切入

阿凱每天花 30 分鐘手抄新聞，能不能自動化？

阿凱是行銷工作室的行銷專員，他每天早上第一件事都是打開 BBC News、TechCrunch、競品官網、3 個產業部落格，花 30 分鐘看有沒有值得記錄的新文章，抄進 Google Sheet 給老闆週報用。

三個月下來他算了一下：每天 30 分鐘「抓網頁內容」，一週 2.5 小時。一小時 300 塊，一個月浪費的人力成本超過 6,000 塊。還不包括漏看、重複記錄的成本。

他聽同事提過 Make.com 可以自動化，打開註冊頁看了 3 分鐘——介面上冒出 Scenario、Module、Trigger、Connection 四個名詞，他看不懂哪個是哪個，關掉頁面回去手抄。

本單元不會把 Make 教完，
只會教一條最小可用路線：
把網頁 RSS 自動抓進 Sheet。
(02) · 關鍵概念

Make 只要搞懂這四個詞

一句話記住每一個，後面都是拼積木。

Scenario
流程 · 最外層
一條完整的自動化流程，就像一張 A4 紙上畫的工作流。你建立的「自動做某件事的流程」，在 Make 裡就叫 Scenario。
Module
節點 · 做事單位
Scenario 這張紙上每一個節點，一個節點做一件事（抓網頁、寫 Sheet、發信）。
Trigger
觸發器 · 第一個 Module
Scenario 的第一個 Module，決定這條流程「什麼時候開始跑」。可以是每天 09:00、收到新郵件、網頁有更新。
Connection
連線設定 · 帳號憑證
當 Module 要碰外部帳號（Gmail、Sheets、Notion）時，Make 需要一個「驗證過的帳號憑證」。一次設定，多個 Scenario 共用。
💡
層級關係：Scenario 最外層、裡面放多個 Module、第一個 Module 是 Trigger、用外部帳號的 Module 會綁一個 Connection。Connection 是「帳號級」設定，跨 Scenario 共用；Module 是「流程級」配置。
(03) · STEP BY STEP · 操作示範（15 分鐘）

三個 Module，串一條完整流程

下面 5 個步驟會從零建一個 Scenario：Schedule（觸發）→ RSS（抓資料）→ Google Sheets（寫入）。

1
登入 Make.com 並建立新 Scenario
  1. 打開 make.com，用 Google 帳號登入（Free Plan 每月 1,000 ops，這支 scenario 一天跑 1 次、一個月 90 ops 綽綽有餘）
  2. 左側 Scenarios → 右上 Create a new scenario
  3. 畫面中央出現一個大加號 +——這就是第一個 Module 位置
2
加入 Schedule Trigger（每日 09:00）
  1. 點中央 + → 搜尋 Schedule → 選 Every Day
  2. 設定 Run scenario：Every day、Time：09:00、Time zone：Asia/Taipei
  3. 按 OK。畫面出現一顆時鐘圖示的圓球，右下顯示「Next run at 2026-04-16 09:00」
3
加入 RSS Module（Retrieve RSS feed items）
  1. 第一個 Module 右邊出現小 +，點它
  2. 搜尋 RSS → 選 Retrieve RSS feed items
  3. URL：https://feeds.bbci.co.uk/news/rss.xml，Maximum number of returned items：5
  4. 在 Module 圓球按右鍵 → Run this module only 測試
  5. 點圓球上的泡泡查看 Output Bundle，可見 title、link、description、pubDate 等欄位
4
加入 Google Sheets Module（Add a Row）
  1. RSS Module 右邊點 + → 搜尋 Google Sheets → 選 Add a Row
  2. 第一次會要求建 Connection：Add → Sign in with Google → 允許 Sheets + Drive 權限 → 命名 gsheets-pilot
  3. 回設定：Spreadsheet 選你事先建好的 Make 抓取紀錄，Sheet 選「工作表 1」
  4. Values 出現 4 欄位，從右邊面板拖 RSS Output 欄位進去：抓取時間 ← now、標題 ← Title、連結 ← Link、摘要 ← Description
5
Run once 確認、切 ON 進入排程
  1. 畫面左下 Run once（播放鍵）
  2. 三個 Module 依序跳「1 / 5 / 5」泡泡（觸發 1 次、抓 5 筆、寫 5 列）
  3. 開 Google Sheet 應該看到 5 筆新列
  4. 回 Make 畫面，右下角切換鈕從灰（OFF）切綠（ON），Scenario 正式排定每日 09:00 自動執行
🎯
Run once 和 ON 的差別：Run once 只跑一次測試，不影響排程；切成 ON 才是真的「每天 09:00 自動跑」。

換你做：同時監控 3 個競品 RSS

延續阿凱的情境。他真正的業務需求是同時監控 3 個競品的 RSS，把結果寫進同一個 Sheet，並在「來源」欄標記是哪個競品。

競品 A · HubSpot
→ https://blog.hubspot.com/marketing/rss.xml
競品 B · CMI
→ https://contentmarketinginstitute.com/feed/
競品 C · Marketo
→ https://blog.marketo.com/feed

完成標準（self-check）：

結構
Scenario 含 1 Schedule + 3 RSS + 1 Sheets Add Row Module
資料流
3 個 RSS Module 的 output 都流到同一個 Sheets Module
來源標記
Sheet 有「來源」欄，正確標記 HubSpot / CMI / Marketo
執行驗證
Run once 成功、Sheet 同時看到 3 來源、不重不漏
排程

做到這三件事才算學會

(07) · PITFALLS · 常見錯誤

三個最容易踩到的坑

Sheets 選單看不到你的 Sheet
原因：Connection 建立時沒勾 drive.file scope，OAuth 只給 Make 讀寫它自己建的檔案。解法：刪掉 Connection 重新建立，授權畫面確認 Sheets 權限有勾。
RSS Module 紅色驚嘆號
錯誤訊息：Parse error at line 1 column 1。原因：URL 不是 RSS 是一般網頁 HTML。解法：找網站的 /rss、/feed、/rss.xml 路徑。
Scenario 切 ON 後隔天沒跑
狀態：Paused by system。原因：Free Plan 連續 14 天沒登入會自動暫停；或 Scenario 連續失敗 5 次。解法：每週登入一次或升級付費；檢查 History 看失敗原因。
(08) · QUIZ · 自我檢核

兩題試試看你真的懂了

Q1（概念驗證）：下列關於 Make 四核心概念，哪個描述是錯的？
顯示答案
正確答案：C。Connection 是「帳號級」設定，建立一次就可以被多個 Scenario 共用。這也是為什麼 Connection 獨立在 Scenario 之外管理。
Q2（應用驗證）：阿凱建的 Scenario 每天 09:00 執行，3 天後發現 Sheet 裡每天都有資料，但每天都是同一筆 5 年前舊新聞。最可能的原因？
顯示答案
正確答案：B。排除順序：(1) 瀏覽器直接打開 RSS URL 確認源頭有新內容；(2) 若有新內容但抓到舊的，改用「watch」類型或調整 max items；(3) 也可能 RSS 有快取，URL 加 ?t={{now}} 避快取。
