## 教學流程（Teaching Flow）

### 破題 / Hook

阿凱是弄一下行銷工作室的行銷專員。兩個月前老闆把「做內容 SEO」這個任務丟給他——寫 blog、衝自然流量、減少廣告預算。他列了 20 個關鍵字，挑了 5 個搜尋量最高的開始寫，每篇 2500 字，他自己覺得寫得不錯。

三個月後回頭看 GSC：5 篇文章平均每月點擊 **12 次**。老闆不爽。

問題出在哪？阿凱查了 5 篇的關鍵字排名，其中 3 篇根本沒進前 100。可是他寫得比現有前 10 的文章都詳細啊？

真正的原因不是「寫得不夠好」——是他**把想買東西的人，帶到了想找答案的頁面**。關鍵字「crm 推薦」的搜尋者，九成九在「挑產品」，但他寫了一篇「CRM 是什麼？完整解析」。Google 不是笨蛋，知道這篇不符合使用者想要的，所以排不上去。

這就是**搜尋意圖錯位**——內容 SEO 最常見、也最容易避開的死法。今天這 60 分鐘，要讓阿凱（和你）學會四種搜尋意圖類型怎麼區辨，下次再拿到關鍵字清單，能先判定「這人要什麼」，再決定寫什麼。

### 概念 / Concepts

搜尋意圖四類型，是 Google 在 2019 年 QRG（品質評分指南）正式定調的分類法，至今業界通用。一句話記住每一個：

- **Informational（資訊型）** — 使用者要「找答案、學東西」。動詞線索：是什麼、怎麼做、為什麼、如何。範例：「n8n 是什麼」「python 迴圈怎麼寫」
- **Navigational（導航型）** — 使用者要「找某個特定網站或品牌頁」。線索：搜尋詞含明確品牌名 + 功能/頁面名。範例：「notion 登入」「蝦皮 客服」「gmail 設定」
- **Commercial Investigation（商業評估型）** — 使用者正在「選產品/服務，還沒決定買哪個」。線索：「比較」「推薦」「vs」「ptt 評價」「2026 排名」。範例：「crm 推薦 中小企業」「iphone 16 vs pixel 9」
- **Transactional（交易型）** — 使用者「準備掏錢或執行動作」。線索：含價格、折扣、下載、訂購、購買。範例：「notion pro 訂閱價格」「airbnb 台北 訂房」「chrome 下載」

這四類型**不是互斥座標，而是漸進光譜**：從「我不懂」→「我要找」→「我在挑」→「我要買」。多數關鍵字有一個**主意圖**，少數混合（這是本課進階重點）。

「但我怎麼知道一個關鍵字落哪一類？」——靠兩條線索：
1. **搜尋詞本身的動詞與修飾詞**（「是什麼」「推薦」「下載」）
2. **SERP（搜尋結果頁）前 3–5 名長什麼樣**（教學文 / 官方頁 / 比較表 / 產品頁）

第 2 條是**終極仲裁**：Google 排出來的內容反映了它判定的主流意圖。當你的判斷和 SERP 不符，**以 SERP 為準**。

### 案例 / Case Study

阿凱拿出他那 5 篇失敗文章的關鍵字，我們逐一解剖：

**案例 1：「crm 推薦」→ 阿凱寫成「CRM 是什麼？完整解析」**
- 動詞線索：「推薦」= Commercial Investigation
- SERP 前 5 名：4 篇「10 款 CRM 比較」、1 篇「中小企業 CRM 選購指南」
- **阿凱錯位**：把 Commercial 詞寫成 Informational 文。正確寫法是「10 款 CRM 比較 · 中小企業版」這種比較文

**案例 2：「notion 教學」→ 阿凱寫成長篇入門教學**
- 動詞線索：「教學」看起來像 Informational
- **陷阱**：SERP 前 3 名都是 notion.com 官方的 help 頁面 → 這其實是 **Navigational**（多數人是要找 Notion 官方教學）
- **阿凱錯位**：寫一篇搶不贏官方的教學文。正確策略是寫 notion 官方「沒教」的進階應用（例「Notion 搭 Zapier 自動化」）把意圖改向 Informational

**案例 3：「n8n 是什麼」→ 阿凱沒寫這個**
- 動詞線索：「是什麼」= 純 Informational
- SERP 前 5 名：全是 blog 文、百科式定義
- **這篇他該寫**：阿凱的產品是 Make，寫一篇「n8n 是什麼、跟 Make 比較」可以同時吃 Informational + Commercial 兩批流量

**案例 4：「notion pro 訂閱價格」→ 阿凱沒寫**
- 線索：「價格」= Transactional
- SERP：90% 是 notion.com 的 pricing 頁、10% 是第三方整理「notion 各方案比較」
- **策略建議**：這種詞**不適合第三方寫**——官方頁永遠比你可信。繞過這類詞

**案例 5：「客戶管理 工具」→ 阿凱寫成「你需要的客戶管理工具」**
- 線索：「工具」模糊，無明確意圖動詞
- SERP 前 5 名：3 比較文、1 教學文、1 產品頁 → **混合意圖，偏 Commercial**
- **阿凱錯位**：寫的是 Informational 視角，但 SERP 主力是 Commercial。應改成「10 款客戶管理工具比較（2026）」

五個案例的共通錯誤：**阿凱只看搜尋詞、沒看 SERP**。

### 反思 / Reflection

停下來對照你自己（或你公司）的情境，回答三個問題：

**反思 1：你最近寫過（或準備寫）的 3 個關鍵字，分別是哪一類？**
現在就打開 Google 搜尋那 3 個詞，看 SERP 前 3 名：
- 前 3 名都是教學文/百科 → Informational
- 前 3 名都是品牌官方頁 → Navigational
- 前 3 名都是比較文/評價 → Commercial Investigation
- 前 3 名都是產品頁/購買頁 → Transactional
- 前 3 名混合 → 混合意圖（看哪個型式占比高）

**反思 2：你公司的現有內容，有沒有「寫對關鍵字但用錯意圖型式」的文章？**
檢查 GSC：有沒有某文章的「曝光高但點擊率低於 1%」？很可能意圖錯位——使用者搜尋時看到你的標題但預覽不像他要的，所以不點。

**反思 3：你手邊有哪些關鍵字是「SERP 告訴你的意圖」和「你直覺的意圖」不一樣？**
這些是**需要特別標註**的陷阱詞。通常是「教學」「設定」「使用方法」這類看似 Informational 但實際被品牌官方壟斷（Navigational）的詞。

## 試跑包需求清單（Verification Asset Spec）

**課程類型**：concept

延續大綱「試跑包交付規格」，設計師層級細化成設計師可執行度：

### 1. Keyword List CSV（檔名 `keyword-list.csv`）

10 個中文真實搜尋詞。請**不要**編造，從下面三個來源任選 10 個（可用 Google Trends / Ubersuggest 免費版驗證有搜尋量）：

- 你自己行銷工作室的潛在客戶詞池
- 阿凱案例裡出現過的詞（crm 推薦、notion 教學、n8n 是什麼 等可直接用）
- 通用上班族痛點詞（excel、notion、gmail、ai 相關）

**必含分布**（缺任一扣分）：
- Informational ≥ 2 個
- Navigational ≥ 2 個
- Commercial Investigation ≥ 2 個
- Transactional ≥ 2 個
- **陷阱詞 2 個**（表面像 A 類、實際 B 類）

格式：
```csv
keyword,monthly_volume,difficulty
"n8n 是什麼",1200,32
"notion 教學",9500,52
```

### 2. 分類解答本（檔名 `classification-key.md`）

10 題對應解答 + 3 題陷阱詞解析。每題 4 欄位：

| 欄位 | 內容 |
|------|------|
| 搜尋詞 | 來自 csv |
| 預期分類 | 四類之一 |
| 判斷線索 | 2 條：搜尋詞動詞 + SERP 前 3 名觀察 |
| 適合內容格式 | 2 種具體格式（呼應 Outcome 4）|

陷阱詞解析另列一節，每個陷阱詞含「表面像 → 實際是 → SERP 怎麼告訴你」三段。

### 3. 內容策略輸出範本（檔名 `content-plan-template.md`）

完全空白的填寫範本（**不含任何解答**）：

```markdown
| 搜尋詞 | 我判斷的分類 | SERP 前 3 名觀察 | 對應內容格式 | 我能寫嗎？ |
|--------|------------|----------------|------------|-----------|
| | | | | |
| | | | | |
```

「我能寫嗎？」是讓學員做商業判斷：有些意圖（Transactional / Navigational）第三方寫了也搶不贏官方，該放棄。

### 4. SERP 截圖樣本（3 張）

3 個陷阱詞各一張 Google SERP 截圖（你自己打開 Google 搜尋後截圖）：
- 「notion 教學」（顯示 notion.com 占前 3 → Navigational）
- 「crm 推薦」（顯示全部是比較文 → Commercial Investigation）
- 「客戶管理 工具」（顯示混合但偏 Commercial）

每張附一行註解：「SERP 告訴你這是 X 類，因為...」

## 動手練習題（Hands-on Exercise）

**題目**：

阿凱準備這週把剩下 15 個關鍵字重新分類。他從工具匯出關鍵字清單（你會收到試跑包裡的 `keyword-list.csv` 10 個詞），**你要幫他做完整流程**：

1. 打開 `keyword-list.csv`
2. **不要先看解答本**，自己對每個詞做分類判斷（直接看搜尋詞猜類型）
3. 填寫 `content-plan-template.md` 的「我判斷的分類」欄
4. **然後**逐個用 Google 搜尋、觀察前 3 名 SERP，填「SERP 前 3 名觀察」欄
5. 兩個欄位不一致時，**以 SERP 為準**修正分類，並填「對應內容格式」
6. 對「我能寫嗎？」做商業判斷：Navigational 被官方壟斷的就填「放棄」、Transactional 非官方難贏的也填「放棄」、Informational / Commercial 填「可寫」
7. 全部做完後打開 `classification-key.md` 對答案

**預期成果**：一份填滿的 `content-plan-template.md`，10 個詞各有分類、SERP 觀察、對應格式、可寫與否判斷。

**完成標準**（self-check）：
- [ ] 10 個詞的「我判斷的分類」全部填完（未對答案前）
- [ ] 10 個詞的「SERP 前 3 名觀察」欄實際打開 Google 查過
- [ ] 對答案後，你分類與解答**至少 8 題相符**（達 Outcome 2 的 80% 標準）
- [ ] 不相符的 ≤ 2 題，你能說出「SERP 觀察的哪個線索你錯判了」
- [ ] 「我能寫嗎？」欄每個都有填寫（不能空白），並能口頭解釋為何放棄某幾個

## 常見錯誤 3 條（Common Pitfalls）

1. **錯誤現象**：把「notion 教學」「gmail 設定」「excel 函數」這類詞全分成 Informational，寫了長文後排不上去。
   **原因**：這些詞 SERP 多數被品牌官方幫助頁面壟斷（notion.com/help、support.google.com），實際主流意圖是 Navigational——使用者想找「該品牌的官方教學」，不是要讀第三方解析。
   **解法**：看 SERP 前 5 名網域，若 3 個以上是品牌官方，分類為 Navigational；作為第三方第四方的你，要寫就寫官方沒教的進階應用（改變意圖為 Informational）或比較文（改變為 Commercial Investigation）。

2. **錯誤現象**：把「crm 推薦」「最好的 project management 工具」分成 Transactional，寫了產品頁/訂購頁，結果跳出率 90%+。
   **原因**：「推薦」「最好的」是**評估階段**，使用者還沒決定要買哪個品牌——這是 Commercial Investigation，要的是比較、評論、選購指南，不是立刻買的頁面。Transactional 的信號是「價格」「下載」「訂購」「購買」這類明確交易動詞。
   **解法**：寫「10 款 CRM 完整比較」「2026 project management 工具排行」這種評估型內容，裡面才放產品頁連結。讓使用者從 Commercial 階段被你教育到 Transactional 階段。

3. **錯誤現象**：一個關鍵字同時有 Informational + Commercial 意圖時，選擇寫「純概念解析」，結果夾在中間兩邊不討好。
   **原因**：混合意圖時，SERP 通常呈現多型式共存（前 10 名有比較文、教學文、產品頁混雜）。寫純 Informational 會搶不贏主流 Commercial 內容；寫純 Commercial 又會漏掉想學原理的一批人。
   **解法**：混合意圖時以 SERP 主流格式為主（通常是 Commercial 比較文佔多數）、在文章前 1/3 用 Informational 概念鋪陳，後 2/3 轉 Commercial 比較分析。這種「混合型寫法」在混合意圖 SERP 上表現最好。

## 檢核題 2 條（Quiz）

**Q1（概念驗證）**：下列關鍵字與搜尋意圖類型的配對，哪一個**錯了**？

- [ ] A. 「python 迴圈怎麼寫」→ Informational
- [ ] B. 「蝦皮 客服 電話」→ Navigational
- [ ] C. 「iphone 16 折扣碼」→ Commercial Investigation ←（**正確答案**：這個錯了）
- [ ] D. 「2026 最好的 VPN」→ Commercial Investigation

預期答案：C 錯。「折扣碼」是 Transactional 的強信號（準備交易），不是還在評估。Commercial Investigation 的詞通常含「比較」「推薦」「排名」「最好的」。

**Q2（應用驗證）**：阿凱手上有個關鍵字「Notion template」，Google 搜尋量 8200/月。他直覺認為這是 Informational（「要學 template 是什麼」），想寫一篇「Notion Template 是什麼？完整介紹」。你會建議他怎麼判斷並採取什麼行動？

- **預期答案要點**（給設計師寫解說用）：
  1. **先打開 Google 搜尋** — 這是 SERP 驗證的必要步驟，不要只靠直覺
  2. **觀察前 5 名型式** — 「Notion template」SERP 主力是 notion.com/templates、以及第三方 template gallery（notiontemplates.com 等），屬於 **Navigational + Transactional 混合**（使用者要找 template 下載）
  3. **正確策略** — 寫「Notion 是什麼」這種 Informational 文搶不贏；應改為製作「15 款最實用 Notion Template（附下載連結）」這種整合清單型內容，跨 Navigational（幫他找到）+ Commercial（幫他比較）兩種意圖，流量才會進來
