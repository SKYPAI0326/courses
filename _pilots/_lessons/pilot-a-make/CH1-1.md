## 教學流程（Teaching Flow）

### 破題 / Hook

阿凱是弄一下行銷工作室的行銷專員。他每天早上第一件事都是打開 BBC News、TechCrunch、競品官網、3 個產業部落格，**花 30 分鐘看有沒有值得記錄的新文章**，抄進一個 Google Sheet 給老闆週報用。三個月下來他算了一下：他每天花在「抓網頁內容」的時間是 30 分鐘，一週 2.5 小時。

一小時 300 塊，一個月浪費的人力成本超過 6,000 塊。還不包括漏看、重複記錄的成本。

他聽同事提過 Make.com 可以自動化，打開註冊頁看了 3 分鐘——介面上冒出 Scenario、Module、Trigger、Connection 四個名詞，他看不懂哪個是哪個，關掉頁面回去手抄。

本單元要幫他解決的就是這個 30 分鐘的重複工作。我們不會把 Make 的所有功能教完，只會教**一條最小可用路線**：把網頁 RSS 自動抓進 Sheet。跑起來，他以後只要看 Sheet 就好。

### 概念 / Concepts

Make 裡面只要搞懂這四個詞，後面都是拼積木。一句話記住每一個：

- **Scenario（情境流程）** — 一條完整的自動化流程，就像一張 A4 紙上畫的工作流。你每次建立的「自動做某件事的流程」，在 Make 裡面就叫 Scenario
- **Module（模組）** — Scenario 這張紙上每一個節點，一個節點做一件事（抓網頁、寫 Sheet、發信）
- **Trigger（觸發器）** — Scenario 的第一個 Module，決定這條流程「什麼時候開始跑」。可以是「每天 09:00」、「收到新郵件」、「網頁有更新」
- **Connection（連線設定）** — 當 Module 要碰外部帳號（Gmail、Google Sheets、Notion）時，Make 需要一個「驗證過的帳號憑證」。Connection 就是這個憑證，一次設定完，多個 scenario 都能共用

四個概念的層級關係：Scenario 最外層，裡面放多個 Module，第一個 Module 是 Trigger，用到外部帳號的 Module 會指定一個 Connection。Connection 是跨 Scenario 共用的「帳號級」設定，Module 是「流程級」配置。

### 操作示範 / Demo

以下是完整的 3-Module Scenario 建置流程，總共 15 分鐘完成。

**步驟 1：登入 Make.com 並建立新 Scenario**（2 分鐘）

1. 打開 `https://www.make.com/`，用 Google 帳號登入（Free Plan 免費給 1,000 ops/月，這支 scenario 每天跑 1 次，一個月花 90 ops，綽綽有餘）
2. 左側選單點 **Scenarios** → 右上 **Create a new scenario**
3. 畫面中央會出現一個大大的加號（`+`）——這就是你的第一個 Module 位置

**步驟 2：加入 Trigger Module — Schedule（3 分鐘）**

1. 點中央的 `+` → 搜尋 `Schedule`
2. 選 **Every Day**（內建 Trigger）
3. 設定：
   - **Run scenario**: `Every day`
   - **Time**: `09:00`
   - **Time zone**: `Asia/Taipei`
4. 按 **OK** 關閉設定
5. 畫面上出現一個時鐘圖示的藍色圓球——這是你的 Trigger Module。右下角會顯示「Next run at 2026-04-16 09:00」，確認時間對就好

**步驟 3：加入 RSS Module — Retrieve RSS feed items（4 分鐘）**

1. 在第一個 Module 右邊出現一個小`+`，點它
2. 搜尋 `RSS` → 選 **Retrieve RSS feed items**
3. 設定：
   - **URL**: `https://feeds.bbci.co.uk/news/rss.xml`
   - **Maximum number of returned items**: `5`（避免第一次爆太多資料）
4. 按 **OK**
5. 在 Module 圓球上按滑鼠右鍵 → **Run this module only** → 按 `OK`
6. 測試執行後，Module 上會出現一個泡泡標示「1」（成功執行一次）。點泡泡可以看到抓到的資料，有 `title`、`link`、`description`、`pubDate` 等欄位——**這就是 Output Bundle**

**步驟 4：加入 Google Sheets Module — Add a row（4 分鐘）**

1. 在 RSS Module 右邊點`+`
2. 搜尋 `Google Sheets` → 選 **Add a Row**
3. 第一次會要求你建 Connection：
   - 按 **Add** → **Sign in with Google**
   - 選你要用的 Google 帳號 → 允許 Make 存取 Sheets 和 Drive
   - Connection 命名為 `gsheets-pilot`，按 Save
4. 回到 Module 設定畫面：
   - **Spreadsheet**: 選下拉，選你事先建好的 `Make 抓取紀錄` Sheet
   - **Sheet**: 選 `工作表 1`
   - **Values** 會出現 4 個欄位（對應你表頭的`抓取時間`/`標題`/`連結`/`摘要`）
   - 每個欄位從右邊面板拖 RSS Module 的 output 欄位進去：
     - `抓取時間` → 拖 `now`（Make 內建函數，代表當下時間）
     - `標題` → 拖 RSS 的 `Title`
     - `連結` → 拖 RSS 的 `Link`
     - `摘要` → 拖 RSS 的 `Description`
5. 按 **OK**

**步驟 5：執行一次確認成功（2 分鐘）**

1. 畫面左下角按 **Run once**（一個播放鍵）
2. 看三個 Module 上的泡泡依序跳出「1」/「5」/「5」（表示 Schedule 觸發 1 次、RSS 抓 5 筆、Sheet 寫 5 列）
3. 打開你的 Google Sheet，應該看到 5 筆新列，每列都有抓取時間、標題、連結、摘要
4. 回到 Make 畫面，**右下角的切換鈕**從灰色（OFF）切到綠色（ON），Scenario 正式排定每天 09:00 自動執行

### 動手 / Hands-on

看完示範，換你做一次。我們**換一個 RSS 來源**，讓你不是照抄：

**你的任務**：把上面 BBC 換成 **Hacker News**（`https://news.ycombinator.com/rss`），抓 10 筆（不是 5 筆），存到同一個 Sheet（附加在下面）。

**提示**：
- 你不需要建新 Scenario——複製既有的（右鍵 Scenario → Clone），改 RSS URL 和 max items 就好
- 如果 Hacker News RSS 的欄位結構跟 BBC 不同（不一定有 description），你需要重新綁 Value 對應
- 試跑時先用 Run once 測試，**不要直接 Turn ON**，避免雜訊寫進 Sheet

### 檢核 / Verification

以下三件事你同時做到，才算學會：

1. **開啟你的 Google Sheet**，裡面同時有 BBC（5 筆）和 Hacker News（10 筆）的資料
2. **在 Make 畫面**，兩個 Scenario 都切成 ON，下次執行時間顯示為明天 09:00
3. **口頭解釋**：「Schedule、RSS、Google Sheets 這三個 Module，哪一個是 Trigger？哪一個需要 Connection？Connection 裡面裝的是什麼？」——三題能答對

第 3 題的答案：Schedule 是 Trigger；Google Sheets 需要 Connection；Connection 裡面裝的是你授權給 Make 存取 Google 帳號的 OAuth token。

## 試跑包需求清單（Verification Asset Spec）

**課程類型**：skill-operation

延續大綱 `試跑包交付規格` 區塊，設計師層級再細化具體可執行度：

### Credential / Connection

- **Google Sheets Connection**
  - 名稱：`gsheets-pilot`
  - OAuth scope：
    - `https://www.googleapis.com/auth/spreadsheets`（讀寫 Sheets）
    - `https://www.googleapis.com/auth/drive.file`（存取透過 Make 建立的檔案）
  - 授權流程：Make.com Connection 畫面 → Add → Sign in with Google → 選帳號 → Allow

### Module 組成（依序 3 個）

| 順序 | Module 官方名 | Module ID（Make blueprint 內） | 關鍵設定 |
|------|---------------|-----------------------------|---------|
| 1 | Schedule > Every Day | `builtin:BasicTrigger` | `scheduling.type: "every"`、`scheduling.interval: 1440`（分鐘）、`hour: 9`、`timezone: "Asia/Taipei"` |
| 2 | RSS > Retrieve RSS feed items | `rss:ActionReadFeedItems` | `url: "https://feeds.bbci.co.uk/news/rss.xml"`、`maxResults: 5` |
| 3 | Google Sheets > Add a Row | `google-sheets:addRow` | `spreadsheetId`（從 URL 取）、`sheetName: "工作表1"`、`values[0..3]` 分別綁 `{{now}}` / `{{2.title}}` / `{{2.link}}` / `{{2.description}}` |

注：`{{2.xxx}}` 的 `2` 代表第 2 個 Module 的 output，Make 的變數引用語法。

- `setup.md`：含 Google Sheet 範本連結、Connection 建立步驟截圖說明

## 動手練習題（Hands-on Exercise）

**題目**：

延續阿凱的情境。他覺得 BBC + Hacker News 只是練習，真正的業務需求是**同時監控 3 個競品的 RSS**：
- 競品 A：`https://blog.hubspot.com/marketing/rss.xml`
- 競品 B：`https://contentmarketinginstitute.com/feed/`
- 競品 C：`https://blog.marketo.com/feed`

你要幫阿凱建一個進階版 Scenario：**每天 09:00 同時抓 3 個 RSS，結果都寫進同一個 Sheet，並在「來源」欄標記是哪個競品**。

**預期成果**：

阿凱每天早上打開 Sheet 會看到類似這樣的資料（每日新增 15-30 列）：

| 抓取時間 | 來源 | 標題 | 連結 | 摘要 |
|----------|------|------|------|------|
| 2026-04-16 09:00 | HubSpot | 「2026 年 B2B 行銷趨勢」| http://... | ... |
| 2026-04-16 09:00 | CMI | 「內容策略該怎麼做」| http://... | ... |
| ... | ... | ... | ... | ... |

**完成標準**（self-check）：

- [ ] Scenario 有 1 個 Schedule Trigger + 3 個 RSS Module（並行或串行皆可）+ 1 個 Sheets Add Row Module
- [ ] 每個 RSS Module 的 output 都能流到同一個 Sheets Module
- [ ] Sheet 裡有「來源」欄且正確標記 HubSpot / CMI / Marketo
- [ ] Run once 成功後，Sheet 裡能同時看到 3 個來源的文章，不重複、不遺漏
- [ ] Scenario 切為 ON、下次執行時間為明天 09:00

**提示**：Make 有個 **Router** Module 可以把一個 Trigger 輸出分流到多個路線；或者你也可以乾脆建 3 個 Scenario，各抓一個 RSS 寫進同一個 Sheet。兩種都可以，看你覺得哪個直觀。

## 常見錯誤 3 條（Common Pitfalls）

1. **錯誤現象**：Google Sheets Module 設定時，Spreadsheet 下拉選單點開後**看不到你的 Sheet**，只看到「No matching results」。
   **原因**：Connection 建立時沒勾到 `drive.file` scope，OAuth 只給了 Make 讀寫「它自己建立的檔案」權限，所以看不到你手動建的 Sheet。
   **解法**：到左側 **Connections** 頁面 → 刪掉 `gsheets-pilot` Connection → 重新建立時，在授權畫面務必點「全選」或確認 Google Sheets 權限有勾。

2. **錯誤現象**：按 Run once 後，RSS Module 上面有紅色驚嘆號，錯誤訊息顯示 `RuntimeError: Parse error at line 1 column 1`。
   **原因**：你輸入的 URL 不是 RSS feed，是一般網頁。很多網站首頁是 HTML、不是 XML 格式的 RSS，Make 無法解析。
   **解法**：到目標網站底部或 `/rss`、`/feed`、`/rss.xml`、`/feed/` 等常見路徑找 RSS URL；或用瀏覽器 View Source 搜尋 `type="application/rss+xml"` 看有沒有 `href` 指向真正的 RSS。

3. **錯誤現象**：Scenario 設定好、切成 ON，但隔天 Sheet 裡沒有新列。Make 畫面看 Scenario 右上角狀態是 `Paused by system`。
   **原因**：Free Plan 有連續 14 天沒手動登入就自動暫停的保護，避免你設定了忘記又燒光 ops。
   **解法**：每週至少登入 Make 一次（看個儀表板就好）；或升級到付費 Plan 解除這限制。另外 Scenario 執行失敗超過 5 次也會被暫停，這種就要去 History 看失敗原因。

## 檢核題 2 條（Quiz）

**Q1（概念驗證）**：下列關於 Make 四個核心概念，哪個描述是**錯的**？

- [ ] A. 一個 Scenario 裡可以有多個 Module
- [ ] B. Trigger 一定是 Scenario 的第一個 Module
- [ ] C. Connection 只能被一個 Scenario 使用，不同 Scenario 必須各自建 Connection ←（正確答案：這是錯的描述）
- [ ] D. Schedule 是 Trigger 的其中一種

**Q2（應用驗證）**：阿凱建了一個抓 RSS 寫 Sheet 的 Scenario，設定每天 09:00 執行。執行了 3 天後發現：Sheet 裡**每天都有資料，但每天都是同一筆 5 年前的舊新聞**。你判斷最可能的原因是什麼？該怎麼排除？

- **預期答案要點**：
  1. 目標 RSS 本身停更（源頭沒新內容，不是 Make 的問題）——先用瀏覽器直接打開 RSS URL 確認
  2. 若 RSS 有新內容但 Make 抓到舊的，可能是 `Maximum number of returned items` 設太小、排序是由舊到新——檢查 RSS Module 的設定，改用「watch」類型（只抓新的）
  3. 也可能是 RSS 有快取，Make 抓到的是快取版本——在 URL 後加 `?t={{now}}` 試看看
