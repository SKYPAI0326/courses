## 教學流程（Teaching Flow）

### 需求 / Requirements

阿凱是行銷工作室的行銷專員，上個月他讓 ChatGPT 寫了一段 Python 幫他整理每日廣告花費資料。程式碼看起來很短、很合理，他複製貼上到 VS Code，按下執行——`ModuleNotFoundError: No module named 'gspread'`。他改到「請幫我裝 gspread」，ChatGPT 說 `pip install gspread`。他在 terminal 打了這行——`command not found: pip`。

阿凱花了一整個下午，重裝 Python 三次，換用 `pip3`、`python3 -m pip`、甚至 `sudo pip install`，還是過不了「第一個 import」。最後他關掉 VS Code，回去 Excel 繼續手動整理。

**問題不是 Python 難，是他缺了「跑完整支 .py」的起點經驗**。ChatGPT 寫的程式碼都對，但 ChatGPT 不會陪他按下每個 Enter 鍵。所以本單元只教他一件事：**從零建 credential → 裝套件 → 跑一支 10 行以內的 .py 讀 Sheet → 修一行跑 writer 寫進 Sheet**。目的是讓他**親眼看到 terminal 印出資料、瀏覽器 Sheet 多一列**，跨過那條「我真的跑起來了」的心理線。

跨過之後，以後 ChatGPT 生的程式碼他才有底氣自己動手試，而不是看到錯誤訊息就關視窗。

**為什麼選 Sheets + Python（不選 Excel VBA 或 Zapier）？**
- Google Sheets 有 Python SDK（gspread），中文文件豐富、免費
- 阿凱已經用 Sheets 做報表了，Python 介入後資料還在原處，不用切換工具
- Zapier/Make 是零代碼，但 Python 是**可擴充性最高的工具**——學會讀寫一個 Sheet，以後分析 10 萬行資料、串 AI API、自動發信都只是「再加幾行」的事

### 設計 / Design

本單元的程式架構，**只有 3 個步驟**。看懂這 3 步，你就懂了這類程式的骨架：

- **Step 1：Auth（認證）** — 告訴 Google：「我是這個 Service Account，它有權讀你家 Sheet」。用 `credentials.json` 載入憑證、`gspread.authorize(creds)` 取得一個「已驗證的 client 物件」叫 `gc`
- **Step 2：Open（開檔）** — 用 `gc.open_by_key("SHEET_KEY")` 打開指定 Sheet，拿到一個 `sheet` 物件，代表 Python 手上握著「這個 Sheet 的遙控器」
- **Step 3：Read or Write（讀或寫）** — 用 `sheet.get_all_values()` 讀全部資料、`sheet.append_row([...])` 新增一列

**三步合起來就是 7–8 行 Python**。不管未來你要讀一個 Sheet 還是讀 100 個，骨架都是這三步；換個 API（例如 Airtable、Notion）也是同樣三步，只是換 import 的套件名。

**Service Account vs OAuth 使用者流程**的差別（這個選擇很關鍵）：
- **OAuth 使用者流程**：每次執行 .py 都會彈一個瀏覽器視窗要你登入——適合互動 script，不適合排程自動化
- **Service Account**：一個「機器人帳號」，credentials.json 就是它的身分證，沒彈窗、可背景執行——適合本單元這種「給 Python 跑、以後可排程」的情境

本單元選 Service Account 因為這是**工作自動化的標準作法**，學會它之後你未來排 cron job / GitHub Actions 跑 Python 都是同一套機制。

### 實作 / Implementation

**預設**：你已完成「課前預備」Service Account 4 步驟設定，`credentials.json` 在專案資料夾內，且 Service Account email 已加到目標 Sheet 共用清單（權限：編輯者）。

**實作步驟**：

**步驟 1：建立專案資料夾並安裝套件**（3 分鐘）

```bash
mkdir pilot-c && cd pilot-c
echo "gspread==6.0.0
google-auth==2.29.0
google-auth-oauthlib==1.2.0" > requirements.txt
pip install -r requirements.txt
```

執行後 terminal 跑完會印「Successfully installed gspread-6.0.0 ...」。**如果 pip 命令找不到**，改用 `python3 -m pip install -r requirements.txt`。

**步驟 2：把 `credentials.json` 放進 `pilot-c/` 資料夾**（已預備完成）

**步驟 3：建立 `reader.py`**（8 行程式碼）

在 `pilot-c/` 資料夾建一個叫 `reader.py` 的檔案，貼入：

```python
import gspread
from google.oauth2.service_account import Credentials

creds = Credentials.from_service_account_file("credentials.json",
    scopes=["https://www.googleapis.com/auth/spreadsheets"])
gc = gspread.authorize(creds)

sheet = gc.open_by_key("YOUR_SHEET_KEY_HERE").sheet1

for row in sheet.get_all_values()[:5]:
    print(row)
```

**這 8 行每一行做什麼**（你要能口頭解釋）：
1. `import gspread` — 載入 gspread 套件（這是我們主要工具）
2. `from google.oauth2...` — 載入 Credentials class（用來讀 JSON 憑證）
3. `creds = Credentials.from_service_account_file(...)` — 讀取 credentials.json、設定要哪個權限範圍
4. `gc = gspread.authorize(creds)` — 用憑證建立一個「已認證的 gspread client」叫 gc
5. `sheet = gc.open_by_key("...")` — 用 Sheet 的 key 打開指定 Sheet，`.sheet1` 取第一個工作表
6. `for row in sheet.get_all_values()[:5]` — 取得所有列、只要前 5 列、逐列迭代
7. `print(row)` — 把每一列印到 terminal

**步驟 4：把 `SHEET_KEY` 換成你的**（最關鍵的一行）

打開你的目標 Google Sheet，看網址列：
```
https://docs.google.com/spreadsheets/d/1AbCdEfGhIjKlMnOpQrStUvWxYz/edit
                                       ^^^^^^^^^^^^^^^^^^^^^^^^^^^
                                       這一段就是你的 SHEET_KEY
```

把 `reader.py` 第 7 行的 `YOUR_SHEET_KEY_HERE` 換成你的 key（**保留雙引號**）。

**步驟 5：執行 reader.py**

```bash
python3 reader.py
```

成功的話 terminal 會印出 5 列資料，每列是一個 list：
```
['timestamp', 'message', 'source']
['2026-04-15', 'hello', 'manual']
['2026-04-15', '測試資料', 'manual']
...
```

**步驟 6：建立 `writer.py`**

```python
import gspread
from google.oauth2.service_account import Credentials
from datetime import datetime

creds = Credentials.from_service_account_file("credentials.json",
    scopes=["https://www.googleapis.com/auth/spreadsheets"])
gc = gspread.authorize(creds)
sheet = gc.open_by_key("YOUR_SHEET_KEY_HERE").sheet1

sheet.append_row([datetime.now().isoformat(), "hello", "from python"])
```

**步驟 7：修改、執行、驗證**

1. 把 `YOUR_SHEET_KEY_HERE` 換成你的（同 reader.py）
2. 修改**最後一行**的 `"hello"` 改成你的名字、`"from python"` 改成你想說的話
3. 執行：`python3 writer.py`
4. 打開瀏覽器的 Google Sheet，**應該看到最後多一列**，是你剛剛改的資料

### 驗證 / Testing

你做到以下**全部**才算學會：

- terminal 執行 `python3 reader.py` 沒報錯、印出 ≥ 5 列資料
- 你能口頭指著 reader.py 的每一行說「這行在做什麼」（用自己的話，不是照著我的註解念）
- terminal 執行 `python3 writer.py` 沒報錯
- 瀏覽器打開 Google Sheet 能看到**你改的那一行**新增在最下面
- Sheet 新增那列的第 1 欄是**你執行當下的時間**（代表 `datetime.now().isoformat()` 真的被執行）

## 常見錯誤 3 條（Common Pitfalls）

1. **錯誤現象**：`python3 reader.py` 執行後噴 `gspread.exceptions.APIError: [403] ...does not have write permission` 或 `APIError: [404] The caller does not have permission`。
   **原因**：你雖然建了 Service Account 並下載 JSON，但**忘了把 Service Account 的 email（client_email 欄位，形如 `pilot-c-sa@xxx.iam.gserviceaccount.com`）加到目標 Sheet 的「共用」清單**，權限設「編輯者」。Service Account 對 Google 來說就是另一個「使用者」，你的 Sheet 預設不會讓它進去。
   **解法**：(1) 打開 `credentials.json`，找到 `client_email` 欄位，複製那個 email；(2) 打開你的目標 Google Sheet，右上「共用」→ 貼上 email → 權限選「編輯者」→ 傳送；(3) 重新執行 `python3 reader.py` 應該就通了。

2. **錯誤現象**：`pip install -r requirements.txt` 執行後印 `ERROR: Could not find a version that satisfies the requirement gspread==6.0.0` 或 `command not found: pip`。
   **原因**：你的系統可能有兩個 Python（系統內建 + 你自己裝的），`pip` 命令指到錯的版本；或 PATH 沒把 pip 加進去。
   **解法**：改用 `python3 -m pip install -r requirements.txt`——這個寫法明確告訴系統「用 python3 這個解譯器的 pip 模組」，避免 PATH 混亂。若還是失敗，執行 `python3 --version` 確認 Python ≥ 3.10；仍失敗則可能要用 `brew install python` 或重裝 Python 3.12 installer（macOS）/從 python.org 下載（Windows）。

3. **錯誤現象**：`python3 reader.py` 噴 `FileNotFoundError: [Errno 2] No such file or directory: 'credentials.json'`。
   **原因**：你執行 `python3 reader.py` 時的**當下工作目錄**不是 `pilot-c/`——Python 在找 `credentials.json` 是以「執行當下的目錄」為基準的相對路徑。如果你從家目錄 `~/` 執行 `python3 ~/pilot-c/reader.py`，Python 會在 `~/` 找 credentials.json 而不是 `~/pilot-c/`。
   **解法**：在 terminal 先 `cd ~/pilot-c/` 切到專案資料夾，再執行 `python3 reader.py`。或把 reader.py 裡的 `"credentials.json"` 改成絕對路徑（例 `"/Users/yourname/pilot-c/credentials.json"`）。前者較推薦，因為絕對路徑會讓程式不好分享。

## 檢核題 2 條（Quiz）

**Q1（概念驗證）**：本單元的 reader.py 用 Service Account 認證，不用一般 OAuth 使用者流程。下列關於 Service Account 的敘述，哪一個**錯了**？

- [ ] A. Service Account 是「機器人帳號」，沒有真人在後面
- [ ] B. credentials.json 相當於 Service Account 的身分證，洩漏了等於任何人都能冒用它
- [ ] C. Service Account 預設可以存取 Google 帳號下所有 Sheet，不需額外授權 ←（**正確答案**：錯的）
- [ ] D. 適合排程自動化（cron / GitHub Actions），不會彈瀏覽器視窗

預期答案：C 錯。Service Account **預設不能**存取任何 Sheet，你**必須把它的 email 加到目標 Sheet 的共用清單**它才有權限——這正是「常見錯誤 1」的原因。Service Account 跟一般使用者帳號一樣是獨立身分，要一個一個授權。

**Q2（應用驗證）**：阿凱執行 `python3 reader.py` 噴 `APIError: [403] Request had insufficient authentication scopes`。ChatGPT 說「加 scope」。他該修改哪裡、加什麼？

- **預期答案要點**：
  1. **修改檔案**：`reader.py` 的 `Credentials.from_service_account_file(...)` 那行
  2. **修改參數**：`scopes` 參數陣列裡只有 `spreadsheets`，如果要讀寫檔案本身（不只是儲存格內容）可能要加 `drive.file`
  3. **完整寫法**：`scopes=["https://www.googleapis.com/auth/spreadsheets", "https://www.googleapis.com/auth/drive.file"]`
  4. **注意**：scope 改了 credential 不用重下載，但必須確認 Service Account 有 Google Cloud 上開啟對應 API（這題情境下要開 Google Drive API）
