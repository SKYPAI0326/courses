Python 讀 Google Sheets API入門

1 小時從零跑一支 8 行的 .py 讀取 Google Sheet，並改一行程式碼寫入新資料——目標是跨過「我真的跑起來了」的心理門檻。

學習成果 · OUTCOMES（課前預備 10 分鐘 + 課中 50 分鐘）
（課前）能完成 Google Cloud 4 步驟：建 Project、啟用 Sheets API、建 Service Account、下載 credentials.json，並把 SA email 加到目標 Sheet 共用清單
能執行 pip install -r requirements.txt、在 .py 用 import gspread 無 ImportError
能用自己的話逐行重新註解 reader.py（8 行全部）
能執行 python3 reader.py，terminal 印出 Sheet 讀出的 5 列資料
能修改 writer.py 的 append_row() 內容，執行後在瀏覽器 Sheet 看到你改的新列
(01) · REQUIREMENTS · 需求

阿凱花一下午沒過第一個 import

阿凱是行銷工作室的行銷專員。上個月他讓 ChatGPT 寫了一段 Python 幫他整理每日廣告花費資料。程式碼看起來很短、很合理，他複製貼上到 VS Code，按下執行——

ModuleNotFoundError: No module named 'gspread'

他改口「請幫我裝 gspread」，ChatGPT 說 pip install gspread。他在 terminal 打了這行——

command not found: pip

阿凱花了一整個下午，重裝 Python 三次，換用 pip3、python3 -m pip、甚至 sudo pip install，還是過不了「第一個 import」。最後他關掉 VS Code，回去 Excel 繼續手動整理。

問題不是 Python 難，
是他缺了「跑完整支 .py」的起點經驗。
ChatGPT 寫的程式碼都對，
但 ChatGPT 不會陪他按下每個 Enter。

本單元只教他一件事：從零建 credential → 裝套件 → 跑一支 10 行以內的 .py 讀 Sheet → 修一行跑 writer 寫進 Sheet。目的是讓他親眼看到 terminal 印出資料、瀏覽器 Sheet 多一列，跨過那條心理線。跨過之後，ChatGPT 生的程式碼他才有底氣自己動手試。

💡
為什麼選 Sheets + Python（不選 Excel VBA 或 Zapier）？Sheets 有 gspread（中文文件豐富、免費）；阿凱已用 Sheets 做報表；Python 是可擴充性最高的工具——學會讀寫一個 Sheet，以後分析 10 萬行資料、串 AI API、自動發信都只是「再加幾行」的事。
(02) · KEY CONCEPTS · 關鍵概念

本單元的程式架構只有 3 步

看懂這 3 步，你就懂了這類程式的骨架。不管未來你要讀一個 Sheet 或 100 個、換成 Airtable/Notion API，都是同樣這三步，只是換 import 的套件名。

Step 1 · Auth
認證
告訴 Google「我是這個 Service Account、它有權讀你家 Sheet」。用 credentials.json 載入憑證、取得 gc 物件。
Step 2 · Open
開檔
用 gc.open_by_key() 打開指定 Sheet、拿到 sheet 物件。代表 Python 握著「這個 Sheet 的遙控器」。
Step 3 · Read / Write
讀或寫
讀：sheet.get_all_values()；寫：sheet.append_row([...])。三步加起來 7–8 行 Python。
⚠️
Service Account vs OAuth 使用者流程（關鍵選擇）：OAuth 使用者流程每次執行 .py 都會彈瀏覽器視窗要你登入，適合互動 script、不適合排程。Service Account 是「機器人帳號」，credentials.json 就是它的身分證，沒彈窗、可背景執行——本單元選它，因為這是工作自動化的標準做法，以後排 cron / GitHub Actions 跑 Python 都是同一套機制。
(03) · STEP BY STEP · 實作流程（50 分鐘）

7 個步驟，完整跑起來

預設前提：你已完成「課前預備」4 步驟 Service Account 設定，credentials.json 在手、且 Service Account email 已加到目標 Sheet 共用清單（權限：編輯者）。

1
建立專案資料夾並安裝套件（3 分鐘）

開 terminal，執行以下：

# 建資料夾、切進去 mkdir pilot-c && cd pilot-c # 寫 requirements.txt echo "gspread==6.0.0 google-auth==2.29.0 google-auth-oauthlib==1.2.0" > requirements.txt # 安裝 pip install -r requirements.txt

執行後 terminal 會印「Successfully installed gspread-6.0.0 ...」。

2
把 credentials.json 放進 pilot-c/ 資料夾

從課前預備下載的 JSON 拖進 pilot-c/。確認檔名是 credentials.json（不是 credentials.json.template 或其他）。

3
建立 reader.py（8 行程式碼）

在 pilot-c/ 建立 reader.py，貼入：

import gspread from google.oauth2.service_account import Credentials creds = Credentials.from_service_account_file("credentials.json", scopes=["https://www.googleapis.com/auth/spreadsheets"]) gc = gspread.authorize(creds) sheet = gc.open_by_key("YOUR_SHEET_KEY_HERE").sheet1 for row in sheet.get_all_values()[:5]: print(row)
4
這 8 行每一行做什麼（你要能口頭解釋）
  1. import gspread — 載入 gspread 套件
  2. from google.oauth2... — 載入 Credentials class（讀 JSON 憑證）
  3. creds = Credentials.from_service_account_file(...) — 讀 credentials.json、設權限範圍
  4. gc = gspread.authorize(creds) — 用憑證建立已認證 client 叫 gc
  5. sheet = gc.open_by_key("...") — 用 Sheet key 打開 Sheet、.sheet1 取第一個工作表
  6. for row in sheet.get_all_values()[:5] — 取全部列、只要前 5、逐列迭代
  7. print(row) — 把每列印到 terminal
5
把 SHEET_KEY 換成你的（最關鍵一行）

打開目標 Google Sheet、看網址列：

https://docs.google.com/spreadsheets/d/1AbCdEfGhIjKlMnOpQrStUvWxYz/edit ^^^^^^^^^^^^^^^^^^^^^^^^^^^ 這一段就是你的 SHEET_KEY

把 reader.py 第 7 行的 YOUR_SHEET_KEY_HERE 換成你的 key（保留雙引號）。

6
執行 reader.py
python3 reader.py

成功的話 terminal 印出 5 列資料、每列是一個 list：

['timestamp', 'message', 'source'] ['2026-04-15', 'hello', 'manual'] ['2026-04-15', '測試資料', 'manual'] ...
7
建 writer.py、修改一行、執行、驗證

建立 writer.py：

import gspread from google.oauth2.service_account import Credentials from datetime import datetime creds = Credentials.from_service_account_file("credentials.json", scopes=["https://www.googleapis.com/auth/spreadsheets"]) gc = gspread.authorize(creds) sheet = gc.open_by_key("YOUR_SHEET_KEY_HERE").sheet1 sheet.append_row([datetime.now().isoformat(), "hello", "from python"])
  1. 把 YOUR_SHEET_KEY_HERE 換成你的（同 reader.py）
  2. 修改最後一行的 "hello" 改成你的名字、"from python" 改成你想說的話
  3. 執行：python3 writer.py
  4. 打開瀏覽器的 Google Sheet，應該看到最後多一列，是你剛改的資料
🎯
做到這一步你就跨過心理門檻了。之後 ChatGPT 生任何 Python + API 的程式碼，你都有能力貼上執行、看錯誤訊息、調整 scope 或檔名——這是最大的收穫，不是這 8 行程式碼本身。

(07) · PITFALLS · 常見錯誤

三個最常卡住的錯誤

APIError: [403/404] does not have permission
原因：Service Account 建了、JSON 下載了，但忘了把 client_email（形如 pilot-c-sa@xxx.iam.gserviceaccount.com）加到 Sheet 共用清單。Service Account 對 Google 就是另一個「使用者」，預設不能進去。解法：(1) 打開 credentials.json 找 client_email；(2) Google Sheet「共用」→ 貼 email → 權限「編輯者」→ 傳送；(3) 重跑 reader.py。
command not found: pip / 套件版本錯
原因：系統有兩個 Python（內建 + 自裝）、pip 指到錯版本；或 PATH 沒把 pip 加進去。解法：改用 python3 -m pip install -r requirements.txt——明確告訴系統「用 python3 的 pip 模組」，避免 PATH 混亂。還失敗就 python3 --version 確認 ≥ 3.10；或重裝 Python 3.12。
FileNotFoundError: credentials.json
原因：執行 python3 reader.py 時當下工作目錄不是 pilot-c/，Python 以「執行目錄」為相對路徑基準。解法：先 cd ~/pilot-c/ 再執行；或把 "credentials.json" 改成絕對路徑（前者較推薦，程式可分享）。
(08) · QUIZ · 自我檢核

兩題試試你真的懂了

Q1（概念驗證）：下列關於 Service Account 的敘述，哪一個錯了？
顯示答案
正確答案：C 錯了。Service Account 預設不能存取任何 Sheet，必須把它的 email 加到目標 Sheet 共用清單才有權限——這正是「常見錯誤 1」的原因。Service Account 跟一般使用者帳號一樣是獨立身分，要一個一個授權。
Q2（應用驗證）：阿凱執行 python3 reader.py 噴 APIError: [403] Request had insufficient authentication scopes。ChatGPT 說「加 scope」。他該怎麼改？
顯示答案
正確答案：C。修改 Credentials.from_service_account_file(...) 的 scopes 參數、加上需要的 scope URL（例 "https://www.googleapis.com/auth/drive.file"）。scope 改了 credential 不用重下載，但要確認 Service Account 對應的 Google Cloud Project 有啟用該 API（這題情境下要開 Google Drive API）。
