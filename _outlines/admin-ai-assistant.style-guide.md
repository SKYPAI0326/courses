# 行政 AI 虛擬助理實戰 · Style Guide 覆寫

## tone_register

**實作為主、不講概念**。每頁開頭直接給「你今天要產出什麼」，不繞圈解釋 AI 是什麼、提示詞工程是什麼（這些只在 CH1 講透一次，之後不回頭）。

行文：
- 短句、動詞開頭、第二人稱「你」
- 不要寫「您」、不要寫「我們」（除非話術引號內）
- 結論前置，不要鋪陳
- 2 小時精簡 DEMO 定位：每頁字量要能在時限內讀完＋動手做完，不塞多餘背景說明

## mood_keywords

`固定流程` `不是單次問答` `裸問對照` `帶走` `2 小時上手`

## differentiation

- 與 office-ai-cases（多角色、多平台不指派）的差異：本課**指定 Gemini（含 Gem）為主要工具**，主案例 prompt 直接寫「在 Gemini」，不用「在任一主流 AI 工具」這種寫法；只有 CH6 才加入 NotebookLM
- 與 gemini-notebooklm-workshop（8h、5 部門變體、跨來源整合為主軸）的差異：本課**時數短、單一主角、不做部門變體**，NotebookLM×Gemini 串聯只在 CH6 出現一次，是「延伸方案的示範」而非全課主軸
- **核心教學主張（G1 檢核定案）**：每個操作單元（CH2~CH5）都要有「裸問 vs 結構化提示詞」並排對照，用真實輸出差異（誤植決議、捏造數字、遺漏待填、語氣格式不符）說服學員，而非用文字告知「精確提示詞比較好」——這是本課與其他課程最大的內容設計差異，course-designer 寫教案時必須落實，不可省略

## 案例主角設定（固定使用、貫穿全課 · 覆寫 template）

**公司**：弄一下顧問股份有限公司（虛構中小型企業管理顧問公司，30 人，服務中小企業客戶）

**主角**：**欣怡**（26 歲，行政專員，隸屬總務行政部）
- 負責：內部會議紀錄、對外/對內 Email、行政公告、部門資料彙整、協助主管準備簡報素材
- 卡住的事：會議常有「討論中」跟「拍板」混在一起沒人分清楚、資料來源散在好幾個試算表跟信件、主管常臨時要三點摘要

**配角**：**協理**（不命名，欣怡的直屬主管）
- 用於「主管摘要」「向上匯報」情境，只在需要「主管要看什麼版本」時出現
- 風格：要重點、要數字有依據、不看過程

**使用守則**：
- 全課只用欣怡當 POV，維持單一主角一致性（本課無 PRAC 頁，不需切換學員自身視角）
- **禁用**：客戶 / 學員 / 老師 / 阿凱 / 雯姊（office-ai-cases 主角）/ 小李 / 怡君 / 凱倫 / 阿翔 / 采蓁（gemini-notebooklm-workshop 主角）
- **不要創造新主角**，協理必要時可簡稱「主管」

## 受眾稱呼

「你」（禁「您」、禁技術黑話、禁「賦能」「數位轉型」「高效」「敏捷」「賽道」）

| 禁用 | 改用 |
|------|------|
| 賦能 | 讓 / 幫 |
| 數位轉型 | 用 AI / 用工具 |
| 高效 | 更快 / 更省時 |
| 敏捷 | 快速調整 |
| 賽道 | 領域 |
| 您 | 你 |
| 我們 | 我 / 你（依語境） |

## 平台引用規範（覆寫 office-ai-cases）

- CH1~CH5：✓ **直接指定「在 Gemini」**（工具已在大綱 tools 欄位指定，不用「在任一主流 AI 工具」這種寫法）
- CH6：✓ 明寫「在 NotebookLM」與「在 Gemini」的切換時機（取證/累積筆記用 NotebookLM、產出成果回 Gemini）
- ❌ 不寫「ChatGPT / Claude / Copilot」（本課只指定 Gemini + NotebookLM）
- 付費功能一律加 💰 標記＋括弧註明（例：「💰 Gemini Advanced：更長 context / Deep Research」），不加標記的功能必須是製作當下真實可用的免費版能力

## 簡報承諾邊界（沿用 office-ai-cases）

**禁用**：「直接生成漂亮 PPT」「下載即用 .pptx」「自動產投影片」
**改用**：「可貼進簡報工具的內容包」「簡報內容素材組裝」「給 Google 簡報 / Gamma / PowerPoint 當輸入材料」

## 裸問 vs 結構化提示詞對照規範（本課專屬 · 每單元必備）

CH2~CH5 每單元至少 1 組對照範例，格式固定：

1. **裸問版**：學員可能直覺打出的簡短提示詞（無限制條件）
2. **裸問輸出的真實問題**：具體指出會出錯在哪（不是「不夠精確」這種模糊講法，要寫出實際錯誤內容，例如把某項議題誤植為已決議）
3. **結構化版**：加上角色/格式/限制/檢核要求的提示詞（逐字可複製）
4. **結構化輸出改善了什麼**：對照前後差異，具體到哪一句話不一樣

不得只給結構化版本、省略裸問對照——這是本課說服學員「提示詞有差」的核心手法。
