## 課程定位（Positioning）

建立「**AI 設計流水線的產品經理**」思維：你不學畫設計，你學怎麼下對需求、驗收 AI 產出、滾出可繼承的骨架、讓 AI 產出守在品牌框架內。不會 Figma / PS 也能產出成系列的商業物料——因為做設計的是 AI，你做的是**需求判定、結構驗收、系列把關**。

## 受眾畫像（Audience Profile）

- **職業**：小公司行銷主管 / 行政、新創老闆、課程設計師、自媒體經營者、PM 兼美術
- **技術底子**：會用 ChatGPT 或 Gemini 其中一個即可；不需要 Figma / PS / Illustrator 背景；不需要 ControlNet / ComfyUI 技術底
- **現有工具棧**：ChatGPT / Gemini（手機或網頁任一）、Canva 免費帳號；可能用過 AI 畫圖但產出品質不穩、每次都要重試很多次
- **痛點 3 條**：
  1. 老闆或客戶臨時要一張廣告圖、貼文底圖、招生 banner，不會設計軟體也沒預算找設計師——**但不想自己去學設計**，只想有工具幫我做
  2. 用 AI 畫圖每次都要重下 prompt，好的版面靠運氣、做不出「同品牌連續 5 張視覺一致」的系列感——**想知道怎麼讓 AI 穩定產出同風格系列**
  3. 看到 AI 生圖社群的高品質作品，但自己跑出來總是差強人意——**不知道是 prompt 問題、還是自己的驗收標準不清楚、還是需求沒講清楚**

## Brand Brief（品牌調性）

- **tone_register**：對話式（像資深 PM 教你怎麼跟 AI 協作，不是設計系課堂演講）
- **mood_keywords**：決策感、克制、專業不玄學（不走誇張、不走「AI 改變世界」行銷話術）
- **differentiation**：本課不是「教你用 MJ / SD 調參數」的工具課，也不是「教你成為設計師」的美學課。本課鎖定「**AI 協作 × 需求判定與驗收**」的流程課——你的角色是 PM / 藝術總監、不是設計師。核心價值承諾：上完課帶走「**需求判定 → AI 執行 → 結構驗收 → 系列把關**」一整條完整流水線，配 5 個跨風格通用的結構骨架 + 1 套品牌 DNA 模板。與 simple-ai（創業生存課）、prompt-basic（公部門交辦課）明顯區隔——本課聚焦「視覺產出的可規模化協作能力」

## 學習成果（Outcomes，6 條 · 動詞開頭 · 可驗證 · 可量化）

1. 能把任意廣告需求**轉成 brief 六件套**（業種 / 商品 / 客群 / 訴求 / 必要元素 / 風格 preset），並判定自己是**需求方**——下指令給 AI 執行，而非自己畫設計
2. 能辨識並命名 **5 種跨風格通用的版面結構骨架**（商品左文字右 / mockup 右文字左 / 風景底資訊下 / 商品中心道具環繞 / 大字堆疊小圖輔助），並說出各自的視覺動線（Z / F / 中央放射）與適用業種——這是驗收 AI 產出時的判準基礎
3. 能**驗收 AI 產出**：拿到一張完稿後能在 30 秒內判斷「結構是否對、是否值得繼續往下做線稿反推、是否需要讓 AI 重跑」——驗收是結構判斷、不是美感判斷
4. 能在同一對話內**指揮 AI 反推 L2 視覺化線稿**，並**判定骨架是否可繼承**（結構完整、佔位符正確、可跨業種套用）——不符要求時能下迭代指令讓 AI 修正
5. 能為自家品牌或假想品牌建立一份 **brand-dna.md**（色票 3 色 + 字族 2 種 + 語氣 + 禁用清單 + 風格 preset），並把關 AI 後續產出的 3-5 張系列完稿**是否守在品牌框架內**（視覺一致度可被第三方辨識為同品牌）
6. 能處理 **6 種常見協作失敗**：需求不清（AI 瞎猜）/ 驗收漏判（把差的當好的收） / L2 反推保留原照片 / 文字未換佔位符 / 結構走樣 / 系列失控——每種能說出症狀、原因、下給 AI 的修正指令

## 試跑包交付規格（Verification Assets）

course_type: skill-operation

每單元提供：
- **Prompt 模板**：生圖 prompt（brief 六件套版）、L2 反推 prompt、跨產業套用 prompt、品牌繼承 prompt 各 1 份，含填空提示與變體範例
- **建議工具標籤**：每則 prompt 標示適用平台（Gemini / ChatGPT / Canva 並列、不偏好任一家、表現浮動時學員自己切換）
- **預期輸出示例**：每則 prompt 附 1-2 組示範輸出（完稿 + 對應 L2 線稿）
- **卡關備案**：常見失敗（保留照片、文字未換佔位符、結構走樣、像 PPT）的症狀描述 + 修正 prompt + Plan B 平台切換

### 課前必備教材資產（🔴 必備，以文字 / 引導清單為主）

> **設計決策（2026-04-24）**：本課採用**引導發現法**，**不備固定示範圖**。學員現場用自己的帳號跑 AI 或用 IG / Pinterest 真實案例對照，避免看到固定範例後誤以為「跑不出那樣是自己失敗」。下列資產全部為文字 / 清單 / 印刷品 / 連結，無圖庫依賴。

1. **L2 骨架庫文件**：5 種版型骨架（商品左 / mockup 右 / 風景底 / 商品中心 / 大字堆疊）的文字描述 + L0 結構 JSON + ASCII 線框示意 + 學員自跑引導（IG hashtag 清單）
2. **10 業種主題庫**（🔴 本課招牌資產，避免學員沒想法）：每業種含 1 份填好的 brief + 1 份可直接丟 AI 的 prompt + 1 個建議骨架 + 預期差異聲明，涵蓋美妝保養、餐飲、App/SaaS、旅遊、教育課程、活動講座、醫美健康、寵物、居家生活、健身運動
3. **brand-dna.md 填空範本**：含色票（primary / accent / bg）、字族（headline / body）、語氣（3-5 詞）、禁用清單，附 2 個完整示範品牌 DNA
4. **brief 六件套模板**：業種 / 商品 / 客群 / 訴求 / 必要元素 / 風格 preset 的填空表
5. **反面教材 4 種**：常見失敗症狀（保留照片 / 文字未換佔位符 / 結構走樣 / 像 PPT）文字描述 + 對應修正 prompt 加強句
6. **骨架對照速查卡**（1 張 A4 雙面，唯一固定印刷資產）：正面 5 骨架 × 5 欄速查表；背面 5 骨架 ASCII 線框示意
7. **IG / Pinterest 推薦搜尋詞清單**（15 組以上）：5 骨架 × 每骨架 3 組搜尋詞，供學員 5 min 內找到真實世界作品

### 明確不備的資產

- ❌ 固定示範圖（固定 L4 / L2 對照圖庫）
- ❌ 固定 12 題骨架辨識題庫
- ❌ 「標準答案圖」

**理由**：AI 生圖本質不穩定（同 prompt 每次輸出都有差異），給固定範例會讓學員誤以為「跑不出那樣就是失敗」。本課核心是**結構判讀能力**而非圖像複製。

## 單元矩陣

### Module 1：開場｜創作閉環（預計 1h）

- CH1-1：你不是設計師、你是 AI 的需求方 — 能說出跟 AI 協作時自己要做的 3 件事（講清楚要什麼 / 看 AI 做得行不行 / 確認系列像同品牌）、用「講 → 等 → 看 → 改」4 步流程跟 AI 工作（concept，30 分鐘）
- CH1-2：三段工作流地圖 + 平台選擇 — 能繪出「生圖 → 轉線稿 → 線稿反控」工作流圖，並依預算決定走 Gemini Nano Banana 堪用版或 Banana Pro 完善版（concept，30 分鐘）
  - 💡 **Tip**：平台配額與付費分級速覽（Gemini Free ~30 張/日 / ChatGPT Plus $20 / Banana Pro 訂閱方案）

### Module 2：起｜商業廣告生圖（預計 2h，含 1 PRAC）

- CH2-1：商業廣告的 8 種用途版型 — 能說出 8 種版型（A 主視覺 / B 橫幅 / C 社群 / D 系列 / E 禮盒 / F 特寫 / G 促銷 / H 生活）的用途與場景；能用「目的 → 版型」3 問速查、為任意需求挑出 1 個合適版型（skill-operation，45 分鐘）
- CH2-2：從 brief 到完稿的 prompt 結構 — 能用 brief 六件套把任意廣告需求寫成 prompt，並用 Gemini / ChatGPT 產出一張符合業種調性的 1:1 完稿（skill-operation，45 分鐘）
  - 💡 **Tip**：文字亂碼處理——讓 AI 只產空版、文字交給 Canva 補
  - 💡 **Tip**：版權邊界——AI 產出僅供參考、商用前重製
- PRAC2：多主題生圖演練 — 從 10 業種主題庫（美妝/餐飲/App/旅遊/教育/活動/醫美/寵物/居家/健身）挑 2-3 個自己最常接案的業種，各跑 1 張完稿；產出要求：版型正確、字階三層、配色 ≤ 3 色（skill-operation，30 分鐘）

### Module 3：承｜圖轉線稿（預計 2h，含 1 PRAC）

- CH3-1：L2 視覺化線稿反推 — 能在同對話延續下把剛產出的完稿轉為 L2 視覺化線稿：保留物件輪廓、移除色彩紋理、文字替換為日文佔位符；能辨識 L0/L1/L2/L3/L4 五個 wireframe 等級的差異與用途（skill-operation，45 分鐘）
  - 💡 **Tip**：Gemini / ChatGPT 做 L2 的品質會浮動——卡關時切換另一家試試、不綁死任一家
- CH3-2：從線稿讀出結構語言 — 能看著 L2 線稿寫下「結構三要素」（視覺重心、焦點層級、對齊網格），並預測該骨架適合什麼業種（concept + skill，45 分鐘）
- PRAC3：線稿反推演練 — 從教材庫挑 5 張現成商業圖（自選競品或假想品牌），每張跑一次 L2 反推；產出要求：5 張 L2 線稿 + 每張 100 字結構筆記（skill-operation，30 分鐘）

### Module 4：轉｜線稿反控圖片（預計 2.5h，含 1 PRAC）

- CH4-1：同線稿 × 跨產業套用 — 能用同一個 L2 骨架 + 新業種 brief 產出至少 3 個不同產業的新完稿，結構保留、內容全換；能說出「線稿是空房間、主題是新住戶」的思維切換（skill-operation，45 分鐘）
  - 💡 **Tip**：道德線——抽象結構可用、具體商標不可抄
- CH4-2：同線稿 × 多變體 — 能用同一個 brief 產出 4 個版面變體（左右對調 / 上下對調 / 中心 / 不對稱），並能說明哪個變體適合 IG 哪個適合 EDM（skill-operation，30 分鐘）
- CH4-3：線稿 × 品牌 DNA 繼承 — 能填寫一份 brand-dna.md（色票 / 字族 / 語氣 / 禁用），並用同骨架 + DNA 產出 3 張視覺一致的系列完稿（skill-operation，45 分鐘）
  - 💡 **Tip**：品牌 DNA 檔的維護建議（每季 review、新增「禁用」條目）
- PRAC4：品牌系列演練 — 學員為自家品牌或假想品牌建一份 brand-dna.md，用同骨架產 3 張春夏秋冬其中任三季的系列圖；產出要求：3 張完稿 + 1 份 brand-dna.md，視覺一致度可被第三方辨識為「同品牌」（skill-operation，30 分鐘）

### Module 5：合｜整合實戰（預計 0.5h）

- CH5-1：從 brief 到品牌系列整合實戰 — 能在 30 分鐘內跑完一次完整閉環：brief → 完稿 A（生圖）→ L2 線稿（轉）→ 線稿 + 新主題 B → 完稿 B（復用）→ 線稿 + 新主題 C → 完稿 C（復用）；產出「一個品牌 3 張系列完稿 + 1 份 brand-dna.md」的交付包（programming / 整合題，30 分鐘）
  - 💡 **Tip**：商用前走一次 Firefly 或 Canva 合法素材重製（版權保險）
