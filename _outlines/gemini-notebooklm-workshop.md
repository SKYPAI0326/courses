## 課程定位（v3 基礎加強班）

學員上過 office-ai-cases，但 Gemini 和 NotebookLM 在日常工作中還沒真正用起來。本課**不是重新教一遍 AI 原理**，而是從「工具介面打開 → 基本操作練熟 → 一個完整工作流走通」三步扎實補強：

- **Gemini**：從基本 prompt 寫法開始，練熟會議紀錄整理 + 彙報初稿兩個核心場景
- **NotebookLM**：功能選單逐一認識，練熟上傳文件 + 查詢引用 + 建個人知識庫
- 最後用一個**個人場景的完整工作流**把兩個工具串起來

**節奏設計**：每個動作講師先示範，學員再跟著做；Day 2 緩衝 65 min，讓卡住的學員有時間補齊。

## 受眾畫像（v3）

- **職業**：跨部門代表（財務 / 業務 / 電商 / 倉儲 / 商品採購）
- **既有訓練**：上過 office-ai-cases 6h，但工具操作不夠熟悉，日常很少主動使用
- **技術底子**：零代碼、Office 一般熟，Gemini 偶爾用過但沒系統化
- **本課切入點**：從「偶爾試過」到「工作中真的會用」

## 學習成果（v3，4 條）

1. **能登入並操作** Gemini 和 NotebookLM 的基本介面，知道每個主要功能的用途
2. **能用 NotebookLM** 上傳 3-5 份工作文件、建立個人知識庫、查詢並確認引用對應到原文
3. **能用 Gemini** 寫出 3 條骨架 prompt，完成會議紀錄整理和一份彙報初稿
4. **能走完一個完整工作流**：NotebookLM 取資料 → Gemini 整合成文，並知道什麼時候用哪個工具

## 單元矩陣（8h = 480 min · 2 天 × 4h）

### Day 1：雙工具環境確認 + 基礎操作（240 min）

#### CH0：雙工具環境設定 + 基礎認識（50 min）

**為何從 25→50 min**：學員雖上過 office-ai-cases，但對工具介面不熟，需要確認每個人都能打開工具並理解基本差異。

- **環境確認（15 min）**：
  - Gemini：帳號登入確認、介面快速導覽（對話框 / 新對話 / 上傳按鈕位置）
  - NotebookLM：帳號登入確認、介面快速導覽（建立 Notebook / Sources / Chat / Studio）
  - 常見卡點排除：帳號問題、瀏覽器相容性、手機版 vs 電腦版
- **兩工具基本差異（15 min）**：
  - Gemini：你問什麼它就回什麼，沒有限定範圍——沒有資料時它會自己補充（可能不準）
  - NotebookLM：你上傳什麼它就只根據那些資料回答——不上傳就不答
  - 一句話原則：**「不確定資料哪來的，先去 NotebookLM」**
  - office-ai-cases 學過的 prompt 骨架在 Gemini 一樣有用（銜接已知，不重教）
- **同一個問題分問兩個工具（15 min）**：
  - 講師示範：同一個業務問題分別問 Gemini 和 NotebookLM，觀察答案差異
  - 學員各自試一次
- **免費版限制說明（5 min）**：
  - Gemini：每日使用次數、32K 輸入上限、上傳檔案格式
  - NotebookLM：每個 notebook 最多 50 個 source、500K token / source

#### CH1-1：NotebookLM 功能選單完整介紹 + 上傳實作（70 min）

**為何從 45→70 min**：這是學員信裡明確要求的單元。v2 的 CH1-1 只做 Gemini 熱身，NotebookLM 基礎被壓縮到 CH1-2 裡，這次要給 NotebookLM 一個完整的「從介面到上機」時段。

- **功能選單逐一導覽（20 min）**：
  - **Sources（來源管理）**：支援格式（PDF / Google Docs / 網址 / 貼上文字）、上傳上限、新增 / 刪除方式
  - **Chat（對話查詢）**：問問題的方式、引用標記（🔗 角標）怎麼點、回原文確認
  - **Audio Overview（音頻摘要）**：生成播客風格摘要、適合快速了解文件重點
  - **Study Guide（學習指南）**：自動生成摘要 / 重點 / 測驗題——適合快速消化長文件
  - **Notebook Guide**：告訴你這份 notebook 有哪些 source 以及整體架構
- **講師示範：上傳 + 查詢（15 min）**：
  - 上傳一份 Word 部門 SOP（或會議紀錄）
  - 問 3 個問題：「這份文件的重點是什麼？」「第三條規定是什麼？」「有沒有提到異常處理？」
  - 示範點擊引用標記 → 回到原文對應段落
  - 示範 Study Guide 生成摘要
- **學員上機：上傳第一份文件（30 min）**：
  - 每位學員選一份自己部門的文件上傳
  - 練 3 個查詢題（根據文件內容自己設計）
  - 確認：引用標記能對回原文嗎？
- **常見問題 + 驗收（5 min）**：
  - 上傳失敗：檔案過大、格式不支援的處理方式
  - 中文文件注意事項（斷行、掃描版 PDF 無法解析）

#### 中場（15 min）

#### CH1-2：Gemini 基礎操作 + 會議紀錄整理（55 min）

**定位調整**：v2 的 CH1-2 是「三層知識庫設計」（本課招牌但偏難）。v3 這個時段改為讓學員把 Gemini 的基本 prompt 操作練熟，搭配最常用的「會議紀錄整理」場景。

- **Gemini 介面操作說明（5 min）**：
  - 新對話 vs 繼續對話的差異
  - 上傳檔案方式（拖放 / 點擊）
  - 輸出格式切換（清單 / 表格 / 段落）
- **Prompt 三件套原則（15 min）**：
  - 三件套：**角色 + 任務 + 格式**（辦公室場景最實用的骨架）
  - 示範：爛 prompt vs 好 prompt 的輸出差異
  - 常見錯誤：問太模糊（「幫我整理一下」）、一次問太多事
  - 銜接 office-ai-cases：「這跟你們上次學的 5 大骨架一致，三件套是其中最常用的三個」
- **講師示範：會議逐字稿 → 會議紀錄（10 min）**：
  - 上傳一份跨部門逐字稿
  - 用三件套 prompt 生成：決議清單 + 個人行動表 + 跟進信
  - 示範如何要求調整格式
- **學員上機（20 min）**：
  - 5 部門變體選自己的：月底結帳會議 / 業務週會 / 電商月度檢討 / 庫存盤點 / 供應商議價
  - 每位學員產出一份整理好的會議紀錄
- **驗收（5 min）**：確認格式符合自己部門習慣，知道怎麼要求 Gemini 調整

#### Day 1 收尾 + 工作流卡 v1（10 min）

- 每位學員寫工作流卡 v1（1 條「本週想試的場景」+ 1 條「還不確定的事」）
- Day 2 路線圖預告：明天繼續深化兩個工具，最後把它們串起來用

#### Day 1 緩衝（40 min · Q&A + 上機卡關補強）

### Day 2：深化練習 + 雙工具串接（240 min）

#### Day 2 開場（10 min）

- 快速回顧 Day 1 三件事：兩工具差異 / NotebookLM 上傳查詢 / Gemini 三件套
- 工作流卡 v1 各自看一遍，今天結束前要更新為 v2

#### CH1-3：NotebookLM 個人知識庫建立（50 min）

**v3 定位**：v2 的三層知識庫（個人 / 部門 / 跨部門）對這批學員太重，v3 只做**個人層**——把自己最常查的工作文件整進一個 notebook。

- **什麼文件適合上傳（10 min）**：
  - 適合：部門 SOP、政策文件、會議紀錄、產品規格書、歷史報告
  - 不適合：含客戶個資的名單、高度機密合約（資安提醒）
  - 命名建議：讓一個月後的自己找得到（日期 + 主題）
  - 每個 notebook 建議 5-15 份文件（太少查不到、太多引用混亂）
- **學員建立個人知識庫（25 min）**：
  - 上傳 3-5 份自己部門的常用工作文件
  - 設定好 notebook 名稱（例：「業務月報庫」「採購規格庫」）
  - 講師巡場協助卡關的學員
- **查詢練習（15 min）**：
  - 每位學員對自己的知識庫問 5 個實際工作問題（「這個流程的負責人是誰？」「交期規定幾天？」「上季最高銷量是哪個品項？」）
  - 練習用 Study Guide 生成一份文件的快速摘要
  - 驗收：引用標記能對回原文嗎？

#### 中場（15 min）

#### CH1-4：Gemini 輔助彙報初稿（單部門）（45 min）

**v3 定位**：v2 的 CH1-4 是「5 主管追問風格 × 攻防壓力測試」，這批學員連基本彙報初稿還不夠熟，v3 改為**從資料生成一份完整初稿**的基本流程。

- **場景說明（5 min）**：
  - 目標：給 Gemini 你的部門資料，讓它幫你整理成彙報初稿
  - 不是讓 Gemini 全自動憑空生成，是**「給素材讓它整理」**
  - 本人要確認數字正確，不能直接 copy 上交
- **講師示範（10 min）**：
  - 上傳一份月報 Excel / CSV
  - Prompt：「根據這份資料，用條列方式整理出 3 個本月重點 + 1 個建議行動，給主管看的格式」
  - 示範如何確認數字來自資料而非 Gemini 自行推算
  - 示範如何要求格式調整
- **學員上機（25 min）**：
  - 每位學員用自己部門資料（或課程提供的範例資料）
  - 生成一份 3 重點 + 1 建議的彙報初稿
  - 嘗試一次格式調整（要求換成表格、或改成 5 條）
- **驗收（5 min）**：確認數字可追溯，知道哪些地方 Gemini 補充的需要自己核實

#### CH2-1：雙工具串接複習（個人場景）（35 min）

**v3 定位**：v2 的 CH2-1 是跨部門綜合彙報（72 min，分組角色扮演）。v3 改為個人場景的**串接複習**——用一個完整情境把 CH1-1 到 CH1-4 學過的兩個工具串起來走一遍。

- **情境設定（5 min）**：
  - 個人場景：「我要準備本月向主管報告，手上有部門 SOP + 本月數據」
  - NotebookLM 的角色：查出 SOP 裡的相關規定和歷史紀錄
  - Gemini 的角色：把查到的內容 + 本月數據整合成彙報段落
- **講師完整示範一遍（10 min）**：
  - Step 1：打開 NotebookLM → 查詢 3 個需要引用的點 → 截圖或複製引用
  - Step 2：打開 Gemini → 把 3 個引用 + 本月數據貼進 prompt → 生成彙報段落
  - Step 3：確認引用來源正確 → 調整格式
- **學員上機（15 min）**：
  - 用自己 CH1-3 建好的知識庫 + CH1-4 的彙報場景，串接走一遍
  - 不強求完整，完成 2 個步驟就算過
- **分享（5 min）**：1-2 位學員分享今天最有收獲的一個操作

#### 結訓 + 工作流卡 v2（20 min）

- 每位學員寫**工作流卡 v2 承諾版**（基於 v1 更新）：
  - 1 條「**本週要試的場景**」（含：用哪個工具 / 什麼資料 / 期望得到什麼）
  - 1 條「**還需要多練的地方**」
- 課後素材包導覽：雙工具速查表 + 5 部門變體 prompt + 範例 dataset
- 維護邊界說明：工具介面有變動 → 業主發信通知
