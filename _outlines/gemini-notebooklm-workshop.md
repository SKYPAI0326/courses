## 課程定位（Positioning v2）

學員已從 office-ai-cases 學過「多平台通用 prompt + 5 大骨架 + 56 套素材」，本課**不是換工具重學一次**，而是把雙工具升級成「**可信來源驅動的工作流**」——

- **Gemini 強在生成 / 推演 / 改寫**，但「沒給資料時會自己編」是它的核心風險
- **NotebookLM 強在引用來源 / 限定範圍 / 降幻覺**，但「不上傳就不答」是它的核心邊界
- 本課反覆訓練「**什麼時候不能直接問 Gemini，要先去 NotebookLM 取證**」——這比介紹功能更接近真實工作場景

8 小時內帶著 **4 件套產物**（知識庫設計表 + 來源可信度檢核表 + 追問補強包 + 工作流卡 v2）回部門，搭配底層的 dataset 與 prompt 集，直接套用。

## 受眾畫像（Audience Profile）

- **職業**：跨部門代表（財務 / 業務 / 電商 / 倉儲 / 商品採購），業務數據導向
- **既有訓練**：**已上過 office-ai-cases 6h**（多平台 prompt 骨架 / 4 部門案例 / 56 prompts / 4 datasets），對 Gemini 與 NotebookLM 有基本印象但未系統化使用
- **技術底子**：零代碼、Office 一般熟，ChatGPT / Gemini 用過幾次
- **現有工具棧**：Office 三件 + Gmail / Outlook + ERP / 電商平台後台 / 採購系統 / WMS / CRM
- **資料現況**：日常處理多為從上述系統匯出的 CSV / Excel，常含部門代碼、合併儲存格、空白列、欄位名稱不統一——**手上資料不一定乾淨**
- **痛點 3 條**（v2 改寫，與 office-ai-cases 痛點切開）：
  1. office-ai-cases 帶走的 prompt 庫能用，但**不知道哪些情境一定要先去 NotebookLM 取證**才能讓 Gemini 答得準
  2. 試過 NotebookLM 但建出來的知識庫**查詢結果不準、引用對不上**，不知道是 source selection 還是 prompt 寫法問題
  3. 主管彙報時 Gemini 給的初稿被追問「**這個數字哪來的**」就回不上來——需要一條能把證據鏈跟生成串起來的工作流

## 學習成果（Outcomes，7 條，動詞開頭、可驗證）

1. **能判斷** 在 6 個常見業務場景中「該先去 NotebookLM 取證、還是可直接用 Gemini 生成」，並說出判斷依據
2. **能完成** 個人層 NotebookLM notebook（1 notebook + 3 上傳規則 + 5 查詢題）+ 知識庫設計表 **Min 版**（第一週能做的）；理解三層架構（個人/部門/跨來源）作為 3-6 個月期程的 Pro 版藍圖
3. **能填寫** 來源可信度檢核表（每個 claim 標 🟢🟡🔴 source 等級 + 引用追溯位置），把競品/標竿研究的 5-7 條主張都掛回來源
4. **能用兩洞補強法**（1 數字洞 + 1 風險洞）跑「NotebookLM 補證 → Gemini 重寫」攻防，產出 **追問補強包**；能在 Gemini 改寫段標明引用是「**公司系統來源（ERP/CRM）**」還是「**外部行業參考**」，回應主管 ERP 數字追問
5. **能在 60 分鐘內** 完成跨來源綜合彙報（NotebookLM 多 source 並行取證 → Gemini 跨源整合 → 1 頁彙報 + **跨來源決策備忘**「可用/不可用/待授權」三層），用責任分工卡處理 3 種現實阻力（敏感資料/版本不明/部門不願共享）
6. **能完成** 個人工作流卡 v1→v2 兩輪迭代（Day 1 寫 v1 / Day 2 互審改 v2 / 課後 1 週業主收 v3）
7. **能識別** 3 大現實阻力（資安禁上傳 / SOP 沒版本 / 主管要 ERP 數字），並對每條選對應策略（sample 練 / LMD 代替 / 雙來源類型標記）

## 單元矩陣（8h = 480 min · 2 天 × 4h）

### Day 1：建立可信來源（240 min）

#### CH0：兩工具切換決策入口（25 min · 已壓縮，不重講 AI 基本盤）

**為何壓縮 35→25 min**：學員上過 office-ai-cases，AI 運作 / 5 失敗模式 / prompt 5 大骨架 / 風險 4 條已學過，本課不重講，只做差異化建立。

- **不再講**（office-ai-cases 已教）：AI 運作 / Token / Hallucination 機制 / Prompt Engineering 5 失敗 / 風險 4 條
- **本課重點**：
  - 兩工具「**信任模式**」對比（Gemini 強在生成推演 + 沒資料會編 / NotebookLM 強在引用限定範圍 + 不上傳不答）
  - 6 題切換決策樹（同一問題丟兩工具看差異）
  - Decision checkpoint 概念引入：**之後每個 CH 開場 3 min 會回到這 6 題問你一次**
  - 跨來源整合路線預告（CH2-1 是壓軸，從這裡就埋鉤）
  - 3 條免費版紅線（Gemini 32K / NotebookLM source 上限 / 隱私邊界）—— 不講通用資安，只講「**免費版會在哪邊撞牆**」

#### CH1-1：會議紀錄整理熱身（45 min · 含 3 min decision checkpoint）

**為何壓縮 70→45 min**：場景仍是 30 min 跨部門逐字稿 → 決議卡 / 個人行動表 / 跟進信，但 office-ai-cases 已教 prompt 5 大骨架，本課只做「Gemini 在會議文本上的特化」+ 為 CH1-2 的「個人知識庫」鋪源。

- **Decision checkpoint（3 min）**：今天這個場景「為什麼用 Gemini 不用 NotebookLM」？（答：要生成新內容、且原始素材是非結構文字、追溯需求低）
- **講師示範（10 min）**：跨部門逐字稿 + 工作日誌 → 決議卡 / 個人行動表 / 跟進信
- **Prompt 拆解（5 min）**：明示「這套 prompt 是 office-ai-cases 的 5 大骨架在 Gemini 會議場景上的特化，不再重講骨架本身」
- **學員上機（20 min）**：5 部門變體（月底結帳 / 業務週會 / 電商月度 / 庫存盤點 / 供應商議價）
- **驗證 + CH1-2 鋪源（7 min）**：把這份會議紀錄存起來，CH1-2 的「個人層知識庫」會用到它作 source

### Day 2：把可信來源變成可彙報決策（240 min）

#### Day 2 開場 + v1 同儕互審（15 min）

- 兩兩交換工作流卡 v1，**挑一條最不切實際的標出來**（不切實際 = 沒有 source / 沒有驗證 / 沒有具體下一步）
- 互審反饋寫在 v1 卡背面，留到結訓寫 v2 時參考

#### CH1-3：競品 / 標竿資料整理（60 min · 含 3 min decision checkpoint）

**為何 60 min**：本案實質是「NotebookLM 取證 + Gemini 整合」的進階版，與 office-ai-cases CH2-1 的電商銷售分析切開——本案不是表格 → 報告，**是文字研究 → 主張 + 證據鏈**。

- **Decision checkpoint（3 min）**：競品研究的哪些主張一定要有 NotebookLM 來源？哪些可放心讓 Gemini 推演？
- **講師示範（15 min）**：
  - NotebookLM 取證：5 篇同業競品文章 + 3 段訪談（含明顯偏見來源 + 高可信度來源混雜）
  - Gemini 整合：把 NotebookLM 抓出的引用片段組成 SWOT + 主管報告草稿
  - 來源可信度標記：每個 claim 標 source 等級（一手 / 二手 / 推論）
- **Prompt 拆解（5 min）**：含「**要 Gemini 標明哪段沒來源**」的 wrapper
- **學員上機（30 min）**：5 部門變體（同業財報 / 客戶分析 / 平台競品 / 倉儲效率 / 同業採購條件）
- **產物：來源可信度檢核表（7 min）**：每位學員的競品研究 5-7 條主張都掛回 source 等級（產物 #2）

#### 中場（10 min）

#### CH1-4：Gemini 輔助彙報壓力測試（60 min · 含 3 min decision checkpoint · **與 office-ai-cases CH2-2 徹底切開**）

**為何不是「資料 → 6-8 頁簡報內容包」**：office-ai-cases CH2-2 已完整教過「資料生成簡報初稿」流程，本案改為「**初稿存在後，主管會怎麼問？怎麼補？怎麼改？**」。產物是攻防後的「追問補強包」，不是初稿本身。

- **Decision checkpoint（3 min）**：當主管追問「這個數字哪來的」時，先回 Gemini 重寫還是先回 NotebookLM 補證？（答：先 NotebookLM 補證 → Gemini 改寫）
- **講師示範（15 min）**：
  - 用 office-ai-cases 帶走的範例初稿（或本課快速生一份）作攻防起點
  - **5 主管追問風格**輪流質問：保守型「這數字可靠嗎」/ 數字型「明細呢」/ 風險型「最壞情況」/ 戰略型「跟競爭對手比」/ 細節型「第三條第二句來源」
  - 每個追問 → 回 NotebookLM 補證（30 秒查 + 引用截圖）→ Gemini 重寫補強段（30 秒）
- **Prompt 拆解（5 min）**：證據缺口清單 + NotebookLM 補證 wrapper + Gemini「明示來源 + 限制風險」改寫 wrapper
- **學員上機（25 min）**：每位學員拿自己 CH1-3 產出的主管報告草稿，由同儕扮 1 種主管追問風格，跑完一輪攻防
- **產物：追問補強包（12 min）**：5 部門變體下，每位學員的追問補強包包含「5 追問 × 證據缺口 + NotebookLM 補證表 + Gemini 重寫段」（產物 #3）

#### CH2-1：跨來源綜合彙報（72 min · 含 3 min decision checkpoint · **占 15% 業主上限**）

**為何拉到 72 min（15.0%）**：跨來源整合是本課唯一無 office-ai-cases 對應的單元，作為品牌錢幣必須有足夠時間落地。72 min 守住業主上限，再透過 CH1-2 / CH1-3 / CH1-4 的能力滲透補足。

- **Decision checkpoint（3 min）**：跨來源綜合彙報的責任分工——誰建知識庫？誰整合？誰驗證？
- **情境設定（8 min）**：新品引進綜合決策（跨採購 + 業務 + 倉儲三部門），主版素材已就位
- **NotebookLM 多 source 並行示範（12 min）**：3 個獨立 notebook 並行取證 + 結果交叉比對
- **Gemini 跨源整合示範（12 min）**：把 3 個 NotebookLM 取證結果整合成 1 頁綜合彙報 + 不確定性標記
- **分組上機（25 min）**：3 人 1 組（採購 + 業務 + 倉儲角色扮演）完成跨來源彙報
- **組間互審 + 改進（10 min）**：A 組評 B 組「責任分工卡是否清楚 + 不確定性標記是否充分」
- **產物：跨來源責任分工卡 + 1 頁綜合彙報範本（2 min 收）**

#### 結訓 + 工作流卡 v2（15 min）

- 每位學員寫 **工作流卡 v2 承諾版**（基於 v1 + 同儕互審反饋）：
  - 1 條「**本週要試的場景**」（含 source 來自哪 / 要用哪個工具 / 怎麼驗證）
  - 1 條「**本月要導入的工作流**」（含三層知識庫的哪一層先建 / 預期完成時間）
- 課後素材包導覽（4 件套模板 + 雙工具速查表 + 5 部門變體 dataset）
- **v3 課後回收機制說明**：1 週後業主發信收 v3（學員實際試完後的反饋版），作為下一梯次課程改進依據
- 3 個月維護邊界：工具介面變動 / 模板更新 → 設計師修網頁 → 業主發信通知

## 品牌風格指引（與 design-tokens.md + 操作型講義.md 對齊）

- **禁用詞**：賦能 / 數位轉型 / 高效 / 痛點 / 您 / 我們（除引號講師話術）
- **改用**：做法 / 卡住的事 / 你
- **平台指派**：**本課指定 Gemini + NotebookLM，主案例 prompt 直接寫「在 Gemini」「在 NotebookLM」**
- **付費功能標記**：免費版限制要明寫，付費版（Gemini Advanced）僅在 trouble-shooting 段提及，標 💰
- **簡報承諾邊界**：「可貼進簡報的內容包」「追問補強包」「主張 + 證據鏈」，禁「自動生成可直接交付的報告」「直接下載 .pptx」
- **不重講基本盤**：office-ai-cases 已教過的內容（AI 運作 / Token / Hallucination / Prompt 5 大骨架 / 5 失敗 / 風險 4 條），本課**只在 1 句話帶過 + 引用「上次學過」**
