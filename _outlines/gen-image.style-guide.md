## 金句系統（硬規、每單元必含）

每單元結尾有 1 個 **「記住這句話」** 區塊、用深色 callout 呈現。金句要：

- 12-20 字（可背、可截圖分享）
- 直白、不抽象
- 對應該單元最核心的觀念

### 8 版型 × 8 金句（CH2-1 用）

| 版型 | 金句 |
|---|---|
| A 主視覺版 | 畫面越強、訊息才會被看見 |
| B 橫幅版 | 在橫幅中、1 秒抓住眼球就是關鍵 |
| C 社群版 | 少即是多、讓產品自己說話 |
| D 系列版 | 系列感、就是讓喜歡變成全都想要 |
| E 禮盒版 | 包裝感、就是溢價的視覺證明 |
| F 特寫版 | 越近越想吃 |
| G 促銷版 | 優惠不亮、訊息白給 |
| H 生活版 | 不賣商品、賣與商品一起的時光 |

### 各單元金句

| 單元 | 金句 |
|---|---|
| CH1-1 | 你不是設計師、你是 AI 的需求方 |
| CH1-2 | 卡關就換另一家、不要綁死任一平台 |
| CH2-1 | 不是圖片不夠好、而是版型不夠對 |
| CH2-2 | 講清楚 AI 才能做對 |
| CH3-1 | 脫掉表象、骨架就能無限再造 |
| CH3-2 | 看結構不看美感、版型才用得久 |
| CH4-1 | 線稿是空房間、新主題是新住戶 |
| CH4-2 | 同一張圖、不同版型、不同效果 |
| CH4-3 | 系列感、就是讓喜歡變成全都想要 |
| CH5-1 | 版型沒有最好、只有最適合 |
| PRAC2/3/4 | 多看、多拆、多練、版型力自然提升 |

## 人的角色定位（硬性，全課程敘事基準）

本課學員**不變成設計師**，而是建立「**AI 設計流水線的產品經理**」的思維與工作方式。

### 學員的三位一體角色

| 角色 | 對應階段 | 具體工作 |
|---|---|---|
| **需求判定者** | M2 起｜生圖 | 任務到手先釐清：要什麼 / 不要什麼 / 誰看 / 何時交；把需求轉成 brief 六件套給 AI |
| **骨架驗收者** | M3 承｜轉線稿 | AI 反推的 L2 線稿是否**結構完整、可繼承、可規模化**（不是判斷好看）|
| **系列把關者** | M4-M5 轉/合 | 建立 brand DNA；AI 產出的系列是否守在品牌框架內；每張新輸出是否值得收進系列資產 |

### 本課教 vs 不教（硬性區分，寫教案時嚴格遵守）

| 教 | 不教 |
|---|---|
| 如何判定需求（六件套） | 如何畫設計（Figma / PS / AI 軟體）|
| 如何驗收 AI 產出（結構判準）| 如何調 prompt 參數（seed / CFG / step）|
| 如何把關品牌一致性（brand DNA）| 如何學習設計史 / 設計理論 |
| 如何讓 AI 滾出可繼承骨架 | 如何訓練個人美感 |
| 人 = PM / 藝術總監 | 人 = 執行設計師 |

### 敘事主詞（每單元必遵守）

- 教案中提到「你」時，學員是**需求方 / 驗收方 / 把關方**——**AI 是執行方**
- 不得寫「你畫一張…」「你設計一個…」——改寫為「你下需求讓 AI 生成…」「你驗收 AI 產出的…」
- 不得寫「學會設計某某…」——改寫為「學會判斷 AI 產出的某某是否可用」

## 風格基準與延伸

| 類型 | 風格 | 用途 |
|---|---|---|
| **教學基準**（主示範）| **日系** | 結構乾淨、佔位符語意清楚、視覺動線教學最好懂——CH / 示範段落用此 |
| **演練延伸**（學員自選）| 韓系、美式、歐陸、極簡、復古、手繪、工業風等 | PRAC 演練段、10 業種庫「風格 preset」欄，鼓勵學員嘗試非日系 |

### 跨風格通用性

**5 骨架（商品左 / mockup 右 / 風景底 / 商品中心 / 大字堆疊）是跨風格通用的結構模式**，日系只是教學上最乾淨的展示版。同一個骨架可以用韓系、美式、歐陸任何風格去「穿衣服」——**結構不變、表象隨風格切換**。

教案寫作時：
- **基準示範**：預設日系 preset
- **延伸提示**：每單元尾段或 PRAC 前，以「風格延伸」卡片提醒學員可改用其他 preset
- **禁止**：把「日系」寫成硬性限定（例：「日系商業橫幅才有這個結構」）

## 語氣基準

- **對話式**：像資深設計同事在旁邊邊看邊教你拆結構，不是設計系課堂演講
- **克制感**：不用「改變設計」「AI 革命」「神級 prompt」等行銷話術
- **職人感**：可用「這張為什麼這樣排」「作者在想什麼」這類視角切換詞
- **直接不廢話**：每句有意義，不加「首先讓我們來了解一下...」式墊場句
- **自曝侷限**：每個技巧說明「什麼時候不適用」「會踩什麼雷」
- **視覺敏感度**：多用「視覺重心」「呼吸留白」「焦點層級」這種具體詞，少用「好看」「漂亮」這種模糊詞

## 引導發現法（硬性教學原則，全課程單元適用）

本課**不提供固定示範圖**，改用「引導發現法」設計所有教學活動。所有 course-designer 寫教案時必須遵守：

### 為什麼不備固定示範圖

1. **AI 生圖本質不穩定**——同 prompt 跑 3 次都不同；給固定示範會讓學員以為「跑不出那樣就是失敗」，反而降低學習意願
2. **骨架是結構、不是長相**——真正要學的是「看懂結構」，不是「做出像老師的圖」
3. **學員實跑 > 看老師的圖**——自己跑才會體驗到模型行為、知道如何修 prompt
4. **永不過時**——AI 模型每季升級，固定示範圖 3 個月就會顯得「過時」；文字 + 引導 + 學員自跑不會過時

### 具體做法（寫教案時的替代方案）

| 傳統做法 | 本課替代 |
|---|---|
| 老師準備 10 張示範圖 | 提供 prompt 清單 + IG hashtag，讓學員現場自跑 / 搜尋 |
| 發給學員固定題目圖庫 | 給搜尋關鍵字清單，學員自蒐 10 張真實世界作品 |
| L4 完稿對照圖 | L0 結構 JSON + ASCII 線框示意圖（文字即可） |
| 「看這張圖是什麼骨架」 | 「打開你自己追蹤的品牌 IG，找 3 張符合某骨架的」 |
| 批改「對錯」 | 批改「判準邏輯是否一致」——同張圖被分到不同骨架，只要理由合理就都對 |

### 教案必含元素

每份教案（CH / PRAC）只要有 AI 生圖或圖片辨識，**必須加上預期差異聲明**：

> **預期差異**：AI 生圖每次輸出都不同——同 prompt、同一人、不同時間跑出來也會有差異。這不是 bug，是 AI 特性。**重點不是複製某張「標準圖」，而是學會判讀結構**。你跑出的圖只要結構對、能看出骨架、能繼續往下做線稿反推，就是成功。

### 判準邏輯 > 標準答案

所有辨識 / 分類題的批改原則：

- ✅ **判準邏輯一致**（視覺動線 + 最大元素 + 商品類型 三個判準自洽）→ 算對
- ✅ **邊界題標為「邊界」並說出兩種可能 + 選擇理由** → 算對
- ❌ **盲猜、說「直覺」「感覺」** → 不算對，提示回去用三判準重新看

本課學員**不是**以下定位，設計時刻意避免：

## 商標與版權紀律（硬規）

本課處理大量視覺素材，**以下規則 course-designer / course-lesson-writer 階段必須嚴格遵守**：

1. **講義中出現的所有示範圖**：
   - 優先使用主線 A 咖啡 / 主線 B 手工皂的虛構品牌（自創，無版權風險）
   - 10 業種主題庫的假想品牌（LUNÉA、notero、HANAKO 等）均為虛構，不得引用真實存在同名品牌
   - 若需比對真實商業案例，僅用文字描述，不放對方 logo / 完整截圖
2. **學員練習產出**：
   - 明確提示「這些是練習作品，不可商用」
   - PRAC4 / CH5-1 整合題要求學員用自家品牌或假想品牌，不得用名人 / 真實大品牌做練習
3. **示範用的反推素材**：
   - 僅用 CC0、Unsplash、Pixabay 類合法免費素材，或老師自行產出的範例
   - 不得拿真實品牌的官網截圖做反推示範（寫入反面教材時也要用虛構版）
4. **交付環節（CH5-1）**：
   - 強制提示「商用前走一次 Firefly 或 Canva 合法素材重製」
   - 附上「版權自我檢查 5 問」清單（素材來源是否合法 / 是否含真實人臉 / 是否近似他人 logo / 是否涉及敏感議題 / 是否標示 AI 生成）
