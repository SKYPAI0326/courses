# Style Guide — prompt-basic

> 本檔覆蓋 `_style_guide_template.md`，course-designer / course-lesson-writer 讀本檔，不讀 template。

---

## 受眾稱呼

- 稱呼學員：**你**（不用「您」）
- 不使用具名角色（無「阿凱」「雯姊」等人設）
- 學員代入身分：「一位承辦人員」「你的機關」「你的主管」

---

## 案例情境規範

### 情境白名單（只用這些）

| 情境類型 | 說明 |
|---|---|
| 陳情案回函 | 婉拒或說明處理結果 |
| 會議紀錄整理 | 決議清單、行動清單、待辦追蹤 |
| 公文撰寫 | 簽呈、開會通知、函稿 |
| 長文摘要 | 施政說明、政策文件、會議記錄 |
| 潤稿翻譯 | 公文語氣調整、簡短英文翻譯 |
| 向長官呈報 | 200 字以內簡短說明 |

### 情境禁用清單

- 行銷文案、廣告設計
- 程式碼撰寫
- 技術文件
- 學術論文
- 創意寫作（小說、詩、劇本）
- 求職、人資篩選
- 客服對話設計

### 通用化原則

- 機關名稱：用「某縣市政府○○局」「本機關」「主管機關」等通用表達
- 人名：用「陳○○先生」「李○○女士」「○○主任」等去識別格式
- 案號：用「某案件」「案號 XXXXXX」等虛構格式
- **不寫入水利署、河川、水利等專業術語**（除非明確標記為水利署版素材）

## 術語對照表（硬性，全課程一律）

| 禁用 | 替代 |
|---|---|
| Prompt / 提示詞 / 指令工程 | 交辦單 / 交辦 |
| Few-shot | 給範例 |
| Chain-of-Thought / CoT | 請它一步步想 |
| Self-consistency | （不教，整個概念不出現） |
| Context / Context window | 背景資料 |
| Token | 字數 |
| Structured output | 指定回答的樣子 |
| Meta-prompt | 問題清單（必要時才出現） |
| Eval / Evaluation set | 幾題固定考題 |
| A/B testing | 新舊版本比一比 |
| Sycophancy | AI 的奉承傾向（必要時才提） |
| RTCF / CRISPE / 任何英文框架縮寫 | 不出現 |

## 每單元必要元素（course-designer 強制要求）

### CH 頁面（概念/技巧單元）

| 元素 | 要求 |
|---|---|
| 爛問法 vs 好問法對照 | **每個 CH 單元至少 1 組**，需包含實際 AI 輸出文本（不是示意） |
| 使用時機說明 | 每個技巧必須說明「什麼時候最好用 / 什麼時候不適用」 |
| 公務情境類比 | 每個核心概念用 1 個公文/陳情/會議情境解釋 |

### PRAC 頁面（練習單元）

| 元素 | 要求 |
|---|---|
| 預填素材 | 情境、背景資料、任務說明**全部由課程提供**，學員不自己想 |
| 填空而非開放題 | 學員填空格或選選項，不要求從零寫作 |
| 驗收標準 | 每題有「老師標準答案」或「評判清單」 |

## 語氣基準

- **直接，不廢話**：每句有意義，不加「首先讓我們來了解一下...」式的墊場句
- **不誇張**：不用「神奇」「驚人」「保證有效」等詞
- **自曝侷限**：每個技巧說明「什麼時候不適用」，而非一味推銷
- **工程師式乾幽默可出現**，但不強求

---

## 禁用行銷話術

- 「AI 會取代人力」「效率提升 10 倍」「AI 革命」
- 引用 Google / OpenAI / Anthropic 的公司故事
- 「神級 prompt」「終極咒語」「絕密技巧」等玄學詞彙
