# Style Guide — line-stickers

> 本檔覆蓋 `_style_guide_template.md`，course-designer / course-lesson-writer 讀本檔，不讀 template。
> 本課定位：全免費工具 + 自用導向 + 12h 兩階連續課程（Part 1 入門 4h + Part 2 進階 8h），與 gen-image（商業用付費圖片生成）刻意區隔。

## 案例主角（雙主線示範角色）

本課用 AI 產 LINE 貼圖，所以「示範角色」就是課程裡反覆出現的虛構貼圖角色，全課程鎖定兩條主線方便學員代入：

### 主線 A：橘貓上班族（簡稱「橘貓」）

- **角色設定**：圓滾滾橘色虎斑貓，戴白領巾，個性溫和但偶爾無語；扁平卡通風格、粗線條、暖色調
- **使用情境**：日常表情組（笑、哭、生氣、無語、晚安、求拜託、嚇到、得意）
- **主要出現單元**：CH1-2、CH1-3、CH1-4、CH1-5、CH1-6、CH2-2、CH2-3、CH2-4

### 主線 B：粉紅水母（簡稱「水母」）

- **角色設定**：透明粉色小水母，圓眼睛、軟身體，個性活潑可愛；漸變水彩風格（為了示範「不是只有卡通能做」）
- **使用情境**：節慶表情組（生日、聖誕、新年、加油）+ 進階多風格示範（Q 版 / 寫實版 / 季節版）
- **主要出現單元**：CH2-5、CH2-6、CH2-7、CH2-8

### 使用原則

- **每單元鎖定一條主線**，不要兩條來回跳
- **橘貓/水母是「校準範例」，不是「作業題材限制」**：學員自選角色時不強制這兩個，但講師示範一律用這兩個，避免每張示範圖都不同學員更亂
- CH1-2 的角色設定卡示範必須包含**「換成你的角色」橋接模板**，至少覆蓋 4 類起手式：寵物擬人（橘貓 / 柴犬 / 倉鼠）/ 真實人物擬卡通（自己 / 伴侶 / 小孩，需肖像授權）/ 物品擬人（咖啡杯 / 行李箱）/ 幻想生物（水母 / 雲朵 / 小怪獸）。讓學員能對號入座
- 角色一致性檢查清單的「對照圖」一律用橘貓的 8 張正解
- **水母（粉紅水母漸變水彩風）難度高於橘貓**：去背邊緣不易乾淨、APNG 壓縮較吃資源。CH2-5 第一次出現時必須提示「水彩風較難壓縮，進階用；入門請先用輪廓清楚的角色」

### 規格術語（本課核心，鎖定）

不要換別的說法，每單元引用同一組命名：

1. **主貼圖**（370×320 px PNG，套組裡的 8/16/24/32/40 張那種）
2. **主圖**（240×240 px PNG，封面那張）
3. **Tab 圖**（96×74 px PNG，鍵盤上的小圖示）

## 語氣基準

- **親切務實**：像鄰居教你做手作，不是設計師講座
- **手作感**：可用「自己做」「做給朋友」「我家貓」「老婆生日」這種生活詞
- **直接，不廢話**：每句有意義，不加「首先讓我們來了解一下...」式的墊場句
- **不誇張**：不用「驚豔」「神圖」「秒過審」「100% 通過」等行銷詞
- **自曝侷限**：每個技巧說明「AI 在什麼時候會崩 / 角色容易跑掉的時機」
- **退件當常態**：本課定位「被退是過程不是失敗」，避免把過審塑造成終點

## 每單元必要元素

### CH 頁面（本課 14 個 CH 單元 + 2 個 PRAC 實作）

| 元素 | 要求 |
|---|---|
| 開場情境一句話 | 每單元第一段用「橘貓」或「水母」主角開場（「橘貓上班族今天要產『笑著苦笑』這張表情，但 AI 第三次又畫成不同隻貓...」） |
| Prompt / 操作範本 | 每單元至少 1 則完整 Prompt 或操作步驟，含填空提示 + 示範輸出 |
| 建議工具 tag | 每則操作標示建議用 Gemini / ChatGPT / Pixian / Photopea / Creators Market |
| 視覺示範圖 | 每單元至少一張「正解圖 vs 失敗圖」對照（角色一致性、邊緣處理、規格錯誤等）|
| 邀請實作 callout | 每單元結尾一個「現在換你」的 60 秒小行動 |
| 卡關備案 | 常見失敗（AI 拒答、額度用盡、Photopea 卡頓、APNG 預覽異常、上架表單卡關）的 Plan B |
| 規格速查 | 涉及 LINE 規格的單元，附「規格速查卡」連結 |

### PRAC 實作頁

- 學員提交：作品集（8 張或 24 張）+ 上架截圖 + 角色一致性自查表
- 因 LINE 審核需 1-7 天，PRAC 為**課後作業**，不在課堂內驗收最終過審結果
- 提供「過審」與「被退」兩種狀況的後續行動指引

### 補充頁面

- **index.html**：課程總覽 + 兩階段路徑圖 + 規格速查卡入口
- **module1.html / module2.html**：各自單元導覽
- **specs.html**：LINE 貼圖規格速查頁（三類貼圖 + 退件 SOP）
- **prompt-library.html**：本課 Prompt + 操作 SOP 索引（角色設定卡 ×3、表情變體 ×8、動作關鍵幀 ×5、訊息貼圖版型 ×3、退件修補 ×5）

## 與兄弟課程的差異化

本課學員**不是**以下定位，設計時刻意避免混淆：

| 課程 | 差異 |
|---|---|
| gen-image（奶茶棕）| 商業用、付費工具（含 MJ）、商業視覺；line-stickers 自用、全免費、單一格式（LINE 貼圖）|
| simple-ai（鼠尾草綠）| 創業者文字 AI 入門；line-stickers 個人圖像創作 |
| ai-workshop（霧藍）| 12h 生產力精通桌面工具；line-stickers 12h 連續但鎖單一交付物（一套貼圖）|
| office-ai（芥末黃）| Office Copilot；line-stickers 完全不碰辦公套件 |

**核心區別**：只有 line-stickers 鎖定「LINE 上架完整流程 + 全免費 + 自用」三條件。不追求商業價值，追求學員拿到一套自己滿意、能跟朋友炫耀的貼圖。

## Designer / Lesson-Writer 寫作優先級

製作教案與 HTML 時的硬性提醒：

1. **CH1-2、CH1-3 是課程命脈**（135 分鐘合計）：要寫到極致細，多 Prompt 模板、多失敗對照、多自查清單。學員是否做出 8 張一致表情，就是 Part 1 成敗指標。
2. **每個示範圖請放兩組**：「角色一致 vs 角色跑掉」對照、「邊緣乾淨 vs 邊緣有殘影」對照、「尺寸正確 vs 規格錯誤」對照——本課重度依賴視覺示範。
3. **Part 1 教案不討論 Part 2 內容**：CH1-6 結尾可以「埋種子」（例：「想升級到動態貼圖、24 張大套組、訊息貼圖嗎？Part 2 給你」），但不教動畫原理。
4. **Part 2 開頭不可重講 Part 1**：CH2-1 直接從動態貼圖原理切入；只允許「上次你做完 8 張靜態，現在我們把它變成會動的」這種一句話橋接，不重炒冷飯。
5. **退件範例必須去識別化**：用虛構的「橘貓挨打範本」，不是真實學員的退件信。
