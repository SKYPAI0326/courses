# Style Guide — career-pivot-mid

> 本檔覆蓋 `_style_guide_template.md`，course-designer / course-lesson-writer 讀本檔，不讀 template。
> 本課定位：轉職實戰 / 8h 一日密集 / 互動工作坊型（PDP × MBTI 雙測為核心 + 履歷面試實戰）/ AI 是輔助不是主軸。
> **核心心法**：年紀沒有幫助。年資 ≠ 競爭力，可遷移的實力才是。市場會問你年紀，要的不是用年紀辯護，是用實力站著。

---

## 受眾稱呼

- 稱呼學員：**你**（禁用「您」）
- 不使用具名虛構主角（不像 simple-ai 有「老闆」「阿凱」「雯姊」）。資深學員自我認同強，**直接「你」對話**比設定第三人稱代入更有效
- 講師自稱：**「我」**（對話式，像「比你經驗多 5 年的前輩」）

---

## 案例主角（雙主線示範角色）

本課用兩條虛構主線跑「轉職」具體情境。所有範例履歷 / 面試 / 薪資談判都掛這兩條，方便學員代入。**主角設定強調年資（工作經驗事實），不強調年齡**：

### 主線 A：製造業中階主管 → 新創 PM（簡稱「主角 A」）

- **背景**：製造業（工具機）QA 主管，**工作 22 年資**，年薪 NT$ 120 萬
- **轉職動機**：產業外移、AI 自動化衝擊、想做更貼近終端用戶的工作
- **目標職位**：新創公司產品經理 / 製造業數位轉型顧問
- **痛點**：履歷 22 年資寫不完、新創怕他「保守」、薪資可能要降到 NT$ 90 萬
- **PDP × MBTI**：貓頭鷹 C 主導 + INTJ（分析型 + 規劃型）
- **可遷移技能**：QA 流程 / 客戶溝通 / 跨部門協調 / 數據判讀
- **主要出現單元**：CH2-1（履歷示範）、CH2-3（為什麼這年紀換）、CH2-5（接受降薪劇本）

### 主線 B：行政 → 行銷企劃（簡稱「主角 B」）

- **背景**：國貿公司行政主管 8 年（前 6 年是業務助理），**工作 14 年資**，年薪 NT$ 75 萬
- **轉職動機**：行政天花板、想往創意 / 內容方向走、想多挑戰
- **目標職位**：B2B 公司行銷企劃 / 內容運營 / 客戶成功
- **痛點**：行政被當「萬年螺絲釘」、無行銷作品集、年輕競爭者一堆
- **PDP × MBTI**：孔雀 I 主導 + ENFP（影響型 + 探索型）
- **可遷移技能**：內外溝通 / 文件撰寫 / 跨部門協調 / 客戶關係維護
- **主要出現單元**：CH1-4（可遷移技能盤點示範）、CH2-1（履歷示範另一條）、CH2-4（模擬面試示範）

### 使用原則

- **每單元主要鎖定一條主線**：CH2-1 履歷示範雙主線並陳（看不同行業切入點）；其他單元擇一主示範
- 講不同 PDP × MBTI 組合的職位建議時，要點名「主角 A 是貓頭鷹 INTJ 所以...」「主角 B 是孔雀 ENFP 所以...」**用具體角色帶觀念**
- **不貼年齡標籤**：不寫「48 歲主角 A」「42 歲主角 B」，改寫「22 年資主角 A」「14 年資主角 B」。年資是工作事實，年齡不是課程主軸

### 情境禁用清單

- **不討論「斜槓」「自由工作者」「數位遊牧」**：這是另一門課的範圍，本課鎖定「找下一份正職工作」
- **不討論創業**：本課不教創業前準備（CH2-6 副業試水僅限「為轉職鋪路」，不是創業）
- **不討論移民 / 海外工作**：本課鎖定台灣就業市場
- **不討論退休後二度就業**：受眾主軸是「還有 10-20 年職涯」的人
- **不深入單一產業**：可舉例不可深入。要深入某產業學員去找該產業專書 / 顧問

### 核心命名（本課鎖定，每單元引用同一組）

不要換別的說法：

1. **PDP × MBTI 雙測**（不是「人格測驗」、不是「性向測驗」）
2. **工作質性對應表**（不是「適合什麼工作」）
3. **可遷移技能盤點**（不是「技能盤點」）
4. **斷捨離履歷**（不是「精簡履歷」）
5. **轉職面試 5 大問題**（不是「常見面試題」）
6. **薪資談判 3 劇本**（守住 / 接受降薪 / 主動降換成長）
7. **30/60/90 天計畫**（不是「新人計畫」）
8. **不裸辭策略**（不是「轉職方法」）

## 語氣基準

- **務實沉穩**：像「比你經驗多 5 年的職涯前輩坐你對面講」，不是激勵演講
- **重量感**：不用「機會」「夢想」「人生」這種大字眼，用「下一份工作」「未來 10 年」這種具體詞
- **直接，不廢話**：每句有意義，不加「首先讓我們來想想...」式的墊場句
- **不諂媚**：不誇學員「你很棒」「你有潛力」，誇要具體（「你過去 10 年的客戶溝通能力 = 新領域的可遷移技能」）
- **承認侷限**：每段方法說「什麼時候不適用 / 什麼樣的學員需要客製」
- **不販賣焦慮**：不用「再不轉就晚了」「AI 要取代你」這種威脅句；用事實數據降溫
- **不用年紀辯護**：講主角 / 範例時，要把焦點放在「實力」「可遷移技能」「市場條件」，不要拿年紀當理由（無論是優勢還是劣勢）

## 禁用詞（轉職實戰課空氣詞黑名單）

這批詞會讓學員覺得「你不懂在想什麼」：

- 「實現自我」「活出自己」「找到天命」（過度勵志）
- 「跨出舒適圈」「擁抱改變」（陳腔濫調）
- 「AI 革命」「AI 浪潮」「下一個十年屬於 ___」（恐嚇行銷）
- 「斜槓青年」「數位遊民」「自由工作者」（受眾不符）
- 「年齡只是數字」「永遠不嫌晚」（學員心裡知道嫌晚；本課直接講「年紀沒有幫助」更誠實）
- 「成功學」「複利效應」「個人 IP」（培訓黑話）
- 「賦能」「生態系」「閉環」（管理顧問廢話）
- 「再不換就被淘汰」「躺平」「內捲」「卷王」（販賣焦慮）
- 「大叔大媽」「老白男」（標籤化年齡群體）

## 每單元必要元素

### CH 頁面（本課 13 個 CH 單元 + 2 個 PRAC 實作）

| 元素 | 要求 |
|---|---|
| 開場情境一句話 | 每單元第一段用主角 A 或 B 開場（「主角 A 坐在白板前發呆，22 年資的履歷要砍成 2 頁——他不知道該砍哪些...」） |
| 心理門檻提醒 | 每單元至少一句「轉職特化」提示：「這段你會覺得『年輕人才需要』，但其實...」 |
| 紙本 / 表單 | 每單元至少 1 份紙本或 fill-form 給學員當場動手 |
| 「為什麼」連結 | 每個方法步驟附「為什麼這樣做」一句話（資深學員需要理解 why，不接受「照做」）|
| 反例展示 | 每單元至少一個「轉職常見誤區」反例：「這樣寫履歷 / 這樣回答面試會 NG，原因是...」|
| 量化數據 | 涉及市場 / 薪資 / 通過率的地方一律引具體數字，不模糊講「越來越多人」|
| 結尾 1 分鐘反思 | 每單元結尾「請你寫下：這 X 分鐘對你最有用的 1 件事」（PRAC 用） |

### PRAC 實作頁

- 課後 1 週內完成，不擠進 8h
- PRAC1：個人 SWOT + 3 目標職位排序表 + 雙測結果截圖
- PRAC2：完整履歷 PDF + 5 個面試題自己的答法 + 90 天入職計畫
- 講師批改回饋（不在課堂內）

### 補充頁面

- **index.html**：課程總覽 + 雙 Part 路徑圖 + 6 紙本素材入口 + 雙測測驗入口
- **module1.html / module2.html**：各自單元導覽
- **personality-test.html**：PDP × MBTI 雙測獨立頁（可在課前或課中跑）

## 與兄弟課程的差異化

本課與既有 14 門課**完全不在同個賽道**——既有都是「工具技能 / AI 應用」，本課是「人 + 職涯思考」工作坊。

| 課程 | 差異 |
|---|---|
| simple-ai（鼠尾草綠）| 創業者用 AI 工具；本課是轉職思考，AI 只輔助 |
| line-stickers（陶土橘）| AI 自製貼圖技能；本課完全不教 AI 工具操作 |
| ai-workshop（霧藍）| 12h 桌面 AI 工作坊；本課完全不教 AI 應用 |
| ntub-seo-ga4（霧紫）| **同色但完全不同主題**——SEO / 數據分析 vs 轉職實戰 |
| 所有「Xh 課程」| 多是技能精通；本課是「職涯體檢工作坊」，**唯一一門非工具課** |

**核心區別**：只有 career-pivot-mid 鎖定「轉職 + 跨領域 + 實戰準備 + 思考型工作坊」四條件。不追求工具精通，追求學員當天帶走「一份履歷骨架 + 一個 3 年職涯計畫 + 一份心理鬆綁」三件事。

## Designer / Lesson-Writer 寫作優先級

製作教案與 HTML 時的硬性提醒：

1. **CH1-2 PDP × MBTI 雙測是命脈**（60 min 佔 Part 1 25%）：教案要寫到極致細，30 題問卷 + 16 題問卷 + 計分腳本 + 16 型 × 工作質性完整對應表（不是 4 個範例，要 16 型都寫到位）。學員是否能在這 60 分鐘內找到「3 個適合的職位類型」，就是本課成敗指標。
2. **CH2-1 履歷 STAR 是命脈**（40 min 佔 Part 2 17%）：教案要附「進階版量化動詞清單」+ 雙主線完整履歷範例。
3. **示範圖 / 範例履歷雙主線並陳**：每個單元都盡量「主角 A + 主角 B 各示範一次」，學員從中找最像自己的那條。
4. **AI 工具只在 CH2-2 / CH2-3 出現**：其他單元嚴禁帶 AI 工具操作（會破壞「思考工作坊」定位）。
5. **不販賣焦慮**：所有方法說明都要附「為什麼這樣做」+「什麼時候不適用」。資深學員不接受「照做」，要理解 why。
6. **量化數據必須具體**：CH1-5 市場地圖、CH2-5 薪資談判等涉及數據的地方一律引具體數字（薪資中位數、轉職成功率、AI 衝擊職位列表），Designer 階段要實際調研 2026-Q2 當下資料。
7. **退路 / 救援設計**：CH1-2 雙測結果矛盾、CH2-1 寫不出量化成就、CH2-4 模擬面試學員不敢說話——每個命脈段落都要有「卡關備案」。
8. **核心心法貫穿**：所有單元提到「為什麼這年紀換」「為什麼跨領域」等話題時，回到「年紀沒有幫助、年資 ≠ 競爭力、用實力站著」這個基準。不要寫成「中年特有優勢」式的安慰文案。
