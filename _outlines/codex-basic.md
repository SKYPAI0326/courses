## 課程定位（Positioning）

帶非技術背景的會計／財務與一般辦公族，用 Codex 桌面 App 把「整理檔案、彙整表格、草擬文件郵件、封裝重複流程、外掛擴充」這套日常工作交給 AI 助理；3 小時走完一支真實能力巡禮，學完能在自己的工作資料夾裡指揮 Codex 完成一輪辦公／財會雜務。

## 受眾畫像（Audience Profile）

- **職業**：會計／財務／行政人員為主，兼及任何想用 AI 助理處理檔案的辦公族
- **技術底子**：非技術背景、零代碼；會用 ChatGPT、Excel、Word
- **現有工具棧**：Excel／CSV、Word、Gmail；尚未用過 AI agent 類工具
- **痛點 3 條**：
  1. 收到多人傳來、命名雜亂的檔案，手動改名與彙整耗時
  2. 月結／對帳要重複做合併、去重、檢查，每次重來
  3. 聽過 AI 很強但不知道邊界，怕它亂補資料、怕出錯沒人擋

## Brand Brief（品牌調性）

- **tone_register**：技術友善（像同事手把手帶，不堆術語）
- **mood_keywords**：沉穩、職人感、可信賴
- **differentiation**：兩個差異點——(1) 介面友善、全程不碰終端（GUI App、零代碼），與偏技術的 AI／自動化課區隔；(2) 不是「AI 無所不能」的炫技課，而是「會用、也守得住底線」，每個功能都配一句「哪裡要人把關」，貼合財會高責任場景

## 學習成果（Outcomes）

> 每條成果都對應主線情境裡一個學員親手做出的產出物（見單元矩陣「→ 產出物」），用「做得出來」驗證學會，而非「聽過」。
> **可重現成果**（學員課後能自己重做）＝下列 1–5（CH1-1～PRAC2 + CH3-1）。Part 3-2 起的外掛、Part 4 的排程/部署/Computer Use 屬**能力地圖展示**——版本/方案/帳號相依，課堂示範為主，不列為可重現成果。

1. 能在 Codex 桌面 App 建立工作資料夾、撰寫 `AGENTS.md` 操作守則並開出可運作的專案
2. 能指揮 Codex 完成日常檔案整理（批次重新命名、比對、抽取資訊）並產出處理日誌
3. 能用 Codex 把多份雜亂表格彙整成一張 Excel、合併去重並建立 Checks 檢查表（含供應商主檔欄位、發票複合鍵、稅務/科目待覆核紅線）
4. 能把重複流程封裝成專案層級／全域層級 Skill（一個 skill 資料夾含 `SKILL.md`），並用自動或斜線方式觸發
5. 能判斷哪些動作必須人工把關（寄出、覆寫、最終決策、稅務/科目判斷），落實權限模式（最小權限）與「不得補值」底線
6. （能力地圖，非可重現成果）能在講師示範下說明外掛擴充、排程、Sites 部署、Computer Use 各能做什麼、各自哪裡要人把關

## 主線情境（貫穿全課的案例）

**情境設定**：學員扮演「弄一下工作室」的行政會計人員，6 月底要做月度報帳與供應商整理。一包雜亂資料丟在桌上，全課就把它一步步變成乾淨的彙整表、通知信、可重用流程，最後做出進度儀表板與部門須知小站。所有單元的跟做都在「同一個工作資料夾、同一包資料」上累積——學員體驗的是一條真實工作流，不是各自獨立的小練習。

**核心技能走跟做、後段炫技走講師示範**（半實作）；財會案例貼在 P1–P3，P4 的進階能力用同情境延伸示範。

**設計原則（回應 G1：避免「設定先行、成果後到」）**：

1. **成果先行**：CH1-1 開場先讓學員看到終點——講師 60–90 秒示範把一包雜亂報帳單丟給已設定好的 Codex，當場產出統一命名 + Excel 彙整。先看到「我也想會」，再回頭學設定。
2. **設定即兌現**：每個設定步驟都綁一個馬上看得到的回報，不讓學員空設定。自訂指令→等下它用你的身份口吻寫信；`AGENTS.md`→等下不用每次重講命名規則；建完專案→立刻請它讀資料夾回報「看到哪些檔、發現哪些問題」，學員第一次看到 Codex 真的看懂自己的工作資料。
3. **每單元都有產出物**：每個單元收尾，學員手上都多一個具體成果（乾淨檔、Excel、Skill、草稿信、儀表板…），用產出物驗證對應的學習成果。

## 單元矩陣

> 配重為暫定，現場可微調。形式：講述 / 示範+跟做 / 實作。

### Part 1：上手與第一個專案（~38min）

- **CH1-1：Codex 是什麼 + 成果先行 demo**（5min・講述+示範）— 與 ChatGPT 的差異：會動手做事、在指定資料夾工作、會產出檔案而非只回答；一句責任邊界：它不知道自己做不到，出錯會一路狂奔。**開場 60–90 秒先讓學員看終點**：講師把一包雜亂報帳單丟給已設定好的 Codex，當場變成統一命名 + Excel 彙整。
  - **→ 產出物**：學員心中有「終點長相」，知道後面的設定是為了什麼成果而做。
- **CH1-2：安裝、個人化與權限**（13min・示範+跟做）— 安裝 → 自訂指令（身份/職務/口吻）→ 記憶開關 → 核准提示與權限模式（自動審核 vs 完整存取）。每步都講「這設定等下兌現在哪」。
  - **案例**：填入「弄一下工作室 行政會計」自訂指令與回覆偏好。
  - **→ 產出物**：一個認得學員身份的 Codex（CH3-2 用它寫信時，口吻即此設定兌現）。
- **CH1-3：工作資料夾、AGENTS.md 與第一次看懂資料**（20min・示範+跟做）— 建「弄一下工作室_6月月結」資料夾 → 寫 `AGENTS.md` 操作守則 → 以「使用現有資料夾」建專案 → **即時兌現**：丟入主線資料包，請 Codex 列出讀到哪些檔、發現哪些問題（空值/重複/異常）→ 對話管理（封存、新對話 vs 專案 + 號）。
  - **案例**：寫含命名規則與「不得補值」硬規則的 `AGENTS.md`；請 Codex 回報它在資料夾看到什麼。
  - **→ 產出物**：可運作的專案 + 一份「Codex 讀檔回報」——學員第一個有感成果（它真的看懂我的檔）。

### Part 2：檔案與表格實作（~52min）

- **CH2-1：整理檔案**（17min・示範+跟做）— 批次重新命名、多檔比對、抽取指定資訊、`@` 引用檔案、並行任務、產出處理日誌。
  - **案例**：把 6 份命名雜亂的報帳單統一成固定格式，並請 Codex 建一份「已處理檔名」日誌。
  - **→ 產出物**：6 份統一命名的報帳單 + 一份處理日誌。
- **CH2-2：彙整清理表格**（20min・示範+跟做）— 多份表格彙整成一張 Excel、合併兩份清單並去重、建立 Checks 工作表檢查空值/重複/總額一致。
  - **案例**：合併 `A.csv`＋`B.csv` 供應商清單去重；把報帳單資料彙整進 Excel；對 `transactions.csv` 跑 Checks，找出空值與重複發票號（體驗「缺資料即停、不得補值」）。
  - **→ 產出物**：一張彙整 Excel + 去重後供應商清單 + 一張標出異常的 Checks 工作表。
- **PRAC2：跟做演練**（15min・實作）— 學員獨立跑一輪。
  - **案例**：給一包「隔天補交」的新雜亂檔，學員自行完成重新命名 + 彙整進既有 Excel + 去重，並更新處理日誌。
  - **→ 產出物**：學員獨力完成的第二批整理成果（驗收前面學的）。

### Part 3：Skill 與外掛擴充（~45min）

- **CH3-1：把流程做成 Skill**（18min・示範+跟做）— `AGENTS.md` vs `Skill` 的取捨（規則寫太多會擠壓 AI 思考空間、拖累品質）、專案層級 vs 全域層級、自動觸發與斜線手動觸發。
  - **案例**：把「月結整理流程（命名＋登記 Excel）」做成**專案 Skill**，下個月一句話觸發；把「公文／簽核表格排版」做成**全域 Skill**跨專案用。
  - **→ 產出物**：一個可一句話觸發的「月結整理」專案 Skill + 一個「表格排版」全域 Skill。
- **CH3-2：外掛擴充能力**（27min・示範）— 外掛 marketplace 與 Skill 的差別；裝 Gmail 外掛草擬郵件存草稿、裝 Presentations 生簡報、用內建生圖 + PDF skill 批次產出。
  - **案例**：用 Gmail 外掛草擬「對帳確認信／催收信」存草稿（強調寄出要人按）；用 Presentations 生一份「6 月月結管理層摘要」PPT；用生圖 + PDF skill 批次做部門識別吊牌（示範圖像+批次能力）。
  - **→ 產出物**：Gmail 草稿信數封 + 一份月結摘要 PPT + 一批識別吊牌 PDF。

### Part 4：能力地圖——自動化、部署與進階展示（~35min・全示範，不列為可重現成果）

> Part 4 定位是「讓學員看見 Codex 還能做到哪」，不是要學員課後自己重現。功能版本/方案/帳號相依，講師示範為主、每項備案。

- **CH4-1：排程與儀表板**（15min・示範）— 排程讓 Codex 每天固定時間自動查信箱（**第一版只產「待更新清單/回覆摘要」，不直接改正式 Excel**；第二版才示範寫入工作副本）；用已安裝/內建的設計能力做 HTML 儀表板整合關鍵數據（**不臨時貼網址裝 skill**）。
  - **案例**：排程每早檢查「廠商對帳回覆」產待更新清單；做一個顯示應收回覆率與月結進度的 HTML 儀表板。
  - **→ 產出物（示範產出）**：一個排程任務 + 一個 HTML 進度儀表板。
- **CH4-2：部署與 Computer Use**（20min・示範）— 規劃模式先擬計畫再動工、`/pet`（寵物通知，Settings > Appearance）、**Codex 內建 Sites plugin 部署**（OpenAI 託管；deployment URL 為 production，但可設 access modes 限制存取）、註解功能改設計、Computer Use 讓 Codex 像真人操作瀏覽器測試。
  - **案例**：用規劃模式做一頁「部門報帳須知」小站 → Codex Sites 部署（先設受限 access、審內容再放寬）→ Computer Use 自動填一筆測試表單驗證。結尾用一句話預告 6h 會計事務班（深度責任治理、月結/對帳/差異分析 SOP）。
  - **→ 產出物（示範產出）**：一頁已部署的「部門報帳須知」小站（受限存取）+ Computer Use 測試通過紀錄。

## 安全／責任（穿插全課，非獨立單元）

深度治理留給 6h 會計班；本課把底線拆成隨堂提醒：
- 權限用最小範圍（官方 profile：`:read-only` 唯讀 / `:workspace` 只允許工作資料夾寫入 / `:danger-full-access` 解除沙盒）；不要一開就最高權限，背景排程更要避免 full access（CH1-2、CH4-1）
- 缺資料即停、不得補值、模糊標待覆核；稅務/科目/入帳/扣抵/資本化一律標待覆核（CH1-3、CH2-2、PRAC2）
- 不說「工具不能寄信」；課程規範＝只做草稿、寄出由人手動按（CH3-2）
- 覆寫/部署/對外輸出前人工確認，原始檔留一份不動；Sites 部署先設受限存取再審內容（CH4）
