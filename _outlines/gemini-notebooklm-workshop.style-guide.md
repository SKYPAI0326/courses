# Gemini × NotebookLM 工作坊 · Style Guide 覆寫

## tone_register

**實作為主、不講概念**。每頁開頭直接給「你今天要產出什麼」，
不繞圈解釋 AI / Gemini / NotebookLM 是什麼（這些只在 CH0 講透一次，後不回頭）。

行文：

- 短句、動詞開頭、第二人稱「你」
- 不要寫「您」、不要寫「我們」（除非講師話術引號內）
- 結論前置，不要鋪陳

## mood_keywords

`上機` `案例` `帶走` `零繳交` `5 部門變體` `指定雙工具` `跨來源整合`

## differentiation

- 與 office-ai-cases（多平台不指派）的核心差異：**本課指定 Gemini + NotebookLM 雙工具**，
  主案例 prompt 直接寫「在 Gemini」「在 NotebookLM」，不再用「在任一主流 AI 工具」這種寫法
- 每案例配 **5 部門變體素材**（財 / 業 / 電 / 倉 / 採），學員從自己部門變體切入
- **跨來源整合作為品牌線索從 CH0 貫穿到 CH2-1 壓軸**

## 案例主角設定（固定使用、貫穿全課 · 覆寫 template）

**公司**：弄一下精選商行（虛構中型電商貿易公司，20–30 人，含財 / 業 / 電 / 倉 / 採 5 部門）

5 部門對應 5 位主角（跨單元固定）：

| 部門 | 主角 | 設定 | 卡住的事 |
|------|------|------|---------|
| 財務部 | **凱倫**（30 歲財務專員） | 月底結帳跑報表 | 報表合併 / 預算執行追蹤 / 應收應付 |
| 業務部 | **小李**（32 歲業務副理） | 週報、業績、客訴整理 | 客戶資料分散 / 不會做客戶分析 |
| 電商部 | **怡君**（29 歲電商營運經理） | 蝦皮 + 自架網店 | 商品分析 / SKU 矩陣 / 活動規劃 |
| 倉儲部 | **阿翔**（35 歲倉儲主管） | 進出貨 / 盤點 / 調撥 | 庫存週轉 / 安全庫存 / 缺料預警 |
| 採購部 | **采蓁**（28 歲採購專員） | 供應商管理 + 報價比較 | 供應商評鑑 / 議價 / 品項追蹤 |

**使用守則**：

- 每個 CH 的「主版示範」用 1 個主角當 POV
- 5 部門變體 demo 段，每個變體用對應部門的主角（不混雜）
- **禁用**：客戶 / 學員 / 老師 / 阿凱 / 雯姊（office-ai-cases 主角）
- **不要創造新主角**

## 受眾稱呼

「你」（禁「您」、禁技術黑話）

## 禁用詞清單（沿用 office-ai-cases，加 1 條）

繼承 office-ai-cases 的禁用詞表。新增：

| 禁用 | 改用 |
|------|------|
| 跨部門協作 | 跨來源整合 |
| 賦能 | 讓 / 幫 |
| 數位轉型 | 用 AI / 用工具 |
| 高效 | 更快 / 更省時 |
| 痛點 | 卡住的事 / 麻煩的事 |
| 場景 | 案例 / 情境（看狀況） |
| 您 | 你 |
| 我們 | 我 / 你（依語境） |

理由：業主明示跨部門占比 ≤15%，本課語言上避免讓學員以為主軸是跨部門協作。

## 平台引用規範（覆寫 office-ai-cases）

**本課主案例（CH1-1 ~ CH2-1）**：

- ✓ **指定 Gemini 或 NotebookLM**（直接寫「在 Gemini」「在 NotebookLM」）
- ✓ 每個案例 frontmatter 明寫主工具
- ✓ 跨工具切換時機要明寫（例：「取證用 NotebookLM、整合用 Gemini」）
- ❌ 不寫「在任一主流 AI 工具」（與 office-ai-cases 區隔）
- ❌ 不寫「ChatGPT / Claude / Copilot」（業主指定雙工具，其他工具不在課程範圍）

**例外**：trouble-shooting 段提到「免費版限制可選付費」時，明示 💰 標記。

## 簡報承諾邊界

**禁用**：「直接生成漂亮 PPT」「下載即用 .pptx」「自動產投影片」

**改用**：「6–8 頁可貼進簡報的內容包」「給 Gemini / Canva / Gamma 當輸入材料」「可人工檢核改寫的初稿」

## 付費功能標記規範

所有需付費才能達成的功能必須加 💰：

- 例：「💰 Gemini Advanced：1M context + Deep Research，免費版本案例不需要」
- 例：「💰 NotebookLM Plus：更多 source + 更高查詢 quota」

不加 💰 的功能必須是 2026-Q2 真實可用的**免費版能力**。

## 案例頁面結構（每案例 CH 必照）

承大綱 CH1-4 的拆分：

1. **場景設定 + 5 部門變體預告**（5 min）
2. **講師示範主版 + 5 部門變體 demo**（15–20 min · 主版 5 min + 每變體 ~3 min × 5）
3. **Prompt 拆解 + 為何這樣寫**（10 min · 含免費版 wrapper）
4. **學員上機演練**（35–40 min · 每人選自己部門變體跑）
5. **驗證 + 變奏 + 跨來源伏筆**（5 min）
