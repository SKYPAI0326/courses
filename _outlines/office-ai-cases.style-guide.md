# office-ai-cases · Style Guide

## tone_register

**實作為主、不講概念**。每一頁的開頭直接給「你今天要產出什麼」，
不繞圈解釋 AI 是什麼、Prompt Engineering 是什麼（這些只在 CH0 講透一次、之後不再回頭）。

行文：
- 短句、動詞開頭、第二人稱「你」
- 不要寫「您」、不要寫「我們」（除非講師話術引號內）
- 結論前置，不要鋪陳

## mood_keywords

`上機` `案例` `帶走` `零繳交` `通用 prompt` `Wrapper`

## differentiation

- 與既有 office-ai：本課**不教「全景圖 / 三工具差異」**（CH0 平台差異 5 min 帶過），
  而是直接進案例。學員結束後不會記得「ChatGPT 強在哪、Gemini 強在哪」——
  學員會記得「**週報這樣寫 prompt**、**異常費用這樣抓**、**簡報這樣組**」。

## 案例主角設定（固定使用、貫穿全課）

兩個跨案例可重複登場的虛構主角：

- **小李**（業務 + 財務跨職）：32 歲，中部某傳產業務副理兼財務代理，
  經常要寫週報、跑業績、整理費用，是 CH1-1 / CH1-2 的主角
- **怡君**（電商 + 行政跨職）：29 歲，蝦皮 + 自架網店電商營運經理，
  負責商品銷售分析 + 主管報告，是 CH2-1 / CH2-2 的主角

**禁用**：客戶 / 個案 / 學員 / 老師（用真實角色名稱）

## 受眾稱呼

「你」（禁「您」、禁技術黑話、禁「賦能」「數位轉型」「高效」「敏捷」「賽道」）

## 禁用詞清單

| 禁用 | 改用 |
|------|------|
| 賦能 | 讓 / 幫 |
| 數位轉型 | 用 AI / 用工具 |
| 高效 | 更快 / 更省時 |
| 敏捷 | 快速調整 |
| 賽道 | 領域 |
| 您 | 你 |
| 我們 | 我 / 你（依語境） |
| 痛點 | 卡住的事 / 麻煩的事 |
| 場景 | 案例 / 情境（看狀況） |
| 解決方案 | 做法 / 辦法 |

## 平台引用規範（重要 — Codex 第二輪建議）

**主課 4 案例**（CH1-1 / CH1-2 / CH2-1 / CH2-2）：
- ❌ **禁止寫「請用 ChatGPT」「Gemini 比較好」**等指派
- ✓ 改寫「在任一主流 AI 工具」「依平台限制加 wrapper」
- 每頁末尾固定附「平台差異備註」3 行：
  - 檔案處理（CSV 在哪些免費版可上傳）
  - 常見限制（5h 訊息 / Office 整合 / Workspace 整合）
  - 跑不出時怎辦（換 wrapper 或換平台）

**NotebookLM bonus**：
- ✓ 可指定 NotebookLM（因為 source-grounded 問答是不可替代功能）
- 標清「這是少數需要指定工具的案例，其他案例都通用」

## 簡報承諾邊界（Codex 第一輪建議）

**禁用**：「直接生成漂亮 PPT」「下載即用 .pptx」「自動產投影片」

**改用**：「6-8 頁可貼進簡報的內容包」「簡報內容素材組裝」「給 Gemini / Canva / Gamma 當輸入材料」

## 付費功能標記規範

所有需付費才能達成的功能必須加 💰 + 括弧註明：
- 例：「💰 Plus US$20/月：可解 5h 10 訊息限制」
- 例：「💰 M365 Copilot 授權：Word/Excel/PPT 內整合需付費」
- 例：「💰 Gemini Advanced：1M context + Deep Research」

不加 💰 標記的功能必須是 2026-05 真實可用的**免費版能力**。

## 案例頁面五段結構（每案例 CH 必照）

1. **場景設定 + 客戶痛點**（5 min · 短）
   - 主角 + 情境一句話
   - 學員會帶走的具體成果

2. **講師示範**（15 min · 多平台對比）
   - 至少 2 個平台跑同一 prompt 看結果差異
   - 不指派「哪個比較好」，讓學員自己評

3. **Prompt 拆解 + 為何這樣寫**（10 min）
   - 通用 prompt 5 大骨架（角色 / 資料 / 任務 / 輸出 / 驗證）對照
   - Wrapper 加在哪裡、為何加

4. **學員上機演練**（40 min · 主軸）
   - 套課程素材跑（必做）
   - 用自己工作場景試跑（鼓勵）
   - 講師走場 + 即時答疑

5. **結果驗證卡 + 變奏 + 風險提醒**（5 min）
   - 結果驗證卡固定 4 欄（總筆數 / 加總金額 / 前三大 / 異常規則）
   - 變奏：對象 / 場景 / 平台 wrapper
   - 風險：本案例最常見的 1-2 個機密外洩 / 幻覺風險

## 結果驗證卡固定格式（每案例必含）

```
┌─────────────────────────────────────┐
│ 結果驗證卡（離場前 30 秒打勾）              │
├─────────────────────────────────────┤
│ ☐ 總筆數：AI 說的 vs 你資料實際筆數一致？      │
│ ☐ 加總金額：AI 加總 vs 試算表 SUM 對得起來？  │
│ ☐ 前三大：AI 列的前三大 vs 你眼睛掃的前三大？   │
│ ☐ 異常規則：AI 用什麼規則判定異常？合理嗎？     │
└─────────────────────────────────────┘
```

## SEO 與 metadata 規範

- 每頁 `<title>` 含「辦公室 AI 工具案例應用」全名（不要簡寫）
- meta description 強調「案例 / 上機演練 / 帶走素材包」
- footer `data-platform-version` 必含 2026-05 各家版本：
  `2026-05（ChatGPT GPT-5.5 free / Gemini 32K free / Copilot Edge free / NotebookLM free / Claude.ai free）`
- footer `data-built-at` 為今日（產製日）
