## 課程定位（Positioning）

兩階段連續課程，把「我也想做一套自己的 LINE 貼圖」從幻想變成可上架的成品。Part 1 用一個下午做出 1 套 8 張靜態貼圖並送審；Part 2 用一整天升級到 24 張動態 + 訊息貼圖。全程免費工具、不討論變現、不教畫畫——靠 AI 把學員想像中的角色固定下來，再走完 LINE 規格與上架流程。

## 受眾畫像（Audience Profile）

- **職業**：上班族、學生、退休族、自由工作者；任何想做專屬貼圖送朋友家人或自用的人
- **技術底子**：零繪圖、零設計軟體經驗；會用手機拍照、能打字、能複製貼上即可
- **現有工具棧**：智慧手機 + 一台筆電（家用、公司、學校都行）；可能用過 ChatGPT 但只用過文字
- **痛點 3 條**：
  1. 想要「我自己的貼圖」但畫不出來，請朋友畫又不好意思開口
  2. 看過很多 AI 圖很美，但**同一個角色**畫 8 張就跑掉，做不出一致的表情組
  3. LINE 上架流程看起來很複雜，怕填錯被退件不知怎麼修

## Brand Brief（品牌調性）

- **tone_register**：親切務實（像鄰居教你做手作，不是設計師講座）
- **mood_keywords**：手作感、專屬、可帶走
- **differentiation**：本課鎖定「全免費 + 自用 + 完整跑上架」三個定位，與弄一下既有「商業用圖片生成 gen-image」（付費 / 商用導向）區隔。不追求視覺最強，追求「真的能做出 8 張一致表情並通過 LINE 審核」

## 學習成果（Outcomes，7 條 · 動詞開頭 · 可驗證）

1. 能用 Gemini Nano Banana 上傳參考圖 + 結構化 Prompt 在同一個對話內產出 **8 張同角色不同表情的圖**，且 8 張都通過自我「角色一致性檢查清單」（5 條判準）
2. 能用 Pixian.AI 或 Erase.bg 把 AI 產出的圖去背成**透明背景 PNG**，邊緣無鋸齒、無殘影；能在 Photopea 內把單張壓到 LINE 規格 370×320 px、≤1MB
3. 能在 LINE Creators Market 完成註冊、填寫個人資料、上傳 8 張主貼圖 + 1 張主圖 + 1 張 Tab 圖、選定上架地區、提交審核
4. 能識別 LINE 審核 5 大退件原因（低品質 / 角色不一致 / 邊緣殘影 / 含他人 IP / 含真實人物），並對自己作品自查；遇到退件能用 SOP 修補後再次提交
5.（進階）能用 Gemini 多輪對話 + Photopea + APNG Assembler 製作出 **1 張合格動態貼圖**（5–10 frame APNG，≤300KB，≤4 秒）；能在當堂示範「眨眼 / 揮手 / 跳起來」三類常見動作其中一種
6.（進階）能在課堂內提交 **24 張套組企劃表**（含日常 ×8 / 節慶 ×4 / 職場 ×4 / 戀愛 ×4 / 加分 ×4 的選題卡）；24 張完整作品集為 PRAC2 課後作業，不在課堂內驗收
7.（進階）能設計訊息貼圖（含可換字）、設定文字疊圖位置與字型；能規劃**同角色 Q 版 / 寫實版 / 季節限定的延伸流程**（不限定本課堂內完成多套，只要產出延伸計畫表）

## 單元矩陣

### Part 1：入門班｜一個下午做出 8 張靜態貼圖並送審（預計 4 小時 / 240 分鐘）

> **時數契約**：課堂目標是讓學員當天帶走「可送審素材包」（8 張處理完的 PNG + 主圖 + Tab 圖 + Creators Market 帳號）。實際送審按鈕學員可選擇當堂按或當晚回家按，不強迫——避免最後 20 分鐘卡在帳號驗證簡訊。

- **CH1-1**：開場 + LINE 貼圖規格 + 免費工具地圖 — 能說出 LINE 三類貼圖（靜態 / 動態 / 訊息）的規格差異；能列出本課用到的 5 個免費工具（Gemini / ChatGPT / Pixian.AI / Photopea / Creators Market）各自的角色（**20 分鐘**）
- **CH1-2**：角色設定 + AI 角色一致性的 3 種手法 — 能寫出一份「角色設定卡」（種族 / 體型 / 配色 / 個性 / 風格參考）；能用 3 種手法（上傳參考圖 / 同對話多輪 / 一次產多張變體）讓 AI 守住同一隻角色（**35 分鐘**）
- **CH1-3**：8 張表情產出實戰 — 能在同一個 Gemini 對話內依序產出 8 張同角色不同表情（笑、哭、生氣、得意、求拜託、無語、晚安、嚇到），並用「角色一致性檢查清單」自查（**75 分鐘**）
- **CH1-4**：透明背景 + 尺寸調整（規格符合） — 能用 Pixian.AI 把 8 張圖去背為 PNG；能在 Photopea 內把單張調整到 370×320 px、檔案 ≤1MB；能匯出主圖（240×240）+ Tab 圖（96×74）（**40 分鐘**）
- **CH1-5**：LINE Creators Market 上架實作 — 能完成註冊、填個資、上傳套組、寫描述、選地區、按下提交審核（或保存為草稿待當晚補完）（**50 分鐘**）
- **CH1-6**：退件 SOP + 第一階段收尾 — 能識別 5 大退件原因；能對自己作品自查；接上 Part 2 預告（**20 分鐘**）
- **PRAC1**：Part 1 實作驗收（課後作業） — 學員提交自己的 8 張貼圖 + 上架截圖（課後 1–7 天等審核結果，過審 / 被退兩種狀況都有對應 SOP）

**Part 1 時間表**：20+35+75+40+50+20 = **240 分鐘**整。無 buffer，每段都收緊 5 分鐘換取「準時下課」。CH1-3、CH1-4 兩段最易爆量，教案要嚴格執行示範比例（示範 50%、學員操作 30%、緩衝 20%）。

### Part 2：進階班｜一整天升級到 24 張動態 + 訊息貼圖（預計 8 小時 / 480 分鐘）

#### 上半場：動態貼圖（4 小時 / 240 分鐘）

- **CH2-1**：動態貼圖原理 + APNG 規格 — 能說出 APNG 與 GIF 的差異；能列出 LINE 動態貼圖規格（≤300KB、5–20 frames、不可超過 4 秒）；能識別「會動的點」與「不該動的點」（**30 分鐘**）
- **CH2-2**：AI 產關鍵幀技巧 — 能用 Gemini 多輪對話從靜態圖延伸出起始 / 中間 / 結尾 3–5 個關鍵幀；能用 prompt 控制「眨眼」「揮手」「跳起來」三類常見動作（**90 分鐘**）
- **CH2-3**：APNG 組裝實戰（Photopea + APNG Assembler） — 能用 Photopea 把多張關鍵幀對齊位置；能用 APNG Assembler 設定每幀 delay 並輸出符合 LINE 規格的 APNG；能用 EZGIF 線上預覽與微調（**90 分鐘**）
- **動態輸出除錯 buffer**：30 分鐘專留給「APNG 壓不到 300KB / 預覽閃爍 / Photopea 對齊跑掉」這類學員會卡住的地方，講師走場一對一支援

#### 下半場：訊息貼圖 + 大型套組 + 進階手法（4 小時 / 240 分鐘）

- **CH2-4**：訊息貼圖（含可換字疊圖） — 能設計訊息貼圖版型（圖 + 文字疊圖）；能設定文字位置 / 字型 / 顏色；能完成上傳前自查清單，並用「模擬退件案例」驗收訊息貼圖品質（實際送審非課堂內可完成）（**55 分鐘**）
- **CH2-5**：大型套組（24 張）設計策略 — 能依「日常 × 8、節慶 × 4、職場 × 4、戀愛 × 4、加分 × 4」框架填出**24 張選題企劃表**；能避免重複又不疲乏（**45 分鐘**）
- **CH2-6**：角色一致性進階 — 能讓同一角色畫出側面、後面、半身、全身；能做雙角色合照（角色 + 寵物 / 角色 + 道具）（**55 分鐘**）
- **CH2-7**：多風格延伸 — 能用同一角色設定卡延伸出 Q 版、寫實版、節慶限定（聖誕、新年、生日）三種風格；能在不破壞角色辨識度前提下做變化（**40 分鐘**）
- **CH2-8**：進階上架實作 + 結訓 — 能完成多語系填寫（中 / 英 / 日）；能規劃「主套組 + 季節限定」的長線延伸計畫；拿到結訓 SOP 卡（**25 分鐘**）
- **檔案壓縮 / 上傳失敗處理 buffer**：20 分鐘專留給結訓前的最後關卡（檔案太大、瀏覽器崩、Creators Market 表單卡關）

**Part 2 時間表**：上半場 30+90+90+30(buffer) = **240 分鐘**；下半場 55+45+55+40+25+20(buffer) = **240 分鐘**。合計 480 分鐘整。明確留 buffer 是因為動態貼圖 ≤300KB 壓縮、訊息貼圖文字位置調整、APNG 預覽異常都是「學員必卡」的點，不留時間整堂課會推遲。

- **PRAC2**：Part 2 實作驗收（課後作業） — 學員提交 1 套 24 張作品集（含至少 5 張動態 + 訊息貼圖）+ 上架截圖；課堂內只驗 1 張合格 APNG + 24 張企劃表，完整 24 張作品為課後完成

## 連續課程設計理由

兩階段拆開的兩個用意：

1. **降低首堂門檻**：4h 一個下午能交付完整作品（8 張靜態 + 上架），學員當天就能跟朋友炫耀「我做了一套自己的貼圖」。如果直接合 12h，第一天 4h 還沒走完上架流程，學員會覺得「卡住了」。
2. **進階自選**：Part 1 學員裡會有 60-70% 滿足於 8 張靜態（自用送朋友夠了）；30-40% 想升級到動態。把 Part 2 設成「主動報名加碼」而不是強制連堂，課程設計師可以針對「真的想學動態」這群人寫進階班，不用顧及「只想做一次」的入門學員。

## G1 強制檢核紀錄（PM 與 PO 對話結論）

| 檢核題 | 答案 | 對 G2 的影響 |
|---|---|---|
| Q1：14 個 CH 中你最沒把握的？ | **規格符合（CH1-4）+ 去背難度 + 工具額度限制**——這是 Part 1 真正的失敗點，AI 繪圖反而比想像簡單 | CH1-4 教案投放最多模板與失敗對照；CH1-1 / CH1-2 / CH1-3 都要埋「省額度」教學點；風險表 5 條補強條全部標 P0 |
| Q2：受眾畫像對不上？ | 與預估一致，沒有偏差 | 受眾段照原案進 G2，無需重寫 |
| Q3：學習成果哪條最浮泛？ | 7 條都 OK | Outcomes 鎖定，不重寫 |
| Q4：品牌調性 OK？ | OK | Style Guide 鎖定，不重寫 |

## Designer 階段（G2）優先級提示

對 course-designer 寫教案時的特別提醒：

### P0（最高優先 · 對應 Q1 主要擔憂）

- **CH1-4 去背 + 規格符合是 Part 1 真正的隱形殺手**（不是 AI 繪圖）：
  - 教案必備「**正解 vs 失敗對照圖庫**」5 組以上：邊緣乾淨 vs 殘影、毛髮通透 vs 切斷、半透明 vs 不透明、白邊 vs 無邊、尺寸正確 vs 規格錯誤
  - **三層救援階梯**：(1) Pixian.AI 一鍵 → (2) Erase.bg 備援 → (3) Photopea 魔術棒 + 橡皮擦手動補救（救援級，學員操作 5–10 分鐘）
  - **「不該救的圖直接重產」決策樹**：去背超過 5 分鐘還救不起來 → 回到 CH1-3 重產同表情，省時不省力
  - 教案要明示「Photopea 在低階筆電會卡頓」應對：開新分頁不要疊太多、處理完一張關掉再開下一張

- **工具額度限制是橫跨 CH1-2 / CH1-3 / CH2-2 的全課程風險**：
  - CH1-1 招生信附「課前一週準備清單」：要求學員**同時註冊 Gemini + ChatGPT 兩家**（一家用光無縫切換）
  - CH1-2 教「**同對話多輪精修**」（同對話的 context 不耗新額度）vs「重開新對話從零講」（每次都是新額度）的對比
  - CH1-3 提供「**示範圖包**」備援：講師事先產好的橘貓 8 張正解圖，學員額度用光時可下載繼續走流程，至少能完成 CH1-4 / CH1-5 的後半流程不被卡死
  - 教案要列**當前免費額度對照表**（platform_version 標日期，半年後重檢）：Gemini 每日 X 張、ChatGPT 每日 Y 張、Bing Image Creator Z boosts 等

### P1（核心教學）

- **CH1-2 與 CH1-3 是 Part 1 上半場核心**：35 + 75 = 110 分鐘佔 Part 1 將近一半時間。教案要給**極多的 Prompt 模板與失敗案例對照**
- **CH1-5 上架實作必須有逐畫面截圖**：LINE Creators Market 介面學員從未見過，光講不夠；至少 30 步逐畫面
- **CH2-2 動態關鍵幀**：示範要用「眨眼」這種 2 frame 的最小可行案例起步，避免直接跳到「跳舞」這類複雜動畫嚇跑學員

### P2（驗收設計）

- **PRAC1 / PRAC2 設計成課後作業**：因為 LINE 審核要 1-7 天，當天驗收不到實際過審結果。提交內容應為「上傳截圖 + 8/24 張作品集 + 自查清單」三件
- **CH2-4 訊息貼圖驗收**：用「模擬退件案例」當場驗收（不等實際送審）
