# Style Guide — simple-ai

> 本檔覆蓋 `_style_guide_template.md`，course-designer / course-lesson-writer 讀本檔，不讀 template。
> 本課定位：手機 + 免費版 AI + 入門體驗（3h / 2 × 90min），與工作坊型課程（ai-workshop-12h）刻意區隔。

---

## 受眾稱呼

- 稱呼學員：**你**（禁用「您」）
- 不使用具名學員角色（無「阿凱」「雯姊」等人設）
- 學員代入身分：**「老闆」**「你」「一人創業者」
- 講師自稱：**「我」**（對話式，不要「筆者」「本課程」）

---

## 案例主角（雙主線）

本課鎖定兩條虛構主線，全課程反覆出現，方便學員代入：

### 主線 A：虛構咖啡館老闆

- **角色設定**：台北單店咖啡館，一人兼老闆+店員+行銷；開店 2 年
- **使用情境**：社群貼文、客戶評論整理、品牌定位、FAQ、菜單升級
- **對應手冊素材**：NotebookLM 品牌庫範例（官網文案 + 菜單 + 客戶評論）
- **主要出現單元**：CH2-1 / CH2-2 / CH2-3 / CH2-4

### 主線 B：虛構手工皂創業者

- **角色設定**：一人創業在家製作手工皂，以 IG + 蝦皮販售；創業半年
- **使用情境**：會議紀錄、報價單拍照、漲價 Email、短影音腳本
- **對應手冊素材**：會議錄音虛構版、報價單照片、漲價通知範例
- **主要出現單元**：CH1-2 / CH1-3 / CH1-4

### 使用原則

- **每單元鎖定一條主線**，不要兩條來回跳
- 需要第三種情境時，用「你的另一位朋友開了○○店」這種臨演角色，不新增固定人設
- 示範輸出的假資料都掛這兩條主線（「拿鐵 120 → 140」「玫瑰手工皂 250」等）

## 術語對照表（硬性，全課程一律）

| 禁用 | 替代 |
|---|---|
| Prompt engineering / 提示詞工程 | 五零件 / 把話講清楚 |
| Few-shot | 給範例 |
| Chain-of-Thought / CoT | 請它一步一步想 |
| Context window | 背景資料 / 給它看的東西 |
| Token | 字數 |
| Multimodal | 會看圖、會聽聲音 |
| RAG / 檢索增強 | NotebookLM 會先讀你的資料 |
| Fine-tuning | （不出現） |
| Agent / 代理 | （不出現） |
| API / Webhook | （不出現，非 non-goal 的延伸提示才帶過）|
| RTCF / CRISPE 等英文框架 | 不出現；統一叫「五零件」 |
| 自動化工作流 | 一條龍 / 從頭做到尾 |

### 五零件命名（本課核心框架，全課程鎖定）

不要換別的說法，每單元引用同一組命名：

1. **身份**（你是誰）
2. **任務**（你要做什麼）
3. **格式**（要什麼樣子）
4. **限制**（不可以做什麼）
5. **退路**（不會時怎麼辦）

## 工具稱呼

| 禁用 | 替代 |
|---|---|
| ChatGPT 免費版 / GPT-4o mini 等版本號 | ChatGPT |
| Gemini 2.5 Flash / Pro 等版本號 | Gemini |
| NotebookLM Plus / Pro | NotebookLM |
| 「AI 工具」統稱 | 點名具體工具（ChatGPT / Gemini / NotebookLM） |

- 不提版本號，避免 2026-Q3 課程重跑時失準
- 介紹工具差異時用「ChatGPT 比較萬用、Gemini 讀長文件、NotebookLM 只讀你的資料」這種人話對比

## 語氣基準

- **對話式**：像跟鄰居老闆聊天，不是上課演講
- **在地感**：可用「老闆」「開店」「下班後抓空檔」這種場景詞
- **直接，不廢話**：每句有意義，不加「首先讓我們來了解一下...」式的墊場句
- **不誇張**：不用「神奇」「驚人」「改變人生」「AI 革命」等行銷詞
- **自曝侷限**：每個技巧說明「什麼時候不適用 / 會踩到什麼雷」
- **工程師式乾幽默可出現**，但不強求

---

## 禁用詞（入門課空氣詞黑名單）

這批詞在入門課通通踩雷，因為學員聽不懂也不在乎：

- 高效、賦能、生態系、數位轉型
- 效率提升 10 倍、AI 革命、AI 會取代人力
- 神級 prompt、終極咒語、絕密技巧、黑科技
- 一鍵生成、秒速完成、自動化到飛起來
- 業界領先、行業標竿、企業級
- 「讓我們一起探索 AI 的無限可能」及同類開場白
- 引用 Google / OpenAI / Anthropic 的公司故事

## 每單元必要元素

### CH 頁面（本課全部 8 個單元都是 CH）

| 元素 | 要求 |
|---|---|
| 開場情境一句話 | 每單元第一段用主線角色開場（「咖啡館老闆週一早上打開 IG 想發貼文，但腦袋一片空白...」） |
| Prompt 範本 | 每單元至少 1 則完整 Prompt，含五零件拆解 + 填空提示 + 示範輸出 |
| 建議工具 tag | 每則 Prompt 標示建議用 ChatGPT / Gemini / NotebookLM |
| 手機操作重點 | 每個示範至少標註一個「手機上怎麼操作」的小提示（例：語音輸入按麥克風鍵）|
| 邀請實作 callout | 每單元結尾一個「現在換你」的小行動，60 秒內可完成 |
| 卡關備案 | 常見失敗（配額用盡、上傳失敗、輸出太長）的 Plan B |

### 補充頁面

- **index.html**：課程總覽 + 兩堂導覽 + Prompt Library 入口
- **module1.html / module2.html**：各自 4 單元導覽
- **prompt-library.html**：55 條 Prompt（EF × 8 + MK × 10 + CS × 6 + RS × 4 + TK × 4 + NB × 3 + WF × 10）索引頁，支援類別篩選、搜尋、一鍵複製

## 與兄弟課程的差異化

本課學員**不是**以下定位，設計時刻意避免混淆：

| 課程 | 差異 |
|---|---|
| ai-workshop-12h（霧藍）| 12h 桌面工作坊，教生產力精通；simple-ai 是 3h 手機入門 |
| gen-ai-140h（職訓）| 140h 職訓班，教全棧 AI 應用；simple-ai 是體驗課 |
| prompt-basic（灰藍綠）| 公部門情境，教交辦單方法；simple-ai 是創業者情境，教五零件 |
| office-ai（陶土橘）| 辦公室職員，教 Copilot 套件；simple-ai 是老闆，教手機免費版 |
| gemini-ai（芥末黃）| Gemini 專課；simple-ai 是三工具比較+選用 |

**核心區別**：只有 simple-ai 鎖定「手機 + 免費版 + 當天帶走明天用得到」的組合。不追求工具精通，追求可持續的 AI 使用習慣。
