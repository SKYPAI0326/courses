Module 01
SECTION 1.3

JSON 基礎與 Prompting 技巧

學會理解 JSON 這種通用資料格式，掌握在 n8n 中讀寫數據的語法，並用 Prompting 讓 AI 穩定輸出結構化資料，讓後續節點能自動處理。

⏱預估時間：45 分鐘
📋7 個步驟
💡不需寫程式背景
1
什麼是 JSON？資料的「名牌標籤」

JSON 是電腦之間傳遞資料的通用格式。想像每件商品都有一張名牌，上面寫著品名、價格、顏色——JSON 就是這種結構化的「名牌」，讓不同的服務（n8n、Make、Google AI Studio）能讀懂彼此傳過來的資料，不會雞同鴨講。

生活類比
一張合約如果只是純文字，電腦不知道哪段是「廠商名稱」、哪段是「金額」。但如果用 JSON 格式，就像幫每個資訊貼上標籤："廠商": "台積電"、"金額": 50000，電腦立刻知道要讀哪個欄位。
💡 n8n 的每個節點之間傳遞的資料都是 JSON 格式。學會讀懂 JSON，就等於學會看懂 n8n 的「數據流」。
2
認識 JSON 物件 {} 與陣列 []

JSON 有兩種基本結構。物件（大括號 {}）用來描述「一筆資料」，裡面是一組組的「欄位名稱 : 值」。陣列（中括號 []）則是「多筆資料的清單」，裡面可以放很多個物件。以下是一個真實的訂單範例：

// 一筆完整的訂單資料（物件 + 陣列） { "客戶姓名": "王小明", "訂單編號": "A20260315-001", "總金額": 3150, "商品列表": [ { "名稱": "無線耳機", "數量": 1, "單價": 2800 }, { "名稱": "充電線", "數量": 1, "單價": 350 } ] }
🗂
n8n 節點輸出視窗中，展開 JSON 結構，看到物件欄位與陣列清單
3
用 Expression 讀取 JSON 欄位

在 n8n 的任何欄位裡，點擊右側齒輪圖示切換到 Expression 模式，就可以用 {{ $json.欄位名稱 }} 的語法讀取資料。若要讀取巢狀欄位（物件裡面的物件），用「.」點下去一層層讀取。

// 以上方的訂單資料為例 {{ $json.客戶姓名 }} → "王小明" {{ $json.訂單編號 }} → "A20260315-001" {{ $json.總金額 }} → 3150
✏️
n8n 節點欄位切換到 Expression 模式，輸入 {{ $json.客戶姓名 }} 後顯示預覽值
🔄 對應 Make 的概念：這就像 Make 中在欄位裡點擊變數、選擇要帶入的資料欄位，只是 n8n 用文字語法表達，更靈活。
4
讀取陣列中的特定項目

陣列的索引從 0 開始，[0] 是第一筆，[1] 是第二筆。讀取陣列裡某個物件的欄位，語法是：先用中括號指定第幾筆，再用點號讀取欄位名稱。

// 讀取商品列表中的第一筆和第二筆 {{ $json.商品列表[0].名稱 }} → "無線耳機" {{ $json.商品列表[0].單價 }} → 2800 {{ $json.商品列表[1].名稱 }} → "充電線" {{ $json.商品列表[1].單價 }} → 350
⚠️ 陣列索引從 0 開始，不是 1。要取第一筆用 [0]，這是初學者最常犯的錯誤。
5
用 Set 節點組合新的 JSON 物件

當你需要把從不同地方讀到的資料重新組合，可以用 n8n 的 Set 節點。點擊「Add field」，設定欄位名稱，值的欄位切換成 Expression 後填入對應語法，就能組出一個全新的 JSON 物件，傳給後續節點使用。

// 在 Set 節點新增以下三個欄位： 欄位名: 客戶簡報 值（Expression）: {{ $json.客戶姓名 }} 欄位名: 消費總額 值（Expression）: {{ $json.總金額 }} 欄位名: 訂單編號 值（Expression）: {{ $json.訂單編號 }}
🔧
n8n Set 節點介面，Add Field 後設定欄位名稱與 Expression 值
6
Prompting 技巧：讓 AI 回傳結構化 JSON

直接問 AI 問題，通常得到的是一段自由格式的文字，n8n 沒辦法自動讀取。解法是在 Prompt 中明確要求 AI 以 JSON 格式回覆，並附上你想要的格式範例——AI 就會照著結構輸出，後續節點才能精準讀取每個欄位。

// Prompt 範本：讓 AI 讀合約並輸出結構化資料 請讀取以下合約文字，並以 JSON 格式回覆， 格式如下，不要包含其他說明文字： { "廠商名稱": "字串", "合約金額": 數字（不含符號）, "簽約日期": "YYYY-MM-DD", "合約類型": "採購單 | 服務合約 | 其他" } 合約文字如下： {{ $json.contract_text }}
🤖
n8n 中連接 Google AI Studio 節點，Prompt 欄位填入上方範本
✅ 關鍵句型：「不要包含其他說明文字」這句很重要——沒有這句，AI 可能會在 JSON 前後加上「以下是回覆：」等文字，導致 JSON Parse 失敗。
7
用 JSON Parse 節點解析 AI 回傳的字串

AI 輸出的 JSON 在 n8n 裡通常是一段「文字字串」，不能直接用 $json.廠商名稱 讀取。需要先接一個 JSON Parse 節點（搜尋「JSON」可找到），選擇 AI 輸出的欄位，它會把字串轉換成 n8n 可以操作的 JSON 物件。

// 設定 JSON Parse 節點 // Mode: Parse JSON // JSON String: 選擇 AI 輸出的文字欄位 // 例如 {{ $json.message.content }} 或 {{ $json.text }} // 解析後，下一個節點就可以直接讀取： {{ $json.廠商名稱 }} → "台積電" {{ $json.合約金額 }} → 50000 {{ $json.簽約日期 }} → "2026-03-15"
🔄
JSON Parse 節點設定，選擇 AI 輸出的字串欄位，執行後下一節點可讀取個別欄位值
⚠️ AI 輸出不穩定時：如果 JSON Parse 偶爾報錯，通常是 AI 在 JSON 外加了說明文字。解法是在 Prompt 前後加上 ```json ... ``` 的格式要求，或加一個 Code 節點先用正規表達式截取 JSON 部分。
完成驗證 CHECKPOINT
你應該能做到這些才算完成

在 n8n 中建立一個測試 Workflow：Webhook 接收資料 → Set 節點讀取並組合欄位 → JSON Parse 節點解析一個模擬的 AI 輸出字串，最後確認所有欄位值都能正確讀取。能做到這三件事，JSON 基礎就算通關。

常見問題
Expression 顯示 undefined 或空白
通常是欄位名稱拼錯（包含大小寫），或路徑層級有誤。在 n8n Expression 編輯器的左側面板會列出前一節點的實際資料結構，可以直接點擊欄位名稱自動帶入，避免手打錯誤。
JSON Parse 節點報錯：Invalid JSON
AI 在 JSON 前後加了說明文字（如「以下是結果：」）。在 Prompt 中加上「只回覆 JSON，不要任何說明文字」，並測試幾次確認 AI 穩定遵守。若問題持續，加一個 Code 節點，用 JSON.parse(input.match(/\{[\s\S]*\}/)[0]) 先截取 JSON 部分。
AI 回傳的數字欄位變成字串
在 Prompt 中明確標示哪些欄位是數字（number），例如 "合約金額": 數字（不含貨幣符號）。若 AI 仍回傳字串，在 Set 節點中用 {{ parseInt($json.合約金額) }} 強制轉換成數字型別。
