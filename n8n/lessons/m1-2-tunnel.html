SECTION 1.2

Cloudflare Tunnel 設定

為本機 n8n 建立一個公開網址，讓 Make、Google Forms 等雲端服務可以傳送 Webhook 到你的電腦，完成雲地協作的關鍵橋接。

⏱預估時間：15–20 分鐘
📋7 個步驟
✅前置：完成 1.1 n8n 安裝
為什麼需要這個步驟

n8n 裝在你的電腦裡，但 Make 或 Google Forms 在網際網路上「看不到」你的電腦——它沒有公開的 IP，也沒有網址。Cloudflare Tunnel 解決這個問題：它幫你的本機 n8n 建立一個公開網址，雲端服務就能透過這個網址把任務傳進來。

1
建立或登入 Cloudflare 帳號

前往 Cloudflare 官網，若尚未有帳號請先免費註冊。完成後，從儀表板左側導覽點擊進入「Zero Trust」（也可能顯示為「One」）。Zero Trust 是管理 Tunnel 的入口，完全免費使用。

☁️
Cloudflare 儀表板，左側找到 Zero Trust 入口
💡 Cloudflare 免費方案已包含完整的 Tunnel 功能，無需升級付費方案。
2
建立新的 Tunnel

進入 Zero Trust 後，從左側選擇「Networks」→「Tunnels」→「Create a tunnel」。選擇「Cloudflared」作為連線方式，為 Tunnel 命名（例如 n8n-local），點擊「Save tunnel」繼續。

🔧
Zero Trust → Networks → Tunnels，點擊「Create a tunnel」，輸入名稱
3
安裝 cloudflared 程式（Mac）

Cloudflare 會依你的作業系統顯示安裝指令。Mac 用戶最簡單的方法是用 Homebrew 安裝：打開終端機（Terminal），輸入以下指令並按 Enter。若尚未安裝 Homebrew，請先至 brew.sh 安裝。

brew install cloudflared
💡 不熟悉終端機？找到 Mac 的「終端機」(Terminal) App，直接貼上指令按 Enter 即可。過程中輸入的內容不會顯示星號，這是正常的。
3b
安裝 cloudflared 程式（Windows）

Windows 用戶從 Cloudflare 提供的頁面直接下載 cloudflared.exe 執行檔，放到容易找到的資料夾（例如 C:\cloudflared\）。接著打開命令提示字元（搜尋 cmd），切換到該資料夾後執行安裝指令。

# 切換到 cloudflared 所在目錄
cd C:\cloudflared
# 執行 Cloudflare 頁面提供的 token 指令
cloudflared.exe service install <你的token>
4
執行 Tunnel 連線指令

回到 Cloudflare 的設定頁面，複製頁面上顯示的完整啟動指令（包含你的 Token），貼到終端機並按 Enter 執行。成功後終端機會持續顯示連線狀態，表示 Tunnel 正在運作中。

⚡️
終端機執行 cloudflared 指令後，顯示「Connection registered」或「Connected」的成功訊息
⚠️ 保持終端機開著：只要 Tunnel 在跑，這個終端機視窗就不能關閉，否則連線會中斷。課程進行時請讓它在背景運作。
5
設定 Public Hostname 指向 n8n

回到 Cloudflare Tunnel 頁面，進入「Public Hostnames」分頁，點擊「Add a public hostname」。設定一個子網域名稱（例如 n8n-webhook），Service 類型選「HTTP」，URL 填入 localhost:5678（n8n Desktop 的預設連接埠），儲存。

🌐
Public Hostname 設定：子網域名稱、Service 選 HTTP、URL 填 localhost:5678
📌 5678 是 n8n Desktop 的預設 port。若你有更改過 n8n 的設定，請填入對應的 port 號。
6
取得並記錄你的 Tunnel 網址

完成 Public Hostname 設定後，Cloudflare 會產生一個公開網址，格式為 你的子網域.你的網域.com（若無自訂網域則為 Cloudflare 提供的預設格式）。複製這個網址並記下來——這是後續讓 Make 找到你 n8n 的核心地址。

🔗
Tunnel 設定完成後，Public Hostname 列表顯示你的公開網址
7
測試：從外部觸發本機 n8n Webhook

開啟 n8n Desktop，進入你的測試 Workflow，點擊 Webhook 節點並選擇「Test step」進入等待狀態。將步驟 6 取得的 Tunnel 網址加上 /webhook-test/ 和你的 Webhook 路徑，貼到瀏覽器新分頁訪問。n8n 若顯示收到資料，代表整個通道已打通。

🎉
n8n Webhook 節點成功接收到來自 Cloudflare Tunnel 網址的資料
✅ 雲地通道打通：你現在可以在 Make.com 的 HTTP 模組中填入這個 Tunnel 網址，讓 Make 的流程觸發你本機 n8n 的 Webhook，雲地協作正式成立。
完成驗證 CHECKPOINT
你應該看到這個畫面才算成功

透過 Cloudflare Tunnel 提供的公開網址，成功觸發本機 n8n 的 Webhook 節點，n8n 顯示已接收到外部資料。此時若在 Make.com 的 HTTP 模組填入此網址並執行，也應能同樣觸發 n8n 接收資料。

常見問題
Tunnel 網址無法連線（瀏覽器顯示錯誤）
優先檢查：(1) 終端機的 cloudflared 是否仍在執行（視窗有無關閉）；(2) Zero Trust 的 Public Hostname 中 localhost 的 port 是否設定為 5678；(3) n8n Desktop 應用程式是否有在運作中。
Cloudflare 頁面找不到 Zero Trust 或 Tunnels 選項
Cloudflare 介面偶有改版。若找不到「Zero Trust」，可嘗試搜尋「Cloudflare One」或在帳號儀表板頂部搜尋「Tunnel」。也可直接至 one.dash.cloudflare.com 進入。
Make.com 的 HTTP 模組打到 Tunnel 網址沒反應
確認在 Make 模組中填入的是完整網址（含 https://），且 n8n 的 Webhook 節點已進入「Waiting for test event」等待狀態。Make 的測試執行預設有 30 秒 timeout，確保在時間內完成觸發。
