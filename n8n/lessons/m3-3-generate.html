Excel 轉 Word：批次生成客製文件

自動化邀請函、結業證書等客製化文件生成——一份 Excel 名單，一鍵產出 100 份完整文件。

生活類比 · ANALOGY
想像你要寄出 100 份邀請函，每份只有姓名和時間不同。以前你需要一份一份修改。現在，你只需要準備一份「範本」（Word）和一份「名單」（Excel），n8n 的 Loop 就像一條生產線，自動將每筆數據填入範本，一鍵產出 100 份完整文件。
STEP BY STEP · 操作步驟
1
準備 Excel 名單（格式規範）
在開始建流程之前，先確認 Excel 檔案的格式正確。第一行必須是欄位標題（Header），後面每一行是一筆數據。以下是本範例使用的欄位結構：
姓名單位日期課程名稱
王小明資訊工程系2026-04-15生成式AI工作坊
李美華企業管理系2026-04-15生成式AI工作坊
張建國外語學院2026-04-15生成式AI工作坊
提示：欄位名稱（第一行）不要有空格或特殊符號，後續在 n8n 中用 {{ $json.姓名 }} 直接引用。
2
Read Excel File 節點：讀取 Excel 名單
新增「Read Excel File」節點。在「File Path」填入 Excel 完整路徑，勾選「First Row is Header」讓 n8n 用第一行作為 JSON 的欄位名稱。執行後，每一行數據會變成一個獨立的 JSON 物件。
// 節點設定 File Path: /Users/你的名稱/Documents/名單.xlsx First Row is Header: ✓ 勾選 Sheet Name: Sheet1（或留空使用第一個工作表） // 輸出範例（每行數據） { "姓名": "王小明", "單位": "資訊工程系", "日期": "2026-04-15", "課程名稱": "生成式AI工作坊" }
提示：若讀取失敗，確認 Excel 格式是 .xlsx（不是 .xls 或 .csv），且檔案沒有被其他程式開啟鎖定。
3
Loop Over Items 節點：迭代每一筆資料
連接「Loop Over Items」節點。這個節點會把 Excel 輸出的多筆資料，一筆一筆送進後面的節點處理——第一次送王小明的資料，第二次送李美華的，依此類推，直到所有人都處理完畢。
💡
Loop Over Items 是批次處理的核心。它不需要特別設定，直接連接到 Read Excel File 輸出即可。n8n 會自動偵測有多少項目並依序迭代。
提示：Loop 內的節點每次只處理「當前那一筆」的資料，用 {{ $json.欄位名 }} 即可取得當次資料。
4
Code 節點：組合文件內容字串
在 Loop 內新增「Code」節點，把當前這筆 Excel 資料組合成完整的文件文字內容。這個文字就是最終文件的主體——你可以根據需求改成邀請函、結業證書或通知書格式。
// Code 節點 - 組合文件內容 const name = $input.item.json['姓名']; const dept = $input.item.json['單位']; const date = $input.item.json['日期']; const course = $input.item.json['課程名稱']; const content = `結業證書 茲證明 ${name} 同學（${dept}） 已完成「${course}」課程， 修習日期：${date}， 特頒此證書，以資證明。 淡江大學 生成式AI工作坊`; const fileName = `${name}_結業證書.txt`; return [{ json: { content, fileName, name, dept, date, course } }];
提示：n8n 原生不支援直接產生 .docx，建議先輸出 .txt 純文字檔，或使用 HTTP Request 呼叫 Google Docs API 填入範本文件。
5
Write Binary File 節點：儲存文件到本機
新增「Write Binary File」節點。先用「Convert to File」節點把文字轉為二進位，再在 Write Binary File 的「File Name」填入動態檔名，將每份文件儲存到指定的輸出資料夾。
// Convert to File 節點 Source Data: JSON Field: content（Code 節點產生的字串） // Write Binary File 節點 File Name: /Users/你的名稱/Documents/結業證書/{{ $json.fileName }} Data: data（Convert to File 的輸出）
提示：確保輸出資料夾（如「結業證書」）已存在，n8n 不會自動建立資料夾。
6
批次驗證：確認產出數量與內容
執行完整流程後，打開輸出資料夾，確認生成的文件數量是否與 Excel 名單的總行數相符。隨機開啟幾份文件，確認每份裡的姓名、單位、日期都正確填入了對應那個人的資料。
提示：在 Loop Over Items 節點的執行記錄中，可以看到總共跑了幾次迭代，與 Excel 行數對照確認無誤。
完成驗證 · VERIFY
輸出資料夾中的檔案數量與 Excel 名單行數完全一致（例如 3 筆資料 → 3 個文件）。每份文件的姓名、單位、日期都正確替換，不存在任何顯示 undefined 或空白的欄位。
常見問題 · TROUBLESHOOT
Excel 讀取後欄位顯示 undefined 或找不到欄位？
請確認 Read Excel File 節點有勾選「First Row is Header」。若欄位名稱含有空格（如「學 生 姓 名」），在 Code 節點中用 $input.item.json['學 生 姓 名'] 方括號語法取值（而非點語法）。
Loop 只執行了一次，只產出一份文件？
確認 Read Excel File 節點輸出的是多個 items（Output 面板右上角顯示「3 items」等）。若輸出只有 1 item（整個 Excel 是一個陣列），需要在 Loop 前加一個「Split Out」節點先把陣列拆開成個別 items。
Write Binary File 失敗，說資料夾不存在？
手動在 Finder（Mac）或檔案總管（Windows）中先建立好輸出資料夾。n8n 的 Write Binary File 節點不會自動建立不存在的資料夾。
