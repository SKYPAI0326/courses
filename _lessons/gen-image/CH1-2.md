## 教學流程（Teaching Flow）

### 看 / Observation（5 min）

#### 開場情境（90 秒，純口述）

上一堂我們學了「參考 → 解構 → 再造」的思維閉環。現在把它套到 AI 生圖的工作流上：

- **參考**：去 Pinterest / IG 找好作品（CH1-1 已教）
- **解構 + 再造**：你猜該怎麼用 AI 做？

最常見的錯誤答案：「下 prompt 給 AI、AI 吐一張圖、我就用這張。」

這是**一次性生圖思維**——跟 CH1-1 的「模仿 → 複製」一模一樣，只是把「複印」的動作交給 AI。結果就是：每次都從零開始、每次品質靠運氣、每次拿到的圖都無法跟上次連動。

本課要教你的是**三段閉環工作流**——讓每次產出都累積可重用的「結構資產」，下次做圖從資產開始，不是從零開始。

---

#### 三段工作流總圖（3 min，口述 + 紙本示意）

老師在白板 / 投影片畫以下流程圖，學員跟畫在筆記本上：

```
┌──────────────────────────────────────────────────────────┐
│                                                          │
│   [起]            [承]             [轉 / 合]              │
│                                                          │
│   brief ─→ L4 ─→  L2 線稿  ─→  新 brief → 新 L4         │
│            完稿    (骨架)        新變體    新系列          │
│                                                          │
│   M2 生圖      M3 線稿反推     M4 線稿反控 + M5 整合      │
│   2h           2h              2.5h + 0.5h                │
│                                                          │
└──────────────────────────────────────────────────────────┘

每一段的產出 = 下一段的素材
每次跑完 = 累積一個可重用的骨架 + 品牌 DNA
```

**重點提問**（給學員 30 秒思考）：

- 為什麼中間要多一個「L2 線稿」環節？直接從完稿 A 到完稿 B 不行嗎？
- 不行。因為完稿包含太多表象（色彩、字體、紋理），直接從完稿 A 做完稿 B，AI 會連表象一起複製過去——做出來的就是「把 A 改成 B」的山寨貨，不是屬於 B 的作品。
- **L2 線稿是「脫掉表象、只留結構」的中轉站**。透過線稿，結構可以乾淨地傳承到下一段，表象則完全替換。

### 學 / Learn（18 min）

#### 三段工作流逐段解說（10 min）

---

**第一段｜起：商業廣告生圖（對應 M2，約 2h）**

從 **brief** 到 **L4 完稿**。

- **輸入**：業種 / 商品 / 客群 / 訴求 / 必要元素 / 風格 preset（brief 六件套，CH2-2 會詳教）
- **工具**：Gemini 或 ChatGPT 生圖
- **產出**：1 張 1:1（或其他比例）的商業橫幅 L4 完稿（日系為教學基準、其他風格 preset 為演練延伸）
- **人的角色**｜**需求判定者**——把模糊想法結構化成 brief 六件套、選骨架、選風格 preset

**本段要學會**：
- 5 種跨風格通用的版面骨架（CH2-1）
- brief 六件套怎麼組合成可丟 AI 的 prompt（CH2-2）
- 多主題 + 多風格 preset 演練（PRAC2）

**這段產出的價值**：一張完稿 + 一個「我跑過這個骨架」的經驗。但**完稿本身不是終點**——它是下一段的素材。

---

**第二段｜承：圖轉線稿（對應 M3，約 2h）**

從 **L4 完稿** 到 **L2 視覺化線稿**。

- **輸入**：上一段的 L4 完稿（或任何現成商業廣告）
- **工具**：同對話裡延續 Gemini 或 ChatGPT（關鍵：**同對話**）
- **產出**：1 張灰階線稿（保留物件輪廓、去掉色彩紋理、文字換佔位符）
- **人的角色**｜**骨架驗收者**——用驗收提問 B 檢查 AI 反推的 L2 是否可繼承（輪廓清晰、色彩移除、佔位符正確、形狀一致、位置不變、風格像 wireframe）

**本段要學會**：
- L2 視覺化線稿反推（CH3-1）
- 從線稿讀出結構語言（CH3-2）
- 線稿反推演練（PRAC3）

**這段產出的價值**：**結構骨架**——一張可重複使用的、跨業種的、跨配色的、跨風格的「版面藍圖」。

---

**第三段｜轉 / 合：線稿反控圖片（對應 M4 + M5，約 3h）**

從 **L2 線稿** 反控，產出 **新的 L4 變體 / 系列**。

有三種應用方式：

1. **同線稿 × 跨產業套用**（CH4-1）——線稿是「空房間」、新業種是「新住戶」
2. **同線稿 × 多變體**（CH4-2）——AB 版面探索，挑最適合的
3. **同線稿 × 品牌 DNA**（CH4-3）——繼承式設計，產出視覺一致的系列

最後整合實戰（CH5-1）：一次跑完 brief → 完稿 A → 線稿 → 完稿 B → 完稿 C，產出「一個品牌 3 張系列」。

- **人的角色**｜**系列把關者**——建立 brand-dna.md、用驗收提問 C 檢查 AI 產出的 3 張系列是否都守在品牌 DNA 內（色票、字族、禁用、骨架、語氣）

**這段產出的價值**：**規模化與一致性**——不是單張作品，而是可複製、可傳承、可建品牌系列的能力。

---

#### 為什麼要分三段？（3 min）

**一次性生圖的問題**：
- 每次都從零開始，結構沒累積
- 修一張只能重跑，不能局部調整
- 跨系列（春夏秋冬 4 張）沒辦法保持風格一致
- 模型升級後，舊 prompt 可能跑不出來

**三段閉環的好處**：
- 結構（L2 線稿）可以保存，重複使用
- 新主題只要換 brief、不用重新發明版面
- 品牌 DNA + 骨架 = 系列化資產
- 模型升級不影響骨架（骨架是結構知識、不綁定模型）

**核心觀念**：你不是在學「用 AI 畫圖」，是在用 AI 當**執行團隊**去實踐 PM 的三位一體協作（CH1-1 已建立：需求判定 / 骨架驗收 / 系列把關）。AI 負責生成，你負責**下需求、驗收、把關**。

---

#### 平台地圖（5 min）

**本課不指定特定平台**——你已經有帳號的、表現穩定的、就是好平台。

但本課工作流對平台有 4 個**能力需求**——不論你用哪家、必須能做到：

| 能力需求 | 為什麼需要 | 哪些平台目前有 |
|---|---|---|
| **同對話延續編輯** | 第二段 L4 → L2 反推需要 AI 看到原圖當 context | Gemini / ChatGPT 都原生支援 |
| **文字渲染**（中 / 日 / 英）| 主標 / 商品名 / CTA 可能要直接產文字 | Gemini / ChatGPT 都基本可用、卡關可改 Canva 後製補字 |
| **結構保留能力** | L2 反推時必須保留物件位置與形狀 | Gemini / ChatGPT 都做得到、表現浮動 |
| **多比例切換**（1:1 / 4:5 / 9:16） | IG / Story / EDM 不同尺寸 | 兩家都支援、但偶爾要在 prompt 強調 |

**2026-04 當下推薦組合**（時間點記著、模型會變）：

- **免費路線**：Gemini Free（Nano Banana）或 ChatGPT Free 任一個——你現有帳號的那個
- **單平台付費**：Gemini Advanced（含 Banana Pro）或 ChatGPT Plus（含 GPT-image-2）任一個約 US$20/月
- **雙平台**：兩個都付——適合高頻商用、能在卡關時隨時切換

**Canva / Firefly 是後期保險**（不論主平台選誰都該用）：

- AI 生圖有版權疑慮（訓練資料來源不透明）
- 商用交付前**建議**用 Canva 替換為合法授權素材、或 Firefly（Adobe 訓練資料版權乾淨）重跑
- 這兩個**不在生圖階段用**、只在「最後交付前的版權保險」階段出現

**模型表現變動頻繁的提醒**（重要）：

> 上面寫「2026-04」是因為**生圖模型每 2-3 個月就會升級或改政策**——配額會變、品質會變、強弱項也會變。**不要綁死任一平台**——遇到卡關就**測試另一個**，課後養成「定期測試各家當前表現」的習慣，永遠選**對你當下任務表現最好的那個**。

**本課明確不教的工具**：
- Midjourney / Stable Diffusion / Flux / Grok / 文心 / 通義萬相 / 可靈
- 理由：**同對話延續編輯**這個本課工作流核心能力不穩定（無法穩定走 L4 → L2 反推），或需要額外技術棧（ComfyUI / API / Discord bot）——不適合本課的 PM 協作節奏
- **不是說它們不好**——只是不適合本課工作流。你若已熟悉 MJ / SD、可繼續用、但本課的 L2 反推階段必須切到 Gemini 或 ChatGPT
- 不是說它們不好，只是**不適合本課的三段閉環工作流**

> **預期差異**：平台模型每季更新、介面每月微調、配額每年調整。本單元列的配額與定價以 **2026-04** 為基準，開課時請學員現場查各平台官網確認最新數字。你看到的定價頁跟同學的可能不一樣（促銷方案、地區差異），這是正常的。

**決策建議（給學員）**：

1. **零預算路線**：Gemini Free 或 ChatGPT Free 任一個 + Canva Free
   - 可走完整個課程、付費版的繼承設計穩定度拿不到
   - 適合學習體驗 + 個人練習、不適合高頻商用

2. **單平台付費路線**（約 US$20/月）：升 Gemini Advanced 或 ChatGPT Plus **任一個**
   - **挑你已經有帳號的那家升級**——介面熟悉省學習成本
   - 沒有任一家是「絕對更好」——當前表現各有強弱、且每季會浮動
   - 可走完整個課程、品質拿到 90%

3. **雙平台付費路線**（約 US$40/月）：Gemini Advanced + ChatGPT Plus
   - 左手 Gemini、右手 ChatGPT，依場景分工
   - 適合專業工作者、課程設計師、自媒體經營者

### 做 / Hands-on（7 min）

#### 活動｜選定路線 + 繪工作流圖

**階段 1｜繪工作流圖（3 min）**

學員拿出筆記本或手機備忘錄，畫出或寫出三段工作流：

- 三個方框：起 / 承 / 轉-合
- 每個方框內寫：輸入 / 工具 / 產出
- 方框之間用箭頭連接

**預期成果**：1 張工作流圖（手繪或文字條列皆可），能在 30 秒內解釋給別人聽。

---

**階段 2｜查平台定價 + 選路線（3 min）**

學員打開手機 / 電腦，查以下 3 個平台的**最新定價**（本課不給死資料）：

- Gemini Advanced 定價頁
- ChatGPT Plus 定價頁
- （可選）Canva Pro 定價頁

查完後回答：

- 我今天要走**哪一條路線**？（零預算 / 單平台付費 / 雙平台付費）
- 我的**決策理由**是什麼？（預算限制 / 工作頻率 / 品質要求）

**預期成果**：1 個路線選擇 + 50 字內的決策理由。

---

**階段 3｜分享（1 min）**

2-3 人一組，快速分享各自的路線選擇。

- 注意：**沒有「正確答案」**，每個人的預算、頻率、品質需求不同
- 重點是你**能說出理由**，不是你選哪條

---

#### 完成標準（self-check）

- [ ] 工作流圖含三段（起 / 承 / 轉-合）+ 箭頭方向正確
- [ ] 每段寫出輸入、工具、產出 3 個欄位
- [ ] 能用 30 秒口述說出三段的意義與產出
- [ ] 選定一條平台路線（零預算 / 單平台付費 / 雙平台付費）
- [ ] 決策理由含「我的預算」「我的使用頻率」「我的品質要求」三項考慮

## 商業情境案例（Case）

**角色**：主線 A 老闆（弄一下咖啡工作室單店老闆）

**公司**：弄一下咖啡工作室

**任務**：老闆上完 CH1-1 後，知道「參考 → 解構 → 再造」的閉環。現在他想知道：這個閉環在 AI 工具上**實際怎麼跑**？要用什麼工具？要花多少錢？

**本單元要他學會**：

- 把抽象的「參考 → 解構 → 再造」對應到具體的「起 → 承 → 轉-合」三段工作流
- 認識主打工具（Gemini / ChatGPT / Canva / Firefly）各自的定位
- 依自己的預算（單店老闆、收入有限）選定路線——很可能是零預算或單平台付費，而不是雙平台付費

**情境擴展**（本單元可用）：

- 老闆的朋友 A 開線上課程（收入穩定、IG 發圖頻率高）→ 會選雙平台付費路線
- 老闆的朋友 B 是剛畢業的設計學生（預算極緊、練習量大）→ 會選零預算路線
- 老闆本人（單店、月收入有限、IG 發圖頻率中）→ 選單平台付費路線（Gemini Advanced）

**本單元的 POV**：

主要走第二人稱「你」，讓學員把自己代入「一個要做平台選擇的工作者」。情境擴展部分用第三人稱介紹 3 種不同使用者畫像，供學員對照自己屬於哪類。

## 動手練習題（Hands-on Exercise）

**題目**：

請在 7 分鐘內完成以下 3 件事：

1. **繪工作流圖**：畫出或寫出三段工作流（起 / 承 / 轉-合），每段標註輸入 / 工具 / 產出
2. **查定價**：查 Gemini Advanced、ChatGPT Plus、Canva Pro 三家的最新定價頁
3. **選路線**：從「零預算 / 單平台付費 / 雙平台付費」選一條，寫 50 字決策理由

**預期成果**：

- 1 張工作流圖（手繪 / 文字條列皆可）
- 1 個平台路線選擇（明確選一條）
- 1 段 50 字內的決策理由（含預算 / 頻率 / 品質三項考慮）

**完成標準**（self-check）：

- [ ] 工作流圖三段齊全、箭頭方向正確
- [ ] 每段寫出輸入 / 工具 / 產出
- [ ] 能用 30 秒口述說明三段各自產出什麼
- [ ] 路線選擇明確（不是「再想想」「不確定」）
- [ ] 決策理由提到至少 2 項具體考慮（例：「我每週要發 3 篇 IG，收入穩定，所以選單平台付費」）

**預期差異提醒**：你選的路線跟隔壁同學**不會一樣**——這是正常的。預算、頻率、品質需求每個人都不同，沒有「標準答案」。唯一要避免的是「盲選」——請基於自己的真實情境做決策。

## 常見錯誤 3 條（Common Pitfalls）

### 錯誤 1｜一上來就用付費版（消費者思維、非 PM 思維）

**錯誤現象**：

學員上完本單元後，當天就刷卡升級 Gemini Advanced + ChatGPT Plus，之後整週在跑課程練習時，每次都用 Banana Pro 最高品質模式。一個月後信用卡帳單來了，發現花了 US$60，但產出的圖**跟用免費版差不多**——因為 prompt 寫得還不夠好，模型再強也發揮不出來。

**原因**：

這是典型的**消費者思維**（「想買最好的工具」），不是 **PM 思維**（「選合適的工具解決當前問題」）。

- 消費者思維：工具越貴越好、功能越多越好
- PM 思維：**先釐清需求 → 再選工具**。需求（練習 / 學習階段）對應的工具應該是**免費版**（配額夠用、差異不大），而不是付費版

付費版的優勢在「最終品質」與「穩定度」，不在「剛學時的練習」。剛學時學員的 prompt 結構還不成熟，用付費版只是浪費配額——免費版已經能展示所有教學重點。

**解法**：

用 **PM 需求判定思維** 做平台決策——**分階段升級**：

1. **第 1-2 週**（CH1 + CH2 + CH3）：需求 = 學工作流、學骨架、學 L2 反推。對應工具 = **全程免費版**（需求簡單、免費版夠用）
2. **第 3 週**（CH4 繼承設計）：需求 = 開始要求品質穩定。對應工具 = **升級一個平台**（建議 Gemini Advanced）
3. **第 4 週以後**（正式商用）：需求 = 頻繁出物料 + 高品質。對應工具 = 考慮雙平台付費

**原則**：**工具升級應該被「具體需求」推動，不是被「想用好的」推動**——這是 PM 與消費者的根本區別。

---

### 錯誤 2｜堅持用 Midjourney / Stable Diffusion / Flux 走本課工作流

**錯誤現象**：

某些學員原本是 Midjourney 或 SD 重度使用者，聽完本單元後堅持「我已經習慣 MJ 了、為什麼不能用 MJ 走完整個課程？」於是用 MJ 跑第一段生圖，到第二段 L4 → L2 反推時卡住——MJ 沒有「同對話延續編輯」的原生功能，反推線稿要另外用 --cref 或 --sref 參數，且結構保留度不穩定。

**原因**：

本課三段工作流的**關鍵能力是「同對話延續編輯」**——第二段必須能吃第一段的完稿作為 context，直接轉成線稿。Gemini / ChatGPT 是原生支援這個能力的少數平台；MJ / SD / Flux 雖然單張品質可能更高，但工作流斷裂，不適合本課的閉環設計。

**解法**：

- 若你已經是 MJ 重度使用者，**第一段生圖可以用 MJ**（產出 L4 完稿）
- 但**第二段 L4 → L2 反推必須切換到 Gemini 或 ChatGPT**，讓新對話吃 MJ 產出的圖當參考
- 第三段反控也一樣，在 Gemini / ChatGPT 走
- **不要強求用單一工具走完三段**——工具是手段，工作流閉環才是目的

---

### 錯誤 3｜不做平台分工（全部塞給一個工具）

**錯誤現象**：

學員選了零預算路線後，整個課程全部用 Gemini Free 跑——包括第一段的高複雜度 brief prompt、第二段的 L4 → L2 反推、第三段的品牌 DNA 繼承。結果發現：
- 複雜 prompt 時 Gemini 遵循度不夠高（該強項是 ChatGPT）
- 多比例切換時 Gemini 頻頻回原尺寸（ChatGPT 做這個比較穩）
- 一整天下來 Gemini Free 配額用爆

**原因**：

每個平台都有強項與弱項、且強弱項**每季都在變**。「全部塞給一個工具」會讓你在工具當前弱項的任務上挫折——若你有兩家帳號、就該交叉用。

**解法**：

**基本分工建議**（**任務導向、不指定品牌**）：

- **第一段生圖**：簡單 brief 用兩家任一、複雜 brief（多必要元素 / 多比例切換）用 prompt 遵循度當下表現較好的那家
- **第二段 L4 → L2**：哪家**同對話延續編輯**最穩、就用哪家
- **第三段品牌繼承**：哪家**identity 一致性**當前表現好、就用哪家
- **最後交付**：Canva 替換合法素材（不用 AI 生圖）

**心法**：**「能力導向、不偏好品牌、卡關就換另一家」**——當前表現好的那家、不論是 Gemini 還 ChatGPT、就是你的「主力」。三個月後可能換。

**測試方法**：每季花 30 分鐘把同一份 brief 各跑 2 張、自己對比兩家當前在你業種上的表現——這比看別人推薦更可靠。

## 檢核題 2 條（Quiz）

### Q1（概念驗證｜簡答）

請**用自己的話**寫或畫出「生圖 → 轉線稿 → 線稿反控」三段工作流，並說明：**為什麼要多中間「線稿」這段？一步到位從完稿 A 直接做完稿 B 不行嗎？**

**預期答案要點**（至少命中 3 點算通過）：

1. 三段工作流圖：起（brief → L4 完稿）→ 承（L4 → L2 線稿）→ 轉-合（L2 + 新 brief → 新 L4）
2. 中間要有線稿是因為**完稿包含太多表象**（色彩、字體、紋理），直接從完稿做完稿會把表象複製過去
3. **線稿脫掉表象、只留結構**，讓結構可以乾淨傳承到下一段
4. 線稿是**可重複使用的資產**（同一張線稿可套多個新業種 / 多個變體 / 多個品牌主題）
5. 一步到位等於「模仿 → 複製」，三段閉環等於「參考 → 解構 → 再造」（加分項：扣回 CH1-1 閉環思維）
6. 線稿讓**結構知識可保存、可累積、經驗會複利**（加分項）

**不及格的答案**：
- 只畫三個方框沒寫理由
- 「因為老師說要這樣」
- 「這樣比較好看」（錯誤：線稿跟好不好看無關，是關於結構傳承）

### Q2（應用驗證｜PM 決策情境題）

**情境**：

你作為老闆的 PM（或顧問），主線 A 老闆想報名本課但有以下限制：

- 預算有限（月可支配約 NT$1,500 用於工具訂閱）
- 時間有限（每週只有週日晚上 2 小時可以練習）
- 品質要求中等（不是專業設計交付，只是 IG 自用）
- 目前已有 Canva Free 帳號、Gmail / Google 帳號

請你用 **PM 的需求判定思維**（而非「工具越貴越好」或「免費就夠」的消費者思維）幫老闆做平台配置。具體說明：

1. 選**哪條路線**（零預算 / 單平台付費 / 雙平台付費）
2. 哪個平台**升級** / 哪個**用免費**？
3. 為什麼這樣配？（必須對應老闆的 3 個限制：預算 / 時間 / 品質）

**預期答案要點**：

**推薦方案**：

- **路線**：**單平台付費**（約 US$20 / 約 TWD 620，在 NT$1,500 預算內）
- **配置**：
  - 升級：Gemini Advanced（含 Banana Pro）
  - 免費：ChatGPT Free、Canva Free
- **理由**：
  1. 挑老闆已經有帳號的那家（Gemini 或 ChatGPT 任一個）升級——介面熟悉省學習成本
  2. 另一家的免費版仍當備援（每週 2 小時練習量、免費配額夠）
  3. Canva Free 足以應付交付（IG 自用不需要 Pro）
  4. 預算剩餘 NT$880 作為**緩衝**（應付未來突發的高頻需求、或省下來作為其他訂閱）
  5. 品質要求中等 → Banana Pro 可以拿到，不用升到雙平台付費
  6. 時間有限 → 不要讓自己「因為付費了就硬要用」，保持鬆弛感

**加分答案**：

- 建議老闆前 2 週先走零預算路線練基本功（CH1-CH3），第 3 週再升級（CH4 繼承設計時才需要品質提升）
- 提醒老闆 Gemini Advanced 月費通常有年繳折扣（可再省 20%）
- 提醒老闆試用期（14-30 天免費）可以先試用再決定

**不及格的答案**：

- 「選雙平台付費，反正多花錢品質一定更好」（沒對應老闆的預算與頻率限制）
- 「選零預算路線，免費就夠了」（沒考慮老闆的品質要求是「中等」而非「只求體驗」）
- 「都可以啊」（沒做出具體建議）
