## 教學流程（Teaching Flow）

### 需求 / Need（5 min）

#### 開場：把整套課程串起來看一遍（90 秒，純口述）

恭喜你撐到最後一堂。

前面幾堂你學了一堆東西——PM 三位一體、5 骨架、brief 六件套、L2 反推、跨產業套用、多變體、品牌 DNA、4 種驗收提問⋯⋯。

聽起來很多、其實只是**一個工作流的不同段**：

> **需求 → 執行 → 驗收 → 迭代**

每次做新廣告、就是這 4 步、循環一輪。

本單元 30 分鐘做兩件事：

1. **看一張總圖**——讓你把整套課程學過的東西**串起來看一遍**
2. **挑一個案例跑跑看**——本單元準備了 5 個現成整合案例（餐飲 / 保健 / 教育 / 旅遊 / App），你挑 1 個最有感的、複製貼上跑——10 分鐘內可以完成

跑得好不好不重要、跑通流程就行。**這不是畢業考、是收尾**。

---

#### 整合工作流總圖（3 min）

老師在白板畫出（或學員看講義）：

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   需求                  執行                  驗收            迭代│
│  (PM 你)              (AI 它)               (PM 你)         (PM 你)│
│                                                                 │
│   brief 六件套  ──→   L4 完稿 A    ──→  驗收 A 8 項清單  ──→  保留X+修Y│
│        ↓                                                        │
│   同 brief     ──→   L2 線稿       ──→  驗收 B 6 項清單  ──→  迭代或重跑│
│        ↓                                                        │
│   L2 + 新主題  ──→   新 L4 B / C   ──→  驗收 A           ──→  迭代│
│        ↓                                                        │
│   L2 + 全套DNA ──→   3 張系列      ──→  驗收 C 5 項清單  ──→  迭代│
│                                                                 │
│   你的角色                AI 的工作           你的判斷           │
│   ─────────              ─────────           ─────────          │
│   需求判定者              執行設計師           骨架驗收者         │
│   系列把關者                                  系列把關者         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

  M2 起｜生圖    →    M3 承｜轉線稿    →    M4 轉/合｜反控
  CH2-1/2/PRAC2     CH3-1/2/PRAC3        CH4-1/2/3/PRAC4
```

**口訣**（給學員記）：

> **「需求我寫、執行 AI、驗收我看、迭代我下」**

每個你經手的廣告都跑這 4 步、累積成資產。

---

#### 一個提醒（1.5 min）

**今天不做新東西**——這 30 分鐘是**整理 + 跑一次**。

- 你**不需要**重填 DNA（用 PRAC4 那份就好）
- 你**不需要**重跑 L2 骨架（用 PRAC3 那張就好）
- 你**只需要**挑 1 個整合案例（或用自家素材）、跑 1-3 張新 L4

跑得卡也沒關係——卡住就**放下、看下個案例的 prompt**。本單元的目標是「**整合工作流的手感**」、不是「**完美交付物**」。

---

### 設計 / Design（10 min）

> **5 個整合演練案例**——每個都是「完整 brand-dna + L2 骨架描述 + 3 主題 brief + 3 份品牌繼承 prompt」可直接複製貼到 AI 跑。
>
> 從中挑 1 個最有感的（跟你業務最相關、或最想體驗的風格）——也可全部不用、改用自家素材。

---

#### 案例 A｜咖啡店季節新品（餐飲、骨架 4、日系）

**情境**：弄一下咖啡工作室年度行銷、3 季新品

**brand-dna**（可複製）：

```
brand: 弄一下咖啡工作室
風格 preset: 日系職人
色票:
  - primary: 鼠尾草綠 #5a7a5a
  - accent:  奶茶棕 #8a6a4a
  - bg:      奶油白 #f5efe3
禁用色: 純黑、漸層、霓虹色
字族:
  - headline: Shippori Mincho 700
  - body:     Noto Sans JP 400
語氣: 手作、職人、季節感、療癒
禁用語氣: 促銷轟炸、網路梗、CP 值口號
禁用視覺: 寫實人像、玻璃態、Comic Sans
主力骨架: 骨架 4 商品中心 × 道具環繞
```

**L2 骨架描述**：

- 商品（中央偏右、玻璃杯狀）
- 主標 2 行（左上）+ accent 手寫字
- 道具（左下原料盤 / 底部湯匙 / 右上葉片）
- 價格（左下）+ CTA（左下膠囊）
- 徽章（右下圓形）

**3 主題（春 / 夏 / 秋）**：

| 主題 | 商品 | 季節徽章 | 道具暗示 |
|---|---|---|---|
| 春 | 抹茶提拉米蘇拿鐵 ¥680 | 春季限定 | 抹茶粉盤 + 木匙 + 綠葉 |
| 夏 | 柚子氣泡美式 ¥580 | 夏季限定 6/1-7/31 | 柚子切片 + 冰塊 + 薄荷葉 |
| 秋 | 焙茶拿鐵 ¥620 | 秋季限定 | 乾茶葉 + 木匙 + 楓葉 |

**品牌繼承 prompt 模板**（每張只換 5 行變量）：

```
吃下以下品牌 DNA：
[貼上述 brand-dna 全部]

結構沿用：骨架 4 商品中心 × 道具環繞

本次主題：
- 商品：{春 抹茶提拉米蘇拿鐵 / 夏 柚子氣泡美式 / 秋 焙茶拿鐵}
- 季節徽章：{對應季節}
- 價格：{對應價格}
- 道具：{對應道具描述}
- 主標：{季節 + 商品名}

請產出 1:1 L4 完稿。
產完自檢：配色 / 字族 / 禁用清單是否全遵守？
```

### 實作 / Implementation（10 min）

#### 跑 1-3 張系列

挑好案例後、依下面的步驟跑。**不必跑完 3 張、跑 1 張就有完整體驗**：

**Step 1｜複製 brand-dna**（30 秒）

從你選的案例段直接複製 brand-dna 那塊文字、貼進 AI 對話。

**Step 2｜複製品牌繼承 prompt + 換主題**（1 分鐘）

從案例段複製 prompt 模板、把 5 行變量換成「3 主題其中 1 個」（例：案例 A 的春季抹茶拿鐵）。

**Step 3｜跑 AI**（1-2 分鐘）

等 AI 產出。

**Step 4｜貼驗收 A 自檢**（1 分鐘）

```
請對剛才的 L4 對照 8 項結構清單自檢：
1. 商品位置 2. 主標位置 3. 徽章位置 4. CTA 位置
5. 配色（DNA 3 色） 6. 字族 7. 視覺動線 8. 多餘元素
通過 __/8 項。
```

**Step 5（可選）｜換主題、再跑一張**

把同 prompt 的主題從春 → 夏（或 A → B）、再跑一張。

**Step 6（可選）｜跨張驗收 C**

3 張都跑完、貼驗收 C 5 項清單看跨張一致性。

---

#### 用自家素材跑（替代方案）

如果你 PRAC4 已經跑出 brand-dna + 3 張系列、本實作段可以**直接拿那組來再跑 1 張新主題**——相當於把 PRAC4 的工作延伸 1 張、不用重做。

> **預期差異**：你跑出的圖跟同學完全不一樣、跑案例 A 的 3 個學員 3 張結果都不同——這是 AI 特性。重點是**你體驗到完整工作流的順暢感**、不是「跑出最美的圖」。

---

#### 卡住怎麼辦（給沒跑通的）

| 卡點 | 怎麼辦 |
|---|---|
| AI 跑出來配色不對 | 看 brand-dna 裡的「禁用色」是否有貼進 prompt——沒貼就補 |
| 結構跑掉、變成別的骨架 | 在 prompt 加一句「結構必須與 L2 骨架描述完全一致、不可重新構圖」|
| 跑了 3 次還是不行 | **直接放下、看下一個案例的 prompt**——本單元目標是體驗、不是達標 |
| AI 配額爆 | 切換另一個平台（Gemini ↔ ChatGPT）、或紙上想像「如果這個 prompt 跑出來、應該長什麼樣」 |

### 驗證 / Testing（5 min）

#### 你帶走的東西（2 min）

打開**你的個人品牌資產庫**——你今天學完帶走的：

```
個人品牌資產庫 v1
│
├─ brand-dna.md（PRAC4 那份、或今天案例其中一個）
├─ L2 骨架（PRAC3 那張、或今天案例的描述）
├─ 系列完稿（PRAC4 + 今天跑的 = 3-6 張）
│
├─ prompt 模板（5 類）
│   ├─ 生圖 prompt（brief 六件套）
│   ├─ L2 反推 prompt
│   ├─ 跨產業套用 prompt
│   ├─ 品牌繼承 prompt
│   └─ 驗收提問 A / B / C / D
│
└─ 思考工具
    ├─ PM 三件事（講清楚 / 看做得行不行 / 確認系列像同品牌）
    ├─ 4 步閉環（講 → 等 → 看 → 改）
    ├─ 8 用途版型速查表（A 主視覺 / B 橫幅 / ... / H 生活）
    ├─ 7 風格 preset 系統（簡約 / 活力 / 科技 / 自然 / 復古 / 奢華 / 藝術）
    └─ 4 種常見失敗症狀 + 修正句
```

**你不會做的事**（這也是好事）：

- 你不會用 Figma / PS 畫圖（**那是設計師的事**）
- 你不會調 ControlNet / seed / CFG（**那是技術工作流課的事**）
- 你不會憑空判定美感（**你用清單**）

#### 課後一週使用計畫（2 min）

寫下你**本週**會做的 1 件事（很小、很具體）：

| 時間點 | 你會做什麼？ |
|---|---|
| 今天回家 | __________________________________________ |
| 明天 | __________________________________________ |
| 這週末 | __________________________________________ |

**範例**：

- 今天回家：把 brand-dna.md 存進 Notion（5 分鐘）

#### 結尾（1 min，老師帶）

> 整套課程下來、你帶走的不是「5 骨架」「11 preset」「4 驗收」這些名詞——
>
> 帶走的是**「不變成設計師、用 PM 思維跟 AI 協作、把每張廣告變成可重複用的品牌資產」**的工作方式。

>
> 你會說：
>
> **「吃下這份 DNA + 骨架、為本週新品『冬季薑汁可可』產出 L4、自檢 8 項清單。」**
>
> 這就是你課後的日常工作語言。
>
> 課程結束、祝你接下來每張廣告都是品牌資產、而不是一次性物料。

## 進階挑戰｜多頁品牌識別 Lookbook（選做、不是必修）

> **適用對象**：學完整套課程、想再進一階的學員（**不做也能畢業**）。
>
> **產出規格**：1 張 4:3 圖、內含 6-8 格網站頁面拼貼（首頁 + 故事 + 產品 + 系列 + 聯絡 + 形象）—— **模擬完整品牌官網視覺**、適合放 portfolio 或客戶提案。

### 為什麼這是「進階」？

前面 4 步閉環 + 8 版型 + 品牌系列、學員產出的是**單張廣告**或**3 張系列**。
進階挑戰要把同一個品牌做成**完整網站視覺**——一次跑 6-8 頁（首頁 / 產品 / 故事 / 聯絡 / ...）、視覺要**互相呼應、像同一個牌子**。

這是 PM 在實務最常遇到的需求——「**幫我提案這個品牌的官網長什麼樣**」。

### 流程（30 min、選做）

#### Step 1｜選風格（3 min）

從 **7 風格 preset 系統**選 1 個（見 `8-prompt-examples.md` § 7 風格 preset 系統 或 `7-brand-style-system.md`）：

| 風格 | 適合業種 | 配色 |
|---|---|---|
| 簡約 minimal | 家具、香氛、生活選品、美容 | 米白 + 灰 + 淺木 |
| 活力 playful | 氣泡飲、零食、潮牌、年輕 App | 黃 + 紫 + 綠 |
| 科技 futuristic | 3C、SaaS、雲端 | 深灰 + 銀 + 螢光綠 |
| 自然 organic ⭐ | 保養、有機食、瑜伽、永續 | 米色 + 草綠 + 淺木 |
| 復古 retro | 餐酒館、潮牌、精釀啤酒 | 深棕 + 米黃 + 磚紅 |
| 奢華 luxury | 精錶、珠寶、頂級香水 | 黑 + 金 + 深酒紅 |
| 藝術 conceptual | 展覽、創意工作室、時尚 | 黑 + 螢光綠 + 純白 |

⭐ 本課程主推：自然 organic（與你 PRAC4 的 brand-dna 屬同色家族）

#### Step 2｜meta-prompting 自動產 prompt（10 min）

把這段貼進 ChatGPT / Gemini（**不是直接給生圖 AI**）：

```
我想為一家名為 [品牌名稱] 的 [產品 / 服務] 品牌、設計一份 [選定的風格] 風格的多頁網站品牌識別。
請依以下需求、產出最終生圖指令、讓我貼給生圖 AI。
請直接生成中文文字指令、不用解釋。

整體品牌應呈現出 [核心關鍵詞 3 個]、結合 [業種特性]、定位 [高端 / 親民 / 實驗性]。

視覺需呈現 [對比特質 A] 與 [對比特質 B]（如：趣味性 vs 專業感）。

採用 [配色策略] 視覺識別、搭配 [視覺元素：幾何 / 流體 / 網格] 與 [字體風格] 與 [節奏：模組化 / 動畫感]。

影像風格 [寫實產品 / 3D 渲染 / 拼貼 / 自然光]。

頁面包含：首頁、產品頁、品牌故事頁、產品系列頁、聯絡頁、形象照頁。

輸出比例 4:3、品牌 Lookbook 方式呈現、採用非對稱、解構式、模組錯位排列。
```

LLM 會幫你把這段需求**自動轉成完整生圖 prompt**——拿產出的 prompt 貼給 Nano Banana / GPT Image。

#### Step 3｜生圖 + 驗收（15 min）

把 LLM 產出的 prompt 貼到 Nano Banana / GPT Image / Imagen → 跑 4:3 Lookbook → 用以下 4 件事自檢：

| 自檢項 | 看什麼 |
|---|---|
| 風格一致性 | 6 格的配色、字體是否屬於同一風格 |
| 結構合理性 | 拼貼是否模組錯位、不平均、有節奏感 |
| 業種對位 | 看起來像「該業種」的品牌嗎？ |
| 整體完成度 | 像真的網站、還是像粗稿？ |

**收成標準**：4 項都對 → 收進 portfolio / 提案；1-2 項不對 → 跟 LLM 說「請重新生成指令、加強 [某項]」、再跑一次。

#### Step 4｜2 個迭代（選做、5 min）

進階變體：

- **同品牌 × 換 1 個風格**（如：你跑了「自然」、再跑「奢華」、看品牌升級的視覺感）
- **同風格 × 換 1 個品牌**（用同樣風格的 Lookbook 結構、套到不同業種、體驗風格的通用性）

### 你會帶走什麼？

| 收成物 | 用途 |
|---|---|
| 4:3 多頁 Lookbook（PNG / JPG） | 個人 portfolio / 客戶提案 / 自家品牌參考稿 |
| Meta-prompt 模板（你的版本） | 下次接案、再用 1 次就會 |
| 風格 preset 經驗 | 7 風格中你跑過哪 1-2 個、之後有手感 |

### 為什麼是選做、不是必修？

- **時間成本高**：1 張 Lookbook 約 30 min、新人需 60 min
- **學習曲線陡**：要熟悉 meta-prompting 思維、初學者直接寫六件套就好
- **CP 值看用途**：日常單張廣告、不需要 Lookbook；客戶提案 / portfolio、Lookbook 大幅加分

**建議優先順序**：先跑 1 張 8 版型單品 → 再跑品牌 3 張系列 → **進階挑戰再做 Lookbook**。

> 📚 **完整教學資源**：`_sources/gen-image/7-brand-style-system.md`（含 7 風格 × 8 版型對應 + 6 格 Lookbook prompt 模板）

- [ ] 挑了 1 個案例（或用自家素材）

## 常見疑惑 3 條（Common Questions）

> ⚠️ 用「常見疑惑」替代「常見錯誤」——這堂課收束、不打分、不嚇人。

### 疑惑 1｜「我課後真的能用得上嗎？」

**疑惑**：

跑完 5 個案例感覺都好順——但回到自家業務時、面對「真實老闆 / 真實客戶」會不會 prompt 寫不出來、AI 跑出來的還是一團糟？

**回應**：

- 真實業務的 prompt 跟案例庫的差異**只有六件套變量**——業種 / 商品 / 客群 / 訴求 / 必要元素 / 風格 preset
- 你的 brand-dna.md 是**一次寫、半年用**——不是每張廣告重寫
- 第 1 個月會不順——這是**正常學習曲線**、不是你不會
- **建議**：第 1 個月只用 1 個骨架 + 1 個 DNA、跑 5-10 張、累積手感再擴張

---

### 疑惑 2｜「AI 跑出來不夠美怎麼辦？」

**疑惑**：

跑出來結構對了、但**美感不如預期**——某些細節（光影 / 排版精緻度）跟付費設計師作品差一截。

**回應**：

- AI 的天花板就是**「中等偏上的設計平均值」**——不會超越專業設計師、但對中小品牌已夠用
- 你的角色是 **PM、不是美術總監**——不是讓你判斷「美不美」、是判斷「能不能用」
- 真要更美：**最後交付前用 Canva 補素材 / Firefly 重製**——AI 產出當參考稿、不是最終稿
- **長期心態**：AI 跑出 80% 美感 + 你的 100% 結構控制 = 中小品牌的最佳成本平衡

---

### 疑惑 3｜「DNA 寫一次真的能用半年嗎？會不會品牌升級就沒用了？」

**疑惑**：

聽起來 brand-dna 像是「寫一次永遠用」——但品牌偶爾要升級、要換新風格、DNA 不會卡住自己嗎？

**回應**：

- DNA 不是**永久不可動的合約**——是**當前階段的視覺規格書**
- 通常一份 DNA 的有效期：**3-12 個月**（看品牌變化頻率）
- **半年後重新 review**：哪些禁用項實戰證明過嚴 / 哪些色票需要微調 / 風格 preset 是否仍適合
- 升級 DNA 不是「重寫」、是「**改 1-3 段**」——其他保留
- 真要大改版（換 logo / 換主視覺）：**重新填一份 DNA、跟舊 DNA 並存**——舊系列不動、新系列吃新 DNA

## 自我反思題 2 條（Reflection）

> ⚠️ 用「自我反思」替代「檢核題」——本單元不是考試、是給自己做總結。

### 反思 1｜整合工作流的 4 步、各自我做了什麼？

寫下：在「需求 → 執行 → 驗收 → 迭代」4 步中、你**作為 PM** 各自做了什麼動作？AI 又做了什麼？

**參考答案**（沒有標準、看你怎麼描述）：

- **需求**：你做——填 brief 六件套、選骨架、選 preset。AI 做——什麼也沒做（等你給指令）
- **執行**：你做——按下送出。AI 做——產出 L4 完稿（這段最辛苦的是 AI）
- **驗收**：你做——貼驗收提問清單、判讀 AI 自檢結果、決定收 / 迭代 / 重跑。AI 做——對照清單回答
- **迭代**：你做——下「保留 X / 修 Y」具體指令。AI 做——按你的指令重產

**心法**：**4 步中只有「執行」是 AI 主導、其他 3 步都是你**——這就是 PM 的工作占比。

---

### 反思 2｜本週你會做的 3 件具體事

寫下：

- **今天回家**會做的 1 件事（5-10 分鐘可完成）：
- **明天**會做的 1 件事（10-20 分鐘可完成）：
- **這週末**會做的 1 件事（30-60 分鐘可完成）：

**範例**：

- 今天回家：把 brand-dna.md 上傳 Notion / Google Drive、命名好（5 分鐘）
- 明天：把今天案例庫的 5 份 prompt 模板存到工作筆記本、設快捷標籤（10 分鐘）
- 這週末：用 PRAC4 + 案例 A 的組合、為下個月新品跑 1 張廣告（30 分鐘）

**不要寫**：

- 「我要把所有產品都做一遍系列」（太大、難啟動）
- 「我會持續學習」（空泛、不可量化）
- 「之後再說」（不會啟動）

**心法**：**小、具體、可完成的 1 件事**——比「完美計畫」實用 100 倍。
