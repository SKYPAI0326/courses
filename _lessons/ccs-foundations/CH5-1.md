## 一、問題提出（5 min）

**場景**：學生考完 CCS、拿到證照，回到自己的報告 / 作業 / 打工職場。他現在知道 CoT 是什麼、Few-shot 是什麼，但真正坐在電腦前要用 AI 寫一份報告摘要時——他只會打「幫我寫一份 XX 的報告摘要」。

**為什麼會這樣？** 認證教的是**名詞 → 定義**的方向，但實務需要的是**任務 → 技巧**的反方向。這一章補這個缺口。

## 二、核心概念：四象限決策樹（15 min）

用兩個軸把任務切四塊：

|  | **單步可解** | **需多步推理** |
|---|---|---|
| **有格式 / 範例可參照** | Few-shot | Chain + Few-shot |
| **無現成格式** | Zero-shot | CoT |

### 四類任務的判斷關鍵詞

1. **Zero-shot**：任務本身很明確，例「翻譯這段」「摘要這篇」「改寫成正式語氣」
2. **Few-shot**：要 AI 模仿某種**格式或語調**，例「幫我寫商品描述，風格像我之前寫的這三則」
3. **CoT**（思維鏈）：任務需要**推理**而非直接給答案，例「這份財報看得出公司健康嗎？」「這段話哪裡邏輯不通？」
4. **Chain**（提示鏈）：任務有**多個明顯步驟**且每步要獨立檢查，例「先找資料→歸類→產出簡報大綱→每段寫稿」

### 30 秒判斷口訣

> 問自己三題：
> 1. 我能不能一句話講清楚要什麼？（能 → Zero-shot；不能 → 往下）
> 2. 我手邊有沒有「好的範例」？（有 → Few-shot；沒有 → 往下）
> 3. 這件事需要分步驟做嗎？（需要分且每步要驗收 → Chain；只需要 AI 自己想清楚 → CoT）

## 三、示範：一個任務跑四種技巧（15 min）

**任務**：「幫我分析這三則商品評價，整理出消費者最關心的三個痛點。」

- **Zero-shot 版**：直接丟任務 → 輸出通常過於表面、分類鬆散
- **Few-shot 版**：先給一則「理想分析範本」→ 輸出格式整齊但分類可能照抄範本邏輯
- **CoT 版**：要求「一步步推理：先列每則評價的負面詞 → 歸類 → 統計頻率 → 排序」→ 輸出深度最好但較長
- **Chain 版**：把上述 CoT 拆成 4 個獨立對話輪次 → 每輪都能檢查、可控性最高但耗時

**結論**：同一個任務，根據「時間 / 精準度 / 可控性」的權衡，選不同技巧。沒有唯一正解。

## 四、動手練習題（8 min）

給 5 個任務描述，學生圈出該用哪種技巧（附參考答案與推理說明）：

1. 把這段英文郵件翻成中文 → Zero-shot
2. 寫 10 則商品標題，風格類似「蝦皮爆款」 → Few-shot
3. 判斷這份合約對我方有哪些風險條款 → CoT
4. 從客戶訪談逐字稿產出使用者旅程地圖 → Chain
5. 把這段會議錄音摘要成重點五條 → Zero-shot（但附格式限制）

## 五、常見錯誤

1. **萬事皆 CoT**
   - 現象：看到任何任務都加「請一步步思考」
   - 原因：CoT 被神化為「萬能強化符」
   - 解法：簡單任務加 CoT 只是拖時間、耗 token，沒有提升準確度

2. **Few-shot 範例全是同一類**
   - 現象：給 3 個範例全是正面情緒，結果模型學到「所有輸入都輸出正面」
   - 原因：範例缺乏多樣性
   - 解法：範例要涵蓋邊界與反例，3–5 則含正/反/中性

3. **把 Chain 做成單輪 prompt**
   - 現象：在一個 prompt 裡寫「先做 A，再做 B，最後做 C」
   - 原因：混淆了 CoT（單輪內部推理）與 Chain（多輪獨立任務）
   - 解法：Chain 的每一步要開獨立對話或明確分段輸出，前一步的結果要能人工檢查再餵下一步

## 六、檢核題

1. 你要幫同學寫一份「自傳範文，風格像某知名作家」。該用哪一類技巧？為什麼？
   - 答：Few-shot，因為你有明確的風格參照對象（範例）且任務單步可解。
2. 老師要你「分析這五份問卷，找出共同抱怨點並提出三個改善建議」。該用哪一類技巧？為什麼？
   - 答：Chain，因為「分析」「歸類」「產出建議」是三個可獨立驗收的步驟，每步錯了會傳染到下一步。
