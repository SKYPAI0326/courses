## 教學流程（Teaching Flow · 55 min）

### 1. 破題（3 min）

業務副理小李剛開完業務週會——業績檢討 + 兩件客訴處理。25 min、4 人、發言亂序。

會後他要產出 3 件：
- 本週業績問題決議清單
- 自己的個人行動表（誰負責 / 何時前 / 卡關找誰）
- 1-2 封跟進信草稿

手寫 60 min，Gemini 跑得好 15 min。

**這單元的任務**：先把 Gemini 介面操作摸熟，再用三件套 prompt 跑完小李這場會議的整理流程，最後換成你的部門變體跑一遍。

---

### 2. 介面操作說明（5 min）

不需要講原理，只講三個「第一次用容易卡住」的操作。

#### 新對話 vs 繼續對話

開新對話 = Gemini 忘掉前面講的全部。繼續對話 = 它記得。

**什麼時候開新對話**：換完全不同的任務（例：剛才整理會議紀錄，現在要寫招募文案）。

**什麼時候繼續對話**：同一件事要接著做（例：決議卡跑完，接著跑個人行動表，再接著跑跟進信）。

本單元三步 prompt 要在**同一個對話視窗**跑完，這樣第二步直接說「結合剛才的決議」就有效，不用重貼資料。

#### 上傳檔案方式

兩種：
1. **拖放**：把逐字稿 .txt 或 .md 拖進輸入框
2. **點擊迴紋針圖示**：選從裝置上傳

本課逐字稿素材是 .txt 檔，拖放最快。上傳後 Gemini 輸入框上方會顯示檔名，確認有顯示再送出。

> 免費版 Gemini 每次對話支援上傳 2-3 份檔案，超過要拆批次或改貼文字。

#### 輸出格式切換

Gemini 預設可能給段落文字，但你要的是表格或清單。有兩種改法：

1. **Prompt 裡直接寫**：「輸出：markdown 表格，4 欄」或「輸出：條列清單，每條 < 30 字」
2. **跑完再追問**：「請改成表格格式」「請把每條壓短在 20 字以內」

Prompt 裡先寫清楚比事後追問省時，本單元三個 prompt 的「輸出」段都已寫好格式要求，照用就行。

### 3. Prompt 三件套原則（15 min）

你在 office-ai-cases 學過 5 大骨架。三件套是其中最常用的三個：**角色 + 任務 + 格式**。

辦公室場景 80% 的情況這三件就夠了。

#### 三件套拆解

| 件套 | 問自己什麼 | 範例 |
|---|---|---|
| **角色** | 你要 Gemini 扮演什麼身份？ | 「你是小李的會議紀錄整理助手」 |
| **任務** | 具體做什麼？輸入是什麼？ | 「把以下 25 min 業務週會逐字稿，整理成決議清單」 |
| **格式** | 輸出長什麼樣？ | 「markdown 表格，4 欄：誰 / 做什麼 / 何時前 / 找誰確認」 |

#### 常見錯誤：這兩種問法最常讓人卡

**錯誤 1：問太模糊**

「幫我整理一下」「幫我看看這份會議」「整理出重點」——Gemini 不知道你要什麼格式、幾欄、幾筆，輸出就會每次不一樣。

**修法**：任務段加一句「整理成 XX 格式」或「輸出要包含 XX 欄位」。

**錯誤 2：一次問太多件事**

「幫我整理決議清單、個人行動表、寫跟進信、還有摘要一份 100 字給老闆看」——Gemini 通常只把第一件做好，後面的越來越草率。

**修法**：三步走。每一步一個 prompt，同一個對話視窗繼續，就不用重貼資料。

<div class="demo-takeaway">三件套 = 角色 + 任務 + 格式。這三件寫清楚，Gemini 的輸出可預測、可直接用。</div>

<div class="dialog-box">

> 三步完成了。現在回頭看，每步的三件套：角色都一樣（小李的整理助手），任務每步不同，格式每步不同。骨架不變，只換任務和格式，就能跑完整份會議整理。

**如何要求 Gemini 調整格式**（額外示範 1 min）：

<div class="dialog-box">
  <div class="dialog-label">如果決議清單用 Google Docs 貼上後跑版，可以這樣追問</div>
  請把剛才的決議清單改成純文字條列（不用 markdown 表格），每條格式：「誰：做什麼（何時前）」。
</div>

<div class="demo-takeaway">三步 prompt，同一個對話視窗跑完。骨架不變，換素材就能套到你的部門會議。</div>

#### 上機步驟（20 min）

1. **（3 min）** 打開 Gemini，確認帳號登入；把你部門的逐字稿 .txt 找好準備拖入
2. **（5 min）** 第 1 步 prompt：跑出決議清單，確認 4 欄齊全、筆數 ≥ 3
3. **（6 min）** 第 2 步 prompt：同一對話接續，跑出個人行動表（只列自己要做的、含「卡關找誰」）
4. **（4 min）** 第 3 步 prompt：選一封最急的跟進信草稿跑出來，確認 ≤ 150 字、結尾有行動句
5. **（2 min）** 對照驗收清單（見第 6 段），確認格式符合自己部門習慣

#### 講師走場重點

巡場時抓三件事：
- 三步有沒有在同一個對話視窗跑？（開了新對話要它接前面脈絡就麻煩了）
- 決議清單的「卡關找誰」有沒有填具體人名？（「主管」這種說法回家會卡住）
- 跟進信有沒有用「您」？（有的話要教追問修正方法）

### 6. 驗收（5 min）

對照以下清單自己確認，有打勾的才算完成。

**本單元驗收清單**：

- [ ] 決議清單：4 欄齊全（誰 / 做什麼 / 何時前 / 找誰確認）
- [ ] 決議清單：筆數 ≥ 3、閒聊段沒混進來
- [ ] 個人行動表：只有自己的項目、每條有具體人名的「卡關找誰」
- [ ] 跟進信：用「你」非「您」、≤ 150 字、結尾有行動句
- [ ] 知道怎麼用追問要求 Gemini 調整輸出格式

**驗收問兩個問題**（口頭確認）：

1. 三件套是哪三件？（角色 + 任務 + 格式）
2. 如果 Gemini 跑出來的格式不對，你怎麼改？（追問：「請改成 XX 格式」）

### Prompt 集（試跑包隨附）

- 三件套 prompt 範本 × 3 步 × 5 部門 = 15 個範本（可直接複製用）
- 追問 prompt 2 條：「請改成純文字條列」/ 「請把每條壓短在 20 字以內」
- 禁用「您」提示句：「語氣：禁用『您』改『你』、每封 ≤ 150 字、結尾必須是 XX」

### 帶走項

- 5 部門變體逐字稿素材包
- 三件套 prompt 範本 × 5 部門（學員帶走換成自己的會議資料）
- 本單元驗收清單 × 5 種（A4 一頁）

## 商業情境案例（Case）

**主角**：小李（32 歲業務副理）

**公司**：弄一下精選商行

**任務**：業務週會 25 min 結束，下班前要產出決議清單 + 個人行動表 + 跟進信草稿。

**小李的真實痛苦**：

不是不會開 Gemini，是「不知道 prompt 要怎麼寫才能跑出可以直接用的東西」。每次試，Gemini 給他的東西格式不對，要花 20 min 改成能用的樣子。

三件套解決的是這個問題：角色寫清楚讓它知道情境，任務寫具體讓它知道做什麼，格式寫死讓它輸出可以直接用。

**5 部門各自的場景連結**：

- 凱倫（財務）：月底結帳 deadlines 很緊，決議清單要今天出，不能明天再整理
- 小李（業務）：客訴要 24h 內回覆，跟進信需要今晚寄出
- 怡君（電商）：月度 KPI 差距要在週報前寫出行動計畫
- 阿翔（倉儲）：盤點後缺貨項目要立刻跟採購對齊，不能等到週會
- 采蓁（採購）：議價會議後供應商等確認，行動表 48h 內要出

## 動手練習題（Hands-on Exercise）

**題目**：選你的部門變體（5 選 1），用三件套 prompt 跑完完整 3 步，產出一份整理好的會議紀錄。

**步驟**：

1. 用第 1 步 prompt 跑出決議清單（4 欄 · ≥ 3 筆 · 閒聊不列入）
2. 同一對話視窗接著跑第 2 步，產出個人行動表（只列自己 · 有「卡關找誰」）
3. 跑第 3 步，產出 1 封跟進信草稿（≤ 150 字 · 用「你」不用「您」）
4. 對驗收清單 5 欄自我確認

**預期成果**：

- 決議清單 ≥ 3 筆，4 欄齊全
- 個人行動表每條都有具體「卡關找誰」人名
- 跟進信 ≤ 150 字，語氣直接，結尾有行動句
- 格式符合你部門平常用的樣子（不是非 markdown 不可，知道怎麼追問調整就行）

**完成標準**（self-check）：

- [ ] 三步都在同一個對話視窗跑完
- [ ] 決議清單 4 欄齊全 + 筆數 ≥ 3
- [ ] 個人行動表「卡關找誰」每條有具體人名
- [ ] 跟進信用「你」非「您」、≤ 150 字
- [ ] 遇到格式不對時知道怎麼追問修正

## 常見錯誤 3 條（Common Pitfalls）

### 1. 開新對話跑第 2 步，Gemini 忘掉決議清單

**錯誤現象**：第 1 步跑出決議清單後，學員開了新對話視窗跑第 2 步「依剛才的決議，整理個人行動表」。Gemini 說「請提供決議清單讓我整理」——它根本不記得剛才做的。

**原因**：每個 Gemini 對話都是獨立的，開新對話 = 清空記憶。

**解法**：三步 prompt 在**同一個對話視窗**跑完。第 2 步說「繼續」或「依剛才的結果」就有效，不用重貼資料。只有在換完全不同任務時才開新對話。

---

### 2. 「某人提到想做 X」被寫進決議清單

**錯誤現象**：怡君說「我覺得下個月可以試試直播帶貨，看看大家意見」——決議清單寫成「怡君：下個月開直播帶貨（月底前）」。怡君下週被問進度，說「我沒答應啊」。

**原因**：Gemini 會把「想法拋出」合理化成「決定要做」，沒加限制 instruction 就會這樣。

**解法**：prompt 在「格式」段加一句「仍在討論的提案不列入，只列已確認的決議」。跑完後對照逐字稿核對一遍，確認每筆決議真的在會議中被確認了。

---

### 3. 跟進信用「您」+ 寫成 300 字商業書信

**錯誤現象**：Gemini 寫出「您好，承蒙協助，特此說明本次客訴處理進度如下……」—— 400 字，起承轉合，同事看完一肚子火，覺得很假。

**原因**：Gemini 預設「商業書信」模板會用「您」+ 鋪陳。不寫格式限制就跑出這種輸出。

**解法**：prompt 格式段明寫「禁用『您』改『你』、每封 ≤ 150 字、結尾必須是 XX 一句話」。如果已經跑出 400 字版本，追問「請縮短到 100 字以內，只保留最重要的一個行動句」。

## 檢核題 2 條（Quiz）

**Q1（概念驗證）**：

以下哪個 prompt 設計**最能確保**決議清單的格式固定、可直接複製使用？

- [ ] A. 「幫我整理這個會議的重點」
- [ ] B. 「請整理決議，格式自訂，讓我容易讀就好」
- [ ] C. 「任務：把逐字稿整理成決議清單。格式：markdown 表格，4 欄 — 誰 / 做什麼 / 何時前 / 找誰確認；閒聊段落不列入」 ← **正確答案**
- [ ] D. 「請整理成 5 筆決議」

**理由**：

- A 沒有格式限制，Gemini 輸出不可預測
- B 格式讓 Gemini 自己決定，每次可能不一樣
- **C 任務 + 格式都寫清楚：欄位明確、過濾條件明確，輸出可預測**
- D 只限定筆數，沒限定欄位，還是會跑出不一樣的格式

---

**Q2（應用驗證）**：

**情境**：你是采蓁（採購），剛用三件套 prompt 跑完供應商議價會議逐字稿，Gemini 跑出來的決議清單是 markdown 表格，但你要貼進主管要求的 Google Sheets 格式，表格貼上去會跑版。

請說出：你下一個追問要怎麼寫？

**預期答案要點**：

- 直接追問格式：「請把剛才的決議清單改成純文字，每條格式：誰 · 做什麼 · 何時前 · 找誰確認，每條一行，不用 markdown 符號」
- 加分：能說出「不需要重新貼逐字稿，因為是在同一個對話視窗接著問，Gemini 記得剛才的結果」
- 加分：能說出「格式調整是三件套的格式件套的功能，改 prompt 的格式段就能控制」
