## 教學流程（Teaching Flow · 60 min）

### 1. Decision Checkpoint（3 min）

開場直接問你 1 句話：

> **「競品研究的哪些主張一定要有 NotebookLM 來源？哪些可放心讓 Gemini 推演？」**

預期答（討論完當場給）：

- **一定要 NotebookLM 取證**：數字、引述、競爭對手主張、客戶說過的話
- **可放心 Gemini 推演**：趨勢推論、SWOT 的「對我方意義」、改寫主管讀得懂的語氣

口訣：**「資料是什麼」要去 NotebookLM，「資料代表什麼」可問 Gemini。**

把這條口訣寫在便利貼上，今天的上機段你會用它判斷 5-7 次。

#### 第 3 步：來源可信度標記（3 min）

把 7 條 claim 一條條過一遍，標 source 等級：

- 🟢 **一手**：直接引用、有來源 URL / 文件位置、第三方可查證
- 🟡 **二手**：他人引用、有來源但未對到原文，或單一來源無交叉驗證
- 🔴 **推論**：Gemini 補的話、沒來源、或來源帶明顯偏見（如 A 同業自家部落格）

采蓁現場填的判斷示範（節錄 3 條）：

<div class="dialog-box">
  <div class="dialog-label">采蓁在來源可信度檢核表（手動判斷 · 不是貼到 AI）</div>
  Claim 1：「月結 60 天 + 滿 100 萬現金 1% 折扣」
  → 🟢 一手：產業媒體報導 + 已點開引用對到原文

  Claim 2：「同業議價門檻 500 萬給 5% 折讓」
  → 🔴 推論：A 同業自家部落格、行銷話術、未經第三方驗證

  Claim 3：「我方供應商關係 2 年穩定」
  → 🔴 推論：Gemini 標 (推論)、未取證
</div>

<div class="demo-takeaway">🟢🟡🔴 不是給自己看的，是給主管問「這數字哪來的」時 30 秒內答得出來的。🔴 的主張不要寫進主管報告，或寫進去要明示「待補佐證」。</div>

### 3. Prompt 拆解（5 min）

3 個 wrapper 的設計理由：

| Wrapper | 用在哪一步 | 為什麼這樣寫 |
|--------|---------|----------|
| 「每條 claim 必附引用原文片段 + 來源檔名 + 第幾段」 | NotebookLM 取證 | 不寫的話 NotebookLM 雖會引用，但**不會主動列「在第幾行」**。要對回原文核對偏見來源時找不到位置 |
| 「同一面向不同 source 矛盾請並列兩條」 | NotebookLM 取證 | 偏見來源（A 同業自家部落格）vs 中性來源（業界訪談）數字常打架。**並列才能讓你看到差異**，不會被單一偏見來源帶風向 |
| 「對每段陳述標註『來自取證材料』或『推論』，後者用 (推論) 標明」 | Gemini 整合 | Gemini 預設會把推論寫得像事實。**強制它自標 (推論)**，你看到括號就知道那段沒來源、要降為 🔴 |

特別點名 **「偏見來源」的處理**：

- A 同業自家部落格那條 claim，NotebookLM 抓進來時會標來源 → 你**不能因為「有引用」就當 🟢**
- 偏見來源的判斷不在 NotebookLM 那一步，而在你**回頭點開引用看原文**那一步
- 看到「我們公司一直以最優惠的條件回饋客戶」這種行銷話術 → 直接降為 🔴
- 規則：**自家發布的內容（部落格 / 官網 / 新聞稿）→ 預設 🔴，除非有第三方交叉驗證**

### 4. 動手 / Hands-on（30 min · 選自己部門變體跑）

挑你部門變體（5 部門變體素材詳列於試跑包附錄，每包含至少 1 篇偏見來源），跑「**NotebookLM 取證 → Gemini 整合 → 標 source 等級**」完整流程。

**5 步驟**：(1)(5 min) 開 NotebookLM 建 notebook + 上傳 8 個 source → (2)(10 min) 貼取證 prompt（5 面向依部門調整）抓 5-7 條 claim → (3)(3 min) 點開引用對回原文 + 標偏見來源 → (4)(7 min) 複製 claim 到 Gemini 跑整合 prompt → (5)(5 min) 填可信度檢核表 5-7 條全掛 🟢🟡🔴。

觸發口訣：Gemini 數字找不到引用 → 🔴；NotebookLM 引用對到原文是行銷話術 → 🔴；兩 source 數字打架 → 並列 + 🟡。

## 常見錯誤 3 條（Common Pitfalls）

### 1. 把「有引用」當「可信」

**錯誤現象**：NotebookLM 引用了 A 同業自家部落格的數字，學員看到有引用就標 🟢，結果這數字是行銷話術。主管問「為什麼相信這數字」答不出來。

**原因**：NotebookLM 只負責「指出來源在哪」，不負責「判斷來源可不可信」。判斷可信度是你的工作。

**解法**：規則寫死——**自家發布內容（部落格 / 官網 / 新聞稿 / 投資人關係 / 公關稿）預設 🔴**，除非另有第三方來源交叉驗證才升到 🟡 或 🟢。第 1 步取證後要**點開引用看原文**，行銷話術就降級。

---

### 2. Gemini 沒標 (推論) 就當事實用

**錯誤現象**：Gemini 整合 SWOT 時把推論寫得像事實（例：「我方供應商關係 2 年穩定」），學員沒注意，直接寫進主管報告。主管追問「2 年穩定的根據是？」答不出來。

**原因**：Gemini 預設「商業分析報告」會用肯定句寫推論，wrapper 沒寫嚴格就會省略 (推論) 標記。

**解法**：(1) Gemini prompt 必加「對每段陳述標註『來自取證材料』或『推論』，後者用 (推論) 標明」(2) 整合段回來後**搜尋 (推論) 字串**，每個都對應到檢核表的 🔴 (3) 主管報告中的 🔴 主張要嘛刪掉、要嘛明示「待內部資料佐證」。

### 3. 檢核表全標 🟢 或全標 🔴

**錯誤現象**：學員填檢核表時，要嘛全部 🟢（樂觀派、沒辨識偏見）、要嘛全部 🔴（保守派、覺得反正都不可靠）。兩種都沒做到「**分級可信度**」的訓練目的。

**原因**：沒理解 🟢🟡🔴 的判斷標準——把標籤當形容詞用，不是當決策工具用。

**解法**：強制檢查「**🟢🟡🔴 三種都至少有 1 條**」。沒有的話：

- 全 🟢 → 你沒對回原文、沒辨識偏見來源 → 回頭點開引用
- 全 🔴 → 你的取證 prompt 沒抓到中性來源的 claim → 重抓
- 缺 🟡 → 你沒注意「單一來源」vs「多來源交叉」的差別 → 加一條「獨立可查證」欄打 ✗ 的就是 🟡

## 檢核題 2 條（Quiz）

**Q1（概念驗證）**：以下哪種來源 **預設應標為 🔴**？（複選）

- [ ] A. 產業媒體 2025/11 報導，引用第三方數據
- [ ] B. A 同業自家部落格「我們是業界最優條件」 ←（**正確**）
- [ ] C. NotebookLM 抓出來、但你還沒點開引用對回原文的單一來源
- [ ] D. Gemini 整合段標了 (推論) 的陳述 ←（**正確**）
- [ ] E. 上市公司年報的「投資人關係 / Q&A」段落自我評價 ←（**正確**）

**預期答案要點**：

- B / D / E 都是 🔴：自家發布內容（B、E）+ Gemini 推論（D）
- A 是 🟢：產業媒體 + 第三方數據 + 對回原文（假設你已對過）
- C 是 🟡：有引用但**未對回原文** + 單一來源未交叉驗證

**情境**：你是阿翔（倉儲主管），做「同業倉儲效率標竿」研究。NotebookLM 取證抓出 4 條代表性 claim：
