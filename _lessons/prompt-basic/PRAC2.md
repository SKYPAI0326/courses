## 教學流程（Teaching Flow）

### 破題 / Hook

你剛收到一份跨局處協調會議的會議紀錄，要請 AI 幫你整理行動清單、起草追蹤函。你打開文件，全文貼進去——然後想到 CH2-2 說的話：「上面有沒有不能貼的東西？」

這份紀錄大約 1,500 字。你會怎麼掃？從哪裡開始？

這個練習的目的不是「找到」就好——而是練習用系統化的方式一次掃完，不漏掉、也不過度刪改到影響後續作業。

### 概念 / Concepts

（本單元為練習頁，概念在 CH2-2 已涵蓋。此處僅列出操作中需要用到的工具定義。）

- **個人識別資訊（PII）** — 可用來直接或間接識別特定個人的資料，包含姓名、電話、身分證字號、地址等
- **去識別化替換** — 以「○○○」「[移除]」「某機關」等佔位符取代原始資料，使文件無法直接指向特定人或案件
- **敏感業務資訊** — 未公開的預算數字、內部案號、決策草案等，不得貼入公共 AI 平台
- **行動清單** — 從決議事項提煉的格式化追蹤清單，含「負責人 + 事項 + 期限」三要素

### 操作示範 / Demo

**尋找個資的系統化順序**（老師說明，30 秒）：

掃會議紀錄時，依序找這五類：

| 順序 | 找什麼 | 出現位置 |
|---|---|---|
| 1 | 完整姓名 | 出席人員、發言記錄 |
| 2 | 電話 / 手機號碼 | 聯絡方式討論、附件說明 |
| 3 | 身分證字號 | 陳情人資料、附件說明 |
| 4 | 未公開預算 / 金額 | 討論事項、報告段落 |
| 5 | 內部案號 | 標題、決議、附件 |

找到之後，套用替換規則卡（見下方），**每個陷阱只換一次，不要改到其他無關文字**。

### 動手 / Hands-on

**學員練習流程（單人，25 分鐘）**：

- **步驟 1**（8 分鐘）：閱讀下方會議紀錄原文，用筆圈出所有個資陷阱。對照「五類尋找清單」逐類掃一次。
- **步驟 2**（7 分鐘）：依替換規則卡，將原文中 5 個陷阱逐一替換為去識別化版本。可在紙本上直接塗改或手寫覆蓋。
- **步驟 3**（7 分鐘）：閱讀「決議事項」與「待辦事項」兩段，整理為行動清單（格式：負責人 + 事項 + 期限）。
- **步驟 4（延伸，3 分鐘）**：將步驟 3 的行動清單，用四欄框架寫一份交辦單，請 AI 幫你起草追蹤函草稿。

### 檢核 / Verification

學員完成後，老師請 2 位學員唸出自己找到的陷阱清單，全班確認：

- [ ] 找出 5 個陷阱，沒有漏項
- [ ] 每個陷阱使用規定的替換格式（不是自己發明）
- [ ] 行動清單每條都有「負責人 + 事項 + 期限」三要素
- [ ] 去識別化後文意仍通順，決議脈絡未因替換而喪失

## 常見錯誤 3 條（Common Pitfalls）

1. **錯誤現象**：只替換「出席人員名單」欄位的姓名，忽略正文段落中再次出現的姓名（如「陳志明科長回應」「吳俊宏副局長表示」）。
   **原因**：視覺上先掃完表格就覺得「姓名已處理」，沒有繼續掃正文段落。
   **解法**：去識別化時依段落順序從頭掃到尾，不按資料類型跳著掃；或在表格處理完後，搜尋已替換的原始姓名是否還出現在其他段落。

2. **錯誤現象**：把「候選場址 A（位於○○鄉）」的「○○」當成個資陷阱，一起移除。
   **原因**：大綱中已用「○○」通用化地名，學員誤以為這是需要處理的真實資訊。
   **解法**：「○○鄉」是原文就已去識別化的地名，不是陷阱。只處理「出現真實可識別資料」的位置，不要過度移除。

3. **錯誤現象**：行動清單只寫「事項」，漏掉「負責單位」或「期限」，例如只寫「完成交通量測作業」。
   **原因**：學員直接複製待辦表格的「事項」欄，沒有確認三要素都在。
   **解法**：整理前先在紙本上標記三個欄位頭（負責 / 事項 / 期限），每條填完都確認三格都有字。
