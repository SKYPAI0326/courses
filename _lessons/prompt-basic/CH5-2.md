## 教學流程（Teaching Flow）

### 破題 / Hook

你有沒有看過同事轉傳一份「超好用 AI 交辦單」，貼進 ChatGPT 卻發現回答怪怪的？

不是這份交辦單沒用。是因為它是別人機關的版本，用的語氣、假設的背景、要的格式，都是「別人的規矩」。你直接用，AI 會依照別人的規矩回答你——而那些規矩跟你機關的不一樣。

這一節要學的，就是怎麼把別人的版本改造成你的版本，四個步驟：**看、拆、改、問**。

---

### 概念 / Concepts

- **借力改造法** — 不從零寫交辦單，而是找一份現成的，拆解它的結構，補上自己的情境，變成可直接使用的版本。

- **四步驟（看、拆、改、問）** — 每個步驟有明確的操作動作，不用靠直覺判斷交辦單的好壞。

- **四欄完整性** — 借來的交辦單通常只填了「任務」和「要什麼樣子」，缺「你是誰」和「背景」。找缺口比從頭寫還快。

- **機關味道** — 每個組織有自己的語氣慣例（公部門要正式、內部簡報要簡潔）。改造的核心動作就是「洗掉別人的味道，補上你的味道」。

### 操作示範 / Demo

> **示範素材：一份有缺陷的流傳交辦單**

以下是一份在網路社群流傳、用於「請 AI 整理會議紀錄」的交辦單：

---

**【流傳版交辦單】**（約 100 字，完全虛構）

```
請你幫我整理這份會議紀錄，輸出一份乾淨的整理稿。

整理時請：
1. 列出所有討論事項
2. 標出每個人的發言重點
3. 把決議清楚寫出來
4. 行動清單也要列進去

[會議紀錄貼在這裡]
```

---

**四步驟完整示範腳本**

---

#### 看：這個交辦單的四欄有沒有填滿？

講師說明：快速掃描四個欄位的填寫狀況。

| 欄位 | 有沒有填 | 說明 |
|---|---|---|
| 你是誰（AI 扮演什麼身分） | ❌ 沒有 | 沒說 AI 應以什麼專業角度整理 |
| 你要做什麼（具體任務） | ✅ 有 | 「整理會議紀錄」清楚 |
| 有什麼背景要讓它知道 | ❌ 沒有 | 沒說這是什麼類型的會議、什麼機關、用途是什麼 |
| 要什麼樣子的回答 | △ 模糊 | 只說「乾淨的整理稿」，沒有字數限制、沒說格式（表格/條列/段落） |

**看的結論**：這份交辦單缺了「你是誰」和「背景」，格式要求也含糊。多數流傳的交辦單都有這個問題——只填了任務和樣子，但沒說身分和背景。

---

#### 拆：請 AI 幫你分析這個交辦單的優缺點

講師操作：把上面那份交辦單貼給 AI，加上一句話：

**給 AI 的指令**：

```
請分析這個交辦單的優缺點，以及可以如何改善。

[貼上上面的交辦單]
```

**AI 的輸出（預期文本）**：

```
這份交辦單的分析如下：

優點：
1. 任務說明清楚：「整理會議紀錄」目標明確，不含糊
2. 輸出項目列得具體：討論事項、發言重點、決議、行動清單四個面向都有提到
3. 結構簡潔，容易閱讀

缺點及改善建議：
1. 缺少身分設定：沒有說明 AI 應以什麼角色或專業角度來整理（例如「會議記錄人員」或「行政承辦人」），可能導致整理風格不符合需求

2. 缺少背景資訊：不知道這是什麼性質的會議（業務協調、跨局處會議、首長主持？）、也不知道整理結果給誰看（主管？對外存檔？），輸出內容會難以針對化

3. 格式要求模糊：「乾淨的整理稿」沒有指定字數上限、段落數、或呈現格式（條列 vs. 表格 vs. 段落文字），AI 可能給出過長或過短的版本

4. 沒有說明輸出用途：整理出來是要存檔、寄給與會人員、還是向長官報告？用途不同，整理的重點也不同

整體評估：這份交辦單可以直接用，但結果品質取決於 AI 的預設判斷，不同 AI 工具的輸出會有明顯差異。建議補上身分、背景、格式規範三個欄位後再使用。
```

講師說明：你不需要自己判斷哪裡好哪裡壞——讓 AI 當你的分析師，它會告訴你。

---

#### 改：把別人機關的味道洗掉，補上你的狀況

講師說明：根據「拆」的分析結果，補上你的機關情境。

示範情境：你是某縣市政府○○局的承辦人員，需要整理跨局處協調會的會議紀錄，整理結果要提給科長看，格式需符合機關公文慣例。

**改造後的交辦單**：

```
你是一位熟悉地方政府行政流程的會議記錄整理人員。

請幫我整理這份跨局處協調會的會議紀錄。
整理結果提供給科長，作為內部存檔與後續追蹤使用。

背景說明：
- 這是某縣市政府○○局召開的跨局處業務協調會
- 與會人員包含本局科長、承辦人員及相關局處代表
- 會議目的：協調業務分工與期程

請依以下格式輸出，全文不超過 600 字：

一、會議基本資訊（時間、地點、主持人、出席單位）
二、討論事項（條列，每項 2–3 行）
三、決議事項（條列，標示負責單位與完成期限）
四、行動清單（表格格式，欄位：事項 / 負責人 / 期限）

語氣使用正式公文體，不使用口語或縮寫。

[會議紀錄貼在這裡]
```

講師說明：改造的核心動作只有兩個——刪掉不適用的設定、補上你機關的規矩。不需要重新發明，只需要替換。

---

#### 問：請 AI 問你 3 個問題，回答完就變成你的版本

講師說明：如果你不確定自己的情境應該補什麼，讓 AI 來問你。

在改造後的交辦單最後加上這一句：

```
在你幫我執行前，請先問我 3 個最重要的問題，讓你能給出更準確的結果。
```

**AI 的三個問題（預期輸出）**：

```
在我開始整理之前，請回答以下三個問題，幫助我給出更符合你需求的整理稿：

1. 這份會議紀錄有逐字稿嗎，還是只有重點記錄？
   （影響我是否需要主動精簡發言內容，或只需歸納段落）

2. 決議事項有沒有已確認的負責人和期限？
   （如果有，我會直接填入行動清單；如果沒有，我會標注「待確認」）

3. 這份整理稿除了提給科長，之後有沒有可能對外發送或公開存檔？
   （影響去識別化的程度和措辭的正式程度）
```

**學員回答範例**：

```
1. 只有重點記錄，沒有逐字稿
2. 決議事項部分有負責人，但期限大部分還沒確認
3. 只內部用，不對外
```

**AI 接收回答後的最終輸出（預期文本）**：

```
收到，依據你的說明，以下是整理後的會議記錄稿：

一、會議基本資訊
時間：[依紀錄填入]
地點：[依紀錄填入]
主持人：○○科長
出席單位：本局承辦科室、○○局、○○處

二、討論事項
1. 【事項名稱】
   討論重點：[整理為 2–3 行摘要]
   各方意見：[若有不同意見，分別列出]

2. 【事項名稱】
   討論重點：[整理為 2–3 行摘要]

三、決議事項
1. [決議內容]——負責單位：○○局，完成期限：待確認
2. [決議內容]——負責單位：本局承辦科，完成期限：[依紀錄填入]

四、行動清單

| 事項 | 負責人 | 期限 |
|---|---|---|
| [行動 1] | ○○局 | 待確認 |
| [行動 2] | 本局承辦 | [依紀錄填入] |
| [行動 3] | [依紀錄填入] | 待確認 |

（本稿為內部用途，語氣以正式公文體呈現，去識別化程度維持一般行政慣例）
```

講師說明：你不需要預先想清楚所有細節。讓 AI 問你，你回答，它自動把問題的答案整合進去。這三個問題問的，往往是你自己沒想到但其實很重要的事。

### 動手 / Hands-on（選用，課堂時間允許時執行）

> 本段標注「選用」：若時間不足，講師直接跳至「檢核」段。課堂示範優先。

**題目**：

你的同事傳來一份「請 AI 幫你寫陳情回函」的交辦單。以下是原版：

```
請幫我寫一封回覆陳情人的信。陳情人對目前的處理結果不滿意，
希望機關重新審查。請用婉轉但明確的語氣說明機關立場，
並感謝對方的反映。

[陳情內容貼這裡]
```

執行「看拆改問」四步驟，把這份交辦單改造為你機關的版本。

**預期成果**：
- 一份填入你機關情境的改造版交辦單（包含完整四欄）
- 若執行「問」步驟，附上 AI 問的三個問題

**完成標準**（self-check）：
- [ ] 已補上「你是誰」欄位（AI 的身分設定）
- [ ] 已補上「背景」欄位（機關類型、函文用途、收件對象）
- [ ] 格式要求已改為具體描述（字數範圍 or 段落數）
- [ ] 已洗掉原版語氣中不符合你機關慣例的部分

---

### 檢核 / Verification

講師結尾確認學員理解的三個問題（口頭確認，不用書面）：

1. 「看」這個步驟，你在找的是什麼？
   → 預期答案：找四欄有沒有填滿，特別是「你是誰」和「背景」

2. 為什麼要執行「拆」？自己看不就好了？
   → 預期答案：讓 AI 當分析師比自己判斷快，也比較不會漏掉問題

3. 「問」步驟的那句話加在哪裡？
   → 預期答案：加在交辦單的最後，執行前要求 AI 先問你問題

## 常見錯誤 3 條（Common Pitfalls）

1. **錯誤現象**：學員在「改」的步驟直接修改原版交辦單，但保留了原版的「你是誰」欄位設定（例如：「你是一位文案撰寫師」）不覺得有問題
   **原因**：改造過程中習慣關注「任務」和「格式」兩欄，容易忽略身分欄位也需要替換
   **解法**：講師提示「看」步驟時，特別強調身分欄是最常被忽略的缺陷；改造後用四欄清單逐一核查

2. **錯誤現象**：學員執行「問」步驟，把那句「請先問我 3 個問題」加在交辦單的開頭，AI 先問問題但後來忘記執行原始任務
   **原因**：AI 看到「先問我」在開頭，容易進入純問答模式，後來沒有自動執行任務
   **解法**：那句話必須加在交辦單的最後，格式是「在你幫我執行前，請先問我 3 個最重要的問題」

3. **錯誤現象**：學員跳過「拆」步驟，直接進入「改」，結果改造後的交辦單仍有沒注意到的缺陷（例如：背景補了，但格式還是模糊的）
   **原因**：「拆」的目的是讓 AI 做系統性分析，跳過後改造就只靠直覺，容易漏改
   **解法**：強調「拆」是省力工具，不是多餘步驟；讓 AI 做一次分析只要 10 秒，但自己逐項想需要 3 分鐘

## 檢核題 2 條（Quiz）

**Q1（概念驗證）**：「看」這個步驟，主要是在確認什麼？

- [ ] A. 這份交辦單的字數夠不夠長
- [ ] B. 這份交辦單的作者是誰
- [ ] C. 四欄框架（你是誰 / 任務 / 背景 / 樣子）有沒有填完整 ← 正確答案
- [ ] D. 這份交辦單用了哪個 AI 平台

**Q2（應用驗證）**：一位同事給你看一份流傳的「撰寫開會通知」交辦單，你執行「看」的步驟後發現：任務清楚、格式有說，但缺少「你是誰」和「背景」。下一步你應該怎麼做？

預期答案要點：
- 執行「拆」步驟：把這份交辦單貼給 AI，加上「請分析這個交辦單的優缺點，以及可以如何改善」
- 根據「拆」的分析結果，執行「改」：補入機關類型、會議性質、收件對象（你的機關同仁？對外？）
- 選擇性執行「問」：若不確定要補什麼背景，在交辦單後加上「在你幫我執行前，請先問我 3 個最重要的問題」
