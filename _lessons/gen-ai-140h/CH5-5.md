## 教學流程（Teaching Flow）

### 破題 / Hook（3 分鐘）

行政小陳上週末跟 Gemini 聊了一小時，做出一個漂亮的個人介紹單頁，朋友看了都說讚。但**朋友打不開**。

不能用 LINE 傳網頁、不能用 Email 寄 HTML 檔（朋友手機點開只看到一堆程式碼）、不能截圖（截圖不能點按鈕）。她卡在這裡兩小時，最後放棄。

問題不是她的 HTML 不夠好，問題是她**沒有一個網址**。

這 1.5 小時要解決的，就是「**把 AI 給你的程式碼變成一個能分享的 https:// 網址**」。

只要學會這 5 步，你以後做的所有 AI 工具、遊戲、單頁網站，都能丟出去給朋友、客戶、面試官看。**全程不打開 Terminal、不裝任何軟體、不背任何 git 指令**。我們的口號是：

> **「你不需要懂 GitHub，你只需要會點按鈕。」**

這節課之後，CH5-6（多檔場景）、PRAC5-13（個人單頁）、PRAC5-14（小遊戲）、PRAC5-15（AI 工具自用 demo）全部都會回頭叫你「請執行 CH5-5 五步」。所以這 5 步**務必記熟**。

### 概念 / Track 判定（2 分鐘）

開始之前先問自己一個問題，**只有兩個答案**：

> **AI 給你的程式碼，是不是「一個 `index.html` 檔，裡面什麼都包了」？**

判斷準則：

| 觀察 | 判定 |
|------|------|
| 所有 CSS 寫在 `<style>...</style>` 內 | ✅ 是單檔 |
| 所有 JavaScript 寫在 `<script>...</script>` 內 | ✅ 是單檔 |
| 圖片是 `https://...` 開頭的網址（公開 CDN） | ✅ 是單檔 |
| 程式碼裡有 `<script src="./script.js">` | ❌ 不是 — 有本地相對路徑 |
| 程式碼裡有 `<link href="./style.css">` | ❌ 不是 — 有本地相對路徑 |
| 程式碼裡有 `<img src="photo.jpg">` 但 AI 沒給你 photo.jpg | ❌ 不是（或修一下，見下面） |

**結論：**
- ✅ 是單檔 → 直接走本單元教的 5 步部署
- ❌ 不是（有本地相對路徑）→ 先去 CH5-6 學「請 AI 幫我整合成單檔」提示詞，再回來走本單元 5 步

**重要例外（灰色地帶）：** 看到 `<script src="https://cdn.jsdelivr.net/npm/chart.js"></script>` 這種**公開 CDN 連結** —— **算單檔**，可走本 SOP。因為 CDN 是網路上的公開檔案，瀏覽器自己會去拿，不需要你提供。

簡單判斷：**`src=` 或 `href=` 後面是 `https://` 開頭 = OK；是 `./` 或 `photo.jpg` = 不 OK**。

### 概念 / 4 種來源「複製程式碼按鈕在哪」對照表（5 分鐘）

不同 AI 工具產出程式碼的「複製按鈕」位置不同。記不住沒關係，這頁是隨時翻的查表。

| AI 工具 | 找哪個按鈕 | 通常情況 |
|--------|----------|---------|
| **Claude Artifacts** | Artifact 視窗右上「**Copy**」（兩個方塊重疊的圖示） | 預設就單檔，最順 |
| **ChatGPT Canvas** | Canvas 右上「下載」（向下箭頭）→ 存成 .html 檔；或全選 Cmd+A → Cmd+C | 預設單檔 |
| **Gemini Canvas** | Canvas 右上「**Copy code**」（兩個方塊圖示） | **常常多檔**，先跑 CH5-6 整合提示詞 |
| **Google AI Studio** | Code 區塊右上的複製圖示 | **常常多檔**，先跑 CH5-6 整合提示詞 |

> **講師端示範：** 上課時打開 4 個分頁（Claude、ChatGPT、Gemini、AI Studio）給學員看一遍每個按鈕長相，現場錄影下來放講義。

**注意：** Gemini 與 AI Studio 是多檔重災區。你會發現它們**只在 Canvas 視窗顯示一個主檔案**，但其實內部還有 `style.css`、`script.js`、甚至 `assets/` 資料夾。**Phase 1 階段我們先學單檔場景**（Claude Artifacts、ChatGPT Canvas 最常成功），等熟練後再回 CH5-6 解多檔。

### 操作示範 / 5 步部署動線（30 分鐘）

這段是本單元的**核心**。講師現場示範一次、學員現場跟做一次。

**前置條件（開課前一週應已完成）：**
1. 學員已註冊 GitHub 帳號（github.com 用 Email 註冊免費）
2. 學員已完成 GitHub 2FA 設定（GitHub 現在強制要求，可裝 Google Authenticator App 掃 QR Code）
3. 學員已在課堂用講師示範 repo 驗證網路通（**部分學校 / 公司網路會擋 GitHub Pages，要當場確認**）

**準備：** 從 Claude Artifacts 隨便產一個單檔 HTML（例如「請做一個顯示時鐘的單檔 HTML 頁」），複製整段程式碼放剪貼簿備用。

---

#### Step 1 — 開新 repo（1 分鐘）

> 動作：開 [github.com](https://github.com) → 右上「**+**」→ **New repository**

填寫表單：

- **Repository name**：`my-clock`（**全英文小寫、用連字號連單字、不要中文**）
- **Description**：可空白
- **Public** ✅（必須 Public，免費版 GitHub Pages 只支援 Public repo）
- **Add a README file** ✅（勾起來、不然 repo 是空的進不去）
- 其他不用動

點下面綠色按鈕 **Create repository**。

**畫面跳到：** 新 repo 頁面，左上會有 `{你的帳號}/my-clock`。

---

#### Step 2 — 按鍵盤的 . 鍵進 github.dev（30 秒）

> 動作：**在 repo 頁面任何位置按一下鍵盤上的 `.`**（句點鍵）

不騙你，就是這個鍵。

**畫面跳到：** 一個跟 VS Code 長得一模一樣的編輯器、但**在瀏覽器裡**。網址會從 `github.com/...` 變成 `github.dev/...`。

> **這是什麼？** github.dev 是 GitHub 免費送的線上版 VS Code，**不需要安裝任何東西**。在這裡改檔案、commit、push，全部用滑鼠點。

**如果按 `.` 沒反應**：可能你還沒從 GitHub 登入。重新整理頁面、確認右上頭像顯示你的帳號名，再試一次。

---

#### Step 3 — 新建 index.html 貼上程式碼（2 分鐘）

> 動作：左側檔案區點「**+**」（新檔案）→ 命名 `index.html`（**全小寫**）→ Enter

**重要：** 檔名一定是 `index.html`，不是 `Index.html`、不是 `INDEX.html`、不是 `首頁.html`。**檔名錯一個字母 Pages 就 404**。

檔案打開後右側出現空白編輯區：

> 動作：在右側編輯區點一下游標進去 → Cmd+V（Mac）/ Ctrl+V（Windows）**貼上剛才從 AI 複製的程式碼**

**畫面跳到：** 整段 HTML 出現在編輯器，左側檔案列表的 `index.html` 旁邊會出現一個藍色 **U**（代表 Untracked，還沒儲存進 git）。

---

#### Step 4 — Commit + Sync（2 分鐘）

> 動作：左側最左邊的圖示列 → 找到一個「**像三個圈圈連著線的圖示**」（Source Control / 原始碼控制）→ 點一下

左側區域變成 Source Control 面板：

> 動作：在「Changes」清單找到 `index.html` → **滑鼠移上去會出現一個「+」** → 點下去（這叫 stage）

接著上方有個輸入框寫「Message (press Cmd+Enter to commit)」：

> 動作：輸入框打字 `first deploy`（隨便寫什麼都行）→ 點上方藍色按鈕 **Commit & Push**（或 Mac 上按 Cmd+Enter）

**畫面變化：** 左下角會出現一個小轉圈圈、然後變成「Sync successful」之類的訊息。代表程式碼已經傳到 GitHub 主站。

---

#### Step 5 — 啟用 GitHub Pages（1 分鐘 + 等 1~3 分鐘）

> 動作：回到 GitHub 主站（重新整理 github.com 那個分頁）→ 進 repo 頁面 → 點上方 **Settings**（齒輪圖示）

進設定頁後左側選單：

> 動作：左側捲到 **Pages**（在 Code and automation 區段下面）→ 點它

右側 Pages 設定畫面：

> 動作：
> 1. **Build and deployment** 區的 **Source** 下拉選 `Deploy from a branch`
> 2. **Branch** 下拉選 `main` → 旁邊資料夾選 `/ (root)` → 點 **Save**

**畫面變化：** 上方會出現一行藍色或黃色的訊息「Your site is being built...」或「Your site is live at https://{你的帳號}.github.io/my-clock/」。

> ⏰ **等 1~3 分鐘**。第一次部署 GitHub 要建環境。喝口水。

**最後一步：** 把那個 `https://...github.io/...` 網址複製貼到新分頁打開 → **應該看到你的時鐘** 🎉

---

#### 完成驗收

到這裡你應該有：
- ✅ 一個 GitHub repo（`https://github.com/{你}/my-clock`）
- ✅ 一個 GitHub Pages 網址（`https://{你}.github.io/my-clock/`）
- ✅ 朋友、客戶、面試官、隨便誰打開都能看到你的東西

你剛剛**沒有**：
- ❌ 開 Terminal
- ❌ 打任何 git 指令
- ❌ 裝任何軟體
- ❌ 理解 git 是什麼

這就是傻瓜模式。

### 卡關地圖 / 6 個必教的常見錯誤（10 分鐘）

學員會在這裡卡住，先講過一次省得到時候助教救火。

#### 1. 檔名打成 `index.HTML`（大寫）→ Pages 404

**症狀：** Pages 啟用後打開網址，看到「404 - There isn't a GitHub Pages site here.」

**原因：** GitHub Pages 只認識小寫 `index.html`。

**修法：** 進 github.dev → 對檔名按右鍵 → Rename → 改成全小寫 → commit & push → 等 1 分鐘。

---

#### 2. repo 名用中文 → URL 亂碼、Pages 失敗

**症狀：** 想用「我的時鐘」當 repo 名 → GitHub 顯示 `我的時鐘` 但網址變 `%E6%88%91%E7%9A%84...`。

**修法：** 直接 **Settings → 拉到最下面 → Delete this repository** → 重做一個英文小寫的。

> 💡 **記住：** repo 名一律全英文小寫、用連字號（`my-clock` 不是 `MyClock`）。

---

#### 3. Pages 啟用後沒等 1~3 分鐘就試 → 看到 404 以為壞了

**症狀：** Step 5 做完馬上打開 URL，看到 404。

**原因：** GitHub 第一次部署要建環境，**沒有錯**。

**修法：** **喝口水**。3 分鐘後重新整理（**Cmd+Shift+R** 強制重新整理、繞過快取）。

---

#### 4. 改完忘了 commit → 看到舊版

**症狀：** 在 github.dev 改了程式碼，但打開 Pages 網址還是舊內容。

**原因：** Step 4 的 commit & push 沒做。改完一定要 commit。

**修法：** 回 github.dev → Source Control → 看「Changes」有沒有東西 → 有就 stage + commit & push。

---

#### 5. 直接把 Canvas 截圖傳上去 → 不是 HTML 是圖片

**症狀：** 學員把 Claude / Gemini Canvas 視窗截圖、存成 `index.html` 想上傳。

**原因：** **截圖是圖片，不是程式碼**。GitHub Pages 看到 `index.html` 裡只有圖片二進位資料、跑不起來。

**修法：** 回 AI 工具找「Copy」或「下載」按鈕，**複製的是文字**（程式碼），不是截圖。

---

#### 6. AI 產出含 `<script src="./local.js">` → 找不到檔案

**症狀：** 部署上去，網頁打開沒互動、開瀏覽器開發者工具看到「找不到 local.js」紅字。

**原因：** AI 給的不是單檔，是多檔。`./local.js` 是「本地檔案」、但你只傳了 index.html 沒傳 local.js。

**修法：** 這是「不是單檔」的訊號 → 回 AI 工具丟這句：

> 「請把這個專案重新整理成**單一 HTML 檔的版本**：所有 CSS 寫進 `<style>` 標籤、所有 JavaScript 寫進 `<script>` 標籤、圖片用 `https://placehold.co/` 連結代替。我要複製整段就能存成 index.html 直接用的版本。」

拿到單檔後回 Step 3 重貼。

---

### 操作示範 / 「錯了就重做」復活術（5 分鐘）

新手最大的敵人**不是錯誤本身，是「卡住超過 5 分鐘還在試」**。

**鐵律：** 任何卡關**超過 5 分鐘** → 你**不是 debug、是重做**。

#### 怎麼重做

> 動作：進 repo 頁面 → **Settings** → 拉到最下面紅色區塊 **Danger Zone** → 點 **Delete this repository** → 照指示輸入 repo 名確認 → Delete

repo 沒了。回 Step 1 重做一遍。

**為什麼這樣最快：**
- GitHub 免費 repo 數量**沒上限**（理論上可以開幾千個）
- 你做第二次只要 5 分鐘（第一次 30 分鐘是因為在學）
- 你不需要學怎麼修 bug，因為**這個 repo 你只有 1 個檔案、5 分鐘工作量**

> 💡 **這不是放棄，這是「砍掉重練比修慢炸還快」的工程智慧**。

### 操作示範 / 除錯回貼三段式貼紙（5 分鐘）

當你**真的**不知道怎麼辦（不只是 5 分鐘卡住、是 15 分鐘卡住）—— 你會去做兩件事：
1. 找助教 / 同學 / 講師
2. 把錯誤丟回 AI 問

**問題是：** 學員常常只貼一句「**它壞了**」「**不能用**」「**怪怪的**」—— 沒人能診斷。

**解法：** 用這個三段式格式：

> ```
> 【我做的事】
> [剛剛點了什麼、貼了什麼、做了哪一步]
> 
> 【我看到的】
> [螢幕上實際出現的訊息 / 截圖 / 錯誤碼]
> 
> 【我期待的】
> [原本以為應該會發生什麼]
> 
> 請問我下一步該做什麼？
> ```

**範例：**

> 【我做的事】
> 我在 github.dev 把 index.html 貼好程式碼，點 Source Control 的 Commit & Push，等了 5 分鐘。
> 
> 【我看到的】
> Pages 設定頁顯示「Your site is live at https://chen.github.io/my-clock/」，但我點進去看到「404 - There isn't a GitHub Pages site here.」
> 
> 【我期待的】
> 看到我的時鐘畫面。
> 
> 請問我下一步該做什麼？

**這樣貼，助教 / AI 在 30 秒內就能診斷。**（上面這個範例答案是「等久一點 + 強制重新整理 Cmd+Shift+R」）

> 💡 **把這個三段式存成手機備忘錄**，卡關時直接套。

### 動手做 / 學員自己跑一遍（30 分鐘）

現在輪到你了。

**任務：** 從 Claude Artifacts 產一個你想要的單檔 HTML（任何東西都行：個人介紹頁、時鐘、計時器、抽籤工具…），走完 5 步部署到 GitHub Pages。

**驗收門檻：**
- ✅ 你有一個 `https://{你的帳號}.github.io/{repo-name}/` 網址
- ✅ 用手機打開（不是電腦）也能正常顯示
- ✅ 把網址貼給旁邊同學，他能打開看到你的東西

**講師 / 助教 SOP（給講師看的）：**
- 助教 A 守在 GitHub 帳號 / 2FA / 信箱驗證關卡
- 助教 B 守在 github.dev 進不去、Pages 404 關卡（教重新整理 + 等 3 分鐘）
- 助教 C 守在 AI 產的程式碼有問題的關卡（教「除錯三段式」貼回 AI）
- 講師親自處理「Pages 啟用後 5 分鐘還是 404」這種需要決策的個案

> **記住口號：** 你不需要懂 GitHub，你只需要會點按鈕。

2. 助教分工列出，貼在黑板 / 投影邊：A 帳號 / B github.dev + Pages / C AI 程式碼
3. 把「除錯回貼三段式」印 A4 貼紙，發每人一張

**課中觀察：**
- Step 2「按 `.` 鍵」最容易卡（學員會直接打字「.」進 README、不是按鍵盤的點）
- Step 5「等 1~3 分鐘」最容易誤判（學員會在 30 秒後就開始焦慮）
- 80% 學員卡關超過 5 分鐘 → 講師當場示範「Delete repository 重做」復活術

**課後**：把今天學員實際拿到的 GitHub Pages 網址收集起來、放講義「同學作品」區作下次招生素材。
