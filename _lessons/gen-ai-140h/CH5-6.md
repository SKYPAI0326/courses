## 教學流程（Teaching Flow）

### 破題 / Hook（3 分鐘）

設計助理小薇用 Gemini Canvas 做了一個「3 頁個人作品集」，畫面超漂亮。她照 CH5-5 五步部署，結果打開網址——**版面全垮、圖片全破、按鈕沒反應**。

她沒做錯任何一步。問題是 Gemini 把網站做成了**四個檔案**：

```
index.html      ← 她只貼了這個
style.css       ← 沒貼 → 所以版面垮了
script.js       ← 沒貼 → 所以按鈕沒反應
assets/me.jpg   ← 沒貼 → 所以圖片破了
```

CH5-5 教的是「**單檔**就能部署」。但 Gemini、AI Studio 很愛把網站拆成多檔。傳統解法是學 git 一次 push 多檔——但那要懂 commit、branch、資料夾結構，**對不想學技術的你是地獄**。

這 1.5 小時教你另一條路，一句話總結：

> **「不要學怎麼傳多檔，要學怎麼叫 AI 把多檔變回一檔。」**

你會學會一段「整合單檔」提示詞。貼回去給 AI，它就把四個檔案重新打包成一個 `index.html`，你再回頭走 CH5-5 的五步。**改變 AI 產出的形態，比學 git 通用一萬倍**——這是元能力。

---

### 概念 / 為什麼 AI 會給你多檔（2 分鐘）

先理解「多檔不是 bug，是 AI 在模仿真實工程習慣」。

| 來源 | 通常 | 為什麼 |
|------|------|--------|
| Claude Artifacts | 多半單檔 | 設計成「一個 artifact 一個檔」 |
| ChatGPT Canvas | 多半單檔 | 同上 |
| **Gemini Canvas** | **常多檔** | 模仿工程師「CSS/JS 分開」的習慣 |
| **Google AI Studio** | **常多檔** | 同上，還常多一層 `assets/` |

**怎麼一眼看出「這是多檔」？**（沿用 CH5-5 的判準）

- 程式碼裡有 `<link rel="stylesheet" href="./style.css">` → 多檔訊號
- 程式碼裡有 `<script src="./script.js">` → 多檔訊號
- 程式碼裡有 `<img src="me.jpg">`、`./assets/...` → 多檔訊號
- 反之 `https://` 開頭的（公開 CDN）→ 不算，那是單檔 OK 的

看到任何 `./` 或本地檔名 = 「該整合了」。**不用看懂程式碼，看到 `./` 就動手。**

### 操作示範 / 「整合單檔」提示詞拆解 + 現場示範（30 分鐘）

這段是本單元核心。講師現場示範一次，學員現場跟做一次。

#### 核心：背下這段「整合單檔」提示詞

> **【整合單檔提示詞範本】**（課程素材包有「複製」按鈕，按了就進剪貼簿）
>
> 「請把你剛剛做的這個網站，**重新整理成單一 HTML 檔的版本**：
> - 所有 CSS 寫進 `<style>` 標籤
> - 所有 JavaScript 寫進 `<script>` 標籤
> - 圖片用 `https://placehold.co/` 或 `https://picsum.photos/` 連結代替本地檔案
> - 字型用 Google Fonts 的 CDN 連結，不要下載到本地
> - 如果有多頁（about / contact 等），改成單頁內用錨點切換
>
> 我要可以**複製整段就存成 `index.html` 直接用**的版本。請開始重做。」

**逐句為什麼（給學員一句話理解，不用深究）：**

| 那句要求 | 解決什麼 |
|---------|---------|
| CSS 寫進 `<style>` | 不用再傳 style.css |
| JS 寫進 `<script>` | 不用再傳 script.js |
| 圖片用線上連結 | 不用再傳 assets 圖片 |
| 字型用 CDN | 不用下載字型檔 |
| 多頁變單頁錨點 | 不用傳 about.html / contact.html |

一句話：**把所有散出去的東西全部塞回一個檔。**

#### 現場示範動線（講師做給學員看）

1. 用 Gemini Canvas 做一個多檔網站（例：「做一個 2 頁的咖啡店網站，共用 CSS」）
2. 觀察：Gemini 給了 `index.html` + `style.css`（多檔）
3. **貼上整合單檔提示詞** → Gemini 重做 → 給出一個全包的 `index.html`
4. 複製整段 → **回頭走 CH5-5 五步部署**
5. 打開網址 → 版面、圖片、互動全正常 🎉

> **最低完成線（傻瓜路徑，人人能過）：** 你只要把多檔產出貼上整合提示詞、拿到「一段就是全部」的單檔，就算過。**真的部署上 GitHub 是加分**——沒環境的同學交「整合前 vs 整合後」對照截圖也算完成。

### 卡關地圖 / 兩條備援逃生路徑 + 常見整合失敗（15 分鐘）

整合提示詞**大多數時候會成功**。但偶爾 AI 整合不乾淨，這時不要硬修，走備援。

#### 備援 A —— GitHub 網頁 Upload files（拖拉 ZIP，3 步）

當整合反覆失敗、或檔案就是多到不想整合：

> 1. 把 AI 產出的整個資料夾，在電腦上**壓成 ZIP**（或直接選全部檔案）
> 2. 進你的 repo 頁面 → 點「**Add file**」→「**Upload files**」
> 3. 把檔案/資料夾**整個拖進去** → 下面打一行字 → **Commit changes**

**不需要 Terminal、不需要 github.dev、不需要懂 git。** 拖進去就好。

#### 備援 B —— 請 AI 一檔一檔給我（github.dev 手建）

當你想保留多檔結構：

> 回 AI 丟這句：「請列出這個專案的所有檔案，每個檔案前面用 markdown ``` 包住檔名、後面接整段內容。我會一個一個建檔貼上去。」

拿到後在 github.dev 按「+」一個一個建檔貼進去（沿用 CH5-5 Step 3 的建檔手法）。

#### 三個常見整合失敗 + 一句話修法

| 症狀 | 修法（回貼給 AI） |
|------|------|
| 整合後圖片破圖（placehold.co 404） | 「請把圖片連結改用 `https://picsum.photos/600/400`」 |
| 整合後**還是有** `<link href="./style.css">` | 「你還留著 style.css 連結，請把那個 CSS 內容也直接內嵌進 `<style>`」 |
| 多頁變單頁後切換很突兀 | 「請幫切換加上 CSS transition 讓它平滑」 |

> **鐵律（沿用 CH5-5）：** 任何卡關**超過 5 分鐘** → 不要 debug、直接走備援 A 拖拉 ZIP。把東西弄上線比弄漂亮重要。

### 動手做 / 學員自己跑一遍（30 分鐘）

**任務：** 用 Gemini Canvas（或 AI Studio）做一個**至少 2 檔**的小網站，用整合單檔提示詞壓成 1 檔，走 CH5-5 部署。

**驗收門檻：**
- ✅ 最低完成線：拿到「一段就是全部」的單檔版本（可交整合前後對照截圖）
- ✅ 加分：實際部署到 `https://{你}.github.io/{repo}/`、版面/圖片/互動正常
- ✅ 若整合失敗 → 改走備援 A 拖拉 ZIP 成功，也算完成（並寫一句「我為什麼改走備援」）

**講師 / 助教 SOP：**
- 助教 A 守「整合提示詞貼了但 AI 還是給多檔」→ 教追問「請把 style.css 內容也內嵌」
- 助教 B 守「整合成功但部署破圖」→ 教換 picsum.photos
- 助教 C 守「乾脆走備援 A」→ 示範 Upload files 拖拉
- 講師處理「ZIP 也傳不上去」這種需要決策的個案

---

### 收尾 / 你現在會做什麼（5 分鐘）

走完這節，你多了一個**形態轉換**的元能力：

- AI 給多檔 → 你會叫它變單檔
- 變不出來 → 你會拖 ZIP 或一檔一檔貼
- 你**還是沒學 git**，也**不需要**

> **記住口號：** 不要學怎麼傳多檔，要學怎麼叫 AI 把多檔變回一檔。

下一個 PRAC5-16 就是拿 Gemini Canvas 的多檔產出，完整跑一次「整合 → 部署」。PRAC5-17（可分享版 Vercel proxy）等講師備好範本 repo 再開。

## 自學包（1 小時）

**任務 A：** 請 Gemini Canvas 做一個「3 頁餐廳網站」（首頁 / 菜單 / 訂位），用整合單檔提示詞壓成 1 檔並部署。
**任務 B：** 故意叫 AI「用 style.css 分開寫」做一個網站，再用整合提示詞救回單檔，體會前後差別。

**完成標準：** 任務 A 拿到一個能打開、三區塊用錨點切換正常的 GitHub Pages 網址。

**卡關：** 套「除錯回貼三段式」（CH5-5 教過）貼回 AI。不要 google（google 出來都是給工程師的 git 教學，會更亂）。

## 講師端準備 SOP

**開課前：**
1. 先用 Gemini Canvas 親手產一個多檔網站、跑一次整合提示詞，錄成示範影片
2. 準備一個「故意多檔」的範例（含 style.css + script.js + 一張本地圖）當現場示範素材
3. 確認課程素材包的「整合單檔提示詞」複製按鈕可用

**課中觀察：**
- 學員最常卡：整合後 AI 仍留一行 `<link href="./style.css">`（要教追問內嵌）
- 第二常卡：placehold.co 偶發 404（直接教換 picsum.photos）
- 約 20% 學員整合反覆失敗 → 講師當場示範備援 A 拖拉 ZIP，強調「上線比漂亮重要」

**課後：** 收集學員「整合前多檔 vs 整合後單檔」對照，作下次教學示範素材。
