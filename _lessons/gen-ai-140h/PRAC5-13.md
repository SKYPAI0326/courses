## 教學流程（Teaching Flow）

### 你會練到的能力（明示開頭，與 PRAC5-1 差異化）

> **本單元練「部署閉環」**：一次提示詞 → AI 一次產出 → 部署成可分享連結，**1 小時內完成**。
> 
> 跟 **PRAC5-1 個人作品頁** 不同的是：PRAC5-1 練「**需求迭代**」（多輪對話打磨需求、調整版面、迭代細節，是慢工出細活）；**PRAC5-13 練「部署閉環」**（最短路徑、最快有成品、馬上能丟連結給人看）。
> 
> **學完今天，你會在 60 分鐘內擁有一個「`https://{你的帳號}.github.io/{repo}`」的個人介紹網址。**

---

### 破題 / Hook（5 分鐘）

行政小陳要轉職找新工作了。下週要面試一家數位行銷公司。

履歷投出去後，她想到一件事：「**面試官 google 我的名字，會看到什麼？**」

答案是：LinkedIn 半成品、十年前的部落格、噗浪上的廢文、一張高中畢業照。

她想做一個「**個人介紹單頁**」放最新的自己 —— 但她不會寫程式碼。

平常做 PowerPoint 沒問題，但 PowerPoint 不能變網頁；用免費 Notion 模板可以，但**那不像「自己做的」**；用 Wix / Webflow 又要學一套新介面。

她在課堂上學過一個提示詞 —— **⑦ 一鍵生個人單頁網站**。

3 分鐘把資料貼給 Gemini，3 分鐘走 CH5-5 五步上 GitHub Pages，**6 分鐘她有了 `https://chen-hr.github.io/about-me/`**。

面試官打開來，看到一頁簡潔現代、米白底、手機打開也漂亮的個人介紹。**她錄取了**。

這 1 小時，我們把⑦這張提示詞卡做成你的第一個「**60 分鐘交付成品**」。

---

### 概念 / 提示詞卡（3 分鐘）

> 💡 **這張卡來自 `fun-apps.html` ⑦，是課堂彩蛋中學員反應最好的一張**。今天我們不只貼提示詞，還要走完整部署流程。

#### 提示詞全文（複製貼上不要改）

```
請用單一 HTML 檔做一份「個人介紹單頁」，包含：姓名與一句話自我定位、關於我、3 項技能、3 個作品占位區、聯絡方式。風格簡潔現代、米白底、手機可讀。我先貼上我的資料你再產生程式碼，請先回我「請貼上你的資料」。
```

**為什麼這張提示詞會贏？**
- **限定單檔 HTML** → 直接走 CH5-5 主路徑（不會卡多檔場景）
- **「請先回我『請貼上你的資料』」** → AI 變成填空主導者、你只負責貼資料
- **米白底、手機可讀** → 預設好看 + 預設手機友善（不用學 RWD）

**今天的演練 AI：** Gemini Canvas（最常用）、Claude Artifacts（最穩）、ChatGPT Canvas 都可以。**任三選一**，貼提示詞、看回應、貼資料。

### 操作示範 / AI 對話完整示範（15 分鐘）

我們用「**行政小陳的轉職單頁**」當主線示範。**講師現場跑一次**。

#### 對話流程

**[學員]** （複製⑦提示詞全文，貼進 Gemini Canvas）

> 請用單一 HTML 檔做一份「個人介紹單頁」，包含：姓名與一句話自我定位、關於我、3 項技能、3 個作品占位區、聯絡方式。風格簡潔現代、米白底、手機可讀。我先貼上我的資料你再產生程式碼，請先回我「請貼上你的資料」。

**[Gemini Canvas]**

> 好的，請貼上你的資料：
> 1. 姓名與一句話自我定位
> 2. 關於我（背景、興趣、目前狀態）
> 3. 3 項技能（每項一句話描述）
> 4. 3 個作品占位區（標題 + 一句話描述，可先用「待補」）
> 5. 聯絡方式（Email / LinkedIn / 社群連結）

> 💡 **第一個關鍵**：AI 真的會「先回問」、不會馬上產 code。這就是「先回我」式提示詞的威力。

**[學員]**（貼自己的資料）

> 1. 陳怡君 / 從行政到數位行銷的轉場者
> 2. 5 年外商行政專員、2024 修完 Google Digital Marketing 證照、目前找品牌行銷新人職缺
> 3. 三項技能：
>    - 跨部門溝通協作（過去主辦 12 場跨國視訊會議）
>    - Google Ads 操作（個人作品集累積 60+ 廣告組合）
>    - AI 工具應用（ChatGPT / Gemini / NotebookLM 日常工作流）
> 4. 三個作品：
>    - 「咖啡店週末活動」Meta 廣告投放案（提案中）
>    - 「個人轉職 90 天計畫」Notion 公開頁
>    - 「行銷術語小抄」（自己用 Gemini 整理的）
> 5. Email: chen.yj@example.com / LinkedIn: linkedin.com/in/chenyj

**[Gemini Canvas]**（產出完整單檔 HTML，約 200 行）

```html
<!DOCTYPE html>
<html lang="zh-TW">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>陳怡君 — 從行政到數位行銷的轉場者</title>
  <style>
    /* 米白底 + 簡潔現代風 + 手機友善 */
    body { background: #f5f3ee; ... }
    /* ...省略，AI 會自動完整生成... */
  </style>
</head>
<body>
  <header>
    <h1>陳怡君</h1>
    <p class="tagline">從行政到數位行銷的轉場者</p>
  </header>
  <section id="about">
    <h2>關於我</h2>
    ...
  </section>
  ...
</body>
</html>
```

> 💡 **第二個關鍵**：Gemini 會自動把 CSS / HTML / JS 全部寫在同一個 `<style>` 和 `<script>` 標籤裡 → **這就是「單檔 HTML」**，走 CH5-5 沒問題。

#### 預期產出長相

| 桌機（1440px） | 手機（375px） |
|---------------|---------------|
| 米白底、置中容器、襯線標題、3 欄技能、3 卡片作品、聯絡按鈕 | 同樣米白、單欄堆疊、字體放大、卡片變單列 |

**現場示範要展示：** Gemini Canvas 右上「Preview」按鈕點下去 → 預覽視窗有完整網頁 → 把瀏覽器拉窄、看是不是真的手機友善。

### 操作示範 / 部署到 GitHub Pages（10 分鐘）

> 🔁 **這部分回頭走 CH5-5 五步**。不重複教，只給你今天的對應動作。

| Step | 動作 |
|------|------|
| 1 | 開 github.com → New repository → 命名 `about-me`（或你的英文名 `chen-yj`） → Public + 勾 README → Create |
| 2 | 在 repo 頁面按 `.` 鍵進 github.dev |
| 3 | 左側新建 `index.html` → **把 Gemini Canvas 給的整段 HTML 貼進去** |
| 4 | Source Control → stage → commit message `init` → Commit & Push |
| 5 | 回 repo → Settings → Pages → Branch 選 main → Save → **等 1~3 分鐘** |

最後拿到 `https://{你的帳號}.github.io/about-me/`。

> 💡 **沒做過 CH5-5？** 回去看 CH5-5 教案再來。本單元預設你已經跟著 CH5-5 跑過一次。

### 卡關地圖 / 常見錯誤（10 分鐘）

#### 1. AI 把照片佔位區寫成 `<img src="photo.jpg">` 但你沒上傳照片

**症狀：** 部署後打開網頁，照片區是一個破損的「X」圖示。

**原因：** AI 隨手寫的 `photo.jpg` 是「假檔名」，但你只傳了 `index.html`，沒傳 photo.jpg。

**修法（兩個選擇）：**

**選擇 A：用免費網路圖片代替**

回 Gemini 問：

> 「請把所有 `photo.jpg` 之類的本地圖片連結，**改成 `https://placehold.co/400x400` 之類的網路 placeholder 圖**。」

Gemini 會把所有 `<img src="photo.jpg">` 改成 `<img src="https://placehold.co/400x400">`，重貼回 github.dev → commit → 推上去。

**選擇 B：先把照片區拿掉**

回 Gemini 問：

> 「請先把 3 個作品占位區的圖片**改成純文字框**（深灰底、白字寫『作品 1』『作品 2』『作品 3』），不要用圖片。」

兩個選擇都 OK，看你想花多少時間找圖。

> 💡 **未來如果想換真實作品圖**：把作品圖上傳到 Imgur / 公開 Google Drive、拿到 `https://...` 連結，回 Gemini 說「請把作品 1 的圖改成 `https://i.imgur.com/xxxxx.jpg`」。**不要把圖檔上傳到 GitHub repo**（會搞複雜）。

---

#### 2. AI 給的字型用 Google Fonts 但你看不到效果

**症狀：** 預覽看起來很美，部署後字型變預設 Times New Roman。

**原因：** 通常是 `<link href="https://fonts.googleapis.com/...">` 被 AI 漏寫進去。

**修法：** 回 Gemini 問「請把字型 link 加進 `<head>` 區、確認字型在所有裝置都能載入」。重貼 + 部署。

---

#### 3. 手機打開字太大、跑版

**症狀：** 桌機看正常、手機看每個字都頂到邊。

**原因：** AI 寫的 RWD 沒做透。

**修法：** 回 Gemini 問「請優化手機版面：標題不要超過 2 行、padding 縮小到 16px、字級在手機降到 14px」。重貼 + 部署。

---

#### 4. 改完忘了重新整理（卡關 5 分鐘的 #1 元兇）

**症狀：** 在 github.dev 改完、commit 完，**打開 Pages 網址還是舊版**。

**原因：** 瀏覽器有快取，看到的是舊內容。

**修法：** **Cmd+Shift+R**（Mac）/ **Ctrl+Shift+R**（Windows）強制重新整理。

### 動手做 / 學員自己跑一遍（15 分鐘）

現在輪到你。

**任務：** 用⑦提示詞 + 你自己的資料 + CH5-5 五步，**做出一個你自己的個人介紹頁**。

**資料準備（5 分鐘想好再貼）：**
1. 姓名 + 一句話定位（你是誰、做什麼、想去哪）
2. 關於我（一段 50-100 字）
3. 3 項技能（每項 1 句話）
4. 3 個作品（可填「待補 + 一句話」）
5. 聯絡方式（Email 或 LinkedIn）

**驗收門檻（要拿到 1 個成品）：**
- ✅ 拿到 `https://{你的帳號}.github.io/{repo}/` 網址
- ✅ 手機打開字級正常、不跑版
- ✅ 把網址貼給旁邊同學或助教，他能打開看到你的頁面

**講師建議：** 學員若卡 5 分鐘，用 CH5-5 的「除錯回貼三段式」貼回 AI 問。

## 自學包（1 小時）

### 變化版練習（三選一或全做）

**變化版 A — 夜間風（深色底版）**

在原提示詞後加：

> 「請把風格改成**夜間風**：深灰底（#1a1a1a）、淺色文字、霓虹紫色（#a855f7）強調色、字型用襯線 + 等寬字混搭。」

**變化版 B — 印刷風（雜誌封面感）**

> 「請把風格改成**印刷雜誌風**：米黃紙質底（#f4e9d0）、襯線標題（Playfair Display）、黑字、加細格線分隔、像 The Economist 雜誌封面排版。」

**變化版 C — 賽博龐克風**

> 「請把風格改成**賽博龐克風**：黑底、霓虹粉紫漸層、發光邊框、字型用未來感無襯線、像 Cyberpunk 2077 主機版選單。」

**做法：** 同一份個人資料、開三個 Gemini Canvas 分頁、各跑一個變化版、各部署一個 repo（`about-me-night` / `about-me-print` / `about-me-cyber`）。

**最後挑一個最喜歡的版本變成「正式版」 → 把網址寫進 LinkedIn / 履歷。**

---

### 自學完成標準

- ✅ 至少完成 1 個變化版部署
- ✅ 三個版本（包含主版）並排對比 → 寫一句「我最喜歡哪個版本、為什麼」筆記
- ✅ 把最終版網址分享到課程群組「同學作品」討論串
