## 教學流程（Teaching Flow）

### 你會練到的能力（明示開頭）

- 故意讓 Gemini 產出多檔網站，再用一段提示詞把它壓回單檔（CH5-6 的實戰）
- 把整合後的單檔走 CH5-5 五步部署成可分享網址
- 整合失敗時，冷靜改走備援 A（拖拉 ZIP）而不是卡在那硬修

這一頁是 CH5-5（部署）+ CH5-6（整合）的**綜合演練**。跑完你就有一條「AI 多檔產出 → 上線」的完整肌肉記憶。

---

### 破題 / Hook（3 分鐘）

CH5-6 教了「整合單檔」提示詞，但你可能還沒真的拿一個**多檔**產出練過。這一頁就是要你**故意製造多檔、再親手壓成單檔**。

為什麼故意？因為 Gemini Canvas、AI Studio 在真實使用時**很常**給你多檔——與其上線時才踩雷，不如現在先在演練裡踩一次、把救援流程練熟。

> **這節口號：** 先在練習裡製造麻煩，真實上線就不怕。

---

### 概念 / 提示詞卡（5 分鐘）

#### 提示詞①：故意做一個多檔網站（複製貼上）

> 「請用 **Gemini Canvas** 幫我做一個**3 頁個人作品集**網站：
> - 首頁（自我介紹）、作品頁、關於我頁
> - 三頁**共用一份 CSS**
> - 手機可讀、米白色底、簡約風
> 請開始製作。」

Gemini 很可能把它做成 `index.html` + `style.css`（+ 各頁）——**這正是我們要的多檔**。

#### 提示詞②：整合單檔（沿用 CH5-6，課程素材包有複製按鈕）

> 「請把你剛剛做的這個網站，**重新整理成單一 HTML 檔的版本**：所有 CSS 寫進 `<style>`、所有 JS 寫進 `<script>`、圖片用 `https://picsum.photos/` 連結、字型用 Google Fonts CDN、多頁改成單頁錨點切換。我要複製整段就能存成 `index.html` 直接用的版本。請開始重做。」

---

### 操作示範 / 觀察多檔 + 整合動線（25 分鐘）

講師現場示範一次、學員現場跟做一次。

**完整動線（5 步）：**

1. **產出**：貼提示詞① → Gemini Canvas 做出多檔作品集
2. **觀察**：打開看到 `<link rel="stylesheet" href="./style.css">` 或分頁檔案 → 確認「這是多檔」
3. **整合**：貼提示詞② → Gemini 重做 → 給出一個全包的 `index.html`
4. **部署**：複製整段 → 走 **CH5-5 五步**（開 repo → 按 `.` → 建 index.html 貼上 → commit & push → 啟用 Pages）
5. **驗證**：打開 `https://{你}.github.io/{repo}/` → 三個區塊用錨點切換、CSS 正常、圖片不破

> **最低完成線（傻瓜路徑）：** 拿到整合後的單檔即算過；實際部署成功是加分；沒環境者交「多檔 vs 單檔」對照截圖。

---

### 操作示範 / 「失敗就走備援」演習（15 分鐘）

這段刻意演練「整合不順」時怎麼辦，免得真實上線慌。

**情境 A — 整合後檔案太大 / 整合反覆失敗：**
→ 走 **CH5-6 備援 A**：把 Gemini 給的整個資料夾壓 ZIP → repo 頁「Add file → Upload files」→ 拖進去 → Commit。3 步上線，不碰 Terminal。

**情境 B — 整合後樣式跑掉：**
→ 回貼 AI：「目前畫面是 ___（描述你看到的），我預期是 ___（描述你要的），請修。」用「描述現況 + 描述期望」讓 AI 自己改。

> **演習要求：** 每位學員至少**主動走一次備援 A**（就算整合成功了也練一次拖拉 ZIP），把救援動作變肌肉記憶。

---

### 卡關地圖 / 常見錯誤（10 分鐘）

#### 1. 整合後圖片變 404（用了 placehold.com）
**修法：** 回貼「請把圖片連結改用 `https://picsum.photos/600/400`」。picsum 比較穩。

#### 2. 多頁變單頁錨點後，切換很生硬沒動畫
**修法：** 回貼「請幫區塊切換加上 CSS transition 讓它平滑滑動」。

#### 3. Gemini 給的單檔**仍含** `<link rel="stylesheet" href="./style.css">`
**修法：** 回貼「你還留著 style.css 的連結，請把那份 CSS 內容**直接內嵌**進 `<style>` 標籤，不要外連」。這是最常見的「整合不乾淨」。

#### 4. 部署後看到舊版（改完忘了 commit）
**修法：** 沿用 CH5-5——回 github.dev → Source Control → stage + commit & push → 強制重新整理（Cmd+Shift+R）。

---

### 動手做 / 學員自己跑一遍（25 分鐘）

**任務：** 用 Gemini Canvas（或 AI Studio）做一個你自己的多檔小網站（作品集 / 活動頁 / 商店頁皆可），整合成單檔並部署。

**驗收門檻：**
- ✅ 最低完成線：拿到整合後的單檔版本
- ✅ 加分：部署到 GitHub Pages、錨點切換 + 圖片 + CSS 都正常
- ✅ 至少示範一次備援 A 拖拉 ZIP（截圖為證）

**講師 / 助教 SOP：**
- 助教 A 守「Gemini 整合後仍外連 style.css」→ 教追問內嵌
- 助教 B 守「部署破圖」→ 教換 picsum.photos
- 助教 C 守備援 A 拖拉 ZIP 動作
- 講師處理「Gemini 怎麼弄都給多檔」的個案（直接判走備援 A）

---

### 收尾 / 你現在會做什麼（5 分鐘）

跑完這頁你掌握了一條完整動線：**AI 多檔產出 → 整合單檔 → 上線**，而且備援也練過。

- 整合得出來 → 走 CH5-5 部署
- 整合不出來 → 拖 ZIP 上線
- 你**依然沒學 git**，但任何 AI 產出都能變成可分享網址

> **記住：** 你練的不是 git，是「用提示詞改變 AI 產出形態」——這個能力換到任何 AI 工具都通用。

想做「可以分享給別人用、KEY 不外洩」的工具？那是 PRAC5-17（Vercel proxy 可分享版），等講師備好範本 repo 再開。

## 自學包（1.5 小時）

### 變化版練習（三選一或全做）
1. **多頁店家網站**：首頁 + 菜單 + 聯絡，整合單檔部署
2. **活動報名頁**：主視覺 + 流程 + 報名表單（表單先做樣式即可），整合部署
3. **作品集 v2**：在課堂作品上加一個「作品 lightbox 放大」互動，整合時注意 JS 也要內嵌

### 自學完成標準
任選一個拿到能打開、切換正常的 GitHub Pages 網址；過程中至少遇到一次整合不乾淨並用追問修好。

## 講師提示

- 先親手用 Gemini Canvas 跑一次提示詞①②，確認當前 Gemini 仍會產多檔（平台會變，課前複查）
- 準備一個「整合失敗」的真實案例截圖，讓學員看到備援 A 不是恥辱是常態
- 課後收集學員作品網址 + 「整合前後對照」，作招生與下次示範素材
