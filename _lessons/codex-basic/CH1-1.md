## 教學流程（Teaching Flow）

### 破題 / Hook（1.5 min）

你是弄一下工作室的行政會計。6 月底，月結要做。

桌面上一包資料：6 份各班…不對，各部門傳來的報帳單，每個人命名方式都不一樣（`報帳.docx`、`六月-業務部.xlsx`、`費用單(最終版2).docx`…）；兩份不同來源的供應商清單，裡面有重複的廠商；一份交易明細，有幾格空白、有一筆金額大得不尋常。

以前你會花一個下午：手動改檔名、一份份打開複製貼上、肉眼抓重複。今天先不教你怎麼做——**先看 Codex 怎麼在 90 秒做完這件事**。看完你再決定值不值得學。

### 概念 / Concepts（1.5 min）

- **Codex** — OpenAI 推出的 AI 助理 App。和 ChatGPT 最大的不同：它不只「回答」，還會**動手做事**——讀檔、改檔、產生新檔案、寫文件、做表格。
- **與 ChatGPT 的三個關鍵差異** —
  1. **以你指定的資料夾為主要工作範圍**：它能讀取那個資料夾的真實檔案，不是你貼一段文字它才知道。（實際能讀寫的範圍，取決於你設定的權限模式、有無啟用外掛與 Computer Use，不是絕對隔離。）
  2. **會產出檔案，不是只給你一段字**：它會真的在你電腦裡生出 Excel、Word、PDF。
  3. **能照你給的規則重複做**：規則寫一次，下個月同一件事一句話就重跑。
- **責任邊界（這條全課回扣）** — Codex **不知道自己做不到**。規則沒講清楚，它會把錯的東西「很順地」做出來，而且一路狂奔不會停。所以一句話先記著：**凡是它產出的，最後都要人看過**——做得順不等於做得對。

### 操作示範 / Demo（成果先行，1.5 min・講師示範）

**主線：播放預錄影片（90 秒，建議優先）**

播放預先錄好的 demo 影片——一個已設定好的 Codex 專案（弄一下工作室月結），只給一句話：

> 把這個資料夾裡的報帳單，檔名統一成「部門-月份-報帳單」格式，再彙整成一張 Excel。

影片呈現：6 個亂命名檔案 → 全部改成整齊格式；產生一張 Excel 彙整表。

**備線：現場最小版（網路 / 額度無問題時可現跑，限縮範圍）**

僅用 **3 個已知格式的 .docx 測試檔**，送出：

> 把這 3 個報帳單的檔名改成「部門-月份-報帳單」格式，並產一份處理日誌。

學員看到：3 個檔名統一改好、產出一份 `處理日誌.txt`（記錄改了哪幾個檔）。
Excel 彙整**不在本單元現場跑**，留到 CH2-2 學員親手做。

---

講師收束一句：**你今天接下來學的每一個「設定」，就是為了讓 Codex 能像剛剛這樣，替你把事做完——而且做對。**

### 觀察單 / Watch（0.5 min・不繳交）

看 demo 時，請你心裡記三件事（不用寫下來給誰，是給你自己建立判斷）：

1. 它是**真的改了**你的檔，不是嘴上說說。
2. 它有沒有**自己亂補**不存在的資料？（後面 CH2 會專門練怎麼防）
3. 如果這是**真的帳**，哪一步你會想先攔下來、自己核一下再用？

然後寫一句話：**你最想讓 Codex 幫你做掉的一件月結雜務是什麼？**（這句之後每個單元都會用得到。）

### 檢核 / Verification（口頭・認知）

學會的標準（不必上機）：

- 你能用自己的話講出 **Codex 跟 ChatGPT 的 3 個差異**。
- 你能指出剛剛 demo 裡，**哪一步是「要人把關」的**（例：採用 Excel 前要先抽查金額）。

兩件都答得出來，就可以進 CH1-2 把 Codex 裝起來。

## 動手練習題（Hands-on Exercise）

本單元無上機實作（學員尚未安裝 Codex）。以「觀察＋一句話」替代：

**題目**：看完成果先行 demo，寫下**一件**你每個月都要做、最想丟給 Codex 的月結／行政雜務。

**預期成果**：一句具體的話。好：「把 3 個部門的費用單併成一張、抓出重複的發票號」。不好：「做帳」（太籠統）。

**完成標準**（self-check）：
- [ ] 寫的是一件**具體、會重複**的雜務（不是一次性任務）
- [ ] 講得出這件事**現在**手動要花多久
- [ ] 講得出這件事裡**哪一步一定要你自己核**（不能全丟 AI）

## 常見錯誤 3 條（Common Pitfalls）

> 本單元學員尚未上機，這三條是「認知誤區」，先破除，避免後面踩雷。

1. **錯誤現象**：把 Codex 當成「換皮的 ChatGPT」，以為它只會回字，於是把要測試的檔案跟正式月結檔丟在同一個資料夾給它玩。
   **原因**：沒意識到 Codex 會**真的改／覆寫**資料夾裡的檔案。
   **解法**：正式檔永遠留一份副本不動；練習、測試用獨立資料夾。這也是 CH1-3 建「專案資料夾」要強調的。

2. **錯誤現象**：看到 demo 一句話就生出 Excel，覺得「太強了，以後直接交出去」，跳過覆核。
   **原因**：把「做得順」當成「做得對」。Codex 文字流暢、表格漂亮，不代表金額對、沒漏列。
   **解法**：建立反射——**它產出、你覆核**。金額抽查、總額對回來源，是你的責任，不是它的。

3. **錯誤現象**：聽到「AI 助理、會寫程式」就先卻步，覺得「我不會 coding 用不了」。
   **原因**：誤以為要懂技術／打指令。
   **解法**：本課全程用 Codex 桌面 App、講中文指令即可，不碰終端機、不寫一行程式。

## 檢核題 2 條（Quiz）

**Q1（概念驗證）**：Codex 和 ChatGPT 最關鍵的差異是哪一個？
- [ ] A. Codex 回答速度比較快
- [ ] B. Codex 會在你指定的資料夾裡動手改檔、產出檔案 ←（正確答案）
- [ ] C. Codex 完全免費、ChatGPT 要付費
- [ ] D. Codex 中文講得比較好

**Q2（應用驗證）**：demo 裡 Codex 把 6 份報帳單彙整成一張 Excel。如果這是真的月結帳，你拿去用之前，**哪一步一定要先自己把關**？
- 預期答案要點：
  1. **抽查金額／分類**是否正確（它可能引用錯範圍、分類錯）
  2. 確認它**沒有自己亂補**不存在的資料（缺值該停而不是硬填）
  3. **總額對得回**原始來源（沒漏列、沒重複加總）
