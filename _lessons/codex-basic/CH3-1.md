## 教學流程（Teaching Flow）

### 破題 / Hook

CH2 整理好了。6 份報帳單命名統一、彙整 Excel 做出來、供應商重複項標好了。

下個月呢？7 月那批報帳單送進來，你要重新開對話，重新跟 Codex 說：「幫我把檔名改成部門-月份-報帳單格式、彙整進那張 Excel、重複發票號碼標出來……」一整段交代，再說一遍。

**做完這個單元，你只需打這一句：**

> 幫我跑月結整理

Codex 就會執行完整流程，不需要再說明步驟。

**本單元你會做出兩個具體成果**：
1. 一個**「月結整理」專案 Skill**——只在月結專案裡生效，封裝 CH2-1/CH2-2 的完整流程
2. 一個**「表格排版」全域 Skill**——跨所有專案都能叫用，讓 Word 表格格式不用每次重新說

做完後，這兩個 Skill 都能在 Codex 的「外掛程式 → 技能」頁看到、管理。

---

### 概念 / Concepts

#### AGENTS.md vs Skill：何時用哪個

先搞清楚這兩個各自適合放什麼，再決定該封裝哪些東西。

**AGENTS.md（操作守則）**

每次開新對話，Codex 都會把 AGENTS.md 整份讀入一次。適合放「每次都需要知道的背景」：

- 公司名稱、你的職稱、部門
- 不得補值的底線規則
- 輸出格式的通用要求（例：日期欄一律 YYYY-MM-DD）

**不要放在 AGENTS.md 的**：一次性的 SOP、按需才用的流程說明。塞太多會讓 AGENTS.md 變成每次對話都要消化一大包背景、占掉 Codex 的可用思考空間，讓它在執行實際工作前就先「疲勞」了。

**Skill（技能）**

一個 Skill 是一個**資料夾**，資料夾內必含 `SKILL.md`（主說明），可再放 `scripts/`、`references/`、`assets/` 等子資源。打開 `SKILL.md`，裡面就是人讀得懂的中文步驟說明，不是程式碼。

Skill 的設計是 **progressive disclosure**：你沒有呼叫它時，Codex 完全不載入它的內容；你呼叫它的瞬間，Codex 才讀取完整 `SKILL.md` 並執行。這意味著：

- 你可以把一整套 SOP 的細節都寫進 Skill，Codex 也不會在你做其他事時被這些細節干擾
- 平時 context 空間留給當前工作，不被你還沒要用的 SOP 佔走
- 可以在「外掛程式 → 技能」頁觀察到這個行為：停用一個 Skill，自動觸發就停了，但 Skill 設定還在

這個機制就是 AGENTS.md 做不到的事：AGENTS.md 沒有「按需載入」——它永遠全部進 context。

#### 決策表：遇到新規則，放哪裡？

| 這條規則的使用頻率 | 適用範圍 | 放哪裡 |
|-------------------|---------|--------|
| 每次開對話都需要 | 這個專案 | AGENTS.md（專案資料夾內） |
| 按需叫用 | 這個專案特有流程 | **專案層級 Skill** |
| 按需叫用 | 跨所有工作都通用 | **全域層級 Skill** |

#### 專案層級 vs 全域層級

**專案層級 Skill**：存在這個專案的資料夾內（`弄一下工作室_6月月結/.codex/skills/月結整理/`），只在這個專案的對話下才看得到。適合「這個專案特有的流程」。

**全域層級 Skill**：Codex App 全域位置管理，跨所有專案的對話都能叫用。適合「任何工作都會用到的格式或規範」。

做完後可到「外掛程式 → 技能」頁，層級欄會標「專案」或「全域」，讓你一眼確認。

#### 兩種觸發方式

**自動觸發**：你打的話和 Skill 的描述夠接近，Codex 自行判斷啟用。例如「幫我跑月結整理」——不需要加指令，Codex 比對描述後自動啟動。

**斜線手動觸發**：在輸入框打 `/`，從彈出選單指定 Skill。適合「需要精確控制用哪個 Skill、不想讓 Codex 自行判斷」的情況。

### 操作示範

#### 示範一：建立「月結整理」專案 Skill

打開「弄一下工作室_6月月結」專案，開一個新對話。在對話框輸入以下提示詞（可直接複製）：

> 請把我的月結整理流程封裝成一個專案層級的 Skill，命名為「月結整理」，步驟如下：
> 1. 把資料夾內的報帳單檔名統一改成「部門-月份-報帳單」格式
> 2. 把各份報帳單的資料彙整進「月結彙整.xlsx」
> 3. 標記重複的發票號碼（用複合鍵：發票字軌號 + 廠商統編 + 日期 + 金額）
> 4. 有缺值或異常金額的欄位標「待覆核」，不補值

Codex 完成後，可到專案資料夾確認：`弄一下工作室_6月月結/.codex/skills/月結整理/` 資料夾已建立，內含 `SKILL.md`。打開 `SKILL.md`，裡面就是中文步驟說明——確認它寫的和你剛才描述的步驟一致。

到「外掛程式 → 技能」頁，確認「月結整理」出現、層級顯示「專案」。

接著示範自動觸發，在輸入框輸入：

> 幫我跑月結整理

Codex 自動啟用 Skill，開始執行命名和彙整步驟。

> **把關提醒**：Skill 把流程固化了，第一次就要確認流程正確。若建立時描述的步驟有誤（例如欄位名稱與實際 Excel 不符），Codex 每次執行都會穩定照著錯誤走，而且輸出格式漂亮，表面上看不出來哪裡不對。
>
> 第一次觸發後的核對方式：
> - 逐列比對彙整 Excel 的資料是否來自所有 6 份報帳單（數量對）
> - 抽查至少 2 個金額欄位，核回原始來源
> - 確認重複發票標記列確實存在、且複合鍵欄位填對
>
> 核對沒問題，之後才讓它一句話批次跑。

---

#### 示範二：建立「表格排版」全域 Skill

切到任意對話（不需要在月結專案下）。在對話框輸入：

> 請建立一個全域 Skill，命名為「表格排版」，內容是：
> 製作 Word 表格時，一律遵守以下格式：
> - 標題列：深灰底（#404040）白字
> - 內容列：白底黑字
> - 欄寬等分
> - 字體：標楷體 12pt
> - 表格外框：粗線 1.5pt；內框：細線 0.5pt

完成後到「外掛程式 → 技能」頁，確認「表格排版」出現、層級顯示「全域」。

示範斜線手動觸發：在輸入框打 `/`，從彈出選單選「表格排版」，接著輸入：

> 幫我替志工名單做一份簽到表

確認 Codex 照「表格排版」Skill 的規格輸出，不是按它自己的預設格式。

> **把關提醒**：全域 Skill 影響所有專案的對話。若要針對單一專案修改格式規範，建專案層級 Skill，不要動全域的——動了全域，你其他所有工作的表格輸出都跟著變。

---

**Skill 管理補充**

「外掛程式 → 技能」頁每個 Skill 右上角有開關：暫時停用（Codex 停止自動觸發，但 Skill 設定保留）、重新啟用。點「解除安裝」可移除 Skill。

### 檢核 / Verification

能做到以下三件，代表本單元學會：

1. 不打開資料夾，說得出「月結整理 Skill 在哪裡找到」（外掛程式 → 技能頁，層級：專案）
2. 以下三個情境，說得出分別用哪個方案：
   - 「每次對話都要知道的公司名稱和月份格式」 → **AGENTS.md**
   - 「每月做一次的月結 SOP，不是每天都用」 → **專案 Skill**
   - 「所有工作都適用的 Word 表格排版格式」 → **全域 Skill**
3. 說得出 Skill 第一次觸發後要核對的三個點（數量對、金額抽查、標記欄位正確）

## 常見錯誤 3 條（Common Pitfalls）

1. **錯誤現象**：「幫我跑月結整理」送出後，Codex 回「請問你要整理哪些步驟？」——沒有觸發 Skill。
   **原因**：當前對話不在「弄一下工作室_6月月結」專案下。這個 Skill 是專案層級，Codex 在其他專案的對話裡看不到它。
   **解法**：確認左側面板顯示的是「弄一下工作室_6月月結」專案，不是點「新增對話」開的普通對話。若需要在任何專案都能用，改建全域 Skill；或在當前對話用斜線 `/月結整理` 手動觸發。

2. **錯誤現象**：Skill 建好第一次觸發，Codex 執行了但彙整 Excel 裡有欄位名稱對不上、金額不對。
   **原因**：建立 Skill 時描述的欄位名稱與實際 Excel 的欄名不符（例如說「金額」，但 Excel 欄名是「報帳金額」）。Skill 固化了這個錯誤，每次觸發都穩定地照著走，而且輸出格式整齊，表面上不容易察覺。
   **解法**：發現問題立即重新提示 Codex 更新 Skill 步驟，說明正確的欄位名稱。更新後觸發一次，確認輸出正確，再正式複用。第一次觸發後就核對，不要「先跑幾次看看」。

3. **錯誤現象**：把整份月結 SOP 塞進 AGENTS.md，之後 Codex 回覆偶爾漏掉某些步驟。
   **原因**：AGENTS.md 每次對話都整份載入。長的 SOP 和其他背景規則一起塞進去，Codex 的可用思考空間被背景資訊占走，執行時偶爾就會漏步驟。Skill 的 progressive disclosure 設計就是為了解決這個問題：SOP 的細節只在你呼叫時才進 context，其他時候不占空間。
   **解法**：AGENTS.md 只留「每次都需要的背景」（公司名、底線規則、通用格式要求）；按需才用的 SOP 改封裝成 Skill。兩邊各司其職。

## 檢核題 2 條（Quiz）

**Q1（概念驗證）**：和把流程寫進 AGENTS.md 相比，封裝成 Skill 最主要的優點是什麼？

- [ ] A. Skill 可以自動排程，AGENTS.md 不行
- [ ] B. Skill 平常不載入到對話，只有你呼叫時才讀取，不占 Codex 的思考空間——AGENTS.md 每次開對話都整份讀入 ←（正確答案）
- [ ] C. Skill 可以包含圖片，AGENTS.md 只能放文字
