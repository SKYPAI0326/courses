## 教學流程（Teaching Flow）

### 破題 / Hook（2 min）

月結資料整理好了——命名也統一、彙整 Excel 也做出來了。

下個月呢？又一批命名雜亂的報帳單送進來，你要重新開一個對話，重新跟 Codex 說一遍：「幫我把檔名改成部門-月份-報帳單格式、然後彙整進那張 Excel、重複的發票號碼標出來……」

**做完這個單元，你不用再重說那一段**。你只需打：「幫我跑月結整理。」Codex 就知道要做什麼。

這個功能叫 Skill（技能）。接下來 18 分鐘，你會做出兩個：一個只在「6 月月結」這個專案生效，一個全域都能用。

---

### 概念 / Concepts（4 min）

- **AGENTS.md（操作守則）** — 每次開新對話，Codex 都會把它讀一遍。適合放「每次都需要知道的背景」，例如公司名稱、你的職稱、不得補值的底線。**什麼不該放**：一次性或按需才用的 SOP；放太多會讓 AGENTS.md 變成每次都要消化的背景負擔。

- **Skill（技能）** — 把一套重複流程打包成一個資料夾（內含 `SKILL.md` 主說明，可再放 scripts/、references/ 等子資源）。**平常不載入到對話**，只有你呼叫時才讀取完整內容（progressive disclosure）。適合放「不是每次都用、但用到就要精準執行」的 SOP。

- **兩種層級的差別** —
  - **專案層級 Skill**：存在這個專案的資料夾裡，只在這個專案的對話中才看得到。適合「這個專案特有的流程」，例如月結整理。
  - **全域層級 Skill**：跨所有專案都能叫用。適合「任何工作都會用到的通用格式規範」，例如公文表格排版。

- **兩種觸發方式** —
  - **自動觸發**：你打的話和 Skill 的描述夠接近，Codex 自行判斷啟用。
  - **斜線手動觸發**：在輸入框打 `/`，從彈出選單指定要啟用哪個 Skill。需要精確控制時用這種。

- **「外掛程式 → 技能」管理頁** — 所有 Skill 都能在這裡看到。可以停用（暫時不要自動叫）、重新啟用、或解除安裝（刪掉這個 Skill）。

---

### 操作示範 / Demo（6 min・講師示範）

**示範一：建立「月結整理」專案 Skill**

打開「弄一下工作室_6月月結」專案，開一個新對話。輸入：

> 請把我這個月結整理流程封裝成一個專案層級的 Skill：
> 1. 把資料夾裡的報帳單檔名統一改成「部門-月份-報帳單」格式
> 2. 把各份報帳單的資料彙整進「月結彙整.xlsx」
> 3. 標記重複的發票號碼
> Skill 命名為「月結整理」

Codex 幾秒後完成。講師說明：「它現在在你的資料夾裡建了一個 `.codex/skills/月結整理/` 的資料夾，裡面有一份 `SKILL.md`——打開來看，就是一份中文步驟說明，不是程式碼。」

帶學員到「外掛程式 → 技能」頁確認「月結整理」已出現，層級顯示「專案」。

接著示範觸發：輸入

> 幫我跑月結整理

Codex 自動啟用 Skill，開始執行命名和彙整步驟。

> ⚠️ **把關提醒**：Skill 固化了流程，**第一次就要確認流程對**。如果你在建立 Skill 時描述的步驟有誤（例如欄位名稱講錯），Codex 之後每次執行都會穩定地照著錯誤走，而且你看到輸出可能還覺得「格式對就沒問題」。建好後第一次跑，務必逐步確認輸出是否符合預期，再正式交給它一句話批次執行。

---

**示範二：建立「表格排版」全域 Skill**

切到任意對話（不在月結專案也無妨）。輸入：

> 請建立一個全域的 Skill，命名為「表格排版」，內容是：
> 製作 Word 表格時，一律遵守以下格式——
> 標題列深灰底白字（#404040 / 白）、內容列白底黑字、欄寬等分、字體標楷體 12pt、
> 表格外框粗線 1.5pt 內框細線 0.5pt。

完成後，帶學員到「外掛程式 → 技能」頁，找到「表格排版」——層級顯示「全域」。

說明：「這個 Skill 不管你在哪個專案，打斜線都能叫出來用。」

示範斜線手動觸發：在輸入框打 `/`，彈出選單選「表格排版」，再輸入「幫我替志工名單做一份簽到表」，送出。

> ⚠️ **把關提醒**：全域 Skill 影響範圍廣，修改它之前要想清楚——你在六月月結這個專案調整的格式規範，可能同時影響你另一個採購專案的輸出。要針對單一專案修改，建專案層級 Skill，不要動全域的。

---

**管理補充：停用 / 重新啟用 / 解除安裝**

「外掛程式 → 技能」頁右上角有開關，可以把 Skill 暫時停用（讓 Codex 停止自動觸發，但保留設定）；不需要的 Skill 可以點左下角「解除安裝」移除。

---

### 動手 / Hands-on（4 min・跟做）

學員動手完成「月結整理」專案 Skill 的建立與觸發：

**步驟 1**：打開自己的「弄一下工作室_6月月結」專案，開新對話。

**步驟 2**：輸入以下提示詞（可依實際情況微調）：

> 請把月結整理流程封裝成一個專案 Skill，步驟如下：
> 1. 把資料夾內報帳單的檔名改成「部門-月份-報帳單」格式
> 2. 把資料彙整進「月結彙整.xlsx」
> 3. 標出重複發票號碼
> 命名為「月結整理」

**步驟 3**：Skill 建立完成後，到「外掛程式 → 技能」頁確認它出現、層級顯示「專案」。

**步驟 4**：新開一個對話（同一個月結專案），輸入「幫我跑月結整理」，確認 Codex 正確啟動 Skill 並開始執行。

**步驟 5**（完成後自行確認）：
- [ ] 資料夾內出現 `.codex/skills/月結整理/SKILL.md`（打開確認內容是你描述的步驟）
- [ ] Codex 真的執行了重新命名與彙整，而不是再問一次「你要做什麼？」

### 檢核 / Verification（2 min）

能做到以下兩件，代表本單元學會：

1. **不必打開資料夾**，說得出「我的月結整理 Skill 在哪個地方找到」（外掛程式 → 技能頁，層級：專案）
2. **遇到新情境能判斷**：下面哪個情況選哪個方案？
   - 每次對話都需要知道的「公司名稱和月份」→ **AGENTS.md**
   - 每月做一次、流程固定但不是每天用的月結 SOP → **專案 Skill**
   - 跨所有工作都適用的「Word 表格排版格式」→ **全域 Skill**

## 動手練習題（Hands-on Exercise）

**題目**：你已經把月結整理做成專案 Skill 了。現在陳主管說：「我們每次出公文，Word 表格格式都不統一，能不能設定一次，之後就自動套？」

請你判斷這個需求該用哪種方式處理，並動手建出來。

**預期成果**：
- 一個名為「表格排版」（或類似命名）的全域 Skill
- 內容至少包含：標題列格式、內容列格式、字體、線條粗細
- 在「外掛程式 → 技能」頁可見，層級為「全域」

**完成標準**（self-check）：
- [ ] Skill 名稱讓人看名字就知道用途（「表格排版」「Word 表格」可以，「skill1」不行）
- [ ] 打開 Skill 資料夾內的 SKILL.md，裡面的步驟是人讀得懂的中文說明
- [ ] 用斜線 `/表格排版` 能叫出 Skill，確認 Codex 真的照格式規範做出表格

## 常見錯誤 3 條（Common Pitfalls）

1. **錯誤現象**：「幫我跑月結整理」送出後，Codex 回覆「請問你要整理哪些步驟？」——沒有觸發 Skill。
   **原因**：Skill 建在另一個專案，或當前對話不在月結專案下，Codex 看不到這個專案層級的 Skill。
   **解法**：確認你是在「弄一下工作室_6月月結」**專案**下開的對話（左側面板顯示的是這個專案），不是點「新增對話」開的普通對話。或改用斜線 `/` 手動觸發。

2. **錯誤現象**：Skill 建好後第一次觸發，Codex 執行了但輸出結果不對（欄位名稱錯、格式跑掉）。
   **原因**：建立 Skill 時描述的步驟本身有誤（例如 Excel 欄位名稱和實際檔案不符），Skill 忠實固化了這個錯誤。之後每次觸發都會穩定地照著錯走。
   **解法**：第一次觸發後逐步核對輸出，發現問題立即重新描述 Skill 步驟、讓 Codex 更新內容，確認正確後才正式複用。

3. **錯誤現象**：把所有規則都塞進 AGENTS.md，Codex 回覆偶爾會漏掉某些步驟。
   **原因**：AGENTS.md 每次對話都整份載入；長的 SOP 一起塞進去，context 空間就被背景資訊占走。Skill 的設計是 progressive disclosure——只有被選中時才讀完整內容，平時不進 context，所以適合放按需才用的 SOP。
   **解法**：AGENTS.md 只留「每次都需要知道」的背景（公司名、底線規則）；按需才用的 SOP 改成 Skill。兩邊各司其職，context 才不會虛耗。

## 檢核題 2 條（Quiz）

**Q1（概念驗證）**：和把流程寫進 AGENTS.md 相比，封裝成 Skill 最主要的優點是什麼？
- [ ] A. Skill 可以自動排程，AGENTS.md 不行
- [ ] B. Skill 平常不載入對話，只有叫用時才讀取，不佔 Codex 背景思考空間 ←（正確答案）
- [ ] C. Skill 可以包含圖片，AGENTS.md 只能放文字
- [ ] D. Skill 建好後不能修改，AGENTS.md 可以隨時更新

**Q2（應用驗證）**：你想把「每份月結報告都要在第一頁加封面、封面格式固定」這條規範設定好。陳主管說這個格式其實所有報告都適用，不只月結。這個情況你應該怎麼做？
- 預期答案要點：
  1. **選全域 Skill**：因為這條格式規範跨所有報告都會用到，不只限月結專案
  2. **不要放進 AGENTS.md**：格式規範只有「做報告時」才需要，不是每次對話都要載入的背景資訊
  3. **說得出確認做對的方式**：建好後在「外掛程式 → 技能」頁確認層級顯示「全域」；用斜線觸發測試一次，確認封面格式符合預期，再正式複用
