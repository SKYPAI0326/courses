## 教學流程（Teaching Flow）

### 破題 / Hook

6 月底，你已經有一包整理好的帳——命名統一的報帳單、一張彙整 Excel、可重用的月結 Skill。但月結還沒結束：陳主管要一份給老闆看的管理層摘要 PPT；行政用的部門識別吊牌還沒做；幾封對帳確認信和催收信還在排隊。

**這個單元，你會看到 Codex 裝三個外掛之後能做出以下成果**：

1. 一份「6 月月結管理層摘要」PPT（Presentations 外掛生的）
2. 一批部門識別吊牌 PDF（內建生圖 + PDF Skill 批次出的）
3. Gmail 草稿信數封（Gmail 外掛草擬，寄出由你按）

三條外掛線，每條都有一個「哪裡要你自己攔下來核」。看完本頁你會知道三條底線在哪裡。

> **外掛可用性**：Presentations、Gmail、PDF Skill 等功能依帳號方案與版本而異，不保證你的帳號當下可用。這一頁用看示範＋觀察，重點是看清楚操作邏輯與把關位置，課後再在自己帳號上跑。

### 概念 / Concepts

- **Skill（技能）** — 可重用的工作流文件，描述 Codex 執行一類任務的步驟與規則。可以自建（CH3-1），也可以從 marketplace 安裝別人寫好的。不含外部服務連線。
- **App / Connector（應用程式/連接器）** — 讓 Codex 連接外部服務（如 Gmail、Google Drive）的橋接模組。安裝後 Codex 才能讀寫那個服務的資料，通常需要授權流程。
- **外掛（Plugin）** — 安裝與分發的打包單位，一個外掛可以同時包含 Skill、App/Connector、MCP 設定等多個元件。你從外掛程式頁安裝的「Gmail 外掛」或「Presentations 外掛」就是這一層。安裝一個外掛，可能同時帶進一條 Skill 和一個 App 連線。
- **外掛 marketplace** — Codex 外掛程式頁（左上角「外掛程式」）。裡面有「外掛程式」和「技能」兩個標籤，分開瀏覽。需要哪條能力，進去找、點安裝、照授權流程走。
- **最小必要權限** — 裝外掛時，Codex 會請你授權某些帳號存取。只給這個任務真正需要的權限，不需要的存取不要勾選。

### 操作示範 / Demo（三條外掛線依序）

---

**外掛 ① Presentations — 生管理層摘要 PPT**

目標：讓 Codex 生出一份可以給老闆看的「6 月月結管理層摘要」PPT，不用自己一頁一頁排版。

步驟 1：進外掛程式頁，安裝 Presentations 外掛。

步驟 2：回到首頁（或你的月結專案對話），送出這段提示詞：

> 請用 Presentations 外掛生一份「弄一下工作室 6 月月結管理層摘要」PPT，大約 8–10 頁，包含：本月費用總覽、各部門費用分項、供應商付款進度、下月待追事項。數字從我的彙整 Excel 取。

過一兩分鐘，Codex 啟用 Presentations 外掛，出 PPT 檔。

步驟 3：打開 PPT，看結構與數字呈現。

> **把關紅線：對外簡報的數字，一定要能回得到來源**
> Codex 從你的 Excel 引數字，但有沒有引錯欄、合計對不對，它不會告訴你。把這份簡報轉給老闆之前，**陳主管要核過一遍，確認每個數字有來源、能回得去**，不能直接就發出去。

---

**外掛 ② 內建生圖 + PDF Skill — 批次做識別吊牌**

目標：行政組要給部門人員識別吊牌，有人名清單，每個人名字不一樣。Codex 內建生圖 + PDF Skill 批次搞定。

步驟 1：向 Codex 描述設計需求，送出這段提示詞：

> 幫我生一張部門識別吊牌的底圖，尺寸比例 54×86mm（名片橫式），風格：簡潔白底、左上角放弄一下工作室 Logo、中央保留姓名和職稱的空間、底部一條橘色色塊（色碼 #b5703a）。

Codex 生出底圖設計稿（這只是範本，名字還沒填）。

步驟 2：確認底圖後，請 Codex 再生一張乾淨底圖（不含示範用的名字佔位符），把這兩個圖檔拖曳到工作資料夾。

步驟 3：進外掛程式頁「技能」標籤，安裝 PDF Skill。

步驟 4：回首頁，送出這段提示詞：

> 請使用剛才生的識別吊牌底圖，把「弄一下工作室_6月月結」資料夾裡的員工名單（姓名、部門、職稱）逐一填入，批次生成每人一份識別吊牌 PDF，存到「識別吊牌/」子資料夾。

Codex 跑完後，打開子資料夾，每個人一份 PDF。

> **把關紅線：批次輸出前確認名單正確**
> PDF 一次生很多份——名字打錯、職稱填錯，全批都錯。批次跑之前，先確認來源名單沒問題；跑完再抽查幾份核人名與職稱，確認無誤再送去印。

---

**外掛 ③ Gmail — 草擬對帳確認信 + 催收信存草稿**

目標：本月有幾家廠商的帳還沒收到回覆確認，要發對帳確認信；另外有兩張發票過期了，要發催收信。讓 Codex 依廠商資料客製化草擬，存到 Gmail 草稿，寄出的按鈕你自己按。

步驟 1：把廠商清單（有公司名、聯絡人、未確認金額）拖曳到工作資料夾。

步驟 2：進外掛程式頁，安裝 Gmail 外掛，照畫面走授權流程。

> **授權底線：最小必要權限**
> Gmail 外掛請你授權時，只給「建立草稿」這個範圍的存取就夠了，不需要「讀取你所有郵件」或「代你傳送」。不需要的帳號存取不要勾選。

步驟 3：回首頁，送出這段提示詞：

> 請幫我草擬對帳確認信，依廠商清單裡的每家公司各寫一封，開頭依廠商名稱和聯絡人姓名客製化，信末請對方確認「6 月對帳金額」是否與附件一致，口吻正式但不生硬。先給我看第一封範本。

步驟 4：Codex 生出第一封，確認語氣與客製化的地方（公司名、金額、聯絡人都對）。

步驟 5：確認沒問題後，送出：

> 樣本沒問題，請把其餘廠商的信全部草擬好並存到 Gmail 草稿匣。

過一兩分鐘，Codex 完成。打開瀏覽器登入 Gmail，確認所有信已在草稿匣。

步驟 6（催收信）：再送出：

> 另外有兩張逾期發票，請草擬催收信，語氣禮貌但明確，請對方在三個工作天內回覆付款時程，同樣存到 Gmail 草稿。

> **把關紅線：信在草稿，寄出的按鈕你自己按**
> **課程規範：本課程只建立草稿，不授權、不要求、不示範寄出；寄出一律由你手動按。**
> 每封信打開、逐封核一遍——收件人、金額、措辭——確認對了才按寄出。對外信件寄錯一封就是問題。

### 觀察單 / Watch（不繳交）

三條外掛線看下來，記住這三個問題——這是你自己跑之前要先想好的：

1. 裝外掛授權時，**你打算給它哪些範圍的存取？** 最小必要夠不夠？
2. Codex 生出的 PPT 數字，**你要怎麼核對回來源？**（翻哪張 Excel、查哪個欄位？）
3. 草稿信看起來語氣對，但**收件人、金額有沒有對應到正確的廠商？** 這你必須逐封確認。

三個問題都答得出來，就掌握了本頁的把關邏輯。

### 檢核 / Verification（口頭·認知）

下一頁 CH4-1 之前，確認自己能答出：

- 「外掛」和 CH3-1 的「Skill」差在哪裡——各自適合什麼情境。
- 裝 Gmail 外掛時授權應給的最小範圍是什麼。
- Gmail 草稿的「把關閘」在哪一步（誰按寄出？在哪裡看到信？）。

## 商業情境案例（Case）

**角色**：你（弄一下工作室 行政會計）
**公司**：弄一下工作室（約 10 人的小型工作室）
**任務**：6 月底月結收尾——給老闆看的管理層摘要 PPT 要做、部門識別吊牌要印、幾家廠商的對帳確認信和催收信要發。

## 常見錯誤 3 條（Common Pitfalls）

1. **錯誤現象**：Gmail 外掛裝好、信也存到草稿了，以為「Codex 已經幫我寄出去了」，沒有再去 Gmail 確認。
   **原因**：混淆「存草稿」和「寄出」。課程規範只建立草稿；寄出一律由人手動按。
   **解法**：裝完 Gmail 外掛後立刻到瀏覽器開 Gmail 草稿匣，確認信在那裡、還沒寄出。信送出之前，每封都要打開看一遍收件人和金額。

2. **錯誤現象**：PPT 生出來，直接截圖傳給老闆，沒有核數字——後來老闆發現合計不對，質疑整份帳的可信度。
   **原因**：把「Codex 從 Excel 引數字」當成「數字一定正確」。Codex 可能引錯欄、算錯總額。
   **解法**：PPT 生出來之後，找到原始彙整 Excel，把 PPT 上的每個關鍵數字跟 Excel 對一遍。對外簡報要能「每個數字都能說出來自哪裡」，陳主管確認過再轉給老闆。

3. **錯誤現象**：授權 Gmail 外掛時，不看授權範圍就全勾，給了「讀取全部郵件」或「代你傳送」的權限。
   **原因**：趕快點「下一步」，沒有停下來讀授權項目。
   **解法**：安裝任何外掛授權時，停一下看清楚 Codex 要什麼存取權。這個任務只需要「建立草稿」——其他的不需要也不要給。如果已經給了過寬的權限，到 Google 帳號安全設定裡的「第三方應用程式存取」把它撤銷，重新只給需要的範圍。

## 檢核題 2 條（Quiz）

**Q1（概念驗證）**：外掛（Plugin）、Skill（技能）、App/Connector 三者最關鍵的差別是什麼？
- [ ] A. 外掛是免費的，Skill 要額外付費，App 是付費升級版
- [ ] B. Skill 是可重用工作流文件；App/Connector 是連接外部服務的橋接模組；Plugin 是安裝分發的打包單位，可同時包含 Skill 和 App ←（正確答案）
- [ ] C. Skill 比外掛更強，外掛只是 Skill 的簡化版
- [ ] D. 三者完全一樣，只是叫法不同

**Q2（應用驗證）**：你請 Codex 裝好 Gmail 外掛，草擬了 5 封廠商對帳確認信並存到草稿匣。陳主管問你：「信寄出去了嗎？」你應該怎麼回答，而且接下來要做什麼？
- 預期答案要點：
  1. **信還在草稿匣、還沒寄出**——課程規範只建立草稿，寄出由人手動按
  2. 寄出之前，**你要逐封打開確認**：收件人名稱和 email 對不對、金額是否對應到正確的廠商、語氣是否合宜
  3. 確認沒問題之後，**自己按發送**，不能叫 Codex 代你寄出
  4. 如果有一封措辭不對或對象搞混，**只改那一封**，其他沒問題的照寄——不要因為一封有問題就整批重來

## 講師授課筆記（不進講義）

> 本區塊不轉成學員 HTML（course-lesson-writer 跳過）。授課調度全寫這裡。

**時間分配（27 min）**：破題 2｜概念 3｜外掛① Presentations 7｜外掛② 內建生圖+PDF Skill 7｜外掛③ Gmail 7｜觀察單+收束 1

**外掛可用性（課前必確認）**：
- Presentations、Gmail、PDF Skill 是帳號方案／版本相依功能，講師用自己帳號課前逐項驗證三條線可用。
- **備案**：
  - Presentations 外掛不可用 → 講師直接呈現預先生好的 PPT 範本，說明生成過程
  - Gmail 外掛不可用 → 示範 Codex 草擬郵件文字後「複製貼上到 Gmail 自行填寄」的替代流程
  - PDF Skill 不可用 → 播放預錄操作短片（5 min）替代現場跑

**講師端準備（三條線各需備妥）**：

外掛① Presentations：
- 已在月結專案裝好 Presentations 外掛（免得現場等安裝）
- 「弄一下工作室_6月月結」彙整 Excel（CH2-2 產出的）放在工作資料夾內——Codex 引數字用
- 備妥提示詞範本（見教學流程操作示範段）
- 預期輸出：一份 8–10 頁 PPT，含費用總覽、部門分項、供應商付款進度、下月追蹤

外掛② 內建生圖 + PDF Skill：
- 員工名單（姓名、部門、職稱，至少 5–6 人）放在工作資料夾；名單格式：CSV 或 Excel
- 備妥設計描述文字（尺寸、風格、色碼）；示範前不要預先生圖，讓學員看到生成過程
- 已在外掛程式頁確認 PDF Skill 安裝連結可用
- 預期輸出：6 份識別吊牌 PDF，存於「識別吊牌/」子資料夾

外掛③ Gmail：
- 廠商清單：3–4 家廠商，含公司名、聯絡人姓名、未確認金額、發票到期狀態（其中 2 家逾期）
- 講師帳號已完成 Gmail 外掛授權（現場授權要輸入帳號密碼會打斷節奏）
- 備妥提示詞範本（見教學流程操作示範段）
- 預期輸出：草稿匣裡 3–4 封對帳確認信 + 2 封催收信（共 5–6 封草稿）

**轉場節奏**：
三條線的順序邏輯——Presentations 一句話出 PPT，先讓學員看到外掛「裝了就能做以前不行的事」；生圖 + PDF Skill 拆兩步展示，讓學員感受 Skill 的擴充邏輯；Gmail 涉及帳號授權，把關底線最多放最後。三條線之間轉場各約 30 秒，簡短說「下一個外掛，先裝再用，同一套邏輯」，不需長過場。

**收束話術**：「裝了外掛，Codex 的手可以伸到更多地方——但每伸一個地方，你就要多一條底線。本頁三條紅線，上機時記得對號入座。」
