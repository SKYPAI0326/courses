## 教學流程（Teaching Flow）

### 破題 / Hook（2 min）

6 月底，你已經有一包整理好的帳——命名統一的報帳單、一張彙整 Excel、可重用的月結 Skill。但月結還沒結束：陳主管要一份給老闆看的管理層摘要 PPT；行政用的部門識別吊牌還沒做；幾封對帳確認信和催收信還在排隊。

這些事情，以前你要靠 Word 拼 PPT、用相片編輯軟體做識別牌、一封一封寫信。今天你會看到 Codex 裝三個外掛之後，全部能處理。

看完這一頁，你手上多三樣東西：
- 一份「6 月月結管理層摘要」PPT（Presentations 外掛生的）
- 一批部門識別吊牌 PDF（內建生圖 + PDF Skill 批次出的）
- Gmail 草稿信數封（Gmail 外掛草擬，寄出由你按）

### 概念 / Concepts（3 min）

- **Skill（技能）** — 可重用的工作流文件，描述 Codex 執行一類任務的步驟與規則。可以自建（CH3-1），也可以從 marketplace 安裝別人寫好的。不含外部服務連線。
- **App / Connector（應用程式/連接器）** — 讓 Codex 連接外部服務（如 Gmail、Google Drive）的橋接模組。安裝後 Codex 才能讀寫那個服務的資料，通常需要授權流程。
- **外掛（Plugin）** — 安裝與分發的打包單位，一個外掛可以同時包含 Skill、App/Connector、MCP 設定等多個元件。你從外掛程式頁安裝的「Gmail 外掛」或「Presentations 外掛」就是這一層。安裝一個外掛，可能同時帶進一條 Skill 和一個 App 連線。
- **外掛 marketplace** — Codex 外掛程式頁（左上角「外掛程式」）。裡面有「外掛程式」和「技能」兩個標籤，分開瀏覽。你需要哪條能力，進去找、點安裝、照授權流程走就好。
- **最小必要權限** — 裝外掛時，Codex 會請你授權某些帳號存取。只給它這個任務真正需要的權限，不需要的存取不要勾選。這是這一頁最重要的一條底線之一。

### 操作示範 / Demo（三條外掛線依序示範，21 min）

> ⚠️ **外掛可用性說明（講師必讀）**
> Presentations、Gmail、生圖、PDF Skill 等外掛能力是 Codex plan/workspace/版本相依的功能，**不保證學員帳號都可用**。
> - **本單元定位：講師示範為主，學員觀察跟進；不要求學員當場在自己帳號完成全部步驟。**
> - **課前逐項確認**：講師用自己帳號驗証三條線可用，並備妥下列備案：
>   - Presentations 外掛不可用 → 講師直接呈現預先生好的 PPT 範本，說明生成過程
>   - Gmail 外掛不可用 → 示範 Codex 草擬郵件文字後「複製貼上到 Gmail 自行填寄」的替代流程
>   - PDF Skill 不可用 → 講師播放預錄操作短片（5 min）替代現場跑

**外掛 ① Presentations — 生管理層摘要 PPT（7 min）**

目標：讓 Codex 生出一份可以給老闆看的「6 月月結管理層摘要」PPT，不用你自己一頁一頁排版。

步驟 1：進外掛程式頁，安裝 Presentations 外掛。

步驟 2：回到首頁（或你的月結專案對話），寫一段提示詞，例如：

> 請用 Presentations 外掛生一份「弄一下工作室 6 月月結管理層摘要」PPT，大約 8–10 頁，包含：本月費用總覽、各部門費用分項、供應商付款進度、下月待追事項。數字從我的彙整 Excel 取。

送出。Codex 啟用 Presentations 外掛，過一兩分鐘出 PPT 檔。

步驟 3：講師打開生出的 PPT 讓學員看結構與數字呈現。

> 🔴 **把關紅線：對外簡報的數字，一定要能回得到來源**
> Codex 生出的 PPT 數字，它是從你的 Excel 引的——但有沒有引錯？合計對不對？你在把這份簡報轉給老闆之前，**陳主管要核過一遍，確認數字有來源、能回得去**，不能直接就發出去。

---

**外掛 ② 內建生圖 + PDF Skill — 批次做識別吊牌（7 min）**

目標：行政組要給部門人員識別吊牌，有人名清單，每個人名字不一樣。不用手工一張一張做，Codex 內建生圖 + PDF Skill 批次搞定。

步驟 1：向 Codex 描述設計需求，例如：

> 幫我生一張部門識別吊牌的底圖，尺寸比例 54×86mm（名片橫式），風格：簡潔白底、左上角放弄一下工作室 Logo、中央保留姓名和職稱的空間、底部一條橘色色塊（色碼 #b5703a）。

Codex 生出底圖設計稿，講師讓學員看圖（這只是範本，名字還沒填）。

步驟 2：確認底圖後，請 Codex 再生一張乾淨底圖（不含示範用的名字佔位符），把這兩個圖檔拖曳到工作資料夾。

步驟 3：進外掛程式頁「技能」標籤，安裝 PDF Skill。

步驟 4：回首頁，寫提示詞，例如：

> 請使用剛才生的識別吊牌底圖，把「弄一下工作室_6月月結」資料夾裡的員工名單（姓名、部門、職稱）逐一填入，批次生成每人一份識別吊牌 PDF，存到「識別吊牌/」子資料夾。

送出，Codex 跑完後打開子資料夾，每個人一份 PDF。

> 🔴 **把關紅線：批次輸出前確認名單正確**
> PDF 一次生很多份——名字打錯、職稱填錯，全批都錯。批次跑之前，你先確認來源名單沒問題，跑完再抽查幾份核人名與職稱，確認無誤再送去印。

---

**外掛 ③ Gmail — 草擬對帳確認信 + 催收信存草稿（7 min）**

目標：本月有幾家廠商的帳還沒收到回覆確認，要發對帳確認信；另外有兩張發票過期了，要發催收信。讓 Codex 依廠商資料客製化草擬，存到 Gmail 草稿，寄出的按鈕你自己按。

步驟 1：把你的廠商清單（有公司名、聯絡人、未確認金額）拖曳到工作資料夾。

步驟 2：進外掛程式頁，安裝 Gmail 外掛，照畫面走授權流程。

> 🔴 **授權底線：最小必要權限**
> Gmail 外掛請你授權時，只給「建立草稿」這個範圍的存取就夠了，不需要「讀取你所有郵件」或「代你傳送」。不需要的帳號存取不要勾選。

步驟 3：回首頁，寫提示詞，例如：

> 請幫我草擬對帳確認信，依廠商清單裡的每家公司各寫一封，開頭依廠商名稱和聯絡人姓名客製化，信末請對方確認「6 月對帳金額」是否與附件一致，口吻正式但不生硬。先給我看第一封範本。

步驟 4：Codex 生出第一封，講師確認語氣與客製化的地方（公司名、金額、聯絡人都對）。

步驟 5：確認沒問題後，寫下一句：

> 樣本沒問題，請把其餘廠商的信全部草擬好並存到 Gmail 草稿匣。

過一兩分鐘，Codex 完成。講師打開瀏覽器登入 Gmail，讓學員看到所有信已在草稿匣。

步驟 6（催收信）：再補一段提示詞：

> 另外有兩張逾期發票，請草擬催收信，語氣禮貌但明確，請對方在三個工作天內回覆付款時程，同樣存到 Gmail 草稿。

> 🔴 **最重要的把關紅線：信在草稿，寄出的按鈕你自己按**
> **課程規範：本課程只建立草稿，不授權、不要求、不示範寄出；即使工具在技術上支援送出動作，「寄出」在本課程視為禁用動作，寄出一律由你手動按。**
> 外掛連接外部服務後理論上可能具備傳送能力，但課程現場絕不走那條路。**每封信你要自己打開、逐封核一遍——收件人、金額、措辭——確認對了才按寄出**。對外信件不比對內備忘錄，寄錯一封就是問題。

### 觀察單 / Watch（1 min・不繳交）

三條外掛線看下來，請你記住這三個問題（這是你自己跑之前要先想好的）：

1. 裝外掛授權時，**你打算給它哪些範圍的存取？** 最小必要夠不夠？
2. Codex 生出的 PPT 數字，**你要怎麼核對回來源？**（翻哪張 Excel、查哪個欄位？）
3. 草稿信看起來語氣對，但**收件人、金額有沒有對應到正確的廠商？** 這你必須逐封確認。

如果三個問題你都答得出來，代表你已經掌握了本頁的把關邏輯，上機跑的時候不會卡。

### 檢核 / Verification（口頭・認知）

學會的標準（不必全部在課堂跑完，但要答得出來）：

- 能說出「外掛」和 CH3-1 的「Skill」差在哪裡——各自適合什麼情境。
- 能指出裝 Gmail 外掛時授權應給的最小範圍是什麼。
- 能說出 Gmail 草稿的「把關閘」在哪一步（誰按寄出？在哪裡看到信？）。

### 節奏備注（講師參考）

三條外掛線的順序設計邏輯如下，可幫助講師掌握轉場節奏：

1. **先裝 Presentations（最快看到成果）**：一段提示詞出一份 PPT，視覺衝擊最直接，放在最前面提振注意力。重點示範外掛「裝了就能做以前不行的事」這個概念。
2. **再裝生圖 + PDF Skill（拆成兩步展示）**：先示範 Codex 內建生圖能力（不用外掛），再示範裝 PDF Skill 把生圖能力延伸成批次 PDF——兩步之間的差距讓學員直觀感受到 Skill 的擴充邏輯。
3. **最後裝 Gmail（最多把關提醒）**：Gmail 外掛涉及帳號授權，把關底線最多，放最後讓前面兩條先暖身，學員心態準備好了，再仔細講授權與草稿邊界。

三條線之間轉場各約 30 秒，簡短說「下一個外掛，先裝再用，照同一套邏輯」，不需要長過場。

## 常見錯誤 3 條（Common Pitfalls）

1. **錯誤現象**：Gmail 外掛裝好、信也存到草稿了，但學員以為「Codex 已經幫我寄出去了」，沒有再去 Gmail 確認。
   **原因**：混淆「存草稿」和「寄出」——課程規範只建立草稿；即使外掛技術上可能支援傳送，寄出也屬禁用動作，後者一律由人手動按。
   **解法**：裝完 Gmail 外掛後立刻到瀏覽器開 Gmail 草稿匣，確認信在那裡、還沒寄出。信送出之前，每封都要打開看一遍收件人和金額。

2. **錯誤現象**：PPT 生出來，直接截圖傳給老闆，沒有核數字——後來老闆發現合計不對，質疑整份帳的可信度。
   **原因**：把「Codex 從 Excel 引數字」當成「數字一定正確」。Codex 可能引錯欄、算錯總額。
   **解法**：PPT 生出來之後，找到原始彙整 Excel，把 PPT 上的每個關鍵數字跟 Excel 對一遍。對外簡報要能「每個數字都能說出來自哪裡」，陳主管確認過再轉給老闆。

3. **錯誤現象**：授權 Gmail 外掛時，不看授權範圍就全勾，給了「讀取全部郵件」或「代你傳送」的權限。
   **原因**：趕快點「下一步」，沒有停下來讀授權項目。
   **解法**：安裝任何外掛授權時，停一下看清楚 Codex 要什麼存取權。這個任務只需要「建立草稿」——其他的不需要也不要給。如果已經給了過寬的權限，到 Google 帳號安全設定裡的「第三方應用程式存取」把它撤銷，重新只給需要的範圍。

**Q1（概念驗證）**：外掛（Plugin）、Skill（技能）、App/Connector 三者最關鍵的差別是什麼？
- [ ] A. 外掛是免費的，Skill 要額外付費，App 是付費升級版
- [ ] B. Skill 是可重用工作流文件；App/Connector 是連接外部服務的橋接模組；Plugin 是安裝分發的打包單位，可同時包含 Skill 和 App ←（正確答案）
- [ ] C. Skill 比外掛更強，外掛只是 Skill 的簡化版
- [ ] D. 三者完全一樣，只是叫法不同

**Q2（應用驗證）**：你請 Codex 裝好 Gmail 外掛，草擬了 5 封廠商對帳確認信並存到草稿匣。陳主管問你：「信寄出去了嗎？」你應該怎麼回答，而且接下來要做什麼？
- 預期答案要點：
  1. **信還在草稿匣、還沒寄出**——課程規範只建立草稿，寄出屬禁用動作，按鈕要自己按
  2. 寄出之前，**你要逐封打開確認**：收件人名稱和 email 對不對、金額是否對應到正確的廠商、語氣是否合宜
  3. 確認沒問題之後，**自己按發送**，不能叫 Codex 代你寄出
  4. 如果有一封措辭不對或對象搞混，**只改那一封**，其他沒問題的照寄——不要因為一封有問題就整批重來
