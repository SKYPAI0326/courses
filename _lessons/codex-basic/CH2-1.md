## 教學流程（Teaching Flow）

### 破題 / Hook

CH1-3 結束時，你讓 Codex 看了一遍「弄一下工作室_6月月結」資料夾——它列出了哪些檔案、哪些地方有問題。現在要進入第一個動手環節。

**這 17 分鐘結束，你手上會多兩樣東西：**

1. **6 份統一命名的報帳單**——素材包裡 6 份命名各自不同的報帳單全部改成「部門-月份-報帳單」格式。
2. **一份處理日誌.txt**——記錄 Codex 改了哪些檔、原始名稱是什麼，方便你之後對帳或回溯。

在進入操作之前，有一件事要先說清楚。

**把關紅線：改檔名是覆寫，不是另存**

Codex 批次改名不是「另存新檔」——它把原始檔案的名稱直接換掉。一旦改完，舊名稱就不在了，OS 的版本紀錄雖然有時能救回，但不保證每次都方便查。

所以課堂的做法是：**操作前先把正式檔案複製一份到別的地方作備份，同時讓 Codex 產生一份 rename log（改名對照表）**。日後萬一要追溯原始檔名，對照這份記錄就能找到。今天這個練習用的是試跑包裡的副本，可以直接玩。真實工作上每次批次改名前，備份是第一步，不能省。

### 概念 / Concepts

**批次重新命名**

一句提示詞，Codex 掃遍整個資料夾裡的指定檔案，把命名全部統一成你要的格式，不用一個一個手動改。

**它怎麼從內容裡找部門**：Codex 依序打開每份文件，在內文、表頭、頁首等位置找「部門」相關的欄位或字眼（例如「業務部」「行政部」）。找到明確的詞就用那個詞，找不到就停下來問——前提是提示詞裡有「抓不到就停、不要猜」這句話。如果提示詞沒說清楚，它可能自己推斷補值（如「未知部門」）或直接跳過，這是你要防的情況。

**@ 引用（@ vs 不引用的差別）**

在提示詞裡打 `@`，畫面跳出候選清單，讓你指定要讓 Codex 讀哪一份檔案。

「不引用」的情況：Codex 有可能根據對話 context 自己判斷要讀哪些檔案，或把整個資料夾都掃過去——你沒有辦法確保它只動你想動的那份。

「用 @ 引用」的情況：你明確指定一份報帳單，Codex 只讀那份，其他的不動。要從多份報帳單裡只抽某一份的資料時，@ 是保險做法。

什麼情境一定要用 @：你只想讓 Codex 看某一份、或某幾份指定的檔案，不想讓它自己決定範圍時。

**並行任務（時序）**

第一句送出後，Codex 開始跑，還沒完成時你可以**在同一個對話框**繼續送第二句指令，它會兩件一起做完。

時序說明：不是同時打兩個對話框，也不是等第一件完全跑完再送第二件。是第一句送出後，趁 Codex 還在處理，緊接著送第二句。批次改名 + 建處理日誌這兩件事可以這樣並行——因為它們屬於同一個作業（整理報帳單），Codex 的 context 不會混亂。

但不是所有任務都適合並行：跟改名完全不同性質的獨立任務（例如「@ 引用某份單一檔案做抽取」），最好等前面兩件確認跑完後再送，避免 Codex 把兩件事的輸出混在一起。

**處理日誌**

請 Codex 在跑完任務後另外建一份文字記錄，列出它處理了哪些檔案、做了什麼動作。這份日誌是批次改名後唯一可靠的回溯依據（原始檔名 → 新檔名的對照），之後補交、追帳、對帳都用得到。

### 操作示範 / Demo

以下示範用試跑包裡的 6 份報帳單：

- `報帳.docx`
- `六月-業務部.xlsx`
- `費用單(最終版2).docx`（刻意不含部門欄位）
- `2026-06 採購.docx`
- `六月費用_行政.xlsx`
- `報帳0610設計部.docx`

**步驟 1：備份確認**

操作前先確認試跑包副本已準備好（不要對著正式原檔操作）。打開 Codex，進入「弄一下工作室月結」專案的對話框，點專案資料夾旁的 + 號，開新對話。

**步驟 2：送出批次改名提示詞**

在輸入框輸入以下提示詞，按 Enter 送出：

> 把資料夾裡所有報帳單的檔名，統一改成「部門-月份-報帳單」的格式。部門和月份從每份文件的內容裡抓。如果哪份文件抓不到部門名稱，不要自己猜——停下來告訴我哪份有問題，等我說怎麼處理。

送出後，Codex 可能先列出「計畫做什麼」請你確認——仔細看它列的清單，確認對應正確再按「繼續」。

**Codex 遇到費用單(最終版2).docx 時的停頓長相**

因為那份文件裡沒有部門欄位，Codex 的輸出看起來類似這樣：

> 有一份報帳單我找不到部門名稱：費用單(最終版2).docx。請告訴我要用哪個部門名稱，或確認部門欄位在哪裡，我再繼續其餘 5 份。

這個停頓是正確的行為，不是錯誤。它是「提示詞裡有寫缺資料就停」這句話產生的效果。如果沒有這句，Codex 可能自己補上「未知部門」或直接略過，你就不會發現資料缺漏。

收到這段訊息後，你有兩個選擇：告訴 Codex 那份要用哪個部門名稱（如果你知道），或先跳過那份、讓它改好其餘 5 份再說。

**步驟 3：並行送出處理日誌任務**

Codex 還在改名的同時（或改完後立刻），在同一個對話框送出：

> 幫我建一份「處理日誌.txt」，列出你處理了哪些報帳單、原始檔名和新檔名各是什麼，每行一筆。格式：原始檔名 → 新檔名。

這份日誌是批次改名後的回溯依據，每次批次改名都要請 Codex 一起建，不能省。

**步驟 4：用 @ 引用特定檔案示範**

等前兩件事確認跑完後，示範 @ 引用的用途。假設你想從「六月-業務部.xlsx」這份報帳單裡抽出金額欄位：

在輸入框打 `@`，等候選清單出現，點選「六月-業務部.xlsx」，檔名自動帶入。接著繼續打：

> 從這份報帳單裡，抓出所有的支出項目和金額，列成表格格式。如果有欄位是空的，不要補值，直接標「待補」。

送出後，Codex 只讀你 @ 指定的那份，其他報帳單不動。

**把關提醒**：抽出來的表格要核一遍再用——金額欄尤其容易識別錯，抽查總額是否和原始報帳單相符。

### 動手 / Hands-on

打開「弄一下工作室_6月月結」專案，開新對話，完成以下三件事：

**任務 1：批次改名**

複製以下提示詞，送出：

> 把資料夾裡所有報帳單的檔名，統一改成「部門-月份-報帳單」的格式。部門和月份從每份文件的內容裡抓。如果哪份文件抓不到部門名稱，不要自己猜——停下來告訴我哪份有問題，等我說怎麼處理。

Codex 如果停下來問，告訴它要怎麼處理那份缺資料的檔案（或先跳過），不要讓它自己猜。

**任務 2：建處理日誌（並行或改名確認後）**

複製以下提示詞送出：

> 幫我建一份「處理日誌.txt」，列出你處理了哪些報帳單、原始檔名和新檔名各是什麼，每行一筆。格式：原始檔名 → 新檔名。

**任務 3：@ 引用抽取**

等任務 1 + 2 確認完成後，用 `@` 選任一份已改名的報帳單，送出：

> 從這份報帳單裡，抓出所有的支出項目和金額，列成表格格式。如果有欄位是空的，不要補值，直接標「待補」。

**完成標準**（self-check）：
- [ ] 資料夾裡的報帳單已改名（缺資料那份已由你確認處理方式，不是 Codex 自己猜的）
- [ ] 處理日誌.txt 存在，裡面有「原始檔名 → 新檔名」格式的對照記錄
- [ ] @ 引用成功，提示詞裡看得到你選的那個檔名被帶入
- [ ] 整個過程沒有對正式原檔直接操作（試跑包副本）

## 常見錯誤 3 條（Common Pitfalls）

1. **錯誤現象**：批次改名跑完後，費用單(最終版2).docx 被改成「未知部門-06-報帳單」（或直接跳過、靜默處理），不是停下來問你。
   **原因**：提示詞沒有明確寫「抓不到就停、不要猜」。沒有這句，Codex 會自己推斷或填預設值。AGENTS.md 裡若有「缺資料即停」規則但提示詞沒呼應，有時規則也不會生效。
   **解法**：提示詞一定要含「如果哪份文件抓不到部門名稱，不要自己猜——停下來告訴我哪份有問題，等我說怎麼處理」。若已改壞，從備份還原那份，重跑。

2. **錯誤現象**：批次改名還沒確認完，就送出 @ 引用任務，Codex 的輸出混在一起，不知道哪段是改名結果、哪段是抽取結果。
   **原因**：@ 引用抽取跟批次改名是兩件性質完全不同的任務，並行送出時 Codex 的 context 容易混。批次改名 + 建處理日誌屬於同一件事的延伸，可以並行；但獨立的 @ 抽取任務不適合混進同一波。
   **解法**：改名 + 日誌並行；等這兩件確認完成後，再開 @ 引用的那句指令。

3. **錯誤現象**：打了 `@` 之後候選清單沒有出現，或選了檔名後提示詞裡只剩 `@`，沒有帶入檔名。
   **原因**：① `@` 後要等一到兩秒讓候選清單載入，不要馬上按 Enter。② 資料夾裡那份檔案還沒被加進目前的 Codex 專案——Codex 找不到就不會列出。③ 有時要再多打幾個字（如 `@報帳`）縮小候選範圍才會出現。
   **解法**：打 `@` 後等候選清單出現再點選，不要直接 Enter。若清單仍沒出現，先確認那份檔案確實在「弄一下工作室_6月月結/」資料夾內，且這個資料夾是目前專案的工作資料夾。
