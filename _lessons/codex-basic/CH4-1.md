## 教學流程（Teaching Flow）

### 破題 / Hook（2 min）

6 月底，弄一下工作室的廠商對帳作業進入收尾。你發了 12 封確認信給廠商，請他們回覆當月帳款金額是否正確。

問題是：這 12 封信的回覆分散在 Gmail 裡，每天早上得一封封點開看、再手動把「有回 / 未回」記到 Excel，做完已經過了半個小時。更麻煩的是，老闆早上進公司第一件事就問「現在回覆率多少？還差幾家？」——你得打開 Excel 算一遍再回報。

這一頁做的事很直接：**讓 Codex 在你還在睡覺的時候把信掃完、結果記進 Excel，然後做一個儀表板，讓老闆一眼看到進度**。先看示範，再談哪裡要你自己攔。

### 概念 / Concepts（3 min）

- **排程（Schedule）** — 你事先告訴 Codex「每天早上幾點做什麼」，到時間它自動在背景啟動、做完關掉，不需要你開著 Codex 視窗等候。可以同時設多個時段（例如早上 8 點 + 下午 3 點）。
- **排程的安全邊界** — 排程執行時你不在場，Codex 做的每個動作你事後才知道。**這正是要把指令寫清楚的原因**：什麼可以動（讀信、記錄狀態），什麼不能動（寄信、修改原始帳冊、刪除信件）——排程前想清楚，不是做完才後悔。
- **Frontend Design skill** — 網路上有人做好的 Skill，專門用來幫 Codex 產出有設計感的 HTML 介面。安裝方式：把 Skill 的網址貼進輸入框，Codex 直接幫你裝，不需要下載任何東西。
- **HTML 儀表板** — Codex 讀取 Excel 的數據、生成一個 HTML 檔案。用瀏覽器打開就能看到整合好的圖表與數字。**關鍵限制**：儀表板是某個時間點的快照，資料更新後要重跑（或靠排程定時更新），它不會自己即時抓最新數字。

### 操作示範 / Demo（7 min・講師示範）

**A 段：建立排程（3.5 min）**

講師打開「弄一下工作室_6月月結」專案，進入 Codex 側邊欄，找到「自動化」（Automation）或「排程」（Schedule）設定入口。（畫面上通常是一個時鐘或鬧鐘圖示；不同版本介面位置可能略有不同，但功能名稱相同。）

講師說明：

> 排程的概念很簡單——你告訴 Codex「每天幾點、做什麼事」，到了時間它在背景自己跑，你不需要開著 Codex 等。跑完它會把結果記下來，你可以在執行紀錄看到它做了什麼。

輸入排程指令（**第一版：只產清單，不寫 Excel**——講師打字輸入，讓學員看清楚內容）：

> 每天早上 8:00，掃描 Gmail 信箱中主旨含「對帳」或「月結確認」的信件，找出廠商回覆的確認信，整理成「待更新清單」（包含：廠商名稱、信件摘要、建議填入的回覆狀態），**不要修改任何 Excel 檔案**，掃完後把清單回傳讓我確認。若找不到對應廠商名稱，在清單中標記「待人工確認」。

按確認儲存後，講師展示「新增時段」——再加一個下午 3:00 的掃描。

講師說明分兩階段的邏輯：

> 第一版的排程只產清單、不寫 Excel——你拿到清單，核對一遍，確認每筆都對了，才自己複製貼到 Excel，或者再下一個指令讓 Codex 更新。為什麼這樣設計？排程在背景跑、你不在場；萬一它判斷錯一筆廠商名稱，直接寫進去你不一定馬上發現。先看清單、確認兩三次輸出都正確，才考慮讓它自己寫入——而且要寫入的話，**只寫工作副本，正式帳冊永遠留一份人工確認的版本**。

講師接著示範「如果要讓 Codex 更新工作副本」的進階指令（補充示範，不進排程）：

> 請把剛才的清單，更新到「供應商彙整表_工作副本.xlsx」的「回覆狀態」欄（以廠商名稱比對）。若找不到對應廠商名稱，跳過該筆並記錄待確認，不要自行填入或合併。

講師強調：這份是「工作副本」，正式的「供應商彙整表.xlsx」不讓 Codex 直接寫入。

**這裡停一下，講師逐字指出排程指令裡的兩個護欄：**

1. 「若找不到對應廠商名稱，標記待人工確認」——這是防止排程靜默亂配的關鍵。沒有這句，Codex 遇到名稱不完全一致時可能自己猜了填進去。
2. 排程指令裡沒有「寄信」「刪信」「修改 Excel」——排程只做讀取和整理清單。要不要催款、什麼時候寄、什麼時候真的更新帳冊，那是你自己決定的事，不讓排程代勞。

講師補充：

> 排程設定完成不代表萬事 OK。**前幾次排程跑完，一定要打開執行紀錄確認它做了哪些事**、有沒有跳過任何筆、有沒有出現「待人工確認」的標記。確認兩三次輸出都符合預期，才算真的可以信任它。背景自動化的原則是：測過才信任，不是設完就放著。

🔴 **把關紅線：** 排程會在你不在場時自動動作。**第一版排程只輸出清單、不寫 Excel**——確認兩三次輸出符合預期，才考慮讓它寫入；寫入時只寫工作副本，正式帳冊不讓排程直接碰。每次設新排程後，前幾次一定要打開執行日誌確認它做的事跟你預期的一致。

**B 段：做 HTML 儀表板（3.5 min）**

⚠️ **本段標注：講師示範、不保證學員帳號可用。**
外掛 Skill 功能取決於帳號版本與設定；課堂上若無法安裝，改用下方備案（Codex 內建 HTML 生成），產出結果相同。

講師說明本段使用的方式（依當下帳號狀況二擇一）：

**主線（帳號已有相容 Skill）：**

> 我的帳號裡已有一個輔助 HTML 排版的 Skill，我先確認一下它在清單裡……（講師切換到「技能」頁面，確認 Skill 名稱給學員看）……好，這個 Skill 可以讓生出來的版面更整齊。

**備案（Codex 內建 HTML 生成，任何帳號皆可用）：**

> Codex 本身就能生 HTML，不需要額外安裝 Skill；版面乾淨程度稍有不同，但資料和邏輯完全一樣。這個方式你回家自己試沒有任何門檻。

講師回到對話框，輸入儀表板指令（主線或備案均使用同一份 prompt）：

> 讀取「供應商彙整表.xlsx」，做一個 HTML 儀表板，要顯示：（1）廠商對帳回覆率（已回覆/總數的比例，用長條或圓餅呈現）；（2）各廠商回覆狀態清單（已回覆、未回覆用不同顏色區分）；（3）本月月結總金額（已對帳 vs 未對帳）。輸出為「月結進度儀表板.html」存入工作資料夾。

Codex 讀 Excel、生成 HTML，約 1 分鐘後在資料夾出現 `月結進度儀表板.html`。

講師用瀏覽器打開，學員看到：
- 廠商回覆率（圓餅或長條，當下數字）
- 各廠商狀態清單，顏色區分（已回覆 / 未回覆）
- 月結總金額摘要（已對帳金額 vs 尚未確認金額）

講師說明：

> Codex 不管有沒有外掛 Skill，都能做出這個頁面——差別在版面精緻度，資料內容一樣。你回家自己試的時候，直接用備案的指令就好，不用等 Skill 能不能裝。

**這裡停一下，講師強調儀表板的核心限制：**

> 這個儀表板裡的數字，是剛剛 Codex 讀 Excel 當下的快照。如果等一下排程跑完、Excel 有新的回覆狀態，儀表板不會自己更新——你要重新請 Codex 跑一次，或者把「重建儀表板」這個步驟也加進每天的排程，才能讓兩件事連動起來。

🔴 **把關紅線：** 把儀表板拿給老闆看之前，打開 Excel 對一下數字。儀表板好看不代表數字對——來源 Excel 有沒有人工漏填的格子、廠商名稱有沒有寫法不一致，這些只有你看得出來，Codex 不知道。

### 收束（1 min）

講師在示範結束後，指著資料夾說：

> 這個資料夾現在多了什麼？一個排程任務設定，加一個 HTML 儀表板檔。這就是今天這 15 分鐘的實際產出。

**本單元產出物確認**：
- **排程任務**：每天早上 8:00 + 下午 3:00 自動掃 Gmail、輸出廠商回覆待更新清單（第一版）；確認輸出正確後才進階到寫入工作副本
- **月結進度儀表板.html**：整合廠商對帳回覆率 + 各廠商狀態清單 + 月結總金額摘要（Codex 內建 HTML 生成；帳號有相容 Skill 可提升排版精緻度）

講師最後一句話：

> 排程跑在背景、儀表板讓老闆看數字——這兩件事加在一起，你就不用每天花半小時盯 Gmail、也不用口頭報進度。但記住：排程第一版先輸出清單讓你確認，不是設完就直接改 Excel；儀表板要給老闆看之前，你要對一次數字。省下的是重複時間，不是你的判斷責任。

### 觀察單 / Watch（2 min・不繳交）

看示範時，請記三件事：

1. **排程的「靜默執行」**：Codex 在你看不到的時候動你的信箱和 Excel。你有沒有看到講師在排程指令裡設了什麼「護欄」來限制它只做哪些事？
2. **儀表板的時間戳**：這個頁面的數字是「什麼時候」讀 Excel 產出的？如果你早上 8 點才收到廠商信、排程是早上 7 點跑的，儀表板顯示的是哪個狀態？
3. **哪一步你一定要攔**：儀表板做好要給老闆看之前，你會先自己做哪一個確認動作？

然後回答一個問題（給自己，不用交）：**如果要讓儀表板顯示「每小時最新狀態」，你認為需要多加什麼步驟？**（不用答對，先想，下頁 CH4-2 會有部署與更新的進一步示範。）

### 檢核 / Verification（口頭・認知）

學會的標準：

- 你能說出排程跑在背景時「最不該讓它做的一件事」。
- 你能說出儀表板數字「什麼情況下會和 Excel 現狀不一致」，以及怎麼讓它更新。

兩個都答得出來，就可以進 CH4-2 看 Sites 部署與 Computer Use 自動測試。

（示範主線到這裡告一段落。CH4-2 會銜接這個儀表板，把它部署上線成一個真正能分享網址的頁面。）

## 試跑包需求清單（Verification Asset Spec）

**課程類型**：skill-operation（本單元為純示範，學員端不需環境）

- **講師端（demo 用，必備）**：
  - 「弄一下工作室_6月月結」Codex 專案：已安裝 Gmail 外掛（沿用 CH3-2）、Gmail 授權已完成、信箱中已有 2–3 封含「對帳」主旨的廠商回覆信（可預先寄給自己）
  - `供應商彙整表.xlsx`：有「廠商名稱」「對帳金額」「回覆狀態」三欄、至少 8 筆資料、「回覆狀態」大部分為空白（1–2 筆已填「已回覆」作為對比）
  - Frontend Design Skill（若使用）：課前確認講師帳號該 Skill 仍可用、網址有效；備案為 Codex 內建 HTML 生成，任何帳號皆可用，不需另裝
  - 備用影片：錄好「排程跑完後，待更新清單出現」的 30 秒片段（demo 中無法等待真實排程時播放）；另備「示範寫入工作副本」的片段
  - `月結進度儀表板.html` 預跑結果：備一份事先生好的版本（以備案方式生成），萬一現場 demo 卡住直接開給學員看
- **學員端**：本單元不需任何環境，只需能看到投影
- **credential / 權限**：Gmail 外掛授權需「信箱讀取」最小權限（第一版排程只讀取不修改）；demo 期間使用預設沙盒模式，不開 full access（full access 在背景自動化情境風險高）
- **現場 Q&A 預備（常見問題）**：
  - Q：「排程會一直跑嗎？電費／額度怎麼算？」→ 排程只在你設的時間點啟動一次，不是 24 小時待機；執行消耗的是 Codex 的對話次數配額，建議 Plus 方案每天 2–3 次排程綽綽有餘。
  - Q：「關電腦之後排程還會跑嗎？」→ 不會，Codex 排程依賴本機 App，電腦睡眠或關機時排程不觸發；常見錯誤第 3 條有說明。
  - Q：「為什麼第一版不直接讓它改 Excel？」→ 排程跑在背景、你不在場；先看清單確認幾次輸出都對了，再進階到寫入，而且只寫工作副本。一旦靜默寫錯了正式帳冊，補救比等多一天確認麻煩得多。
  - Q：「外掛 Skill 要付費嗎？我找不到安裝入口？」→ 本單元以 Codex 內建 HTML 生成為主線，不依賴特定 Skill；如帳號有相容 Skill 可加用，沒有也不影響學習成果。

## 商業情境案例（Case）

**角色**：你（弄一下工作室 行政會計）
**公司**：弄一下工作室（約 10 人的小型工作室）
**任務**：6 月底對帳期，12 家廠商的回覆信散落在 Gmail；老闆每天早上問進度。你要設一個排程讓 Codex 每天早上自動掃信、整理出待更新清單，確認正確後再更新工作副本；並做一個 HTML 儀表板讓進度一眼看清楚。
**情境細節**：
- 供應商彙整表有 12 筆廠商資料，「回覆狀態」欄目前只有 2 筆填了「已回覆」，其餘 10 筆空白
- 陳主管要你每天早上 9 點前把對帳回覆率更新好；但 Gmail 裡廠商的信有時當天凌晨才進來
- 老闆問「還差幾家」的方式是走進來直接問，你沒時間當場打開 Excel 算——需要一個一眼看完的版本
**本單元要你學會**：看懂「排程如何設、護欄怎麼寫、儀表板數字從哪來、哪裡必須人工核」——這四件事都在示範裡，示範後你自己試之前先走過觀察單的三個問題。

## 動手練習題（Hands-on Exercise）

本單元無上機實作（排程設定涉及背景執行 + Gmail 讀取，新手直接設有誤操作風險；形式為示範，觀察後再回家嘗試）。以「設前把關清單」替代：

**題目**：如果你自己要設一個「每天早上 8 點掃廠商回覆信 + 更新 Excel」的排程，在輸入指令之前，你會先確認哪三件事？

**預期成果**：三個具體問題或確認點。好的答案例如「確認 Codex 的 Gmail 只有讀取授權、沒有刪信或寄信權限」。不好的答案：「確認 Codex 設定正確」（太籠統）。

**完成標準**（self-check）：
- [ ] 有一條是關於**權限範圍**（它被允許做哪些動作）
- [ ] 有一條是關於**找不到對應資料時 Codex 該怎麼辦**（停下 or 亂填）
- [ ] 有一條是關於**執行後你怎麼確認它做的事符合預期**

> 這三條合在一起就是一個排程設置的最低安全檢查清單。回家自己設排程之前，把這三條回答完，再動手。

## 常見錯誤 3 條（Common Pitfalls）

1. **錯誤現象**：排程指令寫「把廠商回覆整理好並更新 Excel」，隔天發現 Codex 把幾格廠商名稱改掉了——原來有兩家廠商簡稱不同（「弄一下」vs「弄一下工作室」），Codex 自己做了合併，直接寫進了 Excel。
   **原因**：第一版就讓排程直接寫入 Excel，沒有先輸出清單讓人確認；指令也沒有說「找不到對應廠商名稱時停下、不自行判斷」，Codex 在沒有護欄的情況下靜默做了決定。
   **解法**：第一版排程只輸出待更新清單，確認兩三次輸出都正確再進階到寫入；寫入時只寫工作副本。排程指令裡一定要加「若無法確認對應關係，跳過並記錄待確認，不自行填入或合併」。護欄要在設排程前就寫進去，事後補救比較麻煩。

2. **錯誤現象**：儀表板顯示「已回覆率 50%」，拿給老闆看，但實際上 Excel 在排程跑後又多了 3 家廠商回覆，儀表板沒更新——老闆以為還差 6 家，其實只差 3 家。
   **原因**：儀表板是某個時間點的快照，沒有「即時抓資料」能力。生成後 Excel 更動，頁面不會跟著變。
   **解法**：每次拿儀表板給人看之前，先重跑一次（重下指令讓 Codex 用最新 Excel 重生成）；或者把「重建儀表板」這個步驟也排進每天的排程，確保兩者時間點一致。

3. **錯誤現象**：排程設好了，但隔天 Codex 沒有跑——原來電腦睡眠了，或 Codex App 沒有開著。
   **原因**：Codex 排程依賴 App 本地執行，電腦關機、睡眠或 App 退出時排程不會觸發。
   **解法**：排程時間設在你通常電腦開著的時段（不要設凌晨）；第一次設完，隔天早上主動確認一次執行紀錄，確認排程有跑到再信任它。

## 檢核題 2 條（Quiz）

**Q1（概念驗證）**：Codex 的 HTML 儀表板做好之後，廠商又寄來一封確認信，Excel 更新了回覆狀態。此時儀表板裡顯示的回覆率會怎樣？
- [ ] A. 自動更新，即時顯示最新數字
- [ ] B. 不會自動更新，數字停在 Codex 生成那個時間點的快照 ←（正確答案）
- [ ] C. 儀表板關掉重開就會更新
- [ ] D. 要等下一次排程跑完，儀表板才會自動刷新

**Q2（應用驗證）**：你設了一個排程「每天早上 8 點，Codex 自動掃 Gmail 找廠商回覆信，把結果記到 Excel」。你隔天早上打開 Excel，發現一筆廠商名稱被填成「弄一下」而不是 Excel 裡的「弄一下工作室」，而且 Codex 還自己判斷成功更新了。你下一步應該怎麼做？
- 預期答案要點：
  1. **先確認這筆填入是否正確**（兩個名稱是不是同一家廠商）——不能直接信任 Codex 的判斷；若確認是同一家，要在 Excel 補齊廠商名稱寫法，讓日後比對不再出現差異
  2. **修改排程指令**，加入「找不到完全一致廠商名稱時停下、記錄待確認、不自行填入或合併」的護欄；排程護欄要在事前設好，不是出事後才補
  3. **回查這次排程的其他輸出**，看有沒有其他筆也被靜默改動——不只修那一筆，要全面確認執行日誌，確保只有這一筆例外
