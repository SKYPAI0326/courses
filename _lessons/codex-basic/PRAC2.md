## 教學流程（Teaching Flow）

### 破題 / Hook（2 min）

昨天你把弄一下工作室月結的第一批資料整理完了。報帳單統一命名、彙整進 `月結彙整.xlsx`、供應商清單去重、Checks 工作表也跑過了。以為可以收工。

今天早上，陳主管傳訊息過來：

> 「業務部昨天忘記交，剛剛傳過來了，麻煩幫我整進月結表。對了，他們也補了一份供應商更新清單，一起處理一下。」

三份報帳單、一份供應商補交清單，都丟在你的信箱附件裡，命名依然一團亂。

這 15 分鐘，**你要獨力完成**：

1. 把三份補交報帳單的檔名改成統一格式
2. 把資料彙整進原本的 `月結彙整.xlsx`（不能覆蓋掉舊資料）
3. 把補交的供應商清單合併去重進 `供應商_合併去重.xlsx`
4. 更新 `處理日誌.txt`，把這次處理的三份檔案補記進去

不會有人在旁邊帶你。這正是演練的目的——確認你真的學會了，而不是只是跟著做過。

---

### 概念 / Concepts（2 min）

這次用到的技能，CH2-1 和 CH2-2 都學過。做之前先確認自己記得：

- **批次重新命名** — 一句提示詞讓 Codex 掃資料夾裡的指定檔案，統一格式。抓不到欄位值時，要求它停下來告你，**不准它猜**。

- **@ 引用** — 在輸入框打 `@`，從候選清單選定你要讓 Codex 操作的那份檔案，精確指定、不波及其他的。

- **彙整（Consolidate）** — 把新的來源資料列，**附加**進既有的 Excel，不覆蓋舊資料。提示詞要明確說「加在原有資料下方」。

- **去重（Deduplication）** — 合併兩份清單時，告訴 Codex 以哪個欄位判斷重複，以及衝突時保留哪份。這個判斷必須由你決定，不能讓 Codex 自己定。

- **更新處理日誌** — 用 @ 引用現有的 `處理日誌.txt`，請 Codex 把新處理的三筆追加到文件尾端，不刪舊紀錄。

把這五個技能確認過，準備進任務。

---

### 題目情境 / Scenario（3 min）

**情境**：你是弄一下工作室的行政會計。今天早上收到業務部補交的三份報帳單，附件檔名分別是：

- `業務費用0613.docx`
- `六月業務報帳-最終.xlsx`
- `業務部_報帳單_陳小輝.docx`

同時補交的還有一份供應商更新清單：`業務部廠商0613.csv`

你要完成的事：

**第一件：把三份報帳單改名**

統一成「部門-月份-報帳單」格式（和 CH2-1 一樣的格式），部門從檔案內容裡抓。

> ⚠️ 把關：改名前先把這三份原檔複製一份到另一個地方備用。正式的 `月結彙整.xlsx` 已有舊資料，這批是新的——不要把備份存在同一個資料夾裡弄混。

**第二件：彙整進既有 `月結彙整.xlsx`**

不是建新的 Excel，而是把這三份的資料追加到 `月結彙整.xlsx` 原有資料的下方。欄位對齊方式和第一批一樣（日期、部門、品項、金額、核銷人）。

> ⚠️ 把關：彙整完後，立刻打開 `月結彙整.xlsx`，數一下總列數是否等於原來的列數加三份的資料列數。Codex 不會告訴你它有沒有漏掉哪一份——你要自己算。

**第三件：合併去重供應商清單**

把 `業務部廠商0613.csv` 和現有的 `供應商_合併去重.xlsx` 合併。以「統一編號」為準去重，有差異的筆在備注欄標出來請人工確認。

> ⚠️ 把關（會計紅線）：去重條件由你在提示詞裡指定——「以統一編號判斷重複」「衝突時保留哪份」這兩件事 Codex 不知道，必須你說清楚才能做對。重複廠商的複合判斷鍵為：**發票字軌號＋廠商統編＋日期＋金額**，缺任一項即視為「疑似重複報支—待確認」，不得讓 Codex 自行判斷是否為真重複。稅別、科目、入帳方式、扣抵資格、是否資本化等判斷，Codex 一律標「**待覆核**」，不自行填入。

**第四件：更新處理日誌**

用 `@處理日誌.txt` 引用現有日誌，請 Codex 把這次三份補交報帳單的原始檔名和新檔名追加到文件末尾，**不刪除之前的紀錄**。

> ⚠️ 把關：更新後打開日誌，確認舊紀錄還在（CH2-1 的 6 筆沒有消失），新增的 3 筆在最下方。如果舊紀錄不見了，不要覆蓋存檔，先找備份還原。

---

### 動手 / Hands-on（6 min・學員獨立完成）

打開 Codex，進入「弄一下工作室_6月月結」專案，開一個新的專案對話（點專案資料夾旁的 + 號）。

**使用講師發的 checkpoint 資料包**：解壓後，把整個資料夾直接載入 Codex 專案——裡面已包含 `月結彙整.xlsx`、`供應商_合併去重.xlsx`、`處理日誌.txt`、`AGENTS.md` 以及 `PRAC2_補交/` 資料夾（含三份報帳單和一份供應商清單）。不需要自己準備前面 CH2-1/CH2-2 的輸出物。

確認資料包已載入後，依序完成四件事。

**不要照著別人做——先想好提示詞再送出**。每一步做完，自己確認後再繼續下一步。

提示：若 Codex 中途遇到缺欄位值的情況，它應該停下來告你。如果它沒有停、反而填了一個猜測值進去，表示你的提示詞沒有說清楚「缺資料即停」——在那句指令前加上這條約束，重新送出。

### 檢核 / Verification（2 min）

四件事都做完後，逐項對照下方清單。**全部打勾才算完成**。

**完成標準**（self-check）：

**命名**
- [ ] 三份補交報帳單的檔名已改成「部門-月份-報帳單」格式
- [ ] 原始三份檔案有備份（不在同一個月結資料夾內）
- [ ] 若有報帳單抓不到部門，Codex 有停下來告知而非自行補名（如果遇到這種情況）

**彙整**
- [ ] `月結彙整.xlsx` 的列數 = 舊資料列數 + 這三份的資料列數（打開算一遍）
- [ ] 新資料的欄位名稱和舊資料對齊（沒有多出奇怪的欄位、也沒有欄位對到錯的那一列）
- [ ] Codex 沒有在空白欄位自行填值（空著就是空著，或標「待補」）

**去重**
- [ ] `供應商_合併去重.xlsx` 有更新（最後修改時間是剛才）
- [ ] 備注欄有標出差異廠商（你能指出是哪幾家需要人工確認）
- [ ] 去重條件是你在提示詞裡指定的（不是讓 Codex 自己決定）
- [ ] 疑似重複報支的筆，備注欄標「疑似重複報支—待確認」（複合鍵：發票字軌號＋廠商統編＋日期＋金額，缺任一即標）
- [ ] 稅別、科目、入帳、扣抵、資本化等欄位，Codex 輸出一律標「待覆核」而非自行填入判斷

**日誌**
- [ ] `處理日誌.txt` 打開後，CH2-1 的舊紀錄還在（6 筆沒有消失）
- [ ] 新增的 3 筆在日誌最下方，原始檔名和新檔名都有記錄

**本次演練產出物**：
- 三份已命名的業務部補交報帳單
- 更新後的 `月結彙整.xlsx`（含原有 + 新增共 9 份報帳單資料）
- 更新後的 `供應商_合併去重.xlsx`
- 更新後的 `處理日誌.txt`（共 9 筆，6 舊 + 3 新）

這四樣都確認完，弄一下工作室 6 月月結的第二批補交資料整理完成，可以交給陳主管。

## 動手練習題（Hands-on Exercise）

**題目**：

月結第二天早上，業務部傳來三份補交報帳單和一份廠商清單。

使用**講師發的 checkpoint 資料包**（解壓即得），裡面已有：
- `月結彙整.xlsx`（含 6 份已整理的報帳單資料）
- `供應商_合併去重.xlsx`（A.csv + B.csv 去重後的版本）
- `處理日誌.txt`（記錄了 CH2-1 處理的 6 份報帳單）
- `AGENTS.md`（含硬規則）
- `PRAC2_補交/`（三份補交報帳單 + 供應商清單）

你要用 Codex 完成四件事：

1. 把三份補交報帳單改名為「部門-月份-報帳單」格式
2. 把三份報帳單的資料彙整進 `月結彙整.xlsx`（附加在舊資料下方，不覆蓋）
3. 把業務部廠商清單合併去重進 `供應商_合併去重.xlsx`
4. 把這三份補交報帳單的原始檔名和新檔名追加進 `處理日誌.txt`

## 常見錯誤 3 條（Common Pitfalls）

1. **錯誤現象**：彙整完後打開 `月結彙整.xlsx`，發現原本的 6 筆資料不見了，整份 Excel 只剩這次新加的 3 筆。
   **原因**：提示詞只說「把資料彙整進 月結彙整.xlsx」，沒有明確說「附加在現有資料下方，不刪除也不覆蓋原有資料」。Codex 的預設行為有時是直接產一張新表——它不知道你要的是「追加」而不是「重建」。
   **解法**：在提示詞裡加上「在現有 Excel 資料的最後一列下方追加，不刪除任何現有列，欄位標題不重複插入」。如果舊資料已消失，從備份還原後重新跑，並在提示詞裡補上這條約束。

2. **錯誤現象**：Codex 把補交的日期欄空白報帳單，自己填了「2026-06-13」進去，沒有停下來告知。
   **原因**：提示詞沒有說「缺資料就停、不要補值」，或 `AGENTS.md` 的硬規則有，但學員的提示詞沒有把這條約束帶進去（兩者要互相補強，不能只靠其中一個）。
   **解法**：提示詞結尾加上「如果任何欄位抓不到確定值，不要自行填入推測值——停下來列出哪份有問題，等我告訴你怎麼處理」。已被填值的欄位，從備份還原原始資料，重新跑。確認 `AGENTS.md` 裡也有這條規則。

3. **錯誤現象**：去重後供應商清單裡，某家廠商的統一編號消失了——既不在主清單，也不在備注欄，就像「被刪掉了」。
   **原因**：提示詞只說「去除重複」，沒有指定去重欄位，也沒說「差異筆要留在備注欄」。Codex 可能用廠商名稱精確比對（格式略有差異就不視為重複，或反過來多刪掉了），備注欄保留規則也不清楚。
   **解法**：提示詞要明確：「以統一編號為準判斷重複；有差異的筆保留在備注欄標「請人工確認」，不要靜默刪除任何一筆」。已跑壞的結果不要存檔，從原始兩份 CSV 重跑，提示詞補上以上兩條。

## 檢核題 2 條（Quiz）

**Q1（概念驗證）**：你要把業務部補交的三份報帳單資料彙整進已有舊資料的 `月結彙整.xlsx`，提示詞裡最重要的一句話是什麼？

- [ ] A. 「幫我彙整這三份進 月結彙整.xlsx」
- [ ] B. 「把這三份的資料，附加在 月結彙整.xlsx 現有資料的最後一列下方，不刪除也不覆蓋任何現有資料」 ←（正確答案）
- [ ] C. 「把 月結彙整.xlsx 重建一次，這次把所有報帳單都放進去」
- [ ] D. 「先把 月結彙整.xlsx 備份一份，再幫我彙整」

**Q2（應用驗證）**：業務部廠商清單和現有的供應商清單合併去重後，你要怎麼確認這次去重跑對了、沒有漏掉或多刪掉廠商？

- 預期答案要點：
  1. **對照列數**：合併後的清單列數，應介於「A+B+業務部補交三份來源總列數」和「只保留一份的最大清單列數」之間，偏差太大代表有問題
  2. **看備注欄**：有差異的廠商必須在備注欄標出來，如果備注欄是空的但你知道兩份清單有相同廠商，代表去重條件或備注規則沒有生效
  3. **去重欄位要核對**：提示詞是否真的說了「以統一編號判斷重複」？如果只說「去除重複」，Codex 可能用名稱比對，格式略有差異就不視為同一家
