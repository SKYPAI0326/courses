## 教學流程（Teaching Flow）

### 破題 / Hook

這 20 分鐘結束，你手上有兩個具體成果：

1. 一個可運作的「弄一下工作室_6月月結」Codex 專案（含 `AGENTS.md` 操作守則）
2. 一份 Codex 讀檔回報——它列出資料夾裡看到哪些檔、指出空值、重複、命名不一致、金額異常

CH1-2 做完後，Codex 認得你的身份（弄一下工作室的行政會計、你的口吻偏好），但它還不知道你這個月在做什麼、資料夾裡有哪些檔、命名要用什麼格式。本單元補上這三件：

1. 建「弄一下工作室_6月月結」資料夾——Codex 這個月處理月結時讀寫的範圍。
2. 在資料夾裡放一份 `AGENTS.md`（操作守則）——寫清楚怎麼幫你工作、哪些事不能做。
3. 把素材包放進去，請 Codex 讀完回報它看到什麼、發現哪些問題，當場驗證 `AGENTS.md` 真的生效。

---

### 概念 / Concepts

- **工作資料夾** — Codex 動手的範圍。一般專案工作以你指定的資料夾為主；實際能讀寫的範圍取決於權限模式、外掛與 Computer Use。把所有月結相關檔案丟進來，它才看得到。
- **`AGENTS.md`（操作守則）** — 一份放在工作資料夾裡的純文字文件，檔名就叫 `AGENTS.md`。每次 Codex 開啟這個專案的對話，它都會先把這份文件讀過一遍。你在裡面寫的規則，它每次都會記得。
- **專案（Project）** — Codex 裡的一個工作單位，對應你的工作資料夾。建專案時你選哪個資料夾，Codex 就在哪裡工作、`AGENTS.md` 也從那裡讀。
- **三者的關係** — 工作資料夾是 Codex 工作的位置，`AGENTS.md` 是它每次開工先讀的規則，專案把這兩者綁在一起：開了專案，它才同時知道「去哪裡工作」和「按什麼規則工作」。
- **為什麼用 `AGENTS.md`，而不是每次在對話裡講一遍** — 寫進 `AGENTS.md`，規則每次開新對話自動帶入、每次一致；用講的，你會漏講、會講得每次不一樣，Codex 的行為就跟著不穩。`AGENTS.md` 要短而精準，只放「每次都用得到」的規則。一次性的細節（這次某張表要特殊處理）寫在當下的提示詞就好，不要堆進去——它越長，Codex 每次讀的負擔越重、抓重點越差。真正冗長的固定流程之後做成 Skill（CH3-1），不塞在這裡。

### 操作步驟（跟著做）

四個步驟，每步做完確認後再繼續。

---

#### 步驟 1：建工作資料夾

在電腦桌面或你習慣的位置，建立一個新資料夾，命名為：

```
弄一下工作室_6月月結
```

命名就用這個，後面所有操作都要對到這個資料夾。

---

#### 步驟 2：寫 `AGENTS.md` 操作守則

打開電腦的文字編輯工具（Windows：記事本；Mac：文字編輯器，記得選純文字模式），輸入以下內容：

```
# 弄一下工作室 6月月結 — Codex 操作守則

## 背景
公司名稱：弄一下工作室（約 10 人的小型工作室）
任務週期：2026 年 6 月月度報帳與供應商整理

## 檔案命名規則
- 報帳單：部門-月份-報帳單（例：業務部-2026-06-報帳單）
- 供應商清單：供應商清單-來源名稱-日期（例：供應商清單-A源-2026-06-15）
- 彙整輸出：必須在檔名結尾加「-彙整」（例：費用彙整-2026-06-彙整）

## 硬規則（不得違反）
1. 缺資料即停：欄位空白或資料不足時，停下並回報哪裡缺、缺什麼，不得繼續處理。
2. 不得自行補值：任何欄位的數值或文字，只能來自原始檔案，不得推估、填入預設值或自行造數字。
3. 模糊標待覆核：遇到無法判斷分類、金額異常、重複項目等，在輸出檔案旁邊標「待覆核」，不得自行決定。

## 輸出習慣
- 每次處理完畢，用一段話列出做了什麼、產出哪些檔案
- 如有發現異常，一定要在回報裡單獨列出，不可略過
```

素材包裡已附這份 `AGENTS.md` 範本，可以直接複製貼入、存檔，不必從頭手打。

存檔時注意兩件事：

> **把關提醒 1：存檔位置與檔名**
> 必須存在「弄一下工作室_6月月結」資料夾**裡面**，檔名必須完整打成 **`AGENTS.md`**（大寫、.md 結尾，不能是 `AGENTS.txt`、`agents.md` 或 `操作守則.md`）。
> Codex 只認這個精確的檔名與位置——名稱或位置錯了，它讀不到，規則完全不發揮作用。

確認資料夾裡出現 `AGENTS.md` 一個檔案後，繼續步驟 3。

---

#### 步驟 3：回 Codex 建專案、選模型

打開 Codex 桌面 App，在首頁找到建立新專案的入口，選擇「**使用現有資料夾**」，進入到剛才建立的「弄一下工作室_6月月結」資料夾，點「打開」。

模型設定：
- 模型：**GPT-5.5**（預設值，維持不動）
- 速度：**標準**
- 推理強度：**中**

這組設定處理月結這類日常財會任務綽綽有餘，不用調。

專案建立完成後，你會看到 Codex 的對話視窗，旁邊的側欄有你的專案名稱。到這步，Codex 已經準備好了：它知道要去「弄一下工作室_6月月結」資料夾工作、`AGENTS.md` 也已經在裡面。

---

#### 步驟 4：丟入素材包，請 Codex 讀檔回報

把素材包資料夾裡的全部檔案拖曳到「弄一下工作室_6月月結」資料夾裡。素材包包含：

- 報帳單 ×6：`報帳.docx`、`六月-業務部.xlsx`、`六月費用_行政.xlsx`、`報帳0610設計部.docx`、`費用單(最終版2).docx`，以及另一份（命名雜亂，`.docx`／`.xlsx` 混雜）
- 供應商清單：`供應商_A來源.csv`、`供應商_B來源.csv`（兩來源，有重複廠商）
- 交易明細：`transactions.csv`（含空值、異常金額、重複發票號碼）

拖曳完成後，回到 Codex 的對話框，在專案的 + 號開一個新對話，輸入：

> 請你列出這個資料夾裡看到哪些檔案，並指出空值、重複、命名不一致、金額異常。

送出，等 Codex 回報。

Codex 會列出它讀到的每個檔名，並指出它看到的問題，例如：`transactions.csv` 有空白欄位、`供應商_A來源.csv` 與 `供應商_B來源.csv` 有重複廠商名稱、報帳單檔名不符「部門-月份-報帳單」格式。

**怎麼判斷這份回報可信：** 它列的檔名要對得上你資料夾裡實際的檔。如果它列出你沒放進去的檔、或把檔名講錯，代表它沒真的讀到（可能資料夾選錯、或檔還沒同步）——把資料夾和專案對一次再重送。它連你的檔都讀不準，後面的彙整更不能信。

> **把關提醒 2：回報 ≠ 數據正確**
> Codex 說「我看到 transactions.csv 有 3 個空值」——這句話本身是它掃描資料夾的觀察，通常是對的。但它接下來如果說「我建議這三格填入 X」，那就要你停下來判斷：`AGENTS.md` 裡說的硬規則是「不得補值」，空值應由你決定怎麼處理，不是讓它填。讀檔回報是讓你掌握資料狀況，不是授權它自己解決問題。

---

### 動手練習

1. 確認你的「弄一下工作室_6月月結」資料夾裡有 `AGENTS.md` + 素材包的全部檔案。
2. 在 Codex 的專案裡，用 + 號開一個新對話，送出上方那一句讀檔請求。
3. 看 Codex 的回報，確認它有列出：
   - 資料夾裡的所有檔名
   - 至少一項異常（空值、重複名稱、命名不規則，任一種都算）

如果 Codex 的回報完全沒有提到任何異常，確認 `AGENTS.md` 裡的第三條「模糊標待覆核」有沒有存進去——有時存檔時若副檔名錯了，Codex 讀不到硬規則，就可能漏掉異常。

---

### 對話管理 + 收束

現在你有一個專案，裡面已經有第一條對話記錄。以後開新的月結工作，有兩種情況：

- **新的月結任務**（和「弄一下工作室_6月月結」有關）→ 點專案資料夾旁邊的 **+ 號** 開新對話。Codex 每次從 + 號開的對話，都會重新讀 `AGENTS.md`，規則全在。
- **和月結無關的一般問題**（問它下週行事曆、請它幫你查個資料）→ 點左下角的**「新增對話」**按鈕。這種對話不在專案範圍內，`AGENTS.md` 不會讀入。

> **把關提醒 3：開錯對話型別，規則就失效**
> 如果月結相關任務跑去點「新增對話」，Codex 沒有讀 `AGENTS.md`，你寫的所有命名規則和「不得補值」硬規則都不在它的認知裡。最明顯的症狀：你要它改檔名，它問你要什麼格式——但你明明寫在 AGENTS.md 了。遇到這個，關掉對話，從 + 號重開就好。

不需要的舊對話可以執行「封存」，保留記錄但收乾淨畫面。

## 常見錯誤 3 條（Common Pitfalls）

1. **錯誤現象**：Codex 每次都要重新問你檔案命名規則，好像沒有讀到 `AGENTS.md`。
   **原因**：`AGENTS.md` 存成了 `.txt`（記事本預設存法），或是檔名打成 `agents.md`（小寫 a），Codex 不認識。也可能是 AGENTS.md 存在桌面而不是資料夾裡面。
   **解法**：打開資料夾確認有一個叫做 `AGENTS.md` 的檔案（大寫、.md）。如果是記事本，存檔時把「存檔類型」改成「所有檔案」，並在檔名欄位打 `AGENTS.md`（含副檔名），這樣才不會被加上 .txt。確認正確後，從 + 號重開一個對話，再送一次讀檔請求。

2. **錯誤現象**：Codex 讀完素材包後，主動建議說某個空白欄位可以填入某個值，或者直接把空值填掉了。
   **原因**：`AGENTS.md` 的「不得補值」規則可能沒寫進去，或 Codex 這次對話沒有正確讀到 `AGENTS.md`。
   **解法**：先停下，不要採用它建議或填入的值。打開 `AGENTS.md` 確認三條硬規則都在。如果規則有漏，補上存檔，從 + 號重開對話，重新請它讀檔。缺資料的欄位，怎麼處理由你決定，不是讓 Codex 猜。

3. **錯誤現象**：點「新增對話」開了一個新對話來做月結任務，Codex 不知道公司名稱，也不知道命名格式。
   **原因**：「新增對話」是不帶任何專案脈絡的通用對話，不會讀 `AGENTS.md`。月結相關任務必須從**專案旁邊的 + 號**開對話，才會讀操作守則。
   **解法**：關閉這個對話，回到 Codex 左側的專案列表，找到「弄一下工作室_6月月結」旁邊的 + 號，重新開對話。跟月結有關，走 + 號；跟月結無關，才走「新增對話」。

## 檢核題 2 條（Quiz）

**Q1（概念驗證）**：`AGENTS.md` 必須存在哪裡、叫做什麼名字，Codex 才會讀它？
- [ ] A. 存在電腦桌面，叫做「操作守則.md」就可以
- [ ] B. 存在 Codex App 的設定資料夾，叫做 `AGENTS.md`
- [ ] C. 存在你的工作資料夾裡面，叫做 `AGENTS.md`（大寫、.md 結尾）←（正確答案）
- [ ] D. 放在哪裡都可以，只要在 Codex 裡面手動選取它

**Q2（應用驗證）**：Codex 讀完 `transactions.csv` 後，回報說「第 12 行的金額欄位是空白，建議填入 0」。依照你在 `AGENTS.md` 裡寫的規則，你應該怎麼做？
- 預期答案要點：
  1. 不接受它的建議，不讓 Codex 自行填值——`AGENTS.md` 的硬規則是「不得自行補值」
  2. 自己判斷這格空白的原因——可能是資料來源漏填，可能是那筆就是 0，需要確認原始單據
  3. 確認後才決定怎麼填，或在彙整表上把那格標「待覆核」讓陳主管確認

## 講師授課筆記（不進講義）

> 本區塊不轉成學員 HTML（course-lesson-writer 跳過）。授課調度全寫這裡。

**時間分配（20 min）**：
- 3 min 破題（成果先行）
- 3 min 概念（工作資料夾 / AGENTS.md / 專案 三者關係）
- 10 min 操作示範（跟做 4 個步驟）
    步驟 1：建資料夾（2 min）
    步驟 2：寫 AGENTS.md（4 min，含逐句解釋三條硬規則）
    步驟 3：回 Codex 建專案、選模型（2 min）
    步驟 4：即時兌現——丟入素材包，讓 Codex 讀檔回報（2 min）
- 3 min 動手跟做（學員補完自己的 AGENTS.md，觸發讀檔回報）
- 1 min 對話管理 + 收束（封存 / 新增對話 / + 號）

**授課節奏**：每個步驟講師做完後讓學員跟著做，確認完成再進下一步。不要連續示範多步再讓學員一次補做。

**步驟 4 示範**：講師先建好完整專案（含 AGENTS.md + 素材包），速度比學員快一步。送出讀檔請求後一邊等一邊說明預期回報長相，結果出來再回看。

**備案（如學員 Codex 反應慢）**：步驟 4 改由講師單獨示範，學員在課後用自己的素材包補跑。

**把關提醒講解順序**：
- 步驟 2 存檔後立即講「把關提醒 1」（存錯副檔名）
- 步驟 4 回報出來後立即講「把關提醒 2」（回報 ≠ 數據正確）
- 收束時講「把關提醒 3」（開錯對話型別）

**示範用 AGENTS.md 反例**（選用）：預備一個存成 `AGENTS.txt` 的版本，示範 Codex 讀不到時的症狀（讀檔回報沒有應用命名規則）。

**對話管理收束話術**：「+ 號開的對話才帶著月結的規則，以後凡是月結的事，記得走 + 號。」
