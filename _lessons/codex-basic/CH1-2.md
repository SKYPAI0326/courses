## 教學流程（Teaching Flow）

### 破題 / Hook（1.5 min）

CH1-1 你看到 Codex 把一包亂命名的報帳單整理成乾淨的 Excel，但那個版本是**講師預先設定好**的 Codex——它知道主人是誰、在哪間公司做什麼。

你的 Codex 剛裝好時，什麼都不認識你。你問它「這封對帳信怎麼寫？」，它不知道你是弄一下工作室的行政會計、不知道你要用正式還是輕鬆口吻、不知道你要繁體中文簡短回答。它只能給你一個通用版的回答，跟問路人借火柴一樣尷尬。

這一頁做完，你的 Codex 就認得你。CH3-2 你請它幫你草擬對帳催收信的時候，口吻和語氣就是今天填的設定在兌現。

### 概念 / Concepts（1.5 min）

- **自訂指令（Custom Instructions）** — 告訴 Codex「你是誰、做什麼、喜歡怎樣的回答」。設定一次，之後每次對話都帶著走。填了「弄一下工作室行政會計、繁體中文、簡短重點式」，它之後不會跟你廢話，也不會突然改英文。
- **記憶（Memory）** — 啟用後，Codex 從你的對話中累積你的偏好與習慣，提供更個人化的服務。開關在設定頁面，隨時可以開或關。
- **核准提示（等待核准）** — Codex 每次要動你的檔案，預設會跳出「等待核准」視窗，讓你決定是否放行。這是你的第一道防線，**不要急著關掉它**。
- **權限模式（Permission Mode）** — 控制 Codex 的讀寫範圍。官方三層：`:read-only`（唯讀，不寫任何檔）/ `:workspace`（只允許在工作資料夾內寫入）/ `:danger-full-access`（解除本機沙盒限制，任意讀寫）。GUI 可能顯示友善標籤，概念一致。差別見下方操作示範。

### 操作示範 / Demo（6.0 min）

#### 步驟 1：下載安裝（1 min）

在瀏覽器開啟 ChatGPT 官網（chatgpt.com），找到 **Codex** 的下載連結，下載對應你的作業系統（Windows / macOS）的安裝檔，照一般 App 安裝流程完成。

安裝完成後打開 Codex：介面和 ChatGPT 非常類似，中間是輸入框，左側是對話記錄面板。先別急著打字——先讓它認識你。

#### 步驟 2：自訂指令——讓它認得你（1.5 min）

點左下角「**設定**」→ 切換到「**個人化**」類別 → 找到「**自訂指令**」欄位。

在欄位裡填入你的身份、職務與回覆偏好。以本課的情境為例：

```
我是弄一下工作室的行政會計人員，主要負責月結報帳、供應商對帳與費用彙整。
請用繁體中文回覆，語氣正式但簡潔，以重點條列為主，不需要過多解釋說明。
不確定的地方請直接告訴我，不要自己補填資料或猜測金額。
```

填好後點「**儲存**」。

> 🔴 把關提醒：這裡的「不確定的地方請直接告訴我，不要自己補填資料或猜測金額」這句話，是在提示詞層面預裝一道防線。它不保證 Codex 100% 遵守，但能大幅降低它自己腦補資料的機率。在 CH1-3 的 `AGENTS.md` 還會再加一層。

#### 步驟 3：記憶開關（0.5 min）

在「個人化」頁面的下方，有「**記憶**」設定。啟用後，Codex 會從對話中記錄你的偏好（例如：你之前說過喜歡月結 Excel 統一用某格式）。下次不用重複交代。

建議：現在啟用。如果日後覺得它記錯什麼，可以在記憶管理頁面手動刪除特定項目。

#### 步驟 4：認識「等待核准」彈窗（1 min）

關掉設定頁面，回到主畫面。等一下在 CH1-3 建立專案、請 Codex 動你的工作資料夾時，它每次要讀取或修改檔案，都會彈出「**等待核准**」的提示視窗。

你會看到兩個選項：
- **是**：只允許這一次
- **此工作階段不要再詢問**：本次工作階段內不再問（重新開啟後恢復）

這個彈窗是 Codex 問你「我可以動這個檔嗎？」——**這是好事，不是麻煩**。月結帳的情境下，每一步你都應該知道它在動哪個檔。

> 🔴 把關提醒：不要一看到彈窗就習慣性按「此工作階段不要再詢問」。尤其第一次使用，建議一個一個按「是」，確認它動的是你要它動的那個檔。

#### 步驟 5：選對權限模式——這頁最重要的一個決定（2 min）

彈窗下方有一個按鈕，讓你切換整體的「**權限模式**」。官方三層概念如下：

| 權限層 | 官方標識 | 讀寫範圍 | 風險 | 適合情境 |
|--------|---------|---------|------|---------|
| 唯讀 | `:read-only` | 只讀、不寫任何檔 | 最低 | 查閱、分析、不需改檔 |
| **工作資料夾**（本課起點） | `:workspace` | 只允許在指定工作資料夾內寫入 | 中等 | 月結整理、日常工作 |
| 解除沙盒 | `:danger-full-access` | 解除本機沙盒限制，任意讀寫 | 最高 | 能不開就不開 |

GUI 介面可能顯示不同的友善標籤（如「預設核准」「自動審核」「完整存取」），概念對應上表三層。

**本課從 `:workspace`（工作資料夾模式）起步——把 Codex 的讀寫限制在你指定的工作資料夾內，原始財務檔案放在資料夾外就動不了。月結帳情境下，`:danger-full-access` 能不開就不開。**

為什麼？`:danger-full-access` 解除本機沙盒限制，等於告訴 Codex：「你想改哪個檔都行，不用問我。」如果它誤會了你的指令、或 `AGENTS.md` 沒寫清楚，它可能動到你沒預期的檔，而實際讀寫範圍還會受外掛與 Computer Use 影響，不容易事後追查。

> 🔴 把關提醒（本單元最重要）：`:danger-full-access` 模式下，Codex 不會問你就改檔，且實際讀寫範圍取決於權限模式、外掛與 Computer Use。月結帳、對帳明細、供應商清單——這些都是有責任的數字。遵守最小權限原則：**從 `:workspace` 起步，`:danger-full-access` 是你評估清楚背景排程風險後才開的選項，不是預設值**。

### 動手 / Hands-on（3.0 min）

你來跟做：

**步驟 1**：打開剛安裝好的 Codex。

**步驟 2**：進「設定」→「個人化」→「自訂指令」，填入你自己的身份、職務與回覆偏好。至少要包含：
  - 公司名稱（弄一下工作室）
  - 你的職務（行政會計）
  - 回覆語言（繁體中文）
  - 回覆風格（簡短重點式）
  - 一句「不確定就停、不要自行補資料」的指示

**步驟 3**：確認「記憶」設定已啟用。

**步驟 4**：回到主畫面，在輸入框打這句話送出：

> 請用我設定的身份和口吻回我：我是什麼角色、平常負責哪些工作？

看 Codex 怎麼回答。它應該根據你剛才填的自訂指令，用你給的身份口吻和語言回答你——這就是「設定即兌現」的第一個確認點。

**步驟 5**：確認你的權限模式設定在 `:workspace`（工作資料夾）或更保守的選項。不要切換成 `:danger-full-access`（解除沙盒）。

### 檢核 / Verification（1.0 min）

學會的標準（上機確認）：

- Codex 在剛才那句測試問話中，**用繁體中文、用你填的身份口吻**回答了你。
- 你能說出 **`:workspace` 和 `:danger-full-access` 的差別**，以及為什麼月結帳情境下要遵守最小權限原則、不應該一開就解除沙盒。

兩件都確認，就可以進 CH1-3 建工作資料夾和 `AGENTS.md`——下一步，Codex 不只認識你，還會認識你正在做的這份月結工作。

## 動手練習題（Hands-on Exercise）

**題目**：填完自訂指令後，送出這句測試問話：「請用我設定的身份和口吻回我：我是什麼角色、平常負責哪些工作？」

1. 它有沒有用繁體中文？
2. 它有沒有提到「弄一下工作室」和「行政會計」？

**完成標準**（self-check）：
- [ ] 自訂指令已填寫並點「儲存」
- [ ] 記憶設定已啟用
- [ ] 測試問話的回覆符合你填的語言和口吻
- [ ] 權限模式確認在 `:workspace` 或更保守的選項，未切換成 `:danger-full-access`

## 常見錯誤 3 條（Common Pitfalls）

1. **錯誤現象**：自訂指令填完後，再請 Codex 用你的身份口吻回話，它還是用通用語氣回答、完全沒提到你是弄一下工作室的行政會計，像沒讀到你填的資訊。
   **原因**：填完後忘記按「儲存」，或是按了儲存但還在設定頁面沒有回到對話主畫面——設定沒有生效。
   **解法**：點「儲存」確認彈窗消失後，回到主畫面重新開啟一個新對話再測試。如果原本的對話視窗還開著，要重新開一個對話，設定才會套進去。

2. **錯誤現象**：看到「等待核准」彈窗就覺得很煩，立刻按「此工作階段不要再詢問」，之後 Codex 動了哪些檔案完全不知道。
   **原因**：把「彈窗」當作程式錯誤或打擾，急著消除，結果讓 Codex 在沒有確認的情況下動了不該動的檔。
   **解法**：在 CH1 和 CH2 的學習階段，**每個彈窗都應該逐一確認**。等你熟悉 Codex 的行為模式、確認它真的讀懂了你的指令之後，再考慮切換到「自動審核」——不是現在。

3. **錯誤現象**：權限模式切到「完整存取權」，覺得這樣 Codex 做事最順、彈窗最少。結果在 CH2 練習彙整 Excel 時，發現原始報帳單已被 Codex 改掉了幾個格子，但不確定從哪步開始改的。
   **原因**：「完整存取」讓 Codex 在任何檔案上都不問就動。月結帳的原始來源一旦被改，你再拿去核對就是在對比兩份都可能不準的資料。
   **解法**：切回 `:workspace`（工作資料夾）或更保守的模式。原始報帳單、transactions.csv 這類有責任的源頭檔，在 CH1-3 建工作資料夾時就應該留一份副本不動——讓 Codex 只碰工作副本，且工作資料夾外的原始版不在 `:workspace` 的讀寫範圍內。

## 檢核題 2 條（Quiz）

**Q1（概念驗證）**：在 Codex 的三層權限模式中，`:workspace` 和 `:danger-full-access` 最關鍵的差異是什麼？
- [ ] A. `:workspace` 速度比較慢，`:danger-full-access` 速度比較快
- [ ] B. `:danger-full-access` 不會動你的檔案，`:workspace` 才會 ←（錯誤）
- [ ] C. `:workspace` 把讀寫限制在工作資料夾內；`:danger-full-access` 解除本機沙盒限制，任意讀寫 ←（正確答案）
- [ ] D. 兩者完全相同，只是名稱不同

**Q2（應用驗證）**：你是弄一下工作室的行政會計。今天第一次使用 Codex，準備讓它幫你整理 6 份命名雜亂的報帳單。你應該把權限模式設定成哪一個，並說明理由？
- 預期答案要點：
  1. 選 **`:workspace`**（工作資料夾）——把讀寫範圍限制在你指定的資料夾，遵守最小權限原則。
  2. 不選 `:danger-full-access`——月結報帳單是有責任的財務資料，解除沙盒限制後，Codex 的實際讀寫範圍還受外掛與 Computer Use 影響，超出預期也難追查。
  3. （加分）說出要準備一份「原始檔副本」，讓 Codex 只碰工作副本，不碰源頭檔。
