## 教學流程（Teaching Flow）

### 破題 / Hook（3 分鐘）

晚上 9 點 20 分，手工皂老闆把最後一鍋熱製皂倒進模具，IG 私訊跳了兩封。

一封是原料商寄來的紙本報價單照片——「玫瑰精油、橄欖油、定製皂盒」一口氣 12 項，王經理問「這禮拜方便確認嗎」。另一封是一位老顧客上週才回購第三次，但你下午才從供應商那收到通知：玫瑰精油漲了 22%，你撐不住了，這個月非漲不可。

兩件事都得今晚處理完。你明早 8 點要去市集擺攤，回來後才有空回信會太晚——客戶會跑掉，原料商會以為你不認真。但你只剩 20 分鐘。

「手動把 12 個品項打進 Excel 再寫信」「硬擠出『不得不調整價格』那封要寫五次才不冒犯人的漲價通知」——兩件事加起來，一個晚上就沒了。

今天這 25 分鐘，我要給你兩個 Prompt，把這 20 分鐘的事壓到 5 分鐘：
- **EF-02**：拍照 → 結構化資料 → Email 草稿，一條龍做完
- **EF-04**：一次生三版本語氣（正式 / 溫和 / 友善），你只要挑一版改兩句就能送出

### 概念 / Concepts（2 分鐘）

- **多模態** — AI 除了會讀字，也會看圖、會聽聲音。手機拍一張照片直接丟給它，就能讀出裡面的字，不用自己打。本課只要記住一件事：**免費版的 ChatGPT 跟 Gemini 都會看圖**。
- **結構化資料** — 把雜亂的文字整理成有欄位的表格樣子（項目 / 單價 / 數量 / 小計），比讀原圖還快。AI 整理完你複製貼上就能直接用。
- **三版本策略** — 棘手的信一次請 AI 寫三種語氣給你挑，比一次寫一版要求它改五次還快。你挑對語氣只要改人名、數字、關鍵句，就送得出去。
- **退路（不會時怎麼辦）** — 五零件第五件。拍照模糊看不清、資料抓不到，要 AI 標記 `[無法辨識]` 不要猜。這是 EF-02 的命脈。
- **下班腦** — 累的時候最容易寫信寫到情緒上頭。這時候讓 AI 先打草稿、你只做最後一關的挑版與改字，比從零寫安全。

### 操作示範 / Demo（10 分鐘）

#### 情境 1：EF-02 拍照報價單（講師示範 3' + 學員動手 7'）

**示範步驟（講師在前面用手機操作，投影到螢幕）**：

1. 打開 ChatGPT App（或 Gemini App）
2. 點輸入框左邊的 **+** 號 → **拍照** 或 **從相簿選擇**
3. 對著紙本報價單拍一張（提醒：光線夠、不歪斜、整張入鏡）
4. 貼上 EF-02 Prompt（見試跑包），填好兩個空格：
   - `[文件類型]` 填「報價單」
   - `[你的身份]` 填「手工皂創業者 / 小晴工坊」
5. 送出，等 15–30 秒
6. AI 會同時給你：**結構化資料清單** + **Email 草稿**
7. 長按草稿 → 複製 → 貼到 Gmail App

**手機操作重點**：
- 拍照前先擦鏡頭。指紋油漬會讓 AI 把「45」看成「46」
- 紙本反光時側一點角度拍，不要硬用閃光燈
- ChatGPT App 上傳圖片有時要等 loading 轉完再打 Prompt，提早貼會卡住

**卡關三種處理**：

| 症狀 | 原因 | Plan B |
|---|---|---|
| AI 把 45 看成 48 | 照片模糊或反光 | 補拍一張清楚的，或直接改字。Prompt 裡加一句「這張若看不清請標記 `[無法辨識]`」 |
| AI 抓錯欄位（把「數量」當成「單價」） | 報價單版型亂、AI 腦補 | 把 Prompt 裡的欄位寫得更白：「項目 / 每單位 NT$ / 幾個 / 小計 NT$」 |
| Email 草稿落款名字不對 | 沒填 `[你的身份]` 空格 | 回到 Prompt 把空格補齊再送一次，不要自己手改——下次又會忘 |

#### 情境 2：EF-04 漲價通知（講師示範 4' + 學員動手 8'）

**示範步驟**：

1. 打開 ChatGPT（EF-04 不需要看圖，用 Gemini 也行，但 ChatGPT 的語氣變化比較細，這裡建議 ChatGPT）
2. 貼上 EF-04 Prompt，把 5 個空格填掉：
   - `[棘手的主題]`：漲價通知
   - `[對方是誰]`：長期 VIP 客戶
   - `[我們的關係]`：合作半年、本月第三次回購
   - `[我想達到什麼]`：把玫瑰皂從 250 漲到 275（+10%），保住客人
   - `[我不想達到什麼]`：顯得冷漠、讓人覺得我在趁機漲價
3. 送出，AI 會一次給你 A / B / C 三版
4. 挑一版當底稿，改人名、改金額、改一兩句聽起來不像自己的話
5. 複製貼到 Gmail

**挑版邏輯（講師現場示範怎麼挑）**：

- **A 正式 + 保守**：適合「對方是陌生客戶」「金額大」「得罪不起」。今晚場景不適合——人家都回購三次了，發公文感
- **B 溫和 + 堅定**：今晚的最佳解。有人味、有原因（原料漲）、有行動（月底前下單原價）
- **C 友善 + 清楚**：適合「交情夠老、可以開玩笑」。但漲價信太輕鬆反而怪，備用

**怎麼「局部改」不用整篇重寫**：
- 選 B 版本後，直接回覆 AI：「把第二段的『基於原物料成本』改成『玫瑰精油漲了快一成』」
- 或者：「結尾加一句：本月底前回購仍享原價」
- **不要**每次都重產三版。你會在挑選上耗掉整晚。

**手機操作重點**：
- 三版太長在手機畫面滑來滑去容易看亂，直接叫 AI「把 B 版獨立完整貼一次」
- 改完的版本請 AI **用條列式列出還建議再檢查哪 3 個地方**——避免寄出後才發現漏了「下一步行動」那句

### Prompt 原始碼

#### EF-02 拍照報價單 → Email 確認信

```
這張照片是一份 [文件類型：報價單 / 合約 / 發票 / 收據]。請執行以下步驟：

1. 辨識所有文字內容
2. 整理成結構化資料（欄位：項目、單價、數量、小計、聯絡人、日期）
3. 模糊或看不清處標註 [無法辨識]，不要猜測
4. 根據結構化資料，草擬一封確認 Email

Email 要求：
- 收件者：[對方公司的聯絡人]
- 我的身份：[你的名字 / 公司]
- 目的：確認上述報價內容並詢問下一步流程
- 語氣：正式但不僵硬
- 150 字內
```

**建議工具**：ChatGPT / Gemini（兩者免費版都支援看圖）

#### EF-04 得罪人風險 Email 三版本

```
我要寫一封 [棘手的主題，例：漲價通知 / 婉拒合作 / 催款]。

背景：
- 對方是誰：[長期客戶 / 新客戶 / 朋友 / 供應商]
- 我們的關係：[合作多久、交情如何]
- 我想達到什麼：[保住關係 / 明確拒絕 / 讓對方付款]
- 我不想達到什麼：[得罪對方 / 顯得冷漠 / 讓步]

請給我「三個不同語氣」的版本：
A. 正式 + 保守：最安全，但可能偏冷
B. 溫和 + 堅定：平衡版本
C. 友善 + 清楚：保留關係感但立場清楚

每則 150 字內。結尾都要給對方一個「下一步行動」。
```

**建議工具**：ChatGPT（語氣變化細膩度 > Gemini）

### 卡關備案（Plan B）

| 場景 | 症狀 | 處理 |
|---|---|---|
| 拍照模糊 | AI 把數字看錯或整欄標 `[無法辨識]` | 重拍（擦鏡頭、側角度避反光）；或直接在 Prompt 回覆「第 2 項單價是 8 元」請它更新 |
| AI 抓錯欄位 | 把「數量」填到「單價」、小計算錯 | 在 Prompt 明確改欄位名：「每單位 NT$ / 共幾個 / 小計 NT$」；再不行請它把原圖文字先「全部逐字列出來」再結構化 |
| 三版本都不滿意 | 全部太冷 / 太卑微 / 太長 | 不要重生三版（浪費時間）。選最接近的那一版，回覆 AI：「B 版再溫一點，但把『撐到現在』改成『實在撐不住』」，局部動 |
| 免費版當天次數用光 | ChatGPT 跳「Free tier limit」 | 切到 Gemini（另一個帳號）繼續，或隔 3–5 小時再試；兩家輪著用幾乎不會卡 |
| 上傳圖失敗 | App 卡在轉圈 | 關掉 App 重開；或把照片先存到相簿再從相簿選；或縮圖後再上傳（手機「編輯→裁切」減檔） |

## 商業情境案例（Case）

**角色**：小晴（手工皂創業者，Style Guide 主線 B）
**背景**：一人創業在家做手工皂，IG + 蝦皮販售，創業半年。忙到每天開工都在跟時間賽跑，下班只剩 20 分鐘處理文書。

**任務 1（EF-02）**：原料商青禾包材王經理傳來紙本報價單，12 個品項、合計三萬多。小晴用手機拍照 → 丟 ChatGPT → 3 分鐘後拿到結構化清單 + 可直接寄的確認 Email，送出收工。

**任務 2（EF-04）**：玫瑰精油漲 22%，撐不住了。小晴要寄漲價通知給 VIP 阿芳姐（半年來回購三次）。用 ChatGPT 一次產三版語氣，選 B 溫和 + 堅定，改兩句話送出。阿芳姐回「理解、這個月先下 10 塊」，客情不破、價格也調上去了。

**本單元要他學會**：
- 拍照 → 結構化 → Email 一條龍做完（EF-02）
- 難寫的信不要硬寫，讓 AI 產三版挑一版（EF-04）
- 挑版本 > 重寫，局部修 > 整段重產

## 動手練習題（Hands-on Exercise）

### 練習 1：EF-02 拍照萃取

**題目**：拿出你手上隨便一張紙本文件（收據、菜單、名片、報價單都行；不方便的用現場發的虛構報價單），用 EF-02 Prompt 跑一次。填好兩個空格：`[文件類型]`、`[你的身份]`。

**預期成果**：
- 手機畫面上同時有「結構化資料清單」與「Email 草稿」
- 若原圖有模糊處，草稿中該欄位應為 `[無法辨識]` 而非 AI 亂填數字

**完成標準**（self-check）：
- [ ] 結構化資料至少 4 個欄位正確（自己對著原文件核對）
- [ ] Email 有明確收件者、自己的身份、主旨（確認報價）、150 字內
- [ ] 落款是你自己的名字，不是 `[你的名字 / 公司]` 還沒填的樣子
- [ ] **資安自檢**：照片裡若有客戶姓名 / 電話 / 地址 / 身分證號，你有沒有先遮掉再上傳？（這題接 CH1-4 資安紅線）

### 練習 2：EF-04 棘手 Email 三版本

**題目**：想一件你最近下不了筆的信（漲價 / 婉拒 / 催款 / 拒合作 / 推掉飯局），用 EF-04 Prompt 跑一次三版本。填好 5 個空格。

**預期成果**：
- 手機畫面上有 A / B / C 三個版本
- 你當場選一版，並能用一句話解釋「為什麼選它」（例：「我跟阿芳姐熟、A 太冷」）
- 對選的那版做至少一處局部修改（改名字、改金額、或改一句話）

**完成標準**（self-check）：
- [ ] 三版本語氣明顯不同（不是只換了開頭的稱謂）
- [ ] 每版結尾都有「下一步行動」（付款期限 / 回覆期限 / 下次見面）
- [ ] 你選出一版並說得出選它的理由
- [ ] 改完的版本不含 `[棘手的主題]` `[對方是誰]` 這種沒填完的方括號
- [ ] 150 字以內（手機畫面一螢幕看得完，不用滑）

## 常見錯誤 3 條（Common Pitfalls）

1. **錯誤現象**：拍報價單時把客戶姓名、電話、地址一起拍進去就直接丟上 AI。AI 正常回你結構化資料，但你已經把客戶個資餵給雲端了。
   **原因**：多模態會把整張圖讀進去，不會挑資訊。你看得到的它都讀得到。
   **解法**：上傳前先用手機「編輯 → 標記 → 塗黑」把個資區塊塗掉，再上傳。或者裁切到只剩「品項 + 單價 + 數量 + 小計」區塊。這件事在 CH1-4 會展開，今天先記住「拍照前看一眼」。

2. **錯誤現象**：AI 把報價單的「內襯貼紙 NT$ 8」看成「NT$ 3」，Email 草稿的金額整個錯。你沒核對就按寄送，對方回信「報價是 8 元不是 3 元」。
   **原因**：相信 AI 的 OCR 是 100% 正確。紙本印刷 + 手機拍照 + AI 辨識，每一關都有可能掉字。
   **解法**：**AI 給的結構化資料，小計總額一定要自己心算一次**。Prompt 加一句「模糊處標 `[無法辨識]` 不要猜」已經能擋掉大多數幻覺，但最後一關還是你自己的眼睛。

3. **錯誤現象**：EF-04 三版本跑出來都不滿意，於是重新填一次 Prompt 再生三版，又不滿意，又重跑。20 分鐘過去，還沒寄出任何一封。
   **原因**：把 EF-04 當成「生成器」而不是「起草器」。AI 不會通靈，你越改背景資訊越模糊，它越寫越飄。
   **解法**：**三版本裡一定有一版最接近**。選那版，回 AI 說「B 版把 X 改成 Y」做局部修改，不要整個重生。通常改 2–3 處就能送出，花不到 3 分鐘。

## 檢核題 2 條（Quiz）

**Q1（概念驗證）**：你要寄一封催款信給遲交兩週的客戶，用 EF-04 生出三個版本後，發現 A、B、C 都太客氣，你希望語氣再硬一點。下列哪個做法最有效率？
- [ ] A. 重新填一次 Prompt，把「我想達到什麼」改成「讓對方立刻付款」，再生三版
- [ ] B. 把三版都複製下來，自己動手融合改寫
- [ ] C. 選最接近的那一版（例如 B 版），回覆 AI：「B 版請再堅定一點，加上『逾期將暫停後續出貨』這句」 ←（正確答案）
- [ ] D. 換去 Gemini 重跑 EF-04，可能語氣會不一樣

**Q2（應用驗證）**：小晴剛收到一張紙本發票，想用 EF-02 拍照產 Email 回傳給會計。發票上印著客戶姓名、電話、送貨地址、信用卡末四碼。她該怎麼處理這張照片才送上 AI？
- 預期答案要點：
  1. **上傳前先遮資**：用手機內建的「標記 → 塗黑」或裁切工具，把客戶姓名、電話、地址、信用卡末四碼全部遮掉或裁掉，只留「品項 / 金額 / 發票號碼 / 日期」這些會計需要的欄位
  2. **拍照前做資安自檢（呼應 CH1-4）**：每次上傳圖片前問自己一句「照片裡有沒有個資 / 機密 / 金流資訊」。有，就先處理；沒有，才上傳
  3. **不要省這 30 秒**：AI 會把整張圖讀進雲端，不會幫你分辨哪些不該讀。個資一旦上去了就拿不回來，這 30 秒的遮圖比你事後解釋便宜太多
