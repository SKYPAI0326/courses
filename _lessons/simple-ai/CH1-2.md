## 教學流程（Teaching Flow）

### 破題 / Hook（3 分鐘）

週三晚上七點半，手工皂創業者跟原料商朋友約在巷口的麵店吃飯。

本來只是想敘舊，結果聊著聊著就聊到「欸你下一批玫瑰精油要不要多叫一點，我給你九折」「可是我還沒確認下個月市集的量」「那你最晚禮拜五前告訴我，我才能併單」。

你手上沾著滷味醬汁，筆記本沒帶、手也不方便打字，乾脆把手機錄音打開放在餐桌上，錄了大約 2 分鐘對話。

吃完回到家，打開錄音一聽——整個人累了。兩個人一來一往，夾著閒聊、夾著報價、夾著承諾，你要從裡面挖出「我要在禮拜五前做什麼」這件事，就得把 2 分鐘錄音再聽兩次、邊聽邊抄。

**這個單元要幹一件事**：讓 AI 聽你的錄音（或貼上逐字稿），三分鐘還你一份乾淨的會議紀錄——**逐字稿 + 5 點摘要 + 結構化待辦（誰 / 做什麼 / 何時前）**。一次解決。

> 💡 講師內部提示：開場放一小段 30 秒的錄音（講師預錄的「朋友聊進貨」片段）讓現場感受「雜訊量」，再接「如果你要自己整理要花多久？」這個反問。不要超過 3 分鐘。

---

### 概念 / Concepts（3 分鐘）

處理會議錄音，腦袋裡記住三個階段就好：

- **錄音** — 手機內建錄音 App 錄下來，或直接貼逐字稿。不需要特別 App，iPhone 語音備忘錄 / Android 錄音機都行。
- **摘要** — 把雜訊（口語贅詞、閒聊）壓掉，留下決策與承諾。AI 最擅長這段。
- **行動** — 把摘要再往前推一步，變成「誰 / 做什麼 / 何時前」可以直接貼到待辦 App 或 LINE 群的格式。這是本單元的核心價值。

你只要負責錄，剩下兩階段 AI 幫你做完。

本單元要用的兩則 Prompt（來自 CH1-1 已經教過的五零件框架）：

- **EF-01（主力）** — 會議錄音 → 逐字稿 + 摘要 + 待辦，一條龍。
- **EF-05（延伸）** — 長訊息（500 字以上的 LINE / Email / 公文）→ 三句話重點 + 對方立場判讀 + 回覆優先序。

為什麼要學兩則？因為會議有兩種——**一種是面對面有聲音的**（EF-01），**一種是線上打字一長串貼來貼去的**（EF-05）。一個創業者一週兩種都會遇到。

**工具建議**：
- **EF-01**：**ChatGPT**（免費版可直接上傳短音檔、多模態處理順手）或 **Gemini**（如果錄音轉出來的逐字稿很長，超過 2000 字，Gemini 吃長文件比較穩）
- **EF-05**：**ChatGPT** 或 **Gemini** 都可以，長訊息兩邊都吃得下

### 操作示範 / Demo — EF-01 現場跑一遍（7 分鐘）

**情境設定**：講師現場示範，用手工皂創業者跟原料商朋友的 2 分鐘錄音（試跑包附的虛構素材，文字版見下方「試跑包需求」區）。

**步驟 1：打開工具（30 秒）**

在手機上打開 ChatGPT App（已在 CH1-1 設定好個人化 AI 助理）。確認助理設定還在——直接問「你是誰？」如果它回「我是你的手工皂創業助理」就代表 CH1-1 的設定有效，直接往下走。

如果沒設定好，用 EF-08 重跑一次再回來。

**步驟 2：上傳錄音或貼逐字稿（1 分鐘）**

兩條路擇一：

- **A 路（有音檔）**：點輸入框旁邊的「＋」→「上傳檔案」→ 選錄音檔（.m4a / .mp3 都行）。ChatGPT 免費版可以直接讀音檔。
- **B 路（沒音檔只有文字）**：把已經轉好的逐字稿（或你現場打的重點筆記）貼在輸入框裡。

> 💡 手機操作重點：iPhone 語音備忘錄的檔案，點一下錄音 → 右上「...」→「分享」→ 選 ChatGPT，就會直接丟進去，不用先存到檔案 App。

**步驟 3：貼 EF-01 Prompt（30 秒）**

直接把 EF-01 的完整 Prompt 貼在音檔或逐字稿下方（Prompt 全文見下方「試跑包需求」區）。

**步驟 4：讀輸出、挑錯（3 分鐘）**

AI 會回你三塊：

1. **逐字稿**：檢查人名有沒有標錯（A 和 B 哪個是你、哪個是朋友），時間數字有沒有被 AI 亂補
2. **摘要 5 點**：每點一句，讀起來該像「決定了什麼 / 要做什麼」而不是「我們聊到什麼」
3. **待辦清單**：格式是「誰 / 做什麼 / 何時前」。聽不清楚的地方應該有 **[待確認]** 標記（這是 Prompt 裡強制要求的退路）

**步驟 5：把待辦複製到行動裝置（1 分鐘）**

長按待辦那段文字 → 複製 → 貼到你平常用的待辦 App（iOS 提醒事項 / Google Keep / LINE Keep 都行）。**關鍵動作是「離開 AI 介面，把結果落地到你會再看到的地方」**——不然三天後你找不到。

**步驟 6：把原始對話關掉，開新對話（30 秒）**

下次會議是新的會議，不要在同一個對話裡塞第二場。開新對話等於幫 AI 清空背景資料，輸出會更準。

> 💡 講師內部提示：示範全程站在投影前，手機鏡像到大螢幕。不要偷偷剪輯，學員要看到「AI 也會把人名搞錯、我也要自己挑錯」這件事——這是課程信任感的關鍵。

### 動手 / Hands-on（7 分鐘）

**你現在做一件事**：

選一條：

- **A 條（有設備）**：拿出你自己的手機，錄一段 30 秒的獨白（例如對著手機講「我下週要去市集擺攤，需要準備玫瑰手工皂 50 顆、薰衣草 30 顆、提袋 100 個，禮拜五前要跟原料商確認精油量」）。然後丟進 ChatGPT 跑 EF-01。
- **B 條（不想錄）**：用講義附的文字版虛構對話（手工皂創業者 vs. 原料商朋友，2 分鐘版本，見試跑包素材），直接貼進 ChatGPT 跑 EF-01。

**預期產出**：
- 一份逐字稿（贅詞已被刪除）
- 5 點以內的摘要
- 至少 2 條「誰 / 做什麼 / 何時前」格式的待辦
- 聽不清楚的資訊（時間、數字、人名）有 **[待確認]** 標記

**self-check 清單**：
- [ ] 摘要每點是「決策 / 承諾 / 結論」，不是「聊天過程」
- [ ] 待辦三欄齊全（誰 / 做什麼 / 何時前），缺的地方有標 [待確認]
- [ ] 逐字稿人名沒被搞混（你是你、朋友是朋友）
- [ ] 已複製待辦到你平常用的待辦 App / LINE / 紙筆

做完後，轉頭跟旁邊的人互看輸出——**別人看得懂你的待辦是誰要做什麼嗎？** 如果要解釋才懂，Prompt 結果還要再跑一次。

> 💡 講師內部提示：這 7 分鐘講師不要講課，走動看學員螢幕，遇到「AI 把兩個人講話搞混」的立刻拉出來當例子：提醒「這就是為什麼 Prompt 裡要求保留發言順序」。

### 延伸 / EF-05 長訊息摘要（5 分鐘）

**情境**：會議只是一種。另一種常見的——客戶在 LINE 上傳了一則 **600 字**的長訊息，裡面夾著：抱怨（上次訂單晚到）、需求（想加訂一組禮盒）、期待（希望下次包裝更好）。你要回覆，但手機只看得到前三行。

**怎麼辦？用 EF-05。**

示範（講師口頭帶，不要求學員跟做）：

1. 把 LINE 那則 600 字長訊息複製貼到 ChatGPT（素材見試跑包）
2. 貼上 EF-05 Prompt（全文見試跑包）
3. AI 回你三塊：**三句話以內的核心重點 / 對方情緒立場 / 建議先處理哪一點**
4. 你就有了回覆的方向——**先道歉晚到 → 再處理禮盒訂單 → 最後承諾下次包裝改善**

**什麼時候不適用？**

- 訊息裡有個資（真實姓名 / 電話 / 地址 / 身分證）→ 先遮掉再丟（這是 CH1-4 要教的資安紅線，本單元先打預防針）
- 訊息是合約 / 法律文件 → AI 摘要會漏掉條款細節，只能當「大概在講什麼」用，不能當結論

> 💡 講師內部提示：EF-05 不要求學員現場跑，口頭講完範例即可。本單元核心是 EF-01，EF-05 是「你知道有這條路」的延伸提示，讓學員回家遇到長訊息時想起來。

### 檢核 / Verification（3 條自我驗證，約 2 分鐘）

離開這單元前，你手機上應該有這三個東西，缺一就代表還沒學會：

1. **一份跑完的 EF-01 輸出**：打開 ChatGPT / Gemini 的對話紀錄，找得到一份含「逐字稿 + 5 點摘要 + 待辦三欄」三段式結果（可以是自己錄的 30 秒、也可以是手冊提供的虛構對話文字版）
2. **待辦清單三欄齊全**：誰 / 做什麼 / 何時前三欄都要有；若 AI 漏抓哪欄就追問「是誰要做」或「什麼時候前」——不允許「某人之後處理」這種空值
3. **走完全流程 ≤ 10 分鐘**：從「打開對話」到「拿到可貼回 LINE 的待辦清單」，碼錶計時超過 10 分鐘就代表 Prompt 太囉嗦或步驟散、回家要練一次短版

> 💡 講師提示：學員要是卡在第 3 條，最常見是 Prompt 貼太長（複製整個手冊範本）。告訴他們 EF-01 的核心只有「三段輸出」這一句，其他都是可選微調。

### Prompt 原始碼（完整版）

#### EF-01 — 會議錄音 → 摘要＋待辦

```
請幫我把這段會議內容整理成三部分：

1. 逐字稿：保留發言順序，刪除口語贅詞（嗯、那個、對啊）
2. 會議摘要：5 個 bullet points 以內，每點一句話
3. 待辦事項：格式為「誰 / 做什麼 / 何時前」

若某些資訊不完整（例如聽不清楚時間），標註 [待確認]，不要自行補充。

用繁體中文台灣用語。
```

**填空提示**：沒有填空。直接上傳音檔或貼逐字稿即可。

**建議工具**：ChatGPT（短音檔直接上傳）/ Gemini（逐字稿超過 2000 字時）

#### EF-05 — 長訊息 → 重點摘要

```
請幫我摘要以下內容：

[貼上長訊息]

輸出要求：
1. 三句話以內的核心重點
2. 對方的情緒或立場（憤怒 / 友善 / 商業 / 模糊）
3. 如果我要回覆，建議我先處理哪一點
4. 不要加油添醋，只摘要原文有的內容
```

**填空提示**：直接貼原文，不用自己先摘要。

**建議工具**：ChatGPT 或 Gemini 皆可。

### 情境素材 1：2 分鐘虛構會議錄音（文字版）

**場景**：週三晚上，麵店，手工皂創業者（你）和原料商朋友阿賢。

```
你：哎欸這次精油批次是不是也差不多該下訂了，你上次說玫瑰的那批貨到了？

阿賢：到了啦，上禮拜進的保加利亞玫瑰，這批味道我覺得比上次好。你如果要，我可以給你九折，不過要三公斤起跳。

你：三公斤喔，我算一下...我下個月市集要做 50 顆玫瑰皂，一顆吃 3 克精油大概，那才 150 克欸。

阿賢：可是你除了市集還有蝦皮單啊，你算一下一個月總量。

你：啊對，蝦皮那邊每個月固定出個 30 顆左右，那就差不多 80 顆，240 克。還是不夠三公斤。

阿賢：不然這樣啦，你要不要乾脆併我這批，我幫你留 500 克，九折價，但你要在這禮拜五前跟我說要不要，不然我要併其他客戶的單。

你：禮拜五喔...我禮拜四晚上剛好要盤點庫存，盤完就知道。我禮拜五中午前給你答覆。

阿賢：好啊。那薰衣草的呢？

你：薰衣草我上個月才補貨，還夠用到下下個月。這次先跳過。

阿賢：OK，那精油部分就等你禮拜五。啊對了，你之前說想試試看的那個甜橙香精我拿到樣品了，下次見面給你聞看看。

你：好啊好啊，下次來載精油的時候順便。

阿賢：那就這樣，吃麵吃麵。
```

```
我知道這可能是運送的問題不是你們的問題，但我想說還是跟你講一下，下次如果要出禮盒，是不是可以外面多一層紙箱或是有防撞的那種？因為我看你們蝦皮評論有蠻多人也是說這個問題的，我想說如果改一下應該對生意會比較好。

然後如果下次也是禮盒的話，真的拜託包裝弄好一點，不然我下次可能就不敢買了，因為要送人的話品相很重要欸。

啊還有一個問題，我看你們 IG 上面有出新的檜木皂，但蝦皮上面沒有看到，那個有要上架嗎？如果有我也想一起買。

不好意思打擾囉，期待回覆！
```

（實際字數約 600 字，混合抱怨、需求、期待、提問，符合 EF-05 預期情境）

### 預期輸出示例

#### EF-01 預期輸出（針對上面 2 分鐘會議）

**一、逐字稿**
```
你：這次精油批次是不是差不多該下訂了，玫瑰的那批貨到了？
阿賢：到了，上禮拜進的保加利亞玫瑰，味道比上次好。九折價，但要三公斤起跳。
你：三公斤太多。市集 50 顆加蝦皮 30 顆，一個月總共 240 克而已。
阿賢：幫你留 500 克九折，禮拜五前回覆要不要，不然併給其他客戶。
你：禮拜四盤庫存，禮拜五中午前給答覆。
阿賢：薰衣草呢？
你：還夠，這次跳過。
阿賢：甜橙香精拿到樣品，下次見面給你聞。
```

**二、會議摘要**
- 阿賢手上有保加利亞玫瑰精油，九折價可留 500 克
- 手工皂創業者一個月玫瑰精油用量約 240 克，需評估庫存後決定是否併單
- 盤點時間定在禮拜四晚上
- 回覆截止時間：禮拜五中午前
- 薰衣草本輪跳過；甜橙香精下次碰面試聞

**三、待辦事項**
- 你 / 盤點玫瑰皂原料庫存 / 禮拜四晚上
- 你 / 回覆阿賢是否併單 500 克玫瑰精油 / 禮拜五中午前
- 阿賢 / 下次碰面帶甜橙香精樣品 / [待確認]

#### EF-05 預期輸出（針對上面 600 字客戶訊息）

**【核心重點】**
- 客戶上月收到的玫瑰禮盒包裝外盒壓損，雖然皂無事但品相差

## 常見錯誤 3 條（Common Pitfalls）

1. **錯誤現象**：AI 輸出的待辦清單裡有時間，但那個時間**是 AI 自己補的**——例如原始對話只說「禮拜五前」，AI 卻寫成「2026/04/25 下午 6:00 前」。學員沒檢查就複製到行事曆，結果行事曆多了一個根本不存在的 deadline。
   **原因**：Prompt 裡「若某些資訊不完整，標註 [待確認]，不要自行補充」這句如果漏掉，AI 為了讓輸出「看起來完整」會自己腦補具體日期時間。這是典型的 AI 幻覺（AI 亂掰）。
   **解法**：確認每次都用 EF-01 完整版 Prompt，不要自己改短。如果已經跑出來了，檢查時用眼睛掃一遍每個時間是不是原文講過——沒講過的全部改成 [待確認] 或刪掉。

2. **錯誤現象**：兩個人在對話裡，AI 把 A 的話標成 B 講的、B 的話標成 A 講的，整份逐字稿角色錯置。學員照著摘要「你答應禮拜五前回覆」結果發現自己根本沒答應，是朋友答應的。
   **原因**：音檔裡兩人聲紋相近，或逐字稿貼上時沒標記誰是誰，AI 只能用「語氣提示詞」猜。一猜就錯。
   **解法**：貼逐字稿前自己先在每段前加「我：」「朋友：」標記；或在 EF-01 Prompt 前面加一句「發言人 A 是手工皂創業者自己（你），發言人 B 是原料商朋友阿賢」幫 AI 定錨。

3. **錯誤現象**：學員把整週 5 場不同會議的錄音，在同一個 ChatGPT 對話視窗裡連續丟進去跑 EF-01。第 3 場之後，輸出開始混到前面幾場的內容（出現第一場會議講過的人名或數字）。
   **原因**：同一個對話視窗裡，AI 會把前面的對話內容都當成背景資料一起讀。五零件學過的「限制」這塊沒有清乾淨，就會串味。
   **解法**：**每場會議開一個新對話**（ChatGPT 左上角「新對話」按鈕 / Gemini 左側「開始新對話」）。跟洗鍋子一樣，上一餐的味道別留到下一餐。

## 檢核題 2 條（Quiz）

**Q1（概念驗證）**：以下四種情境，哪一種**最適合**用 EF-01？

- [ ] A. 客戶傳了一則 800 字的 LINE 抱怨訊息，要萃取重點
- [ ] B. 團隊週會 45 分鐘的錄音，要整理成會議紀錄 + 下週待辦 ←（正確答案）
- [ ] C. 一份 30 頁的合約 PDF，要找出所有跟付款條件相關的條款
- [ ] D. IG 限時動態的回覆有 50 則，要統計客戶情緒

**解題要點**：
- A 是 EF-05 的情境（長訊息摘要，非會議）
- C 是超長文件處理，且合約需要法務檢視，AI 摘要不足以當結論（本課 non-goal）
- D 是資料分類/情緒分析，不是會議處理
- 只有 B 完全符合 EF-01「錄音 → 摘要 → 結構化待辦」的設計目標

---

**Q2（應用驗證）**：你跟客戶通話 20 分鐘談新包裝設計，通話錄音已經存在手機。你要在 15 分鐘內產出一份能寄給設計師同事的「客戶需求摘要 + 要辦事項」。描述你會怎麼做（3 個步驟以內）。

**預期答案要點**：
1. **工具選擇**：用 ChatGPT 上傳音檔（或先用手機內建語音備忘錄轉逐字稿再貼），呼叫 CH1-1 已設定好的個人化 AI 助理
2. **Prompt 套用**：直接貼 EF-01 完整 Prompt（逐字稿 + 5 點摘要 + 待辦「誰/做什麼/何時前」），保留「不清楚的標 [待確認]」這句防 AI 幻覺
3. **落地動作**：把「三、待辦事項」那段複製到 Email 寄給設計師同事，或貼到共用的待辦工具；摘要部分附在 Email 開頭讓同事有背景；原始音檔留在手機雲端備份，不丟上去 AI 以外的公開空間（資安紅線預告，CH1-4 會詳細講）

**加分答案**：提到「通話裡有客戶真實姓名/電話的話，先用文字編輯把個資遮掉再丟 AI」——代表學員已經預先連結到 CH1-4 資安紅線的概念。
