## 教學流程（Teaching Flow）

### 破題 / Hook（3 分鐘）

你是那位台北單店咖啡館的老闆，開店兩年，一人兼行銷。週二晚上 10 點，收完最後一組客人，打開 IG 想發明天的貼文。

你照著慣例打開 ChatGPT，打「幫我寫一則咖啡館的 IG 貼文，介紹新的桂花烏龍茶」。

十秒後 AI 丟回一段文字：「在這個忙碌的午後，讓桂花烏龍茶陪你度過慵懶的時光...」

你讀完第一句就關掉。這不是你家的口氣。你家客人來是因為你會在吧台跟他們聊豆子怎麼烘、上次的手沖為什麼比較酸，不是因為「慵懶的午後」。

**問題在哪**：ChatGPT 沒看過你家網站、沒看過你 Google 評論、沒看過你過去三個月的貼文。它只會用「咖啡館貼文的平均值」回你——那是別人家的咖啡館。

**這一堂要解的事**：今天起，我們不再拿「產業常識」湊貼文。我們要讓 AI 先讀完你家的資料，再回答你。用的工具叫 **NotebookLM**。

---

### 概念 / Concepts（5 分鐘）

講兩件事就夠：差在哪、為什麼這一堂不用 ChatGPT。

- **ChatGPT vs NotebookLM 的差別** — ChatGPT 沒讀你的資料，靠它看過的所有網路文章給你「咖啡館產業常識」的平均答案；NotebookLM 只讀你餵進去的檔案，只根據這些檔案回答，而且每句話都附來源出處。
- **為什麼這堂要用 NotebookLM** — 今天目標是做出「一份讀起來像你家的品牌聲音摘要」。這件事 ChatGPT 做不到，因為它不認識你家。NotebookLM 讀完你家三份文件之後，才叫「你自己的答案」。
- **什麼叫「品牌聲音」** — 不是 logo、不是配色，是你家文字的口氣。讀一段話，客人就知道「這是他家」。例如你家可能常用「這批豆子」「老闆」「吧台」「試喝」這種詞，不用「品味」「尊榮」「頂級」。
- **NotebookLM 的運作規則** — 它只會從你上傳的來源回答，來源沒寫的事它會說「資料中未出現」，不會亂掰。這個特性對品牌工作超重要——你要的是「我家真的是這樣嗎」的答案，不是 AI 幫你編一個理想版。
- **三份文件為什麼夠** — 官網文案給它看你「對外怎麼介紹自己」，菜單給它看你「賣什麼、怎麼命名」，客戶評論給它看「客人眼中你是誰」。三邊交叉，聲音就跑出來了。

**一句話總結**：要新內容找 ChatGPT，要搞懂自家資料找 NotebookLM。

### 操作示範 / Demo（7 分鐘｜建立 Notebook + 上傳三份文件）

打開手機上的 NotebookLM App（Android / iOS 都有；沒下載的現在裝，免費）。登入你的 Google 帳號。

**步驟 1：建立新 Notebook**

- 右下角點「+ 新 Notebook」或「Create new」
- 命名為「○○咖啡館－品牌資料庫」（換成你自己的店名）
- 手機操作提示：App 介面中文／英文會切換，如果看到「Create」就是中文版的「新建」

**步驟 2：加入第一份來源——官網文案**

- Notebook 開啟後，點中間或下方的「+ 新增來源」
- 選「貼上文字」或「Text」
- 把官網的「關於我們」「品牌故事」「選豆理念」三段文字複製貼上（約 300 字）
- 命名這份來源叫「官網文案」

**步驟 3：加入第二份來源——菜單**

- 再點「+ 新增來源」
- 一樣選「貼上文字」
- 把菜單 10 項品項（名稱 + 價格 + 簡短描述）貼上
- 命名「菜單」
- 手機操作提示：菜單如果是照片或 PDF，NotebookLM 也吃 PDF，不過這堂示範用最穩的貼文字

**步驟 4：加入第三份來源——客戶評論摘要**

- 再點「+ 新增來源」
- 把 Google 評論整理過的摘要（正面 + 抱怨 + 建議約 300 字）貼上
- 命名「客戶評論摘要」
- 手機操作提示：原始 Google 評論太多會吃配額，先手動挑 15-20 則比較有代表性的，整成一份文件再貼

**步驟 5：確認三份都吃進去了**

- 左側（手機版通常在上方）應該看到三份來源，每一份左邊有勾選框，預設都勾起來
- 這代表接下來問問題時，AI 會同時讀這三份
- 如果哪一份沒勾，它就不會讀

**常見卡關**：
- **上傳失敗**：免費版每份來源文字上限大概 50 萬字，超過會擋下來。我們三份加起來不到 2000 字，理論上不會卡。卡了就檢查是不是複製到奇怪的格式符號。
- **看不到「新增來源」**：App 版本更新會換位置，找「+」或「Add」就對了。

### 動手 / Hands-on（10 分鐘｜NB-01 萃取品牌聲音 + NB-02 帶過）

三份來源都進去了，現在問它第一個問題。

**打開 NotebookLM 底下的對話輸入框，把下面這段完整貼進去**（這就是 NB-01）：

```
你是我的品牌文字顧問。

請根據我提供的三份來源（官網文案、菜單、客戶評論摘要），分析我的品牌聲音：

1. 用 5 個形容詞描述我家的品牌聲音，每個形容詞要附具體例子（從哪份來源的哪段話看出來）
2. 列出我家常用的 3 個句型或用詞習慣（例：常用什麼開頭、常用什麼結尾、常出現哪些詞）
3. 指出我家刻意避開的詞彙類型（從來源中觀察：哪些常見的咖啡館行銷詞我完全沒用）
4. 列出客戶評論中，客人描述我家時最常用的 3 個詞（這是外部怎麼看我家）
5. 給我一段 80 字內的「品牌聲音摘要」，寫給我自己看，下次寫貼文時可以對照

規則：
- 每條結論都要附來源引用（來自哪份來源的哪段話）
- 如果某項在三份來源中找不到足夠證據，直接說「資料不足」，不要推測
- 用繁體中文台灣用語回答
```

**動手練習題詳見下方「動手練習題」段落**。這裡先講講等它出結果後你要看什麼。

**預期輸出長什麼樣**（示意，你家實際產出會不一樣）：

```
【5 個形容詞】
1. 在地 — 來源「官網文案」第 2 段：「我們不找產地標很漂亮的豆子，找喝起來有記憶點的」
2. 誠實 — 來源「客戶評論摘要」：客人 3 次提到「老闆會直接說哪款不合你口味」
3. 慢工 — 來源「菜單」：手沖 180 元、備註「每杯約 8 分鐘，建議坐下來喝」
4. 不裝 — 來源「官網文案」第 4 段：「我們店不大，老闆就我一個」
5. 有溫度 — 來源「客戶評論摘要」：客人形容「像朋友家的吧台」

【3 個常用用詞習慣】
1. 用「老闆」「吧台」「這批豆子」這種工作現場的具體詞
2. 菜單命名偏口語（「招牌焦糖拿鐵」而不是「經典招牌拿鐵」）
3. 常用「其實」「沒有想太多」這種卸下姿態的開頭

【刻意避開】
「頂級」「匠心」「尊榮」「臻選」「限定」——三份來源全部 0 次出現

【客人眼中的你家】
1. 「親切」（8 次提及）
2. 「手沖好喝」（6 次提及）
3. 「空間舒服」（5 次提及）

【品牌聲音摘要】
我家的聲音像吧台邊的老闆在跟熟客聊天——在地、誠實、有溫度，願意承認自己的侷限。
用具體的工作場景詞（豆子、手沖、吧台），避開行銷味重的詞。
下次寫貼文時：想像你正在跟坐在吧台前的客人說話，不是在跟 IG 演講。
```

看到這種結果，你就完成今天的核心目標了：一份讀起來像自家的品牌聲音摘要。

**再玩一下（NB-02 帶過，5 分鐘內結束）**：

同一個 Notebook，接著問下面這段（這是 NB-02，先走過一次，下一單元 CH2-2 會深用）：

```
在我提供的來源中（特別是客戶評論摘要），幫我找出：

1. 客戶最常抱怨的 3 件事，各附一句原文引用
2. 客戶提到「希望有...」「如果有...就好了」類型的需求
3. 這些需求中，哪些是我（小店、一人經營）本週可以動手做的，哪些太大做不了

規則：每點都附原文引用；資料中沒出現的直接說「來源中未出現」，不要推論。
```

這會吐出一份「客戶痛點清單」，例如：

- **抱怨 1**：等太久（3 次提及）——「週末要等 20 分鐘才拿到手沖」
- **抱怨 2**：週末沒位（2 次提及）——「兩次假日過去都客滿」
- **抱怨 3**：價格偏高（2 次提及）——「一杯手沖 180 有點貴」
- **希望有**：乳糖不耐選項（燕麥奶、豆奶）、可加訂位功能
- **本週可動手**：公告「手沖繁忙時段需等 15-20 分鐘」、上架燕麥奶選項
- **做不了**：店面擴張、價格全面下調

CH2-2 會把「抱怨」+「品牌聲音」兩份東西合起來，寫一句話品牌定位。今天先拿到這份清單就好，不用急著行動。

### 檢核 / Verification（2 分鐘｜自我驗收）

這一單元結束前，自己確認：

1. **手機上的 NotebookLM 裡有一個以你店名命名的 Notebook**，裡面有三份來源（官網文案、菜單、客戶評論摘要），三份都勾起來了

4. **NB-02 跑完有一份客戶痛點清單**（3 條抱怨 + 需求清單），留著下一單元用

#### 文件 1：官網文案（約 300 字）

```
【關於我們】

弄一下咖啡，開在台北永康街的一間巷弄小店，店不大，吧台 6 個位、內用 4 張小桌。
老闆就我一個人，從烘豆、手沖、收銀到擦桌子。

【選豆】

我們不找產地標很漂亮的豆子，找喝起來有記憶點的。
淺焙為主，偶爾上中焙。每批豆子到店我自己杯測三輪才決定要不要上架。
每一支豆子會在菜單旁邊手寫小卡片，寫我喝到的味道——不是寫 SCA 評分。

【空間】

店裡放黑膠，音量壓低到剛好蓋過磨豆機。
沒有 wifi 密碼。插座只有吧台兩個。
我們希望你坐下來就是為了那杯咖啡，不是把這裡當作免費辦公室。

【最後】

我們店不大，老闆就我一個。有時候你來可能要等一下，有時候豆子剛好賣完。
不好意思是第一句，謝謝你願意走進來是第二句。
```

## 商業情境案例（Case）

**角色**：你（老闆）——即學員本人代入
**場景主線**：A 咖啡館老闆

## 常見錯誤 3 條（Common Pitfalls）

1. **錯誤現象**：上傳的文件太少（只有官網文案一份，或只有菜單），NB-01 輸出流於通用，寫的東西任何一家咖啡館都適用。
   **原因**：NotebookLM 只從你上傳的來源回答。來源少 + 資訊單薄 = AI 沒東西可分析，只好給最安全的通用答案（「友善、專業、有質感」這種誰家都能貼的形容詞）。
   **解法**：最少要三份來源，而且要涵蓋三個視角——對外文案（官網）、產品現況（菜單）、客人眼中（評論）。只有對內視角（老闆自己寫的）AI 會被你的自我認知帶偏；加進客人的話才會交叉出真實的聲音。

2. **錯誤現象**：AI 輸出了一堆形容詞（「溫暖、親切、專業、優質、用心」），但讀起來都是抽象空話，你拿回去寫貼文完全套不進去。
   **原因**：你沒在 Prompt 裡要求「附具體例子 / 附來源引用 / 舉句子」，AI 預設會給最省力的抽象答案。
   **解法**：Prompt 裡一定要加「每個形容詞要附一句來源原文」「列出 3 個具體的句型或用詞習慣」。抽象的形容詞只是標籤，真正能拿去用的是「你家實際出現過的句型和詞彙」。NB-01 的範本已經幫你寫好了這幾條，照著用就會拿到能落地的結果。

3. **錯誤現象**：上傳後，問 AI 問題它回答「抱歉，來源中未出現相關資訊」，或直接跳到它訓練資料裡的通用答案。
   **原因 1**：三份來源有上傳但**沒勾選**，AI 當那些資料不存在。
   **原因 2**：問題太偏離來源內容（例如問「我家開分店的財務規劃」，來源只寫品牌和菜單，當然答不出來）。
   **原因 3**：來源文件只有一兩句話，資訊量太薄。
   **解法**：先檢查左側三份來源是不是都打勾（勾選狀態代表 AI 會讀）；再確認你問的問題是三份來源有可能覆蓋到的範圍（品牌聲音、客戶反饋、菜單命名風格 = 可以答；財務、法律、擴張策略 = 答不了）；最後如果文件太短就補一份，例如補 20 則過去 IG 貼文。

## 檢核題 2 條（Quiz）

**Q1（概念驗證）**：週三下班前，你想要 AI 幫你寫一篇「介紹本週新上架的薩爾瓦多豆」的 IG 貼文，語氣要像你家平常的風格。下列哪個選擇最合理？

- [ ] A. 打開 ChatGPT，輸入「幫我寫一則咖啡館介紹薩爾瓦多豆的 IG 貼文，語氣要自然」
- [ ] B. 打開 ChatGPT，先告訴它你家過去 3 則貼文的連結，再請它寫
- [ ] C. 打開 NotebookLM，進入已建好的「弄一下咖啡－品牌資料庫」Notebook（裡面有官網、菜單、客戶評論、過去貼文），請它依照你家口氣寫一則介紹薩爾瓦多豆的貼文 ← 正確答案
- [ ] D. 兩個都問，比較哪個寫得比較好

**正確答案**：C
**為什麼**：ChatGPT 沒讀過你家資料，只會給「咖啡館貼文的平均值」（A 錯）；ChatGPT 即使你貼連結它也不穩定吃長網頁，而且每次要重貼（B 錯）；NotebookLM 已經吃過你家三份來源，只根據這些回答，語氣會貼近你家（C 對）；兩個都問在實務上浪費時間，而且 ChatGPT 的版本會誘惑你選「比較漂亮但不像自家」的那一版（D 錯）。

---

**Q2（應用驗證）**：你照著課堂步驟跑完 NB-01，拿到的品牌聲音摘要是這樣：

> 「你的品牌聲音溫暖、專業、有質感，給人值得信賴的感覺。你常用正向積極的語言，傳遞優質服務的理念。建議持續強化這種調性。」

你怎麼判斷這份輸出有問題？該怎麼修？

**預期答案要點**（學員答出其中 2-3 點即算通過）：

1. **沒有來源引用**：合格的 NB-01 輸出每條結論都要附「來自官網文案第 X 段」或「來自客戶評論第 Y 則」。這段輸出全部沒有，代表 AI 沒真的在讀你的來源，或你 Prompt 沒要求它附引用。
2. **形容詞全是通用詞**：「溫暖、專業、有質感、值得信賴、優質」——這套形容詞貼在任何一家店都成立（手搖店、銀行、保險業都能用），沒抓到你家的獨特性。真實的品牌聲音會包含具體特徵（例如「帶點自嘲」「用吧台口語」「常用『其實』開頭」），不是一串安全形容詞。
3. **沒有具體句型或用詞例子**：NB-01 Prompt 有要求「列出 3 個常用句型或用詞習慣」，這段輸出完全沒有，代表 Prompt 被簡化或 AI 偷懶。
4. **修法**：
   - 檢查三份來源有沒有勾選（可能 AI 根本沒讀到）
   - 重貼一次完整 NB-01 Prompt（不要自己縮減條文）
   - 追問一句：「請附具體來源引用，並舉出 3 個我家真實出現過的句子當例子。如果某條結論找不到足夠來源，直接說資料不足」
   - 如果追問後還是通用答案，可能三份來源資訊太薄，補一份過去 20 則 IG 貼文進去
