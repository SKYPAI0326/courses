## 教學流程（Teaching Flow · skill-operation 5 段式）

### 破題 / Hook（5 分鐘）

橘貓上班族今天要產一整套表情組。你打開 Gemini，輸入「畫一隻圓滾滾的橘色虎斑貓，戴白領巾，正在大笑」——第一張很可愛，你開心地按下「再畫一張，這次是哭」。Gemini 給你一張哭的橘貓。但你仔細一看：耳朵變尖了、領巾不見了、配色從橘色變成偏紅褐。

第三張「生氣」更慘——AI 直接畫成另一隻貓，連品種都換了。

這就是你在 CH1-2 學完「角色一致性 3 手法」之後，最容易踩的坑：**手法你都會了，但實戰時還是會跑掉**。原因不是技術不夠，是**操作順序錯了**。

這 75 分鐘做一件事：在**同一個 Gemini 對話**內，依序產出 **8 張同角色不同表情**（笑、哭、生氣、得意、求拜託、無語、晚安、嚇到），並用一份 5 條判準的檢查清單自查每一張。8 張裡如果有 5 張以上跑掉，你的角色設定卡要回 CH1-2 重寫；如果跑掉 1–2 張，你會學到怎麼用「補張」技巧救回來。

> 你**不是來學畫畫的**。你是來學「怎麼跟 AI 講同一句話 8 次，讓它記得這是同一隻貓」。

### 概念 / Concepts（10 分鐘）

這 5 條概念會反覆出現在示範與動手環節，先記下：

- **同對話多輪（context 持續）** — 在 Gemini 同一個對話視窗內連續發 8 個 prompt，AI 會看到前面所有圖當作 context，比較容易守住角色。**重開新對話 = 角色記憶重置**。
- **角色設定卡（character sheet）** — CH1-2 寫好的那份：種族 / 體型 / 配色 / 個性 / 風格參考。本單元每輪 prompt 都會引用這張卡，不重寫。
- **表情變化版本 prompt 結構** — 固定句型：「同一隻【角色設定卡】，這次表情是【某情緒】，動作是【某動作】」。三段式，不要自由發揮。
- **角色一致性檢查清單（5 條判準）** — 每張產出後立刻自查的工具，本單元的硬產出之一。5 條：
  1. **同一隻角色**（不是同類型）— 比對：耳朵形狀、眼睛形狀、配色
  2. **同一種風格** — 線條粗細、上色方式（卡通 / 水彩 / 寫實）
  3. **同一個比例** — 頭身比、五官位置（眼睛在臉的上半還是中間）
  4. **同一個視角** — 正面 / 側面 / 3/4 面要分清，不要 8 張各種角度都有
  5. **同一個色域** — 不會某張突然變鮮豔、變暗沉，整組要在同一個色彩飽和度區間
- **省額度技巧（critical）** — 同對話多輪 = 1 次「啟動成本」+ N 次「續寫成本」，總額度約等於重開新對話的 1/3。8 張都重開 = 額度 8 倍速燒光。
- **補張（patch shot）** — 同對話內針對跑掉的某一張重畫，不重啟對話。句型：「上一張的【某部位】跟第 1 張不一樣，請參考第 1 張的【某部位】重畫一次第 N 張的【某表情】」。本單元下半場學員每個人至少要用一次補張。
- **校準張（baseline shot）** — 8 張中第 1 張的特殊地位：後面 7 張的 context 都以第 1 張為準。第 1 張畫得好不好決定整套的天花板。如果第 1 張就跑掉（例如你想要扁平卡通結果 AI 給你水彩風），**立刻在原對話內重畫第 1 張**，不要繼續往下畫。

### 操作示範 / Demo（25 分鐘）

我會在 Gemini 同一個對話裡，**從頭到尾**示範產出橘貓的 8 張表情。你不要動手，先看完整流程，等下換你跑自己的角色。

#### 為什麼示範要佔 25 分鐘（不是 10 分鐘）

CH1-2 教完 3 種角色一致性手法後，學員的「腦袋知道」程度大概 70%；但實際手敲時還是會卡——卡在「第 2 輪 prompt 怎麼寫才不會 AI 又重置角色」「第 3 張耳朵跑掉時要不要立刻補」「補張的 prompt 句型怎麼開頭」這類**操作層的細節**，CH1-2 沒空細談。25 分鐘示範的目的是讓學員看見「真正在做的人會在哪裡停下來、會說什麼話、看到什麼會皺眉」。看完之後 25 分鐘自己動手，才不會卡在第 2 輪就放棄。

> 如果這 25 分鐘節省成 10 分鐘趕進度，後面 25 分鐘動手會多出至少 15 分鐘卡關時間，整體反而拖延。本單元時間表是緊的，但示範時間絕對不能省。

#### 示範用角色設定卡（橘貓）

```
我要做一隻虛構卡通角色，角色設定如下：
- 種族：圓滾滾的橘色虎斑貓
- 體型：胖胖短手短腳，圓頭圓身，頭身比 2:1（大頭小身體）
- 配色：身體橘色虎斑（淺橘 + 深橘條紋），白色肚子，戴白領巾
- 個性：上班族，溫和但偶爾無語，眼神有點疲憊
- 風格：扁平卡通風格、粗黑線條、暖色調、平塗上色（不要漸變）
- 視角：正面，置中構圖
- 背景：純白背景

請依此設定卡，畫一張這隻橘貓「正在大笑」的圖。表情要明顯，瞇眼開口大笑，
保持上面所有設定不變。
```

#### 示範流程：8 輪 prompt（同對話）

**第 1 輪（基準張）**：上面那段完整 prompt → 產出「笑」。
這張是**校準張**，後面 7 張全部以這張為準。

**第 2 輪**（簡化句型，引用前圖）：
```
保持剛才那隻橘貓的所有設定（同隻角色、同風格、同比例、同視角、同色域），
這次表情換成「在哭」，眼淚在眼角，嘴巴往下彎。其他不變。
```

**第 3 輪**：「生氣」（眉毛皺起、嘴巴抿成一條線、頭頂可加怒氣符號）
**第 4 輪**：「得意」（瞇眼微笑、頭微抬、可加閃光點綴）
**第 5 輪**：「求拜託」（雙手合十、眼睛圓睜含淚、嘴巴小小 O 型）
**第 6 輪**：「無語」（眼神放空、嘴巴一條線、頭微側）
**第 7 輪**：「晚安」（閉眼、頭側靠、可加 Z 字符或月亮）
**第 8 輪**：「嚇到」（眼睛圓睜、瞳孔縮小、嘴巴 O 型、毛微豎）

#### 講師示範重點（你看著我做的時候要注意）

1. **每輪只改「表情 + 動作」這一段**，其他完全不動。如果你連風格、視角都重講一次，AI 會以為你想換新角色。
2. **每張產出後立刻講出 5 條檢查**，不等 8 張產完才檢查。發現第 3 張耳朵變了，馬上「**補張**」（下一個 prompt 加一句「上一張的耳朵跟第 1 張不一樣，請參考第 1 張的耳朵形狀重畫一次第 3 張」）。
3. **不重開對話**。即使第 4 張畫壞，也是補張，不是重啟。重啟 = 前面 3 張的 context 全丟。

#### 示範圖包對照（投影片同步呈現）

我手上會放兩組圖：
- **A 組「8 張一致」**：示範一致性都過的標準成品（橘貓 8 張、耳朵眼睛都對、配色一致、視角全正面）。這組是學員自查的對照基準。
- **B 組「8 張跑掉」**：刻意 prompt 失誤的反例（第 1 張橘貓 → 第 3 張變成花貓、第 5 張變成側面、第 7 張線條變細、第 8 張背景變藍）。這組讓學員看到「跑掉長什麼樣」，比文字描述清楚。

兩組對照看完，5 條判準你會立刻記得起來。

#### 講師預備：示範圖包備援（重要）

我課前會預先在 Gemini 跑完 8 張橘貓正解圖，存成 PNG 圖包。**任何學員額度用光時，可以直接下載這份示範圖包**，繼續走 CH1-4（去背 + 規格符合）和 CH1-5（上架）的流程，不會被卡死在「我畫不出來所以後面都做不了」。

備援使用規則：
- 額度沒用光的學員：自己的圖優先，示範圖包只看不用
- 額度用光的學員：下載橘貓 8 張 PNG，當作「自己的素材」走完 CH1-4 / CH1-5
- 課後 Part 1 PRAC 驗收時，**必須提交自己角色的 8 張**（不能交橘貓圖包當作業）；本堂課用備援圖只是為了不卡關

### 動手 / Hands-on（25 分鐘）

換你做。拿出 CH1-2 寫好的**你自己的角色設定卡**（不是橘貓，是你選的那個——可能是你家貓、自己擬卡通、咖啡杯擬人、雲朵小怪獸都行）。

#### 動手任務

在**同一個 Gemini 對話**內，依序產出你的角色 8 張表情：
1. 笑（大笑、瞇眼開口）
2. 哭（眼淚、嘴巴往下）
3. 生氣（眉毛皺、嘴抿）
4. 得意（瞇眼微笑、頭微抬）
5. 求拜託（雙手合十、眼睛圓睜）
6. 無語（眼神放空、嘴一條線）
7. 晚安（閉眼、Z 字符 / 月亮）
8. 嚇到（眼睛圓睜、嘴 O 型）

#### 操作節奏（建議分配）

- 0–10 分鐘：完成第 1–4 張
- 10–20 分鐘：完成第 5–8 張，邊做邊用 5 條判準自查每一張
- 20–25 分鐘：找出跑掉的 1–2 張，用「補張」技巧救回來

#### 8 張表情題目（固定不能改）

這 8 種表情**為什麼是這 8 個**：
- 笑、哭、生氣 = LINE 貼圖最高使用率前 3 名（佔總使用量 60%+）
- 得意、求拜託 = 對話互動高頻表情
- 無語、晚安 = LINE 對話「下台階」與「結束話題」必備
- 嚇到 = 反應系表情代表

這 8 張全部產出 = 你的貼圖套組「能用」的最低門檻。少於 8 張的套組 LINE 不收（最少 8 張）。

#### 講師走場（你做的時候我會做的事）

我會繞場一對一看每個學員的螢幕。看 3 件事：
1. **你有沒有重開新對話**（最常見錯誤）—— 如果有，我會立刻喊停，讓你回到上一個對話續寫
2. **你的 prompt 有沒有引用「保持剛才那隻」這類 context 連結句**—— 沒有的話我會給你模板貼上去
3. **你有沒有 5 條判準逐張自查**—— 沒有的話我會把檢查清單貼到你螢幕旁邊提醒

#### 商業情境（3 個示範代入點）

雖然本課自用導向，動手過程我會帶 3 組情境讓你思考「為什麼要這 8 張」，方便你之後選自己的題：

**情境 1：橘貓上班族（主示範）** — 你想做一套貼圖送給一起加班的同事。8 張要對應的場景：開會大笑 / 被罵想哭 / 客戶白目氣到 / 提案過了得意 / 跪求 RD 改 bug / 老闆又改需求無語 / 加班到深夜想睡 / 系統當機嚇到。每張表情都有對應的職場情境，朋友看到立刻會用。

**情境 2：媽媽群組角色（學員可自選 1）** — 假設你想做一隻「媽媽日常」的貼圖角色（你媽 / 你 / 朋友媽媽都行）。同 8 張表情對應：孩子做得好開心笑 / 看到聯絡簿哭 / 老公又忘記倒垃圾生氣 / 孩子考試 100 分得意 / 拜託孩子吃青菜 / 看到帳單無語 / 半夜餵奶想睡 / 孩子發燒嚇到。

**情境 3：寵物擬人化（學員可自選 2）** — 你家狗 / 貓 / 倉鼠擬卡通。8 張表情對應：飯來了笑 / 主人出門哭 / 不給玩玩具生氣 / 偷吃成功得意 / 拜託加飯 / 主人講話聽不懂無語 / 趴著睡晚安 / 雷聲嚇到。

選一個你最有共鳴的情境，當作這 8 張的「敘事主軸」。**有故事線的 8 張，比 8 張隨機表情，做完更有成就感**。

#### 省額度反覆強調（critical）

- ✅ **同對話多輪**：8 張在同一個對話視窗，平均每張耗 1 個額度單位
- ❌ **每張重開對話**：8 張各 1 個對話，每次「重講角色設定卡」都耗額外額度，總計約 2-3 倍
- ✅ **補張在原對話內**：第 3 張跑掉，下一個 prompt 直接補，不用重啟
- ❌ **跑掉就重新開始**：第 3 張跑掉就重開對話從第 1 張開始畫——免費額度 5 分鐘內見底

如果你第 5 張時額度耗盡：
1. 第一備援：切到 ChatGPT，**把已經產好的前 4 張上傳當參考圖**，請它畫第 5–8 張。**切平台後角色風格會略偏，第 5–8 張產完每一張都要重跑一次「角色一致性檢查清單 5 條判準」**——通過率明顯下降是正常的，但若 5 條中跑超過 2 條就改用備援 2。
2. 第二備援：下載講師預備的橘貓示範圖包，當作「素材」繼續走 CH1-4。回家額度恢復後再補自己的角色。
3. 不要備援：放棄當天上架，回家明天額度恢復後再做。本課允許這條路（但 CH1-5 課堂上你只能看別人按）。

#### 動手過程的常見體感（你可能會這樣想）

- **「我覺得我畫的跟橘貓示範比起來醜很多」** —— 醜不醜不是本單元判準。本單元只查「8 張像不像同一隻」（5 條判準），不查「美感」。你的角色就算是隨便畫的飯糰擬人，只要 8 張一致也算過關。
- **「第 3 張畫了 5 次都跑掉，我是不是該放棄」** —— 不是放棄，是 prompt 寫法的問題。叫講師過來看你的 prompt，多半是「保持剛才那隻」這句少寫了，或是改了視角描述。修一句話通常就能救回。
- **「我看不出第 4 張到底有沒有跑掉」** —— 把第 1 張和第 4 張縮圖並排放在螢幕上 30 秒，5 條判準逐條看。如果還是看不出來 = 沒跑掉，繼續。本單元判準是「明顯跑掉」，模糊地帶不算。
- **「我的角色設定卡好像有問題，可不可以現在重寫」** —— 不可以。本單元先用 CH1-2 的卡跑完 8 張，看看哪些判準會卡，再回家重寫 CH1-2 的卡。當堂重寫等於把這 25 分鐘讓給 CH1-2 補課，整套流程沒走完。

### 檢核 / Verification（10 分鐘）

下課前 10 分鐘，每個學員要做完 3 件事才算這單元完成：

#### 自查表（必填）

把你的 8 張產出，對照 5 條判準逐張勾選：

| 表情 | 1.同隻 | 2.同風格 | 3.同比例 | 4.同視角 | 5.同色域 |
|---|---|---|---|---|---|
| 笑 | ☐ | ☐ | ☐ | ☐ | ☐ |
| 哭 | ☐ | ☐ | ☐ | ☐ | ☐ |
| 生氣 | ☐ | ☐ | ☐ | ☐ | ☐ |
| 得意 | ☐ | ☐ | ☐ | ☐ | ☐ |
| 求拜託 | ☐ | ☐ | ☐ | ☐ | ☐ |
| 無語 | ☐ | ☐ | ☐ | ☐ | ☐ |
| 晚安 | ☐ | ☐ | ☐ | ☐ | ☐ |
| 嚇到 | ☐ | ☐ | ☐ | ☐ | ☐ |

**通過標準**：8 張 × 5 條 = 40 個勾，至少要 32 個勾（80%）。少於 32 個 = 角色設定卡不夠強，下一單元前回 CH1-2 重寫。

#### 觀察成果（可被講師檢查）

- 你的 Gemini 對話視窗能看到從第 1 張到第 8 張**在同一個對話**內
- 你能說出**自己這 8 張裡哪一張最跑掉**，跑在哪條判準（例：「第 5 張耳朵變尖了，是第 1 條判準」）
- 你能把跑掉的 1–2 張用「補張」prompt 救回來（在我面前現場補一張）

#### 8 張一致 vs 8 張跑掉的對照範例（學員端可帶回複習）

**A 組「8 張一致」標準範例描述**（橘貓示範）：
- 全 8 張都是同一隻橘色虎斑貓，圓頭圓身、白領巾、頭身比 2:1
- 全 8 張視角統一正面（含晚安那張頭微側但仍是正面構圖）
- 線條粗細一致（粗黑線、無細描）
- 配色一致（淺橘 + 深橘條紋 + 白肚子，無突然變紅或變黃）
- 8 張排成一排，看起來「明顯是一套」，不會有任何一張「不像同一系列」

**B 組「8 張跑掉」反例範例描述**（同樣是橘貓題目，但失誤）：
- 第 1 張：標準橘貓，OK
- 第 2 張：耳朵尖了一點（小幅跑掉，仍可辨識）
- 第 3 張：變成花貓（黑白條紋而非橘色虎斑）—— **跑了第 1 條（同隻）**
- 第 4 張：突然變成側面構圖 —— **跑了第 4 條（同視角）**
- 第 5 張：線條變細變多 —— **跑了第 2 條（同風格）**
- 第 6 張：頭身比變成 1:1（不再是大頭小身體）—— **跑了第 3 條（同比例）**
- 第 7 張：背景變成深藍漸變 —— **跑了第 2 條 + 第 5 條（同風格 + 同色域）**
- 第 8 張：直接變成另一隻黑貓 —— **跑了第 1 條 + 第 5 條**

把這兩組擺在一起，學員一眼就能看出「跑掉是什麼意思」。比看 5 條文字判準有效 10 倍。

#### 5 條判準的優先順序（當判準互相矛盾時的決策）

實戰上偶爾會遇到「這張第 2 條過了第 4 條沒過」的兩難——例如線條風格一致但視角從正面變成 3/4 面。學員問「這算過還是不過」？

優先順序由高到低：
1. **第 1 條（同隻角色）** — 最重要，這條沒過 = 整張作廢，任何其他條過了也救不了
2. **第 4 條（同視角）** — LINE 貼圖套組視覺一致性的關鍵，視角跑了套組會看起來「亂」
3. **第 3 條（同比例）** — 頭身比跑了角色辨識度會降低
4. **第 2 條（同風格）** — 線條粗細、上色方式微跑可接受，極端跑（卡通變水彩）不可接受
5. **第 5 條（同色域）** — 色域微偏可在 CH1-4 後處理時微調，是 5 條中最「可救」的

**實戰決策**：第 1、4 條任一不過 = 必須補張；第 2、3、5 條微跑（程度可接受）= 可放行，整套還是能用。

## 常見錯誤 3 條（Common Pitfalls）

### 錯誤 1：重開新對話畫第 5 張表情，AI 把橘貓變成花貓

**錯誤現象**：學員第 4 張畫到「得意」覺得很滿意，關掉對話視窗去喝水，回來開新對話貼上原本的 prompt 想畫「求拜託」。結果 AI 畫出的不是橘色虎斑貓，是黑白花貓——配色完全不一樣，連「圓滾滾」的體型都丟了。

**原因**：Gemini 的「同一隻角色」記憶**只在同一個對話視窗內生效**。重開新對話 = AI 完全忘記前面 4 張長什麼樣，只剩 prompt 文字描述（文字描述對「視覺一致」的力道遠不如「看過前面那張圖」）。學員以為 prompt 一樣就會畫一樣，但 AI 對「橘色虎斑貓」的解讀有 100 種版本，每次重啟可能挑不同版本。

**解法**：
1. **絕對不重開對話**。喝水、上廁所、滑手機都可以，但對話視窗保持開著。
2. 不小心關掉了：到 Gemini 左側對話歷史找回原本對話，繼續續寫（不要按「新對話」）。
3. 對話歷史也找不回了：把已經產的 4 張 PNG **上傳**到新對話當參考圖，再請 AI 「依這 4 張的角色畫第 5 張」。這比純文字重啟好，但仍會有風格小偏移。

### 錯誤 2：每輪 prompt 都重講一次完整角色設定卡，額度燒得超快

**錯誤現象**：學員第 2 輪、第 3 輪、第 4 輪每次都複製貼上完整的 5 段角色設定卡（種族 / 體型 / 配色 / 個性 / 風格），然後加上「這次表情是 XX」。每輪 prompt 都是 200+ 字。第 5 張畫到一半就跳出「You've reached your daily limit」。

**原因**：Gemini 的免費額度按「token 量 × 圖片產出次數」計算。長 prompt + 8 次產圖 = 額度燒得比短 prompt + 8 次快很多。同對話多輪的整個價值就是「AI 已經知道角色長什麼樣，你不用每次重講」——學員每輪重講等於放棄了 context 記憶帶來的省額度紅利。

**解法**：
1. **第 1 輪用完整版**（角色設定卡完整 5 段 + 表情 + 動作）
2. **第 2–8 輪用簡化版**：「保持剛才那隻角色的所有設定（同隻、同風格、同比例、同視角、同色域），這次表情是 XX，動作是 YY。其他不變。」
3. 整段 prompt 控制在 50 字以內。8 張總 prompt 量約 1 個完整 + 7 個短句 = 比每輪重講省約 60% 額度。

### 錯誤 3：8 張全部產完才開始檢查，發現第 3 張就跑掉了，整套要重來

**錯誤現象**：學員一鼓作氣 8 個 prompt 連發，看 8 張縮圖排起來才發現「第 3 張的耳朵跟其他 7 張不一樣」「第 5 張變成側面」「第 7 張的橘色比較鮮豔」。這時候第 3、5、7 張都已經影響了後面 4、6、8 張的 context（AI 看到第 3 張耳朵尖了，第 4 張開始也跟著畫尖耳朵）。要修就要從第 3 張開始全部重來。

**原因**：Gemini 同對話多輪的 context 是**累積的**——前面所有圖都會被 AI 當作「這隻角色該長什麼樣」的參考。中間有一張跑掉沒及時修，後面會延續錯誤，一錯到底。學員憑「全部做完再檢查」的習慣，等於放任錯誤累積。

**解法**：
1. **每張產出後立刻 5 條判準自查**，不等全部產完。
2. 發現跑掉立刻補張：「上一張的耳朵跟第 1 張不一樣，請參考第 1 張的耳朵形狀重畫一次第 3 張。」AI 會在同對話內重畫，後續的第 4 張會以「修正後的第 3 張」為 context，不會繼續錯。
3. 講師現場示範時刻意「**每張都停下來檢查**」，讓學員看到節奏感（不是 8 張連發 = 快、是「產 + 查 + 補」循環 = 穩）。

## 檢核題 2 條（Quiz）

### Q1（概念驗證）：以下 4 種做法，哪一種最能讓 8 張表情保持同一隻角色？

- [ ] A. 開 8 個不同的 Gemini 對話視窗，每個視窗各畫 1 張表情，prompt 文字保持一致
- [ ] B. 在同一個對話視窗依序畫 8 張，每輪 prompt 都複製貼上完整的角色設定卡
- [ ] **C. 在同一個對話視窗依序畫 8 張，第 1 輪用完整角色設定卡，第 2–8 輪用「保持剛才那隻角色」的簡化句型 ←（正確）**
- [ ] D. 先單獨產 8 張不同表情，最後再用 AI 把它們「風格統一化」處理

**解析**：
- A 錯：每個對話視窗的 context 是獨立的，AI 每次都重新解讀「橘色虎斑貓」，8 張會有 8 種版本，是最嚴重的跑掉做法
- B 錯：同對話多輪有效，但每輪重講完整設定卡浪費額度（錯誤 2），且 AI 會以為你想重置角色，反而更不穩
- C 對：同對話多輪 + 引用 context 的句型（「保持剛才那隻」）= 角色一致性最強 + 省額度
- D 錯：「風格統一化」這種 AI 後處理目前免費平台沒這個功能，且就算有，也救不了「角色不同」（D 不是風格問題，是角色問題）

### Q2（應用驗證）：你正在做你家貓「小橘」的 8 張表情貼圖。產完第 3 張「生氣」之後，你發現小橘的耳朵變得比第 1 張尖很多、線條也粗了。這時候你該怎麼處理？

**情境細節**：
- 你已經在同一個 Gemini 對話內產了 3 張
- 第 1 張「笑」OK、第 2 張「哭」OK、第 3 張「生氣」耳朵跑掉
- 額度還有，但不確定夠不夠畫到第 8 張

**預期答案要點**（學員應答出這 3 點中至少 2 點）：

1. **不要重開對話**：重開會丟掉前 2 張正確的 context，反而讓後面更亂。維持原對話。

2. **用補張 prompt 在原對話內修第 3 張**：寫一個短 prompt 例如「上一張第 3 張的耳朵跟第 1 張不一樣，耳朵變太尖且線條變粗。請參考第 1 張的耳朵形狀（圓潤、線條粗細跟第 1 張一致）重畫一次第 3 張的生氣表情」。AI 會在同對話內產出修正版。

3. **修正版過自查後，繼續第 4 張**：把修正後的第 3 張當作新的 context 基準，第 4 張 prompt 用「保持剛才那隻角色」即可。**不要從第 1 張重來**，那會浪費額度且前 2 張本來是對的。

加分要點：
- 如果學員提到「每張產出立刻自查不等全部產完」（呼應錯誤 3），加分
- 如果學員提到「擔心額度時可以先看講師示範圖包，當天先學會流程」（呼應額度備援 SOP），加分
- 如果學員只說「重新產一張就好」沒講補張的具體 prompt 寫法，扣分（沒抓到「補張在原對話內」的核心）
