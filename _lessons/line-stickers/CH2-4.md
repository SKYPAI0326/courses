## 教學流程（Teaching Flow）

> **5 段時間配置（合計 55 min）**：Hook 4 / Concepts 6 / Demo 25 / Hands-on 15 / Verification 5。Demo 內部再細拆 3 步（規格與版型 8 / 文字三大設計重點 12 / 模擬退件案例驗收 5）。

### 破題 / Hook（4 min）

橘貓上班族的 8 張靜態送審過了、PRAC1 結束了，今天進階班下半場第一堂要做的，是 LINE 三類貼圖裡**最有趣、也最容易被忽略**的一種——**訊息貼圖**。

訊息貼圖跟你之前做的「主貼圖（370×320）」完全不一樣。它有兩個特色，先講清楚再開始：

1. **尺寸不同**：訊息貼圖是 **270×270 px PNG**（正方形），不是 370×320。第一次做的人很容易拿著上次的橘貓直接丟上來，結果格式被擋下來。
2. **可換字**：每張訊息貼圖會在固定位置疊上**對方的名字**或一段**收件人專屬的話**。例如「**OOO 早安！**」這張貼圖，朋友 A 收到看到的是「**小美 早安！**」、朋友 B 收到看到的是「**小華 早安！**」——OOO 那塊是 LINE 自動把對方名字代入的。

這兩個特色合起來，意思是：**你今天設計的不是「一張畫」，而是一個「版型」**。版型上「圖」是固定的、「字」是可換的、「位置/字型/顏色」是你決定的。學員拿到的訊息貼圖比靜態貼圖更黏——朋友每次傳「OOO 早安！」配上你做的橘貓，朋友的朋友看到也會問「這套貼圖哪裡買的」。

但麻煩的地方也在這裡：因為要疊字，**字的位置不能擋到角色五官**、**字的顏色要跟底圖夠對比看得清楚**、**字型要跟整體角色風格搭**。這三件事做不好，整張貼圖就是「文字模糊看不清」或「字壓在貓臉上像被打馬賽克」——這是 R1 退件最常見的原因。

今天 55 分鐘的契約：你要產出 **3 種版型**的訊息貼圖（早安 / 偷懶 / 加油），通過「上傳前自查清單」，並用我準備的「**模擬退件案例**」幫自己驗收。實際送審 1-7 天回覆，那是課後的事；課堂內你要學會的是「**送審前我自己先當審核員**」。

### 概念 / Concepts（6 min）

- **訊息貼圖（Message Sticker）** — LINE 三類貼圖（靜態 / 動態 / 訊息）裡的第三類，用戶傳訊時選用，會把**對方在 LINE 的顯示名字**疊到貼圖預設的「OOO」位置。視覺上是「圖 + 可變文字」的組合版型，不是單純一張圖。
- **訊息貼圖規格** — **270×270 px PNG**（正方形，與主貼圖 370×320 不同），**檔案 ≤ 1MB**、**透明背景**、**8 / 16 / 24 張選一**。本課示範以「8 張版本」為主，跟 Part 1 對齊。
- **可換字疊圖（Custom Text）** — LINE 訊息貼圖的核心機制：你預先在版型上留**文字位置**（一個矩形區），LINE 會在傳送時把對方的暱稱/名字代入這個位置。文字字型、顏色、大小都由你在 Creators Market 後台設定，**不由 LINE 自動配**。
- **文字疊圖三大設計重點** — 訊息貼圖好不好看、能不能過審，靠這三點：(1) **文字位置**（避開角色五官、放在留白區）、(2) **字型選擇**（LINE 預設字型 vs 自訂上傳）、(3) **顏色對比**（淺色字配深底 / 深色字配淺底，AAA 對比為佳）。
- **訊息貼圖最常見退件** — 雖然 R1-R5 五大原因都適用，但訊息貼圖最容易踩兩條：**R1 文字模糊**（字太小 / 對比不夠 / 字型太細）和 **R3 邊緣殘影**（去背沒做乾淨，疊上字後白邊更顯眼）。R2 角色不一致比例反而少（每張通常獨立設計）。
- **訊息貼圖的 R2 / R4 / R5 自查**（較低頻但仍要看）：
  - **R2 角色不一致**：8 張並排看是不是同一隻角色（即使各張獨立設計，主角仍不能換）
  - **R4 含他人 IP**：訊息貼圖最常踩——文字示範用了「**麥當勞**」「**iPhone**」「**LINE Pay**」這類品牌名 / IP 名當可換字範例
  - **R5 含真實人物**：可換字機制本身沒事，但**送審時附的範例文字**若打了真實朋友本名 + 配真人風角色，就踩 R5。範例文字一律用「小明 / 同事 / 老闆」這種通用代稱
- **模擬退件案例** — 因為 LINE 真實審核要 1-7 天，課堂無法等到結果，所以本單元用「**講師預先準備好的失敗範本**」當驗收基準。學員把自己的版型跟失敗範本對照，看自己有沒有犯同樣的錯，等於提前自我退件 + 修補一輪。

### 操作示範 / Demo（25 min · 3 步流程拆解）

> 本段配對「3 種版型範例」+「文字位置範本」+「模擬退件案例」三組試跑包資產（lesson-writer 後製階段補上）。學員先看講師走完，再自己跑一輪。

#### Step 1｜訊息貼圖規格 + 3 種版型介紹（8 min）

訊息貼圖跟主貼圖最大差異就是「正方形 270×270」+「文字位置預留」，所以版型設計從第一筆開始就要不一樣。我準備了 3 種**橘貓上班族訊息貼圖版型**示範，覆蓋最常見的三種疊字情境：

##### 版型 A｜「OOO 早安！」（問候型 · 文字在上方）

- **構圖**：橘貓在畫面**下半部**（佔約 60% 高度），眼睛瞇成笑彎彎、揮手；上半部留 30% 高度純色或漸層底，給文字疊圖用
- **文字位置**：畫面上方 1/3 區域，水平置中，「**OOO**」+「**早安！**」分兩行或一行排
- **字型**：粗體圓潤字（搭橘貓的可愛感），字級夠大（OOO 那塊至少佔畫面寬度的 60%）
- **顏色**：深咖啡色文字（#5C3317）配淺米色底（#FFF8E7）；對比夠、不刺眼、跟橘貓主題色 #b5703a 同色系
- **使用情境**：朋友早上第一次傳訊時用，「小美 早安！」「家人 早安！」「老闆 早安！」都套得進去

##### 版型 B｜「OOO 在偷懶」（吐槽型 · 文字在下方）

- **構圖**：橘貓在畫面**上半部偏左**（佔約 50%），表情無語、白眼翻到天上、手叉腰；右側留空、下半部全留給文字
- **文字位置**：畫面下方 1/3，水平置中，「**OOO**」用第一行強調（粗體、深色），「**在偷懶**」第二行較細較輕
- **字型**：手寫感粗體（增加吐槽的親切調侃感），不要太正式
- **顏色**：黑色文字 + 黃色螢光底色塊（像便利貼那種劃重點感）；對比強到一秒看清楚
- **使用情境**：朋友群組起鬨吐槽時用，「老王 在偷懶」「阿明 在偷懶」秒傳

##### 版型 C｜「OOO 加油」（鼓勵型 · 文字在角色旁邊）

- **構圖**：橘貓在畫面**左下角**（佔約 40%），舉起拳頭做加油手勢、眼神堅定；右側 60% 空間留給文字 + 簡單裝飾元素（小星星、小愛心）
- **文字位置**：畫面右側中央，垂直置中，「**OOO**」上面一行（大字、粗體），「**加油！**」下面一行（更大、加粗、可加感嘆號驚嘆效果）
- **字型**：穩重粗體無襯線字（給人「我認真在挺你」的感覺），不要用花俏字型
- **顏色**：深藍色文字（#1E4A6B）配米白底，「加油！」那行可改成橘色 #b5703a 跟橘貓呼應
- **使用情境**：朋友考試 / 面試 / 比賽前傳，「妹妹 加油！」「同事 加油！」

**講師現場示範時必說**：「3 種版型對應 3 種使用場景，**它們的差異不只是文字內容、更是文字位置**——版型 A 文字在上、版型 B 文字在下、版型 C 文字在右。**位置決定氣氛**：上方=正式、下方=吐槽、右側=陪伴。學員回去做自己 8 張時要知道這套對應關係，不是隨便挑一個位置擺。」

**規格速查（鎖死，不要記錯）**：
| 項目 | 訊息貼圖 | 對比：主貼圖 |
|------|--------|------|
| 尺寸 | **270×270 px** | 370×320 px |
| 形狀 | 正方形 | 長方形 |
| 檔案 | ≤ 1MB | ≤ 1MB |
| 背景 | 透明 PNG | 透明 PNG |
| 張數 | 8 / 16 / 24 | 8 / 16 / 24 / 32 / 40 |

#### Step 2｜文字疊圖三大設計重點實戰（12 min · **本單元核心**）

這 12 分鐘是 CH2-4 的精華。我帶著學員把 3 種版型的「文字位置 / 字型 / 顏色」三件事做到位。

##### (A) 文字位置：避開五官、放在留白區（4 min）

訊息貼圖最常踩的雷是「**字壓在角色臉上**」。看起來像被打馬賽克，審核會被退 R1 低品質。

**位置黃金法則 3 條**：

1. **避開五官**：眼睛、鼻子、嘴巴、表情核心區一律不能被文字壓到。學員設計時先**畫一個矩形把五官圈起來當禁區**，文字只能放禁區外。
2. **放在純色 / 漸層留白區**：避開有圖案、有細節的區域（橘貓的毛紋、衣服花紋）。背景越單純、文字越容易看清。
3. **與角色保持 ≥ 20px 距離**：文字邊緣跟角色邊緣不要貼太近，會擠在一起、像「字長在貓身上」。預留 20-30px 安全距離。

**示範操作**：
- 打開 Photopea → 開新檔 270×270 → 拉一條輔助線把畫面分成上 1/3、下 1/3、左 1/3、右 1/3（井字構圖）
- 把橘貓 PNG 拖進去，**先放好角色位置**（決定角色在哪 1/9 格）
- 用矩形工具框出**文字疊圖區**，這個矩形要 (a) 不壓到五官 (b) 落在純色區 (c) 與角色距離 ≥ 20px
- **這個矩形之後到 LINE Creators Market 後台會用「Text Area」設定對應**（學員不用真的畫線到輸出檔上，輔助線只是設計輔助）

##### (B) 字型選擇：LINE 預設字型 vs 自訂上傳（4 min）

訊息貼圖的文字其實有兩條路：

**Path 1：用 LINE 後台預設字型**（推薦給入門學員）
- LINE Creators Market 訊息貼圖後台提供約 **8-10 款預設字型**（中英日各幾種）
- 優點：免費、不用自己處理、字型授權無風險
- 缺點：選擇有限、跟其他人撞字型機率高
- **本課推薦**：版型 A（早安）用「**圓體粗黑**」、版型 B（偷懶）用「**手寫粗體**」、版型 C（加油）用「**標楷粗體**」

**Path 2：把文字當圖層直接燒進 PNG**（進階做法 · 本課不推）
- 學員自己在 Photopea 用任何字型把文字打上去 → 連同角色一起匯出 PNG
- 優點：字型自由、可以用任何特殊字型
- 缺點：**OOO 那塊就不能換字了**（因為已經燒進圖裡）→ 失去訊息貼圖「可換字」的核心特色 → 等於做出一張「假訊息貼圖」
- **本課明確不推此路**：學員想用特殊字型 → 改回去做主貼圖系列就好；訊息貼圖的價值就是可換字

**講師現場示範重點**：「我推 Path 1。你做訊息貼圖的目的就是『可換字』，如果為了用花俏字型把字燒進圖，就是把訊息貼圖做成假的靜態貼圖。**寧可字型普通一點，換字功能保留**——這是訊息貼圖跟主貼圖的核心差異。」

##### (C) 顏色對比：淺字配深底 / 深字配淺底（4 min）

訊息貼圖第二大退件原因是「**字看不清楚**」。原因都是顏色對比沒做夠。

**對比黃金法則**：

| 底色 | 文字色 | 對比結果 | 是否安全 |
|------|--------|---------|---------|
| 米白 #FFF8E7 | 深咖啡 #5C3317 | 高 | ✅ 安全 |
| 深咖啡 #5C3317 | 米白 #FFF8E7 | 高 | ✅ 安全 |
| 米白 #FFF8E7 | 黃色 #FFD700 | 低（同色系） | ❌ 退件 R1 |
| 漸層紫粉 | 白色 | 中（漸層底會吃字） | ⚠️ 加描邊或陰影才安全 |
| 純黑 #000000 | 螢光黃 #FFFF00 | 極高 | ✅ 安全（吐槽型常用）|

**3 條判準**：

1. **AAA 等級對比**：用免費工具 [contrast-ratio.com](https://contrast-ratio.com) 檢查，比例 ≥ **7:1** 才算 AAA、≥ 4.5:1 為 AA。訊息貼圖建議至少 AA 4.5:1 以上。
2. **避開漸層底放純色字**：如果底是漸層（像版型 B 偷懶的便利貼黃漸層），文字要加 **2-3px 黑色描邊**或淡淡的陰影，才能在漸層上保持可讀
3. **手機小螢幕測試**：把設計好的訊息貼圖縮到 **120×120 px** 預覽（手機聊天室實際大小），如果這時候字看不清，就回去加大字級或換顏色

**示範操作**：
- 在 Photopea 把字打上去 → 截圖 → 用 contrast-ratio.com 上傳檢查 → 看 ratio 數字
- 把畫布縮成 120×120 px 預覽 → 自己問自己「現在這字我用手機聊天時看得清嗎」
- 不行就改：放大字級 / 換對比更強的顏色 / 加描邊

#### Step 3｜模擬退件案例驗收（5 min）

本單元最特別的一段：**用講師預先做好的「失敗範本」當驗收標準**，學員把自己 3 種版型的設計跟失敗範本對照，自己當審核員幫自己挑毛病。

##### 模擬退件案例 1｜版型 A 失敗版（R1 文字模糊）

| 對照 | 正解版（橘貓早安） | 失敗版 |
|------|------------|------|
| 文字大小 | OOO 佔畫面寬 60% | OOO 只佔 25%（太小，120×120 預覽看不清）|
| 文字顏色 | 深咖啡 #5C3317 | 淺米色 #E8D8B5（跟底色幾乎同色，對比 < 1.5:1）|
| 模擬退件理由 | — | **R1 低品質**：文字模糊、對比不足，無法辨識 |

##### 模擬退件案例 2｜版型 B 失敗版（R3 邊緣殘影 + 字壓臉）

| 對照 | 正解版（橘貓偷懶） | 失敗版 |
|------|------------|------|
| 角色去背 | 邊緣乾淨無殘影 | 橘貓邊緣有白光暈、耳朵尖殘留底色 |
| 文字位置 | 下方 1/3 純色區 | 「在偷懶」三個字壓在橘貓臉上、擋到眼睛 |
| 模擬退件理由 | — | **R3 邊緣殘影 + R1 文字遮擋表情** |

##### 模擬退件案例 3｜版型 C 失敗版（規格錯誤）

| 對照 | 正解版（橘貓加油） | 失敗版 |
|------|------------|------|
| 尺寸 | 270×270 px PNG | 370×320 px（誤用主貼圖規格）|
| 檔案大小 | 600KB | 2.3MB（PNG 沒壓縮）|
| 模擬退件理由 | — | **規格不符**：尺寸錯誤、檔案超過 1MB 上限 |

**驗收流程**：
1. 學員把自己做的 3 種版型擺出來
2. 對照三組「正解 vs 失敗」範本，**逐項勾選自己有沒有犯同樣的錯**
3. 如果犯了 → 當下修補（位置調整、顏色加深、檔案壓縮）
4. 修完再對照一輪
5. **不對自己滿意之前不放行**——這就是「模擬退件」的意義：在送審前先當一輪審核員

### 動手 / Hands-on（15 min · 3 種版型實作）

**任務**：在這 15 分鐘內，依 Step 1-2 做出 **3 種版型**的訊息貼圖（OOO 早安 / OOO 在偷懶 / OOO 加油），通過「上傳前自查清單」+ 用 Step 3 模擬退件案例自我驗收。

**進度檢核點**（每 5 分鐘自檢一次）：

| 時間 | 應該完成 |
|------|---------|
| 5 min | 版型 A（早安）：構圖+文字位置+顏色定案 |

- 失敗版 PNG（刻意做爛）
- 失敗版退件理由說明卡（A4 一頁，紅字醒目）
- 修補步驟對照（學員看完知道怎麼從失敗版改回正解版）

> **失敗版必須由講師親手刻意做爛**，不要用 AI 隨意生成的圖湊數——刻意做的失敗點要明顯到肉眼一秒看出，這是「模擬退件」的教學價值所在。

## 商業情境案例（Cases）— 3 個自用情境

> Style Guide 規定本課用「**你**」+ 虛構主角開場，**不用阿凱 / 雯姊**等具名學員。下列三個情境皆以橘貓上班族為主角串起，呈現訊息貼圖在不同生活場景下的使用樣態。

### 情境 1｜橘貓「OOO 早安」：給家人群組的早安問候

**身分**：30 出頭的上班族，家裡有一個 5 人的家人 LINE 群組（爸媽 + 兄弟姊妹），每天早上會輪流傳早安貼圖打招呼。

**任務**：你做一張訊息貼圖「OOO 早安！」，傳到家人群組會自動帶入家人的暱稱——爸傳出去顯示「老爸 早安！」、媽傳出去顯示「老媽 早安！」、你的妹妹傳出去顯示「小妹 早安！」。一張貼圖、五個人輪用，每個人都覺得「這張是專門給我的」。

**本單元設計重點**：
- **構圖**：橘貓在下半部（揮手笑臉），上半部留白給文字
- **文字位置**：上方 1/3，水平置中
- **字型**：圓體粗黑（搭橘貓可愛感）
- **顏色**：深咖啡 #5C3317 字配米白 #FFF8E7 底（高對比、暖色調）
- **可換字效果**：「OOO 早安！」會變成「老爸 早安！」「小妹 早安！」每個人收到都像專屬

**本單元終點**：你做出版型 A 的 270×270 PNG，上傳到 LINE Creators Market（CH2-8 才送審），預覽縮成 120×120 仍可讀，自查清單 8 條全打勾。

### 情境 2｜橘貓「OOO 在偷懶」：辦公室群組吐槽神器

**身分**：30 出頭的上班族，公司的同事群組裡有 8 個人，每天午休大家互相吐槽誰在偷懶（其實都很努力，這是一種辦公室幽默）。

**任務**：你做一張訊息貼圖「OOO 在偷懶」，傳到辦公室群組會自動帶入被吐槽對象的暱稱——你看到老王在滑手機就傳「老王 在偷懶」、看到阿明跑去買咖啡就傳「阿明 在偷懶」。一張貼圖、整個辦公室輪流被吐槽。

**本單元設計重點**：
- **構圖**：橘貓在上半部偏左（白眼翻到天上），下半部留白
- **文字位置**：下方 1/3，水平置中
- **字型**：手寫粗體（增加吐槽的調侃感）
- **顏色**：黑色字配螢光黃底色塊（像便利貼那種劃重點，吸睛度極高）

**本單元終點**：版型 B 通過自查清單，文字壓臉問題在動手段就被你自己抓出來修正。

### 情境 3｜橘貓「OOO 加油」：朋友考試 / 比賽 / 面試前的鼓勵

**身分**：大學生，朋友圈裡常常有人面臨關卡——期中考、研究所推甄、駕照路考、找工作面試。你想做一張可以套用在所有情境的「加油」貼圖。

**任務**：你做一張訊息貼圖「OOO 加油！」，誰要面對挑戰就傳給誰——朋友考試前傳「小華 加油！」、室友面試前傳「室友 加油！」、家人手術前傳「老媽 加油！」。一張貼圖適配所有「我陪你撐過去」的情境。

**本單元設計重點**：
- **構圖**：橘貓在左下（舉拳加油手勢），右側 60% 留給文字
- **文字位置**：右側中央，垂直置中
- **字型**：穩重粗體（給人「我認真在挺你」的感覺）
- **顏色**：深藍 #1E4A6B 主色 + 橘色 #b5703a 輔色（深藍給穩定感、橘色呼應角色主題色）

## 常見錯誤 3 條（Common Pitfalls）

### 1. 把文字直接燒進 PNG 裡 → 失去「可換字」核心、做出假訊息貼圖

**錯誤現象**：學員為了用花俏字型或想要更精緻的視覺效果，把「OOO 早安」幾個字當圖層直接打進 Photopea，跟橘貓一起匯出 PNG。上傳到 LINE Creators Market 訊息貼圖後台後，**OOO 那塊變成「OOO」三個字面字**，朋友傳出去顯示的還是「OOO 早安」，沒有自動代入名字的功能。

**原因**：學員不理解「訊息貼圖的可換字機制」是 LINE 後台在傳訊時動態疊上去的——你的 PNG 裡如果已經把字燒進去了，LINE 沒有「文字疊圖區」可以動態替換，自然就變成靜態文字。學員把訊息貼圖當成「會疊字的主貼圖」，把字當成設計元素而不是替換位置。

**解法**：訊息貼圖的 PNG **只放角色 + 背景，不要放文字**。文字部分要用 LINE Creators Market 後台的「**Text Area（文字疊圖區）**」設定（CH2-8 進階上架實作會詳細教，本單元只要學員知道「字不要燒進圖」這個原則）。如果學員真的很想要花俏字型 → 改去做 CH1 系列的主貼圖，主貼圖沒有可換字限制可以隨便燒字。

---

### 2. 文字壓在角色五官上 → 模擬退件案例 R1 + R3 雙重退件

**錯誤現象**：學員設計版型 B（偷懶）時，因為想把橘貓放大讓表情更有戲，結果橘貓佔了 80% 畫面，文字「OOO 在偷懶」只能擠在橘貓臉上、剛好壓住眼睛和鼻子。送審後（或對照模擬退件案例後）會看到 **R1 低品質（文字遮擋表情）+ R3 視覺擁擠** 雙重問題。

**原因**：學員的本能是「主角越大越好」，但訊息貼圖的核心是「圖 + 字共存」，不是「圖最大」。學員沒有把「文字位置」當成構圖的第一個決策——應該先決定文字放哪，再決定角色放哪，反而本末倒置。

**解法**：構圖第一步**先框出文字疊圖區**（用 Photopea 矩形工具拉一個矩形），這個矩形決定了角色不能進的禁區。然後再把角色 PNG 拖進去、避開文字區擺放。這樣設計就不會出現「角色擠到文字區」的問題。**口訣：先文字、後角色，文字位置決定構圖。**

---

### 3. 文字顏色與底色對比不足 → 縮到 120×120 預覽時看不清

**錯誤現象**：學員設計版型 A（早安）時，覺得淺色系比較溫柔，選了「淺米黃 #FFD7A0 字 + 米白 #FFF8E7 底」這種同色系搭配。Photopea 全螢幕看還算可讀，但縮成 120×120（手機聊天室實際大小）後，文字幾乎跟底色融在一起、像沒寫字一樣。送審大概率被退 R1 文字模糊。

**原因**：學員以「設計圖大小看起來覺得 OK」當判準，沒考慮**手機聊天室實際呈現大小**。LINE 審核員會在手機端看，貼圖在聊天室是縮放顯示，對比不夠直接看不見。同色系搭配在大圖時還能看，縮小後就崩。

**解法**：每張訊息貼圖設計完成後，**強制縮成 120×120 預覽**，自問「我用手機聊天時看得清這幾個字嗎」。看不清就 (a) 改用對比更強的顏色（深 vs 淺、暖 vs 冷）(b) 加 2-3px 黑色描邊或淡淡陰影 (c) 放大字級。**用 contrast-ratio.com 線上工具檢查色號對比 ≥ 4.5:1（AA 等級）才放行**，不要靠肉眼判斷。

## 檢核題 2 條（Quiz）

### Q1（概念驗證）｜訊息貼圖的「可換字」機制是怎麼運作的？

- [ ] A. 我在 PNG 裡把「OOO」三個字打上去，LINE 會自動把它替換成名字
- [ ] B. PNG 裡只放角色和背景、不放文字；文字位置由 LINE Creators Market 後台的「Text Area」設定，傳訊時 LINE 動態把對方名字疊到那個位置 ←（正確答案）
- [ ] C. 我每個朋友都要做一張不同名字的貼圖（朋友 A 一張「小美 早安」、朋友 B 一張「小華 早安」）
- [ ] D. 訊息貼圖跟主貼圖一樣，都是把字燒進圖裡，只是尺寸不同

**講解**：B 是正解。訊息貼圖的核心精神是「圖固定、字動態」——你做一張版型，LINE 在傳訊時根據收件人的名字把字疊上去。所以你的 PNG 裡**不能**放文字（A 錯：「OOO」會變成字面字）；也**不需要**為每個朋友做一張（C 錯：那是主貼圖的做法）；訊息貼圖**不是**燒字進圖（D 錯：燒字 = 失去可換字 = 做出假訊息貼圖）。記住「先文字、後角色」這個原則，文字位置由後台設定、不由 PNG 本身決定，這就是訊息貼圖跟主貼圖的最大差異。

### Q2（應用驗證）｜情境題

> 你在做版型 B（橘貓 OOO 在偷懶），第一版設計：橘貓佔畫面 80%（白眼翻到天上很有戲），文字「OOO 在偷懶」只好擠在橘貓肚子上，因為其他地方都放不下。底色是純白，文字用深灰色。Photopea 全螢幕看覺得有點擠但還能接受。問：你接下來最該做哪 3 件事？

**預期答案要點**（口頭或舉手作答即可）：

1. **承認現在文字壓到角色身上是設計失誤**——不是「有點擠但還能接受」，而是「對照模擬退件案例 2 就是 R3 視覺擁擠 + R1 文字遮擋的雙重退件原因」
2. **重新構圖：先決定文字位置再放角色**——把「OOO 在偷懶」的文字疊圖區用矩形框出來放在畫面下方 1/3、純色區，然後把橘貓縮小到上半部偏左
3. **縮成 120×120 預覽再驗收**——確認手機聊天室實際大小下文字仍可讀；如果還是不清，改顏色（純白底配深灰字 → 改成黃色底色塊配黑字增加對比）

**錯誤答案**（會聽到的常見誤解）：

- ❌「全螢幕看 OK 就好，反正聊天室裡大家也不會仔細看」→ 錯。LINE 審核員會看，使用者體驗也差。120×120 預覽是強制檢查項。
- ❌「橘貓 80% 看起來主角感很強，這版設計很到位」→ 錯。訊息貼圖核心是「圖 + 字共存」，不是「圖最大」。本末倒置。
- ❌「壓在貓肚子上沒關係，反正不是壓到臉」→ 錯。R1 退件不只看「壓到五官」，還看「文字 vs 圖混雜程度」。文字壓到角色任何部位都增加退件機率，安全做法是「文字完全不壓到角色」。
