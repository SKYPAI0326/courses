## 教學流程（Teaching Flow）

### 破題 / Hook（5 min）

橘貓上班族今天要產 8 張表情貼圖。第一張「笑」畫得超可愛，圓滾滾橘色虎斑貓戴著白領巾，剛剛好。接著第二張「哭」——AI 畫了一隻長得有點像、但花紋比較斜、領巾變成藍色、體型也瘦了一圈的橘貓。第三張「生氣」——這次直接變成花貓，連橘色都沒了。

如果你今天什麼都不做，只是用 AI 隨便產 8 張，你會得到 **8 隻不同的貓**。LINE 審核員看到這個套組，會直接退件，理由就一條：「角色不一致」。

這堂 35 分鐘只做兩件事：

1. **教你寫一份「角色設定卡」**——一份結構化文件，把你腦中的角色寫到 AI 看得懂
2. **教你 3 種手法把這份文件用出來**——上傳參考圖 / 同對話多輪 / 一次產多張變化版本，三種各有優劣，知道什麼時候用哪一種

下一堂 CH1-3 你會花 75 分鐘真的去產 8 張。今天 35 分鐘是「上戰場前裝彈」。彈裝不好，下一堂就是空槍。

---

### 概念 / Concepts（7 min）

這堂課的全部內容濃縮成 5 個關鍵詞。後面所有操作都是這 5 個詞的展開。

- **角色設定卡** — 一份 5 格的表（種族 / 體型 / 配色 / 個性 / 風格參考），把「我腦中的那隻」寫到 AI 一字一句能複製貼上的程度。寫不出來 = 你還沒想清楚，不是 AI 的錯。
- **角色一致性** — 同一個角色出現在 8 張不同情境的圖時，外觀辨識度仍然成立。本單元的「角色設定卡 5 格」（種族 / 體型 / 配色 / 個性 / 風格參考）是**事前的設計卡**，幫你把腦中的角色寫清楚；CH1-3 還有另一份「**角色一致性檢查清單 5 條判準**」（同隻 / 同風格 / 同比例 / 同視角 / 同色域）是**事後的自查表**，產出後逐張勾。兩組都是 5 條但角色不同——一個寫設定、一個查產出，本課唯一的「自查表」是 CH1-3 那組。
- **AI 漂移（drift）** — 同對話內 AI 連續產圖到第 4-5 張，角色開始「慢慢變樣」（毛色變淡、體型變瘦、配件消失）。這是 AI 的常態，不是你哪裡設錯。
- **參考圖（reference image）** — 上傳一張現成的圖給 AI 看，叫它「照這隻畫」。最強但也最耗額度的手法。
- **起手式（template prompt）** — 角色設定卡寫完後，第一次餵給 AI 的開場 Prompt。本課給你 4 類現成起手式（寵物 / 真人 / 物品 / 幻想），照填空格就能用。

### 操作示範 / Demo（15 min）

接下來分三段：先教「角色設定卡怎麼填」，再教「4 類起手式怎麼用」，最後教「3 種一致性手法怎麼選」。

#### Demo 1：角色設定卡（5 格填空模板）

打開一張白紙或記事本，把這 5 格抄下來：

```
【角色設定卡】

1. 種族 / 物種：________________
   （是什麼生物 / 物品 / 人？例：橘色虎斑貓 / 粉紅水母 / 戴眼鏡的女生 / 咖啡杯擬人）

2. 體型 / 比例：________________
   （Q 版圓滾滾 / 寫實正常比例 / 二頭身 / 大頭小身）

3. 配色（最多 3 色 + 1 配件色）：________________
   （主色：________ / 副色：________ / 點綴：________ / 配件：________）

4. 個性 / 表情常態：________________
   （溫和 / 活潑 / 慵懶 / 暴躁；眼睛常態：圓眼 / 半瞇 / 細線；嘴巴常態：微笑 / 嘟嘴 / 大口）

5. 風格參考（畫風 + 線條 + 色調）：________________
   （扁平卡通 / 漸變水彩 / 馬克筆塗鴉 / 像素風；粗線條 / 細線條 / 無線條；暖色調 / 冷色調 / 高彩度）
```

**示範：橘貓上班族的設定卡**

```
1. 種族 / 物種：橘色虎斑貓（站立兩腳走，會講話的擬人貓）
2. 體型 / 比例：Q 版圓滾滾，二頭身，肚子微凸
3. 配色：主色橘 #E89B5A / 副色奶油白（肚子和臉頰）/ 點綴深橘色虎斑紋 / 配件白色領巾
4. 個性：溫和但偶爾無語的上班族；眼睛常態半瞇圓眼，嘴巴常態微微嘟嘴
5. 風格參考：扁平卡通、粗線條（外輪廓 3px）、無漸層、暖色調（橘 + 米白 + 淺褐）
```

**為什麼這 5 格非寫不可**：

- 沒寫種族 → AI 隨機選貓 / 狗 / 狐狸
- 沒寫體型 → 第 2 張變寫實貓、第 3 張變二頭身
- 沒寫配色 → AI 隨便挑橘的色號，每張都不同
- 沒寫個性 → AI 隨機選表情常態，溫和的貓會被畫成兇狠
- 沒寫風格 → 8 張可能 5 張卡通 3 張水彩

寫完這張卡，後面 8 張就是「拿這張卡去配 8 種情境」，而不是「每張重講一次」。

#### Demo 2：4 類起手式 Prompt 模板

不同學員想做的角色不一樣，這 4 類起手式覆蓋大部分情境。每類都附**示範角色**和**填空版**。

##### 類型 A：寵物擬人（最大宗，零授權問題）

> **示範角色**：橘貓上班族 / 柴犬店長 / 倉鼠房東

**填空起手式**：

```
請畫一隻擬人化的【種族】，
外觀：【體型】，主色【主色號】+ 副色【副色】，配件是【配件】。
個性是【個性形容詞】，眼睛【眼睛狀態】、嘴巴【嘴巴狀態】。
風格：【畫風】，【線條粗細】，【色調】。
背景：純白色。
這是 LINE 貼圖角色設定圖（第一張，正面站姿，比讚）。
```

**橘貓上班族示範填法**：

```
請畫一隻擬人化的橘色虎斑貓，
外觀：Q 版圓滾滾二頭身，主色橘 #E89B5A + 副色奶油白（肚子和臉頰），配件是白色領巾。
個性是溫和但偶爾無語的上班族，眼睛半瞇圓眼、嘴巴微微嘟嘴。
風格：扁平卡通，粗線條（外輪廓 3px），暖色調無漸層。
背景：純白色。
這是 LINE 貼圖角色設定圖（第一張，正面站姿，比讚）。
```

##### 類型 B：真實人物擬卡通（自己 / 伴侶 / 小孩 / 朋友）

> **示範角色**：把自己畫成 Q 版上班族 / 把女兒畫成卡通公主 / 把伴侶畫成漫畫主角

**【紅線提醒，不可跳過】**：

- **自己**：可以，你授權自己。
- **伴侶 / 小孩 / 父母**：自用送他本人 OK；但如果你要做成 LINE 貼圖**上架可下載**，等同公開使用對方肖像，**必須事先取得對方口頭或書面同意**。
- **朋友 / 同事 / 不熟的人**：上架等同公開散布，**不要做**。連自用都建議避開——LINE 審核可能因為「真實人物」直接退件，你還白白用人家的臉。
- **公眾人物 / 名人**：絕對禁止，會違反 LINE 條款 + 肖像權，帳號可能被處分。

**填空起手式**（先擬卡通化，避免太像真人）：

```
請把這張照片裡的人物畫成 Q 版卡通角色（不是寫實肖像）。
保留：【保留特徵：髮型 / 髮色 / 眼鏡 / 標誌性配件】。
卡通化：【體型】，五官簡化（眼睛變圓點或半月、嘴巴變一條線）。
配色：主衣著【顏色】，背景純白。
風格：【畫風】，【線條】，【色調】。
這是 LINE 貼圖角色設定圖（第一張，正面 Q 版，揮手）。
不要畫成寫實人像，要明顯卡通化、辨識度低於原照片，但保留特徵 1-2 個。
```

**為什麼要強制「卡通化、辨識度低於原照片」**：

- 真實人物擬卡通的合規邊界很模糊。畫得太像會引發肖像爭議；畫得太抽象又認不出。本課建議「保留 1-2 個特徵 + 整體 Q 版化」，視覺上像本人但不會被誤認為照片。
- 上架前再讀一次 LINE 條款的「禁止上傳真實人物」段落（CH1-1 已連結），自評是否會被退件。

##### 類型 C：物品擬人（咖啡杯 / 行李箱 / 鬧鐘 / 包包）

> **示範角色**：愛抱怨的咖啡杯 / 老是被遺忘的行李箱 / 起床氣鬧鐘

**填空起手式**：

```
請畫一個擬人化的【物品】（物品本體保持原形，但加上五官和四肢）。
物品本體：【顏色】+ 【材質：陶瓷/塑膠/布/金屬】，【其他特徵：把手/拉鍊/按鈕】。
擬人五官：【眼睛位置：上半部 / 中央】，【眼睛形狀】，【嘴巴形狀】。
四肢：【手腳設計：細線手腳 / 圓潤短手短腳 / 完全沒有四肢只有眼睛】。
個性：【個性形容詞】。
風格：【畫風】，【線條】，【色調】。
背景：純白色。
這是 LINE 貼圖角色設定圖（第一張，正面，舉手打招呼）。
```

**示範填法（愛抱怨的咖啡杯）**：

```
請畫一個擬人化的馬克杯（杯子本體保持原形，但加上五官和四肢）。
物品本體：白色陶瓷 + 一條藍色橫線裝飾，有把手，裝著咖啡（看得到液面）。
擬人五官：眼睛在杯身上半部，半瞇圓眼，嘴巴微微嘟嘴像是在抱怨。
四肢：細線手腳，從杯身兩側伸出。
個性：愛抱怨但其實很可靠的咖啡杯。
風格：扁平卡通，粗線條，暖色調。
背景：純白色。
這是 LINE 貼圖角色設定圖（第一張，正面，舉手打招呼）。
```

##### 類型 D：幻想生物（雲朵 / 小怪獸 / 水母 / 自創角色）

> **示範角色**：粉紅水母 / 慵懶雲朵 / 三隻角小怪獸 / 自創精靈

**填空起手式**：

```
請畫一隻原創的【幻想生物名稱】，沒有對應現實生物。
外觀：【主形狀：圓 / 水滴 / 雲朵 / 蘑菇】，【體型】，【特徵：觸手 / 耳朵 / 角 / 翅膀 / 尾巴】。
配色：主色【主色號】，半透明 / 不透明，【點綴色】。
五官：【眼睛數量+位置+形狀】，【嘴巴形狀】。
個性：【個性形容詞】。
風格：【畫風】，【線條】，【色調】。
背景：純白色。
這是 LINE 貼圖角色設定圖（第一張，正面，比愛心）。
```

**示範填法（粉紅水母）**：

```
請畫一隻原創的粉紅水母，沒有對應現實生物。
外觀：水滴型半透明身體，下方有 6 條柔軟觸手。
配色：主色粉紅 #FFB6C1，半透明可看穿，點綴色淺紫色。
五官：兩個圓眼睛在頭頂上方，嘴巴是一個小三角形微笑。
個性：活潑可愛、容易害羞。
風格：漸變水彩風，細線條，冷色調。
背景：純白色。
這是 LINE 貼圖角色設定圖（第一張，正面，比愛心）。
```

> **難度提醒**：粉紅水母這種「半透明 + 漸變水彩」的角色設定卡，**比扁平卡通難度高**。原因：去背時透明度容易失真、APNG 壓縮較吃資源。如果你是入門學員，第一次做建議選類型 A 或 C（輪廓清楚、無透明度）。水母風格本課 Part 2 才會大量出現，現在記住「水彩風較難壓縮，進階用」就好。

#### Demo 3：3 種角色一致性手法的優劣對比

填好設定卡 + 寫好起手式之後，「同一角色 8 張」實際上有 3 種做法。每種各有優劣，課堂上會教你**根據情境選**而不是死記。

| 手法 | 操作 | 優點 | 缺點 | 額度成本 | 適用情境 |
|------|------|------|------|---------|---------|
| **A：上傳參考圖** | 先用任何方法做出 1 張正解圖（或自己畫 / 找風格類似的圖），上傳給 AI 說「照這隻畫」 | 一致性最強（AI 直接看圖比對）| 第一張要先有，等於先要產出 1 張或自備 | 高（每張都重新讀圖）| 角色已經定型、想做完整套組 |
| **B：同對話多輪** | 開一個對話，先用起手式產第 1 張；然後在**同一個對話**繼續說「換成哭的表情」「換成生氣」 | 省額度（同對話 context 不重算）；快 | 連續到 4-5 張會「漂移」（毛色變淡 / 體型變瘦） | 最低 | 額度緊、想快速跑完 8 張的學員 |
| **C：一次產多張變化版本** | 在一個 Prompt 裡直接說「請產 4 張同角色不同表情」 | 速度最快、額度最省 | 風格容易跑掉（每張比重不同）；不能精準控制每張的情境 | 最低 | 探索期、快速看「這個角色畫起來好看嗎」 |

**本課推薦組合（與 CH1-1 預設策略一致）**：

- **第 1 張**：用**起手式 + 設定卡**直接產（這張是「校準範例」）
- **第 2-4 張**：用**手法 B（同對話多輪）**省額度，快速跑
- **第 5-8 張**：如果第 4 張開始漂移，**仍在同對話內**切到**手法 A**——把第 1 張當參考圖**上傳到同一個對話**請 AI 校準，**不要重開新對話**（重開等於 AI 忘記前 4 張，等於繞遠路）
- **手法 C（一次產多張變化版本）** 不在 8 張流程裡用，留給「想快速試風格」時用
- **唯一允許開新對話的時機**：原對話視窗被關掉且歷史救不回，或切到備援平台（ChatGPT / Bing）

> **省額度提醒**：CH1-1 招生信已要求你註冊 Gemini + ChatGPT 兩家。今天教的 3 手法，**手法 B 最省、手法 A 最耗**。Gemini 免費版每天有產圖上限，用光了就切到 ChatGPT 繼續。額度規劃失敗 = 課堂中段集體卡。

### 動手 / Hands-on（5 min）

現在輪到你。打開記事本（或紙本），花 5 分鐘填**一張你自己的角色設定卡**。不要想商業價值，不要想「會不會紅」，就想「我想做給誰看 / 我家的什麼東西可以擬人化」。

**作業規格**：

1. 寫滿 5 格（種族 / 體型 / 配色 / 個性 / 風格參考），**每格至少一句完整的話**，不可只寫「可愛」「療癒」「卡通風」這種一個詞了事
2. 從 4 類起手式裡選 1 類（寵物 / 真人 / 物品 / 幻想），把對應的填空模板填好
3. 寫完先**自己讀一次**：如果你只看這張卡和起手式，能不能在腦中想出這個角色長什麼樣？想得出來才合格

**完成後（如果還有時間）**：選一個手法（A 上傳參考圖、B 同對話多輪、C 一次產多張變化版本）試跑。如果你還沒想好用哪個，預設用**手法 B（同對話多輪）**最省額度。CH1-3 會教 8 張完整流程，這裡只是讓你先碰一下 AI 的反應。

> **如果你寫不出來**：通常是**第 4 格（個性）**和**第 5 格（風格）**最卡。卡的時候直接抄橘貓的設定當骨架，把種族換成你的角色，配色換成你想要的，先讓 AI 動起來再說。完美主義會讓你卡 30 分鐘什麼都沒產出。

### 檢核 / Verification（3 min）

填完設定卡後，用這 **5 條設定卡驗收清單** 自查（注意：這是**檢查你的設定卡寫得夠不夠具體**，跟 CH1-3 的「角色一致性檢查清單 5 條判準」是不同的兩張表）：

- [ ] **第 1 格寫得夠具體**：不是「貓」而是「橘色虎斑貓 + 兩腳站立 + 會講話」
- [ ] **第 2 格體型有量化**：不是「可愛」而是「Q 版二頭身 + 圓滾滾 + 肚子微凸」
- [ ] **第 3 格配色寫了顏色名稱或色號**：不是「暖色」而是「主色橘 #E89B5A + 副色奶油白 + 配件白色領巾」
- [ ] **第 4 格個性有「常態表情」**：不是「個性溫和」而是「溫和但偶爾無語，眼睛半瞇圓眼，嘴巴微微嘟嘴」
- [ ] **第 5 格風格有「畫風 + 線條 + 色調」三段**：不是「卡通風」而是「扁平卡通 + 粗線條 3px + 暖色調無漸層」

**5 條全勾才算合格**。少 1 條，下一堂 CH1-3 產 8 張時會在那一格上吃虧——AI 看不懂的格，AI 就自己亂發揮。

如果你只勾了 2-3 條：不用重來，先**把橘貓的設定整段複製過去當骨架**，再把你的角色名稱和配色塞進對應位置。CH1-3 我們會帶著走流程，那時候會自然補齊。

## 商業情境案例（Case）

> 本課定位是「自用」，不討論商業情境。以下三個案例是「自用導向」的具體場景，對應 4 類起手式裡各挑一個示範。學員代入身分為「你」（不具名）。

### 案例 1：橘貓上班族（類型 A 寵物擬人 · 主示範）

**情境**：你家養了一隻橘色虎斑貓，名字叫芒果。每次跟朋友 LINE 聊天，你都用文字說「我家芒果今天又翻肚討摸了」。你想做一套芒果的貼圖，送幾組給家人朋友當禮物。

**本單元要你學會**：

- 寫一張「擬人化版芒果」的角色設定卡（不是真實芒果照片，而是 Q 版二頭身的擬人版本）
- 用類型 A 起手式產出第一張「芒果比讚」的設定圖
- 知道下一堂 CH1-3 要怎麼從這張延伸出 8 張表情

**為什麼用這個示範**：寵物擬人是課程主示範，授權零問題，視覺辨識度高，新手最容易走完整流程。

### 案例 2：物品擬人 — 愛抱怨的咖啡杯（類型 C）

**情境**：你每天早上都要喝一杯咖啡才會清醒。你想做一組「我的早晨」貼圖，主角是擬人化的咖啡杯，從「沒被裝水的空杯」「早上六點才被叫醒」「下午三點被洗的怒火」一路演下去。

**本單元要你學會**：

- 寫一張「咖啡杯擬人」的角色設定卡（杯子本體保留 + 加五官四肢）
- 用類型 C 起手式產出第一張「咖啡杯舉手打招呼」的設定圖
- 知道物品擬人的關鍵是「物品本體不變 + 加最少必要的擬人元素」（眼睛+嘴巴），太多反而失去物品感

**為什麼用這個示範**：物品擬人在自用學員裡比例不低（約 15-20%），而且視覺結構簡單（背景一律純白、物體本體形狀固定），是個容易成功的入門類型。

### 案例 3：幻想生物 — 粉紅水母（類型 D · Part 2 預埋）

**情境**：你想做一組「會發光的小生物」貼圖，沒有對應現實動物，純粹自創。你想像中是一隻粉紅色半透明的水母，飄浮在純白背景上，做各種可愛動作。

**本單元要你學會**：

- 寫一張「原創幻想生物」的角色設定卡（不能用「水母」這個常見名稱當作 AI 的快捷鍵，要詳細寫外觀）
- 用類型 D 起手式產出第一張「粉紅水母比愛心」的設定圖
- 知道幻想生物的關鍵是「外觀寫到極具體」（AI 對「水母」一詞會自由發揮，必須鎖死形狀+觸手數+顏色）

**為什麼用這個示範**：水母是 Style Guide 主線 B，預告 Part 2 進階班會大量出現。本課提早曝光讓選 Part 2 的學員心理有準備，也讓學員看到「AI 不只能畫卡通」。**注意**：這類「半透明 + 漸變水彩」的角色去背 + APNG 壓縮較難，CH1-4 / CH2-3 會再強調，本單元只當作起手式示範。

## 動手練習題（Hands-on Exercise）

**題目**：當場填一張「你自己想做的角色」的角色設定卡。

不要選橘貓 / 咖啡杯 / 水母（這三個是課堂示範角色，你要練的是「自己選一個」的能力）。從以下 4 類起手式裡選 1 類，填好對應的設定卡 + 填好對應的起手式 prompt：

- 類型 A 寵物擬人：你家寵物 / 朋友家寵物（虛構名字也可）

**預期成果**：

1. **一張寫滿 5 格的角色設定卡**（紙本手寫或筆電打字皆可）
2. **一份填好的起手式 prompt**（從你選的類型對應模板填空完成）
3. （選做，時間夠才做）**用手法 B 同對話多輪**：把這份 prompt 貼進 Gemini 對話，產第一張設定圖

**完成標準**（self-check）：

- [ ] 角色設定卡 5 格全寫完，每格至少一句完整的話（不是單詞）

- [ ] 起手式 prompt 全部空格都填掉了，沒有遺留 `【】` 中括號
- [ ] 把 prompt 從頭讀一次，沒有矛盾（例如：寫了「Q 版二頭身」又寫「寫實正常比例」）

5 分鐘內如果無法全部勾完，先求把第 1、3、5 格填到具體；第 2、4 格課後再補。**寧可粗胚 5 格全有，不要精修 2 格其他空白**。

## 常見錯誤 3 條（Common Pitfalls）

### 錯誤 1：角色設定卡只寫「可愛 / 療癒」這種抽象詞，AI 每次畫出來都不同

**錯誤現象**：學員在第 4 格「個性」寫「可愛」、第 5 格「風格參考」寫「療癒風」。產出第一張看起來 OK，但第二張表情變了之後，整個角色長相完全不同——上一張是 Q 版圓臉，這張變成寫實長臉；上一張是粗線條，這張變成水彩漸層。

**原因**：「可愛」和「療癒」是**對結果的形容詞**，不是**對畫面的描述**。AI 看到「可愛」會自由發揮（每個 AI 認知的可愛都不同），看到「Q 版二頭身 + 圓眼睛 + 粗線條外輪廓 3px」才會固定下來。「療癒風」更糟，這個詞對 AI 完全沒意義（你問 10 個人「療癒風」是什麼樣，會得到 10 種答案）。

**解法**：把抽象詞翻譯成具體畫面：

- 「可愛」→「Q 版二頭身、圓滾滾、大眼睛、嘴角微微上揚」
- 「療癒」→「暖色調（橘 + 米白 + 淺褐），無強對比，柔和線條」
- 「酷酷的」→「冷色調（藍 + 灰），眼睛半瞇細線，嘴角下垂」

**自查方法**：寫完設定卡，蓋住「種族」那一格，只看其他 4 格。如果你的家人朋友讀完這 4 格，能想像出**同一隻**生物（不是各自亂想），才算合格。

---

### 錯誤 2：4 張之內就直接切換對話 / 重開頁籤，每張都從零講設定卡

**錯誤現象**：學員產完第 1 張之後，覺得這張不錯，**關掉頁籤、開新對話**，把整份設定卡重貼一次想產第 2 張「哭的表情」。產出來的第 2 張和第 1 張差了 30%——配色變了、體型瘦了、配件不見了。學員以為是 AI 不穩定，繼續關頁籤、繼續開新對話、每張都重貼。8 張產完得到 8 隻不同的貓。

**原因**：AI 在「同一個對話」內有 **context 記憶**，前面產過的圖會被當作參考留在對話 context 裡。你開新對話 = 把這份記憶清空 = AI 從零重畫。每次重貼設定卡，AI 都會在「設定卡的描述空間」裡找一個解，每次找到的解略有不同，所以 8 張會漂移到 8 個方向。

更糟的是，每開一個新對話 = 燒一次完整額度。手法 B（同對話多輪）的省額度優勢被完全抵消。

**解法**：

- **預設用手法 B**：產完第 1 張之後，**不要關對話**，直接在同一個對話框輸入「同一隻角色，換成哭著流眼淚的表情」「同一隻角色，換成生氣鼓臉頰」這種短指令
- 短指令直接複用前面 context，不重講設定卡
- 連續到第 4-5 張感覺開始漂移（毛色變淡 / 體型變瘦）時，**才**切到手法 A（上傳第 1 張當參考圖重新校準）
- 不要為了「乾淨整潔」開新對話，AI 不在乎對話亂不亂，AI 在乎你給它的 context 是否連貫

---

### 錯誤 3：示範角色直接抄「皮卡丘 / 哆啦 A 夢 / 史努比」，覺得反正 AI 畫得出來就 OK

**錯誤現象**：學員為了快，第 1 格「種族」直接寫「皮卡丘風格的小怪獸」「類似史努比的小狗」。AI 確實畫得出來——而且畫得很像。學員覺得超級成功，產完 8 張準備上傳。**結果 LINE 審核直接退件，理由：含他人 IP（智慧財產權）。**

更嚴重的情況：學員不只被退件，還可能被 LINE 警告「禁止再次上傳此類內容」，或者帳號被處分。

**原因**：

- 皮卡丘 = 任天堂 / The Pokémon Company 的著作權與商標
- 哆啦 A 夢 = 小學館 / 藤子・F・不二雄 PRO 的著作權與商標
- 史努比 = Peanuts Worldwide 的著作權與商標

「我只是參考風格、又不是複製貼上」不成立。LINE 的 IP 審核非常嚴，連「明顯類似」都會退件。AI 並沒有義務告訴你這隻畫出來像誰，它只是照你寫的「皮卡丘風格」去畫，責任在你。

**解法**：

- **設定卡的「種族」欄位禁止寫任何已存在角色的名字**：禁皮卡丘 / 哆啦 A 夢 / 史努比 / 米老鼠 / Hello Kitty / 角落生物 / 小熊維尼 / 任何動畫遊戲漫畫角色
- **設定卡的「風格參考」欄位也禁止寫公司名 / 動畫名**：禁「吉卜力風」「迪士尼風」「皮克斯風」「三麗鷗風」
- 想做類似的可愛動物：寫**生物本身**（黃色老鼠 / 藍色機器貓 / 白色米格魯狗），加上**自己定義的特徵**（不是電擊頰、不是任意門、不是紅項圈），讓 AI 從零畫一隻**你的**角色
- 自查方法：產出第一張時問自己「這隻像不像任何已存在角色？」，像就立刻調整設定卡，不要硬上送審

## 檢核題 2 條（Quiz）

### Q1（概念驗證）

下列哪一條，是把「療癒風 Q 版可愛貓咪」翻譯成 AI 能穩定守住的角色設定卡描述？

- [ ] A：「療癒風的可愛貓咪，Q 版」
- [ ] B：「Q 版貓咪，療癒色調，可愛配件」
- [ ] C：「橘色虎斑貓，Q 版二頭身圓滾滾，主色橘 #E89B5A + 副色奶油白，扁平卡通粗線條（外輪廓 3px），暖色調無漸層」 ←（正確答案）
- [ ] D：「一隻很可愛的貓，療癒風格，給人家溫暖的感覺」

**解析**：A、B、D 全部用「療癒 / 可愛 / 溫暖」這種對結果的形容詞，AI 看到會自由發揮。C 把所有形容詞都翻譯成可量化的具體描述（顏色名稱 + 色號、體型量化、線條粗細、是否漸層），AI 看了能穩定產出。這是本單元最核心的能力。

### Q2（應用驗證）

你今天已經用手法 B（同對話多輪）在 Gemini 同一個對話內產了 4 張橘貓的不同表情：笑、哭、生氣、得意。第 1-3 張角色都很一致；但第 4 張「得意」感覺貓的毛色變淡了一點，肚子的奶油白範圍也縮小了一點。你還有 4 張要產（求拜託、無語、晚安、嚇到）。請問下一步應該怎麼做？

**預期答案要點（學員講出 2 點即合格，3 點優秀）**：

1. **不要直接繼續在同對話產第 5 張**——AI 漂移已經出現，再產下去會越來越偏，4 張全部報廢
2. **切到手法 A（上傳參考圖）—— 仍在同對話內**：把第 1 張（最一致的那張）下載下來，**在同一個 Gemini 對話視窗上傳**這張圖，附 prompt「請照這張畫的角色，畫一個求拜託的表情」，這樣 AI 重新從圖校準，不再依賴漸漸漂移的 context。**不要重開新對話**——重開等於 AI 忘記前 4 張的 context，等於繞遠路
