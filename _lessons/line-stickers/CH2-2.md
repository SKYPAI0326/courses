## 教學流程（Teaching Flow · skill-operation 5 段式）

### 破題 / Hook（5 分鐘）

CH2-1 你看完了：APNG ≤300KB、**5–20 frames（最少 5 幀是 LINE 硬規格）**、不超過 4 秒、會動的點要少不要多。原理你聽得懂，看到範例 APNG 也覺得「嗯，這我做得出來」。

但今天 90 分鐘你會發現：**動畫做不出來，不是因為 APNG 規格難，是因為角色一動就跑掉**。

橘貓上班族 Part 1 你已經會產 8 張一致的表情了。今天你想讓他「眨眼」——直覺會想：「不就 2 張嗎？睜眼 + 閉眼」。**錯**。LINE 動態貼圖最少 5 幀（CH2-1 已鎖規格），眨眼也不例外——你需要 **5 幀**：睜眼 → 半閉 → 全閉 → 半開 → 睜眼。少一幀都不能上架。

聽起來變難了？其實不會。技巧是這樣：你還是先產**`f01`（睜眼）和 `f03`（全閉）**這 2 張**關鍵幀**，然後用「**補張 prompt**」請 AI 從這兩張關鍵幀**延伸出 3 張中間幀**（f02 半閉、f04 半開、f05 結尾睜眼），補滿到 5 張。今天的核心技巧就是這個——**用 2 張關鍵幀生 5 張完整序列**。

你打開 Gemini，在 CH1-3 同一個對話接續輸入「同一隻橘貓，這次閉眼」。AI 給你一張「閉眼版橘貓」——但耳朵變尖了一點、領巾的位置歪了 5 度、配色變淡了一點。這張單獨看沒問題，但跟「睜眼版」放在一起當動畫播放時，你會看到**整個角色在抖**——不是眼睛在動，是「整隻貓在跳格」。

這就是動態貼圖的核心難題：**一致性在「同一張」可以靠 prompt 守住，在「相鄰兩幀」要靠 prompt + 對話順序 + 補張一起守**。Part 1 的 8 張表情是「並列」（每張獨立看），動態貼圖的關鍵幀是「序列」（兩張要無縫接續），標準完全不同。

這 90 分鐘的**當堂硬目標：完成 1 組眨眼 5 幀並過 5 條判準**（這是合規 LINE 最少 5 幀的最小可上架單位）。教學主軸：

1. **眨眼（5 frame · 當堂硬目標）**：先產 2 張關鍵幀（睜眼 / 全閉）→ 用補張 prompt 延伸出 3 張中間幀（半閉 / 半開）→ 完整 5 幀過 5 條判準
2. **揮手（4 frame · 進階挑戰）**：示範會帶到 prompt 模板，動手段時間有餘力者可做；無暇做 = 課後練習
3. **跳起來（5 frame · 進階挑戰 / 課後練習）**：示範會帶到「形變允許範圍」概念，但**當堂不要求動手**

跑完當堂硬目標，你會帶走 **1 組眨眼 5 幀 PNG 序列**（命名 `{角色名}_anim_blink_f01.png` 到 `_anim_blink_f05.png`）。下一單元 CH2-3 會把這 5 張 PNG 組成 APNG（最低門檻能上架）。今天**只做產關鍵幀，不做 APNG 組裝**——別跳級。

> 你**不是來學動畫原理的**。你是來學「怎麼跟 AI 講『畫第 N 幀』，讓相鄰兩幀像同一隻貓在連續動」。

### 概念 / Concepts（10 分鐘）

這 6 個概念會反覆出現在示範與動手環節，先記下：

- **關鍵幀（key frame）** — 動畫裡的「明顯姿勢張」。眨眼的關鍵幀只有 2 個（睜眼 / 全閉），但 LINE 動態貼圖規格最少 5 幀，所以**還要補 3 張中間幀**（半閉、全閉、半開——全閉那張就是第 2 個關鍵幀）。本課眨眼組教你「**先產 2 張關鍵幀，再用補張 prompt 延伸出中間幀補到 5 張**」這個技巧。揮手 4 frame 與跳起來 5 frame 的關鍵幀數示範會帶到，但當堂硬目標只要求眨眼 5 幀。
- **中間幀（in-between frame · 補張產出）** — 兩張關鍵幀之間的過渡張。眨眼的「半閉」「半開」就是中間幀——它們不是新的關鍵姿勢，而是讓「睜眼 → 全閉」的過程看起來平滑的中段樣本。本課中間幀也用 AI 產（不靠 CH2-3 的工具），方法是給 AI 看「前一個關鍵幀 + 後一個關鍵幀」，請它畫「介於兩張之間的某個比例」（例：眼皮蓋下 50%）。
- **基準幀（baseline frame · 第 1 張的特殊地位）** — 每組關鍵幀的第 1 張是「這個動作組」的角色基準，後面所有幀都以第 1 張為對齊標準。第 1 張畫不好，整組會抖。延續 CH1-3 的「校準張」概念，但這裡是「動作組的基準」不是「整套表情組的基準」。
- **同對話多輪（context 持續）** — Part 1 教過：在 Gemini 同一對話視窗內連續發 prompt，AI 會看到前面所有圖當作 context。**今天當堂硬目標的眨眼 5 張全部在同一個對話內**（揮手 / 跳起來進階做的話也接在同一個對話）。重開新對話 = 角色記憶重置。
- **5 條判準（同隻 / 同風格 / 同比例 / 同視角 / 同色域）** — CH1-3 的角色一致性檢查清單，**動態幀間更容易跑掉**。Part 1 是 8 張並列、肉眼可一一檢查；動態幀是 2-5 張要連續播放，**任一幀跑掉，動畫播起來就會抖**。今天每幀產出後立刻 5 條自查，不可累積。
- **形變範圍（deformation budget）** — 跳起來這類「身體會擠壓 / 拉長」的動作，本來就會違反「同比例」一條（蹲下蓄力時頭身比變窄、起跳時拉長）。本課把這類「動作必要的形變」標為**允許範圍**，跟「跑掉」要分清楚——蹲下蓄力時身體變寬扁是正常，但**配色 / 視角 / 同隻**這 3 條無論姿勢怎麼變都不能跑。
- **補張（patch shot）** — Part 1 教過的概念延續：發現某一幀跑掉，在原對話內針對該幀重畫。動態幀的補張比 Part 1 更常用，因為「兩幀相鄰看會抖」的肉眼判準比 Part 1「8 張並列看哪張像不像」嚴格 3-5 倍。**幾乎每組都至少要補 1-2 張**。

### 操作示範 / Demo（30 分鐘）

我會在 Gemini 同一個對話裡，**從頭到尾**示範產出橘貓的眨眼 5 幀（當堂硬目標）；揮手 4 frame 與跳起來 5 frame 我也會帶到 prompt 模板與「形變允許範圍」概念，但只示範**關鍵卡關**那一兩張，不從頭跑完 14 張全套。你不要動手，先看完整流程。30 分鐘看似很長，但要看完才不會卡。

#### 為什麼示範要佔 30 分鐘（不是 15 分鐘）

CH1-3 你已經會「8 張表情」的同對話多輪。今天看似只是「換成連續動作」——但**動作幀比表情幀的容錯率低**。表情組的 8 張是「並列」，每張獨立可看；眨眼 5 張要「連續播」，肉眼會立刻抓到「f01 和 f02 之間配色變了 0.5%」這種 Part 1 完全不會抓的細節。30 分鐘示範要演**眨眼 5 幀的完整流程（產 2 關鍵幀 + 補 3 中間幀 + 卡關補張）**，加上揮手 / 跳起來的「重點卡關示範」（不全跑、只演每組最容易跑的那一張），這些卡關示範是學員等下動手最大救命線。

> 如果這 30 分鐘節省成 15 分鐘，後面 30 分鐘動手會多出至少 20 分鐘卡關時間，整體反而拖延。本單元 90 分鐘極緊（30 示範 + 30 動手 + 5 破題 + 10 概念 + 15 檢核），示範時間絕對不能省。

#### 示範用對話起點（重要）

**今天的 Gemini 對話不從零開始**——我們延續 CH1-3 那個「橘貓 8 張表情」的對話。原因：那個對話裡 AI 已經看過橘貓 8 張不同表情，**對「這隻橘貓」的 context 記憶已經很穩**。今天直接在那個對話接續產動作關鍵幀，比重開新對話省至少 50% 額度，且角色一致性更穩。

如果你 CH1-3 用的是 ChatGPT 不是 Gemini，或對話視窗已經關掉找不回來，**唯一允許的處理**：開新 Gemini 對話，**先上傳 CH1-3 已產的 8 張橘貓表情圖**當參考，然後跑下面流程。**絕對不要從零文字描述開始**——那等於把 Part 1 累積的 context 全部丟掉。

#### 示範組 A：眨眼（5 frame · 當堂硬目標 · 合規 LINE 最少 5 幀）

當堂硬目標，從這裡起步。技巧三步走：(1) 先產 2 張關鍵幀（睜眼 + 全閉）→ (2) 用「補張 prompt」延伸 3 張中間幀（半閉、半開、結尾睜眼）→ (3) 5 張連起來檢查。

**整組 5 幀構成**：
- f01（睜眼 · 起點關鍵幀）
- f02（半閉 · 中間幀）
- f03（全閉 · 終點關鍵幀）
- f04（半開 · 中間幀）
- f05（睜眼 · 終點 = 起點 · 循環回到 f01）

**第 1 步：產 2 張關鍵幀**

**`f01`（基準幀 · 睜眼）**：

```
延續上面對話的橘貓上班族（同隻角色、同風格、同比例、同視角、同色域）。
這次請畫眨眼動畫的 f01：橘貓站立正面，眼睛完整睜開（圓睜的圓眼），
表情中性，嘴巴微閉。背景純白。

備註：這是「眨眼動畫」的第 1 幀（睜眼 · 起點關鍵幀），
等下會接著產 f03（全閉 · 終點關鍵幀），最後在中間補出 f02 / f04（半閉 / 半開），
湊成 LINE 規格最少 5 幀的眨眼序列。
請保持構圖完全一致，背景純白，畫面比例正方形。
```

**講師示範重點**：產出後**立刻 5 條判準自查**——同隻（橘色虎斑、白領巾、圓臉）、同風格（扁平卡通、粗線條）、同比例（二頭身、頭身比 2:1）、同視角（正面）、同色域（暖橘調）。5 條過了再繼續 f03，沒過就**補張**重產 f01（不繼續往下）。

**`f03`（終點關鍵幀 · 全閉）**：

```
保持 f01 的所有設定（同隻、同風格、同比例、同視角、同色域、同構圖、同背景）。
請畫眨眼動畫的 f03：眼睛「完全閉合」（眼皮完全蓋下，眼睛變成一條向上彎的弧線，
像睡著或眨眼最深的那一瞬間）。

關鍵限制（極重要）：
- 角色位置不能移動（跟 f01 完全一樣的構圖、頭部位置、領巾位置）
- 嘴巴、手腳、領巾、毛色、線條粗細、背景顏色，全部跟 f01 完全一樣
- 只有「眼睛狀態」改變，其他「一根毛都不能動」

這是眨眼動畫的「全閉」終點關鍵幀。等下會在 f01 和 f03 之間補一張 f02（半閉），
在 f03 和 f05 之間補一張 f04（半開）。
```

**講師示範重點**：f03 產出後，**把 f01 和 f03 並排放在螢幕上**，逐條 5 條判準對比。動畫的判準比 Part 1 嚴：
- 第 1 條（同隻）：耳朵形狀、領巾打結處要完全一樣
- 第 2 條（同風格）：線條粗細不能變（變細 0.3px 連續播會看到「閃爍」）
- 第 3 條（同比例）：頭身比不能變（眨眼動作不該改變身體比例）
- 第 4 條（同視角）：頭部不能轉（很多 AI 會「順手」把頭微側，連續播就抖）
- 第 5 條（同色域）：橘色飽和度要完全一致（差 5% 連續播會看到「閃光」）

**第 2 步：用「補張 prompt」延伸 3 張中間幀**

**這是本單元的核心技巧**——學員平常只想到「再產一張新圖」，不會想到「請 AI 畫介於兩張之間的某個比例」。

**`f02`（中間幀 · 半閉 · 介於 f01 和 f03 之間）**：

```
我已經產出 f01（眼睛完全睜開）和 f03（眼睛完全閉合）兩張關鍵幀。
請畫 f02：眼睛「半閉」——眼皮蓋下大約 50%，
眼睛還看得到下半部（黑色瞳孔露出下半），但上半被眼皮遮住。

這是 f01（睜開）和 f03（全閉）之間的過渡幀，
眼皮位置應該大約是 f01 和 f03 的中間值。

關鍵限制：
- 除了「眼皮位置」介於 f01 和 f03 之間之外，其他全部跟 f01 完全一樣
- 構圖、頭部位置、領巾、嘴巴、手腳、毛色、線條、背景全部不變
- 不要改成「半睜」這種模糊的描述，請畫「眼皮從上往下蓋 50%」的明確中間態
```

**`f04`（中間幀 · 半開 · 介於 f03 和 f05 之間）**：

```
保持 f03（全閉）的所有設定。
請畫 f04：眼睛「半開」——眼皮從全閉狀態往上掀開大約 50%，
跟 f02 的「半閉」是鏡像關係（f02 是從睜眼蓋下，f04 是從閉眼掀開）。

關鍵限制：
- 眼皮位置可以跟 f02 完全一樣（半閉 = 半開，視覺上一致）
- 其他全部跟 f01 / f03 完全一樣（構圖、領巾、嘴巴、手腳、毛色、線條、背景）
```

**`f05`（終點 = 起點 · 睜眼 · 循環回到 f01）**：

```
請畫 f05：眼睛「完全睜開」，跟 f01 一模一樣。
這是眨眼動畫的最後一幀，等下播放時會循環回到 f01。
構圖、領巾、嘴巴、手腳、毛色、線條、背景全部跟 f01 完全一樣。
```

> 講師可選：時間夠的話直接「複製 f01 當 f05」（PNG 在電腦上複製檔案改名），不用再耗一次額度產。但如果學員是循 prompt 練習，產一張當作「結尾要回到起點」的概念示範也合理。

**第 3 步：5 張並排對比 5 條判準**

把 f01 → f02 → f03 → f04 → f05 橫排放螢幕，**逐條** 5 條判準對比；眨眼動畫的「眼皮位置」應該呈現 0% → 50% → 100% → 50% → 0% 的平滑過渡，其他元素（耳朵、領巾、線條、配色、視角）5 張完全一致。

**示範卡關（必演）**：f02（半閉）產出後，故意挑出「眼皮蓋下 50% 但耳朵微尖了 1px」這種微跑，講解「Part 1 你會放行，Part 2 不能」。立刻補張：

```
上一張 f02 的耳朵形狀跟 f01 不一樣（耳朵尖了一點），
請參考 f01 的耳朵形狀（圓潤、無尖角），
重畫一次 f02 的「半閉」版本（眼皮蓋下 50%）。其他全部不變。
```

**示範卡關 2（必演）**：學員容易遇到「f02 和 f04 看起來不像同一個閉眼程度」——明明都該是「眼皮 50%」但 f04 比 f02 蓋得更深。補張：

```
上一張 f04 的眼皮位置跟 f02 不一致（f04 蓋得比 f02 深）。
f02 和 f04 應該是「鏡像關係」，眼皮蓋下程度要一模一樣，
請參考 f02 的眼皮位置，重畫 f04。其他全部不變。
```

#### 示範組 B：揮手（4 frame · 進階挑戰 / 課後練習）

> **本組為進階挑戰**：當堂硬目標只要求眨眼 5 幀通過。揮手 4 frame 示範會帶到 prompt 模板與最容易跑的卡關張，學員動手段時間有餘力者可做；無暇做就把這份模板帶回家當作課後練習。

#### 揮手動畫的 4 幀解構

不要直接想「揮手是什麼樣子」。揮手在 LINE 動態貼圖裡的標準解法是 **4 幀循環**：手畫一個「左下 → 左上 → 右上 → 右下」的方框，連續播會看起來像「左右揮手」。命名規格：`{角色名}_anim_wave_f01.png` 到 `_anim_wave_f04.png`。

**`f01`（揮手起點 · 手在右下）**：

```
延續上面對話的橘貓上班族（同隻、同風格、同比例、同視角、同色域）。
這次請畫揮手動畫的 f01：橘貓站立正面，眼睛圓睜（同眨眼 f01），
右手抬起到「右下方位置」（手肘彎曲、手掌張開朝鏡頭），
左手自然垂在身體左側不動。

關鍵限制：
- 身體、頭、領巾、表情、左手，全部跟眨眼 f01 完全一樣
- 只有「右手位置」抬到右下方
- 構圖維持正方形、背景純白、角色置中

這是 4 frame 揮手動畫的第 1 幀（手起點）。
```

**`f02`（手到右上）**：

```
保持揮手 f01 所有設定。
請畫揮手 f02：右手從右下抬到「右上方位置」（手臂上舉約 45 度、手掌依然張開朝鏡頭）。
身體、頭、領巾、表情、左手位置，全部跟 f01 完全一樣。
只有「右手位置」從右下變到右上。
```

**`f03`（手到左上）**：

```
保持上一幀所有設定。
請畫揮手 f03：右手從右上「越過頭頂」到「左上方位置」
（手臂依然上舉、但越過身體中線到角色的左上方、手掌朝鏡頭）。
身體、頭、領巾、表情、左手位置，全部跟前幾幀完全一樣。
```

**`f04`（手到左下）**：

```
保持上一幀所有設定。
請畫揮手 f04：右手從左上落到「左下方位置」
（手臂下垂約 45 度、依然在身體左側、手掌朝鏡頭）。
身體、頭、領巾、表情、左手位置，全部跟前幾幀完全一樣。

這是揮手動畫的最後一幀，等下播放時會循環回到 f01。
```

**講師示範重點**：4 張產完後，把 4 張橫排放在螢幕上：
- **手的軌跡**應該是個「方框」（右下 → 右上 → 左上 → 左下），不是隨機亂飄
- **身體 / 頭 / 表情 / 左手** 4 張完全一樣（5 條判準的 1、2、3、4 條每張都過）
- **第 5 條（同色域）** 在揮手動作裡最容易跑掉，AI 有時會在手抬到上方時「順手」把整體配色提亮 3%，連續播就會看到「閃光」

**示範卡關（必演 · 揮手組唯一示範重點）**：f03（手到左上）AI 容易把整個身體跟著向左偏移 5px——示範「手要動，身體不能跟著飄」的補張：

```
上一張揮手 f03 的身體位置跟 f02 不一樣（整個角色向左偏移了一點）。
請參考 f02 的身體位置（角色置中構圖），保持身體在正中央，
只有右手位置從右上換到左上，重畫 f03。
```

#### 示範組 C：跳起來（5 frame · 進階挑戰 / 課後練習）

> **本組為進階挑戰**：當堂硬目標只要求眨眼 5 幀通過。跳起來 5 frame 示範重點放在「形變允許範圍」概念與最容易跑的蹲下幀；學員當堂**不要求動手做這組**，模板帶回家當作課後練習。

#### 跳起來動畫的 5 幀解構

跳起來比眨眼、揮手難，因為**全身會形變**（蹲下時身體擠壓變扁、起跳時拉長）。形變範圍要在「動作必要」內，不能跑到「角色長相變了」。命名規格：`{角色名}_anim_jump_f01.png` 到 `_anim_jump_f05.png`。

**`f01`（站立 · 起點）**：

```
延續上面對話的橘貓上班族（同隻、同風格、同比例、同視角、同色域）。
這次請畫跳起來動畫的 f01：橘貓站立正面，全身放鬆，
雙手自然垂在身體兩側，雙腳併攏站立。表情中性。
構圖：角色置中，畫面下方留 30% 空白（為跳起來時上移留空間）。

這是 5 frame 跳起來動畫的第 1 幀（站立起點）。
```

**`f02`（蹲下蓄力）**：

```
保持上一幀所有設定。
請畫跳起來 f02：橘貓「蹲下蓄力」——
身體高度壓縮到原本的 70%（頭身比從 2:1 變成 1.5:1，身體變得更扁更寬），
雙腳微彎、雙手向後拉準備擺動。
表情：眼睛微皺（蓄力專注貌）。
位置：角色仍置中，但因為高度壓縮所以視覺上往畫面下方靠。

關鍵限制：
- 配色、線條粗細、視角、同一隻角色（耳朵、領巾、配色）全部跟 f01 完全一樣
- 唯一允許的形變是「身體高度壓縮 + 雙腳彎曲」（這是跳起來動作的必要形變）
- 不要把橘貓改成不同物種、不要改視角、不要改背景
```

**`f03`（起跳瞬間）**：

```
保持上一幀所有設定。
請畫跳起來 f03：橘貓「起跳瞬間」——
身體開始往上拉長（高度恢復到原本的 110%，身體比 f01 稍長），
雙腳開始離地約 5px、雙手向上揮、表情：眼睛圓睜（驚喜貌）。
位置：角色比 f01 略往上方移動（畫面位置高 10%）。

形變允許範圍：身體拉長到 110%（不要超過 120%，超過會變成「不同角色」）。
配色、線條、視角、同隻判準全部維持。
```

**`f04`（最高點）**:

```
保持上一幀所有設定。
請畫跳起來 f04：橘貓「跳到最高點」——
身體在空中（雙腳離地至少 30px），姿勢舒展（手腳張開但不誇張），
表情：眼睛圓睜、嘴巴微 O（開心貌）。
位置：角色明顯在畫面上方（比 f03 又高一些）。
身體高度恢復到 f01 的 100% 比例（不再拉長）。

關鍵限制：配色、線條、視角、同隻判準全部維持。
不要把橘貓畫成「飛起來的不同姿勢角色」（保持是同一隻橘貓在跳）。
```

**`f05`（落地）**:

```
保持上一幀所有設定。
請畫跳起來 f05：橘貓「落地」——
身體回到地面（雙腳併攏站立、姿勢跟 f01 幾乎一樣），
表情：眼睛微瞇（剛落地專注貌）、嘴巴微閉。
位置：角色置中（跟 f01 同位置）。
身體高度恢復到 f01 的 100% 比例。

這是 5 frame 跳起來動畫的最後一幀，等下播放時會循環回到 f01。
配色、線條、視角、同隻判準全部維持。
```

**講師示範重點**：5 張產完後，把 5 張橫排放在螢幕上，**重點看「形變允許範圍」是不是有破**：
- f02（蹲下）身體變扁是 OK 的，但**橘色變淡**就不 OK——形變只允許「形狀」變，**配色 / 視角 / 同隻不能形變**
- f04（最高點）AI 最容易「順手」把橘貓畫成不同姿勢的角色（例如多了翅膀、手腳變細像舞者）——這是「同隻」判準破了
- f05（落地）AI 容易回不到 f01 的姿勢——示範「最後一幀要明確跟第 1 幀對齊，不然循環會抖」

**示範卡關（必演 · 跳起來組唯一示範重點）**：f02（蹲下蓄力）橘貓比例變得比預期窄太多、頭看起來像不同的貓——示範「形變允許範圍」的補張：

```
上一張跳起來 f02 的橘貓比例跟 f01 差太多（頭看起來像不同的貓），
請保持「身體高度壓縮 70% + 雙腳彎曲」的形變，
但「頭部形狀、五官位置、耳朵、領巾、配色」要跟 f01 完全一樣，
重畫 f02。
```

#### 示範圖包對照（投影片同步呈現）

我手上會放兩組對照圖：

- **A 組「3 動作正解」**：橘貓眨眼 5 frame（含 2 關鍵幀 + 3 中間幀）+ 揮手 4 frame + 跳起來 5 frame，全部 5 條判準都過、相鄰幀無抖動

兩組對照看完，學員會明白「動畫的『跑掉』比表情組更隱形——一張單看 OK，兩張連續播才會發現」。

### 動手 / Hands-on（30 分鐘）

換你做。延續 CH1-3 那個你自己角色的對話視窗（最好是 Gemini，CH1-3 用 ChatGPT 的學員見上面備援處理）。

**當堂硬目標**：完成你自己角色的眨眼 5 幀（f01 睜眼 → f02 半閉 → f03 全閉 → f04 半開 → f05 睜眼），5 張全過 5 條判準。揮手 4 frame 與跳起來 5 frame 是進階挑戰：眨眼做完且時間有餘力者可挑揮手做；跳起來原則上**不要求當堂動手**，模板帶回家當課後練習。

#### 動手任務（眨眼為主，揮手 / 跳起來為加分）

**1. 眨眼 5 frame（25 分鐘 · 當堂硬目標 · 必做）**

在你的對話視窗接續，依序產：

- **f01（睜眼 · 起點關鍵幀）**：產出後**立刻 5 條判準自查**，5 條過了再繼續
- **f03（全閉 · 終點關鍵幀）**：跟 f01 並排自查，5 條都要過
- **f02（半閉 · 中間幀）**：用「補張 prompt」請 AI 畫介於 f01 / f03 之間的眼皮 50% 中間態
- **f04（半開 · 中間幀）**：請 AI 畫跟 f02 鏡像關係的眼皮 50% 中間態
- **f05（睜眼結尾 · = f01）**：可直接複製 f01 改名為 f05，或請 AI 再畫一張等同 f01 的睜眼

5 張產完後**全部橫排自查**——眼皮位置應呈 0% → 50% → 100% → 50% → 0% 平滑過渡，其他元素 5 張完全一致。任一條跑掉**立刻補張**，不要繼續往下做。

**2. 揮手 4 frame（5 分鐘 · 進階挑戰 · 眨眼做完且時間夠才做）**

在同一個對話接續，產 f01-f04（手在右下 → 右上 → 左上 → 左下）。

- 4 張產完後**全部橫排自查**——身體 / 頭 / 表情 / 左手在 4 張要完全一致
- 沒做完不影響當堂通過，模板帶回家補做

**3. 跳起來 5 frame（課後練習 · 當堂不要求動手）**

當堂時間若還有，可挑「示範卡關 1 張」做（例如只產 f02 蹲下蓄力，體驗形變允許範圍判斷），不必跑全 5 張。完整 5 張請回家做。

#### 操作節奏（建議分配）

- 0–25 分鐘：眨眼 5 張（含 2 關鍵幀產出 + 3 中間幀補張 + 自查 + 救張）
- 25–30 分鐘：揮手 4 張（時間夠且眨眼已過再做），或停下來檢查眨眼 5 張、補弱項

#### 為什麼當堂只做眨眼（揮手 / 跳起來不必當堂硬幹）

很多學員會想：「眨眼太簡單了，我直接挑戰跳起來吧」。**不要**。眨眼 5 frame 不是「最簡單」——它是 LINE 動態貼圖**最小合規上架單位**，且包含本單元的兩個核心技巧：(1) 產 2 張關鍵幀、(2) 用補張 prompt 延伸出中間幀。把這兩個技巧練熟，揮手與跳起來都是「同樣技巧 + 不同動作」，回家就做得出來。

如果你眨眼還沒過就跳到揮手 4 frame：
- 你會把「身體微偏」當成正常（在 Part 1 表情組這算正常）
- 你會把「線條粗細微變」當成正常
- 你會把「色域微偏」當成正常

**結果是 4 張都跑、組成 APNG 播起來會看到「整隻貓在抖」**，回頭重做。

從眨眼 5 幀起步的理由：眨眼**只動一個地方**（眼睛），其他全部不能動。這個極端條件會強迫你看到「Part 2 的判準比 Part 1 嚴 3 倍」。先在眨眼建立判準感與「補張延伸中間幀」的肌肉記憶，再做揮手、跳起來，你會自動帶入這個嚴格度。

#### 講師走場（你做的時候我會做的事）

我會繞場一對一看每個學員的螢幕。看 4 件事：

1. **你有沒有重開新對話**（最常見錯誤）—— 如果有，我會立刻喊停，讓你回到上一個對話續寫；如果原對話已經關掉找不回來，請我給你「示範圖包備援 SOP」
2. **你是不是從眨眼 5 幀開始做**—— 如果你眨眼還沒做完就跳到揮手或跳起來，我會請你回到眨眼。連「2 關鍵幀 + 3 中間幀」的判準感都還沒建立，做其他組會是災難
3. **你的 prompt 有沒有寫「同隻、同風格、同比例、同視角、同色域」這 5 條限制句**—— 沒有的話我會把模板貼到你螢幕旁
4. **你有沒有 5 條判準逐張自查**—— 沒有的話我會把檢查清單貼到你螢幕旁邊提醒

#### 商業情境（3 個示範代入點 · 全用橘貓）

雖然本課自用導向，動手過程我會帶 3 組情境讓你思考「為什麼要這 3 種動作」，方便你之後選自己的題：

**情境 1：橘貓眨眼（眨眼 5 frame · 當堂硬目標 · 主示範）**

你想做一張「橘貓眨眼」的動態貼圖，傳給朋友當作「我在聽你說話」「我懂你」的回應。對話情境：朋友傳訊息訴苦，你不知道怎麼回，傳一張「橘貓緩慢眨眼」的動態貼圖，比文字「嗯」「我懂」有溫度。

**為什麼眨眼**：LINE 動態貼圖最高使用率前 5 名永遠有「眨眼」一張。動作小、檔案小、含義豐富（同理 / 在聽 / 害羞 / 困惑都能用），**新手做動畫的第 1 個必做題目**。雖然關鍵幀只有 2 個，但合規 LINE 規格需要 5 幀，所以是練習「2 關鍵幀 + 3 中間幀補張」最佳起點。

**情境 2：橘貓揮手（揮手 4 frame · 進階挑戰）**

你想做一張「橘貓揮手 hi」的動態貼圖，當作 LINE 對話的開場語。對話情境：早上跟朋友的 LINE 群打招呼、跟同事 LINE 群「hi 大家早安」、新群組第一次發言時的破冰。

**為什麼揮手**：揮手是 LINE 對話開場高頻動作前 3 名（搭配「早安」「大家好」「我來了」這類文字）。4 frame 循環的揮手檔案小、循環自然、含義單一不歧義。

**情境 3：橘貓跳起來（跳起來 5 frame · 進階 / 課後練習）**

你想做一張「橘貓開心跳起來」的動態貼圖，當作 LINE 對話的「太好了」「我很開心」回應。對話情境：朋友通知好消息（生小孩、升職、加薪、考上）、自己有好事想分享（拿到票、抽到獎）、慶祝週末來了。

**為什麼跳起來**：跳起來是「正向情緒高強度」的代表動作，3 種動作裡視覺最豐富（身體會形變）、檔案最大（5 frame）、APNG 壓縮挑戰最高。這個動作做完，你會有「動態貼圖整套都會做」的自信。

選一個你最有共鳴的動作當主軸練；3 個都做完當然最好。

#### 省額度反覆強調（critical）

- ✅ **同對話多輪 + 眨眼 5 張一條龍**：眨眼 5 張（2 關鍵幀 + 3 中間幀）全在同一個對話內，平均每張耗 1 個額度單位
- ❌ **每組重開對話**：眨眼做完換對話再做揮手 = 額外耗倍數額度，且角色會跑
- ✅ **補張在原對話內**：f02 跑掉，下一個 prompt 直接補，不用重啟
- ❌ **跑掉就重新開始**：f02 跑掉就重開對話從 f01 開始畫——免費額度 5 分鐘內見底
- ✅ **f05 直接複製 f01**：眨眼結尾 = 起點，PNG 在電腦上複製改名即可，不耗額度

如果你做到眨眼 f03（全閉）時額度耗盡：
1. **第一備援（切平台）**：開 ChatGPT 新對話，把已經產好的 f01 / f03 兩張關鍵幀上傳當參考圖，請它畫 f02 / f04 兩張中間幀。**切平台後角色風格會略偏，產完每一張都要重跑一次 5 條判準**——通過率明顯下降是正常的，但若 5 條中跑超過 2 條就改用備援 2
2. **第二備援（用講師圖包）**：下載講師預備的橘貓眨眼 5 張示範圖包，當作「素材」繼續走 CH2-3。回家額度恢復後再補自己的角色
3. **不要備援**：放棄當天，回家明天額度恢復後再做。本課允許這條路（但 CH2-3 課堂上你只能看別人按）

#### 動手過程的常見體感（你可能會這樣想）

- **「我覺得我畫的相鄰兩幀『看起來一樣』，這樣對嗎？」** —— 對！動畫的目標不是「兩幀很不同」而是「兩幀只有要動的地方在動」。眨眼 5 幀「除了眼皮位置變化全部一樣」是正解，不是過度保守
- **「f02 半閉幀我請 AI 畫『眼皮蓋下 50%』，但它畫成幾乎全閉了，怎麼辦？」** —— 這是補張中間幀最常見的卡關。用補張 prompt 明確寫「眼皮從上往下蓋 50%，黑色瞳孔還露出下半」，把「50%」量化、把「露出下半」當作可驗證的視覺特徵，AI 比較不會跑到 80% 或全閉
- **「為什麼揮手 f03 身體一直跟著手往左偏，我寫了『身體不動』還是偏」** —— Gemini / ChatGPT 對「身體位置」的描述需要極具體。試試「角色置中構圖、頭部在畫面正中央、左肩在畫面中央偏左、右肩在畫面中央偏右」這種座標式描述，比「身體不動」精準
- **「跳起來 f02 的橘貓變得『有點不像同一隻』，但我說不上哪裡跑了」** —— 把 f01 和 f02 並排放大看臉部：耳朵形狀（圓 vs 尖）、領巾打結處（在脖子正下 vs 偏一側）、眼間距。這 3 個地方是「同隻」判準在動態幀最容易跑的點
- **「我做完眨眼 5 張覺得每張都過 5 條判準，是不是太順利了」** —— 可能是你把判準放太鬆。試試把 f01 和 f05 並排，5 條都比對一遍。動畫的 5 條判準比 Part 1 嚴 3 倍，5 張全 1 次過很少見，正常都會至少補 1-2 張
- **「我做不到 5 張，只做了 f01 / f03 兩張關鍵幀，這樣算過關嗎」** —— **不算**。本單元當堂硬目標是眨眼 **5 幀**（合規 LINE 最少 5 幀），少 1 幀就不能上架。如果你卡在中間幀，請集中時間用補張 prompt 把 f02 / f04 補出來。如果額度耗盡，走備援 SOP（用講師圖包）至少湊出 5 張帶到 CH2-3 組裝

- 你的 Gemini 對話視窗能看到從 CH1-3 接續的眨眼 5 張（含補張）**全部在同一個對話**內
- 你能說出**眨眼 5 張裡哪 1-2 張最跑**（通常是 f02 / f04 中間幀），並指出跑哪一條判準（例：「f02 半閉幀同隻判準破了，AI 把耳朵畫尖了」）
- 你能把至少 1 張跑掉的幀用「補張」prompt 救回來（在我面前現場補一張）

#### 動作幀 vs 表情幀的判準差異（學員端可帶回複習）

跟 CH1-3 的 8 張表情組對比：

| 對比項 | CH1-3 表情組（8 張） | CH2-2 動作關鍵幀（眨眼 5 張當堂） |
|---|---|---|
| 並列 vs 序列 | 並列（每張獨立看） | 序列（相鄰兩幀要無縫接續） |
| 5 條判準嚴格度 | 8 張裡跑 1-2 張可放行 | 任一幀跑 1 條 = 整組要補 |
| 同視角的容忍 | 微側可（晚安那張頭微側 OK） | 完全不容忍（眨眼所有幀視角必須完全一致） |
| 配色微偏的容忍 | 5% 內可放行 | 1% 都會看到「閃光」 |
| 補張頻率 | 8 張產 1-2 張補 | 5 張產 1-2 張補（中間幀 f02/f04 預期最常補） |

把這張表記下來，下次做 Part 2 任何動態相關單元都用得到。

#### 5 條判準的優先順序（動畫版 · 跟 CH1-3 一致但加註動畫特性）

實戰上偶爾會遇到「這幀第 2 條過了第 4 條沒過」的兩難。優先順序由高到低（**跟 CH1-3 一致**）：

1. **第 1 條（同隻角色）** — 最重要，這條沒過 = 整幀作廢
2. **第 4 條（同視角）** — **動態貼圖的關鍵**，視角跑了連續播會抖到無法看
3. **第 3 條（同比例）** — 跳起來這類動作允許「形變範圍內」的變化（蹲下變扁、起跳拉長），但角色辨識度不能跑
4. **第 2 條（同風格）** — 線條粗細微跑可接受、但連續播會看到「閃爍」，跳起來這類大動作跑風險最高
5. **第 5 條（同色域）** — Part 1 是「最可救」，Part 2 變「次嚴格」（連續播放配色微偏會看到「閃光」）

**動畫版實戰決策**：第 1、4 條任一不過 = 必須補張；第 2、3、5 條微跑是否可放行，取決於「相鄰兩幀並排播放是否有抖動感」——肉眼有感 = 補；無感 = 放行。

## 商業情境案例（Case · 3 組 · 全用橘貓對應 3 種動作）

本課自用導向，「商業情境」此處轉譯為「具體的對話場景 × 為什麼要這個動作」。3 組全用橘貓，呼應 Style Guide 主線 A，且學員 CH1-3 已經會做橘貓 8 張表情，這裡直接接續「給橘貓加動作」最自然。

### 情境 1：橘貓眨眼（眨眼 5 frame · 當堂硬目標 · 講師主示範）

**角色**：橘貓上班族（同 CH1-2 / CH1-3 / CH2-1 主線 A）
**對話場景**：朋友傳訊息訴苦，你不知道怎麼回，傳一張動態貼圖代替文字
**任務**：做一張「橘貓眨眼」的動態貼圖，當作「我在聽 / 我懂 / 同理」的非語言回應
**幀構成（合規 LINE 最少 5 幀）**：f01 睜眼（起點關鍵幀）→ f02 半閉（中間幀）→ f03 全閉（終點關鍵幀）→ f04 半開（中間幀）→ f05 睜眼（= f01 · 循環回起點）

**為什麼是這個動作**：
- LINE 動態貼圖最高使用率前 5 名永遠有「眨眼」一張
- 動作小（只動眼皮）、檔案小（5 frame 約 80-120KB 內）、含義豐富（同理 / 在聽 / 害羞 / 困惑）
- **新手做動畫的第 1 個必做題目**——所有要會做動態貼圖的人都從這個起步
- **包含本單元兩個核心技巧**：(1) 產 2 張關鍵幀 (2) 用補張 prompt 延伸 3 張中間幀。學會眨眼 = 學會所有動態貼圖的底層方法

**本單元要你學會**：在你的 CH1-3 對話延續，先產 f01 + f03 兩張關鍵幀，再用補張 prompt 延伸出 f02 / f04 兩張中間幀，最後 f05 直接複製 f01 改名即可。5 張全過 5 條判準是當堂硬目標。

### 情境 2：橘貓揮手（揮手 4 frame · 進階挑戰）

**角色**：橘貓上班族
**對話場景**：早上跟朋友 LINE 群打招呼、跟同事群「hi 大家早安」、新群組第一次發言破冰
**任務**：做一張「橘貓揮手 hi」的動態貼圖，當作對話開場語
**幀構成**：f01（手在右下）→ f02（手在右上）→ f03（手在左上）→ f04（手在左下）；4 幀循環看起來像「左右揮手」

**為什麼是這個動作**：
- LINE 對話開場高頻動作前 3 名（搭配「早安」「大家好」「我來了」）
- 4 frame 循環檔案小（約 100KB 內）、循環自然、含義單一不歧義
- **動畫的「中間幀」概念第一課**——4 個位置形成手的軌跡，比 2 frame 多一層挑戰

**本單元要你學會**：4 張的「身體 / 頭 / 表情 / 左手」要完全一致（同隻 / 同風格 / 同比例 / 同視角 4 條），「手部位置」是唯一變化變數；發現身體跟著手偏移時用補張。

### 情境 3：橘貓跳起來（跳起來 5 frame · 進階挑戰 / 課後練習）

**角色**：橘貓上班族
**對話場景**：朋友通知好消息（生小孩、升職、加薪、考上）、自己有好事想分享（拿到票、抽到獎）、慶祝週末來了
**任務**：做一張「橘貓開心跳起來」的動態貼圖，當作「太好了」「我很開心」的高強度回應
**幀構成**：f01（站立）→ f02（蹲下蓄力）→ f03（起跳）→ f04（最高點）→ f05（落地）；循環播放看起來像連續跳

**為什麼是這個動作**：
- 「正向情緒高強度」的代表動作，3 種裡視覺最豐富、檔案最大、APNG 壓縮挑戰最高
- 涉及「形變允許範圍」概念（身體會壓縮、拉長），是動畫進階的入門
- **學員做完這個會有「動態貼圖整套都會做」的自信**

**本單元要你學會**：分清「動作必要的形變」（身體高度壓縮 / 拉長）和「跑掉」（配色變、視角變、變不同貓）。形變只允許在「形狀」維度上發生，**配色 / 視角 / 同隻無論姿勢怎麼變都不能跑**。

## 常見錯誤 3 條（Common Pitfalls）

### 錯誤 1：眨眼中間幀畫太多（變成全閉而非半閉）——把「補張中間幀」誤解成「再產一張新的關鍵幀」

**錯誤現象**：學員產眨眼 f02（半閉中間幀）時，prompt 寫「眼睛半閉，介於 f01 和 f03 之間」，但 AI 給的是「眼睛 80% 閉合」幾乎是 f03 的副本。學員沒注意，繼續產 f04（半開）也得到 70% 閉合的圖。最後 5 張並排：f01 全睜 → f02 幾乎全閉 → f03 全閉 → f04 也幾乎全閉 → f05 全睜——看起來像「快速閉眼 → 停留 3 幀全閉 → 突然睜開」，沒有平滑眨眼感。

**原因**：學員把「中間幀」想成「再產一張」，沒意識到中間幀的本質是「**精確的中間值（50%）**」。AI 收到「半閉」這種模糊形容詞，會偏向「更明顯的動作」（更接近全閉，因為 AI 訓練資料的「眨眼」多半是接近全閉的瞬間）。學員沒寫死「眼皮蓋 50%、瞳孔露出下半」這種**量化 + 視覺驗證點**，AI 就會跑掉。

**解法**：

1. **中間幀補張 prompt 必寫「量化 + 視覺驗證點」**：
   ```
   請畫 f02：眼皮從上往下蓋 50%（不是 80%、不是 30%，是正好一半），
   黑色瞳孔還露出下半（這是「半閉」的視覺驗證點）。
   不要畫「幾乎全閉」或「眼睛微眯」，要畫「眼皮蓋下一半，瞳孔下半還看得到」。
   ```

2. **產出後立刻把 f01 / f02 / f03 三張並排**，看眼皮位置：
   - f01：眼皮 0%（完全打開）
   - f02：眼皮 50%（正中間）
   - f03：眼皮 100%（完全蓋下）
   - 三張連續播應該像 0% → 50% → 100% 平滑過渡，不是 0% → 80% → 100% 的突兀跳變

3. **f04（半開）必須跟 f02（半閉）視覺一致**：
   ```
   請畫 f04：眼皮位置跟 f02 完全一樣（從全閉狀態掀開 50%，不是 30%、不是 70%）。
   f02 和 f04 是「鏡像關係」——眼皮高度應該一模一樣。
   ```

4. **判準口訣**：「中間幀不是模糊張，是量化張」——「半閉」要寫成「眼皮 50%、瞳孔露下半」這種可驗證的描述，不要讓 AI 自由發揮。

---

### 錯誤 2：眨眼中途重開新對話，f01-f03 變成 f04-f05 的不同角色

**錯誤現象**：學員產完眨眼 f01（睜眼）+ f03（全閉）覺得很順利，**關掉對話視窗**去喝水，回來開新對話想接續產 f02（半閉）和 f04（半開）。中間幀畫得也不錯，但跟 f01 / f03 並排放在一起時，學員傻眼——新對話畫的 f02 / f04 版本橘貓**領巾位置變了、橘色色調偏紅了 5%、線條粗細變細了**。5 張連起來是「2 隻不同的橘貓交替在眨眼」，組成 APNG 後相鄰幀都會抖。

**原因**：CH1-3 教過「同對話多輪」的 context 記憶——AI 在同一對話內看過前面的圖會記著。學員以為「角色設定卡 + prompt 文字」就能跨對話守住一致性，但實際上**文字描述對「視覺一致」的力道遠不如「看過前面那張圖」**。每開新對話，AI 都會在「角色描述空間」裡重新找一個解，每次找的略不同——眨眼 5 張就漂移到不同方向。

更糟的是，每開一個新對話 = 燒一次完整額度。本單元眨眼 5 張原本可省成 1 個完整 prompt + 4 個短句，學員每動作開新對話等於燒倍數額度，後段就會耗盡。

**解法**：

1. **整個 Part 2 上半場（CH2-1 → CH2-2 → CH2-3）就用一個對話**——延續 CH1-3 那個橘貓 8 張表情的對話，全部在裡面接續。喝水、上廁所、滑手機都可以，但**對話視窗保持開著**。

2. 不小心關掉了：到 Gemini 左側對話歷史找回原本對話，繼續續寫（不要按「新對話」）。

3. 對話歷史也找不回了：開新對話，**第一件事**：上傳 CH1-3 已產的 8 張橘貓表情圖 + 已產的部分關鍵幀，當參考圖。Prompt：「這些圖是同一隻橘貓的不同表情和動作，請延續這隻角色的設定（顏色、線條、比例、視角）」。再開始接續產關鍵幀。**絕對不要從零文字描述開始**——那等於把 Part 1 的 context 全部丟掉。

4. **判準口訣**：「同隻角色 = 同個對話」。對話切換 = 角色切換，沒有例外。

---

### 錯誤 3：眨眼 5 frame 連發不檢查，f02 跑掉繼續產 f03-f05 → 全錯

**錯誤現象**：學員做眨眼 5 frame 時為了趕時間，5 個 prompt 連發，等 5 張全部產完才開始檢查。看 5 張橫排才發現「f02（半閉）橘貓的耳朵變尖了」，但這時 f03-f05 都已經受到 f02「尖耳朵」的 context 污染——AI 看到 f02 的尖耳朵，f03 也跟著畫尖耳朵、f04 繼續尖、f05 回不到 f01 的圓耳朵。要救就要從 f02 開始全部重產。

**原因**：CH1-3 教過「全部產完才檢查」的問題，動畫版本更嚴重。Gemini 同對話多輪的 context 是**累積的**——前面所有圖都會被 AI 當作「這隻角色該長什麼樣」的參考。中間有一幀跑掉沒及時修，後面會延續錯誤，一錯到底。

**動畫版差異**：表情組是 8 張並列、肉眼可一一看；眨眼 5 幀要連續播，**f02 的污染會讓 f03-f05 全變樣**——比表情組的影響更深。

**解法**：

1. **每幀產出後立刻 5 條判準自查**，不等全部產完。動畫版甚至要更嚴：每幀產完**並排上一幀放大看臉部**（耳朵、五官、領巾打結處），確認沒跑再繼續。

2. 發現跑掉立刻補張：「上一張 f02 的耳朵形狀跟 f01 不一樣（耳朵變尖了）。請保持眼皮蓋下 50% 的中間幀設定，但耳朵形狀、五官、領巾、配色全部跟 f01 完全一樣，重畫 f02。」AI 會在同對話內重畫，後續 f03 會以「修正後的 f02」為 context，不會繼續錯。

3. 講師現場示範時刻意「**每幀都停下來檢查 + 並排對比**」，讓學員看到節奏感（不是 5 幀連發 = 快、是「產 + 並排對比 + 補」循環 = 穩）。

4. **判準口訣**：「動畫的判準是『不可累積』」。表情組可以「8 張產完一起檢查」，動畫**必須一張一張產一張一張查**，否則 context 污染後面全錯。

## 檢核題 2 條（Quiz）

### Q1（概念驗證）：以下 4 種做法，哪一種最能用 AI 產出符合 LINE 規格「眨眼 5 frame」的合規序列？

- [ ] A. 跑 5 個獨立 prompt 產 5 張閉眼程度遞增的圖，每次重開對話避免 AI 把前一張當參考
- [ ] B. 只產 f01（睜眼）和 f03（全閉）兩張關鍵幀，剩下 f02 / f04 / f05 用相同檔名複製貼上充數
- [ ] **C. 同對話依序產 f01（睜眼）+ f03（全閉）兩張關鍵幀 → 用「補張 prompt」在 f01 / f03 之間延伸 f02（眼皮 50%）+ f04（眼皮 50% 鏡像）→ f05 直接複製 f01 改名 ←（正確）**
- [ ] D. 一次連發 5 個 prompt 產出完整序列，最後並排檢查時才修跑掉的張

**解析**：

- A 錯：每個對話視窗的 context 是獨立的，AI 每次都重新解讀「橘色虎斑貓」，5 張會有 5 種版本，是最嚴重的跑掉做法（呼應錯誤 2）
- B 錯：複製貼上等同關鍵幀只給 2 張，眼皮位置 0% → 100% → 100% → 100% → 0% 不是平滑眨眼，是「靜止 → 突然全閉 → 突然睜開」，動畫看起來會跳幀；且這違反「中間幀提供平滑過渡」的眨眼設計
- C 對：「2 關鍵幀 + 3 中間幀（其中 f05 = f01）」+ 同對話多輪 + 立刻自查補張 = 本單元當堂硬目標的標準做法
- D 錯：5 frame 連發完才檢查 = 中間幀的污染會傳染後面所有幀（呼應錯誤 3）

### Q2（應用驗證）：你正在做你家貓「小橘」的眨眼 5 frame。產完 f01（睜眼）和 f03（全閉）兩張關鍵幀都通過 5 條判準。接下來你要產 f02（半閉中間幀），但第一次產出的 f02，AI 給的眼皮幾乎全閉（看起來像 80% 蓋下而不是 50%），且耳朵也變尖了一點。f04 / f05 都還沒產。這時候你該怎麼處理？

**情境細節**：
- 你已經在同一個 Gemini 對話內產到 f02
- f01（睜眼）+ f03（全閉）OK 通過 5 條判準
- f02（半閉）有兩個問題：(a) 眼皮蓋太多（80% 而非 50%）(b) 耳朵變尖
- 額度還有，但不確定夠不夠畫到 f05

**預期答案要點**（學員應答出這 4 點中至少 3 點）：

1. **不要繼續產 f04**：f02 已經污染 context，繼續產 f04 會讓 AI 以「跑掉的 f02」當基準，f04 會繼承「眼皮 80% + 尖耳朵」的錯誤一錯到底（呼應錯誤 3）。

2. **不要重開新對話**：重開會丟掉 f01 / f03 的 context（包含 CH1-3 的 8 張表情），反而讓 f02 / f04 更亂，且燒額外額度（呼應錯誤 2）。

3. **在原對話內用補張 prompt 修 f02 · 必須量化 + 視覺驗證點**：寫一個明確的補張 prompt，例如「上一張 f02 有兩個問題：(a) 眼皮蓋太多（看起來像 80% 而不是 50%），請畫『眼皮從上往下蓋正好一半，黑色瞳孔露出下半』；(b) 耳朵變尖了，請參考 f01 的圓潤耳朵形狀。其他（領巾、嘴巴、配色、線條、視角）跟 f01 完全一樣。重畫 f02。」（呼應錯誤 1：中間幀必須量化）。

4. **修正版過自查後，繼續 f04**：把修正後的 f02 當作 context，f04 prompt 寫「眼皮位置跟 f02 完全一樣（鏡像關係，從全閉掀開 50%），其他跟 f01 / f03 完全一致」。**不要從 f01 重來**，那會浪費額度且 f01 本來就是對的。

**加分要點**：

- 如果學員提到「中間幀必須量化（眼皮 50%、瞳孔露下半）」的概念（呼應錯誤 1），加分
