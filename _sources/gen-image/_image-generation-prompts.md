## ⚠️ 商業廣告資訊密度基準（每張 Tier 1 廣告必讀）

> **2026-04-25 補充**：v01 測試結果顯示、若不明確指定「商業廣告應有的資訊元素」、Nano Banana 會輸出**產品攝影風格**（純商品 + 1 行字）、不像商業廣告。
>
> **核心區別**：
> - **產品攝影** = 商品 + 美感 + 0-1 行文字 ← Nano Banana 的預設
> - **商業廣告** = 商品 + 美感 + **4-8 個商業資訊元素**、層次分明 ← 我們要的

### 7 種商業資訊元素

| 元素 | 用途 | 字級提示 |
|---|---|---|
| **主標** | 廣告核心訊息（必要）| 最大、明朝體 / 粗黑體 |
| **副標 / catchphrase** | 補充主標氛圍 | 中、細體 |
| **商品名 + 規格** | 第三人辨認商品 | 小、明確 |
| **品牌名 / logo** | 商家識別（必要）| 小、可放角落、可燙金 |
| **季節 / 期間 tag** | 「春の限定」「新発売」「2026 SPRING」 | 小徽章 / 標籤式 |
| **價格** | 視版型而定 | 小、靠商品名旁 |
| **CTA** | 行動指令（B/G 必要）| 中、按鈕 / 框線 |
| **accent**（加分）| NEW! 徽章、燙金、手寫 accent | 點綴 |

### 各版型最低資訊密度

| 版型 | 最少元素 | 重點 |
|---|---|---|
| A 主視覺 | **6 元素**（主標 + 副標 + 商品名 + 價格 + 品牌 + 季節 tag）| 視覺最強、但商業資訊不省 |
| B 橫幅 | **6-7 元素**（+ CTA）| 結構分明、左商品右資訊 |
| C 社群 | **3-4 元素**（主標 + 商品名 + 品牌 + catchphrase）| 留白、最少但仍須品牌呼應 |
| D 系列 | **8-10 元素**（每款 3 件 + 系列總標 + 品牌）| 資訊密集、每款都要有 |
| E 禮盒 | **5 元素**（主標 + 商品名 + 品牌 + 節慶限定 + 價格 / 內容物）| 質感為主、資訊精緻 |
| F 特寫 | **0-2 元素** | 零文字或極少品名、靠視覺 |
| G 促銷 | **7-8 元素**（大字優惠 + 限時 + 商品 + 品牌 + CTA）| 訊息密度最高 |
| H 生活 | **3-4 元素**（情境文案 + 商品名 + 品牌 + 落款）| 情境主導、資訊輕量 |

### 文字渲染建議（避免拼字錯誤 / 重複 / 字體名外洩）

- **日文 / 英文** Nano Banana 渲染穩定 → 主用
- **中文** 容易拼錯 → 用品牌名 / 標題、其他資訊用日英
- **太小的字** Nano Banana 會糊掉 → 9pt 以下避免
- **每個文字元素只能 1 次** → 在 prompt 加「⚠️ 每個資訊元素只能出現 1 次、不可重複」、防 AI 第一次寫錯後補寫但兩行都留
- **特別注意「ふわふわ」** → 是 ふ・わ・ふ・わ 4 個假名、AI 常拼成「Wろい」「ふぁふぁ」、需在 prompt 顯式提醒
- **⚠️ 字體名外洩問題（重要）** → AI 可能把 `Shippori Mincho` `Noto Sans JP` 等字體規範**誤當成要顯示在圖中的文字**、特別在文字內容少的版型（C / F / H / 系列版）。**所有 prompt 末尾都應加**：「⚠️『Shippori Mincho』『Noto Sans JP』『Helvetica』等字體名稱僅為 AI 渲染指令、絕對不要寫進圖中作為文字」
- **若文字錯**：同對話請 AI「請只保留正確的『XXX』那一行、刪除錯字版本」、或請整體重產
- **若字體名跑進圖中**：同對話跟 AI 說「請移除圖中的『Shippori Mincho』『Noto Sans JP』字樣、這些是字體指令、不是要顯示的內容」
