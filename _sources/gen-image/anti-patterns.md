# 反面教材｜6 種常見失敗症狀與修正

> 本檔為 `gen-image` 課程的「錯誤診斷教學」材料。course-designer 寫「常見錯誤」段落時沿用此 6 種症狀。
>
> 分為兩類：
>
> - **症狀 1-4**：AI 產出問題（L2 反推 / 結構走樣 / 風格偏差類）
> - **症狀 5-6**：人的協作問題（需求不清 / 驗收漏判類）—— 本課人作為需求方 / 把關方，這兩類比 AI 產出問題更根本

## 人的協作問題（症狀 5-6）

### 症狀 5｜需求不清（你給 AI 的是模糊想法、不是 brief）

#### 表現

學員下指令像這樣：
- 「幫我做一張好看的咖啡店廣告」
- 「幫我做 IG 貼文」
- 「我要一張類似這種的圖」（只附一張參考圖）

AI 的回應通常是：
- 結構隨便、每次都不一樣
- 色彩、字體、版面全憑 AI 想像
- 學員拿到後想修，又不知道該從哪一句話開始修

#### 根因

**學員還在「丟想法給 AI」模式**，不是「下需求書給 AI」模式。模糊指令 → AI 只能猜 → 產出當然不穩定。本課教的「brief 六件套」（業種 / 商品 / 客群 / 訴求 / 必要元素 / 風格 preset）正是為了解決這個問題——把模糊想法**結構化**再給 AI。

這不是 AI 的問題，是**學員作為需求方沒有把需求寫清楚**。

#### 解法

**執行驗收**：任何要丟給 AI 的指令，先自己檢查六件套是否齊全：

| 欄位 | 是否填寫？ | 具體內容 |
|---|---|---|
| 業種 | [ ] | |
| 商品 | [ ] | |
| 客群 | [ ] | |
| 訴求 | [ ] | |
| 必要元素 | [ ] | |
| 風格 preset | [ ] | |

六件套**缺一個就不要按送出**。花 2 分鐘把缺的補齊，比讓 AI 猜 10 次重跑省時間。

**補強技巧**：若某欄填不出來，代表**你作為 PM 還沒搞清楚這個需求**——回去跟老闆 / 客戶問清楚再動手，不要浪費 AI 配額。

### 症狀 6｜驗收漏判（把結構錯誤的圖當成功收下）

#### 表現

學員跑完一張 L4 完稿後，第一眼覺得「還行」，直接拿去用——沒對照結構驗收清單。一週後真的上線才發現：
- CTA 按鈕跑到左邊（骨架 4 應該在左下沒錯、但跑到錯的左中央）
- 徽章變成矩形不是圓形
- 主標只有 1 層沒有字階
- 多了一段沒要求的浮水印

到這時候才要改，可能要重跑 3-5 次、浪費配額也浪費時間。

#### 根因

**學員還在用「感覺好不好看」驗收**，不是用「結構清單」驗收。AI 產出的圖乍看都不錯（AI 美感平均值高），但**結構可能跑掉** —— 而學員的眼睛被表象吸走，看不到結構問題。

這就是本課強調「**驗收是結構判斷、不是美感判斷**」的核心理由。

#### 解法

**每張 AI 產出都跑一次驗收提問 A（結構對齊檢核）**：

1. 把完稿丟回給 AI（同對話內）
2. 附上骨架清單 + 8 項自檢題
3. 讓 AI 自己逐項打勾
4. 達標率 ≥ 6/8 才收，< 6/8 回到「迭代修正指令」重跑

**關鍵心法**：
- 不要用眼睛驗收（會漏）
- 用結構化清單驗收（AI 也能幫你看）
- 看不出是否好看沒關係，看得出結構是否對**才是 PM 的本職**

實作參考：`prompt-templates.md` § 類型 5 驗收提問 prompt。

---

## 症狀 1｜L2 反推後仍保留原照片（色彩 / 紋理 / 人像未移除）

### 表現

學員用 Prompt B 轉 L2，但模型回傳的圖：
- 商品照還是彩色照片、沒轉線稿
- 背景風景還是照片質感、沒灰階化
- 人像或具體紋理還保留

### 根因

1. Prompt 的「移除」指令用語太弱（「轉換為線稿」≠「完全移除照片」）
2. 模型處於「稍微修改」模式，而非「重新渲染」模式
3. 同對話的 context 太長，模型忽略了 L2 規則

### 修正 prompt 加強

```
【絕對禁令】
- 絕對不可出現任何照片、漸層、色彩
- 商品必須重新以細黑線條描繪，不得保留原照片質感
- 如果不確定，寧可過度簡化也不要保留照片元素
```

### 修正流程

1. 貼上「絕對禁令」強化句，再跑一次
2. 若仍失敗，開新對話，從頭貼 L4 圖 + Prompt B
3. 切換平台（ChatGPT → Gemini 或反之）

## 症狀 2｜文字佔位符未替換（保留原始文案）

### 表現

L2 線稿圖形結構 OK，但文字還寫著「抹茶ティラミスラテ」「¥680」等原始文案，沒變成「見出し（大）」「価格（大）」。

### 根因

1. Prompt 未明確列出每個位置該用哪個佔位詞
2. 模型預設延續上一張的文案、不做替換
3. 日文佔位符對部分模型（尤其非日文主場的）不熟悉

### 修正 prompt 加強

```
【文字替換強制規則】
- 原圖中所有文字，無論中文 / 日文 / 英文，一律替換
- 替換對照表：
  - 主標（最大字）→ 見出し（大）
  - 副標 → サブコピー
  - 商品名 → 商品名
  - 價格數字 → 価格（大）
  - CTA 按鈕文字 → CTA ボタン
  - 徽章內容 → バッジ / 割引・特典
- 禁止保留任何原始文案片段
- 若某處不確定對應哪個佔位詞，統一用「テキスト」代替
```

### Canva 補救

若 AI 死活不改文字，Plan B：

1. 把 L4 原圖匯入 Canva
2. 加一個「白色半透明方塊」覆蓋所有文字區
3. 手動在對應位置打上佔位詞

## 症狀 3｜結構走樣（位置、比例、對齊跑掉）

### 表現

- 徽章從右上跑到中間
- 主標從大字變成小字
- CTA 從底部膠囊變成矩形按鈕在側邊
- 整個構圖重新排了

### 根因

1. 模型進入「重新創作」模式而非「去風格化」模式
2. Prompt 沒強調「結構必須完全相同」
3. 同對話太長，模型遺失上一張 context

### 修正 prompt 加強

```
【結構強制鎖定】
- 結構必須與前一張完全相同，僅改變呈現方式
- 禁止重新構圖、禁止移動元素位置、禁止調整尺寸比例
- 每個元素的位置（以畫面為座標系）、大小（佔畫面百分比）、形狀（圓/方/膠囊）必須與原圖一致
- 若不確定某元素的原位置，保持原圖不動

【自檢清單】
產完請回覆是否遵守以下結構：
1. 徽章圓形是否在原位置？
2. CTA 形狀是否與原圖一致？
3. 主標位置是否未變？
```

## 症狀 4｜輸出像 PPT 草稿不像 wireframe

### 表現

線稿被畫成粗黑色塊、像簡報配圖，不是 Figma wireframe kit 風格；或變成卡通插畫、塗鴉風。

### 根因

1. Prompt 沒指定「參考風格」
2. 模型默認走「可愛 / 卡通」路線（Nano Banana 有時會這樣）
3. 線條粗細沒規範

### 修正 prompt 加強

```
【風格強制規範】
- 參考 Figma wireframe kit / Sketch wireframe kit 的視覺語言
- 線條統一細黑（0.5-1pt），禁止粗線條或馬克筆感
- 背景純白，禁止紙質感或髒汙
- 禁止卡通 / 塗鴉 / 手繪粗線 / 水彩 / 插畫風
- 允許極少量（5-10%）淡灰填色塊暗示層次（如天空、背景）
- 整體調性：專業、精準、像設計師交付給客戶的低保真稿
```

### 品質對比範例

| ✅ 像 wireframe | ❌ 像 PPT / 塗鴉 |
|---|---|
| 細黑線條 0.5-1pt | 粗黑線條 3pt+ |
| 純白背景 | 淡黃 / 紙質感背景 |
| 幾何精準（圓就圓、方就方） | 手繪感扭曲 |
| 物件線稿概括但比例正確 | 過度簡化為方框 |
| 文字佔位符字體統一（黑體）| 手寫感字體 |

## 綜合診斷流程（給學員的快速檢查表）

當 L2 轉出失敗時，依序檢查：

```
[ ] 1. 有沒有色彩殘留？→ 若有，加「絕對禁令」修正
[ ] 2. 文字是不是原始文案？→ 若是，加「文字替換強制規則」
[ ] 3. 結構有沒有跑掉？→ 若跑掉，加「結構強制鎖定」
[ ] 4. 風格像不像 wireframe？→ 若不像，加「風格強制規範」

全部修正後仍失敗：
[ ] 5. 是不是對話太長？→ 開新對話重試
[ ] 6. 是不是平台問題？→ 切換 Gemini ↔ ChatGPT
[ ] 7. 是不是原圖太複雜？→ 先簡化原圖再反推
```

## 給 course-designer 的提醒

1. **每個 CH 的「常見錯誤」段落選 2-3 個症狀即可**，不要一次塞 6 個（會讓學員先產生挫敗感）
2. **建議分配**：
   - CH1-1 / CH1-2（概念單元）：症狀 5（需求不清）+ 症狀 6（驗收漏判）作為 PM 思維入門
   - CH2-2（brief 六件套）：症狀 5 深入
   - CH3-1（L2 反推教學）：症狀 1 + 症狀 4
   - CH3-2（讀結構）：症狀 2
   - PRAC3：症狀 3 + 綜合檢查表
   - CH4-3（品牌繼承）/ PRAC4：症狀 6 深入（驗收清單對照 DNA）
3. **每個症狀的「修正 prompt 加強句」要獨立成可複製區塊**，學員遇到時直接貼
4. **不要自己編造新症狀**——這 6 種是實測最常見的，夠用
5. **症狀 5 / 6 優先級最高**——這兩個是學員作為 PM 的核心毛病，比 AI 產出問題更根本。CH1-CH2 教案的「常見錯誤」段優先選這兩個
