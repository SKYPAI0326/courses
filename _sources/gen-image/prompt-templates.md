# Prompt 模板集｜生圖 / L2 反推 / 跨產業套用 / 品牌繼承 / 驗收提問

> 本檔收錄 `gen-image` 課程核心工作流的 **5 類** prompt 模板。course-designer 寫 CH2-2 / CH3-1 / CH4-1 / CH4-3 / PRAC 時直接取用。
>
> **使用原則**：
> 1. 所有 prompt 預設 1:1 輸出；其他比例（4:5 IG、16:9 EDM、9:16 Story）在【尺寸】欄調整
> 2. 「完善版」標註給 Gemini Banana Pro / ChatGPT GPT-image-2 使用
> 3. 「堪用版」標註給 Gemini Nano Banana / ChatGPT GPT-image-1 / 免費額度使用
> 4. **風格 preset**｜日系為教學基準（主示範），其他風格（韓系 / 美式 / 歐陸 / 矽谷 / 極簡 / 復古 / 手繪 / 工業 / 街頭）為演練延伸。見 `10-industries-brief-library.md` § 風格 preset 對照表
> 5. **人的角色**｜本課學員是**需求方 / 驗收方 / 把關方**——prompt 是「下給 AI 的需求書」，不是「自己畫圖的教學」

## 類型 1｜生圖 prompt（M1 → L4）

### 完整版（建議 Banana Pro / GPT-image-2）

```
請生成一張 {ratio} 商業橫幅，骨架為「{骨架代號中文}」，風格 preset「{風格}」：

【品牌】{品牌名}（{是否虛構}）
【商品】{商品名與規格}
【場景】{應用場景，例：新品上市 / 季節限定 / 會員招募}
【客群】{年齡 + 性別 + 生活風格}

【必要元素】
- {位置 1}：{元素描述}
- {位置 2}：{元素描述}
- ...
- 主標：{主標文字，限 2 行內；語言可日 / 中 / 英對應風格}
- 副標：{副標}
- 徽章內容：{若有圓徽章，內容是什麼}
- 3 項特徵條列（每項配 icon）：
  1. {特徵 1}
  2. {特徵 2}
  3. {特徵 3}
- CTA 按鈕文案：{CTA 文字}

【風格 preset】
- 類型：{日系 / 韓系 / 美式復古 / 矽谷科技 / 北歐極簡 / 歐陸法式 / 美式海報 / 瑞士排版 / 手繪童趣 / 工業風 / 街頭潮流 —— 預設日系}
- 配色 ≤ 3 色：{primary 色名+hex} + {accent 色名+hex} + {bg 色名+hex}
- 字體：主標 {字族}、副標 {字族}
- 視覺調性：{配合 preset 給具體形容詞，例：療癒 / 極簡 / 力量 / 復古}
- 禁用：{該 preset + 該品牌的禁用項，例：漸層、純黑、霓虹色}

【尺寸】{ratio，預設 1:1}

⚠️ 字體名稱（Shippori Mincho / Noto Sans JP / Helvetica 等）僅為 AI 渲染指令、絕對不要寫進圖中作為文字
```

> **風格 preset 速查**：見 `10-industries-brief-library.md` § 風格 preset 對照表（11 種 preset × 配色 / 字體 / 質感 / 業種）。**日系是教學基準預設值**，演練時可替換其他 preset 體驗「結構不變、表象切換」。

### 堪用版（Gemini Free / ChatGPT Free）

免費版模型對 prompt 長度敏感，壓縮成 8-10 行：

```
1:1 商業橫幅，{骨架類型}，{風格 preset}。
品牌：{品牌名} — {商品}
客群：{一句話}
必要元素：商品在{位置}、主標「{主標}」、副標「{副標}」、徽章「{徽章內容}」、CTA「{CTA}」
風格：{風格 preset 關鍵字}，配色 {3 色} 不超過 3 色
字階三層分明
```

### 使用變化：比例調整

| 比例 | 用途 | 主標字數建議 |
|---|---|---|
| 1:1 | IG post、FB 方形廣告 | ≤ 12 字 |
| 4:5 | IG 直式廣告 | ≤ 12 字 |
| 16:9 | YouTube thumbnail、EDM | ≤ 15 字 |
| 9:16 | IG Story、TikTok | ≤ 10 字 |
| 3:4 | Pinterest、印刷 | ≤ 14 字 |

## 類型 2｜L2 視覺化線稿反推 prompt（L4 → L2）

### 完整版（必須在同對話內接續 L4 生成後）

```
請將剛才那張圖轉為 Level 2 視覺化 wireframe：

【必須保留】（視覺結構層）
1. 所有商品物件的輪廓線稿
   - 瓶子 / 杯子 / 手機 mockup / 包裝盒 的外輪廓與必要結構線完整保留
   - 不畫材質、反光、陰影
2. 所有場景元素的輪廓線稿
   - 山脈、建築、雲、植物、道具轉為極簡白描
   - 允許 5-10% 灰階填色塊暗示層次（天空、遠景）
3. 所有 UI 元件的幾何形狀
   - 圓形徽章、膠囊 CTA、方形 icon、分隔線全部保留
   - 圓形直徑、膠囊長寬比、位置與原圖一致
4. 紋理的極簡暗示
   - 水珠 → 留幾個圓圈
   - 粉末 → 留幾個點
   - 熱氣 → 留 2-3 條曲線

【必須移除】
- 所有色彩（全圖灰階）
- 所有照片質感、漸層、反光
- 所有原始文字內容
- 所有品牌識別元素（logo 換成「ロゴ」方框）

【文字佔位符對照】
- 主標 → 見出し（大）
- 副標 → サブコピー
- 內文 → 本文テキスト
- 商品名 → 商品名
- CTA → CTA ボタン
- 價格 → 価格（大）
- 徽章 → バッジ / 割引・特典 / 期間・日付
- 特徵條列 → 特徴・効果 1 / 2 / 3
- App 相關 → アプリアイコン / アプリ名 / 読み仮名
- 標籤 → ラベル / タグ
- 特徵欄位 → 特徴ポイント 1 / 2 / 3

【風格】
Figma wireframe kit + 極簡白描手繪混合。
線條細黑（0.5-1pt），背景純白。
允許極少量灰色填色暗示層次。
整體觀感：看得出「這裡是什麼東西」，但看不出「是哪個品牌的什麼商品」。

【尺寸】1:1（與原圖一致）
```

### 堪用版（免費版）

```
同對話內，將剛才那張圖轉成 Figma 風格灰階 wireframe：
- 商品、道具、mockup 的輪廓線保留（線稿化）
- 所有色彩移除（全灰階）
- 文字全換日文佔位符：見出し / サブコピー / 本文 / CTAボタン / バッジ / 価格
- 結構位置與原圖完全一致
- 細黑線條、純白背景、允許淡淡灰色塊暗示層次
```

### 常見失敗與修正 prompt 加強

| 症狀 | 修正加強句 |
|---|---|
| 仍保留照片或色彩 | 加入「絕對不可出現任何照片、漸層、色彩、光影」|
| 文字還是原始內容 | 加入「所有文字必須替換為日文佔位符，不得保留任何中文 / 英文 / 原始文案」|
| 結構走樣 | 加入「結構必須與前一張完全相同，僅改變呈現方式。不得重新構圖」|
| 像 PPT 草稿 | 加入「參考 Figma wireframe kit 的視覺風格，不是 PowerPoint」|
| 物件輪廓過度簡化為方框 | 加入「商品的形狀必須用線稿精準描繪，不是簡單的矩形或圓形代替」|

## 類型 3｜跨產業套用 prompt（L2 → 新業種 L4）

### 完整版（吃結構 JSON）

```
吃下以下結構 JSON：

[貼入 L0 JSON schema，見 l2-skeleton-library.md]

不要改結構，只改內容。為以下新情境產出 L4 完稿：

【新業種】{業種}
【新品牌】{品牌名}（虛構 / 真實）
【新商品】{商品名與規格}
【新客群】{客群描述}
【新訴求】{訴求語句}

【對應替換】
- 原骨架中的 product 位置 → 放 {新商品視覺描述}
- 原骨架中的 headline → 改為「{新主標}」
- 原骨架中的 subcopy → 改為「{新副標}」
- 原骨架中的 badge → 改為「{新徽章內容}」
- 原骨架中的 features × 3 → 改為：
  1. {新特徵 1}
  2. {新特徵 2}
  3. {新特徵 3}
- 原骨架中的 cta → 改為「{新 CTA 文案}」

【風格】
- 配色 ≤ 3 色：{新 3 色}
- 視覺調性：{新業種調性}
- 其他保持骨架原則（字階三層、日系、留白）

【驗證自檢】
產完請自我檢查並回覆：
1. 所有 elements 位置是否與原 JSON 完全對應？
2. visual_hierarchy 是否遵守？
3. 配色是否只用指定 3 色？
```

### 堪用版（無 JSON，直接用完稿圖作為結構參考）

```
以下是結構參考圖 [附 L2 線稿或原 L4]，保持相同版面結構，套用新情境：

新業種：{業種}
新商品：{商品}
新主標：{主標}
新副標：{副標}
新徽章：{徽章}
新 3 特徵：{列 3 項}
新 CTA：{CTA}

配色換成：{新 3 色}
所有元素位置、大小比例、對齊方式與參考圖完全一致。
```

## 類型 4｜品牌 DNA 繼承 prompt（同骨架 + 同 DNA × 新主題）

### 完整版

```
吃下以下品牌 DNA：

[貼入 brand-dna.md 完整內容]

---

結構沿用：{骨架代號，例：骨架 4 商品中心・道具環繞}

本次主題：
- 商品：{新商品}
- 季節 / 場景：{例：夏季限定}
- 徽章內容：{例：6/1-7/31 限定}
- 價格：{例：Regular ¥580 / Tall ¥630}
- 道具暗示：{例：柚子切片、冰塊、氣泡水玻璃杯}

請依以上 DNA + 骨架 + 主題產出 L4 完稿。

【強制自檢】
產完請自我檢查並回覆：
1. 色票是否只用了 DNA 指定的 3 色？（列出實際用色 hex）
2. 字族是否符合 DNA 指定？（列出實際字族）
3. 禁用清單是否全部遵守？（逐條比對）
4. 語氣與 DNA 是否一致？（用 DNA 的「示範句對比」比對）
```

### 使用時機提示

- **同品牌連續 5 張系列**：每張換 { 商品、季節、徽章、價格、道具 }，其他全保持
- **跨骨架同 DNA**：主力骨架做 3 張、次要骨架做 2 張，測試 DNA 的泛用性
- **同 DNA × 多比例**：同主題產 1:1 + 4:5 + 9:16 三比例，測試 DNA 的擴展性

## 類型 5｜驗收提問 prompt（人對 AI 做品質把關）

本課**最關鍵的 prompt 類型**。學員的核心能力是**驗收**而非設計——拿到 AI 產出後，不靠美感主觀判斷，改用「結構化驗收清單」做判定。

### 為什麼要有驗收提問？

學員常見的挫敗：**「這張圖好像還行、好像又差一點，但我說不出哪裡不對」**——這是**沒有結構化判準**的症狀。本類型的 prompt 把「我的主觀感覺」外化成**可丟給 AI 自己回答**的問題，讓 AI 當協作驗收官。

### 驗收提問 A｜結構對齊檢核（L4 完稿驗收）

丟給 AI（同對話內，延續上一張完稿輸出）：

```
請檢查剛才產出的這張圖，對照以下結構清單逐項回答：

骨架：{骨架代號，例：骨架 1 商品左・文字右}

自檢項：
1. 商品是否在指定位置？（例：左 40%）→ 答：是 / 否 / 部分
2. 主標是否在指定位置？（例：右上）→ 答：是 / 否 / 部分
3. 徽章是否在指定位置？（例：商品右上，圓形）→ 答：是 / 否 / 部分
4. CTA 是否在指定位置與指定形狀？（例：右下、膠囊形）→ 答：是 / 否 / 部分
5. 3 個特徵條列是否清晰可辨？→ 答：是 / 否 / 部分
6. 配色是否在指定 3 色內？若超出，列出多的顏色
7. 視覺動線是否符合 {Z / F / T / 中央放射 / 左上→右下} 型？
8. 有無多出來的、我沒要求的元素？（多餘文字 / 多餘裝飾 / 錯字等）

最後給出總結：「結構達標率 X/8」。若 ≥ 6/8 我會收，若 < 6/8 請你自己指出要修哪三項最優先。
```

### 驗收提問 B｜L2 線稿可繼承性檢核（M3 承段專用）

丟給 AI（同對話內，產完 L2 線稿後）：

```
請對剛才產出的 L2 線稿做繼承性自檢：

1. 物件輪廓是否清晰可辨（不是黑塊 / 不是糊掉）？
2. 是否完全移除原圖的色彩 / 紋理 / 人像質感？
3. 所有文字是否替換為指定佔位符（見出し / サブコピー / CTA ボタン 等）？
4. 徽章、按鈕、icon 的幾何形狀是否與原圖一致？
5. 元素位置、大小比例是否與原圖完全對應？
6. 整體風格是否像 Figma wireframe kit（不是 PPT、不是卡通）？

若有任何 1 項不符，請用 1 句話說：「這張 L2 線稿不建議繼承，因為 {X}」。
若全部符合，請說：「這張可以作為骨架資產繼承到下一輪」。
```

### 驗收提問 C｜品牌系列一致性檢核（M4-M5 合段專用）

丟給 AI（產出 3 張系列後，同對話）：

```
請拿這 3 張（A / B / C）做品牌系列一致性自檢，對照以下 DNA：

[貼 brand-dna.md]

檢核項：
1. 3 張的配色是否都只用了 DNA 指定的 3 色？（列出實際用色 hex）
2. 3 張的字族是否符合 DNA 指定？
3. 3 張是否都遵守禁用清單？（逐條比對、逐張比對）
4. 3 張的結構骨架是否一致？（例：都是骨架 4）
5. 3 張的語氣調性是否一致？（對照 DNA「示範句對比」）

若任一張偏離，指出「第 {A/B/C} 張在第 {1-5} 項不符，具體是 {...}」。
通過條件：3 張在 5 項全部一致 = 可作為同系列資產收藏。
```

### 驗收提問 D｜迭代修正指令（不滿意時用）

當 AI 產出不符驗收清單，學員要發出**結構化修正指令**而非「再畫一張」：

```
剛才那張圖的結構驗收結果：{貼驗收 A 的結果，例：達標率 5/8}

需要修的 3 項：
1. 第 {N} 項（例：徽章位置跑到左上，應該在商品右上）
2. 第 {N} 項（例：配色多了橘色，應該只用 3 色）
3. 第 {N} 項（例：多出一個我沒要求的裝飾邊框）

請**保留其他 5 項通過的部分**，只修上述 3 項，產出修正版。
```

### 給 course-designer 的提醒

- 本類型的 prompt 在 **CH3-1（L2 反推）、CH4-3（品牌繼承）、PRAC4（品牌系列）、CH5-1（整合實戰）** 都要出現
- 每個 PRAC 的自評清單都應該有這樣一份「丟給 AI 自檢」的 prompt 卡片
- **教學意義**：讓學員建立「AI 也能當驗收官」的協作習慣——人不用每一處都用眼睛判斷，把結構化問題丟回 AI，由 AI 自己對照答案，是 PM 思維的核心

## 通用提醒（寫入所有 prompt 的隱性 context）

### 文字亂碼預防

AI 生圖模型對中文 / 日文字渲染不穩定，建議：

1. **主教策略**：讓 AI 只產「空版 + 圖像元素」，文字交給 Canva / Figma 後製補
2. **若要 AI 嘗試產文字**：限主標 + CTA 兩個元素，其他細節文字一律留白
3. **Fallback**：產完若文字錯誤，同對話內說「請將圖中所有文字替換為 [新文案]」再試
4. **ChatGPT GPT-image-2 在日文處理上通常優於 Gemini Nano Banana**，若需日文精準可優先用 ChatGPT

### 版權邊界

- AI 產出**僅作為參考稿與 wireframe 結構教學**，不得直接商用
- 商用流程必須：
  1. 用 Canva 替換為合法授權素材（Canva Free / Pro 的內建庫）
  2. 或用 Adobe Firefly 重製（Firefly 訓練資料版權乾淨）
  3. 自家拍攝 / 購買素材
- 講義示範與練習產出統一加註：「本頁圖片為 AI 生成練習作品，不可商用」

### 配額管理

| 平台 | 免費配額（2026-04）| 適合場景 |
|---|---|---|
| Gemini 免費版 | 約 30-50 次/日 | 一般練習與作業（兩家擇一） |
| Gemini Advanced | 訂閱制 | 升級品質、降低重試次數 |
| ChatGPT 免費版 | 每日有限 | 一般練習與作業（兩家擇一） |
| ChatGPT Plus | 訂閱制 | 升級品質、降低重試次數 |

**本課程建議學員至少持有 Gemini 或 ChatGPT 任一家帳號**——兩家當前在本課工作流的核心能力都能達標、挑你已有帳號的那家、介面熟悉省學習成本。模型表現會隨升級浮動、卡關時切換另一家試試。

## 給 course-designer 的提醒

1. **每類 prompt 都有「完善版」與「堪用版」兩個版本**，CH2-2 / CH3-1 要對兩種學員分別示範
