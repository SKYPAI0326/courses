# 飛輪規則（錯誤轉規則）

每當 Claude 犯錯（格式不對、命名跑掉、連結失效、邏輯錯誤），立刻將該錯誤轉為一條具體規則加入此文件的對應區塊。規則會因此持續進化，同樣的錯誤不再發生第二次。

---

## 複製按鈕定位規則

- **絕對定位的複製按鈕必須同步設定父容器 padding-right**：若 `.step-example-copy`（或同類 copy button）使用 `position:absolute;top:Xpx;right:Ypx`，其父容器必須加 `padding-right:72px`，否則文字會被按鈕覆蓋
- 格式：`padding:14px 72px 14px 16px`（四值寫法，保留左側 16px 不變）
- 建立新頁面時，有 absolute copy button 的地方一律加 `padding-right:72px`

---

## WordPress 資料庫操作規則

- **禁止**用 Python mysql.connector 讀出 `post_content` 再寫回——會雙重跳脫 `\n` 與 `\u002d`（`--`），導致排版崩潰與 CSS 變數失效
- **必須**用 MySQL `REPLACE()` 直接在資料庫內操作，例如：
  ```sql
  UPDATE wp_posts SET post_content = REPLACE(post_content, '舊值', '新值') WHERE ID=N;
  ```
- 修改後用 `HEX()` 確認關鍵字元（換行應為 `0A`，連字符應為 `2D`）

## 教案 → HTML 渲染邊界規則

教案 `_lessons/{slug}/{unit-id}.md` 的 6 大區塊**並非全部**該渲染為學員可見頁面。學員視角與課程內部視角必須分開：

| 教案區塊 | 學員可見？ | 處理 |
|---|---|---|
| 教學流程（破題/概念/操作/動手/檢核） | ✅ | 完整渲染 |
| 商業情境案例 | ✅ | 完整渲染 |
| 動手練習題 | ✅ | 完整渲染 |
| 常見錯誤 3 條 | ✅ | 完整渲染 |
| 檢核題 2 條（Quiz） | ✅ | 完整渲染 |
| **試跑包需求清單（Verification Asset Spec）** | ❌ | **只給 PM/講師規劃用，不渲染** |
| **「為什麼這樣設計」「課程設計理由」段** | ❌ | **內部設計理由，不渲染** |

### 觸發場景

寫給 `course-lesson-writer` / `build-course-page` subagent 的 prompt 必須**明確說「不渲染」的區塊**。預設「忠於教案、完整呈現所有區塊」會把講師端準備清單（如「示範機器準備」「USB 隨身碟」「截圖序列」「錯誤範例展示包」）整段搬上學員頁。

### 偵測訊號（學員頁面上不該出現的字眼）

- 「講師端」「示範素材」「速查卡（給學員帶走）」
- 「course_type = …」「skill-operation 類型」
- 「為什麼拆 N 頁」「為什麼這樣設計」
- 「示範機器準備」「USB 隨身碟」「離線安裝包」
- 「Win 學員自行確認」（這種以「學員」為第三人稱的描述）

### 修法

1. subagent prompt 加 negative list：「教案的『試跑包需求清單』區塊**不要**渲染為 HTML section」
2. 若已誤渲染，整段移除 + 重編後續 section eyebrow
3. 例外：troubleshoot 排錯手冊的「三行診斷指令」原本在試跑包段，但實際是學員工具——保留 callout 並把 section heading 改為「三行診斷指令」，body 改寫為學員視角
