PART 1 · 觀念建構與環境準備
1.4

串接容器：
將 GTM 容器碼安裝到網站中

網站就緒後，這是我們最後一步基礎建設。我們將去 Google 取得專屬的「容器碼」，並透過超強的 GTM4WP 外掛，不寫任何程式碼就完成專業級的埋設。

學完這節，你會
學會如何從 GTM 後台建立並複製容器 ID
掌握 GTM4WP 外掛的基本設定與啟動方式
使用 Tag Assistant 確認 GTM 是否成功連線
Section 01

第一步：取得 GTM 容器 ID

請先前往 tagmanager.google.com 並登入你的 Google 帳號。建立一個新的容器（帳戶名稱填寫你的公司，容器類型選擇「網頁」）。

建立完成後，系統會自動彈出一個包含程式碼的視窗，但你不需要複製那一大串程式碼。

💡
重點資訊：關閉那個視窗。請看 GTM 介面頂端的藍色橫幅，那裡有一組像是 GTM-XXXXXXX 的字串。這就是你的容器 ID，請把它複製下來，在 WordPress 裡我們只需要這個 ID 就夠了。
Section 02

第二步：設定 GTM4WP 外掛

為了避免手動修改程式碼出錯，以及為了未來能輕鬆抓取「資料層 (Data Layer)」的高級數據，我們在範例網站中已經幫你預裝了由 Thomas Geiger 開發的神級外掛：GTM4WP。

01
進入設定畫面
登入你的測試網站後台。點擊左側選單的 設定 > Google Tag Manager。
02
填入 GTM ID
在「Google Tag Manager ID」欄位中，貼上你剛剛複製的 GTM-XXXXXXX。
03
啟動容器並儲存
確定下方的「Container code on/off」狀態是勾選為 ON。拉到頁面最下方，點擊「儲存設定」。
Section 03

驗證：成功了嗎？

最簡單的驗證方式是回到 GTM 後台，點擊右上角的 「預覽 (Preview)」 按鈕。

  • 在彈出的視窗中輸入你本機測試網站的網址（如 http://teaching-demo.local）。
  • 如果順利彈出一個新視窗，且原視窗顯示 "Connected!"，這代表 GTM 已經成功進駐你的網站了！
🎉
恭喜完成基礎建設！
