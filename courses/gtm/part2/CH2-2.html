PART 2 · 基礎追蹤與 Chrome 工具運用
2.2

脫離工程師的必殺技：
看懂網頁骨架與 CSS 選擇器

如果網頁上有兩個按鈕都叫「立即購買」，GTM 該怎麼分辨你要追蹤哪一個？答案就藏在網頁的原始碼裡。這節課我們要學會使用 Chrome 開發者工具，讓你具備「透視」網頁的能力。

學完這節，你會
學會使用 Chrome 「檢查 (Inspect)」工具
看懂什麼是 HTML 的 id 與 class
掌握撰寫 CSS 選擇器的基本規則（含複合選擇器）
用 GTM 預覽模式驗證「真正被點擊的是哪一層」
Section 01

呼叫透視鏡：Chrome 檢查工具

在你的測試網站上，找到那個我們準備好的「特定按鈕」。對著它 按擊滑鼠右鍵，選擇最下方的 「檢查 (Inspect)」。

這時畫面右側或下方會彈出一個充滿程式碼的面板。別慌！Chrome 已經幫你把剛才點擊的那個按鈕對應的程式碼標示為底色了。

Section 02

尋找獨一無二的身分證

在那段被標底色的程式碼中，我們要在 <a>（連結）或 <button>（按鈕）標籤裡，尋找兩個關鍵字：id 或 class。

<!-- ① 外層 div：我們加的自訂名稱在這裡 -->
<div class="uagb-buttons__outer-wrap promo-btn-01">
  <!-- ② 內層 a：使用者實際「點到」的那層 -->
  <a class="uagb-buttons-repeater wp-block-button__link" href="#">
    領取優惠券
  </a>
</div>
01
ID (身分證字號)
如果標籤上有 id="..." 屬性，這是最精準的追蹤標的，因為 ID 在整個頁面中必須唯一。不過許多頁面編輯器（包含 Spectra）預設不加 ID，而是將自訂名稱放在 class 裡。
02
Class (制服 / 特徵)
我們的練習網站就是這個情況。在 Spectra 的「進階」設定中加入的自訂名稱 promo-btn-01，會出現在外層 <div> 的 class 裡——但注意：這只是「識別這個按鈕區塊的標籤」，不代表使用者點擊時 GTM 記錄的就是這一層。
03
用 GTM 預覽模式確認真正的點擊層
進入 GTM 預覽模式後，點擊一次按鈕。在 Tag Assistant 左側點擊「Click」事件，切換到「Variables」分頁，找到 Click Element 這一列，看它顯示的是哪個標籤與 class——那個才是你之後要填入選擇器的依據。如果 Click Element 顯示的是 <a class="uagb-buttons-repeater...">，代表你需要把 <a> 那層也寫進去。
Section 03

如何寫成 CSS 選擇器？

在 GTM 裡，如果我們想告訴系統「我要抓有這個特徵的東西」，我們必須用 CSS 選擇器（CSS Selector）的語法來寫。規則超級簡單：

★
記住這三個寫法：
1. 如果是 ID，前面加井字號 #。例如：#main-cta
2. 如果是 Class，前面加小點 .。例如：.promo-btn-01
3. 如果點擊層在外層元素的內部，用空格串接兩層。例如：.promo-btn-01 a.uagb-buttons-repeater（意思是「在 .promo-btn-01 裡面、帶有 uagb-buttons-repeater class 的 <a>」）
✓
實際填寫前先用 GTM Preview 確認：看 Click Element 顯示的是哪一層，選擇器就寫到那一層為止。外層 class 只是幫你「定位」這個按鈕區塊，真正要比對的是使用者手指碰到的那個元素。

把 GTM 預覽模式確認過的選擇器記錄下來，下一節我們要把它填入觸發條件。
