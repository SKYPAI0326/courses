專案執行 Check-list

今日演練情境
客戶要追蹤「預約課程體驗」的表單送出
客戶網站上有一個 SureForms 表單，送出後會出現「感謝您，我們將盡快與您聯繫。」的字樣。他想知道每週有多少人送出這個表單。請與你的同學，帶著以下 Check-list 完成整個設定流程。

依序點擊每個步驟，確認完成後勾選：

0 / 5 項完成
定義轉換行為
確認要追蹤的是「感謝字樣出現（元素可見度）」還是「表單 ID 提交」？本情境使用元素可見度觸發條件。
檢查變數是否開啟
進入 GTM「變數」分頁，確認已開啟 Element Visibility 相關的內建變數。
建立並過濾觸發條件
建立「元素可見度」觸發條件，使用感謝訊息的 CSS 選擇器，並勾選「DOM 變更時觀察」。
在預覽模式中親自操作一次
進入 Tag Assistant 預覽，送出表單，確認 generate_lead 代碼有 Fire。若沒有 Fire，確認 CSS 選擇器是否正確。
填寫版本名稱並發布
版本名稱建議：「新增：預約表單轉換追蹤（generate_lead）」，發布後進入 PRAC4-2 進行報表驗收。
5 個步驟全部完成
設定完成。接下來前往下一頁，確認 GA4 報表真的有收到數據。
