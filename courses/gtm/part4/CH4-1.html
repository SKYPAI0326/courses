PART 4 · 資料管理與品質控制
4.1

打通任督二脈：
什麼是資料層 (Data Layer)？

這是區分「新手」與「進階玩家」的分水嶺。如果不懂資料層，你就只能追蹤表面上的點擊；懂了資料層，你就能追蹤訂單金額、商品分類、甚至是使用者的會員等級。

學完這節，你會
用白話文理解 Data Layer 的運作邏輯
在 GTM 預覽模式中找出現有的資料層訊息
建立「資料層變數」來接收網頁傳來的暗號
Section 01

網頁的地下秘密通道

在前面的章節，我們都是用 GTM 主動去「偷看」網頁上的按鈕長什麼樣子（CSS 選擇器）。但有些資訊寫在網頁看不到的地方，例如：使用者的 User ID、或是結帳時的訂單總金額。

Data Layer 就像餐廳的「點餐單」
如果 GTM 是結帳櫃檯，那網頁就是外場服務生。

外場服務生（網頁）會在客人點完餐後，寫一張標準格式的「點餐單（Data Layer）」，並透過送餐通道傳遞給結帳櫃檯（GTM）。

櫃檯不需要自己跑去客人的桌上看他吃了什麼，只需要讀懂那張點餐單上的 "price": 1500 或是 "item_name": "T-shirt" 就可以了。
Section 02

尋找隱藏的 Data Layer

還記得我們在 PART 1 安裝的 GTM4WP 外掛嗎？它的強大之處就在於：它是一位非常勤奮的服務生，已經幫我們寫好了一大堆「點餐單」。

你可以打開 GTM 的「預覽模式 (Preview)」，點進你測試網站的一篇文章（不是首頁）：

  • 在 Tag Assistant 的左側時間軸，你會看到很多事件。
  • 點擊任何一個事件，然後切換到上方的 「Data Layer」分頁。
💡
請在「文章」頁面操作，不要在首頁
Data Layer 的內容會依頁面類型不同而改變。首頁是 WordPress 的「頁面（page）」類型，不具備分類欄位，所以 pagePostCategory 不會出現，pagePostType 也會顯示 "page" 而非 "post"。

請在測試網站後台新增一篇文章並設定分類，或直接瀏覽已有的文章頁面，才能看到以下完整的資料層內容。
// 在文章頁面的 Data Layer 分頁裡，你會看到類似這樣的程式碼：
{
  "pagePostType": "post",
  "pagePostCategory": "行銷科技",
  "pagePostAuthor": "admin"
}

這些就是外掛自動推播給 GTM 的精美數據！（作者欄位顯示的是 WordPress 的使用者帳號名稱）

Section 03

接收暗號：建立資料層變數

現在櫃檯（GTM）看到這張單子了，我們要教它如何把單子上的資訊「抄寫」到報表裡。

🛠️
操作步驟：
1. 在 GTM 左側選單點擊「變數」，往下捲到「使用者定義的變數」，點擊「新增」。
2. 點擊右上角的鉛筆圖示或「變數類型」區塊，選擇 「資料層變數 (Data Layer Variable)」。
3. 在下方出現的「資料層變數名稱」欄位中，一字不差地填入 pagePostCategory（大小寫需完全一致）。
4. 「資料層版本」保持預設的「版本 2」即可，無需更改。版本 1 是舊版的點式記法（如 page.category），版本 2 支援現代的巢狀物件結構，GTM4WP 外掛推送的資料也是版本 2 格式，因此一律選版本 2。
5. 點擊頁面左上角的變數名稱欄位（預設顯示「未命名的變數」），將它改為 DLV - 文章分類，然後點擊右上角「儲存」。
★
恭喜升級！
現在你已經學會建立「資料層變數」了！未來在發送 GA4 事件時，你就可以把這個變數當作「參數」送出去（例如：紀錄使用者最常看哪個「分類」的文章）。
