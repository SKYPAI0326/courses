PART 6 · 資料管理、除錯與治理
6.4

Debug 全流程：五層工具交叉比對

單靠 GTM Preview 不夠。事件沒收到？可能是 trigger 沒觸發、Tag 沒跑、平台沒收、平台收到但欄位空。一層一層剝開才找得到真相。

學完這節，你會
用 Preview Mode 找出 trigger / variable 的錯
用 DebugView、Pixel Helper 驗證平台收到什麼
用 DevTools Network 看最終送出的 request
(01)

五層排查順序

埋碼出問題時，從最接近瀏覽器的那一端往回找：

層工具看什麼
1. dataLayerDevTools Console 打 dataLayer事件 push 進來了嗎？欄位對嗎？
2. GTMGTM Preview Mode（Tag Assistant）Trigger 有觸發嗎？Tag 有 fire 嗎？Variable 值對嗎？
3. 平台接收GA4 DebugView / Meta Pixel Helper / TikTok Events Manager平台真的收到了嗎？
4. RequestDevTools Network → 過濾 collect/tr實際 HTTP request 的 payload 對嗎？
5. 後端（CAPI）Server log / Meta Events Manager Test Events後端 API 有成功接到嗎？
✓
排查原則：從 1 往 5 逐層找。若 Layer 2 沒事件 → 回頭查 Layer 1；Layer 3 沒收到 → 查 Layer 4 request。不要亂跳。
(02)

GTM Preview Mode 三個必看分頁

點 Preview → 輸入網址 → Tag Assistant 視窗會打開。網頁上做動作，左側 event 列表會累積。

  • Tags 分頁：哪些 Tag Fired / Not Fired。Not Fired 的滑鼠移上去看「沒觸發的原因」（通常是 trigger 條件不符）
  • Variables 分頁：點任一 event，下方顯示當下所有 Variable 的值。最常用於確認 DLV 有正確取值
  • Data Layer 分頁：看 dataLayer.push 的完整 payload。沒看到自訂事件 → 是前端沒 push，不是 GTM 問題
⚠
Preview 不 work 時：(1) 用 Chrome 無痕；(2) 關閉 AdBlock / Ghostery 類擴充；(3) 網站若有 CSP 嚴格設定可能擋 Preview iframe —— 跟工程師要求把 tagassistant.google.com 加白名單。
(03)

平台層：DebugView vs Pixel Helper

GA4 DebugView（GA4 管理介面 → DebugView）：啟用條件是 Tag 送出時帶 debug_mode: true，或使用 Chrome 擴充「GA Debugger」。Preview Mode 開啟時 GTM 會自動加 debug flag，所以 Preview 期間的事件都會出現在 DebugView。

Meta Pixel Helper（Chrome 擴充）：點圖示會列出當前頁面的 Pixel 事件，顯示 Pixel ID、event name、參數、match quality。紅色 = 錯，黃色 = 警告（常見如 AEM 未設定），綠色 = OK。

Meta Events Manager → Test Events：專門用於測 CAPI。複製測試 token → 後端事件塞 test_event_code → Events Manager 即時顯示收到的事件，還會提示「Pixel 跟 CAPI 的 event_id 是否配對成功」，查 dedup 超好用。

TikTok Events Manager → Diagnostics：介面類似 Meta，顯示事件覆蓋率、match quality、有無重複送。

(04)

最硬核：DevTools Network

所有工具都說 OK 但數據就是不對？打開 DevTools → Network，用關鍵字過濾看實際 HTTP request：

平台過濾字串檢查重點
GA4collectQuery String 看 en（event name）、ep.xxx（event params）、tid（Measurement ID）
Meta Pixeltr?Query String 看 ev（event）、cd[value]、eid（event_id for dedup）
Google Adsgoogle-analytics.com/g/collect看 _et（conversion tracking）與 tid 開頭 AW-
TikTokanalytics.tiktok.com看 POST body 的 event、properties
◆
疑難雜症指南：(1) GA4 事件沒進 → 檢查 tid 是否對應正確 property；(2) Meta value 為 0 → 常是 DLV 取不到值，回 GTM Preview 確認；(3) Pixel fire 兩次 → 找重複 Tag 或 DOM 連續觸發事件；(4) CAPI 跟 Pixel dedup 失敗 → event_id 必須兩邊完全一樣（字串、大小寫、前後空白）。
