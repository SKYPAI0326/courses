PART 6 · 資料管理、除錯與治理
6.3

Consent Mode v2：不能忽略的合規紅線

2024 年 3 月之後，Google 官方宣布：廣告主如果接觸到歐盟流量，Ads 與 GA4 必須使用 Consent Mode v2，否則再行銷受眾會停止累積。台灣接海外單、跨境電商、政府案，這個坑都必須填。

學完這節，你會
分清 Consent Mode v1 / v2 / Advanced / Basic 的差別
在 GTM 設定預設 consent state 與 update 機制
把 CookieYes / Cookiebot 等 CMP 與 GTM 接起來
(01)

Consent Mode 在解什麼問題

歐盟 GDPR 要求使用者必須主動同意才能被追蹤。但「拒絕的人」也是廣告主想知道的 —— 至少能知道「拒絕率多少、他們看了什麼頁」。傳統 cookie banner 的做法是「按拒絕 → 全部代碼都不跑」，結果數據全黑。

Google Consent Mode 的解法：追蹤代碼依然載入，但根據 consent state 決定送什麼資料。拒絕的用戶依然有匿名的流量計數，但不會被 cookie 識別。

模式使用者狀態Tag 行為
Advanced同意前Tag 會跑，但只送 cookieless ping（給 Google 模型估算流量）
Advanced同意後完整 Tag 行為，帶 cookie 識別
Basic同意前Tag 根本不載入（保守版）
Basic同意後載入後開始追蹤
ℹ
v1 vs v2 差別：v2 增加 ad_user_data 與 ad_personalization 兩個同意旗標，廣告再行銷清單必須在這兩個旗標都為 granted 時才累積。2024 年 3 月後 EEA 流量強制 v2。
(02)

GTM 設定三步曲

步驟 1：容器設定 → 開啟 Consent Overview（管理員 → 容器設定 → 啟用同意設定覆寫）。這會讓每個 Tag 都多出一個「Consent Settings」分頁。

步驟 2：建一個 Custom HTML Tag，類型「Consent Initialization」，在容器其他 Tag 之前先跑：

<script> window.dataLayer = window.dataLayer || []; function gtag(){dataLayer.push(arguments);} // 預設全部 denied（歐盟合規的保守預設） gtag('consent', 'default', { ad_storage: 'denied', ad_user_data: 'denied', // v2 新增 ad_personalization: 'denied', // v2 新增 analytics_storage: 'denied', functionality_storage: 'granted', security_storage: 'granted', wait_for_update: 500 }); </script>

觸發：Consent Initialization - All Pages（GTM 提供的特殊觸發條件，保證跑在最前面）。

步驟 3：用戶按「接受」時更新 state。這段通常寫在 Cookie Banner 的「接受按鈕」onclick：

gtag('consent', 'update', { ad_storage: 'granted', ad_user_data: 'granted', ad_personalization: 'granted', analytics_storage: 'granted' });
(03)

不想自己寫？用 CMP

市面上有現成的 Consent Management Platform 可以直接整合 GTM：

CMP價位特色
CookieYes免費 / $10/月起台灣中小企業常用，介面中文友善
Cookiebot$11/月起Google 認證 CMP，合規最嚴
OneTrust企業級大企業、跨國公司首選
Iubenda€27/年歐洲來源，對 GDPR 文案細節最精

它們都會提供一段 GTM 相容的 Template 或 Consent Initialization 範本，複製貼上即可。唯一要注意：CMP 提供的 default / update 一定要包含 v2 的三個新旗標，沒有就是舊版，要找客服升級。

(04)

Tag 要不要等同意？

每個 Tag 的 Consent Settings 要選對：

  • 不需要額外同意（No additional consent required）— 必要性 Tag，如 Consent Initialization 自己、網站核心功能
  • 需要額外同意（Require additional consent for tag to fire）— 行銷/分析 Tag。選對應的 consent type：
    • GA4 → analytics_storage
    • Google Ads Conversion → ad_storage + ad_user_data + ad_personalization
    • Meta Pixel → ad_storage（GTM 沒直接整合 Meta，要在 Custom HTML 裡自己判斷 dataLayer 中 consent 狀態）
⚠
常見漏洞：只在 GA4/Google Ads Tag 設 consent，忘了在 Meta Pixel 的 Custom HTML 也要檢查。拒絕的用戶照樣被 Pixel 抓，直接違反 GDPR —— 最高罰全球營業額 4%。做跨境生意請嚴格。
◆
台灣內需業主怎麼辦？目前台灣個資法沒有明確規範 cookie 同意，理論上不裝也沒事。但 2024/Q4 個資會草案已提議引入類似 GDPR 的 cookie consent 條款。提前做比較保險，且政府案招標規格書越來越常寫「需支援 Consent Mode v2」。
