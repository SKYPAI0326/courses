4.2

拯救雜亂報表：運用 Lookup Table (查表變數)

當你追蹤了網站上十幾個不同的表單送出時，GA4 報表裡的「表單網址」可能會非常雜亂難讀。這節課教你如何用一個簡單的「翻譯對照表」，讓所有數據變得清爽易讀。

學完這節，你會
理解 Lookup Table (對照表) 的運作概念
在 GTM 中建立並設定查表變數
將長串網址自動轉換為好讀的中文名稱
(01)

為什麼需要對照表？

假設你的網站有三個不同的表單頁面：聯絡我們、報價申請、售後維修。如果你直接把 {{Page Path}} 當作參數送進 GA4，報表裡會出現：

  • /contact-us-2026-v2/
  • /request-quote-main/
  • /support-center/

這些網址不僅難看，如果哪天網址改了，報表就會斷掉。對照表 (Lookup Table) 就是為了解決這個問題而生的。它能自動擔任翻譯官，當 GTM 看到 A 網址，就自動把它翻譯成 B 名字。

(02)

設定 Lookup Table

🛠️
操作步驟：
1. 在 GTM 新增一個「使用者定義的變數」，變數類型選擇 「對照表 (Lookup Table)」。
2. 輸入變數（Input Variable）欄位：點右側的樂高積木圖示，選擇 {{Page Path}}。這代表「以當前頁面路徑作為查表依據」。
3. 點擊下方的「新增列」，每按一次新增一組對照。每列有兩個欄位：
   ・左欄「輸入值 (Input)」：填入頁面路徑，例如 /contact/
   ・右欄「輸出值 (Output)」：填入對應的可讀名稱，例如 一般聯絡表單
4. 依照下表重複新增，直到所有頁面都建立對照。
輸入 (當 Page Path 是...) 輸出 (就把這個變數的值變成...)
/contact/ 一般聯絡表單
/request-quote/ 報價申請
/support/ 售後服務維修

所有列填完後，勾選下方的「設定預設值 (Set Default Value)」，填入 「其他表單」。若不勾選，當使用者從不在清單上的頁面送出表單，這個變數的值會是空白，GA4 報表就會出現空值列。建議一律勾選並填入備用名稱。

將這個變數命名為 Lookup - 表單名稱（左上角欄位），然後儲存。

(03)

讓報表煥然一新

現在，回到我們上一節建立的「表單送出 GA4 事件代碼」。

在事件參數裡新增一個參數名為 form_name，右邊的參數值請選擇剛剛建好的 {{Lookup - 表單名稱}} 變數。

✨
魔法發生了：
從今以後，不論使用者在哪個頁面送出表單，你的 GA4 報表裡只會出現乾淨俐落的「一般聯絡表單」、「報價申請」等中文標籤。如果網址變了，你只需要在 GTM 修改對照表，完全不用動代碼！
