4.3

確保萬無一失：
預覽模式與版本發布守則

在 GTM 中，如果設定錯誤，錯誤的數據就會直接灌進你的 GA4 報表裡，而且「無法修改或刪除」。因此，在按下那個藍色的提交按鈕前，你必須經過嚴格的測試與紀錄。

學完這節，你會
熟練運用 Tag Assistant 判斷「未觸發」的原因
了解「環境與版本管理」的概念
掌握專業行銷人的 GTM 發布 SOP
Section 01

預覽模式是你的沙盒

每次在 GTM 做完任何修改（哪怕只是改了一個變數的拼字），都應該點擊右上角的 「預覽 (Preview)」。

在 Tag Assistant 中，最實用的排錯技巧是點開 Tags Not Fired (未觸發的代碼)：

  • 在左側時間軸點擊你執行操作的那個事件（例如「Click」）。
  • 右側面板切換到 「Tags」分頁，往下拉可以看到「Tags Fired（已觸發）」與「Tags Not Fired（未觸發）」兩個區塊。
  • 點開 Tags Not Fired 裡的代碼，再展開「Firing Triggers（觸發條件）」區塊。
  • GTM 會用一個紅色的叉叉 (❌) 告訴你：「是哪一個過濾條件沒有被滿足」。
  • 例如：你設定 Click Text 要等於「馬上購買」，但網頁上的字其實是「立即購買」，GTM 就會在這裡明確指出錯誤。
Section 02

版本管理的救命恩人

確認測試都沒問題後，回到 GTM 點擊右上角的藍色 「提交 (Submit)」 按鈕。此時會彈出一個要求你填寫「版本名稱」與「說明」的視窗。

⚠️
絕對不要空白！
請養成寫 Release Note 的好習慣。例如：
版本名稱： v1.2 - 新增 SureForms 追蹤與對照表
說明： 1. 新增 generate_lead 事件 2. 加入 Lookup Table 處理表單網址

為什麼這很重要？因為如果明天你發現網站追蹤整個掛掉了，你可以進到 GTM 的「版本」分頁，一鍵 「還原 (Restore)」 到昨天正常的版本。如果你沒寫名稱，你根本不知道該還原到哪一個版本！

Section 03

結語：你的專屬追蹤流

恭喜你完成了所有課程的核心知識點！現在你已經掌握了：

1. 工具運用： Chrome 檢查工具、Tag Assistant
2. 基礎追蹤： 頁面瀏覽、按鈕點擊 (CSS Selector)
3. 進階轉換： AJAX 表單可見度、外部連結、電話點擊
4. 數據整理： Data Layer 概念、Lookup Table 翻譯對照表
