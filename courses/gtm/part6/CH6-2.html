PART 5 · 電商追蹤
5.2

WooCommerce：最常踩到的電商戰場

台灣中小電商大宗跑在 WordPress + WooCommerce 上。PHP hook、動態 JavaScript、結帳頁 AJAX —— 這堂把 GA4 Ecommerce 四個關鍵事件的實作完整拆給你。

學完這節，你會
用「GTM4WP」外掛 5 分鐘接好 dataLayer
自己寫 PHP hook 控制 view_item / purchase 輸出
搞定 add_to_cart（AJAX）與 begin_checkout（SPA）的觸發
(01)

最快路線：GTM4WP 外掛

由 Thomas Geiger 開發的免費外掛，WooCommerce 官方生態圈最成熟的 GTM 整合。裝好後自動產生合規的 GA4 dataLayer。

  1. WP 後台 → 外掛 → 搜尋 「GTM4WP」 → 安裝啟用
  2. 設定 → 填入 GTM Container ID（GTM-XXXXXX）→ 勾 「放進 <body> 第一個子元素」（這是 Google 建議位置）
  3. 切到 Integration → WooCommerce 分頁 → 勾「Track classic e-commerce」跟「Track enhanced e-commerce」→ 選 「GA4 events」（不要用舊 UA 格式）
  4. 儲存後，去商品頁打開 Console 打 dataLayer，應該看到 view_item 跟 ecommerce.items。有就成功
ℹ
GTM4WP 自動包辦：view_item_list（分類頁）、select_item（點擊清單）、view_item（商品頁）、add_to_cart（加購）、remove_from_cart、begin_checkout、purchase。台幣商店記得 WooCommerce 貨幣選 TWD，外掛會直接帶進 currency。
(02)

客製場景：手寫 PHP hook

外掛無法滿足時（例如要加非標準欄位、公司 SKU 與 WooCommerce product ID 不一致），用 WooCommerce 提供的 action hook 自己輸出 dataLayer：

// functions.php 或自訂 plugin add_action('woocommerce_after_single_product', function(){ global $product; if(!$product) return; ?> <script> window.dataLayer = window.dataLayer || []; dataLayer.push({ ecommerce: null }); // 清上一筆，避免污染 dataLayer.push({ event: 'view_item', ecommerce: { currency: '<?= get_woocommerce_currency() ?>', value: <?= $product->get_price() ?>, items: [{ item_id: '<?= $product->get_sku() ?>', item_name: '<?= esc_js($product->get_name()) ?>', price: <?= $product->get_price() ?>, quantity: 1 }] } }); </script> <?php });
⚠
先 push ecommerce: null 再 push 新事件 —— 這是 GA4 官方建議。WooCommerce 結帳頁會經過多個步驟，如果不清除，前一筆 items 會殘留到下一個事件。忘了這一步，報表會出現買了一堆但金額為 0 的幽靈訂單。
(03)

AJAX 加購：cart hook 接手

WooCommerce 大多數主題的「加入購物車」按鈕是 AJAX 觸發，頁面不重新載入。GTM4WP 會自動接 added_to_cart 的 jQuery 事件。如果自己寫，可以監聽：

// 放在 WP footer.php 或 child theme 的 JS 檔 jQuery(document.body).on('added_to_cart', function(evt, fragments, cart_hash, btn){ var pid = btn.data('product_id'), name = btn.data('product_name') || btn.closest('.product').find('.woocommerce-loop-product__title').text(), price = parseFloat(btn.data('product_price')) || 0; dataLayer.push({ ecommerce: null }); dataLayer.push({ event: 'add_to_cart', ecommerce: { currency: 'TWD', value: price, items: [{ item_id: pid, item_name: name, price: price, quantity: 1 }] } }); });

Quick-add 按鈕的 data-product_* 屬性是 WooCommerce 核心原生提供；如果主題把它們去掉了，改用 AJAX response 的 fragments 去抓。

(04)

purchase：訂單完成才能信任

thank-you 頁（order-received/）是唯一安全的 purchase 觸發點。用 woocommerce_thankyou hook 拿到訂單物件再輸出：

add_action('woocommerce_thankyou', function($order_id){ $order = wc_get_order($order_id); if(!$order) return; $items = []; foreach($order->get_items() as $item){ $product = $item->get_product(); $items[] = [ 'item_id' => $product->get_sku() ?: $product->get_id(), 'item_name' => $item->get_name(), 'price' => (float)$order->get_item_subtotal($item, false), 'quantity' => $item->get_quantity(), ]; } $payload = [ 'event' => 'purchase', 'ecommerce' => [ 'transaction_id' => $order->get_order_number(), 'currency' => $order->get_currency(), 'value' => (float)$order->get_subtotal(), // 不含稅運 'tax' => (float)$order->get_total_tax(), 'shipping' => (float)$order->get_shipping_total(), 'items' => $items, ] ]; echo '<script>dataLayer.push({ecommerce:null});dataLayer.push('.wp_json_encode($payload).');</script>'; });
◆
踩雷常見：(1) 用戶重新整理 thank-you → 事件重送，transaction_id 讓 GA4 自動去重；(2) 用 LINE Pay / 綠界的「跳轉付款」流程，thank-you 可能沒真的被訪問 → 改用 woocommerce_order_status_completed 搭配 Webhook 進 GA4 MP API；(3) Meta Pixel 與 CAPI 的 event_id 統一用 order_number。
