PART 5 · 電商追蹤
5.1

GA4 Ecommerce：items array 才是核心

GA4 電商追蹤的精髓全在 items array —— 產品名、分類、價格、數量、折扣、幣別都綁在一起送。只要這一層結構對了，後面所有報表、歸因、再行銷受眾才能漂亮地跑。

學完這節，你會
背熟 GA4 Ecommerce 10 個標準事件與觸發時機
寫出符合規範的 items array（必填 vs 選填、命名規則）
判斷 currency、value、transaction_id 的正確用法
(01)

10 個標準事件對應漏斗

GA4 Ecommerce 事件是官方固定的名稱與欄位，不要自己發明。按漏斗順序：

事件觸發時機是否必備
view_item_list分類頁、搜尋結果頁載入選
select_item從清單點進商品選
view_item商品頁載入必
add_to_cart加入購物車必
view_cart購物車頁載入選
remove_from_cart從購物車移除選
begin_checkout進入結帳流程必
add_payment_info填完付款方式選
add_shipping_info填完寄送地址選
purchase訂單成立（thank-you 頁）必
✓
最小可行集（3 必 +1 加碼）：view_item / add_to_cart / purchase + begin_checkout。這四個能撐起 80% 的電商報表。有預算再往上加 view_item_list、add_payment_info 提升 funnel 解析度。
(02)

items array 標準欄位

所有 Ecommerce 事件的主體都是 items（array of object），每個 object 代表一個商品：

dataLayer.push({ event: 'view_item', ecommerce: { currency: 'TWD', value: 1280, items: [{ item_id: 'SKU-A001', // 必填，唯一碼 item_name: '經典白T', // 必填 item_brand: 'MUJI style', item_category: '服飾', item_category2: '上衣', item_variant: '白色-M', price: 1280, quantity: 1, discount: 0, index: 0 // 在 list 中的位置，view_item_list/select_item 需要 }] } });
欄位規則
item_id必填。SKU，唯一且穩定。用資料庫主鍵而不是商品名
item_name必填。與網站顯示一致，避免用內部代號
price單價，數字不帶幣符。用打折後價格（若 discount 單獨送則用原價）
quantity商品頁通常為 1；add_to_cart / purchase 填實際購買數量
currencyISO 4217 三碼，TWD / USD / JPY。與 value 一起在 ecommerce 外層
value這個事件的總金額 = items 中 price × quantity 的總和（purchase 時不含運費稅）
(03)

purchase 的關鍵欄位

購買事件比其他事件多四個必備欄位：

dataLayer.push({ event: 'purchase', ecommerce: { transaction_id: 'ORD-20260418-00021', // 必填，唯一 currency: 'TWD', value: 2580, // 商品總額（不含稅費） tax: 129, shipping: 80, coupon: 'SPRING10', items: [ /* ... */ ] } });
⚠
transaction_id 是 GA4 判斷「重複購買事件」的唯一鍵。用戶重新整理 thank-you 頁、網速慢按兩次下單、被導回網站，事件都可能重送。有 transaction_id，GA4 會自動去重；沒有就每次都計 1 單。訂單編號直接用，不要自己組字串。
◆
value 的陷阱：value = sum(items.price × items.quantity)。寫 dataLayer 時不要把 tax、shipping 加進 value，GA4 會重複計算。tax 跟 shipping 獨立欄位送就好。ROAS 報表才能分開看淨商品銷售 vs 總營收。
(04)

GTM 端怎麼取

GA4 Tag 預設就認得 ecommerce 物件。設定很簡單：

  • Tag 類型 → Google Analytics: GA4 Event
  • Event Name → {{Event}}（用內建 Event 變數自動帶入 view_item / purchase）
  • 下方 More Settings → Ecommerce → Send Ecommerce data 勾起來，Data Source 選 Data Layer
  • Trigger → Custom Event，Event name 填一個 regex ^(view_item|add_to_cart|begin_checkout|purchase)$ 一網打盡

這樣一個 Tag 就能處理全部 Ecommerce 事件。不要為每個事件建一個 Tag，那是維護災難。
