演練 2

三平台轉換同時回報驗證

終極挑戰：一筆 purchase 事件同時送到 GA4、Meta Pixel、Google Ads 三個平台，每個平台都要能正確收到、金額一致、不重複計算。這是廣告操盤手最常拿來考人的情境題。

這個演練你會
跑完 15 項 checklist，驗證三平台的 purchase 事件
用三款 Helper 擴充並排比對結果
產出一份「轉換驗收報告」可交付給客戶
Step 1

情境與dataLayer 範本

假設你在 thank-you 頁用了這段前端程式：

// thank-you.html const tx = 'TX20260418001'; dataLayer.push({ event: 'purchase', transaction_id: tx, value: 2990, currency: 'TWD', items: [{item_id: 'SKU-A01', item_name: '年費會籍', price: 2990, quantity: 1}] }); fbq('track', 'Purchase', {value: 2990, currency: 'TWD'}, {eventID: tx});

GTM 裡應該有三個轉換 Tag 都綁到 Custom Event: purchase：GA4 Event (purchase)、Meta Pixel (透過 dataLayer 的 Custom HTML)、Google Ads Conversion。

Step 2

15 項驗收 Checklist

  • GTM Preview 面板看到 purchase 事件觸發，左側列表有 3 個 Tag 全部 Fired（GA4 / Meta / Google Ads）
  • GTM Preview Data Layer 面板看到 transaction_id、value、currency、items 四個欄位齊全
  • GA4 DebugView（Google Analytics 介面 → 設定 → DebugView）看到 purchase 事件，參數展開有 value: 2990、currency: TWD、items 陣列
  • GA4 即時報表「轉換事件」列表出現 purchase、事件數 = 1
  • Meta Pixel Helper（Chrome 擴充）顯示綠色 Purchase、value=2990、currency=TWD、eventID 存在
  • Meta 事件管理工具 Test Events 頁籤看到 Purchase，Event Match Quality 顯示分數（≥ 5.0 合格）
  • Meta 事件管理工具 診斷頁籤無黃色/紅色警告
  • Tag Assistant Legacy（Chrome 擴充）看到 Google Ads Conversion Tag 綠色勾勾、Conversion ID 正確
  • Google Ads「轉換動作」狀態為「記錄中」（不是「驗證中」）
  • Google Ads「轉換動作」→ 診斷：近 7 天至少有 1 筆測試轉換被記錄
  • dedup 驗證：連續刷新 thank-you 頁 3 次，GA4/Meta/Google Ads 三邊的 purchase 事件 count 都要等於 1（因為 transaction_id / eventID / order_id 相同）
  • 金額一致性：GA4 report 的 Revenue = Meta 的 Purchase value total = Google Ads 的 Conv. value
  • 沒有額外的 Purchase：不該有 Meta 標準事件 Subscribe、Google Ads 額外轉換動作被觸發
  • Enhanced Conversions / CAPI：若有開啟，Meta EMQ ≥ 7、Google Ads Enhanced Conversion 比對率 ≥ 70%
  • Cookie 同意：在 Consent Banner 拒絕追蹤時，三個 Tag 都 不應該觸發（Consent Mode v2 運作中）
⚠
15 項全過才算驗收完成。任何一項沒過，到對應的課程章節去找答案：3、4 項不過 → CH2-1；5-7 → CH4-1/4-2；8-10 → CH4-3；11-12 → CH4-3 的 Order ID；13 → 事件規劃表重審；15 → CH4-4 Consent Mode。
Step 3

產出驗收報告交付客戶

完成後整理成 1 頁 PDF：

# 轉換追蹤驗收報告 【網站】 fitgym.com.tw 【驗收日期】 2026-04-18 【執行人員】 {你的名字} ## 一、追蹤平台清單 - GA4 Property: 123456789 - Meta Pixel: 987654321012345 - Google Ads: AW-1234567890 ## 二、驗收結果 | 項目 | GA4 | Meta | Google Ads | |---|---|---|---| | 基底碼 | ✅ | ✅ | ✅ | | Purchase 事件 | ✅ | ✅ | ✅ | | 金額參數 | ✅ 2990 | ✅ 2990 | ✅ 2990 | | Dedup 驗證 | ✅ | ✅ eventID | ✅ Order ID | | CAPI / Enhanced | — | ✅ EMQ 8.2 | ✅ 82% | ## 三、截圖存檔 （附 DebugView、Pixel Helper、Tag Assistant 各一張）
💡
這份報告的價值：客戶會用它跟後續來接手的廣告代理商溝通；你會用它在月報證明「我埋的碼沒壞，是廣告投手的事」。爭議時它就是你的免死金牌。
