PART 4 · 廣告像素與轉換回報
4.1

Meta Pixel：把 FB/IG 廣告的眼睛裝上你的網站

GTM 對廣告操盤手最大的價值，就是 5 分鐘裝好 Meta Pixel、5 分鐘掛完 9 個標準事件、5 分鐘把購買金額動態帶進去。這一節我們一次做完。

學完這節，你會
在 GTM 用 Custom HTML 部署 Meta Pixel 基底碼
熟記 9 個 Meta 標準事件與漏斗階段對應
用 dataLayer 把 purchase 的 value / currency / content_ids 動態帶給 Pixel
(01)

先取得Pixel ID

  • 進入 Meta 事件管理工具（business.facebook.com/events_manager）
  • 建立新資料來源 → 選「網站」→ 選「Meta Pixel」→ 命名（建議格式：{品牌}_web_pixel）
  • 複製 16 位數的 Pixel ID（如 1234567890123456）
  • 不要選「直接加入代碼」，也不要用 Meta 的「合作夥伴整合 → GTM」自動安裝 —— 那個安裝法之後改事件會綁死，一定要在 GTM 裡自己建
(02)

GTM 部署基底碼（PageView）

在 GTM 新增 Tag，類型選「Custom HTML」，內容貼上：

<script> !function(f,b,e,v,n,t,s){if(f.fbq)return;n=f.fbq=function(){n.callMethod? n.callMethod.apply(n,arguments):n.queue.push(arguments)};if(!f._fbq)f._fbq=n; n.push=n;n.loaded=!0;n.version='2.0';n.queue=[];t=b.createElement(e);t.async=!0; t.src=v;s=b.getElementsByTagName(e)[0];s.parentNode.insertBefore(t,s)}(window, document,'script','https://connect.facebook.net/en_US/fbevents.js'); fbq('init', '{{FB_PIXEL_ID}}'); fbq('track', 'PageView'); </script>

觸發條件：All Pages。把 Pixel ID 抽成 GTM 常數變數 {{FB_PIXEL_ID}}，之後其他事件 Tag 都引用同一個變數，換客戶時只改一處。

💡
驗證：裝 Chrome 擴充「Meta Pixel Helper」，進你的網站，點擴充圖示應該看到綠色勾勾 + PageView 事件。沒看到？先檢查 GTM Preview 裡 Tag 有沒有開槍，再檢查 Pixel ID 有沒有填錯。
(03)

9 個標準事件對應表

Meta 有預定義的「標準事件」清單，照著用，演算法才有辦法優化。自訂名稱（Custom Event）演算法學得慢。對應漏斗如下：

漏斗Meta 事件什麼時候開槍關鍵參數
TOFUPageView全站—
MOFUViewContent商品/服務頁content_ids, value
MOFUSearch站內搜尋search_string
BOFUAddToCart加入購物車content_ids, value, currency
BOFUInitiateCheckout進結帳頁 / 開表單value, currency
BOFUAddPaymentInfo填信用卡value
CONVLead表單送出成功value (選填)
CONVCompleteRegistration註冊會員value
CONVPurchase結帳完成value, currency, content_ids
(04)

動態參數：把購買金額帶進去

Purchase 事件如果沒帶 value，Meta 廣告看到「有轉換但不知道值多少」，ROAS 算不出來。正確做法是讓前端在 thank-you 頁 push dataLayer：

// 結帳成功頁（thank-you.html）前端 dataLayer.push({ event: 'purchase', transaction_id: 'TX20260418001', value: 2990, currency: 'TWD', content_ids: ['SKU-A01', 'SKU-B07'] });

GTM 建 3 個 Data Layer Variables（value / currency / content_ids），建一個 Custom Event 觸發「purchase」，Custom HTML Tag：

<script> fbq('track', 'Purchase', { value: {{DLV - value}}, currency: '{{DLV - currency}}', content_ids: {{DLV - content_ids}}, content_type: 'product' }); </script>
⚠
常見錯誤：value 一定要是數字不能是字串，currency 用 ISO 4217 三碼（TWD / USD / JPY），content_ids 一定要是陣列即使只有一個。錯一個 Meta 都會拒絕這筆轉換。
