4.4

LINE LAP + TikTok Pixel：
兩個在地戰場

LINE 是台灣業主一定會問的，TikTok 是成長最快的新流量池。這一節不重複 Meta/Google 的細節，只講兩個平台獨有的設定與坑。

學完這節，你會
用 GTM 部署 LINE Tag 與 TikTok Pixel 基底碼
掌握 LINE Cross Tag（跨網站再行銷）的串接點
了解 TikTok Events API（CAPI 版）的實作路徑
Section 01

LINE LAP：基底碼

LAP（LINE Ads Platform）後台 → 轉換 → 建立 LINE Tag。會拿到一個 Tag ID，例如 abc123def。GTM 部署範本：

<!-- Custom HTML Tag，觸發：All Pages --> <script> (function(g,d,o){ g._ltq=g._ltq||[];g._lt=g._lt||function(){g._ltq.push(arguments)}; var h=location.protocol==='https:'?'https://d.line-scdn.net':'http://d.line-cdn.net'; var s=d.createElement('script');s.async=1; s.src=h+'/n/line_tag/public/release/v1/lt.js'; var t=d.getElementsByTagName('script')[0];t.parentNode.insertBefore(s,t); })(window,document); _lt('init', {customerType: 'lap', tagId: '{{LINE_TAG_ID}}'}); _lt('send', 'pv', ['{{LINE_TAG_ID}}']); </script>

轉換事件（以 Purchase 為例）新建一個 Custom HTML Tag：

<script> _lt('send', 'cv', { type: 'Conversion', args: {value: {{DLV - value}}, currency: 'TWD'} }, ['{{LINE_TAG_ID}}']); </script>
Section 02

LINE 獨有的Cross Tag

LINE 後台的「Cross Tag」是跨網站再行銷用的 —— 如果業主同時經營兩個品牌站，可以建 Cross Tag 把兩站的流量歸併成同一個 audience pool。設定時在 GTM 多一個 Custom HTML Tag，呼叫：

_lt('send', 'ct', {argsName: 'cross_tag'}, ['{{LINE_TAG_ID}}']);
ℹ
LINE 廣告與 Meta/Google 最大的差別：LINE 官方帳號本身是流量來源。一個有 LINE OA 的業主，真正的歸因應該是「LINE 廣告 → 加入好友 → 在 OA 內消費」，這段 OA 內的轉換要靠 LINE 官方帳號後台的 API 回傳，GTM 管不到。這一塊屬於進階議題。
Section 03

Section 04

TikTok 的CAPI 版：Events API

TikTok 也有類似 Meta CAPI 的「Events API」，從後端直送事件。2026 年已是必備：

  • 端點：POST https://business-api.tiktok.com/open_api/v1.3/event/track/
  • 需要 Access-Token（TikTok 開發者後台申請）
  • 用 event_id 做 dedup（跟 Meta CAPI 同思路）
  • Events API 的 Event Match Quality 分數 > 70 才算健康
⚠
2024 後 TikTok 廣告成效開始重度依賴 CAPI 資料，只裝 Pixel 的帳戶 CPA 會比同行貴 30-50%。真要投 TikTok 必補 Events API。
