PART 3 · 進階互動與轉換追蹤
3.2

流量去哪了？
追蹤外部連結與撥打電話

當使用者點擊了連往 FB 粉專、Line 客服或是直接用手機撥打電話時，他們就「離開」了你的網站。本節教你如何設定過濾條件，精準捕捉這些「導外點擊」。

學完這節，你會
建立「僅限連結」的觸發條件
掌握排除自家網址的過濾技巧（追蹤導外）
運用 `tel:` 特徵追蹤手機撥打電話行為
Section 01

追蹤外部連結 (Outbound Click)

如果你的網站有很多導購連結或是社群媒體連結，追蹤這些點擊能幫你評估：哪些外部渠道最受歡迎？

01
建立「僅限連結」觸發條件
新增觸發條件，類型選擇 「點擊 - 僅限連結 (Just Links)」。預設是「所有點擊」，將它改為「部分點擊 (Some Clicks)」，畫面出現條件列後點擊「新增條件」。
02
設定過濾邏輯
左側下拉選擇 Click URL，中間條件選擇 「不包含 (doesn't contain)」，右側填入你的本機網站網域（例如：localwp.me）。
💡
為什麼用「不包含」？
因為只要點擊的連結網址裡沒有出現你自家的網域名稱，就代表這個連結是通往外部的。這是一個非常聰明且省力、能一次抓到所有外部連結的設定方式。
ℹ
GTM 條件邏輯速查：何時用哪個？

包含 (contains)：URL 或文字中某個片段存在時。例：Click URL 包含 facebook.com
不包含 (doesn't contain)：排除特定片段。例：排除自家網域
等於 (equals)：完整字串完全一致時。例：Page Path 等於 /thank-you
開頭為 (starts with)：URL 或值的開頭固定時。例：Click URL 開頭為 tel:
符合 CSS 選擇器 (matches CSS selector)：比對 HTML 元素的 class 或 id。例：Click Element 符合 .promo-btn a

外部連結 + 電話各建一個觸發條件，再各自對應一個 GA4 事件代碼，不要合併在同一個觸發條件裡，這樣事件命名才能清楚區分。
Section 02

追蹤撥打電話 (Click to Call)

很多服務型網站會在頁首放上電話號碼，並加上 href="tel:0912..." 的連結，讓手機用戶點擊就能直接撥打。這也是一個非常高價值的轉換動作。

01
建立電話觸發條件
一樣選擇 「點擊 - 僅限連結 (Just Links)」，預設改為「部分點擊」後點擊「新增條件」。
02
鎖定 tel: 特徵
左側下拉選擇 Click URL，中間條件選擇 「開頭為 (starts with)」，右側填入 tel:。儲存！
ℹ
發送事件的建議：
在建立對應的 GA4 事件代碼時，建議將事件名稱取為 click_phone_number。你也可以新增一個參數 phone_number，並帶入 {{Click URL}} 變數，這樣 GA4 就能紀錄使用者撥打的號碼（值會包含 tel: 前綴，例如 tel:+886912345678，屬正常現象）。
💡
讓參數出現在 GA4 報表的必要步驟：自訂維度
GTM 送出的事件參數，GA4 不會自動開放在報表中查詢。你必須手動登錄，才能在「探索」報表裡將 phone_number 當作維度使用：

GA4 → 管理 → 自訂定義 → 建立自訂維度
・維度名稱：phone_number（自訂顯示名稱）
・範圍：事件
・事件參數：phone_number（須與 GTM 參數名稱完全一致）

儲存後約 24 小時，資料才會開始在報表中可見。若要立即確認參數值是否正確傳送，可使用 GA4 → 管理 → DebugView，GTM 預覽模式下的事件會即時顯示在此，包含所有參數內容。
