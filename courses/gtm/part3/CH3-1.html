PART 3 · 進階互動與轉換追蹤
3.1

獲取名單：
追蹤表單送出與元素可見度

對行銷人來說，「填寫聯絡表單」往往是比點擊按鈕更重要的轉換指標。然而現代網頁的表單送出後經常「不換頁」，這會讓傳統追蹤方式失效。本節教你如何見招拆招。

學完這節，你會
理解 AJAX 表單（不換頁表單）的特性
設定「元素可見度」來追蹤成功訊息
將轉換資料記錄為 GA4 的 generate_lead 事件
Section 01

現代表單的追蹤難題

在你的測試網站中，我們使用的是 SureForms 這個現代化的表單外掛。你可以試著去網站前台填寫並送出一次表單：

你會發現，送出後網址沒有改變，也沒有跳轉到傳統的「感謝頁 (Thank You Page)」，而是在按鈕下方直接浮現一行綠色的「感謝您的填寫」。這種不換頁的設計稱為 AJAX 表單，會讓依賴「網址跳轉」的傳統追蹤方式完全無效。

Section 02

解法：元素可見度追蹤

既然網址沒變，那我們就讓 GTM 去「監視」那段綠色的感謝訊息！只要這段訊息一出現在畫面上，就代表表單成功送出了。

01
尋找感謝訊息的特徵
送出表單後，用 Chrome 右鍵點擊感謝訊息文字 →「檢查 (Inspect)」。右側開發者工具會直接定位到該文字所在的 HTML 標籤，往外看幾層，找到最靠近這段文字、且帶有明確 class 的那個標籤，記下它的 class 名稱。在我們的測試網站中，實際的 class 是 .srfm-success-box。
注意：class 名稱因環境而異
這個 class 會隨使用的表單外掛或佈景主題不同而改變。SureForms 用的是 srfm- 前綴；其他常見的有 .wpcf7-response-output（Contact Form 7）、.gform_confirmation_message（Gravity Forms）等。每次實作前請務必用「檢查」親自確認，不要直接套用範例 class。
02
建立「元素可見度」觸發條件
在 GTM 新增觸發條件，選擇 「元素可見度 (Element Visibility)」。選取方法設定為 「CSS 選擇器」，並填入剛剛找到的 class。
03
設定啟動時機
將「何時觸發此觸發條件」設定為 「每個網頁一次」，並勾選「觀察 DOM 變更 (Observe DOM changes)」。儲存！

為什麼必須勾選這個？AJAX 表單的感謝訊息是在送出後才由 JavaScript 動態插入頁面，GTM 預設在頁面載入時掃描一次元素，此時訊息還不存在。勾選「觀察 DOM 變更」後，GTM 會持續監聽頁面結構的任何變動，一旦感謝訊息出現就立即觸發。若不勾選，這個觸發條件永遠不會啟動。
Section 03

建立 GA4 事件代碼

最後，建立一個對應的代碼：

  • 代碼類型：GA4 事件。
  • 事件名稱：建議填入 generate_lead（這是 GA4 預設的潛在客戶開發事件名稱）。
  • 觸發條件：選擇剛剛建立的 「元素可見度 - 表單成功訊息」。
💡
進階備註：有些外掛支援直接推送資料層 (Data Layer) 事件，我們會在 PART 4 介紹這個更高級的作法。但「元素可見度」是你面對任何未知表單都能使用的萬能解法。
★
完成實作！
現在，當有潛在客戶在你的網站留言時，GA4 就會立刻記錄下一筆高價值的轉換數據。下一節，我們來學習如何追蹤那些「點擊後會離開網站」的連結與電話號碼。
