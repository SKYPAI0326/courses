PART 3 · 進階互動與自訂事件
3.4

dataLayer.push：讓 GTM 接得住任何前端事件

React / Vue 的 SPA、AJAX 送表單、Shopify 動態結帳，這些場景 GTM 內建觸發條件都抓不到。解法是讓前端「主動通知」GTM。這一節我們把 dataLayer 從黑盒打開。

學完這節，你會
寫出符合 GA4 推薦格式的 dataLayer.push
在 GTM 建立對應的 Custom Event 觸發條件 + Data Layer Variable
排除 SPA route 切換抓不到 page_view 的經典問題
(01)

什麼時候非用 dataLayer.push 不可

GTM 內建觸發條件（Click、Form Submit、Element Visibility）解決得了 80% 場景，但以下情境它幫不了你：

  • SPA 路由切換：React/Vue 用 history.pushState 切頁，瀏覽器不重載，GTM 的 Page View 觸發不會再開槍
  • AJAX 表單送出：表單用 fetch/XHR 送出，沒有真正的 form submit 事件，GTM Form Submit 觸發抓不到
  • 第三方結帳回跳：綠界 / TapPay / Stripe 結帳完跳回 thank-you 頁，但訂單細節要從前端變數拿
  • 動態載入內容：無限捲動載入的文章、彈窗開啟、影片章節切換
💡
共通特徵：DOM 上沒有對應的「自然」事件可以監聽。解法是讓前端工程師在事件發生的那一刻，主動 push 一筆資料到 GTM 的資料層。
(02)

dataLayer.push 的標準結構

GTM 會監聽全域變數 window.dataLayer，任何人 push 進去的物件，GTM 都能讀到。GA4 推薦的格式長這樣：

// 前端工程師在事件發生時呼叫 window.dataLayer = window.dataLayer || []; window.dataLayer.push({ event: 'generate_lead', // 事件名（對應 GTM 觸發條件） form_id: 'contact_main', form_location: 'footer', value: 1, currency: 'TWD' });

三個要件：① event 鍵必填（GTM 用這個辨識）、② 其他鍵是自訂參數（給 GA4 report 用）、③ 參數命名遵守 1.4 的字典。

⚠
順序超重要：如果前端在 GTM 容器碼載入「之前」就 push，資料會遺失。SPA 場景通常把 dataLayer = [] 的初始化放在 <head> 最上方，早於 GTM snippet。
(03)

GTM 端三步接球

前端 push 出來後，GTM 裡需要配三個元件：

  • 建立 Data Layer Variable：變數類型選「資料層變數」，名稱填 form_id。之後在 GTM 裡可以用 {{DLV - form_id}} 引用
  • 建立 Custom Event 觸發條件：觸發類型選「自訂事件」，事件名稱填 generate_lead（與 dataLayer.push 的 event 值一字不差）
  • 建立 GA4 Event Tag：Event Name 填 generate_lead，Parameters 加入 form_id = {{DLV - form_id}}，觸發器掛上剛建好的 Custom Event
ℹ
驗證：打開 GTM 預覽模式 → 操作前端 → 在 Preview 面板左側應該看到 generate_lead 事件出現，右側 Data Layer 面板能看到你 push 的所有欄位。看不到就是前端沒 push 成功，先去 Console 跑 dataLayer 檢查。
(04)

SPA 經典坑：page_view 只開一次

React/Vue 站最常見 bug：用戶從首頁點到商品頁，網址變了、內容變了，但 GA4 只記錄到一次 page_view。原因：GTM 的 Page View 觸發條件綁在瀏覽器 DOMContentLoaded，SPA 只觸發一次。

解法：攔截 history.pushState，每次路由切換時 push 一筆虛擬 page_view：

// 放在 Vue Router / React Router 的 afterEach hook router.afterEach((to) => { window.dataLayer.push({ event: 'virtual_page_view', page_path: to.fullPath, page_title: document.title }); });

然後在 GTM 建 Custom Event 觸發「virtual_page_view」，綁 GA4 的 page_view 事件 —— GA4 報表就能正確看到每個前端路由的流量。
