演練 5

演練 5：
從訪談到報表交付的完整 SOP

模擬客戶「小美咖啡」—— WooCommerce 電商、月廣告費 15 萬（Meta+Google）、月營收 60 萬、希望三週內上線完整追蹤並交付週報。你是接案工程師或內部數據負責人，完整走過這套 SOP，下次真案子就能照抄。

這個演練會讓你
熟悉五階段交付的產出與時程規劃
寫得出每個階段對客戶要說的話、要簽的確認單
建立自己的交付 checklist，避免漏掉常見項目
Phase 01 · Day 1–2

需求訪談：問對 8 個問題

最多 60 分鐘的會議，帶離現場的必須是「可以開始動工」，不是還要再問一輪。

商業背景（必問）
  1. 現在最關心的商業問題是什麼？（營收？新客？回購？）
  2. 目前怎麼看這個數字？（GA 舊版？Shopify？還是沒看）
  3. 每月廣告預算多少？分哪些渠道？
技術背景（必問）
  1. 網站技術 stack（WooCommerce / Shopify / 自建）？誰能改網站？
  2. 已有的追蹤（GTM 容器？Meta Pixel？GA4？）
  3. 客戶資料管理（LINE 官方？CRM？電子報？）
交付期待（必問）
  1. 什麼時候要上線？（決定優先級）
  2. 誰是最終看報表的人？ 他想看哪些數字？
產出：訪談紀錄 1 頁、初步報價單（按小時 or 固定）、NDA（若需要）
Phase 02 · Day 3–5

埋碼規劃：寫出那張 Tracking Spec

用本課程 1.3 的「漏斗→KPI→事件對應表」方法論，輸出一張 Google Sheet。8 欄：漏斗階段、KPI、事件名、觸發頁、DLV 欄位、優先級、前端是否需動、預計工時。

Phase 03 · Day 6–12

GTM 設定 + 前端埋入：分工與順序

分工：前端工程師做 dataLayer push，你（GTM Specialist）在 GTM 取值 + 建 Tag。兩條線並行，每天同步。

GTM 設定順序
  1. 建容器、放 snippet、確認 Preview 能跑
  2. 建 Consent Initialization Tag（Mode v2 預設 denied）
  3. 建 CMP Template（CookieYes 等）→ update consent
  4. 建 GA4 Config Tag（All Pages，Measurement ID 填入）
  5. 建「GA4 Event - Ecommerce」（Custom Event trigger regex）
  6. 建 Meta Pixel base + standard events（Custom HTML，參考 CH5-1）
  7. 建 Google Ads Conversion Linker + Tag（CH5-3）
  8. 建 LINE LAP Tag（若有 LAP 投放）
  9. 每個事件在 Preview + DebugView + Pixel Helper 三路確認
  10. 提交版本 → 發布到生產環境
產出：GTM 容器版本號 + 發布說明、前端 commit、技術文件（Variable 命名規則、Tag 分類命名）
Phase 04 · Day 13–16

驗收：15 項 checklist 逐條過

參考演練 2 的清單，一項一項打勾。客戶與你兩人共同過，現場看 GA4 DebugView、Meta Test Events、Google Ads Conversion Tracking 三個平台。

必過 15 項
類別項目
GA4page_view 有資料
view_item / add_to_cart / purchase 事件進得去
transaction_id 正確、不重複
items array 完整（item_id / item_name / price / quantity）
MetaPixel 所有事件在 Events Manager 無紅字
CAPI 與 Pixel 的 event_id 對齊、dedup rate > 50%
user_data 有 hash 過的 email / phone
Google AdsConversion Tracking 狀態「已記錄」（Conversion Diagnostics）
Enhanced Conversions 顯示「已啟用」
LINE / TikTok（如有）LAP Pixel 追蹤狀態為正常
TikTok Events Manager 覆蓋率 > 70%
Consent拒絕同意的用戶 GA4 只收 cookieless ping
Meta Pixel 在拒絕同意時不觸發
整體三平台 purchase 數量差異 < 15%（跨平台誤差合理範圍）
測試訂單用 IP 過濾或自訂維度標記排除
產出：驗收報告 PDF、已簽名的 sign-off sheet、設定備份（GTM 匯出 .json）
Phase 05 · Day 17–21

報表交付 + 培訓：讓客戶能自己跑

Looker Studio 五頁模板（Overview / Media / Funnel / Product / Campaign）填上客戶資料源，clone 給客戶當 owner，排程週一早上 9 點寄 PDF 給三個指定收件者。

30 分鐘教育訓練
  • 10 分鐘：儀表板走一次，五張頁面各代表什麼、哪些數字要每週檢視
  • 10 分鐘：什麼訊號代表要行動 — ROAS < 2 要調價 / Funnel 某步流失暴增要查 UX
  • 10 分鐘：客戶自己試著用篩選器看「這週 vs 上週」、「新客 vs 舊客」
產出：Looker Studio 報表（客戶為 owner）、操作手冊、Q&A FAQ、下一步建議清單（可做的優化 5 項）
◆
結案關鍵：最後一封信附「三個月內 30 分鐘免費諮詢」連結，讓客戶有問題願意找你。這是下一個合約的伏筆 — 月維運 / 季度優化 / 活動加碼都是續約機會。沒有這封信，案子結束後客戶很快就忘了。
Section 06

畢業：下一步往哪走

走完這 56 小時，你有能力獨立交付中小電商的追蹤專案。進階方向：

  • Server-side GTM（sGTM）：本課程沒教，但在追蹤生態圈越來越重要。一個 Cloud Run 實例月費 < 500 元，可繞過 iOS 追蹤限制、降低前端 script 負擔
  • BigQuery 離線分析：GA4 免費匯出到 BQ，用 SQL 寫 LTV、歸因路徑報表
  • CDP 整合：把 GA4 受眾 + CRM + 客服記錄打通，做 RFM 分群個人化行銷
  • A/B Testing：接 Google Optimize（已停用）替代方案如 VWO / Kameleoon，搭配 GTM 做變體追蹤
  • 考取證照：Google Analytics Certification（免費）、Google Ads Certification、Meta Blueprint
✓
做作品集：把這五個演練整理成 Notion / GitHub repo 當求職作品。面試台灣數位代理商、in-house growth team、跨境電商都會問實際案例，「我做過一套完整的 GTM 追蹤，這裡是我的 Tracking Spec 跟驗收清單」遠比說「我會 GTM」有說服力。
