PART 7 · 分析與交付
7.1

GA4 分析工具：讓事件長出洞察

埋好 custom event 只是起點。自訂維度讓事件參數在報表中「可被分群」，受眾把行為打包成可投廣告的清單，Exploration 則是真正拿來回答商業問題的地方。

學完這節，你會
把 event params 升級成 custom dimension、讓它出現在報表
建立事件型、漏斗型、使用者屬性型 受眾
用 Exploration 的 Funnel、Path、Segment overlap 找答案
(01)

Custom Dimension：讓參數可以被分群

GA4 的 event 可以帶任意參數（如 article_category、video_length），但這些參數 預設不會出現在報表。必須先註冊成「自訂維度」才能用。

  1. GA4 後台 → 管理 → 自訂定義 → 建立自訂維度
  2. 範圍選 「事件」（一般參數）或 「使用者」（要跟著 user 跨 session 留存，如會員等級）
  3. 維度名稱填顯示名（e.g. 「文章分類」），事件參數填 dataLayer 送的 key（e.g. article_category）
  4. 儲存後 24–48 小時才會有資料，之前送的資料無法回填
⚠
配額限制：Event 範圍 50 個、User 範圍 25 個。不要把所有事件參數都註冊，挑真的會拿來分群的。常見優先順序：content_category、logged_in、member_tier、plan_type、coupon_used。
✓
自訂指標（custom metric）用於數值型參數要做「加總 / 平均」，例如 video_play_seconds。維度是分群用、指標是加總用，別搞混。
(02)

受眾建立：從行為到廣告清單

GA4 受眾（Audience）能直接推給 Google Ads 做再行銷清單。進「管理 → 受眾」→「新增受眾」，三種常見模式：

受眾類型條件範例用途
事件型add_to_cart 次數 ≥ 1 AND purchase 次數 = 0購物車拋棄再行銷
序列型view_item → add_to_cart（30 天內）有意圖但沒加購的高價值流量
預測型預測 7 天內購買機率 > 90 百分位給 PMax、Max Conversion Value 用

存續時間（Membership duration）預設 30 天，最長 540 天。電商再行銷常設 14–30 天；B2B 線索型業務設 90–180 天。

ℹ
要讓受眾被 Google Ads 用，必須先在 GA4「管理 → 產品連結 → Google Ads 連結」接好。LINE LAP、Meta 需要用匯出方式（Audiences → Export to CSV）或 CDP 中介。
(03)

Exploration：回答問題的地方

標準報表是快速瀏覽用，真要做分析用「探索」（Explore）。五種技術常用：

技術回答什麼
自由形式（Free form）Excel 樞紐分析，快速交叉維度與指標
漏斗探索（Funnel）view_item → add_to_cart → purchase 每步流失率
路徑探索（Path）用戶看完首頁後最常去的三個頁面
區隔重疊（Segment overlap）「高消費者 ∩ 手機用戶 ∩ 回訪者」有多少
使用者生命週期（Cohort）1 月新客的 30 天 / 90 天留存
◆
漏斗探索祕技：勾「使用開放漏斗」 → 允許用戶中間可以有其他步驟（現實中用戶很少一路連續走完）。還能加上「區隔」做對照：行動 vs 桌機、新客 vs 舊客的漏斗表現差多少，老闆最愛這種圖。
⚠
資料抽樣：Explore 的事件超過 1,000 萬會觸發抽樣（標準版）、GA4 360 是 10 億。抽樣會導致數字不精確，遇到要改用 BigQuery 匯出做離線分析（GA4 免費版也可以匯出）。
(04)

每週例行五張表

老闆每週想看的通常是這五張，先把自訂 Explore 存成收藏：

  1. 來源媒介 × 營收（Free form：維度=session_source_medium，指標=sessions / conversions / revenue）
  2. 活動 × ROAS（維度=session_campaign，指標=ad_cost / purchase_revenue / ROAS [計算欄位]）
  3. 電商漏斗（Funnel：view_item → add_to_cart → begin_checkout → purchase）
  4. 熱門商品（Free form：維度=item_name，指標=items_purchased / item_revenue）
  5. 新 vs 舊客表現（Free form：維度=new_vs_returning，指標=users / avg_session_duration / purchase_rate）
