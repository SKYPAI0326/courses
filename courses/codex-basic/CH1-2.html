安裝、個人化與權限

這頁結束，你會有一個認得你身份的 Codex——知道你是弄一下工作室的行政會計、偏好繁體中文簡短條列、遇到不確定的事不能自己補資料。

學習成果
能完成 Codex 安裝並填好自訂指令（弄一下工作室行政會計、繁中、簡短、不確定就停）
能說出 :workspace 和 :danger-full-access 的差別，並把權限設在 :workspace
送出 role-based 測試問話後，Codex 用行政會計視角、繁體中文條列回覆
(01)

這頁的任務：讓 Codex 認識你

CH1-1 的示範，你看到 Codex 把一包亂命名的報帳單整理成乾淨的 Excel。那個版本的 Codex 已經被設定好——它知道主人是弄一下工作室的行政會計、偏好繁體中文簡短條列、遇到不確定的事不能自己補資料。

你剛裝好的 Codex 不認識你。給它一張廠商對帳單，問它「下一步怎麼確認？」，它不知道你負責月結、不知道你要正式口吻、不知道你要繁體中文。它只能給一個通用版回答。

本單元明確任務（具體成果）
  • 自訂指令填好並儲存（弄一下工作室行政會計、繁中、簡短、不確定就停）
  • 記憶開關啟用
  • 權限模式設在 :workspace，不是 :danger-full-access

驗收動作：送出一句 role-based 問話，Codex 用行政會計視角、繁體中文簡短條列回覆。CH3-2 你請它草擬對帳催收信的時候，口吻和語氣就是今天這組設定在兌現。

(02)

四個關鍵設定項與權限三層

設定 01
自訂指令
告訴 Codex「你是誰、做什麼、喜歡怎樣的回答」。設定一次，之後每次對話都帶著走。填了「弄一下工作室行政會計、繁體中文、簡短重點式」，它之後不會跟你廢話，也不會突然改英文。
設定 02
記憶
啟用後，Codex 從你的對話中累積你的偏好與習慣。開關在設定頁面，隨時可開或關。
設定 03
等待核准彈窗
Codex 每次要動你的檔案，預設會跳出「等待核准」視窗，讓你決定是否放行。這是你的第一道防線，不要急著關掉它。
設定 04
權限模式
控制 Codex 的讀寫範圍。官方三層：唯讀（:read-only）、工作資料夾（:workspace）、解除沙盒（:danger-full-access）。
權限層 官方標識 讀寫範圍 風險
唯讀 :read-only 只讀、不寫任何檔 最低
工作資料夾（本課起點） :workspace 只允許在指定工作資料夾內寫入 中等
解除沙盒 :danger-full-access 解除本機沙盒限制，任意讀寫 最高

GUI 介面可能顯示不同的友善標籤（如「預設核准」「自動審核」「完整存取」），概念對應上表三層。

!
:workspace 把 Codex 的讀寫限制在你指定的工作資料夾：原始財務檔案放在資料夾外，Codex 就動不了。:danger-full-access 解除本機沙盒限制，等於告訴 Codex「你想改哪個檔都行，不用問我」；加上實際讀寫範圍還受外掛與 Computer Use 影響，超出預期也難事後追查。月結帳情境下，:danger-full-access 能不開就不開。
(03)

步驟 1–5：從安裝到設定完成

按以下順序跟做，每個步驟結束時都有一個可以看見的回報。

1
下載安裝
在瀏覽器開啟 ChatGPT 官網（chatgpt.com），找到 Codex 的下載連結，下載對應你作業系統（Windows / macOS）的安裝檔，照一般 App 安裝流程完成。安裝完成後打開 Codex：介面和 ChatGPT 類似，中間是輸入框，左側是對話記錄面板。先別打字——先讓它認識你。
2
自訂指令——讓它認得你
點左下角「設定」→ 切換到「個人化」類別 → 找到「自訂指令」欄位。填入以下文字（本課標準版，你也可依自己的實際職務調整），填好後點「儲存」。
自訂指令範本（複製後貼入設定頁面）
我是弄一下工作室的行政會計人員，主要負責月結報帳、供應商對帳與費用彙整。
請用繁體中文回覆，語氣正式但簡潔，以重點條列為主，不需要過多解釋說明。
不確定的地方請直接告訴我，不要自己補填資料或猜測金額。
!
把關提醒：「不確定的地方請直接告訴我，不要自己補填資料或猜測金額」這句，是在提示詞層面預裝的防線。它不保證 Codex 100% 遵守，但能大幅降低它自己腦補資料的機率。CH1-3 的 AGENTS.md（操作守則）還會再加一層。
3
記憶開關
在「個人化」頁面的下方找到「記憶」設定，啟用它。啟用後，Codex 會從對話中記錄你的偏好（例如：你說過月結 Excel 要用特定格式），下次不用重複交代。若日後覺得它記錯什麼，可在記憶管理頁面手動刪除特定項目。
4
「等待核准」彈窗的作用
關掉設定頁面，回到主畫面。CH1-3 建立專案後、請 Codex 動你的工作資料夾時，它每次要讀取或修改檔案，都會彈出「等待核准」提示視窗。你會看到兩個選項：「是」（只允許這一次）、「此工作階段不要再詢問」（本次工作階段內不再問，重新開啟後恢復）。這個彈窗是 Codex 在問你「我可以動這個檔嗎？」——這是好事，不是麻煩。月結帳的情境下，每一步你都應該知道它在動哪個檔。
!
把關提醒：不要一看到彈窗就習慣性按「此工作階段不要再詢問」。第一次使用，建議一個一個按「是」，確認它動的是你要它動的那個檔。
5
設定權限模式
彈窗下方有一個按鈕，讓你切換整體的「權限模式」。把模式設定在 :workspace（工作資料夾），不要切換成 :danger-full-access（解除沙盒）。
!
把關提醒（本單元最重要）：:danger-full-access 模式下，Codex 不問你就改檔，且實際讀寫範圍取決於權限模式、外掛與 Computer Use。月結帳、對帳明細、供應商清單是有責任的數字。從 :workspace 起步，:danger-full-access 是你評估清楚背景排程風險後才開的選項，不是預設值。
(04)

動手跟做：完成設定並驗收

按以下步驟操作，完成後用測試問話驗收自訂指令是否生效。

1
打開剛安裝好的 Codex
2
進「設定」→「個人化」→「自訂指令」，填入上方範本
至少要包含：公司名稱（弄一下工作室）、你的職務（行政會計）、回覆語言（繁體中文）、回覆風格（簡短重點條列）、一句「不確定就停、不要自行補資料」的指示。填完後點「儲存」，確認彈窗消失。
3
確認「記憶」設定已啟用
4
回到主畫面，送出這句測試問話
測試問話（複製後貼入對話輸入框）
我收到一張廠商的報價單，請問我作為弄一下工作室的行政會計，下一步要做哪些確認工作？

看 Codex 怎麼回答。它應該用繁體中文、行政會計的視角、條列式列出確認步驟——這就是自訂指令生效的確認點。如果回答不符，回到「設定」→「個人化」確認自訂指令已儲存，重新開一個對話再送一次。

5
確認權限模式設在 :workspace
確認你的權限模式設定在 :workspace（工作資料夾）或更保守的選項。不要切換成 :danger-full-access（解除沙盒）。
(05)

進下一頁前確認

學會的標準
  • 測試問話的回覆，Codex 用繁體中文、行政會計視角、條列式回答確認工作步驟。
  • 你能說出 :workspace 和 :danger-full-access 的差別，以及月結帳情境下為何從 :workspace 起步。

兩件都確認，就可以進 CH1-3 建工作資料夾和 AGENTS.md——下一步，Codex 不只認識你，還會認識你正在做的這份月結工作。

(06)

常見錯誤——提前知道，避免踩雷

錯誤 1
填了忘記按儲存
現象：自訂指令填完後送出測試問話，Codex 還是用通用語氣回答、沒有表現出行政會計的視角。
原因：填完後忘記按「儲存」，或是按了儲存但沒有回到對話主畫面重開一個新對話——設定沒有套進當前對話。
解法：點「儲存」確認彈窗消失後，回到主畫面重新開啟一個新對話再測試。原本還開著的對話視窗不會自動套入新設定。
錯誤 2
習慣性消掉核准彈窗
現象：看到「等待核准」彈窗就急著按「此工作階段不要再詢問」，之後 Codex 動了哪些檔案完全不知道。
原因：把彈窗當作程式錯誤或打擾，急著消除，讓 Codex 在沒有確認的情況下動了不該動的檔。
解法：CH1 和 CH2 學習階段，每個彈窗都應該逐一確認。等你熟悉 Codex 的行為模式、確認它真的讀懂了你的指令之後，再考慮切換到「自動審核」——不是現在。
錯誤 3
切換完整存取權
現象：權限模式切到「完整存取權」（:danger-full-access），覺得這樣做事最順、彈窗最少。結果在 CH2 練習彙整 Excel 時，發現原始報帳單已被 Codex 改掉了幾個格子，但不確定從哪步開始改的。
原因：「完整存取」讓 Codex 在任何檔案上都不問就動，加上實際讀寫範圍還受外掛與 Computer Use 影響。月結帳的原始來源一旦被改，再拿去核對就是在對比兩份都可能不準的資料。
解法：切回 :workspace（工作資料夾）。原始報帳單、transactions.csv 這類有責任的源頭檔，在 CH1-3 建工作資料夾時就留一份副本不動——讓 Codex 只碰工作副本，且工作資料夾外的原始版不在 :workspace 的讀寫範圍內。
(07)

確認你學到了什麼

Q1：在 Codex 的三層權限模式中，:workspace 和 :danger-full-access 最關鍵的差異是什麼？
查看解析
正確答案是 C。:workspace 把 Codex 的讀寫限制在你指定的工作資料夾內，原始財務檔案放在資料夾外就動不了。:danger-full-access 解除本機沙盒限制，Codex 可在任意位置讀寫檔案，且實際範圍還受外掛與 Computer Use 影響，月結帳情境下風險過高。
Q2：你是弄一下工作室的行政會計。今天第一次使用 Codex，準備讓它幫你整理 6 份命名雜亂的報帳單。你應該把權限模式設定成哪一個，並說明理由？
查看解析
正確答案是 B。應選 :workspace（工作資料夾）：
  • 把讀寫範圍限制在你指定的工作資料夾，遵守最小權限原則
  • 不選 :danger-full-access——月結報帳單是有責任的財務資料，解除沙盒限制後，Codex 的實際讀寫範圍還受外掛與 Computer Use 影響，超出預期也難追查
  • （加分）說出要準備一份「原始檔副本」，讓 Codex 只碰工作副本，不碰源頭檔
