(10)

排程與儀表板

讓 Codex 每天早上自動掃信、整理廠商回覆待更新清單，再做一個 HTML 儀表板讓老闆一眼看到進度。本單元是能力地圖示範。

學習成果
能說出排程指令裡「護欄」的作用，以及第一版為何只輸出清單、不寫 Excel
能說出 HTML 儀表板的數字「什麼情況下會和 Excel 現狀不一致」，以及如何讓它更新
能列出設排程前的三個最低安全確認點（權限範圍、找不到資料時怎麼辦、如何驗收執行結果）
(01)

破題：12 封回覆信散落 Gmail，老闆每天早上問進度

情境
6 月底，弄一下工作室的廠商對帳作業進入收尾。你發了 12 封確認信給廠商，請他們回覆當月帳款金額是否正確。這 12 封信的回覆分散在 Gmail 裡，每天早上得一封封點開看、再手動把「有回 / 未回」記到 Excel，做完已經過了半個小時。更麻煩的是，老闆早上進公司第一件事就問「現在回覆率多少？還差幾家？」——你得打開 Excel 算一遍再回報。

這一頁做兩件事：

本單元你會看到的成果
  • 一個每早查信的排程（第一版只產待更新清單，不寫 Excel）：每天早上 8:00 Codex 自動掃 Gmail、整理出哪幾家廠商已回覆、哪幾家沒有，輸出清單讓你確認。確認兩三次輸出都正確，才進階到讓它更新工作副本。
  • 一個 HTML 儀表板：顯示廠商對帳回覆率、各廠商狀態、本月月結金額摘要，讓老闆一眼看完進度。

看示範時先記住：排程在你看不到的時候動你的信箱，儀表板的數字是生成那一刻的快照。這兩條限制貫穿本單元，示範裡你會看到怎麼把關。

(02)

概念：排程、安全邊界、HTML 儀表板

概念 01
排程（Schedule）
你事先告訴 Codex「每天幾點做什麼」，到時間它自動在背景啟動、做完關掉，不需要你開著 Codex 視窗等候。可以同時設多個時段（例如早上 8:00 + 下午 3:00）。
概念 02
排程的安全邊界
排程執行時你不在場，Codex 做的每個動作你事後才知道。什麼可以動（讀信、記錄狀態），什麼不能動（寄信、修改原始帳冊、刪除信件）——要在設排程前就寫進指令裡，不是出事後才補。
概念 03
HTML 儀表板
Codex 讀取 Excel 的數據、生成一個 HTML 檔案。用瀏覽器打開就能看到整合好的數字與圖表。關鍵限制：儀表板是某個時間點的快照，資料更新後要重跑（或靠排程定時重建），它不會自己即時抓最新數字。
概念 04
外掛 Skill（示範、不保證學員帳號可用）
有人做好的 Skill 可讓 HTML 排版更整齊；帳號沒有相容 Skill 時，Codex 內建 HTML 生成一樣可以完成儀表板，資料和邏輯完全相同，只差版面精緻度。
(03)

A 段：建立排程

開啟「弄一下工作室_6月月結」專案，進入 Codex 側邊欄，找到「自動化」（Automation）或「排程」（Schedule）設定入口（畫面上通常是時鐘或鬧鐘圖示；不同版本介面位置可能略有不同，功能名稱相同）。

排程指令（第一版只產清單、不寫 Excel）：

排程指令
每天早上 8:00，掃描 Gmail 信箱中主旨含「對帳」或「月結確認」的信件，找出廠商回覆的確認信，整理成「待更新清單」（包含：廠商名稱、信件摘要、建議填入的回覆狀態），不要修改任何 Excel 檔案，掃完後把清單回傳讓我確認。若找不到對應廠商名稱，在清單中標記「待人工確認」。

按確認儲存後，再加一個下午 3:00 的掃描時段。

看示範時注意指令裡的兩個護欄：

1
「若找不到對應廠商名稱，標記待人工確認」
沒有這句，Codex 遇到名稱不完全一致時可能自己猜了填進去。這句話讓它在不確定時停下、標記，而不是靜默做決定。
2
指令裡沒有「寄信」「刪信」「修改 Excel」
排程只做讀取和整理清單。催款、什麼時候更新帳冊，是你自己決定的事，不讓排程代勞。

確認了幾次清單都正確、要讓 Codex 更新工作副本時，進階指令如下：

進階指令（寫入工作副本）
請把剛才的清單，更新到「供應商彙整表_工作副本.xlsx」的「回覆狀態」欄（以廠商名稱比對）。若找不到對應廠商名稱，跳過該筆並記錄待確認，不要自行填入或合併。

這份是工作副本，正式的「供應商彙整表.xlsx」不讓 Codex 直接寫入。

!
把關紅線：排程會在你不在場時自動動作。第一版排程只輸出清單、不寫 Excel——確認兩三次輸出符合預期，才考慮讓它寫入；寫入時只寫工作副本，正式帳冊不讓排程直接碰。每次設新排程後，前幾次一定要打開執行日誌確認它做的事跟你預期的一致。
(04)

B 段：做 HTML 儀表板

本段示範可能因帳號版本不同而有差異；若帳號無相容 Skill，改用 Codex 內建 HTML 生成，產出結果相同。

儀表板指令（有無外掛 Skill 都用同一份）：

儀表板指令
讀取「供應商彙整表.xlsx」，做一個 HTML 儀表板，要顯示：（1）廠商對帳回覆率（已回覆/總數的比例，用長條或圓餅呈現）；（2）各廠商回覆狀態清單（已回覆、未回覆用不同顏色區分）；（3）本月月結總金額（已對帳 vs 未對帳）。輸出為「月結進度儀表板.html」存入工作資料夾。

Codex 讀 Excel、生成 HTML，約 1 分鐘後在工作資料夾出現 月結進度儀表板.html。用瀏覽器打開，你會看到：

儀表板包含的三個區塊
  • 廠商回覆率（圓餅或長條，顯示的是生成當下的數字）
  • 各廠商狀態清單，顏色區分（已回覆 / 未回覆）
  • 月結總金額摘要（已對帳金額 vs 尚未確認金額）

不管有沒有外掛 Skill，Codex 都能做出這個頁面——差別只在版面精緻度，資料內容和邏輯一樣。你自己試的時候，直接用上面這份指令就好。

!
把關紅線：把儀表板拿給老闆看之前，打開 Excel 對一下數字。儀表板好看不代表數字對——來源 Excel 有沒有人工漏填的格子、廠商名稱有沒有寫法不一致，這些只有你看得出來，Codex 不知道。
(05)

觀察單：看示範時記三件事

觀察重點
  • 排程的「靜默執行」：Codex 在你看不到的時候動你的信箱和 Excel。你有沒有看到排程指令設了什麼護欄來限制它只做哪些事？
  • 儀表板的時間戳：這個頁面的數字是「什麼時候」讀 Excel 產出的？如果你早上 8 點才收到廠商信、排程是早上 7 點跑的，儀表板顯示的是哪個狀態？
  • 哪一步你一定要攔：儀表板做好要給老闆看之前，你會先自己做哪一個確認動作？

回答一個問題（給自己，不用交）：如果要讓儀表板顯示「每小時最新狀態」，你認為需要多加什麼步驟？（先想，CH4-2 會有部署與更新的進一步示範。）

(06)

設前把關清單：這一頁的思考題

題目
如果你自己要設一個「每天早上 8 點掃廠商回覆信 + 更新 Excel」的排程，在輸入指令之前，你會先確認哪三件事？

預期成果：三個具體問題或確認點。好的答案例如「確認 Codex 的 Gmail 只有讀取授權、沒有刪信或寄信權限」。不好的答案：「確認 Codex 設定正確」（太籠統）。

自我檢核（三條都答得出來，才動手設排程）
  • 有一條是關於權限範圍（它被允許做哪些動作）
  • 有一條是關於找不到對應資料時 Codex 該怎麼辦（停下還是亂填）
  • 有一條是關於執行後你怎麼確認它做的事符合預期

這三條合在一起就是一個排程設置的最低安全檢查清單。回家自己設排程之前，把這三條回答完，再動手。

(07)

常見錯誤：設排程前先認識這三條

錯誤 1
第一版就讓排程寫入 Excel
現象：排程指令寫「把廠商回覆整理好並更新 Excel」，隔天發現 Codex 把幾格廠商名稱改掉了——原來有兩家廠商簡稱不同（「弄一下」vs「弄一下工作室」），Codex 自己做了合併，直接寫進了 Excel。
原因：第一版就讓排程直接寫入 Excel，沒有先輸出清單讓人確認；指令也沒有說「找不到對應廠商名稱時停下、不自行判斷」，Codex 在沒有護欄的情況下靜默做了決定。
解法：第一版排程只輸出待更新清單，確認兩三次輸出都正確再進階到寫入；寫入時只寫工作副本。排程指令裡一定要加「若無法確認對應關係，跳過並記錄待確認，不自行填入或合併」。護欄要在設排程前就寫進去，事後補救比較麻煩。
錯誤 2
把儀表板快照當即時數字
現象：儀表板顯示「已回覆率 50%」，拿給老闆看，但實際上 Excel 在排程跑後又多了 3 家廠商回覆，儀表板沒更新——老闆以為還差 6 家，其實只差 3 家。
原因：儀表板是某個時間點的快照，沒有「即時抓資料」能力。生成後 Excel 更動，頁面不會跟著變。
解法：每次拿儀表板給人看之前，先重跑一次（重下指令讓 Codex 用最新 Excel 重生成）；或者把「重建儀表板」這個步驟也排進每天的排程，確保兩者時間點一致。
錯誤 3
設好排程隔天卻沒跑
現象：排程設好了，但隔天 Codex 沒有跑——原來電腦睡眠了，或 Codex App 沒有開著。
原因：Codex 排程依賴 App 本地執行，電腦關機、睡眠或 App 退出時排程不會觸發。
解法：排程時間設在你通常電腦開著的時段（不要設凌晨）；第一次設完，隔天早上主動確認一次執行紀錄，確認排程有跑到再信任它。
(08)

確認你學到了什麼

Q1：Codex 的 HTML 儀表板做好之後，廠商又寄來一封確認信，Excel 更新了回覆狀態。此時儀表板裡顯示的回覆率會怎樣？
查看解析
正確答案是 B。HTML 儀表板是 Codex 在某個時間點讀 Excel 生出來的快照，不具備即時抓資料的能力。生成後 Excel 有任何更新，頁面都不會自動反映——需要重新下指令讓 Codex 用最新 Excel 重生成，或者把「重建儀表板」這個步驟也加進每天的排程。D 選項聽起來合理，但排程跑完只是更新了 Excel，儀表板本身不會因此自動刷新。
Q2：你設了一個排程「每天早上 8 點，Codex 自動掃 Gmail 找廠商回覆信，把結果記到 Excel」。你隔天早上打開 Excel，發現一筆廠商名稱被填成「弄一下」而不是 Excel 裡的「弄一下工作室」，而且 Codex 還自己判斷成功更新了。你下一步應該怎麼做？
查看解析
正確思考方向是 B，三個步驟缺一不可：
  • 先確認這筆填入是否正確——「弄一下」和「弄一下工作室」是不是同一家廠商？不能直接信任 Codex 的判斷；若確認是同一家，要在 Excel 補齊廠商名稱寫法，讓日後比對不再出現差異。
  • 修改排程指令，加入「找不到完全一致廠商名稱時停下、記錄待確認、不自行填入或合併」的護欄；排程護欄要在事前設好，不是出事後才補。
  • 回查這次排程的其他輸出，看有沒有其他筆也被靜默改動——不只修那一筆，要全面確認執行日誌，確保只有這一筆例外。
