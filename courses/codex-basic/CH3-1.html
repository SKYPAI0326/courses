(03-01)

把流程做成 Skill

月結整理做完了——但下個月同樣一套流程，不能每次重來跟 Codex 說清楚。這頁教你把「說清楚那段」打包成 Skill，之後一句話就能叫出來。18 分鐘，做出兩個 Skill。

學習成果
能說出 AGENTS.md 和 Skill 各適合放什麼，不會把 SOP 塞錯地方
能做出「月結整理」專案層級 Skill 並用一句話自動觸發
能做出「表格排版」全域層級 Skill 並用斜線手動觸發
(01)

破題：下個月同樣一包，還要重說一遍嗎？

CH2 整理好了。6 份報帳單命名統一、彙整 Excel 做出來、供應商重複項標好了。

下個月呢？7 月那批報帳單送進來，你要重新開對話，重新跟 Codex 說：「幫我把檔名改成部門-月份-報帳單格式、彙整進那張 Excel、重複發票號碼標出來……」一整段交代，再說一遍。

做完這個單元，你只需打這一句：

觸發提示詞
幫我跑月結整理

Codex 就會執行完整流程，不需要再說明步驟。這個功能叫 Skill（技能）。

這頁結束你會做出的兩個成果
  • 「月結整理」專案 Skill——只在月結專案裡生效，封裝 CH2-1／CH2-2 的完整流程，一句話觸發
  • 「表格排版」全域 Skill——跨所有專案都能叫用，Word 表格格式不用每次重新說

做完後，這兩個 Skill 都能在 Codex 的「外掛程式 → 技能」頁看到、管理。

(02)

AGENTS.md vs Skill：各放什麼、怎麼決定

先搞清楚這兩個各自適合放什麼，再決定該封裝哪些東西。

AGENTS.md（操作守則）
每次開對話都整份讀入
適合放「每次都需要知道的背景」：公司名稱、你的職稱、不得補值的底線規則、輸出格式的通用要求（例：日期欄一律 YYYY-MM-DD）。

不要放：一次性的 SOP、按需才用的流程說明。塞太多會讓每次對話都先消化一大包背景，讓 Codex 在執行實際工作前就先「疲勞」了。
Skill（技能）
只有被呼叫時才讀取
一個 Skill 是一個資料夾，資料夾內必含 SKILL.md（主說明），可再放 scripts/、references/、assets/ 等子資源。打開 SKILL.md，裡面就是人讀得懂的中文步驟說明，不是程式碼。

Skill 的設計是 progressive disclosure：你沒有呼叫它時，Codex 完全不載入它的內容；你呼叫它的瞬間，Codex 才讀取完整 SKILL.md 並執行。平時 context 空間留給當前工作，不被你還沒要用的 SOP 佔走。
專案層級 Skill
只在這個專案的對話裡看得到
存在這個專案的資料夾內（例：弄一下工作室_6月月結/.codex/skills/月結整理/）。適合「這個專案特有的流程」，跨到其他專案的對話就不會出現。
全域層級 Skill
跨所有專案都能叫用
由 Codex App 全域位置管理。任何專案的對話都能呼叫。適合「任何工作都會用到的格式或規範」，例如公文表格排版。
決策表：遇到新規則，放哪裡？
這條規則的使用頻率適用範圍放哪裡
每次開對話都需要這個專案AGENTS.md（專案資料夾內）
按需叫用這個專案特有流程專案層級 Skill
按需叫用跨所有工作都通用全域層級 Skill
!
在「外掛程式 → 技能」頁停用一個 Skill，自動觸發就停了，但 Skill 設定還在。這個行為可以用來觀察 Skill 是否真的「只有被選中才讀入」——停用後再呼叫，Codex 確實不認識它了。
(03)

建立「月結整理」專案 Skill

打開「弄一下工作室_6月月結」專案，開一個新對話。在對話框輸入以下提示詞：

建立專案 Skill 的提示詞（可直接複製）
請把我的月結整理流程封裝成一個專案層級的 Skill，命名為「月結整理」，步驟如下：
1. 把資料夾內的報帳單檔名統一改成「部門-月份-報帳單」格式
2. 把各份報帳單的資料彙整進「月結彙整.xlsx」
3. 標記重複的發票號碼（用複合鍵：發票字軌號 + 廠商統編 + 日期 + 金額）
4. 有缺值或異常金額的欄位標「待覆核」，不補值

Codex 完成後，到專案資料夾確認：弄一下工作室_6月月結/.codex/skills/月結整理/ 資料夾已建立，內含 SKILL.md。打開 SKILL.md，裡面就是中文步驟說明——確認它寫的和你剛才描述的步驟一致。

到「外掛程式 → 技能」頁，確認「月結整理」出現、層級顯示「專案」。

接著測試自動觸發，在輸入框輸入：

自動觸發提示詞
幫我跑月結整理

Codex 自動啟用 Skill，開始執行命名和彙整步驟，不再問「你要整理什麼」。

!
把關提醒：Skill 把流程固化了，第一次就要確認流程正確。若建立時描述的步驟有誤（例如欄位名稱與實際 Excel 不符），Codex 每次執行都會穩定照著錯誤走，而且輸出格式漂亮，表面上看不出哪裡不對。

第一次觸發後的核對方式：
· 逐列比對彙整 Excel 的資料是否來自所有 6 份報帳單（數量對）
· 抽查至少 2 個金額欄位，核回原始來源
· 確認重複發票標記列確實存在、且複合鍵欄位填對

核對沒問題，之後才讓它一句話批次跑。
(04)

建立「表格排版」全域 Skill

切到任意對話（不需要在月結專案下）。在對話框輸入以下提示詞：

建立全域 Skill 的提示詞（可直接複製）
請建立一個全域 Skill，命名為「表格排版」，內容是：
製作 Word 表格時，一律遵守以下格式：
· 標題列：深灰底（#404040）白字
· 內容列：白底黑字
· 欄寬等分
· 字體：標楷體 12pt
· 表格外框：粗線 1.5pt；內框：細線 0.5pt

完成後到「外掛程式 → 技能」頁，確認「表格排版」出現、層級顯示「全域」。

接著測試斜線手動觸發：在輸入框打 /，從彈出選單選「表格排版」，再輸入：

斜線觸發後接的提示詞
幫我替志工名單做一份簽到表

確認 Codex 照「表格排版」Skill 的規格輸出，不是按它自己的預設格式。

!
把關提醒：全域 Skill 影響所有專案的對話。若要針對單一專案修改格式規範，建專案層級 Skill，不要動全域的——動了全域，你其他所有工作的表格輸出都跟著變。

「外掛程式 → 技能」頁每個 Skill 右上角有開關：暫時停用（Codex 停止自動觸發，但 Skill 設定保留）、重新啟用。點「解除安裝」可移除 Skill。

(05)

動手做：把月結流程封裝成你自己的 Skill

把 CH2-1／CH2-2 完成的月結整理流程，封裝成你自己的「月結整理」專案 Skill，並觸發確認它能正確執行。

步驟 1
開新對話
打開你的「弄一下工作室_6月月結」專案，開一個新對話。確認左側面板顯示的是這個專案名稱——不是點「新增對話」開的普通對話。
步驟 2
輸入提示詞
在對話框輸入：「請把月結整理流程封裝成一個專案 Skill，命名為「月結整理」，步驟如下：1. 把資料夾內報帳單的檔名改成「部門-月份-報帳單」格式；2. 把資料彙整進「月結彙整.xlsx」；3. 標出重複發票號碼（複合鍵：發票字軌號 + 廠商統編 + 日期 + 金額）；4. 缺值或異常金額標「待覆核」，不補值」
步驟 3
確認技能頁
Skill 建立完成後，到「外掛程式 → 技能」頁，確認「月結整理」出現、層級顯示「專案」。
步驟 4
觸發 Skill
新開一個對話（同一個月結專案），輸入「幫我跑月結整理」，確認 Codex 直接開始執行步驟，不再問「你要整理哪些內容？」
步驟 5
自行核對
專案資料夾內出現 .codex/skills/月結整理/SKILL.md，打開確認步驟描述與你輸入的一致。Codex 執行後，彙整 Excel 有 6 份報帳單的資料（不是空的、不是少的）。重複發票標記欄位存在，缺值欄位標「待覆核」而非自行補填。
(06)

情境案例：陳主管的兩個要求

情境
你（弄一下工作室行政會計）剛完成 6 月月結整理。陳主管說：「這套流程下個月還要跑，能不能設定好讓你不用每次重新說一遍？另外，出公文的表格格式每個人都不一樣，能不能統一設定一次就好？」

判斷這兩個需求各該用哪種方式處理：

月結流程
下月重複用
→ 專案層級 Skill
這套流程只在月結專案下用，封裝成專案 Skill，下個月一句話就能叫出來跑，不用重新說步驟。
公文表格
全部統一格式
→ 全域層級 Skill
表格格式規範跨所有工作都適用，不只月結。放全域 Skill，不管在哪個專案的對話都能用斜線叫出來。

動手完成「表格排版」全域 Skill 的建立（步驟詳見上方第 (04) 節），並用斜線 /表格排版 觸發確認 Codex 照格式規範輸出。

(07)

常見錯誤——先知道，再動手

錯誤 1
Skill 沒觸發
現象：「幫我跑月結整理」送出後，Codex 回「請問你要整理哪些步驟？」——沒有觸發 Skill。
原因：當前對話不在「弄一下工作室_6月月結」專案下。這個 Skill 是專案層級，Codex 在其他專案的對話裡看不到它。
解法：確認左側面板顯示的是「弄一下工作室_6月月結」專案，不是點「新增對話」開的普通對話。若需要在任何專案都能用，改建全域 Skill；或在當前對話用斜線 /月結整理 手動觸發。
錯誤 2
輸出結果不對
現象：Skill 建好第一次觸發，Codex 執行了但彙整 Excel 裡有欄位名稱對不上、金額不對。
原因：建立 Skill 時描述的欄位名稱與實際 Excel 的欄名不符（例如說「金額」，但 Excel 欄名是「報帳金額」）。Skill 固化了這個錯誤，每次觸發都穩定地照著走，而且輸出格式整齊，表面上不容易察覺。
解法：發現問題立即重新提示 Codex 更新 Skill 步驟，說明正確的欄位名稱。更新後觸發一次，確認輸出正確，再正式複用。第一次觸發後就核對，不要「先跑幾次看看」。
錯誤 3
全塞進 AGENTS.md
現象：把整份月結 SOP 塞進 AGENTS.md，之後 Codex 回覆偶爾漏掉某些步驟。
原因：AGENTS.md 每次對話都整份載入。長的 SOP 和其他背景規則一起塞進去，Codex 的可用思考空間被背景資訊占走，執行時偶爾就會漏步驟。Skill 的 progressive disclosure 設計就是為了解決這個問題：SOP 的細節只在你呼叫時才進 context，其他時候不占空間。
解法：AGENTS.md 只留「每次都需要的背景」（公司名、底線規則、通用格式要求）；按需才用的 SOP 改封裝成 Skill。兩邊各司其職。
(08)

做到這三件，代表本單元學會了

本單元完成標準
  • 不打開資料夾，說得出「月結整理 Skill 在哪裡找到」（外掛程式 → 技能頁，層級：專案）
  • 以下三個情境，說得出分別用哪個方案：「每次對話都要知道的公司名稱和月份格式」→ AGENTS.md；「每月做一次的月結 SOP，不是每天都用」→ 專案 Skill；「所有工作都適用的 Word 表格排版格式」→ 全域 Skill
  • 說得出 Skill 第一次觸發後要核對的三個點（數量對、金額抽查、標記欄位正確）
(09)

確認你學到了什麼

Q1：和把流程寫進 AGENTS.md 相比，封裝成 Skill 最主要的優點是什麼？
查看解析
正確答案是 B。Skill 的核心優勢在於 progressive disclosure——它平常不會載入到對話 context，只有你呼叫時才讀取完整內容。AGENTS.md 每次對話都整份讀一遍，長的 SOP 塞進去會讓背景資訊占走 context 空間，導致 Codex 回覆容易漏步驟。Skill 讓兩邊各司其職：常態背景放 AGENTS.md，按需 SOP 放 Skill。
Q2：你想把「每份月結報告都要在第一頁加封面，封面格式固定」這條規範設定好。陳主管說這個格式所有報告都適用，不只月結用。你應該怎麼做？
查看解析
正確答案是 C。這個情境的判斷要點有三：
  • 選全域 Skill：格式規範跨所有報告都會用到，不只限月結專案，放全域才能每個工作都叫用
  • 不放進 AGENTS.md：格式規範只有「做報告時」才需要，不是每次開對話都要載入的背景，放進 AGENTS.md 只是讓它每次都白白占 context
  • 確認做對的方式：建好後到「外掛程式 → 技能」頁確認層級標示「全域」；用斜線觸發測試一次，確認封面格式符合預期，再正式複用
