(03-01)

把流程做成 Skill

月結整理做完了——但下個月同樣一套流程，不能每次重來跟 Codex 說清楚。這頁教你把「說清楚那段」打包成 Skill，之後一句話就能叫出來。18 分鐘，做出兩個 Skill。

學習成果
能說出 AGENTS.md 和 Skill 各適合放什麼，不會把 SOP 塞錯地方
能做出「月結整理」專案層級 Skill 並用一句話觸發
能做出「表格排版」全域層級 Skill 並用斜線手動觸發
(01)

破題：下個月同樣一包，還要重說一遍嗎？

這頁結束你會做出什麼
一個可以一句話觸發的「月結整理」專案 Skill，和一個跨所有工作都能叫的「表格排版」全域 Skill。下個月月結，你只需輸入：「幫我跑月結整理」。

月結資料整理好了——命名也統一、彙整 Excel 也做出來了。

下個月呢？又一批命名雜亂的報帳單送進來，你要重新開一個對話，重新跟 Codex 說一遍：「幫我把檔名改成部門-月份-報帳單格式、然後彙整進那張 Excel、重複的發票號碼標出來……」

做完這個單元，你不用再重說那一段。你只需打：「幫我跑月結整理。」Codex 就知道要做什麼。這個功能叫 Skill（技能）。接下來 18 分鐘，你會做出兩個：一個只在「6 月月結」這個專案生效，一個全域都能用。

(02)

關鍵概念：AGENTS.md vs Skill，兩種層級，兩種觸發

概念 01
AGENTS.md（操作守則）
每次開新對話，Codex 都會把它讀一遍。適合放「每次都需要知道的背景」，例如公司名稱、你的職稱、不得補值的底線。

什麼不該放：一次性或按需才用的 SOP；放太多會讓 AGENTS.md 變成每次都要消化的背景負擔。
概念 02
Skill（技能）
把一套重複流程打包成一個資料夾（內含 SKILL.md 主說明，可再放 scripts/、references/ 等子資源）。平常不載入到對話，只有你呼叫時才讀取完整內容（progressive disclosure）。適合放「不是每次都用、但用到就要精準執行」的 SOP。
概念 03
專案層級 Skill
存在這個專案的資料夾裡，只在這個專案的對話中才看得到。適合「這個專案特有的流程」，例如月結整理。
概念 04
全域層級 Skill
跨所有專案都能叫用。適合「任何工作都會用到的通用格式規範」，例如公文表格排版。
概念 05
兩種觸發方式 + 管理頁
自動觸發：你打的話和 Skill 的描述夠接近，Codex 自行判斷啟用。
斜線手動觸發：在輸入框打 /，從彈出選單指定要啟用哪個 Skill。需要精確控制時用這種。

「外掛程式 → 技能」管理頁：所有 Skill 都能在這裡看到，可以停用（暫時不要自動叫）、重新啟用、或解除安裝（刪掉這個 Skill）。
(03)

操作示範：建立兩個 Skill

講師示範
以下為講師示範流程。你先看步驟，下一節再跟做。

示範一：建立「月結整理」專案 Skill

1
打開專案，開新對話
打開「弄一下工作室_6月月結」專案，開一個新對話。輸入以下提示詞：

「請把我這個月結整理流程封裝成一個專案層級的 Skill：
1. 把資料夾裡的報帳單檔名統一改成「部門-月份-報帳單」格式
2. 把各份報帳單的資料彙整進「月結彙整.xlsx」
3. 標記重複的發票號碼
Skill 命名為「月結整理」」
2
確認 Skill 建立成功
Codex 幾秒後完成。它在你的資料夾裡建了一個 .codex/skills/月結整理/ 的資料夾，裡面有一份 SKILL.md——打開來看，就是一份中文步驟說明，不是程式碼。
帶學員到「外掛程式 → 技能」頁確認「月結整理」已出現，層級顯示「專案」。
3
觸發 Skill——一句話啟動
輸入：「幫我跑月結整理」。Codex 自動啟用 Skill，開始執行命名和彙整步驟。
!
把關提醒：Skill 固化了流程，第一次就要確認流程對。如果你在建立 Skill 時描述的步驟有誤（例如欄位名稱講錯），Codex 之後每次執行都會穩定地照著錯誤走，而且你看到輸出可能還覺得「格式對就沒問題」。建好後第一次跑，務必逐步確認輸出是否符合預期，再正式交給它一句話批次執行。

示範二：建立「表格排版」全域 Skill

4
切到任意對話，輸入全域 Skill 提示詞
切到任意對話（不在月結專案也無妨）。輸入：

「請建立一個全域的 Skill，命名為「表格排版」，內容是：
製作 Word 表格時，一律遵守以下格式——
標題列深灰底白字（#404040 / 白）、內容列白底黑字、欄寬等分、字體標楷體 12pt、
表格外框粗線 1.5pt 內框細線 0.5pt。」
5
確認全域 Skill 出現在管理頁
完成後，到「外掛程式 → 技能」頁，找到「表格排版」——層級顯示「全域」。這個 Skill 不管你在哪個專案，打斜線都能叫出來用。
6
示範斜線手動觸發
在輸入框打 /，彈出選單選「表格排版」，再輸入「幫我替志工名單做一份簽到表」，送出。
把關提醒：全域 Skill 影響範圍廣，修改它之前要想清楚——你在六月月結這個專案調整的格式規範，可能同時影響你另一個採購專案的輸出。要針對單一專案修改，建專案層級 Skill，不要動全域的。
7
管理補充：停用 / 重新啟用 / 解除安裝
「外掛程式 → 技能」頁右上角有開關，可以把 Skill 暫時停用（讓 Codex 停止自動觸發，但保留設定）；不需要的 Skill 可以點左下角「解除安裝」移除。
(04)

動手跟做：建立你的「月結整理」專案 Skill

現在換你動手，完成「月結整理」專案 Skill 的建立與觸發。

步驟 1
開新對話
打開自己的「弄一下工作室_6月月結」專案，開新對話。確認左側面板顯示的是這個專案名稱（不是點「新增對話」開的普通對話）。
步驟 2
輸入提示詞
輸入以下提示詞（可依實際情況微調）：「請把月結整理流程封裝成一個專案 Skill，步驟如下：1. 把資料夾內報帳單的檔名改成「部門-月份-報帳單」格式；2. 把資料彙整進「月結彙整.xlsx」；3. 標出重複發票號碼。命名為「月結整理」」
步驟 3
確認管理頁
Skill 建立完成後，到「外掛程式 → 技能」頁確認它出現、層級顯示「專案」。
步驟 4
觸發 Skill
新開一個對話（同一個月結專案），輸入「幫我跑月結整理」，確認 Codex 正確啟動 Skill 並開始執行。
步驟 5
自行確認
資料夾內出現 .codex/skills/月結整理/SKILL.md（打開確認內容是你描述的步驟）。Codex 真的執行了重新命名與彙整，而不是再問一次「你要做什麼？」
(05)

驗證：兩件做到就學會了

本單元完成標準
  • 不必打開資料夾，說得出「我的月結整理 Skill 在哪個地方找到」（外掛程式 → 技能頁，層級：專案）
  • 遇到新情境能判斷：每次對話都需要知道的「公司名稱和月份」→ AGENTS.md；每月做一次、流程固定但不是每天用的月結 SOP → 專案 Skill；跨所有工作都適用的「Word 表格排版格式」→ 全域 Skill

(07)

常見錯誤——先知道，再動手

錯誤 1
Skill 沒有觸發
現象：「幫我跑月結整理」送出後，Codex 回覆「請問你要整理哪些步驟？」——沒有觸發 Skill。
原因：Skill 建在另一個專案，或當前對話不在月結專案下，Codex 看不到這個專案層級的 Skill。
解法：確認你是在「弄一下工作室_6月月結」專案下開的對話（左側面板顯示的是這個專案），不是點「新增對話」開的普通對話。或改用斜線 / 手動觸發。
錯誤 2
輸出結果不對
現象：Skill 建好後第一次觸發，Codex 執行了但輸出結果不對（欄位名稱錯、格式跑掉）。
原因：建立 Skill 時描述的步驟本身有誤（例如 Excel 欄位名稱和實際檔案不符），Skill 忠實固化了這個錯誤。之後每次觸發都會穩定地照著錯走。
解法：第一次觸發後逐步核對輸出，發現問題立即重新描述 Skill 步驟、讓 Codex 更新內容，確認正確後才正式複用。
錯誤 3
全塞進 AGENTS.md
現象：把所有規則都塞進 AGENTS.md，Codex 回覆偶爾會漏掉某些步驟。
原因：AGENTS.md 每次對話都整份載入；長的 SOP 一起塞進去，context 空間就被背景資訊占走。Skill 的設計是 progressive disclosure——只有被選中時才讀完整內容，平時不進 context，所以適合放按需才用的 SOP。
解法：AGENTS.md 只留「每次都需要知道」的背景（公司名、底線規則）；按需才用的 SOP 改成 Skill。兩邊各司其職，context 才不會虛耗。
(08)

確認你學到了什麼

Q1：和把流程寫進 AGENTS.md 相比，封裝成 Skill 最主要的優點是什麼？
查看解析
正確答案是 B。Skill 的核心優勢在於 progressive disclosure——它平常不會載入到對話 context，只有你呼叫時才讀取完整內容。AGENTS.md 每次對話都整份讀一遍，長的 SOP 塞進去會讓背景資訊占走 context 空間，導致 Codex 回覆容易漏步驟。Skill 讓兩邊各司其職：常態背景放 AGENTS.md，按需 SOP 放 Skill。
Q2：你想把「每份月結報告都要在第一頁加封面、封面格式固定」這條規範設定好。陳主管說這個格式其實所有報告都適用，不只月結。你應該怎麼做？
查看解析
正確答案是 C。這個情境的關鍵有三點：
  • 選全域 Skill：因為這條格式規範跨所有報告都會用到，不只限月結專案
  • 不要放進 AGENTS.md：格式規範只有「做報告時」才需要，不是每次對話都要載入的背景資訊
  • 確認做對的方式：建好後在「外掛程式 → 技能」頁確認層級顯示「全域」；用斜線觸發測試一次，確認封面格式符合預期，再正式複用
