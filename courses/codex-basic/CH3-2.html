(03-2)

外掛擴充能力

裝三個外掛之後，PPT、識別吊牌批次 PDF、廠商對帳確認信——6 月底月結這三件收尾工作，Codex 都能草擬或批次產出。本頁的重點不只是「裝得起來」，而是知道每條線哪裡你要自己攔下來核。

學習成果
能安裝 Presentations、PDF Skill、Gmail 外掛並說明各自的用途與差別
能說出三條把關底線：PPT 數字要回得到來源、批次 PDF 跑前先確認名單、Gmail 草稿信逐封核過再寄
能指出裝 Gmail 外掛時授權應給的最小範圍，以及「寄出」為何一律由人手動按
(01)

破題：6 月底還剩三件事

6 月底，你已經有一包整理好的帳——命名統一的報帳單、一張彙整 Excel、可重用的月結 Skill。但月結還沒結束：陳主管要一份給老闆看的管理層摘要 PPT；行政用的部門識別吊牌還沒做；幾封對帳確認信和催收信還在排隊。

這個單元，你會看到 Codex 裝三個外掛之後能做出以下成果：

本單元明確任務
  • 一份「6 月月結管理層摘要」PPT（Presentations 外掛生的）
  • 一批部門識別吊牌 PDF（內建生圖 + PDF Skill 批次出的）
  • Gmail 草稿信數封（Gmail 外掛草擬，寄出由你按）

三條外掛線，每條都有一個「哪裡要你自己攔下來核」。看完本頁你會知道三條底線在哪裡。

!
外掛可用性：Presentations、Gmail、PDF Skill 等功能依帳號方案與版本而異，不保證你的帳號當下可用。這一頁用看操作流程加觀察，重點是看清楚操作邏輯與把關位置，課後再在自己帳號上跑。若外掛暫不可用，課程備有替代流程。
(02)

外掛、Skill、App：三個概念先分清楚

這三個詞在 Codex 裡各有位置，裝錯或混用，你會找不到要裝的東西。

Skill（技能）
可重用的工作流文件
描述 Codex 執行一類任務的步驟與規則。可以自建（CH3-1），也可以從 marketplace 安裝別人寫好的。不含外部服務連線。
App / Connector（應用程式/連接器）
連接外部服務的橋接模組
讓 Codex 連接外部服務（如 Gmail、Google Drive）的橋接模組。安裝後 Codex 才能讀寫那個服務的資料，通常需要授權流程。
外掛（Plugin）
安裝與分發的打包單位
一個外掛可以同時包含 Skill、App/Connector、MCP 設定等多個元件。你從外掛程式頁安裝的「Gmail 外掛」或「Presentations 外掛」就是這一層。安裝一個外掛，可能同時帶進一條 Skill 和一個 App 連線。
最小必要權限
裝外掛時只給這個任務真正需要的存取
裝外掛時，Codex 會請你授權某些帳號存取。只給它這個任務真正需要的權限，不需要的存取不要勾選。
(03)

操作流程：三條外掛線依序

外掛 ① Presentations — 生管理層摘要 PPT
目標：讓 Codex 生出一份可以給老闆看的「6 月月結管理層摘要」PPT，不用你自己一頁一頁排版。
1
進外掛程式頁，安裝 Presentations 外掛
點左上角「外掛程式」，找到 Presentations 外掛，點安裝。
2
回首頁送出提示詞
回到首頁（或你的月結專案對話），送出以下提示詞：
提示詞
請用 Presentations 外掛生一份「弄一下工作室 6 月月結管理層摘要」PPT，大約 8–10 頁，包含：本月費用總覽、各部門費用分項、供應商付款進度、下月待追事項。數字從我的彙整 Excel 取。
3
打開生出的 PPT，看結構與數字
過一兩分鐘，Codex 啟用 Presentations 外掛，出 PPT 檔。打開 PPT，看頁面結構與數字呈現。
!
把關紅線：對外簡報的數字，一定要能回得到來源。Codex 從你的 Excel 引數字，但有沒有引錯欄、合計對不對，它不會告訴你。把這份簡報轉給老闆之前，陳主管要核過一遍，確認每個數字有來源、能回得去，不能直接就發出去。
外掛 ② 內建生圖 + PDF Skill — 批次做識別吊牌
目標：行政組要給部門人員識別吊牌，有人名清單，每個人名字不一樣。Codex 內建生圖 + PDF Skill 批次搞定。
1
向 Codex 描述設計需求，生底圖
向 Codex 描述設計需求，送出以下提示詞：
提示詞
幫我生一張部門識別吊牌的底圖，尺寸比例 54×86mm（名片橫式），風格：簡潔白底、左上角放弄一下工作室 Logo、中央保留姓名和職稱的空間、底部一條橘色色塊（色碼 #b5703a）。
2
確認底圖後，把圖檔拖曳到工作資料夾
Codex 生出底圖設計稿（這只是範本，名字還沒填）。確認底圖後，請 Codex 再生一張乾淨底圖（不含示範用的名字佔位符），把這兩個圖檔拖曳到工作資料夾。
3
進外掛程式頁「技能」標籤，安裝 PDF Skill
點外掛程式頁，切換到「技能」標籤，找到 PDF Skill，點安裝。
4
回首頁送出提示詞，批次生成每人一份 PDF
回首頁，送出以下提示詞：
提示詞
請使用剛才生的識別吊牌底圖，把「弄一下工作室_6月月結」資料夾裡的員工名單（姓名、部門、職稱）逐一填入，批次生成每人一份識別吊牌 PDF，存到「識別吊牌/」子資料夾。
Codex 跑完後，打開子資料夾，每個人一份 PDF。
!
把關紅線：批次輸出前確認名單正確。PDF 一次生很多份——名字打錯、職稱填錯，全批都錯。批次跑之前，先確認來源名單沒問題；跑完再抽查幾份核人名與職稱，確認無誤再送去印。
外掛 ③ Gmail — 草擬對帳確認信 + 催收信存草稿
目標：本月有幾家廠商的帳還沒收到回覆確認，要發對帳確認信；另外有兩張發票過期了，要發催收信。讓 Codex 依廠商資料客製化草擬，存到 Gmail 草稿，寄出的按鈕你自己按。
1
把廠商清單拖曳到工作資料夾
把你的廠商清單（有公司名、聯絡人、未確認金額）拖曳到工作資料夾。
2
進外掛程式頁，安裝 Gmail 外掛，照畫面走授權流程
點外掛程式頁，找到 Gmail 外掛，點安裝，照畫面走授權流程。
!
授權底線：最小必要權限。Gmail 外掛請你授權時，只給「建立草稿」這個範圍的存取就夠了，不需要「讀取你所有郵件」或「代你傳送」。不需要的帳號存取不要勾選。
3
回首頁，先草擬第一封範本
回首頁，送出以下提示詞：
提示詞
請幫我草擬對帳確認信，依廠商清單裡的每家公司各寫一封，開頭依廠商名稱和聯絡人姓名客製化，信末請對方確認「6 月對帳金額」是否與附件一致，口吻正式但不生硬。先給我看第一封範本。
4
確認第一封語氣與客製化的地方
Codex 生出第一封，確認語氣與客製化的地方（公司名、金額、聯絡人都對）。
5
確認樣本沒問題，請 Codex 生成其餘並存草稿
確認沒問題後，送出：
提示詞
樣本沒問題，請把其餘廠商的信全部草擬好並存到 Gmail 草稿匣。
過一兩分鐘，Codex 完成。打開瀏覽器登入 Gmail，確認所有信已在草稿匣。
6
補一段催收信，同樣存草稿
再送出：
提示詞
另外有兩張逾期發票，請草擬催收信，語氣禮貌但明確，請對方在三個工作天內回覆付款時程，同樣存到 Gmail 草稿。
!
把關紅線：信在草稿，寄出的按鈕你自己按。課程規範：本課程只建立草稿，不授權、不要求、不示範寄出；寄出一律由你手動按。每封信打開、逐封核一遍——收件人、金額、措辭——確認對了才按寄出。對外信件寄錯一封就是問題。
(04)

觀察單：看完操作，問自己三個問題

三條外掛線看下來，記住這三個問題——這是你自己跑之前要先想好的：

觀察單（不繳交）
  • 裝外掛授權時，你打算給它哪些範圍的存取？最小必要夠不夠？
  • Codex 生出的 PPT 數字，你要怎麼核對回來源？（翻哪張 Excel、查哪個欄位？）
  • 草稿信看起來語氣對，但收件人、金額有沒有對應到正確的廠商？這你必須逐封確認。

三個問題都答得出來，就掌握了本頁的把關邏輯。

動手選一條線練習
  • A：請 Codex 用 Presentations 外掛，生一份 5 頁的「部門月費摘要」PPT
  • B：用 Gmail 外掛，草擬一封給特定廠商的對帳確認信（一封就好），存到草稿，打開核一下
  • C：請 Codex 生一張吊牌底圖（你描述風格），再用 PDF Skill 批次生 2–3 份含姓名的 PDF

完成標準：成果確實存在（PPT 打得開 / 草稿信在 Gmail / PDF 在子資料夾）；能說出這個成果裡你核了哪一步；授權外掛時沒有給超出需要的權限。

(05)

課後練習環境

課後在自己帳號跑三條線，需要備妥以下環境：

學員端需求清單
  • Codex 桌面 App 已安裝並登入（CH1-2 完成）
  • 月結工作資料夾與彙整 Excel（CH2-2 產出）
  • Gmail 帳號（自己的）可用於授權外掛
  • 廠商清單（自己整理一份；不需要真實廠商，測試用即可）

授權建議：Presentations 外掛與 PDF Skill 只需本機資料夾讀寫；Gmail 外掛只給「建立草稿」，不給讀取全部郵件、代你傳送、管理聯絡人。

!
若外掛在你的帳號暫不可用：Presentations 外掛不可用時，可手動整理一份 PPT 架構後交給 Codex 補齊文字；Gmail 外掛不可用時，請 Codex 草擬郵件文字後複製貼上到 Gmail 自行填寄；PDF Skill 不可用時，先做完底圖設計這一步，批次 PDF 待帳號權限就緒後跑。
(06)

常見錯誤——這三條踩過的人不少

下面三個現象，提前知道就不用繞那個彎。

錯誤 1
以為 Codex 已經幫我寄出去了
現象：Gmail 外掛裝好、信也存到草稿了，以為「Codex 已經幫我寄出去了」，沒有再去 Gmail 確認。
原因：混淆「存草稿」和「寄出」。課程規範只建立草稿；寄出一律由人手動按。
解法：裝完 Gmail 外掛後立刻到瀏覽器開 Gmail 草稿匣，確認信在那裡、還沒寄出。信送出之前，每封都要打開看一遍收件人和金額。
錯誤 2
PPT 數字沒核就給老闆
現象：PPT 生出來，直接截圖傳給老闆，沒有核數字——後來老闆發現合計不對，質疑整份帳的可信度。
原因：把「Codex 從 Excel 引數字」當成「數字一定正確」。Codex 可能引錯欄、算錯總額。
解法：PPT 生出來之後，找到原始彙整 Excel，把 PPT 上的每個關鍵數字跟 Excel 對一遍。對外簡報要能「每個數字都能說出來自哪裡」，陳主管確認過再轉給老闆。
錯誤 3
授權時全勾不看範圍
現象：授權 Gmail 外掛時，不看授權範圍就全勾，給了「讀取全部郵件」或「代你傳送」的權限。
原因：趕快點「下一步」，沒有停下來讀授權項目。
解法：安裝任何外掛授權時，停一下看清楚 Codex 要什麼存取權。這個任務只需要「建立草稿」——其他的不需要也不要給。如果已經給了過寬的權限，到 Google 帳號安全設定裡的「第三方應用程式存取」把它撤銷，重新只給需要的範圍。
(07)

確認你學到了什麼

Q1：外掛（Plugin）、Skill（技能）、App/Connector 三者最關鍵的差別是什麼？
查看解析
正確答案是 B。三者各有位置：Skill 是工作流文件（不含外部連線）；App/Connector 是讓 Codex 連接 Gmail、Drive 等外部服務的橋接模組；Plugin 是安裝分發的打包單位，裝一個外掛可能同時帶進 Skill 和 App。費用和強弱不是核心差別。
Q2：你請 Codex 裝好 Gmail 外掛，草擬了 5 封廠商對帳確認信並存到草稿匣。陳主管問你：「信寄出去了嗎？」你應該怎麼回答，而且接下來要做什麼？
查看解析
正確答案是 B。要點有四：
  • 信還在草稿匣、還沒寄出——課程規範只建立草稿，寄出一律由人手動按
  • 寄出之前，你要逐封打開確認：收件人名稱和 email 對不對、金額是否對應到正確的廠商、語氣是否合宜
  • 確認沒問題之後，自己按發送，不能叫 Codex 代你寄出
  • 如果有一封措辭不對或對象搞混，只改那一封，其他沒問題的照寄——不要因為一封有問題就整批重來
