MODULE 3 · SECTION 3.1

GTM 八種觸發條件

Page View / Click / Form / History / Timer / Custom Event / Scroll / Element Visibility——這 8 種覆蓋 90% 追蹤場景。

為什麼這很重要 · WHY THIS MATTERS
GTM 的觸發條件有 20+ 種，但實務上只有 8 種高頻使用。背熟這 8 種，遇到新需求秒選型。
STEP BY STEP · 本節重點
1
1 Page View 系列（3 種）
Page View：DOM 還沒 ready 就 Fire，最早。DOM Ready：DOM 解析完成，可以抓元素了。Window Loaded：所有資源（圖、script）都載完，最晚。預設 GA4 Config 用 Page View；若 Tag 需要讀 DOM 元素用 DOM Ready。
錯用 Page View 去抓尚未渲染的元素，Tag 會 Fire 但變數是 undefined——最常見 debug 困擾之一。
2
2 Click 系列（2 種）
All Elements：任何元素被點都 Fire，配條件過濾（Click Classes 含某值）。Just Links：只聽 <a> 元素，附贈 Click URL 變數。外連追蹤用 Just Links；按鈕（非 a）用 All Elements。
記得先到『變數』頁勾選內建 Click 變數（Click Element / Classes / URL / Text），不然觸發條件裡這些欄位根本不存在。
3
3 Form Submission
Form 觸發有坑：Wait for Tags（等 Tag Fire 完再提交）、Check Validation（等表單驗證通過）。大多要兩個都勾。測試用 LocalWP 的聯絡表單。
SPA 表單（Vue/React）往往不發 form submit 事件——這時要改用 Custom Event 聽 dataLayer.push。
4
4 History Change（SPA 神器）
偵測 pushState/popState，SPA 網站的『假換頁』靠這個。Trigger Type → History Change → 附贈 New History Fragment 變數。與 GA4 Config 搭配做 virtual_page_view。Module 4.3 會深入。
非 SPA 網站不要用 History Change——會被其他 pushState（GA 本身的）誤觸發。
5
5 Timer / Custom Event / Scroll / Element Visibility
Timer：每 N 秒 Fire（追停留時間、心跳）。Custom Event：監聽 dataLayer 的自訂事件名（Module 4 主角）。Scroll Depth：捲動百分比（GA4 加強型評估已覆蓋，少用）。Element Visibility：元素進入視口才 Fire（影片播放、Promo Banner 曝光）。
這五個用到再學即可。重要性：Custom Event > Element Visibility > Timer > Scroll Depth（GA4 已免費提供）。
6
實戰案例｜一個錯誤 trigger 扣了我 3 萬預算
某線上課程用 Click URL contains 'checkout' 當 Purchase trigger，結果每次使用者點擊「回到結帳頁」按鈕也被算成 Purchase，Meta/Google Ads 回傳假轉換——演算法以為『點到這個受眾都會買』，CPA 看起來爆降到 $30，實際成交 CPA 仍是 $400。廣告機器學歪 3 天後被發現，燒了 3 萬預算。改用 Custom Event = purchase（從 dataLayer 讀）後恢復正常。
URL-based trigger 是最脆弱的，因為 URL 會因 UTM、錨點、query 變形。Purchase 這種關鍵轉換 一律用 dataLayer Custom Event。要改前一定先看 GA4 DebugView 驗證觸發條件。
完成驗證 · VERIFY
每種觸發至少在 LocalWP 試跑一次，截圖存證至少 4 種的 Preview Fire 畫面。
常見問題 · TROUBLESHOOT
觸發條件同時符合多個，會 Fire 多次嗎？
只 Fire 一次。GTM 會合併判斷。但若同一個事件綁多個 Tag，每個 Tag 各自 Fire 一次（這是正確行為）。
Click 觸發抓不到？
8 成是 (1) 變數沒勾（Click Element/Classes 都沒開）；(2) Selector 寫錯；(3) 元素是 iframe 內部（GTM 管不到 iframe）。先用 Preview Mode 點擊後看 Debug 裡 Click 事件有無出現，再往下追。
Form Submission 一直 Fire 但我沒送出？
通常是 Wait for Tags 沒勾，導致 submit 事件在驗證失敗前就被偵測到了。勾 Wait for Tags + Check Validation 可解。
