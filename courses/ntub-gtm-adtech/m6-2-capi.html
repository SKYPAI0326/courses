MODULE 6 · SECTION 6.2

Conversion API（CAPI）部署

iOS ATT 後 Browser Pixel 量減 30%——CAPI 從後端直送 Meta，不怕 adblocker 與 ITP。

為什麼這很重要 · WHY THIS MATTERS
三條部署路徑：GTM Server-Side（最乾淨）、Make.com（零程式）、直接 PHP（最便宜）。挑一條走完整。
STEP BY STEP · 本節重點
1
1 CAPI 是什麼
Browser Pixel 走使用者瀏覽器；CAPI 走你的 server → Meta。CAPI 不會被 adblocker／iOS ITP／第三方 cookie 限制影響。Meta 官方建議兩者並用 + event_id 去重——Browser 主打速度與 cookie 匹配，CAPI 主打可靠性。
CAPI 本身不會讓 ROAS 增加——它只是讓『已發生的事件』不漏掉。重點是能把 EMQ 推到 7+ 時才真正影響廣告效能。
2
2 路徑 A：GTM Server-Side Container
建立第二個 GTM 容器類型『Server』→ 佈署到 Google Cloud Run 或 Stape.io → Web 容器把事件 forward 給 Server 容器 → Server 容器用官方 Meta Client 送到 Meta CAPI。優點：前後統一 GTM 管理、event_id 自動帶；缺點：GCP 成本（~$5/月）+ 學習曲線。
對中大型客戶值得；小客戶走路徑 B 或 C。
3
3 路徑 B：Make.com 中繼
WooCommerce 訂單建立 → Webhook → Make.com → 用 Meta Conversions API module 送到 Meta。Make.com 模板市集有現成範本。優點：零程式、可以插入其他動作（寫 Sheet、Slack 通知）；缺點：每筆訂單一個 operation，量大會吃額度。
50 單/日以內 Make.com 免費方案夠用；破 100/日考慮直送。
4
4 路徑 C：PHP 直送（最便宜）
WooCommerce woocommerce_order_status_completed hook 裡 curl_post 到 https://graph.facebook.com/v18.0/PIXEL_ID/events?access_token=...。需要 Meta 後台生成 Access Token。優點：零月費；缺點：要寫／維護 PHP。
用 Meta 的 facebook-business PHP SDK 比 curl_post 更穩，有重試與錯誤處理。
5
5 event_id 產生與帶入（關鍵）
Browser 與 CAPI 的同一筆 purchase 必須共用同一個 event_id，Meta 才能去重。做法：前端送 purchase 時產生 UUID（crypto.randomUUID()）push 進 dataLayer，也寫進 sessionStorage；後端觸發 CAPI 時從 order meta 讀回同一個 event_id。
沒有 event_id 去重＝雙算。Meta Events Manager Diagnostics 會顯示『Event Match Quality』低——立刻修。
6
實戰案例｜iOS 14 後 CAPI 救了美妝電商的 ROAS
某美妝電商 2021 iOS 14 上線後 Meta 廣告 ROAS 從 4.2 掉到 2.1——Browser Pixel 事件量減 35%，演算法不知道誰買了，優化失靈。導入 CAPI + event_id 去重後，事件覆蓋率從 65% → 94%，EMQ 從 5.1 → 7.8，3 週後 ROAS 回到 3.9。成本：Cloud Run $8/月 + 工程師 16 小時部署。ROI 明顯。
CAPI 不是萬能解藥：如果 EMQ 只停在 6 以下，等於只是把事件送過去但 Meta 匹配不到使用者——沒效果。關鍵是使用者資料的 hash 品質（email + phone + fbp + fbc + client_ip + user_agent 至少 5 個）。
完成驗證 · VERIFY
成功從 LocalWP／Make.com／PHP 其中一條路徑送 Purchase 到 Meta CAPI，在 Events Manager 看到『此事件同時經 Browser 與 Server 接收』標記。
常見問題 · TROUBLESHOOT
CAPI 的 Access Token 會過期嗎？
系統使用者 Access Token 可設永久。個人 Access Token 有 60 天效期——不要用。
event_time 要傳什麼？
實際事件發生的 Unix timestamp（秒）。Meta 要求在 7 天內。訂單是 2 天前才送也沒關係，但寫 event_time=現在會被拒絕（ValidationError）。
可以只用 CAPI 不用 Browser Pixel 嗎？
技術上可以，但 Meta 建議雙軌——Browser 的 fbc/fbp cookie 幫助 match rate。只用 CAPI 的 EMQ 通常 <5。
