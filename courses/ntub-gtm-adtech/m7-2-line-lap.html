MODULE 7 · SECTION 7.2

LINE LAP Pixel + Cross Tag

LAP（LINE Ads Platform）Tag 追轉換、Cross Tag 做跨站再行銷受眾——台灣必備平台。

為什麼這很重要 · WHY THIS MATTERS
LINE 在台灣佔社群 85% 以上——沒 LAP 的投放等於放棄主流市場。但 LAP 文件少、介面中文但繁瑣，這節把重點抓出來。
STEP BY STEP · 本節重點
1
1 LAP Tag 類型
LAP 後台分兩種 Tag：LINE Tag（base）＝全站基礎 Pixel；LINE Custom Tag＝特定事件（conversion、add to cart）。兩個都要裝——base 追基礎、Custom 追轉換。
LAP Tag 的 base 必須在每一頁，Custom 在對應事件頁。
2
2 GTM 佈署：自訂 HTML Tag
GTM 沒有 LAP 官方 template。用 Custom HTML Tag 貼 LAP 提供的 script。Base Tag 綁 All Pages；Conversion Tag 綁 purchase Custom Event、並在 script 裡填 _lt('send', 'cv', { type: 'Conversion', value: '{{DLV - value}}' });。
LAP 的參數用底線前綴而不是 fbq 或 gtag——容易跟其他平台混。
3
3 Cross Tag：受眾同步
LAP → 受眾設定 → Audience → 用 LINE Tag 的條件建受眾（例：過去 30 天造訪但未購買）。這個受眾可直接做 LAP 再行銷廣告。Cross Tag 額外功能：同一個 LINE 官方帳號的多個廣告帳戶共用受眾。
LAP 的受眾至少需 1000 個 LINE 使用者才能投（低於會拒絕出稿）。小客戶初期可能要等量累積。
4
4 LINE Official Account 連動
LAP 的『LINE 官方帳號』類型廣告能把廣告互動的使用者自動加入官方帳號好友。需要先綁定 LINE OA → LAP 廣告。這是 LINE 特有的獲客路徑。
綁定後 LAP 後台會多出『官方帳號成長』報表，追好友增加的 CPA。
5
5 驗證：LAP Tag Helper
Chrome 裝 LINE Tag Helper 擴充 → 打開站 → Helper 角落顯示 base 與 conversion 是否 Fire + 帶的參數。類似 Meta Pixel Helper 操作。
LAP 後台的 Tag 狀態報表延遲 1–2 小時才更新。急著 debug 用 Helper 不要看後台。
6
實戰案例｜LINE Cross Tag 把老客喚回來
某生鮮電商 LINE 官方帳號 50 萬好友，但 LAP 廣告 CPA 一直比 Meta 高 40%。導入 Cross Tag（跨媒體追蹤）後發現：30% 的 LINE 廣告點擊者其實是『封鎖官方帳號 30 天內』的人，LAP 把他們當新客撒廣告、但他們早就在黑名單。排除黑名單受眾後 CPA 降 38%、且能針對 7 天未開訊者做廣告推播召回，召回率 12%。
Cross Tag 要填的 lap_id 是 LINE LAP 後台的 Tag ID（網址 /tag/xxxxxx），不是 Ad Account ID、不是 Pixel ID。常被填錯，填錯就完全沒數據回傳、也不會報錯。
完成驗證 · VERIFY
LAP Tag Helper 顯示你的 LocalWP 站 base + conversion 都成功 Fire、LAP 後台顯示至少 1 筆 conversion 資料。
常見問題 · TROUBLESHOOT
沒有 LINE 官方帳號可以投 LAP 嗎？
不行。LAP 帳號註冊時要綁 LINE OA。先申請 LINE OA（免費）再開 LAP。
LAP Conversion Value 要不要設？
電商要——不然 ROAS 報表是 null。LINE 廣告的自動最佳化吃 value 欄位。
LAP 有 CAPI 嗎？
2024 推出 LINE Conversions API，概念同 Meta CAPI。中大型客戶值得接；小客戶用 base + custom tag 即可。
