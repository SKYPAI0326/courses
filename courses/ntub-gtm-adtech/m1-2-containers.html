MODULE 1 · SECTION 1.2

容器 × 觸發 × 變數 × 版本

GTM 的四大基石一次拆清楚——帳戶層級、容器內資源結構、工作區如何讓團隊不互相踩線。

為什麼這很重要 · WHY THIS MATTERS
很多學員卡在『變數和觸發到底誰先誰後？』『為什麼我同事改了東西我的預覽就變了？』——這些都是因為沒弄懂容器的層級結構。這一節用一張關係圖解掉。
STEP BY STEP · 本節重點
1
1 層級：Google 帳戶 → GTM 帳戶 → 容器
一個 Google 帳戶可以建立多個 GTM 帳戶（通常一個公司一個 GTM 帳戶）；一個 GTM 帳戶可以有多個容器（通常一個站一個容器）。命名習慣：GTM 帳戶名 = 公司名，容器名 = 站點域名（例：acme.com - 主站）。
不要為『開發環境』另開一個容器——用同一個容器 + 工作區（Workspace）就好。多容器同步改動很容易走鐘。
2
2 容器內四大資源
Tags（標籤）：要 Fire 的東西（GA4 事件、Meta Pixel、Ads Conversion）。Triggers（觸發條件）：什麼時候 Fire（頁面瀏覽、點擊特定元素、自訂事件）。Variables（變數）：Fire 時要帶的動態值（URL、點擊文字、dataLayer 欄位）。Folders（資料夾）：分類整理（建議依平台或依事件類型分）。
想像：Tag 是『發射器』、Trigger 是『發射按鈕』、Variable 是『彈藥』。缺一不能射。
3
3 版本（Version）與發布（Publish）
每次按『提交』＝產生一個版本快照，並同步發布到線上。版本號自動遞增（v1, v2...）。可以回滾到任一舊版本。版本命名建議寫清楚『這次改了什麼』，例如：v14 - 加入 TikTok Pixel + 修 purchase 重複 bug。
重要：『預覽模式』裡看到的是你的工作區內容；但『發布』才會讓線上使用者感受到改動。新手最常犯的錯是預覽成功就以為上線了。
4
4 工作區（Workspace）：團隊協作利器
一個容器可以同時開多個工作區，每個人在自己的工作區改，互不干擾。發布時再合併到主分支。免費版限制 3 個工作區，360 版無限制。適合大型團隊做『同時改 A/B 兩個 campaign』。
工作區邏輯像 Git branch——改完要 merge 才算進入 main。如果多人同時改同一個標籤會有衝突提示。
5
5 預覽（Preview / Tag Assistant）工作流
流程：改東西 → 點右上『預覽』 → 輸入要測的 URL → 開新分頁，頁面角落會出現 Debug 訊息。Tag Assistant 會即時顯示每個 Tag 的 Fire/Not fire 狀態、觸發理由、帶過去的變數值。每改必預覽、每預覽必驗收，是 GTM 的紀律。
Preview 模式下你在其他分頁打開同個網域也會被偵測到——所以可以用 Incognito 測試『剛登入的新使用者流程』。
6
實戰案例｜多品牌集團的容器架構
某母公司下有 A/B/C 三個子品牌站點，行銷共用一組投放團隊。早期用「一個 GTM 容器 + 條件判斷 hostname」，tag 數量暴增到 80+、Preview Mode 卡頓、改一個品牌的 pixel 常影響另一個。重構成 三個獨立容器 + 一個共用 Server 容器：每個品牌 Web 容器各 20 個 tag、Server 容器集中處理 CAPI 與 Google Ads Enhanced Conversions，單一品牌上線 tag 只影響自己。
隱藏坑：三容器架構下，_ga cookie 若沒設 domain=.parent.com，A 品牌跳 B 品牌會被視為兩個使用者——記得在 GA4 共享時設 cookie_domain 與 linker。
完成驗證 · VERIFY
能畫出容器層級圖、列出四大資源、解釋工作區與版本差異；能打開自己容器的 Preview 模式並看到至少一個 Tag Fire。
常見問題 · TROUBLESHOOT
我建了帳戶但找不到容器在哪？
在左側『所有容器』頁面——帳戶可以有很多容器。新建帳戶時系統會預設建一個同名容器，如果你選錯類型（網頁/AMP/iOS/Android）可以直接建新的容器，舊的停用就好。
預覽模式一直顯示『未連線』？
三個常見原因：① GTM script 沒植入到站（用 DevTools Elements 搜 GTM-XXX 確認）。② 植入的 container id 和你改的不同（很常見：複製舊 script 忘了換 id）。③ 有廣告阻擋外掛擋 googletagmanager.com。
發布後多久才會生效？
幾乎即時（快取最多 1 分鐘）。但瀏覽器與 CDN 會快取 GTM 的 JS，一般使用者要重新開頁面才會拿到新版。『沒看到』通常是自己沒清快取。
