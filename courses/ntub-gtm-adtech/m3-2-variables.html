MODULE 3 · SECTION 3.2

DLV ／ Lookup Table ／ RegEx Table 三大變數

把髒資料變成乾淨欄位——這三種變數是 GTM 工具箱中最常被用到的三把刀。

為什麼這很重要 · WHY THIS MATTERS
你會 80% 的變數工作都在這三種：從 dataLayer 取值、把值映射到新值、用正則把 URL 拆成欄位。
STEP BY STEP · 本節重點
1
1 Data Layer Variable（DLV）：最基礎
從 window.dataLayer 取一個 key 的值。例：ecommerce.value 或 items.0.item_name。支援巢狀路徑用點號（dot notation）。預設回傳 undefined 時要設 Default Value（空字串或 0）。
習慣把所有常用的 dataLayer 欄位都建成 DLV，命名 DLV - value，方便 Tag 拉用。不要每次在 Tag 裡打 {{Data Layer Variable}}。
2
2 Lookup Table：多對一映射
把一個變數值映射到另一個值。例：把 page hostname 映射到 GA4 Measurement ID（acme.com → G-AAA、acme.jp → G-BBB）。輸入欄位 Input Variable，幾個 key → value 對，加一個 Default Value。
常見用途：多站共用容器時決定送到哪個 GA4；依 country code 切換 currency；依 page path 決定 content group。
3
3 RegEx Table：模糊匹配的 Lookup
同 Lookup Table 但 key 是正則表達式。例：URL path ^/product/ → 'product_page'、^/blog/ → 'blog_page'。比一個個列 URL 有效率。
先用 regex101.com 驗證你的 regex，再貼進 GTM。regex 錯誤常常 Tag 看起來有 Fire 但變數是空的。
4
4 Custom JavaScript Variable：萬能後備
寫一個回傳值的函式。例：從 Cookie 讀 user_id、把時間戳轉格式、拼 event_id。⚠️ 盡量少用：難 debug、難維護。能用 DLV/Lookup/RegEx 就不要寫 JS。
如果真的需要 JS，把函式寫清楚（加註解）、取名有意義、單一職責。不要寫『包 5 個 if-else』的怪獸函式。
5
5 變數命名紀律
DLV：dlv - {欄位名}（例：dlv - value）。Lookup：lookup - {用途}（例：lookup - ga4_measurement_id）。RegEx：re - {用途}。變數頁用資料夾分類（Data Layer / Lookup / Custom）。
一個成熟容器的變數數量可能破 50。沒命名紀律找變數會找到瘋。
6
實戰案例｜用 Lookup Table 把 10 個 tag 壓成 1 個
某內容站要針對 5 個產品類別 × 2 種受眾 = 10 組不同的 Google Ads Remarketing tag。原本 10 個 tag 的容器，Preview Mode 載入 3 秒。重構：用 Lookup Table 變數把 {{Page Path}} 對應到不同的 conversion_label，只留 1 個 tag 動態帶入 label——容器大小 -60%、Preview 秒開、維護只改一張表。
Lookup Table 的 Default Value 一定要設，不然未命中的路徑會傳 undefined到 Google Ads，變成『廣告收到空標籤』的隱性錯誤（後台不會紅字警告）。
完成驗證 · VERIFY
在你的 GTM 容器建至少：3 個 DLV（value/currency/transaction_id）、1 個 Lookup（page path → content group）、1 個 RegEx Table（URL 分類）。
常見問題 · TROUBLESHOOT
DLV 一直回傳 undefined？
三個檢查：(1) dataLayer 有真的 push 這個 key 嗎（Preview Mode 的 Data Layer 頁可看）；(2) 路徑寫對嗎（點號注意大小寫）；(3) Tag Fire 的時機點有沒有比 push 晚（Page View Trigger 太早）。
Lookup Table 的 Default 什麼時候會觸發？
Input 變數值沒匹配任何 key 時。Default 要填有意義的值（例：unknown、0）而不是空字串，否則 GA4 報表會出現『(not set)』很難 debug。
RegEx 比對是否區分大小寫？
預設區分大小寫。要忽略大小寫加 (?i) 於開頭，例 (?i)^/product/。
