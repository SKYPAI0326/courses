MODULE 4 · SECTION 4.1

dataLayer 是什麼

window.dataLayer 是個 JavaScript 陣列——GTM 的血液、前後端追蹤的合約、跨頁傳資料的容器。

為什麼這很重要 · WHY THIS MATTERS
GTM 會不會用進階，關鍵就在 dataLayer 觀念。把這個搞懂，後面所有動態追蹤都是小菜。
STEP BY STEP · 本節重點
1
1 dataLayer 本質：一個陣列
實際就是 window.dataLayer = [];。頁面任何地方都可以 window.dataLayer.push({...}) 一個物件。GTM 會監聽這個陣列的新增——每個 push 就像丟一封信到 GTM 信箱。
為什麼是陣列不是物件？因為同一頁會有多次事件（view_item、add_to_cart、purchase）——每一次 push 獨立留存。
2
2 GTM 與 dataLayer 的接續時序
頁面載入順序：(1) dataLayer 宣告（GTM script 最上方）；(2) GTM container.js 載入；(3) 頁面 HTML 開始填入 dataLayer.push；(4) GTM 監聽到 event → Fire 對應 Tag。重點：GTM script 一定要在所有 push 前就載入，否則前面的 push 會漏。
GTM4WP 外掛會自動處理這個順序，不用擔心。手動埋的站要格外小心 script 位置。
3
3 內建事件：gtm.js / gtm.dom / gtm.load
GTM 自己會 push 三個事件：gtm.js（容器載入完）→ gtm.dom（DOM ready）→ gtm.load（全部載完）。分別對應 Trigger 的 Page View / DOM Ready / Window Loaded。這就是 3.1 提到的三種時機。
理解這三個事件＝理解 GTM 觸發時機。不理解這個會一直卡在『為什麼我的 Tag 偶爾 Fire 不到』。
4
4 實戰：在 Console 觀察 dataLayer
打開任何用 GTM 的站 → DevTools Console → 輸入 dataLayer → 看完整陣列。每個元素都是一個事件。看到 event=view_item 的那筆，點開看 items 陣列結構。這是 Module 5 要產出的資料。
生產環境禁用 Console 的站，用 GTM Preview Mode 的 Data Layer 分頁看同樣資料。
5
5 dataLayer 的哲學：解耦
最重要的思維：前端負責 push，GTM 負責消費。前端不應該關心『這個資料要送 GA4 還是 Meta』——那是 GTM 的事。前端只要忠實 push『商業事件』（user 買了什麼、做了什麼）。這個解耦讓後續加新平台不用動前端程式碼。
這個原則叫 '事件規格 = 合約'：前端與 GTM 工程師共同維護一份 dataLayer schema 文件，改動要雙方同意。
6
實戰案例｜dataLayer 命名不一致毀掉半年數據
某電商前端團隊 6 個工程師各自埋 dataLayer，product id 有人寫 product_id、有人 productId、有人 sku——GA4 items 欄位一半空值，Meta DPA（Dynamic Product Ads）匹配率只有 40%。建 dataLayer schema 文件 + Schema validator GTM 變數（發現命名不對就在 DebugView 打紅字）後，三個月內匹配率拉到 92%，DPA ROAS 提升 2.1 倍。
dataLayer schema 要放在前端 repo 的 README（工程師看得到）+ Notion（行銷看得到）雙份。每個欄位註明：型別、範例、必填與否、什麼時機 push。沒有文件＝亂命名的起點。
完成驗證 · VERIFY
能在 LocalWP 站的 Console 看到 dataLayer 內容並解釋至少 3 筆的 event 值；能畫出 dataLayer → GTM → 平台 的資料流圖。
常見問題 · TROUBLESHOOT
dataLayer 一頁最多幾筆？
技術上無限制，但實務上越多瀏覽器越慢。一頁 50 筆以內健康；破 200 筆代表 push 邏輯有問題（可能 infinite loop）。
換頁後 dataLayer 會保留嗎？
不會。dataLayer 是 window 的屬性，換頁 window 重建。要跨頁傳值用 sessionStorage / localStorage / cookie / URL params。
可以直接操作 dataLayer 陣列嗎（push 以外）？
不建議。直接 splice / pop 會破壞 GTM 的監聽器。只用 push()。要『重置』用 GA4 的 ecommerce: null 模式（Module 5 會講）。
