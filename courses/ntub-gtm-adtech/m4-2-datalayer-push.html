MODULE 4 · SECTION 4.2

dataLayer.push 自訂事件實作

寫第一個 dataLayer.push——把點擊按鈕的行為變成 GTM 能消費的事件。

為什麼這很重要 · WHY THIS MATTERS
大部分非 Ecommerce 的自訂追蹤，本質就是一句 dataLayer.push。這節把這句話寫對。
STEP BY STEP · 本節重點
1
1 基本語法
window.dataLayer = window.dataLayer || [];
window.dataLayer.push({
  event: 'submit_contact_form',
  form_id: 'contact-main',
  form_location: 'footer'
});
第一行確保 dataLayer 存在（免外層未定義報錯）。event key 必填——GTM 靠這個欄位觸發 Custom Event Trigger。
event 名＝你在 GTM 建 Custom Event Trigger 時要填入的值。兩邊要完全一致（大小寫敏感）。
2
2 綁到使用者動作：以聯絡表單為例
WordPress Contact Form 7 提交成功後會觸發 wpcf7mailsent JS 事件。用 document.addEventListener('wpcf7mailsent', function(e){ dataLayer.push({...}) }) 在表單成功時 push。
不同 CMS／表單外掛事件名不同——Gravity Forms、Ninja Forms、Elementor Form 都有各自的 JS 事件，查文件即可。
3
3 GTM 端接住：Custom Event Trigger
GTM → Triggers → New → Custom Event → Event name 填 submit_contact_form → 儲存。建立一個 GA4 Event Tag，Event Name 用 generate_lead（GA4 推薦事件），params 用 {{DLV - form_id}}、{{DLV - form_location}}。
注意：dataLayer event 名（submit_contact_form）不一定等於 GA4 event 名（generate_lead）。前者是『合約名』、後者是『平台名』。
4
4 參數設計四原則
① 只 push 必要資料（避免一坨 debug 資訊）；② 命名 snake_case 一致；③ 值的型別統一（form_id 永遠字串，value 永遠 number）；④ 避免 PII（email、phone 除非明確要用於 enhanced conversions）。
常見錯誤：push 了一整個 user object 含 email／電話，這違反 GA4 TOS 且會被平台自動過濾掉。
5
5 Debug：驗證 push 真的被 GTM 收到
Preview Mode → 觸發一次表單送出 → 看左側事件列應該新增一行 submit_contact_form（非 gtm.* 開頭的事件）→ 點進去看 Data Layer 分頁 → 應該看到你 push 的 form_id / form_location。
看不到事件 90% 是 dataLayer.push 根本沒跑到——去前端 console.log 確認。
6
實戰案例｜SPA 路由變化漏追 70% 頁面
某 Vue.js SPA 站 GA4 page_view 只算進站那一次，之後使用者切到 10 個頁面都沒回報——實際頁面瀏覽量被低估 70%。修法：在 Vue Router 的 afterEach hook push {event:'virtual_pageview', page_path: to.fullPath}，GTM 用 Custom Event 觸發 GA4 page_view。部署後 PV 從 8 萬 → 27 萬／月，不是流量變多，是終於看到真實狀況。
React 用 useEffect(()=>{},[location])、Next.js 用 router.events.on('routeChangeComplete')、Nuxt 同 Vue。關鍵字：『framework + virtual pageview GTM』Stack Overflow 有現成範本。
完成驗證 · VERIFY
LocalWP 站成功用 dataLayer.push 追到 1 個非電商事件（表單、按鈕、影片），並在 GA4 DebugView 看到對應事件。
常見問題 · TROUBLESHOOT
為什麼要設 window.dataLayer = window.dataLayer || [] ？
保險。若 GTM script 還沒載入就 push，沒這行會報 TypeError。GTM 官方範例都這樣寫。
push 的值可以是函式嗎？
不要。dataLayer 值要 serializable（能 JSON.stringify）。函式雖技術上能 push 但會破壞 GTM 內部處理。
同一個事件 push 多次會怎樣？
GTM 會每次都 Fire 對應 Tag（正常）。但 GA4 會認為是多次事件——這時你要問自己『使用者真的做了多次嗎』，如果是 bug（e.g. 按鈕 double click）要在前端加 debounce。
