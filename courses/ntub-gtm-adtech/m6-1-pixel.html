MODULE 6 · SECTION 6.1

Meta Pixel 九大標準事件

PageView / ViewContent / AddToCart / InitiateCheckout / AddPaymentInfo / Purchase / Lead / CompleteRegistration / Search——九件覆蓋電商與 B2B。

為什麼這很重要 · WHY THIS MATTERS
Meta 的廣告演算法吃標準事件訓練；自訂事件很難被最佳化。先把標準事件做對，ROAS 就贏一半。
STEP BY STEP · 本節重點
1
1 九大標準事件分層
一般瀏覽：PageView（自動）、ViewContent、Search。購物動作：AddToCart、InitiateCheckout、AddPaymentInfo。轉換：Purchase、Lead、CompleteRegistration。每個事件綁 GA4 對應事件（view_item → ViewContent，add_to_cart → AddToCart）。
Meta 名稱用 PascalCase（Purchase），GA4 用 snake_case（purchase）——在 GTM Tag template 轉換就好，dataLayer 用一套即可。
2
2 必填參數：value / currency / content_ids / content_type
value：整筆金額（數字）；currency：ISO 4217；content_ids：SKU 陣列；content_type：'product' 或 'product_group'。缺 value 會變 $0 歸因；缺 content_ids 無法做商品目錄再行銷。
DPA（動態商品廣告）靠 content_ids + content_type 連回 Meta Catalog。沒對上就無法 DPA。
3
3 GTM 佈署路徑：自訂 HTML Tag
Meta 沒有官方 GTM Template（有社群版本），傳統做法是自訂 HTML Tag 貼 Pixel base code + fbq('track', 'Purchase', {value, currency, content_ids})。Trigger 綁 purchase Custom Event。
推薦用 Stape 或 Facebook Pixel by Simo Ahava 的 community template，比手刻更穩。
4
4 Advanced Matching：提升 EMQ 的 user data
Pixel base code 裡加 fbq('init', 'PIXEL_ID', { em: hashEmail, ph: hashPhone, fn: firstName });。這些 first-party user data 被 hash 後送 Meta 幫忙 match 到 Facebook 使用者。EMQ 6.0+ 是廣告效能良好的門檻。
email/phone 要用 SHA-256 hash 後再送，千萬不要送明文。GTM 可以用 Custom JS Variable 做 hash。
5
5 驗證：Events Manager Test Events
Meta Business → Events Manager → 選 Pixel → Test Events 分頁 → 輸入你的 LocalWP 站 URL → 操作一次購買 → 應看到 Purchase 事件即時出現 + 所有 params + Deduplication 欄位（為 6.2 CAPI 做準備）。
Test Events 與 Diagnostics 分頁是 debug 主戰場，不要只看 Overview（有延遲）。
6
實戰案例｜Meta Pixel 裝錯位置延後 8 秒觸發
某電商把 Meta Pixel base code 放在 </body> 前、PageView 事件又自己寫在按鈕 onclick，結果 PageView 平均 8 秒後才觸發——使用者很多在 8 秒內就關掉，Pixel 覆蓋率只有 62%。改放 <head> 內、PageView 由 base code 自動 fire 後，覆蓋率拉到 96%，受眾規模翻倍、廣告學習期更短。
Pixel base code 一律放 <head>，PageView 讓 base code 自動 fire。只有自訂事件（ViewContent, AddToCart）才需要另外觸發。位置錯＝無聲的覆蓋率掉點，後台不會警告。
完成驗證 · VERIFY
Meta Events Manager Test Events 看到你 LocalWP 站送出的 Purchase 完整 9 大事件至少 5 個，value 與 currency 正確。
常見問題 · TROUBLESHOOT
Pixel base code 要裝一次還是每頁？
一次。裝在 全站 template 即可（GTM 的 All Pages Trigger）。每頁不用重裝。
fbq is not defined 怎辦？
Pixel base code 沒載入或被 adblocker 擋。檢查 Tag 順序（base 先 Fire），推薦 Meta CAPI（6.2）做 server-side 備援。
Meta 的轉換數和 GA4 差很多？
歸因窗口不同（Meta 預設 7 天點擊+1 天檢視 vs GA4 的 Data-driven）+ iOS ATT 造成 Browser Pixel 漏事件——CAPI（6.2）補。差 20–30% 以內算正常。
