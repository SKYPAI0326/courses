MODULE 7 · SECTION 7.1

Google Ads Conversion + Enhanced Conversions

Conversion Action、Enhanced Conversions、離線匯入——讓 Google Ads 的 ROAS 看到 GA4 看到的那筆轉換。

為什麼這很重要 · WHY THIS MATTERS
Meta 靠 Pixel+CAPI 雙軌；Google Ads 靠 Conversion Tag + Enhanced Conversions。觀念類似、指令不同。
STEP BY STEP · 本節重點
1
1 Conversion Action 建立
Google Ads → Tools → Conversions → New → 選 Website → 輸入 Conversion name（例：Purchase）→ Category 選 Purchase → Value 選『Use different values for each conversion』（電商必選）→ Count 選 Every（電商）或 One（單次 Lead）。
Count=Every 的意思：同一使用者多次下單都算獨立轉換。錯設 One 會漏掉回購。
2
2 GTM Conversion Tag
GTM → Tags → New → Google Ads Conversion Tracking → 填 Conversion ID（AW-XXX）+ Conversion Label → Conversion Value = {{DLV - value}}、Currency = {{DLV - currency}}、Transaction ID = {{DLV - transaction_id}}。Trigger 綁 purchase Custom Event。
Conversion ID 和 Label 從 Google Ads 的 Conversion 設定頁『Use Google Tag Manager』那邊複製。
3
3 Enhanced Conversions：提升匹配率
GTM → 剛建的 Conversion Tag → 展開 Advanced settings → 勾『Include user-provided data』 → 新建一個 User-provided data variable → email、phone 等欄位 map 到對應 dataLayer 值（email/phone_number/first_name/last_name/address）。Google 會自動做 hash。
不要自己 hash 再給 Google——Google 要明文（它自己 hash）。和 Meta CAPI 的習慣相反，小心別混。
4
4 Google Tag（全域）
Google Ads 現在推薦用 Google Tag（gtag.js 整合）統一發送 GA4 + Ads 事件。GTM 佈署時用『Google Tag』類型 Tag（不是 GA4 Configuration）兩邊一起發。
舊版 GA4 Configuration Tag 仍能用，但新站建議直接用 Google Tag 整合。
5
5 離線轉換匯入（OCI）
CRM 裡的線下成交（例：電話成單、實體門市）可以用 Offline Conversion Import 回送 Google Ads。透過 GCLID（廣告點擊 ID）比對，讓 Ads 歸因到原本的點擊。做法：CRM 記 GCLID → 業務完成成單時 CSV/API 匯入 Ads。
GCLID 保存期最長 90 天——Lead 週期過長的 B2B 要特別注意這個窗。
6
實戰案例｜Enhanced Conversions 讓 Smart Bidding 變聰明
某保險比價網 Google Ads 轉換只有 GCLID + form_submit，實際成交是 7 天後電話達成，Google 看不到——Smart Bidding 一直優化錯的人群。導入 Enhanced Conversions for Leads：成交時把 hashed email 回傳 Google Ads，演算法把電話成交的歸因接回點擊那一刻。2 個月後 CPA 降 28%、優質 lead 比例從 18% → 34%。
Enhanced Conversions 前提：必須有使用者的 email／phone（hashed）。若你的 form 連 email 都沒收，這條路走不通。另一條路是用 Offline Conversions Import（GCLID 回傳）——成本稍高但同樣有效。
完成驗證 · VERIFY
Google Ads → Conversions → 你的 Purchase Action 顯示 Enhanced Conversions 診斷通過、至少 1 筆成功轉換資料。
常見問題 · TROUBLESHOOT
Enhanced Conversions 會把使用者資料送給 Google 嗎？
會（hash 後）。使用者需知悉——Privacy Policy 要寫清楚。歐盟需走 Consent Mode 管。
Google Ads 轉換數 vs GA4 轉換數不一致？
歸因模型不同（Ads 預設 Last click 或 Data-driven；GA4 可自選）+ 轉換時間窗不同（Ads 30/60 天 vs GA4 session-based）。差 10–20% 正常。
Google Tag 和 GA4 Configuration 有什麼差？
Google Tag 是新統一方案，一個 tag 同時送 GA4 + Ads。GA4 Configuration 是舊的只送 GA4。新建議用 Google Tag。
