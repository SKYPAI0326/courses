MODULE 1 · SECTION 1.3

建容器 × LocalWP × 首次 Preview

實作：開一個 GTM 帳戶、建容器、把 script 植入 LocalWP 的 WordPress 站、開 Preview Mode 看到 GA4 Config Tag 成功 Fire。

為什麼這很重要 · WHY THIS MATTERS
理論講完一天，實作才一小時就能跑通——這節把第一次的手忙腳亂降到最低。
STEP BY STEP · 本節重點
1
1 建立 GTM 帳戶與容器
到 tagmanager.google.com → 建立帳戶 → 帳戶名填公司或自己的名字 → 容器名填網站域名 → 平台選『網頁』 → 同意服務條款。系統會立即產出 GTM-XXXXXXX 與兩段 script（<head> 一段、<body> 一段）。
先複製這兩段 script 存到 Sheets 或筆記軟體——之後每個新站都要貼。
2
2 LocalWP 啟動 WooCommerce 測試站
LocalWP → Create a new site → 站名 acme-lab → 選 PHP 8.1 + MariaDB → 啟動後進 wp-admin 裝 WooCommerce 外掛 → 設一家虛擬商店（繁中、TWD）。這個站會跟完整整學期。
LocalWP 的 SSL 不可信——如果 GTM Preview 報錯 'blocked by mixed content'，先關 LocalWP 的 HTTPS 選項，之後再處理 SSL。
3
3 植入 GTM script（三種方法擇一）
(A) 手改佈景 header.php / footer.php：兩段 script 依指示貼進去。最直接但佈景主題更新會丟失。(B) 用 GTM4WP 外掛：只需填入 GTM-XXX，自動處理兩段 script + 解鎖 Ecommerce dataLayer（後面 Module 5 會用到）。(C) 用 Site Kit：Google 官方但只支援基本模式。推薦 (B)。
GTM4WP 是 WooCommerce 標配——先裝先爽。Module 5 直接就能用它送出 view_item / purchase 事件。
4
4 建立第一個 Tag：GA4 Configuration
GTM → Tags → New → 選『Google Tag』 → 填入你的 GA4 Measurement ID（G-XXXXXXX）→ Trigger 選 All Pages → 取名 GA4 Config - 主站 → 儲存。這是後續所有 GA4 事件的基礎，缺它一切不會動。
GA4 Measurement ID 在 GA4 後台 → 管理 → 資料串流 → 選你的網站串流 → 右上角 G-XXX 複製。
5
5 開 Preview，驗證 Tag 有 Fire
GTM 右上『預覽』 → 填入你的 LocalWP 站 URL → 點 Connect → 新分頁會打開你的站，角落出現 Tag Assistant → 回到 Tag Assistant → 看 Tags Fired 那欄應該有 GA4 Config - 主站 → 打開 GA4 即時報表應該看到自己的造訪 → 截圖存證。
看不到 Fire 的三個檢查點：(1) GTM script 在頁面上嗎（View Source 搜 GTM-）(2) container id 一致嗎 (3) GA4 串流正確嗎。
6
實戰案例｜Preview Mode 搶救上線前 1 小時
某 3C 電商在大促前 1 小時做最後一次檢查，Preview Mode 連上生產站後發現 add_to_cart 完全不觸發。Tag Assistant 顯示 trigger 有 fire，但 GA4 Tag 說 `Missing required parameter: items`。回去看 dataLayer push——前端工程師把 items 拼成 item（少 s）。5 分鐘改完、重推、Preview 再驗過、發佈——趕在大促開始前修好。
沒有 Preview Mode 這個場景就是：大促開始 48 小時後才從 GA4 報表發現『咦怎麼加購事件 0？』——已經流失一週的再行銷受眾。規則：每次發佈前 Preview 至少走一次完整購物流程。
完成驗證 · VERIFY
GTM-XXXXXXX + LocalWP 成功佈建截圖 + Preview Mode 顯示 GA4 Config Tag Fire 的截圖，全部存進本週資料夾。
常見問題 · TROUBLESHOOT
Preview 模式下 GA4 即時報表沒看到我？
（1）確認 Measurement ID 正確；（2）關掉所有廣告阻擋；（3）Chrome 開無痕重來。GA4 即時報表有 30 秒以內延遲是正常。
GTM4WP 裝了但 dataLayer 空的？
GTM4WP 設定頁需要手動勾『Track enhanced e-commerce』才會開始推 dataLayer。另外 WooCommerce 至少要有 1 個商品才會推事件。
可以一個容器追兩個網站嗎？
技術上可以，但強烈不建議。後續你會發現觸發條件、變數、命名全部要加 host 判斷，複雜度爆炸。一站一容器是紀律。
