MODULE 3 · SECTION 3.3

Debug 五層排查法

當事件沒進 GA4，從上到下檢查：dataLayer → GTM → 平台（GA4/Meta）→ Network 請求 → 後端日誌——五層任一層斷，事件就丟。

為什麼這很重要 · WHY THIS MATTERS
『客戶說 purchase 沒追到』這種問題，沒有系統化的排查就是三天浪費。這節給你一張固定流程表。
STEP BY STEP · 本節重點
1
1 第一層：dataLayer 有沒有 push？
開 Preview Mode → 操作一次購買 → 看 Data Layer 頁 → 應該看到 event: 'purchase' 的一筆 push。如果沒有——問題在前端／dataLayer，GTM 是無辜的。去修 GTM4WP 或 PHP hook。
DevTools Console 輸入 dataLayer 可以直接看完整陣列。比 Preview Mode 更直接但沒有 event 分段。
2
2 第二層：GTM Tag 有沒有 Fire？
Preview Mode → Tags 分頁 → 應該看到對應的 Tag 列在 Tags Fired 區。沒 Fire 的三個原因：(1) Trigger 條件沒命中（看 Trigger 那列為什麼 false）；(2) 另有 Blocking Trigger；(3) 工作區改動沒保存。
Tag 顯示 Fire 但 GA4 沒收到——往第三層走。
3
3 第三層：平台有沒有收到？
GA4 DebugView（Admin → DebugView）即時顯示當前裝置所有事件。Meta Events Manager Test Events 相同功能。LINE LAP / TikTok 也有各自的測試工具。看不到就代表事件被平台拒收（格式錯、token 錯、地區被擋）。
注意 DebugView 只在『GA4 debug mode on』時啟動——GTM Preview Mode 會自動開，但正式流量不會。
4
4 第四層：Network 請求有沒有送出？
DevTools Network 篩 collect（GA4）、tr（Meta）、conversion（Ads）。找不到請求代表 Tag 根本沒送；有請求但 4xx/5xx 代表平台端拒收（看 Response 找原因）。
常見 400 Bad Request 原因：(1) JSON 格式錯（items 不是陣列）；(2) 必填欄位缺（value、currency）；(3) 字串太長（>100 字）。
5
5 第五層：後端／Server Side 事件
若有用 Meta CAPI、GTM Server-Side、或 Make.com 代送——還要看後端 log。事件在 Browser Pixel 看不到但 Events Manager 看得到 server 事件，代表 Browser 那邊失敗但 server 成功（或反之）。
最煩的 bug 是 event_id 去重失敗導致看起來雙算——Module 6 會專門處理。
6
實戰案例｜Preview Mode 正常、生產掛掉的詭異案
某客戶反應『GA4 沒收到 Purchase』，但 Preview Mode 完全正常。折騰一天後發現：Preview Mode 會 bypass Consent Mode，但生產站啟用了 Consent Mode v2 且 analytics_storage 預設 denied——使用者沒按同意，GA4 tag 根本沒 fire。加上 CookieBot 設對後 GA4 事件立刻回來。
Debug 流程必備：除了 Preview Mode，還要在實際瀏覽器用無痕視窗+Tag Assistant Companion 檢查「真實使用者同意狀態下」tag 有沒有 fire。Consent Mode 是 2024 後最常見的『看起來好了其實沒收到』陷阱。
完成驗證 · VERIFY
拿一個真實案例（可自製缺事件情境）跑完五層排查，產出 1 頁 debug 報告：現象 → 第幾層卡住 → 根因 → 修法 → 驗證。
常見問題 · TROUBLESHOOT
Tag 顯示 Fire 但 GA4 DebugView 沒出現？
95% 是 Measurement ID 錯／串流選錯／GA4 debug mode 沒開（GTM Preview 會自動開，正常使用者要裝 GA Debugger 擴充）。
為什麼同樣的事件手機有送 桌機沒送？
通常是 (1) 響應式版面下某元素不存在（Click Trigger 找不到 selector）；(2) 手機版有廣告阻擋 App；(3) iOS Safari ITP 阻擋第三方 cookie。優先用 Server-Side 送。
事件送成功但 GA4 報表隔天才出現——正常嗎？
正常。GA4 標準報表有 24–48 小時處理延遲；即時報表只保留過去 30 分鐘。要看即時狀態用 DebugView 或 Realtime。
