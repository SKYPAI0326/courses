MODULE 8 · SECTION 8.1

Consent Mode v2 實作

GDPR 2024 新規——歐盟用戶來訪必須遵守 Consent Mode v2。這節把四旗標、default／update、CMP 整合拆解。

為什麼這很重要 · WHY THIS MATTERS
不遵守 Consent Mode v2 的後果：Google Ads Enhanced Conversions 被停用、Meta 歐盟轉換數被扣分。就算你不投歐盟客戶，架構上補起來也無害。
STEP BY STEP · 本節重點
1
1 四個 consent 旗標
analytics_storage：GA4 分析用 cookie 權限。ad_storage：廣告 cookie 權限。ad_user_data：把 user data 傳給廣告平台的權限（v2 新增）。ad_personalization：依 user data 做個人化廣告的權限（v2 新增）。四者各自可 granted / denied。
v1 只有前兩個；v2 加後兩個是因為 GDPR 對『廣告個人化』與『資料傳輸』要分開授權。
2
2 Default Consent：使用者還沒選之前
頁面載入時先 push gtag('consent', 'default', {analytics_storage: 'denied', ad_storage: 'denied', ad_user_data: 'denied', ad_personalization: 'denied', region: ['EEA'], wait_for_update: 500})。region 限定歐盟使用；wait_for_update 等 CMP 決定。
非歐盟流量不設 region 或設 region 為 ['US','TW']——那些地區預設 granted 即可。
3
3 Update Consent：使用者按同意後
CMP（同意管理平台，如 Cookiebot／Usercentrics／OneTrust）在使用者按『同意所有』後 push gtag('consent', 'update', {...: 'granted'})。GTM 內建的 Consent Overview 會立刻反映。
OneTrust 的 auto-blocking 模式會自動管 default＋update——適合大型站。小站用 Cookiebot 更便宜。
4
4 GTM Tag 的 Consent 設定
每個 Tag → Advanced Settings → Consent Settings → 設定該 Tag 需要哪些 consent 才能 Fire。GA4 Config 通常設 analytics_storage；Meta/Ads Conversion Tag 設 ad_storage + ad_user_data + ad_personalization。Tag 會自動在 consent denied 時不 Fire。
『Additional consent checks → Require additional consent』是保險做法——讓 Tag 層級的 consent 與全站設定雙重檢查。
5
5 Consent 驗證：GTM Preview 的 Consent 欄
GTM Preview → 每個 Tag 的 details → 看 Consent 欄顯示目前使用者的四旗標狀態。測：開無痕 → 載入頁面（Consent 應全 denied）→ 點 CMP 同意 → 應看到 Tag Fire + Consent 變 granted。
Consent 行為 bug 最難 debug——Preview 的這個欄位是救命稻草。
6
實戰案例｜Consent Mode v2 上線 EEA 流量腰斬的驚魂
某跨境電商 2024/3 Google 強制 Consent Mode v2 上線那週，歐洲流量的 GA4 事件掉 55%、Google Ads 轉換掉 60%。檢查：CookieBot 預設值設成 denied 但沒 flex（允許建模），加上 consent banner 同意率只有 30%——70% 訪客完全看不到。修法：(1) 改用 Advanced Consent Mode（未同意時送 cookieless ping）；(2) banner UX 優化（大綠色同意鈕、小灰色拒絕）→ 同意率拉到 58%。
未同意不等於完全沒資料——Google 會用 consent modeling把同意者的行為模式套到未同意者上做估算。但前提是：(1) 用 Advanced mode；(2) 要有足夠同意樣本（>1000/天）；(3) Consent Mode v2 正確 push ad_user_data + ad_personalization。
完成驗證 · VERIFY
LocalWP 站加上 CMP 外掛（Cookiebot 或 WP Consent API），Preview 顯示預設四旗標 denied、同意後 granted、Tag Fire 正確。
常見問題 · TROUBLESHOOT
非歐盟流量要不要用 Consent Mode v2？
非必要，但建議架構都補——未來歐盟規範擴散到其他地區（美國 CCPA、台灣個資法）不用重做。
consent denied 時完全沒資料嗎？
不是。Google 用『Consent Mode 建模』從 granted 使用者的行為推估 denied 的轉換——大約能補回 50–70%。無建模則全丟。
CMP 一定要付費嗎？
Cookiebot 有免費版（月 PV < 50000）。Usercentrics 與 OneTrust 都付費。WordPress 也有免費 WP Consent API 搭配。
