MODULE 5 · SECTION 5.2

WooCommerce dataLayer 實作

GTM4WP 外掛 + woocommerce hooks——把上節學到的 items array 真的產出來。

為什麼這很重要 · WHY THIS MATTERS
WooCommerce 是台灣最常見的自建站方案。這節用 GTM4WP 把整個電商 dataLayer 跑通，省下 80% 手刻工作。
STEP BY STEP · 本節重點
1
1 GTM4WP 設定核心勾選
WordPress 後台 → GTM4WP → Basic data 填 GTM-XXX；Integration → WooCommerce 勾 Track enhanced e-commerce + 勾 Track classic e-commerce（舊 UA 相容）；Events 勾 view_item / add_to_cart / remove_from_cart / begin_checkout / purchase。儲存。
Track enhanced e-commerce 是重點——沒勾 dataLayer 就不會自動 push 電商事件。
2
2 驗證：產品頁開 Console 看 dataLayer
打開任一商品頁 → Console → dataLayer → 找 event: 'view_item' → 展開 items 檢查 item_id（= product SKU）、price、currency='TWD'。沒看到代表 GTM4WP 沒正確裝或商品沒 SKU。
WooCommerce 商品編輯頁 → Inventory 分頁 → SKU 欄位。空的話 item_id 會 fallback 到 post_id 但不建議，請每個商品都設 SKU。
3
3 AJAX add_to_cart 的特殊處理
WooCommerce 預設『加入購物車』按鈕是 AJAX 模式（點了不跳頁）。GTM4WP 已內建監聽 added_to_cart jQuery 事件並 push add_to_cart dataLayer。驗證：點加車 → Preview Mode Data Layer 應出現新的 add_to_cart push。
自訂佈景若覆蓋了 AJAX 按鈕行為，GTM4WP 監聽可能失效——要手動加 listener。
4
4 purchase 觸發點：woocommerce_thankyou hook
GTM4WP 在 thank-you 頁（order-received）hook 裡自動 push purchase 事件。PHP 層級（非 JS）所以不怕被 ad blocker 擋。檢查：下一單訂單 → 到 order-received 頁 → Console 看 dataLayer 最後一筆應為 purchase 含 transaction_id=order_id。
若客戶是『付款 redirect 回站才到 thank-you』，redirect 失敗 purchase 就丟——要考慮 Server-Side 備援（Module 6.2）。
5
5 客製化：加自訂參數
要多 push 欄位（例：會員等級、是否首購）用 filter hook：add_filter('gtm4wp_datalayer_purchase', function($data){ $data['customer_tier'] = ...; return $data; }); 放在 theme functions.php 或自訂外掛。
filter hook 清單在 GTM4WP 文件 developers.google.com/tag-platform/tag-manager/web/datalayer。別硬改外掛本體——下次更新會被覆蓋。
6
實戰案例｜GTM4WP + Purchase 重複計算
某 WooCommerce 站用 GTM4WP 插件部署 Purchase 事件，但同時在 thank-you 頁手動放了一段舊的 gtag purchase code——每筆訂單被算兩次，月營收數字膨脹 100%。靠 GA4 DebugView 看到同一 transaction_id 被送兩次才發現。拔掉手動 code + 啟用 GTM4WP 的 Thank you page de-dup 設定後恢復。
GTM4WP 有『事件去重』選項（基於 transaction_id），設定裡要打開。另外 WooCommerce 訂單若改狀態（pending→processing→completed）可能觸發多次 hook，要選一個穩定點（通常是 processing 或 completed）送，不要兩個都送。
完成驗證 · VERIFY
LocalWP WooCommerce 成功在 Preview 裡看到完整四漏斗事件（view_item / add_to_cart / begin_checkout / purchase）並截圖。
常見問題 · TROUBLESHOOT
GTM4WP 的 dataLayer 和 GA4 Tag template 格式不一致？
有小差異——GTM4WP 是 GA4 舊格式。GTM 的 GA4 Event Tag template 開『Send Ecommerce data = Data Layer』會自動對應。若有欄位對不起來用 Variables 手動覆蓋。
Shopify 沒有 GTM4WP 對等的東西？
Shopify 走自己的 checkout.liquid + web pixels API，這課程不涵蓋。Shopify 的建議是用 Elevar 或 Analyzify 付費 app。
GTM4WP 免費版有限制嗎？
免費版可以完整 push 核心電商事件（view_item / cart / checkout / purchase）。Pro 版加商品曝光 impression tracking 與更細 user data——大部分店家免費版夠用。
