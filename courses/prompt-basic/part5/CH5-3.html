把提示詞做成你的工具

學會交辦，下一步是包裝。把你最常用的提示詞框架做成一個可以反覆填入、隨手分享的 HTML 小工具——用 AI Studio 或 Gemini Canvas，五分鐘搞定，不用學程式。

識別哪種日常工作場景適合包裝成提示詞工具

HOOK 破題

你現在已經會交辦了，接下來呢？

你學完了四欄框架、背景資料、三個技巧、格式控制、輸出驗收、借力改造。每次打開 ChatGPT 或 Gemini，你已經知道怎麼說話了。

但有一個問題沒有解決：你最常用的提示詞，每次都要重新組一遍。每週固定的進度回報、每次收到陳情的標準回覆、每次開會前的任務拆解……這些結構你早就熟了，卻還是要把四個欄位從頭填過。

這一節要解決這件事：把那些重複用的提示詞包裝成一個工具。不用學程式，不用找工程師，用 AI Studio 或 Gemini Canvas，五分鐘做出來。

CONCEPT 核心概念

什麼是「提示詞工具」？

簡單說：把一段常用的提示詞框架，做成一個有輸入格、即時預覽、複製按鈕的 HTML 表單。學員或同事打開這個 HTML 檔案，只要填幾個欄位，下面就自動組出一段完整的 AI 指令，直接複製貼入 ChatGPT 或 Gemini 就能用。

1
不用每次重寫
把框架固定下來，每次只填變動的部分——收件人、事項、背景——其餘自動補齊。
2
可以分享
存成一個 HTML 檔案，傳給同事直接在瀏覽器開啟。對方不需要懂提示詞也能使用。
3
五分鐘可以做
你不寫程式，你寫「指令」——用 AI Studio 或 Gemini Canvas，把需求說清楚，AI 幫你寫好整個工具。
怎麼做到的？
1
打開 aistudio.google.com（推薦，免費額度較高）；或 gemini.google.com → 右上角切換「Canvas」模式——兩者都能即時預覽生成的 HTML
2
把下面的 meta-prompt（工具製作指令）完整複製，貼入 AI Studio 或 Gemini Canvas，直接送出
3
等 AI 生成 HTML（約 30 秒），點「下載」或「複製原始碼」，儲存為 .html 檔，用瀏覽器開啟就能使用

QUIZ · 檢核題
Q1. 以下哪種工作情境，最適合包裝成提示詞工具？
正確。提示詞工具的核心價值是「重複使用」——固定結構加上固定格式的任務，最值得包裝。一次性或高度對話性的任務反而不適合。
提示詞工具的核心價值在於「重複使用」，最適合的是結構固定、每次只有少數變數不同的任務，例如每週固定撰寫的報告或回信。
Q2. 寫 meta-prompt 讓 AI Studio 或 Gemini Canvas 生成工具時，哪個部分最關鍵？
正確。AI 生成工具最需要的是明確的「要什麼功能」和「長什麼樣」——也就是本課一直強調的四欄框架中的「你要做什麼」和「要什麼樣子」。給角色有幫助，但不是最關鍵的。
AI 生成工具最需要明確的功能描述和設計要求——對應四欄框架的「你要做什麼」和「要什麼樣子」。越具體越好，例如「有三個按鈕」「有即時預覽」「背景色 #f5f3ee」。
