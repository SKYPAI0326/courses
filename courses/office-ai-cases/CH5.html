(05) · 加深章節 · 40 MIN

Prompt 設計決策樹

CH0 教了 5 大骨架（角色 / 資料 / 任務 / 輸出 / 驗證）——但每次新任務你還是會卡「該用哪些骨架、怎麼組」。
這 40 分鐘給你一張決策樹：6 個分支問題答完，prompt 雛形就出來了。

你會帶走
能用 6 個決策問題 3 分鐘內定出任何任務的 prompt 結構
能套用 5 種常見任務模板（寫信 / 分析 / 摘要 / 翻譯 / 腦力激盪）
能依「輸出不對」的 4 種症狀對症下藥微調 prompt
回辦公室遇到沒練過的新任務，能 5 分鐘寫出可跑的第一版 prompt
(01) · 5 MIN

5 大骨架 → 決策樹的起點

骨架不是「都要寫」、是「依任務選用」
CH0 的 5 大骨架是「最完整版」——但不是每個任務都 5 件全寫。簡單任務只用 2-3 個就夠、複雜任務 5 個全上。決策樹的作用就是幫你決定這次該用哪幾個。

5 個骨架的「必要程度」依任務性質而定：

任務類型 角色 資料 任務 輸出 驗證
簡單問答（單一事實查詢）——★——
輕度寫作（寫一封信 / 一段文案）—★★★—
分析任務（數據分析 / 整理）—★★★★
對外發送（給客戶 / 主管 / 法務看）★★★★★
創意 / 腦力激盪★—★★—
◎
5 大骨架的核心心法：你給 AI 越多正確的 context、越明確的任務、越具體的輸出要求，它預測得越準。但「越多」不等於「全寫」——對的東西寫對的量。下面決策樹幫你判斷「對的量」是多少。
(02) · 15 MIN

決策樹本體：6 個分支問題

3 分鐘把這 6 題答完，prompt 結構就出來了
每題回答後，會告訴你「該加哪個骨架 / 該加什麼具體內容」。題目順序不能亂——前面題目的答案會影響後面題目該問什麼。
Q1
這次的輸出「給誰看」？
給自己看（私人筆記 / 整理思緒）→ 不需「角色」骨架，輸出格式可寬鬆 給同事看（內部討論 / 提案草稿）→ 不需「角色」、但要明確「對象」（部門 / 職級）讓 AI 抓對語氣 給主管 / 客戶 / 法務看（對外或對上）→ 必須「角色」+「對象」，且後續 Q5 驗證必開
Q2
這次任務「需要 AI 知道你哪些既有資訊」？
不需要（純創意 / 通用問答）→ 跳過「資料」骨架，直接寫任務 需要少量（1-2 條背景：你的角色 / 公司類型）→ 在 prompt 開頭寫 1-2 句即可 需要大量（一整份文件 / CSV / 對話記錄）→ 必須「資料」骨架，用 [資料] 標籤明確圈出
Q3
任務本身「能用一句話講清楚動詞嗎」？
能（如「翻譯」「摘要」「分類」）→ 一句話寫任務即可 能但要加條件（如「翻譯成主管能看的繁體中文、保留專業術語」）→ 任務句後面加 1-2 條規則 不能、要分步（如「先 X、再 Y、最後 Z」）→ 拆成 3-4 個編號步驟，每步一句
Q4
輸出「長什麼樣」你能明確描述嗎？
能（有範例）→ 用範例（few-shot）：「請依以下格式輸出：『例子』」 能（有結構但無範例）→ 列結構規格：行數 / 字數 / 分區 / 必要欄位 不能（這次想看 AI 自由發揮）→ 至少寫「請給 3 個方案讓我選」，避免拿到不能比較的單一輸出
Q5
這個輸出「錯了會出事嗎」？
不會（自己看的草稿 / 內部討論）→ 跳過「驗證」骨架 會但能 catch（給同事看，他們會幫你檢）→ 加「請列出你不確定的地方」就好 會且要負責（對外發送 / 進報告 / 含數字）→ 必加「驗證」骨架：要求 AI 自報「引用了哪幾條 / 用什麼公式算 / 哪裡是推測」，並走 CH3 驗證卡 v2
Q6
這個任務「跑 1 次就完了還是會跑很多次」？
1 次性（一次性報告）→ 寫個能用的 prompt 跑完即可，不用美化 反覆會用（每週週報 / 每月分析）→ 把 prompt 做成「模板」：把每次會變的資料用 [佔位符] 框出來，存成可重用版 批次處理多筆（一次回 30 封信）→ 在 prompt 內加「請依序處理每一筆，輸出格式：『輸入 → 輸出』成對顯示」
★
6 題答完 = prompt 結構出爐。下面 5 個常見任務示範就是把這 6 題答完後組出來的——你回去也用一樣的順序套即可。第一次套會慢、跑 3-5 次後 1 分鐘就答完。
(03) · 15 MIN

5 個常見任務示範

每個任務都展示「決策樹 6 題答案 → 最終 prompt」
下面 5 個是辦公室最常見的任務類型。每個任務都展示：6 題答案速覽 → 組出來的 prompt。你回去遇到沒看過的新任務，就照這個流程跑一次。
任務 A · 寫信
寫一封 200 字內的延遲交付道歉信給客戶
Q1 給誰
客戶（對外）→ 要角色 + 對象
Q2 資料
少量：訂單延遲幾天 / 原因 / 補償方案
Q3 任務
一句話 + 規則：道歉 + 解釋 + 補償 + 後續
Q4 格式
結構規格：200 字內 + 4 段
Q5 風險
會出事（對外 + 含承諾）→ 必加驗證
Q6 重複
會反覆用→ 做成模板
【角色】你是專業的客戶服務代表，回信對象是企業採購窗口。 【背景】訂單 [編號]，原訂 [日期] 出貨，因 [原因] 延遲 [N] 天，補償方案為 [方案]。 【任務】寫一封道歉信，包含： 1. 真誠致歉（1 句，不卑微） 2. 解釋原因（1 句，不甩鍋） 3. 補償方案說明（1 句） 4. 後續預防（1 句） 【格式】繁體中文、200 字內、4 段分開、語氣專業溫暖。 【驗證】最後請列：你引用了哪些我給的事實、有沒有加入我沒給的承諾。
任務 B · 分析數據
分析一張 100 筆退貨清單找出退貨主因 Top 3
Q1 給誰
主管（對上）→ 要角色 + 對象
Q2 資料
大量（100 筆 CSV）→ 用 [資料] 標籤
Q3 任務
分步：分類 → 統計 → 排序 → 結論
Q4 格式
結構規格：Top 3 + 每項佔比 + 1 句結論
Q5 風險
會出事（含數字 + 進報告）→ 必加驗證
Q6 重複
每月用→ 模板化
【角色】你是擅長退貨分析的營運分析師，分析結果要給營運主管看。 【資料】以下是 [N] 筆退貨清單，欄位：退貨日期 / 商品 / 退貨原因 / 金額。 [貼 CSV] 【任務】請依序： 1. 把「退貨原因」依語意分類成 5-7 個類別（同義或近義合併） 2. 統計每類的筆數 + 金額 3. 列出筆數 Top 3 + 金額 Top 3 4. 寫 1 句結論：最值得優先處理的退貨類型是哪個、為什麼 【格式】表格：類別 / 筆數 / 占比% / 金額 / 占比%。結論獨立一段。 【驗證】最後請列： - 你共處理了幾筆（應為 N） - 你的「分類規則」是什麼（讓我能複核） - 哪些筆數是你不確定如何分類的
任務 C · 寫摘要
把一份 5 頁會議紀錄濃縮成 1 段 100 字摘要
Q1 給誰
同事 / 自己（內部）→ 不需角色，但說「對象」
Q2 資料
大量（5 頁紀錄）→ 用 [資料] 標籤
Q3 任務
一句話 + 規則：摘要 + 保留決議與 owner
Q4 格式
結構規格：100 字內 + 1 段
Q5 風險
能 catch（同事會看）→ 加「列你不確定的地方」
Q6 重複
每次會議→ 模板化
【對象】這份摘要要給沒參與會議的同事看，幫他們 30 秒內了解會議重點與待辦。 【資料】以下是 [日期] [主題] 會議紀錄，共 [N] 頁。 [貼紀錄] 【任務】請寫成 1 段 100 字以內的摘要，必須包含： - 核心決議（1 個） - 待辦事項 + 負責人（最多 3 條） - 下次會議重點（若有） 【格式】純文字 1 段、不分點、不超過 100 字、不要開場白。 【驗證】最後列出： - 紀錄裡你看不懂或資訊不完整的段落（如有） - 待辦事項中沒有明確負責人的條目
任務 D · 翻譯
把一份 500 字英文產品說明翻成繁體中文（給業務團隊用）
Q1 給誰
同事（內部業務）→ 寫對象
Q2 資料
大量（500 字原文）→ 用 [資料]
Q3 任務
一句話 + 規則：翻譯 + 保留專業術語 + 加註解
Q4 格式
結構規格：原文 + 譯文成對 + 術語表
Q5 風險
能 catch（業務會用）→ 加「不確定的地方」
Q6 重複
會反覆→ 模板化
【對象】這份譯文要給台灣業務團隊用於客戶提案。讀者是有產業背景但英文一般的業務人員。 【資料】以下是 [產品名] 英文說明： [貼原文] 【任務】請翻成繁體中文，規則： 1. 保留專業術語（如 SaaS / API / KPI），第一次出現時括號加 1 句中文白話解釋 2. 行銷話術（如 best-in-class / world-class）請轉成具體描述、不要直譯 3. 數字 / 單位 / 產品名 / 公司名一律保持原樣 【格式】先列「全譯本」，再列「術語對照表」（英文 / 中譯 / 白話解釋 3 欄）。 【驗證】最後列：你不確定如何翻譯的字句、有歧義的段落。
任務 E · 腦力激盪
想出 5 個提升舊客戶回購率的活動點子
Q1 給誰
自己 / 同事→ 不需角色但說「立場」
Q2 資料
少量：產業 / 客單價 / 預算上限
Q3 任務
一句話：給 5 個點子
Q4 格式
結構規格：每個點子含成本 / 預期效果 / 風險
Q5 風險
內部腦力激盪 → 跳過交差驗證；但因模板要 AI 出「成本/效果數字」，加最低「假設透明」要求避免學員把估算當預測用
Q6 重複
1 次性→ 不用模板化
【背景】我是 [產業] 公司的 [角色]，客單價 [X] 元，活動單次預算上限 [Y] 萬，目標是提升「過去 6 個月未回購的舊客戶」的回購率。 【任務】請給我 5 個活動點子，要求點子之間差異大（不要全部都是折扣促銷）。 【格式】每個點子用以下結構： - 點子名稱：[一句話] - 做法：[3-4 句具體執行] - 估算成本：[數字]（請標明：依據什麼假設算出，例：「假設 100 名客戶 × 200 元成本」；無法估算請寫「需實測」，不要硬猜） - 預期效果：[數字 + 單位]（請標明假設來源；無業界 benchmark 請寫「需 A/B 測試」） - 主要風險：[1-2 條] 【假設透明要求】最後請列「本提案所有數字的假設清單」（每個數字來自什麼假設），讓我能判斷哪些數字可信、哪些只是粗估。 請按「成本由低到高」排列。
📋
5 個任務的共通結構：6 題答完直接組裝。你會發現 Q1+Q2 變成 prompt 開頭、Q3 是中段、Q4 是格式段、Q5 是驗證段、Q6 決定要不要把可變部分換成 [佔位符]。練習 5-10 次後你不用再「想」，看到新任務直接組。
(04) · 5 MIN

輸出不對時的 4 種微調

對症下藥
prompt 寫好跑出來不滿意，不要整份重寫——看症狀加 1-2 句微調就好。下面 4 種症狀對應 4 種微調招式。
症狀 原因 微調招式（在原 prompt 末尾加 1-2 句）
太籠統 / 廢話多 沒給字數上限 / 沒禁空話 「每條不超過 30 字，不接受『持續推進』『相關單位協助』這類空話」
格式不對 沒給範例 / 沒寫「嚴格依照」 「請嚴格依照以下格式輸出：[範例]，不要增加多餘段落或前言」
編造資訊（幻覺） 沒給驗證要求 「每個結論後請註明『引自我提供的第 X 段』；沒有對應的請刪掉、不要補」
語氣不對 沒給對象 / 沒給範例語氣 「對象是 [誰]，語氣應 [專業 / 親切 / 直接]，避免 [客套話 / 行話 / 縮寫]」
★
微調原則：在原 prompt 後面用「請特別注意：[微調]」這個格式追加，不要重寫整份。AI 重跑時會把追加的指令視為「優先級最高」，比你整份重寫更穩。連續微調 3 次仍不對就停手——這時不是 prompt 問題，是任務本身不適合 AI 做（回 CH4 §05 停手訊號）。
(05) 本章帶走清單

確認這些都在你手上

工具 6 題決策樹（給誰 / 資料 / 任務 / 格式 / 風險 / 重複）— 新任務 3 分鐘定 prompt 結構
範本 5 個任務模板（寫信 / 分析 / 摘要 / 翻譯 / 腦力激盪）— 含 6 題答案速覽 + 完整 prompt，可直接改 [佔位符] 套用
速查 5 大骨架 × 5 種任務類型必要程度矩陣— 知道哪個任務該寫到哪幾個骨架
招式 4 種輸出不對微調招（太籠統 / 格式錯 / 幻覺 / 語氣錯）— 對症下藥，不重寫整份
