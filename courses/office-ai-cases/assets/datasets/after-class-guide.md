# 回家 30 分鐘復跑手冊（L5 升級包）

> 課堂上跑得動，回家用真實資料跑不出來？這是最常見的問題。
> 本手冊針對 4 大案例，每個給「5 步驟 30 分鐘」復跑流程，含資料整理 / 貼資料 / 跑 prompt / 追問 / 驗證。

---

## 共同前置：第一次跑前確認 3 件事（5 分鐘）

1. **資料份量**：你打算貼的資料有幾筆？幾欄？
   - < 50 筆／< 10 欄 → 直接貼文字
   - 50-300 筆 → 用 ChatGPT/Gemini/Copilot 免費版的 CSV 上傳
   - > 300 筆 → 分批（用 platform-guide 第 1 套 wrapper），或先抽樣 100 筆熟悉 AI 輸出
2. **資料敏感度**：是否含「客戶名單 / 採購金額 / 員工薪資」？
   - 是 → **不要上傳免費版 AI**，先去識別化（化名、模糊金額），或改用付費版商務帳號
   - 否 → 可上傳
3. **平台選擇**：依案例查 platform-guide.html 的「平台選擇建議」段落
   - 純文字（如週報）：任一平台都穩
   - CSV 稽核：首選 Gemini / ChatGPT
   - 雙 CSV 合併（電商）：首選 Gemini
   - 多素材組裝（簡報）：首選 Claude

## 案例 ① 週報生成 — 30 分鐘食譜

### 步驟 1：整理資料欄位（5 分鐘）
- 蒐集這週的工作記錄來源：LINE 工作群訊息 / 工作日誌 / 行事曆 / Todo
- 整理成 3 欄：**日期** / **事項** / **狀態（完成 / 進行中 / 卡住）**
- 至少 20 條，越完整 AI 越有東西可寫

### 步驟 2：貼資料（3 分鐘）
- 開啟 prompt-library.html 的 P1-1-3（進階版週報）
- 用「貼上資料」段落，把整理好的工作記錄貼進去
- 在 prompt 開頭加：「我這週是 [部門名稱] 的 [職位]，這份週報要交給 [主管職稱]」

### 步驟 3：跑 prompt（5 分鐘）
- 把 P1-1-3 完整貼進 ChatGPT / Gemini / Claude
- 等 AI 跑完（通常 30 秒）
- **不要急著貼到 Word**，先看一遍

### 步驟 4：追問修正（10 分鐘）
若結果有以下狀況，用對應追問：
- 太空泛 → 「請把第 2 點改寫，加入具體日期和成果數字」
- 漏掉某事項 → 「[事項] 你怎麼漏了？請補進去」
- 語氣不對 → 「請改成 [更正式 / 更謙虛] 的語氣，給 [主管職稱] 看的版本」
- 太長 → 「請壓縮到 200 字內，只保留最重要的 3 件事」

### 步驟 5：驗證輸出（7 分鐘）
- **必查 3 件事**：① 沒有亂編你沒做的事 ② 沒有遺漏卡住的事 ③ 下週計畫是你真的打算做的
- 通過後複製到 Word / Notion / Email
- **時間留 5 分鐘給自己再讀一次**，週報是要簽名負責的

## 案例 ②C 費用稽核 — 30 分鐘食譜

### 步驟 1：整理資料欄位（8 分鐘）
- 從公司 ERP / Excel 匯出本月費用單
- **必含欄位**：編號 / 日期 / 部門 / 項目 / 分類 / 金額 / 申請人 / 憑證類型
- **加一欄空白「備註」**（即使原始資料沒有也要留）
- 存成 CSV（UTF-8 編碼）

### 步驟 2：去識別化（5 分鐘）— 機密資料才需做
- 申請人名字 → 化名（員工 A、員工 B...）
- 公司名 → 化名（客戶 X、廠商 Y）
- 部門名可以保留（業務 / 行政 / 財務等通用）
- 金額**不要改**，AI 需要看到真實金額分布

### 步驟 3：跑 P1-2C-2 進階 prompt（5 分鐘）
- 開啟 prompt-library 的 P1-2C-2
- 上傳 CSV（Gemini 或 ChatGPT 免費版）
- **若用 Copilot**：加 platform-guide 的「案例 ②C wrapper」（強制先確認欄位再跑）

### 步驟 4：追問修正（7 分鐘）
- 若 AI 漏掉某類異常 → 「我覺得 [某筆編號] 有異常但你沒抓，請說明為何不算」
- 若 AI 抓太多誤判 → 「請只保留你 90% 確定的，其他列為『需人工複查』」
- 若 Block A / Block B 沒分欄 → 「請嚴格按 Block A 主要規則、Block B 輔助規則分欄輸出」

### 步驟 5：驗證輸出（5 分鐘）— **這步絕對不能省**
- 拿 Excel 開原始 CSV，依 AI 抓出的編號**逐筆對應原始資料**
- 看 AI 的「主要規則代號」是否合理（不要直接信備註欄）
- 不確定的筆數 → 在 Excel 標黃色，**主管最後簽核前由你人工複查**
- **AI 是稽核助手，不是稽核主管。簽核責任在你。**

## 案例 ③ 電商銷售 ABC + 長尾 — 30 分鐘食譜

### 步驟 1：整理兩份資料（10 分鐘）
- **訂單檔**：從蝦皮 / 自架網店 / momo / PChome 匯出本月訂單
  - 必含：訂單號 / 日期 / SKU / 商品名稱 / 數量 / 金額
  - 額外欄位（活動折扣 / 運費 / 退貨）建議**單獨另存一份**，先不丟給 AI
- **SKU 主檔**：商品基本資料
  - 必含：SKU / 商品名稱 / 分類 / 成本 / 售價 / 庫存
  - 沒有「分類」就先用「品牌」或「類別」代替

### 步驟 2：欄位對照（3 分鐘）
- 跑 prompt-library 的 **NB-3 引用式問答風格**（先問 AI 你的欄位 vs 課程範例對應關係）：
  ```
  我有兩份 CSV：
  訂單檔欄位：[列出你的欄位]
  SKU 主檔欄位：[列出你的欄位]
  請告訴我哪些欄位等於課程範例的哪個欄位，缺哪些建議補。
  ```

### 步驟 3：跑 P2-1-2（5 分鐘）
- 用 Gemini 上傳兩份 CSV
- 貼 P2-1-2 進階版（拆 A/B 詳列 + C 摘要 + 4 元長尾決策）
- 若用 Copilot / Claude：加 platform-guide 的「案例 ③ 雙 CSV 合併 wrapper」

### 步驟 4：追問修正（7 分鐘）
- ABC 數字怎麼算的 → 「請列出計算公式，與我每個 A 類 SKU 的累積佔比」
- 零銷售 SKU 沒抓出 → 「請列出 sku-50 中所有訂單檔沒出現的 SKU」
- 長尾決策太籠統 → 「請對 [某 SKU] 給出具體建議（清倉 / 觀察 / 曝光 / 停採購其一）+ 一句理由」

### 步驟 5：Excel 驗算（5 分鐘）— **這步絕對不能省**
- 在 Excel 用樞紐表：SKU + SUM(金額) → 排序，**前 5 名應該與 AI 的 Top 5 一致**
- 算總銷售額（SUM(金額)），與 AI 的數字對比
- **任何 5% 以上差異** = AI 漏算或算錯，重跑或追問
- 通過後才把分析貼進報告

## 案例 ④ 簡報組裝 — 30 分鐘食譜

### 步驟 1：先跑前 3 個案例的 -7 整理 prompt（10 分鐘）
- 用 P1-1-7（週報重點）、P1-2B-7（業績趨勢）、P1-2C-7（費用異常）、P2-1-7（電商分析）分別跑出各自的「投影片素材」
- 每個案例 1-3 張，總共 5 張左右
- **此時數字已驗證過、語氣已調整好**

### 步驟 2：把 5 張素材合併文字（3 分鐘）
- 新開一個 Notion / Google Docs
- 把 5 個 -7 prompt 的輸出貼進去，每段加標題（如「素材 ②B：業績趨勢」）
- 確認 5 段都不超過 150 字

### 步驟 3：跑 P2-2-2 進階組裝 prompt（5 分鐘）
- 開啟 prompt-library 的 P2-2-2（含防幻覺三鐵律）
- 把【資料】段落填入剛才合併的 5 段素材
- 用 Claude（語氣與來源約束最穩）或 Gemini

### 步驟 4：追問修正（7 分鐘）
- 沒標來源 → 「請對每個數字標來源（[來源：①/②A/②B/②C/③]）」
- 多出陌生數字 → 「[某數字] 是哪來的？請列出原素材，沒有就刪掉」
- 一句話結論不夠精確 → 「請給 3 個候選一句話結論，我選最好的」

### 步驟 5：人工最後校對（5 分鐘）— **這步絕對不能省**
- **逐頁**對照 5 張素材，確認每個數字都有來源
- AI 輸出的「可能需要人工確認的數字清單」逐一查證
- **不確定的數字標 [TBD] 而不是硬填**
- 貼進 PowerPoint / Google Slides 時，把 [TBD] 設成紅色，會議前補上

## 共同尾聲：跑出問題時的 3 個求救步驟

1. **回查 platform-guide.html 的「wrapper」段落** — 對應你遇到的問題（檔案太大 / 幻覺 / 格式不對 / context 不夠 / Block 分欄破格 / 雙檔合併混淆 / 反問空泛 / 簡報擴寫）
2. **切換平台再試一次** — Copilot 不順試 Gemini、Gemini 不順試 ChatGPT
3. **拆任務** — 一個 prompt 跑不出，把它拆成 3 個小 prompt 串接

**最後一個重要原則**：AI 是助手不是主管，**簽核責任永遠在你**。
任何要交出去的成果（週報、費用報告、電商分析、主管簡報），都要你自己看過一遍。
AI 加快草稿、AI 提醒盲點、AI 校對數字——但「我同意這個結論」的責任是你的。
