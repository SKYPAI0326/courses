PLATFORM GUIDE · 2026-05

平台速查 · 通用 Prompt + Wrapper + 限制速查

本課不指派「哪案例用哪平台」——你帶來什麼工具就用什麼。本頁是 6h 課程的「平台中性手冊」：通用 prompt 5 骨架、4 套 wrapper、5 家免費版能力、付費版必要情境、機密外洩實例。

本頁用途
課堂中：講師指向本頁、學員手機/筆電開著對照
課後：學員回工作場合套用前的「最後 30 秒檢查」清單
爭議時：當學員問「ChatGPT 還是 Gemini 比較好」、講師指這頁說「都可以、看情境」
(01)

通用 Prompt 5 大骨架

為什麼這很重要
這 5 個骨架是本課所有 56 套 prompt 的「結構底盤」。學員把這 5 個記住、其他平台限制都靠 wrapper 補強——這就是「通用 prompt + 平台 wrapper」的核心邏輯。
① 角色
告訴 AI 它是誰（含領域 + 資歷 + 風格）。
例：你是資深電商營運顧問，擅長從訂單資料找商品結構問題。
為何重要：讓 AI 進入專業視角，不要泛泛而答。
② 資料
給 AI 它要處理的東西（檔案 / 表格 / 文字）。
例：以下是 500 筆訂單明細（CSV 格式）...
為何重要：AI 不能讀心，必須給具體輸入。
③ 任務
明確說「要它做什麼」（一次一個任務，不要 4 件混在一起）。
例：請做 ABC 商品分類（依銷量），列出 A / B / C 三層各包含哪些 SKU。
為何重要：避免「五個失敗」中的「一次問太多事」。
④ 輸出
指定輸出格式（純文字 / Markdown 表格 / JSON / 字數限制）。
例：輸出格式：Markdown 表格（SKU / 分類 / 銷量 / 金額），最後加 100 字內結論。
為何重要：讓結果可以直接貼進你下游用的工具。
⑤ 驗證
要求 AI 列出「計算依據」+「不確定的地方」。
例：請列出 ABC 分類的閾值規則（百分比）+ 你最不確定的 3 個 SKU。
為何重要：降低亂編機率並留下人工核對線索、給你可驗證的線索。
★
口訣記法：「我是誰 → 給什麼 → 做什麼 → 要什麼格式 → 怎麼驗證」
5 個骨架不必每個 prompt 都寫到、但進階版 prompt（200 字以上的長 prompt）必須 5 個都含。
(02)

4 套 Wrapper — 平台適配補強

當你的「通用 prompt」遇到平台限制，加 1-2 句 wrapper 補強。這 4 套覆蓋本課所有案例會遇到的限制。

檔案太大 / 行數太多 / Context 不夠
我會分批貼資料給你，先確認你準備好接收第 1 批。 第 1 批：前 100 筆（[貼資料]） 分析完先給我這批的初步結論，等我貼第 2 批再繼續。
加在 prompt 開頭。適用情境：CSV > 200 筆、ChatGPT 免費版 context 16K 不夠。
結果有幻覺風險（數字 / 引用 / 算總）
最後請列出： 1. 你計算用的具體欄位 + 公式 2. 取樣行數（用了幾筆資料） 3. 你最不確定的 3 個推論 不要省略這 3 項，這是我驗證用的。
加在 prompt 結尾。適用情境：要 AI 算金額 / 找前三大 / 引用特定資料。
輸出格式不對（想貼進 Excel / Slides）
最後給我可貼進 [Excel / Slides / Markdown] 的純文字 / Markdown 表格格式。 表格欄位：[列出你要的欄位] 不要加任何前後說明，只輸出表格本體。
加在 prompt 結尾。適用情境：要把 AI 結果直接拷貝到 Excel / PPT，避免 AI 寫廢話。
想用同一個 prompt 跨平台對比
[原 prompt 內容] 備註：請用「最保守」的計算方式，不要創意發揮。 如果你不確定某個欄位的計算方式，直接回「我不確定」，不要猜。
加在 prompt 結尾。讓不同平台跑出來的結果落差變小、可比較。
◎
Wrapper 使用原則：不是每個 prompt 都要加 wrapper。只在你「跑出結果發現有問題」時，回頭加對應的 wrapper 再跑一次。Wrapper 多加會稀釋主任務、結果反而變差。

資料型任務的「驗證 wrapper」（Codex L4b 升級）

L4b 抽樣驗證發現：純文字案例（① 週報、④ 簡報）4 家平台都很穩；CSV/雙檔/強格式案例（②C 費用、③ 電商）落差明顯。針對這類資料型任務，在主 prompt 之外**強制加下方驗證 wrapper**，可拉平各家表現。

案例 ②C 費用偵測（Block A/B 分欄、①-⑦ 規則）
請先輸出「我將使用哪些欄位、哪些規則、總筆數」三行確認，再產出結果。 若無法完整遵守 Block A/B 與 ①-⑦ 格式，請停止並說明缺哪一項，不要改用自由摘要。
加在 P1-2C-2 開頭。Copilot / Claude 最需要（容易破格式）。
案例 ③ 電商雙 CSV 合併（sales × sku）
先確認兩份檔案角色：sales 檔是交易資料，sku 檔是商品主檔。請用 SKU 欄位合併後再算 ABC；不得只根據商品名稱或單檔資料推論。 若兩份檔案沒成功合併，請停止並回報「無法合併」，不要硬跑。
加在 P2-1-2 開頭。Copilot / Claude 最需要（容易把兩份檔案混讀）。
反問型驗證 prompt（P1-2C-6 / P2-1-6 / P2-2-6）
你的任務不是找更多內容，而是稽核前面的判斷。每一項都必須列： 1. 計算規則（公式） 2. 套用欄位 3. 樣本筆數 4. 至少 2 筆 row id / SKU / 頁碼 缺任一項就標「無法驗證」，不要含混過去。
加在所有風險檢查 prompt 結尾。Copilot 最需要（最容易回「看起來合理」）。
案例 ④ 簡報組裝（防新增 / 來源標註）
每一頁都要列「使用素材來源」與「引用的具體數字」。 沒有出現在素材中的數字不可新增；若需要估算，請標「估算」並放在備註，不得放入主結論。
加在 P2-2-2 結尾。Copilot 最需要（容易自行擴寫）；ChatGPT/Gemini/Claude 也建議加。
▣
平台選擇建議（L4b verdict 整理）：
  • 案例 ①（純文字週報）：4 家都穩；首選 Claude 或 ChatGPT
  • 案例 ②C（CSV 費用稽核）：首選 Gemini 或 ChatGPT；Copilot 需加 wrapper；Claude 對數字較弱
  • 案例 ③（雙 CSV 電商）：首選 Gemini；ChatGPT 次之；Copilot / Claude 需加合併 wrapper
  • 案例 ④（多素材簡報）：首選 Claude（語氣與來源約束最穩）；Gemini 次之
平台中立精神不變——學員可用任何平台跑，但 CSV/數字任務遇到問題時，第一個動作是換上方建議平台，不是放棄。
!
驗收標準鬆綁：學員跑同一 prompt 在不同平台會有落差。不以「答案完全相同」為通過標準，改以「是否能列出依據（公式 / 欄位 / 樣本）+ 是否能回查素材」為通過標準。

(04)

講師課堂話術 3 段

遇到學員問這 3 個問題時，講師可直接念這段話作為標準回答。

話術 1：為何不指定平台
「我教的是通用 prompt 寫法，因為三年內你的平台會換好幾次——但通用 prompt 不會過時。今天你用 ChatGPT 跑得起來、明天 ChatGPT 改規則了，你換 Gemini 用同一份 prompt 也跑得起來。平台會變，但清楚描述任務、資料、格式與驗證的方法可以跨平台使用。」
話術 2：免費版該注意什麼
「免費版有兩個現實限制——第一是訊息額度（ChatGPT 每 5 小時 10 則最容易踩到）；第二是 Office 內整合（Copilot 在 Word/Excel/PPT 裡的功能基本要付費）。

不過今天 4 個案例都是『瀏覽器內貼 prompt』的玩法，這些免費版全都跑得起來。真的要把錢花在哪？當你發現自己每週用 AI 超過 20 次、且常因為額度被擋下來，那就是該付 US$20/月的時候——US$20/月。」
話術 3：跑不出時怎辦
「跑不出來的時候照這個順序試：
1. 檔案太大？用 wrapper 1（分批貼資料）
2. 結果怪怪？用 wrapper 2（要 AI 列計算依據）
3. 格式不對？用 wrapper 3（指定 Markdown 表格）
4. 都試了還是不行？換另一個平台跑同一個 prompt——通常 ChatGPT 跑不動的，Gemini 32K context 可以；Gemini 跑出格式怪的，ChatGPT 比較聽指令。

記住：平台是工具、prompt 才是你的資產。」
(05)

機密外洩風險 3 個情境案例

這 3 個是為教學用情境，模擬企業常見風險。學員回去用 AI 前必須認知這些。

實例 1：員工把客戶名單貼到 ChatGPT 排序 → 5 個月後競爭對手挖角
某 B2B 公司業務把 200 個客戶名單（含聯絡人 / 電話 / 採購金額）貼到 ChatGPT 免費版「請幫我依潛在價值排序」。5 個月後該員工離職、競爭對手主動聯絡這 200 個客戶——AI 模型訓練 + 員工自帶資料離職，無法證明因果但客戶來源可疑。
避免做法：客戶名單一律先去識別化（名字改編號、金額模糊化）再上傳。
實例 2：HR 把員工薪資表上傳 AI 整理 → 內網匿名爆料
某中型公司 HR 把 80 人薪資表貼到 Gemini 免費版整理成「部門 / 職等 / 中位數」格式。3 個月後內網匿名平台出現「公司部門薪資範圍」精確爆料——可能是員工偶然在 AI 對話中看到、可能是模型訓練間接洩漏。
避免做法：薪資 / 績效 / 評估資料一律用付費版的 enterprise / business 方案（明確有資料保護承諾的）。免費版禁止上傳。
實例 3：把未公開財報貼進 AI 寫摘要 → 被視為內線交易
某上市公司中階主管把未公開 Q3 財報草稿貼到 AI 寫主管簡報摘要。雖然 AI 沒洩漏資料，但公司內控查到「未公開資料外傳行為」，視同內線交易未遂處理。
避免做法：上市公司、敏感行業、政府機關員工任何未公開資料禁止上傳免費版 AI。改用公司部署的私有 AI / Copilot for M365 Compliance 版本。
⚠
本課所有 dataset 都是虛構假資料（人名 / 公司 / 金額皆編造）。學員回去套自己工作場景時，務必先去識別化。如果你的工作涉及客戶 / 員工 / 財務敏感資料，本課的免費版玩法只能用「練習」、不能用「實戰」——實戰請走公司 IT 核可的工具路徑。
(06)

付費版必要情境 — 什麼時候真的該付 US$20

課堂上「我可以不付費嗎」是高頻問題。這 3 個情境是「真的該付」的判準。其他情境免費版都夠用。

情境 1：每週使用 AI > 20 次、且常因訊息額度被擋
💰 ChatGPT Plus US$20 / 月　or　Gemini Advanced US$20 / 月
如果你每天工作要用 AI 4 次以上、每週超過 20 次，免費版的「ChatGPT 5h 10 則」「Gemini 簡報 20 份」會經常打斷你。若常被額度中斷、可以評估付費版是否值得。
情境 2：要把 AI 整合進 Word / Excel / PowerPoint 工作流
💰 Microsoft 365 Copilot（商務授權，個人約 US$30 / 月、企業另計）
如果你的工作 80% 在 Office 內、想「在 Excel 裡叫 AI 修活頁簿」「在 Word 裡叫 AI 改段落」——這個是真的要付費。Edge / copilot.com 的免費 Copilot 做不到 Office 內整合。
情境 3：要處理敏感資料、需要資料保護承諾
💰 ChatGPT Team / Enterprise　or　Gemini for Workspace　or　Copilot for M365（Compliance 版）
客戶名單、員工資料、未公開財報——這些不能上傳免費版。需要付費版「enterprise / business / team」方案，這類方案明確承諾「你的資料不會用於模型訓練」。
💡
本課所有 4 個案例都用免費版可完成。如果你剛開始學 AI、且工作場景符合「練習 / 學習 / 非敏感資料」範圍——免費版就夠。等你發現自己每週撞限制 20 次以上，那就是「該付」的訊號。
