(01) · 10 MIN

AI 運作機制 — 3 個關鍵字搞懂

為什麼這很重要
很多人覺得 AI「聰明」或「笨」是看運氣——其實不是。AI 的行為由三個機制決定：它怎麼讀你的字（Token）、它能記多少（Context）、它為什麼會胡謅（Hallucination）。搞懂這三個，後面 5 小時的 Prompt 設計才有依據。
Token
AI 怎麼讀你的字
AI 不是用人類的「字數」讀文字，而是把文字切成 token——可能是一個英文短字、單字的一部分、一個中文字、幾個中文字、或標點符號。

2026 年的現代模型（GPT-5.5 / Claude Sonnet 4.5 / Gemini 2-3），中文常接近「1 字 ≈ 1 token」，但專有名詞、程式碼、表格格式會讓比例改變（舊 GPT-4 / GPT-3.5 約 1.3-2.0 倍）。

影響：prompt 太長或直接貼大量文字 / CSV 到對話框，會超過 context 限制或被截斷。大表格優先用「上傳檔案」讓資料分析工具讀，再請 AI 摘要 / 篩選 / 計算（這比貼文字省 token 也更穩）。
Context Window
AI 能記多少
AI 對話的「短期記憶」上限。2026-05 主流免費版：ChatGPT 16K、Gemini 32K、Copilot 較小、Claude 較大。

影響：超過 context 時 AI 會「忘記前面講的」。後面案例會教 分批貼資料 wrapper 處理大檔。
Hallucination（幻覺）
AI 為什麼會胡謅
AI 的本質是「預測下一個最像的字」——當它沒看過正確答案時，會「補一個聽起來合理的答案」。

典型場景：算錯加總、編造不存在的人名、捏造法規條文。後面每個案例都會教 結果驗證卡 抓幻覺。
Temperature（溫度）
AI 的「自由度」
控制 AI 輸出的隨機性。Temperature 高 = 創意多樣（適合寫文案）、低 = 結果穩定（適合數據分析）。

多數免費版不開放調整，但你可以用 Prompt 寫「請給確定的答案、不要創意發揮」達到類似效果。
★
記這句話就好：把 AI 理解成「根據上下文產生最可能答案的工具」——它能推理與整理，但不保證每次都正確——所以給它越多正確的 context、越明確的任務、越具體的輸出要求，它預測得越準。這就是 Prompt Engineering 的本質。
(02) · 10 MIN

Prompt Engineering 起因 — 5 個常見失敗示範

為什麼網路上一堆「會 AI 的人」效率高很多？不是他們聰明——是他們知道怎麼避開以下 5 個失敗。每個失敗都對應一個 Prompt 設計原則。

1
失敗 1：太短太籠統
❌ 「幫我寫週報」 → AI 不知道你做什麼工作、給誰看、要多長
✓ 「我是電商營運經理，把下面 50 條工作日誌整理成 3 段週報（150 字內），分主要成果/雜事/下週計畫，給直屬主管看」
原則：把「角色 + 資料 + 任務 + 輸出格式 + 對象」5 件事都講清楚
2
失敗 2：沒給「驗證要求」
❌ 「幫我分析這 200 筆費用找異常」 → AI 給你 5 筆異常，但不知道是不是 hallucinate
✓ 「...找異常，每筆異常請列出（編號 + 金額 + 為何判定異常 + 是依據哪個規則）」
原則：要求 AI 列出計算依據，降低亂編機率並留下人工核對線索
3
失敗 3：一次問太多事
❌ 「分析業績 + 找異常 + 寫主管簡報 + 預測下季」 → AI 每件都做一半、品質都不好
✓ 拆成 4 個 prompts，每次只做 1 件事，結果接起來
原則：一個 Prompt 解一個問題，複雜任務分階段
4
失敗 4：沒給範例
❌ 「整理成主管要的格式」 → AI 不知道你主管要什麼格式
✓ 「整理成以下格式：『[日期] [結論] [數據] [行動建議]』，每行不超過 30 字」
原則：給範例（few-shot prompting）比給形容詞有效 10 倍
5
失敗 5：不檢查結果就用
❌ AI 給的數字直接貼進報告 → 加總對不起來被主管抓包
✓ 用「結果驗證卡」4 欄打勾：總筆數 / 加總金額 / 前三大 / 異常規則
原則：AI 是助手，不是負責人——你才是負責的人，必須驗證
◎
後面 5 小時的 56 套 Prompt 都圍繞這 5 個原則設計。今天課堂結束你不必記住 5 個原則的名字——但你會把這 5 個原則的應用「透過反覆練習熟悉這 5 個原則」。
(03) · 5 MIN

風險與瑕疵 — 4 大避雷

把 AI 用在公司工作前，你必須知道這 4 個風險。記住：你才是負責的人，AI 出包，責任在你。

風險類型 嚴重度 具體場景 怎麼避
機密外洩 高 把客戶名單 / 員工薪資 / 未公開財報貼到免費版 AI，這些資料可能被用於模型訓練 客戶/員工/財務敏感資料一律先去識別化（換名稱、模糊金額）才上傳；或用公司付費版（💰 ChatGPT Team / Gemini Workspace / Copilot for M365 有資料保護承諾）
過時資料 中 問 AI「2026 年最新稅法是什麼」，AI 給的可能是 2024 年的資訊（模型訓練截止日） 涉及法規 / 政策 / 時事 / 即時股價的問題，不要信 AI 的回答，去政府官網 / 公司公告查最新；或用有 web search 功能的版本（ChatGPT / Gemini 部分免費功能有）
偏見 中 用 AI 篩履歷時，AI 可能會帶有訓練資料中的性別 / 年齡 / 種族偏見 涉及人事決策 / 評估 / 篩選的場景，AI 只當建議、不當決策；最後決策由人做、有書面理由
幻覺 高 AI 算 200 筆費用加總是 1,234,567 但實際是 1,289,432（誤差 4%）；AI 引用「2023 年 X 法規第 5 條」但實際沒這條 數字類結果一律用「結果驗證卡」4 欄打勾；引用法規 / 數據 / 人名一律自己再查一次
⚠
機密外洩風險最容易踩——你以為「就只是貼客戶名單請 AI 排序」很安全，但免費版資料使用政策各平台不同、可能隨設定改變，敏感資料不要上傳未經公司核可的第三方 AI。後面案例的 dataset 都是虛構假資料，你回去用自己工作場景時務必先去識別化。
(04) · 5 MIN

平台差異 — 5 家免費版 30 秒看懂

本課不指派「哪個案例用哪個平台」——你用任一主流 AI 工具都能跑完 4 個案例。這 5 分鐘只告訴你每家「擅長什麼、付費才有什麼」，後面遇到限制時你知道怎麼換工具。

ChatGPT
最普及、整合最廣
免費版能力：GPT-5.5 / 16K context / 每日上傳 3 個檔案 / CSV 約 50MB / 資料分析
主要限制：訊息額度動態調整（依政策與負載）上限
💰 Plus US$20/月：解額度限制 + Thinking 模式 + 更大 context
Gemini
Google 生態整合
免費版能力：32K context / 一次 prompt 最多 10 檔 × 100MB / 簡報生成額度依帳號與地區顯示為準 / 額度依帳號顯示為準圖
主要限制：Google Workspace 深度整合需付費
💰 Advanced：1M context + Deep Research
Microsoft Copilot
Office 整合（要付費）
免費版能力：Edge / copilot.com 可上傳 PDF/DOCX/XLSX/PPTX/CSV，單次最多 20 檔 × 50MB
主要限制：Word/Excel/PowerPoint 內整合需付費
💰 Pro / M365 Copilot：Office 內深度整合（價格依組織授權）
NotebookLM
本課唯一指定工具的場景
免費版能力：100 個 notebook / 每個 50 sources / 每日 50 chat / source-grounded 引用問答
適合：多份內部文件整合、會議紀錄摘要、引用式 QA
主要限制：Slide deck 標 limited
💰 Advanced：更多 sources / 更高額度
Claude.ai
長文改寫補位（非主線）
免費版能力：適合主管語氣改寫、報告草稿、長文摘要 / 中文質感較細
不推薦做：CSV / 大量數據分析（不如其他三家穩）
💰 Pro / Max：更穩定額度、Projects 功能
💡
本課的平台策略：核心 prompt 通用（你寫一份能在任何平台跑），平台限制用 1-2 句 wrapper 適配（這在每案例的「平台差異備註」段都會教）。

除了案例 ④ 結尾的 NotebookLM bonus 會明確說「請用 NotebookLM」之外，其他 4 大案例都不指派平台——用你手邊有的就好。

30 分鐘的概念段結束。接下來 5 小時你會：

  • 案例 ② 業務+財務雙合（75 min）：行程 + 業績 + 費用三件套，21 套 prompt
  • 案例 ③ 電商銷售分析（75 min）：500 筆訂單，學 ABC 分類 + 商品矩陣

◎
本課不繳交。學員上機演練的目的不是交作業——是讓你跑過知道「成果長什麼樣」。你帶回去的不只是「概念」，是「跑過一次的 prompt 庫」。
