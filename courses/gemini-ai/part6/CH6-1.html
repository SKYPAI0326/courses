PART 6 · 串聯 LLM 與上架
6.1

當單機工具不夠用：把 AI 裝進工具裡

你在 Part 1 到 4 做出來的工具，邏輯都寫死在程式碼裡——快、穩，但不會「判斷」。有些任務需要 AI 現場理解內容再給答案，這時候與其每次回聊天視窗重新描述一遍，不如把這個判斷力直接裝進一個工具裡，讓任何人打開就能用。

學完這節，你會有
分辨任務該用「不會思考」的單機工具、還是「會思考」的 AI app
用 AI Studio 的描述框生成一個內建 Gemini 判斷力的 app
理解 API key 在背後怎麼被安全管理，不會外洩到前端
(01)

情境：三個回聊天視窗的代價

假設你每天都要幫忙判斷一批客戶留言的情緒（正面／負面／中立），或者把一段會議逐字稿整理成摘要。這種任務不是「套公式算出來」就能解決，它需要 AI 讀懂內容、做出判斷。目前你唯一能做的，是打開 Gemini 聊天視窗，把指令重新打一次。

這麼做有三個代價：每次要重新描述指令——同樣的分類規則、同樣的輸出格式，每次都得重打一遍，或東翻西找之前存的 prompt；對話越長越被舊內容污染——同一個對話視窗聊到後面，前面聊過的話題會影響到後面的回答，你要的其實是「每次都乾淨重來」的判斷，不是延續對話；同事拿到你的 prompt 也不一定會用——你把 prompt 複製貼給同事，他還要自己開 Gemini、貼上、等結果，操作門檻沒有消失，只是換了個人做。

ℹ
共同解法：把「AI 判斷」包進一個工具的介面裡——使用者只要填欄位、按按鈕，AI 判斷的邏輯（連同你寫好的固定指令）在背後自動執行，介面固定、輸出格式固定，不需要每次重新溝通。這就是接下來要做的「AI app」。
(02)

核心概念：兩個工具的分工

到目前為止你認識的是 Gemini Canvas——生成單一 HTML 檔案、存在本機、打開瀏覽器就能用，但它沒有連線 AI 的能力，做出來的是「不會思考」的工具。當任務需要現場判斷（分類、摘要、生成文字），你需要另一個工具：AI Studio 的 Build 功能，它做出來的是 React app，內建呼叫 Gemini API 的能力，跑在雲端專案裡。

不會思考
Gemini Canvas
單檔 HTML，本機開啟即可用
免帳號、零成本
存檔即部署，沒有雲端依賴
沒有連線 AI 的能力
會思考
AI Studio Build
內建 Gemini API，能即時判斷內容
可分享網址、可正式上架
對話式修改，隨時能回滾版本
需要 Google 帳號與雲端專案

判斷準則很簡單：這個工具本身要不要「思考」？只是執行固定邏輯（計算、排版、排序）——用 Canvas。需要現場理解內容、做判斷、生成文字——用 AI Studio。

(03)

認識 AI Studio

入口在 aistudio.google.com，左上角選單分成三大類，先花一分鐘認熟位置，之後找功能不會迷路。

01
EXPLORE
Playground（跟 Gemini 對話測試）、History（過去的對話紀錄）。
02
BUILD
New app（開新專案）、My apps（管理你做過的所有 app）、Gallery（瀏覽別人做的範例 app）。這一節的重點都在這裡。
03
MANAGE
Dashboard、Documentation——帳號層級的總覽與說明文件，先知道它存在就好。
(04)

動手做：生成你的第一個 AI app

進入 Build 首頁，會看到「Build your ideas with Gemini」的描述框，跟著下面四步走完，就會有一個可以用的 AI app。

01
在描述框寫清楚 app 的用途與介面
例如：「幫我做一個貼上客戶留言、自動判斷情緒是正面／負面／中立的小工具，用卡片呈現結果」。也可以用 ⊕ 選單匯入現成檔案（GitHub、Drive、Upload Files、Camera）當起點。
02
等 Gemini 生成
左側是跟 Gemini 對話的欄位，右側即時顯示 Preview，生成過程可以看到進度。預設模型是 Gemini 3.5 Flash。
03
在 Preview 測試、用對話欄修改
直接在 Preview 操作測試效果；不滿意就在左側對話欄追加需求。每次修改都會留一個 Checkpoint，可以用 View differences 看差異，或用 Restore code from this checkpoint 整個回滾。
04
打開 Code 分頁確認程式碼
可以看到完整的 React + TypeScript 程式碼（App.tsx、components/、services/geminiService.ts），透過 @google/genai SDK 呼叫 Gemini。
(05)

背後機制：API key 去哪了

這個 app 要呼叫 Gemini，需要的 API key 是誰的？打開設定面板的 Secrets 分頁，會看到 GEMINI_API_KEY 這個項目，用下拉選單直接選用 AI Studio 提供給你的 Gemini API Key，也可以自己 Add secret 加其他金鑰。這把 key 只存在伺服器端，不會出現在前端程式碼裡，使用者打開你的 app 也看不到它。

ℹ
用量算在誰頭上：這代表 app 每一次呼叫 Gemini，用的都是你自己的免費額度（免費層有每分鐘／每日次數上限）。適合做內部小工具，不適合預期會有大量使用者同時打開的公開產品。
(06)

延伸提醒：把黃金公式寫進描述框

★
描述框也適用黃金公式：在 Build 首頁的描述框，把「角色＋任務＋限制＋呈現」直接寫進去，例如：「你是一個客服助理（角色），幫我判斷留言情緒分成正面／負面／中立（任務），只能輸出這三個分類其中一個（限制），用顏色卡片呈現（呈現）」——描述越具體，Gemini 生成的 app 越貼近你要的樣子。下一節看怎麼把做好的 app 分享出去或正式上架。
