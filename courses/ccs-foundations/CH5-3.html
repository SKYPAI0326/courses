範本改寫工作坊 — 同一模板套 5 個不同場景

能拿一個現成的提示詞範本，辨識出「可替換零件」與「不可動的骨架」，在 5 分鐘內改寫成自己場景的版本。

學完這章，你會
拆解任何範本的三層結構：骨架、可替換零件、場景補丁
在改寫時不動骨架，只換零件與加補丁
對照 5 個真實場景，看懂「改了哪些、沒改哪些」的差異
獨立完成一次分析類範本的改寫練習
(01)

為什麼「照抄範本」通常行不通

先猜再讀 · 01
「拿 CH4 範本直接照抄」最常失敗的原因是什麼？
想想「實務上的場景」跟「範本預設情境」有多少機率剛好吻合。
正解 B。範本是通用版，但你的場景永遠是特殊版——80% 用範本的工作都是「改寫」不是「照抄」。下面要拆開講改寫該動哪、不該動哪。

CH4 範本庫提供了 4 個場景模板，但你在實務上遇到的場景不會剛好是這 4 個。80% 的「用範本」都是改寫，不是照抄。

問題是：學生不會改。改出來常常改到骨架（把結構搞壞），或是沒改到位（只換了主詞，結果語氣完全不通順）。這章要解決這個問題。

📌
前置條件：本章預設你已熟悉 CH4 範本庫的基本結構，以及 CH5-1 的任務決策思路。遇到不熟悉的範本格式，先回去看 CH4。
驗收 · 對應 Q21考綱二
Kenneth 要做「美國西部 10 天旅程，含戶外景點 / 機場 / 租車」。下列哪個提示詞最像「為自己場景補完的版本」，最不容易失敗？
A。A 把「天數 / 地點範圍 / 活動類型 / 物流」四個維度都填滿——這就是「拿到通用範本後，把每個槽位都換成自己場景變數」的成果。B/D 漏了天數，C 漏了「景點接近機場」這個約束，照抄就是踩雷。
(02)

核心概念：範本的三層結構

先猜再讀 · 02
範本句子「你是一位行銷經理。請針對 [產品] 寫一則貼文。三段式：痛點 → 解方 → CTA。」哪一部分屬於「骨架（不該動）」？
「骨架」承載的是結構效能——拿掉它輸出品質會直接掉。
正解 C。三段式是骨架（決定輸出品質的結構）；A 是角色零件、B 是產品零件、D 是任務零件——三者都可換。下面三層結構表會把這件事系統化。

任何提示詞範本都可以拆成三層。理解這三層，你就知道改寫時哪裡可以動、哪裡不能動。

01
骨架（不動）
範本之所以有效的核心結構。包含：
  • 任務類型：「你是 X 專家」「請幫我 Y」
  • 輸出結構：「分三段」「先列要點再給細節」
  • 驗收標準：「字數不超過 300」「避免籠統敘述」
動了骨架 = 等於重寫，不叫「改範本」。
02
可替換零件（改）
依場景替換的變數。常見的零件有：
  • 角色：專家 → 老師 / 學長 / 部落客
  • 領域：行銷 → 財務 / 醫療 / 教育
  • 受眾：一般大眾 → 高中生 / 主管 / 同行
  • 語氣：正式 → 口語 / 活潑
  • 舉例風格：國際品牌 → 本土案例
03
場景補丁（加）
改寫最後補上的在地化條件。範本是通用版，補丁讓它貼合你的場景：
  • 「本校是私立大學，舉例請用我們能接觸到的場景」
  • 「我是新手，請避開專業術語，要解釋就附白話版」
改寫最後一定要問自己：「我的場景有什麼特殊條件範本沒涵蓋？」
骨架留、零件換、場景補
💡
記憶方法：把範本想像成一份合約範本。條款結構（骨架）不能亂改，但簽約方名稱、金額、日期（零件）當然要換。特殊附加條款（場景補丁）則視你的情況加入。
驗收 · 對應 Q20考綱二
「你是一位在目標受眾領域有豐富經驗的高級行銷經理」這句話，對應到提示詞的哪一個結構要素？
A。「你是 X 專家 / 經理 / 老師」屬於角色設定，是本章「可替換零件」中最常換的一塊。對齊到 Q20 的四要素模型：Role 設身份，Instruction 給任務，Context 給背景，Constraints 約束輸出。改寫範本要先認清每句話踩在哪一格。
(03)

示範：一個範本套 5 個場景

先猜再讀 · 03
「同一份行銷文案範本，套到『社團招生』vs『道歉信』兩個場景」，最該換掉的零件是？
「受眾」是決定整個 prompt 語感的關鍵零件。
正解 C。骨架（A/B/D）不動，受眾 + 語氣 + 補丁三個零件決定輸出的「氛圍」——這就是下面表格 5 個場景中變動的全部維度。

下面是一個來自 CH4 的行銷文案範本。我們要把它改寫成 5 個完全不同的場景版本，同時確保骨架（三段式結構、字數限制、要有實例）一字不動。

原範本（CH4 行銷文案類型） 你是一位 [行銷] 領域的專家。請針對 [產品] 寫一則 [文案]，
要求：
1. 三段式結構：痛點 → 解方 → 行動呼籲
2. 每段不超過 60 字
3. 口吻符合 [目標受眾] 的語感
4. 避免籠統形容詞，要有具體數字或實例

注意骨架（四條 requirement）在所有改寫版本中都保留不動。以下是五個場景的改寫對照：

場景 領域 產品 文案 受眾 場景補丁
社團招生 校園社團 攝影社 IG 限動文案 大一新生 加「用新鮮人會懂的語言，不要說術語」
打工履歷 求職 我自己 履歷自我介紹 咖啡店老闆 加「要凸顯抗壓與學習力，不要炫耀成績」
社群貼文 學習歷程 我的程式作品 LinkedIn 分享文 工程師前輩 加「承認新手身分，請教而不是展示」
問卷招募 學術研究 期末專題問卷 FB 貼文邀請 同校學生 加「強調只花 3 分鐘，結果會回饋給填答者」
道歉信 人際 弄丟朋友東西 道歉訊息 好朋友 加「要誠懇不要客套，承認錯誤不要找藉口」
ℹ️
關鍵觀察：每一個改寫版本，骨架（三段式、60 字、要有實例）都沒動，只改了零件跟加補丁。但產出的文案感覺完全不同——因為受眾、語氣、場景都換了。這就是改寫的精髓。
驗收 · 對應 Q25考綱二
Jasmine 用「為一家環保草坪服務公司製作 Logo」這個範本，產出太抽象。下列哪一個是「補上補丁、把場景變具體」的最佳做法？
A。「青草 + 鮮花 + 蜜蜂」是具象的視覺隱喻補丁，把「環保」這個抽象概念綁定到 AI 能畫出的具體元素。其他選項都是「再來一次」「更多」這種模糊指令——這正是本章一直警告的「沒補丁照抄」失敗模式。
(04)

動手練習：改寫一個分析類範本

先猜再讀 · 04
把分析類範本改寫成「分析我的期末報告初稿」時，你「絕對」要動的至少有哪兩格？
A/C/D 都是骨架，動了等於重寫。
正解 B。領域是零件、補丁是場景條件——兩者一起換才會貼合自己場景。骨架（A/C/D）一動就壞，這是動手練習的關鍵紀律。

現在換你練習。以下是一個「分析類」範本，用於文件摘要任務。請完成三個步驟：

練習用範本（分析類 · 文件摘要） 你是一位 [文件分析] 領域的專家。請閱讀以下 [文件]，
要求：
1. 找出三個最重要的核心論點
2. 每個論點用兩句話說明（不超過 50 字）
3. 最後給一句整體評語
1
標記骨架與零件
閱讀範本，用紙筆或心中標記：哪些是骨架（三要點、字數限制、評語結尾），哪些是零件（文件分析、文件這兩個詞）。
2
改寫為期末報告版本
目標：把範本改成「分析我的期末報告初稿，指出三個該改善的地方」。替換零件，並加上場景補丁（例如：你是大學生、寫作能力中等、報告主題是 OO）。
3
執行並對比差異
用原範本跑一次，用你改寫後的版本再跑一次，貼相同的一段文字進去。對比兩個輸出：哪個版本更貼近你的需求？差異在哪裡？
驗收 · 對應 Q37考綱二
你要請 AI 生成小說封面草圖，下列哪個提示詞包含「最完整的可替換零件」，最值得當作改寫範本的起點？
A。A 涵蓋了「視覺氛圍」需要的所有可替換槽位（書名、主題、背景、風格、內容）——這就是好範本的特徵：每個槽位都對應一個明確可替換的零件。其他選項要嘛資訊冗（藝術家姓名）、要嘛資訊少（頁數、尺寸），不利於改寫複用。
(05)

常見錯誤

先猜再讀 · 05
「拿主管報告範本去寫給同學看的分享文，產出滿是商業術語」——這屬於哪種範本改寫錯誤？
「商業術語」直接洩漏出原本針對的對象沒換掉。
正解 B。受眾沒換，零件留在「主管」上，整個語氣 / 詞彙 / 例子全跟錯目標。受眾是「幾乎每次都要換」的零件，這也是錯誤 02 的核心。
錯誤 1
改到骨架
現象：學生把「三段式結構」改成「想寫幾段寫幾段」。
原因：覺得骨架太死，想要更自由。
解法：骨架是範本之所以有效的原因。動骨架等於重寫，那就不叫「改範本」了。如果骨架真的不適合，應該換一個範本，不是硬改。
錯誤 2
沒換受眾
現象：套用「給高階主管看的報告」範本去寫「給同學看的分享文」，結果產出滿是商業術語。
原因：只換了主題，沒換受眾。
解法：受眾是關鍵零件，幾乎每次都要換。受眾決定了語氣、詞彙、舉例風格。
錯誤 3
漏掉補丁
現象：產出「技術上完美但跟你的場景格格不入」。
原因：範本是通用版，沒有補上在地化條件。
解法：改寫最後一定要問自己：「我的場景有什麼特殊條件範本沒涵蓋？」加一兩條場景補丁。
驗收 · 對應 Q47考綱三
Brianna 用「制定每週鍛煉計畫」範本，AI 給的計畫沒考慮她的時間表與身體限制。下列哪個改進方式最對應「補上場景補丁」？
A。「時間表 + 身體限制」就是場景補丁——通用範本沒涵蓋的個人特殊條件。漏掉補丁的後果就是「技術上完美但跟你格格不入」，這正是錯誤 03 的場景。
(06)

檢核題

先猜再讀 · 06
你發現範本的「骨架」根本不適合你的場景。最該做的事是？
「骨架不動」的反面意思就是——不適合就換，不要硬改。
正解 B。骨架承載結構效能，動它等於丟掉範本的核心價值。骨架對不上你的場景，代表你拿錯範本——換一份就好，不要硬改。
1. 請拿 CH4「會議紀要整理」範本，改寫成「整理社團幹部會議錄音為 IG 限動公告」的版本。你需要改哪些零件？加哪些補丁？
點此查看示範答案
零件替換：
領域：會議紀要 → 社群貼文；文件類型：文字紀錄 → 錄音轉文字；受眾：公司同仁 → 社員（大學生）；輸出格式：條列紀要 → IG 限動文案格式（短句、視覺化）。

場景補丁：
加「語氣輕鬆活潑，符合大學生社群風格」；加「重點只取行動項目和活動公告，不需要討論過程」；加「字數限制在 50 字以內，適合限動展示」。

骨架保留：原範本的「先找重點、再整理輸出」結構不動。
2. 為什麼「骨架留」是改寫最重要的原則？如果你真的覺得骨架不適合你的場景，該怎麼做？
點此查看示範答案
骨架承載了範本的結構效能（例如：三段式確保訊息層次清晰，字數限制避免冗長）。動了骨架等於丟了範本的核心價值，輸出品質無法保證。

如果骨架真的不適合你的場景，代表這個範本不是正確的範本。正確做法是：換一個更適合的範本，或是自己重寫 prompt。不要硬改現有骨架——那只是浪費時間又破壞結構。
驗收 · 對應 Q44考綱三
Nicole 改寫「分析類範本」協助她編輯英文議論文。下列哪一種改寫最符合「AI 當輔助、人類負責原創」的紀律？
A。「分析 + 提供建議」是 AI 編輯角色；C/D 都跨界變成代筆，違反「原創在人類、輔助在 AI」的範本改寫紀律。改寫範本不只是改字，更要守住「AI 該做什麼、不該做什麼」的邊界。
