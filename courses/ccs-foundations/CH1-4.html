CH 1-4

提示詞優化：RTSC 公式、角色設定與 JSON 輸出格式

優化不是調整措辭，而是讓提示詞結構化、可重用、輸出可預期。這章整合考綱三的核心技巧，從公式框架到具體輸出格式控制。

學完這章，你會
運用 RTSWC 公式設計高品質提示詞
掌握角色設定（Role Prompting）的效果與應用
理解 JSON 輸出格式的優勢與使用場景
區分圖表類型（散點圖、折線圖、圓形圖）的適用場景
(01)

RTSWC 公式：提示詞的完整藍圖

先猜再讀 · 01
「你是一位有十年行銷經驗的數位策略師」是 RTSWC 公式中的哪一段？
關鍵句型「你是一位...」——這在框架裡是哪一段？
正解 A。賦予身份 = Role。下面 RTSWC 5 格圖會把另外四段的判斷依據說清楚，並接到具體 prompt demo。

RTSWC 是提示工程的最佳實踐公式，將一個好提示詞所需的所有元素結構化，讓 AI 從「你是誰」到「你要怎麼做」再到「邊界在哪」都有明確指引。

RTSWC 公式結構
Role
角色
+
Task
任務
+
Skills
技能
+
Workflow
工作流程
+
Constraints
限制條件
Role
定義 AI 身份與專業背景
Task
明確描述需要完成的任務
Skills
指定 AI 需要具備的特定能力
Workflow
定義任務的執行步驟順序
Constraints
邊界條件：長度、格式、禁止事項

以下是一個完整的 RTSWC 公式範例（顏色對應各元素）：

「你是一位有十年行銷經驗的數位策略師，幫我為新款咖啡機撰寫三則 Instagram 貼文，你特別擅長用故事型文案引發情感共鳴。每則貼文請先寫標題，再寫正文（不超過 150 字），最後加上 3 個相關主題標籤。貼文語氣活潑但不要過度口語化。」
ℹ️
公式不必僵化使用：RTSWC 是框架，不是規定。簡單任務可以省略 Skills 和 Workflow，但 Role（角色）+ Task（任務）+ Constraints（限制）三個核心幾乎任何時候都應該包含。
驗收 · 對應 Q20考綱二
「我們需要為清單（連結）中的每個人生成電子郵件」這句話在 RTSWC / 四要素提示詞中對應？
C。「清單（連結）」這個資料來源是上下文（Context）：告訴 AI 該依據哪份數據工作。指令是「做什麼」，這裡的「依據此清單」是「給 AI 工作的依據」。
(02)

角色設定（Role Prompting）：讓 AI 成為專家

先猜再讀 · 02
想讓 AI 把複雜光合作用講給小學生聽，最有效的提示詞改寫是？
告訴 AI「對象是誰」會自動拉動它的詞彙、比喻、難度。
正解 B。明確的「受眾設定」會讓 AI 切換到對應的心智模型，比模糊的「簡單一點」精準十倍。下面有/沒有角色設定的對比卡會把這個效應視覺化。

給 AI 一個具體的角色身份，它的輸出會從「通用助理的建議」變成「領域專家的判斷」。這個技巧能提升輸出的專業性和個人化程度高達 30%（研究數據）。

沒有角色設定
「幫我寫一則廣告」
AI 給你一個中規中矩、像模板填空的廣告，語氣平淡，缺乏針對性。
有角色設定
「你是個超潮的行銷大師，幫我寫一則吸睛廣告」
AI 會用更具感染力的語氣、更大膽的表達方式，符合「潮」這個角色所呈現的風格。
📌
常考情境：Q47（Brianna 的健身計畫）、Q55（向小學生解釋光合作用）都考的是「受眾設定」的重要性。要讓 AI 對「兒童受眾」說話，提示詞就要明確寫出「兒童目標受眾」，而不是模糊說「解釋得更簡單一點」。
驗收 · 對應 Q47考綱三
Brianna 給 AI「制定每週鍛煉計畫」，計畫無法配合她的身體狀況與工作時間。最有效的下一步是？
A。個性化失敗的根因是「缺少受眾/約束條件」。把時間表與身體限制塞進提示詞，AI 才能輸出可執行版本。B 不改變輸入；C 偏離目標；D 自我犧牲。
(03)

JSON 輸出格式：結構化資料的力量

先猜再讀 · 03
要求 AI 輸出 JSON 格式，主要解決什麼問題？
想一下 Make.com、n8n、Python 拿到 AI 的輸出後要做什麼。
正解 B。結構化輸出讓自動化流程「不用 regex 解析自然語言段落」，這是 AI 接 workflow 的關鍵橋。下面四優勢卡會把使用情境列齊。

當你需要 AI 的輸出「被程式讀取」或「整合到工作流程」時，指定 JSON 格式是最佳選擇。JSON 讓每筆資料都有明確標籤，機器可以直接解析，不需要人工整理。

"貼文清單": [ { "編號": 1, "標題": "每個早晨都值得一杯好咖啡", "內文": "探索新款 XX 咖啡機，讓清晨的儀式感升級...", "標籤": ["咖啡控", "早晨儀式", "生活品質"], "適合平台": "Instagram" } ]

JSON 輸出的核心優勢：

結構清晰
鍵值對讓資料一目了然
每個欄位都有標籤，不需要人工從段落文字中提取特定資訊。
機器可讀
直接接入自動化工作流
JSON 可以直接被 Make.com、n8n、Python 等工具解析，無縫整合到自動化流程。
批量處理
同一格式的多筆資料
要求 AI 生成多個產品描述時，JSON 確保每筆資料的欄位結構一致，方便匯入資料庫。
減少歧義
精確控制輸出結構
若不指定格式，AI 的輸出段落結構可能每次不同，難以程式化處理。
驗收 · 對應 Q49考綱三
AI 給的 AI 趨勢內容太籠統，下列哪個 follow-up 提示詞最能拉到「具體、有結構」的回覆？
A。Drill-down 是針對初步回覆鑽進「具體範例 + 產業 case」，把空泛敘述具象化。B/C/D 都是換主題（規格/歷史/未來），不算把「機器學習突破」這個點挖深。
(04)

資料視覺化選擇：圖表類型快速判斷

先猜再讀 · 04
要呈現「2000-2005 西班牙與南非每年人口的歷史變化」，最適合的圖表是？
關鍵詞「每年」「歷史變化」——時間軸上的「持續變化」是哪種圖的強項？
正解 B。折線圖把時間 x 軸上的斜率（成長速率）視覺化。下面四圖表對照會把另外三種圖的適用場景也釐清。

Q45、Q56 考的是：給定一個資料分析目標，應該用哪種圖表？這是提示詞優化的延伸——你必須能指定「正確的視覺化類型」，AI 才能生成有意義的圖表。

散點圖（Scatter Plot）
探索兩個連續變數之間的關係
範例：消費者年齡（X）vs 購買產品數量（Y）。看相關性（正相關、負相關、無相關）。

考點：Q45
折線圖（Line Chart）
顯示數據隨時間的持續變化趨勢
範例：2000–2005 年西班牙與南非的人口變化。看增長/下降斜率。

考點：Q56（從橫條圖改為折線圖）
圓形圖（Pie Chart）
顯示各部分占整體的比例
範例：各產品線占總銷售額的百分比。注意：不適合時序資料。
橫條圖 / 長條圖（Bar Chart）
比較不同類別的數值大小
範例：比較各城市的銷售額。不適合展示相關性或時間趨勢。
💡
記憶規則：時序 → 折線；相關性 → 散點；比例 → 圓形；比較 → 長條。在提示詞中明確指定圖表類型，AI 才能生成符合分析目的的視覺化。
驗收 · 對應 Q45考綱三
要探討「消費者年齡與購買產品數量兩個變數之間的關係」，最該用的圖表類型是？
A。兩個連續變數的相關性 → 散點圖唯一解。圓形圖是比例、長條圖是類別比較、橫條圖只看單個變數分布——都不對。
