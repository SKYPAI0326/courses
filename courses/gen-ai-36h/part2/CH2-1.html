四大文字任務：
摘要 · 分類 · 改寫 · 潤稿

上班族 80% 的文字工作，本質都是這四個動作之一。很多人「寫 prompt 卻還是卡」的原因是：他們不知道自己要做的是哪一類任務。本節把這四類拆到骨子裡，每類給完整 prompt 範本 + 3 個業界變化 + 踩坑診斷。

SECTION 01 · 總覽10 分鐘

上班族文字工作的四種型態

把你一天做的文字工作攤開來看，大多會落到以下四類。先學會「辨識任務類型」，比直接學寫 prompt 還重要——類型不同，prompt 結構差很多。

改寫

真實情境 · PM 阿凱的週二下午
阿凱 14:00 收到工程團隊寄來的 30 頁 Sprint Review 報告。主管要 16:00 前看到「3 分鐘能讀完的給執行長版」——摘要。同時客服轉來 80 筆客戶意見問他能不能看出趨勢——分類。順便把自己的週報草稿改成「給業務部的版本」給另一個會議用——改寫。寄前最後一次檢查用詞、去掉 AI 味——潤稿。一個下午四類任務全遇上。不會辨識的人寫一個通用 prompt 然後花 2 小時修；會辨識的人開四個分開對話，40 分鐘全部完工。
核心觀念 · 為什麼不能用「一個萬用 prompt」？
每個任務需要 AI 用不同的「思考模式」。摘要要它「壓縮」，分類要它「聚類」，改寫要它「換衣服」，潤稿要它「微調」。混在一起問（「幫我整理這份文件」），AI 會給你「四不像」——摘要不夠短、分類不夠清、改寫沒改到、潤稿沒潤徹底。先判斷類型、再開對應 prompt，這是本節最重要的一課。
SECTION 02 · 任務 1 摘要20 分鐘
TASK 01 · SUMMARIZE
摘要：從長變短但不失真
最常見也最容易寫爛的任務。大多數人寫「幫我摘要這份文件」——AI 不知道你的「重要」是什麼，給你隨機版本。

2.1 · 摘要三要素

寫摘要 prompt 時必須給 AI 三個資訊：① 給誰看 ② 要多長 ③ 留住什麼重點。缺一個就會寫不到位。

① 對象
董事會？主管？新人？客戶？不同對象在意的點完全不同。
董事會在意「錢 / 風險 / 決策」；新人在意「流程 / 怎麼做」
② 長度
「500 字」「3 分鐘讀完」「1 頁 A4」都可以。越具體越好。
3 分鐘 ≈ 600-750 字；1 頁 A4 ≈ 400 字中文
③ 保留欄位
告訴 AI「這些一定要有」：數字、決議、責任人、時程、風險、下一步。
保留：預算、責任人、截止日
可砍：討論過程、發言者

2.2 · 三種典型摘要情境

「摘要」依對象不同至少有三種版本。下面三個完整 prompt 對應三種最常遇到的情境：

情境 A · 給董事會 / 高階主管
決策型摘要 · 300 字內
對象極少時間，要的是「能做 yes/no」的濃縮。
# 角色 你是策略分析師。 # 任務 把以下會議紀錄濃縮成「給 CEO 看的執行摘要」。 # 限制 - 對象：時間極少的 CEO，要做 yes/no 決策 - 字數：300 字內 - 必須保留：決議事項、預算數字、責任歸屬、下一步時程 - 可以砍掉：個人發言、討論過程、問候寒暄 - 不加入會議中沒提到的推測 # 輸出格式 ## 會議結論（一段話） ## 關鍵決議（1./2./3.） ## 預算與資源（金額 / 負責） ## 下一步（日期：xxx） # 素材 """ [貼上會議紀錄] """
情境 B · 給沒到會的同事
補課型摘要 · 500 字內
同事沒時間看 1 小時錄音，但要能「接得上」後續工作。
# 角色 你是會議書記。 # 任務 把以下會議內容摘要成「給沒到場的同事看的補課版」。 # 限制 - 對象：沒參與但需要掌握結論與脈絡的同事 - 字數：500 字內 - 結構：背景 / 主要決議 / 待解決事項 / 下次會議 - 不寫個人發言，只寫共識與決策 - 含「我怎麼接」的具體引導 # 輸出格式 # [會議主題] 摘要 · YYYY/MM/DD **參與**：xxx **背景**：[一段話] **主要決議**：1. / 2. / 3. **待解決事項**：- xxx **下次會議**：[日期 / 主題] **你該接的事**：[指引] # 素材 """ [貼上會議內容] """
情境 C · 給自己未來回顧用
備忘型摘要 · 含個人觀察
不只記結論，還記「我當下的判斷」——是個人知識庫素材。
# 角色 你是個人知識管理助理。 # 任務 把以下會議整理成「我的個人備忘錄」——不只記結論，記我的觀察。 # 限制 - 對象：未來的我（3 個月後回看） - 必須保留：客戶 / 同事說的具體句子（能讓我回想當下氛圍） - 特別標記：我「當下的判斷」、我「心裡有疑問的點」 # 輸出格式 ## 會議概況 ## 重要片段（原始引述） > "..." ## 我的觀察 - 當下我覺得 xxx - 我對 yyy 還有疑問 ## 待追蹤 - 未解議題 - 3 個月後回看時要問自己的問題 # 素材 """ [貼上會議紀錄] """
⚠️
摘要最常見陷阱：只寫「請幫我摘要這份文件」。AI 無從判斷你的「重要」是什麼——數字？決議？風險？行動？指定保留欄位才能避免漏掉關鍵。
SECTION 03 · 任務 2 分類20 分鐘
TASK 02 · CLASSIFY
分類：給 AI 一個「分類規則」
分類任務有兩種：你已經有分類框架 / 還沒有——兩種寫法完全不同。
✓ 場景 A · 已有框架
何時：你知道類別（如客訴分 5 類）。
Prompt 關鍵：把類別清單寫進 prompt，AI 只歸類不歸納。
品質高：有規則書，不會發明奇怪類別。
✓ 場景 B · 讓 AI 歸納
何時：資料是新的、不知道有哪些主題。
Prompt 關鍵：請 AI 先「歸納 3-5 類」再分類，設上限防止爆炸。
品質：需 review 調整。

3.1 · 場景 A：已有分類框架

# 角色 你是客服主管。 # 任務 把以下 [N] 筆客訴依下列分類整理： - 商品瑕疵 - 物流問題 - 客服態度 - 退款糾紛 - 其他（說明子類） # 限制 - 每筆附：分類 / 嚴重度 / 建議處理人 - 無法分類時放「其他」並標註原因 - 不編造客訴沒提到的資訊 - 嚴重度定義： * 高：含「退款」「賠償」「投訴」「負評」 * 中：明確抱怨但平穩 * 低：建議性 / 輕微不滿 # 輸出格式 | # | 客戶 | 分類 | 嚴重度 | 處理人 | 摘要 | 【客訴資料】 """ [貼上] """

3.2 · 場景 B：讓 AI 先歸納

# 角色 你是用戶研究員。 # 任務 分析以下 [N] 筆使用者意見，兩步驟： 步驟 1：歸納 3-5 個主題類別（不超過 5） 步驟 2：每筆意見對應類別 # 限制 - 類別名稱精準、不重疊 - 每類給：名稱 / 數量 / 代表引述 2 句 / 核心訴求 - 若無法歸類放「其他」並說明 - 附「建議優先處理的類別」+ 理由 # 輸出格式 ## 識別的主題 1. **[類別名]**（N 筆） - 引述 1：「...」 - 引述 2：「...」 - 核心訴求：[一句] ## 每筆分類 | # | 意見片段 | 類別 | ## 建議優先 [一段話] 【意見資料】 """ [貼上] """
⚡
進階 · Few-shot：丟給 AI 前先自己標 5-10 筆當示範。「以下 5 筆我已分好類，請按相同邏輯處理剩下 95 筆」——分類準度大幅提升。尤其當分類標準有主觀性時。
SECTION 04 · 任務 3 改寫20 分鐘
TASK 03 · REWRITE
改寫：有方向的轉換
改寫不是「換句話說」，而是依某個方向重構。必須明確指定「從什麼換成什麼」。

4.1 · 改寫的 4 個維度

維度 01 · 換受眾
同內容給不同人看：技術人員 → 業務 / 高層 / 客戶
「把技術規格改成給業務看的版本，重點放在能對客戶講的賣點」
維度 02 · 換語氣
正式 ↔ 口語、強硬 ↔ 委婉、生硬 ↔ 親切
「改成『更口語、更親近，但維持專業』的版本」
維度 03 · 換語言
中 ↔ 英、台灣 ↔ 香港 / 中國中文
「翻成英文，使用台灣商業界常用英文，避免英美口語」
維度 04 · 換格式
長文 → 條列、條列 → 表格、文字 → 簡報大綱
「改成 6 頁簡報大綱，每頁一個重點 + 3 個 bullet」

4.2 · 四維度通用範本

# 角色 你是中文文案編輯，擅長多語境切換。 # 任務 把以下內容改寫。 # 改寫指示 維度：[受眾 / 語氣 / 語言 / 格式 · 選 1-2 個] 從：[原本是什麼] 到：[要改成什麼] # 限制 - 保留所有事實（數字、時程、承諾） - 字數不膨脹超過 [20%] - 不加入原文沒有的資訊 - 不改變立場 # 輸出格式 直接給改寫成品，不列修改說明。 # 原文 """ [貼上原文] """ # 情境（若有） [對方是誰 / 場合 / 禁忌]
💡
多維度組合：常見複合改寫是「換受眾 + 換格式」——「把給工程團隊的技術文件（條列）改成給客戶的行銷文案（故事形式）」。雙維度一次指示即可。
SECTION 05 · 任務 4 潤稿15 分鐘
TASK 04 · POLISH
潤稿：去掉「AI 味」
大家都看得出「這是 AI 寫的」——因為有 6 個典型徵兆。

5.1 · AI 味的 6 個徵兆

✗ AI 味的六個徵兆
1. 起承轉合套詞：首先、其次、綜上所述
2. 過度形容詞：卓越、傑出、頂尖
3. 並列結構多：X 與 Y、A 及 B
4. 套話開場：在當今快速變化的時代
5. 結論空話：對未來具有重要意義
6. 敬語堆疊：敬請、惠請、不便之處
✓ 潤稿後的人味文
1. 直接切入，第一句講重點
2. 用具體例子（「比去年成長 23%」）
3. 短句、單一動作
4. 口語化開場（「這週我們決定⋯」）
5. 結論給「下一步行動」
6. 用一般禮貌語

5.2 · 完整潤稿 prompt

# 角色 你是中文編輯，擅長把 AI 文章改成有人味的版本。 # 任務 把以下文章潤稿，消滅「AI 味」。 # 限制（6 項全部消除） - 移除「首先 / 其次 / 綜上所述 / 整體而言」套詞 - 形容詞改成具體例子 - 並列結構改成單句 - 開場直接寫第一重點 - 結論段必須是「下一步行動」 - 客套敬語改一般禮貌語 # 要保留 - 所有事實、數字、人名、決議 - 原文立場 - 字數 ±10% # 不要做 - 增加新資訊 - 改變立場 # 輸出格式 直接給潤稿後成品，不列修改說明。 # 原文 """ [貼上原文] """
💡
自我檢查：潤完念一遍——像你會說的話就 OK；像演講稿就還要再潤。文字節奏對齊口語節奏是去 AI 味的標準。
SECTION 06 · 串接工作流15 分鐘

把四任務串成 pipeline

真正進階的用法——不是四任務各自跑，而是「串」起來。一份原料丟進去，AI 連跑四道工序。

原始逐字稿
→
摘要
→
分類待辦
→
改寫成 email
→
潤稿
# 一 prompt 跑完四道工 你是專案助理。 請依序處理以下會議逐字稿： 步驟 1（摘要） 把逐字稿濃縮成「會議結論」段落（200 字內），保留： 決議、數字、責任人、時程。 步驟 2（分類） 從會議萃取所有「行動項目」，依「我方 / 客戶方 / 待確認」三類整理。 步驟 3（改寫） 針對「我方」的行動項目，產出一封中文 email 給內部團隊， 宣布負責人與截止日。 步驟 4（潤稿） 把 email 潤成「直接、不囉嗦、無 AI 味」的版本。 # 輸出格式 【步驟 1：摘要】 ... 【步驟 2：待辦分類】 ... 【步驟 3：email 初稿】 ... 【步驟 4：email 成品】 ... # 素材 """ [貼上逐字稿] """
⚠️
兩種實作法：① 單一 prompt 四步驟（上面）——簡單情境。② 多輪對話——每步要人工確認，適合複雜情境。新手先用多輪，熟練後合併。
SECTION 07 · 踩坑博物館10 分鐘

四大任務的8 個典型踩坑

摘要 1沒指定保留欄位
症狀：摘要缺金額、或缺負責人、或缺時程——每次少不同東西。
修：必寫 「必須保留：金額 / 責任人 / 時程 / 風險」。
摘要 2還是太長
症狀：說「500 字內」AI 給 800；說「1 頁」給 2 頁。
修：雙重限制 「500 字內，每段 3 句內，總共 4 段內」。
分類 1類別爆炸
症狀：讓 AI 歸納出 15 類，看不出重點。
修：嚴格上限 「不超過 5 類，超過請合併」。
分類 2邊界模糊
症狀：同筆意見被分 A 又像 B，連 AI 都猶豫。
修：類別名互斥。如「客訴」vs「建議」改成「需立即處理」vs「未來改善」，用時間維度互斥。
改寫 1改成另一件事
症狀：「5nm 製程」被改成「性能卓越」——量化資訊跑光。
修：加保留 「量化數字必保留（5nm、23%、NT$100 萬），不泛化成形容詞」。
改寫 2字數爆量
症狀：原文 300 字改寫後 800 字。
修：明指 「字數不膨脹超過 20%」。
潤稿 1潤成白話失專業
症狀：潤後像聊天，但要寄客戶。
修：加對象 「對象 B2B 客戶，專業但不正式，像 3 年合作夥伴」。
潤稿 2套話還在
症狀：潤後句子順了但「首先 / 在當今」還在。
修：逐一列禁用詞 「不要用：首先、其次、綜上所述、在當今、整體而言」。
SECTION 08 · 課堂動手15 分鐘
PRACTICE 01 · 四任務各跑一次
挑你最近一週工作，每類找實例
題目：回想最近一週實際做過的工作，每類找實例，用本節 prompt 結構跑一次：
① 摘要：挑一份長文件跑「給主管看」版
② 分類：挑一批雜亂資料請 AI 歸納 3-5 類
③ 改寫：挑一份要轉發文件改成「給不同對象」版
④ 潤稿：挑一篇 AI 寫的跑 6 徵兆檢查
提示：不用四個都完整——每類最有感的情境即可。重點是親身感受「不同任務不同 prompt」的差異。
PRACTICE 02 · 串接 pipeline
一 prompt 完成整個會議流程
題目：找一份會議錄音逐字稿（或用 NotebookLM 快速生一份）。用 Section 6 完整 pipeline prompt 一次跑完四步驟。計時——從「貼逐字稿」到「拿到可寄 email」花多久。
提示：第一次某步不夠好（通常是分類或潤稿）別重跑整條——追問「請把步驟 2 重做，類別不超過 3 類」。這是串接工作流除錯技巧。
▸ 預期省時數據
傳統手動：摘要 15-20 分 + 整理待辦 10-15 分 + 寫 email 10-15 分 + 潤稿 5-10 分 = 40-60 分
Pipeline：貼稿貼 prompt 2 分 + AI 產出 1 分 + review 調整 5-10 分 = 8-13 分

省時 75-85%。這就是「串接」而非「單步」的威力。
SECTION 09 · 課後行動5 分鐘

建「任務類型 → prompt 範本」的自動反應

本節最值錢不是 4 個範本，而是「辨識任務類型」的直覺。下週刻意練習：

Day 1-2
