Part 2 · 帶走範本
PRAC 2 · Part 2 共 6h 的整合成果

辦公室文書工作包8 情境 Email 腳本 + 完整會議 Pipeline

這是 Part 2 的成果——兩件可立刻帶回公司用的工具：Tool A 給 Email 工作流的 8 種典型情境 4 回合對話腳本；Tool B 給會議錄音 → 紀錄 → 待辦的完整 pipeline；Tool C 整合進 Claude Project 的 SOP。每段都能一鍵複製。

怎麼使用：① Tool A 適合「我有要寫的 Email」時，打開對應情境的腳本，照 4 回合走。② Tool B 適合「我有會議錄音」時，按步驟接著跑 prompt。③ Tool C 教你把最常用的 prompt 固化成永久助手。
RO I 粗估：按此腳本實際跑過 10 次之後，你會省下每週 1.5-2 小時的文書時間——一年 80-100 小時。
TOOL A
Email 4 回合對話腳本 · 8 個高頻情境
上班族最常需要寫的 8 種 Email 情境。每張卡按「列方向 → 寫草稿 → 修細節 → 多版本」的 4 回合工作流（CH 2.2 教過）。用法：找對應情境 → 點每一回合的「複製」→ 貼 AI → 收到回應後貼下一回合。
情境 01 · 危機溝通
通知客戶案件延期
案子卡關需延後交付，要承擔但保留信任
1
回合 1 · 列方向
我要寫信給客戶 [姓名]，告知 [案件名] 因 [原因] 需延後 [天數]。 請列 3 種開場方向（直接認錯 / 中性說明 / 強調補救），各自適用情境與風險。
AI 給 3 種選項後，告訴它你選哪個。
2
回合 2 · 寫草稿
我選 [方向 N]。請寫完整草稿。 角色：8 年經驗的專案經理 任務：寫中文 email 通知延期 限制：250 字內、提 2 個補救方案、不用「萬分抱歉」 格式：主旨 / 正文 / 署名（[你的姓名]）
3
回合 3 · 修細節
整體可以，但： - 開場太鋪陳，砍成一句話直接講重點 - 第二段「綜合考量」改「我們的判斷」 - 結尾去掉「不便之處敬請見諒」
這步根據 AI 版本實際調整不滿意的地方。
4
回合 4 · 多版本
給我 3 個版本： A. 更正式精準 B. 更口語親近 C. 精簡 100 字內 我可以挑一個或拼裝。
情境 02 · 追蹤
追蹤未回覆的信件
寄出的信對方沒回，要禮貌地追，不顯急躁
1
回合 1 · 列方向
我寄了 [事項] 給 [對方]，已過 [天數] 沒回。 請給我 3 種追蹤方向（提醒 / 補充資訊 / 給對方下台階），各自用法與時機。
2
回合 2 · 寫草稿
我選 [方向 N]。 角色：業務專員 任務：寫追蹤 email 限制：100 字內、不顯急躁、提供「需要更多資訊嗎」下台階 格式：主旨 [Re: 原信主旨] - 追蹤 / 正文
3
回合 3 · 修細節
調整： - 開頭不要「再次聯繫」套話 - 中段加「若需要更多資料我隨時可提供」 - 結尾不要「再次叨擾」
4
回合 4 · 多版本
給我 2 版： A. 偏正式（上下游廠商） B. 偏口語（已熟識的客戶）
情境 03 · 業務
回客戶詢價（含產品說明）
客戶來信問報價，要快速、專業、含後續步驟
1
回合 1 · 解讀來信
客戶來信如下： """ [貼上客戶來信] """ 先做： 1. 萃取客戶關心的 3 個重點 2. 識別未明說但需釐清的問題 3. 建議我回信要強調哪 2 個賣點
2
回合 2 · 寫草稿
基於以上分析，寫回信。 角色：B2B 業務經理 任務：回覆詢價，提供 [產品/服務] 報價說明 限制：250 字內、必含「報價區間 + 交期 + 下一步行動」、語氣正式但不冷 格式：主旨 / 正文 / 署名 [你的姓名+職稱]
3
回合 3 · 加賣點
第二段加一句強調 [賣點]，但不寫成廣告詞。 中段補「適合的客戶情境」一句話讓對方覺得對到他。
4
回合 4 · 多版本
給我 2 版： A. 含詳細報價數字（已決定採購的客戶） B. 含「需 30 分鐘電訪確認需求才能精準報價」（還在比價的客戶）
情境 04 · 向上溝通
內部請主管做決策
事情卡在主管，需他決定，要清楚但別命令
1
回合 1 · 整理脈絡
我需請主管 [姓名] 決定 [事項]。 背景：[背景] 已知選項：[A / B / C] 我傾向：[X]，理由：[原因] 整理成「給主管看的決策請求」結構： - 一句話核心問題 - 3 個選項各自取捨（成本 / 風險 / 時間） - 我的建議與理由 - 需主管在哪時間點決定
2
回合 2 · 寫 email
基於以上寫中文 email 給主管。 角色：直屬下屬 任務：請主管做決策 限制：300 字內、不「煩請」「敬請」、結尾明確「希望 X 月 Y 日前回覆」 格式：主旨 [決策請求] xxx / 4 段（問題 / 選項 / 建議 / 時程）
3
回合 3 · 修語氣
調整： - 我的建議那段，加「如果採取 X，預期能 [量化效益]」 - 時程改直接：「為配合 [時程]，希望你在 [日期] 前回覆」 - 主旨改具體：[決策請求] [事項] - 需 X/Y 前回覆
4
回合 4 · 反向檢查
換從主管角度讀這封信： - 他會抓到什麼點？ - 他可能不滿意 / 會反問什麼？ - 我有沒有遺漏關鍵資訊？ 如有風險，幫我預先補強。
情境 05 · 婉拒
拒絕客戶要求 / 不合理請求
客戶要超出範圍的服務 / 折扣，要拒但不傷關係
1
回合 1 · 找替代方案
客戶 [姓名] 要 [不合理請求]，理由是 [客戶說的]。 我不能答應，因為 [我方原因]。 幫我想： - 3 個「替代方案」可以提供，部分滿足客戶 - 拒絕的核心話術（不傷感情） - 避免讓客戶覺得「被一刀切回」的措辭技巧
2
回合 2 · 寫信
寫拒絕信。 角色：業務主管 任務：婉拒客戶請求 + 提供 2 個替代方案 限制：250 字內、不要先講「無法」再講替代（順序倒過來：肯定關係 → 提替代 → 解釋原要求限制） 格式：主旨 / 正文 / 署名
3
回合 3 · 軟化
這封太硬。調整： - 第一段加一句感謝對方提出需求 - 解釋限制用「業界慣例」「公司政策」這類非個人化說法 - 結尾留一個「未來合作」鉤子，不關門
4
回合 4 · 預測反應
客戶收到這封信，最可能 3 種反應是？ 針對每種，我該準備什麼後續話術？ 給我「應對劇本」表格：客戶反應 / 我方回應 / 進一步動作
情境 06 · 開發
冷開場 · 第一次聯絡潛在客戶
想開發新客戶，對方完全不認識你，要讓人想回信
1
回合 1 · 做功課
我想聯絡 [公司名] 的 [職稱 · 人名]。他們做 [業務]。 我方能提供 [服務]。 幫我想： - 他們公司的可能痛點 3 個（從他們產業推測） - 我方哪項能力最能解他們的點 - 開場「鉤子」句 3 種（不是廣告式，是有價值的觀察）
2
回合 2 · 寫冷開場信
用第一個鉤子寫冷開場信。 角色：同行或異業的資深同業（看 [服務] 是什麼） 任務：第一次聯絡，目的是讓對方願意 15 分鐘視訊 限制：150 字內！不要介紹公司、不要「你好 我是某某」自我介紹開場、不要廣告詞、開頭就是「有價值的觀察」 格式：主旨（引發好奇、不是廣告） / 正文 / 簡短署名
3
回合 3 · 主旨測試
給我 5 個不同的主旨（不是 3 個），測試哪個最可能被打開： - 含問號的 - 含具體數字 / 名字的 - 很短（< 15 字）的 - 看起來像內部 email 的 - 含對方公司名的 各附「為什麼這招會有效」的理由。
4
回合 4 · 多版本
給我 2 版： A. 非常短（80 字內）+ 最強鉤子 B. 略長（150 字）+ 2 個鉤子 + 具體 CTA 我會都寄出測 A/B 哪個回覆率高。
情境 07 · 內部協調
跨部門找人協助
你的專案需要別部門幫忙（對方沒義務），要讓人願意幫
1
回合 1 · 找共贏點
我在做 [我的專案]。需要 [對方部門]的 [人名] 幫忙 [具體事項]。 先幫我想： - 對他的真實價值是什麼？（不只「幫忙」這個理由） - 可能他會擔心的 3 件事？ - 我可以回報的 2 件事（讓這不是單方請求）
2
回合 2 · 寫信
基於以上寫 email。 角色：平輩同事 任務：請跨部門協助 限制：200 字內、具體說「要他做什麼 / 花多少時間 / 截止日」、不模糊說「需要你幫忙」、含回報 格式：主旨（具體、能被他看出要他做什麼） / 正文 4 段（請求 / 為什麼找他 / 對他的價值 / 時間與回報）
3
回合 3 · 減阻力
調整讓對方更容易答應： - 把「需要幫忙」的範圍縮到最小（他具體要做什麼，30 分內能做完的事） - 附上「若他說不行」的備選指示 - 提供「不想幫也沒關係，我這邊有其他辦法」的退路（反而提高答應率）
4
回合 4 · CC 是否加
分析一下：這封信我要不要 CC 他主管？CC 會讓他： - 更有壓力答應（可能正向） - 或覺得被施壓不爽（可能負向） 根據 [關係狀況]，給我建議 CC 或不 CC，以及「不 CC 版本」和「CC 版本」的措辭差異。
情境 08 · 道歉
對方已被惹毛的道歉信
你或同事搞砸了，客戶 / 主管已經不爽，需要挽回
1
回合 1 · 定錯的深度
發生了 [事件]。對方 [客戶 / 主管] 的反應是 [生氣 / 失望 / 威脅退單]。 幫我分析： - 這是哪種等級的錯？（一般失誤 / 結構性問題 / 重大違約） - 對方真正不爽的是什麼？（錯本身 / 過程態度 / 信任感） - 道歉的「核心」應該是什麼？（錯誤 / 影響 / 補救） 依此決定道歉的「深度」。不同深度措辭完全不同。
2
回合 2 · 寫道歉信
依分析寫道歉信。 角色：直接負責人（不是組織代表） 任務：正式道歉 + 具體補救 + 取信行動 限制：不寫「抱歉」三個字（太輕）、用「我們做錯了」具體承認、不辯解、不找藉口、必含 3 個具體補救行動與時程 格式：主旨（含「已啟動補救」這類字） / 正文 5 段（承認 / 具體做錯什麼 / 影響評估 / 補救行動 / 後續追蹤）
3
回合 3 · 真誠感檢測
用「嚴格顧問」視角重讀這封信，找出 3 個「讀起來像推託 / 公關話術」的地方，改掉。 標準：真誠的信能讓對方即使還生氣，也感覺到「你有理解問題」。
4
回合 4 · 後續劇本
客戶收到這封可能的 3 種反應： A. 接受但提條件 B. 仍然生氣要求更多 C. 冷處理不回 每種給我「下一步該做什麼」的劇本，含第二封後續信的草稿（若需要）。
TOOL B
會議 Pipeline · 錄音 → 紀錄 → 待辦 → Email
完整 5 步流程，從一個會議錄音檔開始，產出可以直接寄給與會者的「會後 email + 待辦表」。每步附完整 prompt 可一鍵複製。建議工具：步驟 1 用 NotebookLM；步驟 2-5 用 Claude（接續對話，不要重打）。
傳統 vs Pipeline：1 小時會議傳統手動整理 1 小時；Pipeline 10 分鐘完成。
1
步驟 1 · 錄音 → 逐字稿（NotebookLM）
把錄音檔（可達 200MB）上傳到 NotebookLM，自動產生逐字稿。複製整段備用。
操作：
notebooklm.google.com → New notebook → Sources → Upload → 選錄音檔 → 等 1-3 分鐘 → 點 source 看逐字稿 → 全選複製
2
步驟 2 · 逐字稿 → 結構化紀錄（Claude）
在 Claude 開新對話，把以下整段 prompt + 逐字稿丟進去。
# 角色 你是熟悉行動項目追蹤的會議記錄秘書，擅長從亂糟糟的逐字稿整理出結構化會議紀錄。 # 任務 把以下逐字稿整理成「給沒到場的人看的會議紀錄」。 # 限制 - 移除：贅字、口語停頓、寒暄、離題討論 - 保留：所有事實、數字、決議、人名、時程 - 不要編造任何沒講到的內容 - 同一主題的討論合併（即使在逐字稿裡是分段討論的） - 區分「我方」「客戶方」「待確認」三種發言來源 # 輸出格式 # [會議主題] · YYYY/MM/DD **參與**：[姓名/單位列表] **會議目的**：[一句話] ## 主題 1：xxx [整理後的討論與結論] ## 主題 2：xxx [整理後的討論與結論] ## 主題 3：xxx [整理後的討論與結論] ## 待解決問題 - xxx 【逐字稿】 """ [在此貼上 NotebookLM 給的逐字稿] """
3
步驟 3 · 紀錄 → 行動項目（待辦）
在同一個 Claude 對話接續輸入（不要重開）：
# 接續任務 從以上會議紀錄萃取所有「行動項目」（action items），整理成可追蹤的待辦清單。 # 限制 - 只列「明確要有人去做」的事，不列討論性事項 - 每項註明：項目 / 負責人 / 截止日 / 優先級（高中低） - 如果負責人或日期會議中沒講清楚，標 "[待確認]" - 「待確認」項目下方加一句「需追問的具體問題」 - 不要編造會議沒提到的事 # 輸出格式 | # | 項目 | 負責人 | 截止日 | 優先級 | 備註 | |---|------|--------|--------|--------|------| | 1 | xxx | 王xx | 2026/04/22 | 高 | — | | 2 | xxx | [待確認] | [待確認] | 中 | 需追問：xxx |
4
步驟 4 · 待辦 → 會後 email
同一對話繼續：
# 接續任務 基於以上紀錄與待辦，寫一封會後確認 email 給所有與會者。 # 限制 - 中文，簡潔有溫度 - 包含：感謝出席（一句話）、會議摘要（80 字內）、行動項目表格、回覆截止 - 不要寫「以上紀錄敬請指教」這類客套 - 結尾請對方在 2 個工作天內回覆若內容有誤 - 署名：[在此填你的名字] # 輸出格式 主旨：[會議主題] 會後紀錄 · YYYY/MM/DD [正文] [行動項目表格] 如有任何補充或修正，請於 [日期] 前回覆。 謝謝， [你的名字]
5
步驟 5 · 潤稿（去 AI 味）
最後一輪潤稿：
# 接續任務 最後一輪潤稿。 # 修正項目 - 開頭直接寫第一個重點，不要「茲將會議結論彙整如下」這類開場 - 砍掉「綜上所述」「整體而言」這類詞 - 「請對方回覆」要直接，不要「煩請」「敬請」 - 結尾去掉敬語套話 - 整體正式但不生硬，避免公文語，保留 1-2 句口語 直接給最終成品。
ROI 粗估：1 小時會議 × 每月 8 場 = 每月 8 小時會議。傳統整理 = 8 小時整理（1:1）；Pipeline = 80 分鐘整理。每月省 6.5 小時，一年 78 小時——等於你每年多 2 個工作週的時間。
TOOL C
固化成 Claude Project · 永久助手
上面 Tool A 的 8 張情境卡、Tool B 的 5 步 pipeline，每張 / 每步都是可複製的 prompt。但更進階的用法是把最常用的 3-5 個「固化」成 Claude Project——每次只需貼素材、不用每次貼 prompt。以下給完整 SOP：
1
挑最常用的 3 個情境
從 Tool A 的 8 情境中，挑你每週會用 3 次以上的 3 個。多數上班族會選：
- 情境 03（回客戶詢價）—— 業務每天用
- 情境 05（婉拒）—— 業務 / 客服每週用
- 情境 07（跨部門協調）—— PM / 經理每週用
2
建 3 個 Claude Projects
打開 claude.ai → 左側 Projects → + New project。每個 Project 命名為情境名（「詢價回覆助理」「婉拒信助理」「跨部門協調助理」）。
3
把「回合 2 的角色 + 限制」放 Instructions
每個 Project 的「Edit project details → Project instructions」欄位，貼對應卡片的回合 2 的 prompt（Role + 限制 + 格式部分，不要貼「任務：」那行因為每次都變）。再加上：
# System Prompt 範本（放 Project Instructions） [從對應情境卡的「回合 2」複製角色 + 限制 + 格式部分] # 我的固定資訊 - 姓名：[你的名字] - 職稱：[職稱] - 公司：[公司] - 常用禁用詞：首先、其次、綜上所述、敬請、萬分抱歉、不便之處敬請見諒 # 回覆流程 1. 讀懂我丟的素材（客戶信 / 背景） 2. 若資訊不足，先問我 1-2 個釐清問題 3. 確認後寫完整成品 4. 若我說「調整」，依我指示修改 5. 最後一步主動問「要 2-3 個版本嗎？」
4
實測 5 次，調整 Instructions

文書工作流精通階梯 · 你現在在哪一階？
LEVEL 0
從頭手寫每封信 / 每份紀錄——AI 對你沒影響。
LEVEL 1
會用單點 prompt 寫信——一次到位但品質不穩（CH 1.2 的程度）。
LEVEL 2
會用 4 回合對話——從「寫信」進化到「迭代寫信」。本 PRAC 的 Tool A 就是這階的實戰。
LEVEL 3
會做會議 pipeline——錄音 → 紀錄 → email 一氣呵成。本 PRAC 的 Tool B。
LEVEL 4
把常用流程固化成 Claude Project——從「每次貼 prompt」到「每次丟素材即可」。本 PRAC 的 Tool C。
LEVEL 5
接自動化流程——Gmail 收到詢價信 → Make 自動呼叫 Claude Project → 自動產出回信草稿。Part 5 會教。
