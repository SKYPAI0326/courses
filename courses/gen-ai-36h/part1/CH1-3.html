CH 1.3 · 1 小時（60 分鐘）

System Prompt 與提示詞除錯

前節你學會寫 4 欄公式。但若每次寫信都要重貼一遍 300 字的 prompt，太累。本節教你「把 prompt 嵌進工具裡」變成專屬助手——之後只需丟素材、AI 自動套用你的設定。還會教你當 AI 產出不對勁時，怎麼 5 秒內找出是哪一欄的問題。

(01) · 兩層 Prompt10 分鐘

System 是「長期身份」，User 是「當下任務」

所有 AI 工具底下都有兩層提示詞——一層是「永久設定」、一層是「單次任務」。理解這個分層，你的 prompt 使用方式會從「每次重複」進化成「一次設定、長期使用」。

System Prompt
永久 · 設定一次
告訴 AI 「你是誰、你要怎麼工作」。寫進去後，這個對話視窗 / 這個 GPT / 這個 Claude Project 裡每次提問 AI 都會自動套用。

適合放：角色（Role）、語氣、產業背景、固定格式、禁忌事項、知識邊界——這些不會每次改變的東西。
User Prompt
單次 · 每次都寫
告訴 AI 「這次具體要做什麼」。每次提問都要寫，內容隨任務變化。

適合放：任務內容、當次的限制（字數、受眾）、這次輸出想要的格式——這些每次都會變的東西。
類比 · 像員工的「職務說明書」 vs 「每日工作指令」
System Prompt 像是員工的「職務說明書」——說明他是會計、要遵守會計準則、用台幣、回覆語氣專業但親切。User Prompt 像是主管每天交辦的「工作指令」——「處理一下這份報帳」「整理昨天客戶的發票」。因為員工知道自己是會計（System），所以你每次交辦只要說「處理一下這份報帳」（User）即可——不用重講「你是會計、要遵守準則⋯⋯」這些背景。
案例 · 行銷 Lily 的一個月變化
Lily 以前每次要 Claude 改文案都貼 300 字 prompt（「你是資深文案編輯⋯⋯品牌調性⋯⋯目標受眾⋯⋯」）。一天改 5 篇文案、光貼 prompt 就花 15 分鐘。後來她把那 300 字的 Role + Constraints 放進一個 Claude Project「品牌文案助手」，每次只要貼原文 + 一句「請改」即可。一個月下來省下 7.5 小時，幾乎等於多出整整一天。
🎯
怎麼判斷哪些放 System、哪些放 User？問自己一個問題：「這個資訊下一次用還會一樣嗎？」。一樣 → 放 System；每次變 → 放 User。「你是行銷顧問」永遠一樣 → System。「這次幫我改 X 文案」每次變 → User。
(02) · 三大工具實作15 分鐘

Claude / ChatGPT / Gemini 怎麼設定 System Prompt

三大工具都有 System Prompt 設定入口，名稱不一樣但概念相同。以下是本節會實際演練的三個操作。建議你邊讀邊跟著設好自己的第一個。

Claude
功能名稱：Projects
位置：左側選單 → Projects
① 點「+ New project」建專案
② 專案名稱如「Email 助理」
③ 點「Edit project details」
④ 在「Project instructions」欄位貼入 System Prompt
⑤ 之後在此 project 內所有對話
　 都自動套用
ChatGPT
功能名稱：GPTs（自訂 GPT）
位置：左下角 「Explore GPTs」→「Create」
① 「Create a GPT」
② 用對話式設定引導（Configure tab）
③ Instructions 欄位貼 System Prompt
④ 可加 Knowledge（上傳文件）
⑤ 發佈後從 Sidebar 直接進入
Gemini
功能名稱：Gems
位置：左側選單 → Gem manager
① 「New Gem」建 Gem
② Name 填 Gem 名稱
③ Instructions 欄位貼 System Prompt
④ 可選是否加入 file references
⑤ 可切換 Gem 當對話時套用

完整 System Prompt 範例 · Email 草稿助理

下面是一份實戰可用的 System Prompt——貼進 Claude Projects 的 Instructions 欄位，就變成你的「專屬 Email 助理」。之後只需告訴它「幫我回這封信」並貼上對方信件，它就會用下面的設定回覆：

# 角色設定 你是 5 年資歷的行政專員，協助我寫中文商務 email。 # 固定原則 - 語氣：正式、簡潔、有同理心 - 開頭：直接切入主題，不寫「希望你一切安好」這類套話 - 結尾：用「謝謝，[我的名字]」 - 字數：控制在 200 字內（不含問候與署名） - 若我的指令模糊，先問清楚再寫，不要自己腦補 # 禁用詞清單 不要使用：首先、其次、綜上所述、敬請查照、不便之處敬請見諒、 萬分抱歉、煩請撥冗、惠請、敬請、謹此等制式套話 # 我的身份 - 姓名：王小明 - 職稱：弄一下工作室 業務經理 - 常往來對象：客戶、供應商、內部同事 # 回覆流程 1. 先讀懂我貼給你的客戶信（或需求描述） 2. 按上述原則寫回信草稿 3. 附註「若要調整語氣 / 加減內容，請告訴我」
使用示範 · 設定好之後怎麼用
設定完成後，你的對話變得超簡單：

過去（沒 System Prompt）：每次要貼 300 字 prompt + 100 字客戶信，共 400 字
現在（有 System Prompt）：只需說「幫我回這封」+ 貼 100 字客戶信，共 110 字

省下的 290 字 × 每天 5 次 = 一天省 1,450 字的打字時間，大約 10 分鐘；一週省下 50 分鐘。
⚠️
資安邊界：不要把敏感資料放 System Prompt。客戶名單、未公開財報、員工薪資、帳密——這些都不要寫進 System Prompt，因為這層設定會跟著對話一起送到 AI 平台。免費版更要小心（可能用來訓練模型）。System Prompt 放「怎麼做事」而非「做事的資料」。
(03) · 提示詞除錯20 分鐘

AI 產出不對勁時，用症狀找原因

即使你寫了完整 4 欄 prompt，AI 有時還是會給你不滿意的東西。這時不是「prompt 沒救」——通常是某一欄沒寫清楚。下面 6 種症狀是新手最常遇到，每種都有對應的「怎麼診斷 + 怎麼修」方法。

症狀 01產出太「萬用」、像罐頭範本
症狀：產出感覺「每個字都對但沒什麼用」——就像範本填字練習，沒有行業特性、沒有決策點、沒有你公司的影子。
根因：缺 Role（身份不明）或 受眾描述（不知道給誰看）。AI 只能用「最安全」的版本回答。
修法：補一句具體 Role：「你是 X 行業 Y 職務的 Z 年資顧問」。再補受眾：「對象是不懂技術的董事長」。兩句下去產出立刻有差。
症狀 02產出太長、重點散落
症狀：你要 200 字，它給 800 字；你要一句話結論，它給 5 段話。AI 的默認寫法是「全面性」——但你多數時候需要的是「精簡」。
根因：缺 字數限制 與 格式指定。
修法：明確指定：「200 字內 / 用 3 個 bullet point / 結論寫在第一行」。越具體越好。若它還是給你一坨，追問「請再精簡一半」。
症狀 03用詞太「AI 味」、套話多
症狀：出現「首先」「其次」「綜上所述」「敬請」「不便之處」——同事一看就知道你用 AI 寫的。
根因：缺明確的 禁用詞清單。AI 預設用「最禮貌、最學術」的寫法，但那不是職場實用的語氣。
修法：限制區加禁用清單：「不要用『首先』『其次』『綜上所述』『敬請』『不便之處』這類套話」。可以建一個你的「個人禁詞清單」貼在 System Prompt。
症狀 04方向完全不對 · 跟你想的差十萬八千里
症狀：你要的是 A 調性，它給你 B 調性；你要的是工程角度，它寫成行銷角度。
根因：你的描述太抽象，AI 自行腦補。缺 Few-shot 範例。
修法：給 1-2 個「我要的樣子」範例，比寫 100 字描述還有效。「請參考以下範例的寫法：[範例 1] [範例 2]。現在請用同樣風格處理新的：[新素材]」。AI 會「模仿」範例風格。
症狀 05AI 編造資訊 · 幻覺 · 講出不存在的事
症狀：AI 回答得很有自信，但資料是假的——講出根本不存在的條款、數字、人名。處理合約、法規、事實性資料時特別危險。
根因：沒設 知識邊界。AI 被訓練成「盡量有幫助」，即使不知道也會猜。
修法：限制區強制加：「資料不足時直接說『我沒有這個資訊』，不要推測或編造」。處理文件時最好改用 NotebookLM（它只會回答你給的資料裡有的內容）。
症狀 06第二輪回答跟第一輪矛盾
症狀：第一輪說「建議 A」，你追問「為什麼不是 B」，它立刻說「你說的對，B 才對」——然後下次又說 A。AI 為了迎合你不斷變立場。
根因：AI 被訓練成「配合使用者」，缺乏「堅持觀點」的指示。
修法：要求它「扮演批判顧問」：「請扮演嚴格的顧問，若你認為我的看法錯，請明確反駁；即使我追問，也要堅持你認為對的立場，除非我提出新證據」。

除錯 SOP · 追問代替重打

5 步除錯流程
1. 看到爛輸出時 不要重開新對話——重開會丟掉 AI 已累積的 context
2. 先問自己「是哪一欄的問題？」（6 症狀對應表）
3. 在同一對話裡追問「修正指示」：「剛剛那版：① 太長 ② 有套話 ③ 方向偏。請重寫，限制 200 字、去套話、角度改成 X」
4. AI 會帶著前文 context 修改，通常 2-3 輪內可解決
5. 若是 System Prompt 問題（每次都犯同樣毛病），改 System Prompt 而非追問 User——追問是短期修正，改 System 是長期治本
💡
什麼時候該重開？若發現「整個方向都錯了」「Role 本身設錯了」「對象想錯了」——這時 context 已污染，重開比較快。但若只是「某段不對」「語氣稍偏」「字數過長」——永遠追問，不重開。
(04) · 課堂動手10 分鐘
PRACTICE 01 · 建第一個 Claude Project
把 CH 1.2 寫的週報 prompt 變成永久助手
題目：把 CH 1.2 Practice 01 中你寫的「週報 prompt」，放進 Claude Project 的 Instructions 欄位，建出「我的週報助手」。之後每週只需貼流水帳就能生成週報。
提示：① 打開 claude.ai → 左側 Projects → New project ② 命名「週報助手」③ Edit project details → Instructions 貼入整段 prompt（含 Role + Constraints + Format）④ 儲存後在該 Project 內新開對話，貼你本週流水帳試跑一次
PRACTICE 02 · 除錯診斷
對照 6 症狀，診斷你過去的爛輸出
題目：打開你最近一次對 AI 結果「不滿意」的對話。對照 6 症狀表，找出「是哪幾項症狀」；然後寫一段「修正追問」貼回該對話，看結果是否變好。
提示：若最近 AI 對話都還滿意——恭喜，你已經寫 prompt 寫得比 80% 的人好了。可以找一個故意模糊的 prompt（「幫我寫一段 AI 的好處」）跑一次，體驗各種症狀一次發作，再用追問修回來。
(05) · 課後行動5 分鐘

建 3 個專屬助手

Part 1 到這裡，你已經累積了 3 個可以永久使用的 prompt（CH 1.2 Practice 01 + 02 + 03 的成果）。本節課後就是把這 3 個 prompt 全部「扎進」對應工具：

Action 1
15 分鐘
在 Claude Projects 建 3 個 Project：Email 助理、週報助理、會議紀錄整理助理。每個 Project 的 Instructions 貼入對應 prompt。
Action 2
下週實測
下週遇到對應情境時實際使用這 3 個 Project，而非開新對話。紀錄「省下多少時間」。這個紀錄是 PRAC 1 要用到的素材。
Action 3
3 天後
回顧實測結果，調整 System Prompt。若某個 Project 產出總是某個毛病——回去改 Instructions（而非每次追問修）。這叫「治本」的除錯。
