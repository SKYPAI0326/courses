Part 4 · 零代碼 AI 工具開發
CH 4.4 · 2 小時

UI/UX 風格覆寫
與部署上線

前 3 節做出來的工具好用，但「視覺都一樣」——預設都是 Tailwind 標配藍紫色。本節教你用一句話改造整個工具的設計風格，並把 HTML 部署到 GitHub Pages 變成永久可分享網址。

學習成果 · OUTCOMES
會用「風格描述語言」指定設計：色系、字型、間距、配色基調、視覺氣質
能套用 5 種常用風格範本（無印 / 蘋果 / 暗黑 / 企業 / 新聞），一鍵切換
會把 Claude Artifact 下載成 HTML，部署到 GitHub Pages 變成永久公開連結
SECTION 01 · 為什麼要改視覺

預設視覺洩露你「用 AI 做」的事實

Claude / ChatGPT 預設生成的 web 工具，大多是 Tailwind + Lucide icons + 紫藍漸層 + 白色陰影卡。看一眼就知道「AI 做的」。對個人練習沒差，但若要：

  • 給客戶用、要匹配公司品牌
  • 放進公司網站 / 簡報
  • 顯得你有用心做（而不是 5 秒生的）

就需要「視覺風格覆寫」。重點是：你不需要會設計，只需要會「描述風格」。

🎨
核心心法：視覺風格 = 用形容詞 + 配色 + 字型 + 間距感覺描述。AI 會把這些抽象描述翻成 CSS。你不必懂 CSS。
SECTION 02 · 風格描述語言

5 個維度描述你要的風格

配色基調
「米白 + 深灰 + 一個強調色」
「全暗黑、霓虹點綴」
「企業藍 + 純白」
字型
「明朝體做標題、無襯線做內文」
「全 sans-serif、字重對比強」
「等寬字 + 終端機感」
間距與留白
「大量留白、卡片寬鬆」
「資訊密度高、空間緊湊」
「日式排版、行高大」
圓角與陰影
「直角、無陰影、扁平」
「圓角 16px、柔和陰影」
「圓角 4px、邊框細線」
氣質形容詞
「沉靜、克制」「銳利、未來」「溫暖、人文」「專業、穩重」「活潑、年輕」
💡
最簡公式：「主色 X + 字型 Y + 氣質 Z」三元素就能定調。例：「米白主色 + Noto Serif 標題 + 沉靜克制」。AI 會自己補完所有細節。
SECTION 03 · 5 種常用風格範本

5 個一鍵切換的風格 prompt

以下是上班族最常用的 5 種風格。針對任何已有的工具，跑「請把整體視覺風格改成 [某風格]」即可整體換臉。

無印良品 / 日式
沉靜、人文
米白底、深褐文字、明朝體標題、單一強調色（霧藍 / 鼠尾草綠）、大量留白、無陰影、直角邊框
請把整體視覺風格改成無印良品 / 日式風：背景 #f5f3ee 米白、主文字 #2c2b28 深褐、Shippori Mincho 標題、Noto Sans TC 內文、單一霧藍 #6b7fa3 強調色、大量留白、卡片白底細邊框、圓角 6px、無漸層、無陰影、整體沉靜克制
蘋果風 / Modern
乾淨、銳利
純白底、深灰文字、SF Pro 字型、藍 / 黑強調色、適度圓角（10-12px）、輕陰影、玻璃感模糊背景
改成蘋果官網風：背景純白 #fff、主色系統黑 / 藍 #007AFF、字型用 -apple-system / SF Pro Display、字重 400/600 對比、圓角 12px、玻璃感（backdrop-filter blur）、輕微陰影 0 4px 16px rgba(0,0,0,.06)、整體乾淨銳利
企業簡報 / 商務
專業、穩重
深藍 + 灰階配色、Roboto / Inter 字型、強對比、明確區塊邊界、傳統商務感
改成企業簡報風：背景 #f8f9fa、主色深藍 #1e3a5f、文字 #212529、Inter 字型、卡片白底 + 1px 灰邊框、圓角 4px、強對比表頭（深藍底白字）、整體專業可放進公司 PPT
暗黑 / Dev Mode
銳利、未來
深色底、霓虹強調色、等寬字、終端機感、低彩度高對比
改成暗黑開發者風：背景 #0d1117、卡片 #161b22、文字 #c9d1d9、強調色霓虹綠 #00ff88 或藍 #58a6ff、JetBrains Mono 字型、邊框 1px #30363d、整體有 IDE / 終端機質感
新聞網站 / 編輯風
優雅、可讀
米白或淡黃底、襯線標題、正文行高大、強調色克制、可讀性優先
改成新聞網站 / The Guardian 風：背景 #fefdf9、襯線標題（Playfair Display 或 Noto Serif TC）、正文 Source Sans Pro / Noto Sans TC、行高 1.85、單一強調色酒紅 #8b1a1a 或墨綠 #2c5f4f、無多餘陰影、強調可讀性
活潑年輕 / 社群
明亮、有活力
高彩度配色、圓角大、emoji 友善、漸層按鈕、活潑互動
改成年輕社群風：背景 #fff5f7、主色珊瑚紅 #ff6b6b 或薄荷綠 #4ecdc4、輔色芥末黃 #ffe66d、Quicksand 圓潤字型、按鈕大圓角 24px + 漸層、卡片有彈跳 hover 動畫、適度 emoji 點綴
🎯
用法：做完工具後，直接接續對話貼這些 prompt 之一即可全改。不需要重做——AI 會基於現有 HTML 改 CSS。
SECTION 04 · 細節微調

套完風格後的5 個必修微調

字級對比
「標題太小，加大到 2.5rem；正文太大，縮到 0.95rem」
主色強度
「主色太搶眼，請降低飽和度 20%」
「主色太淡，加深 15%」
間距 / 留白
「卡片內距太擠，加大到 32px」
「整體太鬆，縮 15%」
行動裝置
「手機畫面太擠，請優化響應式：螢幕 <768px 時垂直堆疊、字體略縮」
細節動畫
「按鈕 hover 加 ease 0.2s 過渡」
「主數字加 fade-in 動畫」
SECTION 05 · 部署上線

把 HTML 變成永久可分享網址

Claude Artifacts 的分享連結會綁在你的對話上——當對話被歸檔或刪除，連結會失效。要永久公開，得部署到自己的網址。

GitHub Pages
本課推薦
免費、永久、有自訂網域。一個 repo 可放數十個工具。
適合：個人工具庫、長期累積
Netlify Drop
最快
拖一個 HTML 進去就有網址，5 秒上線。免費版有限速。
適合：一次性 demo、給客戶看
Vercel
最進階
適合連結 GitHub repo、自動部署、有後端能力。
適合：進階使用者、有後端需求

GitHub Pages 5 步部署

1
下載 Artifact 成 HTML
在 Claude Artifact 右上角點「⋯」→ Download 或 Copy code，存成 index.html
2
建 GitHub repo
github.com 註冊（免費）→ 新建 repository，命名如 my-tools，勾選「Add README」，建立
3
上傳 HTML 檔
repo 頁面點「Add file」→「Upload files」→ 拖 index.html 上去 → Commit
4
啟用 Pages
repo 頁面 Settings → Pages → Source 選「Deploy from a branch」→ Branch 選 main + / (root) → Save
5
取得永久網址
2-3 分鐘後，網址會出現在 Pages 頁面：https://[帳號].github.io/my-tools/ — 這就是永久連結，可分享給任何人。
⚡
多工具集中管理：同一個 repo 內可建多個資料夾（quote-tool/, utm-tool/），每個資料夾放一個工具的 index.html。網址自動變成 github.io/my-tools/quote-tool/。3 個月後你會有自己的工具集首頁。
⚠️
注意：GitHub Pages 是公開的，任何人有連結都能看。不要放敏感資訊（客戶名單、價格表、內部資料）。若需私密，用 Netlify Drop 的密碼保護功能，或 Vercel + 簡單密碼關卡。
SECTION 06 · 本節行動

課後練習：把一個工具部署上線

挑 CH 4.2 / 4.3 做的任一個工具，跑這個流程：

  1. 套一個風格範本（無印良品最容易，跟本網站一致）
  2. 跑 2-3 輪細節微調直到你滿意
  3. 下載成 index.html
  4. 建 GitHub repo + 啟用 Pages
  5. 把網址貼給朋友 / 同事，請對方試用並反饋

完成這一輪後，你已經是個能獨立做工具上線的人。下一個 Part 4 的 PRAC 會帶你做 3 個正式作品上線。

📝
