(01)15 分鐘

預設視覺洩露你「用 AI 做」的事實

情境 A · 業務經理把工具給客戶用
業務經理 Alex 做了一個報價試算器給客戶。客戶打開看到藍紫色漸層按鈕、Tailwind 預設字型、Lucide icons——客戶說「這是 ChatGPT 做的吧？」雖然功能沒問題，但客戶對精緻度的印象打了折扣。Alex 後來把風格改成公司的深藍 + 白底商務風，客戶再打開時沒有任何質疑。視覺決定了「專業感」的第一印象。
情境 B · 主管看到你做的 Dashboard
你做了一個 KPI dashboard，功能完美。但主管在週會上投影時，旁邊的副總問「這是什麼外部工具？」——因為風格跟公司內部系統完全不搭。如果你花 2 分鐘把它改成跟公司簡報同一套配色，主管會覺得「這是我們自己的系統」。風格一致 = 歸屬感 = 被採用的機率。

Claude / ChatGPT 預設生成的 web 工具通常是：Tailwind + Lucide icons + 紫藍漸層 + 白色陰影卡。看一眼就知道是 AI 生成。要改視覺，你不需要會 CSS，只需要會「描述風格」——AI 會把你的形容詞翻成程式碼。

觀念 · 風格描述語言 = 形容詞 + 配色 + 字型
AI 需要的不是 CSS 指令，而是你用人話描述「你要什麼感覺」。最簡公式：「主色 X + 字型 Y + 氣質 Z」三元素就能定調。例：「米白主色 + 明朝體標題 + 沉靜克制」。AI 會自己補完 padding、line-height、hover 動畫等所有細節。

5 個維度描述你要的風格

配色基調
「米白 + 深灰 + 一個強調色」
「全暗黑、霓虹點綴」
「企業藍 + 純白」
字型
「明朝體做標題、sans-serif 做內文」
「全 sans-serif、字重對比強」
「等寬字 + 終端機感」
間距與留白
「大量留白、卡片寬鬆」
「資訊密度高、空間緊湊」
「日式排版、行高大」
圓角與陰影
「直角、無陰影、扁平」
「圓角 16px、柔和陰影」
「圓角 4px、邊框細線」
氣質形容詞
「沉靜、克制」「銳利、未來」「溫暖、人文」「專業、穩重」「活潑、年輕」
(02)40 分鐘

6 種風格範本：一句話全換臉

以下每種風格都附完整的 prompt。你只需要在做完工具後，接續對話貼這些 prompt 即可整體改風格——不需要重做，AI 會基於現有 HTML 改 CSS。

無印良品 / 日式
沉靜、人文
米白底、深褐文字、明朝體標題、單一強調色（霧藍 / 鼠尾草綠）、大量留白、無陰影
請把整體視覺風格改成無印良品 / 日式風：背景 #f5f3ee 米白、主文字 #2c2b28 深褐、Shippori Mincho 標題、Noto Sans TC 內文、單一霧藍 #6b7fa3 強調色、大量留白、卡片白底細邊框、圓角 6px、無漸層、無陰影、整體沉靜克制
蘋果風 / Modern
乾淨、銳利
純白底、深灰文字、SF Pro 字型、藍 / 黑強調色、圓角 10-12px、輕陰影、玻璃感模糊背景
改成蘋果官網風：背景純白 #fff、主色系統黑 / 藍 #007AFF、字型用 -apple-system / SF Pro Display、字重 400/600 對比、圓角 12px、玻璃感（backdrop-filter blur）、輕微陰影 0 4px 16px rgba(0,0,0,.06)、整體乾淨銳利
企業簡報 / 商務
專業、穩重
深藍 + 灰階、Roboto / Inter 字型、強對比、明確區塊邊界、傳統商務感
改成企業簡報風：背景 #f8f9fa、主色深藍 #1e3a5f、文字 #212529、Inter 字型、卡片白底 + 1px 灰邊框、圓角 4px、強對比表頭（深藍底白字）、整體專業可放進公司 PPT
暗黑 / Dev Mode
銳利、未來
深色底、霓虹強調色、等寬字、終端機感、低彩度高對比
改成暗黑開發者風：背景 #0d1117、卡片 #161b22、文字 #c9d1d9、強調色霓虹綠 #00ff88 或藍 #58a6ff、JetBrains Mono 字型、邊框 1px #30363d、整體 IDE / 終端機質感
新聞網站 / 編輯風
優雅、可讀
米白 / 淡黃底、襯線標題、正文行高大、強調色克制、可讀性優先
改成新聞網站 / The Guardian 風：背景 #fefdf9、襯線標題（Playfair Display 或 Noto Serif TC）、正文 Noto Sans TC、行高 1.85、強調色酒紅 #8b1a1a 或墨綠 #2c5f4f、無多餘陰影、強調可讀性
活潑年輕 / 社群
明亮、有活力
高彩度配色、圓角大、漸層按鈕、活潑 hover 動畫
改成年輕社群風：背景 #fff5f7、主色珊瑚紅 #ff6b6b 或薄荷綠 #4ecdc4、輔色芥末黃 #ffe66d、Quicksand 圓潤字型、按鈕大圓角 24px + 漸層、卡片有彈跳 hover 動畫
觀念 · 為什麼不是選一個「最好的」？
風格沒有對錯，只有「適不適合你的情境」。給客戶的報價工具 → 企業簡報風。自己用的 KPI 儀表板 → 無印 / 暗黑。簡報用的計時器 → 暗黑。Instagram 抽獎活動頁 → 活潑年輕。先想清楚「誰在什麼場景用」，風格自然就選對了。
情境 · 同一工具三種面貌
設計師 Jess 做了一個專案進度追蹤器。跑完功能後，她測試三種風格：① 無印良品——自己每天用，看了不煩。② 企業簡報——週會投影給主管看，專業感。③ 暗黑——晚上加班時切換，不刺眼。同一份 HTML，三段 CSS 變數切換。她甚至在工具裡加了一個「主題切換」下拉——是的，你可以追問 AI「請加一個深色 / 淺色模式 toggle」，2 分鐘搞定。
GUIDED PRACTICE 01
給你的工具換一套風格
打開 CH 4.2 或 4.3 做的任何一個工具的 Claude 對話。接續對話，貼上面任一風格範本的 prompt。觀察 AI 回傳的成品，然後做一輪微調。
重點：切完風格後，問自己「這個風格適合這個工具的使用情境嗎？」如果不適合，換另一個風格。
進階：追問「請加一個右上角的深色 / 淺色模式 toggle 開關，用 CSS 變數切換」，讓工具同時支援兩種風格。
展開參考做法
假設你做了報價試算器：
Step 1：在同一個對話中貼「改成企業簡報風：背景 #f8f9fa、主色深藍 #1e3a5f...」
Step 2：AI 會改全部 CSS，保留功能不變
Step 3：微調：「表頭文字太小，加大到 1.1rem。合計數字改用主色 #1e3a5f 加粗。」
Step 4：追問：「右上角加一個 toggle，可切換深色 / 淺色模式。深色用 #1a1a2e 底 + #e8e8e8 字。」
(03)20 分鐘

套完風格後的5 項必修微調

AI 套完風格後，80% 到位但 20% 需要你用人眼調整。以下 5 項是最常需要微調的，每項附追問範例。

1. 字級對比
「標題太小，加大到 2.5rem」
「正文太大，縮到 0.92rem」
「數字區太擠，加大到 2rem + 粗體」
2. 主色強度
「主色太搶眼，降低飽和度 20%」
「主色太淡看不到，加深 15%」
「邊框太明顯，改成 0.5px」
3. 間距留白
「卡片內距太擠，padding 加到 32px」
「整體太鬆散，卡片間距從 24 縮到 14px」
4. 行動裝置
「手機畫面太擠，< 768px 時垂直堆疊、字體略縮」
「按鈕在手機上太小，改成 width 100%」
5. 細節動畫
「按鈕 hover 加 ease 0.2s 過渡」
「數字載入時加 fade-in 動畫」
「卡片 hover 加邊框變色」
BEFORE · AI 預設
紫藍漸層按鈕 + Tailwind 預設圓角 8px + 大片陰影 + Lucide icons + Inter 字型。
「看起來像 AI 做的。」
AFTER · 一句話覆寫
主色改公司藍 + 圓角 4px + 無陰影 + 邊框細線 + 明朝體標題。
「看起來像公司的內部系統。」
GUIDED PRACTICE 02
做 5 項微調
延續 Practice 01 的工具，逐項追問上面 5 個微調。每追問一個，看結果是否符合你的期待。
目標：調到你「會願意把這個網址發給主管或客戶」的程度。
捷徑：你可以一次給完 5 個微調指令（用列表），AI 會一次改好。但建議初學者一個一個來，看清楚每一步改了什麼。
展開參考做法
一次性追問範例：「請做以下 5 項微調：
1. 標題加大到 1.8rem，正文改 0.95rem
2. 主色飽和度降 10%，改成 #5a6f8f
3. 卡片內距改 28px，卡片間距 16px
4. 手機 < 768px 時 KPI 卡垂直堆疊、按鈕 width 100%
5. 合計數字載入時加 0.5s fade-in 動畫」
(04)30 分鐘

GitHub Pages 5 步部署

Claude Artifacts 的分享連結綁在你的對話上——對話歸檔或刪除，連結就失效。要永久公開分享，得部署到自己的網址。

情境 · 從「分享連結」到「我的工具網站」
PM 小美做了一個會議準備卡工具，Claude 分享連結有效了 3 個月。某天她清理對話，連結突然 404。同事問她要連結，她才發現。後來她把 HTML 放到 GitHub Pages，網址永久有效：xiaomei.github.io/tools/meeting-prep/。再也不擔心連結失效，還可以持續更新。
1
下載 Artifact 成 HTML
在 Claude Artifact 右上角點「...」→ Download 或 Copy code → 貼到文字編輯器 → 存成 index.html。檔名必須是 index.html（小寫）。
2
建 GitHub 帳號 + 新 repo
github.com 註冊（免費）→ 右上角「+」→ New repository → 命名如 my-tools → 勾選「Add README」→ 建立。
3
上傳 HTML 檔
repo 頁面點「Add file」→「Upload files」→ 拖入 index.html → 在 Commit changes 欄寫「add tool」→ Commit。多個工具可分別放子資料夾。
4
啟用 Pages
repo 上方 → Settings → 左欄 Pages → Source 選「Deploy from a branch」→ Branch 選 main + / (root) → Save。
5
取得永久網址
等 2-3 分鐘，Pages 頁面會出現網址：https://[帳號].github.io/my-tools/。這就是永久連結，可分享給任何人。
觀念 · 多工具集中管理
同一個 repo 內可建多個資料夾：quote-tool/index.html、utm-tool/index.html、kpi/index.html。網址自動變成 github.io/my-tools/quote-tool/。3 個月後你會有自己的工具集首頁。建議根目錄也放一個 index.html 當目錄頁，列出所有工具的連結。

3 個部署平台比較

GitHub Pages
本課推薦
免費、永久、有自訂網域。一個 repo 可放數十個工具。學會 Git 之後可版本控制。
適合：個人工具庫、長期累積、需要版本管理
Netlify Drop
最快 · 5 秒上線
拖一個 HTML 進去就有網址。免費版有月流量限制。可加密碼保護（付費）。
適合：一次性 demo、臨時給客戶看、需要密碼保護
Vercel
最進階
連結 GitHub repo 自動部署、有 serverless 後端能力、速度極快。
適合：進階使用者、需要 API route、團隊協作
⚠️
安全提醒：GitHub Pages 是完全公開的，任何人有連結都能看。不要放敏感資訊——客戶名單、報價明細、內部資料都不行。若需要私密工具，用 Netlify 的密碼保護或 Vercel 加簡單 auth。
GUIDED PRACTICE 03
把一個工具部署到 GitHub Pages
選 CH 4.2-4.3 做的任何一個工具，按上面 5 步部署。
Step 1：下載 Artifact 成 index.html
Step 2：建 GitHub repo（如果已有可用現有的）
Step 3：上傳 + 啟用 Pages
Step 4：拿到網址後，用手機打開確認能正常顯示
Step 5：把網址發給一位同事或朋友，請對方試用並給一句反饋
常見卡關：Pages 啟用後要等 2-3 分鐘才生效。如果超過 5 分鐘還 404，檢查檔名是否為 index.html（小寫）、Branch 是否選對。
展開常見問題
Q：我沒有 GitHub 帳號怎麼辦？
A：用 Google 或 email 免費註冊，2 分鐘完成。

Q：我想放多個工具在同一個 repo？
A：建子資料夾，每個資料夾放一個 index.html。網址自動是 username.github.io/repo-name/folder-name/。

Q：更新工具怎麼辦？
A：回到 repo → 點進 index.html → 右上角筆 icon → Edit → 貼新版 → Commit。Pages 會自動重新部署。
(05) · 踩坑博物館10 分鐘

6 個常見災難與修復方法

01改完風格後功能壞了
症狀：套完新風格後，按鈕點了沒反應、下拉選單消失。
修法：AI 改 CSS 時可能動到了 class 名或 id。追問：「改風格後某個功能壞了，請只改 CSS 樣式，不要動 HTML 結構和 JavaScript 邏輯。」
02Google Fonts 載入失敗，字型跑掉
症狀：標題用了 Playfair Display 但顯示為預設 serif，明顯跑版。
修法：檢查 <link> 是否正確引入。追問：「請確認 Google Fonts 的 link 標籤正確，並在 CSS 的 font-family 加 fallback：font-family: 'Playfair Display', Georgia, serif」
03暗黑模式下文字看不清
症狀：深色背景但某些文字還是深色，幾乎看不到。
修法：追問：「暗黑模式有些文字看不清。請確認所有文字（包括 placeholder、label、小字）都改成淺色。用 color: #c9d1d9 做主文字，#8b949e 做次要文字。」
04手機上排版炸裂
症狀：電腦上很美，手機上元素重疊、超出螢幕。
修法：追問：「請加 @media (max-width: 768px) 響應式：卡片改單欄、按鈕 width 100%、字型縮小 10%、padding 改 16px。」
05GitHub Pages 一直 404
症狀：啟用 Pages 後等了 10 分鐘還是 404。
修法：三個檢查點：① 檔名是否為 index.html（不是 Index.html） ② Branch 是否選對（main） ③ 路徑是否在根目錄。如果都對，到 repo → Actions 頁面看部署是否有報錯。
06風格改到一半 AI 回傳不完整
症狀：AI 只改了一半 CSS 就截斷了，頁面半新半舊。
修法：追問：「你上次的回應被截斷了，請從 [被截斷的地方] 繼續給我剩下的完整 CSS」。或更好的做法：一開始就說「請只修改 CSS 變數區塊（:root），用 CSS custom properties 控制全部配色」。
(06) · 課後行動

課後行動：一個工具完整上線

挑 CH 4.2 / 4.3 做的任一個工具，完成這個閉環：

Step 1：套一個風格範本（無印良品最容易上手）。
Step 2：跑 2-3 輪細節微調，調到你滿意。
Step 3：下載成 index.html。
Step 4：建 GitHub repo + 啟用 Pages。
Step 5：把網址貼給同事 / 主管，請對方試用 + 給一句反饋。

完成這一輪後，你已經是一個能獨立把工具做到上線的人。PRAC 4 會帶你做 3 個正式作品，建立你的個人工具集。

觀念 · 部署不是終點，而是「被用」的起點
做完 → 部署 → 有人用 → 收到反饋 → 改版 → 更多人用。這個循環才是價值所在。一個沒人用的部署只是佔了一個 URL。所以 Step 5 的「分享給人用 + 收反饋」是最重要的步驟，不是選配。
