Part 4 · 零代碼 AI 工具開發
CH 4.1 · 2 小時

零代碼工具方法論：
輸入 · 邏輯 · 輸出

這節是 Part 4 的核心觀念。學完之後，你看任何一個 web 工具都能拆解成「輸入 / 邏輯 / 輸出」三層——這就是讓你能用 Claude / Gemini 直接做出工具的思維基礎，不需要寫一行程式。

學習成果 · OUTCOMES
能用「輸入 / 邏輯 / 輸出」三層架構描述任何小工具，這是寫 prompt 的基礎
能在 Claude Artifacts 內 30 秒做出第一個能用的小工具（計時器 / 試算器）
分得清「適合零代碼做」與「不適合做」的工具邊界，避免浪費時間踩坑
SECTION 01 · 重新理解「工具」

所有 web 工具都是同一個結構

打開任何一個你常用的 web 工具——線上時鐘、BMI 試算器、QR Code 產生器、待辦清單、IG 文案產生器——它們本質上都是同一個結構：使用者輸入某些東西，工具用某種邏輯處理，產出某種結果。

過去要做出這種工具，需要會 HTML、CSS、JavaScript。現在你只需要：用中文清楚描述「輸入 / 邏輯 / 輸出」，丟給 AI，它直接生出可以即時使用的成品。

LAYER 01
輸入
使用者要填什麼
LAYER 02
邏輯
怎麼處理輸入
LAYER 03
輸出
給使用者看什麼
🧭
關鍵心法：先想清楚這三層，再去寫 prompt。最常見的失敗是「省略思考、直接喊 AI 做」，得到的東西方向不對還要重來。30 秒的設計，省你 30 分鐘的修改。
SECTION 02 · 三層拆解

每一層該想清楚什麼

LAYER 01 · 輸入
使用者要填 / 選 / 上傳什麼？
想清楚：欄位有幾個、每個欄位是什麼類型（文字 / 數字 / 日期 / 下拉選單 / 上傳檔案）、有沒有預設值、有沒有限制（最少 / 最多 / 必填 / 選填）。
例：BMI 試算器的輸入
· 身高（數字，cm，必填，120-220）
· 體重（數字，kg，必填，30-200）
· 性別（下拉，男 / 女，預設男）
LAYER 02 · 邏輯
輸入要怎麼變成輸出？
想清楚：用什麼公式 / 規則 / 條件來處理。如果是計算類就寫公式；如果是分類類就寫條件分支；如果是文字處理就寫規則。用「如果 X 就 Y」這種白話寫，AI 會自己翻成程式。
例：BMI 試算器的邏輯
· BMI = 體重 / (身高/100)²
· 如果 BMI < 18.5 → 偏瘦
· 如果 18.5 ≤ BMI < 24 → 正常
· 如果 24 ≤ BMI < 27 → 過重
· 如果 BMI ≥ 27 → 肥胖
LAYER 03 · 輸出
使用者要看到什麼？怎麼呈現？
想清楚：要顯示什麼數字 / 文字 / 圖表、用什麼版面、要不要顏色提示、要不要按鈕（重置 / 複製 / 分享）。「能看到什麼」直接決定使用者爽不爽。
例：BMI 試算器的輸出
· 大字顯示 BMI 數值
· 顯示分類（偏瘦 / 正常 / 過重 / 肥胖），用顏色區別
· 一句話建議（如「正常範圍，繼續保持」）
· 「重新計算」按鈕
⚡
練習：挑你常用的 web 工具（Google 翻譯、QR Code 產生器、UTM Builder…），試著把它拆成三層。會發現絕大多數工具都很單純，背後沒什麼魔法。
SECTION 03 · 工具選擇

用哪個 AI 工具做？

2026 年有三個主流選擇，本課程主要使用 Claude Artifacts（最穩、最快、輸出能即時看到效果）。

Claude Artifacts
本課首選
在 Claude 對話中直接生成可預覽的 HTML/JS 工具，右側面板即時跑、可分享連結。免費版即可用。
適合：幾乎所有個人小工具、簡報嵌入式 demo、客戶展示
Gemini Canvas
Google 整合
類似 Artifacts，內建在 Gemini，適合需要與 Google Workspace（Docs、Sheets）整合的工具。
適合：要存進 Google Drive、給團隊用的工具
ChatGPT Canvas
付費為主
類似但要 ChatGPT Plus 才能用 Code Interpreter 跑 Python 工具。互動 UI 還是 HTML。
適合：需要 Python 處理資料的工具（如分析 Excel）
v0.dev / Bolt.new
進階選項
專業 AI 寫網頁工具，可生成完整 React app。學習曲線略高，免費額度有限。
適合：已試過 Artifacts 後想做更複雜的應用
⚠️
本課程從 Claude Artifacts 開始。免費版每天可生 10-20 個 Artifacts，足夠學習。學會原理後切換到任何工具都通——因為「三層架構」與「prompt 寫法」是通用的。
SECTION 04 · 第一個工具

30 秒做出會議倒數計時器

直接示範。打開 Claude（claude.ai），把以下 prompt 貼進去：

# 完整 prompt 請幫我做一個會議倒數計時器，HTML 單檔可即時預覽。 輸入 - 會議時長：分鐘數，預設 30 分鐘 - 會議主題：文字（選填） 邏輯 - 按「開始」後倒數 - 剩餘時間 > 5 分鐘 → 綠色 - 5 分鐘 ≥ 剩餘 > 1 分鐘 → 黃色 - 剩餘 ≤ 1 分鐘 → 紅色，並開始閃爍 - 倒數結束時播放系統提示音 + 顯示「時間到」 輸出 - 大字（>100px）顯示 MM:SS 倒數 - 上方顯示會議主題（若有填） - 下方有「開始 / 暫停 / 重置」三個按鈕 - 簡潔風格，黑底白字、現代感 直接生成可預覽的 Artifact，不要解釋程式碼。

按下 Enter，Claude 在 5-10 秒內會在右側 Artifacts 面板做出可即時點按的計時器。這就是你的第一個零代碼工具。

💡
為什麼這個 prompt 寫得好？因為「輸入 / 邏輯 / 輸出」三層都明確了，包括邊界條件（顏色變化的閾值）、互動細節（按鈕、音效）、視覺風格。AI 不需要猜，可以一次到位。
SECTION 05 · 工具邊界

什麼適合、什麼不適合零代碼做

零代碼工具強大，但有邊界。試圖用它做超出邊界的東西，會浪費好幾天還做不出來。

✓ 適合用零代碼做
- 計算 / 試算（BMI、利率、報價）
- 資料轉換（單位、時區、格式）
- 文字處理（產生器、模板填充）
- 視覺呈現（圖表、Dashboard）
- 互動展示（流程圖、簡報嵌入）
- 個人化清單（待辦、檢查表）
✗ 不適合（要找專業工具）
- 需要登入 / 帳號管理
- 需要存資料庫（給多人共用）
- 串接外部 API（Email、付款）
- 大量檔案處理 / 後端運算
- 即時通訊 / 多人協作
- 複雜的金流 / 安全性需求
⚠️
判斷標準：能不能在「一個 HTML 檔內」跑完？如果可以，零代碼能做。如果需要「儲存到雲端、登入、跨人協作」，這超出 Artifacts 的範圍——這時候要轉用 Make / n8n 自動化流程（Part 5）或真正的開發工具。

本課程 Part 4 全部聚焦在「能在 HTML 檔內跑完」的工具——這已經涵蓋上班族 80% 的個人生產力需求。

SECTION 06 · 本節行動

課後練習：跑一遍倒數計時器

就照本節 Section 4 的 prompt，到 Claude 跑一次。重點不是「做出計時器」（你大概不缺這個），而是親身體驗「描述 → 即時預覽」這個流程的順暢度。

跑完後，挑你工作中「想要但網路上找不到剛好對的」小工具，照三層架構寫成 prompt，跑第二個。例如：

# 練習題目（任選一個） 1. 跨時區會議時間轉換器（輸入：來源時區、目標時區、時間） 2. 客戶報價試算器（輸入：方案類型、數量、折扣；輸出：總金額 + 條款） 3. 課程倒數時計算器（輸入：開課日；輸出：剩餘工作天數） 4. 簡報計時器（輸入：總時長、頁數；輸出：每頁建議停留時間） 5. 業績達成率視覺化（輸入：目標、實際；輸出：進度條 + 預警顏色）
