SECTION 01 · 重新理解「工具」15 分鐘

所有 web 工具都是同一個結構

打開任何一個你常用的 web 工具——線上時鐘、BMI 試算器、QR Code 產生器、待辦清單、IG 文案產生器——它們本質上都是同一個結構：使用者輸入某些東西，工具用某種邏輯處理，產出某種結果。

過去要做出這種工具，需要會 HTML、CSS、JavaScript。現在你只需要：用中文清楚描述「輸入 / 邏輯 / 輸出」，丟給 AI，它直接生出可以即時使用的成品。

LAYER 01
輸入
使用者要填 / 選 / 上傳什麼
LAYER 02
邏輯
怎麼處理輸入資料
LAYER 03
輸出
給使用者看什麼結果
真實故事 · HR 小美的招募計算器
小美每次算「offer 薪資包」都要開 Excel——底薪 × 月數 + 保險 + 年終 + 分紅。公式超複雜，而且每次都怕算錯。她在 Claude 花 30 秒描述三層架構，AI 10 秒生出一個互動式的「Offer 試算器」——貼到 Chrome 分頁加 bookmark，從此每次 hiring 都秒算。她第一反應是「這個我以前覺得要找工程師才能做的」。
核心心法 · 30 秒設計省 30 分鐘修改
先想清楚三層，再去寫 prompt。最常見的失敗是「省略思考、直接喊 AI 做」，得到的東西方向不對還要重來。30 秒在腦中（或紙上）畫三層圖 → 再花 30 秒打 prompt → AI 10 秒給你成品。整個流程 70 秒。沒畫圖直接寫？70 秒變 30 分鐘修。

1.1 · 為什麼要「自己做」而不用現成工具？

網路上明明有 timezone.com、一堆現成的 UTM 產生器——為什麼還要自己做？三個原因：

理由 01
符合你的工作流
你常跟那 4 個城市合作、你的報價方案只有那 3 種——現成工具為「全部人」設計，反而不夠精準。自己做的工具精準到你的使用情境，不多也不少。
理由 02
輸出能直接用
現成工具給你一個計算結果，你還要複製貼上到 email。自己做可以直接在工具裡生成「可貼的 email 文字」「可列印的報價單」「可加到日曆的連結」。少一步就省一分鐘。
理由 03
隨時能改
想加個功能？直接追問 AI「加個 XX」10 秒搞定。現成工具只能等開發者。你的工具會「演化」跟著你工作變化——這是現成工具永遠做不到的。
✋
暫停 30 秒想一下：你工作中有哪些「有現成工具但每次用都卡卡」的情境？可能是功能太多（你只用到 10%）、格式不對（輸出要手動改）、或單純「懶得每次都找那個網址」。這些就是你做零代碼工具的最佳題目。
SECTION 02 · 三層架構深度拆解30 分鐘

每一層該想清楚什麼

LAYER 01 · INPUT
輸入：使用者要填 / 選 / 上傳什麼？
想清楚：欄位有幾個、每個欄位是什麼類型（文字 / 數字 / 日期 / 下拉選單 / 上傳檔案）、有沒有預設值、有沒有限制（最少 / 最多 / 必填 / 選填）。欄位越少，使用者越願意用——每多一個欄位流失 10% 使用率。
BMI 試算器的輸入
· 身高（數字，cm，必填，120-220）
· 體重（數字，kg，必填，30-200）
· 性別（下拉，男 / 女，預設男）
報價試算器的輸入
· 客戶名稱（文字）
· 方案類型（下拉 3 種）
· 期間（下拉 3/6/12 月）
· 折扣（滑桿 0-30%）
· 付款條件（單選：月付 / 季付 / 年付）
設計要領 · 為什麼每個欄位要標類型
「身高」寫成文字框？使用者可能打「175公分」。改成數字框？只能打數字。加驗證「120-220」？不合理的值被擋住。每個設計決定都在減少「使用者犯錯」的機會。你在寫 prompt 前想清楚，AI 才能精準產出。不想清楚就讓 AI 猜——它猜的方向 50% 不是你要的。
LAYER 02 · LOGIC
邏輯：輸入要怎麼變成輸出？
想清楚：用什麼公式 / 規則 / 條件來處理。如果是計算類就寫公式；如果是分類類就寫條件分支；如果是文字處理就寫規則。用「如果 X 就 Y」這種白話寫，AI 會自己翻成程式。
BMI 的邏輯
· BMI = 體重 / (身高/100)²
· 如果 BMI < 18.5 → 偏瘦（黃）
· 如果 18.5 ≤ BMI < 24 → 正常（綠）
· 如果 24 ≤ BMI < 27 → 過重（橘）
· 如果 BMI ≥ 27 → 肥胖（紅）
報價試算器的邏輯
· 小計 = 單價 × 期間
· 折扣金額 = 小計 × 折扣%
· 若年付 → 再乘 0.97（額外 3%）
· 含稅 = 折扣後 × 1.05
實戰觀察 · 邏輯層是「業務 know-how」的所在
邏輯層是你的工具跟通用工具「不同」的地方。通用 BMI 工具只算數字；但如果你做的是「員工健康管理工具」——邏輯層可能加入「年齡 + 久坐時數」做更精確的判斷。你的業務 know-how 寫在邏輯層裡，就是你工具的核心價值。
LAYER 03 · OUTPUT
輸出：使用者要看到什麼？怎麼呈現？
想清楚：要顯示什麼數字 / 文字 / 圖表、用什麼版面、要不要顏色提示、要不要按鈕（重置 / 複製 / 分享）。「能看到什麼」直接決定使用者爽不爽——好的輸出讓人 2 秒掌握重點，爛的輸出讓人還要自己「解讀」。
BMI 的輸出
· 大字顯示 BMI 數值
· 分類標籤用顏色區別（綠 / 黃 / 橘 / 紅）
· 一句話建議（「正常範圍，繼續保持」）
· 「重新計算」按鈕
報價試算器的輸出
· 報價單區塊（客戶名 / 方案 / 折扣 / 合計大字）
· 「複製成 email」按鈕（自動帶文字）
· 「列印成 PDF」按鈕
· 「加到 Google Calendar」（若是時區工具）
⚡
練習：挑你常用的 3 個 web 工具（Google 翻譯、QR Code 產生器、UTM Builder⋯⋯），試著把每個拆成三層。你會發現絕大多數工具都很單純，背後沒什麼魔法——就是 Input → Logic → Output。拆完這 3 個，你就有信心自己做了。
SECTION 03 · 工具選擇15 分鐘

用哪個 AI 工具做？

2026 年有四個主流選擇。本課程主要用 Claude Artifacts——最穩、最快、輸出能即時看到效果、免費版即可用。

Claude Artifacts
本課首選
在 Claude 對話中直接生成可預覽的 HTML/JS 工具，右側面板即時跑、可分享連結。免費版即可用（每天 10-20 個 Artifacts）。
適合：幾乎所有個人小工具、簡報嵌入式 demo、客戶展示。學習成本最低。
Gemini Canvas
Google 整合
類似 Artifacts，內建在 Gemini，適合需與 Google Workspace 整合的工具。可直接存 Drive。
適合：要存進 Google Drive、給團隊用的工具。
ChatGPT Canvas
付費為主
類似但要 ChatGPT Plus 才能完整使用。互動 UI 仍是 HTML 但 Code Interpreter 可跑 Python。
適合：需要 Python 處理資料的工具（如分析 Excel）。
v0.dev / Bolt.new
進階選項
專業 AI 寫網頁工具，可生成完整 React app。學習曲線略高，免費額度有限。
適合：已試過 Artifacts 後想做更複雜的應用。
⚠️
本課程從 Claude Artifacts 開始。免費版每天可生 10-20 個 Artifacts，足夠學習與實戰。學會原理後切換到任何工具都通——因為「三層架構」與「prompt 寫法」是通用的。不要同時學 3 個工具——先精通 1 個。
SECTION 04 · 第一個工具實作25 分鐘

30 秒做出會議倒數計時器

直接示範。先用三層架構畫草稿（10 秒），再打開 Claude 貼 prompt（20 秒）：

4.1 · 三層草稿（先想再做）

INPUT
會議時長
分鐘數（預設 30）+ 會議主題（選填）
LOGIC
倒數 + 顏色變化
>5min 綠 / 5-1min 黃 / <1min 紅閃爍 / 到 0 音效
OUTPUT
大字 MM:SS
+ 主題顯示 + 開始/暫停/重置 3 按鈕

4.2 · 完整 prompt

# 完整 prompt（貼到 Claude） 請幫我做一個會議倒數計時器，HTML 單檔可即時預覽。 輸入 - 會議時長：分鐘數，預設 30 分鐘 - 會議主題：文字（選填） 邏輯 - 按「開始」後倒數 - 剩餘時間 > 5 分鐘 → 綠色 - 5 分鐘 ≥ 剩餘 > 1 分鐘 → 黃色 - 剩餘 ≤ 1 分鐘 → 紅色，開始閃爍 - 倒數結束時播放系統提示音 + 顯示「時間到」 輸出 - 大字（>100px）顯示 MM:SS 倒數 - 上方顯示會議主題（若有填） - 下方「開始 / 暫停 / 重置」三按鈕 - 簡潔風格，黑底白字、現代感 直接生成可預覽的 Artifact，不要解釋程式碼。

4.3 · 實測 · 學員視角

實測紀錄 · 第一次看到 Artifact
觀察 01
按下 Enter 後 Claude 在 5-10 秒內在右側面板生出一個可互動的計時器。黑底白字、有 3 個按鈕、輸入 30 分鐘按開始後開始倒數。第一個零代碼工具完成。
觀察 02
但可能遇到的問題：① 音效沒響（Artifact sandbox 限制）② 閃爍太快或太慢 ③ 手機版排版歪。這些都不影響核心功能，可在下一節 CH 4.2 的微調中解決。
為什麼這個 prompt 寫得好？
因為「輸入 / 邏輯 / 輸出」三層都明確了，包括邊界條件（顏色變化的閾值 5min / 1min）、互動細節（3 個按鈕）、視覺風格（黑底白字現代感）。AI 不需要猜任何東西，可以一次到位。

如果你只寫「做一個計時器」——AI 會猜你要白底還是黑底、要不要暫停、顏色要不要變，8 成猜錯，你要改 3-5 次。
SECTION 05 · 工具邊界15 分鐘

什麼適合、什麼不適合零代碼做

零代碼工具強大，但有邊界。試圖用它做超出邊界的東西，會浪費好幾天還做不出來。在動手之前先過這張表。

✓ 適合用零代碼做
- 計算 / 試算（BMI、利率、報價、ROI）
- 資料轉換（單位、時區、格式、URL encode）
- 文字處理（產生器、模板填充、UTM 組裝）
- 視覺呈現（圖表、Dashboard、進度條）
- 互動展示（流程圖、簡報嵌入、Demo 頁）
- 個人化清單（待辦、檢查表、評分卡）

判斷標準：能在「一個 HTML 檔內」跑完嗎？
✗ 不適合（需專業工具）
- 需要登入 / 帳號管理
- 需要存資料庫（多人共用的資料）
- 串接外部 API（Email、付款、CRM）
- 大量檔案處理 / 後端運算
- 即時通訊 / 多人協作
- 複雜的金流 / 安全性需求

需要這些 → 用 Make / n8n（Part 5）或找開發者
實戰判斷 · 3 秒決策法
看到一個需求時問自己：「使用者打開這個工具，用完後關掉，不需要帳號也不需要跟別人連線嗎？」如果是 → 零代碼能做。如果需要「記住我上次填了什麼」「我的同事也能看到」「串接公司的 CRM」→ 這超出 Artifacts 範圍。

不過「localStorage」可以讓工具在瀏覽器內記住設定（CH 4.2 會教），這能解決「記住我上次填了什麼」的部分需求。

本課程 Part 4 全部聚焦在「能在 HTML 檔內跑完」的工具——這已經涵蓋上班族 80% 的個人生產力需求。剩下的 20% 需要自動化（Part 5 的 Make / n8n）或專業開發。

SECTION 06 · 踩坑博物館10 分鐘

第一個工具的6 個典型踩坑

坑 01省略三層草稿直接寫 prompt
症狀：寫了「做一個 XX 工具」，AI 猜了一個方向，跟你想的差十萬八千里。改 3 次還不對。
修：強制自己先花 30 秒畫三層（紙上 / 腦中都行）。先問自己「輸入幾個欄位？每個什麼類型？邏輯怎麼算？輸出要什麼？」想清楚再打字。
坑 02「做一個 XX 工具」prompt 太短
症狀：AI 給的工具「能用但不對」——按鈕風格不是你要的、欄位名稱奇怪、輸出排版亂。
修：prompt 最少要有 3 個段落：輸入規格 + 邏輯規則 + 輸出設計。加上「設計風格」（米白底 / 暗黑 / 蘋果風）會更精準。
坑 03想一次做太多功能
症狀：第一版就想加「歷史紀錄 + 匯出 CSV + 多語言 + 手機版」，結果什麼都做到一半，沒一個能用。
修：v1 只做核心 1 個功能。確認核心能跑後，v2 再加第二個功能。每次只加 1 個功能。
坑 04手機版排版壞掉
症狀：桌面看正常，手機打開欄位擠在一起或滾動條超醜。
修：prompt 最後加一句 「行動裝置友善（螢幕 <768px 時垂直排列）」。這一句讓 AI 自動處理 responsive。
坑 05Artifact 內 clipboard 不靈
症狀：按「複製」但剪貼簿沒東西。Claude Artifact 的 sandbox 有時限制 clipboard API。
修：prompt 加 「複製成功後加 toast 提示；若 clipboard API 不可用，fallback 到選取文字供使用者手動複製」。
坑 06不知道怎麼分享
症狀：工具做完很滿意，但不知道怎麼給同事看。
修：Claude Artifact 右上角有「Share」按鈕（取得分享連結）。但連結綁你的帳號——若要永久分享，下載成 HTML 丟 GitHub Pages（CH 4.4 會教）。
SECTION 07 · 課堂動手15 分鐘
PRACTICE 01 · 跑一遍計時器
親手體驗「描述 → 即時預覽」
題目：照 Section 4 的 prompt，到 Claude 跑一次。重點不是「做出計時器」（你大概不缺這個），而是親身體驗「描述 → 即時預覽」這個流程的順暢度。跑完後試著改一件事：把黑底改成白底、或把按鈕改成圓形、或加一個「主題」的下拉選單。體驗微調的手感。
提示：微調時不要重打整個 prompt——直接追問「把背景改成白底、文字改深灰」。AI 會基於前面的 context 修改，比重來快 3 倍。
PRACTICE 02 · 拆解 + 設計你自己的工具
從你工作的痛點出發
題目：挑你工作中「想要但網路上找不到剛好對的」小工具。先用三層架構（紙上或腦中）畫出 Input / Logic / Output，每層至少 3 個具體描述。然後寫成完整 prompt 跑一次。

題目參考（任選一個或自己想）：
① 跨時區會議時間轉換器（下節會深入做）
② 客戶報價試算器
③ 課程倒數天數計算器
④ 簡報計時器（每頁建議停留時間）
⑤ 業績達成率視覺化（進度條 + 預警顏色）
提示：不用做到完美。v1 能跑就算成功——CH 4.2 會教你怎麼微調成「真的會用」的版本。
▸ 參考答案（業績達成率）
三層草稿：
Input：業務員姓名 × 5、各自目標金額、各自實際金額
Logic：達成率 = 實際 / 目標 × 100%；≥100% 綠、80-99% 黃、<80% 紅；按達成率排序
Output：水平進度條 × 5、每條顯示「實際 / 目標 (達成 X%)」、頂端顯示團隊整體達成率

這就是 CH 4.3 視覺化 dashboard 的基礎版。先做出來、跑通了再看下節怎麼加 Chart.js 互動。
SECTION 08 · 課後行動5 分鐘

帶著三層架構的眼光過一天

今天回家後做一件事：用你一天中遇到的 3 個「小不便」，各用三層架構拆一次（用手機備忘錄記）。例如：每天算加班費很煩（→ Input：上班時間 + 下班時間；Logic：超過 8h 的部分 × 1.34 倍；Output：加班費金額 + 複製功能）。
