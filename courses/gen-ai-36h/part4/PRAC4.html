Part 4 · 帶走範本
PRAC 4 · 含於 Part 4 共 8h

個人零代碼工具集
3 件作品上線

Part 4 的成果——3 個完整工具藍圖。每個都給三層架構規劃 + 完整 prompt + 風格指定 + 部署提示。跑完之後，你的 GitHub Pages 上會有 3 個可以分享的工具，這是 Part 4 最有形的 deliverable。

建議節奏：① 一週做一個（不要一次做完）。② 每個跑完整流程：prompt → 微調 → 風格 → 部署 → 分享。③ 第一個工具預期花 1-2 小時（含部署），熟練後第二、三個 30-40 分鐘。
作品 01 · DASHBOARD
個人 KPI 追蹤 Dashboard
追蹤你自己每月的 3-5 個關鍵指標（業績、運動天數、學習時數、客戶數⋯⋯）。每月底花 30 秒輸入數字，立即看到趨勢與達成率。可分享給主管當「自我管理證據」。
三層架構
輸入
  • 3-5 個 KPI 名稱
  • 每月目標
  • 每月實際數字
  • 備註（選填）
邏輯
  • 計算各月達成率
  • 計算 12 個月趨勢
  • 判斷異常月份
  • localStorage 永久保存
輸出
  • 當月達成率儀表盤
  • 12 個月趨勢圖
  • 各 KPI 進度條
  • 「分享連結」按鈕
完整 Prompt
請幫我做一個個人 KPI 追蹤 Dashboard，HTML 單檔可預覽。 輸入區 - 上方有「設定」按鈕，可定義 3-5 個 KPI（名稱、每月目標、單位） - 主畫面：當月份的 KPI 輸入欄位（每個 KPI 一個數字框） - 「儲存本月」按鈕 邏輯 - 用 localStorage 保存所有月份的 KPI 數據 - 自動計算：當月達成率、12 個月平均、趨勢方向（上升/下降） - 異常標記：偏離 12 個月平均超過 1 個標準差的月份 輸出區 頂端：當月狀態 - 大字顯示當月份 - 每個 KPI 一個小卡：實際 / 目標、達成率（顏色：≥100 綠、80-99 黃、<80 紅） 中段：12 個月趨勢 - 每個 KPI 一條折線圖（用 Chart.js） - 標出目標線（虛線） 底部： - 「匯出 JSON」按鈕（備份用） - 「匯入 JSON」按鈕（換裝置 / 重灌時恢復） - 「分享當月狀態截圖」按鈕（用 html2canvas） 設計 - 米白底 #f5f3ee - 主色 #7a9ea3 - Shippori Mincho 標題、Noto Sans TC 內文 - 卡片白底細邊框、無漸層、無陰影 - 行動裝置 friendly 特殊需求 - 第一次打開要有引導 onboarding（3 步：設定 KPI → 輸入第一筆 → 看結果） - 沒資料時要顯示空狀態插圖（用 SVG）+ 引導文字
風格選擇
建議無印良品風（沉靜，適合每天看不會煩）。若要更俐落，可改成蘋果風（白底 + 圓角 12px）。
部署提示：建一個 repo 叫 my-kpi，把 index.html 丟進去 → 啟用 Pages → 永久網址。建議加自己 bookmark 列為「我的 KPI」，每月底打開填一次。
作品 02 · 工作流
客戶會議準備卡產生器
每次跟客戶開會前，花 2 分鐘填表單，自動生成「會前 3 個關鍵問題 + 5 個對方可能丟出的難題 + 應對方向 + 最低 / 最高目標」的會議準備卡。可印出帶進會議室。
三層架構
輸入
  • 客戶名 / 公司
  • 會議目的
  • 我方角色
  • 已知背景
邏輯
  • 呼叫 Claude API（或讓使用者複製 prompt 自行貼）
  • 生成 3 區塊內容
  • localStorage 存歷史
輸出
  • 會議準備卡（A4 大小）
  • 「列印 PDF」按鈕
  • 「複製成 markdown」
完整 Prompt
請幫我做一個「客戶會議準備卡產生器」，HTML 單檔可預覽。 輸入欄位 - 客戶 / 公司名（文字必填） - 會議日期時間（datetime 預設明天 14:00） - 會議目的（textarea：「希望達成什麼？」） - 我方角色（下拉：業務 / PM / 主管 / 顧問 / 其他） - 已知背景（textarea，可選填） - 客戶可能在意的議題（textarea，可選填） 邏輯 此工具不直接呼叫 AI（避免 API key 問題）。 按「生成準備卡」後： 1. 將輸入組合成一個結構化 prompt（用以下範本，見下方） 2. 自動複製到剪貼簿 3. 顯示提示：「已複製，請貼到 Claude 或 ChatGPT，把回應貼回下方」 4. 提供大 textarea 讓使用者貼回 AI 回應 5. 解析回應後顯示成準備卡 組合的 prompt 範本（複製到剪貼簿用）： """ 你是資深會議顧問。我即將參加： - 客戶：[客戶名] - 會議目的：[目的] - 我的角色：[角色] - 已知背景：[背景] - 客戶在意：[議題] 請給我會議準備卡，包含： 1. 開場前 3 分鐘該確認的事 2. 我必須問清楚的 3 個關鍵問題（附『為什麼這個重要』） 3. 客戶可能丟出的 3 個難題 + 我的應對方向 4. 本場會議最低目標、最高目標 5. 會後立即要做的 2 件事 格式用 markdown，每段加標題。 """ 輸出 - 會議準備卡用 A4 直式排版 - 上方：客戶名 + 日期 + 大字「會議準備卡」 - 中段：5 個區塊（用 Card 樣式）依序展示 - 下方：「列印 PDF（A4）」按鈕、「複製整張為 markdown」按鈕 - 右上角：「歷史紀錄」按鈕（顯示過去 10 張準備卡，可重新打開） 設計 - 印刷友善：列印時用純白底、深灰文字、適合 A4 邊距 - 螢幕版用米白底 - 主色 #7a9ea3 - 字型：Shippori Mincho 標題 + Noto Sans TC 內文
風格選擇
建議「新聞網站 / 編輯風」——襯線標題增加正式感，列印出來像一份專業文件。
部署提示：這個工具會「跨會議使用」，建議獨立 repo 或路徑 my-tools/meeting-prep/。可將網址加進 Chrome bookmark bar，會議前 5 分鐘打開就好。
作品 03 · 簡報神器
簡報計時器 + 提詞器
講課 / 演講 / 內訓時放在副螢幕上：上方倒數計時、中央顯示當前要講的重點、下方有上一頁 / 下一頁按鈕。讓你不用偷看手機就能掌握時間與重點。
三層架構
輸入
  • 總時長（分鐘）
  • 每張投影片重點
  • 各段預計時間
邏輯
  • 倒數計時
  • 顏色階段警示
  • 記住偏好時長
輸出
  • 大字計時器
  • 當前重點顯示
  • 進度條
完整 Prompt
請幫我做一個「簡報計時器 + 提詞器」，HTML 單檔可預覽，全螢幕友善。 設定模式 - 上方欄位：演講主題、總時長（分鐘） - 大 textarea：每行一個重點，格式「[時間]||[重點內容]」 範例： 3||歡迎、自我介紹 5||議題背景與動機 10||核心方法 (深入) 3||案例展示 4||Q&A 與結論 - 「開始」按鈕 演講模式（按開始後進入） - 隱藏所有設定，全螢幕模式 - 頂端區塊：總倒數時間（大字 100px）+ 當前段落剩餘時間 - 中央區塊（最大）：當前段落的重點文字（粗體大字 60px） - 下方區塊： - 進度條（按段落分段，當前段落高亮） - 「← 上一段」「下一段 →」按鈕 - 小字：當前段落 / 總段落（如「2 / 5」） - 「重置」「離開全螢幕」按鈕 邏輯 - 自動倒數總時間 - 當前段落時間到 → 自動跳下一段（可由設定關閉，預設開） - 顏色階段： - 總剩餘 ≥ 30% → 綠 - 10-30% → 黃 - < 10% → 紅 + 閃爍 - 鍵盤快捷鍵：space 暫停 / 繼續、← 上一段、→ 下一段、F 全螢幕 輸出 - 全螢幕模式時整個畫面是計時器 - 適合放在副螢幕（演講者監看） - 結束時跳出「演講結束！總時長 XX:XX，預計 XX:XX」摘要 設計 - 暗黑背景 #1a1a1a（避免反光） - 文字白色 #fff - 強調色（時間警示）：綠 #5a7a5a、黃 #c9963a、紅 #b5703a - 字體無襯線、超大字重 700 - 按鈕大、易點 資料保存 - 設定用 localStorage 保存，下次打開預設用上次的
風格選擇
這個用「暗黑 / Dev Mode」——演講場景燈光通常昏暗，深色背景才不刺眼，且避免投影到副螢幕的反光。
部署提示：這個工具是「演講當下」用的，網址要好記。建議用 my-tools/timer/ 或自訂網域 timer.yourdomain.com。第一次正式用前先在家排練一次。

3 個工具的 prompt 都跑過、第一版 Artifact 完成
每個工具跑了 2-3 輪細節微調
每個工具套用了風格範本
下載成 HTML 檔備份
建立 GitHub repo（可一個 repo 放三個資料夾）
啟用 GitHub Pages 取得永久網址
3 個網址分別實際使用一次（不只「跑出來」）
