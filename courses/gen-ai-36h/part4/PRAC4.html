Part 4 的成果——3 個完整工具藍圖。每個都有真實場景、三層架構規劃、完整 prompt、微調示範、風格指定與部署提示。跑完之後，你的 GitHub Pages 上會有 3 個可分享的工具。這是整個 Part 4 最有形的 deliverable。

建議節奏：一週做一個，不要一次做完。每個跑完整流程：prompt → 微調 → 風格 → 部署 → 分享 → 收反饋。第一個工具預期花 1.5-2 小時（含學部署），熟練後第二、三個 30-40 分鐘。

作品 01 · DASHBOARD
個人 KPI 追蹤 Dashboard
追蹤你每月的 3-5 個關鍵指標（業績、客戶數、運動天數、學習時數）。每月底花 30 秒輸入數字，立即看到趨勢與達成率。可分享給主管當「自我管理證據」。
情境 · 年度考核的秘密武器
業務 Eric 每月底打開自己的 KPI Dashboard，花 30 秒填入當月業績、客戶拜訪數、新客數。12 個月下來，年度考核時他打開這張圖——12 個月的趨勢折線圖、每個 KPI 的達成率進度條、異常月份自動標紅。主管問「你覺得今年表現如何？」他直接投影 dashboard：「上半年業績平均達成 105%，Q3 因為某客戶流失掉到 78%，Q4 回升到 112%。」主管不用翻報表，當場給了 A 評。
三層架構
INPUT 輸入
  • 「設定」按鈕：定義 3-5 個 KPI（名稱、每月目標、單位）
  • 主畫面：當月 KPI 輸入欄位
  • 「儲存本月」按鈕
  • 月份選擇器（回填歷史）
LOGIC 邏輯
  • localStorage 持久化所有月份數據
  • 計算：當月達成率、12 月平均、趨勢方向
  • 異常偵測：偏離均值 > 1 SD 標紅
  • 數據匯出 / 匯入（JSON）
OUTPUT 輸出
  • 當月 KPI 卡片（實際/目標/達成率）
  • 12 月趨勢折線圖（Chart.js）
  • 各 KPI 進度條（綠黃紅）
  • 「分享截圖」按鈕（html2canvas）
完整 Prompt
# Tool A · 個人 KPI 追蹤 Dashboard 請幫我做一個個人 KPI 追蹤 Dashboard，HTML 單檔可預覽。 設定模式 - 上方有「設定」按鈕，點開 modal 可定義 3-5 個 KPI - 每個 KPI 欄位：名稱、每月目標數字、單位（萬元/個/次/小時） - 預設 3 個示範 KPI：業績(100萬)、客戶拜訪(20次)、新客戶(5個) - 設定用 localStorage 保存 主畫面：輸入 + 顯示 - 頂端：當前月份（可切換月份下拉） - 每個 KPI 一個輸入框 + 「儲存本月」按鈕 - 按儲存後進入「顯示模式」 顯示模式 頂端卡片列（3-5 張小卡並排）： - 每個 KPI 一張卡：實際 / 目標、達成率% - 顏色：≥100% 綠 #5a7a5a、80-99% 黃 #c9963a、<80% 紅 #b5703a - 卡片內顯示 vs 上月箭頭（↑↓） 中段趨勢區： - 每個 KPI 一條折線圖（Chart.js） - 目標線用虛線疊加 - 異常月份（偏離均值 > 1 SD）用紅色大點標記 - 可切換只看單一 KPI 或全部疊加 底部功能列： - 「匯出 JSON」按鈕（下載備份檔） - 「匯入 JSON」按鈕（上傳恢復資料） - 「分享當月截圖」按鈕（html2canvas 截圖 → 下載 PNG） Onboarding 引導 第一次打開（localStorage 無資料）時顯示 3 步引導： ① 「先設定你的 KPI」→ 打開設定 modal ② 「輸入第一筆資料」→ 跳到輸入模式 ③ 「看看結果」→ 顯示 Dashboard 空狀態用 SVG 插圖 + 引導文字 設計 - 米白底 #f5f3ee、卡片白底 #fff - 主色 #7a9ea3 - Shippori Mincho 標題、Noto Sans TC 內文 - 卡片 border 1px solid #d8d4cb、圓角 6px、無漸層無陰影 - 響應式：手機卡片垂直堆疊、圖表 100% 寬

作品 02 · 工作流工具
客戶會議準備卡產生器
每次跟客戶開會前，花 2 分鐘填表單，自動組裝成結構化的 AI prompt。把 prompt 貼到 Claude/ChatGPT，將回應貼回來，工具自動解析成一張漂亮的「會議準備卡」。可印出帶進會議室。
情境 · 客戶會議前的 2 分鐘
業務 Grace 明天要跟一個不太熟的客戶談續約。她打開會議準備卡工具，填了 4 個欄位：客戶名、會議目的「討論明年續約條件」、我的角色「主談業務」、已知背景「客戶今年縮減預算 20%」。工具自動組成一段 AI prompt，她複製貼到 Claude，30 秒後收到：3 個必問問題、3 個客戶可能刁難的點 + 應對策略、最低 / 最高目標。她列印出來帶進會議室——會議結束後客戶竟然主動加碼了 10%，因為 Grace 精準命中了客戶最在意的痛點。

INPUT 輸入

完整 Prompt
# Tool B · 客戶會議準備卡產生器 請幫我做一個「客戶會議準備卡產生器」，HTML 單檔可預覽。 輸入欄位 - 客戶 / 公司名（text，必填） - 會議日期時間（datetime-local，預設明天 14:00） - 會議目的（textarea：「你希望這場會議達成什麼？」） - 我方角色（下拉：業務 / PM / 主管 / 顧問 / 技術 / 其他） - 已知背景（textarea，可選填，placeholder：「關於這個客戶你已經知道什麼？」） - 客戶可能在意的議題（textarea，可選填） 邏輯（不直接呼叫 AI API） 按「生成準備卡」後執行： 1. 將所有輸入組合成以下結構化 prompt： --- 你是資深會議策略顧問。我即將參加以下會議： - 客戶：[客戶名] - 日期：[日期] - 會議目的：[目的] - 我的角色：[角色] - 已知背景：[背景] - 客戶在意：[議題] 請產出完整的會議準備卡，包含以下 5 個區塊： 區塊 1：開場前 3 分鐘確認事項（3 條） 區塊 2：必問的 3 個關鍵問題（每個附「為什麼重要」） 區塊 3：客戶可能丟出的 3 個難題 + 應對方向 區塊 4：最低目標 / 理想目標 / 最高目標 區塊 5：會後立即要做的 2 件事 格式用 markdown，每個區塊用 ## 標題。 --- 2. 自動複製到剪貼簿 3. 顯示提示框：「prompt 已複製！請貼到 Claude / ChatGPT，然後把回應貼回下方。」 4. 顯示一個大 textarea：「貼入 AI 回應」 5. 按「解析回應」後，將 markdown 解析成準備卡版面 準備卡版面 - A4 直式排版（列印友善） - 頂端：客戶名 + 日期 + 大字「會議準備卡」 - 中段：5 個區塊依序展示，每區塊用 Card 樣式 - 底部按鈕列： ① 「列印 PDF（A4）」（window.print + @media print 優化） ② 「複製整張為 Markdown」 ③ 「新建準備卡」（重置） 歷史紀錄 - 右上角「歷史」按鈕 → 顯示過去 10 張準備卡（localStorage） - 點任一張可重新展開查看 - 可刪除單張紀錄 設計 - 螢幕版：米白底 #f5f3ee - 列印版：純白底、深灰文字、適合 A4 邊距、隱藏按鈕和輸入區 - 主色 #7a9ea3 - Shippori Mincho 標題、Noto Sans TC 內文 - 區塊卡片有淺灰背景、細邊框 - @media print 規則要完善

作品 03 · 簡報神器
簡報計時器 + 提詞器
講課、演講、內訓時放在副螢幕上：大字倒數計時 + 當前段落的提詞重點 + 進度條。讓你不用偷看手機就能掌握時間與要講的內容。全螢幕暗黑設計，避免反光。
情境 · 內訓的完美時間控制
講師小如每次內訓都超時。她做了這個計時器，設定好每段主題的時間分配，放在筆電螢幕上（投影看大螢幕，筆電看計時器）。螢幕上永遠顯示「你現在該講：核心方法（剩 7:23）」+ 整體剩餘時間。當剩餘 < 10% 時文字變紅閃爍——提醒她該收尾了。從此她的內訓從來沒超時過，學員評價「內容精準不拖泥帶水」。
情境 · 客戶簡報的安全網
業務 Jason 要對客戶做 20 分鐘的產品簡報。他設了 5 個段落：開場(2 分) → 痛點(3 分) → 方案(8 分) → 案例(4 分) → 結論(3 分)。每段的提詞欄寫了要講的關鍵數字和金句。即使緊張忘詞，低頭看一眼筆電就知道「現在該講方案特色，還剩 5 分 12 秒」。
三層架構
INPUT 輸入
  • 演講主題（文字）
  • 總時長（分鐘）
  • 每段：時間 || 提詞內容
  • 是否自動跳下一段
LOGIC 邏輯
  • 倒數計時（總 + 段落）
  • 顏色階段警示
  • 鍵盤快捷鍵控制
  • localStorage 保存設定
OUTPUT 輸出
  • 大字計時器（100px）
  • 當前提詞（60px）
  • 分段進度條
  • 結束摘要
完整 Prompt
# Tool C · 簡報計時器 + 提詞器 請幫我做一個「簡報計時器 + 提詞器」，HTML 單檔可預覽，全螢幕友善。 設定模式（預設顯示） - 演講主題（text） - 總時長（number，分鐘） - 大 textarea：每行一個段落，格式「分鐘||提詞內容」 範例： 3||歡迎、自我介紹、破冰 5||議題背景與動機（為什麼要做這件事） 10||核心方法（深入講解，重點數字：轉換率 37%、效率提升 2.5x） 3||案例展示（A 客戶：省 200 小時/年） 4||Q&A 與結論（Call to action：本週五前回覆） - Checkbox：「段落時間到自動跳下一段」（預設開） - 「開始演講」按鈕 演講模式（按開始後進入） 頂端區塊（20% 高度）： - 總倒數時間（大字 100px，粗體） - 右側小字：「當前段落剩餘 X:XX」 中央區塊（60% 高度，最大）： - 當前段落的提詞文字（粗體大字 48-60px） - 淺色小字：段落標題（如「3 / 5 · 核心方法」） 底部區塊（20% 高度）： - 分段進度條（每段一格，當前段落高亮 + 動畫填充） - 「← 上一段」「下一段 →」大按鈕 - 小字：「2 / 5」（當前段/總段） - 「暫停」「重置」「離開全螢幕」按鈕 邏輯 - 總時間倒數，每秒更新 - 段落時間到 → 自動跳下一段（或手動） - 顏色階段（全域，看總剩餘）： - ≥ 30% → 綠 #5a7a5a - 10-30% → 黃 #c9963a - < 10% → 紅 #b5703a + 閃爍動畫 - 鍵盤快捷鍵： Space = 暫停/繼續 ← = 上一段 → = 下一段 F = 全螢幕切換 Esc = 退出全螢幕 結束畫面 倒數到 0 或手動結束後： - 顯示「演講結束」大字 - 摘要：實際花了 XX:XX（預計 XX:XX） - 每段實際用時 vs 預計用時對比表 - 「回到設定」按鈕 設計 - 設定模式：米白底 #f5f3ee（正常閱讀） - 演講模式：暗黑背景 #1a1a1a（避免反光投影） - 文字白色 #fff - 顏色警示：綠 #5a7a5a、黃 #c9963a、紅 #b5703a - 字體全無襯線、超大字重 700 - 按鈕大而明顯（44px+ 高），方便快速點擊 - 全螢幕時 requestFullscreen API 資料保存 - localStorage 保存上次的設定（主題、段落） - 下次打開自動載入

3 件作品的常見踩坑

01localStorage 資料消失
症狀：換瀏覽器或清除快取後，KPI Dashboard 資料全沒了。
修法：localStorage 是綁瀏覽器的。所以 prompt 裡才要「匯出 / 匯入 JSON」功能——每月備份一次 JSON 檔到 Google Drive。追問 AI：「加一個定期提醒，每月 1 日打開工具時彈出提示：『建議匯出備份』」
02會議準備卡的 AI 回應解析失敗
症狀：把 Claude 回應貼回來，但工具沒辦法正確拆分成 5 個區塊。
修法：AI 的回應格式可能不穩定。追問：「解析回應時，請用寬鬆匹配：用 ## 或 ### 標題拆區塊，如果沒找到標題就依空行拆。如果完全無法解析，直接以 markdown 原文顯示，不要報錯。」
03計時器全螢幕在手機上沒用
症狀：手機上按「全螢幕」沒反應，或進去後字太小。
修法：iOS Safari 不支援 requestFullscreen。追問：「手機上不要用 Fullscreen API，改為隱藏設定區 + 按鈕放大 + 文字縮放到 viewport 寬度。並在 meta viewport 加 user-scalable=no。」
04Chart.js 在舊手機上跑很慢
症狀：KPI Dashboard 在 3-4 年前的手機上載入很久、切換卡頓。
修法：Chart.js 的動畫是效能殺手。追問：「手機版 (max-width: 768px) 時請關閉 Chart.js 動畫：animation: false。同時把圖表高度固定為 200px，不要 responsive 縮放。」
05列印 PDF 排版跑掉
症狀：會議準備卡按「列印」，但紙本排版跟螢幕完全不同。
修法：追問：「請完善 @media print 規則：① 隱藏所有按鈕、輸入區、navigation ② 背景改純白 ③ 文字改純黑 ④ 設定 page-break-inside: avoid 在每個卡片區塊 ⑤ 邊距 15mm。」
GUIDED PRACTICE
挑一個工具，完整走完從 Prompt 到部署
選 Tool A / B / C 其中一個，完成以下完整流程：
① Prompt：複製上面的完整 prompt，貼到 Claude
② 實測：用真實資料（或模擬資料）測試功能是否正確
③ 微調 2 輪：至少修 2 個你不滿意的地方
④ 風格覆寫：套一個風格範本（參考 CH 4.4）
⑤ 部署：下載 HTML → 上傳 GitHub → 啟用 Pages
⑥ 分享：把網址發給 1 位同事或朋友，請對方試用 + 給一句反饋
建議順序：第一個做 Tool A（KPI Dashboard），因為有 localStorage + Chart.js 的學習價值最高。第二個做 Tool C（計時器），全螢幕 + 暗黑是新挑戰。第三個做 Tool B（準備卡），涉及 AI prompt 組裝流程，最有創意空間。
展開進階挑戰
做完 3 個工具後的進階挑戰：

挑戰 1：做一個「工具集首頁」——在 GitHub repo 根目錄建 index.html，列出 3 個工具的連結 + 簡介 + 截圖。像是你的個人工具 portfolio。

挑戰 2：給 KPI Dashboard 加一個「團隊版」功能——可以追加多個成員的 KPI，比較誰達成率最高。

挑戰 3：給計時器加一個「匯入 Markdown 大綱」功能——貼入演講的 markdown 大綱，自動拆段落。

3 個工具的完整 prompt 都跑過，第一版 Artifact 生成完成

每個工具套用了一個風格範本（無印 / 蘋果 / 企業 / 暗黑 / 新聞 / 活潑）
3 個工具都下載成 HTML 檔備份在本機

啟用 GitHub Pages，取得 3 個永久網址
