AI 自動化場景設計從發想到上線的完整流程

Part 5 的最終成果——帶你從「發想 → 設計 → 實作 → 上線」做完一條真能用的自動化工作流。4 個工具依序使用：盤點需求 → 設計流程 → 動手做 Demo → 上線前 Checklist。

使用方式：依序走 Tool A → B → C → D。Tool A 幫你找出「真的值得自動化的事」。Tool B 用 7 格工作單設計完整流程。Tool C 給你一個完整 Demo 照著做。Tool D 上線前跑一次 10 項 checklist。全程預期 2-4 小時。

TOOL A
需求盤點：8 問自評
不是所有事都該自動化。用這 8 個問題盤點——5 題以上答「是」才值得投入時間。答不到的，手動處理反而更快。
Q1
這件事每週至少發生 5 次？
低頻事件（一週 1-2 次）手動做比建自動化快。自動化的 ROI 來自頻率。
Q2
每次處理至少花 10 分鐘？
建一條自動化通常花 2-4 小時。如果每次只省 1 分鐘，要跑 120-240 次才回本。每次省 10 分 = 12-24 次就回本。
Q3
步驟可以描述成「如果 X 就 Y」？
有明確規則才適合自動化。「看情況」「憑感覺」「要跟客戶聊過才知道」的事，AI 做不好。
Q4
錯了能承受？
自動化偶爾會出包（AI 分錯、API 當機）。「錯一筆沒關係，補就好」可做。「一錯就完蛋」的事（金流扣款、法律合約）保留人工把關。
Q5
輸入來源是數位的？
Email、表單、檔案可以自動接。還要客戶「口頭告訴你」「拍照傳 LINE」「手寫傳真」的事，自動化斷點太多。
Q6
你能清楚說出「成功的樣子」？
「客戶填表後 10 分鐘內收到專屬報價信」——這種具體目標。說不清楚的，做完你也不知道有沒有成功。
Q7
涉及的工具都有 API / 整合？
在 Make / n8n 搜尋一下——你需要的 app 有模組嗎？沒有的話要用 HTTP + API，門檻高一截。
Q8
你有權限改變這個流程？
有些流程公司規定必須人工簽核。你能跳過/改變的才適合自動化。否則做了也用不上。
核心觀念 · 不該自動化的 3 種事
① 一個月才做一次的事（手動 15 分鐘 vs 建自動化 4 小時 → 要 16 個月才回本）。② 需要創意判斷的事（如「這篇文案好不好」——AI 能幫忙但不能自動決定）。③ 錯不得的高風險事（如自動轉帳、自動簽約——留人工把關）。
真實情境 · 行政主管 Ruby 的選擇
Ruby 列了 5 個想自動化的事。用 8 問盤點後：「報名表處理」8/8 全是 → 最優先。「月報整理」只有 3/8（頻率低、每月一次）→ 不做。「客訴分派」6/8 → 第二做。不是做最酷的，是做最值得的。
TOOL B
流程設計工作單
通過 Tool A 的需求，用這份 7 格工作單把流程「畫」出來。先在紙上想清楚，再去 Make/n8n 拖模組——沒做這步的人花 80% 時間在除錯。
1
觸發事件
什麼事件觸發？
如：收到新 email / 表單新提交 / 每天 9 點 / 新檔案上傳
頻率？
每天幾次？即時需求還是可以延遲 15 分鐘？
工具選擇
Make（需 24/7）/ n8n（本機/高頻）/ 串接
2
輸入資料
資料長什麼樣？
幾個欄位？文字/數字/檔案？中文/英文？長度？
需要清洗嗎？
有沒有多餘空白、格式不統一、需要合併欄位的情況？
3
AI 處理
AI 做什麼？
分類 / 摘要 / 評估 / 翻譯 / 產生回覆？
輸出格式
JSON 欄位有哪些？每個欄位的值範圍？
用哪個模型？
Claude / ChatGPT 不同檔次？成本考量？
4
條件分支
分幾條路徑？
每條路徑的觸發條件是什麼（AI 回傳的哪個欄位、什麼值）？
Fallback 路徑
AI 判斷不出來（confidence 低）走哪裡？通常是人工處理。
5
動作
每條路徑做什麼？
歸檔 Sheet / 寄 email / 發 Slack / 建 Notion 頁 / 回覆客戶...
通知誰？
哪些人需要被通知？用什麼管道？
6
錯誤處理
API 失敗怎辦？
重試？通知管理員？存到待處理清單？
資料消失怎辦？
是否全程記錄到 Sheet？失敗的資料有備份嗎？
7
成功指標
怎麼知道成功了？
具體的驗收標準（如：客戶填表後 5 分鐘內收到回覆信）
月省多少時間？
估算：每次省 X 分 × 每月 Y 次 = 月省 Z 小時
TOOL C
完整 Demo：客戶詢價自動分派
以下是一個完整的 14 步 Demo，用 Make 建出「客戶詢價表單 → AI 分類 → 自動分派 + 通知」。你可以照做，也可以改成你的場景。
DEMO 場景
公司官網有一個「聯絡我們」Google Form。每天收到 10-30 筆。內容混雜詢價、客訴、求職、合作。目前是一位行政助理每天手動分類轉寄，每天花 45 分鐘。自動化後歸零。

PHASE 1 · 準備工作
1
建 Google Form
4 個欄位：姓名（短文字）、Email（短文字）、主旨（短文字）、詳細內容（長文字）。表單標題「聯絡我們」。
2
建 Google Sheet
建一個「客戶分類紀錄」Sheet，表頭：A:時間 B:姓名 C:Email D:主旨 E:AI分類 F:摘要 G:緊急度 H:原始內容。
3
準備 Claude API Key
到 console.anthropic.com → API Keys → Create Key。複製備用。免費額度 $5 夠跑這個 Demo 很久。
PHASE 2 · 建 Make Scenario
4
建新 Scenario + Google Forms Trigger
Make → + Create a new scenario → 搜尋 Google Forms → 選 Watch Responses → 授權 Google → 選你的表單。
5
加 Anthropic Claude 模組
搜尋 Anthropic → 選 Create a Message → 貼 API Key → Model: claude-3-5-sonnet → System prompt 和 User message 見下方完整 prompt。
6
加 JSON Parse 模組
搜尋 JSON → 選 Parse JSON → JSON string 選 Claude 模組的 Content[].Text。這會把 AI 回傳的 JSON 字串解析成可用的欄位。
7
加 Router（4 條路徑）
在 JSON Parse 後加 Router。設定 4 條路徑的 Filter：
路徑 1: category = inquiry
路徑 2: category = complaint
路徑 3: category = partnership
路徑 4: Fallback（其他）
PHASE 3 · 設定每條路徑
8
路徑 1（inquiry）→ Sheet + Slack #sales
Sheet: Add Row（寫入分類紀錄）→ Slack: Send Message 到 #sales channel，內容含客戶名、摘要、Email。
9
路徑 2（complaint）→ Sheet + Slack #cs-urgent
Sheet: Add Row → Slack: Send Message 到 #cs-urgent，加上 @channel 確保所有人看到。
10
路徑 3（partnership）→ Sheet + Email 主管
Sheet: Add Row → Gmail: Send Email 寄給業務主管，附上 AI 摘要和客戶聯絡資訊。
11
路徑 4（other / fallback）→ Sheet only
只存 Sheet 不通知。每週人工巡一次這類資料即可。
PHASE 4 · 測試 + 上線
12
準備 4 筆測試資料
在 Google Form 分別填入 4 種類型的訊息：① 詢價信 ② 客訴信 ③ 合作邀約 ④ 不明訊息。確保每條路徑都能被測到。
13
Run once 逐筆測試
按 Run once → 確認 AI 分類正確、Sheet 有新行、Slack/Email 收到通知。如果分類錯誤 → 回去調 prompt。
14
開啟排程 + 切 ON
Scheduling → Every 15 minutes → 左下角開關切 ON。你的第一條 AI 自動化正式上線了。
# System Prompt（放在 Make Anthropic 模組的 System 欄位） 你是客服分類助理。只回傳 JSON，不要加任何說明文字。 # User Message（放在 Message 欄位） 以下是客戶透過官網「聯絡我們」表單送來的訊息。 請分類並產出結構化摘要。 姓名：{{Google Forms.姓名}} Email：{{Google Forms.Email}} 主旨：{{Google Forms.主旨}} 內容：{{Google Forms.詳細內容}} 分類規則（只能分為以下 4 類之一）： - inquiry：詢價、想了解方案、要報價 - complaint：客訴、不滿、要退款、功能壞了 - partnership：合作邀約、媒體、異業結盟 - other：求職、無關、垃圾、判斷不出來 輸出格式： { "category": "inquiry|complaint|partnership|other", "confidence": 0.0-1.0, "summary": "一句話摘要（30字內）", "urgency": "high|medium|low" } 規則： - 只回傳 JSON，不要用 markdown code block - confidence 低於 0.7 的歸 other - complaint 一律 urgency "high" - inquiry 一律 urgency "medium"
Demo 坑 1Claude 回傳帶 markdown
症狀：Claude 回 ```json {...} ```，JSON Parse 模組失敗。
修：在 System prompt 加 「不要用 markdown code block 包裝，直接回傳 JSON」。如果還是有，在 JSON Parse 前加一個 Text Parser 用 regex 去掉 ```json 和 ```。
Demo 坑 2Router 沒有 Fallback
症狀：AI 回傳一個意料之外的 category（如 "general"），4 條路徑都沒進，資料消失。
修：路徑 4 不要設 Filter（設成 Fallback）——只要不符合前 3 條的，全走路徑 4。永遠要有一條兜底路徑。
Demo 坑 3Google Forms 授權每 7 天過期
症狀：跑了一週後突然不觸發，log 顯示 401。
修：Make 的 Google 授權用的是 OAuth refresh token。到 Connections → Google → 按 Reauthorize。或把 Google API 的 OAuth Client 設成 Production 模式（需 Google 審核）。
Demo 坑 4表單沒有新回覆但 ops 一直跑
症狀：沒人填表但 ops 每天少 96（15 分鐘 polling × 96 次/天）。
修：Google Forms Watch 是 Polling 觸發，有沒有新回覆都耗 ops。改成每小時觸發（24 ops/天）或用 Apps Script Webhook 觸發（0 ops when idle）。
Demo 坑 5API 費用意外暴增
症狀：月底 Anthropic 帳單比預期高。
修：用便宜的小模型（如 Claude Haiku 檔次）做分類就夠了——分類不需要頂級模型的推理能力。在 Anthropic console 設 usage limit（月上限 $10）。
GUIDED PRACTICE · 跟著 Demo 做
照 14 步完成你的第一條 AI 自動化
選擇一：完全照著 Demo 做（客戶詢價分派），體驗完整流程。
選擇二：把 Demo 的場景改成你在 Tool A/B 設計的場景——改表單欄位、改 AI prompt 的分類、改 Router 路徑。

目標：跑通 Run once 並看到 Sheet 新增一行 + Slack/Email 收到通知。
提示：先完全照做跑通一次（選擇一），確認流程理解正確後再改成自己的版本。兩次都做最扎實。
TOOL D
上線前 Checklist
Workflow 跑通了不等於能上線。上線前跑一次這 10 項 checklist，避免上線第一天就出包。
1. 測試資料覆蓋率——每條 Router 路徑至少跑過 1 筆測試。特別測「模糊 / 邊界」案例。
2. Error Handler 設好了——API 失敗、JSON Parse 失敗時不會讓資料消失。至少存到 error sheet。
3. Fallback 路徑存在——AI 回傳意料之外的值時有兜底。不要假設 AI 100% 正確。
4. 額度估算完成——月 ops 用量算過，不會月中爆額。（觸發頻率 x 模組數 x 30）
5. API 費用設限——OpenAI / Anthropic console 設了月消費上限（建議 $10-20）。
6. 授權不會過期——Google OAuth 確認是長期有效。設日曆提醒每月 1 號檢查 Connections。
7. Sheet 表頭固定——歸檔 Sheet 的表頭不要手動改動（會影響 Make 的欄位映射）。
8. 通知管道確認——Slack channel / Email 收件人都確認正確。不要發到錯的 channel。
9. 排程頻率合理——不需要即時的不要設太頻繁。每小時就夠的別用每 15 分鐘。
10. 告知團隊——讓相關同事知道「這條流程自動化了」、通知會從哪個 channel 來、出問題找誰。
GUIDED PRACTICE · Checklist 實跑
對你的 Workflow 跑一次 10 項 Checklist
題目：用你剛才建好的 Workflow（Demo 或自訂版本），逐項對照 Checklist 的 10 條。每條標記「已完成 / 需改善 / 不適用」。至少 8 條「已完成」才適合上線。
提示：最常被忽略的是 #2（Error Handler）和 #4（額度估算）。這兩條在 Demo 階段不會出事，但上線 1 週後一定會遇到。
GUIDED PRACTICE · 時間 ROI 估算
算出你的自動化每月省多少時間
題目：填入你的場景數字：
A = 每次手動處理花 _____ 分鐘
B = 每月發生 _____ 次
C = 建這條自動化花了 _____ 小時

月省時間 = A x B 分鐘 = _____ 小時
回本天數 = C 小時 / (月省時間 / 30 天) = _____ 天
提示：常見結果：每次 10 分鐘 x 每月 100 次 = 月省 16.7 小時。建置 4 小時 → 7 天回本。之後每月都是純賺。
