Part 5 · 帶走範本
PRAC 5 · 含於 Part 5 共 6h

AI 自動化場景設計
一條完整工作流上線

Part 5 的最終成果——帶你從「發想 → 設計 → 實作 → 上線」做完一條真能用的自動化。含 4 工具：① 需求盤點 8 問 ② 流程設計工作單 ③ 完整實作 Demo ④ 上線 Checklist。

怎麼使用：① 先用 Tool A 找出「你真的該自動化的事」（不是酷炫的事） ② 用 Tool B 的 7 格工作單設計流程 ③ 對照 Tool C 的 Demo 實作 ④ 用 Tool D 上線前跑一次 checklist。全程預期 2-4 小時，依流程複雜度而定。
TOOL A
需求盤點：8 問自評
不是所有事都該自動化。用這 8 個問題盤點——只要 5 題以上答「是」，就是值得花時間做的。否則自動化反而浪費精力。
Q1
這件事每週至少發生 5 次？
低頻事件手動處理比建自動化快。一週 1-2 次的事通常不值得做。
Q2
每次處理至少花 10 分鐘？
省下的時間要能抵過建置時間（通常 2-4 小時）。一次只省 1 分鐘的事多半不划算。
Q3
步驟可以描述成「如果 X 就 Y」？
流程有明確規則才適合自動化。「看情況」「憑感覺」的事 AI 做不好。
Q4
錯了能承受？
自動化偶爾會出包。「錯一筆沒關係」可做；「一錯就完蛋」（如金流、法律）的事保留人工把關。
Q5
輸入來源是數位的？
Email、表單、檔案可自動接。還要請客戶「口頭告訴你」「拍照傳 LINE」的事，自動化斷點多。
Q6
你能清楚說出「成功的樣子」？
「客戶填表後 10 分內收到專屬報價信」這種具體目標。說不清楚的，自動化做完你也不知道有沒有成功。
Q7
可以追蹤結果（知道跑了幾次、對了幾次）？
沒 monitoring 的自動化就是黑箱。一定要有 Google Sheet log 或其他記錄。
Q8
相關的人都會同意用自動化？
若客戶 / 老闆 / 同事期待「一定要是人回」的場景，硬上自動化會被嫌。先溝通再建。
TOOL B
流程設計工作單（7 格）
把你腦中的想法「結構化」到這張表上。填完表，Make / n8n 的實作就是照著拖節點而已。建議用筆或 Docs 填寫（不要直接開 Make）——設計階段停在紙上，才能想清楚。
1
場景與目標
流程名稱
例：「客戶詢價自動分派」——要夠具體，能對同事說清楚
痛點
例：「客戶來詢價平均回覆需 8 小時，錯過很多案子」
成功指標
例：「回覆時間 < 30 分、分類正確率 > 90%」
2
Trigger（觸發）
觸發條件
例：「Gmail 收到主旨含『詢價』的信」
觸發工具
Gmail / Google Form / LINE Bot / Stripe / Webhook 等
預估頻率
每天幾次？每週幾次？——影響要選 Make（低頻）或 n8n（高頻）
3
輸入欄位
必要欄位
條列出來：寄件人、主旨、內文、時間、金額⋯⋯
資料品質
這些欄位會「乾淨」嗎？需要先清理嗎？（如亂打的金額「NT$ 十萬」要用 AI 先規範化）
4
AI 處理
AI 任務
分類？摘要？評分？萃取？（從 CH 2.1 的四大任務想）
輸出格式
JSON（強烈建議）：列出要回傳的欄位名與類型
用哪個 AI
Claude（中文寫作）/ OpenAI（便宜廣泛）/ Gemini（免費額度大）
5
分支條件
路徑 A
例：「若金額 > 10 萬 → 優先處理」
路徑 B
例：「若客戶有提及『急』→ 加急通知 LINE」
預設路徑
其他都進一般處理——一定要有，不然會掉資料
6
動作（Actions）
Action 1
例：「寫入 Google Sheet 追蹤」——所有流程都該先有這步
Action 2
例：「Slack 通知業務」
Action 3（選）
例：「自動回一封制式確認信」
7
異常處理
AI 失敗
若 AI 回傳格式錯或超時 → 通知你人工處理（絕不能吞掉）
動作失敗
若寄信 / 寫 Sheet 失敗 → Make 會 retry 3 次，之後寄警示 email
降級方案
若整條流程都失敗，最起碼保證「原始訊息有收到、你會知道」
TOOL C
完整實作 Demo：客戶詢價自動分派
拿 CH 5.3 Workflow 01 的「客戶詢價 → AI 分派」當實例，從 Make 上開新 scenario 到啟用，給你完整 step-by-step。走完這個 demo，你就有一條真能用的自動化。
Phase 1 · 準備（15 分鐘）
1
建 Google Form
欄位：姓名、Email、公司、主旨、訊息。回應寫入 Google Sheet（Form 選項自動建）。
2
準備 Slack / LINE 通知管道
到 Slack 建 channel 叫 #sales-inquiry，取得 webhook URL；或到 LINE Notify 申請 token。
3
Anthropic / OpenAI 申請 API key
console.anthropic.com 或 platform.openai.com 申請 API key，儲值 $5 USD 足夠測試。
Phase 2 · 在 Make 建 Scenario（30 分鐘）
4
新建 scenario，加 Trigger
「Google Sheets: Watch Rows」→ 選對應 Sheet → Limit 10 → 每 15 分檢查一次新資料。
5
加 AI 處理
加「Anthropic Claude: Messages」→ 填 API key → System prompt 貼下面那段 → User message 用 Name: {{name}}...Message: {{message}} 動態填入。
6
加 JSON parser
「Tools: Parse JSON」→ 把 Claude 回傳的文字解析成可用的 JSON 物件，之後才能用 {{category}} 這種寫法。
7
加 Router 分流
4 條路徑：詢價 / 客訴 / 求職 / 其他。每條 Filter 設 category = "詢價" 等條件。
8
每條路徑加動作
詢價：「Slack: Create Message」到 #sales-inquiry；客訴：「LINE Notify」給老闆；求職：寄 HR；其他：寄你自己 inbox。
9
加共用的記錄節點
在 AI 處理之後、Router 之前，加「Google Sheets: Add a Row」到一個叫 自動化 log 的 sheet，欄位含：時間、輸入、AI 判斷、分派結果。
AI Prompt（貼到 Claude 模組）
# System 你是客服分類助理，嚴格按 JSON 格式回覆，不要加任何說明文字。 # User 表單內容： 姓名：{{1.name}} Email：{{1.email}} 公司：{{1.company}} 主旨：{{1.subject}} 訊息：{{1.message}} 請分類並回覆這個 JSON： { "category": "詢價 / 客訴 / 求職 / 其他", "priority": "high / medium / low", "summary": "30 字內摘要", "suggested_reply_hours": 2 / 8 / 24 } 規則： - 「詢價」：提到產品、報價、購買 - 「客訴」：抱怨、退款、投訴 - 「求職」：應徵、履歷、職缺 - 其他：以上都不是 - priority high：大金額、客訴、VIP 公司名 - suggested_reply_hours：高優先 2，中 8，低 24
Phase 3 · 測試（30 分鐘）
10
手動填表送 10 筆測試資料
每種分類至少送 2 筆，還要故意送「邊緣 case」——太短的訊息、中英混雜、很模糊的需求。
11
檢查 AI 分類對了幾筆
分錯的要回去調 prompt：「『退款』這個詞該歸客訴不是詢價」這類明確規則加進 prompt。
12
確認通知都有到
Slack 訊息、LINE 訊息、log sheet 都要檢查。任何一個環節沒通知 = 自動化失敗。
Phase 4 · 啟用與監控
13
切 ON
Scenario 右下角 schedule 設 every 15 min → 左下角 Scheduling 切 ON。從此新 Form 送件會自動跑。
14
設定每週檢查 log
加個行事曆提醒，每週五 17:00 打開 log sheet 看：跑了幾次、分錯幾次、有沒有漏。月底依 log 調整 prompt。
TOOL D
上線前 Checklist
跟著 Tool C 做完 demo 後，正式上線前跑一次這個 checklist——確保你不會上線後才發現問題（例如「忘了加 log」「通知沒通到」）。
Log 記錄：所有跑過的流程都有寫入 Google Sheet，包含輸入、AI 判斷、最終動作
測試 10 筆不同情境，包含 3 個邊緣 case（超短、中英混、模糊需求），分類正確率 > 90%
通知都到位：Slack / LINE / Email 實際收到測試訊息，格式可讀
異常處理：Make 內設定 Error handler（Tools → Add Error Handler），AI 失敗會寄警示
預設路徑：Router 有「catch-all」路徑，「其他」case 不會掉資料
ops 預估：預計每日觸發幾次 × 每次幾個 ops = 月總 ops，確認在方案額度內
內部溝通：相關同事知道「這個自動化在跑」，並知道收到通知時該做什麼
關閉機制：你知道怎麼「緊急關掉」這條流程（Make 左下角切 OFF），避免失控時停不下來
週五檢查：行事曆已加「每週五 17:00 檢查自動化 log」重複事件
第一週陪跑：前 7 天每天早上看一次 log，確認運作正常、有問題立即修
恭喜完成 Part 5 ✓
你現在有一條真正在跑的自動化流程，這是上班族的分水嶺——你從「重複性工作的執行者」變成「工作流設計者」。此後遇到任何「每週都在做同件事」的情境，你第一反應都是「這可以自動化」，這個心態值 10 萬元以上。
