Part 5 · 自動化流程設計
CH 5.1 · 1.5 小時

Make.com 入門：
視覺化自動化

Make 是最適合上班族的自動化工具——拖拖模組、連連線、流程就會跑。本節從 0 到能跑第一條自動化流程：Gmail 收到信 → 自動歸檔到 Google Sheet。

學習成果 · OUTCOMES
理解 Scenario / Module / Trigger / Action / Filter 五個核心概念
能在 Make 完成第一條自動化流程（Gmail → Google Sheet）
掌握 Operations 額度概念，選對免費版 / 付費版
SECTION 01 · 什麼是 Make.com

把「當 A 發生 → 就做 B」的規則變成可視化流程

Make（前身叫 Integromat，2022 年改名）是一個「視覺化自動化工具」：你在畫布上拖一個「Gmail」模組，再拖一個「Google Sheet」模組，用線連起來——就完成了「收到信 → 存到 Sheet」的自動化。

它解決的是上班族最痛的「重複性手動工作」：每天早上把 Email 分類、每週把報表匯總成總表、每次活動報名後要手動寄確認信⋯⋯這些都是 Make 的典型場景。

🧭
為什麼先學 Make 而不是 Zapier？Make 免費版給的額度更大（1,000 operations/月 vs Zapier 100 tasks/月）、支援的 app 幾乎一樣多、視覺化流程更直觀、對多步驟的複雜流程處理更好。對上班族初學者是最 CP 值的選擇。
SECTION 02 · 5 個核心概念

先搞懂這 5 個字，介面就不會迷路

Scenario
場景
一條完整的自動化流程。每個「要自動化的事」都是一個 Scenario。
例：「Gmail 收到報價請求 → 自動存 Sheet → 通知業務」是一個 Scenario
Module
模組
流程中的一個步驟。Make 有上千個模組（Gmail、Sheet、Slack、OpenAI、Notion⋯⋯）。
例：「Gmail: Watch Emails」「Google Sheets: Add a Row」各是一個 Module
Trigger
觸發器
流程的「起點」——決定什麼條件會啟動整條 scenario。通常是 Scenario 的第一個 module。
例：「收到新 email」「Sheet 新增一行」「每天早上 9 點」都是 Trigger
Action
動作
Trigger 之後要做的事。可以串很多個 action（A → B → C → D），每個用線連起來。
例：「寄 email」「寫入 Sheet」「呼叫 Claude」都是 Action
Filter
過濾器
兩個模組中間的「條件」——只有符合條件的才會繼續往下走。
例：「email 主旨含『報價』才繼續」「金額 > 10 萬才通知主管」
Router
分流器（進階）
把流程分成多條路徑，每條路徑有不同條件。適合「if-then-else」邏輯。
例：「若是 VIP 客戶 → 路徑 A；若是一般 → 路徑 B」
💡
心法：每條 Scenario 都是「1 個 Trigger + N 個 Action + 0-N 個 Filter」的組合。先畫在紙上（或腦海中），再到 Make 上拖模組，準確度高 5 倍。
SECTION 03 · 第一條 Scenario

5 分鐘做完：Gmail → Google Sheet

場景：每次收到包含「詢價」字樣的 Gmail，自動把寄件人、主旨、日期、內容摘要存到 Google Sheet，讓你能回顧追蹤。

Gmail
Watch Emails
→
Filter
主旨含「詢價」
→
Google Sheets
Add a Row
1
註冊 Make 帳號
到 make.com 註冊（免費，可用 Google 帳號登入）。進入後按「+ Create a new scenario」。
2
加第一個 Module：Gmail 觸發器
畫布中點 + 圓圈 → 搜尋「Gmail」→ 選「Watch Emails」。授權你的 Gmail → 選「INBOX」資料夾 → Limit: 10 封 → 存儲。
3
加 Filter：只要含「詢價」的信
連線的中間會出現扳手圖示，點它 → Set up a filter → Condition: Subject contains 詢價。
4
加第二個 Module：Google Sheet
在 Filter 後按 + → 搜尋「Google Sheets」→ 選「Add a Row」。授權 Google 帳號 → 選你的試算表 → 選 sheet → 把 Gmail 的欄位（Sender, Subject, Date, Text Content）拖到對應欄位。
5
測試 + 啟用
按下畫布左下角「Run once」——Make 會跑一次流程，你會看到模擬結果。確認 OK 後右下角開「Scheduling」設定成每 15 分鐘跑一次 → 左下角開關切 ON。完成，之後會自動跑。
🎯
做完後回頭看：這就是你的第一條自動化流程。變化上千種：把 Gmail 換成 LINE、把 Sheet 換成 Notion、在中間加 Claude 做摘要⋯⋯全部都只是「換模組、拖連線」的事。
SECTION 04 · Operations 額度

Make 免費版 vs 付費版

Make 的收費單位是「Operation」——每個模組跑一次算 1 operation。Filter 不算。一條 3 個模組的 scenario 跑一次就是 3 operations。

Free
$0
  • 1,000 operations/月
  • 最多 2 個 scenarios 同時啟用
  • 最低 15 分鐘觸發一次
  • 適合：個人學習、1-2 個自動化
Core
$9/月（年繳）
  • 10,000 operations/月
  • 無限 scenarios
  • 最低 1 分鐘觸發
  • 適合：1-2 人工作室
Pro
$16/月（年繳）
  • 10,000 ops + 可購加購
  • 優先執行
  • Custom variables
  • 適合：小團隊、中型自動化
⚠️
額度陷阱：若 trigger 是「每 15 分鐘檢查 Gmail」、一天 96 次，即使沒有新信，每次仍吃 1 op（96×30 = 2,880 ops/月）。免費版要用「Webhook」觸發型（真正新事件才觸發），而不是 Polling 型，額度才夠用。
SECTION 05 · 本節行動

課後練習：跑一條 Scenario

照 Section 3 的 5 步驟，跑完「Gmail → Google Sheet」這條流程。目標不是「從此自動化詢價」，而是體驗整個設定節奏——你會發現難點不在技術，而在「想清楚流程」。

跑完後嘗試改：把「Gmail」換成「Slack channel 新訊息」、把「Google Sheet」換成「Notion」或「Airtable」。每換一個模組，都在練習「拼積木」的思維。
