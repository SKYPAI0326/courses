SECTION 01 · 什麼是 Make.com10 分鐘

把「當 A 發生 → 就做 B」變成會自己跑的流程

Make（前身叫 Integromat，2022 年改名）是一個「視覺化自動化工具」：你在畫布上拖一個「Gmail」模組，再拖一個「Google Sheet」模組，用線連起來——就完成了「收到信 → 存到 Sheet」的自動化。不用寫程式，不用懂 API，純粹拖拉連線。

它解決的是上班族最痛的重複性手動工作：每天早上把 Email 分類、每週把報表匯總成總表、每次活動報名後要手動寄確認信。這些全都是 Make 一拖一連就能做的事。

真實情境 · 行政助理 Amy 的每日早晨
Amy 每天 9 點到公司，第一件事是打開 Gmail 找「報價需求」的信，手動把寄件人、主旨、日期複製到 Google Sheet，再轉寄給業務部。每天 15 封，每封 3 分鐘，一天浪費 45 分鐘。用 Make 建一條 Scenario 後，這件事自動跑——Amy 到公司直接開始處理更重要的事。一天省 45 分，一個月省 15 小時。
真實情境 · 小型工作室老闆 Kevin
Kevin 的線上課程每次有新學員報名（Google Form），他要手動寄歡迎信、加 LINE 群、更新學員名冊。報名 20 人就花掉一整個下午。用 Make 接 Google Form 觸發器，自動寄信 + 更新 Sheet + 發 LINE Notify 通知自己——零手動，報名到歡迎信 2 分鐘內完成。
核心觀念 · 為什麼先學 Make 而不是 Zapier？
三個原因：① 免費額度更大——Make 每月 1,000 operations，Zapier 只有 100 tasks。② 視覺化更直觀——Make 的畫布式介面讓你「看得到流程」，Zapier 是清單式，流程一複雜就看不懂。③ 多步驟更便宜——Zapier 每一步算一次 task，Make 則是每個模組算一次 operation，Filter 還不算。同樣的流程 Zapier 可能是 Make 的 2 倍價格。
SECTION 02 · 5 個核心概念20 分鐘

搞懂這 5 個字，Make 介面就不會迷路

所有 Make 的操作都建立在這 5 個概念上。你會發現它們就像「積木」——每個概念都有明確角色，組合起來就是一條自動化流程。

Scenario
場景 = 一條完整流程
一個 Scenario 就是一條完整的自動化流程。「客戶下單 → 寄確認信 → 更新庫存」是一個 Scenario。每個「要自動化的事」開一個 Scenario。
真實範例：Kevin 的「新學員報名 → 寄歡迎信 → 更新名冊 → 通知自己」就是一個 Scenario，裡面有 4 個模組串起來
Module
模組 = 流程中的一個步驟
每個模組代表一個動作：「讀 Gmail」是一個模組，「寫入 Sheet」是一個模組。Make 有 1,800+ 模組涵蓋主流 app。
真實範例：「Gmail: Watch Emails」「Google Sheets: Add a Row」「Slack: Send a Message」各是一個 Module
Trigger
觸發器 = 流程起點
Scenario 的第一個模組就是 Trigger——決定「什麼事件會啟動流程」。有兩種：Polling（定時檢查）和 Instant（事件即時觸發）。
真實範例：「收到新 email」「Google Form 新提交」「每天早上 9 點」「收到 Webhook」都是 Trigger
Action
動作 = 觸發後要做的事
Trigger 之後串的每個模組都是 Action。可以串很多個：A → B → C → D，每個之間用線連起來。
真實範例：「寫入 Sheet」「寄 Email」「發 Slack 訊息」「呼叫 Claude API」「建立 Notion 頁面」都是 Action
Filter
過濾器 = 條件判斷
Filter 放在兩個模組中間，設定「只有符合條件的才繼續」。Filter 不消耗 operation 額度，是免費版的省額度利器。
真實範例：「email 主旨含 '報價' 才繼續」「金額 > 10 萬才通知主管」「表單選 '企業版' 才走 VIP 流程」
Router
分流器 = 多路徑分支
把流程分成多條路徑，每條設不同條件。就是 if-then-else。後面 CH5-3 的 AI 分類工作流會大量用到。
真實範例：「若是 VIP 客戶 → 路徑 A 專人回覆；若是一般 → 路徑 B 自動回覆；其他 → 路徑 C 歸檔」
核心觀念 · 公式記起來
每條 Scenario 都是「1 個 Trigger + N 個 Action + 0~N 個 Filter/Router」的組合。先在紙上畫出流程（用→連接方框），再到 Make 上拖模組——先想清楚再動手，效率高 5 倍。拖出來才改的人，花 80% 時間在除錯。
真實情境 · 業務部的智慧信件分流
業務部收到的 email 什麼都有：詢價、客訴、合作邀約、垃圾信。用 Make 做一條 Scenario：Trigger（Gmail 收到新信）→ Router（依主旨關鍵字分流）→ 路徑 A：含「報價」→ 存 Sheet + 通知業務；路徑 B：含「客訴」→ 存另一張 Sheet + 通知客服主管；路徑 C：其他 → 歸檔不處理。一條 Scenario 完成 3 個人的工作。
SECTION 03 · 第一條 Scenario20 分鐘

5 分鐘做完：Gmail → Google Sheet

場景：每次收到包含「詢價」字樣的 Gmail，自動把寄件人、主旨、日期、內容摘要存到 Google Sheet。下面每一步都拆到最細——你跟著做，5 分鐘就能跑通。

(Trigger)
→
Filter
主旨含「詢價」
(不耗 ops)
→
Google Sheets
Add a Row
(Action)
1
註冊 Make 帳號
到 make.com → 點右上「Get started free」→ 用 Google 帳號登入最快。進入 Dashboard 後左側選「Scenarios」→ 右上角「+ Create a new scenario」。你會看到一個空白畫布，中間一個大 + 號。
2
加第一個 Module：Gmail 觸發器
點畫布中央的 + → 搜尋「Gmail」→ 選「Watch Emails」（注意不是 Search Emails）。第一次會跳授權：允許 Make 讀取你的 Gmail。授權後設定：Folder: INBOX、Limit: 10（每次最多拉 10 封）。按「OK」。
3
加 Filter：只要含「詢價」的信
Gmail 模組右邊會有一條虛線，點虛線上的「扳手」圖示 → 「Set up a filter」→ Label 填「只要詢價信」→ Condition：Subject → Contains (case insensitive) → 值填 詢價。這樣只有主旨含「詢價」的信才會往下走。
4
加第二個 Module：Google Sheets
在 Filter 右邊按 + → 搜尋「Google Sheets」→ 選「Add a Row」。授權 Google → 選你事先建好的試算表 → 選工作表（Sheet1）→ 欄位對應：A 欄 = 拖入 Sender Email Address、B 欄 = Subject、C 欄 = Date、D 欄 = Text content。按 OK。
5
測試 + 啟用排程
左下角按 Run once → Make 會讀最近的 Gmail 跑一次流程。若 Sheet 出現新行就成功了。確認 OK 後：右下角齒輪 → Scheduling → 設「Every 15 minutes」 → 左下角開關切 ON。完成，之後自動跑。
🎯
變化無限：把 Gmail 換成 LINE（用 Webhook）、把 Sheet 換成 Notion、在中間加 Claude 做 AI 摘要——全部都只是「換模組、拖連線」。你剛學的這 5 步驟結構，適用於所有 Make scenario。
SECTION 04 · 額度與方案10 分鐘

Operations 額度：免費版 vs 付費版

Make 的收費單位是「Operation」——每個模組跑一次算 1 operation。Filter 和 Router 不算。一條 3 模組的 Scenario 跑一次就是 3 operations。先把這個算清楚，才知道免費版夠不夠用。

Free
$0/月
  • 1,000 operations/月
  • 最多 2 個 scenarios 同時啟用
  • 最低 15 分鐘觸發一次
  • 無 Webhook（限 Polling）
  • 適合：個人學習、1-2 個簡單流程
Core
$9 USD/月（年繳）
  • 10,000 operations/月
  • 無限 scenarios
  • 最低 1 分鐘觸發
  • 支援 Webhook（即時觸發）
  • 適合：1-2 人工作室、小型團隊
Pro
$16 USD/月（年繳）
  • 10,000 ops + 可加購
  • 優先執行佇列
  • Custom variables
  • Full-text log search
  • 適合：小團隊、中型自動化
真實算帳 · 免費版到底夠不夠？
Amy 的「Gmail → Filter → Sheet」scenario 有 2 個模組（Filter 不算）。每 15 分鐘跑一次 = 一天 96 次 = 即使沒新信也消耗 96 ops（因為 Gmail Watch 模組被啟動了）。96 × 30 天 = 2,880 ops/月——遠超免費版 1,000。

破解法：改成「每小時跑一次」（24 × 30 = 720 ops）或用 Google Apps Script 發 Webhook 觸發（只在真正有新信時才跑）。
核心觀念 · Polling vs Instant Trigger
Polling（輪詢）：Make 每隔 N 分鐘去「問」一次有沒有新事件，不管有沒有都吃 1 op。免費版最低 15 分鐘。Instant（即時）：外部主動「告訴」Make 有新事件（Webhook），只在事件發生時才觸發，不觸發就不吃 op。省額度法則：能用 Instant 就不用 Polling。但 Instant Webhook 要 Core 以上方案。

SECTION 06 · 踩坑博物館5 分鐘

Make 新手最常踩的 6 個坑

坑 1Polling 把額度吃光
症狀：月中就收到「Operations limit reached」通知，但你覺得根本沒幾筆資料。
修：把觸發頻率從 15 分鐘改成 1 小時，或改用 Webhook（Instant Trigger）。算式：每日觸發次數 × 模組數 × 30 天 = 月用量。
坑 2授權過期沒發現
症狀：Scenario 突然不跑了，看 log 是「401 Unauthorized」。
修：Google / Slack 的 OAuth token 會過期。到 Connections 頁面 → 點 Reauthorize。建議每月 1 號固定檢查一次。
坑 3Watch Emails 重複觸發
症狀：同一封信被處理 2-3 次，Sheet 出現重複行。
修：Make 用 bookmark 記住「上次讀到哪」。如果你手動改了 Gmail 標籤或移動信件，bookmark 可能失準。勿在 Make 和手動操作之間互搶同一信箱。
坑 4Run once 會跑但排程不跑
症狀：手動 Run once 成功，但開了 Scheduling 後沒有自動跑。
修：檢查三件事：① 左下角開關是否切 ON ② Scheduling 是否有設時間 ③ Trigger 的 "Choose where to start" 有沒有選對（From now on / All）。
坑 5欄位映射拖錯
症狀：Sheet 的 A 欄出現 email 主旨、B 欄出現寄件人——對調了。
修：先在 Sheet 建好表頭（A:寄件人、B:主旨...），Make 會自動抓表頭名稱。拖欄位時看清楚左邊的表頭名再拖。
坑 6免費版只能開 2 個 Scenario
症狀：想建第 3 個 Scenario 發現要付費。
修：把不常用的 Scenario 切 OFF（不算在 2 個限制內）。或把多個小流程合併成一個 Scenario 用 Router 分流。

PRACTICE 01 · 跑通第一條 Scenario
Gmail → Filter → Google Sheet
題目：照 Section 03 的 5 步驟，在 Make 建出「Gmail → Filter（主旨含特定關鍵字） → Google Sheet」的 Scenario，並執行 Run once 成功看到 Sheet 新增一行。

驗收標準
① Scenario 有 3 個元件（Gmail trigger → Filter → Sheets action）
② Run once 執行成功（綠色勾勾）
③ Google Sheet 出現新的一行，欄位正確對應（寄件人、主旨、日期、內容）
PRACTICE 02 · 算你的月額度
Practice 01 的 Scenario 夠不夠用？
題目：用下面公式計算 Practice 01 的 Scenario 若開 Scheduling，每月會消耗多少 operations：
觸發頻率（次/天） × 模組數 × 30 天 = 月消耗
算完回答：免費版 1,000 ops 夠嗎？如果不夠，要改什麼？
提示：每 15 分鐘觸發 = 一天 96 次。Gmail Watch 算 1 op，Sheets Add Row 算 1 op，Filter 不算。
參考答案
96 次/天 × 2 模組 × 30 天 = 5,760 ops/月 → 遠超免費版 1,000

解法：① 改成每小時觸發（24 × 2 × 30 = 1,440，還是超）② 改成每 2 小時觸發（12 × 2 × 30 = 720，OK）③ 改用 Webhook 觸發（只有新信才跑，假設一天 5 封 = 5 × 2 × 30 = 300 ops，超省）
PRACTICE 03 · 加上 Router
挑戰：把 Practice 01 升級成分流版
題目：把 Practice 01 的 Filter 改成 Router，建 2 條路徑：路徑 A「主旨含報價」→ 存到 Sheet1；路徑 B「主旨含客訴」→ 存到 Sheet2。測試跑通兩條路徑。
提示：Router 在 Make 介面的位置：點模組之間的連線 → 會看到「Add a router」選項。每條路徑可以獨立設 Filter。
