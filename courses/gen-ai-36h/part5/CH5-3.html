Part 5 · 自動化流程設計
CH 5.3 · 1.5 小時

AI × 自動化：
3 種典型工作流

Make / n8n 工具學會了，接下來是「實際要做什麼？」上班族最有價值的 3 種自動化場景：表單送件自動處理、會議錄音自動歸檔、客訴自動分派。每個都給完整 workflow 設計圖與 AI prompt。

學習成果 · OUTCOMES
掌握「表單 / 錄音 / Email」三種最常見的自動化觸發來源
學會把 AI 呼叫（Claude / OpenAI）嵌進 workflow 的正確姿勢
能設計「有分支條件」的工作流：依 AI 判斷分派到不同後續動作
SECTION 01 · 設計思維

自動化流程的通用架構

所有 AI 自動化流程幾乎都長同個樣子：

Trigger
事件發生
→
資料準備
擷取欄位、整理
→
AI 處理
分類 / 摘要 / 評估
→
條件分支
if-then-else
→
行動
歸檔 / 通知 / 回信
🧭
心法：AI 在自動化流程裡的角色是「會判斷的過濾器」——它不做決策，只給出結構化的標籤或摘要，讓後續動作能依據這個標籤走不同路徑。AI 回傳的格式一定要讓後續節點能處理（通常是 JSON）。
WORKFLOW 01
表單送件 → AI 分類 → 分派通知
場景：公司官網的「聯絡我們」表單每天收到各種訊息（詢價、客訴、求職、合作）。以前要人工分類後轉給對應部門；現在讓 AI 自動判斷並直接通知對應同事。
Google Form
新送出
→
AI
分類 + 摘要
→
Router
依類別分支
→
Slack
通知對應人
需要的模組
Trigger
Google Form: Watch Responses — 或用 Apps Script 送 webhook 到 n8n
AI 處理
Anthropic Claude 或 OpenAI ChatGPT 模組，丟表單內容進去
分支
Make: Router / n8n: Switch
通知
Slack / LINE Notify / Gmail（擇一）
歸檔
Google Sheets: Append Row 把所有分類結果記下來
AI 分類 prompt
你是客服分類助理。以下是客戶透過官網表單送來的訊息，請分類並摘要。 表單內容： 姓名：{{ name }} Email：{{ email }} 主旨：{{ subject }} 訊息：{{ message }} 請嚴格按以下 JSON 格式回覆，不要加任何其他文字： { "category": "詢價 / 客訴 / 求職 / 合作 / 其他", "priority": "high / medium / low", "summary": "20 字內摘要", "suggested_assignee": "業務 / 客服 / HR / 老闆 / 任何人", "needs_urgent_reply": true 或 false, "has_amount_mentioned": true 或 false } 分類規則： - 「詢價」：有提到產品、方案、報價、購買 - 「客訴」：有抱怨、退款、投訴、品質、服務問題 - 「求職」：有提到應徵、履歷、職缺 - 「合作」：有提到合作、夥伴、轉介 - 「其他」：無法歸類 priority 判斷： - high：含客訴、大金額詢價、知名公司 - medium：一般詢價、合作邀請 - low：求職、其他
Router 分支設定
分支 A
條件：category == "詢價" → 通知業務 Slack channel
分支 B
條件：category == "客訴" AND priority == "high" → 同時通知客服 + 老闆 LINE
分支 C
條件：category == "求職" → 轉寄到 HR email
分支 D（預設）
其他 → 寄到老闆的 inbox 人工判斷
⚡
成效：實測這個 workflow 上線後，客訴處理時間從「平均 4 小時」降到「20 分鐘」——因為第一時間就進到對的人手上。老闆也不用再當「人工轉信機」。
WORKFLOW 02
會議錄音 → AI 摘要 → 自動歸檔寄發
場景：團隊有內部會議錄音習慣，但從來沒人有空寫紀錄。讓自動化接手：會議錄音檔丟進 Google Drive 指定資料夾，AI 自動摘要 + 萃取待辦 + 寄給所有與會者。

→

需要的模組
Trigger

轉文字
OpenAI Whisper（付費 API，便宜）或 AssemblyAI（有免費額度）
AI 摘要
Claude 或 ChatGPT
寄信

歸檔
Google Docs: Create Doc 把紀錄存到「會議紀錄」資料夾
AI 摘要 prompt
你是專業會議書記。以下是會議逐字稿，請整理成結構化會議紀錄。 會議檔名（含元資訊）：{{ filename }} 逐字稿： """ {{ transcript }} """ 請嚴格按以下 JSON 格式回覆（其他文字一律不要）： { "meeting_title": "從檔名或內容推斷會議主題", "date": "YYYY-MM-DD", "participants": ["從內容識別的參與者姓名陣列"], "summary": "200 字內總結會議", "key_decisions": ["決議 1", "決議 2", "..."], "action_items": [ {"task": "具體行動", "owner": "負責人", "deadline": "日期或 待確認"}, {"task": "...", "owner": "...", "deadline": "..."} ], "follow_ups": ["待追蹤的議題"], "email_subject_suggestion": "建議的 email 主旨" } 規則： - action_items 只列「明確有人要做的」，不列討論性事項 - 負責人若沒講清楚標 "待確認" - 不要編造沒在逐字稿裡的內容
💡
升級版：把 Email 寄出內容用 Claude 再潤稿一次（去掉 AI 味），並加一段「若內容有誤請在 24 小時內回覆」。整個流程從「上傳錄音 → 大家收到紀錄」只要 5-10 分鐘。
WORKFLOW 03
客訴 Email → AI 嚴重度判斷 → 升級通知
場景：客訴信件塞滿客服信箱，其中只有 5% 是「真的需要老闆介入」的嚴重案件。讓 AI 幫你先篩一輪，高嚴重度才升級通知，避免雜訊淹沒真正重要的事。
Gmail
含「客訴」標籤
→
AI
評估嚴重度
→
Sheets
記錄所有
→
IF 高
升級 LINE
需要的模組
Trigger
Gmail: Watch Emails，Filter: label = 客訴（客服先貼標）
AI 評估
Claude（中文客訴情緒判讀最穩）
記錄
Google Sheets — 每筆客訴都記下來，可做月報
條件
Filter: severity == "critical"
通知
LINE Notify / Slack 即時通知老闆
AI 嚴重度判斷 prompt
你是客服主管，判斷一封客訴信的嚴重度。 信件內容： 寄件人：{{ sender }} 主旨：{{ subject }} 內文：{{ body }} 請嚴格按以下 JSON 格式回覆： { "severity": "critical / high / medium / low", "sentiment": "憤怒 / 失望 / 擔憂 / 中性", "keywords": ["關鍵字陣列（如：退款、賠償、訴訟、爆料）"], "issue_type": "商品瑕疵 / 物流 / 客服態度 / 退款糾紛 / 其他", "needs_escalation": true 或 false, "suggested_action": "一句話建議下一步", "reply_urgency_hours": 4 / 8 / 24 / 72 } 嚴重度定義： - critical：提到「訴訟」「消保官」「媒體」「爆料」「律師」、或極度憤怒語氣、或威脅退貨/負評 - high：多次抱怨（「第 X 次反應」）、金額大、提到補償要求 - medium：明確問題但語氣平穩、可協商 - low：小抱怨、建議性、未提到補償 needs_escalation 為 true 的情況：severity == critical，或 sentiment == 憤怒 且金額大
⚠️
重要：AI 不是決策者。這個 workflow 的 AI 給的是「建議嚴重度」，不是最終決策。所有 critical 都「通知老闆」而不是「自動發退款」。自動化只做「篩選與通知」，決策永遠在人手上。
SECTION 02 · 通用設計原則

做自動化流程的5 個原則

① 先人工跑一次
先手動把整個流程走一遍，每一步你怎麼判斷寫下來，這就是 AI 的 prompt。自己沒想清楚的流程，AI 也處理不好。
② JSON 是橋樑
AI 回傳一定要用結構化 JSON（不要純文字），後續節點才能穩定讀取欄位。prompt 尾端一定要寫「嚴格按以下 JSON 格式」。
③ 設一個 catch-all
Router 一定要有「以上都不是」的預設路徑，通常是「寄給我人工判斷」。不然遇到邊緣 case 會直接掉資料。
④ 記錄所有結果到 Sheet
在任何 Router 之前都先寫一行到 Google Sheet（含所有欄位 + AI 判斷）。這是你的審計記錄，也是未來優化 prompt 的依據。
⑤ 測試 10 個邊緣 case
流程上線前，拿過去實際的 10 個案例手動輸入，看 AI 都分對嗎？分錯的就補 prompt 規則，直到都對才啟用。
SECTION 03 · 本節行動

課後練習：挑一個你真的缺的流程

看完這 3 個範例，想想你工作中哪件事是「每天都在做、其實有規則可循」的——那就是你該自動化的事。先用紙筆畫出 flow，再到 Make / n8n 實作。

常見練習場景：① 行事曆新增會議後自動提醒準備 ② 客戶 email 收到後 AI 判斷語氣，回覆前給我一個「他可能不高興」警示 ③ 月底自動從 Sheet 拉資料、生成月報 PDF、寄給主管。
