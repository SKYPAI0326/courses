(02) · 場景對照15 分鐘

7 個常見場景，該用誰？

1. 官網表單 → 自動通知團隊
Make
理由：需要 24/7 收表單。資料不敏感。流程簡單（3-4 個模組）。Make 免費版就能搞定。
2. 本機 CSV 批次 AI 分析
n8n
理由：要讀本機檔案（Make 做不到）。2,000 筆資料每筆呼叫 AI = 2,000+ ops（Make 額度爆炸）。n8n 零限制。
3. Shopify 下單 → 出貨通知
Make
理由：Shopify 模組 Make 原生支援、設定最快。客戶下單任何時間都可能發生，要 24/7。一般電商一天幾十單，免費版不夠，Core（$9/月）夠用。
4. 客戶資料報表（含個資）
n8n
理由：含個人資料（姓名、電話、地址）。不想經 Make 的美國伺服器。n8n 跑在自己電腦/NAS，資料不出內網。
5. LINE Bot 自動客服
串接
理由：Make 接 LINE Webhook（24/7、簡單轉發）。收到訊息後 Webhook 打到 n8n，n8n 跑本機 AI 處理後回傳。Make 當門口、n8n 當廚房。
6. 每日爬蟲 + 資料清洗
n8n
理由：爬蟲需要 HTTP Request + 複雜 HTML 解析（Code 節點）。每日跑一次，不需要 24/7。資料量大但只在上班時間需要。
7. Google Calendar → Slack 提醒
Make
理由：Google Calendar + Slack 兩個模組 Make 都原生支援。一天觸發 3-5 次，ops 微乎其微。免費版完全夠。

真實情境 · 行銷公司的雙引擎架構
行銷公司每天收客戶表單（Make 接，24/7）→ Webhook 打到 n8n → n8n 用本機 AI 跑客戶分析 + 讀歷史資料庫 → 分析完 Webhook 回 Make → Make 發 Slack 通知 + 建 Notion 工單。Make 負責「接客」和「通知」，n8n 負責「動腦」和「看資料」。月成本：Make Core $9 + n8n 跑在 NAS 免費 = $9 USD/月。

核心觀念 · 付費策略
能免費的階段就免費——Make 免費 + n8n 本機可以撐很久。第一筆花費：Make Core $9/月——解鎖 10K ops、Webhook、1 分鐘觸發。n8n 需要 24/7 時再花 $5/月租 VPS。不要一開始就買 Make Pro——大多數人用不到 Custom Variables 和 Full-text search。
(05) · 踩坑博物館5 分鐘

雙引擎協作常見的 5 個坑

坑 1Webhook 超時
症狀：Make 打到 n8n 的 Webhook，n8n 要跑 60 秒 AI 分析，Make 等不到回覆就 timeout。
修：改成「非同步」模式：n8n 收到 Webhook 先回 200（「收到了」），AI 分析完再主動打回 Make 的另一個 Webhook。
坑 2n8n 本機關了 Make 打不到
症狀：週末筆電關機，Make 的 Webhook 打到 n8n 失敗，資料遺失。
修：Make 端加 Error Handler → 失敗的資料存到 Google Sheet「待處理」清單。週一開機後 n8n 去讀那張 Sheet 補跑。
坑 3兩邊都存資料，不知道以誰為準
症狀：Make 存了一份 Sheet、n8n 也存了一份 Sheet，資料不同步。
修：約定「Single source of truth」——只有一邊負責寫入最終資料。另一邊只讀取。通常讓 n8n（做完處理的那端）寫入最終版。
坑 4忘記 Webhook URL 有兩種
症狀：n8n 的 Webhook 有 Test URL 和 Production URL，用錯的那個打不通。
修：Test URL 只在 n8n 編輯介面開著時有效。Production URL 要切 Active 才能用。Make 端要填 Production URL。
坑 5兩邊都扣 AI API 費用
症狀：Make 用 Claude 模組 + n8n 也用 Claude 模組，月底 API 帳單是預期的 2 倍。
修：AI 處理只在一邊做（通常是 n8n，因為無 ops 限制）。Make 只負責觸發和通知。一筆資料只經過 AI 一次。
(06) · 課堂動手10 分鐘

Guided Practice：做出決策

PRACTICE 01 · 場景分配
把你的 5 個自動化需求分到 Make 或 n8n
題目：回想你在 Part 5 學到的所有 Scenario / Workflow，加上你自己工作中想做的自動化，列出 5 個需求。用 Section 01 的 6 維度表，分別標記每個需求該放 Make / n8n / 串接。
提示：如果 6 維度有 4 個以上指向同一邊，就選那邊。3:3 平手的，看「24/7 需求」——需要就 Make，不需要就 n8n。
範例對照
① 官網表單通知 → Make（24/7、輕量、公開）
② 月報 AI 分析 → n8n（本機 Excel、大量、一個月跑一次）
③ LINE Bot 客服 → 串接（Make 接 LINE、n8n 跑 AI）
④ Google Calendar 提醒 → Make（最簡單、免費版夠）
⑤ 客戶個資報表 → n8n（敏感資料不出內網）
PRACTICE 02 · 算你的月費
用成本表估算你的真實月費
題目：根據 Practice 01 的 5 個需求，估算每個在 Make 的月 ops 用量（觸發頻率 × 模組數 × 30 天），加總後看需要哪個方案。同時估算 n8n 端需不需要 VPS。算出你的「真實月費」。
提示：常見結果是「Make Core $9 + n8n 本機 $0 = $9 USD/月」。只有需要 24/7 n8n 的人才要加 VPS 費用。
