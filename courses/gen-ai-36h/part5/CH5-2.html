Part 5 · 自動化流程設計
CH 5.2 · 2 小時

n8n Desktop + Cloudflare Tunnel

Make 免費額度有限、付費每月 $9 起。n8n 是開源替代方案——跑在你自己電腦，完全免費、無限制。本節教你安裝 n8n、跑第一個 workflow、用 Cloudflare Tunnel 讓它能接外部 webhook。

學習成果 · OUTCOMES
理解 n8n 與 Make 的定位差異，知道什麼時候該用哪個
能在本機安裝 n8n（Desktop / Docker / npm 三選一）並跑第一個 workflow
會用 Cloudflare Tunnel 把本機 n8n 暴露給外部（接 LINE Bot、Webhook、Stripe 付款通知）
SECTION 01 · 為什麼學 n8n

跟 Make 比，多了什麼、少了什麼

Make.com
雲端 SaaS
  • ✓ 零安裝、瀏覽器打開就能用
  • ✓ 整合 app 數量最多（1,800+）
  • ✓ UI 最直觀、學習成本低
  • ✗ 有額度限制（免費 1,000 ops/月）
  • ✗ 付費從 $9 USD/月起跳
  • ✗ 資料會經過 Make 伺服器（隱私）
n8n
開源自架
  • ✓ 完全免費、無限制
  • ✓ 資料在自己電腦，不經第三方
  • ✓ 可讀取本機檔案（Make 做不到）
  • ✓ 可以用 JavaScript / Python 寫邏輯
  • ✗ 要自己安裝、維護
  • ✗ 電腦關機 = workflow 停擺
🧭
實務建議：不要「二選一」。很多情況是「Make 跑輕量雲端流程，n8n 跑本機 / 高頻 / 隱私需求」並用。這節學會後，下一節 CH 5.4 會深入「雙引擎協作」——什麼情境用哪個、怎麼互相串接。
SECTION 02 · 三種安裝方式

n8n 裝在你電腦上的 3 種方法

n8n Desktop
本課推薦
Mac / Windows 官方 app，雙擊安裝、開啟即用。適合 80% 個人使用者。
選：你是個人使用、想立刻能開始
Docker
穩定
容器化部署，適合想放在公司伺服器或 NAS。要懂一點 Docker 指令。
選：公司 IT 支援、24/7 運作
npm
開發者
npx n8n 一指令跑起，適合偶爾跑一次、或開發環境測試。
選：已有 Node.js 環境、臨時試用

n8n Desktop 安裝 4 步

1
下載 n8n Desktop
到 n8n.io/download，依你的系統（Mac / Windows / Linux）下載安裝檔。
2
安裝並啟動
Mac 拖到 Applications → 第一次開啟會要輸入 email 建本機帳號（資料在本機，email 只是登入用）。
3
開啟 n8n 介面
啟動後會打開 http://localhost:5678，這就是 n8n 的工作區。介面跟 Make 類似但更密一點（節點較多）。
4
建第一個 workflow
左上角「New workflow」→ 開始拖節點。n8n 的節點叫「nodes」，功能跟 Make 的 modules 一樣。
SECTION 03 · 第一個 workflow

在 n8n 做同一個 Gmail → Sheet

為了對照 CH 5.1，做完全一樣的流程看介面差異：

1
加 Gmail Trigger
左側搜尋「Gmail」→ 拖「Gmail Trigger」節點到畫布。點它 → Add Credential → 照精靈走（需要 Google Cloud Console 建 OAuth app）。這步是 n8n 門檻最高的部分，第一次要花 30 分鐘，之後一勞永逸。
2
加 IF 節點（相當於 Make 的 Filter）
拖「IF」節點 → Condition: {{ $json.subject }} contains 詢價。n8n 用雙花括號 {{ }} 取值，這個語法你會反覆用到。
3
加 Google Sheets 節點
接在 IF 的「true」輸出後 → Google Sheets → Action: Append Row → 選試算表與 sheet → 欄位用 {{ $json.sender }} 等拖入。
4
Execute workflow 測試
右上角「Execute workflow」→ 看到每個節點出現綠色勾勾就是通過。打開 Sheet 確認有新增一行。
5
切換為 Active
右上角開關從 Inactive → Active。從此 n8n 會每 1 分鐘檢查 Gmail（比 Make 免費版的 15 分鐘快 15 倍）。但前提是你的電腦要開著。
⚠️
電腦關機的影響：n8n Desktop 依賴你的電腦。若你會下班後關機，n8n 也會停。解決方式：① 放在永遠開機的桌機 ② 裝在 Mac mini / NAS ③ 改用 Docker 部署到公司伺服器 ④ 付費用 n8n Cloud（每月 $20 起）。
SECTION 04 · Cloudflare Tunnel

讓本機 n8n 接外部 Webhook

n8n 跑在 localhost:5678，只有你自己電腦能連。但如果想用 n8n 當「Line Bot 後端」「Stripe 付款通知接收器」「Google Form 送出觸發」這些外部服務要能 call 進來，就需要把本機 n8n 暴露到公網。

最簡單的方法是 Cloudflare Tunnel——免費、安全、不用設定路由器 port forwarding。

1
註冊 Cloudflare 帳號
到 cloudflare.com 註冊（免費）。不需要有自己的網域（2.0 版 Quick Tunnels 可以用他們提供的隨機網址）。
2
安裝 cloudflared
Mac: brew install cloudflared
Windows: 到官方下載 .exe。安裝完在終端機確認：cloudflared --version
3
啟動 Quick Tunnel
終端機輸入：cloudflared tunnel --url http://localhost:5678
會印出一個類似 https://random-words-1234.trycloudflare.com 的網址，這就是你 n8n 的公開入口。
4
在 n8n 用 Webhook 節點
workflow 第一個節點選「Webhook」→ Webhook URL 會顯示 http://localhost:5678/webhook/xxx → 你要給外部服務的網址是把 localhost:5678 換成 Cloudflare 給的網址。
5
測試
在另一個終端機執行：curl -X POST https://random-words-1234.trycloudflare.com/webhook/xxx -d '{"test":"hello"}' — 如果 n8n 收到訊息並觸發 workflow，就成功。
⚡
實用情境：① LINE Bot — 在 LINE Developers 填 webhook URL 指向 Cloudflare 網址；② Stripe 付款通知 — 訂閱 webhook event；③ Google Form — 用 Apps Script 送出時 POST 到 webhook；④ Notion 資料庫變動 — 透過 Notion automation webhook。
⚠️
Quick Tunnel 的限制：網址每次重啟都會變。要固定網址需綁自己的網域 + Named Tunnel（稍複雜但一勞永逸）。個人測試用 Quick Tunnel 即可。
SECTION 05 · JSON 與節點邏輯

比 Make 更進階：用 JS 處理資料

n8n 的資料在節點間是以 JSON 格式流動的。看到 {{ $json.subject }} 就是「取上個節點輸出的 subject 欄位」。Make 把這層包裝得很漂亮，n8n 直接讓你看 JSON——一開始嚇人，熟了才發現超好用。

# n8n 節點間傳遞的 JSON 長這樣 { "sender": "客戶 A <a@example.com>", "subject": "關於產品 X 的詢價", "date": "2026-04-15T10:30:00Z", "body": "您好，想了解 X 產品..." } # 要取欄位的寫法 {{ $json.sender }} // 取寄件人 {{ $json.subject }} // 取主旨 {{ $json.body.substring(0, 100) }} // 取內文前 100 字

要做「Make 做不到」的複雜邏輯時，加一個「Code」節點（或 Function 節點），直接寫 JS：

# Code 節點範例：從 email 主旨萃取金額 // n8n 的 Code 節點裡寫 JavaScript const subject = $input.first().json.subject; // 用 regex 找「$數字」或「NT$數字」 const match = subject.match(/\$?([\d,]+)/); const amount = match ? parseInt(match[1].replace(/,/g, '')) : 0; // 依金額分類 let priority = 'low'; if (amount > 100000) priority = 'high'; else if (amount > 20000) priority = 'medium'; return [{ json: { ...$input.first().json, extracted_amount: amount, priority: priority } }];
💡
不會 JS 怎麼辦？讓 Claude 幫你寫。貼給 Claude：「我在 n8n 的 Code 節點，輸入是 JSON（含 subject 欄位），我要從 subject 中取出金額並分類優先級為 high/medium/low，請給我 JS」——它會直接給可貼上的程式碼。你不用學 JS，只要學「怎麼跟 AI 描述你的需求」。
SECTION 06 · 本節行動

課後練習：n8n 從 0 到 webhook

這節技術細節多，務必動手跑一次：

  1. 安裝 n8n Desktop（約 10 分鐘）
  2. 做一個 Webhook → Slack / Telegram / Email 通知的 workflow（不必做 Gmail，太複雜的 OAuth）
  3. 跑 Cloudflare Tunnel 取得公開網址
  4. 用手機瀏覽器 / Postman call 一下 webhook，看是否觸發通知

這個練習完成 = 你有一個可以接任何外部事件的「自動化大腦」，後面幾節會把它用起來。
