(01) · n8n vs Make15 分鐘

跟 Make 比，多了什麼、少了什麼

n8n 不是 Make 的「免費版替代品」——它是另一種思路的工具。Make 像 SaaS 套裝軟體（穩、簡單、有限制）；n8n 像自組桌機（自由、強大、要動手）。兩者各有適用場景。

真實情境 · 自由工作者 Lisa 的雙引擎
Lisa 是平面設計師。她用 Make 接 Google Form 報名（需要 24/7 收、月用量 300 ops 免費夠用）。但客戶資料要跑 AI 分析報告時，她用 n8n——因為一次跑 200 筆資料在 Make 要 800+ ops（快超額），在 n8n 零限制，還能讀本機 Excel 檔。兩個工具各司其職，不用二選一。
核心觀念 · 什麼時候選 n8n
三個明確訊號告訴你該用 n8n：① 需要處理本機檔案（Make 做不到）。② 高頻 / 大量處理（Make 額度不夠）。③ 隱私敏感（客戶名單、財務資料不想經第三方伺服器）。反過來說，要 24/7 跑、不想維護、流程簡單的——用 Make。
(02) · 3 種安裝方式20 分鐘

把 n8n 裝在你電腦上

n8n Desktop
本課推薦
Mac / Windows 官方 app，雙擊安裝、開啟即用。適合 80% 個人使用者。
  1. 到 n8n.io/get-started
  2. 下載 Desktop app（Mac / Win）
  3. 安裝後開啟 → 自動跑在 localhost:5678
  4. 瀏覽器會自動打開 n8n 介面
Docker
穩定長期
容器化部署，適合放公司伺服器或 NAS。要懂 Docker 基本操作。
  1. 安裝 Docker Desktop
  2. 終端機跑：docker run -d --name n8n -p 5678:5678 n8nio/n8n
  3. 開 localhost:5678
  4. 加 -v 掛載目錄保存資料
npm
開發者
直接用 Node.js 跑。最輕量，適合想自訂或做開發整合。
  1. 安裝 Node.js 18+
  2. 終端機跑：npx n8n
  3. 開 localhost:5678
  4. 要長期跑用 pm2
真實情境 · IT 部門的小王
小王在公司 NAS（Synology）上用 Docker 跑 n8n，讓部門同事都能用。因為是內網，不用擔心安全問題。Docker 版的好處是 NAS 不關機，n8n 就 24/7 跑——解決了「電腦關機流程就停」的問題。一台 NT$15,000 的 NAS 就能替代 Make 年費 $108 USD。
(03) · 第一個 Workflow20 分鐘

跑通：排程 → 抓網頁 → 存 Sheet

n8n 的第一個 workflow 我們不做 Gmail（跟 CH5-1 重複），改做一個更有 n8n 特色的：定時抓網頁資料 → 存 Google Sheet。這展示了 n8n「能做 Make 做不到的事」。

Schedule
每天 9:00
→
HTTP Request
抓匯率 API
→
Code
擷取需要的欄位
→
Google Sheets
新增一行
1
開新 Workflow
在 n8n 首頁按「+ New Workflow」→ 命名為「每日匯率追蹤」。你會看到空白畫布，跟 Make 類似但更寬敞。
2
加 Schedule Trigger
按 + → 搜尋「Schedule Trigger」→ 設定 Rule：Trigger at → 9:00、Interval → Every day。這就是 n8n 版的排程觸發。
3
加 HTTP Request 抓匯率
在 Schedule 右邊按 + → 搜尋「HTTP Request」→ Method: GET → URL: https://api.exchangerate-api.com/v4/latest/USD。按「Test step」確認有回傳 JSON。
4
加 Code 節點擷取欄位
按 + → 搜尋「Code」→ 語言選 JavaScript → 貼入下方程式碼。這會從 API 回傳的 JSON 中取出 TWD 匯率和日期。
5
加 Google Sheets 寫入
按 + → 搜尋「Google Sheets」→ Operation: Append Row → 授權 Google 帳號 → 選試算表 → 欄位映射：A = date、B = rate。按「Test workflow」跑一次確認。
// n8n Code 節點：擷取匯率 const data = $input.all()[0].json; const rate = data.rates.TWD; const date = new Date().toISOString().split('T')[0]; return [{ json: { date: date, usd_to_twd: rate, source: 'exchangerate-api' } }];
核心觀念 · JSON 就是「有結構的文字」
n8n 裡所有資料都是 JSON 格式傳遞。不用害怕——JSON 就是「用大括號和冒號寫的清單」。{"name":"Amy","age":28} 就是一筆 JSON 資料。每個節點收到上一個節點的 JSON，加工後往下傳。你在 n8n 介面看到的「Input / Output」面板，就是在看 JSON 資料流過每個節點。Make 也是 JSON，只是 Make 用拖拉介面幫你處理了，n8n 讓你直接看到。
(04) · Cloudflare Tunnel25 分鐘

讓你的本機 n8n 能接外部 Webhook

n8n 跑在 localhost:5678——只有你自己電腦打得開。但如果你想接 LINE Bot 的訊息、Stripe 的付款通知、Google Form 的 Webhook，這些外部服務需要一個「公開網址」才能打到你的 n8n。Cloudflare Tunnel 就是幫你建一條安全通道：外部 → Cloudflare → 你的電腦。

真實情境 · 微型電商老闆 Eric
Eric 用 n8n 做訂單自動化。客戶在 Shopify 下單 → Shopify 送 Webhook 到 n8n → n8n 自動建出貨單 + 寄通知信。但 Shopify 的 Webhook 需要一個公開 URL。Eric 用 Cloudflare Tunnel 把自己筆電的 n8n 暴露成 https://n8n.eric-shop.com，Shopify 就能打得到。零月費，不用租伺服器。
1
註冊 Cloudflare 帳號
到 dash.cloudflare.com 註冊免費帳號。如果你有自己的網域，把它加到 Cloudflare（改 DNS nameserver）。沒有網域也可以用 Cloudflare 的臨時子域名。
2
安裝 cloudflared
Mac：brew install cloudflared
Windows：到 Cloudflare 官網下載安裝檔
安裝後在終端機跑 cloudflared --version 確認成功。
3
登入 + 建立 Tunnel
終端機跑：cloudflared tunnel login（會跳瀏覽器授權）
然後：cloudflared tunnel create n8n-tunnel
記下 Tunnel ID（後面要用）。
4
設定 config.yml
在 ~/.cloudflared/config.yml 建立設定檔（見下方範本）。把 hostname 設成你要的子域名、service 指向 http://localhost:5678。
5
啟動 Tunnel
先到 Cloudflare DNS 加一筆 CNAME 記錄 → 然後終端機跑：cloudflared tunnel run n8n-tunnel
打開 https://n8n.你的網域.com 確認能看到 n8n 介面。完成——你的 n8n 現在有公開網址了。
# ~/.cloudflared/config.yml tunnel: 你的-tunnel-id credentials-file: /Users/你的帳號/.cloudflared/tunnel-id.json ingress: - hostname: n8n.你的網域.com service: http://localhost:5678 - service: http_status:404
# 快速版（不需要自有網域） # 一行指令，Cloudflare 自動分配臨時子域名 cloudflared tunnel --url http://localhost:5678 # 會輸出類似： # https://random-name-1234.trycloudflare.com # 把這個 URL 填到 LINE Bot / Stripe Webhook 設定 # 注意：每次重新執行 URL 會變
核心觀念 · 為什麼不用 ngrok？
ngrok 也能做同樣的事，但免費版每次 URL 會變（Webhook 要改）、有頻寬限制、免費只能開一條。Cloudflare Tunnel 優勢：① 完全免費 ② URL 固定（用你自己的網域） ③ 自帶 HTTPS ④ 無頻寬限制。唯一缺點是設定稍複雜（上面 5 步），但設定一次就永久。
(05) · 電腦關機怎辦10 分鐘

n8n 的致命傷：電腦關了就停

這是所有人第一個問的問題。5 種解法，從簡單到進階：

1. 不關機
最簡單
筆電蓋上只休眠不關機，桌機設定「永不休眠」。適合家用、電費可接受的情境。
2. 舊手機 / 平板
零成本
把 n8n 裝在一台不用的舊 Android 手機（Termux + npm），插電放著。耗電量比燈泡還低。
3. NAS 跑 Docker
推薦
Synology / QNAP NAS 原生支援 Docker。裝一次，NAS 不關機 n8n 就 24/7。NT$15,000 左右。
4. 雲端 VPS
穩定
在 DigitalOcean / Vultr / AWS Lightsail 開一台最小 VPS（$5-6 USD/月），Docker 跑 n8n。
5. n8n Cloud
付費
n8n 官方的雲端版（$20 USD/月起）。等於 Make 的付費版，但 n8n 的邏輯和彈性更好。
真實情境 · 設計師 Mia 的 Synology 方案
Mia 買了一台 Synology DS224+（NT$12,000）來備份照片。發現它可以跑 Docker，就順便裝了 n8n。現在她的 n8n 24/7 跑在 NAS 上，接 Cloudflare Tunnel，LINE Bot 客服、表單處理、檔案整理全部自動化。NAS 的電費一個月不到 NT$50，等於免費。
(06) · 踩坑博物館5 分鐘

n8n 新手最常踩的 7 個坑

坑 1Desktop 版資料存哪？
症狀：重新安裝後所有 workflow 消失。
修：Desktop 版資料存在 ~/.n8n/。重裝前先備份這個資料夾。也可以用「Export」功能匯出每個 workflow 的 JSON。
坑 2Webhook URL 一直變
症狀：用 cloudflared tunnel --url 快速版，每次重啟 URL 都不同，LINE Bot 要重設。
修：用正式版 config.yml + 自有網域，URL 固定不變。或花 $5/月用 ngrok 固定子域。
坑 3Google 授權一直失敗
症狀：n8n 的 Google Sheets / Gmail OAuth 跳 400 錯誤。
修：n8n 的 Google OAuth 需要你在 Google Cloud Console 自己建 OAuth Client。流程比 Make 多一步：Console → API → Credentials → OAuth 2.0 → 加 redirect URI: http://localhost:5678/rest/oauth2-credential/callback。
坑 4Code 節點 return 格式錯
症狀：Code 節點跑了但下一個節點收不到資料。
修：n8n Code 節點必須 return [{json: {...}}]——外層陣列、內層 json 屬性。漏掉任何一層都不行。
坑 5Tunnel 啟動但外面打不開
症狀：cloudflared 跑了，瀏覽器打 URL 顯示 502。
修：確認 n8n 真的在跑（localhost:5678 能打開）。確認 config.yml 的 service 是 http://localhost:5678 不是 https。確認 DNS CNAME 設定正確。
坑 6Workflow 沒啟用
症狀：Webhook 設好了但外部打進來沒反應。
修：n8n 的 Workflow 預設是「Inactive」。右上角要切成「Active」才會真的監聽。Test mode 只在測試時觸發。
坑 7npm 版更新後壞了
症狀：跑 npm update -g n8n 後啟動失敗。
修：n8n 大版本更新有時會改資料庫結構。先備份 ~/.n8n/ 再更新。Desktop 版會自動處理，npm 版要手動。
(07) · 課堂動手15 分鐘

Guided Practice：裝起來、跑起來

PRACTICE 01 · 安裝 + 第一個 Workflow
在本機安裝 n8n 並跑通匯率追蹤
題目：① 用 Desktop 版安裝 n8n ② 照 Section 03 建「Schedule → HTTP Request → Code → Google Sheets」workflow ③ 按 Test workflow 跑通一次，確認 Sheet 出現新行。
提示：如果 Google 授權卡住，先用「Set」節點（固定值）代替 Sheets，測試前三個節點的資料流沒問題後再接 Sheets。
驗收標準
① n8n Desktop 安裝成功，localhost:5678 能打開
② Workflow 有 4 個節點，連線正確
③ Test workflow 執行成功（所有節點綠勾）
④ Google Sheet 出現一行：日期 + USD/TWD 匯率
PRACTICE 02 · Cloudflare Tunnel 快速版
一行指令讓 n8n 有公開 URL
題目：① 安裝 cloudflared ② 跑 cloudflared tunnel --url http://localhost:5678 ③ 用手機瀏覽器打開 Cloudflare 給的 URL，確認能看到 n8n 介面。
提示：快速版的 URL 每次重啟會變，僅限測試用。正式用途要設定 config.yml + 自有網域。
PRACTICE 03 · Webhook 接收測試
讓外部能打到你的 n8n
題目：① 在 n8n 建一個新 Workflow：Webhook 節點 → Set 節點（回傳 "ok"）② Workflow 切 Active ③ 用 Tunnel 的公開 URL + Webhook 路徑，在另一台裝置（手機）用瀏覽器打開測試。
提示：Webhook 節點會給你一個路徑（如 /webhook/abc123），完整 URL = Tunnel URL + 路徑。
