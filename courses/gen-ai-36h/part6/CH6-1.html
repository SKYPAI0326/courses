Part 6 · 個人 AI 系統整合
CH 6.1 · 2 小時

個人提示詞庫
與工具卡建立

前 5 個 Part 你累積了一堆好用的 prompt 和工具——但散在各個對話、各個平台，用起來還是要從頭打指令。本節教你把它們系統化歸檔，變成隨時能調用的「個人資產」。

學習成果 · OUTCOMES
分得清「提示詞庫」與「工具卡」的差別，兩者一起用才是完整系統
能用標準「工具卡範本」歸檔任何一個 AI 工具（含觸發條件、prompt、連結）
學會「分三層存」的策略：Claude Projects（常用）+ Notion/Obsidian（歸檔）+ GitHub（開源分享）
SECTION 01 · 為什麼要建庫

不建庫，每次都從頭開始

你現在打開 Claude，想起上週那個很順的 prompt——你得翻對話紀錄，找不到就重寫。10 次有 7 次重新打更爛的版本。這就是「沒系統化」的代價。

建庫的意義是：讓過去寫過的好 prompt，變成未來隨時可複製的資產。就像工匠把每次的工法記下來——下次遇到類似工件，拿出工法卡、改改參數即可。

🧭
心態轉變：從「每次重寫」→「一次寫好、永久使用」。Part 1-5 你已經寫了 40+ 個 prompt，本節結束後它們會變成你隨時能調出的工具集，而不是散落的 AI 對話紀錄。
SECTION 02 · 兩個概念

提示詞庫 vs 工具卡：分工不同

提示詞庫（Prompt Library）
純文字
一個個可以直接複製到 AI 對話框的 prompt 文字。每個 prompt 有標題、用途、變數說明。
例：「回客戶詢價信」的 prompt 文字（含四欄：角色、任務、限制、格式）。打開 Claude → 貼 prompt → 填變數 → 送出
工具卡（Tool Card）
結構化
一個「完整可使用的 AI 工具」的描述卡片，含：用途、觸發條件、輸入輸出、prompt、部署位置、使用連結。
例：「報價試算器」工具卡 — 含用途、inputs、Artifact 連結、System prompt（已部署在 Claude Project）、使用說明
💡
關係：提示詞庫是「原材料」（prompt 本身），工具卡是「成品目錄」（包裝好、部署好、可直接用的 AI 工具）。兩者分層存，各有用途——找靈感時翻 prompt 庫，要用工具時開工具卡。
SECTION 03 · 工具卡範本

每張工具卡應該包含什麼

好的工具卡應該讓你 30 秒內能判斷「這個工具是否解決我現在的問題」。標準 7 欄位：

工具卡範本（示例：客戶詢價回信助理）
TOOL · 001
用途
回覆客戶詢價，自動產生專業中文 email 草稿，含報價區間與下一步行動
觸發情境
客戶 email 或 LINE 訊息中含「想了解」「報價」「合作」「多少錢」等關鍵字
輸入
① 客戶訊息原文 ② 產品名稱 ③ 建議報價區間
輸出
① Email 草稿（250 字內） ② 內部記錄摘要（給 CRM 用）
部署位置
Claude Projects「客戶溝通」→ System prompt 已配置，直接丟客戶訊息即可
連結
claude.ai/projects/xxx（Claude Project）
備註
2026-04 建立 · 已處理過 30+ 次詢價，調整過 prompt 3 次
📋
關鍵是「觸發情境」欄：當你工作時遇到某情境，你能 5 秒想起「我有這個工具」。所以觸發情境要寫得具體——寫「當我要寫 email」太模糊，寫「當客戶 LINE 說想了解產品時」才管用。
SECTION 04 · 三層分存

提示詞庫要存在哪？

不要把所有 prompt 都塞一個地方。依使用頻率與場景分三層，效率最高：

層 1：Claude Projects（主力）
每天用
最常用的 5-10 個工具，在 Claude 內建 Project，System prompt 已配好，打開直接用。零複製貼上。
放什麼：回信助理、週報助理、客戶資料卡助理——這些每天都會用的
層 2：Notion / Obsidian（歸檔）
每週翻
所有 prompt 的完整歸檔，按類別組織（信件 / 文書 / 分析 / 工具 / 自動化）。要用時複製出來貼到 AI。
放什麼：每月 1-2 次用的 prompt、實驗中的版本、來自課程 / 書籍的參考範例
層 3：GitHub Pages（公開分享）
選配
打磨成熟、公開有價值的工具，部署到 GitHub Pages（Part 4 教過）。可分享給同事或網路上。
放什麼：已部署的零代碼工具（報價試算器、KPI dashboard）、對外分享用的精華 prompt 集
層 4：Bookmarks（快捷）
1 秒存取
最重要的 5-10 個直接加到 Chrome bookmark bar，使用時一鍵打開 —— 不需要搜尋。
放什麼：你每天一定會開的 Claude Project、NotebookLM、Make scenario 後台
SECTION 05 · 建置 SOP

從 0 開始5 步建立個人系統

1
列出你已有的 prompt
回翻 Part 1-5，把你覺得「會想再用」的 prompt 全部複製到一個 Notion 頁或 Obsidian 檔。先不分類，先收集。預計 20-40 個。
2
分類與命名
用 5 個大類：信件溝通 / 文書產出 / 分析整理 / 工具開發 / 自動化。每個 prompt 給個好懂的標題（「客戶詢價回信」勝過「prompt_1」）。
3
挑 Top 5 升級成 Claude Project
你最常用的 5 個，直接在 claude.ai 建 Project，把 prompt 貼到「Project Instructions」當 System Prompt。之後只要丟輸入即可。
4
寫工具卡
把 Top 5 Claude Project + 所有 Artifacts 工具 + Make/n8n 自動化 scenarios，用 Section 3 的 7 欄格式各寫一張工具卡。所有工具卡放在一個「我的 AI 工具箱」頁面。
5
加到 bookmark
Top 5 Claude Project + 工具箱頁面加進 Chrome bookmark bar。命名用 emoji + 短字（✍ 回信 / 📊 週報 / 🎯 工具箱）。視覺化快捷比長清單更有效。
SECTION 06 · 維護節奏

別讓它變成死資料庫

建好的庫若不維護，3 個月後就沒人用。建議跑以下維護節奏：

每日
用到新的 prompt 就存
發現一個有用的 prompt，立刻複製到「待整理」資料夾（不分類、不寫工具卡，先囤）。
每週
消化待整理（30 分鐘）
週五下午，處理待整理的 prompt：① 重要的 → 寫工具卡 + 建 Claude Project ② 一般的 → 分類歸檔 ③ 沒價值 → 刪。
每月
檢視 Top 5 表現
回顧你的 Top 5 Claude Project：還常用嗎？有哪個該調 prompt？有哪個已沒人用該下架？更新 Top 5 列表。
每季
大整理 + 歸檔
把這季累積的東西做一次完整歸檔：更新工具箱頁、淘汰過期工具卡、把高價值工具考慮部署到 GitHub Pages 公開分享。
⚠️
最常見的失敗模式：建好初始庫後就不再更新，3 個月後庫裡全是過時的 prompt。維護比建立重要 10 倍。把「每週 30 分鐘」當作固定的行事曆事件，像刷牙一樣。
SECTION 07 · 本節行動

課後練習：建你的第一批 5 張工具卡

不要想「一次建完所有 prompt」——那會拖半年永遠做不完。挑 5 個你最近一個月用最多次的 prompt，做以下事：

  1. 按本節格式寫 5 張工具卡（放在 Notion 或 Obsidian）
  2. 選 2-3 個升級成 Claude Project，設好 System prompt
  3. 把 Claude Project 連結加到 bookmark bar
  4. 下週記錄你用了幾次——這是庫有價值的證據
📝
下一節預告：CH 6.2「工作場景對應地圖」——本節建了「工具集」，下節會把工具「串成地圖」：把一天 / 一週的工作場景畫出來，每個場景對應哪個工具，讓 AI 不只是單點用，而是整合進工作流。
