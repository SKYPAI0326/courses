SECTION 01 · 為什麼要建庫15 分鐘

不建庫，每次都從頭開始

你現在打開 Claude，想起上週那個很順的 prompt——你翻了 5 分鐘對話紀錄，找不到就重寫。10 次有 7 次重新打了更爛的版本。這就是「沒系統化」的代價：你花了 5 個 Part 累積的 40+ 個好 prompt，散落在各處等著被遺忘。

真實故事 · 行銷小安
小安學完 Part 2 後寫了一個「社群文案產生器」prompt，效果很好——語氣對、字數準、還會附 hashtag。結果兩週後要用時翻不到，重寫了 3 次都沒上次好。因為那個版本是第 4 次迭代出來的，已經調過角色、限制、格式三個參數。沒存下來等於浪費那 40 分鐘的調校功夫。從那之後她用 Notion 建了一頁「我的 AI prompt 庫」，再也沒重寫過。
觀念深化 · 從「用完即丟」到「資產累積」
建庫的意義：讓過去寫過的好 prompt，變成未來隨時可複製的資產。就像工匠把每次的工法記下來——下次遇到類似工件，拿出工法卡、改改參數即可。

心態轉變：從「每次重寫」→「一次寫好、永久使用」。Part 1-5 你已經寫了 40+ 個 prompt，本節結束後它們會變成你隨時能調出的工具集，而不是散落的 AI 對話紀錄。
真實故事 · 業務經理 Kevin
Kevin 團隊 5 個人各自用 AI，各有一堆好 prompt，但從不分享。結果同事 A 花 30 分鐘調出的「客戶回信模板」，同事 B 不知道，又花 40 分鐘調一個更差的版本。整個團隊每月浪費 10+ 小時在重複造輪子。Kevin 後來建了一個共用的「業務 AI 工具卡」Notion 頁面，現在新人第一天就能用到全隊最佳 prompt。
SECTION 02 · 兩個概念20 分鐘

提示詞庫 vs 工具卡：分工不同

很多人搞混這兩個——以為「存 prompt 就是建庫」。不完全對。提示詞庫和工具卡是兩個層次，各有用途。

提示詞庫（Prompt Library）
本質：純文字。一個個可以直接複製到 AI 對話框的 prompt。
包含：標題、用途說明、可替換的變數、完整 prompt 文字
用法：打開 Claude → 貼 prompt → 填變數 → 送出
像什麼：食譜書——記錄每道菜怎麼做
工具卡（Tool Card）
本質：結構化。「一個完整可使用的 AI 工具」的描述卡片
包含：用途、觸發條件、輸入輸出、prompt、部署位置、使用連結、備註
用法：遇到情境 → 查工具卡 → 點連結直接開用
像什麼：廚房工具掛板——每把刀在哪、什麼時候用
觀念深化 · 兩者的關係
提示詞庫是「原材料」（prompt 本身），工具卡是「成品目錄」（包裝好、部署好、可直接用的 AI 工具）。

兩者分層存，各有用途——找靈感時翻 prompt 庫，要用工具時開工具卡。
一個成熟的工具卡裡面會包含對應的 prompt，但工具卡比 prompt 多了「觸發條件」和「部署位置」，這兩欄決定了你會不會真的用它。
真實故事 · 人資主管 Linda
Linda 存了 30 個 prompt 在 Notion，但從來沒用過。因為每次遇到問題，她不知道「該用哪個 prompt」——目錄太長、標題模糊、分類混亂。後來她把最常用的 8 個升級成工具卡，每張卡多寫了「觸發情境」：「當員工問請假規定時用」「當收到面試履歷時用」。從此遇到情境 → 5 秒想到工具卡 → 點開直接用。
SECTION 03 · 7 欄工具卡範本20 分鐘

每張工具卡應該包含什麼

好的工具卡讓你 30 秒內判斷「這個工具是否解決我現在的問題」。標準 7 欄位：用途、觸發情境、輸入、輸出、部署位置、連結、備註。

完整示例：客戶詢價回信助理
TOOL · 001
1. 用途
回覆客戶詢價，自動產生專業中文 email 草稿，含報價區間與下一步行動建議
2. 觸發情境
客戶 email 或 LINE 訊息中含「想了解」「報價」「合作」「多少錢」等關鍵字
3. 輸入
① 客戶訊息原文　② 產品名稱　③ 建議報價區間
4. 輸出
① Email 草稿（250 字內）　② 內部記錄摘要（給 CRM 用）
5. 部署位置
Claude Projects「客戶溝通」→ System prompt 已配置，直接丟客戶訊息即可
6. 連結
claude.ai/projects/xxx（Claude Project 直連）
7. 備註
2026-04 建立 · 已處理過 30+ 次詢價，調整過 prompt 3 次 · v3 加了語氣切換（正式 / 親切）

3.1 · 第二個完整示例：會議紀錄助理

完整示例：會議紀錄助理
TOOL · 002
1. 用途
會議結束後 5 分鐘內產出結構化會議紀錄（決議 / 待辦 / 下次議程），可直接寄 email
2. 觸發情境
任何會議結束後，手邊有錄音檔 or 手寫筆記 or 記憶還新鮮時
3. 輸入
① 會議錄音（NotebookLM 轉逐字稿）或手寫重點　② 會議議程　③ 出席名單
4. 輸出
① 3 段式紀錄（決議 / 待辦 / 下次）　② email 版（可直接寄出）
5. 部署位置
Claude Projects「會議紀錄」→ System prompt 含格式範本 + 上傳錄音流程
6. 連結
claude.ai/projects/yyy + notebooklm.google.com
7. 備註
搭配 NotebookLM 效果最好 · 內部 / 外部會議通用 · 建議會議結束 30 分鐘內跑完
# 工具卡建立 · Prompt 範本 角色：你是我的 AI 工具庫管理員。 任務：我會告訴你一個我常用的 AI 工具 / prompt， 請幫我用以下 7 欄格式整理成一張工具卡： 1. 用途（一句話說明做什麼） 2. 觸發情境（什麼狀況下用，要具體到場景） 3. 輸入（需要準備什麼資料） 4. 輸出（會產生什麼結果） 5. 部署位置（放在哪裡用） 6. 連結（一鍵可開的網址） 7. 備註（版本、使用次數、已知限制） 限制：觸發情境要寫得夠具體，讓我遇到時 5 秒內想起。 不要寫「需要寫 email 時」，要寫「當客戶 LINE 問報價時」。
SECTION 04 · 三層分存策略20 分鐘

提示詞庫要存在哪？

不要把所有 prompt 都塞一個地方。依使用頻率與場景分三層，效率最高。

層 1 · Claude Projects（每天用）
最常用的 5-10 個工具，在 Claude 內建 Project，System prompt 已配好，打開直接用、零複製貼上。
放什麼：回信助理、週報助理、客戶資料卡——這些每天都會用的
層 2 · Notion / Obsidian（每週翻）
所有 prompt 的完整歸檔，按類別組織（信件 / 文書 / 分析 / 工具 / 自動化）。要用時複製出來貼到 AI。
放什麼：每月 1-2 次用的 prompt、實驗版本、課程 / 書籍的參考範例
層 3 · GitHub Pages（選配 · 公開分享）
打磨成熟、公開有價值的工具，部署到 GitHub Pages（Part 4 教過）。可分享給同事或網路上。
放什麼：已部署的零代碼工具（報價試算器、KPI dashboard）、精華 prompt 集
加分層 · Bookmarks（1 秒存取）
最重要的 5-10 個直接加到 Chrome bookmark bar，一鍵打開不需搜尋。
放什麼：Claude Project、NotebookLM、Make scenario 後台。命名用 emoji + 短字更好辨識

4.1 · 5 步建置 SOP

1
列出你已有的 prompt
回翻 Part 1-5，把你覺得「會想再用」的 prompt 全部複製到一個 Notion 頁或 Obsidian 檔。先不分類，先收集。預計 20-40 個。
2
分類與命名
用 5 個大類：信件溝通 / 文書產出 / 分析整理 / 工具開發 / 自動化。每個 prompt 給好懂的標題（「客戶詢價回信」勝過「prompt_1」）。
3
挑 Top 5 升級成 Claude Project
最常用的 5 個，直接在 claude.ai 建 Project，把 prompt 貼到「Project Instructions」當 System Prompt。之後只要丟輸入即可，省掉每次貼 prompt 的 30 秒。
4
寫工具卡
把 Top 5 Claude Project + 所有 Artifacts 工具 + Make/n8n 自動化 scenarios，用 Section 3 的 7 欄格式各寫一張工具卡。所有工具卡放在一個「我的 AI 工具箱」頁面。
5
加到 bookmark
Top 5 Claude Project + 工具箱頁面加進 Chrome bookmark bar。命名用 emoji + 短字（「回信」「週報」「工具箱」）。視覺化快捷比長清單有效。
# Prompt 庫初始化 · 蒐集整理範本 角色：你是 AI 工具庫整理顧問。 任務：我會貼給你一堆零散的 prompt（可能有重複、格式不一）。 請幫我做以下整理： 1. 去重複（相似的合併成一個最佳版本） 2. 分成 5 大類：信件溝通 / 文書產出 / 分析整理 / 工具開發 / 自動化 3. 每個 prompt 重新命名（用「動詞+對象」格式，如「回覆客戶詢價」） 4. 標注推薦度（★★★ = 常用必備 / ★★ = 偶爾用 / ★ = 備查） 5. 推薦 Top 5 升級成 Claude Project 的候選名單 輸出格式：按分類列表，每個 prompt 含：名稱 / 分類 / 推薦度 / 一句話用途 最後附：Top 5 升級推薦（含推薦理由）
SECTION 05 · 維護節奏20 分鐘

別讓它變成死資料庫

建好的庫若不維護，3 個月後就沒人用。維護比建立重要 10 倍。

日
用到新的 prompt 就存
發現一個有用的 prompt，立刻複製到「待整理」資料夾（不分類、不寫工具卡，先囤）。關鍵是降低門檻——先存再說，5 秒完成。
週
週五 30 分鐘消化待整理
處理待整理的 prompt：重要的 → 寫工具卡 + 建 Claude Project；一般的 → 分類歸檔；沒價值的 → 刪。把這 30 分鐘當行事曆固定事件。
月
檢視 Top 5 表現
回顧你的 Top 5 Claude Project：還常用嗎？prompt 需要調嗎？有沒有用、該下架嗎？更新 Top 5 列表。
季
大整理 + 歸檔
這季累積的東西做一次完整歸檔：更新工具箱頁、淘汰過期工具卡、把高價值工具考慮部署到 GitHub Pages 公開分享。像衣櫥換季一樣。
觀念深化 · 維護的心理學
人類天生不擅長維護——建新東西有多巴胺，維護舊東西沒有。解法：

① 讓維護變得小到不需意志力：每日存 prompt 只要 5 秒（複製到固定資料夾）
② 讓維護變得可見：每週整理後在工具箱頁面上標「更新日期：4/15」
③ 讓維護有回報：記下你「因為有庫所以省了多少分鐘」的次數——第一週通常 3-5 次
SECTION 06 · 踩坑博物館10 分鐘

過來人踩過的坑，你不用再踩

坑 1一次想建完整個庫
症狀：花一整天把 100 個 prompt 全部分類、寫工具卡——累到崩潰，之後再也不碰。
解法：第一天只做 5 張工具卡（你最常用的）。之後每週加 2-3 張。3 個月後自然有 30+ 張。
坑 2觸發情境寫太模糊
症狀：工具卡觸發欄寫「要寫 email 時」。結果遇到真實情境時想不起來要用。
解法：寫具體場景：「當客戶 LINE 說『想了解報價』時」「當週五下午要交週報時」。越具體越容易觸發。
坑 3存了不維護
症狀：初始建庫很認真，3 個月後裡面的 prompt 全是舊版，Claude 更新了好幾次都沒跟上。
解法：行事曆設重複事件：每月最後一個週五「30 分鐘 AI 工具箱維護」。像刷牙一樣，養成習慣。
坑 4工具散在 3 個地方找不到
症狀：有的在 Claude Project、有的在 Notion、有的在 Google Drive——找工具比手動做還慢。
解法：建一個「工具箱索引頁」（Notion 或 Obsidian），所有工具卡集中放這裡，每張卡有直連連結。入口唯一。
坑 5只存 prompt 不存上下文
症狀：存了 prompt 但沒記「這個 prompt 是配合什麼 System prompt 用的」。單獨貼出來效果差 50%。
解法：工具卡第 5 欄「部署位置」要寫清楚：「配合 Claude Project 的 System prompt 一起用」「需要先在 NotebookLM 上傳文件」。
坑 6不敢刪東西
症狀：庫裡 80 個 prompt，其中 50 個半年沒用。捨不得刪，結果好用的被淹沒。
解法：每季大整理時，3 個月沒用的移到「歸檔區」。6 個月沒用的直接刪。工具箱越精簡越好用。
SECTION 07 · GUIDED PRACTICE15 分鐘

動手練習：建你的第一批工具卡

GUIDED PRACTICE 1
寫 3 張工具卡
回想 Part 1-5 你最常用的 3 個 prompt。用 Section 3 的 7 欄格式，把它們各寫成一張工具卡。觸發情境欄要具體到「遇到什麼情境時打開」。
提示：不確定觸發情境怎麼寫？問自己：「上次用這個 prompt 是因為遇到什麼事？」那件事就是你的觸發情境。
驗收標準：3 張工具卡都有 7 欄填完，觸發情境欄不超過 1 句話但夠具體，旁邊同學看得懂「什麼時候用」。
GUIDED PRACTICE 2
挑 1 個升級成 Claude Project
從你的 3 張工具卡中選 1 個最常用的，現在就在 claude.ai 建一個 Claude Project，把 prompt 貼到 Project Instructions。丟一筆真實輸入測試，確認能跑。
提示：建 Project 步驟：claude.ai → 左側 Projects → Create → 取名 → 點「Project Instructions」貼入 prompt → 開始對話。
驗收標準：Claude Project 可正常運作，丟真實輸入後 3 秒內有產出。把 Project 連結加到 Chrome bookmark bar。
SECTION 08 · 課後行動

這週做這些，讓庫活起來

不要想「一次建完所有 prompt」——那會拖半年永遠做不完。先跑完以下最小可行動作：

  1. 按本節格式寫 5 張工具卡（放在 Notion 或 Obsidian）
  2. 選 2-3 個升級成 Claude Project，設好 System prompt
  3. 把 Claude Project 連結加到 bookmark bar
  4. 行事曆設重複事件：「每週五 30 分鐘整理 AI 工具箱」
  5. 下週記錄你用了幾次——這是庫有價值的證據
