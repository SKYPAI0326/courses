透明背景 + 尺寸調整（規格符合）

橘貓表情 8 張做出來了，但 LINE 後台一張都不收——規格全錯。這 40 分鐘你要把「AI 看起來很美但規格全錯的圖」，全部改成「LINE 後台會收的版本」。

📄 跳到 4 張紙本素材（可單張列印）
學習成果
能用 Pixian 把 8 張圖去背為透明背景 PNG，邊緣無鋸齒、無殘影
能在 Photopea 內把單張調到 370×320 px、檔案 ≤1MB；能匯出主圖（240×240）+ Tab 圖（96×74）

(01)

90% 完工的錯覺——規格全錯

為什麼這很重要
AI 產出的圖通常是 1024×1024 px、JPEG 格式、底色不透明、檔案 1.8MB。LINE 主貼圖規格：PNG 透明背景、370×320 px、≤1MB。四項全錯。這 40 分鐘最大的敵人不是技術，是時間管理——一張圖去背順利只要 30 秒，卡住的話 10 分鐘下不來。

你在上一單元產出了 8 張橘貓表情，看著螢幕覺得 90% 完工。但你開啟其中一張「橘貓得意」，發現背景是一塊淺米色帶漸變的方塊——不是透明的。接下來這 40 分鐘，你要走完三件事：去背、改尺寸、統一命名。這段比 AI 繪圖更容易卡你——很多人以為畫圖最難，其實是去背 + 規格才是真關卡。

直接告訴你結論：三層救援階梯 + 不該救的圖直接重產決策樹，照著走絕對不會超時。

▲
5 分鐘還救不起來，就回上一單元重產
去背超過 5 分鐘還沒搞定 → 立刻關掉 Photopea，回 Gemini 重產同表情。重產一張 AI 圖只要 90 秒，你在去背卡的 10 分鐘可以重產 6 張。省時不省力這四個字記下來。
(02)

先把 6 個關鍵詞釘死

這單元用到的術語，後面操作會反覆引用：

術語意思
透明背景圖片背後是「空的」，貼到 LINE 對話框不會帶一個方塊。對應格式：PNG（不是 JPEG）
主貼圖套組裡 8/16/24 張那種。LINE 規格：370×320 px、PNG、≤1MB、透明背景
主圖套組封面縮圖。240×240 px、PNG、≤1MB。從 8 張裡挑一張改尺寸，不是另外畫
Tab 圖LINE 鍵盤上方的小圖示。96×74 px、PNG、≤1MB。從 8 張裡挑線條最清楚的那張改
三層救援階梯去背工具優先級：(1) Pixian 一鍵 → (2) Erase.bg 備援 → (3) Photopea 手動。前一層失敗才往下走，不要直接跳第三層
不該救的圖直接重產去背超過 5 分鐘還救不起來 → 回上一單元重產同表情。省時不省力，不是放棄是時間管理

三類檔案規格速查（背起來）：

用途尺寸格式大小上限數量
主貼圖370×320 pxPNG（透明背景）1MB8 張
主圖240×240 pxPNG（透明背景）1MB1 張
Tab 圖96×74 pxPNG（透明背景）1MB1 張

全部都是 PNG 透明背景——這 40 分鐘你就在做這一件事，不要被別的格式（JPG、WebP、SVG）干擾。

統一檔案命名規則（強制，不要自創）：

orangecat_01.png ← 主貼圖第 1 張 orangecat_02.png ← 主貼圖第 2 張 orangecat_03.png orangecat_04.png orangecat_05.png orangecat_06.png orangecat_07.png orangecat_08.png ← 主貼圖第 8 張 orangecat_main.png ← 主圖（封面縮圖） orangecat_tab.png ← Tab 圖（鍵盤小圖示）

範例以橘貓示範。你的角色名換成英文小寫不空格（orangecat、bluepuppy），不要用中文或大寫。

輸入角色英文名 → 一鍵生成 10 個檔名清單
輸入你的角色英文名（小寫），點按鈕會自動生成全部 10 個檔名（8 主貼圖 + 主圖 + Tab 圖），複製貼到記事本，照著存檔就不會亂。

  


  
為什麼強制？ 下一單元上架時，LINE 後台一個欄位只能上傳一張、要照順序對應。檔名統一的話照順序拖檔 3 分鐘就上完；檔名亂的話每張都要重新對，10 分鐘拖成半小時。










  
(03)

  
三層救援階梯——照順序走，別跳


  


    

      
1

      

        
Pixian.AI 一鍵去背（首選 · 90% 場合搞定）

        

          
Pixian 是主力工具。大多數 AI 卡通圖它一鍵就能搞定，邊緣乾淨、毛髮通透、沒有殘影。

          
    
            
  1. 開瀏覽器到 pixian.ai（注意是 .ai 不是 .com）
    2. 
            
  2. 把橘貓 01（笑）拖進中央大框，或按「Upload Image」
    3. 
            
  3. 等 5 秒，右邊顯示去背後的圖
    4. 
            
  4. 按 Download 前先看 Output Settings：Format = PNG、Resolution = Full 或 Original（不要選 Preview，Preview 是 0.25MP 縮水版，上傳 LINE 會被退「解析度不足」）
    5. 
            
  5. 點「Download」存到桌面
    6. 
            
  6. Finder 按空白鍵預覽，背景應顯示棋盤格（透明），不是白色
    7. 
          

          
Pixian 免費版每月有限額（約 10–20 張全解析度下載），但 8 張主貼圖在額度內可以全部跑完。額度用光跳第 2 層。

        

      

    


    

      
2

      

        
Erase.bg 備援（Pixian 失敗時切換）

        

          
Pixian 失敗的兩種情境：(1) 額度用光、(2) 邊緣處理不乾淨（水彩風或毛髮複雜的圖）。不要硬幹，切到 Erase.bg。

          
    
            
  1. 開 erase.bg
    2. 
            
  2. 拖檔上傳，等 5 秒
    3. 
            
  3. 預覽看邊緣，對比 Pixian 的版本，誰乾淨選誰
    4. 
            
  4. 點「Download HD Image」（不是 Standard，Standard 也是縮水版）
    5. 
            
  5. 同樣 Finder 預覽確認透明
    6. 
          

          

            
情境首選

            
              
卡通扁平風（橘貓）Pixian

              
水彩漸變風（水母）Erase.bg 通常更通透

              
邊緣有 AI 暈染、模糊光暈Erase.bg 切得比較利

              
毛髮細節（橘貓鬍鬚）Pixian 保留比較完整

            
          

        

      

    


    

      
3

      

        
Photopea 手動救援（救援級 · 5 分鐘決策樹）

        

          
什麼時候走第 3 層？ Pixian + Erase.bg 都跑過，邊緣還是有殘影、或者主體被切掉（例如鬍鬚被當背景吃掉）。8 張裡通常會遇到 1–2 張。

          
    
            
  1. 開 photopea.com，把第 2 層失敗的去背檔拖進來
    2. 
            
  2. 左側工具列點魔術棒（W 鍵），上方「Tolerance」設 20（預設 32 太鬆）
    3. 
            
  3. 在殘影區域點一下，選出灰邊那塊
    4. 
            
  4. 按 Delete 鍵刪除選區
    5. 
            
  5. 零碎殘影換橡皮擦（E 鍵），筆刷大小 10–20 px，手動擦
    6. 
            
  6. 完成後按 File → Export As → PNG 下載
    7. 
          

          
Photopea 在低階筆電會卡頓：同時開 8 張會卡死，處理完一張關掉再開下一張；關掉隔壁的 Gemini / ChatGPT 分頁釋放記憶體；風扇開始狂轉時存檔後重整一次。

        

      

    


    

      
4

      

        
Photopea 尺寸調整 + 主圖 / Tab 圖匯出

        

          
去背完成後還有最後一步：把每張改成 LINE 規格尺寸。

          
主貼圖 8 張（每張 30 秒）：

          
    
            
  1. 在 Photopea 開啟去背完的 PNG
    2. 
            
  2. 先 Image → Canvas Size 改成 370×320（背景選透明，把畫布調對）
    3. 
            
  3. 再 Edit → Free Transform（Cmd+T），按 Shift 等比拖角縮放，橘貓對準畫布中央
    4. 
            
  4. File → Export As → PNG，Quality 拉最高（Compress 0）
    5. 
            
  5. 檔名：orangecat_01.png 到 orangecat_08.png
    6. 
            
  6. Finder 看檔案大小，超過 1MB 的：重新匯出把 Quality 降到 9（10 是最高），通常就能壓進 1MB
    7. 
          

          
為什麼不用 Image → Image Size？ 原圖是 1:1（1024×1024），目標是 370×320（約 1.156:1），直接 Image Size 會強制壓扁橘貓。分兩步（Canvas Size 先調畫布 → Free Transform 再縮圖）才能保持比例。

          
主圖 240×240（從第 01 張改）：

          
    
            
  1. 開 orangecat_01.png（去背完的 370×320 版）
    2. 
            
  2. Image → Image Size，改 240×240（這次內容等比縮）
    3. 
            
  3. Export As PNG，存成 orangecat_main.png
    4. 
          

          
Tab 圖 96×74（從第 03 張改，挑線條清楚的）：

          
    
            
  1. 開 orangecat_03.png
    2. 
            
  2. Image → Image Size，改 96×74
    3. 
            
  3. 縮完用肉眼看角色五官能不能辨識——能辨識才算合格，糊成一團換另一張
    4. 
            
  4. Export As PNG，存成 orangecat_tab.png
    5. 
          

          
Tab 圖選圖原則：表情最簡單、線條最粗、輪廓最完整的那張。「03 生氣」「04 得意」這種臉佔滿畫面的最適合；「07 晚安」（眼睛閉著、躺著）Tab 圖看不清楚。

(06)

  
常見錯誤 3 條——踩過才能理解


  

    

      
錯誤 01

      
Pixian 下載 Preview 版本，LINE 退「解析度不足」

    

    

      

        
錯誤現象

        
用 Pixian 跑完去背，看著預覽很滿意按 Download。8 張全做完上傳 LINE Creators Market，卻全部被退件「解析度不足」。回頭才發現下載的是 0.25MP Preview 版本。

      

      

        
原因

        
Pixian.AI 免費版預設下載是 Preview（縮水到 0.25MP），不是 Full Resolution。「Full Resolution」按鈕在 Output Settings 裡，預設沒勾。

      

      

        
解法

        
每次下載前確認 Output Settings「Resolution」= Full 或 Original，不是 Preview。已下載錯的：回 Pixian 重跑（同檔案重跑不耗額度）。

      

    

  


  

    

      
錯誤 02

      
Photopea Image Size 直接改 370×320，圖被壓扁變形

    

    

      

        
錯誤現象

        
開 1024×1024 橘貓去 Image → Image Size 直接打 370×320，橘貓變成上下被壓扁的橘色橢圓，鬍鬚也歪掉。

      

      

        
原因

        
原圖是 1:1，目標是 1.156:1（370÷320）。直接 Image Size 強制塞到目標尺寸，比例改變導致變形。

      

      

        
解法

        
分兩步：先 Canvas Size 改 370×320（保留原圖，只改畫布），再 Free Transform（Cmd+T）等比縮放對準中央。

      

    

  


  

    

      
錯誤 03

      
檔名用中文 + 空格 + 大寫，LINE 後台上傳卡關

    

    

      

        
錯誤現象

        
把檔案命名為「橘貓 笑 01.png」「Orange Cat tab.PNG」，下一單元上架拖檔時部分檔案卡住或上傳後對應錯位，甚至 .PNG 大寫副檔名被 LINE 後台擋下。

      

      

        
原因

        
中文檔名在某些瀏覽器會被 URL encode，LINE 後台相容性差；空格在拖檔時偶爾被當成多檔；.PNG 大寫 MIME type 檢測寫死小寫。

      

      

        
解法

        

          嚴格遵守命名規則：英文小寫 + 底線 + 兩位數字 + 小寫 .png
          

            
錯誤檔名修正後

            
              
橘貓 笑 01.pngorangecat_01.png

              
Orange Cat 01.PNGorangecat_01.png

              
orangecat-01.pngorangecat_01.png（連字號 → 底線）

              
OrangeCat01.pngorangecat_01.png

              
orangecat_1.pngorangecat_01.png（補 0 對齊兩位數）

            
          

          
macOS 小技巧：Finder 選 8 個檔案 → 右鍵「Rename...」→ Format，一次改完。

        

      

    

  










  
(07)

  
檢核——概念驗證 + R1–R5 上架前快篩


  

    
Q1（概念驗證）：你用 Pixian 跑完去背、按 Download 後，在 Photopea 改 370×320 並 Export PNG 上傳 LINE，被退「解析度不足」。最可能的原因是？

    

      
A. Photopea 的 Export 預設壓縮太多

      
B. PNG 透明背景在 LINE 後台不被支援，要改用 JPG

      
C. Pixian 下載時用了預設的 Preview 版本（0.25MP）而不是 Full Resolution

      
D. 370×320 這個尺寸太小，LINE 規格其實是 740×640

    

    

      看解析
      
答案是 C。A 不會導致解析度不足（最多影響檔案大小）；B 完全錯（LINE 主貼圖規格就是 PNG 透明背景）；D 完全錯（370×320 就是規格）。C 是 Pixian 的隱藏雷點：免費版預設下載 Preview 縮水版，肉眼看不出來但 LINE 檢測解析度會擋下。每次下載記得切到 Full / Original。

    

  


  

    
Q2（應用驗證）：你在處理橘貓第 5 張「求拜託」。Pixian 跑完邊緣有殘影、Erase.bg 跑完鬍鬚被切掉一根。你打開 Photopea 開始用魔術棒手動補救，已經調了 6 分鐘還沒搞定。接下來你應該怎麼做？

課堂紙本素材——4 張可列印帶走

  
這單元是 P0 隱形殺手單元，去背 + 規格 + 上架前快篩 三件事容易卡。下面 4 張就是課堂會發給你的紙本素材。每張右上有「📄 列印這張」按鈕——按下去會開新分頁印 A4 一張，操作時放在桌上隨時翻。


  
  

    

      

        
素材 1 · 操作時放桌上

        
三層救援階梯 SOP 卡

      

      
    

    

      
遇到去背問題照三層走，不要直接跳第三層。每一層失敗才往下，前一層成功就停在那層。

      
    
        
  1. 
          第 1 層 · Pixian.AI（首選 · 90% 場合）
    
          網址：pixian.ai（注意是 .ai 不是 .com）
    
          操作：拖檔 → 5 秒 → Output Settings 切 Full / Original（不要 Preview）→ Download
    
          什麼時候切：成功率 90%，卡通扁平風一律先用這層。免費版每月 10–20 張全解析度額度。
        
    2. 
        
  2. 
          第 2 層 · Erase.bg（備援 · 切換用）
    
          網址：erase.bg
    
          操作：拖檔 → 5 秒 → 點 Download HD Image（不是 Standard）
    
          什麼時候切：(1) Pixian 額度用光、(2) Pixian 邊緣處理不乾淨——水彩漸變風或邊緣模糊光暈時 Erase.bg 通常更利。
        
    3. 
        
  3. 
          第 3 層 · Photopea 手動（救援級 · 5 分鐘上限）
    
          網址：photopea.com
    
          操作：開檔 → 魔術棒（W 鍵，Tolerance 20）→ 點殘影區 → Delete → 橡皮擦補（E 鍵，10–20 px）→ Export As PNG
    
          什麼時候切：Pixian + Erase.bg 都跑過、邊緣還是有殘影或鬍鬚被切。8 張裡通常 1–2 張會落到這層。
        
    4. 
      

      
兩家都跑一次挑邊緣乾淨的；不要在單張反覆調 Pixian 進階參數——那不是免費版該花時間的地方，切第 2 層比較快。

素材 2 · 計時用

        
5 分鐘決策樹卡——卡住就重產

      

      
    

    

      
這張卡的目的是壓住「沉沒成本心態」。AI 重產一張只要 90 秒，硬救一張 10 分鐘——怎麼選一目瞭然。照 5 步走，第 5 步觸發就回 CH1-3 重產。

      
    
        
  1. 
          步驟 1 · 跑 Pixian.AI
    
          If 邊緣乾淨、毛髮通透、無殘影 → Then 收檔，跳到 Photopea 改尺寸（這張通關）
    
          If 邊緣有殘影或鬍鬚被切 → Then 進步驟 2
        
    2. 
        
  2. 
          步驟 2 · 評邊緣
    
          If 殘影屬於「光暈一圈」（半透明殘留底色）→ Then 進步驟 3 切 Erase.bg
    
          If 殘影屬於「主體被切掉」（鬍鬚 / 髮絲被當背景吃掉）→ Then 直接跳步驟 5 重產（Erase.bg 通常救不回主體）
        
    3. 
        
  3. 
          步驟 3 · 切 Erase.bg
    
          If 邊緣比 Pixian 乾淨 → Then 收檔通關
    
          If 仍有殘影但範圍已縮小 → Then 進步驟 4
        
    4. 
        
  4. 
          步驟 4 · 試 Photopea 5 分鐘
    
          開始計時 5 分鐘，魔術棒 + 橡皮擦補殘影
    
          If 5 分鐘內救起來 → Then 收檔通關
    
          If 5 分鐘到還沒搞定 → Then 進步驟 5
        
    5. 
        
  5. 
          步驟 5 · 還救不起來就回 CH1-3 重產
    
          關掉 Photopea → 開 Gemini 同對話 → 「換一張同表情，背景改純白／純藍」（重產後背景單純，去背成功率拉滿）
    
          重產 90 秒搞定，比硬救快 10 倍
        
    6. 
      

      
Photopea 手動 10 分鐘是硬上限，5 分鐘還沒救回來就重產比較快。沉沒成本心態（「我已經花這麼久了不甘心」）會把 40 分鐘的單元變成 80 分鐘——這張卡就是來打斷那個念頭的。
