單元 5-4

內容 SEO：讓 Google 和人都看懂

好的內容 SEO 是雙重任務：讓搜尋引擎能正確理解頁面主題，同時讓真實用戶讀起來流暢、滿足意圖。Title、Meta、H 標籤、語意優化、FAQ Schema——每一個元素都在發送信號。

學習目標 OUTCOMES
能為任意頁面寫出符合 SEO 規範的 Title Tag 和 Meta Description（字數、關鍵字、點擊誘因）
能正確設計 H1/H2/H3 標題層級，並在內容中自然融入語意相關詞彙
能為文章加上 FAQ Schema 結構化資料，爭取精選摘要版位
ON-PAGE ELEMENTS · 頁面元素

五個標籤，送給 Google 的信號

<title>
目標關鍵字放在最前面，50–60 字元（中文約 25–30 字）。格式：主關鍵字 | 品牌名。這是 SERP 上用戶看到的第一行藍色文字，直接影響點擊率。
meta desc
150–160 字元，包含目標關鍵字與行動誘因。雖然不直接影響排名，但影響點擊率，點擊率又間接影響排名。把它當作一個 30 字的廣告文案寫。
H1
每頁只能有一個 H1，通常與 Title 相近但可略有不同。H1 是給用戶看的主標題，需要包含主關鍵字，但更重要的是要引人繼續讀下去。
H2 / H3
次標題結構化內容邏輯。H2 是主要章節，H3 是子章節。在 H2 中自然使用相關長尾詞和語意相關詞（LSI Keywords），幫助 Google 理解內容廣度。
URL
簡短、包含關鍵字、用連字號分隔。例：/seo-guide-2025/ 而非 /post?id=1234。中文 URL 會被編碼，建議使用英文 slug。

SEMANTIC SEO · 語意優化

不只關鍵字密度，是語意完整性

現代 SEO 的重心已從「關鍵字密度」轉向「語意完整性」。Google 的 NLP 演算法會評估整篇文章是否完整涵蓋這個主題應有的相關概念。寫一篇 SEO 文章前，先問：「這個主題最完整的討論應該包含哪些子題？」

🔍
語意優化實作方法：搜尋目標關鍵字，看前三名文章有哪些 H2 章節是你沒有的。「People Also Ask」中的問題，是 Google 認為相關的語意範疇——都應該在文章中回答。
📝
LSI 關鍵字（語意相關詞）：寫 SEO 文章時，除了主關鍵字，要自然融入同義詞和相關詞。例如寫「SEO」時，「搜尋引擎優化」「有機流量」「關鍵字排名」都應該自然出現，讓 Google 確認內容的主題範疇。
FAQ SCHEMA · 結構化資料

搶佔精選摘要：FAQ Schema 實作

FAQ Schema 是一段 JSON-LD 程式碼，放在頁面 <head> 或 <body> 中，告訴 Google「這個頁面有問答結構」。Google 可能會直接在 SERP 顯示這些問答——讓你的搜尋結果佔更大的版面空間，提升點擊率。

FAQ Schema 範本（JSON-LD）
<script type="application/ld+json">
{
"@context": "https://schema.org",
"@type": "FAQPage",
"mainEntity": [
{
"@type": "Question",
"name": "SEO 要多久才有效果？",
"acceptedAnswer": {
"@type": "Answer",
"text": "一般而言，新網站的 SEO 效果需要 3–6 個月才開始明顯。..."
}
},
{
"@type": "Question",
"name": "...",
"acceptedAnswer": { "@type": "Answer", "text": "..." }
}
]
}
</script>
