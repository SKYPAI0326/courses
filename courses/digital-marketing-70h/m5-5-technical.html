單元 5-5

技術 SEO：讓 Google 能順暢爬取

最好的內容，如果 Google 爬不到、載入太慢、在手機上破版，排名就是空談。技術 SEO 是 SEO 的地基，確保搜尋引擎能順暢理解並索引你的網站。

學習目標 OUTCOMES
能說明 Core Web Vitals 三項指標的意義與及格門檻，並指出改善方向
能識別常見的技術 SEO 問題（301、Canonical、HTTPS、robots.txt），並說明各自的用途
能用 Google Search Console 的「網址審查」和「Core Web Vitals」報告，診斷基本技術問題
CORE WEB VITALS · 核心網頁指標

Google 的使用者體驗評分卡

Core Web Vitals（CWV）是 Google 從 2021 年起正式納入排名的用戶體驗指標。它測量的不是內容品質，而是用戶打開網頁後的實際感受：載入速度、視覺穩定性、互動回應。

LCP
Largest Contentful Paint
最大內容渲染
良好：≤ 2.5 秒
頁面上最大的視覺元素（通常是首圖或 Hero 區塊）載入完成所需時間。影響因素：圖片大小、伺服器回應速度、阻擋渲染的 JS。
INP
Interaction to Next Paint
互動到下一次渲染
良好：≤ 200ms
用戶點擊/輸入後，頁面多快給出視覺回饋。取代舊指標 FID。影響因素：JS 執行負擔、主執行緒阻塞。
CLS
Cumulative Layout Shift
累積版面移動
良好：≤ 0.1
頁面載入過程中，元素「突然跳位」的程度。影響因素：未設定尺寸的圖片、動態插入廣告、字體載入導致排版跳動。
🔧
診斷工具：Google PageSpeed Insights（免費輸入網址即可診斷）、GSC 的「Core Web Vitals」報告、Chrome DevTools Lighthouse。三個工具免費且夠用。
TECHNICAL CHECKLIST · 技術清單

網站健康度，六個必查項目

🔒
HTTPS 加密：所有頁面都應通過 HTTPS。HTTP 頁面在 Chrome 會顯示「不安全」警告，且排名有懲罰。SSL 憑證現在大多免費（Let's Encrypt）。
📱
行動裝置相容：Google 使用行動版索引優先（Mobile-First Indexing）。用 Google 的「行動裝置相容性測試」工具驗證，確保版面在手機上不破版、按鈕大小合適。
🗺️
XML Sitemap：提交網站的 sitemap.xml 給 GSC，幫助 Google 發現所有頁面。WordPress 安裝 Yoast SEO 或 RankMath 會自動生成。
🤖
robots.txt：控制哪些頁面允許爬蟲抓取。確認沒有意外封鎖重要頁面（常見錯誤：Disallow: / 封鎖整個網站）。網址格式：yoursite.com/robots.txt
🔗
內部連結結構：確保所有重要頁面都能從首頁在 3 次點擊內抵達。「孤立頁面」（沒有其他頁面連結到它）難以被 Google 找到和評估。
🖼️
圖片優化：壓縮圖片（目標 <200KB）、使用 WebP 格式、為每張圖填寫 alt 文字（描述圖片內容 + 自然帶入關鍵字）。圖片是 LCP 分數的主要影響因素。
REDIRECTS & CANONICAL · 重定向與標準化

避免重複內容：Canonical 與 301

常見的 HTTP 狀態碼與 SEO 影響
301
永久重定向——將舊 URL 的 SEO 權重傳遞到新 URL。修改網址時必用。例：將 HTTP 重定向到 HTTPS、舊文章 URL 改變時。
302
臨時重定向——不傳遞 SEO 權重。只用在真正臨時的情況（如：活動頁面暫時轉移）。錯誤地用在永久重定向會損失 PageRank。
404
頁面不存在——過多 404 會影響爬取預算。刪除頁面前，確認是否有外部連結指向，有的話應設 301 到相關頁面。
Canonical
指定標準版本——當多個 URL 有相同內容時（如：分頁、UTM 參數），用 canonical 標籤告訴 Google 哪個是「正式版本」，避免重複內容懲罰。
💡
行銷人的重點：技術 SEO 的大部分工作是開發或 IT 的職責。行銷人需要能診斷問題、提出需求，而不是親自修改 server 設定。學會用 GSC 和 PageSpeed Insights 診斷，再交給技術團隊處理。
