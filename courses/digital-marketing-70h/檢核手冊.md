# 數位行銷人才培訓：整體學習路徑檢核手冊

## 1. 手冊目的

本手冊用於檢核 `digital-marketing-70h` 是否能讓一位只有基礎電腦操作能力、沒有生成式 AI 與數位行銷背景的學員，依照課程順序逐步完成能力建構，最後在 AI 輔助下產出一份完整、可說明、可執行、可衡量的行銷企劃案。

本輪檢核的核心不是「每頁有沒有內容」，而是確認：

1. 每項後期任務需要的能力，是否已在前期適時教過並練習。
2. 同一主題分散在不同模組時，是否形成進階關係，而不是重複或錯過學習時機。
3. 每個模組是否留下可帶入後續模組的階段產物。
4. M10 是否真的整合前面累積的成果，而不是到最後才重新教授核心能力。
5. AI 是否是貫穿全程的輔助方法，而不是與數位行銷分離的獨立知識島。

## 2. 檢核範圍與排除範圍

### 2.1 本輪納入

- 課程總覽、大綱、10 個模組總覽及所有單元頁。
- 模組與單元的順序、先備關係、內容銜接與學習負荷。
- 零基礎學員從概念、示範、練習到整合專案的完整路徑。
- 消費者心理、生成式 AI、市場策略、內容與文案、社群、SEO、數據分析、追蹤、廣告、Email 與整合企劃之間的關係。
- 各階段產物能否累積成 M10 的完整行銷企劃案。
- AI 能力是否在後續行銷任務中持續被正確調用、驗證與修正。

### 2.2 本輪排除

- HTML 視覺樣式、SEO meta、密碼關卡與技術合規；這些另由 `docs/lint-page.py` 檢查。
- 單一工具介面的像素級操作正確性。
- 課程時數是否必須固定分配；僅在時數造成能力無法完成時列為證據。
- 文句潤飾與版面美化。

## 3. 核心方法：學習產物依賴鏈

檢核不從 M1 逐頁讀到 M10 後才憑印象下結論，而是先從最終成果反推必要能力，再回到課程順序驗證每項能力的建構時機。

每一項能力都必須形成以下鏈條：

> 首次教學 → 引導練習 → 獨立應用 → 跨模組調用 → 最終企劃整合

若鏈條中斷，即使相關關鍵字曾在某頁出現，也不能視為已完成教學。

### 3.1 關鍵名詞

| 名詞 | 定義 |
|---|---|
| 能力 | 學員可獨立完成且可觀察的行為，例如「依 Persona 與渠道撰寫三種廣告文案」 |
| 階段產物 | 某模組完成後留下、可被後續模組直接使用的成果 |
| 首次教學 | 第一次完整說明概念、方法、使用時機、步驟與品質判準 |
| 再調用 | 後續模組使用已學能力完成新任務，不重新從零講授 |
| 螺旋深化 | 同一能力再次出現時，任務複雜度、情境或自主程度明顯提高 |
| 延遲教學 | 某能力已被前面任務要求使用，卻到後面才正式教授 |
| 假性覆蓋 | 只提到名詞、給 Prompt 或展示結果，沒有讓學員建立可遷移能力 |

## 4. 最終成果的最低完成線

M10 的「完整行銷企劃案」至少應能整合下列成果。正式檢核前，先以 M10 實際要求校正此清單；若 M10 未要求其中某項，需判斷是合理取捨或成果缺口。

| 最終企劃構成 | 學員要能完成的內容 | 主要先備能力 |
|---|---|---|
| 品牌與問題定義 | 品牌現況、商業問題、企劃範圍 | 市場環境分析、需求辨識 |
| 目標受眾 | STP、Persona、需求與痛點 | 消費者心理、受眾研究 |
| 定位與價值主張 | USP、價值主張、核心訊息 | 競品分析、定位、4P/4C |
| 消費者旅程 | 階段、接觸點、障礙、轉換行動 | 旅程設計、渠道認知 |
| 內容與文案策略 | 核心訊息、內容支柱、社群／短影音／SEO／廣告／Email 文案 | 文案基礎、渠道語境、AI 內容產製 |
| 渠道配置 | 自有、自然、付費與合作渠道的角色分工 | 社群、SEO、廣告、Email |
| 執行計畫 | 活動節奏、內容曆、責任與資源 | 專案拆解、內容管理 |
| 預算與媒體配置 | 預算假設、渠道配置與取捨理由 | 付費／自然流量策略 |
| KPI 與衡量 | 目標、指標、事件、UTM、報表與判斷門檻 | GA4、GTM、漏斗與假設驗證 |
| AI 使用紀錄 | 任務選擇、Prompt、輸入資料、人工判斷、修訂與查核 | AI 素養、Prompt、風險與品質驗證 |
| 優化方案 | 根據模擬或數據提出下一輪行動 | 數據判讀、A/B 假設、決策框架 |

判定原則：若 M10 要求一項產物，前面至少要有一次完整教學與一次非 M10 的練習；不能把 M10 同時當成首次教學與成果驗收。

## 5. 六階段檢核流程

### 階段一：建立課程地圖

逐單元建立一列紀錄，不能只採用頁面標題推定內容。

| 欄位 | 紀錄要求 |
|---|---|
| 模組／單元 | 檔名與頁面標題 |
| 學習目標 | 改寫成可觀察行為，不照抄空泛標語 |
| 新教能力 | 本單元首次完整教授的能力 |
| 調用能力 | 本單元假設學員已會的能力 |
| 學習活動 | 看、跟做、變化練習、獨立完成或發表 |
| 階段產物 | 學員實際留下的檔案、表格、設定或決策 |
| AI 角色 | 研究、生成、分析、查核、自動化或未使用 |
| 後續用途 | 哪個單元或 M10 會再次使用此成果 |

完成後先檢查三件事：單元是否只有知識輸入、模組是否沒有可帶走的產物、頁面宣稱的先備是否與實際任務一致。

### 階段二：反推最終企劃依賴

以第 4 節每項最終成果為起點，反向追蹤：

1. M10 在哪個步驟要求使用？
2. 前面哪個單元第一次完整教學？
3. 第一次練習在哪裡？
4. 是否有第二個情境讓學員獨立應用？
5. 中間是否隔太久且沒有再調用？
6. M10 是整合既有成果，還是臨時補課？

每項成果記錄為：

| 最終成果 | 首次教學 | 首次練習 | 再調用 | M10 整合位置 | 判定 |
|---|---|---|---|---|---|
| 例：廣告文案 | 單元 A | 單元 B | 單元 C | m10-x | 完整／斷鏈／延遲／重複 |

### 階段三：檢查概念時序

對每項核心能力標記第一次出現的性質：

- `提及`：只出現名詞或用途。
- `示範`：講師做給學員看。
- `教學`：包含方法、步驟與品質判準。
- `練習`：學員實際產出並可檢查。
- `整合`：與其他能力共同完成較高階任務。

只有「教學 + 練習」才算完成基礎建構。若前段只提及，後段卻直接整合，列為斷鏈；若後段才首次教學，列為延遲教學。

優先追蹤下列高風險主題：

- 行銷文案與核心訊息。
- 市場研究、受眾、Persona、定位與消費者旅程。
- 內容策略、內容曆、短影音腳本與 CTA。
- SEO 搜尋意圖、內容結構與轉換文案。
- 廣告文案、Email 文案及 Landing Page 轉換。
- KPI、漏斗、UTM、事件追蹤、GA4 與 GTM。
- AI Prompt、資料輸入品質、輸出查核、人工修訂與風險控管。

### 階段四：檢查重複與拆分

同一主題出現在不同單元時，不因名稱不同就視為不同能力。逐組比較以下項目：

| 比較面向 | 合理深化 | 問題性重複 |
|---|---|---|
| 任務 | 新渠道或更複雜的行銷問題 | 只是再次生成類似內容 |
| 輸入 | 使用前面累積的品牌、受眾或數據 | 每次重新使用無關範例 |
| 自主度 | 從示範、跟做到獨立決策 | 每次都照抄同類 Prompt |
| 品質判準 | 新增渠道限制、指標或驗證方式 | 只換文案公式名稱 |
| 產物用途 | 直接成為後續企劃的一部分 | 做完後不再使用 |

以下情況需列問題：

1. **錯序拆分**：基礎方法被拆到後期，前期已要求使用。
2. **平行重複**：兩個單元的學習目標、操作與產物實質相同。
3. **知識孤島**：某單元內容完整，但後續從未調用。
4. **名詞改裝**：以不同術語包裝同一流程，沒有增加能力。
5. **產物重做**：後期要求重建前期已完成的成果，卻沒有說明修訂依據。

文案主題要特別建立「文案能力譜系」，至少區分：核心訊息、一般行銷文案、社群貼文、短影音腳本、SEO 內容、廣告文案、Email 文案與 Landing Page 文案。可以分渠道教，但共通基礎必須先完成；後續應教渠道限制與轉換目標，而不是重新教授同一套基礎。

### 階段五：以零基礎學員走讀

每個單元依序提出以下問題：

1. 學員是否知道這一節為什麼現在學？
2. 所有必要術語是否在使用前已解釋？
3. 是否提供可操作的起始資料或情境？
4. 學員是否看過範例，再進入練習？
5. 練習是否有步驟、預期結果與品質判準？
6. AI 產出後，學員是否知道如何判斷好壞與修正？
7. 不靠講師臨場補充，學員是否能完成本節產物？
8. 學員是否知道這份產物要保存並在何處再使用？

零基礎不代表降低專業要求，而是不能省略專業人士習以為常的中間推理。例如：不能從「知道 Persona」直接跳到「完成媒體配置」，也不能從「AI 生成文案」直接跳到「文案可投放」。

### 階段六：檢查 AI 貫穿性

AI 檢核必須同時確認「會用」與「不盲用」。每個行銷模組至少確認：

- 使用 AI 的任務是否適合交給 AI，而非為使用而使用。
- Prompt 是否承接前面產出的品牌、受眾、旅程、渠道或數據。
- 是否要求學員提供足夠背景資料，而不是只套通用模板。
- 是否有人工判斷標準、事實查核與品牌一致性修訂。
- 是否保存 Prompt、版本差異與採納理由。
- AI 產物是否進入模組成果與最終企劃，而非一次性展示。

若後續模組只附一段「請 AI 生成」提示，沒有輸入品質、評估、修訂與應用流程，不算 AI 能力貫穿。

## 6. 起承轉合判準

本課程的起承轉合不是文學式分段，而是能力狀態的轉變。

| 階段 | 應達成的狀態 | 檢核問題 |
|---|---|---|
| 起：建立共同語言 | 知道行銷解決什麼問題，理解消費者決策，能安全使用 AI | 學員是否先建立問題意識，再接觸工具與渠道？ |
| 承：形成策略骨架 | 能研究市場與受眾，完成定位、價值主張與旅程 | 後續內容與渠道決策是否都能回扣此骨架？ |
| 轉：把策略變成執行與測量 | 能依渠道產製內容、取得流量、設計追蹤並判讀數據 | 技術工具是否服務策略，而非成為孤立操作課？ |
| 合：整合、取捨與辯護 | 能整合成果成企劃，說明選擇、資源、KPI 與優化方式 | M10 是否要求真正取捨，而非把前面作業貼在一起？ |

判定整體路徑合格，四段之間必須有清楚交接產物：

- 起 → 承：需求洞察與 AI 使用規範。
- 承 → 轉：受眾、定位、旅程與核心訊息。
- 轉 → 合：渠道內容、追蹤設計、數據判讀與優化假設。

## 7. 模組級檢核表

每個模組完成下表，不以「有／沒有提到」作為唯一答案。

| 檢核項 | 通過條件 | 結果 |
|---|---|---|
| 角色清楚 | 能用一句話說明本模組在完整企劃中的作用 | 通過／部分／不通過 |
| 先備合理 | 使用的能力皆已在前面教學與練習 | 通過／部分／不通過 |
| 新增價值 | 本模組增加新的決策或執行能力 | 通過／部分／不通過 |
| 產物明確 | 學員完成可保存、可檢查的成果 | 通過／部分／不通過 |
| 後續調用 | 至少一個後續單元或 M10 明確使用該產物 | 通過／部分／不通過 |
| AI 有效介入 | AI 任務含輸入、生成、查核、修訂與採用 | 通過／部分／不通過 |
| 難度遞進 | 任務自主度與整合度較前一階段提高 | 通過／部分／不通過 |
| 無實質重複 | 與其他模組的邊界和深化關係清楚 | 通過／部分／不通過 |
| 零基礎可跟上 | 無未教先用、術語跳躍或隱藏步驟 | 通過／部分／不通過 |

## 8. 問題分級

| 等級 | 定義 | 典型情況 |
|---|---|---|
| BLOCKER | 破壞最終成果或零基礎學習路徑，不修不能視為完整課程 | M10 要求的核心能力此前從未教學；關鍵模組缺失；嚴重未教先用 |
| MAJOR | 明顯造成錯序、重複、斷鏈或能力不足 | 文案基礎到後期才教；階段產物沒有後續用途；AI 只有生成沒有判斷 |
| MINOR | 不致使路徑中斷，但降低銜接、理解或練習品質 | 相鄰單元交接不清；產物保存方式未說明；術語出現稍早 |
| NIT | 小幅改善即可提升一致性 | 標題無法反映實際能力；同一概念命名不一致但不影響理解 |

嚴重度依「對學習路徑的影響」判定，不依問題出現次數判定。相同缺口跨多頁重複發生時，應合併成一條根因問題並列出所有受影響位置。

## 9. Issue 寫法

每條問題必須包含：

```markdown
### [MAJOR｜時序] 文案基礎晚於首次應用

- 位置：前置使用單元、正式教學單元、M10 整合單元
- 證據：各位置要求學員完成的具體任務或原文摘要
- 問題：哪項能力被未教先用、重複教授或延遲教授
- 學習影響：零基礎學員會在哪一步卡住，或形成何種錯誤理解
- 根因：內容錯序、能力定義不清、模組邊界重疊或缺少練習
- 建議：前移、合併、改為深化、補橋接練習或刪除重複
- 驗收：修正後應出現的教學、練習、調用與整合證據
```

禁止只寫「內容不夠深入」「順序需要調整」「建議加強文案」。建議必須指出要移動或改變哪一項能力，以及修正後如何驗收。

## 10. 審核交付物

正式審核應產出四份內容，可整合在同一份報告中：

1. **課程能力地圖**：所有單元的新教能力、調用能力與階段產物。
2. **最終企劃依賴矩陣**：M10 每項成果對應的首次教學、練習與再調用位置。
3. **跨模組問題清單**：依 BLOCKER、MAJOR、MINOR、NIT 分級。
4. **建議後的學習路徑**：只在發現問題時提出調序、合併或橋接方案，並說明改動影響。

報告最後必須回答以下總結題：

- 零基礎學員能否依現有順序完成課程？
- 是否有核心能力未教先用或後期才補教？
- 是否有同一能力被拆成多塊卻沒有形成深化？
- 每個模組是否產生可累積的企劃素材？
- M10 是真正整合，還是集中補課與重做？
- AI 是否在研究、策略、產製、衡量與優化中形成一致工作方法？
- 目前課程是否可放行：`PASS`、`CONDITIONAL PASS` 或 `BLOCK`？

## 11. 放行標準

### PASS

- 無 BLOCKER。
- 核心企劃能力皆具備「教學 → 練習 → 再調用 → 整合」證據。
- MAJOR 不影響零基礎學員完成最終企劃，且有明確修補安排。

### CONDITIONAL PASS

- 無 BLOCKER，但存在少量 MAJOR。
- 學員仍能完成企劃，但部分主題有重複、橋接不足或 AI 查核流程不完整。
- 必須列出修正優先順序與驗收方式。

### BLOCK

- 任一最終核心成果缺乏前置教學或練習。
- 多處未教先用，使零基礎學員無法依順序完成。
- M10 承擔大量首次教學，無法視為整合驗收。
- 模組間存在重大錯序或重複，導致完整學習路徑不成立。

## 12. 執行紀律

1. 先記錄證據，再下判斷；不可只根據標題或大綱推定。
2. 同一檔案在同一檢核階段只讀一次，將證據集中記錄。
3. 不以「講師上課可以補充」作為通過理由；頁面應承載必要學習路徑。
