單元 7-3

自訂事件追蹤：按鈕、表單、捲動、連結

增強型評估涵蓋不了所有行為。當你需要追蹤「特定按鈕點擊」「表單成功送出」「50% 捲動深度」這些自訂行為時，就需要在 GTM 手動建立觸發條件和事件代碼。

學習目標 OUTCOMES
能在 GTM 中建立按鈕點擊追蹤（透過 Click ID 或 Click Text 觸發）
能設定自訂捲動深度追蹤（25%/50%/75%/100%）
能建立事件命名規範，確保 GA4 報告中事件名稱一致且可讀
CUSTOM EVENTS · 四種常見自訂追蹤

GTM 實作，四種最常用的自訂追蹤

01
按鈕點擊追蹤
觸發條件：所有元素點擊 + 篩選條件
GTM 設定步驟
  • 啟用內建變數：Click ID、Click Classes、Click Text
  • 新增觸發條件 → 點擊：所有元素
  • 加入篩選條件：Click ID 等於 cta-button（或用 Click Text 等於按鈕文字）
  • 建立 GA4 事件代碼，事件名稱填 button_click
事件參數建議
  • button_text：{{Click Text}}（記錄按了哪顆按鈕）
  • button_location：手動填入（如 hero_section）
  • 在 GA4 自訂維度中新增 button_text，才能在報告看到
02
表單送出追蹤
觸發條件：表單提交（Form Submission）
GTM 設定步驟
  • 啟用內建變數：Form ID、Form Classes
  • 新增觸發條件 → 表單提交
  • 勾選「等待表單提交」確保送出後才觸發
  • 加入篩選：Form ID 等於你的表單 ID
常見問題
  • 若表單用 AJAX 送出，GTM 的表單觸發不會觸發，需用「感謝頁面 URL」追蹤
  • 感謝頁面法：觸發條件 = 頁面瀏覽，URL 包含 /thank-you
03
自訂捲動深度追蹤
觸發條件：捲動深度
GTM 設定步驟
  • 新增觸發條件 → 捲動深度
  • 勾選「垂直捲動深度」
  • 設定觸發時間點：25、50、75、100（百分比）
  • GA4 事件名稱：scroll_depth
事件參數建議
  • scroll_depth_threshold：{{Scroll Depth Threshold}}（25/50/75/100）
  • 比對不同百分比的觸發次數，找出讀者的放棄點
04
外部連結點擊追蹤
觸發條件：連結點擊 + 外部網域篩選
GTM 設定步驟
  • 啟用內建變數：Click URL
  • 新增觸發條件 → 點擊：僅連結
  • 篩選條件：Click URL 不包含 yourdomain.com
  • GA4 事件名稱：outbound_link_click
應用場景
  • 追蹤用戶點擊合作夥伴連結
  • 追蹤社群媒體連結點擊
  • 事件參數 link_url：{{Click URL}} 記錄去哪了
NAMING CONVENTION · 事件命名規範

好的命名，讓報告自己說話

GA4 的事件名稱會直接出現在報告中。如果命名不一致，同一個按鈕點擊可能有三種不同名稱，讓報告難以解讀。以下是標準命名規範：

📋
命名規則：全小寫、用底線分隔、動詞_名詞格式。例：button_click、form_submit、video_start、file_download、scroll_depth
❌ CTA Button Click
✓ button_click
❌ Form Submitted
✓ form_submit
❌ PDF download clicked
✓ file_download
❌ 電話點擊
✓ phone_click
