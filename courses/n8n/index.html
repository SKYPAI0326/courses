Make 真做不到的事 讀取本機檔案系統

Make 無法主動監控你電腦裡的資料夾、讀取本機 PDF 或寫入本地文件。在本機跑的 n8n 可以——這是兩者最關鍵的分界線。M3「重型文件工廠」整模專攻這個能力。

資料留本機的可選擇性 可選的本機 AI（local LLM）

機密 NDA、合約、客戶資料不想送 Gemini？課程教你用 Ollama / llama.cpp 跑 local LLM，或設計「去識別化摘要送雲端」變體。注意：用 Gemini API 等雲端 LLM 時，資料仍會離開本機。

該選哪個方案？
5 選 1 決策表
不是所有人都該走 self-host。誠實列出 5 種路線的取捨，讓你選對自己的位置。
方案 月成本 技術責任 本機檔案？ 適合誰
Make Cloud (Free / Pro) 0 / $9 / 10K ops SaaS 全包 ❌ 不行 純雲端流程、不碰本機檔案
n8n Cloud $20 起 / 月 SaaS 全包 ❌ 不行（官方明示） 想要 n8n 但不想自架
n8n Local（本課主路線） 電費 ~$2-5 + 維運時間 中（要會 Docker、會看 log） ✅ 完整支援 有本機檔案/內網需求、願意維運的進階使用者
n8n VPS（Hetzner / Linode） $5-15 / 月 中（VPS + Docker） ⚠️ 是 VPS 的「本機」 不想 24/7 開機，要穩定 server
Python cron + Gmail/Gemini API 電費 / VPS 高（會寫 Python） ✅ 完整支援 已有 Code 能力、想要最大彈性
本課定位：n8n Local 路線。如果你不確定是不是該選自架，請看 M4-2 「Make + n8n 雙引擎協作」與 M4-4 「AI 秘書遷移」的真實 TCO 對比示範——最後一次決定要不要轉。
核心工具
3 個
⚙️
n8n（本機 Docker）
本機執行 · 可讀寫本機檔案

用 Docker Desktop + Compose 在自己電腦跑的視覺化自動化中樞，Community Edition 自架不依執行次數收費（但你要承擔電費、維運、備份的隱性成本）。能直接讀寫本機資料夾與檔案，是 Make 無法取代的本機執行引擎。

🚇
Cloudflare Tunnel
雲地橋接 · 免月費

讓 Make、Google Forms 等雲端服務能夠「找到」你電腦裡的 n8n 並傳送任務。Cloudflare Tunnel 在免費方案可用（依 Cloudflare 當前規則），不需申請固定 IP 或設定路由器，是雲地協作的關鍵橋樑。

✨
Google AI Studio
多模態辨識

提供最強大的免費多模態辨識大腦，連圖片檔也能讀。透過 Gemini API，為自動化流程注入 AI 理解與生成能力。
