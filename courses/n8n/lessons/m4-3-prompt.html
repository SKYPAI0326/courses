Prompt Engineering 完整指南

在 n8n 自動化流程中，Prompt 的品質決定 AI 輸出的穩定性。本頁整理讓 Gemini 穩定輸出可解析 JSON 的六大技巧，以及企劃書、郵件、摘要三種任務的完整 Prompt 範本。

📖補充參考
⏱閱讀約 12 分鐘
穩定 JSON 輸出的六大原則
01
明確聲明輸出格式
Prompt 開頭直接說「你必須以 JSON 格式回應，不要包含任何其他文字」。不要含糊。
02
提供 JSON Schema 範例
直接給一個完整的範例 JSON，包含每個欄位的鍵名和值的型別（字串、數字、陣列）。
03
規定不得省略欄位
明確說「即使沒有資訊也必須包含這個欄位，用空字串 "" 填充」，避免 JSON 結構不完整。
04
禁止 Markdown 包裝
說「不要用 ``` 或 ```json 包裝，直接輸出純 JSON 文字」。否則 JSON.parse() 會失敗。
05
指定文字長度範圍
對每個文字欄位說明預期長度（如「50 字以內」），控制輸出成本並提升品質。
06
用繁體中文提問
如果輸出需要中文，用中文提問。Gemini 在中文 Prompt 下輸出中文更穩定。
企劃書生成 Prompt 範本

用於 4.3 主流程的 AI 企劃書生成，動態變數用 n8n Expression 填入：

企劃書 PROMPT TEMPLATE
你是跑過 30+ 場活動的企劃執行，熟悉贊助提案與預算估算。請根據以下資訊生成一份完整的活動企劃書。 活動名稱：{{ $json.taskName }} 活動類型：{{ $json.taskType }} 截止日期：{{ $json.deadline }} 備註：{{ $json.notes || '無' }} 你必須以純 JSON 格式回應，不要包含任何說明文字，不要用 ``` 包裝。 JSON 結構如下（所有欄位必須存在，不可省略）： { "title": "企劃書標題（20字以內）", "summary": "活動簡介（100字以內）", "objectives": ["目標1", "目標2", "目標3"], "schedule": [ {"phase": "階段名稱", "duration": "時間長度", "tasks": "主要工作"}, {"phase": "階段名稱", "duration": "時間長度", "tasks": "主要工作"} ], "budget_estimate": "預算估算說明（50字以內）", "expected_outcome": "預期成效（80字以內）", "risks": ["風險1", "風險2"], "marketing_tagline": "行銷標語（15字以內）" }
n8n 中安全解析 Gemini 回應

即使 Prompt 已限制輸出格式，仍要在 Code 節點加入防禦性解析，避免偶發的格式錯誤讓整條流程崩潰：

// Code 節點：安全解析 Gemini JSON 回應 const raw = $input.item.json; // 取出 Gemini 回應文字（安全讀取） const text = raw.candidates?.[0]?.content?.parts?.[0]?.text || ''; // 嘗試清理並解析 JSON let plan; try { // 移除可能殘留的 markdown 包裝 const cleaned = text .replace(/^```json\s*/i, '') .replace(/^```\s*/i, '') .replace(/\s*```$/, '') .trim(); plan = JSON.parse(cleaned); } catch (e) { // 解析失敗：回傳錯誤，不讓流程繼續 return [{ json: { success: false, error: 'JSON 解析失敗', rawText: text.slice(0, 200) // 截取前 200 字方便 debug } }]; } // 解析成功 return [{ json: { success: true, plan, taskName: raw.taskName } }];
其他常用 Prompt 範本
任務系統角色關鍵 Prompt 元素
郵件摘要 「你是熟悉商務 email 語境的文書助理」 輸入郵件全文，要求輸出 JSON：{"subject_summary", "key_actions", "urgency_level", "reply_needed"}
會議記錄整理 「你是熟悉 OKR 與行動項目追蹤的會議記錄秘書」 輸入會議逐字稿，要求輸出 JSON：{"participants", "decisions", "action_items", "next_meeting"}
行銷文案生成 「你是一位熟悉台灣市場的行銷顧問」 輸入產品資訊，輸出 JSON：{"headline", "body_copy", "cta_text", "hashtags"}
PDF 文件摘要 「你是一位文件分析師」 輸入 PDF base64，要求輸出 JSON：{"document_type", "key_points", "dates", "parties_involved"}
✅ 測試 Prompt 的最快方法：先在 Google AI Studio 的 Prompt 編輯器測試，確認輸出穩定後，再把 Prompt 複製到 n8n 的 HTTP Request Body 中。不要直接在 n8n 裡摸索——AI Studio 的即時反饋比 n8n 的執行日誌快很多。
⚠️ 溫度參數 (temperature)：企劃書等創意任務可設 0.7–1.0；JSON 結構化輸出建議設 0.2–0.4，讓輸出更穩定可預測，減少格式錯誤率。在 HTTP Request Body 加上 "generationConfig": {"temperature": 0.3}。
💼 實戰案例
情境｜某活動公司用同一份 Prompt Template 產出 20 種活動類型的企劃——只換 {{taskType}} 變數。
做法｜Template 放 n8n Set 節點當共用變數，改一處全流程更新，不用改 HTTP Request 裡的 Body。
踩雷｜Prompt 超過 1,000 字時 Gemini 輸出品質反而下降——重要指令放最後一段（『最後強調：以 JSON 格式回應』），AI 會更聽話。
