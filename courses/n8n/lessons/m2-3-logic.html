Module 02
SECTION 2.3

邏輯判斷節點（If / Switch）

讓流程自己做決定：這份合約該存檔？還是通知主管審核？根據資料內容自動分流到不同的處理路徑。

⏱預估時間：25 分鐘
📋4 個步驟
✅前置：完成 2.2
1
新增 If 節點：設定第一個判斷條件

在流程中點擊「+」搜尋並新增 If 節點。在「Condition」欄位切換到 Expression 模式，輸入你的判斷條件。條件成立走 True 分支，不成立走 False 分支。

// 範例：判斷合約金額是否超過 10 萬 // Condition 欄位填入（切換到 Expression 模式）： {{ $json.合約金額 > 100000 }} // 結果： True → 合約金額 > 10萬，走主管審核流程 False → 合約金額 ≤ 10萬，走一般存檔流程
🔀
If 節點設定介面：Condition 填入判斷 Expression，節點右側出現 True / False 兩個連接點
🔄 對應 Make 的概念：就像 Make 的 Router + Filter 組合——If 節點的 True/False 分支等同於 Router 的兩條路徑，各自加條件過濾。
2
連接 True / False 兩條分支到不同節點

If 節點右側有兩個連接點，分別標示 true 和 false。從 true 拉線連接「通知主管」節點，從 false 拉線連接「直接存檔」節點。兩條路徑各自獨立執行，互不干擾。

TRUE 分支（金額 > 10萬）
→ 發送 Email 通知主管
→ 在 Google Sheets 標記為「待審核」
If 節點
↙ ↘
FALSE 分支（金額 ≤ 10萬）
→ 直接存入 Google Drive
→ 更新 Sheets 為「已存檔」
3
Switch 節點：一次分出多條路徑

If 只能分兩條路，Switch 可以分出 3 條、5 條甚至更多。搜尋新增 Switch 節點，在「Value」欄位填入你要判斷的欄位，然後為每個可能的值新增一個 Case，各自連接對應的後續節點。最後還有一個 Default 分支，處理不符合任何 Case 的情況。

// Switch 節點設定範例：依合約類型分流 Value（要判斷的欄位）： {{ $json.合約類型 }} Case 1： "採購合約" → 連接採購部門通知節點 Case 2： "服務合約" → 連接服務部門通知節點 Case 3： "租賃合約" → 連接法務部門通知節點 Default： → 連接「人工處理」通知節點
🔀
Switch 節點：Value 填入判斷欄位，多個 Case 各有獨立出口，Default 處理未定義情況
4
多重條件：AND 與 OR 組合判斷

If 節點支援多個條件同時判斷。點擊「Add condition」可以加入多個條件，並選擇 AND（全部成立才走 True）或 OR（任一成立就走 True）的邏輯組合。

// AND：金額超過10萬，「且」客戶等級是 VIP // → 兩個條件都要成立，才走 True 分支 條件 1：{{ $json.合約金額 }} 大於 100000 條件 2：{{ $json.客戶等級 }} 等於 "VIP" 邏輯：AND // OR：合約狀態是「緊急」，「或」金額超過50萬 // → 任一條件成立，就走 True 分支 條件 1：{{ $json.優先級 }} 等於 "緊急" 條件 2：{{ $json.合約金額 }} 大於 500000 邏輯：OR
✅ n8n 的 If 節點提供視覺化的條件設定介面，不需要手寫 && 或 || 語法，直接用下拉選單選擇 AND / OR，初學者也能輕鬆設定複雜條件。
完成驗證 CHECKPOINT
測試資料能正確分流到對應路徑

送入三筆不同金額的合約資料（如 5萬、15萬、80萬），確認 If 節點依照條件正確分流。再送入不同合約類型的資料，確認 Switch 節點各 Case 都能觸發到對應後續節點，Default 也能正確處理未定義類型。

常見問題
資料全部走 False，沒有走 True
最常見原因：欄位資料型別不符。例如條件設定 金額 > 100000，但欄位實際值是字串 "100000" 而非數字。先用 Set 節點或 parseInt() 轉換型別，或改用「String contains」類型的條件做字串比較。
Switch 節點所有資料都走 Default
確認 Case 的值與資料欄位的值完全吻合，包含大小寫與前後空格。例如資料是 "採購合約 "（有尾部空格），但 Case 設定是 "採購合約"（無空格），就不會匹配。可用 {{ $json.合約類型.trim() }} 先清除空白。
💼 實戰案例
情境｜某客服系統根據訊息內含『緊急』『投訴』『價錢』分流到不同團隊。
做法｜用 IF 節點 + 複合條件 AND/OR 搭配正則，平均回覆時間從 2 小時 → 25 分鐘。
踩雷｜n8n 的 IF 節點一次只能判斷一組條件——多條件用 AND/OR combinator。超過 5 個 OR 可讀性差，改用 Switch 節點或 Code 節點寫 switch case。
