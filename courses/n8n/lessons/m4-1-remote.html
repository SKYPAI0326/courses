REMOTE CONTROL · 4.1

Google Forms 遠端遙控 n8n

填表單 = 下指令。用手機提交 Google Forms，Make 接力轉送，本機 n8n 立刻啟動對應的文件處理任務。

完整資料流向 · FLOW
Google Forms
→
Make.com Webhook
→
Make HTTP Request
→
Cloudflare Tunnel
→
n8n Webhook
→
If / Switch 判斷
STEP BY STEP · 操作步驟
1
建立 Google Forms 表單（遙控介面）
開啟 Google Forms 建立一個新表單。加入以下三個欄位：申請人（簡短答案）、文件類型（選擇題：報告 / 企劃 / 合約）、截止日期（日期）。所有欄位設為必填，確保資料完整。
提示：表單標題建議取名「文件處理申請」，讓填表人清楚知道提交後的效果。完成後複製表單連結，之後可直接用手機開啟填寫。
2
Make.com：建立 Webhook 接收表單資料
在 Make.com 建立新 Scenario，第一個模組選「Webhooks → Custom webhook」。點擊「Add」建立新 Webhook，複製產生的 Webhook URL。Make 會進入 Listening 狀態等待第一筆測試資料。
提示：先不要關閉 Make Scenario，保持 Listening 狀態。接下來設定 Google Forms → Make 的連接，送出測試資料後 Make 會自動識別欄位結構。
3
連接 Google Forms → Make（用 Apps Script）
在 Google Forms 點擊右上角「⋮」→「Apps Script」，貼入以下程式碼，將 Make 的 Webhook URL 填入。儲存後設定觸發條件：「表單提交時」執行 onSubmit 函數。
// 替換為你的 Make Webhook URL const MAKE_WEBHOOK = 'https://hook.make.com/YOUR_WEBHOOK_ID'; function onSubmit(e) { const vals = e.namedValues; const payload = { applicant: vals['申請人'][0], document_type: vals['文件類型'][0], due_date: vals['截止日期'][0] }; UrlFetchApp.fetch(MAKE_WEBHOOK, { method: 'post', contentType: 'application/json', payload: JSON.stringify(payload) }); }
提示：設定觸發條件：Apps Script 左側「觸發條件」→「+ 新增觸發條件」→ 函數選 onSubmit，觸發事件選「表單提交時」。
4
n8n：設定 Webhook 觸發節點
在 n8n 建立新 Workflow，新增「Webhook」觸發節點。Method 設為 POST，Path 可自訂（如 form-request）。n8n 會產生一個本機 URL（localhost:5678/webhook/form-request），透過 Cloudflare Tunnel 對外暴露。
// n8n Webhook 本機 URL http://localhost:5678/webhook/form-request // 透過 Cloudflare Tunnel 的對外 URL（Module 1 設定的） https://YOUR-TUNNEL-NAME.trycloudflare.com/webhook/form-request
提示：確認 Cloudflare Tunnel 仍在運行（命令列視窗還開著）。若 Tunnel 重新啟動，URL 會改變，需更新 Make 的 HTTP Request 目標。
5
Make：新增 HTTP Request 轉發資料到 n8n
在 Make Scenario 的 Webhook 之後，新增「HTTP → Make a request」模組。Method 選 POST，URL 填入 n8n 透過 Cloudflare Tunnel 的外部 URL。Header 加入 Content-Type: application/json，Body 把 Webhook 收到的欄位轉送。
// Make HTTP Request Body（Raw JSON） { "applicant": "{{1.applicant}}", "document_type": "{{1.document_type}}", "due_date": "{{1.due_date}}" }
提示：Body Type 選「Raw」→ Content Type 選「JSON (application/json)」，才能讓 n8n 正確解析傳入的資料。
6
n8n：用 If/Switch 節點根據文件類型分流
在 n8n Webhook 節點後加入「Switch」節點，設定以 {{ $json.body.document_type }} 作為判斷值，建立三個 Case（報告 / 企劃 / 合約），每個 Case 連接對應的處理流程（可接 Module 3 學過的任何節點）。
// Switch 節點設定 Value: {{ $json.body.document_type }} Case 1: 報告 → 連接報告產生流程 Case 2: 企劃 → 連接企劃產生流程 Case 3: 合約 → 連接合約產生流程
提示：提交表單後，先點 n8n Webhook 節點的「Test URL」，讓節點進入接收狀態。Make 送出資料後，n8n 會顯示收到的完整 JSON，確認結構正確再繼續。
完成驗證 · VERIFY
用手機填寫 Google Forms（選擇任一文件類型）並提交。Make Scenario 的執行記錄顯示「Success」，n8n Webhook 節點有收到資料，Switch 節點正確進入對應的分支。若以上三點均成功，代表遠端遙控鏈路已全線打通。
常見問題 · TROUBLESHOOT
Make 收到 Google Forms 資料但 HTTP 轉送失敗（n8n 沒收到）？
確認 Cloudflare Tunnel 仍在運行。若 Tunnel URL 失效，重新啟動 cloudflared 並更新 Make HTTP Request 的目標 URL。另外確認 n8n Workflow 已切換到 Active 狀態。
n8n 收到資料但欄位路徑不對（取不到 applicant）？
Make 送來的資料會包在 $json.body 下，因此完整路徑是 {{ $json.body.applicant }}。在 n8n Webhook 的 Output 面板確認實際 JSON 結構再調整引用路徑。
Apps Script 執行時顯示「未經授權」錯誤？
第一次執行時 Google 會要求授權。在 Apps Script 點「Run」→ 在彈出視窗同意授權。之後設定好觸發條件，就不需要再手動執行。
💼 實戰案例
情境｜某工作室老闆出國仍能透過 LINE 指令觸發本機 n8n 處理客戶訂單。
做法｜LINE Bot → Webhook → Cloudflare Tunnel → n8n Desktop → 處理完 LINE 回覆。完全免費。
踩雷｜LINE Bot Messaging API 免費版每月 500 則主動推播上限；被動回覆無限。設計上別讓 bot 主動推播太多，用『收到才回』的互動式設計。
