AUTOMATED MONITORING · 3.1

Watch Folder 節點：讓資料夾成為自動化的觸發器

Make 做不到的本機觸發，n8n 輕鬆搞定。只要把檔案丟進資料夾，流程自動啟動。

為什麼這很重要 · WHY THIS MATTERS
Make.com 只能被動等待雲端訊號（如 Gmail 收信、Google Forms 提交），無法監控你電腦裡的資料夾。n8n 的 Watch Folder 節點可以——新檔案一落地，流程立刻觸發，完全不需人工介入。這是 n8n Desktop 最關鍵的本機優勢。
STEP BY STEP · 操作步驟
1
新增 Watch Folder 節點並配置監控路徑
在 n8n 工作流程中，點擊「+」搜尋並新增「Watch Folder」節點。在「Path to Watch」欄位填入你想監控的資料夾完整路徑，設定「Polling Interval」為 5 秒，「Events」選擇「Created」（新檔案建立時觸發）。
Mac: /Users/你的名稱/Documents/待處理PDF Windows: C:\Users\你的名稱\Documents\待處理PDF
提示：確保監控資料夾已存在，且 n8n 有讀取該資料夾的權限，否則觸發會靜默失敗。
2
啟用流程並測試觸發
儲存工作流程後，點擊右上角切換鍵將流程設為「Active」（啟用狀態）。然後手動將一個任意檔案（例如 test.txt）拖曳放入監控資料夾，觀察 n8n 介面是否顯示執行記錄。
提示：每次測試請使用不同的檔名，已觸發過的檔案名稱 n8n 會記住，重複放入不會再次觸發。
3
檢視觸發資料，確認輸出結構
點擊 Watch Folder 節點查看執行結果。在 Output 面板中，找到觸發事件的 JSON 資料，定位到 data.name（檔名）和 data.path（完整路徑）兩個關鍵欄位，後續步驟都依賴這兩個值。
{{ $json.data.name }} // 檔名，例如：report.pdf {{ $json.data.path }} // 完整路徑，例如：/Users/.../report.pdf
提示：這兩個欄位是後續所有節點操作檔案的關鍵，務必確認能正確讀到。
4
加入 If 節點，篩選只處理 PDF 檔案
在 Watch Folder 後面連接一個「If」節點。設定條件：Value 1 填入 {{ $json.data.name.endsWith('.pdf') }}，Operation 選「Is True」。這樣只有 PDF 檔案才會進入 True 分支繼續處理。
// If 節點條件設定 Value 1: {{ $json.data.name.endsWith('.pdf') }} Operation: Is True
提示：使用 endsWith() 是判斷副檔名最簡潔有效的方式，注意大小寫（PDF 與 pdf 不同）。
5
測試 If 節點過濾效果
先放入一個 document.txt，確認流程在 If 節點停止（走 False 分支，後面沒有連接任何節點即停止）。再放入一個 report.pdf，確認流程通過 True 分支正常繼續執行。
提示：同時確認 If 節點的 False 分支可連接通知節點（如 Send Email），以便知道有非 PDF 檔案被放入。
完成驗證 · VERIFY
放入 .pdf 檔案時，流程能順利通過 If 節點的 True 分支並有執行記錄；放入 .txt 或其他類型檔案時，流程在 If 節點停止，不繼續執行後續步驟。兩種情況下執行記錄顯示正常（無紅色錯誤）。
常見問題 · TROUBLESHOOT
Watch Folder 沒有觸發，放入檔案沒有反應？
請先確認流程已切換到「Active」啟用狀態（非 Inactive）。再檢查設定的路徑是否正確（注意 Mac 路徑用 /，Windows 用 \）。可以先點「Test Workflow」強制執行一次，再重新放入檔案測試。
If 節點沒有篩選出 PDF，所有檔案都走 True 分支？
請確認 If 節點的 Value 1 表達式輸入正確，且選用 Is True 而非 Equal。另外注意副檔名大小寫：如果你的 PDF 是大寫 .PDF，改用 $json.data.name.toLowerCase().endsWith('.pdf')。
