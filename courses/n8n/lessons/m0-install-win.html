M0 · QUICKSTART · WINDOWS

從零在 Windows 安裝 Lite Pack：60-120 分鐘到 14 個 workflow 全部就緒

這是「先用後學」路線的 Windows 入口頁。Windows 比 Mac 多一層 WSL2（Windows Subsystem for Linux 2）— Docker Desktop 在 Windows 跑容器需要 WSL2 當底層，本頁會帶你裝完。建議 Windows 10 21H2 以上 / Windows 11 任何版本，Home 版以上即可（不用 Pro）。完成後跟 Mac 學員拿到一樣的 14 個 workflow 在你電腦上跑著。

學習成果
能在 Windows 10/11 從零安裝 WSL2 + Docker Desktop + n8n 並完成首次設定
能用 setup-wizard.bat 自動填入 Gemini API key + Telegram bot 完成 Lite Pack 一鍵安裝
能在自己電腦上跑通 5 個招牌 demo workflow 並驗證輸出（剩 9 個 workflow 全裝好可隨時用）
(00) STEP ZERO · 重灌學員必做｜新手跳過

先把 Windows 上既有的 n8n / Docker 殘留清乾淨

為什麼這很重要
如果你以前裝過 n8n（自己摸索、跟別的教學、舊版 Desktop App），這次重灌會撞 port、撞 volume、撞 cloudflared process — 新跑的 n8n 啟動失敗或行為怪異 90% 是這個原因。把舊的全清掉再從新版下手，比邊裝邊修快得多。
🆕
完全沒裝過 Docker / WSL2 / n8n 的新手？Step 0 內所有指令都需要 Docker + WSL2（如 docker info / docker rm）。請直接跳到 Phase A 從零裝，裝完不用回來做 Step 0（你電腦本來就乾淨無殘留）。Step 0 是「以前裝過 n8n / 跟過別的教學留下殘留」的學員才需要做的歸零步驟。
⚠️
會清掉：所有 n8n / postgres container + volume + network、cloudflared 殘留 process、所有 starter-kit / lite-pack 解壓資料夾、所有殘留 zip。
不會動到：Docker Desktop 本身、Docker image cache（重灌會直接拿 cache，不重新下載 GB 級 image）、WSL2 本身、Telegram bot、Gemini API key、Cloudflare 帳號（這些都在雲端）。
STEP 0-1
打開 PowerShell（系統管理員權限）

左下角搜尋「PowerShell」→ 右鍵 →「以系統管理員身分執行」。後面所有指令都在這個視窗跑。

💡
不用系統管理員的 PowerShell 也能跑大部分指令，但 cloudflared service 卸載必須要管理員權限。一次開好省事。
STEP 0-2
確認 Docker Desktop 在跑

桌面右下角工具列應有鯨魚 icon（不再動畫表示就緒）。沒安裝跳到 Phase A 先裝。

docker info | Out-Null
if ($?) { Write-Host "✅ Docker daemon 正常" -ForegroundColor Green } else { Write-Host "❌ 先打開 Docker Desktop" -ForegroundColor Red }
STEP 0-3
清掉所有 n8n / postgres container + volume + network
# 用 docker compose down -v 走正規路徑（如果 starter-kit 還在）
$kit = "$env:USERPROFILE\Downloads\n8n-starter-kit"
if (Test-Path "$kit\n8n-compose.yml") {
  cd $kit
  docker compose -f n8n-compose.yml down -v 2>$null
  cd $env:USERPROFILE
}

# 殘留 container 強制清
docker ps -a --filter "name=n8n" --format "{{.Names}}" | ForEach-Object { docker rm -f $_ }
docker ps -a --filter "name=postgres" --format "{{.Names}}" | ForEach-Object { docker rm -f $_ }

# 殘留 volume 清
docker volume ls --filter "name=n8n" -q | ForEach-Object { docker volume rm $_ }
docker volume ls --filter "name=postgres" -q | ForEach-Object { docker volume rm $_ }

# 殘留 network 清
docker network ls --filter "name=n8n" -q | ForEach-Object { docker network rm $_ 2>$null }
STEP 0-4
殺掉殘留的 cloudflared process
# 之前雙擊 tunnel-quick.bat 開的 cloudflared 可能還在跑
Get-Process cloudflared -ErrorAction SilentlyContinue | Stop-Process -Force
Start-Sleep -Seconds 2
$still = Get-Process cloudflared -ErrorAction SilentlyContinue
if ($still) { Write-Host "⚠️ 還有殘留 cloudflared，看下方 service 卸載" -ForegroundColor Yellow }
else { Write-Host "✅ cloudflared 已清乾淨" -ForegroundColor Green }
⚠️
如果以前裝過 cloudflared service install 的 Windows service，需要卸載： 
cloudflared service uninstall 2>$null
sc.exe delete cloudflared 2>$null
Get-Service cloudflared -ErrorAction SilentlyContinue

最後一行如果回 Cannot find any service 表示已卸載乾淨。

STEP 0-5
刪所有解壓資料夾 + 殘留 zip
# 含 Windows 防覆蓋產生的 starter-kit (1) / (2)
Remove-Item "$env:USERPROFILE\Downloads\n8n-starter-kit*" -Recurse -Force -ErrorAction SilentlyContinue
Remove-Item "$env:USERPROFILE\Desktop\n8n-starter-kit*" -Recurse -Force -ErrorAction SilentlyContinue
Remove-Item "$env:USERPROFILE\Downloads\n8n-lite-pack*" -Recurse -Force -ErrorAction SilentlyContinue
Remove-Item "$env:USERPROFILE\Desktop\n8n-lite-pack*" -Recurse -Force -ErrorAction SilentlyContinue
Remove-Item "$env:USERPROFILE\Downloads\n8n-starter-kit*.zip" -Force -ErrorAction SilentlyContinue
Remove-Item "$env:USERPROFILE\Downloads\n8n-lite-pack*.zip" -Force -ErrorAction SilentlyContinue
Write-Host "✅ 解壓資料夾與 zip 已清" -ForegroundColor Green
STEP 0-6
完全歸零驗證
Write-Host "── container ──"
docker ps -a --filter "name=n8n" --format "{{.Names}}"
docker ps -a --filter "name=postgres" --format "{{.Names}}"
Write-Host "── volume ──"
docker volume ls | Select-String "n8n|postgres" || Write-Host "✅ 無殘留 volume"
Write-Host "── network ──"
docker network ls | Select-String "n8n" || Write-Host "✅ 無殘留 network"
Write-Host "── 解壓資料夾 ──"
Get-ChildItem "$env:USERPROFILE\Downloads\" -ErrorAction SilentlyContinue | Where-Object { $_.Name -match "n8n|lite-pack" } || Write-Host "✅ Downloads 已清乾淨"
Write-Host "── cloudflared process ──"
Get-Process cloudflared -ErrorAction SilentlyContinue || Write-Host "✅ cloudflared 已清乾淨"
Write-Host "── 完成 ──"
通過 STEP 0 的判準
5 個區塊都顯示 ✅ 已清乾淨。任何一行有殘留就重跑對應 step 0-3 ~ 0-5 的指令。
(01) PHASE A

WSL2 + Docker Desktop + n8n 啟動（30-60 分鐘）

這個階段要做的事
先確認 / 安裝 WSL2（Docker Desktop 在 Windows 的底層），裝 Docker Desktop，下載 starter-kit zip，雙擊 start.bat 把 n8n 跑起來，建立 Owner Account。完成後瀏覽器打開 localhost:5678 看到 n8n 主介面。
A1
安裝 / 確認 WSL2

系統管理員 PowerShell 跑：

wsl --status
wsl --version

如果回 'wsl' 不是內部或外部命令 或顯示版本 1，跑安裝指令：

wsl --install
wsl --set-default-version 2

跑完重開機（Windows 強制要求）。重開後再回來繼續 A2。

⚠️
如果 wsl --install 跳「主機板設定 BIOS Virtualization 沒開」→ 重開進 BIOS 找 Virtualization Technology / VT-x / SVM 設成 Enabled 再回來。沒這選項代表你 CPU 不支援虛擬化（極舊 CPU），這種情況本課程整體不適用。
A2
安裝 Docker Desktop

到 https://www.docker.com/products/docker-desktop/ 下載 Windows 版（會自動偵測 64-bit）。

下載 Docker Desktop Installer.exe → 雙擊 → 預設選項全打勾（特別注意「Use WSL 2 instead of Hyper-V」要打勾）→ 安裝完重開機。

重開後第一次啟動 Docker Desktop：

  1. 同意服務條款
  2. 跳註冊頁可以跳過（不註冊也能用本課程所有功能）
  3. 等鯨魚 icon 不再動畫表示就緒

驗證安裝：PowerShell 跑 docker --version 應顯示版本號。

A3
下載 n8n-starter-kit.zip
⬇ 下載 n8n-starter-kit.zip（約 15 KB）

下載後在檔案總管 Downloads 找到 zip → 右鍵「全部解壓縮」→ 解壓到同層 C:\Users\<你的名字>\Downloads\n8n-starter-kit\。

⚠️
Windows 解壓預設會多一層資料夾（n8n-starter-kit\n8n-starter-kit\）。需要把內層的東西搬到外層，最終結構應為 Downloads\n8n-starter-kit\start.bat（start.bat 直接在資料夾底下）。
A4
Windows SmartScreen 解鎖 .bat

Windows Defender SmartScreen 對網路下載的 .bat 會擋。處理方法：

  1. 檔案總管進 Downloads\n8n-starter-kit\
  2. 對 start.bat 右鍵 → 內容
  3. 對話框最下方看到「安全性：這個檔案來自其他電腦，可能被封鎖」 → 勾「解除封鎖」 → 確定
  4. 對 stop.bat / update.bat / tunnel-quick.bat 重複
💡
如果右鍵「內容」沒看到「安全性」那欄表示這台 Windows 沒擋（少見），直接跳 A5。
A5
第一次雙擊 start.bat（建 .env）

檔案總管雙擊 start.bat。第一次會自動把 .env.example 複製成 .env 並打開記事本讓你編輯。

⚠️
改密碼：把 POSTGRES_PASSWORD=please_change_me_to_a_strong_password 那行的值改成自己的強密碼（至少 16 字元、混合英數字符），存檔關閉。
🚨
不要再改第二次：Postgres 密碼第一次寫進 volume 之後，之後改 .env 不會自動同步，會造成「password authentication failed」。設好就不要再改。
A6
再次雙擊 start.bat（真的啟動）

會拉 Docker image + 啟動 container。首次需 1-5 分鐘下載 postgres / n8n image。命令提示字元視窗每 10 秒輸出進度。

預期看到（成功）
命令提示字元顯示 ✅ n8n 已啟動：http://localhost:5678，瀏覽器自動開到 n8n 主介面。
⚠️
如果跳「Port 5678 已被佔用」或「postgres 認證失敗」對話框，按對話框內的修補按鈕（停掉並重啟 / 重置並重啟）。
A7
建立 n8n Owner Account

瀏覽器（用 Chrome / Firefox / Edge，不要用 IE）開 http://localhost:5678。

  1. Email：填你的（任意，不會驗證）
  2. 密碼：自定（至少 8 字元含大小寫與數字）
  3. 跳過行銷問卷
Phase A 通過判準
進入 n8n 主介面（左側選單看到 Workflows / Credentials / Executions / Templates）。
(02) PHASE B

Lite Pack 一鍵安裝（10 分鐘）

這個階段要做的事
下載 Lite Pack 試跑包，雙擊 setup-wizard.bat，依對話框提示貼入 3 個 key（Gemini API key / Telegram bot token / Telegram chat ID）。setup-wizard 會跑 10 個步驟自動完成：建 credentials、匯入 14 個 workflow、做 Telegram + Gemini smoke test。
📋
事前準備：開始 setup-wizard 前先把 3 個 key 拿到手放在記事本上。
1. Gemini API key：到 aistudio.google.com/apikey（免費）→ Create API key → 複製 AIza... 開頭那串
2. Telegram bot token：Telegram 找 @BotFather → /newbot → 給名字 → 拿到 123456:ABC-DEF...
3. Telegram chat ID：對你的 bot 發任意訊息 → 瀏覽器開 https://api.telegram.org/bot<你的TOKEN>/getUpdates → 找 chat.id 那串純數字
B1
下載 Lite Pack zip
⬇ 下載 n8n-lite-pack.zip（約 90 KB · v1.2 · Telegram 可選 + Gemini 限速防禦）

下載後右鍵「全部解壓縮」→ 同層產生 Downloads\n8n-lite-pack\。同樣注意內外兩層問題（最終結構應為 Downloads\n8n-lite-pack\setup-wizard.bat）。

B2
SmartScreen 解鎖 setup-wizard.bat
  1. 對 setup-wizard.bat 右鍵 → 內容 → 勾「解除封鎖」 → 確定
  2. 第一次雙擊執行可能還跳「Windows 已保護你的電腦」 → 點「其他資訊」 → 「仍要執行」
B3
雙擊 setup-wizard.bat 執行

會跳 3 個對話框依序問 Gemini key / TG token / TG chat ID。把記事本上 3 個 key 依序貼進去。

setup-wizard 跑 10 個步驟（約 1-3 分鐘）：

  1. 環境檢查（Docker + n8n 在跑 + 找到 starter-kit）
  2. 收集 3 個 key
  3. 偵測 + patch file access 環境變數
  4. 寫 personalization.env
  5. 建 sample 資料夾
  6. 生成 decrypted credentials JSON
  7. 替換 workflow JSON 內 placeholder
  8. n8n CLI 匯入 2 個 credentials
  9. n8n CLI 匯入 14 個 workflows
  10. Telegram + Gemini smoke test
Phase B 通過判準
命令提示字元跳「Lite Pack 安裝完成」對話框 + 自動開瀏覽器到 n8n。
對話框內容應該齊備：✅ 14 workflows ✅ 2 credentials ✅ Telegram 你手機收到「✅ Lite Pack setup-wizard 安裝完成」 ✅ Gemini reply 「OK」
B4
n8n UI 確認 8 workflow + 2 credentials

瀏覽器到 http://localhost:5678 → 左側 Workflows：應該看到 14 個（編號 01 ~ 14）。
左側 Credentials：應該看到 2 個 — Lite Pack · Gemini API + Lite Pack · Telegram Bot。

(03) PHASE C

5 個招牌 demo 驗證（30-50 分鐘）

這個階段要做的事
逐個跑通 5 個 demo workflow（05 Telegram 通知 / 02 PDF AI 改名 / 03 批次處理 / 04 定時日報 / 06 Webhook AI），驗證 Lite Pack 在你 Windows 上完整可用。剩 9 個 workflow 也都裝好了，課程結束去工作流總覽看其他能力。
C1
準備測試素材

最快方式：下載演練素材包（推薦）

⬇ 下載 n8n-sample-pack.zip（約 660 KB · v1.2 · 加 scale-up 5 個放大版資料夾）

下載後 7-Zip / 內建解壓 → 同層產生 C:\Users\<你的名字>\Downloads\n8n-sample-pack\，內含 7 個子資料夾對應 7 個演練 workflow（pdf-inbox / batch-inbox / daily-input / client-inbox / leads-inbox / knowledge-docs / ops-input）。在 PowerShell 跑這條一次複製到位：

$Src = "$env:USERPROFILE\Downloads\n8n-sample-pack"
$Dst = "$env:USERPROFILE\Downloads\n8n-starter-kit\shared"
foreach ($folder in 'pdf-inbox','batch-inbox','daily-input','client-inbox','leads-inbox','knowledge-docs','ops-input') {
  Copy-Item -Path "$Src\$folder\*" -Destination "$Dst\$folder\" -Recurse -Force
}
Write-Host "✓ 7 個子夾素材複製完成"

替代方式：用自己的檔（如果想測自己的真實情境）

檔案總管打開 C:\Users\<你的名字>\Downloads\n8n-starter-kit\shared\，從你電腦各處挑：

  • 1-3 份合約 / 發票 / 說明 PDF 拖進 shared\pdf-inbox\
  • 3-5 份 PDF（含 1 份故意是壞檔） 拖進 shared\batch-inbox\
  • 1-2 份 .txt / .md（會議記錄、待辦） 拖進 shared\daily-input\

建壞檔測試（PowerShell 跑）：

"this is not a real PDF" | Out-File "$env:USERPROFILE\Downloads\n8n-starter-kit\shared\batch-inbox\壞檔測試.pdf" -Encoding UTF8
C2
驗證 #5 Telegram 通知（最簡單先測）
  1. n8n 左側 Workflows → 點開 05 · Telegram 通知
  2. 下方紫色 Execute workflow
  3. 3 個節點全綠
通過判準
你的 Telegram bot 收到「✅ Lite Pack 通知測試成功」訊息。
C3
驗證 #2 PDF AI 改名（招牌 demo）
  1. 確認 shared\pdf-inbox\ 有 1 份 PDF
  2. n8n 進 02 · PDF AI 改名 → Execute workflow
  3. 看 5 個節點全綠
  4. 點 Code: AI 改名節點看 Output：parseMethod 應該是 ai-plain
  5. 檔案總管看 shared\pdf-renamed\
通過判準
新檔名長相 YYYYMMDD_類型_關鍵字.pdf。
C4
驗證 #3 批次處理錯誤恢復
  1. 確認 shared\batch-inbox\ 有 3-5 份 PDF（含 1 份壞檔）
  2. n8n 進 03 · 批次處理錯誤恢復 → Execute workflow
  3. 看 canvas 雙路徑：合法 PDF → processed/，壞檔 → failed/
  4. 檔案總管驗證 shared\batch-inbox\processed\ 與 shared\batch-inbox\failed\
通過判準
processed/ 內合法 PDF 都改名。failed/ 內壞檔 fileName 正確。
C5
驗證 #4 定時 AI 日報
  1. 確認 shared\daily-input\ 有至少 1 份 .md / .txt
  2. n8n 進 04 · 定時 AI 日報 → Execute workflow（測試用，不等到明早 8:00）
  3. 檔案總管看 shared\daily-output\daily-2026-XX-XX.md，記事本打開檢查
通過判準
Markdown 內容含 4 段結構化內容。
C6
驗證 #6 Webhook AI（外部觸發 AI 鏈）
  1. n8n 進 06 · Webhook → Gemini → 本機檔 → Execute workflow（Webhook 變藍色「Listening」）
  2. 新開 PowerShell（不要關原本的）跑：
curl.exe -X POST http://localhost:5678/webhook-test/ai-ask -H "Content-Type: application/json" -d "{\"question\":\"什麼是 n8n？三句話內回答。\"}"
💡
PowerShell 用 curl.exe（不是 curl，後者在 PowerShell 是 Invoke-WebRequest 別名語法不同）。引號內要 escape \"。
通過判準（4 件齊備）
(1) curl 終端回 {"status":"ok",...}
(2) shared\ai-output\ 多 1 份 .md
(3) Telegram 收到通知
(4) Code 節點 Output 的 aiError 為 null
🎯
Lite Pack 全鏈路通過：5 個 workflow（C2-C6）都跑完代表你的 Windows 已具備真實的 n8n + AI 工作站。下一步去 工作流設計拆解學習每個 workflow 怎麼設計、為什麼這樣設計、自己怎麼改造。
(04) TROUBLESHOOT

卡關時看這（Windows 特有）

A1 wsl --install 失敗 / 重開機後仍說 WSL 沒裝
  1. 系統管理員 PowerShell 跑 dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
  2. 跑 dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
  3. 重開機 → 跑 wsl --set-default-version 2
A2 Docker Desktop 啟動失敗 / 鯨魚 icon 一直紅燈
  1. 確認 BIOS Virtualization 開啟（A1 同樣前置）
  2. 確認 Hyper-V 與 WSL2 都啟用（控制台 → 程式集 → 開啟或關閉 Windows 功能 → 勾「Hyper-V」「Virtual Machine Platform」「Windows Subsystem for Linux」）
  3. 重開機後重啟 Docker Desktop
A4 .bat 雙擊閃一下黑視窗就消失

對 .bat 右鍵 →「以系統管理員身分執行」，這次會留住視窗看錯誤訊息。常見：Docker Desktop 沒開、SmartScreen 沒解鎖、解壓資料夾雙層問題。

A6 終端機顯示「docker: command not found」
  1. 確認 Docker Desktop 在跑（鯨魚 icon 不動畫）
  2. 關掉這個 PowerShell 視窗 → 重開（PATH 環境變數要重抓）
  3. 或重開機後重試
localhost:5678 顯示亂碼或一片白
  1. 確認用 Chrome / Firefox / Edge（IE 不支援）
  2. 清瀏覽器 cache 重整
  3. 檢查 Windows 防火牆是否擋 localhost（一般情況不會擋，但有時企業環境會）
PDF 改名 parseMethod 不是 ai-plain

同 Mac 版處理：點 Code 節點 Output 看 finishReason / aiResponseLength / httpError，依錯誤類型對症下藥。Lite Pack v0.6+ 已預埋 thinkingConfig: { thinkingBudget: 0 } 修補 Gemini 2.5 thinking token 雷。
