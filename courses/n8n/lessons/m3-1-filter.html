副檔名過濾：只處理目標格式

Watch Folder 會監控資料夾內的所有變化。加入副檔名過濾，確保只有 .pdf、.xlsx 等指定格式的檔案才會觸發流程，避免雜訊干擾。

⏱預估時間：15 分鐘
📋3 個步驟
1
Watch Folder 節點的內建過濾設定

Watch Folder 節點本身有一個「Ignore Files Matching」設定（在節點設定的 Options 區塊），可以用 glob pattern 排除不需要的檔案。但這是「排除」邏輯，而非「只接受」邏輯。更直覺的做法是在後面接 If 節點做「允許名單」過濾。

💡 Watch Folder 觸發時，輸出的 $json.name 是完整檔名（如 發票_20260414.pdf），$json.type 是事件類型（如 change 或 rename）。
2
用 If 節點過濾副檔名

Watch Folder 後面接 If 節點，用 Expression 讀取檔名並判斷副檔名。符合的往 True 分支繼續處理，不符合的往 False 分支（可接一個 No Operation 節點讓它靜默結束）。

// If 節點的條件設定（Expression 模式） // 方法一：直接用 endsWith 判斷副檔名 // 條件：$json.name 結尾包含 .pdf {{ $json.name.toLowerCase().endsWith('.pdf') }} // 方法二：用 includes 判斷多個副檔名（OR 條件） {{ ['.pdf', '.xlsx', '.docx'].some(ext => $json.name.toLowerCase().endsWith(ext)) }} // 方法三：用正規表達式（更彈性） {{ /\.(pdf|xlsx|docx|txt|md)$/i.test($json.name) }}
✅ 建議用方法三：正規表達式最彈性，i flag 讓大小寫不敏感（.PDF 和 .pdf 都過）。把副檔名列在括號內，新增格式只要加一個 |xxx 即可。
3
常用副檔名過濾規則
過濾目的正規表達式
只處理 PDF/\.pdf$/i
PDF 和 Word 文件/\.(pdf|docx|doc)$/i
所有 Excel 格式/\.(xlsx|xls|csv)$/i
圖片類/\.(jpg|jpeg|png|gif|webp)$/i
文字類（含 Markdown）/\.(txt|md|markdown)$/i
排除暫存檔（~ 開頭）If 節點加一條 NOT 條件：!$json.name.startsWith('~')
💡 排除 macOS 隱藏檔：macOS 有時會在資料夾產生 .DS_Store 或 ._filename 等隱藏檔。在過濾條件中加入 !$json.name.startsWith('.') 可一次排除所有點開頭的隱藏檔。
💼 實戰案例
情境｜某網路書店每天從供應商 CSV 篩出『庫存 <5 且近 7 天有人搜尋』的書建補貨單。
做法｜Filter 節點兩條件 AND，篩選後 95% 的資料直接丟掉——後面節點處理量從 3,000 筆降到 40 筆，執行時間 5 分鐘 → 12 秒。
踩雷｜Filter 節點只接 items 陣列，若前一 node 回傳單一 JSON 物件（而非 items）會整個 workflow 過不去——要先加 SplitInBatches 或 Code 節點轉成 items 陣列。
