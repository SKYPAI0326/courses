n8n 節點操作速查

新增、連線、命名、啟用與停用節點的完整操作說明，以及與 Make 模組對應的操作邏輯對照。

📖參考頁面，可隨時查閱
⏱閱讀約 5 分鐘
四種基本操作
➕
新增節點
在畫布空白處按 Tab 或點擊右上角「+」按鈕，輸入節點名稱搜尋，點擊即加入畫布。也可拖曳現有節點的輸出點（右側圓點）到空白處，直接進入搜尋。
🔗
連接節點
從節點右側的輸出點（灰色圓點）拖曳到下一個節點左側的輸入點，出現綠色線條表示連接成功。一個輸出點可連接多個下游節點（分支）。
✏️
命名節點
雙擊節點標題，或點開節點設定後點選頂部名稱欄位即可修改。建議使用有意義的名稱，如「讀取 PDF」而非預設的「HTTP Request」，方便後續節點用 $('節點名稱') 引用。
⏸️
停用節點
右鍵點擊節點 → 選擇「Disable」，節點變灰並被跳過，資料直接穿透到下一個節點。適合臨時關閉某個步驟而不刪除它。再次右鍵 → Enable 重新啟用。
執行方式
操作說明適用時機
Execute Node（節點內按鈕）只執行該節點，使用前一節點的輸出作為輸入測試單一節點的設定是否正確
Test Workflow（頂部按鈕）從 Trigger 開始執行整條流程（測試模式）測試完整流程邏輯
Save + Activate儲存並啟動 Workflow，讓 Trigger 開始監聽確認流程正確後正式啟動
💡 測試 vs 啟動的差別：Test Workflow 只跑一次、不持續監聽；Activate 後 Trigger 節點才會真正開始等待觸發（如 Webhook 收到請求、Schedule 到時間）。

與 Make 操作對照
Make 操作n8n 對應操作差異說明
新增模組（點 +）新增節點（Tab 或拖曳）n8n 支援畫布任意位置新增

✅ 善用節點命名：當流程複雜時，有意義的節點名稱讓你能用 ${'讀取 PDF'}.item.json.text 精確引用，而不是靠節點順序猜資料來源。
💼 實戰案例
情境｜某行銷助理用 n8n 替代每天手動整理 Slack 訊息→寫進 Notion 的工作。
做法｜只用三個 node：Slack Trigger + Set（整理格式）+ Notion。不上 Code 節點也能做到每日自動歸檔，節省 40 分鐘/天。
踩雷｜初學者常一個 workflow 塞 20 個 node——難除錯、難維護。規則：超過 10 個 node 就該拆成子 workflow 用 Execute Workflow 串接。
