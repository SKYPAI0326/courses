環境互鎖與自我驗收 — 改 workflow 前先確認在哪跑

本系列到這頁結束。在你開始動手之前，先確認執行環境——同一份 LLM 改寫建議，在 self-host 與 Cloud 上適用程度不同。然後用 5 條硬指標驗收自己學到了什麼。

本頁學習成果
能識別自己跑在 self-host 還是 Cloud，並知道 4 個關鍵差異點
能用 5 條驗收清單自評是否達到本系列學習目標
知道 2 道進階練習題的方向，並能獨立用前 7 頁的方法解題
(01)

為什麼最後要回到「在哪跑」

前提假設
本系列的所有技巧都假設你在 self-host n8n 上跑。如果你準備搬上 Cloud，credential 儲存位置、節點可用性、檔案路徑、execution 配額這四個面向都會有差異——LLM 給的改寫建議不一定能直接沿用。

改 workflow 前確認執行環境，不是多餘的步驟。LLM 建議「用 Read Binary File 節點讀本機 PDF」，在 self-host 完全可行；同一句話在 Cloud 執行必定失敗，因為 Cloud 環境裡根本沒有「本機」的概念。先知道你在哪裡，才能判斷 LLM 的建議是否有效。

(02)

self-host vs Cloud — 4 個關鍵差異

以下 4 點對照是改造 workflow 時最常踩到的環境邊界，搬遷前先逐點確認。

SELF-HOST（本課主路線）
Credential 儲存位置
你的電腦或 VPS，由你負責備份與機密管理。加密金鑰遺失即無法復原。
節點可用性
全部 community node 可安裝（自負風險）。Read/Write Files、Execute Command、本地 LLM 節點均可用。
檔案路徑
可直接指定本機路徑，例如 /files/ 或 /Users/.../folder 等本機絕對路徑。
Execution 配額
無上限，但吃你電腦或 VPS 的資源（電費、CPU、磁碟）。自己控制執行量。
N8N CLOUD
Credential 儲存位置
n8n 雲端伺服器，由 n8n 負責加密儲存。self-host 加密過的 credential 無法直接匯入，需重設。
節點可用性
部分節點受限，community node 需經審核。Read/Write Files、Execute Command、本地 LLM 在 Cloud 環境不可用。
檔案路徑
不能直接讀取本機檔案。必須改走 Google Drive、S3 等雲端儲存節點作為中介。
Execution 配額
依方案有 monthly execution 上限，超過另計費。大量批次處理需留意配額消耗速度。
△
改 workflow 前，先確認你跑在哪個環境。同一份 LLM 改寫建議，在 self-host 與 Cloud 上適用程度不同。

(04)

自我驗收 — 5 條可驗證清單

使用方式
這 5 條是本系列的學習目標，以動詞開頭、結果可驗證。逐條對照自己現在能不能做到，不能做到的回去看對應頁。第 5 條是硬指標，建議實際跑過才算。
  • 能用 4 格便箋拆解一個改造需求（input / output / 保留節點 / workflow 編號），並能對著別人講 1 分鐘不打結。（對應：第 3 頁，步驟 1）
  • 能在不貼 credential 的前提下，把任一 Lite Pack workflow JSON 貼給 LLM 並要求中文流程描述。（對應：第 3 頁，步驟 2）
  • 能下指令讓 LLM 只回傳節點區段、不重寫整份 workflow，並在 n8n UI 上正確完成節點替換。（對應：第 3 頁，步驟 3）
  • 能用「最小可重現訊息」格式回報錯誤——紅燈訊息全文 + 上一節點 input + 當前 parameter 三件套，不貼整份 workflow 重來。（對應：第 4 頁，步驟 5）
  • 能把 #02-pdf-ai-rename 改成處理某種非合約類文件（發票、收據、PO、申請表 4 選 1），跑通 10 筆樣本。硬指標（對應：第 6 頁，完整 Walkthrough）

第 5 條建議到課程社群貼出改造成品截圖，讓其他結業學員看到你的進展——這也是最直接的驗收方式。

(05)

自我練習 — 不給答案的 2 題

設計意圖
這兩題沒有附範本答案，也沒有附 prompt 模板。你需要回去用前 7 頁學的方法自己解：拆需求、讀 JSON、限定修改範圍、匯入測試、錯誤回報、收斂。能獨立跑完一輪，才算真正內化了這套流程。
練習 1
把 #04-daily-ai-report 的輸出端從 Google Sheet 改成 Notion Database
提示：輸出端換新節點類型時，記得先在 n8n UI 拉出一個 Notion 節點、複製出來給 LLM 當「目標形狀」範本，不要讓 LLM 從記憶生成節點 JSON。
練習 2
把 #11-csv-clean-score 的評分維度從通用版改成你業務的特定維度，並跑通 10 筆樣本
提示：先填好 4 格便箋，把「你業務的特定維度是什麼、原本的通用維度是什麼、哪些節點需要改」寫清楚，再開始跟 LLM 對話。需求不清就先別動 JSON。
(06)

系列結語

8 頁學完，下一步是動手。這套流程的核心不是 prompt 技巧，是「縮小範圍、逐步驗收」的工作紀律——第一次走完 6 步可能要花一個下午，第五次就會變成 30 分鐘的例行作業。改造案例歡迎分享到學員社群，你的實際情境對其他結業學員的參考價值，遠比任何教材範例都高。
