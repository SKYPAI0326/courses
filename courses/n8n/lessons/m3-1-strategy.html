監聽策略選擇：即時 vs 定時

Watch Folder 和 Schedule Trigger 是 n8n 兩種最常用的本機自動化觸發方式，各有適合的場景。選對策略，讓資源消耗最小化。

📖參考頁面
⏱閱讀約 8 分鐘
兩種策略對比
👁
Watch Folder（即時監聽）
事件驅動
  • 檔案一落地就立即觸發，延遲極低
  • 不會浪費資源掃描空資料夾
  • 適合需要快速回應的場景
  • n8n 必須一直在執行中才能監聽
  • 只能監控資料夾，不能做其他類型的觸發
⏰
Schedule Trigger（定時執行）
時間驅動
  • 可以在指定時間執行任意任務
  • 不需要外部事件觸發，完全自主
  • 適合每日/每週定時彙整類任務
  • 固定間隔執行，即使沒有新資料也會跑
  • 實時性不如 Watch Folder
選擇決策表
任務特性建議策略理由
客戶上傳 PDF → 立即重命名Watch Folder需要即時反應，不能等下一個時間點
每天早上 8 點產出日報Schedule Trigger固定時間任務，與外部事件無關
每週一統計上週資料Schedule Trigger週期性彙整，時間比事件更重要
收到圖片 → 立即 AI 辨識Watch Folder需要在圖片到達時馬上處理
整理上個月的合約文件手動執行 / Schedule偶發性任務，不需要持續監聽
監控資料夾新增 + 每日彙整兩者並用Watch Folder 處理即時，Schedule 做每日統計
兩者並用的場景

Watch Folder 和 Schedule 並不互斥，可以在同一個系統中各自扮演不同角色：

範例：發票自動化系統

第一條 Workflow（Watch Folder）：監控「收件匣」資料夾 → 新發票 PDF 一到立即 AI 辨識 → 提取金額與廠商 → 存入 SQLite 資料庫。

第二條 Workflow（Schedule，每月1日）：讀取資料庫本月所有發票 → 彙整總支出 → 生成 Markdown 報告 → 存入「月報」資料夾 → 發送郵件通知。

💡 n8n Desktop 需要保持開啟：兩種 Trigger 都需要 n8n Desktop 持續運行。若電腦重開機後 n8n 沒有自動啟動，Watch Folder 和 Schedule 都不會觸發。建議配合 1.2 的 launchd 自動重啟設定（Tunnel 那頁有說明）。
✅ 資源消耗：Watch Folder 持續監控資料夾的 inotify 事件，CPU 消耗幾乎為零。Schedule Trigger 每次執行時才消耗資源，不執行時完全靜默。兩者對一般電腦都是輕量級的。
💼 實戰案例
情境｜某律師事務所整理 500 份舊案卷——一次全跑 AI 分類會破 Gemini 日配額。
做法｜用 SplitInBatches (每批 50) + Wait (30 分鐘)，一天內慢慢跑完、不破額度、半夜掛著執行。
踩雷｜批次策略最容易踩的雷：想用 Loop 節點手動切批——n8n 的 SplitInBatches 已經內建批次邏輯，不要再套 Loop 會變巢狀死迴圈。
