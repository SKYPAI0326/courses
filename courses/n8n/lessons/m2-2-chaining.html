Module 02
SECTION 2.2

Optional Chaining ?.（自動防斷機制）

一個符號，讓流程從脆弱變得堅韌。資料有缺漏時不再當機，繼續往下跑。

⏱預估時間：15 分鐘
📋4 個步驟
✅前置：完成 2.1
1
空值地雷：示範一個會當機的流程

先親眼看一次問題才能理解解法的價值。建立一個讀取客戶地址的 Set 節點，然後故意送一筆沒有 address 欄位的資料進去，看看 n8n 會出現什麼錯誤。

// 送入資料（故意缺少 address 欄位）： { "客戶姓名": "陳大明", "電話": "0912-345-678" } // Set 節點嘗試讀取： {{ $json.address.city }} // 結果：TypeError: Cannot read properties of undefined // 流程中斷，後續所有節點全部停止執行
⚠️ 現實中，表單填寫、API 回傳、上傳的 Excel 都可能有缺漏欄位。若不處理，一筆缺漏資料就能讓整批 1,000 筆的工作流程全部中斷。
2
?. 語法：有就讀，沒有就跳過

在可能不存在的欄位後面加上 ?.，n8n 遇到空值時不會報錯，而是直接回傳 undefined 並繼續執行。你在哪層欄位不確定是否存在，就在哪層加上 ?.。

生活類比
一般語法像直接開門：門不存在就撞牆當機。?. 的語法像是：「如果這扇門存在，進去；不存在，就安靜繼續走。」流程不會因為一筆缺漏資料而整批停擺。
// 原本會報錯的寫法： {{ $json.address.city }} // 加上 ?. 後，address 不存在時回傳 undefined，不報錯： {{ $json.address?.city }} // 多層巢狀也可以每層都加： {{ $json.客戶?.聯絡方式?.電子郵件 }}
3
搭配 ?? 提供預設值，讓輸出更乾淨

?. 讓流程不報錯，但空值仍然是 undefined，下游節點可能顯示空白或繼續出問題。搭配 ??（雙問號）指定一個預設值，當欄位是空值時自動填入，輸出結果更完整。

// 語法：{{ 可能為空的值 ?? '預設值' }} {{ $json.address?.city ?? '未填寫' }} → address 有值：顯示城市名稱 → address 為空：顯示 "未填寫" {{ $json.客戶?.聯絡方式?.電話 ?? '無聯絡電話' }} → 電話有值：顯示電話號碼 → 電話為空：顯示 "無聯絡電話" {{ $json.合約金額 ?? 0 }} → 金額有值：顯示金額數字 → 金額為空：顯示 0（數字型別，不是字串）
4
實際練習：處理可能缺漏欄位的客戶表單

建立一個 Set 節點，處理從 Google 表單收到的客戶資料。表單有姓名、電話、Email、地址四個欄位，但都是選填。用 ?. 和 ?? 讓每個欄位在空值時都有合理的預設值，再送給下一個節點處理。

// Set 節點，四個輸出欄位： 姓名： {{ $json.姓名 ?? '未提供' }} 電話： {{ $json.聯絡資訊?.電話 ?? '無電話' }} Email： {{ $json.聯絡資訊?.email ?? '無Email' }} 城市： {{ $json.地址?.縣市 ?? '未填寫' }}
✅ 完成後，試著送兩筆資料：一筆填完整、一筆故意留空。確認兩筆都能成功通過，且空值欄位顯示預設值，流程沒有中斷。
完成驗證 CHECKPOINT
流程對完整和缺漏資料都能順利執行

送入一筆缺漏 address 欄位的資料，Set 節點不報錯，輸出顯示你設定的預設值（如「未填寫」）；送入完整資料時，正常顯示真實欄位值。這代表 ?. 和 ?? 已正確運作。

常見問題
加了 ?. 還是報錯
確認 ?. 加在正確的層級。例如 $json.客戶?.地址.城市 只保護了「地址」這層，若「客戶」本身不存在仍然報錯。應改為 $json.客戶?.地址?.城市，每個不確定的層都要加。
預設值沒有出現，還是顯示 undefined
?? 只對 null 和 undefined 生效。若欄位值是空字串 ""（使用者送出空表單），?? 不會觸發，欄位仍然是空字串。若要同時處理空字串，改用 || '預設值'（但要注意 || 對數字 0 也會觸發，依情況選擇）。
