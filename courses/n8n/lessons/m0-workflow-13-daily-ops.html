#13 · DAILY OPS SNAPSHOT · 9 NODES · 進階

每日營運快照與異常提醒：n8n 長期運行的代表場景

小團隊每天看訂單數、營收、退款、報名、客服量、廣告花費。痛點是「資料分散 + 異常太晚發現」。這個 workflow 每天 9:00 自動讀今日 CSV → 跟過去 7 天歷史對比 → 異常時 AI 摘要可能原因 + Telegram 推。Schedule + 歷史對比 + 異常偵測組合教學，是 n8n 從「即時通知」升級到「長期營運資料庫」的代表場景。

學習成果
能設計兩條 input 鏈（今日 CSV + 歷史 CSV）合流到同一 Code 節點
能用 jsCode 計算 7 天均值、最大最小、偏差百分比 + 異常閾值（30%）判斷
能設計「only on anomaly」AI 呼叫策略（沒異常不打 AI 省 token）
(01) WORKFLOW 全貌

兩條 input 流合併到 Code

Schedule: 每天 09:00→ 雙 fan-out：

Read 今日 CSV→Extract ↘
Code: 對比 + AI 分析→Write 快照 JSON + Telegram
Read 歷史 CSV→Extract ↗

9 節點。Schedule 同時觸發兩條 input 鏈（今日 + 歷史），都流到 Code 節點用 $('Extract 今日 CSV').all() + $('Extract 歷史 CSV').all() 各取一份，計算對比。

(02) 重點節點拆解

Code: 對比 + AI 分析（核心）

CORE
三段式：聚合 today → 算 7 天均值 → 異常才打 AI

Step 1 · 聚合 today metrics：

// CSV 每行是 {metric: "orders", value: "42"} const today = {}; for (const it of $('Extract 今日 CSV').all()) { today[it.json.metric] = parseFloat(it.json.value); } // today = { orders: 42, revenue: 35000, refunds: 2 }

Step 2 · 算 7 天均值 + 偏差：

const histByMetric = {}; for (const it of $('Extract 歷史 CSV').all()) { if (!histByMetric[it.json.metric]) histByMetric[it.json.metric] = []; histByMetric[it.json.metric].push(parseFloat(it.json.value)); } const anomalies = []; for (const [m, v] of Object.entries(today)) { const hist = histByMetric[m] || []; const avg = hist.reduce((s,x)=>s+x,0) / hist.length; const dev = (v - avg) / avg * 100; if (Math.abs(dev) > 30) anomalies.push({ metric: m, today: v, avg, dev }); }

Step 3 · 只在異常時打 AI：

// anomalies.length === 0 不打 AI（省 token） let aiSummary = ''; if (anomalies.length > 0) { const prompt = '你是營運分析助手...請用 3 句話：(1) 哪些異常最重要 (2) 可能原因 (3) 建議下一步'; aiSummary = await this.helpers.httpRequest({ ... }); }

關鍵設計：「only on anomaly」呼叫 AI — 一個月 30 天平常 25 天沒異常，省下 25 次 API 呼叫。Make 學員會立刻有感（Make 也有 conditional 但 n8n 在 jsCode 內判斷更彈性）。

(03) 改造方向

5 種改造

  1. 換資料來源：本機 CSV 改 HTTP Request 抓 Shopify / GA4 / Google Ads / Stripe API（Make 學員都熟，遷移直觀）。
  2. 加多時段對比：除了 7 天均值，加「同期上月」「同期上週幾」做更精準異常判斷。
  3. 加 dashboard 寫入：snapshot.json 寫到本機後，再用一個 workflow 把 30 天歷史串成 charts.html 開瀏覽器看。
  4. 加多人通知：依異常嚴重度分層通知（10% → Telegram 個人，30% → 群組，50% → email + SMS）。
  5. 串 Schedule + Aggregate 做週報：每週一 9:00 跑一次「過去 7 天每天 snapshot 統整」推給高層。
