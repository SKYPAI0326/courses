#07 · QUICK TUNNEL RECEIVER · 4 NODES · 進階

把本機 n8n 變成公開 URL：Cloudflare Quick Tunnel 雲地橋接

本機 n8n 對外只能 localhost:5678 — 別的服務（Make scenario / Google Form / 學員自家網站）打不到。Cloudflare Quick Tunnel 給你一個免費的 *.trycloudflare.com URL（無需 Cloudflare 帳號、無需綁網域），3 秒開通把本機暴露成公開。這個 workflow 是 tunnel 接到後做什麼事的範例 — 收到外部 ping 推 Telegram + 立刻回 pong。

學習成果
能用 starter-kit 的 tunnel-quick.command / .bat 開 Cloudflare Quick Tunnel 取得公開 URL
能設計外部觸發 webhook + 解析請求參數 + 即時回應的 4 節點 pattern
理解 Quick Tunnel 與 Named Tunnel 的差異與適用場景
(01) WORKFLOW 全貌

節點與資料流

Webhook: external-ping → Code: 解析外部請求

↗ Telegram 通知
↘ Respond: pong

4 節點，類似 #06 的 fan-out pattern 但只有 2 條輸出。Webhook path 設 external-ping，搭 Quick Tunnel 後完整公開 URL：https://xxxx.trycloudflare.com/webhook/external-ping。任何服務（Make / curl / 學員手機）打這個 URL 都會觸發 workflow。

(02) NODE BREAKDOWN

節點逐一拆解

NODE 1 / 4
Webhook: external-ping

Path external-ping、HTTP Method 接受 GET + POST、Response Mode 設 lastNode。

n8n 預設 Respond: When Last Node Finishes，但因為下游有 fan-out 兩條線，n8n 會用「最後執行完的節點」當 response。明確設 Set 節點當 lastNode 比較可控。

NODE 2 / 4
Code: 解析外部請求

從 webhook 取參數，組合 Telegram 訊息文與回應 JSON：

// 從 webhook 取請求方來源 const webhookData = $input.first().json; const source = webhookData.headers?.['user-agent'] || 'unknown'; const ip = webhookData.headers?.['x-forwarded-for'] || 'unknown'; const message = webhookData.body?.message || webhookData.query?.message || '(空訊息)'; const ts = new Date().toISOString(); // 組 Telegram 通知文 const tgMsg = '🌐 外部觸發\n\n來源 IP: ' + ip + '\n訊息: ' + message + '\n時間: ' + ts; // 組 webhook 回應 return [{ json: { pong: true, received_at: ts, echo_message: message, tgMsg } }];
NODE 3 / 4
Telegram 通知

同 #05 / #06，text: {{ $json.tgMsg }}。每次有人從外部打你的公開 URL 你手機就收到通知 — 像個個人「webhook 監控器」。

NODE 4 / 4
Respond: pong

Set 節點輸出 { status: "pong", received_at: ..., echo_message: ... }，n8n 把這個當 webhook response 回給 caller（HTTP 200）。

(03) DESIGN RATIONALE

Quick Tunnel vs Named Tunnel

💭
Quick Tunnel（本課程預設）：
雙擊 tunnel-quick.command / .bat，3 秒開通，給你 https://random-words-123.trycloudflare.com 公開 URL。
優點：免費、不需要 Cloudflare 帳號、不需要綁自家網域、隨開隨用。
缺點：URL 每次重啟會變、預設 24 小時自動斷線、不能指定 hostname。
適用：開發測試、課堂 demo、臨時對外。
💭
Named Tunnel（進階）：
需要 Cloudflare 帳號 + 自家網域 + cloudflared service install 設定，固定 hostname（如 n8n.yourdomain.com）、不會斷線、可永久運作。
適用：生產環境、長期對外服務。
注意：會在系統建 launchd / Windows service，要清乾淨需要 cloudflared service uninstall（連發 #11 學員實測）。
⚠️
Quick Tunnel HTTP/2 vs QUIC 的雷：
Cloudflare Quick Tunnel 預設用 QUIC 協議，部分企業網路（含臺灣某些 ISP / 企業 firewall）會擋 QUIC port 7844。本課程 starter-kit 的 tunnel-quick.command 預設加 --protocol http2 強制走 HTTP/2，避開 QUIC 被擋。如果還連不上代表你的網路完全擋 cloudflared，跳 ngrok 備用方案。
(04) 改造方向

5 種改造方向

  1. Make 觸發 n8n：Make scenario 結束時 HTTP module POST 到 https://xxx.trycloudflare.com/webhook/external-ping，把 Make 雲端訊號接到本機 n8n 跑批次任務。
  2. Google Form → Make → n8n：學員填表 → Google Sheets 新列 → Make 偵測 → 打 n8n webhook → 本機跑 PDF 生成 / 寄信 / 歸檔。
  3. IFTTT / Zapier 接 n8n：把 n8n 變成 IFTTT 的 webhook target，IFTTT 觸發（時間 / 位置 / iOS 捷徑）→ 本機 n8n 跑事。
  4. Telegram bot 指令：在 BotFather 設 setWebhook 指向 trycloudflare URL，使用者對 bot 發訊息 → Telegram 推到 n8n → n8n 解析指令跑 workflow。
  5. 個人 dashboard：寫個簡單 HTML 頁面（GitHub Pages 免費架）含表單，submit 打到 trycloudflare URL，收到 Telegram 通知 + 觸發業務 workflow。
