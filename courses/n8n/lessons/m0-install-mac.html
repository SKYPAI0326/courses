M0 · QUICKSTART · MAC

從零在 Mac 安裝 Lite Pack：60-90 分鐘到 14 個 workflow 全部就緒

這是「先用後學」路線的入口頁。你不用先懂 Docker、不用先懂 n8n、不用先懂 webhook —— 跟著本頁四階段做下去，60-90 分鐘後你的 Mac 上會有一個能持續使用的 n8n 工作站，內含 14 個現成 workflow，10 個是 AI 驅動的真實業務流程（PDF 改名 / 批次處理 / Gmail 分類 / 客戶資料夾整理 / CSV 清洗評分 / 知識庫問答 / 營運快照 / API 監控 等），剩 4 個是基礎設施與練習素材。原理之後再去 Module 1-4 慢慢學。

學習成果
能在 Mac（Intel / Apple Silicon 皆可）從零安裝 Docker Desktop + n8n 並完成首次設定
能用 setup-wizard.command 自動填入 Gemini API key + Telegram bot 完成 Lite Pack 一鍵安裝
能在自己電腦上跑通 5 個招牌 demo workflow 並驗證輸出（Telegram 通知收到、PDF 自動改名、批次分流、AI 日報生成、Webhook 觸發 AI 回應）
(00) STEP ZERO · 重灌學員必做｜新手跳過

先把 Mac 上既有的 n8n / Docker 殘留清乾淨

為什麼這很重要
如果你以前裝過 n8n（自己摸索、跟別的教學、舊版 Desktop App），這次重灌會撞 port、撞 volume、撞 cloudflared process — 新跑的 n8n 啟動失敗或行為怪異 90% 是這個原因。把舊的全清掉再從新版下手，比邊裝邊修快得多。
🆕
完全沒裝過 Docker / n8n 的新手？Step 0 內所有指令都需要 Docker（如 docker info / docker rm）。請直接跳到 Phase A 從零裝，裝完不用回來做 Step 0（你電腦本來就乾淨無殘留）。Step 0 是「以前裝過 n8n / 跟過別的教學留下殘留」的學員才需要做的歸零步驟。
⚠️
會清掉：所有 n8n / postgres container + volume + network、cloudflared 殘留 process、所有 starter-kit / lite-pack 解壓資料夾、所有殘留 zip。
不會動到：Docker Desktop 本身、Docker image cache（重灌會直接拿 cache，不重新下載 GB 級 image）、Telegram bot、Gemini API key、Cloudflare 帳號（這些都在雲端）。
STEP 0-1
先把終端機準備好

打開 Terminal（應用程式 → 工具程式 → 終端機，或 Spotlight 搜「Terminal」）。第一行先跑這個讓 zsh 認得 # 註解：

setopt interactivecomments

沒跑這行的話，後面所有 # 註解會被當成指令導致 zsh 噴 command not found: #（不影響實際執行但很嚇人）。

STEP 0-2
確認 Docker Desktop 在跑

選單列右上角應該有鯨魚 icon（不再動畫表示就緒）。沒安裝的話跳到 Phase A 先裝。

docker info > /dev/null 2>&1 && echo "✅ Docker daemon 正常" || echo "❌ 先打開 Docker Desktop"
STEP 0-3
清掉所有 n8n / postgres container + volume + network
# 用 docker compose down -v 走正規路徑（如果 starter-kit 還在）
[ -f ~/Downloads/n8n-starter-kit/n8n-compose.yml ] && \
  (cd ~/Downloads/n8n-starter-kit && docker compose -f n8n-compose.yml down -v 2>/dev/null) || true

# 殘留 container 強制清
docker ps -a --filter "name=n8n" --format "{{.Names}}" | xargs -r docker rm -f
docker ps -a --filter "name=postgres" --format "{{.Names}}" | xargs -r docker rm -f

# 殘留 volume 清
docker volume ls --filter "name=n8n" -q | xargs -r docker volume rm
docker volume ls --filter "name=postgres" -q | xargs -r docker volume rm

# 殘留 network 清
docker network ls --filter "name=n8n" -q | xargs -r docker network rm 2>/dev/null || true
STEP 0-4
殺掉殘留的 cloudflared tunnel process
# 之前雙擊 tunnel-quick.command 開的 cloudflared 可能還在跑
pkill -f "cloudflared.*trycloudflare" 2>/dev/null || true
pkill -f "cloudflared tunnel" 2>/dev/null || true
ps aux | grep -v grep | grep cloudflared || echo "✅ cloudflared 已清乾淨"
⚠️
如果驗證還看到 root 權限的 cloudflared，是以前裝過 cloudflared service install 的 launchd service，需要 sudo 卸載： 
# 卸載 launchd service
sudo launchctl list | grep -i cloudflare
sudo cloudflared service uninstall 2>/dev/null || true
sudo pkill -f cloudflared 2>/dev/null || true
sleep 2
ps aux | grep -v grep | grep cloudflared || echo "✅ 完全清乾淨"
STEP 0-5
刪所有解壓資料夾 + 殘留 zip
# 含 macOS 防覆蓋產生的 starter-kit 2 / 3
rm -rf ~/Downloads/n8n-starter-kit*
rm -rf ~/Desktop/n8n-starter-kit*
rm -rf ~/n8n-starter-kit*
rm -rf ~/Downloads/n8n-lite-pack*
rm -rf ~/Desktop/n8n-lite-pack*
rm -f ~/Downloads/n8n-starter-kit*.zip
rm -f ~/Downloads/n8n-lite-pack*.zip
STEP 0-6
完全歸零驗證
echo "── container ──"
docker ps -a --filter "name=n8n" --format "{{.Names}}"
docker ps -a --filter "name=postgres" --format "{{.Names}}"
echo "── volume ──"
docker volume ls | grep -E "n8n|postgres" || echo "✅ 無殘留 volume"
echo "── network ──"
docker network ls | grep n8n || echo "✅ 無殘留 network"
echo "── 解壓資料夾 ──"
ls ~/Downloads/ 2>/dev/null | grep -i "n8n\|lite-pack" || echo "✅ ~/Downloads 已清乾淨"
echo "── cloudflared process ──"
ps aux | grep -v grep | grep cloudflared || echo "✅ cloudflared 已清乾淨"
echo "── 完成 ──"
通過 STEP 0 的判準
5 個區塊都顯示 ✅ 已清乾淨。任何一行有殘留就重跑對應 step 0-3 ~ 0-5 的指令直到歸零。
(01) PHASE A

Docker Desktop + n8n 啟動（10-30 分鐘）

這個階段要做的事
先安裝 Docker Desktop（如果你還沒裝過），下載課程提供的 starter-kit zip，雙擊 start.command 把 n8n 跑起來，建立你的 Owner Account。完成後瀏覽器打開 localhost:5678 會看到 n8n 主介面。
A1
安裝 Docker Desktop（如果還沒裝）

到 https://www.docker.com/products/docker-desktop/ 下載 Mac 版安裝。

  • Apple Silicon（M1/M2/M3/M4）→ 選 Apple Silicon 那個下載按鈕
  • Intel Mac → 選 Intel Chip 那個按鈕

下載 .dmg → 雙擊 → 把 Docker.app 拖到 Applications → 第一次打開要 input 密碼授權。等選單列鯨魚 icon 不再動畫表示就緒。

💡
不用註冊 Docker Hub 帳號就能用本課程所有功能。Docker Desktop 啟動時跳註冊頁可以直接 X 關掉。
A2
下載 n8n-starter-kit.zip

從本課程下載試跑包：

⬇ 下載 n8n-starter-kit.zip（約 15 KB）

下載後在 Finder Downloads 雙擊解壓 → 同層產生 ~/Downloads/n8n-starter-kit/。

⚠️
如果你前面 Step 0 沒清，會解壓成 n8n-starter-kit 2（macOS 防覆蓋）→ 回 Step 0 重做歸零。
A3
macOS Gatekeeper 解鎖 4 個 .command

解壓後資料夾內有 start / stop / update / tunnel-quick 共 4 個 .command 腳本。每個第一次雙擊都會被 Gatekeeper 擋。一次處理 4 個比較有效率：

  1. 分別雙擊 4 個 .command → 都會跳「無法打開」對話框 → 按「取消」
  2. 系統設定 → 隱私權與安全性 → 捲到「安全性」區塊
  3. 會看到「'start.command' 已被阻擋使用」→ 按「強制打開」→ 確認密碼
  4. 對 stop / update / tunnel-quick 重複 step 3
完成判準
4 個 .command 都加到「允許執行」清單。之後雙擊不會再被擋。
A4
第一次雙擊 start.command（建 .env）

雙擊 ~/Downloads/n8n-starter-kit/start.command。第一次會自動把 .env.example 複製成 .env 並打開 TextEdit 讓你編輯。

⚠️
改密碼：把 POSTGRES_PASSWORD=please_change_me_to_a_strong_password 那行的值改成自己的強密碼（至少 16 字元、混合英數字符），存檔關閉。
🚨
不要再改第二次：Postgres 密碼第一次寫進 volume 之後，之後改 .env 不會自動同步，會造成「password authentication failed」。設好就不要再改。
A5
再次雙擊 start.command（真的啟動）

這次會真的拉 Docker image + 啟動 container。首次需 1-5 分鐘下載 postgres / n8n image。終端機每 10 秒輸出進度。

預期看到（成功）
終端機顯示 ✅ n8n 已啟動：http://localhost:5678，瀏覽器自動開到 n8n 主介面。
⚠️
如果跳「Port 5678 已被佔用」→ 按「停掉並重啟」自動清理。
如果跳「postgres 密碼認證失敗」→ 按「重置並重啟」自動 down -v 清 volume。
A6
建立 n8n Owner Account

瀏覽器開 http://localhost:5678，看到 「Set up owner account」引導頁。

🚨
用 Chrome / Firefox / Edge，不要用 Safari（Safari 對 localhost cookie 限制嚴格，可能登入後一直跳回登入頁）。
  1. Email：填你的（任意 email，不會驗證）
  2. 密碼：自定（至少 8 字元含大小寫與數字）
  3. 跳過後續行銷問卷
Phase A 通過判準
進入 n8n 主介面（左側選單看到 Workflows / Credentials / Executions / Templates）。
(02) PHASE B

Lite Pack 一鍵安裝（10 分鐘）

這個階段要做的事
下載 Lite Pack 試跑包，雙擊 setup-wizard.command，依對話框提示貼入 3 個 key（Gemini API key / Telegram bot token / Telegram chat ID）。setup-wizard 會跑 10 個步驟自動完成：建 credentials、匯入 14 個 workflow、做 Telegram + Gemini smoke test。
📋
事前準備：開始 setup-wizard 前先把 3 個 key 拿到手放在便利貼上。
1. Gemini API key：到 aistudio.google.com/apikey（免費）→ Create API key → 複製 AIza... 開頭那串
2. Telegram bot token：Telegram 找 @BotFather → /newbot → 給名字 → 拿到 123456:ABC-DEF...
3. Telegram chat ID：對你的 bot 發任意訊息 → 瀏覽器開 https://api.telegram.org/bot<你的TOKEN>/getUpdates → 找 chat.id 那串純數字
B1
下載 Lite Pack zip
⬇ 下載 n8n-lite-pack.zip（約 30 KB）

下載後 Finder 雙擊解壓 → 同層產生 ~/Downloads/n8n-lite-pack/。

B2
macOS Gatekeeper 解鎖 setup-wizard.command
  1. 進 ~/Downloads/n8n-lite-pack/ 雙擊 setup-wizard.command → 跳「無法打開」按取消
  2. 系統設定 → 隱私權與安全性 → 看到「setup-wizard.command 已被阻擋使用」→ 按「強制打開」→ 確認密碼
B3
再次雙擊 setup-wizard.command（真的跑）

會跳 3 個對話框依序問 Gemini key / TG token / TG chat ID。把便利貼上 3 個 key 依序貼進去。

setup-wizard 跑 10 個步驟（約 1-3 分鐘）：

  1. 環境檢查（Docker + n8n 在跑 + 找到 starter-kit）
  2. 收集 3 個 key
  3. 偵測 + patch file access 環境變數（如果還沒設過）
  4. 寫 personalization.env（你的 key 紀錄）
  5. 建 sample 資料夾（pdf-inbox / batch-inbox / daily-input / ai-output 等 7 個）
  6. 生成 decrypted credentials JSON
  7. 替換 workflow JSON 內 placeholder（API key + chat ID）
  8. n8n CLI 匯入 2 個 credentials
  9. n8n CLI 匯入 14 個 workflows
  10. Telegram + Gemini smoke test
Phase B 通過判準
終端機跳「Lite Pack 安裝完成」對話框 + 自動開瀏覽器到 n8n。
對話框內容應該齊備：✅ 14 workflows ✅ 2 credentials ✅ Telegram 你手機收到「✅ Lite Pack setup-wizard 安裝完成」 ✅ Gemini reply 「OK」
B4
n8n UI 確認 8 workflow + 2 credentials

瀏覽器到 http://localhost:5678 → 左側 Workflows：應該看到 14 個（編號 01 ~ 14）。

左側 Credentials：應該看到 2 個 — Lite Pack · Gemini API + Lite Pack · Telegram Bot。

(03) PHASE C

5 個招牌 demo 驗證（30-50 分鐘）

這個階段要做的事
逐個跑通 5 個 demo workflow（05 Telegram 通知 / 02 PDF AI 改名 / 03 批次處理 / 04 定時日報 / 06 Webhook AI），驗證 Lite Pack 在你 Mac 上完整可用。剩 9 個 workflow（含客戶資料夾整理、CSV 清洗、知識庫問答、營運快照等）也都裝好了，課程結束去工作流總覽看其他能力。每個 workflow 給範例輸入 → Execute → 看實際輸出。跑通就代表你已具備跑任何 n8n + AI 自動化的能力，剩下只是改造 prompt / 換資料來源。
C1
準備測試素材

最快方式：下載演練素材包（推薦）

⬇ 下載 n8n-sample-pack.zip（約 130 KB）

下載後 Finder 雙擊解壓 → 同層產生 ~/Downloads/n8n-sample-pack/，內含 7 個子資料夾對應 7 個演練 workflow（pdf-inbox / batch-inbox / daily-input / client-inbox / leads-inbox / knowledge-docs / ops-input）。把各子資料夾**裡面的檔**複製到 starter-kit 對應位置：

# 把素材包內容複製到 starter-kit 對應資料夾
cp -R ~/Downloads/n8n-sample-pack/pdf-inbox/* ~/Downloads/n8n-starter-kit/shared/pdf-inbox/
cp -R ~/Downloads/n8n-sample-pack/batch-inbox/* ~/Downloads/n8n-starter-kit/shared/batch-inbox/
cp -R ~/Downloads/n8n-sample-pack/daily-input/* ~/Downloads/n8n-starter-kit/shared/daily-input/
cp -R ~/Downloads/n8n-sample-pack/client-inbox/* ~/Downloads/n8n-starter-kit/shared/client-inbox/
cp -R ~/Downloads/n8n-sample-pack/leads-inbox/* ~/Downloads/n8n-starter-kit/shared/leads-inbox/
cp -R ~/Downloads/n8n-sample-pack/knowledge-docs/* ~/Downloads/n8n-starter-kit/shared/knowledge-docs/
cp -R ~/Downloads/n8n-sample-pack/ops-input/* ~/Downloads/n8n-starter-kit/shared/ops-input/

替代方式：用自己的檔（如果想測自己的真實情境）

# 開 Finder 看 shared/ 結構
open ~/Downloads/n8n-starter-kit/shared/

# 建一份測試 daily-input/ 內容
echo "今日完成：n8n Lite Pack 安裝。明日待辦：跑 5 個 demo workflow 驗證。" \
  > ~/Downloads/n8n-starter-kit/shared/daily-input/test-meeting.md

另外從你電腦挑：

  • 1-3 份合約 / 發票 / 說明 PDF 拖進 shared/pdf-inbox/（給 02 用）
  • 3-5 份 PDF（含 1 份故意是壞檔可以用 echo 建 .pdf） 拖進 shared/batch-inbox/（給 03 用）
C2
驗證 #5 Telegram 通知（最簡單先測）
  1. n8n 左側 Workflows → 點開 05 · Telegram 通知
  2. 下方紫色 Execute workflow 按鈕
  3. 3 個節點全綠（Manual Trigger → Code → Telegram）
通過判準
你的 Telegram bot 收到「✅ Lite Pack 通知測試成功」訊息。
C3
驗證 #2 PDF AI 改名（招牌 demo）
  1. 確認 shared/pdf-inbox/ 有 1 份 PDF
  2. n8n 進 02 · PDF AI 改名 workflow
  3. Execute workflow → 看 5 個節點全綠（Manual → Read → Extract → Code: AI 改名 → Write）
  4. 點 Code: AI 改名（內含 Gemini call）節點看 Output：parseMethod 應該是 ai-plain
  5. 終端機跑：ls ~/Downloads/n8n-starter-kit/shared/pdf-renamed/
通過判準
新檔名長相 YYYYMMDD_類型_關鍵字.pdf（例：20260506_說明_Adobe創意服務.pdf）。
如果是 YYYYMMDD_AI解析失敗_xxx.pdf → AI 沒回應，去 troubleshoot 看「parseMethod 不是 ai-plain 怎麼辦」。
C4
驗證 #3 批次處理錯誤恢復（資料工廠精髓）
  1. 確認 shared/batch-inbox/ 有 3-5 份 PDF（含至少 1 份壞檔）
  2. 建壞檔測試：echo "fake" > ~/Downloads/n8n-starter-kit/shared/batch-inbox/壞檔.pdf
  3. n8n 進 03 · 批次處理錯誤恢復 → Execute workflow
  4. 看 canvas 雙路徑：合法 PDF 走上排（綠）→ processed/，壞檔走下排（紅）→ failed/
  5. 終端機跑：ls ~/Downloads/n8n-starter-kit/shared/batch-inbox/processed/ ~/Downloads/n8n-starter-kit/shared/batch-inbox/failed/
通過判準
processed/ 內合法 PDF 都改名（不是 AI解析失敗_xxx）。
failed/ 內壞檔 fileName 正確 + 內容是壞檔原內容（用 file 指令驗應顯示 ASCII text）。
C5
驗證 #4 定時 AI 日報
  1. 確認 shared/daily-input/ 有至少 1 份 .md / .txt
  2. n8n 進 04 · 定時 AI 日報 → Execute workflow（測試用，不等到明早 8:00）
  3. 跑完看 cat ~/Downloads/n8n-starter-kit/shared/daily-output/daily-2026-XX-XX.md
通過判準
Markdown 內容含「重點摘要 / 待辦事項 / 關鍵決策 / 風險或卡點」四段結構化內容。
Code 節點 Output 的 aiError 為 null。
C6
驗證 #6 Webhook AI（外部觸發 AI 鏈）
  1. n8n 進 06 · Webhook → Gemini → 本機檔 → Execute workflow（Webhook 變藍色「Listening」）
  2. 新終端機（不要關原本的）跑下面 curl（整行不要換行）：
curl -X POST http://localhost:5678/webhook-test/ai-ask -H "Content-Type: application/json" -d '{"question":"什麼是 n8n？三句話內回答。"}'
通過判準（4 件齊備）
(1) curl 終端回 {"status":"ok",...}
(2) ~/Downloads/n8n-starter-kit/shared/ai-output/ 多 1 份 .md
(3) Telegram 收到通知
(4) Code 節點 Output 的 aiError 為 null
🎯
Lite Pack 全鏈路通過：5 個 workflow（C2-C6）都跑完代表你的 Mac 已具備真實的 n8n + AI 工作站。其餘 3 個 workflow（01 webhook hello-world / 07 quick-tunnel-receiver / 08 expression-practice）作為改造練習素材。
下一步去 工作流設計拆解學習每個 workflow 怎麼設計、為什麼這樣設計、自己怎麼改造。
(04) TROUBLESHOOT

卡關時看這

A4 雙擊 start.command 沒反應 / 終端機馬上關

對 .command 按右鍵 → 用「終端機」開啟，看終端機輸出錯誤訊息。常見：Docker Desktop 沒開（先到選單列點鯨魚 icon 啟動）。

A6 localhost:5678 連不上 / 載入一直轉
  1. 確認 Docker Desktop 鯨魚 icon 不再動畫
  2. 終端機跑 docker ps | grep n8n 應有 2 個 container（n8n + postgres）狀態 healthy
  3. 如果 container 不在，重雙擊 start.command
  4. 瀏覽器換 Chrome/Firefox（Safari cookie 雷）
B3 setup-wizard 跳「找不到 n8n container」

表示 n8n 沒在跑。回 A5 雙擊 start.command 把 n8n 啟動再跑 setup-wizard。

B3 跳「Owner account 未設置」

A6 沒做完。瀏覽器到 localhost:5678 完成 owner account 設置，然後重跑 setup-wizard。

C3 PDF 改名 parseMethod 不是 ai-plain（AI 沒回應）

點 Code 節點 Output 看：
