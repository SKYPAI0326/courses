#11 · CSV CLEAN & SCORE · 8 NODES · 進階

銷售線索 CSV 清洗評分：批次處理 1000 筆名單，n8n 工程化升級

Make 老手最熟悉的場景：團隊從表單 / 展會 / 名單工具匯出 CSV，欄位格式不一、電話缺國碼、公司髒資料、重複 email、品質無法快速排序。這個 workflow 一鍵清乾淨：欄位正規化（電話補 +886）→ email 去重 → AI 評 hot/warm/cold → 輸出 cleaned.csv + Markdown report + Telegram 摘要。

學習成果
能用 n8n Extract CSV + Code 節點做欄位正規化（電話格式、email 小寫、公司去空白）
能用 JS Object key trick 做高效 email 去重（O(N) 而非 O(N²)）
能用「一次性 batch prompt」對整批 N 筆 lead 一次評分（避免 N 次 API call）
(01) WORKFLOW 全貌

節點與資料流

8 節點線性 → 三輸出 fan-out。重點在第一個 Code 節點（清洗 + 去重 + AI 評分一次處理整批）。

(02) 重點節點拆解

Code: 清洗 + 去重 + AI 評分（核心）

CORE
三段式 jsCode：normalize → dedup → batch AI

Step 1 · 欄位正規化：

// 電話自動補國碼 let phone = String(r.phone).replace(/[^0-9+]/g, ''); if (phone.startsWith('09') && phone.length === 10) phone = '+886' + phone.slice(1); // email 統一小寫 out.email = String(r.email).trim().toLowerCase();

Step 2 · email 去重（O(N)）：

const uniq = {}; for (const r of cleaned) if (!uniq[r.email]) uniq[r.email] = r; const dedup = Object.values(uniq);

用 Object key 自動覆蓋，比 array filter + indexOf 的 O(N²) 快很多（1000 筆 lead 差距很明顯）。

Step 3 · 一次性 batch prompt 評分：

// 把 50 筆 lead 一次餵給 Gemini（取代 50 次 API call） const prompt = '對下列每筆 lead 評 hot/warm/cold...\n回 JSON array：[{"score":"hot"}, {"score":"warm"}, ...]'; // 一次 API 呼叫 → 拿 50 筆評分

關鍵設計：限制只給 AI 看欄位（name/email/phone/company），不丟其他敏感資料。AI 評分標準寫在 prompt 而不是 jsCode（學員改 prompt 就能改評分邏輯）。

(03) 改造方向

5 種改造

  1. 換評分標準成你公司 ICP：產業 / 規模 / 預算範圍 / 決策角色 — 全寫在 prompt，學員改 prompt 不用改 jsCode。
  2. 分派負責人：根據 score + 公司規模 dispatch 到不同業務（Switch 節點）：hot 給 A、warm 給 B、cold 進 nurture pool。
  3. 串 CRM：cleaned.csv 改成寫進 HubSpot / Salesforce / Notion（換最後 Write 節點為 HTTP Request）。
  4. 歷史去重：跟過去 90 天的 leads-archive/ 對比，已聯絡過的 lead 標記 stale 不再評分。
  5. 多檔合併：leads-inbox/ 內多份來源（Facebook lead / 表單 / 名片掃描）一起 normalize 再合併去重。
