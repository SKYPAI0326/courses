n8n 術語 × Make 概念速查

已熟悉 Make？這頁幫你把 Make 的術語直接對應到 n8n，加速上手。每個 n8n 概念都有對應的 Make 等價物和差異說明。

📖參考頁面，可隨時查閱
⏱閱讀約 5 分鐘
核心概念對照
n8n 術語Make 對應術語說明與差異
Workflow Scenario（場景） 都是一條自動化流程的容器。n8n Workflow 儲存在本機，Make Scenario 儲存在雲端。
Node（節點） Module（模組） 都是流程中的一個步驟。n8n 節點可自由拖曳排列在畫布上，Make 模組是線性排列。
Trigger Node（觸發節點） Trigger Module（第一個模組） 啟動流程的起點。n8n Trigger 節點有閃電⚡圖示，啟動 Workflow 才開始監聽。
Connection（連線） Route（路由） 節點間的資料傳遞路徑。n8n 需手動拖曳連線，一個輸出可同時連多個下游節點。
Execution（執行記錄） Run（執行紀錄） 每次流程跑完的記錄。n8n 免費版保留最近 200 筆；Make 依方案限制。
Credential（憑證） Connection（連線） 儲存 API Key、帳號密碼的安全容器。可跨 Workflow 共用。
Expression {{ }} Dynamic Variable（變數引用） 在欄位中引用其他節點的資料。n8n 用雙大括號語法，Make 用點擊選取。
Item（項目） Bundle（捆） 一筆資料的單位。n8n 用 $json 讀取當前 Item，Make 用 {{1.欄位}}。
Loop Over Items Iterator + Aggregator 批次處理多筆資料。n8n 的 Loop Over Items 節點整合了 Iterator 和 Aggregator 的功能。
Code Node Tools → Custom Function 執行 JavaScript 自訂邏輯。n8n Code 節點支援完整 Node.js，可 require 模組。
HTTP Request Node HTTP Module 呼叫任意 API。功能相近，n8n 支援更多認證方式。
Webhook Trigger Webhooks Module（接收） 接收 HTTP 請求來觸發流程。n8n 需搭配 Cloudflare Tunnel 才能從外部呼叫。
Schedule Trigger Scheduling（定時執行） 定時觸發流程。n8n 支援 Cron 語法，Make 也有類似設定。
If / Switch Node Filter / Router 條件判斷分支。n8n If = 兩路分支；Switch = 多路分支（相當於 Make 的 Router）。
資料引用語法對照
目的n8n 語法Make 等價寫法
讀取當前節點的欄位{{ $json.欄位名稱 }}{{模組編號.欄位名稱}}
讀取指定節點的欄位{{ $('節點名稱').item.json.欄位 }}需用 Router 或直接引用上游模組編號
讀取陣列第一筆{{ $json.陣列[0].欄位 }}{{模組.陣列[].欄位}}（迭代）
字串拼接{{ '你好，' + $json.姓名 }}你好，{{1.姓名}}
現在時間{{ $now }}{{now}}
格式化日期{{ $now.toFormat('yyyy-MM-dd') }}{{formatDate(now; 'YYYY-MM-DD')}}
💡 最大不同點：Make 採 credits 計費（Free plan 1,000 credits/月，一般模組動作大致 1 credit/次，AI 相關動作另計），自架 n8n Community Edition 不按執行次數計費。這是為什麼批次處理任務（如 PDF 批次改名、大量 AI 呼叫）適合放在 n8n 而非 Make。
💼 實戰案例
情境｜從 Make 轉到 n8n 的使用者常因名詞不同而卡住（Scenario vs Workflow、Module vs Node）。
做法｜建一張對照表貼在螢幕旁或 Notion 首頁，前兩週對照著操作，一個月後自然內化。
踩雷｜特別容易混淆：Make 的 Iterator/Aggregator ≠ n8n 的 SplitInBatches——後者是為了『暫停等批次完成』，功能更接近 Make 的 Sleep+Array Aggregator 組合。
