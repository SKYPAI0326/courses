Make ↔ n8n API 串接詳解

Make 與 n8n 之間的所有資料交換都靠 Webhook + HTTP Request。本頁整理兩個方向的串接方式、認證設定、以及常見問題的排查方法。

📖補充參考
⏱閱讀約 10 分鐘
兩個方向的資料流

Make 與 n8n 之間的通訊有兩個方向，對應不同的使用情境：

Make
雲端觸發
→ HTTP POST →
n8n Webhook

→（執行後）→
Respond

→
Make

Make → n8n：傳送資料到 n8n

在 Make 中用「HTTP · Make a request」模組，向 n8n 的 Webhook URL 發送 POST 請求：

設定項目值說明
URLn8n Webhook URL（含 Cloudflare 網域）從 n8n Webhook 節點複製，格式：https://你的域名/webhook/路徑
MethodPOST固定用 POST，n8n Webhook 節點預設接受 POST
Body typeJSON（application/json）n8n 解析 JSON Body，不要用 Form Data
HeadersContent-Type: application/jsonMake 通常自動加，但確認一下
Timeout40 秒（免費）/ 60 秒（付費）超過時間 Make 會報錯；長時任務用非同步方案
// Make 傳送的 JSON Body 範例 { "task_type": "企劃書", "task_name": "2026 校慶科技展", "deadline": "2026-05-01", "requester_email": "{{1.email}}" ← Make 的動態變數 }
n8n → Make：主動 Push 通知

在 n8n 使用「HTTP Request」節點，向 Make 的 Webhook URL 發送 POST 請求：

// n8n HTTP Request 節點設定 Method: POST URL: https://hook.make.com/xxxxxxxxxxxxxxxx ← Make Webhook URL Authentication: None（Make Webhook 不需要認證） Content Type: JSON Body: { "status": "completed", "task_name": {{ $json.taskName }}, "result_url": {{ $json.docUrl }}, "completed_at": {{ $now.toISO() }} }
認證方案對比
方案設定方法適合場景
無認證 不設定，任何人知道 URL 就能呼叫 本地測試、內網環境
Token 驗證 Webhook URL 加上 ?token=密碼；在 n8n Code 節點驗證 $json.query.token 輕量安全、不需設定 Header
Header 認證 Make 發送時加 Header X-Secret: 密碼；n8n 驗證 $request.headers['x-secret'] 較規範的 API 安全方案
// n8n 驗證 Token 的 Code 節點（放在 Webhook 後第一個節點） const expectedToken = 'your-secret-token'; const receivedToken = $input.item.json.query?.token; if (receivedToken !== expectedToken) { // 驗證失敗，終止流程 return [{ json: { error: 'Unauthorized', code: 401 } }]; } // 驗證通過，繼續後面的節點 return [$input.item];
常見問題排查
問題原因解決方法
Make 顯示「Connection refused」n8n 未啟動，或 Cloudflare Tunnel 斷線確認 n8n 在執行；確認 cloudflared 進程存在（ps aux | grep cloudflared）
Make 顯示「Timeout」n8n 流程執行超過 Make 等待時間改用非同步方案：立即回傳 accepted，完成後 Push
n8n 收到空 BodyMake 傳送了 Form Data 而非 JSONMake 的 Body type 改為 JSON
n8n 解析欄位失敗（undefined）欄位名稱拼錯，或 JSON 結構層級不對在 n8n Webhook 節點的 Last execution 確認實際收到的 JSON 結構
Make 收到 n8n 回傳的 HTML 錯誤頁n8n Workflow 未啟用（Active），Webhook 回傳 404確認 n8n Workflow 右上角 Toggle 為 Active
✅ 除錯技巧：用 Webhook.site（免費工具）接收 Make 的 POST，確認 Body 格式正確後，再換成真正的 n8n URL。反方向也一樣——先把 n8n 的 HTTP Request 打到 Webhook.site 確認格式，再換成 Make 的 URL。
💼 實戰案例
情境｜某地方旅遊網整合中央氣象局 API + Google Maps API 產出『今天適合去哪玩』的每日推薦。
做法｜氣象局 API 免費（需註冊 CWA 授權碼）+ Google Maps 每月 $200 免費額度夠用。整合後站點訪客停留時間 +40%。
踩雷｜API key 不要寫死在 HTTP Request 節點——放 Credentials 集中管理，萬一外洩只要改一處、不用翻遍每個 workflow。
