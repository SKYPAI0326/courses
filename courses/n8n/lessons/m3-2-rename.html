PDF 批次改名：AI 辨識內容自動命名

結合 Gemini API，讓 AI 讀取 PDF 內容、提取廠商名稱與金額，自動按規則重命名整批文件。

實際效果 · BEFORE / AFTER
scan_20260315_001.pdf → 2026_合約_廠商A_50萬.pdf
document_final_v3.pdf → 2026_採購單_供應商B_3.2萬.pdf
STEP BY STEP · 操作步驟
本節流程概覽

1
Read Binary File 節點：讀取 PDF 為二進位資料
承接 3.1 的 If 節點 True 分支，新增「Read Binary File」節點。在「File Path」欄位，使用 Watch Folder 傳來的路徑：{{ $json.data.path }}。執行後，PDF 會被讀取為 Base64 編碼的二進位字串，儲存在 $binary.data.data。
提示：此節點將 PDF 轉換為 Base64，這是上傳至 Gemini API 所需的格式。在 Output 面板確認 binary 欄位已出現。
2
HTTP Request 節點：呼叫 Gemini API 辨識 PDF
新增「HTTP Request」節點，Method 設為 POST，URL 填入 Gemini API 端點（含你的 API Key）。Body 類型選「JSON」，貼入以下結構，讓 Gemini 讀取 PDF 並回傳結構化 JSON。
// URL（替換 YOUR_API_KEY） https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash:generateContent?key=YOUR_API_KEY // Body（JSON 格式） { "contents": [{ "parts": [ { "text": "請解析此 PDF，提取以下資訊並以 JSON 格式回傳，不要包含 markdown：{\"vendor\": \"廠商名稱\", \"amount\": \"金額\", \"date\": \"日期YYYYMMDD\", \"type\": \"文件類型\"}" }, { "inlineData": { "mimeType": "application/pdf", "data": "{{ $binary.data.data }}" } } ] }] }
提示：在 Google AI Studio 的「Get API Key」取得免費 API Key。Gemini 2.0 Flash 可直接讀取 PDF 圖文內容，不需要額外 OCR 節點。免費方案每分鐘上限 15 次請求，批次處理多個 PDF 時，在 Loop 內加入「Wait」節點（間隔設為 5 秒）可避免觸發速率限制。
3
Code 節點：解析 Gemini 回傳的 JSON
新增「Code」節點（Language 選 JavaScript），從 HTTP Request 的回傳中提取 AI 辨識結果。Gemini 的回傳結構為 candidates[0].content.parts[0].text，將其解析為 JSON 物件。
// Code 節點內容 const rawText = $input.item.json.candidates[0].content.parts[0].text; const info = JSON.parse(rawText); const originalPath = $('Watch Folder').item.json.data.path; const dir = originalPath.substring(0, originalPath.lastIndexOf('/')); // 組合新檔名 const newName = `${info.date}_${info.type}_${info.vendor}_${info.amount}.pdf`; const newPath = `${dir}/${newName}`; return [{ json: { originalPath, newPath, newName, ...info } }];
提示：若 Gemini 回傳的文字包含 ```json ... ``` 標記，可用 rawText.replace(/```json\n?|\n?```/g, '') 先清除再 JSON.parse。
4
Move/Rename File 節點：依 AI 結果重命名
新增「Move File」節點。「Source Path」填入原始路徑 {{ $json.originalPath }}，「Destination Path」填入新路徑 {{ $json.newPath }}。執行後，原 PDF 檔案將在原位被重命名。
提示：若想保留原始檔案，在 Move File 前先用「Copy File」節點備份到另一個資料夾，再對備份執行重命名。
5
用 Loop 節點批次處理多個 PDF
若需要一次處理資料夾內所有已存在的 PDF，在流程開始改用「Read All Files In Folder」節點取得檔案清單，然後連接「Loop Over Items」節點，將步驟 1-4 的節點包在 Loop 內，即可自動迭代處理每一個 PDF 檔案。
// Read All Files In Folder 設定 Path: /Users/你的名稱/Documents/待處理PDF File Extension: pdf Return: File paths as strings
提示：Watch Folder 每次觸發處理單一新增檔案；若要批次處理已有的 PDF，改用 Read All Files + Loop 的組合。
完成驗證 · VERIFY
放入一個 PDF 後，觀察資料夾中的檔名是否從原始亂名變成如 20260315_合約_台積電_500萬.pdf 的格式。若 Gemini 辨識的資訊欄位回傳 "未知"，代表 PDF 內容不清晰，可調整 Prompt 加上「若無法辨識請回傳 unknown」。
常見問題 · TROUBLESHOOT
Gemini API 回傳 400 錯誤，或說 inlineData 格式不正確？
確認 HTTP Request 節點的 Body 類型是「JSON」而非「Form Data」。另外檢查 $binary.data.data 是否有值（在 Read Binary File 的 Output 面板確認有 binary 欄位）。API Key 是否正確放在 URL 參數中。
Code 節點 JSON.parse 失敗，報錯 SyntaxError？
Gemini 有時會在 JSON 外包 markdown 標記（```json ... ```）。在 Code 節點加入清除邏輯：const clean = rawText.replace(/```[\w]*\n?/g, '').trim()，再對 clean 做 JSON.parse。
