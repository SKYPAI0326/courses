#05 · TELEGRAM NOTIFY · 3 NODES · 基礎

最簡 Telegram bot 推訊息：3 節點驗證 credentials 通暢

Lite Pack 最簡單的 workflow，但學員必看 — setup-wizard 的 smoke test 用的就是這個 workflow，跑通就證明 Telegram credentials + chat_id placeholder 替換都正確。明天課堂第一個 demo 也是這個（最快讓學員看到結果）。

學習成果
能用 @BotFather 申請 Telegram bot 取得 token
能在 n8n 設定 Telegram credentials 並串接 Telegram 節點
理解 Telegram 節點 typeVersion 1.2 的 parse_mode 雷與 Lite Pack 的解法
(01) WORKFLOW 全貌

節點與資料流

Manual Trigger → Set: 訊息內容 → Telegram: Send Message

3 節點線性鏈，零 AI、零外部依賴。Manual Trigger 觸發 → Set 節點組訊息字串 → Telegram 節點呼叫 Telegram Bot API 送訊息到指定 chat_id。setup-wizard 跑完會自動 Execute 一次當 smoke test。

(02) NODE BREAKDOWN

節點逐一拆解

NODE 1 / 3
Manual Trigger

跟 #02 一樣，純粹的「點 Execute 觸發」按鈕。沒任何設定。

NODE 2 / 3
Set: 訊息內容
set v3.4

用途：組合一個固定字串當訊息。

關鍵設定：Assignments 區設一個欄位 messageText = "✅ Lite Pack 通知測試成功\n\n如果你看到這則訊息，代表 Telegram bot token + chat_id 都設對了。"

注意 \n 是真的換行符（Telegram 會解析成換行）。中文字元、emoji 都安全（不會踩 entities 雷）。

NODE 3 / 3
Telegram: Send Message
telegram v1.2

關鍵設定：

  • Resource: Message
  • Operation: Send Message
  • Chat ID: __TELEGRAM_CHAT_ID__（setup-wizard 自動替換成你的 chat_id）
  • Text: {{ $json.messageText }}（從上一節點 Set 取訊息）
  • Credential: Lite Pack · Telegram Bot（含 bot token）
  • Additional Fields: 留空（不加 Parse Mode 欄位）
(03) DESIGN RATIONALE

為什麼這樣設計（含踩過的雷）

🚨
連發 #15 的雷：Telegram parse_mode 沒有 None 選項
n8n Telegram Node typeVersion 1.2 的 Parse Mode dropdown 只給 3 個選項：Markdown (Legacy) / MarkdownV2 / HTML，沒有 None。如果動態訊息內含 * _ [ 字元（AI 回應常見），任何一種 parse_mode 都會炸 Bad Request: can't parse entities。
解法：Additional Fields 完全留空（不加 Parse Mode 欄位），這樣 n8n 不傳 parse_mode 給 Telegram API，Telegram API 預設不解析訊息 → markdown 字元當純文字顯示。
💭
為什麼 #05 訊息固定字串而不用 AI 生？
#05 是 setup-wizard 的 smoke test，目的是驗證 credentials + chat_id 串接通暢，不是測 AI。固定字串訊息保證 100% 可預測，學員看到「✅ Lite Pack 通知測試成功」就知道 Telegram 部分成功。AI 動態訊息留給 #06 webhook AI workflow。
(04) 改造方向

5 種改造方向

  1. 串到別的 workflow 當「結尾通知」：複製這個 Telegram 節點貼到 #02 / #03 / #04 結尾，跑完 workflow 自動推 Telegram「✅ 處理完成 N 份」。
  2. 改 Resource 推照片 / 檔案：Telegram 節點可選 Resource: Photo / Document / Audio，從 binary input 推給 Telegram。例如 #02 改名後直接推 PDF 縮圖到 Telegram。
  3. 串 Webhook 接 Telegram bot 指令：複製 #06 的 Webhook 節點開 endpoint，加上 Telegram bot 的 setWebhook 設定，讓 bot 收到指令時觸發 workflow（例如打 /report 自動跑 #04 日報）。
  4. 多 chat 群推：把 chat_id 設成 array，loop 推到多個個人 / 群組 / channel。
  5. 替換 Slack / Discord / Line：n8n 內建 Slack / Discord 節點介面類似，credentials 換掉 + 節點換掉就能轉移。
