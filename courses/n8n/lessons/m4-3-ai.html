FINAL PROJECT · 4.3

AI 計劃生成器：課程壓軸實戰

整合所有 Module 的技能——Webhook 接收需求，Gemini 生成完整行動計劃，自動存檔並發送通知。從一個點子到成品，全程自動。

⏩ 時間調配提示：本節為課程壓軸，若前面環節超時，可改由講師現場 Live Demo 完整流程，學員跟著觀摩 Prompt 設計與節點邏輯，課後再自行照步驟實作。重點放在理解「從表單到成品」的全鏈路概念，而非當場完成每個節點設定。
最終成果 · 完整流程
①用手機填寫 Google 表單，輸入一個活動點子（如「2026 校慶科技展」）
②Make 接收表單資料，HTTP 轉送到本機 n8n
③n8n 呼叫 Gemini API，AI 生成結構化行動計劃（JSON 格式）
④解析 AI 回傳，自動格式化成 Markdown 任務清單
⑤Write File 儲存到本機，HTTP 回傳 Make → Gmail 發送完成通知
STEP BY STEP · 操作步驟
1
Google Forms：建立計劃需求表單
建立新的 Google Forms 表單，加入以下欄位：專案名稱（如「2026 校慶科技展」）、目標（如「吸引 500 人參與，展示 AI 應用成果」）、期望交付物（如「活動企劃書、分工表、時程表」）、負責人。提交後由 Make 接收並轉送到 n8n（方法與 4.1 相同）。
提示：如果已完成 4.1，可以直接在原有表單加入「目標」和「期望交付物」欄位，Apps Script 會自動帶入所有欄位。
2
設計能穩定輸出 JSON 的 Prompt
Prompt Engineering 是本步驟的核心。要讓 Gemini 穩定回傳可解析的 JSON，必須在 Prompt 中明確指定格式、禁止 markdown，並提供欄位範例。
Prompt 範本
你是一位專業的專案管理顧問。請針對以下專案需求，生成一個詳細的行動計劃，並以純 JSON 格式回傳（不要包含 markdown 標記或額外說明文字）。 專案名稱：[專案名稱] 目標：[目標] 期望交付物：[期望交付物] JSON 格式如下： {"project": "...", "objective": "...", "tasks": [{"task": "任務名稱", "owner": "建議負責人角色", "start": "YYYY-MM-DD", "end": "YYYY-MM-DD", "status": "未開始", "deliverable": "產出"}]}
提示：明確寫「不要包含 markdown 標記」是關鍵。若 Gemini 仍然輸出 ```json ... ```，在 Code 節點加入清除 markdown 的邏輯。
3
HTTP Request 節點：呼叫 Gemini API 生成計劃
在 n8n Webhook 後新增「HTTP Request」節點，呼叫 Gemini API。將表單欄位動態插入 Prompt，讓 AI 針對這次的具體需求生成計劃。
// HTTP Request 設定 Method: POST URL: https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash:generateContent?key=YOUR_KEY // Body（JSON） { "contents": [{ "parts": [{ "text": "你是一位專業的專案管理顧問。請針對專案「{{ $json.body.project_name }}」，目標「{{ $json.body.objective }}」，期望交付物「{{ $json.body.deliverables }}」，生成詳細行動計劃。以純 JSON 格式回傳，不要包含 markdown：{\"project\":\"...\",\"tasks\":[{\"task\":\"...\",\"owner\":\"...\",\"start\":\"YYYY-MM-DD\",\"end\":\"YYYY-MM-DD\",\"status\":\"未開始\",\"deliverable\":\"...\"}]}" }] }] }
提示：Gemini 2.0 Flash 速度快且免費，適合這類結構化輸出任務。若 AI 生成的計劃不夠詳細，在 Prompt 中加入「請生成至少 6 個具體任務，每項任務需說明負責人角色與具體產出」，無需升級模型即可提升輸出品質。
4
Code 節點：解析 AI 回傳並格式化成 Markdown
用 Code 節點解析 Gemini 回傳的 JSON，然後把結構化任務清單轉換為易讀的 Markdown 格式——包含標題、目標說明、任務表格，以及任務狀態清單。
// Code 節點 const raw = $input.item.json.candidates[0].content.parts[0].text; const clean = raw.replace(/```[\w]*\n?|\n?```/g, '').trim(); const plan = JSON.parse(clean); let md = `# 行動計劃：${plan.project}\n\n`; md += `**目標：** ${plan.objective ?? '未指定'}\n\n`; md += `## 任務清單\n\n`; plan.tasks.forEach(t => { const done = t.status === '已完成' ? 'x' : ' '; md += `- [${done}] **${t.task}**\n`; md += ` - 負責人：${t.owner}　期間：${t.start} ~ ${t.end}\n`; md += ` - 產出：${t.deliverable}\n\n`; }); const ts = new Date().toISOString().slice(0,10).replace(/-/g,''); const fileName = `${ts}_${plan.project.replace(/\s/g,'_')}.md`; return [{ json: { md, fileName, project: plan.project } }];
提示：?.（Optional Chaining） 在這裡很有用——如果 AI 漏掉某個欄位，不會導致整個流程崩潰（例如 t.owner ?? '待指派'）。
5
Write File 儲存計劃文件
用「Convert to File」節點把 Markdown 字串轉為二進位，再用「Write Binary File」儲存到本機的計劃資料夾。檔名包含日期和專案名，格式如 20260402_校慶科技展.md。
// Write Binary File 設定 File Name: /Users/你的名稱/Documents/AI計劃/{{ $json.fileName }} Data: data（Convert to File 的輸出）
提示：Markdown 文件可以直接在 VS Code、Typora 或 GitHub 開啟並渲染，比純 txt 更專業。
6
回傳 Make → 發送 Gmail 完成通知
最後，n8n 把計劃檔名和專案名回傳給 Make（方法與 4.2 相同），Make 發送 Gmail 通知，告知計劃已生成完畢，並附上檔案儲存位置的說明。
// n8n → Make 回傳 Body { "status": "success", "project": "{{ $json.project }}", "fileName": "{{ $json.fileName }}" }
提示：這是整個課程最後一個節點！完成後可以嘗試用手機填表，全程不碰電腦，看著文件自動生成。
完成驗證 · VERIFY
填寫 Google Forms 提交後：（1）本機「AI計劃」資料夾出現 Markdown 文件，（2）打開文件確認任務清單結構完整（有任務名稱、負責人、日期），（3）Gmail 收到完成通知。三點全部達成，代表課程壓軸實戰成功。
恭喜完成整門課程
你已整合了 Module 1 的環境建置、Module 2 的數據引用與邏輯判斷、Module 3 的批次文件處理，以及 Module 4 的雲地協作與 AI 生成——打造出一套屬於自己的全自動 AI 工作流程。
這不是終點，而是你探索更多可能性的起點。
常見問題 · TROUBLESHOOT
Gemini 回傳的 JSON 格式不穩定，有時包含 markdown 有時不包含？
在 Code 節點加入標準化清除邏輯：const clean = raw.replace(/```[\w]*\n?|\n?```/g, '').trim()，無論有沒有 markdown 包裝都能安全解析。同時把 JSON.parse 包在 try-catch 中，失敗時記錄原始 raw 方便 debug。
AI 生成的任務清單只有 2-3 項，感覺不夠詳細？
在 Prompt 中加入數量要求，如「請生成至少 6 個具體任務」。另外在「目標」欄位填入更多細節，AI 會根據輸入的豐富程度調整輸出深度。
整個流程跑到一半停了，不知道哪個節點出錯？
在 n8n 的「Executions」頁籤查看最近的執行記錄，點擊失敗的執行可以看到每個節點的輸入/輸出，找到第一個紅色錯誤節點逐一排查。
