#08 · EXPRESSION PRACTICE · 4 NODES · 基礎

n8n Expression 語法速查表：改造其他 workflow 必看

這個 workflow 不解決業務問題，是「Expression 速查表」— 把 n8n {{ }} 內常用的 10 種寫法集合在一個 workflow，學員可以複製其中任何一個範例直接貼到自己 workflow 的欄位。當你看不懂 #02 / #03 / #06 內某個 expression 寫法時，回來這頁查。

學習成果
能用相對引用 {{ $json.xxx }}、絕對引用 {{ $('NodeName').first().json.xxx }} 取資料
能用 Optional Chaining ?. 處理可能缺失的巢狀欄位避免 undefined 錯誤
能用 n8n 內建變數（$now, $itemIndex, $workflow）+ JavaScript 函式（split / map / filter）
(01) WORKFLOW 全貌

節點與資料流

Manual Trigger → Set: 假資料（巢狀物件 + array） → Set: 10 種 expression 範例 → Code: 複合資料處理

4 節點線性鏈。第 2 節點 Set 假資料當「上游」（含 user / orders / metadata 巢狀結構），第 3 節點 Set 用 10 種 expression 寫法各自抓上游某個欄位（學員看 Output 對照寫法 → 結果），第 4 節點 Code 用 JavaScript 處理同樣資料示範更複雜的邏輯。

(02) 10 種 expression 寫法速查表

最常用的 10 種，搭配 #08 workflow 看實際輸出

用途Expression 寫法結果範例
取上一節點的 json 欄位{{ $json.user.name }}"小明"
取指定節點的 json 欄位{{ $('Set: 假資料').first().json.user.email }}"ming@example.com"
Optional Chaining（防 undefined）{{ $json.user?.address?.city ?? "未知" }}"未知"（如果 address 不存在）
取 array 第 N 個元素{{ $json.orders[0].id }}"ORD-001"
array 長度{{ $json.orders.length }}3
當前時間 ISO 字串{{ $now.toISO() }}"2026-05-06T08:30:00..."
當前時間自訂格式{{ $now.toFormat('yyyy-MM-dd') }}"2026-05-06"
當前 item 的 index（multi-item）{{ $itemIndex }}0, 1, 2...
workflow 名稱{{ $workflow.name }}"08 · Reference 練習"
JavaScript 內建函式{{ $json.user.name.toUpperCase() }}"小明"（中文不變）/ "MING"
💡
n8n Expression 完整能用什麼：所有 JavaScript 內建（Math / String / Array / Date methods）+ Luxon DateTime（$now / $today）+ n8n 內建變數（$json / $node / $workflow / $execution / $itemIndex）+ 引用語法 $('NodeName').first().json / .all() / .item.json。
(03) DESIGN RATIONALE

為什麼要把 expression 集合在一個 workflow

💭
n8n 文件不錯但散：
n8n 官方有完整 Expression 文件，但分散在 Code Node / Set Node / built-in methods / Luxon DateTime 等多個頁面。學員碰到 {{ $json.xxx?.yyy }} 這種寫法時，要查 5 個頁面才能拼出全貌。#08 把最常用的 10 種放一個 workflow，學員執行一次看 Output 就懂全部。
💭
為什麼要 4 個節點而不是 1 個：
第 2 節點假資料是「上游 context」（學員可改成自己業務資料），第 3 節點 Set 是「expression 表達」（學員可複製貼到自己 workflow 欄位），第 4 節點 Code 是「JavaScript 完整版」（複雜邏輯走 Code Node）。3 種寫法的等效對照讓學員理解同樣事情可以用 expression 寫也可以用 jsCode 寫，依複雜度選工具。
(04) 改造方向

3 種改造方向

  1. 把假資料換成真實業務資料：第 2 節點改成你業務的 JSON 結構（學員 CRM 資料 / 訂單記錄 / 文章 metadata），第 3 節點重寫 expression 抓你需要的欄位。這就是「我業務專屬的 expression 速查表」。
  2. 當 sandbox 測試新寫法：碰到不確定的 expression 寫法，先在這個 workflow 試（Set 節點欄位填 {{ ... }} → Execute → 看 Output）。確認對了再貼到生產 workflow。
  3. 學 Luxon DateTime：Set 節點各種 $now.plus({days: 7}).toFormat(...) 寫法測試，做日期計算（這個月最後一天 / 下一個工作日 / 7 天前）。
