多 Workflow 多 Public Hostname

為不同的 Workflow 設定獨立的 Public Hostname，有效區隔測試與正式環境，或讓多個專案各自擁有穩定的 Webhook 入口。

📖進階參考，工作坊後延伸學習
⏱閱讀約 8 分鐘
為什麼需要多個 Hostname？

一個 Cloudflare Tunnel 預設提供一個 Public URL，所有 Webhook 都指向同一個入口（localhost:5678）。n8n 本身會根據 Webhook 路徑區分不同的 Workflow，大多數情況下一個 URL 就夠了。

但在以下情境，你可能需要多個不同的 Hostname：

測試 vs 正式環境
test.你的名稱.com
讓 Make 的測試場景打測試 URL，正式場景打正式 URL，避免測試資料干擾正式流程。
多專案獨立管理
project-a.你的名稱.com
不同客戶或專案各自有一個 Hostname，便於管理與除錯，也方便日後停用特定專案。
設定方法

在 Cloudflare Zero Trust 後台，一個 Tunnel 可以設定多個 Public Hostname，每個 Hostname 可以指向不同的本機服務（甚至不同的 Port）。

設定位置操作步驟
Cloudflare Zero Trust → Networks → Tunnels 點擊你的 Tunnel → 「Configure」→ 「Public Hostname」→「Add a public hostname」
新增第二個 Hostname Subdomain 填新名稱（如 test）、Domain 選你的域名、Service 填 http://localhost:5678，儲存
驗證 用瀏覽器訪問新的 URL，能看到 n8n 回應即成功
💡 路徑比 Hostname 更簡單：大多數情況，不需要多個 Hostname。只要在 n8n Webhook 節點設定不同的 Path（如 /webhook/test-flow 和 /webhook/prod-flow），就能用同一個 URL 區分不同 Workflow。
命名建議
用途Hostname 範例說明
主要 / 正式n8n.你的名稱.com給 Make 正式場景使用
測試n8n-test.你的名稱.com給 Make 測試場景使用，流程出錯不影響正式
特定專案project-abc.你的名稱.com長期合作的特定客戶或專案
⚠️ 免費版限制：Cloudflare 免費版的 Tunnel 功能完整，但需要有一個已在 Cloudflare 管理的域名（可用自己購買的便宜域名，不用 .com，.xyz 每年約 $1 美金）。若沒有域名，只能用 1.2 主頁面介紹的預設隨機 URL。
💼 實戰案例
情境｜某設計工作室三台 Mac 共用同一條 Cloudflare Tunnel，誰在家值班誰接 Webhook。
做法｜每台設 Public Hostname mac-a.studio.com、mac-b.studio.com、mac-c.studio.com，Make 用 Router + 變數決定發給誰。
踩雷｜Hostname 打錯就完全收不到——不會報錯、只會靜默失敗。規則：設完立刻用瀏覽器打開 https://你的hostname/，要能看到 n8n 主介面才算通。
