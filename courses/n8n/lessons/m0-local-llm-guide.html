補充章節：把 Lite Pack 的 Gemini 換成本機 Ollama

Lite Pack 14 個 workflow 預設用 Gemini API（雲端），對「處理客戶機密 / 員工資料 / 合約 / 商務戰略」場景，資料離開本機進到 Google 是不能接受的。這頁教你 30 分鐘把所有 AI 呼叫換成本機 Ollama —— Llama 3 / Gemma 2 / Qwen 2.5 等開源模型，所有 prompt + 資料都不離開你電腦。是 n8n 自架最關鍵的差異化能力。

學習成果
能在 Mac / Windows 安裝 Ollama 並下載 Llama 3.2 / Gemma 2 等開源模型
能改 jsCode 內 Gemini URL 為 Ollama 端點，讓 14 個 workflow 全部走本機 LLM
能評估「哪些 workflow 適合本地 LLM、哪些仍走雲端 API」的工程取捨
(01) 為什麼要本機 LLM

三個情境：本地 LLM 是唯一解

不是每個場景都要本地
本機 LLM 不是萬靈丹。質量普遍比 Gemini 2.5 / GPT-5 / Claude 弱（Llama 3.2 70B 約 ChatGPT 3.5 等級），且需要 16GB+ RAM。但對下列三種情境是唯一解 — 沒得選。
情境為什麼必須本地適用 workflow
客戶 NDA / 合約處理合約內容含對方公司機密，丟到 Google AI 違反 NDA#02 / #03 / #10
員工 / 客戶 PII含身份證、薪資、客戶聯絡資料，資安規範禁止離開本機#11（lead 評分）/ #09（信件分類）
商務戰略 / 產品路線圖競爭情報，丟到 Google 等於告訴 Google 你公司做什麼#04（日報）/ #12（知識庫）/ #13（營運快照）
純技術 demo / 公開資料沒機密 — 用 Gemini 2.5 Flash 質量更好且免費 tier 夠用#06 / #07 / #08（保留雲端）
🎯
建議混搭策略：14 個 workflow 中含敏感資料的 5-7 個換成 Ollama，其他保持 Gemini。同個 n8n 內可以混用。
(02) OLLAMA 安裝（Mac + Windows）

15 分鐘下載 + 跑起來

STEP 1 · 安裝 Ollama
Mac / Windows 一鍵安裝包

到 https://ollama.com/download 下載對應系統的安裝包：

  • Mac：Ollama-darwin.zip → 解壓 → 拖到 Applications → 第一次打開要授權
  • Windows：OllamaSetup.exe → 雙擊執行 → 一路 Next

裝完選單列 / 工作列會有 Ollama icon 表示在跑。預設 listen 在 http://localhost:11434。

STEP 2 · 下載模型
挑一個適合你電腦的

終端機跑（會下載對應大小的模型檔，約 2-40 GB）：

# 推薦：Llama 3.2 3B（小快，4GB RAM 機器都跑得動） ollama pull llama3.2 # 中型：Gemma 2 9B（質量比 3B 好很多，需要 12GB+ RAM） ollama pull gemma2:9b # 大型：Llama 3.1 70B（接近 ChatGPT 4 質量，需要 48GB+ RAM） ollama pull llama3.1:70b
模型大小RAM 需求適合 workflow
llama3.2 (3B)2 GB4 GB+分類 / 標籤（#09/#10/#11）
gemma2:9b5.4 GB12 GB+摘要 / 命名（#02/#03/#04）
qwen2.5:14b9 GB16 GB+中文任務（推薦給繁中使用者）
llama3.1:70b40 GB48 GB+複雜推理（#12 RAG / #14 診斷）
STEP 3 · 測試 Ollama API
確認本地 LLM 能用

終端機跑：

curl -X POST http://localhost:11434/api/generate \ -H "Content-Type: application/json" \ -d '{"model":"llama3.2","prompt":"用一句話介紹 n8n","stream":false}'

應該回 JSON 含 "response": "n8n 是..."。回對了表示 Ollama 跑起來。

(03) 改 LITE PACK WORKFLOW

把 jsCode 內 Gemini URL 換 Ollama

3 處關鍵差異
Gemini API 跟 Ollama API 是不同 schema。改 jsCode 要動 3 個地方：URL / headers / body 結構。response parse 也不同。

Diff 對照

-url: 'https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent',
+url: 'http://host.docker.internal:11434/api/generate',
-headers: { 'Content-Type': 'application/json', 'x-goog-api-key': GEMINI_API_KEY },
+headers: { 'Content-Type': 'application/json' },
-body: { contents: [{role:'user', parts:[{text: prompt}]}], generationConfig: {...} },
+body: { model: 'llama3.2', prompt: prompt, stream: false, options: { temperature: 0.4 } },
-aiResp = response.candidates?.[0]?.content?.parts?.[0]?.text || '';
+aiResp = response.response || '';
🚨
Docker 內 n8n 連本機 Ollama 的關鍵：n8n 跑在 Docker container 內，localhost:11434 指的是 container 自己（沒 Ollama）。要用 host.docker.internal:11434 從 container 連到主機的 Ollama。Mac 跟 Windows 的 Docker Desktop 都支援這個 hostname。

完整改寫範例（#02 PDF AI 改名）

// 原版（Gemini） const response = await this.helpers.httpRequest({ method: 'POST', url: 'https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent', headers: { 'Content-Type': 'application/json', 'x-goog-api-key': GEMINI_API_KEY }, body: { contents: [{ role: 'user', parts: [{ text: prompt }] }], generationConfig: { temperature: 0.4, maxOutputTokens: 200, thinkingConfig: { thinkingBudget: 0 } } }, json: true }); const aiResp = response.candidates?.[0]?.content?.parts?.[0]?.text || ''; // 改版（Ollama 本機 LLM） const response = await this.helpers.httpRequest({ method: 'POST', url: 'http://host.docker.internal:11434/api/generate', headers: { 'Content-Type': 'application/json' }, // 不用 API key body: { model: 'llama3.2', // 換你 ollama pull 過的模型 prompt: prompt, stream: false, options: { temperature: 0.4 } }, json: true, timeout: 120000 // 本機 LLM 比雲端慢，timeout 拉長 }); const aiResp = response.response || '';
(04) 一鍵替換 14 個 workflow（進階）

用 sed 批次改 source workflow JSON

適合進階學員
14 個 workflow 內 jsCode 都用 generativelanguage.googleapis.com。如果你決定全部換本機 LLM，可以用 sed 一次批次替換 source 後重新 import。比 UI 一個個改快 10 倍。
# 1. 進到 lite-pack 解壓資料夾 cd ~/Downloads/n8n-lite-pack/workflows/ # 2. 備份 cp -r . ../workflows-backup-gemini/ # 3. 批次替換 URL（macOS 用 sed -i ''；Linux 用 sed -i） for f in *.json; do sed -i '' 's|https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent|http://host.docker.internal:11434/api/generate|g' "$f" done # 4. body 結構就要手動改了（grep 找出哪些 jsCode 還含 Gemini schema） grep -l 'contents.*parts' *.json # 然後針對每個檔案手動改 body schema + response parse # 5. 用 docker exec 重 import for f in *.json; do docker cp "$f" n8n-starter-kit-n8n-1:/tmp/$f docker exec -u node n8n-starter-kit-n8n-1 n8n import:workflow --input=/tmp/$f done
⚠️
實務建議：不要 14 個一次全換。先換最敏感的 1-2 個（#02 合約改名 / #11 lead 評分），用 1 週驗證 Llama 質量是否夠你業務用，再決定要不要全換。
(05) 質量取捨

本機 LLM vs 雲端 API 真實對比

維度Gemini 2.5 Flash（雲端）Llama 3.2 / Gemma 2（本機）
速度1-3 秒5-30 秒（依模型大小 + RAM）
質量（中文任務）優（接近 GPT-4）中（Qwen 2.5 中文表現最好）
質量（結構化 JSON 輸出）非常穩定偶爾飄（要加更多 fallback parser）
成本免費 tier 通常夠 / 超過後 $/1K tokens$0（電費 + RAM 折舊忽略不計）
隱私內容到 Google（Free tier 用於改善 model）完全本機，零外傳
離線可用否（要連 Google）✅ 飛機上 / 沒網路也能跑
RAM 占用0（雲端）運行時 4-48 GB
🎯
實務最佳策略：含敏感資料的 workflow（含合約 / 客戶 / 員工 / 戰略）→ 本機 Ollama；公開資料 / 純技術任務 / 高 throughput 場景 → Gemini。一個 n8n 同時用兩種 AI，根據 workflow 性質選擇。
(06) 進階補充

3 個進階銜接

ADV 1
自家伺服器 + Ollama（多人共用）

個人筆電跑 Ollama 不夠快 + 多人重複下載模型浪費。可以在公司 NAS / 內網伺服器跑 Ollama，所有人 n8n 連同個 endpoint。把 jsCode 內 URL 改成 http://192.168.1.100:11434/api/generate。

注意：Ollama 預設只 listen 127.0.0.1。要對內網開放需設環境變數 OLLAMA_HOST=0.0.0.0:11434 後重啟。

ADV 2
用 Embeddings 升級 #12 RAG

#12 用「關鍵字 score」做 retrieval（窮人版 RAG）。進階版可換 Ollama embeddings：

curl http://localhost:11434/api/embeddings \ -d '{"model":"nomic-embed-text","prompt":"你的文字"}' # 回 768 維向量

建索引時把每個 chunk 算 embedding 存成 JSON。問答時把 question 也算 embedding，cosine similarity 取 top K。能處理「同義詞 / 換句話問」。

ADV 3
混合策略：先本機後雲端

Code 節點 try/catch 結構：先打本機 Ollama → 質量太差 / fallback parser 偵測 fallback → 再打 Gemini 重做。讓敏感資料優先走本機，邊界模糊的才丟雲端。

let aiResp = await tryOllama(prompt); if (parseMethod(aiResp) === 'fallback-empty') { aiResp = await tryGemini(prompt); // 質量不夠才打雲端 }
總結

本地 LLM 是 n8n 自架最關鍵的差異化能力

Make / Zapier / 其他 SaaS 自動化工具都做不到「資料完全不離開本機」，因為他們本身就是雲端服務。n8n 自架在你電腦 / NAS / 私有伺服器上，配合 Ollama 等本地 LLM，能做出真正「敏感資料管線」。

這是承接 Make 的學員應該帶走的核心 mindset 升級：自動化不只是「在 SaaS 之間搬資料」，是「打造自己擁有的資料處理系統」。
