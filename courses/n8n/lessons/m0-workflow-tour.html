M0 · WORKFLOW TOUR

14 個 workflow 設計拆解總覽：每個都是一個可改造範本

Lite Pack 給你 14 個現成 workflow + 本地 LLM 銜接指引，讓你看得懂「自動化的設計選擇」。從基礎 webhook、AI 整合、批次工程化、本機檔案系統、自家 RAG、長期營運監控到 ops 自動化全覆蓋。每個拆解頁含 4 個區塊：workflow 全圖 → 節點逐一解釋 → 為什麼這樣設計（含踩過的雷）→ 改造方向。

(02) 學習路徑建議

不要 8 個一次看完，依目標選路徑

為什麼分路徑
8 個 workflow 涵蓋的設計概念深淺不同。如果你是「明天課堂前先快速搞懂」，看 3 個就夠；如果你想「徹底改造成自己業務」，那 8 個都要看且要動手改。
PATH A · 1 小時速通
趕時間先看這 3 個

看完能理解 Lite Pack 主要 wow point + 知道 AI workflow 一般長相。

  1. #2 PDF AI 改名 — AI 整合 + fallback 設計
  2. #3 批次處理錯誤恢復 — 多檔批次 + dual-path 路由
  3. #6 Webhook → AI → 三輸出 — webhook 觸發 + 多分支輸出
PATH B · 2-3 小時系統學
想徹底搞懂 AI 自動化的這 5 個

看完能 0 起手寫類似 workflow + 知道踩雷時怎麼修。

  1. 先看 #5 Telegram 通知 — 最簡 3 節點，理解節點 + binding 概念
  2. 再看 #1 Webhook hello-world — Trigger / Code / Respond 三件套
  3. 進入 #2 PDF AI 改名 — AI 整合的 standard pattern
  4. 進入 #4 定時 AI 日報 — Schedule + Aggregate + AI
  5. 進階 #3 批次處理 — 雙路由 + 容錯（Lite Pack 最複雜的一個）
PATH C · 半天動手改
想改造成自己業務的全部都看

看完能把 8 個 workflow 改成自己業務的版本（換 prompt、換資料來源、加 webhook 觸發、串自家 API）。

  1. 依序看 #1 → #2 → #3 → #4 → #5 → #6 → #7 → #8
  2. 每看完一個 workflow，去 n8n UI 對照同個 workflow 看 canvas 跟 jsCode
  3. 每個 workflow 拆解頁最後有「改造方向」清單，挑一個改造練手
  4. 遇到不懂的節點概念回去看 Module 1 / Module 2 / Module 3 / Module 4 對應章節
