n8n Expression 語法速查

常用 Expression 語法整理：資料讀取、字串處理、日期格式化、數字計算、條件表達式，一頁查完不用翻文件。

📖速查參考，可收藏常用頁面

讀取巢狀欄位{{ $json.層一.層二.欄位 }}用點號逐層往下
讀取陣列第一筆{{ $json.陣列[0].欄位 }}索引從 0 開始
讀取指定節點的資料{{ $('節點名稱').item.json.欄位 }}跨節點引用，節點名稱需與畫布一致

找不到時給預設值{{ $json.欄位 ?? '預設值' }}欄位為 null/undefined 時回傳預設值

// 三元運算子：條件 ? 成立值 : 否則值 {{ $json.金額 > 1000 ? '大額' : '一般' }} // 多層條件（巢狀三元） {{ $json.分數 >= 90 ? 'A' : $json.分數 >= 70 ? 'B' : 'C' }} // 檢查欄位是否存在 {{ $json.備註 ? $json.備註 : '無備註' }} // 陣列是否有資料 {{ $json.項目.length > 0 ? '有資料' : '空清單' }}
💡 Expression 只能用在欄位輸入框：把欄位右側切換成 Expression 模式（小齒輪/閃電圖示），才能輸入 {{ }} 語法。在 Code 節點中直接寫 JavaScript，不需要 {{ }}。
⚠️ 中文欄位名稱：n8n 支援中文欄位名稱，但若欄位名稱有空格或特殊符號，用方括號寫法：{{ $json['欄位 名稱'] }}。
💼 實戰案例
情境｜某電商客服用 Expression 判斷客戶訊息關鍵字自動分流（退貨 / 改地址 / 其他）。
做法｜一行 {{ $json.message.toLowerCase().includes('退貨') ? 'refund' : 'other' }} 就能分類，不用 Switch 節點。
踩雷｜大括號數量錯一個整個 Expression 壞掉——n8n UI 會變紅但錯誤訊息不明顯。養成習慣：複雜表達式先在 Chrome DevTools Console 貼上測一次再搬進來。
