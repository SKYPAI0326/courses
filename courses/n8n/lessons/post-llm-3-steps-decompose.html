步驟 1-3 詳解 — 拆需求、讀 JSON、限定範圍

六步流程的前三步是整套 SOP 的地基。第一步刻意不用 LLM；第二步讓 LLM 做它最擅長的事；第三步最容易跳過、也最值得花時間做對。每步都有 prompt 模板可直接複製。

讀完這頁你會能做到 · OUTCOMES
用 4 格便箋獨立拆解一個改造需求，對著別人講 1 分鐘不打結
在不貼 credential 的前提下，把 workflow JSON 貼給 LLM 並要求它翻成流程描述
下指令讓 LLM 只回傳節點區段，不重寫整份 workflow，並在 n8n UI 上正確完成節點替換
(01)

步驟 1：拆需求（學員主導，不用 LLM）

為什麼這一步刻意排除 LLM
LLM 在需求模糊時會自己腦補。你說「幫我改 #02 處理發票」，LLM 不知道你的發票是什麼格式、想改名成什麼樣子、哪些節點一定要保留——它就開始猜，越改越亂。這一步的工作是你把需求說清楚，讓後面的 LLM 對話能有明確的邊界。
動作
用一句話寫下「我要改成什麼」，再用 3 個追問填滿：input 是什麼 / output 是什麼 / 哪些步驟保留不動。填完之後對著另一個人講 1 分鐘，看能不能講清楚。
LLM 做什麼
不做。這一步刻意排除 LLM，等到 4 格都填好了再請 LLM 進場。
你做什麼
填一張 4 格便箋（下方有範本）：原 workflow 編號、目標 input、目標 output、必保留的節點。如果某格填不出來，代表需求還沒想清楚，先想清楚再繼續。
驗收訊號
4 格都有內容，而且能對著別人講 1 分鐘不打結。如果你只會對自己講、對別人講就說不清楚，通常是 output 那格寫得太模糊。
雷區
跳過這一步直接問 LLM「幫我改 #02 處理發票」是新手最常犯的錯。LLM 拿不到 input/output 的具體形狀，會猜，結果就漂掉——通常你會拿到一份「看起來很像」但細節全錯的 workflow。

4 格便箋範本（以 Story A 為例）

原 WORKFLOW 編號
#02-pdf-ai-rename（合約 PDF 改名版）
目標 INPUT
供應商發票 PDF，每月約 200 張，含統編、品項清單、稅後金額、發票日期
目標 OUTPUT
改名後的 PDF，命名格式：{供應商名}_{發票日期}_{稅後金額}.pdf，移入 /歸檔/2026/{月份}/
必保留節點
PDF 讀取節點、Move File 節點（改名後移檔流程保留），不換節點類型
PROMPT 模板 1 — 步驟 1 自我檢查（選用，也可直接手寫便箋）
我要改 n8n workflow #{ID}（{原本做什麼}）。
目標 input：{欄位/格式/來源}
目標 output：{欄位/格式/去處}
保留節點：{節點 1, 節點 2}
請問我的需求有哪一格還沒講清楚？只指出空白，不要幫我補。
常見錯誤示範
直接問：「幫我改 #02，讓它可以處理發票。」
問題：沒有 input 格式（你的發票 PDF 長什麼樣？哪些欄位要抽取？）、沒有 output 形狀（改名規則是什麼？要移到哪個資料夾？）。LLM 會補上自己的假設，結果你拿到的 workflow 是它猜出來的版本，不是你要的。
(02)

步驟 2：讀 JSON（LLM 主導，你來驗收）

讓 LLM 做它最擅長的事
非工程師看 n8n workflow JSON 通常只看到一大堆 typeVersion、parameters、credentials——不知道哪個節點做什麼、資料往哪裡流。LLM 把 JSON 翻成人話，速度比你快 50 倍，而且能點出資料流向。這一步你只需要驗收它說得對不對。
動作
把整份 workflow JSON（移除 credential 後）貼給 LLM，請它做兩件事：(a) 用一段話描述「這份 workflow 做了什麼」，(b) 列出每個節點的角色與資料流向。
LLM 做什麼
把 JSON 翻成人話。回傳格式大約是：節點名稱 → 它輸出什麼 → 下一個節點吃什麼。這就夠了，不需要它解釋原理或建議改進。
你做什麼
用 n8n UI 對照 LLM 的描述。如果 LLM 說某節點是「傳 Email」但 n8n 介面上明明是「寫 Google Sheet」，代表這份 workflow 結構特殊，後面步驟 3 要更小心。
驗收訊號
你能用自己的話複述 LLM 的描述，而且能在 n8n UI 上一一對應到節點。如果有一個節點你講不清楚它的角色，就回頭問 LLM 那個節點的 input/output 形狀是什麼。
雷區
把 credential 區段一起貼給 LLM。n8n workflow JSON 可能含 API key、Gmail OAuth token、Notion integration secret 等敏感欄位。永遠用 n8n UI 的「Export without credentials」選項，貼出去前也搜尋一次確認沒有 apiKey / accessToken / password 字串殘留。
i
匯出步驟：在 n8n UI 開啟 workflow → 右上角三點選單 → 「Download」→ 選「Without credentials」。下載的 JSON 就是安全可以給 LLM 的版本。每次都用這個選項，不要手動刪欄位，因為 credential 欄位散落在各節點內、容易漏刪。
PROMPT 模板 2 — 步驟 2 讀 JSON
我是 n8n 非工程師使用者，請以結果為主、不解釋原理。
以下是一份 n8n workflow JSON（已移除 credential）。請：
1. 用一段話講它做了什麼。
2. 列出每個節點的角色與資料流向（節點名稱 → 它輸出什麼 → 下一個節點吃什麼）。
不要建議改進、不要寫程式碼。
[貼 JSON]
常見錯誤示範
直接貼 JSON 問：「這個 workflow 我要怎麼改成處理發票？」
問題：把步驟 2（讀 JSON）和步驟 3（限定修改範圍）混在一起問，LLM 會跳過「幫你理解現況」直接開始改，而你還沒確認它描述的現況是不是正確的。先讓 LLM 描述，你驗收，再進步驟 3。
(03)

步驟 3：限定修改範圍（學員 + LLM 協同）

最容易跳過、最值得做對的一步
大多數改 workflow 出問題，都是因為讓 LLM 改的範圍太大——它多動了幾個節點、加了你沒要求的防禦邏輯、重寫了整份 JSON 但 typeVersion 對不上你裝的版本。這一步的核心原則是：告訴 LLM 只改哪幾個節點，要它只回傳那幾個節點的 JSON 區段，不是整份。
動作
把步驟 1 的需求 + 步驟 2 的節點清單，明確告訴 LLM「只改節點 X、Y、Z，其他不動」，並請 LLM 回傳「修改後的這幾個節點 JSON 區段」，不是整份。
LLM 做什麼
產出指定節點的區段 JSON。你拿到的應該是一個或幾個節點的 JSON 物件，不是完整的 workflow JSON（即不應該有最外層的 "nodes": [...] 包住整份）。
你做什麼
先在 n8n UI 上把要改的節點選起來、複製、貼到一個臨時筆記檔當備份。LLM 改完後，在 n8n UI 上用「複製貼上節點」功能替換這幾個節點（不是改檔案）。
驗收訊號
n8n UI 上節點還在原位、連線沒斷、其他節點原樣。只有你指定要改的節點有變動。
雷區（示範形狀法）
如果改動牽涉換新類型的節點（例如要把 Google Sheet 節點換成 Notion 節點），先在 n8n UI 拉一個目標節點、隨便填、複製出來給 LLM 當形狀範本，不要讓 LLM 從記憶生成節點 JSON。LLM 的記憶會造成 typeVersion 錯——它以為是 2.x，但你裝的版本是 3.x，匯入就壞。
核心原則 — 只回傳節點區段，不重寫整份
n8n workflow JSON 裡，節點之間靠 id 與 connections 區段串聯。LLM 在整份重寫時極容易動到 ID 或漏掉 connection，你匯入後會發現節點全斷線。

要求 LLM「只回傳節點區段」+ 你在 n8n UI 上做替換，就是為了繞開這個坑。整個 connections 結構保持不動，只換節點本身的設定。

若你確實需要新增或刪除節點（不是修改既有節點），connections 必須回到 n8n UI 上手動重接——UI 拖線 30 秒，LLM 寫錯一個 ID 你 debug 30 分鐘。
PROMPT 模板 3 — 步驟 3 限定修改範圍
我是 n8n 非工程師使用者，請以結果為主、不解釋原理。
基於剛才那份 workflow，只改節點 {X, Y, Z}，不動其他。
我要的修改：{具體描述，含 input/output 形狀}
{若換新節點類型，附：以下是我從 n8n UI 拉出來的目標節點形狀範本：[貼節點 JSON]}
請回傳「只有這 3 個節點」的 JSON 區段。
不要重寫整份 workflow、不要改其他節點。
常見錯誤示範
問：「基於剛才那份 workflow，幫我改成處理發票，給我完整的 JSON。」
問題一：沒有指定只改哪幾個節點，LLM 會自由發揮，可能重寫整份。
問題二：要求「完整的 JSON」正好踩了你最不想踩的雷——LLM 從記憶產出的完整 workflow，typeVersion 漂移、connections 引用錯位，匯入直接失敗，或更糟：匯入成功但資料形狀全錯。

如果你收到 LLM 給的 JSON，不確定它是「區段」還是「整份」，有個快速判斷法：看 JSON 最外層有沒有 "name" 和 "nodes" 兩個頂層 key——有的話代表它回傳了整份 workflow。這時要回頭再問一次：「你給的是整份 workflow。請只給我節點 {X, Y, Z} 的 JSON 物件，不要其他內容。」
