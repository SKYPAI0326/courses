(01) · Module 1 · Section 1.1

流程總覽與前置檢查

安裝之前先看地圖。這 10 分鐘讓你知道接下來要走哪條路、要花多少時間、要先確認什麼——不是現在就要動手裝，是先把全局看清楚。

學習成果 · OUTCOMES
能說出本課程為何使用 Docker Desktop 而非 n8n Desktop App
能依自己的電腦型號選出對應的安裝路徑（A / B / C 三選一）
能完成前置檢查清單 6 項並估算自己走完 1.1 系列的時間
(01)

為什麼是 Docker，不是 n8n Desktop App

情境切入 · HOOK
阿凱這個月做了一檔 PDF 客戶資料抽取的活動，月初才到 12 號，Make 已經跑掉 800 ops，眼看月底還有兩週。他不想升級到 Make 付費版，更不想把客戶 PDF 上傳到第三方 SaaS。雯姊跟他說：「你裝 n8n 在自己電腦跑，免費、跑幾次都不扣點，PDF 也不用離開你的硬碟。」阿凱的第一個問題：「不是要會寫 code 才能裝嗎？」

不用。本單元的 10 分鐘，目標是讓你看完之後知道自己接下來要走哪條路——不是現在就要動手裝，是先把地圖看清楚。

!
n8n Desktop App 已退役：n8n 官方 GitHub 倉庫 n8n-io/n8n-desktop-app 於 2025-08-15 封存（read-only），不再更新。本課程不使用 Desktop App，改採官方推薦的 Docker Desktop + Docker Compose 方案。啟動仍透過課程提供的一鍵腳本，不需要手打 docker 指令。
(02)

關鍵概念

Docker
基礎設施
把「整包跑得起來的軟體環境」裝箱送到你電腦上的工具，你不用懂裡面的細節。
✓不需要懂容器技術
✓Docker Desktop 開著，n8n 就能跑
Docker Desktop
桌面應用
Docker 在 Mac / Win 上的桌面版本，有圖形介面（左上角看到鯨魚圖示），不用打指令。
✓有 GUI，跟開一般 App 一樣
✓免費下載，Mac 與 Win 均支援
Docker Compose
設定檔
把「要跑哪些容器、用什麼設定」寫成一份 yaml 檔，一個指令把整組服務拉起來。
✓試跑包就是一份寫好的 compose 檔
✓學員雙擊腳本即可啟動，不用手打
n8n Community Edition
自架版本
n8n 的自架版本，個人/教學/小型團隊免費（採 Sustainable Use License），跑在你自己電腦上，不按執行次數計費，可以讀寫本機檔案。
✓不按次數計費（告別 Make 1,000 credits/月限制）
✓客戶 PDF 不用離開你的硬碟
(03)

安裝旅程全覽：4 步驟、4 子單元

完整安裝旅程一共 4 步驟，本課程把 1.1 拆成 4 子單元，正好對應 4 步驟：

步驟 1：裝 Docker Desktop ← 1.1.1 install
↓
步驟 2：下載試跑包，改 .env 密碼 ← 1.1.2 launch（前半）
↓
步驟 3：雙擊 start.command/.bat ← 1.1.2 launch（後半）
↓
步驟 4：開 http://localhost:5678 ← 1.1.2 launch（驗證）

中間任何一步卡關 → 跳 1.1.3 troubleshoot 排錯手冊。

3 條安裝路徑分流

依你的電腦選一條走：

路徑 適用情境 預計時間
AMac Intel 2020 之前的 MacBook / iMac 首次 30–40 分鐘
BMac Apple Silicon 2020 之後（M1 / M2 / M3 / M4） 首次 25–35 分鐘
CWindows 10/11 + WSL2 Win 10 Pro / Home 或 Win 11 首次 35–45 分鐘（含 WSL2 啟用）
i
判斷 Mac 是 Intel 還是 Apple Silicon：點左上角 Apple →「關於這台 Mac」→ 看「晶片」欄位。寫 Apple M* 的選路徑 B，寫 Intel Core * 的選路徑 A。

判斷 Win 版本：開「設定 → 系統 → 系統資訊」看 Windows 版本。Win 10 1903（build 19041）以前的版本不支援 WSL2，需要先升級。
(04)

前置檢查清單

動手 · HANDS-ON
依清單逐項確認，全部過了才進 1.1.1。現在就花 3 分鐘做完，比裝到一半才發現磁碟不夠或 Win 沒開虛擬化要省很多時間。

判斷你的時間預算

5 分鐘
已裝過 Docker Desktop 且還在運作，直接進 1.1.2
30–45 分鐘
完全沒裝過 Docker，走完 1.1.1 + 1.1.2
額外 +15 分鐘
Win 用戶且尚未啟用 WSL2，需額外操作

如果現場時間不夠跑完全程，先做完前置檢查清單，把實際安裝放到下課後依 1.1 系列子頁照走。

(05)

完成驗證

本單元完成條件 · VERIFY

跟旁邊的人（或自己對自己）說一遍以下 4 個答案，說得出來就過：

我這台電腦的安裝路徑是 A / B / C 哪一條
我磁碟剩 _____ GB，超過 8GB 沒問題
我有沒有裝過 Docker（已裝 / 沒裝）
我預期會花 _____ 分鐘走完 1.1 全部子單元

動手練習

用你現在這台電腦，完成上方的「前置檢查清單」6 項，並用便條紙（或記事 App）寫下：

寫下安裝路徑
A（Mac Intel）/ B（Mac Apple Silicon）/ C（Win + WSL2），不是「應該是 B 吧」
磁碟剩餘空間
實際數字，Mac 在「關於這台 Mac → 儲存空間」，Win 在「檔案總管」
預期花費時間
依上方時間預算表填入，不是「不知道」

一張寫滿三個答案的便條紙可以拍照存檔，作為自己的「準備好了」憑據。

(07)

常見錯誤

看到「裝 Docker」就想跳過，改用 npx n8n
不建議：npx n8n 雖然 30 秒能跑，但資料不持久（重啟易消失）、沒有 PostgreSQL，Module 3 跑大量 PDF 時會很痛苦。Docker Desktop 啟動後跟「開另一個 App」一樣，平時不用不會打擾你。
Mac 用戶跳過「判斷 Intel / Apple Silicon」，直接下官網第一個 .dmg
官網有兩個下載按鈕：裝錯晶片版本啟動後會報錯。現在就到「關於這台 Mac」看「晶片」欄位，自己確認一次。
Win 用戶沒檢查虛擬化，裝完 Docker Desktop 才被擋下
BIOS 設定需重開機：臨時找不到資料會卡很久。Win 用戶現在就開工作管理員 → 效能 → CPU，看「虛擬化」是否「已啟用」。
(08)

自我檢核

Q1：本課程不使用 n8n Desktop App 的原因是？
顯示答案
正確答案：C。n8n 官方 GitHub 倉庫 n8n-io/n8n-desktop-app 於 2025-08-15 封存（read-only），官方不再更新。本課程改採 Docker Desktop + Compose 方案，功能更完整且持續維護。
Q2：阿凱用 MacBook Pro M2，磁碟剩 12GB，從來沒裝過 Docker。他應該選哪條安裝路徑、預期花多少時間？
顯示答案
正確答案：B。M2 = Apple Silicon，選路徑 B。12GB 超過 8GB 門檻，過關。完全沒裝過 Docker，預期 25–35 分鐘走完 1.1.1 + 1.1.2。
