#10 · CLIENT FOLDER ORGANIZE · 9 NODES · 進階

客戶資料夾自動整理：丟一堆混雜檔案，n8n 自動分類 + 命名 + 產交接 index

顧問 / 設計師 / 課程助教 / 代操常會收到客戶各種檔案：PDF、圖片、合約、簡報、發票全混在一個資料夾。手動整理很耗時，交接時要重做一次。這個 workflow 把「檔案 → AI 分類 → 改名 → 歸檔 → 產 Markdown index」一鍵自動化。本機檔案系統級自動化是 n8n 的代表優勢場景 — 不需要先把檔案丟到 Google Drive 才能處理。

學習成果
能對混雜檔案做副檔名分流 + AI 文字分類雙路徑（圖片直接歸 images，文件交給 AI）
能用 Aggregate 節點把 N 個 items 統計成「分類計數 + Markdown index」一份輸出
能用 fan-out connection 讓 Code 節點同時送 Write + Aggregate 兩條線並行
(01) WORKFLOW 全貌

節點與資料流（fan-out 雙線）

Manual Trigger→Read 所有檔→Extract 文字→Code: AI 分類 + 命名

↗ Write 移到分類資料夾
↘ Aggregate → Code: 組 _index.md + 通知 → Write index + Telegram

9 節點。Code: AI 分類同時 fan-out 兩條線：一條把每個檔分別 Write 到 6 個分類子資料夾（contracts / invoices / presentations / images / docs / others），另一條 Aggregate 全部 N 個 items 後組一份 Markdown index 供交接 + Telegram 摘要通知。

(02) 重點節點拆解

Code: AI 分類 + 命名（核心）

CORE NODE
副檔名分流 + AI 分類雙路徑
code v2runOnceForAllItems + 內 loop

關鍵設計：先看副檔名（jpg/png/heic 等圖片直接歸 images，跳過 AI 呼叫）；不是圖片才丟給 Gemini 用 5 類分類（contracts / invoices / presentations / docs / others）。

為什麼分流：圖片打 AI 浪費（Gemini 還是會回類別但靠檔名而非內容判斷）。直接用副檔名 dispatch 又快又省 token。

JSON 輸出格式：{"category":"xxx","newName":"YYYYMMDD_類型_關鍵字"}，限定 enum 5 類避免 AI 發明新類別。fallback：JSON 解析失敗 → 預設 others。

AGGREGATE
把 N 個 items 收成 1 個 Markdown index

Aggregate 節點 fieldsToAggregate: [indexLine, category] → output 1 個 item，indexLine 變成 array。下游 Code 節點 reduce array 算各分類數量 + 串成 Markdown 列表。

vs 直接 loop：n8n 的 Aggregate 比手寫 loop 在「多 items → 1 個 summary」場景更直觀，少寫一段 jsCode。

(03) 改造方向

5 種改造

  1. 換業務分類術語：把 5 類換成你公司業務（合約 / 客戶 brief / 設計稿 / 報價單 / 簡報），改 prompt 的分類定義即可。
  2. 串 #02 PDF AI 改名：合約類分流後，再呼叫 #02 邏輯做更精準的合約改名（含日期 + 對方公司）。
  3. 加 OCR 處理掃描 PDF：掃描合約沒文字 layer，串 Tesseract / Google Vision OCR 後再 AI 分類。
  4. 加版本控制：在 client-organized/ 用 git init，每次整理後自動 commit，做完整的客戶檔案版本歷史。
  5. 串雲端儲存：分類後同時上傳 Google Drive / Dropbox 對應資料夾，本機 + 雲端雙備份。
