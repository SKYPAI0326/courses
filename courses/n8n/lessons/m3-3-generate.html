定時彙整日報：排程掃描 × AI 摘要

設定好後，n8n 每天自動掃描文件資料夾、AI 彙整摘要，輸出 Markdown 日報——完全不需人工觸發，也不依賴任何雲端服務。

這節的核心優勢

Make 必須由外部事件觸發（有人寄信、有人填表），但 n8n 可以從自身的排程主動啟動。Schedule Trigger 讓 n8n 成為真正自主運行的本機代理人——不需要任何雲端訊號，不需要任何人按下按鈕，到時間就跑。

MAKE.COM
被動觸發
必須等待雲端事件（Gmail 收信、表單提交）才能啟動，無法自行按時執行本機任務。
N8N DESKTOP
主動排程
Schedule Trigger 自行定時啟動，無需任何外部訊號，完全在本機獨立運行。
STEP BY STEP · 操作步驟
本節流程概覽
Schedule Trigger
→
Code（讀取資料夾）
→
HTTP Request（Gemini）
→
Code（格式化）
→
Write Binary File
1
Schedule Trigger：設定每日定時啟動
新建一個 Workflow，點擊「Add first step」，搜尋並選擇「Schedule Trigger」節點。這個觸發器完全不依賴外部訊號，n8n 自己按照排程執行。
設定欄位建議值說明
Trigger IntervalDays每天觸發一次
Days Between Triggers1間隔 1 天
Trigger at Hour17下午 5 點（可依需求調整）
Trigger at Minute0整點觸發
測試方式：開發時不需要等到真正的觸發時間——點擊節點上的「Execute step」可以立即手動觸發，跳過等待直接測試後續節點是否正確。
2
Code 節點：讀取資料夾並合併所有文件內容
新增「Code」節點（Language 選 JavaScript）。這個節點會讀取指定資料夾內所有 .txt 與 .md 文件，將它們的內容合併成單一字串，為後續 AI 摘要做準備。
// Code 節點：讀取資料夾所有文件 const fs = require('fs'); const path = require('path'); const folderPath = '/Users/你的名稱/Documents/每日文件'; const today = new Date().toISOString().slice(0, 10); const files = fs.readdirSync(folderPath) .filter(f => f.endsWith('.txt') || f.endsWith('.md')); let combined = ''; for (const file of files) { const content = fs.readFileSync(path.join(folderPath, file), 'utf-8'); combined += `\n\n## ${file}\n${content}`; } return [{ json: { combined, today, fileCount: files.length } }];
修改路徑：將 /Users/你的名稱/Documents/每日文件 改成你實際要掃描的資料夾路徑。資料夾必須已存在，且裡面有 .txt 或 .md 文件。
3
HTTP Request 節點：呼叫 Gemini API 生成摘要
新增「HTTP Request」節點，呼叫 Gemini API，將合併後的文件內容傳入，請 AI 生成結構化的每日摘要日報。
// URL（替換 YOUR_API_KEY） https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash:generateContent?key=YOUR_API_KEY // Body（JSON 格式） { "contents": [{ "parts": [{ "text": "你是一位工作摘要助理。以下是今天的工作文件內容，請生成一份每日摘要日報，格式為純 Markdown（不要包含 markdown 圍欄標記）。日報包含：今日重點（3-5 項條列）、待處理事項、備注。\n\n文件內容：\n{{ $json.combined }}" }] }] }
Gemini 免費配額：Gemini Flash 免費方案每分鐘 15 次請求，每日 1,500 次。每天跑一次日報完全不會觸及限制。
4
Code 節點：格式化為完整 Markdown 日報
新增第二個「Code」節點，從 Gemini 回傳中提取文字，加上日期標題，組合成完整的 Markdown 日報，並生成含日期的檔名。
// Code 節點：格式化 AI 回傳為日報 const raw = $input.item.json.candidates[0].content.parts[0].text; const clean = raw.replace(/```[\w]*\n?|\n?```/g, '').trim(); const today = $('Code').item.json.today; const fileCount = $('Code').item.json.fileCount; const report = `# 每日工作日報｜${today}\n\n` + `> 本日共掃描 ${fileCount} 份文件，由 AI 自動生成摘要。\n\n` + clean; const fileName = `${today.replace(/-/g, '')}_日報.md`; return [{ json: { report, fileName } }];
引用前一個 Code 節點：$('Code').item.json.today 是用節點名稱引用資料。若你的第一個 Code 節點有自訂名稱，將引號內的名稱替換成實際名稱。
5
Write Binary File：儲存日報到本機
先新增「Convert to File」節點，把 Markdown 字串轉為二進位（Source Field 選 report，Encoding 選 UTF-8）。再接「Write Binary File」節點，設定輸出路徑和動態檔名。
// Write Binary File 設定 File Name: /Users/你的名稱/Documents/日報/{{ $json.fileName }} Data: data（Convert to File 的輸出）
📁
先建立資料夾：手動在 Finder 建立「日報」資料夾，n8n 不會自動建立不存在的目錄。日後每天的日報都會以日期命名（如 20260415_日報.md）存入此資料夾。
6
啟用 Workflow：讓 n8n 自主運行
完成所有節點後，點擊 n8n 右上角的「Active」開關，將 Workflow 狀態切換為啟用。此後，每天指定時間 n8n 會自動執行，不需要任何人工操作。確認 n8n Desktop 應用程式有在執行中（系統列有 n8n 圖示），排程才能正常觸發。
💡
排程要求 n8n 持續運行：Schedule Trigger 的觸發依賴 n8n Desktop 保持開啟狀態。若需要電腦關閉後仍能執行，未來可考慮部署到 n8n Cloud 或使用 Cron 系統服務，但對工作坊情境而言，保持 Desktop 開啟已足夠。
完成驗證 · VERIFY
手動點擊「Execute workflow」測試整條流程後：（1）日報資料夾中出現今日日期命名的 .md 文件，（2）打開文件確認包含「今日重點」、「待處理事項」等結構，（3）文件掃描數量與實際資料夾內的 .txt/.md 數量吻合。三點達成即代表成功。將 Workflow 切換為 Active 後，無需任何人工操作，每天指定時間自動產出。
常見問題 · TROUBLESHOOT
Code 節點報錯：ENOENT no such file or directory
資料夾路徑不存在。確認 folderPath 中的路徑在 Finder 中確實存在，且路徑格式使用正斜線 /（不是反斜線）。Mac 用戶路徑通常以 /Users/你的帳號名稱/ 開頭。
Code 節點回傳 fileCount 為 0，AI 摘要內容空白
資料夾存在但沒有 .txt 或 .md 文件。確認掃描的資料夾中有至少一個文字文件。Word 文件（.docx）和 PDF 不在預設掃描範圍，若要掃描其他格式，修改 .filter() 中的副檔名條件。
Schedule Trigger 設定好後沒有在指定時間執行
兩個可能原因：（1）Workflow 尚未切換為「Active」——確認右上角開關是藍色啟用狀態；（2）n8n Desktop 應用程式在指定時間已關閉——Schedule Trigger 需要 n8n 持續在背景運行才能觸發。
第二個 Code 節點引用 $('Code').item.json.today 報錯
n8n 用節點名稱引用資料。若你的第一個 Code 節點名稱不是預設的「Code」（例如已改為「讀取文件」），將引用語法改為 $('讀取文件').item.json.today。
