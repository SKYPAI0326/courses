Module 02
SECTION 2.1

相對引用與絕對引用（點名抓取法）

學會靈活取用各節點的資料——不管流程有多長，都能精準抓到你要的那一筆。

⏱預估時間：20 分鐘
📋4 個步驟
✅前置：完成 Module 1
1
認識兩種引用方式的差異

$json 永遠代表「當前節點」正在處理的那筆資料，寫起來最短也最常用。但當你需要從更早的節點抓資料時，就要用「點名」語法：$('節點名稱').item.json，明確指定要從哪個節點拿。

相對引用（當前節點）
{{ $json.客戶姓名 }}
讀取「當前節點」的資料
適合：簡單線性流程
絕對引用（指定節點）
{{ $('取得訂單').item.json.客戶姓名 }}
讀取「指定節點名稱」的資料
適合：Loop 內 / 跨多節點
🔄 對應 Make 的概念：就像 Make 中你可以選擇「哪個 Module 的輸出」來帶入欄位，n8n 的絕對引用也是相同概念，只是改用文字語法指定。
2
實作：建立三節點流程，跨節點讀取資料

建立一個包含三個節點的測試流程：Webhook（接收資料）→ Set1（加工資料）→ Set2（組合輸出）。在最後的 Set2 節點中，試著讀取第一個 Webhook 節點的原始資料，而不是 Set1 加工後的資料。

// 流程：Webhook → Set1 → Set2 // Webhook 節點收到：{ "客戶編號": "C-001", "訂單金額": 45000 } // Set1 節點只保留「訂單金額」欄位 // 在 Set2 節點，用絕對引用取回 Webhook 的原始「客戶編號」： {{ $('Webhook').item.json.客戶編號 }} → "C-001" // 用相對引用取 Set1 處理後的金額： {{ $json.訂單金額 }} → 45000
🗂
Set2 節點 Expression 面板，左側列出所有節點，點擊 Webhook 節點展開可選欄位
3
Loop 內部：同時取用迭代資料與外部主資料

最常需要絕對引用的場景是 Loop（迴圈）。當你用 Loop 逐一處理一份商品清單，Loop 內部的 $json 會是「當前這筆商品」的資料；但主訂單資訊（客戶姓名、訂單編號）在 Loop 之前就已讀取，這時就要用絕對引用才能取到。

// Loop 內的節點，同時用兩種引用： // 當前這筆商品（相對引用） {{ $json.商品名稱 }} → "無線耳機"（第N筆商品） {{ $json.數量 }} → 2 // Loop 外的主訂單資訊（絕對引用，指向 Loop 前的節點） {{ $('取得訂單').item.json.客戶姓名 }} → "王小明"（每筆都一樣） {{ $('取得訂單').item.json.訂單編號 }} → "A20260315-001"
⚠️ 在 Loop 內部使用 $json 取主訂單資訊，通常只會得到第一筆商品的資料，不是你要的主資料。遇到這種情況，一定要改用絕對引用指定正確節點。
4
Expression 編輯器：用左側面板快速選取，避免打錯字

點擊任何 n8n 欄位切換到 Expression 模式後，左側會出現一個資料面板，列出流程中所有節點的輸出結構。直接點擊你要的欄位，n8n 會自動帶入正確語法，不需要手動拼欄位名稱，大幅減少錯誤。

✨
Expression 編輯器左側面板：展開不同節點的輸出，點擊欄位自動插入語法
✅ 養成習慣：盡量從左側面板點擊欄位，而非手打語法。這樣節點名稱和欄位名稱都不會出錯，除錯時間大幅減少。
完成驗證 CHECKPOINT
執行三節點流程，確認以下兩件事

Set2 節點的輸出中，同時包含用絕對引用讀回的「Webhook 原始欄位」，以及用相對引用讀取的「Set1 加工後欄位」，兩者都顯示正確值。這代表你已掌握兩種引用方式的使用時機。

✅ 完成檢查 · 跑完本節你應該看到
三節點 + 兩種引用，輸出可逐欄對照
  1. Webhook、Set1、Set2 三個節點全部亮綠（執行成功）。
  2. Set2 Output 面板同時看到三類欄位：（a）相對引用 {{ $json.xxx }} 從 Set1 帶過來的加工值；（b）絕對引用 {{ $('Webhook').item.json.xxx }} 從 Webhook 帶過來的原始值；（c）你自訂的常數欄位。
  3. 欄位值與你在 Webhook 測試打進去的內容一致（沒被無聲覆蓋）；切到 JSON 視圖檢查結構，沒有 undefined 或缺欄。
  4. 本機沒有副作用（本節純記憶體流程，不寫檔、不打外部 API）。
  5. 外部服務狀態：Webhook test URL 可重複觸發，每次 Set2 都產生對應一筆新 output。
任一項對不上 → 看上方「常見問題」對照；或回 Module 2 索引重看流程設定。
常見問題
出現「Node '節點名' not found」錯誤
絕對引用中的節點名稱必須與畫布上的節點名稱完全一致，包含大小寫與空格。最安全的方式是從 Expression 左側面板點擊選取，由 n8n 自動填入，不要手打節點名稱。
Loop 內用絕對引用，每次結果都一樣（只有第一筆）
這是預期行為——絕對引用指定節點時，預設取第一筆 item（index 0）。若需要取對應 index 的資料，需加上 $('節點名').all()[{{ $itemIndex }}].json.欄位，但大多數場景下用主資料第一筆就是正確的。
💼 實戰案例
情境｜某管理顧問公司流程有 15 個節點，用 $items('node-name') 回頭抓第 2 個節點的資料到第 14 個節點。
做法｜給每個 node 明確命名（如 Slack-Fetch-Messages）不用預設 Slack1、Slack2，三個月後回來改流程看名字就懂。
踩雷｜改節點名稱後所有引用會失效——n8n 不會自動 rename 引用。改名前先用 Ctrl+F 搜尋所有引用位置。
