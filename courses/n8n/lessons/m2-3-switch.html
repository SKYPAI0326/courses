Switch 節點：多路分支處理

當任務類型超過兩種，If 節點要層層巢狀才能處理。Switch 節點讓你根據一個欄位的值，把資料分流到不同的處理路徑，設定更直覺。

⏱預估時間：20 分鐘
📋4 個步驟
1
If vs Switch：選擇時機

If 節點只有兩條出口（True / False），適合「是/否」的二元判斷。Switch 節點可以設定任意數量的出口，每個出口對應一個條件，適合「多種類型分流」的場景。

If 節點 — 只適合兩路

→ True
大額處理
→ False
一般處理

→ "報告"

→ "通知"
發送通知
→ "備份"

→ 其他
記錄未知任務
2
新增 Switch 節點並設定路徑

在畫布搜尋「Switch」加入節點。開啟設定後，選擇判斷模式「Rules」（規則模式），然後為每個分支新增一條規則：

每條規則設定三個部分：輸入欄位（如 任務類型）→ 條件（如 Equal to）→ 比對值（如 報告）。符合條件的 Item 從對應的出口輸出。

🔀
Switch 節點設定介面，左側為判斷欄位，中間為條件，右側為比對值。每條規則對應一個輸出連線。
💡 一定要加「其他」分支（Fallback）：在規則列表最下方有「Fallback Output」選項，啟用後會多一條輸出線，讓不符合任何規則的 Item 也有出路，而不是靜默丟掉。
3
連接各分支的下游節點

Switch 節點的每個出口都可以連接不同的後續節點。從 Switch 節點的右側，你會看到對應每條規則的輸出點（Output 0, 1, 2…）。把每個輸出點連接到對應的處理節點，形成多路流程。

⚠️ 多個分支最後要匯合：各分支最終若要執行同一個節點（如「記錄完成」），需要把所有分支的輸出都連線到該節點。n8n 允許一個節點有多個輸入連線——所有輸入的 Items 會合併後一起處理。
4
實際應用：Google 表單任務分流

在 Module 4 的 Google Forms 遠端遙控場景中，表單有一個「任務類型」欄位，值可能是「生成日報」、「PDF 改名」或「發送通知」。用 Switch 節點分流後，各類型走各自的處理邏輯，最後統一回傳完成訊息。

三路 Switch 分流測試

建立 Switch 節點，根據 $json.type 分三路（A/B/C）加一個 Fallback。用 Set 節點輸出三筆不同 type 的測試資料，執行後確認每筆資料都流到對應的正確分支，Fallback 分支在輸入不存在的 type 時正確觸發。

💼 實戰案例
情境｜某電商依訂單金額分級處理：<300 走 A 倉、300-1000 走 B 倉、>1000 走 VIP 專人。
做法｜Switch 節點比 IF 鍊好維護——三個月後新增『國際訂單』只要加一個 output，不用改原條件。
踩雷｜Switch 的『Fall through』預設 OFF——一筆資料只會走第一個符合的 output。若想多條件都觸發要另外用平行 IF 節點。
