啟動 n8n 並完成首次設定

阿凱裝完 Docker Desktop，鯨魚圖示亮了，但 n8n 在哪？這單元 15 分鐘，目標是讓 n8n 從「沒啟動」變成「localhost:5678 主介面開起來 + Webhook 節點亮綠燈」，全程不需要打任何 docker 指令。

學習成果
能雙擊試跑包腳本啟動 n8n，並在瀏覽器開啟 localhost:5678
能完成 Owner Account 首次設定，進入 n8n 主介面
能新增 Webhook 節點、跑通 hello-world 測試請求，確認節點亮綠燈
試跑包 · 一鍵下載
n8n-starter-kit.zip
課程預先準備的「整套可跑設定 + 一鍵腳本」，包含 Docker Compose 配置、.env 範本、各平台雙擊腳本（共 9 個檔案 + shared/ 子資料夾）。本單元所有操作都建立在這份試跑包上。
⬇ 下載 n8n-starter-kit.zip（18 KB · v1.0.2）
⚠️ 解壓後若 .command 雙擊報錯（macOS），點開看修法

原因：macOS 從瀏覽器下載 .zip 解壓後，Gatekeeper 會把 .command 標記為 quarantine，加上某些解壓工具會剝奪執行權限，導致雙擊報錯「無法打開」或「找不到應用」。

解法 1（推薦，圖形介面）：開「系統設定 → 隱私權與安全性」→ 捲到底「安全性」區塊 → 看到被擋的 start.command → 按「強制打開」→ 輸入密碼確認。

解法 2（一行終端機指令解全部）：右鍵點 n8n-starter-kit 資料夾 → 「服務 → 在資料夾位置開新終端機」（要先在 系統設定 → 鍵盤 → 鍵盤快速鍵 → 服務 → 啟用），執行：

chmod +x start.command stop.command update.command xattr -d com.apple.quarantine start.command stop.command update.command 2>/dev/null || true

Windows 不需要這步：.bat 檔雙擊即可，但若被 SmartScreen 擋，按「更多資訊 → 仍要執行」即可。

📂 個別檔案連結（zip 損壞時備援）
README.md n8n-compose.yml .env.example start.command（Mac） start.bat（Win） stop.command stop.bat update.command update.bat
逐個下載後請手動建立 n8n-starter-kit/ 資料夾、放入 9 個檔案、再建一個空的 shared/ 子資料夾（M3 Watch Folder 會用到）。
(01)

為什麼要用試跑包

情境
阿凱裝完 Docker Desktop，看到鯨魚圖示在右上角發著綠光，但畫面打開只看到一個空空的 Containers 列表。「然後呢？n8n 在哪？」他問雯姊。雯姊把試跑包資料夾傳給他：「下載解壓，雙擊 start.command，剩下交給它。」

Docker 本身只是容器引擎，不會自動跑 n8n。你需要告訴它「要跑哪個 image、用什麼設定、資料存哪裡」——這份描述就是 Docker Compose 的 yaml 檔。試跑包把這些全打包好，讓你不用手打指令也能啟動完整的 n8n 環境。

雙擊腳本背後的每一步，本單元都會說明——這樣出問題你才能自己排查，而不是只能重裝。

(02)

啟動前需要知道的五個概念

(A)
Docker Compose
用一份 yaml 檔描述「要跑哪幾個容器、它們怎麼連、用什麼設定」的工具。本課程的 n8n-compose.yml 就是這份描述。
(B)
試跑包（Starter Kit）
課程預先準備的「整套可跑設定 + 一鍵腳本」，把 yaml、環境變數範本、雙擊腳本都包好，降低第一次啟動的門檻。
(C)
.env 檔
環境變數設定檔，存密碼、時區、URL 等敏感或會變動的值。Compose 啟動時自動讀取，讓設定與程式碼分離。
(D)
Volume（資料卷）
Docker 用來持久化資料的機制。你的 n8n flows 與 credentials 存在 volume 裡，停容器再啟不會消失。
(E)
Owner Account
n8n 第一次開啟會引導建立的管理者帳號，只存在你本機，不需要真的去收驗證信。
(F)
Webhook hello-world
最小驗證流程：一個 Webhook 節點等候請求，用瀏覽器打 Test URL 觸發，確認節點能接收外部 HTTP 請求。
(03)

六步驟完成啟動與驗證

1
下載試跑包，解壓到方便管理的位置

從上方下載連結取得試跑包，解壓後放到：

# Mac 建議路徑 ~/n8n-starter-kit/ # Windows 建議路徑 C:\Users\你的名字\n8n-starter-kit\
預期看到
資料夾內含 n8n-compose.yml、.env.example、README.md，以及 start / stop / update 各平台腳本（.command 是 Mac，.bat 是 Windows）。
2
第一次雙擊 start——會自動建 .env，讓你修改密碼

Mac：在 Finder 進入 n8n-starter-kit 資料夾，雙擊 start.command

Windows：在檔案總管進入 n8n-starter-kit 資料夾，雙擊 start.bat

Mac 常見卡點：無法打開，因為來自未識別的開發者
這是 macOS Gatekeeper 的正常安全機制，不是錯誤。處理步驟：
  1. 開「系統設定 → 隱私權與安全性」
  2. 捲到「安全性」區塊，看到「start.command 已被阻擋使用」
  3. 按「強制打開」→ 輸入密碼確認
  4. 之後同一檔案不會再被擋
預期看到
第一次跑會自動把 .env.example 複製為 .env，並打開文字編輯器讓你編輯，同時跳出提示「請修改 POSTGRES_PASSWORD 為強密碼後存檔」。
3
改 .env 裡的 POSTGRES_PASSWORD，存檔關掉編輯器

.env 完整內容如下，只需要改 POSTGRES_PASSWORD 那行：

GENERIC_TIMEZONE=Asia/Taipei POSTGRES_USER=n8n POSTGRES_PASSWORD=please_change_me_to_a_strong_password <--- 改這行 POSTGRES_DB=n8n WEBHOOK_URL=http://localhost:5678/ N8N_HOST=localhost N8N_PROTOCOL=http

改成你自己的強密碼，例如：

POSTGRES_PASSWORD=NongYiXia_n8n_2026!
i
這密碼只用在你本機 PostgreSQL，不需要記住給別人，但不要用 password 之類的弱密碼。建議 16 字元以上，混合英數符號。存檔後關掉編輯器。
4
再次雙擊 start——等 Docker 拉取 image 並啟動容器

回到 n8n-starter-kit 資料夾，再次雙擊 start.command（Mac）或 start.bat（Windows）。

預期看到（終端機日誌）
正在啟動 n8n（首次需下載 image，可能花 1-5 分鐘）... [+] Pulling 12/12 ✔ postgres 16 image 下載完成 ✔ n8nio/n8n:2.17.8 image 下載完成 [+] Running 3/3 ✔ Network n8n-starter-kit_default Created ✔ Container n8n-starter-kit-postgres-1 Started ✔ Container n8n-starter-kit-n8n-1 Started 等候 n8n 服務就緒...
約 30 秒到 1 分鐘後（首次較久），瀏覽器會自動開到 http://localhost:5678。如果沒有自動開，手動在瀏覽器輸入 http://localhost:5678。
5
建立 Owner Account，進入 n8n 主介面

n8n 第一次開啟會引導你建立帳號：

Email
可以填假信箱（例如 me@local.local），不會真的寄信驗證
First / Last Name
隨意填
Password
自訂（至少 8 字元，含大小寫與數字）

按 Next → 跳過後續行銷問卷（按 Skip）→ 進入 n8n 主介面。

預期看到
左側選單有 Workflows / Credentials / Executions / Templates；中央區塊空白或有「Create your first workflow」按鈕。
6
跑 Webhook hello-world 驗證——確認 n8n 能接收外部請求

按「Create Workflow」進入空白編輯器，然後：

  1. 點 canvas 中央的「+」加節點
  2. 搜尋 Webhook，選 Webhook trigger 節點
  3. 節點設定面板：HTTP Method 選 GET，Path 填 hello，其他保留預設
  4. 按 Listen for test event（節點變藍色等候狀態）
  5. 複製 Test URL（形如 http://localhost:5678/webhook-test/hello）
  6. 開新分頁貼上 URL 按 Enter
*
Webhook 驗證成功的兩個信號：
① 剛才開的新分頁顯示 Workflow was started
② 回到 n8n 編輯器，Webhook 節點變綠色，下方 Output 顯示收到的請求資料（headers、query、body）
恭喜
n8n 從零跑到「能接收外部請求」，全程只花了 15 分鐘，且沒打過任何 docker 指令。
✅ 完成檢查 · 跑完本節你應該看到
  1. Docker Desktop：左下角顯示 postgres 與 n8n 兩個容器都是綠色 Running 狀態。
  2. 瀏覽器 http://localhost:5678：能登入到 n8n 主介面，左側看到 Workflows / Credentials / Executions / Templates 四個項目。
  3. Webhook hello-world workflow：Webhook 節點亮綠色（執行成功）；下方 Output 面板顯示收到的請求，至少包含 headers、query、params 三個欄位。
  4. 本機資料夾：n8n-starter-kit/shared/ 已建立，可手動丟測試檔案進去（後續 Module 3 Watch Folder 會用到）。
  5. 外部驗證：在新瀏覽器分頁打開 Test URL，頁面顯示 Workflow was started。
任何一項對不上 → 跳到 1.1.4 排錯手冊 對照排查。
(04)

進階：手動指令速查

雙擊腳本背後的指令是 docker compose。如果你想直接在終端機操作，先 cd 到 n8n-starter-kit/ 資料夾，再執行：

# 啟動所有服務（背景跑） docker compose up -d # 停止所有服務（資料保留） docker compose down # 停止 + 刪 volumes（資料全清，謹慎使用） docker compose down -v # 拉最新 image docker compose pull # 看 n8n 即時日誌（Ctrl+C 退出） docker compose logs n8n -f # 看跑中的容器狀態 docker compose ps
Mac：開終端機
在 Finder 對 n8n-starter-kit 資料夾按右鍵 → 服務 → 「新終端機（位於資料夾位置）」
Windows：開 cmd
在檔案總管網址列輸入 cmd 按 Enter，會直接以該資料夾為位置開 cmd
!
本課程的 stop.command / stop.bat 都是用 down（不加 -v），只要用這兩個檔停就不會丟資料。想清掉重來才用 down -v。
(05)

動手練習：完成六步驟

跟著步驟 1–6 走完，最終要看到 n8n Webhook 節點亮綠燈 + 收到測試請求。

完成檢核
確認以下四件事都成立：
  • 左上角看到「My workflow」（你建的流程）
  • Webhook 節點是綠色，不是紅色或灰色
  • Output 面板看得到 headers、query、body 的資料結構
  • 左側 Executions 選單能看到剛才的執行紀錄
四個全過，1.1.2 完成，可以進 1.2 Cloudflare Tunnel。

預期成果截圖包含：一個綠色的 Webhook 節點 + Output 面板顯示 JSON 格式的 headers / query / body。

(06)

常見錯誤三條

遇到問題時先對照這裡
瀏覽器開 localhost:5678 顯示「Your n8n server is configured to use a secure cookie...」
原因：你下載的試跑包是 2026-04-30 之前的舊版。
解法：捲到本頁最上方綠色按鈕「⬇ 下載 n8n-starter-kit.zip」重新下載最新版，覆蓋掉 ~/n8n-starter-kit/ 整個資料夾，再雙擊 start.command。新版本已預設關閉 secure cookie。
瀏覽器選擇：避開 Safari，用 Chrome / Firefox / Edge 開 localhost:5678（Safari 對本機 cookie 限制嚴格）。
start 腳本跑出 no configuration file provided: not found
原因：你下載的試跑包是 2026-04-30 之前的舊版。
解法：捲到本頁最上方綠色按鈕「⬇ 下載 n8n-starter-kit.zip」重新下載最新版，覆蓋掉 ~/n8n-starter-kit/ 整個資料夾，再雙擊 start.command。新版本的腳本已自動指定 compose 檔。
start 腳本跑出 port is already allocated 或 Bind for 0.0.0.0:5678 failed
原因：你之前已經啟動過 n8n（也許解壓到不同資料夾啟動了多次），舊 container 還佔著 5678 port。
解法：捲到本頁最上方綠色按鈕「⬇ 下載 n8n-starter-kit.zip」重新下載最新版覆蓋資料夾，再雙擊 start.command。新版本啟動前會自動偵測 port 衝突，跳對話框問你「要停掉舊 container 嗎？」按一下就清理好。
start 腳本跑出 password authentication failed for user "n8n"
原因：你之前啟動過 n8n，postgres 用舊密碼建了資料庫；後來改了 .env 密碼，新密碼跟舊資料庫對不上。
解法：直接再雙擊一次 start.command，腳本會自動偵測這個錯誤，跳出對話框問你「要重置嗎？」按「重置並重啟」即可（會清掉舊資料庫，但你還沒建 Owner Account 沒有資料損失）。
雙擊 start.command 跳出「無法打開，因為來自未識別的開發者」
原因：macOS Gatekeeper 第一次擋未簽署的 .command 檔，正常行為。
解法：開「系統設定 → 隱私權與安全性 → 安全性」捲到底，看到被擋的 start.command，按「強制打開」→ 輸入密碼。之後同一檔案不會再被擋。詳見 1.1.3 troubleshoot 案 6。
start 腳本跑完，瀏覽器打開 localhost:5678 顯示「無法連線」或 ERR_CONNECTION_REFUSED
原因：n8n 容器還在啟動中（首次拉 image 較慢），或 5678 埠被其他程式佔用。
解法：等 30–60 秒重試。仍不行去 1.1.3 troubleshoot 案 5（容器在跑但連不上）或案 2（埠被佔用）。
Owner Account 建好之後，下次重啟 Docker Desktop 發現 flows 不見了
原因：之前用 docker compose down -v 而非 docker compose down，-v 會刪 volume 連帶刪資料。
解法：本課程的 stop.command / stop.bat 都是用 down（不加 -v），只要用這兩個檔停就不會丟資料。想清掉重來才用 down -v。詳見 1.1.3 troubleshoot 案 8。

(08)

檢核題

Q1：n8n 的 flows 與 credentials 實際存在哪裡？
A. ~/n8n-starter-kit/ 資料夾的某個檔案
B. .env 檔
C. Docker volume n8n_data（不在你看得到的硬碟資料夾）
D. PostgreSQL 容器的記憶體
看答案
答案是 C。n8n 把資料寫進 Docker volume，volume 是 Docker 管理的持久化儲存空間，不是你日常可以在 Finder 或檔案總管看到的資料夾。這也是為什麼 down（不加 -v）停容器後資料不會消失，但 down -v 會把 volume 一起刪掉。
Q2：阿凱想把 n8n 升級到最新版本，但又怕升級後 flows 不見。他應該怎麼做？
A. 先手動備份 ~/n8n-starter-kit/ 整個資料夾，再跑 down -v 後重拉
B. 雙擊 update.command（Mac）或 update.bat（Windows），升級不會碰 volumes
C. 直接在 Docker Desktop GUI 點「Update image」按鈕
D. 刪掉整個 n8n-starter-kit 資料夾重新下載試跑包
看答案
答案是 B。update.command / update.bat 背後跑的是 docker compose pull && docker compose up -d，只更新 image，不動 volumes，所以 flows 和 credentials 不受影響。切忌用 down -v——那會刪掉所有資料。
