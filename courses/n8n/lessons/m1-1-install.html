Docker Desktop 安裝（Mac / Windows 雙平台）

25 分鐘。目標：從零到「終端機跑 docker --version 吐得出版本號」。選你的路徑（A / B / C），跟著走，不需要看其他路徑。

學習成果
能依自己的 OS 與 CPU 架構選到正確的 Docker Desktop 安裝包
能完成安裝、首次授權、啟動，看到鯨魚圖示且狀態為綠燈
能在終端機 / PowerShell 跑 docker --version 看到版本號
(01)

你是誰？先確認路徑

為什麼這很重要
Mac Intel 和 Mac Apple Silicon 的 Docker Desktop 是兩個不同的安裝包。下錯版本裝完會跑不起來，重裝才能修。30 秒確認 CPU 架構，省掉 10 分鐘除錯。

阿凱的 MacBook 是 M2 晶片，剛剛在 1.1.0 確認自己屬於路徑 B（Mac Apple Silicon）。他打開 docker.com，看到首頁有兩個按鈕：「Download for Mac (Apple Silicon)」和「Download for Mac (Intel)」。旁邊雯姊用的是 Intel iMac，要點另一個。

本頁有 三條路徑，找到你的那條之後，只需要看那一條就好。

路徑 A
Mac Intel（2020 年之前的 Mac）
Apple 選單 → 關於這台 Mac → 晶片顯示「Intel Core i...」
路徑 B
Mac Apple Silicon（M1/M2/M3/M4）
Apple 選單 → 關於這台 Mac → 晶片顯示「Apple M1」或更新
路徑 C
Windows 10/11 + WSL2
Windows 機器。需要先確認 WSL2 是否已啟用（詳見路徑 C 步驟 1）
不確定？
先回 1.1.0 確認架構
前一頁 m1-1-overview.html 有詳細的架構辨識流程，30 秒可確認
(02)

關鍵概念：裝這 4 個詞

安裝過程會遇到這些詞。先知道它們是什麼，遇到時不會慌。

01
Docker Desktop
一個桌面應用，裝它就等於同時裝完 Docker Engine（跑容器的核心）+ 圖形管理介面 + 自動更新機制。
02
WSL2
Windows Subsystem for Linux 2。Win 上跑 Linux 容器需要的底層，Docker Desktop on Win 預設以它作為 Engine 後端。
03
虛擬化（Virtualization）
CPU 提供的硬體支援，讓電腦能跑容器。Mac 預設開啟；Win 多數預設開啟，少數機種需進 BIOS 才能開。
04
Apple Silicon vs Intel
兩種 CPU 架構各自需要對應版本的 Docker Desktop。下錯版本裝完會跑不起來，需重裝正確版本。
(03)

安裝步驟 — 選你的路徑

三條路徑互相獨立。找到自己的，跟著走，不需要看其他路徑。

1
開瀏覽器到 Docker 官網

前往 https://www.docker.com/products/docker-desktop/

預期看到什麼
頁面中央有「Download Docker Desktop」按鈕，下方列出 Mac / Win / Linux 三個分頁。
2
選 Intel 版本下載

點 Mac 分頁 → 選 「Mac with Intel chip」（不是 Apple Silicon）。

預期看到什麼
開始下載 Docker.dmg（約 600MB），下載 1–3 分鐘視網速而定。
3
雙擊 .dmg 開啟安裝視窗
預期看到什麼
彈出小視窗：左邊是 Docker 鯨魚圖示，右邊是 Applications 資料夾捷徑，中間有箭頭。
4
拖曳鯨魚到 Applications
預期看到什麼
顯示複製檔案進度條，約 30 秒完成。
5
到 Launchpad 找 Docker 雙擊打開
若出現「Docker.app 是從網際網路下載的，你確定要打開嗎？」→ 點「打開」。這是 macOS Gatekeeper 第一次驗證，正常現象。
預期看到什麼
Docker Desktop 啟動畫面，要求授權 privileged helper（系統權限）。
6
輸入 Mac 密碼授權 → 同意 service agreement → 跳過教學
預期看到什麼
右上角選單列出現鯨魚圖示，點開應寫 Docker Desktop is running（綠燈）。視窗內顯示 dashboard，左邊有 Containers / Images / Volumes 等選單。
7
驗證：打開終端機跑驗證指令

Cmd+Space 搜尋「終端機」開啟，輸入：

docker --version
預期看到什麼
吐出類似 Docker version 27.3.1, build ce12230 的字串。版本號數字不重要，看到 Docker version 開頭就過關。
1
開瀏覽器到 Docker 官網

前往 https://www.docker.com/products/docker-desktop/

2
選 Apple Silicon 版本下載

點 Mac 分頁 → 選 「Mac with Apple chip」（不是 Intel）。

預期看到什麼
開始下載 Docker.dmg（約 600MB）。
3
安裝步驟 3–6 與路徑 A 相同

雙擊 .dmg → 拖曳到 Applications → 首次啟動授權 → 同意 service agreement → 跳過教學。完成後右上角選單列出現鯨魚，狀態 Docker Desktop is running（綠燈）。

4
驗證：打開終端機跑指令

Cmd+Space 搜尋「終端機」開啟，輸入：

docker --version
預期看到什麼
吐出 Docker version 27.x.x, build ...。看到 Docker version 開頭就過關。
i
Apple Silicon 額外注意：部分 Linux 容器是 x86 架構，在 Apple Silicon 上會有「platform mismatch」黃色警告。這不是錯誤，本課程的 n8n 與 PostgreSQL 都有原生 ARM 版本，不會出現此警告。如果之後跑其他 image 出現此警告，到 1.1.3 troubleshoot 案 3 看怎麼處理。
1
先檢查 WSL2 是否已啟用

打開 PowerShell（以系統管理員身分執行：右鍵點開始 → 終端機（系統管理員）/ Windows PowerShell（系統管理員））。輸入：

wsl --status
預期看到什麼
已啟用且為 v2：顯示「預設版本: 2」（或英文 Default Version: 2）→ 直接跳到步驟 3。
未啟用 / 顯示找不到指令：繼續步驟 2。
2
啟用 WSL2（僅未啟用時需做）

在系統管理員 PowerShell 輸入：

wsl --install
預期看到什麼
自動下載並安裝 WSL2 + Ubuntu 預設發行版，完成後要求重新啟動電腦。重啟之後再跑一次 wsl --status 確認顯示「預設版本: 2」。
如果跳出「需要啟用虛擬化」：到工作管理員 → 效能 → CPU，看右下角「虛擬化」是否「已啟用」。沒啟用要進 BIOS 開：開機按 F2 / Del / Esc（依主機板），找 Intel VT-x / AMD-V / SVM Mode 啟用。詳細流程見 1.1.3 troubleshoot 案 4。
3
下載 Docker Desktop for Windows

開瀏覽器到 https://www.docker.com/products/docker-desktop/，點 Windows 分頁 →「Download Docker Desktop for Windows」→ 下載 Docker Desktop Installer.exe（約 700MB）。

4
執行安裝精靈

雙擊 .exe 開啟安裝精靈。

預期看到什麼
精靈第一頁有兩個勾選框：「Use WSL 2 instead of Hyper-V (recommended)」保持勾選；「Add shortcut to desktop」依個人喜好。
5
按 OK → 等安裝完成 → 重啟

安裝約 3–5 分鐘，提示「Close and restart」按下去重啟電腦。

6
重啟後完成首次設定

重啟後 Docker Desktop 自動啟動。首次啟動會跳：service agreement 同意 → 登入帳號（可跳過，按右上角 Skip）→ 教學影片可跳。

預期看到什麼
右下角通知列出現鯨魚圖示，狀態為 Docker Desktop is running（綠燈）。
7
驗證：打開 PowerShell 跑指令

打開 PowerShell（不需系統管理員），輸入：

docker --version
預期看到什麼
吐出類似 Docker version 27.3.1, build ce12230 的字串。看到 Docker version 開頭就過關。
(04)

動手練習

依你在 1.1.0 確認的路徑（A / B / C），完成從下載 → 安裝 → 首次啟動 → 終端機驗證的全流程。

完成標準（自檢）：

鯨魚圖示
出現在 Mac 右上角選單列 / Win 右下角通知列
狀態燈號
顯示 Docker Desktop is running（綠燈），不是 Starting（黃燈）或 Stopped（紅燈）
終端機指令
docker --version 看到 Docker version 開頭的字串
i
時間預期：下載 1–3 分鐘 + 安裝 2–5 分鐘 + 首次啟動授權 1–3 分鐘 + 驗證 30 秒。Win 若需啟用 WSL2 / BIOS 虛擬化則另加 5–15 分鐘。
(05)

完整驗證 — 跑三個指令

打開終端機（Mac）/ PowerShell（Win）依序跑這 3 個指令，每個都要看到合理輸出：

docker --version # 應吐出版本號 docker info # 應看到一大串系統資訊（Server / Storage / Plugins） docker run hello-world # 應看到 "Hello from Docker!" 訊息
通關確認
最後一個 docker run hello-world 是 Docker 官方的測試 image，跑起來會自動下載並執行一個迷你容器，印出歡迎訊息。看到「Hello from Docker!」表示 Docker Engine + 容器執行整條鏈路都通了。如果這 3 個指令任何一個出錯，跳 1.1.3 troubleshoot 對症下藥。
(06)

常見錯誤 3 條

錯誤 1
架構不符
現象：Mac 下載到 Intel 版裝在 Apple Silicon 上（或反過來），啟動後鯨魚圖示一直 Starting 或顯示 corrupt。
原因：CPU 架構不符，安裝包跟硬體對不起來。
解法：到 Applications 把 Docker.app 丟垃圾桶 → 重新到 docker.com 下載對應晶片版本 → 重裝。判斷晶片：Apple 選單 → 關於這台 Mac → 晶片欄位。
錯誤 2
需要啟用虛擬化
現象：Win 跑 wsl --install 顯示「需要啟用虛擬化」。
原因：CPU 虛擬化（Intel VT-x / AMD-V）在 BIOS 預設關閉。
解法：重開機按 F2 / Del / Esc 進 BIOS（依主機板），在 Advanced / CPU Configuration 找 Intel VT-x / AMD-V / SVM Mode 啟用 → 存檔重啟 → 再跑 wsl --install。完整流程在 1.1.3 troubleshoot 案 4。
錯誤 3
Win 一直 Starting
現象：Win Docker Desktop 安裝完啟動，狀態一直 Starting 不變綠燈，過 10 分鐘還在轉。
原因：WSL2 後端沒裝好，或 Hyper-V 與 WSL2 衝突。
解法：先在 PowerShell 跑 wsl --update（強制升級 WSL 核心）→ 重啟 Docker Desktop。仍卡住跑 1.1.3 troubleshoot 案 4。
(07)

檢核題

Q1（概念驗證）：跑 docker run hello-world 看到「Hello from Docker!」代表什麼？
A. n8n 已經安裝完成
B. Docker Engine 與容器執行整條鏈路都正常
C. 你的網路連得到 Docker Hub
D. WSL2 已啟用
查看解析
正確答案是 B。hello-world 這個指令完整走了「從 Docker Hub 拉取 image → 建立容器 → 執行 → 印出訊息」整條鏈路。這不驗證 n8n（還沒裝），也不單獨驗證網路或 WSL2——而是整條 Docker Engine 執行鏈路。
Q2（應用驗證）：雯姊用 Win 11，跑 wsl --install 顯示「需要啟用虛擬化」，她接下來該怎麼做？
A. 重新安裝 Windows
B. 換用 Mac
C. 進 BIOS 找 Intel VT-x / AMD-V / SVM Mode 啟用後重啟，再跑 wsl --install
D. 直接跳過 WSL2，安裝 Docker Desktop 用 Hyper-V 模式
查看解析
正確答案是 C。先到工作管理員 → 效能 → CPU 確認「虛擬化」狀態：已啟用代表其他問題，已停用才需進 BIOS。進 BIOS 後（開機按 F2 / Del / Esc 依主機板）找 Intel VT-x / AMD-V / SVM Mode 啟用 → 存檔重啟 → 再跑 wsl --install 應該就通。
