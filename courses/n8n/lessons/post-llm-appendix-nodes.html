附錄：Lite Pack 14 workflow 節點對照表

改 workflow 前先查這頁 — 把「保留節點」「要改節點」對應到實際名稱。本頁是 reference card，不需要依序讀，找到你的 workflow 號碼直接跳過去。

這頁的用途 · HOW TO USE
填 prompt 模板 1「必保留的節點」時，先查對應 workflow 的節點清單
確認節點的 typeVersion，避免改動 A5 紅線 3 禁止觸碰的版本號
快速辨識哪些節點是「核心改造點」、哪些「保留即可」
(01)

怎麼用這頁

三個使用場景
填 prompt 模板前查節點清單、改 workflow 時確認保留範圍、遇到 typeVersion 疑問時核對紅線 3。

你打開 LLM 對話、要填 prompt 模板 1 的「必保留的節點」，先查這頁找對應 workflow 的節點清單。

表格說明：名稱欄是你在 n8n UI 上會看到的中文標籤；type 欄是節點的技術識別碼；typeVersion 欄照 workflow JSON 原始值列出，不要主動改（這是 A5 紅線 3）。

📌
「📌 使用說明」是 sticky note（n8n 視覺註解），在 n8n UI 上是一張黃色便利貼，永遠不動。它不參與 workflow 執行，但刪掉會讓改造記錄消失。
🔶
表中橘色底色 = 核心改造點，這些節點有 prompt 或輸出格式需要你依需求調整。其餘白底 = 保留即可（路徑、資料夾名稱等小調整除外）。

(04)

typeVersion 紅線提醒

A5 紅線 3 明確禁止主動修改 typeVersion。這個欄位由 n8n 管理，版本不符會讓節點無法執行。你的工作是改 prompt、改路徑、改分類規則——不是改版本號。

ℹ️
如果你的 n8n 升版後 UI 提示「節點有新版可用」，那是 n8n 自己的升級提示，按照 n8n 的流程操作即可——這跟 LLM Copilot 改 workflow 流程無關，不要混淆。
(05)

找不到節點？三個可能原因

如果你在 n8n UI 上找不到對應的節點名稱，排查這三個方向：

💡
01 · workflow 版本不一致 — 你匯入的 JSON 檔不是這頁對應的版本。重新從課程資產下載 .json 並重新匯入。
💡
02 · 節點被改名了 — 你（或協作者）曾經在 n8n UI 上改過節點標籤。節點標籤可以改，type 不會變。去看節點設定面板右上角的 type 識別碼做對照。
💡
03 · workflow 號碼查錯了 — 回到這頁確認你在改的是哪個號碼的 workflow，不同 workflow 的同名節點（例如「📌 使用說明」）是不同的節點實例。
