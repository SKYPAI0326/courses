# Google Form 欄位範例（for Module 4 演練）

對應單元：`m4-1-remote.html` / `m4-1-data.html` / `m4-1-control.html`

學員建立 Google Form 時，可參考以下欄位設計，方便對照課程教材的 Webhook payload 解析示範。

## 範例 1：客戶訂單表（攝影工作室情境）

| 欄位名稱 | 類型 | 必填 | 說明 |
|---------|------|------|------|
| 客戶姓名 | 簡答 | 是 | 用於建立資料夾 |
| 聯絡 Email | 簡答 | 是 | 用於回覆通知 |
| 拍攝日期 | 日期 | 是 | 用於排期 |
| 服務類型 | 單選 | 是 | 婚禮 / 商品 / 個人寫真 |
| 是否需修圖 | 是非 | 否 | 影響後續流程 |
| 備註 | 段落 | 否 | 自由輸入 |

對應 n8n Webhook 收到的 payload（精簡）：

```json
{
  "客戶姓名": "陳測試",
  "聯絡 Email": "test@example.com",
  "拍攝日期": "2026-02-14",
  "服務類型": "婚禮",
  "是否需修圖": "是",
  "備註": "希望 2 月底前交件"
}
```

## 範例 2：日報觸發表（每日下班按一下）

| 欄位名稱 | 類型 | 必填 |
|---------|------|------|
| 日期 | 日期 | 是 |
| 工作項目（多選） | 核取方塊 | 是 |
| 重點筆記 | 段落 | 否 |
| 是否要產出日報 | 是非 | 是 |

## 範例 3：遠端遙控指令表（簡化版）

| 欄位名稱 | 類型 | 必填 | 說明 |
|---------|------|------|------|
| 指令類型 | 單選 | 是 | scan / report / cleanup |
| 目標路徑 | 簡答 | 否 | 預留參數欄 |
| 通知對象 | 簡答 | 否 | LINE userId 或 email |

## 設計提示

- 欄位名稱建議用**英文小寫底線**（如 `customer_name`）方便 JSON 處理；中文欄位也可用，但 n8n 取值要用 `$json["客戶姓名"]` 帶引號
- 所有欄位的「描述」欄填上預期格式範例，避免學員填錯造成 webhook 解析錯誤
- Form 整合 n8n 的 Apps Script 寫法見 `m4-1-remote.html` 步驟 3
