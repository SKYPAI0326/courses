## 重設計內容

## 第 2 章重設計：六步流程一頁就上手

### 1. 學員此時的痛點

學員看完第 1 章決定「ok 我繼續看下去」，到第 2 章他/她要的是：

- **整體骨架我能不能 5 分鐘看完？**（這章定位是「總覽」，不要塞細節）
- **6 步裡哪幾步是關鍵、哪幾步可以快過？**
- **這章看完我能不能直接動手？**（理想：能；現實：第 2 章不該塞所有 prompt，但要讓學員看完就「知道下一頁讀什麼」）

### 2. 核心要傳遞的 1 個觀念

**6 步流程不是「要做 6 件事」，是「2 個迴圈」：(a) 動工前的 3 步準備（拆需求、讀 JSON、限定範圍）+ (b) 動工後的 3 步驗收迴圈（測試、回報、收斂）。前 3 步沒做好，後 3 步會無限迴圈。**

### 3. 具體 step-by-step

第 2 章學員不動手，但要讀完後能描述：

- 流程進場條件：「我已經 export 了 #02 的 JSON，已經知道我要改成處理發票」
- 流程出場條件：「pdf-renamed 資料夾裡有 200 張改名後的發票」
- 中間 6 步：每步 30 秒 elevator pitch（不是表格，是一句話）
- 迴圈在哪：步驟 5 失敗後**只回到步驟 3**（不是步驟 1）；超過 3 輪不過才退步驟 1

### 4. 完整範例 prompt 模板

第 2 章不獨立給模板（5 個模板在第 3-4 章詳解），但要給**「6 步裡每步的 1 行 prompt 縮影」**讓學員看到風格：

```
步驟 1（拆需求）：不用 LLM
步驟 2（讀 JSON）：「請把這份 workflow 翻成人話，列每節點輸出/輸入」
步驟 3（限定範圍）：「只改節點 X，回傳區段 JSON，不重寫整份」
步驟 4（匯入測試）：不用 LLM
步驟 5（錯誤回報）：「貼錯誤訊息+input+parameter，請只指出錯哪格」
步驟 6（收斂）：「請保留最小修改，移除你新增的防禦邏輯」
```

### 5. 範例 LLM 回應

第 2 章不執行完整 prompt，但要給**「步驟 2 sample LLM 回應」**作為整章的具象 hook（學員第一次看到「啊原來 LLM 會這樣回」）：

```
LLM 會回（範例 — 步驟 2 拿到的 #02 翻譯）：

這份 workflow 做什麼：
從 pdf-inbox 資料夾讀取所有 PDF，用 Gemini 分析內文後給每張 PDF 一個語意化檔名，
並把檔案改名後存到 pdf-renamed 資料夾。整套流程是手動觸發。

節點清單：
1. Manual Trigger — 手動觸發點，不接受輸入。
2. Read PDF — 讀 /files/pdf-inbox/ 資料夾，輸出每張 PDF 的二進位內容。
3. Extract PDF Text — 把二進位 PDF 轉成純文字。
4. Code: AI 改名 — 用 Gemini API 分析 PDF 文字，產出 newFilename 字串。
5. Write — 把原 PDF 用 newFilename 改名後存到 /files/pdf-renamed/。

資料流向：Manual Trigger → Read PDF → Extract PDF Text → Code → Write
```

### 6. 驗收標準

學員讀完第 2 章後應該能回答：

- [ ] 我能用一句話說出 6 步流程的「進場條件」和「出場條件」
- [ ] 我能說出哪 3 步是「動工前準備」、哪 3 步是「動工後驗收」
- [ ] 我知道步驟 5 失敗後迴圈回到步驟 3（不是步驟 1）
- [ ] 我能說出至少 1 個跳步的後果（從原文「為什麼不能跳步」表格挑 1 個）
- [ ] 我看到一段 LLM 翻譯回應（步驟 2 sample），知道大概格式是「先一段話描述 → 再列節點清單」

### 7. 常見錯誤 + 怎麼解

**錯誤 1：「我覺得 6 步太繁瑣，我自己改 1-2 步就好」**
- 解：商業培訓學員第一次走完整流程平均 40 分鐘。從第 3 次開始大概 15 分鐘。短期看似繁瑣，但跳步單次出事 debug 平均 1-2 小時。算總帳是 6 步省時間

**錯誤 2：「步驟 4 跑了 1 次綠燈，我直接放 200 張上去」**
- 解：1 筆綠燈不代表結構正確。用第 2 筆不同類型的樣本跑（例如不同供應商發票）才能發現「prompt 寫得太死」之類的問題。10 筆是經驗值下限

### 8. 回到課程動線

- **下一頁 (第 3 章)**：步驟 1-3 的詳細操作（4 格便箋、模板 1-3、示範形狀法）
- **跳讀**：如果你急著動手，看完第 3 章 + 第 4 章 = 完整六步操作指南；第 5、7 章是 reference

### 第 2 章 post-llm-2-flow — 6 步流程總覽

**整體判定**：這章是表格體，看起來訊息密度很高，但**幾乎全部是抽象動詞填表**（判定 1）。每格用「LLM 翻成人話」「精準回報」「最小修改」這種詞 — 學員看完知道每步要做什麼動詞，但不知道這個動詞落到鍵盤上是什麼動作。

| # | line | 原文摘要 | 違反 | 為什麼會卡住學員 |
|---|---|---|---|---|
| A2-1 | 184-191 | 步驟 1 表格「LLM 做什麼：不做。/ 你做什麼：填一張 4 格便箋」 | 1, 2 | 4 格便箋長什麼樣？只在第 3 章秀。第 2 章是「總覽」應該要可以獨立看完，看完就能用，但這裡學員必須翻到第 3 章才能填。應該至少在表格下方放一張縮圖式的 4 格範例 |
| A2-2 | 207-211 | 步驟 2 表格「驗收訊號：你能用自己的話複述一遍 LLM 的描述，不用看 JSON」 | 2 | 「複述一遍」是模糊驗收。商業培訓學員會問「複述到什麼程度算過？要說 5 句話還是 1 段話？」應該給一個 mini scoring：「能講出 input → 中間做了什麼 → output，3 句話以內」 |
| A2-3 | 225-231 | 步驟 3「動作：告訴 LLM『只改節點 X、Y、Z』，要求它只回傳這幾個節點的 JSON 區段，不是整份」 | 1, 3 | 怎麼告訴？要打哪些字？沒給範例 prompt。學員會問「我直接寫『只改節點 X』就可以了嗎？要不要說它為什麼不能改別的？」這正是空話的核心：動詞 + 賓語但沒給可貼的字串 |
| A2-4 | 249-251 | 步驟 4「驗收訊號：所有節點綠燈，output 結構符合步驟 1 便箋裡寫的目標 output」 | 2, 4 | 「output 結構符合」需要學員看 n8n UI 的 output panel——但這章沒教怎麼看 output panel。學員到 n8n UI 看到 output panel 顯示的是 JSON tree，會迷失。要嘛這章補一段「output panel 怎麼讀」要嘛明確指向「第 4 章 step XX 教你看」 |
| A2-5 | 264-271 | 步驟 5「動作：複製三件事給 LLM：(a) 紅燈節點的錯誤訊息全文，(b) 該節點的 input panel JSON，(c) 該節點當前的 parameter JSON（移除 credential）」 | 1 | 「複製三件事」聽起來很清楚，但實際上 input panel 在 n8n UI 哪裡？parameter JSON 在哪裡複製？「移除 credential」具體要刪哪幾個欄位？這些操作步驟全部沒給 |
| A2-6 | 285-295 | 步驟 6「驗收訊號：10 筆樣本全數綠燈」 | 5 | 10 筆從哪裡來？學員手上可能只有 1 張範本發票，要怎麼湊 10 筆？應該明說「從你 200 張裡隨機挑 10 張」或「沒有 10 張就用 5 張，但低於 5 張不能直接上 200」 |
