# post-llm 系列重設計 — Phase A 診斷與後續建議

> 本檔記錄 2026-05-07 對 8 章「課後用 LLM 改 workflow」原始 HTML 的「空話診斷」共 56 條，以及後續課程設計建議。每章的修法落地在對應的 `post-llm-{N}-*.md` 教案。

## Phase A — 8 章空話診斷

### 判定標準（從用戶 brief 抄回）

一段如果符合**任一**這條就是空話：

1. 只有抽象動詞沒有具體動作（沒給範例 prompt）
2. 沒給「拿到什麼算對」的驗收標準
3. 警告「不要做 X」但沒教「怎麼做 X 才安全」的替代路徑
4. 引用其他章節但該章沒給可獨立讀通的具體要點
5. 純技術名詞堆砌，沒給商業情境 anchor
6. 「步驟 N」標題下面只重述標題、沒實質內容

### 第 1 章 post-llm-1-overview — 開場與設計取捨

**整體判定**：開場章本來就會比較高層，但這章踩了**判定 4**（大量引用其他章節卻沒有 hook 學員「我為什麼要繼續看」）和**判定 5**（自圓自說「教什麼／不教什麼」，學員看不到自己的場景）。

### 第 5 章 post-llm-5-redlines — 8 條紅線

**整體判定**：使用者提示這章可能「只列禁忌沒給安全替代」是準的。8 條紅線裡有 4 條（紅線 1、4、5、6）只說了「不要做」沒給「正確要做什麼」（判定 3）。另外**整章都有判定 5**：每條紅線的「為什麼」段落用大量技術名詞（typeVersion、connections、OAuth callback URL）但沒給商業情境 anchor，學員看完能背但不會用。

### 我自評的「最大改動」1-2 章

#### 最大改動第 1 名：第 6 章（Phase C 對話劇本）

**改動規模**：原 HTML 11 個 placeholder（步驟 2-6 的所有 LLM 對話 + 截圖）全部從「行為大綱」升級為「可貼可跑的對話腳本」。

**為什麼這樣改**：
- 使用者最痛的反饋是「閱讀過後發現純粹空話，作為學員不知從何下手」
- 第 6 章在當前狀態下對學員價值最低（placeholder 占主體）
- 商業培訓非工程師學員最需要的就是「我看別人怎麼做，照做就會」— 這正是 walkthrough 章該交付的，但原版沒交
- Phase C 透過完整實寫 4 個關鍵岔路（整份反例 / null_null 劇情 / 精準診斷 sample / try/catch 拒絕），把第 1-5 章的所有抽象概念**全部具象化**到一個學員可以全程跟著走的 60-90 分鐘體驗
- 改動完後第 6 章從「最浮泛章」變成「整個系列最核心的學習資產」

#### 最大改動第 2 名：第 5 章（紅線從禁忌列表 → 危險+替代雙欄）

**改動規模**：8 條紅線全部從「2 段式（危險+為什麼）」改成「3 段式（危險+為什麼+安全替代）」，特別是紅線 2 的「禁止項目清單」（webhook URL / trigger / schedule / recipient / delete）每條都補了具體替代動作。

**為什麼這樣改**：
- 商業培訓學員的真實情境：他/她**不會**因為紅線禁止就放棄改 webhook URL — 業務需求逼著他/她要改。如果只給「不能讓 LLM 改」沒給「正確怎麼改」，學員只會偷偷把 LLM 給的 webhook URL 直接貼進去（更危險）
- 改動後每條紅線變成「危險動作清單 + 自己在 n8n UI 怎麼安全做」雙欄，學員看完不只知道避坑，還知道「我真的需要做時走哪條路」
- 紅線 4（節點 deprecation）特別重做了「4 順位 fallback」，把 google changelog 這種非工程師做不到的事降級成可執行動作（google 搜尋 → n8n UI 搜尋 → 課程社群 → 不直接信 LLM）
