# 回家 30 分鐘復跑手冊

> 課堂上跟著老師按了幾下、Yoast 燈號變綠、GA4 即時報表看到事件——但回家用真實網站的真實資料跑，第一次幾乎一定卡住。
> 這份手冊是給你回家用「真實工作場景」復跑五個核心案例的食譜。每個案例約 30 分鐘，五個全跑完是 2.5 小時。
> 課程是 30 小時，這 2.5 小時的回家練習，決定你是「上完課」還是「會做」。

---

## 共同前置：第一次跑前確認 3 件事（5 分鐘）

### 1. 資料份量

復跑時你會需要：
- 一個能改的測試網站（推薦：LocalWP 的 ntub-demo，不要用真實公司網站練）
- 一份你業界的關鍵字種子（5 個核心詞就夠開始）
- 一個 Gemini 或 ChatGPT 帳號（免費版即可，付費版品質更穩）
- 一個 Google 帳號（GA4 + GSC + Google Ads 規劃工具共用）

### 2. 資料敏感度

下面三件事**絕對不要在 AI 對話視窗貼出來**：
- 真實客戶的姓名 / 電話 / Email（即使你只是要 AI 幫忙寫信）
- 公司內部的營業額 / 客戶名單 / 員工資料
- 任何含「機密」「內部」「不公開」字樣的文件

如果一定要用 AI 處理真實資料，**先去識別化**：把姓名改 `客戶A`、電話改 `XXXX`、公司名改 `某客戶`。AI 對話歷史可能被用於訓練。

### 3. 工具選擇

| 用途 | 免費首選 | 付費升級時機 |
|------|---------|------------|
| 關鍵字研究 | Google Ads 規劃工具（搜尋量區間） | 接案／真實上線時用 Ahrefs 或 Semrush |
| AI 寫 prompt | Gemini Free / ChatGPT Free | 每天用超過 30 次再考慮 Plus |
| GA4 | 完全免費 | — |
| GTM | 完全免費 | — |
| GSC | 完全免費 | — |

## 案例 ① 關鍵字研究 + AI 標記意圖 — 30 分鐘食譜

> 對應講義：m3-1（Google Ads 關鍵字工具）+ m2-3（關鍵字三分類）

### 步驟 1：整理資料（10 分鐘）

開 Google Ads 規劃工具 → 探索新關鍵字 → 輸入你業界的 3-5 個核心詞（例如：「客製化禮品」「企業送禮」「員工尾牙禮品」）→ 語言「中文（繁體）」、地區「台灣」→ 取得結果。

勾選 50 組關鍵字 → 新增至計畫 → 下載 CSV。

### 步驟 2：套 prompt（3 分鐘）

打開 ChatGPT，複製 [prompts-all.md](datasets/prompts-all.md) 的 **P3-1-A 批次標記意圖** prompt，把 CSV 內容貼進去 prompt 的占位符。

### 步驟 3：跑 AI 並讀（5 分鐘）

讀 AI 給的表格 → 確認所有 50 組都有「意圖類型 / 分類 / 優先級」三欄。

### 步驟 4：追問修正（5 分鐘）

抽 5 組驗證：你直覺認為是「交易型」的詞，AI 標的對不對？看到 AI 把明顯的交易詞標成「資訊型」，追問「為什麼？」AI 通常會承認誤判並重標。

### 步驟 5：驗證輸出（7 分鐘）— **這步絕對不能省**

用 Google 實際搜尋你認為「高優先級」的 3 個詞，看 SERP 第一頁長什麼樣：
- 第 1 頁全是大品牌（蝦皮 / momo） → 你的優先級可能要降一級
- 第 1 頁有部落格文章 / 個人站 → 維持高優先級
- 第 1 頁有 AI Overview → 補做 m3-3 的 FAQ + Schema

## 案例 ② Title + Meta + H1 三件套 — 30 分鐘食譜

> 對應講義：m4-3（On-Page 上篇）

### 步驟 1：整理資料（5 分鐘）

選一篇 LocalWP 上現有的演練文章 → 記下「主要目標關鍵字」「次要關鍵字」「文章受眾」三件事。

### 步驟 2：套 prompt（3 分鐘）

打開 ChatGPT，套 **P4-3-A Title 三件套生成** prompt，把上面三件資訊填進去。

### 步驟 3：跑 AI 並讀（3 分鐘）

AI 會給 5 個 Title 候選 → 標哪個最適合搜尋意圖。

### 步驟 4：追問修正（10 分鐘）

照下面 3 個檢查項追問：
1. **長度檢查**：用 [Yoast SEO 預覽](https://yoast.com/check-title/) 驗證每個 Title 在 SERP 上的視覺長度（約 60 字元 / 920px）
2. **獨特性**：用 Google 搜尋你的 Title 確切字串（加引號），如果第 1 頁很多人在用同樣標題，追問 AI「改寫得更有差異化」
3. **意圖貼合**：用 Google 搜尋「目標關鍵字」，看第 1 頁排名前 5 的 Title 都用什麼角度，追問 AI「改寫成跟競品都不同的角度」

### 步驟 5：驗證輸出（9 分鐘）

把選定的 Title + Meta + H1 貼進 LocalWP 的 Yoast SEO 欄位 → 看燈號是否變橘 / 綠 → 在「重點關鍵字」也填入主關鍵字 → 看 Yoast 給的具體建議（關鍵字密度 / 內文長度 / 副標題分佈）逐一處理。

## 案例 ③ 批次補 Alt 文字 — 25 分鐘食譜

> 對應講義：m5-1（On-Page 下篇）

### 步驟 1：整理資料（5 分鐘）

到 LocalWP 媒體庫 → 找出你演練網站裡所有「沒 alt」或「alt 是檔名」的圖片 → 列一份清單（檔名 + 圖片內容簡述 + 使用情境，每張一行）。

### 步驟 2：套 prompt（2 分鐘）

打開 ChatGPT，套 **P5-1-A 批次補 Alt** prompt，把清單貼進去。

### 步驟 3：跑 AI 並讀（3 分鐘）

AI 會一次給所有 Alt 文字。

### 步驟 4：追問修正（10 分鐘）

**這是最容易踩雷的一步**：
- 同主題（例如 5 張禮盒圖）的 Alt 不能都長很像 → 追問「2/3/5 不需要『客製化』這詞，改寫得有差異」
- 看到 Alt 連續出現「客製化禮盒 + 修飾詞」(客製化禮盒精美款 / 客製化禮盒高級款) → 那是關鍵字填充，不是描述
- 裝飾性圖（分隔線、背景花紋）AI 偶爾會硬寫描述 → 改成 `alt=""`

### 步驟 5：驗證輸出（5 分鐘）

回 LocalWP 媒體庫 → 逐張更新「替代文字」欄位 → 用 [WAVE](https://wave.webaim.org/) 線上工具測你的網頁 → 確認「Missing alternative text」警告變 0。

## 案例 ④ GA4 自訂事件設計 + 即時報表驗證 — 35 分鐘食譜

> 對應講義：m7-2（自訂事件）+ m7-3（DebugView）

### 步驟 1：整理資料（5 分鐘）

列出你的演練網站想追蹤的 3-5 個使用者行為（不要寫太多，新手先求做完 3 個）。範例：
- 「立即詢價」按鈕點擊
- 「企業提案 PDF」下載
- 「免費索取樣品」表單提交

### 步驟 2：套 prompt（3 分鐘）

套 **P7-2-A 事件結構設計** prompt，把行為清單貼進去。

### 步驟 3：跑 AI 並讀（5 分鐘）

AI 給每個事件的事件名 + 參數結構（表格格式）。

### 步驟 4：追問修正（15 分鐘）

逐項檢查：
- **命名規則**：全小寫底線？有沒有用到保留字（ga_/google_/firebase_）？看到大寫或駝峰就追問改 snake_case
- **參數合理性**：3 個參數夠不夠用？AI 偶爾只給 2 個就停 → 追問「補到 3-4 個有意義的維度」
- **是否該標轉換**：哪幾個事件值得被標為「轉換」？AI 通常會建議全標，**那是錯的**——只標真正完成業務目標的事件（表單提交、購買、電話點擊）

### 步驟 5：驗證輸出（7 分鐘）— **這步絕對不能省**

把對應的 gtag JavaScript code 貼進 WordPress 的 Head & Footer Code 外掛 → 在 LocalWP 網站實際觸發（點擊按鈕、下載 PDF、送表單）→ 切到 GA4 → 報表 → 即時 → 「依事件名稱」區塊應該 1-2 分鐘內看到你的自訂事件。

**沒看到怎麼辦？**
1. 確認 gtag.js 載入（DevTools → Network → 搜尋 `collect`）
2. 確認 Head & Footer Code 外掛已儲存
3. 確認自訂腳本放在 gtag.js 之後
4. **不要用 console.log 偵錯 gtag**，gtag 沒有回傳值，用 GA4 DebugView 才看得到

## 案例 ⑤ GTM 觸發條件 + DataLayer push — 40 分鐘食譜

> 對應講義：m9-3（GTM 事件）+ m7-2（自訂事件）

### 步驟 1：整理資料（5 分鐘）

選 1 個你最想追蹤但 GA4 加強型評估抓不到的行為（例如「客製化頁面拖曳滑桿」「特定 div 進入視野」）。記下：
- 觸發的 HTML 元素（class / id）
- 你想記錄什麼維度（產品 ID / 選的材質 / 操作時間 / 等等）

### 步驟 2：套 prompt（3 分鐘）

套 **P9-3-A 業務需求翻 GTM 設定** prompt，把上面資料填入。

### 步驟 3：跑 AI 並讀（5 分鐘）

AI 給「觸發條件 + 變數 + 標籤」三件套設定建議。

### 步驟 4：追問修正（15 分鐘）

**GTM 的設定錯漏是最常見的問題**：
- AI 漏配 currency 參數（GA4 的 value 必須配 currency 才會被算入電商報表）→ 追問補
- AI 用 Click Element 屬性取值不穩定 → 追問改用 DOM Element 變數
- AI 給的 querySelector 只抓單一元素 → 追問「多個元素用 querySelectorAll forEach」
- AI 給的 DataLayer push 把資料寫死 → 追問「改成從 PHP 變數動態帶入，給我 WP 對應的 wp_localize_script 寫法」

### 步驟 5：驗證輸出（12 分鐘）— **這步絕對不能省**

完整三層驗證：
1. **GTM 預覽模式**：開啟 Tag Assistant → 觸發行為 → 確認標籤在「已觸發」清單
2. **GA4 DebugView**：開 DebugView → 等 Tag Assistant 顯示已送出 → DebugView 應該 30 秒內看到事件
3. **GA4 即時報表**：報表 → 即時 → 「依事件名稱」應該 1-2 分鐘內看到事件出現

**任一層失敗就停下排錯，不要 3 層都跑完才回頭**——通常後一層失敗的根因都在前一層。

## 共同尾聲：跑出問題時的 3 個求救步驟

1. **看 lint 和錯誤訊息原文**：不要看「大概意思」，看英文原文搜尋 Stack Overflow 或 GA4 官方說明
2. **GA4 報表的「即時 vs 標準」延遲**：即時 1-2 分鐘可見、標準報表 24-48 小時、轉換報表更慢；不要看不到就以為壞了
3. **試一份「最小可行範例」**：把問題簡化到只剩兩個 HTML 元素 + 一個事件，能不能跑？能 → 是你的程式碼複雜度問題；不能 → 是 gtag/GTM 環境問題

## 最重要的一個原則：AI 是助手不是主管，所有發佈到網站的內容要自己驗證

AI 會自信地給你：
- 看起來合理但完全捏造的 GA4 介面路徑
- 過時的 GTM 變數類型（GA4 推出後改名過很多次）
- 不符合 schema.org 規範的 JSON-LD
- 不適合台灣市場的關鍵字策略（AI 訓練資料偏美國）

**永遠用 Google Rich Results Test、GA4 DebugView、GTM 預覽模式驗證後才發布**。AI 答錯你看不出來，就會帶著錯誤的設定跑半年——這比沒用 AI 更糟。
