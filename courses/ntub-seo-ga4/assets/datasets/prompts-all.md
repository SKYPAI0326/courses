## Part 3 · 關鍵字研究 + AI 搜尋

### P3-1-A 批次標記關鍵字意圖

**使用情境**：Google Ads 規劃工具匯出 30-50 組關鍵字後，用 AI 一次標完三欄。

**完整 prompt**：

```
我從 Google Ads 關鍵字規劃工具匯出了 [N] 組關鍵字（產品類別：[你的產品類別]）。
請幫我為每組加上三個欄位：

1. 意圖類型：資訊型 / 商業型 / 交易型 / 導航型
2. 分類：通用 / 品類 / 品牌
3. 優先級：高 / 中 / 低
   （判斷原則：意圖明確的商業/交易型 + 搜尋量合理 + 競爭低中 = 高）

請回 Markdown 表格，不要解釋，我等下要直接貼回 Google Sheets。

關鍵字資料：
[貼上你的 CSV 內容，格式：關鍵字 | 月搜尋量 | 競爭程度]
```

**後續追問模板**（任選）：

- 把高優先級的 10 組挑出來，每組附一行原因
- [某詞] 為什麼是 [意圖類型]？請重新評估
- 把分類比例算給我（通用 / 品類 / 品牌 各佔幾組）

### P3-3-A 列 5 題 FAQ + 對應答案 + JSON-LD Schema

**使用情境**：要讓內容更容易被 AI Overview / Perplexity 引用，在文章底加 FAQ。

**完整 prompt**（三輪對話）：

**第 1 輪 — 列問題：**

```
我有一篇部落格文章，主題是「[文章主題]」。
我要在文章底部加一個 FAQ 區塊，讓 AI Overview 更容易引用我的內容。

請以「實際在 Google 搜尋這個主題的人」的視角，
列出 5 個他們最可能會問的延伸問題（不要重複文章主標題已回答的問題）。

每題用一行，不要編號。
```

**第 2 輪 — 寫答案：**

```
很好。請為這 5 題各寫一段「直接回答」，每段限制 80 字內。

要求：
- 第一句就給結論，不要鋪陳
- 內含至少一個具體數字或年份
- 避免「可能」「也許」「通常」這類模糊詞
```

**第 3 輪 — 轉 JSON-LD：**

```
把上面 5 組 Q&A 轉成 schema.org 的 FAQPage JSON-LD 格式。

要求：
- 只回純程式碼，不要任何解釋文字
- 用 <script type="application/ld+json"> 包起來
- 我等下會直接貼進 WordPress 文章編輯器的「自訂 HTML」區塊
```

**驗證步驟**：把 JSON-LD 貼進 https://search.google.com/test/rich-results 驗證是否被識別為 FAQPage。

## Part 4 · On-Page SEO 上篇

### P4-3-A Title + Meta Description + H1 三件套生成

**使用情境**：寫好一篇文章但 Title / Meta / H1 沒思路時，請 AI 給 5 個 Title 候選 + 對應 Meta + H1。

**完整 prompt**（兩輪對話）：

**第 1 輪 — 生 Title 候選：**

```
我要為一篇 SEO 部落格文章寫 Title。

主題：[你的文章主題]
目標關鍵字：[主關鍵字]（主）／[次關鍵字]（次）
受眾：[年齡 + 身份 + 階段 + 預算 / 興趣]
品牌名：[你的品牌名]

請給我 5 個 Title 候選：
- 50-60 字元內（中文約 22-28 字）
- 主關鍵字盡量放開頭
- 結尾加品牌名（用「｜」分隔）
- 至少 1 個含具體數字（坪數 / 預算 / 案例數）

請列出後標哪個最適合「[主要搜尋意圖]」的搜尋意圖。
```

**第 2 輪 — 配 Meta + H1：**

```
選定第 [N] 個 Title「[完整 Title]」。

請配對應的：
1. Meta Description（中文 50-65 字、含目標關鍵字、結尾加號召詞）
2. H1（與 Title 相近但用文章語氣，不要重複「[品牌名]」品牌名）

兩項都直接給最終版，不要列候選。
```

**後續追問模板**：

- 第 3 個 Title 太長了，壓回 28 字內
- Meta 沒含「[次關鍵字]」，加上
- H1 跟 Title 太像，改成問句式

## Part 5 · On-Page SEO 下篇

### P5-1-A 批次補圖片 Alt 文字

**使用情境**：上傳多張圖到 WordPress，要一次寫所有 Alt 文字。

**完整 prompt**：

```
我有 [N] 張產品圖要上 WordPress，請幫我批次寫 Alt 文字。

產品定位：[你的產品定位，例如：客製化木製禮品，企業送禮為主]
主要 SEO 目標關鍵字：[3-5 個關鍵字]

每張 Alt 要求：
- 25-35 字內
- 自然帶入關鍵字其中一個（不全部都塞）
- 描述「圖片實際內容 + 使用情境」，不是 SEO 後門
- 避免「[關鍵字][關鍵字]」這種填充

圖片清單：
1. [檔名] — [圖片內容簡述]
2. [檔名] — [圖片內容簡述]
...（最多 10 張）
```

**驗證重點**：

- 同主題圖（5 張禮盒）Alt 不能都長得很像
- 連續出現「客製化禮盒 + 修飾詞」就是關鍵字填充，要追問改寫
- 裝飾性圖（分隔線、背景花紋）AI 偶爾會硬寫描述，改成 `alt=""`

**後續追問模板**：

- 第 2、3、5 張不需要「客製化」這詞，改寫
- 同主題的 5 張圖 Alt 太像，每張用不同切角描述
- 第 7 張是分隔線裝飾，改成空 alt

## Part 7 · GA4 自訂事件

### P7-2-A 自訂事件結構設計

**使用情境**：列出網站要追蹤的 3-5 個行為，請 AI 一次設計事件名 + 參數架構。

**完整 prompt**（兩輪對話）：

**第 1 輪 — 事件結構：**

```
我有一個 [網站類型]，要在 GA4 加自訂事件。
請依 GA4 命名規則（小寫_底線_動詞_名詞、避開保留字 ga_/google_/firebase_）
為下列 [N] 種使用者行為設計事件名與參數結構（每事件最多 25 個參數、每參數值上限 100 字）。

1. [行為 1]
2. [行為 2]
3. [行為 3]
4. [行為 4]
5. [行為 5]

請回 Markdown 表格，欄位：行為 | 事件名 | 參數 1 | 參數 2 | 參數 3。
```

**第 2 輪 — 寫 gtag 程式碼：**

```
選定 [事件名]。請寫對應的 gtag JavaScript 程式碼：

要求：
- 監聽所有 [DOM 選擇器，例如 class 為「.quote-btn」的按鈕]
- 從 [按鈕的屬性 / 隱藏欄位 / sessionStorage] 取值
- page_location 用 window.location.href
- 程式碼開頭加註解說明用途
- 直接給最終版，不要解釋程式碼邏輯
```

**驗證重點**：

- 命名是否全小寫底線（不是駝峰、不是連字號）
- 沒用保留字（ga_、google_、firebase_、event 開頭）
- 多元素場景用 `querySelectorAll().forEach()` 不是 `querySelector()`
- 是否該標為轉換？只有「真正完成業務目標」（表單、購買、電話）才標

## Part 9 · GTM 事件追蹤

### P9-3-A 業務需求翻譯成 GTM 觸發條件 + DataLayer

**使用情境**：你知道要追蹤什麼，但不熟悉 GTM 介面選哪個觸發類型 / 變數類型。

**完整 prompt**（兩輪對話）：

**第 1 輪 — GTM 設定：**

```
我用 GTM 追蹤一個 [網站類型]，想加新追蹤但不想自己對 GTM 介面摸索。

需求：當使用者[做什麼動作]時，記錄這次操作的：[要記錄的維度 1]、[維度 2]、[維度 3]、[維度 4]。

按鈕/元素的 HTML 大概長這樣：
[貼上 HTML 範例]

請幫我設計：
1. GTM 觸發條件（type / 設定 / 條件三項）
2. GTM 變數（要建幾個 DataLayer Variable 或 DOM Element 變數）
3. GA4 事件標籤的事件名 + 事件參數對應
```

**第 2 輪 — 改 DataLayer push 版本（可選）：**

```
剛才的方案依賴 [data-* 屬性]。如果按鈕沒這些屬性，我想改成 DataLayer push 版本。

請給我：
1. 在按鈕的 onclick 事件裡推 DataLayer 的 JavaScript code
2. 對應的 GTM 觸發條件（自訂事件類型）
3. GTM 變數要怎麼取（DataLayer Variable）

DataLayer 事件名用 [事件名]（跟 GA4 事件同名）。
```

**驗證重點**：

- `value` 參數必須配對 `currency` 才會被算入 GA4 電商報表
- 多元素場景用 `forEach` 不是單一 `querySelector`
- 寫死的資料（產品 ID、價格）正式用要改成 `wp_localize_script` 或 PHP 動態帶入
- 表單追蹤優先用「追蹤感謝頁面瀏覽」而非「表單提交」事件（更準）

## 使用建議

### 第一週開始的習慣

1. 挑出最常用 2-3 個 Prompt（建議：P3-1-A 標關鍵字、P4-3-A Title 三件套、P5-1-A 批次補 Alt）
2. 在 [Google Keep](https://keep.google.com/) 或 [ChatGPT 的「自訂指令」](https://chatgpt.com/) 存起來
3. 每次使用後微調模板：發現 AI 常漏的細節，把追問模板補進原 prompt
4. 記錄使用次數，找出 ROI 最高的 prompt 持續優化

### 整個課程結束後

期末發表的「整合提案」段落可以附這份 prompts-all.md 自己改寫的版本，作為「我跑這個專案會用的 AI 工具庫」附錄。實務工作上這個 prompt 庫就是你交付給下一個接手的同事的最大價值。

### 紅線

下面三件事**絕對不要塞進 prompt**：
- 真實客戶 / 員工的姓名、電話、Email
- 公司營業額、客戶名單、員工資料
- 任何「機密」「內部」「不公開」字樣的文件

如果一定要用 AI 處理真實資料，**先去識別化**再貼。
