SECTION 5.3

Technical SEO：Canonical、301、HTTPS、結構化資料

技術 SEO 是「讓 Google 能正確理解你的網站」的基礎建設。這四項不做好，排名的天花板會比內容差勁的網站還低。

為什麼這很重要 · WHY THIS MATTERS
On-Page 做得再好，如果 Canonical 設錯導致 Google 把兩個相同頁面當作重複內容、301 沒設好讓換 URL 後排名歸零、沒有 HTTPS 讓 Chrome 顯示「不安全」——這些都是技術地雷。本節逐一排雷。
CONCEPT & PRACTICE · 概念與實作
1
Canonical Tag：告訴 Google 哪個 URL 是正本
當同樣的內容出現在多個 URL（例如 /product/ 和 /product/?ref=email），Google 不知道哪個是「正版」，會分散 PageRank 給兩個 URL。Canonical Tag 在 <head> 加入 <link rel="canonical" href="正本URL">，告訴 Google「以這個為準」。Yoast SEO 預設會自動為每頁生成正確的 Canonical，通常不需要手動設定。
提示：電商的篩選 URL（顏色、尺寸、排序參數）最容易產生重複內容問題。全部指向無參數的主頁 Canonical 是標準作法。
2
301 轉址：換 URL 不損失排名
301 是 HTTP 狀態碼，代表「永久轉址」，告訴 Google 把舊頁面的 PageRank 轉移到新 URL。在 WordPress 安裝 Redirection 外掛，可視覺化管理所有 301 規則：
舊 URL：/old-product-page/ 新 URL：/client-gift-taiwan/ 類型：301（永久）
302 是「暫時轉址」，PageRank 不轉移。除非真的是暫時（例如促銷頁），否則永遠用 301。
提示：改網域（例如從 HTTP 移到 HTTPS）時，也必須設置 301 確保所有舊 HTTP 連結跳到 HTTPS 版本。
3
HTTPS：信任訊號與排名因素
Google 在 2014 年宣布 HTTPS 是排名訊號。Chrome 對 HTTP 網站顯示「不安全」，直接影響訪客信任度和轉換率。LocalWP 預設就是 HTTPS（https://ntub-demo.local）。真實主機通常透過 Let's Encrypt 免費安裝 SSL——大多數主機後台有一鍵安裝功能。安裝後記得設定 HTTP → HTTPS 的 301 轉址。
提示：安裝 SSL 後，在 WordPress 後台「設定 → 一般」把網址從 http:// 改為 https://，否則前台會有混合內容（Mixed Content）問題。
4
結構化資料（Schema Markup）：讓 Google 更懂你
結構化資料是加在頁面 HTML 的特殊標記（JSON-LD 格式），明確告訴 Google「這是文章」「這是商品」「這是常見問題」。正確的 Schema 讓你的搜尋結果出現額外資訊（豐富摘要 Rich Snippet）——例如文章旁邊的發布日期、商品的評價星數。Yoast SEO 已自動生成基本的 Article Schema，用 schema.org/Article 的完整格式可以進一步強化。
{ "@context": "https://schema.org", "@type": "Article", "headline": "客製化禮品推薦 2026", "author": {"@type": "Person", "name": "小明"}, "datePublished": "2026-01-15" }
MORE SCHEMA TYPES · 其他常用結構化資料類型
FAQPage — 讓 FAQ 出現在搜尋結果展開
{ "@type": "FAQPage", "mainEntity": [{ "@type": "Question", "name": "客製化禮品最快幾天出貨？", "acceptedAnswer": {"@type": "Answer", "text": "標準訂單 3 個工作天，加急服務 1–2 天。"} }] }
適用場景：服務頁底部的 FAQ 區塊、部落格文章的問答段落。Google 可能在搜尋結果展開問答，提升佔版面積。
LocalBusiness — 本地商家資訊
{ "@type": "LocalBusiness", "name": "XX 設計工作室", "address": {"@type": "PostalAddress", "addressLocality": "台北市", "addressRegion": "大安區"}, "telephone": "+886-2-xxxx-xxxx", "openingHours": "Mo-Fr 10:00-18:00" }
適用場景：實體店面、服務型公司、診所、餐廳。配合 Google Business Profile，提升本地搜尋排名。
提示：用 Google 的「豐富結果測試工具」（search.google.com/test/rich-results）驗證 Schema 是否正確，能預覽在搜尋結果的顯示效果。
5
Mobile-First 索引：行動版決定排名
Google 自 2019 年起全面採用 Mobile-First 索引——爬蟲主要用手機版來評估頁面品質和排名。確認你的 WordPress 主題有響應式設計（RWD），在手機上顯示正常。GSC「體驗 → 行動裝置使用體驗」會列出手機版的問題（文字太小、連結太近等）。Twenty Twenty-Four 主題預設是響應式設計。
提示：不要用單獨的 m.yoursite.com 手機站——維護成本高，且 Canonical 設置複雜。用 RWD 一個 URL 服務所有裝置是最佳實務。
完成驗證 · VERIFY
能說出 Canonical、301、HTTPS 各自的用途和差異，用「豐富結果測試工具」驗證 Yoast 自動生成的 Schema 沒有錯誤。
常見問題 · TROUBLESHOOT
設了 301 後舊頁面在 GSC 還是顯示「已排除」？
301 轉址後，Google 重新爬取舊 URL 並跟著轉到新 URL，通常需要 1–4 週才會在 GSC 看到數據更新。若超過一個月沒變化，用 GSC 的「URL 檢查」確認舊 URL 確實回傳 301 狀態碼。
Canonical 要指向自己嗎？
是，每頁都應該有一個 Self-Referential Canonical（指向自己的 Canonical）。Yoast 預設已經這樣做。這樣做的好處：若別人轉貼你的文章時不小心沒加 Canonical，Google 仍能判斷原始出處是你。
結構化資料測試顯示「警告」但沒有「錯誤」，需要修嗎？
警告通常是「建議但非必要的欄位未填」，例如 Article Schema 沒有 author image。有錯誤才影響豐富摘要出現。警告可以忽略或有時間再優化，不是緊急事項。
