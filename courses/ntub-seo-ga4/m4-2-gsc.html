SECTION 4.2

Google Search Console 註冊與索引檢查

GSC 是 Google 官方提供的免費 SEO 儀表板，直接告訴你哪些頁面被收錄、哪些關鍵字帶來流量、哪些頁面有問題——是 SEO 工作的核心工具。

為什麼這很重要 · WHY THIS MATTERS
所有 SEO 優化做完之後，最終驗證場所是 GSC：它告訴你 Google 真正看到了什麼。本節學習設定 GSC、送出 Sitemap、用「URL 檢查工具」確認頁面狀態。本地站 Google 爬不到，所以我們以「真實網站」為範例示範，LocalWP 用 Yoast 分數替代驗證。
STEP BY STEP · 操作步驟
1
進入 GSC 並新增資源
前往 search.google.com/search-console，用 Google 帳號登入。點「新增資源」，選擇資源類型：網域（涵蓋所有子網域和 HTTP/HTTPS，推薦）或 URL 前置字元（精確到某個 URL 前綴）。輸入你的網域後，GSC 會提供幾種驗證方式。
提示：本地站 ntub-demo.local 無法添加到 GSC（Google 爬不到）。如果有真實的 Blogger 或 GitHub Pages 練習站，可以在那上面驗證。
2
驗證網站所有權
最常用的驗證方法：HTML 標籤——GSC 給一個 <meta name="google-site-verification" ...> 標籤，貼到首頁 <head> 中，點驗證即可。在 WordPress + Yoast 環境下，進入 Yoast SEO → 一般 → 網站認證，直接填入驗證碼即可，不需要手動改 HTML。
提示：驗證後不能移除這個 meta 標籤，否則 GSC 所有權失效、資料停止更新。
3
提交 Sitemap
驗證成功後，左側選單 → 索引 → Sitemap → 輸入 Sitemap URL（通常是 yourdomain.com/sitemap.xml）→ 提交。幾天後回來查看，可以看到「發現幾個頁面、成功索引幾個頁面」。若發現數大於索引數，要進一步查原因。
提示：Yoast SEO 的 Sitemap URL 固定是 /sitemap_index.xml（注意有 index 後綴）。
4
認識 GSC 四大報表區
GSC 左側選單的核心四區：
成效
哪些搜尋詞帶來流量、排名、CTR
索引
哪些頁面被收錄、哪些被排除及原因
體驗
Core Web Vitals 分數、行動友善
連結
哪些外部網站連到你、哪些內部連結最多
提示：「成效」報表是找「低 CTR、高曝光」關鍵字的最佳工具——這些詞表示 Google 認為你的頁面相關，但使用者沒有點擊，通常是 Title 或 Meta Description 需要優化。
5
URL 檢查工具：單頁索引診斷
左側點「URL 檢查」，輸入任何一個頁面 URL，GSC 會即時顯示：這頁有沒有被索引、最後一次爬取時間、行動裝置相容性、結構化資料是否有效。若頁面沒有被索引，可以點「要求索引」讓 Google 優先爬取。
提示：「要求索引」不保證立即收錄，只是告訴 Google「這頁更新了，請盡快來看」。通常 1–3 天內會重新爬取。
完成驗證 · VERIFY
若有真實網站：GSC 已新增資源、所有權驗證成功、Sitemap 已提交。若只有 LocalWP：能說出 GSC 四個報表區的名稱和各自的用途。期末報告的 SEO 診斷章節要引用 GSC 成效報表數據。
常見問題 · TROUBLESHOOT
提交 Sitemap 後顯示「無法擷取」？
先在瀏覽器直接打開 Sitemap URL 確認能讀取。若 Yoast 沒有啟用 XML Sitemap 功能（Yoast → 一般 → 功能 → XML Sitemap），啟用後重試。
為什麼 GSC 顯示的流量跟 GA4 不一樣？
正常現象。GSC 計算「搜尋曝光與點擊」，GA4 計算「進站後的工作階段」。同一個訪客可能點了多次（GSC 算多次點擊），但只有一個工作階段（GA4 算一個）。此外 GA4 有 JavaScript 過濾爬蟲，GSC 沒有。
GSC 說「已排除 — 由 noindex 標記」但我沒有設 noindex？
在 WordPress 後台 → 設定 → 閱讀，確認「阻擋搜尋引擎」沒有勾選。也可能是 Yoast SEO 在頁面層級設了 noindex，到該頁面的 Yoast 區塊查看「進階」分頁。
