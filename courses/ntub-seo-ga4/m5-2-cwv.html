SECTION 5.2

Core Web Vitals 與 PageSpeed Insights

網站速度是 Google 的直接排名因素。LCP、INP、CLS 三個指標量化了「使用者感受到的速度體驗」——用 PageSpeed Insights 一次看清問題和解法。

為什麼這很重要 · WHY THIS MATTERS
Google 在 2021 年正式將 Core Web Vitals 納入排名訊號（Page Experience Update）。這不只是技術問題——頁面載入太慢，使用者離開，轉換率下降，排名也跟著掉。本節教你讀懂分數、找到最高優先級的問題，以及在 WordPress 環境下的快速改善方法。
CONCEPT BY CONCEPT · 指標與工具
1
三大 Core Web Vitals 指標
Google 用三個指標衡量「頁面體驗品質」：
LCP
最大內容繪製
頁面最大元素（通常是主圖）出現的時間。良好：< 2.5s
INP
互動到繪製延遲
點擊/輸入後，畫面反應時間。良好：< 200ms（2024年取代 FID）
CLS
累積版面配置偏移
頁面載入時元素是否亂跳。良好：< 0.1
提示：三個指標中，LCP 最常見問題也最容易改善；CLS 問題通常來自圖片沒有設定寬高，或廣告插入時擠掉其他內容。
INP vs FID · 為什麼 2024 年換掉了舊指標
FID（已淘汰，2024/3 前）
只量「第一次互動」的延遲。問題：用戶第一次點擊反應快，之後操作越來越慢，FID 也給好分，無法反映真實體驗。
INP（現行指標）
量整個工作階段中「所有互動」的最慢那一次。能抓出頁面在使用過程中的任何卡頓點，標準更嚴格也更真實。
WordPress 環境 INP 常見壞因子：
• 過多第三方腳本：Google Ads、Facebook Pixel、聊天機器人、熱圖工具——每個都在爭奪主執行緒
• Gutenberg 區塊編輯器的 JS：Gutenberg 在前端也會載入部分 JS，某些區塊的互動延遲明顯
• 大型外掛的 jQuery 依賴：舊外掛仍依賴 jQuery，主執行緒被同步 JS 阻塞
優化方向：延遲載入非必要第三方 JS（在標籤屬性加 defer 或 async）；審查並停用非必要外掛；使用 GTM 的「同意前」模式延遲廣告腳本載入。
2
用 PageSpeed Insights 測試你的網站
前往 pagespeed.web.dev，輸入任何真實網址（本地站無法測試）。PSI 提供兩種資料：「實地資料」（真實使用者的 Chrome UX 數據，更重要）和「實驗室資料」（模擬測試）。若沒有實地資料（流量太低的站），只會顯示實驗室數據。
提示：測試時選「行動裝置」版本——Google 用 Mobile-First 索引，行動版成效決定 SEO 排名。通常行動版分數比桌機低 20–30 分。
3
讀懂 PSI 建議清單，找最高 CP 值改法
PSI 的「建議」區塊按「潛在節省時間」排序。最常見的高影響建議：(1)「提供適當大小的圖片」——未壓縮的大圖是最常見的 LCP 殺手，用 webp 格式可節省 25–35%；(2)「消除阻斷轉譯的資源」——CSS/JS 在 <head> 阻塞首屏渲染；(3)「啟用文字壓縮」——Gzip/Brotli 壓縮，通常主機後台可以一鍵開啟。
提示：WordPress 安裝 WP Rocket 或免費的 W3 Total Cache + ShortPixel，可以一次解決大部分 PSI 建議。
4
LocalWP 的速度測試替代方案
本地站無法用 PSI 測試（爬不到），但可以用 Chrome 開發者工具的 Lighthouse：右鍵 → 檢查 → Lighthouse 分頁 → 選「行動裝置」→「產生報告」。結果和 PSI 類似但基於本地環境，分數通常偏高（沒有真實網路延遲）。主要用來學習「如何看懂報告」而非追求分數。
提示：Lighthouse 在隱私視窗（無痕模式）下執行，避免瀏覽器快取影響結果。
5
GSC「體驗」報表：追蹤真實使用者數據
PSI 是「診斷工具」，GSC 的「體驗 → Core Web Vitals」是「趨勢追蹤工具」——顯示過去 90 天真實使用者的 CWV 達標率。若 GSC 顯示「需要改善」或「差」的頁面，這些頁面在排名上可能已被扣分。搭配 PSI 找原因，GSC 看趨勢，兩者互補使用。
提示：CWV 的排名影響力相對有限（10%以下），不要為了衝 PSI 分數而破壞功能或設計。內容和連結的影響力遠大於速度。
完成驗證 · VERIFY
能說出 LCP、INP、CLS 各自的意思和良好標準，在 Chrome Lighthouse 跑完本地站的報告，並指出至少一個「高潛在節省」的建議是什麼。
常見問題 · TROUBLESHOOT
PSI 分數 40 分，是 SEO 問題嗎？
未必立即影響排名，但確實是使用者體驗問題。先看「實地資料」是否有紅色/橘色指標，若有才是優先修復對象。純「實驗室資料」低分對排名影響有限。
安裝快取外掛後分數反而更低？
快取外掛配置不當（如生成多餘 CSS）可能拖慢速度。可先停用外掛測試，確認是哪個外掛造成，然後調整設定或換外掛。
CLS 高是什麼意思，怎麼修？
頁面元素在載入時位移。常見原因：圖片沒有 width/height 屬性（加上後瀏覽器預留空間）；廣告/嵌入程式碼沒有固定高度；字體延遲載入導致文字大小改變。PSI 的「診斷」區塊會指出具體哪些元素是問題。
