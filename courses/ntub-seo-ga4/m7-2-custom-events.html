MODULE 7 · SECTION 7.2

自訂事件設計與建立

當 GA4 內建事件無法滿足業務需求時，透過自訂事件追蹤任何使用者行為，並將其轉化為轉換指標。

為什麼這很重要 · WHY THIS MATTERS
GA4 的自動收集事件只能涵蓋通用行為（頁面瀏覽、點擊、捲動），但每個網站的核心價值都不同。電商需要追蹤「加入購物車」，部落格需要追蹤「文章閱讀完成率」，服務型網站需要追蹤「表單提交」。自訂事件讓你將業務目標直接翻譯成 GA4 可量化的數據，是 GA4 分析深度的核心所在。
STEP BY STEP · 操作步驟
1
判斷是否需要自訂事件
進入 GA4 → 管理員 → 事件，先確認「現有事件」清單。GA4 預設已收集 page_view、scroll、click、form_submit 等事件。只有在這些事件無法區分你真正想追蹤的行為時，才需要自訂事件。例如：你需要區分「點擊電話連結」與「點擊一般連結」，預設的 click 事件無法做到，就需要自訂。
原則：先窮舉內建事件，再決定是否自訂，避免產生重複或無意義的事件。
2
設計事件命名與參數架構
GA4 事件名稱必須遵守命名規則，否則事件無法正確建立或報表顯示異常。建議使用「動詞_名詞」格式，全部小寫，以底線分隔單字。每個事件最多可附帶 25 個參數，每個參數值上限 100 字元。
正確命名範例
form_submit_contact
video_play_hero
download_pdf_brochure
click_phone_link
scroll_depth_75
錯誤命名（避免）
FormSubmit（大寫）
form-submit（連字號）
ga_session（ga_ 保留）
firebase_xxx（保留前綴）
event name（空格）
注意：以 ga_、google_、firebase_ 開頭的名稱是保留字，GA4 不接受。
3
在 GA4 介面中建立自訂事件（以現有事件為基礎）
GA4 提供一個「從現有事件建立新事件」的功能，不需修改追蹤碼。適合用於：將某一類 click 事件（特定連結 class）單獨命名為新事件。路徑：管理員 → 資源欄 → 事件 → 建立事件 → 建立。
// 建立條件範例： // 將點擊電話連結的行為獨立為 click_phone_link 條件 1：event_name 等於 click 條件 2：link_url 包含 tel: 新事件名稱：click_phone_link 複製條件中的參數：✅ 勾選
提示：「複製條件中的參數」務必勾選，這樣原始 click 的 link_url 等參數才會被帶入新事件。
4
透過 gtag.js 直接傳送自訂事件
若需要追蹤更複雜的行為（如：影片觀看完成、特定 div 可見、表單欄位填寫），需在網站程式碼中加入 gtag 呼叫。LocalWP 環境可直接在佈景主題的 footer.php 或透過 WordPress 外掛「Head & Footer Code」注入。
// 範例 1：追蹤電話連結點擊 document.querySelectorAll('a[href^="tel:"]').forEach(el => { el.addEventListener('click', () => { gtag('event', 'click_phone_link', { 'phone_number': el.href.replace('tel:', '') }); }); }); // 範例 2：追蹤表單送出 document.getElementById('contact-form').addEventListener('submit', () => { gtag('event', 'form_submit_contact', { 'form_location': 'homepage' }); });
注意：gtag 函式必須在 GA4 追蹤碼載入後才能呼叫，確保你的自訂腳本放在 gtag.js 之後。
5
將自訂事件標記為轉換
不是每個自訂事件都值得成為轉換。轉換代表「業務核心目標達成」，通常是：表單提交、電話點擊、購買完成、下載完成。在 GA4 中標記轉換：管理員 → 資源欄 → 轉換 → 新增轉換事件，輸入你的自訂事件名稱（如 form_submit_contact），儲存後該事件就會出現在轉換報表中。
限制：GA4 每個資源最多可標記 30 個轉換事件（含預設的 purchase）。
6
在即時報表確認事件送出
完成設定後，開啟另一個瀏覽器視窗（或無痕模式）造訪你的 LocalWP 網站，實際觸發你設定的行為（點擊電話連結、送出表單）。切換回 GA4 → 報表 → 即時，觀察「事件計數（依事件名稱）」區塊，應該能在 1–2 分鐘內看到自訂事件名稱出現。
提示：GA4 的自訂事件最多需要 24–48 小時才會出現在「事件」報表清單，但即時報表是立即的。
完成驗證 · VERIFY
在 GA4 即時報表中確認自訂事件（如 click_phone_link 或 form_submit_contact）已出現在事件清單。若設定了轉換，在「轉換」頁籤中也應能看到對應事件名稱。DebugView 會在下一節進行更完整的驗證。
常見問題 · TROUBLESHOOT
即時報表看不到自訂事件
確認 gtag.js 追蹤碼已正確載入（打開 DevTools → Network，搜尋 collect 或 analytics.js），且自訂事件腳本放在 gtag 載入之後。LocalWP 網站需確認外掛「Head & Footer Code」已啟用並儲存。
GA4 介面建立的新事件為何沒有觸發？
以現有事件為基礎的「建立事件」功能需要 1–2 天才會生效，且前提是原始的 click 事件確實有觸發。先在即時報表確認 click 事件有出現，再回頭設定條件。
轉換事件的資料從何時開始計算？
GA4 的轉換標記只對「標記後」的新資料生效，無法回溯過去的事件。因此建議儘早設定轉換，讓資料從課程學習期間就開始累積。
