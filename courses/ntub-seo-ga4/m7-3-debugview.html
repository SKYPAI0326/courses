MODULE 7 · SECTION 7.3

DebugView 驗證與資源設定

透過 GA4 DebugView 即時確認事件是否正確送出，並完成資源層級的基礎設定，讓 GA4 資料從一開始就乾淨準確。

為什麼這很重要 · WHY THIS MATTERS
即時報表只能看到「事件發生了」，但 DebugView 能讓你看到「每個事件帶了哪些參數、參數值是否正確、事件是否被標記為轉換」。在正式上線前用 DebugView 驗證，可以避免幾週後才發現資料從一開始就有誤的慘況。
STEP BY STEP · 操作步驟
1
安裝 Google Analytics Debugger 擴充功能
在 Chrome 瀏覽器中安裝「Google Analytics Debugger」擴充功能（Chrome 線上應用程式商店搜尋）。安裝後，點擊擴充功能圖示啟用（圖示變為彩色 = 已啟用）。這個工具會在開發者工具的 Console 輸出所有 GA4 事件的詳細資訊，並觸發 GA4 的 DebugView 模式。
替代方案：在 gtag.js 中加入 debug_mode: true 參數也可以啟用 DebugView，但只限該設備。
2
開啟 GA4 DebugView 介面
確認 Chrome Debugger 已啟用後，在同一瀏覽器造訪你的 LocalWP 網站並執行一些操作（點擊、捲動、填寫表單）。切換到 GA4 → 管理員 → 資源欄 → DebugView。畫面應該會出現你的設備（顯示 Debug 裝置名稱），左側時間軸顯示每個事件的觸發時間，右側顯示事件詳情。
提示：DebugView 的事件延遲約 2–10 秒，看不到時稍等一下再觸發新行為。
3
驗證事件參數與轉換標記
在 DebugView 時間軸上點擊任一事件名稱，右側面板會展開該事件的所有參數。重點確認項目：事件名稱是否符合預期、自訂參數（如 phone_number、form_location）值是否正確、被標記為轉換的事件旁邊是否有旗幟圖示。
// DebugView 應看到的事件範例 ✅ page_view page_location: "http://ntub-demo.local/" page_title: "首頁 - NTUB Demo" ✅ click_phone_link 🚩（轉換） phone_number: "0912345678" page_location: "http://ntub-demo.local/contact/" ✅ form_submit_contact 🚩（轉換） form_location: "homepage"
注意：若事件名稱拼錯（如多了空格），DebugView 仍會顯示，但事件在報表中會歸類到錯誤名稱下，難以查找。
4
設定資源基礎參數（時區與貨幣）
在 GA4 管理員 → 資源欄 → 資源設定中，確認以下設定符合台灣市場需求。這些設定影響報表中的日期顯示與電商交易金額計算。
時區
（台灣）GMT+8
貨幣
新台幣（TWD）
產業類別
依實際業務選擇
事件資料保留
建議選 14 個月
重要：資料保留預設為 2 個月，若要在「探索」報表使用超過 2 個月的原始資料，必須改為 14 個月。
5
啟用加強型評估功能
GA4 的加強型評估可以自動追蹤捲動深度、外部連結點擊、站內搜尋、影片觀看、檔案下載等行為，無需額外程式碼。路徑：管理員 → 資源欄 → 資料串流 → 點擊你的串流 → 加強型評估（滑動啟用）。展開設定後可個別開關每種追蹤類型。
提示：「站內搜尋」功能需要搜尋結果頁的 URL 包含 q= 或 s= 等查詢參數，WordPress 預設使用 ?s=，GA4 已能自動辨識。
6
連結 Google Search Console（選用）
將 GA4 與 Search Console 連結後，可在 GA4「搜尋主控台」報表中直接看到自然搜尋關鍵字的點擊與曝光資料。連結路徑：GA4 管理員 → 資源欄 → Search Console 連結 → 新增連結，選擇已驗證的 Search Console 資源。需要你對兩個帳戶都有管理員權限。
注意：LocalWP 網站通常無法驗證 Search Console（因為沒有公開 URL），這個步驟留待部署到正式環境後再執行。
完成驗證 · VERIFY
DebugView 中能看到你的自訂事件，且參數值正確。GA4 資源設定中時區已設為 GMT+8、資料保留已設為 14 個月、加強型評估已啟用。Module 7 的 GA4 基礎設定至此全部完成。
常見問題 · TROUBLESHOOT
DebugView 顯示「目前沒有偵錯資料」
確認 Chrome Debugger 擴充功能已啟用（圖示為彩色），且你在同一瀏覽器中造訪了 LocalWP 網站。若已確認，嘗試重新整理 LocalWP 網站頁面，並等待 10 秒後再查看 DebugView。
DebugView 中看不到自訂事件，只有 page_view
自訂的 gtag 呼叫可能沒有執行到。開啟 Chrome DevTools → Console，確認沒有 JavaScript 錯誤。最常見的原因是事件監聽器在 DOM 載入前就執行了，解決方法是將程式碼包在 DOMContentLoaded 事件內。
為什麼 DebugView 的事件沒有旗幟（轉換標記）？
GA4 的轉換設定需要最多 24 小時才會在 DebugView 中生效。若你剛剛才設定轉換，當下的 DebugView 可能還不會顯示旗幟，隔天再確認即可。
