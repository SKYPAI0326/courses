MODULE 9 · SECTION 9.2

GTM 容器建立與 LocalWP 佈建

從零開始建立 GTM 帳戶與容器，將追蹤碼正確安裝到 LocalWP WordPress 網站，並透過 GTM 部署 GA4 追蹤。

為什麼這很重要 · WHY THIS MATTERS
GTM 的價值在實際安裝後才能體會：所有追蹤標籤集中管理、改動不需接觸網站程式碼、可以在發布前預覽測試。正確安裝 GTM 容器是後續所有標籤管理的基礎，這個步驟做對了，Module 9.3 的事件追蹤才能順利進行。
STEP BY STEP · 操作步驟
1
建立 GTM 帳戶與容器
前往 tagmanager.google.com，使用 Google 帳戶登入。點擊「建立帳戶」，填入：帳戶名稱（你的組織名稱，如「Demo」）、國家/地區（台灣）。接著建立容器：容器名稱（輸入網站 URL 或名稱，如「ntub-demo.local」）、目標平台選「網頁」。同意條款後，GTM 會顯示兩段追蹤碼。
帳戶 vs 容器：一個 GTM 帳戶可以管理多個容器（如：主網站、行動版 App、AMP 頁面各一個容器）。通常一個網站對應一個容器。
2
複製 GTM 容器代碼（兩段）
GTM 提供的兩段代碼各有用途：第一段放在 <head> 標籤內，儘量靠前，確保標籤管理器在頁面載入初期就初始化；第二段放在 <body> 開頭後，是給瀏覽器不支援 JavaScript 時的 noscript 備援。兩段都需要安裝才算完整。
/* 第一段：放在 <head> 內 */ <!-- Google Tag Manager --> <script>(function(w,d,s,l,i){w[l]=w[l]||[];w[l].push({'gtm.start': new Date().getTime(),event:'gtm.js'});var f=d.getElementsByTagName(s)[0], j=d.createElement(s),dl=l!='dataLayer'?'&l='+l:'';j.async=true;j.src= 'https://www.googletagmanager.com/gtm.js?id='+i+dl;f.parentNode.insertBefore(j,f); })(window,document,'script','dataLayer','GTM-XXXXXXX');</script> <!-- End Google Tag Manager --> /* 第二段：放在 <body> 開頭後 */ <!-- Google Tag Manager (noscript) --> <noscript><iframe src="https://www.googletagmanager.com/ns.html?id=GTM-XXXXXXX" height="0" width="0" style="display:none;visibility:hidden"></iframe></noscript> <!-- End Google Tag Manager (noscript) -->
3
在 LocalWP WordPress 安裝 GTM
方法 A（外掛）：在 WordPress 管理後台搜尋安裝外掛「GTM4WP」（Google Tag Manager for WordPress）。啟用後，到外掛設定頁面，輸入你的容器 ID（格式：GTM-XXXXXXX），儲存。外掛會自動將兩段代碼插入正確位置。方法 B（手動）：在 WordPress 佈景主題編輯器中，直接編輯 header.php 和 header.php，分別貼入兩段代碼。推薦使用方法 A，更安全且在佈景主題更新後不會失效。
安全提醒：手動編輯 header.php 時，若語法錯誤可能導致白版（WSOD）。建議先備份佈景主題，或使用子佈景主題進行修改。
4
在 GTM 中建立 GA4 設定標籤
若你原本用 gtag.js 直接追蹤 GA4，現在改為透過 GTM 管理。在 GTM 容器中：標籤 → 新建 → 選擇「Google Analytics: GA4 設定」，填入你的 GA4 測量 ID（格式：G-XXXXXXXXXX）。觸發條件選「All Pages（所有頁面）」，儲存。記得同時移除網站中直接嵌入的 gtag.js 代碼，避免重複計算。
重要：GTM 的改動必須先「預覽」測試，確認無誤後再「提交發布」，否則變更不會對訪客生效。
CONSENT MODE V2 · 同意模式設定（不可忽略）
Google Consent Mode v2 是 2024 年 Google 推出的隱私合規機制。即使是台灣站，若你未來要跑 Google Ads 再行銷或使用 GA4 的轉換建模功能，都需要正確設定，否則 GA4 的轉換數據會比實際少計 10–40%。
在 GTM 中設定同意模式的最快方法：
①在 GTM 容器中新增「同意總覽」設定：標籤 → 新建 → 同意初始化 → 選「Google 代碼」，設定預設值：analytics_storage: denied、ad_storage: denied
②安裝 Cookie 同意管理外掛（如 Complianz 或 CookieYes），外掛會在使用者接受 Cookie 時自動更新同意狀態
③在 GA4 資源設定中確認「Google 信號資料收集」已啟用，讓 Consent Mode 的建模功能生效
LocalWP 練習環境不需要嚴格執行，但在正式上線的網站中，這個設定影響廣告轉換數據的完整性，屬於上線前必做清單的一部分。
5
使用 GTM 預覽模式驗證安裝
在 GTM 容器頁面右上角點擊「預覽」，輸入你的 LocalWP 網站 URL（如 http://ntub-demo.local）。GTM 會開啟一個除錯視窗（Google Tag Assistant），在底部面板顯示已觸發的標籤、觸發條件狀態、DataLayer 事件流。確認「GA4 設定」標籤出現在「已觸發的標籤」清單中，代表安裝成功。
預覽模式特點：預覽模式只在你的瀏覽器生效，其他訪客看不到你的測試標籤，是安全的測試環境。
6
發布容器版本
預覽確認無誤後，點擊右上角「提交」按鈕。填入版本名稱（如「v1.0 初始安裝 GA4」）和版本說明，點擊「發布」。GTM 會建立一個版本記錄，讓你日後可以回滾到任何歷史版本。發布後，用 GA4 即時報表確認是否有資料進入（造訪 LocalWP 網站後，GA4 應顯示即時活躍使用者）。
版本命名習慣：每次發布都加上日期和變更摘要（如「2026-04-15 新增電話點擊追蹤」），方便日後查閱歷史。
完成驗證 · VERIFY
GTM 預覽模式中能看到 GA4 設定標籤在所有頁面觸發，且 GA4 即時報表在造訪 LocalWP 網站後顯示活躍使用者。GTM 容器已成功發布第一個版本。
常見問題 · TROUBLESHOOT
GTM 預覽模式無法連線到 LocalWP 網站
確認 LocalWP 網站正在運行（LocalWP 軟體中網站狀態為「Running」），且網站 URL 正確（通常是 http://網站名稱.local）。若使用 HTTPS，需確認 LocalWP 的 SSL 憑證已設定並被瀏覽器信任（LocalWP 選單 → Trust 按鈕）。
GTM 安裝後 GA4 開始收到重複的 page_view 事件
這代表網站同時存在 gtag.js 直接追蹤和 GTM 的 GA4 標籤，兩者都在發送 page_view。解決方法：移除網站中直接嵌入的 gtag.js 代碼（或移除 GTM4WP 以外的其他 GA4 外掛），只保留 GTM 管理的 GA4 標籤。
GTM4WP 外掛安裝後，WordPress 後台顯示白版
通常是外掛與其他外掛或佈景主題衝突。嘗試停用其他外掛逐一測試。若問題持續，改用方法 B（手動安裝），透過 phpMyAdmin 或 FTP 直接修改 header.php 檔案。
