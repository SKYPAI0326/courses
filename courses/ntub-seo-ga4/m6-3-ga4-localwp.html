SECTION 6.3

LocalWP 埋碼與即時驗證

把 GA4 追蹤碼裝進 LocalWP 網站，用「即時」報表確認資料有送進來——從這一刻起，你的每一個頁面瀏覽都會被記錄。

為什麼這很重要 · WHY THIS MATTERS
上一節拿到了 GA4 評估 ID（G-XXXXXXXXXX），這節要把它「裝進網站」。安裝方式有兩種：直接在 WP 後台用 Yoast 或外掛插入（本節做法），或透過 GTM 管理（第 9 週做法）。先學最快的直接安裝，之後學完 GTM 再理解為何多數專業環境偏好 GTM。
STEP BY STEP · 操作步驟
1
安裝 Google Site Kit 外掛
WP 後台 → 外掛 → 安裝外掛 → 搜尋「Site Kit by Google」→ 安裝並啟用。這是 Google 官方外掛，可以一鍵連結 GA4、GSC、PageSpeed Insights 到 WordPress，不需要手動修改 PHP 主題檔案。
提示：Site Kit 是快速上手的好選擇。若之後要透過 GTM 管理 GA4，記得移除 Site Kit 避免重複計算。
2
透過 Site Kit 連結 GA4
啟用後，左側選單出現「Site Kit」→ 點「開始設定」→ 用你的 Google 帳號授權。設定流程：連結 Search Console（若有）→ 連結 Analytics → 選擇你剛建立的「Demo Site」資源 → 確認。Site Kit 會自動把 GA4 追蹤碼插入所有頁面的 <head>。
提示：Site Kit 連結本地站（.local 網址）時，Google 授權頁面可能顯示警告，這是因為不是公開網址，按「繼續」即可完成授權。
3
手動方式：直接插入追蹤碼片段
若不用 Site Kit，也可以手動插入。取得 GA4 提供的 gtag.js 代碼片段，放入 WP 主題的 functions.php 或使用 「Insert Headers and Footers」外掛：
<!-- Google tag (gtag.js) --> <script async src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXXXXXX"></script> <script> window.dataLayer = window.dataLayer || []; function gtag(){dataLayer.push(arguments);} gtag('js', new Date()); gtag('config', 'G-XXXXXXXXXX'); </script>
提示：把 G-XXXXXXXXXX 替換成你的實際評估 ID。不要把這段代碼貼進文章內容區塊——要貼在 <head> 區段。
4
用「即時」報表驗證資料進來了
安裝完成後：(1) 打開 LocalWP 瀏覽器，進入 ntub-demo.local 隨意點幾頁；(2) 另一個瀏覽器打開 GA4 你自己的帳戶 → 報表 → 即時。若看到「1 個使用者正在使用中」的地圖顯示台灣，代表追蹤碼運作正常。即時報表最多延遲 1–2 分鐘，不是真正的「零延遲」。
提示：建議用「無痕視窗」瀏覽 LocalWP 站做測試，避免自己的 Cookie 造成工作階段混淆。
5
排除「自己的訪問」污染資料
本地站的訪客只有你自己，若要讓報表乾淨不計算開發/測試流量，可在 GA4 管理 → 資源 → 資料串流 → 你的串流 → 「更多標記設定」→「定義內部流量」，把你的 IP 或 LocalWP 的 localhost 設定為內部流量並在篩選器中排除。
提示：練習帳戶不需要特別排除（流量就是你），但真實客戶網站一定要排除公司 IP，否則員工的瀏覽行為會污染報表。
完成驗證 · VERIFY
GA4「即時」報表看到來自 ntub-demo.local 的訪問（即使只有 1 個使用者，就是你自己）。這代表追蹤碼正常運作。截圖即時報表，作為 GA4 事件規劃作業的第一張驗證截圖。
常見問題 · TROUBLESHOOT
即時報表一直顯示 0 個使用者？
三個排查步驟：(1) 確認 LocalWP 站台已啟動（綠燈）；(2) 用 Chrome 開發者工具的「Network」分頁，搜尋 google-analytics 或 gtag，確認有請求發出；(3) 確認安裝的是正確的評估 ID（G- 而非 UA-）。
Site Kit 授權失敗怎麼辦？
本地站授權有時不穩定，改用「手動插入代碼」方式：安裝「Insert Headers and Footers」外掛，把 gtag.js 代碼貼入 Header 區域，儲存後重新整理驗證。
即時報表顯示位置是美國而不是台灣？
正常現象。GA4 用 IP 判斷地理位置，本地站（127.0.0.1 或 .local）的 IP 無法對應到真實地點，GA4 可能把它歸類為未知或預設地點。這不影響事件資料的收集。
