# office-ai 全課程 Prompt 純文字版

> 把本課程所有 Prompt 集中一份純文字檔，便於：
> - 課後存到 Google Keep 當個人 Prompt 庫（見 CH4-2）
> - 列印貼在桌邊速查
> - 跨平台複製（Gemini / NotebookLM / ChatGPT 都能用）
>
> **使用方式**：找到場景對應的 Prompt，把 `[方括號]` 內容換成你的實際資料，貼進 AI 對話框。

---

## Part 1 · Prompt 黃金公式

### P1-1 通用三元素模板（背景 + 格式 + 情境）
```
我是 [你的角色]，需要 [任務目標]。
情境：[發生什麼事、相關細節、時間地點對象]
格式：[幾段 / 條列幾點 / 字數限制]
語氣：[正式 / 友善 / 簡潔 / 專業]
```

### P1-2 通知信 / 公告
```
請幫我寫一封給 [收件人] 的 [文件類型]。
主題：[主題]
情境：[具體細節，含日期、地點、原因、要求行動]
格式：主旨 + [幾段] + 結尾感謝
字數：[N] 字以內
語氣：[正式但友善]
```

### P1-3 修改指令（給 AI 的回饋）
```
[指方向]：語氣再 [正式/口語/簡潔] 一點，不要 [某種開場 / 句型]
[補細節]：第 [N] 段加上 [具體內容]
[守邊界]：全文壓到 [N] 字內
```

## Part 2 · 文書自動化

### P2-1 週報生成
```
根據以下工作項目，整理成週報格式，分「本週完成」「進行中」「下週計畫」三區塊，
每點一句話、含具體成果或數字，語氣務實謙虛、不誇大，全文 300 字內。

工作項目：
[貼上本週工作記錄]
```

### P2-2 郵件・通知全體員工
```
請幫我寫一封通知全體員工的郵件，主旨是 [主題]，語氣正式但友善，不超過 150 字。
具體細節：[誰、什麼時候、做什麼、要員工配合什麼]
```

### P2-3 郵件・感謝
```
幫我寫一封感謝信給 [對象]，感謝他們 [具體事項]，保持真誠但不過於誇張，三段以內。
```

### P2-4 郵件・請假
```
我要請 [天數] 天假（[起 - 迄日期]），原因是 [原因]，
請幫我寫一封給主管的請假申請信，格式正式，含工作交接安排。
```

### P2-5 郵件・跟進
```
請幫我寫一封跟進郵件，提醒對方 [事項] 還未收到回覆，語氣禮貌不催促，三段內。
背景：[上次溝通日期 + 對方承諾事項]
```

### P2-6 文件・摘要
```
請將以下內容整理成三到五個重點，每點一句話，用條列格式：

[貼上原文]
```

### P2-7 文件・公告
```
請幫我寫一則公司內部公告，主旨是 [主題]，
格式：標題 + 說明段落 + 注意事項，語氣正式。
具體內容：[要公告什麼、為什麼、員工要做什麼]
```

### P2-8 文件・改寫語氣
```
請幫我改寫以下段落，讓語氣更 [正式/口語/簡潔]，保留原意：

[貼上原文]
```

### P2-9 簡報・大綱
```
請幫我生成一份 [N] 頁的簡報大綱，主題是 [主題]，對象是 [對象]，
風格要 [說服力強 / 清楚易懂 / 數據驅動]。
背景資訊：[核心訊息 / 關鍵數據 / 主要論點]
```

### P2-10 簡報・標題優化
```
以下是我的簡報重點，請幫我為每一頁想一個吸引人的標題（10 字以內，動詞開頭）：

[貼上各頁重點]
```

## Part 3 · NotebookLM 與會議準備

### P3-1 NotebookLM・會議整理（需先上傳逐字稿）
```
請根據逐字稿整理本次會議。請嚴格按以下格式：

【決議事項】
（列出所有已確認的決定，每條一句話）

【待辦清單】
（格式：事項 ｜ 負責人 ｜ 截止日）

【下次會議重點】
（這次未解決、需要下次討論的議題）
```

### P3-2 NotebookLM・長報告摘要
```
請從這份報告中提取：
1. 核心論點（3 點以內，每點一句話）
2. 關鍵數據（最重要的 5 個數字 + 其意義）
3. 對 [你的角色 / 你的部門] 的具體影響（3 點）
請每點都標明引用的章節或頁碼。
```

### P3-3 NotebookLM・多份文件交叉比對（需上傳 2+ 份）
```
請比對以下幾份文件：
- [文件 A] 重點是什麼？
- [文件 B] 重點是什麼？
- 兩份的共通點、差異點各列 3 項
- 若有矛盾資訊，請明確指出並引用原文
```

### P3-4 會議前準備・議程草擬
```
請幫我規劃一個 [時長] 的會議議程，主題是 [主題]，出席人員有 [列表]，
列出每個環節與時間，並標註誰主講、誰備詢。
```

### P3-5 會議前準備・對方背景研究（給 Gemini）
```
我要跟 [公司名稱 / 對方職位] 開會，主題是 [主題]。
請幫我列出 3 個可能會被問到的問題，以及我應該事先準備的回答方向。
```

### P3-6 分析・比較方案
```
請比較以下 [N] 個方案的優缺點，用表格呈現，
維度包含：成本、時間、風險、效果、所需資源：

方案 A：[說明]
方案 B：[說明]
方案 C：[說明]
```

### P3-7 分析・問題排查
```
我遇到一個問題：[描述問題]
請列出 3-5 個可能的原因，以及每個原因對應的解決方向。
最後給我一個「最有可能 / 最容易先驗證」的建議起手點。
```

## Part 4 · 個人 Prompt 庫建立

### P4-1 計畫・任務拆解
```
我需要完成 [任務]，截止日是 [日期]。
請幫我拆解成具體執行步驟，每步驟估算所需時間，並標出哪些步驟有相依關係。
```

### P4-2 學習・概念解釋
```
請用簡單語言解釋 [概念]，假設我完全不懂，
舉一個辦公室場景的實際例子說明，最後給我一句話的「實用結論」。
```

### P4-3 回饋・內容改善建議
```
請檢查以下內容，給我三個可以改善的建議，
重點放在 [邏輯清晰度 / 說服力 / 可讀性 / 行動指引]：

[貼上內容]
```

### P4-4 翻譯・商務中翻英
```
請將以下中文翻譯成英文，語氣保持正式精準、不堆疊形容詞，
適合用在商業郵件中：

[貼上中文]
```

### P4-5 翻譯・英文潤飾
```
請幫我潤飾以下英文段落，讓語法更自然流暢，保留原意，
適用 [商業 / 學術 / 一般] 場景：

[貼上英文]
```

## Part 5 · 行銷文案

### P5-1 Facebook 貼文
```
請幫我寫一則 Facebook 貼文：
主題：[你的主題]
目的：[宣傳活動 / 分享知識 / 推廣產品]
語氣：口語化、像在跟朋友聊天
結構：用 AIDA 公式（注意 → 興趣 → 渴望 → 行動）
字數：150-300 字
附上 3-5 個相關 Hashtag
```

### P5-2 Instagram 貼文
```
請幫我寫一則 Instagram 貼文：
主題：[你的主題]
語氣：年輕活潑、有個性
結構：問句開頭 + 3 個重點 + 行動呼籲
字數：80-150 字
附上 10-15 個 Hashtag（混合熱門 + 小眾）
```

### P5-3 LINE 官方帳號訊息
```
請幫我寫一則 LINE 官方帳號貼文：
主題：[你的主題]
語氣：親切直接、像在傳訊息給朋友
字數：60-100 字
不要加 Hashtag（LINE 不支援）
結尾加一個簡單行動指令（如：點下方連結 / 回覆關鍵字）
```

### P5-4 EDM 行銷郵件
```
請幫我寫一封 EDM 行銷郵件，
主題：[活動 / 產品 / 優惠]
對象：[現有客戶 / 潛在客戶 / 會員]
結構：主旨（吸引人開信） + 開場（與讀者連結） + 主體（價值說明） + CTA（明確行動）
字數：300-500 字
語氣：[專業 / 親切 / 急迫]
```

### P5-5 廣告文案・精煉
```
這是我的廣告文案：[貼上原稿]
請幫我精煉成：
1. 一句 slogan（10 字內）
2. 三點賣點（每點一句）
3. 一句 CTA（含明確動作 + 立即性）
語氣：[品牌風格描述]
```

### P5-6 創意・命名
```
請幫我為 [專案 / 活動 / 產品] 想 5 個名稱，
風格要 [正式 / 活潑 / 簡短]，
並說明每個名稱的含義與適用情境。
```

## 使用建議

1. **挑出你最常用的 3-5 個 Prompt**，存到 Google Keep 並加標籤
2. **每次使用後**，回頭微調 Prompt 模板（占位符不夠精準的補上）
3. **記錄使用次數**，找出 ROI 最高的 Prompt 持續優化
4. **不要每次都從頭打 Prompt**，從 Keep 複製比從 AI 對話歷史翻快 10 倍

詳細見 CH4-2 個人 Prompt 庫建立。
