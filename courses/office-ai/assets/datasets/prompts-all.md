# office-ai 全課程 Prompt 純文字版（v4 誠實框架）

> 把本課程所有 Prompt 集中一份純文字檔，便於：
> - 課後存到 Google Keep 當個人 Prompt 庫（見 CH6-1）
> - 列印貼在桌邊速查
> - 跨平台複製（Gemini / ChatGPT 都能用）
>
> **使用方式**：找到場景對應的 Prompt，把 `[方括號]` 內容換成你的實際資料，貼進 AI 對話框。
>
> **鐵則**：所有帶「控制版」的 Prompt 都包含「遇到不確定的標〔待補〕/〔待確認〕」。用 AI 前先確認輸出哪裡是它補的、哪裡是你給的。

## 章一 · 認識文字型 AI

### A1-1 通用三元素模板（背景 + 格式 + 情境）
```
我是 [你的角色]，需要 [任務目標]。
情境：[發生什麼事、相關細節、時間地點對象]
格式：[幾段 / 條列幾點 / 字數限制]
語氣：[正式 / 友善 / 簡潔 / 專業]
```

### A1-2 通知信 / 公告（基礎版）
```
請幫我寫一封給 [收件人] 的 [文件類型]。
主題：[主題]
情境：[具體細節，含日期、地點、原因、要求行動]
格式：主旨 + [幾段] + 結尾感謝
字數：[N] 字以內
語氣：[正式但友善]
```

### A1-3 立即修改指令
```
[指方向]：語氣再 [正式/口語/簡潔] 一點，不要 [某種開場 / 句型]
[補細節]：第 [N] 段加上 [具體內容]
[守邊界]：全文壓到 [N] 字內
```

## 章二 · 指令控制方法

### B2-1 角色設定（讓 AI 用對視角）
```
你是 [具體角色描述，例：嚴謹的主管週報編輯 / 企業公關寫手 / 專業客服回覆員]。
你的任務：[具體任務]
限制：[你不能做什麼，例：不可美化、不可捏造數字]
輸出格式：[表格 / 條列 / 段落]
```

### B2-2 風格語氣控制
```
請把以下內容改寫成 [目標語氣：正式/口語/簡潔/溫暖]，
保留原意，不添加我沒說的內容。
[貼上原文]
```

### B2-3 追問 / Repair 指令
```
你剛才的回覆有問題：[具體指出哪裡不對]
請修正這個部分，其他保持原樣。
```

### B2-4 客訴回覆（三輪控制版，見 CH2-3）
```
第一版：
你是資深客服，回覆以下客訴，語氣誠懇不推責，三段內，不承諾你不確定能做到的補償。
客訴內容：[貼上]

第二輪追問：
承諾「一週內回電」這件事，我要改成「三個工作日內」，其他段落維持原樣。

第三輪追問：
第一段太官腔，用更像真人在說話的口吻重寫第一段，其他段維持。
```

## 章三 · 文書工作演練

### C3-1 週報生成（控制口徑版，見 CH3-1）
```
你是嚴謹的主管週報編輯，把以下工作日誌整理成週報，不准把未完成寫成已完成。
判斷口徑：
1. 只有寫「完成／已確認／已拿到」才列「本週完成」
2.「還沒開始／等回覆／待確認／下週」放「進行中・需追蹤」
3. 金額用實際成交數字，不用交辦的目標數字
4. 回信、簽核、CRM 更新、私人事項不列入成果
5. 不確定的放「需人工確認」，不要美化
輸出：表格欄位＝分類 ｜ 週報文字 ｜ 來源編號 ｜ 需人工確認
工作日誌（每條前面加 [編號] 方便回查）：
[貼上本週工作記錄]
```

### C3-1R 週報自我審核（跑完 C3-1 再用）
```
以下是原始工作日誌，和你剛產出的週報。請審核，找三種問題：
1. 哪些句子在日誌找不到來源？
2. 哪些「還沒開始／等回覆」被寫成已完成？
3. 哪些重要待追蹤事項被漏掉？
輸出表格：問題類型 ｜ 週報原句 ｜ 對應編號（沒有寫「無」）｜ 修正建議
最後重寫一版 180 字內、保留來源編號的週報。
[貼上原始日誌] [貼上上一版週報]
```

### C3-2 表格整理（控制版，見 CH3-2）
```
請只根據我給的資料整理以下表格，不要計算、不要推估、不要補值。
- 遇到空格：填「（無）」
- 遇到矛盾欄位：填「（衝突，請確認）」
- 遇到你不確定的欄位：填「〔待確認〕」，不要猜
輸出格式：[表格名稱欄位列表]
原始資料：
[貼上原始資料]
```

### C3-3 長文摘要（控制版，見 CH3-3）
```
請只整理出現在以下文件裡的重點，不要推算沒有的數字，不要衍伸文件沒說的結論。
如果某個數字是你從其他數字計算出來的，請標「（推算，請核對）」。
輸出格式：條列重點，每點一句話，標明對應段落或頁數。
[貼上原始文件]
```

## 章四 · 簡報製作演練

### D4-1 簡報大綱（控制版，見 CH4-1）
```
請幫我生成一份 [N] 頁的簡報大綱，主題是 [主題]，對象是 [對象]。
限制：只用我提供的資訊，沒有數據不填，沒有策略不寫。
如果你想添加我沒給的資訊，標〔待補〕，不要自行填入。
背景資訊：
[貼上你有的資料]
```

### D4-2 內容展開（控制版，見 CH4-2）
```
請依照我給的要點展開以下每張投影片的內容。
限制：只依照我提供的要點展開，不補充我沒給的例子、數據或策略。
如果某個地方我的資料不夠，標〔待補〕，等我補充。
大綱：
[貼上投影片要點]
```

### D4-3 配圖描述（見 CH4-3）
```
這張投影片的核心訊息是：[一句話說明]
受眾是：[對象]
請描述一張具體、切題的配圖：
1. 主體是什麼（人 / 物 / 場景）
2. 視角與構圖（特寫 / 廣角 / 俯視）
3. 情緒與色調
4. 避免什麼（例：避免過於抽象的意境照）
```

### D4-4 簡報標題優化
```
以下是我的簡報重點，請幫我為每一頁想一個吸引人的標題（10 字以內，動詞開頭）：
[貼上各頁重點]
```

## 章五 · 會議工作演練

### E5-1 逐字稿校對重點提示
```
以下是 AI 轉錄的會議逐字稿，請特別標出可能轉錯的部分：
1. 人名（尤其是罕見字或外文）
2. 數字（金額、日期、百分比）
3. 決議事項（確保措辭精確）
逐字稿：
[貼上]
```

### E5-2 會議紀錄（三區分離版，控制版，見 CH5-2）
```
以下是本次會議的逐字稿。請嚴格依照「三區分離」原則整理，不可把「原則同意、再看看」或「等確認」寫成已決議。

【決議事項】
（只列出明確拍板確認的事項，措辭保留原話，例：「採用 A 方案」）

【待辦清單】
（格式：事項 ｜ 負責人 ｜ 截止日期——只填逐字稿有提到的，沒說截止日填「待確認」）

【待確認 / 下次討論】
（模糊表態、原則同意、需主管批准、尚未決定的，一律放這區）

逐字稿：
[貼上]
```

### E5-3 開會通知（控制版，見 CH5-3）
```
請草擬一封開會通知郵件。
會議資訊：
- 日期：[日期]
- 時間：[開始時間]（結束時間我還不確定，不要填）
- 地點：[地點]（如果有線上連結我會另外補，先寫「連結：〔待填〕」）
- 出席人員：[名單]
議程（只列逐字稿裡出現的待討論事項）：
[貼上待議事項]
截止回覆：（我沒說截止日，不要自行加「幾號前回覆」）
```

## 章六 · 持續應用與安全

### F6-1 個人場景盤點
```
我是 [職位]，以下是我一週的主要工作類型：
[條列 5-10 項]
請幫我找出哪些最適合用文字型 AI 輔助，依「省時潛力×失誤風險」分成三類：
- 高省時、低風險：可直接導入
- 高省時、高風險：需加查核流程
- 低省時或不適合：不建議
```

### F6-2 AI 工作流程規劃
```
我想把 [任務名稱] 導入 AI 輔助流程。
目前流程：[現在怎麼做]
請建議 AI 可以介入的步驟、建議的 Prompt 類型，以及哪些步驟必須人工處理。
```

### F6-3 翻譯・商務中翻英
```
請將以下中文翻譯成英文，語氣保持正式精準、不堆疊形容詞，
適合用在商業郵件中：
[貼上中文]
```

### F6-4 文件改善建議
```
請檢查以下內容，給我三個可以改善的建議，
重點放在 [邏輯清晰度 / 說服力 / 可讀性]：
[貼上內容]
```

## 使用建議

1. **挑出你最常用的 3-5 個 Prompt**，存到 Google Keep 並加標籤
2. **優先用「控制版」Prompt**（帶〔待補〕/〔待確認〕的），可大幅減少 AI 自行腦補的問題
3. **每次使用後**，回頭微調 Prompt 模板（占位符不夠精準的補上）
4. **查核 AI 輸出的三件事**：數字是否有來源、日期是否符合事實、人名是否正確

詳細說明見 CH6-1 建立你的 AI 工作流程。
