# 回家 30 分鐘復跑手冊

> 課堂上跟著老師按了幾下，回家用真實工作資料跑不出來？這是最常見的問題。
> 本手冊針對 5 大辦公室場景，每個給「5 步驟 30 分鐘」復跑流程，含資料整理 / 貼 prompt / 跑 AI / 追問 / 驗證。
>
> **打開 Gemini（gemini.google.com）或 NotebookLM（notebooklm.google）即可開始。全程免費版。**

---

## 共同前置：第一次跑前確認 3 件事（5 分鐘）

1. **資料份量**：你打算貼給 AI 的內容有多少？
   - < 500 字 / < 50 筆 → 直接複製貼上對話框
   - 500-5000 字 / 50-300 筆 → 用 Gemini 的「附件」功能上傳（PDF / DOCX / TXT）
   - > 5000 字 / > 300 筆 / 多份文件 → 改用 NotebookLM（免費版可上傳 50 份來源）

2. **資料敏感度**：是否含「客戶名單 / 員工薪資 / 未公開財務數字 / 個資」？
   - 是 → **不要上傳免費版 AI**，先去識別化（化名「員工 A」「客戶 X」、模糊金額為區間）
   - 否 → 可上傳。但仍建議避開：身分證號、銀行帳號、未公告人事案
   - **公司若有 AI 使用規範，以規範為準**

3. **工具選擇**：依案例選平台
   - 寫郵件 / 通知 / 公告 / 翻譯 → **Gemini**
   - 整理會議逐字稿 / 摘要長報告 / 多份 PDF 比對 → **NotebookLM**
   - 在 Google 文件內潤飾段落 / 改寫語氣 → **Google Workspace AI（內嵌）**

## 案例 ① 通知信 / 公告 — 30 分鐘食譜

對應課程：CH1-3 寫第一封辦公室通知信、PRAC1 Prompt 黃金公式填空台

### 步驟 1：先想清楚要寫什麼（5 分鐘）
打開記事本，回答 4 題：
- **對象**：要寄給誰？（全體員工 / 部門 / 主管 / 特定人）
- **語氣**：正式還是輕鬆？對方跟你的階層關係？
- **具體細節**：日期、地點、金額、聯絡窗口 — 哪些非寫不可？
- **格式**：主旨 + 幾段正文 + 字數限制？

**4 題答不出來 → 不是 AI 寫不好，是你還沒準備好。**先去找資料。

### 步驟 2：套黃金公式組 prompt（3 分鐘）
打開 Gemini，貼入這個結構：
```
我是 [你的角色]，需要寫一封 [文件類型]。
對象：[收件人]
情境：[發生什麼事、具體細節]
格式：主旨 + [幾段] + [結尾形式]，[字數] 字內。語氣：[正式/友善/簡潔]。
```

### 步驟 3：跑 AI 並讀第一版（3 分鐘）
- 等 Gemini 跑完（通常 10-20 秒）
- **不要急著複製**，先讀一遍
- 確認三件事：(1) 主旨清楚？(2) 語氣符合公司風格？(3) 重要細節都到位？

### 步驟 4：追問修正（10 分鐘）
若有以下狀況，在同一個對話框繼續說：
- 語氣不對 → 「語氣再正式一點，不要用『大家好』開頭」
- 漏掉細節 → 「第二段加上『本措施自 6/1 起實施』」
- 太長 → 「請壓到 200 字內，刪掉客套話」
- 太短缺說明 → 「請補充為什麼這樣做的一句話理由」
- AI 自己加了你沒要的內容 → 「請刪掉『經過研究證實』那句，我們是行政通知不是學術報告」

**修改原則：指方向 + 補細節 + 守邊界。**模糊形容詞（「再好一點」「自然一點」）一律無效。

### 步驟 5：驗證輸出（9 分鐘）
- **必查 3 件事**：① 收件人稱謂正確 ② 日期、金額、分機號碼數字都已填入 ③ 沒有 AI 自己編的「研究顯示」「根據調查」這類來源不明的句子
- 複製到 Gmail，補上收件人與主旨
- **發送前自己再讀一次**，這封信會印在公司形象上

## 案例 ② 週報 / 摘要 — 30 分鐘食譜

對應課程：CH2-1 週報、公告、摘要

### 步驟 1：蒐集本週工作記錄（10 分鐘）
從以下來源整理：
- Google 行事曆本週事件
- Gmail 已寄出的工作信主旨
- LINE 工作群本週訊息（截圖或整理重點）
- 自己的 todo / 筆記

整理成簡單格式（直接在記事本）：
```
週一：完成 X / 開了 Y 會議 / 跟 Z 客戶通電話
週二：...
週三：...
週四：...
週五：...
```

**至少 20 條，越完整 AI 越有東西可寫。**

### 步驟 2：選對 prompt（2 分鐘）
打開 CH1-2 的 Prompt 範本 20 句，找到「文件・週報」那一句：
```
根據以下工作項目，整理成週報格式，分「本週完成」「進行中」「下週計畫」三區塊：[清單]
```
複製貼進 Gemini，把 [清單] 換成步驟 1 整理好的內容。

### 步驟 3：跑 AI（3 分鐘）
等 Gemini 產出。第一版通常會：
- 自動分三區塊
- 把工作項目分類
- 可能會自己「補上」一些你沒寫的計畫（要小心）

### 步驟 4：追問修正（10 分鐘）
- AI 自己編的事 → 「請只用我給的內容，不要自己補我沒寫的計畫」
- 太流水帳 → 「請把『本週完成』改寫，每項加一句具體成果或數字」
- 漏掉重要事 → 「[某事項] 是本週最重要的，請放在最前面並強調」
- 語氣不對 → 「這是給主管看的，請改成更謙虛、不誇大的語氣」

**最關鍵的一句追問 ——「Repair 回合」**：與其自己一條條挑錯，不如讓 AI 審自己（套 CH2-1 產出 2.1-D）：
```
請審核你剛才產出的週報，對照原始記錄找三種問題：
1. 哪些句子在我的記錄裡找不到來源？
2. 哪些「還沒開始／等回覆」被你寫成已完成？
3. 哪些重要待追蹤事項被你漏掉？
列表指出後，重寫一版 180 字內的週報。
```
這一步最能擋住「把還沒做的寫成做完了」——AI 第一版最常犯這個錯。

### 步驟 5：驗證輸出（5 分鐘）— **這步絕對不能省**
- **逐條對照原始記錄**，確認沒有 AI 編出來的事
- 「下週計畫」確認是你真的打算做的（不是 AI 替你決定的）
- 數字、客戶名稱、金額逐一核對
- 通過後複製到公司週報系統 / Email

## 案例 ③ 會議紀錄整理 — 30 分鐘食譜

對應課程：CH3-1 NotebookLM 實戰、CH3-2 會議前準備術、PRAC3

### 步驟 1：取得逐字稿（5 分鐘）
方法 A — **會議當下錄音**：
- Google Meet 開啟「會議記錄」自動產出逐字稿（會議結束後存到 Drive）
- 或用 iPhone 內建「備忘錄」錄音轉文字、Otter.ai 免費版

方法 B — **沒錄音時**：
- 把自己手寫的會議筆記打成文字，至少 500 字
- 含：時間、出席者、討論主題、發言重點、爭議點、結論

### 步驟 2：上傳到 NotebookLM（3 分鐘）
- 開 notebooklm.google → 新建筆記本
- 點「新增來源」上傳逐字稿（TXT / DOCX / PDF 都行）
- 等 NotebookLM 處理完（通常 30 秒 — 1 分鐘）

### 步驟 3：跑會議整理 prompt（5 分鐘）
在 NotebookLM 對話框貼：
```
請根據逐字稿整理本次會議。請嚴格按以下格式：

【決議事項】
（列出所有已確認的決定，每條一句話）

【待辦清單】
（格式：事項 ｜ 負責人 ｜ 截止日）

【下次會議重點】
（這次未解決、需要下次討論的議題）
```

**為什麼用 NotebookLM 不用 Gemini？** NotebookLM 只根據你給的文件回答，不會像 Gemini 自己腦補沒提到的決議。

### 步驟 4：追問修正（10 分鐘）
- 漏掉某個討論 → 「[某主題] 在會議中有提到 [某時點]，請補進去」
- 負責人沒抓到 → 「[某事項] 的負責人是誰？請從逐字稿引用原文」
- 太簡略 → 「『下次會議重點』請說明為什麼這次沒決議、有哪些卡點」

**NotebookLM 的優勢**：每個回答都會標來源段落，你可以點開驗證它沒亂編。

### 步驟 5：驗證輸出（7 分鐘）— **這步絕對不能省**
- 點每個決議的「來源」連結，確認真的在逐字稿裡
- 待辦清單的「負責人」確認跟會議共識一致（沒被 AI 張冠李戴）
- 截止日如果是「下週」「月底」這種模糊詞，補成具體日期
- 複製到內部會議系統 / Email 發給出席者，請大家校正一遍再定稿

## 案例 ④ 建立你的個人 Prompt 庫 — 30 分鐘食譜

對應課程：CH4-1 找出三個高價值場景、CH4-2 建立個人 Prompt 庫

### 步驟 1：盤點本週你的 AI 場景（10 分鐘）
回想過去一週，哪些事情你用了 AI？或哪些事情下次想試？列出來：
- 寫了什麼類型的文件？（信件 / 報告 / 簡報 / 翻譯 / 整理）
- 哪些任務是重複性的？（每週週報、每月公告、固定回覆模板）
- 哪些任務你寫得很慢、很痛苦？

**至少列 5 個情境**。重複性越高、痛苦度越高，就是高價值場景。

### 步驟 2：選出 Top 3 並寫成可重用 Prompt（10 分鐘）
對每個高價值情境，套用三原則寫一個「模板 Prompt」（含占位符）：
```
[情境名稱：每週週報生成]

我是 [部門] 的 [職位]，每週要交週報給 [主管職稱]。
請根據以下本週工作清單，整理成週報格式：
- 三區塊：本週完成 / 進行中 / 下週計畫
- 每點一句話、含具體成果或數字
- 語氣務實謙虛、不誇大
- 全文 300 字內

本週工作清單：
[請貼上清單]
```

### 步驟 3：存到 Google Keep（3 分鐘）
- 開 keep.google.com
- 每個 Prompt 新建一張卡片，標題用情境名稱
- 加標籤 #prompt #週報 #郵件（依場景）
- **以後從 Keep 複製比從 Gemini 對話歷史翻快 10 倍**

### 步驟 4：實測 + 微調（5 分鐘）
- 拿每個 Prompt 套真實本週資料跑一次
- 跑完評估：產出是否可直接用？哪個 placeholder 不夠精準？
- 回 Keep 修正模板，越用越貼合自己工作

### 步驟 5：建立「使用日誌」（2 分鐘）
在 Keep 另開一張卡片：
- 每次用 Prompt 紀錄一行：「日期 / 哪個 Prompt / 省了幾分鐘 / 有沒有要改」
- 一個月後回看，找出最高 ROI 的 Prompt 持續優化

**3 個常用 Prompt 用熟 = 一個月省 5 小時。**

## 案例 ⑤ 社群 / EDM 文案 — 30 分鐘食譜

對應課程：CH5-1 社群貼文三平台攻略、CH5-2 EDM 與活動文案、CH5-3 廣告文字精煉術、PRAC5

### 步驟 1：先想清楚 4 件事（5 分鐘）
- **平台**：要發在 FB / IG / LINE 哪個？（語氣、字數差很大）
- **目的**：宣傳活動 / 分享知識 / 推廣產品 / 蓄熱粉絲？
- **受眾**：客戶是哪種人？（年齡、興趣、痛點）
- **核心訊息**：你最想讓他們記住哪一句話？

### 步驟 2：套平台模板 prompt（3 分鐘）
打開 Gemini，依平台貼對應模板：

**FB 模板**：
```
請幫我寫一則 Facebook 貼文：
主題：[你的主題]
目的：[宣傳活動/分享知識/推廣產品]
語氣：口語化、像在跟朋友聊天
結構：用 AIDA 公式（注意 → 興趣 → 渴望 → 行動）
字數：150-300 字
附上 3-5 個相關 Hashtag
```

**IG 模板**：
```
請幫我寫一則 Instagram 貼文：
主題：[你的主題]
語氣：年輕活潑、有個性
結構：問句開頭 + 3 個重點 + 行動呼籲
字數：80-150 字
附上 10-15 個 Hashtag（混合熱門 + 小眾）
```

**LINE 模板**：
```
請幫我寫一則 LINE 官方帳號貼文：
主題：[你的主題]
語氣：親切直接、像在傳訊息給朋友
字數：60-100 字
不要加 Hashtag（LINE 不支援）
結尾加一個簡單行動指令（如：點下方連結 / 回覆關鍵字）
```

### 步驟 3：跑 AI 並讀（3 分鐘）
讀的時候自問：
- 第一句有沒有抓住注意力？（FB 容易第一句就被滑過）
- 重點清楚嗎？還是塞太多訊息？
- 行動呼籲明確嗎？讀者知道下一步做什麼嗎？

### 步驟 4：追問修正（12 分鐘）
- 太通用沒個性 → 「請加入我們品牌特有的詞彙：[你常用的口頭禪 / 招牌句]」
- 第一句不夠勾人 → 「請給 3 個版本的開場句，我選最強的」
- Hashtag 不對 → 「請改成 [產業] 領域常用的 hashtag，不要太大眾化」
- 太正式 → 「再口語一點，像在跟好朋友推薦」
- 太隨便 → 「我們是 [B2B / 高端品牌]，語氣要再專業一點」
- 字數沒控制 → 「請壓到 100 字內」

### 步驟 5：驗證輸出（7 分鐘）— **這步絕對不能省**
- **產品名、價格、活動日期**逐一核對（AI 最容易在這三項出錯）
- 連結網址、聯絡資訊**自己手動補**，不要相信 AI 給的
- 找一位「不在這個專案內」的同事讀一次，看是否清楚易懂
- 發布前先用 FB / IG / LINE 的「預覽」功能看版面效果

## 共同尾聲：跑出問題時的 3 個求救步驟

1. **檢查 prompt 三原則** — 你給的 prompt 有沒有「背景 + 格式 + 情境」？少了哪個？
2. **切換工具再試** — Gemini 跑不順 → 試 NotebookLM；NotebookLM 不夠彈性 → 回 Gemini
3. **拆任務** — 一個 prompt 跑不出，把它拆成 2-3 個小 prompt 串接（例如：先請 AI 列大綱、再請 AI 補內容、再請 AI 修語氣）

---

## 最重要的一個原則

**AI 是助手不是主管，簽核責任永遠在你。**

任何要交出去的成果（通知信、週報、會議紀錄、社群貼文、EDM），都要你自己看過一遍才能發。
AI 加快草稿、AI 提醒盲點、AI 校對文字 — 但「我同意這個內容」的責任是你的。

寄出去那一刻，AI 不會幫你扛任何後果。
