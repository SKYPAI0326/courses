(06) · CH 3.2

從線稿讀出結構語言

同一張 L2，有人看到「只能用在高腳杯飲品」，有人看到「可以跨 5 個業種」——差的是讀結構語言的能力。本單元教你用 3 個白話觀察讀 L2，並判斷骨架能套幾種業種。時間：45 分鐘

學習成果
能從 L2 線稿看出最吸睛的東西（大小 + 位置 + 輪廓 + 對比 ≥ 2 項）、看的順序（P1 / P2 / P3）、空白夠不夠
能用 3 個白話觀察判定骨架能套幾種業種（高 / 中 / 低），說出「元素依賴度」是核心判準
能為 3 張 L2 各填 6 欄觀察表，並判定哪張最值得存入品牌骨架資產庫
今天結束時，你會交出
  1. 1 份 6 欄觀察表（針對你 PRAC3 的 L2 之一）
  2. 1 個泛用度判定（高 / 中 / 低 + 30 字理由：為什麼這個依賴度）
  3. 1 張被你選進「品牌骨架資產庫」的 L2 + 命名規範

完成這 3 件就算過關。讀結構語言是長期能力、今天先建感覺就好。

(01) · 看 / Observation｜10 min

開場情境：同一張骨架，能跨幾個情境？

為什麼這很重要
骨架驗收不只判「能繼承」，還要判「繼承後能套幾種業種」。粗粒度的觀察只能判斷能不能繼承，細粒度才能判斷骨架的資產價值。

弄一下咖啡工作室老闆在 CH3-1 跑出了春季抹茶拿鐵的 L2 線稿，通過了線稿檢查的 6 項清單——可以繼承了。

現在他想知道：這張 L2 能不能套「夏季柚子美式」？能不能套「秋季焙茶拿鐵」？能不能套「冬季薑汁可可」？

直覺上應該可以——因為結構相同、都是「商品中心 × 道具環繞」。但老闆不確定：這張骨架會不會偷偷限定在春季商品？會不會有某些結構元素（例：道具圍繞的空間感）在夏季冰品情境下就失效？

—
答案是：要看你怎麼讀 L2。同一張 L2，有人看出「這張結構只適合高腳杯類飲品」，有人看出「這張結構適合所有需要『商品 + 原料暗示』的業種」——讀結構語言的能力決定你判斷多大的泛用度。

本單元教你讀 L2 的三件事：最吸睛的東西、看的順序、空白夠不夠——這三件事決定一張 L2 的資產價值。

(02) · 看 / Observation｜現場發現活動｜8 min

同線稿、不同觀察

用 2-3 人一組，分享各自 CH3-1 產出的 L2，互相讀結構——體驗「結構語言也有閱讀層次」。

1
交換 L2（2 min）
跟 2-3 個同學交換自己 CH3-1 的 L2 線稿（截圖、手機傳、或 email）。每個人手上要有 2-3 張不同同學的 L2。
2
各自讀結構（3 min）
每張 L2 用以下 3 個問題讀：
  • 最吸睛的東西在哪？（眼睛第一個被吸引到的物件）
  • 看的順序如何？（眼睛從那裡出發，接著往哪裡走？）
  • 元素邊緣有沒有對齊？（留白均勻 vs 跳動、元素排列整齊 vs 亂）

不用寫標準答案，用自己的話描述。

3
互相分享（3 min）
每個人口述自己讀到的結構。不同人讀同一張 L2 通常會有不同觀察——這不是誰對誰錯，是觀察粒度的差異：
粒度 描述方式
粗粒度 「商品在中間、字在左邊」（看得見、沒看深）
中粒度 「商品偏右偏下、主標偏左上、徽章壓右上角、CTA 在左下」（能描述相對位置）
細粒度（本單元目標） 「最吸睛的東西在商品中央偏右、看的順序走中央放射、留白集中在右側以平衡左側的文字密度、元素都對齊 8pt 網格」

為什麼要細粒度？粗粒度判斷「能不能繼承」、細粒度判斷「能繼承到幾個新情境」——後者是 PM 骨架資產的真正價值。

(03) · 學 / Learn｜3 個白話觀察｜15 min

L2 的閱讀層次：3 個白話觀察

讀懂一張 L2，需要拆三個層次。這三個觀察決定一張骨架是「結構穩定」還是「換情境會垮」。

(01)
最吸睛的東西
最吸引眼球的物件——讀者第一眼停留的地方。判定條件：大小、位置、輪廓粗細、對比度，至少 2 項同時符合。
(02)
看的順序
眼睛從最吸睛的東西出發，接著走到哪裡的順序。分三層：P1 主焦點、P2 次焦點、P3 輔助焦點。
(03)
空白夠不夠
元素之間的留白是否均勻，元素邊緣有沒有對齊同一 8pt 網格線。對齊 = 套到新情境時結構不垮。
(03-A) · 最吸睛的東西

觀察 1｜最吸睛的東西（Visual Center）

判斷條件（至少 2 項同時符合）
大小（最大的物件或元素群）／位置（偏畫面中央或黃金分割點）／輪廓粗細（線條最粗、輪廓最明顯）／對比度（跟周圍元素差異最大）

常見誤判

誤判 為什麼錯
「最大的元素就是最吸睛的東西」 若最大物件在邊緣，通常不是最吸睛的東西。位置比大小更關鍵。
「文字越多的地方越重要」 文字密度高反而容易變成「次要閱讀區」，不是最吸睛的東西。

5 骨架的典型最吸睛的東西

骨架 最吸睛的東西位置
1 商品左 × 文字右 商品（左 40%、中央偏下）
2 mockup 右 × 文字左 mockup（右 45%、中央）
3 風景底 × 資訊下 風景上的主標（中上部、大字）
4 商品中心 × 道具環繞 商品（正中央或中央偏右）
5 大字堆疊 × 小圖輔助 主標字群（左上或中上、大字堆疊）
—
快速驗證法：看整張 L2 閉眼 3 秒，睜開第一眼看哪裡——那個通常就是最吸睛的東西。跟條件符合度應該吻合。若不吻合，代表你閱讀習慣影響了判斷，要重新對條件打勾。
(03-B) · 看的順序

觀察 2｜看的順序（Focal Hierarchy）

眼睛從最吸睛的東西出發，接著走到哪裡的順序——扣回 CH2-1 的視覺動線（Z / F / T / 中央放射 / 左上→右下）。

三層焦點
P1 主要焦點：最吸睛的東西（讀者第一眼看到的）
P2 次要焦點：讀完主焦點後，眼睛自然移動到的下一個元素（通常是主標或副標）
P3 輔助焦點：前兩個看完後，補充資訊的元素（CTA、徽章、3 特徵條列、價格）

新手判斷捷徑

  • 看整張 L2 閉眼 3 秒，睜開第一個注意到的 → P1
  • 閉眼，想下一個要看的內容是什麼 → P2
  • 看第三眼的東西 → P3

5 骨架的典型看的順序

骨架 P1 主焦點 P2 次焦點 P3 輔助焦點
1 商品左 商品 主標 特徵條列 + CTA
2 mockup 右 mockup 主標 App icon + CTA + 特徵標籤
3 風景底 主標 / 風景 徽章 3 特徵欄 + CTA
4 商品中心 商品 主標 + accent 價格 + CTA + 道具
5 大字堆疊 主標字群 副標 日期 / 地點 / 講師 + CTA
—
PM 判定要基於目標受眾的視線習慣，不是你自己的。你會先看文字（閱讀習慣），但目標受眾在 IG 滑過時，1 秒內大多先看視覺物件。骨架 5 大字堆疊例外——那是文字主導型，P1 才是主標字群。
(03-C) · 空白夠不夠

觀察 3｜空白夠不夠（留白均勻 + 元素邊緣對齊）

元素之間的留白是否均勻、元素邊緣有沒有對齊同一網格線。對齊 = 設計師級結構，套到新情境時穩定；不對齊 = 結構鬆散，換情境容易垮。

A. 留白均勻檢查
元素跟元素之間的距離是否視覺上看起來一致？畫面四邊（上下左右邊界）的留白是否均勻？有沒有某塊區域特別擁擠、某塊特別空？

均勻留白 = 視覺舒服、結構穩定／不均勻留白 = 視覺緊張、常代表結構有瑕疵
B. 元素邊緣對齊檢查
假想畫面有一張 8pt 網格紙覆蓋在上面。主標的左邊緣、商品的左邊緣、CTA 的左邊緣——這三者是否對齊同一條垂直線？徽章的圓心、主標的中線、CTA 的中線——是否對齊同一條水平線？

對齊網格 = 跨風格切換穩定／不對齊 = 套到新情境時容易走樣
誤判 為什麼錯
「有留白就是好」 不均勻的留白比沒留白更糟（視覺會跳動），均勻才是關鍵。
「元素大致對齊就可以」 0.5mm 的偏差在 AI 反覆產出時會放大，套到新情境後結構會跑掉。
(04) · 學 / Learn｜能套幾種業種｜5 min

L2 資產的價值分級：能套幾種業種？

看懂 3 個白話觀察後，下一步判定這張 L2 值不值得收入品牌骨架資產庫——關鍵指標是能套幾種業種（泛用度）。

級別 特徵 典型骨架 跨業種能力
多（高泛用） 結構要素少依賴、元素可替換性高 骨架 1 商品左 / 骨架 4 商品中心 可跨美妝、餐飲、手工皂、保健、寵物... ≥ 5 業種
一些（中泛用） 結構依賴特定元素類型（如 mockup、特定道具） 骨架 2 mockup 右 / 骨架 5 大字堆疊 跨同類型業種，3-4 業種
不太行（低泛用） 結構依賴特定場景（風景、地點、環境） 骨架 3 風景底 專用 1-2 業種（旅遊、活動、地方創生）

判斷關鍵：元素依賴度

  • 低依賴 = 能套多種業種：商品位置、主標位置、CTA 位置——這些「角色」在任何業種都存在，只是「內容物」替換
  • 高依賴 = 只能套少數業種：mockup 特別形狀、風景照的場景、手寫字體暗示——這些「必要元素」限制了骨架能去哪

資產收納判斷

能套幾種業種 收納決策
多（高泛用） 必收——存入品牌骨架庫、反覆使用
一些（中泛用） 條件收：若你的業種剛好符合就收，否則暫放
不太行（低泛用） 僅在特定情境收（例：你有旅遊客戶才收骨架 3 的 L2）
預期差異：同一張 L2，不同 PM 判定的能套幾種業種可能不同——因為你們的業種背景不同。這不是誰對誰錯，是判定基於你自己的業務情境。PM 的任務是做對自己情境有用的判斷。
(04-B) · 記住這句話
看結構不看美感、版型才用得久

美感放在 L4 完稿階段處理（換風格 preset 就能改），骨架 L2 階段追求「樸素通用」。越樸素的骨架越能跨業種——這是 PM 資產思維的核心反直覺觀念。

(06) · 做 / Hands-on｜15 min

活動：讀 3 張 L2 × 3 個白話觀察 + 能套幾種業種

教學意圖
用 3 個白話觀察判準讀 3 張 L2（自己 CH3-1 + 同學交換 2 張），為每張打出「能套幾種業種」判定。這個動作在 M4 密集用到——今天做的就是為 M4 建立骨架資產庫的第一步。
1
準備 3 張 L2（2 min）
  • 1 張：自己 CH3-1 產出的 L2
  • 2 張：從 2-3 位同學交換到的 L2（建議跟自己不同業種）
2
逐張做 3 個白話觀察分析（9 min，每張 3 min）
對每張 L2 填完以下 6 欄觀察表：
欄位 你的觀察（填入）
對應骨架（1-5） （骨架幾？）
最吸睛的東西（在哪、為什麼） （位置 + 至少 2 項條件符合）
看的順序（P1 / P2 / P3 各是什麼） （三層都要填）
空白夠不夠（均勻 / 有問題？） （具體描述）
能套幾種業種（多 / 一些 / 不太行 + 理由） （附具體依據）
可套用業種（至少列 2 個） （具體業種名稱）
3
分享 + 對照（4 min）
跟原作者（同學）對照：
  • 原作者的 L4 是什麼業種？
  • 原作者原本覺得這張適合幾個業種？
  • 你判定的能套幾種業種跟原作者的預期一致還是不同？

預期發現：有時同學覺得「能套很多業種」、你覺得「只能套一些」——第三方觀察比自己估得準。有時同學覺得「只適合自己業種」、你卻發現可以套到你的業種——結構語言是跨業種的、作者有時看不到自己的盲點。

完成標準（self-check）

—
  • 讀完 3 張 L2（1 自己 + 2 同學）
  • 每張 6 欄填寫完整、不留空
  • 最吸睛的東西判定有至少 2 項符合條件（大小、位置、輪廓、對比）
  • 看的順序有 P1 / P2 / P3 三層，不是只寫「主標很重要」
  • 能套幾種業種判定附具體理由（例：「因為結構依賴 mockup，所以只能套一些業種」）
  • 跟同學對照，有 1 個「觀察差異」紀錄
預期差異：你讀的 3 張 L2 跟其他學員不會一樣，結果也不會一樣。不同人讀同一張可能有不同判定——這不是誰對誰錯，是基於各自業務情境的判斷。PM 的核心能力是判斷對自己有用的答案，不是找「絕對正解」。今天判定「能套多種業種」的那張 L2——會成為 M4 反控階段的核心資產。
(07) · 常見錯誤 3 條

三個讀 L2 時容易踩的雷

錯誤 01
把「最大的元素」當成最吸睛的東西
現象
學員看 L2 線稿，只要看到某個物件占畫面比例最大，就直接判定那是最吸睛的東西。例如一張骨架 5（大字堆疊）的 L2 裡，某張小插圖雖然在畫面邊緣，但面積比主標字單個字大，學員就判「插圖是最吸睛的東西」。
原因
最吸睛的東西不是「最大」單一條件——還要看位置、輪廓、對比。邊緣大物件往往視覺重量低，中央偏中等物件的視覺重量更高。
解法
判定最吸睛的東西用至少 2 項條件同時符合的原則：大小（最大或前幾大）+ 位置（偏中央或黃金分割點）+ 輪廓粗細（線條最粗）+ 對比度（跟周圍差異最大）——2 項以上同時符合才算最吸睛的東西。單項符合（只大不在中央）都不算。驗證：閉眼 3 秒再睜開，第一眼看哪裡通常就是最吸睛的東西。
錯誤 02
誤判看的順序（眼睛只跟文字走）
現象
學員讀 L2 時習慣先看文字區，把主標當成 P1 主焦點。例如骨架 4 的 L2 商品在中央、主標在左側——學員判「主標是 P1」。實際上商品才是 P1，主標是 P2。
原因
你的閱讀習慣（讀者視角）影響判斷——你會先看文字。但目標受眾看廣告的視線路徑不一樣——大多數受眾先看視覺物件（尤其在 IG / LP banner 情境）。
解法
判定看的順序時問自己：「我如果是廣告主，希望讀者先看到哪個元素？」（答案通常是商品或主視覺）；「目標受眾在 IG 滑過時，1 秒內會記住什麼？」（通常是視覺元素，不是文字）。

骨架 5 大字堆疊例外——那是文字主導型，P1 才是主標字群。骨架 1 / 2 / 4 的 P1 都是視覺物件（商品 / mockup），不是文字。
錯誤 03
用「漂亮感」判斷能套幾種業種
現象
學員看到一張「很漂亮、很精緻」的 L2，直接判「能套很多業種」。實際上那張 L2 的結構依賴某個特殊元素（例：複雜的異形徽章、特殊角度的 mockup），換到其他業種反而套不進去。

相反的，一張「樸素、看起來沒什麼」的 L2 反而可能是高泛用——因為元素都是標準槽位（商品在左、主標在右上、徽章圓形壓商品右上），任何業種都能套。
原因
美感 ≠ 能套幾種業種。美感可能來自特殊元素（異形、獨特布局），而這些特殊元素會限制跨業種能力。越樸素的骨架越能跨業種——這是 PM 資產思維的核心反直覺觀念。
解法
判定能套幾種業種用元素依賴度，不是美感：結構的「角色」是誰（商品位置、主標位置、CTA 位置）？這些任何業種都有 → 能套多種業種。結構依賴的「內容物」是什麼（mockup、風景、特殊道具）？這些限制業種 → 只能套少數業種。

PM 收骨架資產時：收樸素的、標準化的（能套多種業種）——百搭款、反覆用；少收花俏的、特殊化的（只能套少數業種）——僅供特定情境。美感放在 L4 完稿階段處理（換風格 preset 就能改），骨架 L2 階段追求「樸素通用」。
(08) · 檢核題 2 條

確認你能讀出結構語言並判定骨架價值

Q1（概念驗證｜簡答）：請用自己的話說明，3 個白話觀察是哪三個？為什麼這三個觀察能用來判斷骨架能套幾種業種（而不是用「看起來漂亮程度」）？

至少命中 4 點算通過：

  • 最吸睛的東西：最吸引眼球的物件（大小 + 位置 + 輪廓粗細 + 對比度，至少 2 項符合）
  • 看的順序：眼睛從 P1 → P2 → P3 的閱讀順序（扣 CH2-1 視覺動線）
  • 空白夠不夠：元素間留白均勻 + 元素邊緣對齊 8pt 網格
  • 三個觀察都是結構屬性（位置、層級、對齊），不是表象屬性（色彩、字體）——跨業種都存在，所以用來判斷能不能跨業種復用最準
  • （加分）美感常來自「特殊元素」，這些限制業種；樸素骨架能套更多業種——越樸素越能跨業種
  • （加分）能套多種業種的 L2 = 可反覆用在多個業種 = 資產複利；判斷 L2 資產價值 = 判斷能套幾種業種 = PM 決定是否存入骨架庫
不及格的答案（確認自己沒有這些）
只列三個觀察名稱沒解釋（沒說各觀察在判斷什麼）；「美感也重要」（不對，本單元明確說美感 ≠ 能套幾種業種）；「差不多就好」（PM 要有結構化判準，不是差不多）。
Q2（應用驗證｜情境題）：你手上有 2 張 L2 線稿的結構描述。L2-A：骨架 4 商品中心・道具環繞，最吸睛的東西在中央商品（大 + 中央 + 輪廓粗），看的順序商品 → 主標 → 價格 → CTA（中央放射），留白均勻、元素邊緣對齊左右兩欄網格，特殊元素無（道具都是標準形狀，如小圓盤、湯匙、葉片）。L2-B：骨架 3 風景底・資訊下，最吸睛的東西在上半部風景中的特定建築物（明顯地標式），看的順序建築 → 主標疊字 → 徽章 → CTA（上→下 T 型），上下分割線明顯、網格整齊，特殊元素建築物輪廓暗示特定地點（例：希臘小屋）。請判定哪張 L2 能套更多業種，並給出具體理由 + 你作為 PM 會把哪張存入骨架庫、哪張暫放？

對照以下要點：

  • L2-A 能套更多業種（核心判定）
  • L2-A 高泛用理由（至少 2 條）：骨架 4 本身是高泛用骨架（可跨餐飲、寵物、手工皂、保健、伴手禮等 ≥ 5 業種）；特殊元素依賴低——道具都是標準形狀（可替換為任何業種的原料暗示）；最吸睛的東西是「商品」這個標準角色，跨業種都存在
  • L2-B 低泛用理由（至少 2 條）：骨架 3 本身是低泛用骨架（專用旅遊 / 活動 / 地方創生）；特殊元素依賴高——建築物輪廓綁定特定地點；換成非旅遊業種時，風景的存在變成「多餘元素」
  • PM 收納決策：L2-A 存入骨架庫——可反覆用；L2-B 暫放（或僅在旅遊業種情境收）
  • （加分）指出「L2-B 不是沒價值，只是專用性高」——若你業務是旅遊，L2-B 反而是核心資產；建議骨架庫應有「通用層（L2-A 這類）+ 專用層（L2-B 這類，按業種分類）」
不及格的答案（確認自己沒有這些）
「L2-B 比較漂亮、更有價值」（用美感判斷，違反錯誤 3）；「都很好」（沒做出 PM 決策）；「L2-A 有商品比較重要」（沒扣回元素依賴度判準）；「我看不出來」（PM 的核心能力就是判斷，這題沒做到就是不通過）。
