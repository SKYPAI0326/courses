(01) · CH 4.1

同線稿 × 跨產業套用

一張高泛用 L2 骨架，能套 3–5 個完全不同的業種——這就是骨架資產的複利起點。本單元教你以 PM 跨情境指揮者的身份，下具體指令讓 AI 執行骨架繼承，並用結構化驗收確認每張新 L4 是否達標。時間：45 分鐘

學習成果
能用同一個 L2 骨架 + 新業種 brief，產出至少 3 個不同產業的 L4 完稿（結構保留、內容全換）
能用驗收提問 A（8 項清單）判定每張新 L4 是否符合骨架，而非憑肉眼感覺
能下具體迭代指令（保留 X / 修 Y）調整偏離項，不說「再跑一次」
(01)

看 — 同骨架、3 個業種

為什麼這很重要
骨架的價值不在做一張——而在同一張能反覆套。你在 PRAC3 花時間跑出的高泛用 L2，不是「咖啡業種的線稿」，而是一個可以讓任何業種住進去的空房間。本段讓你親眼看到：同一個骨架結構、套 3 個完全不同業種，3 張看起來像 3 個獨立品牌。

開場情境

弄一下咖啡工作室老闆在 PRAC3 選出了他的第一張「高泛用 L2 骨架」：骨架 4 商品中心 × 道具環繞，從春季抹茶拿鐵的 L4 反推而來。他想測試：這張真的能套到咖啡以外的業種嗎？

他準備測試 3 個完全不同的業種：寵物鮮食包（朋友的品牌、有實體商品 + 原料道具）、手工皂禮盒（有實體商品 + 可用植物裝飾）、夏季柚子美式（同咖啡業種、但新商品）。

如果 3 個都套得起來——這張 L2 就是他的第一個跨業種骨架資產，以後做新廣告從這張開始、不從零開始。本段教你看懂這件事在結構層面是怎麼發生的。

後面 35 分鐘，教你怎麼指揮 AI 做這件事——並用結構化驗收確認 AI 真的做到了。

參考｜同 wireframe 結構 × 3 個業種

下方 3 張範例都套用「左文字 40% + 右商品 60%」的同一個 wireframe 結構、只換業種與內容——體驗「一張 L2 骨架可以跨業種繼承」的具體樣貌。

(02)

學 — 跨產業套用三原則 + Prompt + 驗收

跨產業套用三原則
原則 1
結構保留

L2 骨架的以下項目不動：元素位置（商品位置、主標位置、CTA 位置）、元素比例（商品占畫面多少 %、主標字級大小）、對齊關係（左右兩欄對齊、元素邊緣對齊網格）、視覺動線（骨架對應的 Z / F / T / 中央放射 / 左上→右下）。

換句話說：L2 的空間邏輯繼承、不重新設計。

原則 2
內容替換

以下項目全換：商品主視覺（拿鐵杯 → 鮮食包 → 手工皂禮盒）、文字內容（主標、副標、商品名、價格、CTA 全換）、道具（抹茶粉盤 → 雞肉切片 → 乾燥花瓣）、配色（鼠尾草綠 → 橘黃 → 米粉）、字體風格。

換句話說：表象 100% 替換、不能偷偷保留原業種的任何痕跡。

原則 3
不可跨越的界線

若新業種的「商品型態」跟骨架不符——換骨架、不是硬套。

新業種骨架 4（商品中心）適合？
寵物鮮食包✅ 有實體商品 + 有原料道具
手工皂禮盒✅ 有實體商品 + 可加裝飾
醫美保健品✅ 有實體罐裝 + 可加成分道具
手機 App❌ 沒實體商品、沒道具可環繞 → 換骨架 2 mockup 右
純粹的服務業（顧問 / 諮詢）❌ 沒實體商品 → 換骨架 5 大字堆疊
旅遊目的地❌ 商品是地方、不是物件 → 換骨架 3 風景底

PM 判斷：新業種的主視覺物件如果跟骨架的「角色槽位」對不上，就是信號要換骨架。硬套會導致版面崩壞，這不是 AI 的問題。

跨產業套用 Prompt 模板

跑跨產業套用前，先備好兩段必填規則——結構鎖定規則和內容替換規則缺一不可，否則 AI 會「幫你優化」或帶入原業種殘影。

完整版（建議搭配付費版生圖平台、品質較穩定）
請使用以下 L2 骨架作為結構基礎，套用新業種 brief、產出 L4 完稿：

【L2 骨架描述】
- 骨架類型：骨架 4 商品中心 × 道具環繞
- 商品位置：中央偏右
- 道具配置：商品左下 / 底部 / 右上三組道具點綴
- 主標位置：左上（2 行）
- CTA 位置：左下膠囊按鈕
- 徽章位置：右下圓形
- 視覺動線：中央放射（商品 → 道具 → 左側文字 → 價格 → CTA）

【新業種 brief】
- 業種：{新業種}
- 商品：{新商品名稱 + 規格}
- 客群：{新客群}
- 訴求：{新訴求 3-5 個調性詞}
- 必要元素：
  - 商品主視覺（中央偏右）：{新商品描述}
  - 左上標籤：{新標籤}
  - 主標（2 行）：{新主標}
  - accent 手寫字：{新 accent}
  - 道具三組：{新道具描述、對應商品原料 / 成分}
  - 價格（左下）：{新價格}
  - CTA（左下）：{新 CTA 文案}
  - 徽章（右下）：{新徽章內容}
- 風格 preset：{新風格 preset，如日系 / 美式 / 韓系}
  - 配色 ≤ 3 色：{具體 3 色}
  - 字體：{主標 + 內文字族}
  - 調性：{3-5 個具體形容詞}

【結構鎖定規則】
- 元素位置、比例、對齊、視覺動線必須與 L2 骨架完全一致
- 不可「優化版面」、不可「重新構圖」
- 若某元素在原 L2 是圓形徽章、新 L4 也必須是圓形徽章
- 若某元素在原 L2 占畫面 30%、新 L4 也必須占約 30%

【內容替換規則】
- 商品、文字、道具、配色、字體全換
- 不保留原業種（咖啡 / 抹茶 / 拿鐵等）的任何視覺暗示
- 新業種應該看起來像獨立品牌作品、不是「原業種的換皮版」

【尺寸】1:1（與 L2 一致）
堪用版（免費平台，上傳 L2 圖 + 描述）
上傳這張 L2 骨架圖，請依以下新業種 brief 產出 L4 完稿：

【新業種】{業種}
【商品】{商品}
【主標】「{主標}」
【必要元素】商品在中央偏右、主標左上、CTA 左下、徽章右下
【風格】{風格 preset 關鍵字 + 3 色}

結構與原 L2 完全一致、只換內容。不可重新構圖。
—
兩段規則缺一不可：結構鎖定規則不貼 → AI 會「幫你優化」版面；內容替換規則不貼 → AI 會帶入原業種殘影（例：新業種是手工皂、但跑出帶有咖啡杯風格的皂）。

預期差異：即使貼了兩段規則，也不保證 100% 對齊——通常會有 1–2 項偏離。這是預期工作流、不是失敗。用驗收提問 A 找出偏離項，再用驗收提問 D 具體迭代。
驗收提問 A｜結構對齊 8 項清單

拿到新 L4 後，在同對話內貼以下驗收清單，讓 AI 對照原 L2 骨架逐項自檢。

驗收提問 A（可直接複製貼上）
請對剛才的 L4 做結構對齊自檢、對照原 L2 骨架逐項回答：

1. 商品位置（中央偏右、占畫面約 X%）：是 / 否 / 部分
2. 主標位置（左上、2 行）：是 / 否 / 部分
3. 徽章位置（右下、圓形）：是 / 否 / 部分
4. CTA 位置（左下、膠囊形）：是 / 否 / 部分
5. 3 特徵 / 道具配置（商品左下 / 底部 / 右上）：是 / 否 / 部分
6. 配色（只用指定 3 色、無多餘色）：是 / 否 / 部分
7. 視覺動線（中央放射、從商品出發）：是 / 否 / 部分
8. 多餘元素（有無 AI 加上你沒要求的元素、如浮水印 / 裝飾邊框）：無 / 有

結論：通過 __/8 項。建議：收 / 迭代 / 重跑。

判定標準：

≥ 6/8 通過
收（存檔進骨架資產庫）
3–5/8 通過
用驗收提問 D 迭代 1 次
< 3/8 通過
推翻重來，檢查 prompt 的結構鎖定規則是否足夠具體
驗收提問 D｜迭代修正指令（本單元新教）

當驗收 A 出現 3–5/8 時，不要說「再跑一次」，而是下結構化迭代指令——保留通過部分、只修不過部分、指令要具體到可打勾。

驗收提問 D 格式
剛才那張 L4 的結構對齊驗收：通過 {X/8}。

保留：第 {通過的項號}（這些正確、不要動）

需要修：
- 第 {偏離項號}：「{項目名稱}」
  原 L2 設定：{原本的正確位置 / 形狀 / 配色}
  剛才輸出：{AI 實際跑出的偏離狀況}
  請修正為：{預期的正確結果}

其他項目請保持不變。只修上述偏離項、不要重新構圖。
範例（通過 5/8）
剛才那張 L4 的結構對齊驗收：通過 5/8。

保留：第 1, 2, 3, 5, 7 項（商品位置、主標位置、徽章位置、道具配置、視覺動線都正確）

需要修：
- 第 4 項「CTA 位置」：原 L2 設定為左下膠囊按鈕、剛才跑出來的 CTA 變成底部寬矩形按鈕。
  請修正回左下膠囊形、不要改尺寸。
- 第 6 項「配色」：原指定鼠尾草綠 + 奶油白 + 抹茶綠 3 色、剛才多出紫色漸層背景。
  請移除紫色、改為純奶油白背景。
- 第 8 項「多餘元素」：右下多了「AI generated」浮水印。請移除。

其他 5 項請保持不變。
(03)

做 — 拿 PRAC3 的高泛用 L2 套 3 個新業種

教學意圖
你拿自己 PRAC3 標記的「高泛用 L2」，套 3 個新業種，產出 3 張新 L4，跑驗收 A，必要時迭代。不是比誰的 L4 漂亮——是比誰能「結構保留 + 內容全換 + 驗收結構化」。
階段 1 · 3 min
挑 1 張高泛用 L2 + 選 3 個新業種

打開 PRAC3 標記的「高泛用」L2 骨架 + 結構筆記（筆記裡有「可套業種」清單）。從 10 業種庫（或自己業務情境）挑 3 個新業種——建議挑自己沒跑過的、測試泛用度。

挑業種規則：

  • 不能 3 個都是你熟悉的業種（AI 容易跑得順，但沒挑戰）
  • 避開骨架不相容的業種（例：骨架 4 商品中心不套 App / 服務業）
  • 建議組合：2 個你的主業種 + 1 個陌生業種
階段 2 · 9 min
跑 3 張新 L4（每張 3 min）
1
組 prompt（30 秒）
用跨產業套用 prompt 完整版（見學段），填入新業種 brief 的六件套。結構鎖定規則和內容替換規則必貼。
2
跑 AI（1 min）
貼 prompt 給 Gemini 或 ChatGPT，等 AI 產出新 L4。
3
跑驗收 A（1.5 min）

同對話內丟驗收提問 A 8 項清單，讓 AI 自檢。

≥ 6/8
存檔、進下一張
3–5/8
用驗收 D 迭代 1 次（再花 1 分鐘）
< 3/8
放棄這張、進下一張、記錄「此業種與骨架不相容」
—
預期成果：3 張新 L4，至少 2 張通過驗收 A ≥ 6/8。
階段 3 · 3 min
橫向比較 + 反思

把 3 張新 L4 並排看（+ 原 L2 骨架）：

  • 結構：3 張是否都保留 L2 的元素位置、比例、對齊？
  • 內容：3 張的商品 / 文字 / 道具 / 配色是否完全不同？
  • 感受：3 張看起來像 3 個獨立品牌嗎？還是仍有「同一底版的換皮感」？

反思題：

  • 哪個新業種跑得最順？為什麼？
  • 哪個新業種跑得最卡？為什麼？
  • 如果下週你要為這張 L2 再套 2 個新業種，你會挑哪 2 個？
完成標準（self-check）
  • 挑 1 張 PRAC3 的高泛用 L2 + 3 個新業種
  • 跑出 3 張新 L4（至少 2 張通過驗收 A ≥ 6/8）
  • 每張都跑過驗收 A（不是憑眼睛判斷）
  • 至少 1 張做過驗收 D 迭代
  • 存檔 3 張新 L4 + 3 份驗收 A 記錄
  • 寫 30 字反思（哪個最順 / 最卡 / 為什麼）
(04)

動手練習

練習目標
在 15 分鐘內完成完整的跨產業套用流程，並留下可重複使用的驗收記錄。

題目：

  1. 挑 1 張 PRAC3 高泛用 L2（你自己標記的）+ 選 3 個新業種（2 主業種 + 1 陌生業種）
  2. 跑 3 張新 L4（每張走「組 prompt → 跑 AI → 驗收 A」3 Step，每張 3 分鐘）
  3. 至少 1 張做驗收 D 迭代（若驗收 A 3–5/8）
  4. 橫向比較 + 30 字反思（哪個最順 / 最卡 / 為什麼）

預期成果：

  • 3 張新 L4（至少 2 張通過驗收 A ≥ 6/8）
  • 3 份驗收 A 記錄（每張 8 項打勾）
  • 至少 1 次驗收 D 迭代記錄（保留 X / 修 Y 的具體指令）
  • 1 段 30 字反思
完成標準
  • 3 個新業種包含至少 1 個陌生業種
  • 至少 2 張通過驗收 A ≥ 6/8
  • 每張都跑過驗收 A（不是憑感覺）
  • 至少 1 張迭代過驗收 D
  • 反思指出「哪個業種最順 / 最卡」+ 具體理由
—
預期差異提醒：你挑的 3 個業種跟同學完全不會一樣；3 張新 L4 也都會不同（同骨架 + 不同業種 + AI 特性）。不是比誰的 L4 漂亮，是比誰能「結構保留 + 內容全換 + 驗收結構化」。

你跑通的這 3 張新 L4 + 原 L2 = 你的骨架資產複利的起點，下次做廣告從這張開始、不從零開始。
(05)

常見錯誤 3 條

錯誤 1 of 3
新 L4 帶入原 L4 的配色 / 道具殘影
錯誤現象
套骨架 4（原 L2 來自春季抹茶拿鐵）到寵物鮮食業種，新 L4 跑出來商品位置對了、但背景仍是奶油白 + 抹茶綠點綴（原咖啡業種的配色）、道具仍隱約像茶葉形狀。變成「披著寵物鮮食皮的抹茶拿鐵」。
原因
即使經過 L2 中轉，AI 的對話記憶仍會回推原 L4 的表象。Prompt 中的「結構鎖定規則」如果不夠強，AI 會偷偷帶入殘影。
解法
在跨產業套用 prompt 中加強「內容替換規則」，明確寫出：
【內容替換規則（加強版）】
- 配色絕對不使用原業種的任何色彩（例：原是綠 + 白、新業種不能出現綠色）
- 道具絕對不使用原業種的原料暗示（例：原有茶葉、新業種不能出現葉片）
- 新業種應該看起來像獨立品牌作品、不是「原業種的換皮版」

預防心法：跑完新 L4，眼睛掃一次——有沒有任何原業種的綠色殘影？有沒有原業種的原料道具？若有，立刻用驗收 D 迭代「移除原業種殘影」。

錯誤 2 of 3
新業種跟骨架不相容還硬套
錯誤現象
手上是骨架 4（商品中心 × 道具環繞）的 L2，想套到「AI 筆記 App」——把 App 名稱當商品、在中央放「手機 mockup」、周圍散布「效率 icon」當道具。結果 L4 跑出來版面崩壞：App icon 太小、mockup 變形、周圍 icon 像亂飛。
原因
骨架 4 的結構前提是「實體物件 + 原料道具」。App 不是實體，效率 icon 不是原料——骨架 4 的「角色槽位」在 App 情境下都錯位。這不是 AI 的問題，是骨架選錯。
解法
判定骨架相容性再套（不跑 AI 前先判斷）：
新業種類型相容骨架不相容骨架
實體包裝商品（美妝 / 保健 / 手工皂）骨架 1 / 骨架 4骨架 2 mockup 右
飲品 / 甜點 / 餐飲骨架 4 商品中心骨架 3 風景底
App / SaaS / 數位服務骨架 2 mockup 右骨架 1 / 4（實體類）
旅遊 / 地方活動骨架 3 風景底骨架 1 / 2（物件類）
課程 / 講座 / 招生骨架 5 大字堆疊骨架 4（商品類）

換骨架、不是硬套——若你的 L2 是骨架 4、但想做 App 廣告，請用骨架 2 的 L2（PRAC3 可能沒跑到，去 CH3-1 補跑一張骨架 2 的 L2）。

錯誤 3 of 3
沒跑驗收 A 就收下 3 張新 L4
錯誤現象
跑出 3 張新 L4 後，眼睛掃一次覺得「看起來還行」，直接存檔——沒跑驗收 A 8 項清單。實際上可能有 2–3 項結構偏離：CTA 位置從左下跑到底部正中央；徽章從圓形變成矩形；多了 AI 產生的浮水印 / 裝飾邊框。
原因
AI 產出的平均美感太高（乍看都 OK），但結構偏離不會被肉眼立刻發現。在規模化產出（一次跑 5–10 張）時，一張偏離 = 整組系列裂。
解法
每張新 L4 都必跑驗收 A——即使你肉眼覺得「還行」，也要讓 AI 對照 8 項清單自檢。理由：AI 的自檢比你的肉眼準（AI 知道原 L2 骨架的精確參數）；跑一次驗收 A 只要 30 秒，省後面 10 分鐘返工時間。

紀律：跑完新 L4 → 立刻貼驗收 A → 看結果再決定收 / 迭代 / 推翻。不跳步、不憑感覺。

檢核題

Q1（概念驗證｜簡答）
請用「空房間 × 新住戶」的比喻，說明跨產業套用的 3 個原則：結構保留、內容替換、不可跨越的界線。每個原則給 1 個具體例子。
展開答案要點（至少命中 4 點算通過）
1
「空房間」= L2 骨架，「新住戶」= 新業種 brief
2
結構保留原則：空房間的格局不動——元素位置、比例、對齊、視覺動線保留。
例：骨架 4 的商品位置在中央偏右，換業種後仍在中央偏右
3
內容替換原則：新住戶的裝潢全換——商品、文字、道具、配色、字體全替換。
例：從咖啡換到手工皂，配色從鼠尾草綠換成米粉色，道具從抹茶粉換成乾燥花瓣
4
不可跨越的界線：新住戶類型若跟房間格局不符，換房間不是硬塞。
例：骨架 4 的商品槽位需要實體物件，若新業種是 App（數位產品），該換骨架 2 mockup 右，而不是強行塞 App icon 當「實體商品」
+
加分項：提到「結構繼承 = 資產複利」——好的 L2 可以反覆套 3–5+ 業種，每套 1 次經驗累積 1 次

不及格的答案：只列原則沒給例子；「不相容就重畫 L2」（應該換骨架，不是重畫）

Q2（應用驗證｜情境題）
你手上有一張 L2 線稿：骨架 1 商品左・文字右，原本是美妝業種（LUNÉA 精華液）反推而來。以下 3 個候選新業種，哪些可套用？哪些要換骨架？可套用的那個（或那些），寫一段跨產業套用 prompt（可簡版）。
候選 A：安活診所乳酸菌保健品（白色罐裝、有成分道具）
候選 B：notero AI 筆記 App（數位產品、有手機 mockup）
候選 C：瀨戶內慢旅 2 天 1 夜遊程（旅遊方案、有風景照）
展開答案要點（至少命中 4 點算通過）
1
骨架相容性判斷：
候選 A 乳酸菌保健品：✅ 可套用（實體罐裝商品 + 成分道具 = 符合骨架 1 商品左的結構）
候選 B notero App：❌ 不可套用，要換骨架（數位產品 + 手機 mockup = 應用骨架 2 mockup 右）
候選 C 瀨戶內旅遊：❌ 不可套用，要換骨架（商品是地方 + 風景 = 應用骨架 3 風景底）
2
候選 A 的跨產業套用 prompt（簡版）：包含 L2 骨架描述（骨架 1 商品左・文字右、左 40% 商品、Z 型視覺動線）+ 新業種 brief（安活診所乳酸菌 30 日份）+ 結構鎖定規則 + 內容替換規則（不保留原美妝業種的淡紫色殘影）
3
跑完後必跑驗收 A（8 項清單）——≥ 6/8 收，3–5/8 用驗收 D 迭代，< 3/8 檢查結構鎖定規則
+
加分答案：指出「候選 B 與 C 不是不能做廣告，只是不適合用這張 L2」；建議「骨架庫應該有多種骨架，不同新業種套不同骨架」——這是 PM 長期骨架資產思維

不及格的答案：3 個都說可套用（沒判斷骨架相容性）；沒寫 prompt 只說「去跑一張看看」；prompt 沒提結構鎖定和內容替換規則
