(01) · 看 / Observation｜5 min

一次性生圖 vs 三段閉環工作流

為什麼這很重要
上一堂學了「參考 → 解構 → 再造」的思維閉環。現在要把它落地：三段工作流是這個閉環在 AI 生圖上的具體操作版。沒有這張地圖，你很容易退回「下 prompt → 拿圖 → 結束」的一次性生圖模式，每次都從零開始。

把 CH1-1 的閉環套到 AI 生圖工作流上：

  • 參考：去 Pinterest / IG 找好作品（CH1-1 已教）
  • 解構 + 再造：用 AI 生圖 → 轉線稿 → 線稿反控，把結構抽出來再重用

最常見的錯誤答案：「下 prompt 給 AI、AI 吐一張圖、我就用這張。」

這是一次性生圖思維——跟 CH1-1 的「模仿 → 複製」一模一樣，只是把「複印」的動作交給 AI。結果就是：每次都從零開始、每次品質靠運氣、每次拿到的圖都無法跟上次連動。

本課要教你的是三段閉環工作流——讓每次產出都累積可重用的「結構資產」，下次做圖從資產開始，不是從零開始。

三段工作流總圖（跟著畫在你的筆記本上）：

┌──────────────────────────────────────────────────────────┐
│                                                          │
│   [起]            [承]             [轉 / 合]              │
│                                                          │
│   brief ─→ L4 ─→  L2 線稿  ─→  新 brief → 新 L4         │
│            完稿    (骨架)        新變體    新系列          │
│                                                          │
│   M2 生圖      M3 線稿反推     M4 線稿反控 + M5 整合      │
│   2h           2h              2.5h + 0.5h               │
│                                                          │
└──────────────────────────────────────────────────────────┘

每一段的產出 = 下一段的素材
每次跑完 = 累積一個可重用的骨架 + 品牌 DNA

方框內的縮寫：L4 = 高保真完稿；L2 = 視覺化線稿（低保真骨架）；M = Module 對應課程模組

思考一下（30 秒）：

為什麼中間要多一個「L2 線稿」環節？直接從完稿 A 到完稿 B 不行嗎？
不行。完稿包含太多表象（色彩、字體、紋理），直接從完稿 A 做完稿 B，AI 會連表象一起複製過去——做出來的就是「把 A 改成 B」的山寨貨，不是屬於 B 的作品。

L2 線稿是「脫掉表象、只留結構」的中轉站。透過線稿，結構可以乾淨地傳承到下一段，表象則完全替換。
(02) · 學 / Learn｜三段工作流逐段解說｜10 min

起 / 承 / 轉-合：三段各自做什麼

三段工作流各有明確的輸入、工具、產出，以及你這位 PM 扮演的角色：

第一段｜起：商業廣告生圖 對應 M2，約 2h
輸入
業種 / 商品 / 客群 / 訴求 / 必要元素 / 風格 preset（brief 六件套，CH2-2 會詳教）
工具
Gemini 或 ChatGPT 生圖（兩家都支援本段需求）
產出
1 張完稿（L4 高保真，日系為教學基準、其他風格 preset 為演練延伸）
你的角色
需求判定者
把模糊想法結構化成 brief 六件套、選骨架、選風格 preset
本段要學
5 種跨風格通用版面骨架（CH2-1）、brief 六件套組成 prompt（CH2-2）、多主題演練（PRAC2）
關鍵認知
完稿本身不是終點——它是下一段的素材。
第二段｜承：圖轉線稿 對應 M3，約 2h
輸入
上一段的 L4 完稿（或任何現成商業廣告）
工具
同對話裡延續 Gemini 或 ChatGPT（關鍵：同對話）
產出
1 張灰階線稿——保留物件輪廓、去掉色彩紋理、文字換佔位符
你的角色
骨架驗收者
用驗收提問 B 檢查 AI 反推的 L2 是否可繼承（輪廓清晰、色彩移除、佔位符正確、形狀一致、位置不變、風格像 wireframe）
本段要學
L2 視覺化線稿反推（CH3-1）、從線稿讀出結構語言（CH3-2）、線稿反推演練（PRAC3）
關鍵認知
結構骨架——一張可重複使用的、跨業種的、跨配色的、跨風格的「版面藍圖」。
第三段｜轉 / 合：線稿反控圖片 對應 M4 + M5，約 3h
輸入
L2 線稿 + 新 brief（新業種 / 新主題 / 品牌 DNA）
產出
新的 L4 變體 / 系列
3 種用法
(1) 同線稿 × 跨產業套用（CH4-1）——線稿是「空房間」、新業種是「新住戶」
(2) 同線稿 × 多變體（CH4-2）——AB 版面探索，挑最適合的
(3) 同線稿 × 品牌 DNA（CH4-3）——繼承式設計，產出視覺一致的系列
整合實戰
CH5-1：一次跑完 brief → 完稿 A → 線稿 → 完稿 B → 完稿 C，產出「一個品牌 3 張系列」
你的角色
系列把關者
建立 brand-dna.md、用驗收提問 C 檢查 AI 產出的 3 張系列是否都守在品牌 DNA 內（色票、字族、禁用、骨架、語氣）
關鍵認知
規模化與一致性——不是單張作品，而是可複製、可傳承、可建品牌系列的能力。
(03) · 學 / Learn｜為什麼要分三段｜3 min

一次性生圖的問題，三段閉環的解法

一次性生圖的問題
每次都從零開始，結構沒累積
修一張只能重跑，不能局部調整
跨系列（春夏秋冬 4 張）沒辦法保持風格一致
模型升級後，舊 prompt 可能跑不出來
三段閉環的好處
結構（L2 線稿）可以保存，重複使用
新主題只要換 brief、不用重新發明版面
品牌 DNA + 骨架 = 系列化資產
模型升級不影響骨架（骨架是結構知識、不綁定模型）
—
你不是在學「用 AI 畫圖」，是在用 AI 當執行團隊去實踐 PM 的三位一體協作（CH1-1 已建立）：需求判定 / 骨架驗收 / 系列把關。AI 負責生成，你負責下需求、驗收、把關。
(04) · 學 / Learn｜適用平台類型與能力需求｜5 min

本課適用的平台類型與工作流能力需求

本課不指定特定平台——你已經有帳號的、表現穩定的、就是好平台。但本課工作流對平台有 4 個能力需求——不論你用哪家，必須能做到：

能力需求 為什麼需要 哪些平台目前有
同對話延續編輯 第二段 L4 → L2 反推需要 AI 看到原圖當 context Gemini / ChatGPT 都原生支援
文字渲染（中 / 日 / 英） 主標 / 商品名 / CTA 可能要直接產文字 Gemini / ChatGPT 都基本可用；卡關可改 Canva 後製補字
結構保留能力 L2 反推時必須保留物件位置與形狀 Gemini / ChatGPT 都做得到、表現浮動
多比例切換（1:1 / 4:5 / 9:16） IG / Story / EDM 不同尺寸 兩家都支援、但偶爾要在 prompt 強調

2026-04 當下推薦組合（時間點記著、模型會變）：

免
免費路線
Gemini Free 或 ChatGPT Free 任一個——你現有帳號的那個。兩家的免費版都能走完本課工作流的主要學習。
升
單平台付費（約 US$20 / 月）
Gemini Advanced 或 ChatGPT Plus 任一個——挑你已經有帳號的那家升級。介面熟悉省學習成本。沒有任一家是「絕對更好」，當前表現各有強弱、且每季會浮動。
雙
雙平台（兩個都付，約 US$40 / 月）
Gemini Advanced + ChatGPT Plus。左手右手都有，任務卡關隨時切換。適合高頻商用、課程設計師、自媒體經營者。
保
Canva / Firefly 是後期版權保險（不論主平台選誰都該用）
AI 生圖有版權疑慮（訓練資料來源不透明）。商用交付前建議用 Canva 替換為合法授權素材、或 Firefly（Adobe 訓練資料版權乾淨）重跑。這兩個不在生圖階段用，只在「最後交付前的版權保險」階段出現。
×
本課明確不教的工具
Midjourney / Stable Diffusion / Flux / Grok / 文心 / 通義萬相 / 可靈。理由：同對話延續編輯這個本課工作流核心能力不穩定（無法穩定走 L4 → L2 反推），或需要額外技術棧（ComfyUI / API / Discord bot）——不適合本課的 PM 協作節奏。不是說它們不好，只是不適合本課的三段閉環工作流。若你已熟悉 MJ / SD，可繼續用，但本課的 L2 反推階段必須切到 Gemini 或 ChatGPT。
模型表現變動頻繁的提醒：上面寫「2026-04」是因為生圖模型每 2-3 個月就會升級或改政策——配額會變、品質會變、強弱項也會變。不要綁死任一平台——遇到卡關就測試另一個，課後養成「定期測試各家當前表現」的習慣，永遠選對你當下任務表現最好的那個。
預期差異：平台模型每季更新、介面每月微調、配額每年調整。本單元列的配額與定價以 2026-04 為基準，開課時請學員現場查各平台官網確認最新數字。你看到的定價頁跟同學的可能不一樣（促銷方案、地區差異），這是正常的。

(06) · 學 / Learn｜平台決策路線

零預算 / 單平台付費 / 雙平台付費——哪條適合你？

依預算、頻率、品質需求三個維度，本課提供三條路線供選擇。沒有哪條路線「絕對更好」，只有哪條最符合你現在的情境：

路線 1｜零預算
配置Gemini Free 或 ChatGPT Free（你有帳號的那家）+ Canva Free 可走完整個課程，但付費版的繼承設計穩定度拿不到。 適合：學習體驗 + 個人練習，不適合高頻商用
路線 2｜單平台付費（約 US$20 / 月）
配置升 Gemini Advanced 或 ChatGPT Plus 任一個 挑你已經有帳號的那家升級——介面熟悉省學習成本。沒有任一家是「絕對更好」，當前表現各有強弱、且每季會浮動。可走完整個課程，品質拿到 90%。 適合：每週有固定練習、品質要求中等的工作者
路線 3｜雙平台付費（約 US$40 / 月）
配置Gemini Advanced + ChatGPT Plus 兩家各有強項，依場景分工；卡關時隨時切換。 適合：專業工作者、課程設計師、自媒體經營者、高頻商用
分階段升級的 PM 思維：工具升級應該被「具體需求」推動，不是被「想用好的」推動。
第 1–2 週（CH1–CH3）：全程免費版夠用；第 3 週（CH4 繼承設計）：考慮升任一平台付費版；第 4 週以後（正式商用）：考慮雙平台付費。

(08) · 常見錯誤 3 條

三個你很可能會踩的雷

錯誤 01
一上來就用付費版（消費者思維、非 PM 思維）
現象
上完本單元後，當天刷卡同時升級兩個平台。一個月後帳單來了，花了 US$60，但產出的圖跟用免費版差不多——因為 prompt 還不夠成熟，模型再強也發揮不出來。
原因
這是消費者思維（「想買最好的工具」），不是 PM 思維（「選合適的工具解決當前問題」）。付費版的優勢在「最終品質」與「穩定度」，不在「剛學時的練習」——此時免費版已足夠。
解法
分階段升級：第 1–2 週（CH1–CH3）→ 全程免費版；第 3 週（CH4 繼承設計）→ 升級一個平台（挑你已有帳號的那家）；第 4 週以後（正式商用）→ 考慮雙平台付費。工具升級應被「具體需求」推動，不是被「想用好的」推動。
錯誤 02
堅持用 Midjourney / Stable Diffusion / Flux 走本課工作流
現象
原本是 MJ 重度使用者，聽完本單元後堅持用 MJ 走完整個課程。到第二段 L4 → L2 反推時卡住——MJ 沒有「同對話延續編輯」的原生功能，結構保留度不穩定。
原因
本課三段工作流的關鍵能力是同對話延續編輯——第二段必須能吃第一段的完稿作為 context，直接轉成線稿。Gemini / ChatGPT 是原生支援這個能力的少數平台；MJ / SD / Flux 單張品質可能更高，但工作流會斷裂。
解法
MJ 重度使用者的折衷方案：第一段生圖可以用 MJ 產出 L4 完稿，但第二段 L4 → L2 反推必須切換到 Gemini 或 ChatGPT，讓新對話吃 MJ 產出的圖當參考。不要強求用單一工具走完三段——工具是手段，工作流閉環才是目的。
錯誤 03
不做平台分工（全部塞給一個工具）
現象
選了零預算路線後，整個課程全部用同一個平台跑。結果：複雜 prompt 時遵循度不夠高；多比例切換頻頻跑回原尺寸；一整天下來免費配額用爆。
原因
每個平台都有強項與弱項、且強弱項每季都在變。「全部塞給一個工具」會讓你在工具當前弱項的任務上挫折——若你有兩家帳號、就該交叉用。
解法
基本分工建議（任務導向、不指定品牌）：
第一段生圖 → 簡單 brief 兩家任一；複雜 brief / 多比例切換 → 用 prompt 遵循度當下表現較好的那家
第二段 L4 → L2 → 哪家同對話延續編輯最穩、就用哪家
第三段品牌繼承 → 哪家 identity 一致性當前表現好、就用哪家
最後交付 → Canva 替換合法素材

心法：「能力導向、不偏好品牌、卡關就換另一家」——當前表現好的那家、不論是 Gemini 還是 ChatGPT、就是你的「主力」。三個月後可能換。每季花 30 分鐘把同一份 brief 各跑 2 張，自己對比兩家當前在你業種上的表現——這比看別人推薦更可靠。
(09) · 檢核題 2 條

確認你已經理解三段工作流與平台決策

Q1（概念驗證｜簡答）：請用自己的話寫或畫出「生圖 → 轉線稿 → 線稿反控」三段工作流，並說明：為什麼要多中間「線稿」這段？一步到位從完稿 A 直接做完稿 B 不行嗎？

寫在筆記本 / 備忘錄，能說出以下要點的 3 點以上即通過：

  • 三段工作流圖：起（brief → L4 完稿）→ 承（L4 → L2 線稿）→ 轉-合（L2 + 新 brief → 新 L4）
  • 中間要有線稿是因為完稿包含太多表象（色彩、字體、紋理），直接從完稿做完稿會把表象複製過去
  • 線稿脫掉表象、只留結構，讓結構可以乾淨傳承到下一段
  • 線稿是可重複使用的資產（同一張線稿可套多個新業種 / 多個變體 / 多個品牌主題）
  • （加分）一步到位等於「模仿 → 複製」，三段閉環等於「參考 → 解構 → 再造」
不及格的答案（確認自己沒有這些）
只畫三個方框沒寫理由；「因為老師說要這樣」；「這樣比較好看」（錯誤：線稿跟好不好看無關，是關於結構傳承）。
Q2（應用驗證｜PM 決策情境題）：主線 A 老闆想報名本課，限制如下：預算有限（月可支配約 NT$1,500 用於工具訂閱）、時間有限（每週只有週日晚上 2 小時可練習）、品質要求中等（IG 自用，不是專業設計交付）、目前已有 Canva Free + Gmail / Google 帳號。請用 PM 需求判定思維幫老闆做平台配置，說明：(1) 選哪條路線、(2) 哪個平台升級 / 哪個用免費、(3) 為什麼這樣配。

對照以下要點檢查你的答案：

  • 路線：單平台付費（約 US$20 / 約 TWD 620，在 NT$1,500 預算內）
  • 配置：挑老闆已有帳號的那家（Gemini 或 ChatGPT 任一個）升級；另一家的免費版仍當備援；Canva Free 維持免費
  • 理由：挑老闆已有帳號的那家升級——介面熟悉省學習成本；另一家免費配額夠當備援（每週 2 小時練習量不高）；Canva Free 足以應付 IG 交付（自用不需要 Pro）
  • 沒有任一家絕對更好——當前表現各有強弱、且每季會浮動；預算剩餘 NT$880 作為緩衝；品質中等 → 付費版品質已夠；不需雙平台
  • （加分）建議前 2 週先走零預算練基本功，第 3 週 CH4 再升級
不及格的答案（確認自己沒有這些）
「選雙平台付費，反正多花錢品質一定更好」（沒對應老闆的預算與頻率限制）；「選零預算路線，免費就夠了」（沒考慮老闆的品質要求是「中等」而非「只求體驗」）；「都可以啊」（沒做出具體建議）。
