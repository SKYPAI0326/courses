(02) Part 1 · Gemini 基礎操作

Gemini 基礎操作 + 會議紀錄整理

手寫整理 60 min，Gemini 跑得好 15 min。這單元的核心就是三件套 prompt：角色 + 任務 + 格式。把三件寫清楚，Gemini 的輸出就可預測、可直接用。先摸熟介面操作，再用小李的業務週會示範完整三步流程，最後換成你的部門變體自己跑一遍。

學習成果 · OUTCOMES
能操作 Gemini 介面三個關鍵動作（新對話 vs 繼續對話、上傳檔案、切換輸出格式）
能用三件套 prompt 跑完完整三步流程（決議清單 + 個人行動表 + 跟進信），同一對話視窗接續不重貼資料
能選自己部門變體（5 選 1）跑出一份整理好的會議紀錄，格式符合自己部門習慣
(01)

破題：小李的會後困境（3 min）

情境設定
業務副理小李剛開完業務週會。25 min、4 人、業績檢討 + 兩件客訴處理。發言亂序，沒人做正式紀錄。

會後小李要產出三件東西：

  • 本週業績問題決議清單
  • 自己的個人行動表（誰負責 / 何時前 / 卡關找誰）
  • 1-2 封跟進信草稿

手寫整理 60 min，Gemini 跑得好 15 min。

小李的真實困難不是不會用 Gemini，是「不知道 prompt 要怎麼寫才能跑出可以直接用的東西」。每次試，Gemini 給他的東西格式不對，要花 20 min 再改。

這單元任務：先把 Gemini 介面操作摸熟，再用三件套 prompt 跑完小李這場會議的整理流程，最後換成你的部門變體跑一遍。
(02)

介面操作說明（5 min）

只講三個「第一次用容易卡住」的操作
不需要講原理，講完就上機，讓操作本身回答疑問。

(1) 新對話 vs 繼續對話

開新對話 = Gemini 忘掉前面講的全部。繼續對話 = 它記得。

什麼時候做法範例
換完全不同的任務開新對話剛才整理會議紀錄，現在要寫招募文案
同一件事要接著做繼續對話決議清單跑完，接著跑個人行動表，再接著跑跟進信
→
本單元三步 prompt 要在同一個對話視窗跑完。這樣第二步直接說「結合剛才的決議」就有效，不用重貼資料。

(2) 上傳檔案方式

兩種方法都行：

  • 拖放：把逐字稿 .txt 或 .md 拖進輸入框
  • 點擊迴紋針圖示：選從裝置上傳

本課逐字稿素材是 .txt 檔，拖放最快。上傳後 Gemini 輸入框上方會顯示檔名，確認有顯示再送出。

→
免費版 Gemini 每次對話支援上傳 2-3 份檔案，超過要拆批次或改貼文字。

(3) 輸出格式切換

Gemini 預設可能給段落文字，但你要的是表格或清單。兩種改法：

  • Prompt 裡直接寫：「輸出：markdown 表格，4 欄」或「輸出：條列清單，每條 < 30 字」
  • 跑完再追問：「請改成表格格式」「請把每條壓短在 20 字以內」

Prompt 裡先寫清楚比事後追問省時。本單元三個 prompt 的「格式」段都已寫好，照用就行。

(03)

Prompt 三件套原則（15 min）

三件套 = 角色 + 任務 + 格式
你在 office-ai-cases 學過 5 大骨架。三件套是其中最常用的三個。辦公室場景 80% 的情況這三件就夠了。
(01) 角色
你要 Gemini 扮演什麼身份？
「你是小李的會議紀錄整理助手」
(02) 任務
具體做什麼？輸入是什麼？
「把以下 25 min 業務週會逐字稿，整理成決議清單」
(03) 格式
輸出長什麼樣？
「markdown 表格，4 欄：誰 / 做什麼 / 何時前 / 找誰確認」

爛 prompt vs 好 prompt 示範

爛 prompt（模糊）
幫我整理一下這個會議紀錄。 [貼逐字稿]
Gemini 的回應（爛 prompt）

Gemini 可能給你一段 400 字的摘要段落，或隨機給 3-5 個重點，格式不確定、欄位不齊，有時連誰負責什麼都講不清楚。

輸出不可預測 = 你要花時間改，或再追問兩輪。
好 prompt（三件套）
角色：你是業務副理小李的會議紀錄整理助手。 任務：把以下 25 min 業務週會逐字稿，整理成本週的決議清單。 格式：markdown 表格，4 欄 — 誰 / 做什麼 / 何時前 / 找誰確認；至少 4 筆；閒聊段落不列入。 [貼逐字稿]
Gemini 的回應（三件套 prompt）

Gemini 產出一張 4 欄表格，5 筆決議，例：

| 誰 | 做什麼 | 何時前 | 找誰確認 |
|---|---|---|---|
| 小李 | 聯絡 B 客戶確認訂單狀態 | 週三前 | 小李主管 |
| 小陳 | 更新業績追蹤表 | 週四前 | 小李 |
| ... | ... | ... | ... |

格式固定、欄位齊全，可以直接 copy 進 Google Docs 或寄信用。

常見錯誤：這兩種問法最常讓人卡

錯誤 1：問太模糊

「幫我整理一下」「幫我看看這份會議」「整理出重點」——Gemini 不知道你要什麼格式、幾欄、幾筆，輸出就會每次不一樣。修法：任務段加一句「整理成 XX 格式」或「輸出要包含 XX 欄位」。

錯誤 2：一次問太多件事

「幫我整理決議清單、個人行動表、寫跟進信、還有摘要一份 100 字給老闆看」——Gemini 通常只把第一件做好，後面的越來越草率。修法：三步走。每一步一個 prompt，同一個對話視窗繼續，就不用重貼資料。

三件套 = 角色 + 任務 + 格式。這三件寫清楚，Gemini 的輸出可預測、可直接用。
(04)

講師示範：業務週會逐字稿 → 會議紀錄（10 min）

主版 POV 小李 · 素材 sales-weekly-25min.txt
三步走完，在同一個對話視窗裡跑。骨架不變，只換任務和格式，就能跑完整份會議整理。

第 1 步：決議清單（3-4 min）

小李在 Gemini（第 1 步）
角色：你是業務副理小李的會議紀錄整理助手。 任務：把以下 25 min 業務週會逐字稿，整理成本週的決議清單。 格式：markdown 表格，4 欄 — 誰 / 做什麼 / 何時前 / 找誰確認；至少 4 筆；閒聊與情緒發洩段落不列入。 [此處拖入或貼上 sales-weekly-25min.txt]
Gemini 大概會回

Gemini 整理出 5 筆決議清單：

| 誰 | 做什麼 | 何時前 | 找誰確認 |
|---|---|---|---|
| 小李 | 聯絡 B 客戶確認本月訂單是否取消 | 週三前 | 業務主管 |
| 小陳 | 更新本週業績追蹤表並標記落後項 | 週四前 | 小李 |
| 全體業務 | 這週每人至少打 5 通開發電話 | 週五前 | 小李 |
| 小李 | 回覆 A 客戶客訴，說明處理方案 | 明天前 | 業務主管 |
| 小陳 | 追 C 供應商缺貨處理進度 | 週三前 | 小李 |

實際輸出每次略有差異。重點是 4 欄齊全、筆數 ≥ 4、閒聊段沒混進來。

第 2 步：個人行動表（3-4 min）

小李在 Gemini（第 2 步 · 同一對話視窗接續）
繼續。依剛才的決議清單，整理小李「本週個人行動表」。 格式：條列清單，每條包含 — 日期 / 要做什麼 / 卡關了找誰；只列小李自己要做的事，其他人的決議不列。
Gemini 大概會回

小李本週個人行動表：

- 明天：回覆 A 客戶客訴，說明處理方案 → 卡關找業務主管
- 週三前：聯絡 B 客戶確認本月訂單是否取消 → 卡關找業務主管
- 週五前：確認全體業務本週開發電話達標情況 → 卡關找小陳查報表

「卡關了找誰」這欄很關鍵。回家自己用時這欄能讓你不卡住。

第 3 步：跟進信（3 min）

小李在 Gemini（第 3 步 · 同一對話視窗接續）
最後一步。針對 A 客戶客訴這件事，幫小李寫一封跟進信草稿。 要求：語氣是「業務副理向客戶說明進度」；禁用「您」改「你」；主旨含日期 + 主題；結尾必須有「有任何問題請回覆這封信」；全信 ≤ 150 字。
Gemini 大概會回

主旨：[0102] A 客戶客訴處理進度說明

你好，感謝你告知上週的問題。我們已確認是出貨流程的疏失，責任部門本週四前完成內部檢討，我會親自追進度。
預計週五前給你書面說明與補救方案。
有任何問題請回覆這封信。

——小李

禁用「您」改「你」要明寫在 prompt，Gemini 預設商業書信會用「您」。

示範結束補一句：三步完成了。回頭看，每步的三件套：角色都一樣（小李的整理助手），任務每步不同，格式每步不同。骨架不變，只換任務和格式，就能跑完整份會議整理。

如何要求 Gemini 調整格式（額外示範 1 min）：

如果決議清單貼上 Google Docs 後跑版，可以這樣追問
請把剛才的決議清單改成純文字條列（不用 markdown 表格），每條格式：「誰：做什麼（何時前）」。
三步 prompt，同一個對話視窗跑完。骨架不變，換素材就能套到你的部門會議。

(08)

3 個會卡關的地方 + 解法

01 開新對話跑第 2 步，Gemini 忘掉決議清單
現象
第 1 步跑出決議清單後，學員開了新對話視窗跑第 2 步「依剛才的決議，整理個人行動表」。Gemini 說「請提供決議清單讓我整理」——它根本不記得剛才做的。
原因
每個 Gemini 對話都是獨立的，開新對話 = 清空記憶。
解法
三步 prompt 在同一個對話視窗跑完。第 2 步說「繼續」或「依剛才的結果」就有效，不用重貼資料。只有在換完全不同任務時才開新對話。
02 「某人提到想做 X」被寫進決議清單
現象
怡君說「我覺得下個月可以試試直播帶貨，看看大家意見」——決議清單寫成「怡君：下個月開直播帶貨（月底前）」。怡君下週被問進度，說「我沒答應啊」。
原因
Gemini 會把「想法拋出」合理化成「決定要做」，沒加限制 instruction 就會這樣。
解法
prompt 在「格式」段加一句「仍在討論的提案不列入，只列已確認的決議」。跑完後對照逐字稿核對一遍，確認每筆決議真的在會議中被確認了。
03 跟進信用「您」+ 寫成 300 字商業書信
現象
Gemini 寫出「您好，承蒙協助，特此說明本次客訴處理進度如下……」—— 400 字，起承轉合，同事看完一肚子火，覺得很假。
原因
Gemini 預設「商業書信」模板會用「您」+ 鋪陳。不寫格式限制就跑出這種輸出。
解法
prompt 格式段明寫「禁用『您』改『你』、每封 ≤ 150 字、結尾必須是 XX 一句話」。如果已經跑出 400 字版本，追問「請縮短到 100 字以內，只保留最重要的一個行動句」。

Q1（概念驗證）：以下哪個 prompt 設計最能確保決議清單的格式固定、可直接複製使用？
顯示答案
正確答案：C

A 沒有格式限制，Gemini 輸出不可預測；B 格式讓 Gemini 自己決定，每次可能不一樣；C 任務 + 格式都寫清楚：欄位明確、過濾條件明確，輸出可預測；D 只限定筆數，沒限定欄位，還是會跑出不一樣的格式。
Q2（應用驗證）：你是采蓁（採購），剛用三件套 prompt 跑完供應商議價會議逐字稿，Gemini 跑出來的決議清單是 markdown 表格，但要貼進主管要求的 Google Sheets 格式，表格貼上去會跑版。你下一個追問要怎麼寫？
顯示答案要點
直接追問格式：「請把剛才的決議清單改成純文字，每條格式：誰 · 做什麼 · 何時前 · 找誰確認，每條一行，不用 markdown 符號」

加分：能說出「不需要重新貼逐字稿，因為是在同一個對話視窗接著問，Gemini 記得剛才的結果」

加分：能說出「格式調整是三件套的格式件套的功能，改 prompt 的格式段就能控制」
