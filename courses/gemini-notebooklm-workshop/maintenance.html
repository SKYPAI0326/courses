3 個月維護清單

開課後 3 個月內，Gemini / NotebookLM 免費版可能改版（介面 / 上下文上限 / 引用呈現 / 上傳能力），本頁列出對應要更新的講義位置 + 嚴重度 + 更新動作，給業主與設計師對齊用。

使用說明

維護觸發 + 動線

什麼時候觸發維護
業主或學員回報「課程操作步驟跟現在 Gemini / NotebookLM 介面不一樣」/ Google 官方公告免費版能力調整 / 設計師月度自查發現截圖過時。觸發後依下方表格找對應頁面 + 動作，由業主發信通知學員、設計師修網頁。

改版類型影響頁面 + 區塊更新動作嚴重度
Gemini 免費版 32K 上下文上限變了 CH1-4 Section 03「Gemini 32K 限制」tool-card / Section 02 第 1-3 步「分批貼 wrapper」說明 / CH1-3 第 1 步「新鮮度核 wrapper」 更新 token 限制具體數字（例：32K → 1M）、檢查 350 筆是否還需要分批、若不需分批則改寫「分批貼 wrapper」概念 高

NotebookLM source 上限或上傳檔型改變 CH1-2 Section 02 第 1 步「建 notebook + 上傳 20 sources」/ Section 06 試跑包「20 份 PDF」 更新「免費版上限是 50 sources」說明、更新允許上傳的檔型清單（若 csv / md 改了） 高

維護表 · 低嚴重度

學員自己看得出的小調整（不強制修）

改版類型影響頁面 + 區塊更新動作嚴重度
UI 文案微調（按鈕字 / 提示語） operational dialog-box 內 UI 引用 學員能自己對應，不強制修。設計師月度自查時順手改 低
競品工具新增 / 退出 CH0 Section 02 雙工具速查表 / CH0 Section 03 Demo 對比 本課指定 Gemini + NotebookLM 雙工具，競品變化不影響課程。除非業主要求加入，否則不修 低
月度自查節奏

設計師每月做的事

  • 每月 1 日：自跑一次 CH1-4 主版 demo（怡君電商月度檢討），確認 dialog-box prompt 仍能跑出 expected-output 預期結構
  • 每月 1 日：自跑一次 CH1-2 主版 demo（怡君電商規範查詢），確認 NotebookLM 引用標號格式未變
  • 每月 1 日：自跑一次 CH2-1 兩工具聯動，確認「複製 NotebookLM 取證結果 → Gemini 整合」工作流仍 work
  • 每月 15 日：檢視業主 / 學員回報，依本頁維護表分嚴重度處理
  • 每月 30 日：若有改動 → 跑 lint 全課 + 重建 search-index + 通知業主發信給學員
→
3 個月期滿後：與業主討論是否續約。續約則每季自查一次。不續約則本頁封存，業主自行維護。
