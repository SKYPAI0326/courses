## 3.1 No Code / Low Code 概念

### A. 基本概念 ⭐🔧

**No Code（無程式碼）**：
> 透過圖形化介面（UI）和拖放操作，讓使用者**無需編寫程式碼**即可開發應用程式。
> 主要面向：**非技術背景使用者**（樣題第 8 題）。
> 適用：快速原型設計、簡易工作流程、內部工具。

**Low Code（低程式碼）**：
> 結合視覺化介面與**少量程式碼**擴充功能，讓開發者能在視覺化基礎上實現複雜邏輯。
> 主要面向：具基本技術背景的開發者。
> 適用：需高自訂性、複雜邏輯、企業級應用。

**樣題第 5 題重點**：Low Code 平台適合開發**靈活且可擴展**的解決方案。

**樣題第 1 題重點**：No Code 主要靠 UI、Low Code 結合簡單程式 + 視覺化工具。

**對應關係**（必背）：
- **No Code → 非技術人員 → 快速原型**
- **Low Code → 有技術背景 → 複雜邏輯與整合**

### B. 6 大應用領域 📌

樣題第 7 題：**自動化生病歷生成 屬於醫療領域**（金融業 = 風險、投資、客服）

| 領域 | 生成式 AI 應用 |
|---|---|
| 醫療保健 | 藥物開發、個人化治療方案、病歷生成、輔助診斷 |
| 製造業 | 產品設計、生產流程優化 |
| 金融業 | 風險評估、投資組合優化、自動化合規監管 |
| 零售業 | 個人化行銷、庫存管理、客戶體驗優化 |
| 教育 | 自動教材生成、個人化學習路徑、智慧評量 |
| 客戶服務 | 虛擬智慧客服、自動回應生成、客訴分析 |

### C. 6 大選擇與評估因素 ⭐

樣題第 10 題（樣題第 10 問新版）：**功能與擴展性 = 平台的技術整合能力與自訂性**

1. **目標用戶與技術需求**（NC 給非技術 / LC 給有技術）
2. **功能與擴展性**（系統整合 CRM/ERP、自訂性）⭐
3. **安全性與合規性**（資料加密、權限管理）
4. **成本與效益**（TCO 總擁有成本、ROI 投資回報率）
5. **技術支援與社群資源**（文件、論壇、客服）
6. **市場評價與案例**

### D. 4 大挑戰 ⭐

1. **模型準確性與可靠性**：輸出錯誤風險、驗證機制不足 → 多層次驗證 + 人工審核
2. **資料隱私與安全性**：敏感資料洩漏 → 加密、匿名化、權限控管（GDPR/CCPA）
3. **道德與倫理風險**：虛假資訊、偏見歧視 → 倫理準則 + 公平性檢測工具
4. **技術整合與平台適配性**：模型整合難、效能優化挑戰 → 開放式 API 平台 + 雲端

### E. AI 民主化 ⭐🔧

> AI Democratization：將 AI 技術從少數專家擴展到更廣泛社會層面，使非技術背景人士也能參與 AI 開發

**「市民開發者」**（Citizen Developer）：用 NC/LC 工具自動化日常工作的非工程師
**5 大影響**：降低技術門檻、擴大可訪問性、推動跨領域創新、強化普及應用、啟動全球性影響

**樣題第 6 題重點**：NC/LC 推動 AI 民主化的挑戰**不包括**「提供昂貴的開發工具」（D 是錯的）

## 3.2 生成式 AI 應用領域與工具使用

### A. 生成式 AI 基本概念 ⭐🔧

> Generative AI = 透過深度學習與大數據集訓練生成新內容（非僅分析或辨識現有數據）
> 通常基於 GAN、VAE、Transformer 架構

**3 大技術架構**：
1. **深度學習網路**：多層神經網路 + **注意力機制**（Self-Attention）
2. **訓練數據處理**：清洗（Data Cleaning）、標記化（Tokenization）、向量化（Vectorization）
3. **推理機制** ⭐
   - **溫度（Temperature）**：低 → 保守可預測；高 → 創意多樣（樣題第 33 題：要邏輯清晰應設 0.1-0.3）
   - **Top-k Sampling**：選機率最高的前 k 個
   - **Nucleus Sampling**（Top-p）：累積機率達閾值的選項

### B. 生成式 AI 技術突破 ⭐

- **GAN**：高品質圖像生成
- **VAE / Flow-based**：穩定性 + 機率生成
- **Transformer + Self-attention**：GPT 系列、DALL-E、Stable Diffusion
- **預訓練 + 微調 + RLHF**（人類回饋強化學習）
- **vLLM / 模型壓縮 / 量化**：邊緣計算與行動裝置

### C. 工具發展趨勢 ⭐

- **專業化垂直整合**（法律/醫療/金融/教育）
- **多模態整合**（DALL-E、Midjourney、Whisper）
- **AI 即服務**（AIaaS：OpenAI API、Hugging Face Spaces）
- **個人化 + 可控性**（Fine-tuning、Prompt Engineering）
- **協作與即時回饋**
- **安全與隱私保護**（本地部署、加密）
- **效能優化**（高效 Transformer、Diffusion 改良）
- **生態系建構**（API、外掛、第三方整合）
- **道德與法律規範**

### D. 5 大應用領域

| 領域 | 應用 |
|---|---|
| 藝術設計/內容創作 | 數位藝術、時尚設計、動畫、音樂創作 |
| 醫療生技 | 藥物開發、醫學影像、精準醫療、醫學教育 |
| 教育培訓 | 個人化學習、智慧教學助理、互動教材、語言學習 |
| 娛樂媒體 | 遊戲開發、劇本創作、虛擬偶像、音樂與影片生成 |
| 產品設計與製造 | 創新設計、快速原型、模擬測試、供應鏈管理 |

### E. 重點生成式 AI 模型清單 ⭐📌

| 模型 | 類型 | 用途 |
|---|---|---|
| GPT-4 / Claude / Gemini | LLM | 對話、生成文本 |
| DALL-E / Midjourney | Diffusion + Multimodal | 文字→圖像 |
| Stable Diffusion | Diffusion | 開源圖像生成 |
| Whisper | Speech | 語音→文字 |
| StyleGAN | GAN | 高品質人臉/圖像 |
| LLaMA / Mistral | LLM | 開源大語言模型 |

### F. 0 基礎工具教學重點

**ChatGPT / Claude / Gemini 操作**：
- Prompt 撰寫基本（角色設定、任務描述、格式要求、範例）
- 對話式生成（追問、迭代）
- 注意：**幻覺（Hallucination）**問題（樣題第 30 題：模型生成虛構但語句通順 → 幻覺現象）

**圖像生成（DALL-E、Midjourney）**：
- 提示詞結構（主題 + 風格 + 細節 + 反向提示）
- 解析度、版權、商用授權考量

**多模態應用**（樣題第 20 題）：
- 客服處理文字 + 調用圖片影片回覆 → **多模態模型**

## 3.3 生成式 AI 導入評估規劃

### A. 4 階段導入流程 ⭐📌

**樣題第 29 題核心：「上線部署→模型監控與優化→AI 價值擴散」順序為 BCA。**

```
(1) 準備階段（挑選 AI 應用方案）
   ├ 掌握企業課題（設立目標、選定範圍、提取改善項目）
   ├ 檢視 AI 方案與企業資源（必要性、方案選擇、資源檢視）
   └ 確定應用領域優先順序（容易程度、優先順序）

(2) 設計階段（確任 AI 生成規格）
   ├ 確認導入最終目標（KPI、可量化標準）
   ├ 確認數據狀態（品質、來源多樣性）
   ├ 確認使用情境
   └ 估算 AI 導入成本（軟體、硬體、雲端、人才培訓）

(3) 驗證 POC（驗證 AI 效果）
   ├ 模型開發至部署（架構選擇、訓練、優化）
   ├ 導入實務運作流程（業務整合、試點、員工培訓、改進）
   └ ROI 計算（NPV、IRR、敏感性分析）

(4) 實施 / 營運（持續發揮價值）
   ├ 專案落地（內部專家團隊）
   ├ 模型監控與重新訓練（Data Drift 偵測、自動化重訓 Pipeline）
   └ AI 價值擴散（單點 → 全企業）
```

### B. 準備階段 — 掌握企業課題 ⭐

樣題第 1 題：導入第一步 = **設定可量化、可追蹤的經營目標**
樣題第 6 題：構想階段應**明確經營目標與核心價值**
樣題第 35 題：客服系統第一階段任務 = **明確定義導入目標與關鍵績效指標**

### C. POC（概念驗證） ⭐

> Proof of Concept = 小規模實驗測試，評估模型效能、與現有作業融合度

**5 大評估方法**：
1. **設定驗證基準**：BLEU、ROUGE、Perplexity 等指標
2. **自動化效能測試與批量測試**
3. **主觀人工檢查**（領域專家 + 用戶回饋）
4. **壓力測試與邊界檢查**
5. **敏感性與公平性檢查**（樣題第 32 題：偏差檢測 = Bias Detection）
6. **A/B 測試與線上驗證**

### D. ROI 計算 ⭐

- **TCO**（Total Cost of Ownership）：軟硬體 + 人力培訓 + 維護
- **ROI**（Return on Investment）：營收增長、客戶滿意度、市場擴展
- **NPV / IRR**：財務淨現值、內部報酬率
- **敏感性分析**：模擬市場條件對方案收益影響

### E. 風險管理 6 大策略 ⭐📌

樣題第 27 題：**外包給第三方 = 風險轉移**

| 策略 | 適用情境 |
|---|---|
| **風險溯源** | 數據合法性、可解釋 AI 透明化 |
| **風險文化** | 員工教育、報告機制 |
| **風險接受** | 影響有限或無法避免 → 制定應急計畫 |
| **風險緩解** | 數據管理 + 內容審查 + 動態調整 |
| **風險迴避** | 技術不成熟或損害過大 → 暫緩開發 |
| **風險轉移** | 購買保險或外包第三方 |

### F. 5 大常見風險識別 ⭐📌

樣題第 28 題：**倫理風險 = AI 生成內容帶有偏見或歧視**

1. **訓練數據洩漏**：模型可能無意重現敏感資訊
2. **反向工程**：分析模型行為推斷參數
3. **提示詞攻擊**：誘導模型洩露訓練數據
4. **對抗性攻擊**：誘導錯誤輸出
5. **倫理偏見**：性別/種族/文化偏見

### G. 防範措施

- 身份驗證與授權機制
- 加密技術（傳輸與儲存）
- **差分隱私**（Differential Privacy） ⭐🔧
- 零信任架構（Zero Trust Architecture，樣題第 31 題）
- AI 內容標示「此內容由 AI 生成」

### H. 法規對應 ⭐

- **GDPR**（歐盟一般資料保護規則）：被遺忘權、數據匿名化要求
- **CCPA**（加州消費者隱私法案）
- 《個人資料保護法》（樣題第 9 題：跨境傳輸限制）
- 智慧財產權：版權侵犯防範
