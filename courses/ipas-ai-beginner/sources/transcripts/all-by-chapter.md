## CH1 人工智慧概念

- **(314p1, line 38-103) AI 三波是「進度」不是「強弱」**：規則式 AI / 機器學習 / 深度學習不是 1 < 2 < 3，第二代到現在還有人用、第一代規則式系統也還有參考價值。它們各有特點—規則式邏輯明確、可解釋性最高（規則人寫的，自己當然知道在幹嘛）；生成式 AI（深度學習主流）反而可解釋性低，「為什麼會這樣判」連工程師都不一定講得出來。
- **(314p1, line 219-275) 專家系統 vs RAG 的關係**：考試常出專家系統。最簡單的對應是現在 Notebook LM 用的 RAG—把已有的資料丟進去，AI 答案就高度 focus 在你給的資料庫裡，能最大限度減少 AI 幻覺（答非所問 / 引用錯資料）。專家系統三件套 = 知識庫 + IF-THEN 推論引擎 + 使用者介面，UI 概念就是現在 ChatGPT/Gemini/Claude 那條輸入框。
- **(314p1, line 380-398) 機器學習 vs 深度學習一句話差異**：差別就在「需不需要人工去標記、設計特徵」這件事。CNN 是深度學習代表性的多層神經網路；機器學習常見應用是信用評分（人工挑特徵），深度學習常見應用是人臉辨識（自動學特徵）。
- **(314p1, line 412-482) 三大領域分法（自製口訣，沒學術依據但好記）**：AI 看懂數據（大數據分析）→ AI 看懂文字（NLP）→ AI 看懂圖片/影片。難度從低到高，文字的難點是上下文承接 + 抓重點誰是主角，這就是 Transformer 注意力機制要解決的；圖片再加一個維度（影片就是連續圖片）。
- **(314p1, line 195-218) AlphaGo 三個月爬到天梯頂的故事**：圍棋是排列組合最多的棋類，AI 做不到的只有「心理博弈」（故意耗對方時間之類），但連這也可以納入勝率學習。AI 算多步永遠贏人類，可以拿來舉例 DL 突破棋類遊戲門檻。
- **(315p1, line 1-16) AI 看懂三件事的順序**：考試題目通常一定要會三大領域分類—資料、文字、圖片，最後一塊（圖片）通常分到 CV / CNN / 生成式 AI 主題裡，前兩塊則散在 ML 和 NLP 範圍。

## CH2 資料處理與分析

- **(314p1, line 547-695) 資料蒐集佔整個 AI 專案 4-6 成的力**：訓練模型時最花時間不是模型本身，是資料蒐集 + 清洗。資料蒐集要 concern 兩個點：(1) 偏差—例：以前西方人臉孔資料集多，亞洲臉孔少，做出來的情緒辨識在亞洲就爛；(2) 代表性—例：銀行升職資料女性少，AI 就會把女性都排除，所以要刻意把女性樣本放多一點、男性壓低一點，達成「資料平衡」。問卷發放的群組也算這個概念—問男生「這個名牌包吸不吸引你」是無效資料。
- **(314p1, line 660-674) 標準化的隱藏陷阱：性別代碼**：登記性別時若把男生 = 1、女生 = 2，沒宣告型別電腦會視為數值，平均出來變成 1.6、1.7—但「沒有人性別叫 1.6 1.7」。資料的標準化格式是非常重要的，不只有日期到秒/到分這種格式問題，類別型欄位也要避免被誤算成數值。
- **(314p1, line 717-732) 重複值用 UUID 防呆**：UUID 是「不可能重複的 ID」，常用電腦隨機碼產生，可以幫你檢查有沒有同一筆資料重複進來。異常值一般直接移除（例：體重 800 公斤），或用相形圖看離峰程度判斷。
- **(314p1, line 1410-1500) 個資三層分級實戰**：(1) 直接識別 = 姓名/身份證/手機/Email/地址；(2) 間接識別 = 年齡/性別/職業/郵遞區—但「姓名 + 職業」在台灣多數情況就足以鎖定一個人；(3) 敏感個資 = 醫療紀錄、宗教/政治傾向、生物特徵。台灣健保資料庫特別龐大（看診頻繁），所以醫療紀錄反向推論身份的風險特別高。
- **(314p1, line 1532-1585) 脫明 vs 假名化 vs 加密 vs 雜湊四層的可逆性**：脫明 = 不可逆刪除（王曉明 → 使用者 A）；假名化 = 用代碼，**有金鑰可以還原**；加密 = 用密碼學算法，「複雜到刪掉一個字都解不出來」；雜湊 = 單向轉換、不可還原但可比對。最容易混的是脫明 vs 假名化—脫明真的把資料砍掉、假名化只是換代號。
- **(314p1, line 1700-1869) 零信任架構（ZTA）三件事**：(1) 身份認證—每次都當外人驗、可能還要 MFA / 二次驗證；(2) 裝置與連線安全—連線裝置本身要安全；(3) 存取權限最小化—學生只能看自己選的課、教授只能看自己開的課。最完整實踐的是金融機構（網銀指紋 / Face ID + 身分證 + 兩種密碼）。
- **(314p1, line 1648-1685) 聯邦式學習的「家庭代工」比喻**：每個人在自己家做完一部分零件（本地訓練），最後上繳總工廠拼裝（模型結合）。原始資料「不動」、只動「訓練好的結果」。常見場景：醫院、銀行、跨國資安。
- **(314p1, line 4040-4144) Z-score 公式 + 為何只記這條**：常態分布 68/95/99% 法則 → Z = (目標值 − 平均) / 標準差。Z 絕對值大於 3 → 非常異常（例：人體溫平均 37.6 度、標準差 1 度，量到 40 度直接觸發）。台灣 PR 值/趴數其實同概念。**老師親口說**：「這是 Z-score 是我覺得唯一你們需要記的數學公式」。
- **(314p1, line 2810-2932) 向量是上百上千維、不只 3D**：高中教三維向量是給人看；電腦做 AI 用的向量上百上千維。貓和狗的向量數值我們不能用「哺乳/四腳/雜食」這些詞去解讀，電腦只是用距離算相似度。應用：文字向量化、影像辨識特徵點、推薦系統、語音音樂分析、SEO（關鍵字向量比對）。
- **(314p1, line 3700-3950) Min-Max 正規化 vs 標準化（Z-score）的差異**：正規化 = 等比例縮放到 0–1；標準化 = 平均值 0、標準差 1，越接近常態分布越好用，可以直接看誰是離峰值/平均/中位。穩健縮放（robust scaling）較少考但會用中位數 + IQR 處理離群值。圖片要降運算量也常做「二階化」—把 RGB 三個數字壓成黑白一個數字，因為 CNN 主要抓輪廓，黑白與否影響不大。

## CH3 機器學習

- **(314p1, line 1975-2200) 監督式/非監督式/強化學習的記憶法**：監督式 = 「有人在看著他」（給 Y 標籤，例：垃圾郵件二分類、房價回歸）。非監督式 = 沒有明確標籤，做分群（例：RFM 八種客群—消費金額/頻率/最近時間，2×2×2）。強化學習 = 策略型、有獎勵分數、最後拿最高分（例：圍棋一步累積到最後勝率）。**重要區分**：分類（有明確類別 = 資源回收、寶特瓶/紙類）vs 分群（看起來像就分一群 = 國際班按長相分歐美 vs 亞洲）—監督式才能做「分類」、非監督式只能做「分群」。
- **(314p1, line 2196-2310) 為何 GPU 取代 CPU 在 AI 場景**：CPU 序列執行（一次一件、作業系統管控用）。GPU 平行多工（同時算很多包），AI 把資料轉成數字後要做大量矩陣運算 → GPU 是最適合的。「機器學習跨到深度學習，最大的轉折就是 GPU 出現」，所以黃仁勳跟 AI 綁定那麼深。TPU 是 Google 自家為 AI 開發的晶片，類似邏輯但比較少人用。
- **(314p1, line 2447-2500) 訓練集 / 測試集 / 驗證集的「課本-習作-考試」比喻**：訓練集 = 課本（70-80%）；測試集 = 老師出的考題（20-30%）；驗證集 = 習作練習，不一定要有。為什麼不能拿全部當訓練集再自己出考題？因為自己出可能會把資料集的規律打亂、變得沒參考性。
- **(314p1, line 2505-2561) 過擬合 = 死讀書 / 欠擬合 = 更爛**：過擬合 = 訓練集表現很好、新資料就爛（記選項排列組合不真理解題目）。欠擬合不是過擬合的反面—**欠擬合是訓練集本身就爛**（「AI 沒有不讀書還考試考很好的情況，要爛就一起爛」）。交叉驗證 = K-fold 平均切，每段輪流當測試，缺點是運算成本 K 倍。
- **(314p1, line 2556-2680) 泛化能力的車牌辨識五因素**：(1) 資料量太少 → 模型死記細節；(2) 資料品質差（磨損嚴重的車牌、車牌邊框）；(3) 模型太複雜 → 過擬合；(4) 沒做正則化；(5) 資料多樣性不足（沒涵蓋軍車、重機四碼、舊式 ABC 開頭）。露天 vs 室內、白天 vs 晚上、起霧、山路水滴—都是泛化能力測試。
- **(314p1, line 2932-3068) 梯度下降的「下山找坑」比喻**：學習率 = 學的速度，太大會跳過整個山谷（彈到很遠，失準），太小會把每個彈珠大小的洞都記下來（學很慢、成本高）。實作技巧：先設適中的學習率，再慢慢往下調。問題：(1) 局部最小值（卡在小坑找不到全局）；(2) 梯度爆炸（權重失控發散）。
- **(314p1, line 3074-3170) CNN 四層的口頭順序**：卷積層（找特徵 = 邊緣/輪廓/材質）→ 池化層（降維 = 留下主要輪廓）→ 全連接層（分類）→ 輸出。卷積層找的「材質」例：毛、鐵、塑膠。貓的辨識會留毛、耳朵、腿長度、臉，把腳掌、肚子、尾巴砍掉降維。
- **(314p1, line 3380-3530) Dropout = 板凳球員都要練**：訓練時刻意關掉一些 feature，讓模型不依賴特定幾個。比喻：帶隊比賽不能只練先發球員，先發出狀況時板凳上去要能打。下次新資料如果有缺值或錯誤，模型才不會崩。Dropout 屬於正則化的一種。
- **(314p2, line 156-265) 混淆矩陣 4 格的中文諧音記法**：T = True / F = False / P = Positive / N = Negative。**準確率（accuracy）** = 預測對的比例，例：100 個人 1 個有病，全猜沒病也有 99% 準確率，但精確率是 0。**精確率（precision）** = 你說 Yes 的有多少真的 Yes（避免誤報）。**召回率（recall）** = 真的 Yes 的有多少被你抓到（避免漏報）。**F1 = 平衡精確 + 召回**。
- **(314p2, line 265-318) 詐騙簡訊偵測的取捨**：FP（誤報詐騙）→ 使用者更小心，沒太大壞處；FN（漏報詐騙）→ 使用者放心點擊、上當。所以詐騙偵測題目要看**召回率**（不能漏）。醫療診斷同樣邏輯—不能漏報病患。
- **(314p2, line 337-405) ROC 曲線面積看法**：曲線下面積 100% = 完美；50% = 全靠猜；0% = 完全反過來（其實只要倒過來解讀就是 100%，所以 0% 通常表示模型定義錯誤）。商用 AI 通常以 80% 為基準往上拉，99% 已經很不錯了，「100% 拿出去討論一定會被打槍」。
- **(314p2, line 580-625) 不平衡資料的判斷準則**：正樣本 200 / 負樣本 9800（2% vs 98%）→ 全猜負樣本準確率就有 98%，但精確率 0%。要解這種題，重點是「少數類別才是任務的重點」—醫療找疾病、金融抓詐欺、製造業找瑕疵、公共安全找異常。看到這四個場景就要警覺要看精確率/召回率而不是準確率。
- **(314p2, line 1018-1095) LIME（局部）vs SHAP（全域）的銀行貸款例子**：LIME = 解釋這「一筆」資料為何被判成這樣（例：某客戶只要月收入再加 X、信用紀錄加 Y、負債比降 Z 就會核准）。SHAP = 計算每個「特徵」對整個模型預測的貢獻度，能講出「在這個模型裡，信用紀錄是最重要的、其次是收入、年齡和負債比」。
- **(315p1, line 254-380) 推論模式四類**：線上推論（即時、低延遲，例：ChatGPT 網頁版、即時影像辨識）、批次推論（一次處理大量，例：每月財報分析）、離線推論（不急著輸出、開發測試常用）、分散式推論（雲端 AI 服務的主流，OpenAI 就是線上 + 分散式）。
- **(408, line 1267-1320) 聯邦學習傳的是「梯度/權重更新」**：再次強調—資料不出本地、只把訓練好的結果（梯度/權重變化）傳到中央做合併。傳統集中式 vs 聯邦式最大差別就是「資料不上傳」這件事，主要避免醫療/金融這類隱私資料的傳遞風險。

## CH4 鑑別式 vs 生成式 AI

- **(314p2, line 3845-3910) GAN「畫家 vs 鑑賞家」的舉例 + 守門員崩潰故事**：生成器一直畫新圖、判別器一直批判，互相督促。但兩個都是 AI 容易「越走越偏」。**啾啾鞋影片可參考**：大學做的足球對抗式 GAN—讓 AI 守門員 vs AI 球員，因為兩 AI 抓的細節太細，到後面球員只要做一些沒意義的抽搐，守門員就直接倒在地上，因為它認不出這是什麼動作就 crash 了。這就是訓練不穩定 / 模式崩潰最直觀的例子。
- **(314p2, line 3917-3950) VAE 的「揉紙球」比喻**：把資料壓縮到潛在空間（揉成紙球放在手上），再展開來重建。重建時會稍微補齊丟失的東西，所以每次出來不太一樣。常見應用：圖片型驗證碼、雜訊類處理。缺點：畫質模糊、不夠真實。
- **(314p2, line 3933-3950) 擴散模型「背單字遮起來再背」比喻**：正向擴散加雜訊（背過後遮住）→ 反向擴散去雜訊（重新背一次）。缺點是計算成本高，但現在多數商用文生圖（DALL·E 2、Stable Diffusion、Midjourney）都用它。
- **(314p2, line 3956-4050) Top-K / Top-P / Temperature 三參數的「行銷 vs 律師」對照**：Top-K = 從最佳前 K 個答案隨機抽，K 越大越多元。Top-P = 抽機率 P% 內的答案。Temperature = 溫度 0–1，越高越「熱情多變」（行銷崗位需要、寫劇本創意），越低越「死板穩定」（律師引條文需要）。重點：「沒有哪個比較好，看你的應用場景」。
- **(408, line 1085-1199) Imagen 模型 + ResNet/SVN 的辨識題重點**：Imagen 是 Google 的擴散模型分支，特點是降維運算（潛在空間做 diffusion，類似 Stable Diffusion）。考題常用的混淆—ResNet 是 CNN 變種（鑑別式）、SVM/SVN 是監督式機器學習（鑑別式），這兩個都不是生成式。生成式三巨頭只記 GAN、VAE、Diffusion。
- **(408, line 1163-1186) CLIP 結合「文字 + 影像」的關鍵**：把文字和圖像對應到「相同的語義空間」—假設文字是一維資料，用文字的方式去堆疊符合圖像資料的算法。也可延伸到音訊。它的功能不是「生成」而是「對齊」，把多模態的向量拉到同一個比對空間。
- **(408, line 1217-1232) Zero-Shot vs Few-Shot Learning**：ZSL = 沒有任何訓練資料下，模型能透過已有知識處理沒見過的新類別/任務。FSL = 在「少量樣本」情況下還能達到不錯表現。差別就在「完全沒看過」vs「看過一點點」。

## CH5 No Code / Low Code 概念

- **(315p1, line 980-1095) Low-code / No-code 平台的本質**：Low-code 用少量程式碼建立工具；No-code 完全不寫程式（圖像化、流程圖、積木排序，例 Scratch）。這兩個東西都是「平台」—**沒有平台這東西多數時候是不能動的**，這是 NC/LC 的根本特性，也是它最大缺點的根源。
- **(315p1, line 1018-1100) NC/LC 的客製化軟體優勢**：餐飲業 POS 系統（iChef / DoDo / 快一點）每家店打卡和出缺席算法都不一樣，沒一個通用系統能完美 fit—這時候 NC/LC 就有優勢，可以完全按自己的規則設計表單請假/打卡系統/自動報表。Gemini 的 Canvas 功能搭配 vibe coding 可以做到中型網站、官網、簡單遊戲的程度。
- **(315p1, line 1095-1170) NC/LC 三大缺點—流量、Token、平台依賴**：(1) **流量費用**：N8N 內容農場真實案例—學員測試期間覺得 OK，試用期一過開出來 6 萬塊台幣/月帳單，「我快吐了」；(2) **Token 計費**：現在多數 AI 工具計費都是按 Token，指令越複雜消耗越大；(3) **平台依賴**：類比早期 WordPress 一更新就把熟手的網站全打爛—平台一變你的開發模式就要跟著變。實務建議：「**內部使用完全 OK，但要對外商用、給外部顧客的話不建議**」。
- **(315p1, line 1218-1320) 平台鎖定三層**：(1) 技術鎖定—平台 A 的拉法和 B 不一樣；(2) 資料鎖定—格式/儲存方式不相容，搬遷要重新設計資料庫；(3) 生態鎖定—外掛功能（購物車、扣款、刷卡）每家平台接口不同，搬家「就像 110 家電全部要換 220 插頭」。
- **(315p1, line 1320-1465) API = 餐廳服務生比喻**：客人（client）目標是「得到服務」，廚房（server）目標是「提供服務」，**服務生就是 API**—點餐單從客人傳到廚房、菜從廚房傳到客人。API 不做運算、只做資料傳遞，「就像插座，電是電，插座是插座，方便你把家電插上去」。串接 ChatGPT API 的真實意義 = 你網頁某個按鈕按下去，把資料用 API 丟給 ChatGPT，ChatGPT 處理完再用 API 回傳到你網頁。
- **(315p1, line 1465-1510) 容器化 = 程式 + 環境一起打包**：寫個翻譯程式，丟給人家不能只丟程式碼，還要丟需要的語言文本（埃及文要丟埃及文文本）。容器化就是把這些必要環境綁在一起，能在手機/平板/筆電到處跑。優點是跨平台運行，缺點是需要規劃打包邏輯。

- **(315p1, line 145-170) RAG 兩個器：檢索器 + 生成器**：檢索器去資料庫撈相關資料 → 生成器用 LLM 寫答案。比喻：「RAG 就是多帶一本工作指南書在身上，做事前要先翻書理解一下，再從 LLM 模型把答案吐出來」。

- **(315p2, line 1080-1235) 系統性防禦五原則**：(1) **最小權限原則**—AI 助手只需讀取就只給讀取，不開寫入/刪除（「警衛拿到 A 樓鑰匙也進不了 BC 樓」）；(2) **人在迴圈內**—電商 AI 助理收到「下架所有紅洋裝」要產報表給人類審核，不直接執行；vs **人在迴圈上** = 人不參與執行但能監看，自駕車就是典型，不能每動作都問人；(3) 風險評估—串金流就要算最壞情況可被刷光多少；(4) 完整 LOG + 即時告警；(5) UX 設計避免使用者輸入不該輸入的東西。
- **(315p2, line 1230-1380) 提示模板 vs 系統提示 vs 角色提示三層**：系統提示 = 模型最上層、AI 的「人設」（例：ChatGPT 預設是 AI 助手）；角色提示 = 你下的「你現在是李白，請用李白口吻聊」；提示模板 = 結構化、可重複套用的指令格式（要它輸出的格式：條列、圖表、字數限制）。多數線上 AI 工具的角色提示是可以蓋掉系統的。
- **(315p2, line 1373-1430) Self-prompt（自動提示生成）的實務技巧**：你打「幫我寫一封行銷信」太籠統，好的 AI 會幫你預設填滿提示詞。「你下指令越詳細越好；建議斷句要明確—逗點、句點不要全用空白鍵；最好的習慣是 AI 給你滿意的答案後，回問它『下次想要這個答案我應該打什麼提示詞』，把回的版本記下來，下次直接複製改寫」。
- **(315p2, line 1530-1565) 否定詞的陷阱（生圖必踩）**：「給我一隻沒有尾巴的狗」—AI 會給你一般的狗（看到的重點是「尾巴」，不是「沒有」）。對策：換成沒有反義詞的正面詞—「禿頭和尚」取代「沒有頭髮的和尚」、「只有一顆眼睛」取代「少一顆眼睛」。英文的 without 也常被忽略。
- **(315p2, line 1567-1650) 短期 vs 長期記憶**：短期 = 這個對話內的記憶（叫它修改 X，後面都記得 X 的長相）。長期 = 跨對話記住偏好（名字、職業、科系、過去問過什麼）。老師個人例：玩 3D 列印透光浮雕一陣子，之後問任何 3D 列印問題它都會問「你是要做透光浮雕嗎？」—**有時候你會煩，但實際上 AI 在你沒注意的地方也是越來越貼近你喜歡的格式**（例：自己愛 Notion 條列格式，AI 後來不用特別講就直接給）。
- **(315p2, line 184-380) AI Agent 四模組 + 三類型**：四模組 = 感知（input：感測器/視覺/文件理解）+ 記憶（儲存經驗、使用者偏好）+ 推理（思考策略）+ 動作（output：可控制的權限）。三類型 = (1) 反射型（即時偵測 + 條件觸發，例：掃地機器人避障）；(2) 模型型（有預訓練知識、看到結果就執行，例：特斯拉自駕、自動停車）；(3) 學習型（自主學習、適應環境變化，例：對話型教學機器人）。
- **(315p2, line 380-470) ERP vs CRM 在 AI 應用**：CRM = 對外（顧客關係，例：美髮店記錄客戶適合的剪髮週期、牙醫每半年自動電話通知洗牙）；ERP = 對內（內部營運，員工排班、毛巾庫存、消毒器具管理）。AI 介入後可全自動發 Email/簡訊/語音電話，把行政人員手動工作變成自動化、把行政流程精簡掉，「**通常不會省略研發、客服和顧客關係維護**」。
- **(315p2, line 470-565) Workflow 三層次**：(1) 自動執行重複動作；(2) 結合 AI 判斷（例：Email 分類）；(3) 跨系統協作（例：採購部門 AI 自動看 ERP 庫存 → 自動寫信叫貨）。最後人的角色只剩「在叫貨單簽名」。

## CH7 生成式 AI 導入評估規劃

- **(314p1, line 1685-1850) 企業資安 vs AI 資安的差異**：AI 資安最怕的不是 AI 模型被操控，是**資料外洩**。AI 模型訓練要海量資料、可能涉及上萬上億人的個資，這些資料一旦外洩價值很高。補習班例：買得到「學生」電話比買「全部人」電話轉換率高百倍—**對的資料對商業上面就是非常非常有價值的**。
- **(314p2, line 1100-1145) AI 風險治理四原則**：(1) 整合性—風險管理要融入企業治理；(2) 比例性—風險管理強度依 AI 影響力決定（手機全權限 AI 自然是高比例）；(3) 動態性—AI 模型存活週期約一年，要持續監視；(4) 透明性—以季/月/年為單位做火眼吸取，公開決策過程。
- **(314p2, line 1320-1390) 「為 AI 而 AI」的真實警告**：美國去年數據—**真正打算或有導入 AI 的企業 60-70%、真正落地的只有 9.2%**。多數公司是「老闆要求做 AI」但想不清楚 AI 到底要解決什麼問題。AI 導入四階段必須走：(1) 問題定義；(2) 資料盤點；(3) 模型設計；(4) 上線 + 持續追蹤。
- **(314p2, line 1485-1525) 「考古題 5 年後失效」案例**：你今年把哈佛考古題丟進 NotebookLM 通過考試，5 年後學弟用同樣的工具卻沒過—因為**他少了中間 5 年的考題趨勢**，AI 模型對時代變遷敏感、不像簡報是一次性產物。「AI 專案不會是一次型專案」。
- **(314p2, line 1830-1900) KPI 質性 vs 量性的雙例**：以「ChatGPT 教英文」為例—質性 KPI = 「降低學生不敢開口的可能性」（一句抽象話）；量性 KPI = 「每週讓一個學生練到 100 個新單字」（可量化）。同個專案兩種 KPI 並存才完整。
- **(314p2, line 2178-2400) ROI vs TCO**：ROI（投資報酬率）關注「丟進去的錢能不能賺回來」；TCO（持有總成本）算開發 + 維運 + 應用 + 合規授權。多數人覺得 ROI 重要但 TCO 也很大—「**不同團隊在不同面向會花到的錢不一樣**，有的公司專養某類人才這塊就便宜」。
- **(314p2, line 2270-2400) AI 失敗的「水土不服」原因**：實務接案時最常遇的不是技術成本問題，是**人文治理**—企業內部行政流程或使用工具上「水土不服」、不願嘗試新東西，導致導入效果差。所以 AI 評估要看「能不能整合到既有系統架構」，不只看技術可不可行。
- **(314p2, line 2092-2110) MLOps 四目標**：自動化 + 可重複 + 持續自我學習 + 可監控/回饋。**核心精神**：人力介入只應該介入在「執行面」，daily 維運要自動化交給機器。
- **(315p1, line 1517-1605) 雲/邊/HPC 三架構**：(1) 雲端運算 = 資料 + 模型放伺服器，主流（GCP/AWS）；(2) 邊緣運算 = 末端裝置運算，延遲低（自駕車、IoT），缺點是受限算力 + 更新困難（例：虎頭山自駕公車要本地端更新）；(3) 高效能運算（HPC）= 超級電腦，做物理模擬、數位孿生—老師親身故事：物理系教授論文要排隊跑超級電腦兩個月才拿到數據。
- **(315p1, line 600-1010) AI 民主化的「爽點 vs 痛點」推廣邏輯**：技術是雲端運算 + 開源生態 + 介面簡化（NC/LC、vibe coding）+ 教育推廣四步。最有效的傳播方式是讓人感受**爽點**（例：蝦皮、Foodpanda 一開始免費讓你狂用，習慣後加運費還是繼續用），痛點就是「沒這個會麻煩」的感受。
- **(315p1, line 800-980) AI 民主化五大挑戰**：(1) 技術落差（「全球自認懂 AI 的只 10%」）；(2) 偏鄉/弱勢設備網路差（中華電信在大學辦樂齡學堂教長輩 P 圖、生圖）；(3) 誤用 + 偏見（同樣事實只要換角度、調順序就立場不同）；(4) 資料隱私；(5) 依賴性（「你已經多久沒自己做過簡報了？」連手 key 也可能撈到 AI 寫的內容）。
- **(315p1, line 1810-1890) AI 導入成本佔比口訣**：人力成本 ≈ 4-5 成（最大宗）、算力成本（GPU/雲帳單/外包）、授權成本（軟體/API/外部模型）、維運成本（伺服器電費 + 重新訓練）。多數軟體專案「人力 + 流量」是兩大頭，AI 把流量換成算力。
- **(315p1, line 1910-2085) 優化策略四方向**：(1) 人力—兩個工程師 + 善用 AI 寫 code 取代 5 個工程師（但「不要變成只會 vibe coding 不會 debug 的學弟妹」）；(2) 算力—模型訓練不要追求完美、學習率適中即可，混合雲；(3) 授權—開源框架便宜但有資安風險（例：XAMPP 被攻破，所有用 XAMPP 的網站連帶遭殃）；(4) 維運—自動備份/歸檔。
- **(315p1, line 2086-2270) AI 導入成效四類**：技術績效（正確率）、營運效率（省時間/省人力）、顧客價值（質性—溫度感不能丟）、財務績效（省錢/賺錢）。**重要警告**：「AI 客服太有效率沒溫度反而讓客人有無名火」—顧客價值這塊跟 AI 是矛盾的，「不要那麼像 AI、不要那麼像 AI」是常見要求。日本是極端例—一整天可以完全不跟人類交流、不點頭也能完成消費。
- **(315p1, line 2218-2310) POC vs Pilot 兩階段**：POC（概念驗證）= 技術可行性，「會不會動」；Pilot（試點/試行）= 實際價值 + 可用性，「動了但客人喜不喜歡」。例：地獄廚房同桌不同料理時間要同步上菜（龍蝦 20 分 + 牛排 15 分，5 分後牛排才下鍋）—這就是 Pilot 階段測試出的流程。**所有非 AI 的服務開發也應該有這兩階段**。
- **(315p1, line 2614-2745) 歐盟 AIA 四級風險清單細節**：禁止使用 = 違反人權的操作（Social Credit、潛意識操作—**演算法社群媒體某方面也碰到這層**）；高風險 = 信用評分/醫療診斷/教育評量/升職系統—**台灣繁星玉田系統其實也算高風險，只是它沒那麼 AI、是大數據跑的**；有限風險 = 聊天機器人（誤導但不直接決策）；最低風險 = 過濾器/遊戲。**Doom 遊戲是純 AI 即時生成的特別例**—每幀畫面都 AI 生、地圖永遠不一樣。
- **(315p1, line 2740-2900) GDPR 七原則**：(1) 合法透明公平；(2) 目的限制（**不能二次運用**—點餐系統的客戶資料不能搬去做旅遊 App，即使可能讓客戶體驗更好）；(3) 資料最小化（同 ZTA 概念—吃個飯不需要家裡地址）；(4) 準確性（要可改但**不能讓使用者隨意改生日領禮券**，要客服 + 證件審核）；(5) 保存期限限制（被遺忘權）；(6) 完整性與保密性；(7) 責任制。**七樣缺一不可、每個都很重要**。
- **(315p1, line 2305-2540) AI 著作權四風險**：(1) 著作權—訓練資料來源是否有版權（例：古阿莫解說電影被告，比電影上映還早出來）；(2) 資料隱私—AI 把訓練時的個人資料當「前世記憶」吐出來，「孟婆湯沒喝乾淨」是完全沒辯解空間的風險；(3) 授權與歸屬—使用者用 AI 工具賺很多錢，工具開發商反過來主張分潤；(4) 責任歸屬—AI 教你做炸藥你照做出事，是你的責任不是 AI 的（「不然化學老師也要被抓」）。
- **(315p1, line 2538-2615) 三種授權模式**：(1) 開源許可—自由使用，附上原作者；(2) 開源衍生注作權—生成作品歸你；(3) 自定義限制授權—暴雪 Dota 故事（Dota 跑去做 Dota2 後暴雪加條款「你做出來的遊戲都算我的」）。
- **(315p2, line 1-130) 風險管理四方向**：(1) 風險轉移—轉嫁第三方（保險、雲端服務商擔保）；(2) 風險接受—小錯誤忽略，「越描越黑」就不處理；(3) 風險緩解—降低發生機率/衝擊，AI 護欄就是這層；(4) 風險迴避—直接停止/換策略（例：肖像權問題就直接買開源資料）。
- **(315p2, line 130-180) 各國 AI 法規重點對比**：歐盟 = 透明性 + 可解釋性（嚴格審查制）；美國 = 市場導向 + 自願性（供需原則 + 資料隱私）；台灣 = 數據保護 / 個資為主，透明性反而比較少。
- **(315p2, line 635-865) 資料譜系（data lineage）的「碳足跡」比喻**：追蹤資料的「Effect / When / Where / Who / Why / How」六件事 + 三階段（原始來源 → 處理轉換 → 模型應用）。商業效益：(1) 降低合規風險（金融業要證明信用評分模型沒歧視，全靠這套留檔）；(2) 加速 debug；(3) 跨團隊協作（**「工程師不喜歡看別人寫的、寧可重寫一個新的」**—所以紀錄越完整越好）；(4) 建立顧客信任。
- **(408, line 348-435) 災害應變/氣象預報的 AI 應用**：救災 = 影像辨識 + 災情監測（水量/降雨量感測 + 即時判斷誰要優先驅離）。氣象 = 路徑預測（以前手算颱風動態現在 AI）。觀測站建置本身不是 AI 應用，但資料收集 + 解讀是。
- **(408, line 395-450) 無人機 + AI 救災 + 火災預防**：無人機核心永遠是「拍攝 + 回傳影像」，不管打仗、救災、農業都是收集資訊為主。AI 在救災 = 框出哪裡有人；火災預防 = 烟霧檢測 + 溫度感測 + 影像分析（不同鏡頭下烟霧長相）。
- **(408, line 451-490) 媒體識讀的 AI 角色 + 假新聞陷阱**：「現在假新聞**大宗都是偏頗角度 + 惡意新聞**而不是完全造假」—講話節目愛引用國外政治名嘴 X 或 FB 發文當作真實新聞來源。AI 也常基於網路抓資料，所以 AI 也會被同樣污染。**媒體識讀必須內化成自己的能力**。
- **(408, line 491-545) 智慧農業 AI 應用 + 番茄園自動養殖案例**：番茄園機械手臂 + 各種感測器讓 AI 完全獨立養活。供應鏈預測（市場波動、配送）也是大宗應用。
- **(408, line 547-580) 漁業 + 食品工廠 AI 案例**：漁業 = 影像辨識魚群種類數量。麵包店電子化困境—生鮮類沒條碼很難盤點，多靠收銀員體感。洋芋片工廠真實 YouTube 案例—馬鈴薯削皮前 AI 影像偵測發芽（中毒不能吃），偵測到後叫聲提醒員工或機械手臂吹/吸/夾出來。
- **(408, line 660-720) 物流 AI 路線優化**：「物流司機收到包裹但收件人不在家」是常見問題，AI Agent 可以即時調整路線、跳到下一家。送物流的個人經歷分享：自己送、自己聯絡、放哪裡、爬樓梯、簽名、貨到付款收錢「司機很累」。
- **(408, line 1042-1085) 智慧製造 + 數位孿生**：自動化 + 瑕疵檢測。**數位孿生 = 把生產線數據化做模擬**，跟做物理實驗類似（試錯成本下降）。火箭故事—不會只蓋一台、會蓋兩台一模一樣的，「上去如果壞了哪修，所以用地球上那一台模擬」，這就是孿生實務。
- **(408, line 1268-1320) 聯邦學習 vs 集中式訓練（再強調）**：差別就在「資料不上傳」這件事。集中式有傳遞風險，聯邦學習資料留本地、只傳訓練結果（梯度/權重變化）。
- **(408, line 1320-1380) 被遺忘權的「停卡」例子**：「有些信用卡是要收年費、有些戶頭要持續交流，有的人很久沒用就主動要求關掉、刪除」—這是被遺忘權的日常實踐，使用者有權要求企業刪除自己的資料。
- **(408, line 1380-1410) 最小權限原則的「人資/財務分區」例子**：公司 ERP 系統，人資只能看人資那塊、財務只能看財務那塊、不能互看，是 ZTA 概念在企業內部管理的延伸。
- **(408, line 1382-1407) SHAP（全模型）vs LIME（局部）再次釐清**：SHAP 判斷「整個模型的可解釋性」（全部特徵對模型預測的貢獻）、LIME 判斷「為什麼此時此刻答這個答案」（局部 / 單筆）。這兩個都「跟模型不可置」（model-agnostic）—可以用在任何模型上。

## CH8 中級延伸（量化 / LoRA / 蒸餾 / Diffusion 細節）

- **(408, line 1085-1199) Imagen 模型細節**：Google 推出，本質是 **降維運算的擴散模型**（Latent Diffusion 變種）。它不是直接在像素空間做 diffusion 而是在低維潛在空間，這是它跟原生擴散模型最大差別—與 Stable Diffusion 同思路但 Google 自己訓練。
- **(314p1, line 2940-3068) 學習率（learning rate）詳細圖示**：學習率太低 → 完全貼合曲線但每個箭頭很短、學很慢；學習率適中 → 箭頭數量約一半、可以包住線；學習率太大 → 瘋狂跳、四個反彈點什麼都學不到。實作上一律先設稍大、再慢慢降。常見問題：局部最小值 + 梯度爆炸（權重失控）。
- **(314p1, line 3500-3570) 正則化的「懲罰項」核心思想**：增加懲罰項或弱化獎勵項，限制每個 feature 的權重大小，「不是單純記住、是要找出整個規律」。AI 模型訓練本質是「**學規則不是學數據**」，這兩件事差非常多。
- **(314p1, line 3700-3850) PCA 的繩索比喻**：把每個向量當成繩子，PCA 就像把一把繩子擰成新的大繩子（編織）。它不是把資料刪掉，是把幾百個相似定位的特徵合成新特徵。應用：降維 + 視覺化（高維資料投影到 2D 才能畫圖）。
- **(314p2, line 1850-1950) MLOps 四工具的目的**：(1) 追蹤資料版本（**比追蹤模型版本更重要**）；(2) 自動訓練測試部署（鑑別型 AI 遇例外條件要自動存下來，下次回頭學進去—類似漁市場老闆看到新魚不會明天忘）；(3) 自動偵測例外條件 + 收集回饋；(4) 統一平台 + 持續監控 + 版本管理。
- **(315p2, line 660-724) 資料生命週期三階段詳解**：(1) 原始來源（從哪抓的決定品質）；(2) 處理轉換程（記錄所有改變的內容 + 步驟，「這份資料是經過哪些步驟變成這樣」）；(3) 模型應用程（資料用於哪個版本的模型訓練）。**核心兩記**：「資料變成什麼」+「為什麼/怎麼變的」。
- **(408, line 1075-1085) 數位孿生（Digital Twin）核心**：把實體生產線/設備全部數據化讓 AI 模擬，「此時此刻這樣設計、最後產量各環節數值會是怎樣」。比起真的建設便宜很多，跟做物理實驗本質一樣。
- **(408, line 1210-1235) ZSL / FSL 細節**：Zero-shot 重點是「沒見過的新類別」也能處理（靠語義對齊）；Few-shot 重點是「少量樣本」就能達到不錯表現。
- **(408, line 1265-1280) Transformer 的注意力機制 = 唯一識別重點**：「Transformer 看到注意力機制就直接綁，沒有別的」。
