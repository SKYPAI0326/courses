## CH2 資料處理與分析

- **(line 105-128) SHAP 抓「邊際貢獻度」的記法**：考試只要看到題目寫「邊際貢獻度」「每個特徵」「整體模型」就直接綁 SHAP；看到「單一筆預測」「局部解釋」綁 LIME。這兩個容易混。SHAP 算的是整個模型乃至於它所看的特徵，每個特徵的貢獻度多少；LIME 是用簡單的線性模型在「單一回答」這個局部去解釋。
- **(line 144-178) 差分隱私 ε（差分預算）的方向感**：ε 越小 → 注入雜訊越多 → 保護程度越高，但資料統計可用性會下降。ε 沒有負數，正常範圍 0.1~10 之間，不會到 100。法規沒有強制規定 ε 數值。
- **(line 727-766) Human in the loop vs Human on the loop**：歐盟規範要求人工介入審查權利時，常考 human in the loop（人在迴圈內）—AI 跟人一起合作，在關鍵執行節點人類必須存在、不能繞過。另一種是 human on the loop（人在迴圈上）—人不用跟著做，但要在上面看得到全局，遇到警報才介入。
- **(line 1395-1435) GDPR 四種資料權利的口語對照**：被遺忘權 = 徹底刪除（含備份）；可攜權 = 用戶可把資料打包到另一家公司（保險業/電信業最常見）；限制處理權 = 暫停使用但不刪除；拒絕權 = 拒絕公司用資料於特定用途（例：留電話給學校 → 拒絕學校把號碼給職涯探訪中心）。考試「歐盟 + 刪除」綁被遺忘權。
- **(line 1438-1496) Lakehouse / ELT 為何取代 ETL**：現代資料倉儲傾向「先丟進來再算」（ELT），原因是 (1) 企業集中算力到中央電算中心，方便管理 (2) 移動端普及，本地端算力差異大，用集中運算可拉齊使用者體驗。所以消費模式從「買斷硬體」變成「服務型訂閱」，背後就是這層運算重心轉移。
- **(line 2438-2463) KL 散度為何能偵測資料漂移**：資料漂移比的是「機率分布的形狀差異」，不是單點距離。KL 散度與 Wasserstein 距離都在量化兩個分布的形狀。模型部署後把線上輸入分布跟訓練時分布做比較，超過設定閾值就觸發警告。
- **(line 2932-2978) Lakehouse（湖倉一體）的本質**：保留 SQL/ACID 的嚴謹結構化管理能力 + 廉價彈性，套用到非結構化海量資料上。和傳統 SQL 關聯式資料庫差別在於資料是非結構化的；和區塊鏈差別在於區塊鏈追求去中心化、不適合儲存海量資料。
- **(line 3318-3375) 爬蟲 vs robots.txt 的對應**：爬蟲只看網頁的「程式碼結構」，每次抓特定結構的資料。robots.txt 是放在網站根目錄的設定檔，宣告禁止特定 AI 爬蟲。網頁上放紅色警告圖只給人看、把文字轉 PDF 圖片爬蟲還是抓得到，都不是標準解。歐盟版規方向就是 robots.txt + AI clause。

## CH3 機器學習

- **(line 297-378) Focal loss 的考試決策樹**：題目出現「極端不平衡」「樣本少類」「希望模型專注少數類」→ focal loss。MSE 是線性回歸（預測天氣、自然災害、線性股價）；交叉熵是標準分類但不擅長極不平衡（會被多數類帶走）；餘弦相似度是多元特徵向量方向比對，不會用在二分法。
- **(line 1244-1310) 召回率 vs 精確度的「捉放法」記憶法**：高風險疾病早期篩檢、漏診代價高 → 看召回率（不能漏）。捉放法的比喻：從真正有病的人裡面，抓回來幾隻、答對幾個。模型如果只看精確度，會變得保守—怕精確度掉，所以盡量都判沒病。題目中「正常人遠多於感染者」+「漏診代價可怕」→ 召回率。特異度是反過來的召回率（捕捉正常人的能力），考試出現率低。
- **(line 605-634) PDP（部分依賴圖）跟 LIME 不要搞混**：LIME 解釋單筆，SHAP 解釋全部，PDP 是把一個特徵在不同分布上的平均趨勢散成圖，三者用途完全不同。
- **(line 781-808) 概念漂移 vs 資料漂移的區分**：概念漂移 = 特徵與目標的「邏輯關係」改變（市場習慣變、客戶行為變、欺詐手法變）。資料漂移 = 純粹輸入資料的分布改變。第 21 題講「半年後流行趨勢改變」+「點擊率下降」→ 兩者同時發生（資料 + 概念都漂）。
- **(line 251-279) 模式崩潰、災難性遺忘、梯度爆炸的場域對照**：模式崩潰是 GAN 專屬（生成只剩一種結果、訓練失敗）；災難性遺忘是「學新忘舊」，跟增量學習相反；梯度爆炸是訓練初期參數不收斂，數值極端兩極。考試題目場域不同就選對應名詞。
- **(line 211-221) 增量學習為何成為現代 AI 標配**：模型要持續學新知識又不能忘舊。如果不保留這能力，時代一變、更新越多東西模型越亂，所以多數線上 AI 模型都會盡可能保留增量學習機制。
- **(line 916-979) 履歷 AI 偏見的前處理三選一**：性別偏見的修法在「資料前處理」階段最直接的做法是「重新抽樣 + 合成資料平衡」。增加深度反而過擬合、放大偏見；直接刪除女性樣本邏輯上等於把它當離異值，但題目沒說它是離異值；one-hot encoding 只改格式不影響偏差統計分布。
- **(line 1311-1395) 特徵庫（feature store）的職責邊界**：「統一管理 + 共享重用」是核心。它不是 SQL 替代品，不是自動標註工具（標註屬於資料工程），不是模型權重存放處（那是 model registry）。記住：訓練 → 資料 → 特徵 → 模型，這四層是分開的。
- **(line 2070-2105) 公平性指標的三層差異**：人口統計對等（demographic parity）= 不同族群核准比例強制一致（保送名額概念）。均等機會 = 真正會還款的人裡面被核准的機率一致（用結果回推）。考試出現「比例完全相等」「不論種族」就綁 demographic parity。
- **(line 2152-2210) QAT vs PTQ 的訓練時機差別**：PTQ（訓練後量化）= 訓練好直接砍掉精度（四捨五入、無條件捨去），會讓資料間關係亂掉、精度雪崩。QAT（量化感知訓練）= 訓練階段就模擬量化誤差，讓神經網路自己調整權重去適應低精度。邊緣裝置算力差，所以要在訓練時就考量「我未來只能用這種精度」。
- **(line 2542-2589) MoE（混合專家）的核心一句話**：每次推論只透過路由網路啟動最相關的少數專家，其他專家不動。考試直接綁這句話。所謂「專家」是神經網路專家，不是真的找人類專家（那是早期 rule-based 專家系統）。
- **(line 2740-2776) Tensor Core 在做的單一件事**：硬體層級加速「混合精度的矩陣乘加運算」。深度學習的計算瓶頸就在矩陣加減乘除（特別大），CUDA 負責損失函數/激勵函數/各種網路算法，Tensor Core 是專門做矩陣那一層硬體加速。

## CH4 鑑別式 vs 生成式 AI

- **(line 252-260) 模式崩潰專屬於 GAN**：題目場域只要寫「生成只剩一種結果」「訓練失敗的表現」一定是 GAN 的模式崩潰，不會出現在判別模型題。
- **(line 700-722) 缺瑕疵樣本要從無到有「生成」**：「擴充資料集」+「需要多樣性」→ 直接綁生成式（GAN、擴散）。BERT 是處理文字（單純自然語言理解）、強化學習是機器學習層級、語言模型不能生成圖片，這三個跟「視覺生成」題沒關係。
- **(line 533-587) 擴散模型推論慢的本質**：原理是把圖片加噪再慢慢還原，每一次還原難免畫不一樣（比喻：給人塗白幾個地方、要他重畫，每次結果都不同）。所以加速的方向是「優化採樣器演算法」把幾百步壓縮成幾步十步。提示詞長度跟運算量無關 → AI 不會因為提示詞越多就跑越久。
- **(line 1969-2021) Stable Diffusion 為何快**：傳統擴散模型在「像素空間」運算（1000×1000 解析度 × 顏色通道），運算量爆炸。Stable 的關鍵是先把高解析度壓到「潛在空間」（極低維、人難以視覺化），算完再解壓回來。這就是 Latent Diffusion 的本質。
- **(line 1497-1537) FID 用來評估生成圖品質**：透過萃取真實影像 + 生成影像兩邊的特徵分布，計算它們的距離。「越像 = 越接近真實世界 = 多樣性越好」。BLEU 是翻譯指標，MSE 是回歸，IS（題目中第三選項）跟 FID 同類但 FID 比較常考。
- **(line 2042-2065) CLIP 的 contrastive loss 一句話**：把配對正確的圖文向量在空間裡拉近、配對錯誤的推遠。它讓多模態模型看圖看文字都能用同一個向量空間比對。

## CH6 生成式 AI 應用工具

- **(line 419-452) RAG 分塊為何要「重疊滑動視窗」**：embedding 有長度上限，所以要把長文件切成小塊。電腦看不出文件「哪裡是階段」，純隨機抽 10% 關鍵字會把語意關聯性整個打散。重疊滑動視窗的核心是讓上下文不會被切斷。
- **(line 638-680) HNSW 的本質**：「近似最近鄰演算法」，讓不同向量以距離方式找出相似的聚集起來，降低運算速度。K-means 是分群（不是搜尋）；TF-IDF 是關鍵字頻率（不是向量）；決策樹是 ML 分類，三者跟 RAG 向量搜尋無關。
- **(line 808-861) Few-shot vs zero-shot vs 系統提示的回答精度差異**：少量樣本提示主要在改回答格式/語氣（給範例），不會限制底層知識。要避免幻覺、要強制只用上下文回答 → 必須在 system prompt 裡用「上下文邊界限制 + 嚴格約束」（題 22）。「提示工程在工作上的應用價值，可能比這次考試的內容更重要。」
- **(line 1722-1761) 自我一致性提示（Self-Consistency）一句話**：要求模型生成多條不同推理路徑，再讓模型自己挑出一致性最高的那條當最終答案。和負面提示（圖像生成常見）、零樣本提示、角色扮演不同。
- **(line 1768-1806) Reranker 的兩段式結構**：檢索器先用 bi-encoder 快速篩 top 100/20，再丟給 cross-encoder 重排序、最後送生成器。這樣可以「先挑出有答案的、再做重新整理」。
- **(line 2030-2039) 思維樹（ToT）vs 思維鏈（CoT）**：CoT 是單一鏈、中途算錯就全錯。ToT 結合「廣度優先 + 深度優先」搜尋演算法，可以探索多條分支、走死路時回溯，適合複雜多步試錯的數理邏輯題。
- **(line 1862-1885) KV Cache 是什麼**：每生一個 token，傳統做法都要重算前面所有 token 的注意力。KV Cache 把已算過 token 的相對向量存下來，後續只需取出做新計算，不用全部重推。「口頭解釋很單純，就是把東西先存下來。」
- **(line 1638-1694) 稀疏注意力 / 滑動視窗注意力**：傳統注意力是平方級爆炸（10 萬字 vs 1 萬字 → 100 倍時間）。稀疏注意力讓每個 token 只關注鄰近局部，把複雜度降到線性。比喻：第 10 段就管好自己，跟第 1 段有沒有 callback 不用管。
- **(line 3300-3317) System / User / Assistant 三角色的權重順序**：核心人格、安全護欄、整體行為準則一律放 System block，因為它在注意力機制裡被賦予最高權重。「定義你是誰」「絕對不能做什麼」這類指令必須在 System。
- **(line 3186-3262) DSPy 的「自動編譯提示詞」概念**：傳統手寫提示詞換模型就要全部重寫。DSPy 讓開發者只寫主要邏輯，由它的編譯器自動為不同模型搜尋最佳提示策略 + few-shot 範例。
  - 講師業界例：Grok（馬斯克）對版權內容寬容很高，OpenAI Sora 一旦提到品牌名就直接拒絕生成；Sora 才剛跟 200 多個知名 IP 簽授權，最近就把 Sora 關掉。「不知道有沒有要賠錢，這也可能會考。」
- **(line 3478-3527) RAGAS 框架的「答案忠實度」**：判斷生成答案是否完全基於檢索出的文件上下文，也就是「有沒有幻覺」。和「上下文關聯性」（檢索文件是否包含關鍵資訊）、回應速度、查詢-文件向量距離不同。

## CH7 生成式 AI 導入評估規劃

- **(line 105-128) 銀行信用評分的 XAI 需求綁 SHAP**：題目場域是法規審查 + 解釋邊際貢獻度 → SHAP。實務上銀行業導入 AI 都會被法規要求高度可解釋性，所以這類業務場景必考 SHAP/LIME。
- **(line 145-178) 醫療資料 + 差分隱私場景**：醫療機構釋出去識別化病例給研究時，怕成員推斷攻擊（membership inference），導入 ε 控制雜訊量。差分隱私在醫療業 + 法規場景幾乎必考。
- **(line 184-221) 醫院/銀行/高機敏資料優先聯邦學習**：「資料不能離開本地」是聯邦學習的觸發條件。各地醫院、各家銀行、跨國資安資料都會走聯邦學習。每個本地端訓練完後上傳的是「梯度/權重更新」，而不是原始資料 / 加密原始資料 / 完整作業系統印象檔（題 60）。
- **(line 1140-1174) 罕見疾病多家醫院共訓練 → 聯邦學習**：「授予各支法規不能離開各家醫院的網絡」是聯邦學習的標準導入情境。
- **(line 2256-2270) 高風險 AI 進入歐盟市場前的 DPIA**：對使用者權益與自由產生高風險的 AI 系統，歐盟強制要求書面評估程序（資料保護影響評估）。
- **(line 2272-2291) 反事實解釋（counterfactual）的金融場景**：銀行拒絕貸款後告訴客戶「月收入再增 5000 + 信用卡遲繳少一次就會核准」就是 counterfactual。這是 XAI 中對使用者最直觀、最有行動建議的解釋方式，金融業正是最常見的應用場域。
- **(line 2592-2640) 機密運算 / TEE 的「運算中保護」位置**：資料三種狀態 = 傳輸中（HTTPS）、儲存中（加密）、運算中（過去無保護）。雲端最高權限管理員也不能窺探記憶體中原始值 → 必須用 TEE / 機密運算硬體技術。這是 2024-2026 年新興的金融/醫療雲端合規重點。
- **(line 2680-2706) 聯邦學習上傳的是「梯度權重變化」而非原始資料**：再強調一次，聯邦學習的核心精神是資料不動、模型動。「加密後的原始訓練資料」也不傳，因為大多數情況下訓練時希望資料完全不動。

## CH8 中級延伸

- **(line 38-72) LoRA 的 R 值對訓練成本的影響**：LoRA 透過低秩矩陣讓模型微調時只調整少量參數。R 值越小 → 訓練越加速、顯存佔用越低、成本越低，但結果不一定最好。R 值的關鍵是「在成本與效果之間取折衷」。
- **(line 76-104) 模型量化 vs 知識蒸餾 vs 網路剪枝**：
  - 量化 = 把資料形態從 FP32 → INT8（改變權重的數值精度）
  - 剪枝 = 把不重要的神經元或連線刪除（跟資料形態無關）
  - 知識蒸餾 = 用大模型當「老師」訓練小模型「學生」（DeepSeek 早期被認為用蒸餾從現有模型反推）
  - 「權重擴增」這個詞不存在，看到直接刪。
- **(line 533-571) 擴散模型加速採樣的演算法本質**：把「慢慢復原」這段演算法用數學方式壓縮抓在一起算，從幾百步壓到 5-10 步。這就是 LCM / DDIM / DPM-Solver 等採樣器在做的事。
- **(line 419-452) 切塊 chunk 的真實限制**：embedding 模型有 token 上限，這才是切塊的根本原因。電腦無法判斷「文件哪裡才是段落分界」，這就是進階遞迴分塊（題 71）能勝出的原因—它先按 \n、句號、段落符號切，再用語義相似度合併。
- **(line 1697-1721) 資料治理 vs 資料譜系（data lineage）**：題目要追蹤「從哪一筆資料 → 哪些步驟 → 最後流到哪」這種「資料碳足跡」就是資料譜系，不是 ER 圖、不是平均值/標準差統計、不是視覺化儀表板。
- **(line 2107-2150) PPO 在 RLHF 微調的角色**：用「剪裁機制」嚴格限制模型每次更新時不能離原模型太遠 →「不要學太快」、避免極端值衝擊。這是 PPO 跟 logistic regression 等其他方法最大的差異。
- **(line 2796-2825) Informer 用「機率稀疏自注意力」處理超長時序**：用數學推導讓注意力只保留最具代表性的幾個位置，把記憶體複雜度從平方級降到 N log N。其他選項（DP / 完全捨棄注意力 / 時序轉 2D 圖譜）都是錯的。
- **(line 2826-2851) Plan and Solve 解 ReAct 的死循環缺點**：ReAct 容易迷失方向、陷入無限迴圈。Plan and Solve 要求模型先生成完整宏觀計畫，再依序執行每個子任務。「限制只能輸出 100 種制式回答」「凍結權重 + 禁止外部工具」「人類接管鍵盤」都不是 agent 設計初衷。
- **(line 2851-2885) Actor-Critic 框架的分工**：Actor = 策略網路（看當前環境決定要做哪一動）；Critic = 價值網路（評估剛剛做的動作是好還是壞，打分數）。沒有所謂「Actor 預測未來、Critic 記憶歷史」這種拆分。
- **(line 2885-2929) D-Fake 隱形浮水印放在解碼器/潛在空間取樣階段**：必須在生成階段植入特定頻率模式才能抗剪裁與壓縮。降低解析度、JPEG header 寫文字、GPU 風扇轉速指紋都能被輕易破壞或本來就沒有這個說法。
- **(line 2982-3010) DPO 取代 RLHF 的數學優勢**：傳統 RLHF 要先訓練大型獎勵模型 + 用 PPO 不穩定地微調。DPO 用數學推導直接把人類偏好資料轉成交叉熵損失，繞過獎勵模型 + 強化學習步驟。缺點是資料偏頗時會亂掉。
- **(line 3070-3127) Perplexity（困惑度）的方向感**：困惑度越低 → 模型對下一個詞越有信心、機率分布越集中於正確答案。困惑度越高 ≠ 越精確（這是反過來的）。它計算的是機率分布，不是錯誤字詞總數，所以是 0-1 的浮點數。困惑度只用於語言模型，不用於圖像生成。
- **(line 3144-3182) 對抗訓練是「打疫苗」**：對抗樣本攻擊的根本對策是訓練時主動帶入帶有對抗擾動的樣本讓模型學習。把圖片轉黑白二值化只方便運算，不會增強對抗韌性—因為 CNN 抓的是輪廓，黑白與否影響不大。
- **(line 3375-3426) DQN 的核心改良**：傳統 Q-learning 用表格記錄每個狀態的報酬，圍棋/高解析畫面的狀態空間太大、表格存不下。DQN 用神經網路（CNN）近似 Q 函數—直接讀畫面當輸入，輸出每個動作的 Q 值。電腦不會理解「這個畫面裡有敵人/隊友/道具」，它就是一幀一幀記下來。
- **(line 3426-3477) Prefix Tuning 的「黏土」比喻**：凍結 LLM 所有預訓練權重，只在 Transformer 每層的注意力機制裡插入一組可訓練的連續向量。這組向量像「黏土」電住輸出，可以即時用數學公式調整，達到輕量微調目的。和 LoRA 不同的是 Prefix Tuning 完全鎖權重。
- **(line 3551-3572) Mamba / 狀態空間模型（SSM）取代 Transformer 處理長序列**：Transformer 自注意力是平方級爆炸。Mamba 基於狀態空間模型（控制論常見），把資訊壓縮成固定大小的隱藏狀態（類似 RNN 的精神）然後讓時間複雜度降到線性。這是 2024-2026 年取代 Transformer 處理超長上下文的重要新架構。
- **(line 3573-3593) 講師備考心法（轉成 metadata 給其他章）**：「重點不是對多少題、不是寫到滾瓜爛熟。重點是把每個選項的專有名詞跟它對應的技術/概念綁起來。一題四個選項就練到四個關鍵字，這樣最大程度幫到備考。」這條備考策略應放在 CH8 末尾或考前心法區。
