## CH3 機器學習｜傳統演算法（PDF p.4-15）

### IF-THEN 規則
- **白話**：「如果…就…」的條件判斷，最早期的 AI 邏輯
- **舉例**：銀行核卡：IF 月收入>5 萬 AND 信用分>700 THEN 通過
- **章節**：CH1（AI 演進）/ CH3（規則式 vs 機器學習對比）

### 鑑別式 AI（Discriminative AI Model）
- **白話**：學會「分類」與「判斷」的 AI（垃圾信判斷、影像辨識）
- **對比**：跟生成式 AI（會「創作」內容）不同
- **章節**：CH4 鑑別 vs 生成核心對比

### 線性回歸（Linear Regression）
- **白話**：找出資料間「直線關係」，最常見的預測方法
- **舉例**：用坪數預測房價、用廣告費預測銷售額
- **章節**：CH3（監督式學習-回歸） | 樣題：S1-Q20

### SVM（Support Vector Machine 支援向量機）
- **白話**：找出最「公平」的分隔線，把兩類資料分開
- **比喻**：在兩群人中間畫線，線離兩邊最近的人都要等距
- **章節**：CH3（監督式學習-分類）

### 決策樹（Decision Tree）
- **白話**：用「一連串是非題」做判斷，畫出來像一棵樹
- **舉例**：醫生問診流程（有發燒嗎→有咳嗽嗎→有外出史嗎）
- **章節**：CH3（監督式學習-分類） | 樣題：S1-Q17

### 隨機森林（Random Forest）
- **白話**：種一片決策樹，多數決定答案；比單一棵樹穩
- **重點**：可透過特徵重要度輔助「特徵選取」（S1-Q22 解析需強調）
- **章節**：CH3（集成式學習）

### Boosting（提升法）
- **白話**：弱模型接力賽，每個都修正上一個的錯
- **代表**：XGBoost、LightGBM
- **章節**：CH3（集成式學習，補充版）

### 貝氏分類器（Naive Bayes）
- **白話**：用「機率」做分類，假設特徵互相獨立
- **舉例**：垃圾信過濾（出現「免費」「中獎」就提高垃圾機率）
- **章節**：CH3（監督式學習-分類）

### 貝氏定理（Bayes' Theorem）
- **白話**：「看到新證據後，重新計算機率」的數學法則
- **章節**：CH3（補充版，可摺疊）

### KNN（K-Nearest Neighbors）
- **白話**：「物以類聚」，看周圍 K 個最近鄰居誰多就分哪類
- **章節**：CH3（監督式學習-分類）

### K-Means
- **白話**：把資料自動分成 K 群，常用於客戶分群
- **限制**：K 要事先指定、對離群值敏感（S1-Q9 重點）
- **章節**：CH3（非監督式學習-分群） | 樣題：S1-Q9

### 階層式分群（Hierarchical Clustering）
- **白話**：像「家族樹」一層層往上合併或往下切分
- **章節**：CH3（非監督式學習-分群，補充版）

### t-SNE
- **白話**：把高維資料壓成 2D / 3D 的「視覺化」工具
- **章節**：CH2（降維） / CH3（補充版）

## CH3-CH4 深度學習與序列模型（PDF p.16-22）

### RNN（Recurrent Neural Network 遞迴神經網路）
- **白話**：「有記憶」的神經網路，能處理一連串的資料
- **舉例**：文字接龍、股票時間序列預測
- **章節**：CH4（深度學習補充）

### LSTM（Long Short-Term Memory 長短期記憶）
- **白話**：RNN 的進階版，會「選擇性記憶 / 遺忘」
- **解決**：傳統 RNN 記不住長句子的問題
- **章節**：CH4（深度學習補充）

### ImageNet 與 AlexNet
- **白話**：ImageNet 是大型圖片資料集；AlexNet 是 2012 年奪冠的 CNN，**開啟了現代深度學習時代**
- **章節**：CH1（AI 演進史關鍵節點） / CH4

### BERT（Bidirectional Encoder Representations from Transformers）
- **白話**：Google 2018 年語言模型，**從前後文**學文字意思
- **比喻**：讓電腦不只「看字」，還能「懂上下文」
- **章節**：CH4（生成式 AI 前置 / Transformer 家族）

## CH3-CH4 NLP 與文字表示（PDF p.22-25）

### TF-IDF（詞頻－逆文檔頻率）
- **白話**：衡量文字在文件中的「重要程度」
- **應用**：搜尋引擎排名、文件分類、關鍵詞擷取
- **章節**：CH3 / CH4（NLP 補充版）

### Word2Vec
- **白話**：把文字轉成「數值向量」，語意相似的字向量距離近
- **經典案例**：King − Man + Woman ≈ Queen
- **章節**：CH4（補充版）

### LDA（Latent Dirichlet Allocation）
- **白話**：自動找出大量文件的「主題分佈」
- **應用**：文件主題分析、新聞分類
- **章節**：CH4（補充版）

## CH2 資料工程（PDF p.26-32）

### 結構化 / 半結構化 / 非結構化資料
- **結構化**：表格資料（關聯式資料庫）
- **半結構化**：有標記但無固定表格（JSON、XML、HTML）
- **非結構化**：文字、影像、音訊、影片
- **章節**：CH2 資料類型（樣題 S1-Q11、S1-Q12 重點）

### 正則化（Regularization）⚠️ 易混淆
- **白話**：防止模型「學太多」過擬合的技巧（給模型加規矩）
- **L1**：Lasso，部分權重變 0（特徵選擇）
- **L2**：Ridge，權重變小但不為 0
- **章節**：CH3（過擬合處理，S1-Q3 重點）

### 正規化（Normalization）⚠️ 易與正則化混淆
- **白話**：資料前處理，把不同特徵的數值縮到同尺度
- **比喻**：先把資料尺度調到「公平的起跑線」
- **方法**：Min-Max（縮到 0-1）、Z-Score（平均 0 標準差 1）、Batch Norm
- **章節**：CH2 資料前處理

### ADASYN（適應性合成採樣）
- **白話**：資料不平衡時，自動生成更多少數類樣本
- **跟 SMOTE 比**：更具自適應性（會看哪裡邊界難分）
- **章節**：CH2（補充版）

### 資料漂移（Data Drift）
- **白話**：模型上線後，輸入資料的分佈跟訓練時不一樣了
- **比喻**：「AI 學的世界」和「現實世界」不再一樣
- **章節**：CH7 模型部署監控

### 概念漂移（Concept Drift）⚠️ 跟資料漂移不同
- **白話**：輸入跟輸出的「關係」變了，不是資料分佈變
- **比喻**：模型學的規則「過時了」（突變、漸變、週期）
- **章節**：CH7 模型部署監控

## CH7 AI 專案實施（PDF p.33-37）

### AI 專案規劃流程（6 步驟）⭐⭐⭐
1. **問題定義** Problem Definition
2. **資料蒐集與清理** Data Collection & Cleaning
3. **模型設計與訓練** Model Design & Training
4. **模型評估與驗證** Evaluation & Validation
5. **部署與監控** Deployment & Monitoring（MLOps）
6. **持續優化與回饋** Iteration & Improvement
- **章節**：CH7 主軸（樣題 S2-Q29、S2-Q35 重點）

### LoRA（Low-Rank Adaptation 低秩適應）
- **白話**：讓 LLM 不用完全重新訓練就能學新任務的技術
- **重點**：訓練成本和儲存比全模型微調低得多
- **章節**：CH7（企業導入 LLM 補充版）

### 知識蒸餾（Knowledge Distillation）
- **白話**：「小模型(學生)」學「大模型(老師)」的輸出，省資源
- **章節**：CH7（模型壓縮補充版）

### AI 加速器（AI Accelerator）
- **GPU**：圖形處理器，深度學習主力
- **TPU**：Google 為 TensorFlow 客製化矩陣運算晶片
- **NPU**：手機/邊緣裝置即時 AI 推論
- **ASIC**：特定模型客製，效率最高彈性最低
- **對比口訣**：CPU 通用 / GPU 並行 / TPU 矩陣
- **章節**：CH7 IT 環境支援（樣題 S2-Q19 重點）

### IaaS / PaaS / SaaS（雲端三層架構）⭐
- **比喻**：從「自己蓋房」到「租現成的房」
- **IaaS**：基礎架構（AWS EC2）— 自己裝 OS 和應用
- **PaaS**：平台（Google App Engine）— 只專注應用開發
- **SaaS**：軟體（ChatGPT）— 即用型軟體
- **章節**：CH7 部署架構（**0 基礎學員必懂**）

## CH4 / CH6 進階 AI 概念（PDF p.50-58）

### Zero-shot / Few-shot Learning ⭐
- **Zero-shot**：模型沒看過任何任務樣本，光靠 Prompt 理解任務

- **白話**：回答前先花更多計算「多步思考」的模型

### CLIP（對比語言-影像預訓練）

## 0 基礎友善設計建議

依「川辰白話版」風格，每個技術名詞在初次出現時都要做到：
1. **中英對照**（含縮寫展開）
2. **一句話白話定義**（用「簡單來說」開頭）
3. **生活/商業舉例**（讓非技術背景能 imagine）
4. **跟相似名詞對比**（例：正則化 vs 正規化、資料漂移 vs 概念漂移）

⚠️ **特別易混淆名詞清單**（教案撰寫時要強化區分）：
- 正則化（Regularization）vs 正規化（Normalization）
- 資料漂移（Data Drift）vs 概念漂移（Concept Drift）
- 鑑別式 AI vs 生成式 AI
- TF / IDF vs TF-IDF（單獨 vs 組合）
- IaaS vs PaaS vs SaaS
- GPU / TPU / NPU / ASIC
- Zero-shot vs Few-shot

⚠️ **0 基礎不必背的（補充版可摺疊）**：
