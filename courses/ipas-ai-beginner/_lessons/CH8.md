## 8.1 中級三科目概覽（30 min）

### 為什麼要先認識中級？

iPAS AI 應用規劃師認證分**初級**與**中級**兩階。**初級**就是你正在準備的考試，重點在「**概念認識與職場應用判斷**」。**中級**則是更進階的層次，重點在「**規劃落地、技術細節與工程實務**」。

> 💡 **白話比喻**：
> - 初級 = 會點菜（知道菜單上的菜大概是什麼、好不好吃、適合誰）
> - 中級 = 會配菜下單給廚房（知道食材怎麼處理、要先煮什麼後煮什麼、出問題該回頭調整哪個步驟）
>
> 兩者用的是同一份菜單，但深度差了一層。

### 中級三科目對照

| 科目 | 中級主題 | 與初級對應章節 |
|---|---|---|
| 科目一 | **人工智慧技術應用與規劃** | 對應初級 CH1 + CH4（深化技術細節與規劃落地） |
| 科目二 | **大數據處理分析與應用** | 對應初級 CH2（深化進階統計與架構選擇） |
| 科目三 | **機器學習技術與應用** | 對應初級 CH3（深化演算法、評估、部署） |

### 與初級的根本差異

| 層面 | 初級（現在這份講義） | 中級（CH8 介紹的） |
|---|---|---|
| **題目深度** | 概念認識、情境判斷 | 概念計算、技術細節、程式碼閱讀 |
| **數學要求** | 看得懂公式意義即可 | 會算 Z-score、IQR 公式、ROC AUC |
| **程式要求** | **完全不需要** | 會讀基本 Python / numpy / sklearn |
| **模型細節** | 知道 CNN 處理影像、RNN 處理序列即可 | 區分 VGG / ResNet / Inception / R-CNN 變體 |
| **題目來源** | 商業情境判斷為主 | 含實際程式碼與架構圖 |

> 📌 **重點**：你看到上面表格的「中級」欄不要慌，**你現在不需要會這些**。這份講義 CH0-CH7 教的內容已經足夠你通過初級認證。

## 8.2 初→中級難度跳躍分析（45 min）

### 跳躍 1：概念深度提升

**初級你已經學過的，中級會深化成這樣：**

| 你在初級學過的 | 中級會問你 | 在初級講義哪一章 |
|---|---|---|
| 「CNN 適合處理影像」 | 區分 VGG / ResNet / Inception / R-CNN 各自的設計目的 | CH4 |
| 「資料隱私重要」 | 差分隱私加噪聲機制、GDPR 被遺忘權細節條款 | CH2、CH7 |
| 「過擬合要避免」 | Early Stopping / L1/L2 / Dropout 各自的數學原理 | CH3 |
| 「假說檢定有 H₀ 和 H₁」 | t 檢定 / F 檢定 / 卡方檢定 / ROC AUC 適用情境區分 | CH2 |
| 「Z-score 用來標準化」 | 給你成績、平均、標準差，**直接算 Z-score** | CH2 |
| 「IQR 可以找離群值」 | **直接算** Q1 - 1.5×IQR、Q3 + 1.5×IQR 範圍 | CH2 |
| 「優化器有很多種」 | Momentum / Adagrad / Adam / SGD 各自適用什麼情境 | CH3 |
| 「Transformer 是新架構」 | Self-Attention 機制細節、Word2Vec 詞嵌入 | CH4 |

> 💡 **學員容易誤解的點**：「我初級學過的是不是中級會重新教一次？」
> **答**：中級**不會重教**初級概念，預設你已經會了。中級題目假設你「概念都懂」，直接考你「能不能算、能不能讀程式、能不能挑變體」。

### 跳躍 2：技能跳躍（初級不考、中級開始考）

| 中級開始要會的 | 中級具體在哪科 |
|---|---|
| **程式碼閱讀**（Python / numpy / sklearn） | 科目二 + 科目三 ⭐ 最大跳躍 |
| **統計實作**（檢定方法選擇、ROC AUC 計算） | 科目二 |
| **模型架構辨識**（電腦視覺 4 大任務圖示題） | 科目一 |
| **模型壓縮、邊緣部署**（Pruning、Quantization） | 科目一 |
| **系統整合測試**（多模型協同） | 科目一 |
| **大數據架構**（MapReduce、串流處理） | 科目二 |
| **激活函數選擇**（Sigmoid / Softmax 用途） | 科目三 |

### 跳躍 3：題型變化

中級題目開始出現以下類型，**初級完全不會出現**：

1. **程式碼閱讀題**：給你一段 sklearn pipeline 或 numpy 操作，問執行結果
2. **架構辨識題**：給你 4 張電腦視覺任務示意圖，要你分辨哪張是 Object Detection、哪張是 Semantic Segmentation
3. **公式計算題**：給你具體數字，要你算 Z-score、Accuracy、IQR 範圍
4. **變體比較題**：列 4 種 CNN 變體，要你選「哪個是把卷積層加寬而不是加深」

> 🛑 **再強調一次**：上面這些題型**不會出現在初級**。你準備初級時不需要練這些。

## 8.3 中級樣題精選賞析（30 min）

> **使用方式**：以下 4 題從中級樣題（114.09 版）抽出，每題附「為什麼這題已超出初級範圍」說明。**讀完當作擴大視野即可，不必試著答對**。

### 樣題賞析 1：科一 Q4（Elman 神經網路）

**題目**：下列關於深度學習的敘述，何者錯誤？
- (A) CNN 透過卷積層提取空間特徵
- (B) RNN 適合處理序列數據
- (C) Transformer 透過自注意力機制處理長距離依賴
- (D) **Elman 神經網路適合處理影像辨識** ← 錯誤敘述（正解）

**詳解**：Elman 神經網路是 RNN 的早期變體之一，**屬於序列處理**，不是影像。本題正解為 (D)。

**為什麼這題超出初級**：
- 初級你只需要知道「CNN 適合影像、RNN 適合序列、Transformer 用注意力」（這在初級講義 CH4 已經教過）
- 中級開始**要求你認得 RNN 的具體變體名稱**（Elman、Jordan、LSTM、GRU 等）
- 初級**不會問**你 Elman 是什麼，只會問你「處理時間序列適合什麼類型」這種大方向

**對應初級講義位置**：CH4 鑑別式 AI 與生成式 AI 概念

---

### 樣題賞析 2：科二 Q10（Z-score 計算）

**題目**：某學生考試成績為 90 分，全班平均 70 分、標準差 10 分，該生的 Z-score 為何？
- (A) 1
- (B) 1.5
- **(C) 2** ← 正解
- (D) 2.5

**詳解**：Z-score = (觀察值 - 平均) / 標準差 = (90 - 70) / 10 = **2**。

**為什麼這題超出初級**：
- 初級你只需要知道「Z-score 是用來看一個數據離平均有多遠」（CH2 已教概念）
- 初級樣題 S1-Q10 也只考「**異常檢測適合用 Z-score**」這種概念判斷，不會給你具體數字
- 中級開始**直接給數字要你套公式算**

**對應初級講義位置**：CH2 統計量與假說檢定

---

### 樣題賞析 3：科二 Q15（CIFAR-10 程式碼閱讀）⭐⭐⭐ 最大跳躍

**題目**：執行下列程式碼後，x_test 共有幾筆資料？
```python
from tensorflow.keras.datasets import cifar10
(x_train, y_train), (x_test, y_test) = cifar10.load_data()
print(x_test.shape)
```
- (A) 5,000
- **(B) 10,000** ← 正解
- (C) 50,000
- (D) 60,000

**詳解**：CIFAR-10 標準切分為 50,000 訓練 + 10,000 測試，所以 x_test 是 10,000 筆。

**為什麼這題超出初級**（⭐ 這是初→中級的**最大跳躍點**）：
- 初級**完全不考程式碼**。本份講義從 CH0 到 CH9 都不會要求你看懂任何 Python
- 中級開始**至少要會讀**基本 sklearn / Keras / numpy 語法
- 中級不要求你**寫**程式，但要求你**看得懂**輸出結果

**對應初級講義位置**：本題在初級**沒有對應章節**（程式碼不在初級範圍）

---

### 樣題賞析 4：科三 Q15（sklearn pipeline 排序）⭐⭐⭐ 第二大跳躍

**題目**：使用 sklearn 訓練機器學習模型的標準流程，下列步驟正確順序為何？
- (a) 切分訓練/測試資料
- (b) 訓練模型 model.fit(...)
- (c) 載入並清理資料
- (d) 預測 model.predict(...) 並評估
- **(A) c → a → b → d** ← 正解
- (B) a → b → c → d
- (C) c → b → a → d
- (D) a → c → b → d

**詳解**：sklearn 標準工作流：先**載入清理**（c）→ 再**切分**訓練測試（a）→ **訓練**（b）→ 最後**預測評估**（d）。

**為什麼這題超出初級**：
- 初級你學過 4 階段導入（準備 → 設計 → POC → 實施，CH7 教過），那是**商業導入流程**
- 中級這題考的是**程式實作流程**（fit、predict 是 sklearn 函數名稱）
- 中級題目假設你**用過 Python 寫過 ML 程式**

**對應初級講義位置**：本題在初級**沒有對應章節**（程式實作不在初級範圍）

## 講師口頭補充（多場整合）

### Imagen 模型 = Google 的 Latent Diffusion

**Imagen** 是 Google 推出的擴散模型，本質是「**降維運算的擴散模型**」（Latent Diffusion 變種）。它不是直接在像素空間做 diffusion 而是在低維潛在空間，這是它跟原生擴散模型最大差別—與 Stable Diffusion 同思路但 Google 自己訓練。考題中與 ResNet（CNN 分類）、SVN/SVM（監督式分類）並排出現時，**只有 Imagen 是生成式**。 —— 來源：[408]

### CLIP 的本質是「對齊」不是「生成」

CLIP 把文字和圖像對應到「**相同的語義空間**」—假設文字是一維資料，用文字的方式去堆疊符合圖像資料的算法。也可延伸到音訊。**它的功能不是「生成」而是「對齊」**，把多模態的向量拉到同一個比對空間。它是 GPT-4o、Gemini 多模態能力的關鍵基礎。 —— 來源：[408]

### Zero-Shot vs Few-Shot Learning

**ZSL** = 沒有任何訓練資料下，模型能處理沒見過的新類別/任務（靠語義對齊）。**FSL** = 在「少量樣本」情況下還能達到不錯表現。**One-shot** 比 Few-shot 更狠—只看一個範例就學。一張圖在 AI 眼中只要翻轉、放大縮小、旋轉、平移就是「不同圖片」，所以一張圖實際上可以變成幾十到幾百張訓練樣本，這也是 FSL/ZSL 能成立的工程基礎。 —— 來源：[314p2] [408]

### Transformer = 注意力機制（單一識別重點）

「Transformer 看到注意力機制就直接綁，沒有別的」。Transformer 解決兩件事：(1) 模型聚焦在最重要的部分；(2) 打破過去 RNN 的「序列依賴瓶頸」（中文/英文詞序改變理解就改變的問題）。Transformer 衍生出 BERT（理解文字）和 GPT（生成）兩支主流，前者是 Google 開發、後者是 OpenAI。 —— 來源：[314p1] [408]

### 學習率（learning rate）的「下山找坑」比喻

學習率太低 → 完全貼合曲線但每個箭頭很短、學很慢；學習率適中 → 箭頭數量約一半、可以包住線；學習率太大 → 瘋狂跳、四個反彈點什麼都學不到。**實作上一律先設稍大、再慢慢降**。常見問題：(1) 局部最小值（卡在小坑找不到全局）；(2) 梯度爆炸（權重失控發散）。這條心法在中級程式碼閱讀題（sklearn pipeline 排序）也會用到。 —— 來源：[314p1]

### MLOps 四工具的目的（中級實作會用到）

(1) 追蹤資料版本（**比追蹤模型版本更重要**）；(2) 自動訓練測試部署（鑑別型 AI 遇例外條件要自動存下來，下次回頭學進去—類似漁市場老闆看到新魚不會明天忘）；(3) 自動偵測例外條件 + 收集回饋；(4) 統一平台 + 持續監控 + 版本管理。**核心精神**：人力介入只應該介入在「執行面」，daily 維運要自動化。中級科目三會考 MLflow / DVC / Kubeflow 等具體工具。 —— 來源：[314p2]

### 數位孿生（Digital Twin）核心

把實體生產線/設備全部數據化讓 AI 模擬，「此時此刻這樣設計、最後產量各環節數值會是怎樣」。比起真的建設便宜很多，跟做物理實驗本質一樣。**火箭故事**—不會只蓋一台、會蓋兩台一模一樣的，「上去如果壞了哪修，所以用地球上那一台模擬」，這就是孿生實務。中級會考 Industrial AI / Industry 4.0 場景。 —— 來源：[408]

### LoRA 的 R 值對訓練成本的影響

LoRA（Low-Rank Adaptation）透過低秩矩陣，讓模型微調時只調整少量參數。**R 值越小 → 訓練越加速、顯存佔用越低、成本越低，但結果不一定最好**。R 值的關鍵是「在成本與效果之間取折衷」，沒有萬用最佳值。實務上做企業微調時 R 值通常從 8、16、32 開始試。 —— 來源：[415]

### 模型量化 vs 知識蒸餾 vs 網路剪枝

三個容易搞混的模型壓縮技術，各自的本質不同：

| 技術 | 本質 |
|---|---|
| **量化（Quantization）** | 把資料形態從 FP32 降到 INT8（改變「數值精度」） |
| **剪枝（Pruning）** | 把不重要的神經元或連線刪除（不改變數值精度） |
| **知識蒸餾（Distillation）** | 用大模型當「老師」訓練小模型「學生」 |

知名例子：DeepSeek 早期被認為用知識蒸餾從現有大模型反推學習，所以表現異常好。

考試陷阱：「權重擴增」這個詞**不存在**——看到直接刪。 —— 來源：[415]

### 擴散模型加速採樣的演算法本質

把「慢慢還原」這段演算法用數學方式壓縮抓在一起算，從幾百步壓到 5-10 步。LCM、DDIM、DPM-Solver 等採樣器都是在做這件事。考試題目寫到「應用採樣器優化的演算法、減少反向去噪的迭代步數」就是這條路線，不是改變雜訊量、不是換 RNN、不是增加提示詞。 —— 來源：[415]

### 資料譜系（Data Lineage）vs 資料治理 vs ER 圖

題目要追蹤「從哪一筆資料 → 經過哪些步驟 → 最後流到哪」這種「資料碳足跡」就是資料譜系，不是 ER 圖、不是統計平均值/標準差、不是視覺化儀表板。資料譜系是企業 AI 合規（特別是金融、醫療業）的核心需求。**實作 6 件事 + 三階段**：(1) Effect / When / Where / Who / Why / How 六件事；(2) 原始來源 → 處理轉換 → 模型應用三階段。 —— 來源：[315p2] [415]

### PPO 在 RLHF 微調的角色

PPO（Proximal Policy Optimization）的核心是「剪裁機制」——嚴格限制模型每次更新時不能離原模型太遠。一句話：**「不要學太快」、避免極端值衝擊**。這是 PPO 跟其他強化學習演算法（如純 policy gradient）最大的差異，也是 RLHF 訓練 LLM 時最常用的演算法。 —— 來源：[415]

### Informer：用「機率稀疏自注意力」處理超長時序

傳統 Transformer 處理超長時序時記憶體會平方級爆增。Informer 用數學推導讓注意力**只保留最具代表性的幾個位置**（高貢獻值的 token），把記憶體複雜度從 O(n²) 降到 O(n log n)。

考試陷阱：完全捨棄注意力回到 RNN、引入差分隱私、把時序轉 2D 圖譜給 CNN——三個都是錯的。 —— 來源：[415]

### Plan-and-Solve 改良 ReAct 的死循環缺點

ReAct（Reasoning + Acting）框架在面對複雜長任務時容易迷失方向、陷入無限迴圈。Plan-and-Solve 的改良核心：**先生成完整宏觀計畫、再依序執行每個子任務**。

考試干擾項都是「限制 / 凍結」方向：

- 限制只能輸出 100 種制式回答
- 凍結權重 + 禁止外部工具
- 人類接管鍵盤

這三個都跟 AI agent 的設計初衷相反——agent 的精神就是給足權限、放手做事。 —— 來源：[415]

### Actor-Critic 框架的分工

| 角色 | 職責 |
|---|---|
| Actor | 策略網路：看當前環境決定要做哪一動 |
| Critic | 價值網路：評估剛剛做的動作是好還是壞，打分數 |

考試陷阱：「Actor 預測未來、Critic 記憶歷史」這種拆分**不存在**，這是錯誤選項。 —— 來源：[415]

### D-Fake 隱形浮水印必須在「生成階段」植入

要對抗 deepfake，浮水印必須能抗剪裁與壓縮。所以正解是「在生成模型的解碼器訓練 / 潛在空間取樣階段植入特定頻率模式」——也就是把浮水印塞進擴散模型的 latent space 取樣過程裡。

考試干擾項：

- 降低解析度（會破壞圖片品質且能被還原）
- 在 JPEG header 寫文字記錄（可被輕易修改）
- 用 GPU 風扇轉速產生硬體指紋（沒這個說法）

—— 來源：[415]

### DPO 取代 RLHF 的數學優勢

傳統 RLHF 流程複雜：先訓練大型獎勵模型 → 用 PPO 不穩定地微調。**DPO（Direct Preference Optimization）用數學推導直接把人類偏好資料轉成交叉熵損失，繞過獎勵模型 + 強化學習步驟**。

DPO 的缺點：訓練資料偏頗時會崩。但它跟 PPO 微調 LLM 比省了一個訓練階段，是 2024 年後業界主流。 —— 來源：[415]

### Perplexity（困惑度）的方向感

| 困惑度 | 含義 |
|---|---|
| 越低 | 模型對下一個詞越有信心、機率分布越集中於正確答案 |
| 越高 | **不代表越精確**（這是反過來的，是錯誤選項） |

困惑度計算的是「機率分布」，所以是 0-1 的浮點數，不是錯誤字詞總數。困惑度只用於語言模型，不用於圖像生成。 —— 來源：[415]

### 對抗訓練是「打疫苗」

對抗樣本攻擊的根本對策：**訓練時主動帶入帶有對抗擾動的樣本**讓模型學習。比喻：本來在學的時候就先學到攻擊長什麼樣子，等於打疫苗。

考試陷阱：「把圖片轉黑白二值化」不會增強對抗韌性——CNN 抓的是輪廓特徵，黑白與否影響不大，所以這條路不能擋對抗攻擊。 —— 來源：[415]

### DQN 的核心改良：用神經網路近似 Q 函數

傳統 Q-learning 用「表格」記錄每個狀態的報酬。但圍棋、高解析度遊戲畫面這類狀態空間太大，表格根本存不下。**DQN 用神經網路（CNN）來近似 Q 函數**——直接讀畫面當輸入，輸出每個動作的 Q 值。

實務細節：電腦不會理解「這個畫面裡有敵人 / 隊友 / 道具」，它就是一幀一幀像素地記下來，所以才需要 CNN 抽取特徵。 —— 來源：[415]

### Prefix Tuning 的「黏土」比喻

Prefix Tuning 是另一種高效微調技術。原理：

1. **凍結** LLM 所有預訓練權重
2. 在 Transformer **每一層**的自注意力機制裡插入一組可訓練的連續向量

這組向量像「黏土」一樣電住輸出，可以即時用數學公式調整，達到輕量微調目的。和 LoRA 不同：Prefix Tuning 完全鎖權重、只調黏土；LoRA 是注入低秩矩陣去調權重。 —— 來源：[415]

### Mamba / 狀態空間模型（SSM）：超長上下文的新解法

Transformer 自注意力是平方級爆炸（O(n²)）。Mamba 基於控制論常見的「狀態空間模型」，把資訊壓縮成「固定大小的隱藏狀態」（類似 RNN 的精神），讓時間複雜度降到 O(n)（線性）。

這是 2024-2026 年取代 Transformer 處理超長上下文（百 K ~ 百萬 token）的重要新架構。考試題目寫到「Mamba」「狀態空間」「線性時間複雜度」就綁這條。 —— 來源：[415]

### 講師備考心法（總結）

> 🎙️ 415 場最後一段口頭評語：

「重點不是對多少題、不是寫到滾瓜爛熟。**重點是把每個選項的專有名詞跟它對應的技術 / 概念綁起來**。一題四個選項就練到四個關鍵字，這樣最大程度幫到備考。」

「一個題目的四個選項就可以讓你練到四個關鍵字。以這樣為目標去做準備，能夠最大程度地幫助到大家。」
