## 9.1 考試流程說明（10 min）

### iPAS AI 應用規劃師初級認證考試格式

| 項目 | 規格 |
|---|---|
| **考試形式** | 線上監考（自家電腦考） |
| **題型** | **單選題** |
| **科目** | 2 科目（科一：人工智慧基礎概論｜科二：生成式 AI 應用與規劃） |
| **每科題數** | 35 題（合計 70 題） |
| **每科時間** | 60 分鐘（合計 2 小時） |
| **每題配分** | 約 2.86 分（每科總分 100） |
| **合格標準** | **兩科皆達 60 分**（即每科答對 21 題以上）<br>⚠️ 本模擬測驗合格門檻：≥ 56 題（70%） |
| **重考限制** | 視官方公告（一般可隔週重考） |

> 📌 **注意**：本模擬測驗 80 題是為了**涵蓋全範圍練習**，比實際考試多 10 題。實際考試是 70 題（兩科各 35），用本份模擬卷練到 70% 以上，實戰應夠用。

### 考試當天流程（線上）

1. **報到**：前 15 分鐘登入考試系統，完成設備檢查（webcam、麥克風）
2. **規則說明**：閱讀考試規則，同意後開始
3. **科目一作答**：60 分鐘，作答中可標記題目稍後回頭看
4. **科目間休息**：依官方公告（通常 5-10 分鐘）
5. **科目二作答**：60 分鐘
6. **交卷**：時間到自動交卷，或可提前手動交卷
7. **成績公告**：依官方公告（一般當週至次週公告）

### 模擬測驗使用方式（本章 9.2 起）

> **建議流程**：
> 1. **先模擬實戰**：找一個 90 分鐘不被打擾的時段，把全部 80 題作答完，**不查資料**
> 2. **記錄答案**：用紙本或電子文件記下你的答案
> 3. **對答案**：作答完才看 9.3 詳解索引，計算總分
> 4. **針對性複習**：對照 9.4 弱項分析，回頭重讀對應章節

#### E2（Q17）Type I / Type II Error 辨識｜CH2 ⚠️ 勘誤後正確版

**題目**：在統計假說檢定中，下列敘述何者**正確**？
- (A) Type I 錯誤是「縱放壞人」，把假的 H₀ 當真
- (B) **Type I 錯誤（α）是「冤枉好人」，拒絕了真的 H₀**
- (C) Type II 錯誤是拒絕真的 H₀
- (D) Type I 與 Type II 錯誤可同時降到 0

**正解**：**B**

**詳解**（依勘誤後 03-erratum.md 第 4 條）：
- α = Type I 錯誤 = **拒絕為真的 H₀**（冤枉好人）
- β = Type II 錯誤 = **接受為假的 H₀**（縱放壞人）
- (A) 描述顛倒（縱放壞人是 Type II，不是 Type I）
- (C) 描述顛倒（拒絕真 H₀ 是 Type I）
- (D) α、β **不能同時降到 0**，兩者有反向關係（樣本固定下）

**0 基礎提醒**：「I = 一」就是「冤枉好人（一好人）」，「II = 二」就是「縱放壞人（二字尾）」，諧音記。

#### E3（Q31）混淆矩陣判讀｜CH3

**題目**：某模型對 100 位病人的疾病檢測結果如下：實際生病 30 人中，模型預測 24 人為生病、6 人為健康；實際健康 70 人中，模型預測 5 人為生病、65 人為健康。下列何者**正確**？
- (A) 精確率（Precision）＝ 24 / 30 ＝ 80%
- (B) **召回率（Recall）＝ 24 / 30 ＝ 80%**
- (C) 準確率（Accuracy）＝ 24 / 100 ＝ 24%
- (D) F1 score 為 100%

**正解**：**B**

**詳解**：
- TP = 24（生病且預測生病）、FN = 6（生病但預測健康）、FP = 5（健康但預測生病）、TN = 65（健康且預測健康）
- 精確率 = TP / (TP + FP) = 24 / 29 ≈ 82.8%（不是 80%）
- 召回率 = TP / (TP + FN) = 24 / 30 = **80%** ✓
- 準確率 = (TP + TN) / 100 = (24 + 65) / 100 = 89%（不是 24%）
- F1 = 2 × Prec × Rec / (Prec + Rec) ≈ 81.4%（不是 100%）

**0 基礎提醒**：醫療場景看「**召回率**」（漏報多嚴重），金融場景看「**精確率**」（誤報多嚴重）。

## 9.5 答題技巧（15 min）

### 技巧 1：時間管理

**60 分鐘 35 題 = 平均每題 1 分 43 秒**。實戰建議：

- **第一輪**（30 分鐘）：快速做完所有「一看就會」的題目（70-80% 題目），不確定的標記跳過
- **第二輪**（20 分鐘）：回頭處理標記題，運用刪去法做最佳猜測
- **第三輪**（10 分鐘）：檢查疑慮、確認答題卡填寫無誤

> ⚠️ **不要在一題卡超過 3 分鐘**。卡住就標記跳過，回頭再處理。

### 技巧 2：刪去法（看不懂題目時的救命稻草）

iPAS 單選題 4 個選項，先刪掉「絕對錯」的，再從剩下選最合理的：

**典型「絕對錯」訊號**：
- 含「**只能**」「**完全不可能**」「**100%**」這種絕對化字眼 → 八成是錯的
- 與題目情境**完全無關**的選項 → 直接刪
- 兩個選項**意思相反**：通常正解在這兩個之中
- 兩個選項**意思幾乎相同**：兩個應該都不是答案（單選題不會兩個正確）

**範例**（用 Q40 VGG 屬鑑別非生成）：
- 看到「VGG」→ 想到 CNN → CNN 屬於鑑別式 → 答案應在「鑑別」相關選項
- 看到含「生成」字眼的選項可以先刪

### 技巧 3：易錯陷阱辨識

**iPAS 樣題常見陷阱**：

#### 陷阱 1：**「不是」題（負面題）**

題幹寫「下列**何者非**……」「下列**不屬於**……」

**對策**：先把每個選項當作「正確的」評估一次，最後反過來選那個「不對的」。
**例**：S1-Q8 大數據特性（D 資料存儲位置固定 → 是錯的，所以選 D）

#### 陷阱 2：**「最」字題**

題幹寫「**最**適合」「**最**不適合」「**最**主要」

**對策**：所有選項可能都「有點對」，要選**最強的**那個。注意「相對比較」而非「絕對對錯」。

#### 陷阱 3：**勘誤後修正版題目**

特別注意 03-erratum.md 整理的：
- **Type I/II Error 表**（拒絕真 H₀ 是 Type I）
- **K-means 原理簡單**（不是複雜）
- **「95% 信心水準下拒絕 H₀」**（嚴謹用語，「我有 95% 信心拒絕」是錯誤敘述）

#### 陷阱 4：**英文縮寫陷阱**

選項裡放陌生英文縮寫嚇人，如 RLHF、CoT、TTQA、PCA、TCO。

**對策**：縮寫不會就**回到題目情境**判斷，從情境推測哪個合理。

### 技巧 4：完全沒看過的題目怎麼辦

1. **不要慌**，讀 2 遍題幹
2. **找關鍵字**：題目情境（醫療？金融？製造？）對應到哪一章
3. **刪去法**：刪 1-2 個明顯離題選項
4. **猜測**：剩下 2-3 個選項，**選最像課本講過的那個**
5. **不要空白**：iPAS **倒扣機制依官方公告**，但一般單選題沒倒扣，**寧可猜也不要空著**

### 技巧 5：答題心態

- **不要追求 100 分**：實際考試門檻是兩科各 60 分（每科 21 題對）；本模擬測驗 56 題（70%）達標即可，不需滿分
- **遇到難題先跳過**：一題只值 ~2.86 分，不值得卡 5 分鐘
- **相信第一直覺**：除非有強烈證據，**不要改答案**（統計上改答案常改錯）
- **看到「不是」題放慢 10 秒**：這是最常被慣性害的題型

#### 2. 業界 AI 應用實際案例

## 常見錯誤觀念釐清

| 錯誤觀念 | 正確認知 |
|---|---|
| 「模擬考一定要 100% 才能去考」 | **70%** 已達及格門檻；80% 較有把握；不需要 100% |
| 「考試會出題庫的原題」 | iPAS 樣題僅供**範例參考**，**實際題目會調整**；要靠觀念答題 |
| 「猜題運氣好就能過」 | 實際考試每科 35 題、60% 門檻 = 21 題對；80 題模擬測驗 70% 門檻 = 56 題對，**靠純猜測機率極低** |
| 「考過初級就完成 AI 學習」 | 認證是「起點不是終點」，CH7 動手活動才是價值核心 |
| 「沒考過就是白學」 | 即使沒考過，本研習教的 4 階段導入流程、Prompt 工程、風險管理都可直接用在工作 |
| 「考試只能考一次」 | 一般可重考（依官方公告週期），失敗不是世界末日 |

### 易錯陷阱（再看一次）

- [ ] **「不是」題**：先評估每個選項，反向選錯的
- [ ] **「最」字題**：選相對最強，不是絕對對錯

- [ ] **K-means 原理簡單**（不是複雜）
- [ ] **VGG 屬鑑別**（不是生成）
- [ ] **Temperature 邏輯題低、創意題高**
