# CH3｜機器學習概念（5 小時）

> ⭐⭐⭐ 本章與 CH2 並列為「題量最大、最容易拉開差距」的兩章。
> 本章對應科目一第 2、3、5、9、13、17、19、20、22-25、35 題，共 **13 題**。
> 機器學習是整套考試的**核心地基**，是分量最重的章節。

---

## 學習目標（這章學完你能回答的工作問題）

讀完本章後，你應該能夠回答以下情境：

1. **「主管說我們要做一個『預測客戶會不會買』的模型，這是什麼類型的學習？」**
 → 你會答出「監督式學習」並說明為什麼（樣題 S1-Q5、Q13）。
2. **「圍棋 AI、自駕車、機器人手臂這類『一直在試錯』的應用屬於哪種 ML？」**
 → 你會答出「強化學習」（樣題 S1-Q2）。
3. **「同事說我們的模型訓練集準確率 99%、測試集只有 60%，怎麼辦？」**
 → 你會說這是過擬合，並提出加正則化、early stopping、增加資料量等方案（樣題 S1-Q3）。
4. **「老闆問我們的工廠異常停機預警模型『漏報率』表現如何，要看哪個指標？」**
 → 你會答出「召回率（Recall）」（樣題 S1-Q24）。
5. **「分析師說要用 K-means 對客戶分群，我們的客戶資料有性別、會員等級這種類別，行不行？」**
 → 你會答出「K-means 不適合類別資料」（樣題 S1-Q9）。
6. **「主管問『迴歸分析』算不算一種特徵選取方法？」**
 → 你會答出**迴歸分析是預測模型，不是特徵選取技術**（樣題 S1-Q22 ⚠️ 必考陷阱題）。

> 💡 **0 基礎友善承諾**：本章每個演算法都會給你「**用一句話辨識**」的記憶法。考試不會考你寫程式、也不會考數學公式推導，只會考「**情境 → 該選哪個模型/方法**」。

## 0 基礎入口：先把機器學習想成「教小孩做事」

> 🔧 **機器學習（Machine Learning, ML）**：白話 — 「**讓電腦從大量資料裡，自己找規律**」，不需要工程師寫死規則。

### 跟「傳統程式」差在哪？

| 傳統程式 | 機器學習 |
|---|---|
| 工程師寫好「**規則**」，電腦照規則做事 | 工程師給「**範例資料**」，電腦自己找規則 |
| 例：信用卡核卡 IF 月薪>5 萬 AND 信用分>700 THEN 通過 | 例：給電腦看 100 萬筆「核過 / 沒核過」的歷史資料，讓它自己學什麼樣的人會核過 |
| 邏輯死板、規則改不動 | 規則隨資料更新自動進化 |

### 教小孩學「認水果」的三種方式

這個比喻會貫穿整章，很重要：

| 教法 | 對應 ML 類型 | 故事 |
|---|---|---|
| **「這是蘋果，那是香蕉」** — 一個個告訴他 | 監督式學習 | 每張圖都標好答案 |
| **「自己分組玩玩看」** — 給他一籃水果不告訴他名字 | 非監督式學習 | 他會自己依顏色 / 形狀 / 大小分群 |
| **「你猜對給糖、猜錯不給」** — 用獎懲讓他學 | 強化學習 | 多次嘗試後找到最佳策略 |

## 3.1 機器學習基本原理（45 分鐘）

### A. 三大核心要素

任何機器學習系統，都由三件東西組成：

| 要素 | 中文 | 白話 |
|---|---|---|
| **Data** | 資料 | 「教材」 — 給模型學的範例 |
| **Model** | 模型 | 「學生」 — 找規律、做預測的那個東西 |
| **Loss Function** | 損失函數 | 「考卷批改老師」 — 評估預測準不準，指導模型修正 |

> 💡 **記法**：「**有教材、有學生、有老師**」，缺一不可。

### B. ML 訓練的三階段

```
1. 資料處理 → 清洗、標準化、特徵選擇/降維（CH2 教過了）
2. 模型訓練 → 選模型、定損失函數、跑優化器、防過擬合
3. 評估與優化 → 指標檢核、交叉驗證、調參
```

> 🎯 本章重點是 **第 2 與第 3 階段**。

### C. 三層名詞關係：AI ⊃ ML ⊃ DL

> 0 基礎學員最常混淆的三個英文縮寫，用同心圓記。

```
 ┌──────────────────────────────┐
 │ AI（人工智慧） │
 │ = 任何讓機器表現出「智慧」 │
 │ ┌──────────────────────┐ │
 │ │ ML（機器學習） │ │
 │ │ = 從資料學規律的 AI │ │
 │ │ ┌──────────────┐ │ │
 │ │ │ DL（深度學習） │ │ │
 │ │ │ = 用多層神經網路 │ │
 │ │ │ 的 ML 子方法 │ │
 │ │ └──────────────┘ │ │
 │ └──────────────────────┘ │
 └──────────────────────────────┘
```

| 縮寫 | 全名 | 一句話定義 |
|---|---|---|
| **AI** | Artificial Intelligence | 模擬人類智慧的所有技術（最大集合） |
| **ML** | Machine Learning | 從資料學規律的 AI 子集 |
| **DL** | Deep Learning | 用多層神經網路的 ML 子集 |

> 💡 **記法**：「**ML 在 AI 裡，DL 在 ML 裡**」。深度學習一定是機器學習，但機器學習不一定是深度學習。

## 3.2 三大學習類型（90 分鐘，必背對比表）

> ⚠️ 這節對比表**幾乎每年都考**，至少出 2-3 題，請務必熟記。

### 三大類型對比表

| 維度 | 監督式學習<br>Supervised | 非監督式學習<br>Unsupervised | 強化學習<br>Reinforcement |
|---|---|---|---|
| **資料是否有標籤** | **有標籤** | **無標籤** | 沒有預先標籤，靠**獎懲訊號** |
| **學習目標** | 學會把輸入 → 對應到正確答案 | 自己找出資料中的潛在結構 | 學會在環境中採取行動，最大化長期累積獎勵 |
| **典型任務** | 分類 / 迴歸 | 分群 / 降維 / 異常檢測 | 序列決策 / 控制 |
| **白話比喻** | 老師批改作業 | 自己分組玩 | 訓練寵物給獎勵 |
| **常見演算法** | 線性迴歸、邏輯迴歸、決策樹、隨機森林、SVM、KNN、Naive Bayes、神經網路 | K-means、階層式分群、DBSCAN、PCA、t-SNE | Q-Learning、PPO、DQN |
| **應用例子** | 垃圾郵件辨識、房價預測、影像分類 | 客戶分群、市場區隔、異常偵測 | 圍棋 AlphaGo、自駕車、機器人控制、遊戲 AI |

#### 樣題練習：S1-Q2（答 D，必考）

**題目**：下列何者最適合訓練電腦下圍棋、自動駕駛等動態重複地互動的問題？
- (A) 監督式學習
- (B) 非監督式學習
- (C) 半監督式學習
- **(D) 強化學習（Reinforcement Learning）** ✓

**詳解**：題目關鍵詞「**動態 + 重複 + 互動**」 = 強化學習的招牌特徵。圍棋每下一步都會收到「贏/輸/盤面好壞」的回饋訊號，模型透過這些訊號慢慢學最佳策略。

#### 樣題練習：S1-Q5（答 A）

**題目**：若要設計一個能夠辨識並過濾垃圾郵件的系統，應該選擇下列哪一種機器學習演算法以實現最佳效果？
- **(A) 監督式學習（Supervised Learning）** ✓
- (B) 非監督式學習
- (C) 半監督式學習
- (D) 強化學習

**詳解**：垃圾郵件辨識需要事先有「**已標記為垃圾 / 非垃圾的歷史郵件**」當訓練資料 → 監督式學習的標準場景。

#### 樣題練習：S1-Q13（答 D）

**題目**：某行銷公司欲針對新客戶進行「行銷活動推播」，希望預測新單一客戶是否可能完成購買行為。資料中包含「客戶基本資料、過往點擊行為、**類似客戶是否曾購買過（已標記「有/無購買」標籤）**」。下列哪一種學習方式與資料搭配最合適？
- (A) 使用非監督式學習，分析所有客戶點擊路徑進行異常偵測
- (B) 使用非監督式學習，將客戶分群後預測其轉換率
- (C) 使用監督式學習，針對未標記資料直接預測轉換可能性
- **(D) 使用監督式學習，以已知購買結果作為標籤進行訓練** ✓

**詳解**：題目明寫「**已標記有/無購買標籤**」 → 一定是監督式學習。(C) 雖也是監督式，但「針對未標記資料訓練」邏輯錯誤。(D) 用已標記資料訓練，再去預測新客戶，是標準監督式流程。

#### 樣題練習：S1-Q35（答 B）

**題目**：某電商平台希望預測商品的退貨機率，輸入是「商品售價、顧客年齡、運送天數」，輸出是「是否退貨（是/否）」。「是否退貨」應歸類為下列哪一類變數？
- (A) 特徵（Feature）
- **(B) 標籤（Label / Target）** ✓
- (C) 超參數（Hyperparameter）
- (D) 正則化係數（Regularization Coefficient）

**詳解**：
- **特徵（Feature）**：模型的**輸入** — 售價、年齡、天數
- **標籤（Label）**：模型要預測的**輸出** — 是否退貨
- **超參數（Hyperparameter）**：人工指定的訓練設定值（如學習率、樹的深度）
- **正則化係數**：控制正則化強度的超參數（用 λ 表示）

> 💡 **核心記法**：「**輸入是特徵、輸出是標籤、設定值是超參數**」。

## 3.3 監督式學習：分類 vs 迴歸（60 分鐘）

監督式學習依「**輸出是什麼類型**」再分兩支：

| 子類 | 輸出 | 例子 | 常見演算法 |
|---|---|---|---|
| **分類（Classification）** | 類別標籤（離散） | 是 / 否、貓 / 狗 / 鳥、A / B / C 級風險 | 決策樹、隨機森林、SVM、KNN、Naive Bayes、邏輯迴歸、神經網路 |
| **迴歸（Regression）** | 數值（連續） | 房價、銷售額、氣溫、剩餘壽命 | 線性迴歸 |

### A. 線性迴歸（樣題 S1-Q20）

> 🔧 **線性迴歸（Linear Regression）**：白話 — 「**找一條最能貼合資料的直線**」，用來預測連續數值。

**比喻**：你想用「廣告費」預測「銷售額」，把過去 100 筆紀錄畫成散佈圖，畫一條最能貼合所有點的直線 — 這就是線性迴歸。

**應用例子**：
- 房價預測（用坪數、地段、屋齡預測價格）
- 銷售額預測（用廣告費、季節、促銷力度預測）
- 氣溫預測

#### 樣題練習：S1-Q20（答 B）

**題目**：線性迴歸模型最適合解決哪種類型的問題？
- (A) 圖像分類
- **(B) 銷售額預測** ✓
- (C) 聚類分析
- (D) 遊戲策略學習

**詳解**：
- (A) 圖像分類 → 是「分類」，要用 CNN / 邏輯迴歸 / SVM
- (B) 銷售額 → 連續數值預測，正解
- (C) 聚類 → 非監督式
- (D) 遊戲策略 → 強化學習

> 💡 看到「**預測一個數字**」就想到線性迴歸；看到「**預測一個類別**」就想到分類演算法。

### B. 邏輯迴歸（補充版）

> 🔧 **邏輯迴歸（Logistic Regression）**：白話 — 名字有「迴歸」但其實是**分類**演算法。
> 透過 Sigmoid 函數把任何數值壓到 0–1 之間，當作「機率」。

**注意陷阱**：邏輯迴歸 ≠ 線性迴歸。
- 線性迴歸：輸出**連續數值**
- 邏輯迴歸：輸出**0–1 機率**，再依閾值判分類

### C. 決策樹（樣題 S1-Q17）

> 🔧 **決策樹（Decision Tree）**：白話 — 「**用一連串是非題畫成一棵樹做決策**」。

**比喻**：醫生問診流程
```
有發燒嗎？
├─ 有 → 有咳嗽嗎？
│ ├─ 有 → 流感
│ └─ 沒 → 中暑
└─ 沒 → 有頭痛嗎？
 ├─ 有 → 偏頭痛
 └─ 沒 → 沒事
```

**特點**：
- ✓ **可解釋性高** — 每個分支都能講出原因
- ✓ 能處理數值 + 類別資料
- ✗ 容易過擬合（樹長太深、規則太細）

#### 樣題練習：S1-Q17（答 A）

**題目**：某醫院研究特定心血管疾病的成因，收集了 50 名病患與 150 名正常人的年齡、血壓、血型等三項屬性變數。此研究適合使用下列哪一種機器學習模型來建立？
- **(A) 決策樹（Decision Tree）** ✓
- (B) 線性迴歸
- (C) DBSCAN
- (D) K-means 聚類

**詳解**：
- 任務性質 = **分類**（病/不病）
- 屬性 = **數值（年齡、血壓）+ 類別（血型）混合**
- 醫療研究需要**可解釋**（醫生要看得懂規則）
- → 決策樹三項都符合，正解

> 💡 **記法**：「醫療研究 + 可解釋」就想到決策樹。

### D. 隨機森林（補充版）

> 🔧 **隨機森林（Random Forest）**：白話 — 「**種一片決策樹，多數決定答案**」。
> 比單一決策樹更穩定、不易過擬合。

**比喻**：醫院召開會診，找 100 個醫生各自獨立診斷，最後採多數決。

**重點屬性**：
- 屬於**集成式學習（Ensemble Learning）**的一種
- 透過 **Bootstrap 取樣**讓每棵樹看到不同資料
- 可透過「**特徵重要度（Feature Importance）**」**輔助特徵選取**（這點是 S1-Q22 的關鍵 ⚠️）

### E. SVM（補充版）

> 🔧 **SVM（Support Vector Machine, 支援向量機）**：白話 — 「**在兩類資料之間畫一條最公平的分隔線**」。
> 線離兩邊最近的資料點都要等距，才算最公平。

**特點**：
- 適合**高維資料**（如文本、影像）
- 用「**核函數（Kernel）**」處理非線性問題
- 訓練資料量小時表現好

### F. KNN（補充版）

> 🔧 **KNN（K-Nearest Neighbors, K 最近鄰）**：白話 — 「**物以類聚**」。
> 看周圍 K 個最近鄰居誰多、就分到哪一類。

**特點**：
- 沒有「訓練」階段（lazy learner）
- 對 K 值很敏感
- 計算成本高

### G. Naive Bayes 貝氏分類器（補充版）

> 🔧 **Naive Bayes**：白話 — 「**用機率做分類，假設每個特徵互相獨立**」。
> 「Naive（天真）」就是因為「假設特徵互相獨立」這個前提其實並不真，但效果意外地好。

**比喻**：垃圾信過濾
- 看「免費」、「中獎」、「點擊」這些字出現的機率
- 用 Bayes 定理算出「這封信是垃圾」的後驗機率

> 🔧 **Bayes 定理白話比喻**：
> 「**看到新證據後，重新計算機率**」。
> 例：原本醫生估計你得病機率 1%，做完檢驗陽性後，重新算後機率變 30%。

公式不必背，知道「貝氏分類器 = 用機率分類 + 假設特徵獨立」就足以應付初級考試。

## 3.4 非監督式學習：分群與降維（45 分鐘）

監督式有「答案」，非監督式則是「**自己找結構**」。

### A. K-means 聚類（樣題 S1-Q9，必考陷阱題）

> 🔧 **K-means 聚類**：白話 — 「**指定要分 K 群，演算法自己把資料分成 K 群**」。

**運作原理（不必背步驟）**：
1. 隨機選 K 個中心點
2. 把每個資料分配到最近的中心
3. 重新計算每群中心
4. 重複 2-3 直到穩定

**K-means 的限制（考試重點）**：

| 限制 | 白話 |
|---|---|
| ❌ **不適合類別資料** | 因為要算「距離」，類別（如顏色、性別）沒有距離概念 |
| ❌ **K 要事先指定** | 不知道要分幾群時很麻煩 |
| ❌ **對離群值敏感** | 一個離群值會把中心拉歪 |
| ❌ **不適合非球形群集** | 例如環狀、長條狀的群會被切錯 |
| ❌ **不同起始點可能得到不同結果** | 隨機性影響穩定性 |

#### 樣題練習：S1-Q9（答 D）

**題目**：關於 K 平均法（K-means），下列敘述何者「**不**」正確？
- (A) 希望找出 k 個互不交集的群集
- (B) 不同的起始群集中心，可能會造成不同的分群結果
- (C) 容易受雜訊與離群值（Outlier）影響其群集中心
- **(D) 可以處理類別型資料** ✓ ← K-means 不能處理類別型

**詳解**：A、B、C 都是 K-means 的真實特性。D 錯誤 — K-means 依賴**距離計算**，類別資料無法直接算距離。
**注意**：本題在問「**不正確**」的選項，看清楚題目要選的方向。

> ⚠️ **勘誤注意**（學習指引模擬考第 3 題）：
> 學習指引原版有一題問「下列何者**並非** K-means 集群法的特點」答案是 (A)「**原理相對其他集群法較為複雜**」 — 因為 K-means 原理其實**簡單**（隨機選中心 → 分配 → 更新 → 收斂）。所以「原理複雜」是錯誤敘述（並非其特點）。
> 重點：**K-means 原理簡單、限制多**。

### B. 階層式分群（補充版）

> 🔧 **階層式分群（Hierarchical Clustering）**：白話 — 「**像家族樹一層層往上合併或往下切分**」。

**特點**：
- 不需事先指定 K
- 可視覺化成「樹狀圖（Dendrogram）」
- 計算成本高，不適合大資料

### C. PCA 主成分分析（已在 CH2 講過）

> 🔧 **PCA**：CH2 說過了 — 找出資料中最重要的幾個方向，用來**降維 + 保留資訊**。
> 在 ML 分類上，PCA 屬於**非監督式 + 降維**方法。

### D. t-SNE（補充版）

> 🔧 **t-SNE**：白話 — 「**把高維資料壓成 2D / 3D 來視覺化**」。
> 跟 PCA 不同：PCA 是線性方法，t-SNE 是非線性方法，更善於保留「相似群聚」結構。

> 0 基礎學員只需要記得：「**降維家族最常考的是 PCA**」。t-SNE 在初級題目少見。

## 3.5 強化學習基礎（30 分鐘）

> 🔧 **強化學習（Reinforcement Learning, RL）**：白話 — 「**透過試錯與獎懲學習最佳策略**」。

### 三大要素

| 要素 | 中文 | 白話 |
|---|---|---|
| **State（狀態）** | 環境當前長什麼樣 | 圍棋盤面、車子目前位置 |
| **Action（動作）** | 模型可以做的選擇 | 下哪個位置、左轉/右轉/煞車 |
| **Reward（獎勵）** | 做完動作後得到的回饋分數 | 贏棋 +1、輸棋 -1、撞車 -100 |

### 核心循環

```
Agent（模型）
 ↓ Action
環境（Environment）
 ↓ State + Reward
Agent 更新策略
 ↓ 下一個 Action
...
```

**白話**：「**做選擇 → 看結果 → 學經驗 → 下次做更好**」，跟養寵物給獎勵的邏輯一樣。

### 典型應用

- AlphaGo 下圍棋
- 自駕車決策
- 機器人手臂操作
- 遊戲 AI（Atari、星海爭霸）
- 推薦系統（探索 vs 開發）

> 💡 強化學習在初級考試只考「**辨識**」 — 看到「動態互動 + 獎懲」就選 RL。內部演算法（PPO、Q-Learning）屬於中級範圍。

## 3.6 模型訓練：過擬合 vs 欠擬合（60 分鐘，重點）

### A. 兩個敵人：Overfitting 與 Underfitting

> 🔧 **過擬合（Overfitting）**：白話 — 「**模型在訓練集表現好，測試集表現差**」。
> 學生背題庫考 100 分，遇新題就掛。

> 🔧 **欠擬合（Underfitting）**：白話 — 「**模型在訓練集都表現不好**」。
> 學生連考古題都看不懂，根本沒學進去。

**判斷表**：

| 訓練集表現 | 測試集表現 | 狀態 | 怎麼辦 |
|---|---|---|---|
| 好 | 好 | 剛剛好 ✓ | 不用調 |
| **好** | **差** | **過擬合** | 加正則化、減少模型複雜度、增加資料、early stopping |
| 差 | 差 | 欠擬合 | 加複雜度、加更多特徵、訓練更久 |
| 差 | 好 | 不正常（資料外洩 / 抽樣錯誤） | 檢查資料分割 |

### B. 防過擬合 3 大策略

| 策略 | 白話 | 比喻 |
|---|---|---|
| **正則化（Regularization）** | 給模型加規矩，不准它變太複雜 | 「考試時限制只能用 3 種解題技巧」 |
| **Early Stopping** | 訓練時觀察驗證集，表現開始下降就停 | 「看到學生開始死背就停下來」 |
| **Data Augmentation** | 用旋轉、翻轉、加噪音等擴增訓練資料 | 「同一張圖以不同角度給模型看」 |

#### 樣題練習：S1-Q3（答 D）

**題目**：深度學習模型中，下列哪一項通常用來降低過擬合（Overfitting）問題？
- (A) 增加訓練數據量
- (B) 增加模型的複雜度
- (C) 增加學習率
- **(D) 增加正則化項** ✓

**詳解**：
- (A) **增加訓練數據量也能防過擬合**（Data Augmentation 的精神），但本題在問「通常用來降低過擬合」的標準技術。
- (B) 增加複雜度 → 過擬合**更嚴重**，反向操作。
- (C) 學習率影響收斂速度 / 穩定性，跟過擬合不直接相關。
- (D) 正則化 = 防過擬合**最直接**的標準方法。

> ⚠️ 嚴格說 (A) 也對，但 (D) 是教科書最常列出的「**直接防過擬合手段**」。考試遇兩個都對的時候，選**最直接**的那個。

### C. 正則化的兩種寫法：L1 vs L2（白話版）

> 0 基礎學員只需要知道概念，公式不必背。

| 名稱 | 中文 | 白話 | 副作用 |
|---|---|---|---|
| **L1 正則化（Lasso）** | L1 範數 | 加總絕對值，**會把不重要的權重壓成 0** | **自動做特徵選擇**（不重要的特徵權重 = 0） |
| **L2 正則化（Ridge）** | L2 範數 | 加總平方值，**會把所有權重壓小但不為 0** | 所有特徵都保留，只是影響變小 |

> 💡 **記法**：
> - L1 像「**斷捨離**」 — 不要的直接歸零
> - L2 像「**節食**」 — 大家都瘦一點

### D. 損失函數選哪個？

> 🔧 **損失函數（Loss Function）**：白話 — 「**告訴模型『你猜得有多離譜』的一個分數**」，分數越低越好。

| 任務 | 適用損失函數 |
|---|---|
| **迴歸**（預測連續數值） | **MSE（均方誤差）** — 預測與真實值的平方差 |
| **分類**（預測類別） | **Cross-Entropy（交叉熵）** — 預測機率分布與真實分布的差距 |

> 💡 **記法**：「**回歸用 MSE、分類用交叉熵**」。

### E. 優化器（補充版）

> 🔧 **優化器（Optimizer）**：白話 — 「**告訴模型『該怎麼往對的方向修正』的方法**」。

| 名稱 | 特點 |
|---|---|
| **BGD（批次梯度下降）** | 每次用全部資料算一次，準但慢 |
| **SGD（隨機梯度下降）** | 每次用一個樣本，快但不穩 |
| **Adam** | 結合動量法 + RMSProp，**業界最常用** |
| **Adagrad** | 對稀疏資料特別好 |

> 0 基礎學員只需記住：「**Adam 是最常用的優化器**」。

## 3.7 模型評估：分類指標四兄弟（60 分鐘）

### A. 混淆矩陣（Confusion Matrix）

> 🔧 **混淆矩陣**：白話 — 「**把模型預測對、預測錯的情況用表格列出來**」。

以「**預測員工會不會離職**」為例：

| 實際 \ 預測 | 預測會離職 | 預測不會離職 |
|---|---|---|
| **實際會離職** | TP（真陽） | FN（偽陰）← **漏報** |
| **實際不會離職** | FP（偽陽）← **誤報** | TN（真陰） |

**4 個專有縮寫**：
- **TP（True Positive, 真陽）**：實際會 + 預測會 → 對
- **TN（True Negative, 真陰）**：實際不會 + 預測不會 → 對
- **FP（False Positive, 偽陽 / 誤報）**：實際不會 + 預測會 → 「冤枉好人」
- **FN（False Negative, 偽陰 / 漏報）**：實際會 + 預測不會 → 「縱放壞人」

> ⚠️ **與 Type I / Type II 對應**：
> - FP = Type I 錯誤（α）= 冤枉好人
> - FN = Type II 錯誤（β）= 縱放壞人

### B. 4 個核心指標

| 指標 | 公式概念 | 白話 | 適合用在 |
|---|---|---|---|
| **準確率（Accuracy）** | (TP+TN) / 全部 | 預測對的比例 | 類別均衡時 |
| **精確率（Precision）** | TP / (TP+FP) | 「**預測會離職的人，真的離職的比例**」 | 不能誤報的場景（醫療、信用評分） |
| **召回率（Recall）** | TP / (TP+FN) | 「**真正會離職的人，被找出來的比例**」 | **不能漏報的場景**（疾病篩檢、設備異常停機） |
| **F1 Score** | 精確率與召回率的調和平均 | 兩者平衡的綜合指標 | 兩個都重要時 |

> 💡 **記法（極重要）**：
> - **精確率（Precision）**：分母 = 「**模型說會的所有人**」 → **避免誤報**
> - **召回率（Recall）**：分母 = 「**真的會的所有人**」 → **避免漏報**

#### 樣題練習：S1-Q24（答 B，必考）

**題目**：一家智慧工廠使用機器學習分類模型預測關鍵設備是否會異常停機。完成模型訓練後，團隊希望全面評估模型在不同面向的表現。請問下列哪項指標最適合用來衡量模型在偵測異常停機時的「**漏報率**」（即未能正確偵測出異常事件的比例）？
- (A) 準確率（Accuracy）
- **(B) 召回率（Recall）** ✓
- (C) F1 分數（F1 Score）
- (D) 假陽性率（False Positive Rate）

**詳解**：
- 「**漏報率**」 = 真的會發生但沒抓到 = FN 比例
- 召回率 = TP / (TP+FN) = 「找出來的 / 真正會的」
- 召回率高 → 漏報少 ✓

> ⚠️ 注意：「漏報率」 = 1 - Recall，但題目問「**衡量漏報率最合適的指標**」 → 直接看 Recall 就能反推漏報。所以答案是 Recall。

> 💡 **記法**：
> - **怕漏抓** → 看 **Recall（召回率）**
> - **怕誤報** → 看 **Precision（精確率）**

### C. 迴歸用什麼指標？

| 指標 | 白話 |
|---|---|
| **MSE（均方誤差）** | 「平均誤差平方多少」，越小越好 |
| **RMSE（均方根誤差）** | MSE 開根號，跟原資料同單位 |
| **R²（決定係數）** | 模型解釋資料變異的比例，0–1 之間，越接近 1 越好 |

### D. 交叉驗證（樣題 S1-Q23）

> 🔧 **交叉驗證（Cross-Validation, CV）**：白話 — 「**把資料切成 K 份，輪流當測試集**」，避免某次抽樣運氣好/壞影響結果。

**最常用：K-Fold Cross-Validation**
- 把資料切成 K 份（常用 K=5 或 10）
- 每次拿其中 1 份當測試、其他 K-1 份當訓練
- 跑 K 次取平均

**目的**：
- ✓ 降低過擬合風險
- ✓ 評估模型**泛化能力**（在沒看過的資料上的表現）
- ✓ 避免單一次切分的運氣影響

#### 樣題練習：S1-Q23（答 C）

**題目**：交叉驗證的主要目的是什麼？
- (A) 提高模型的訓練速度
- (B) 驗證數據是否線性可分
- **(C) 減少模型的過擬合風險** ✓
- (D) 測試模型的容錯能力

**詳解**：
- (A) 交叉驗證**反而會變慢**（要跑 K 次）。
- (B) 跟線性可分性無關。
- (C) 透過多次切分平均，能更穩定地評估泛化能力，**降低過擬合風險** ✓
- (D) 容錯能力是另一回事（如資料缺失下的穩定性）。

## 3.8 特徵選取技術（30 分鐘，必考陷阱題）

> 🔧 **特徵選取（Feature Selection）**：白話 — 「**從一堆欄位中挑出對預測最有用的幾個**」。
> 跟 PCA 的差別：PCA 是把欄位「**轉換**」成新欄位；特徵選取是「**從原欄位中挑**」。

### 常見特徵選取方法

| 方法 | 類別 | 白話 |
|---|---|---|
| **Pearson 相關係數** | Filter Methods | 看每個特徵跟目標的線性相關度，相關度低的丟 |
| **PCA 主成分分析** | 廣義特徵工程（屬降維） | 找最大變異方向組合新特徵 |
| **隨機森林特徵重要度** | Embedded Methods | 用樹模型在訓練時順便算每個特徵的重要度 |
| **L1 正則化（Lasso）** | Embedded Methods | 訓練時自動把不重要的特徵權重壓 0 |
| **遞迴特徵消除（RFE）** | Wrapper Methods | 反覆訓練模型，每次刪掉最不重要的特徵 |

### ⚠️ S1-Q22 必考陷阱題（答 C）

**題目**：請問下列何者**不是**常見的特徵選取技術或方法？
- (A) 皮爾森積差相關分析（Pearson Correlation）
- (B) 主成分分析（Principal Components Analysis, PCA）
- **(C) 迴歸分析（Regression Analysis）** ✓
- (D) 隨機森林（Random Forest）

**詳解**（⚠️ 這題答案常被誤記）：
- (A) Pearson：經典 Filter Methods ✓ **是**特徵選取技術
- (B) PCA：降維 / 廣義特徵工程 ✓ **是**特徵選取相關技術
- (C) **迴歸分析：是預測模型，不是特徵選取技術** ← **正解**
- (D) 隨機森林：透過特徵重要度（Feature Importance）**輔助**特徵選取 ✓ **是**常見方法

> ⚠️ **重要警告**：
> 此題答案是 **C（迴歸分析）**，不是 D。
> **隨機森林**透過特徵重要度可以輔助特徵選取，**屬於常見方法**。
> 「迴歸分析」是預測模型（用來「預測一個數字」），不是用來「挑特徵」的技術。
>
> 千萬不要記成「隨機森林不是特徵選取技術」 — 那是錯的。
>
> 一句話記法：「**迴歸是預測，不是選特徵**」。

## 觀念補充（延伸理解）

### 監督式 / 非監督式 / 強化學習三模式記憶法

監督式 = 「有人在看著他」（給 Y 標籤，例：垃圾郵件二分類、房價迴歸）。非監督式 = 沒有明確標籤，做分群（例：RFM 八種客群—消費金額/頻率/最近時間，2×2×2）。強化學習 = 策略型、有獎勵分數、最後拿最高分（例：圍棋一步累積到最後勝率）。**重要區分**：分類（有明確類別 = 資源回收、寶特瓶/紙類）vs 分群（看起來像就分一群 = 國際班按長相分歐美 vs 亞洲）—**監督式才能做「分類」、非監督式只能做「分群」**。AlphaGo 就是強化學習，它在圍棋網站打到頂用了 3 個月（人類要 6-7 年）。### 召回率的「捉放法」記憶法

高風險疾病早期篩檢題目中，正常人遠多於感染者，漏診（偽陰性）代價可怕——這類題目方向就是看召回率。背的方式：小學自然課的「捉放法」——抓回來幾隻、答對幾個。模型如果只看精確度會變得保守，怕精確度掉就盡量都判沒病；但這樣召回率會變得非常糟。「一個都不能漏」的場景永遠看召回率。詐騙簡訊偵測也是同理—FP（誤報）→使用者更小心、沒太大壞處；FN（漏報）→使用者放心點擊、上當，所以詐騙也看召回率。

特異度是反過來的召回率（捕捉正常人的能力），考試出現率較低。### 混淆矩陣的「驗 100 個人」陷阱例

100 個人中只有 1 人有病，全猜「沒病」**準確率（accuracy）也有 99%**，但**精確率（precision）= 0%**（你說 Yes 的有多少真的 Yes，你都沒說 Yes 所以沒分母）。這個例子是混淆矩陣最經典的「陷阱」，看到「不平衡資料」+「少數類才是重點」（疾病、詐騙、瑕疵、異常）幾乎一定要看精確/召回不是準確。實務應用 = 醫療診斷、金融風暴、製造瑕疵、公共安全。### ROC 曲線面積的判讀

ROC 曲線下面積 100% = 完美（TP 跟 TN 完全分得開）；50% = 全靠猜（亂答都對一半）；0% = 完全反過來（其實只要倒過來解讀就是 100%，所以實務出 0% 通常表示模型定義錯了）。商用 AI 通常以 80% 為基準往上拉，99% 已經很不錯了，「**100% 拿出去討論一定會被打槍**—世界上沒有絕對的事情，模型出 100% 通常代表你的測試集設計有問題」。### 訓練集 / 測試集 / 驗證集的「課本-習作-考試」比喻

訓練集 = 課本（70-80%）；測試集 = 老師出的考題（20-30%）；驗證集 = 習作練習，不一定要有。為什麼不能拿全部當訓練集再自己出考題？因為自己出可能會把資料集的規律打亂、變得沒參考性。**過擬合（overfitting）= 死讀書**（記選項排列組合不真理解題目）；**欠擬合（underfitting）不是過擬合的反面**—欠擬合是訓練集本身就爛，「**AI 沒有不讀書還考試考很好的情況，要爛就一起爛**」。交叉驗證（K-fold）平均切，每段輪流當測試，缺點是運算成本變 K 倍。### 泛化能力的車牌辨識五因素

車牌辨識是泛化能力最直觀的例子。影響因素 5 個：(1) 資料量太少 → 模型死記細節；(2) 資料品質差（磨損嚴重的車牌、車牌邊框）；(3) 模型太複雜 → 過擬合；(4) 沒做正則化；(5) 資料多樣性不足（沒涵蓋軍車、重機四碼、舊式 ABC 開頭、現在的 ABC 開頭）。露天 vs 室內、白天 vs 晚上、起霧、山路水滴—都是泛化能力測試的場景。### 梯度下降的「下山找坑」比喻

學習率（learning rate）= 你學的速度。從山頂往下走、要記下會造成爬山危險的坑洞。學習率太大 → 跳過整個山谷（彈到很遠、失準）；學習率太小 → 把每個彈珠大小的洞都記下來（學很慢、成本高）。實作技巧：先設適中的學習率，再慢慢往下調。**常見問題**：(1) 局部最小值（卡在小坑找不到全局，「以為到山腳了其實還沒」）；(2) 梯度爆炸（權重失控發散，根本找不到任何規律）。### CNN 四層的口頭順序

卷積層（找特徵 = 邊緣/輪廓/材質）→ 池化層（降維 = 留下主要輪廓）→ 全連接層（分類）→ 輸出。卷積層找的「材質」就是電腦學會的紋理：毛、鐵、塑膠等。貓的辨識會留毛、耳朵、腿長度、臉，把腳掌、肚子、尾巴砍掉降維。增加非線性是激勵函數（activation）的工作—讓神經網路可以處理「圖片這種非線性資料」，否則整個網路退化成單層線性回歸。### Dropout = 板凳球員都要練

訓練時刻意關掉一些 feature 的權重，讓模型不依賴特定幾個特徵。比喻：帶隊比賽不能只練先發球員—先發出狀況時板凳上去要能打。下次新資料如果有缺值或錯誤，模型才不會崩。Dropout 屬於正則化的一種；正則化的核心是「**增加懲罰項或弱化獎勵項，限制每個 feature 的權重大小**」，AI 模型訓練本質是「**學規則不是學數據**」，這兩件事差非常多。### Focal Loss 的考試決策樹

題目出現「極端不平衡」、「樣本少類」、「希望模型專注少數類」——直接綁 focal loss。其他三個常見的損失函數要這樣記：

- 均方誤差（MSE）→ 線性迴歸（預測天氣、自然災害、線性股價）
- 交叉熵 → 標準分類，但極不平衡時容易被多數類帶走
- 餘弦相似度 → 多元特徵向量方向比對，不會用在二分法

例如「信用卡盜刷偵測：99.9% 正常、0.1% 盜刷」這種極端不平衡的二分法情境，答案永遠是 focal loss。### 概念漂移 vs 資料漂移的場域差異

**概念漂移**：特徵與目標的「邏輯關係」改變。例：以前流行養狗、現在流行養貓；詐騙手法從現金改成 LINE Pay。

**資料漂移**：純粹輸入資料的分布改變。例：模型部署到新國家、用戶族群結構變了。

兩者可以同時發生。題目描述「半年後流行趨勢改變」+「點擊率下降」就是兩者都有。### LIME（局部）vs SHAP（全域）的銀行貸款例子

兩個都「跟模型不可知」（model-agnostic）—可以用在任何模型上。**LIME** = 解釋這「一筆」資料為何被判成這樣（例：某客戶只要月收入再加 X、信用紀錄加 Y、負債比降 Z 就會核准）。**SHAP** = 計算每個「特徵」對整個模型預測的貢獻度，能講出「在這個模型裡，信用紀錄是最重要的、其次是收入、年齡和負債比」。考試決策樹：「邊際貢獻度」「每個特徵」「整體模型」→ SHAP；「單一筆預測」「局部解釋」→ LIME。模型審查（model audit）的目的就是驗證 model 在挑選參數、學習率調整、決策過程上的合理性與合規性。### 推論模式四類（線上 / 批次 / 離線 / 分散式）

(1) **線上推論** = 即時、低延遲，例：ChatGPT 網頁版、即時影像辨識；(2) **批次推論** = 一次處理大量資料，例：每月財報分析、每月庫存補貨建議；(3) **離線推論** = 不急著輸出，開發測試或非即時場景常用；(4) **分散式推論** = 雲端 AI 服務的主流，OpenAI 就是「線上 + 分散式」結合。各場景的速度/成本/算力需求不同，自駕車這種一定要邊緣 + 線上、不能等雲端來回；批次的優勢就是吞吐量最大，但延遲也最大。### 模式崩潰、災難性遺忘、梯度爆炸的場域對照

這三個容易混，記得各自的場域：

- 模式崩潰（mode collapse）：GAN 專屬，生成只剩一種結果、訓練失敗
- 災難性遺忘（catastrophic forgetting）：學新忘舊，跟增量學習相反
- 梯度爆炸（gradient explosion）：訓練初期參數不收斂、數值極端兩極

題目場域不同就選對應名詞。### 增量學習為何成為現代 AI 標配

模型要「持續學新知識又不能忘舊」。如果不保留這能力，時代一變、更新越多東西模型越亂。所以多數線上 AI 模型都會盡可能保留增量學習機制。### 公平性指標三層差異

- 人口統計對等（demographic parity）：不同族群的核准比例必須一致（保送名額/保底名額概念）
- 均等機會（equal opportunity）：真正會還款的人裡面被核准的機率一致（用結果回推）
- 精準機會差異跟條件機率有關，記得最關鍵的差異是「全體比例 vs 真正陽性裡的比例」

題目寫「不論種族為何、批准機率必須完全相等」就綁 demographic parity。### GPU 為何取代 CPU 在 AI 場景

CPU 序列執行（一次一件、作業系統管控用）。GPU 平行多工（同時算很多包），AI 把資料轉成數字後要做大量矩陣運算 → GPU 最適合。「**機器學習跨到深度學習，最大的轉折就是 GPU 出現**」，所以黃仁勳跟 AI 綁定那麼深、輝達現在那麼搶手。TPU 是 Google 自家為 AI 開發的晶片，類似邏輯但比較少人用。Tensor Core 是 GPU 內部硬體層級加速「混合精度矩陣乘加」的單元，CUDA 負責損失函數/激勵函數/各種網路運算，Tensor Core 專做矩陣那塊。### 不平衡資料的「全猜負樣本」陷阱

正樣本 200 筆 / 負樣本 9800 筆（2% vs 98%）→ 全猜負樣本準確率就有 98%，但精確率 0%。要解這種題，重點是「**少數類別才是任務的重點**」—醫療找疾病、金融抓詐欺、製造業找瑕疵、公共安全找異常。看到這四類場景就要警覺要看精確率/召回率而不是準確率。對策：過採樣（少數類複製/合成）+ 欠採樣（多數類隨機抽）+ 改用 focal loss + 改評估指標。### QAT vs PTQ 的訓練時機差別

模型量化部署到邊緣裝置時兩種做法：

- **PTQ（訓練後量化）**：訓練好直接砍精度（四捨五入、無條件捨去），會讓資料間關係亂掉、精度雪崩
- **QAT（量化感知訓練）**：訓練階段就模擬量化誤差，讓神經網路自己調整權重去適應低精度

邊緣裝置算力差，所以要在訓練時就考量「我未來只能用這種精度」。「邊緣裝置 + 直接 INT8 + 精度雪崩」這組關鍵字一定走 QAT。### 履歷篩選 AI 偏見的前處理對策

性別偏見的修法在「資料前處理」階段最直接的做法是「重新抽樣 + 合成資料平衡」。其他選項的錯誤點：

- 增加深度 → 反而會過擬合、放大偏見，而且不算「資料前處理」
- 直接刪除女性樣本 → 邏輯上等於把它當離異值，但題目沒說它是離異值
- one-hot encoding → 只改格式，不影響偏差統計分布### 特徵庫（Feature Store）的職責邊界

特徵庫的核心是「統一管理 + 共享重用」。它不是 SQL 替代品、不是自動標註工具（標註屬於資料工程）、不是模型權重存放處（那是 model registry）。

請記住四層分工：**訓練流程 → 資料 → 特徵 → 模型**，這四層是分開的、各有對應的管理工具。---
