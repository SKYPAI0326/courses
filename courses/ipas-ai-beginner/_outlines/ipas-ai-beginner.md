## 2. 學習成果（Learning Outcomes）

研習結束後，學員應能：

### 2.1 知識面（Knowledge）

- [ ] 說明 AI 三大類型（分析型 / 預測型 / 生成型）的差異與企業應用
- [ ] 區分結構化、半結構化、非結構化資料，並舉出企業常見實例
- [ ] 解釋 ETL（萃取/轉換/載入）每個階段該處理什麼問題
- [ ] 區分監督式、非監督式、強化學習三大 ML 類型，並判斷情境適用
- [ ] 區分鑑別式 AI 與生成式 AI 的數學意義與應用差異
- [ ] 說明 No Code vs Low Code 平台的定位差異與選型考量
- [ ] 列出生成式 AI 在 6 大產業（醫療/製造/金融/零售/教育/客服）的應用案例
- [ ] 說明生成式 AI 導入的 4 階段流程（準備→設計→POC→實施）
- [ ] 解釋風險管理 6 策略（溯源/文化/接受/緩解/迴避/轉移）

### 2.2 應試面（Exam Skills）

- [ ] 解出官方樣題 35 題（正確率 ≥ 90%）
- [ ] 解出指引內 70 題模擬考題（正確率 ≥ 85%）
- [ ] 完成 1 次模擬測驗（80 題，正確率 ≥ 70%，合格門檻）
- [ ] 識別常見陷阱題型（如「平均差絕對差值」「Type I/II 錯誤」「K-means 限制」）

### 2.3 應用面（Application）

- [ ] 在自己工作場景中提出至少 1 個生成式 AI 應用構想（CH6 動手題）
- [ ] 為某假想企業撰寫 1 份「生成式 AI 導入評估」初步報告（CH7 動手題）

### 2.4 進階面（Bridge to Mid-level）

- [ ] 認識中級三科目（AI 技術應用規劃 / 大數據處理 / 機器學習應用）的範圍
- [ ] 知道「初→中級」的關鍵跳躍點（程式碼閱讀、進階統計、模型架構辨識）
- [ ] 拿到一份「進階學習路線圖」可自行接續學習

**內容**：

**動手活動**：

### CH1 人工智慧概念（3h）

**學習目標**：能說明 AI 是什麼、有哪些分類、企業怎麼用、政府怎麼治理。

**單元結構**：
1.1 AI 的定義與分類（45 min）
- 什麼是 AI？歷史小故事（1956 達特茅斯會議起源）
- 三大類型：分析型 / 預測型 / 生成型（每個給 3 個職場常見例子）
- ChatGPT 屬於哪一類？

1.2 AI 應用領域（45 min）
- 5 大產業案例（醫療/金融/製造/交通/娛樂）
- AI 多層次架構（技術底層 → 開發應用 → 實際運用）

1.3 AI 治理概念（60 min）
- AI 為什麼需要治理？（公平性事件案例）
- 6 大原則：公平、透明、問責、可解釋、隱私、風險分級
- 歐盟 AIA 風險分級：4 級（不可接受/高/有限/低）
- 負責任 AI（Responsible AI）

1.4 章末練習（30 min）
- 樣題第 1、4、6、31 題詳解
- 快問快答 5 題

**0 基礎友善元素**：
- 「AI vs 機器學習 vs 深度學習」用同心圓圖解
- 治理原則用「自駕車事故誰負責」案例導入
- 風險分級對照「日常 AI 應用」舉例

---

### CH2 資料處理與分析概念（5h）⭐ 重點章

**學習目標**：能判斷資料品質問題、選擇適合的處理方式、理解基本統計。

**單元結構**：
2.1 資料的種類與來源（60 min）
- 結構化 vs 半結構化 vs 非結構化（用 Excel/JSON/X 光片三圖對比）
- 資料蒐集方法（問卷、自有、外部、爬蟲）
- 大數據 4V 特性（量、速、多、價）

2.2 資料清洗（60 min）
- 4 大常見問題：遺缺、重複、錯誤、離群
- 每種處理方式（平均填補/中位填補/刪除/業務判斷）
- 案例：員工資料庫整理實作

2.3 ETL 流程（45 min）
- Extract / Transform / Load 三階段
- 「客戶名稱不一致」屬於哪個步驟？（樣題第 18 題）
- 標準化：Min-Max vs Z-score（公式 + 計算範例）

2.4 資料分析 4 大方法（60 min）
- 敘述性、探索性、診斷性、預測性
- 視覺化工具（直方圖、散佈圖、箱型圖、熱圖）

2.5 統計量與假說檢定（90 min）⭐
- 集中趨勢：平均/中位/眾數（適用情境對比）
- 分散程度：全距/IQR/標準差（含勘誤後「平均差」定義）
- 偏態判定（平均 vs 中位）
- 假說檢定：H₀ / H₁ / α / p 值
- **Type I / Type II Error 表**（用勘誤後正確版）
- 特別解析：樣題第 7 題「95% 信心水準下拒絕 H₀」(陷阱題)

2.6 章末練習（45 min）
- 樣題第 8-18、22、27、34 題詳解（共 13 題）
- 模擬題 26 道（從補充題庫抽組）

**0 基礎友善元素**：
- 統計量用「全班 30 個學生身高」具體例子
- 標準差用「品管廠長煩惱」故事帶入
- 假說檢定用「藥廠新藥試驗」場景說明
- 把所有公式放在「速查表」附錄，課堂不背公式

### CH3 機器學習概念（5h）⭐ 重點章

**學習目標**：能說明 ML 的三大類型、判斷情境適用哪種、理解評估指標。

**單元結構**：
3.1 機器學習基本原理（45 min）
- 「機器怎麼學」用兒童識字類比（看多、糾正、進步）
- 3 大核心要素：數據 + 模型 + 損失函數
- 與「傳統程式設計」的本質差異

3.2 三大學習類型（90 min）
- **監督式學習**：標記資料 → 分類/迴歸（垃圾郵件、房價）
- **非監督式學習**：無標記 → 聚類/降維（市場區隔、PCA）
- **強化學習**：試錯 + 獎勵（圍棋 AlphaGo、自駕、機器人）
- 情境判斷練習（樣題第 2、5、13 題）

3.3 模型訓練核心（60 min）
- 損失函數：MSE 用迴歸 / 交叉熵用分類
- 優化器：BGD / SGD / Adam（每個適用情境）
- 防過擬合 3 招：Regularization / Early Stopping / Data Augmentation

3.4 模型評估（60 min）
- 準確率 / 精確率 / 召回率 / F1 / R²（用「醫療誤診」與「漏報」案例對照）
- 混淆矩陣（TP/TN/FP/FN）
- 交叉驗證（K-Fold）目的
- 樣題第 24 題「漏報率 = Recall」精解

3.5 常見演算法概覽（60 min）
- 線性迴歸 / 邏輯迴歸 / KNN
- 決策樹 / 隨機森林（含 Bootstrap）
- SVM
- K-means / DBSCAN（含 K-means 限制：不適合非球形）
- ⚠️ 樣題 S1-Q22 陷阱題正解：**「迴歸分析」是預測模型不是特徵選取技術**（注意：隨機森林透過特徵重要度可輔助特徵選取，屬於常見方法）

3.6 章末練習（45 min）
- 樣題第 2、3、5、9、13、17、19、20、22-25 題詳解（共 12 題）
- 機器學習速查表（演算法 vs 適用情境）

**0 基礎友善元素**：
- 用「水果分類遊戲」帶監督式學習
- 用「Spotify 推薦歌單」帶非監督式
- 用「養寵物給獎勵」帶強化學習
- 演算法用「我該選哪一個」決策樹，避免從數學公式講起

### CH4 鑑別式 AI 與生成式 AI 概念（4h）

**學習目標**：能說清楚兩者本質差異、各自應用、整合應用情境。

**🚨 0 基礎友善分層原則**（CH4 內容密度高，必須分層撰寫）：
- **「考試必懂白話版」**（主線講義內容）：用比喻說明差異、能判題即可（例：條件機率 P(y\|x) → 「給我一張圖，告訴我是什麼」）
- **「補充理解版」**（標註為「想多懂一點？」摺疊區塊）：列出數學式、模型架構細節（VAE 隱變量空間、Diffusion 反向擴散等），不要求學員理解
- 學員不必背數學式即能答題；模型名稱（GAN/VAE/Diffusion/Autoregressive）只需認得用途，不必懂內部機制

**單元結構**：
4.1 鑑別式 AI 概念（60 min）
- 條件機率 P(y|x) 白話版：「給我一張圖，告訴我是貓是狗」
- 典型模型：邏輯迴歸 / SVM / 決策樹 / 隨機森林 / 神經網路
- 應用：分類、預測、辨識

4.2 生成式 AI 概念（60 min）
- 聯合分佈 P(x,y) 白話版：「我能生出一張新的貓的圖」
- 典型模型：GAN / VAE / Diffusion / Autoregressive
- 應用：圖像生成、文字創作、音樂、Deepfake
- 重點 Diffusion Model（DALL-E/Stable Diffusion 用此）

4.3 深度學習三大模型（60 min）
- **CNN**：影像處理（卷積層 + 池化 + 全連接）
- **RNN / LSTM / GRU**：序列數據（語言、時間序列）
- **GAN**：生成器 + 鑑別器對抗
- 樣題第 26 題（GAN 組成）詳解

4.4 整合應用（45 min）
- 數據增強協同：GAN 生成醫療影像 + CNN 分類
- 多模態：自駕模擬 + 環境感知
- 即時分析回饋：客服系統、安全防禦
- 樣題第 7、29、33 題詳解

4.5 章末練習（15 min）
- 樣題第 7、26、28-30、32-33 題詳解（共 7 題）

**0 基礎友善元素**：
- P(y|x) vs P(x,y) 用「考題分類 vs 出新考題」對比
- GAN 用「警察與小偷對抗」故事
- Diffusion 用「拍立得顯影」反向類比

**學習目標**：能區分 NC/LC 適用場景、知道 6 大評估因素、認識 AI 民主化。

**單元結構**：
5.1 NC vs LC 定位（30 min）
- 圖解差異：NC 純拖拉 / LC 加少量程式
- 適用對象：非技術 vs 有技術背景
- 樣題第 1、5、8 題詳解

5.2 6 大應用領域實例（30 min）
- 醫療、製造、金融、零售、教育、客服

5.3 6 大評估因素（30 min）

5.4 4 大挑戰與 AI 民主化（30 min）

- AI 民主化、市民開發者
- 樣題第 6 題詳解

**章末練習**：樣題科目二第 1-8 題詳解 + 5 題小測驗

**0 基礎友善元素**：
- 用「Wix vs WordPress vs 寫網站」對比 NC/LC/Code
- 6 大產業每個給 1 個「你也能用」工具推薦（如 Make.com、n8n）

### CH6 生成式 AI 應用領域與工具使用（3h）

**學習目標**：能說明生成式 AI 5 大領域應用、操作主流工具、寫好 prompt。

**🚨 0 基礎友善分層原則**（CH6 名詞多，要避免轟炸）：
- **「考試必懂白話版」**（主線）：Temperature、Top-k、Top-p 用一句話對應「保守 vs 創意」即可
- **「補充理解版」**（摺疊區塊）：Tokenization、Vectorization、Self-Attention、RLHF、vLLM 等放這裡，不列入主線測驗
- 主線只保留「能判題的差異」（如：「邏輯清晰應用要把 Temperature 設低」），不要求理解採樣演算法數學
- 工具操作（ChatGPT/Claude/DALL-E）才是 CH6 的核心動手練習，理論背景退居輔助

**單元結構**：
6.1 生成式 AI 技術架構（45 min）
- 深度學習網路（多層神經網路 + 注意力機制）
- 訓練數據處理（清洗、Tokenization、向量化）
- 推理機制：Temperature / Top-k / Nucleus Sampling
- 樣題第 33 題：Temperature 設定（0.1-0.3 vs 1.0-1.2）

6.2 5 大領域實際應用（45 min）
- 藝術設計、醫療生技、教育培訓、娛樂媒體、產品設計
- 案例展示：DALL-E、Midjourney、Whisper、ChatGPT、Claude

6.3 主流工具操作（60 min）
- ChatGPT / Claude / Gemini 對話操作（角色設定、任務描述、迭代）
- DALL-E / Midjourney 圖像生成（提示詞結構、版權考量）
- Whisper 語音轉文字
- 多模態應用情境（樣題第 20 題：A 企業客服系統）

6.4 提示工程基礎（30 min）
- Prompt 結構：角色 + 任務 + 格式 + 範例
- Chain-of-Thought（CoT）思維鏈引導（樣題第 6 題）
- 幻覺現象（Hallucination）辨識（樣題第 30 題）

**章末練習**：樣題第 9-22 題詳解 + 動手作業（為自己工作場景寫 1 個應用構想）

**0 基礎友善元素**：
- Prompt 用「點奶茶下單」白話比喻（指定甜度、冰塊、加料 = 角色 + 任務 + 格式）
- 幻覺用「自信的塑膠記者」案例（明明不存在的法規）

### CH7 生成式 AI 導入評估規劃（3h）

**學習目標**：能說明 4 階段導入流程、做基本 ROI 計算、認識風險管理。

**單元結構**：
7.1 4 階段導入總覽（45 min）
- 準備 → 設計 → POC → 實施
- 每階段任務、關鍵交付物
- 樣題第 1、6、29、35 題詳解

7.2 準備階段細節（30 min）
- 掌握企業課題（目標、範圍、改善項目）
- 檢視 AI 方案與企業資源
- 應用領域優先順序評估

7.3 設計階段細節（30 min）
- 確認最終目標與 KPI
- 確認數據狀態
- 估算 AI 導入成本（軟硬體 / 雲端 / 培訓）

7.4 POC 與 ROI 計算（45 min）
- POC 5 大評估方法（驗證基準、自動化測試、人工檢查、壓力測試、A/B 測試）
- ROI 公式 + NPV/IRR 概念
- TCO（總擁有成本）

7.5 實施 / 營運（30 min）
- 模型監控：Data Drift 偵測、自動重訓 Pipeline
- AI 價值擴散

7.6 風險管理（45 min）
- 6 大策略：溯源 / 文化 / 接受 / 緩解 / 迴避 / 轉移
- 5 大常見風險：訓練數據洩漏 / 反向工程 / 提示詞攻擊 / 對抗性攻擊 / 倫理偏見
- 樣題第 27、28、31 題詳解
- 法規對應：GDPR / CCPA / 個資法

**章末練習**：樣題第 23-35 題詳解 + 動手作業（撰寫某假想企業生成式 AI 導入評估報告）

**0 基礎友善元素**：
- 4 階段用「裝潢房子」類比（規劃 / 設計 / 樣品屋 / 入住）
- ROI 用「健身房會員」例子算給看
- 風險策略用「下雨天怎麼辦」生活化例子

## 9. 0 基礎友善設計總原則（硬規則）

每章必須遵守：

1. **章首「這章學完你能回答的日常問題」**：3-5 個問句，直觀勾連工作場景
2. **每個專業名詞首次出現**：附 1-2 句白話定義 + 1 個日常或商業舉例
3. **英文縮寫首次出現**：完整拼字 + 中譯 + 主要用途
4. **公式類概念**：以「為什麼要算這個」帶入，公式列在速查表
5. **章末對照「考題長相」**：放 1-2 題官方樣題，連結到題目原文
6. **「容易混淆點」專欄**：列出常見誤解 vs 正解（如 Type I/II 錯誤）
7. **章末「速查卡」**：1 頁濃縮所有名詞與關鍵公式
