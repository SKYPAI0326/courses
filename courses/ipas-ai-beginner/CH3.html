先把機器學習想成「教小孩做事」

為什麼這很重要
CH3 是 CH2「乾淨資料」的下一步。你在 CH2 把資料整理好之後，就是要餵給機器學習模型。本章 13 題覆蓋全試卷最大比例，每一題都直接影響成績，請特別專心。
名詞定義
機器學習（Machine Learning, ML）：「讓電腦從大量資料裡，自己找規律」，不需要工程師寫死規則。
傳統程式機器學習
工程師寫好「規則」，電腦照規則做事工程師給「範例資料」，電腦自己找規則

邏輯死板、規則改不動規則隨資料更新自動進化

教小孩學「認水果」的三種方式，這個比喻會貫穿整章：

教法對應 ML 類型故事
「這是蘋果，那是香蕉」— 一個個告訴他監督式學習每張圖都標好答案
「自己分組玩玩看」— 給他一籃水果不告訴名字非監督式學習他會自己依顏色 / 形狀 / 大小分群
「你猜對給糖、猜錯不給」— 用獎懲讓他學強化學習多次嘗試後找到最佳策略
💡
AI ⊃ ML ⊃ DL 三層關係：深度學習（DL）在機器學習（ML）裡，機器學習在人工智慧（AI）裡。深度學習一定是機器學習，但機器學習不一定是深度學習。

樣題 S1-Q2（答 D）— 強化學習必考，展開查看詳解
下列何者最適合訓練電腦下圍棋、自動駕駛等動態重複地互動的問題？
(A) 監督式學習
(B) 非監督式學習
(C) 半監督式學習
(D) 強化學習（Reinforcement Learning） ✓
詳解：題目關鍵詞「動態 + 重複 + 互動」= 強化學習的招牌特徵。圍棋每下一步都會收到「贏 / 輸 / 盤面好壞」的回饋訊號，模型透過這些訊號慢慢學最佳策略。
樣題 S1-Q5（答 A）+ S1-Q13（答 D）— 監督式學習
S1-Q5：若要設計一個能夠辨識並過濾垃圾郵件的系統，應該選擇下列哪一種機器學習演算法？
(A) 監督式學習（Supervised Learning） ✓
(B) 非監督式學習
(C) 半監督式學習
(D) 強化學習
詳解：垃圾郵件辨識需要事先有「已標記為垃圾 / 非垃圾的歷史郵件」當訓練資料 → 監督式學習的標準場景。
S1-Q13：某行銷公司希望預測新客戶是否可能完成購買行為。資料中包含「客戶基本資料、過往點擊行為、類似客戶是否曾購買過（已標記「有 / 無購買」標籤）」。下列哪一種學習方式最合適？
(A) 使用非監督式學習，分析所有客戶點擊路徑進行異常偵測
(B) 使用非監督式學習，將客戶分群後預測其轉換率
(C) 使用監督式學習，針對未標記資料直接預測轉換可能性
(D) 使用監督式學習，以已知購買結果作為標籤進行訓練 ✓
詳解：題目明寫「已標記有 / 無購買標籤」→ 一定是監督式學習。(C) 雖也是監督式，但「針對未標記資料訓練」邏輯錯誤。(D) 用已標記資料訓練，再去預測新客戶，是標準監督式流程。
樣題 S1-Q35（答 B）— 標籤 vs 特徵 vs 超參數
某電商平台希望預測商品的退貨機率，輸入是「商品售價、顧客年齡、運送天數」，輸出是「是否退貨（是 / 否）」。「是否退貨」應歸類為下列哪一類變數？
(A) 特徵（Feature）
(B) 標籤（Label / Target） ✓
(C) 超參數（Hyperparameter）
(D) 正則化係數（Regularization Coefficient）
詳解：特徵（Feature）= 模型的輸入（售價、年齡、天數）；標籤（Label）= 模型要預測的輸出（是否退貨）；超參數（Hyperparameter）= 人工指定的訓練設定值（如學習率、樹的深度）。
記法：「輸入是特徵、輸出是標籤、設定值是超參數」。
(03)

3.3 監督式學習：常見演算法速查

監督式學習依「輸出是什麼類型」再分兩支：分類（輸出類別標籤）和迴歸（輸出數值）。

線性迴歸
監督 · 迴歸
找一條最能貼合資料的直線，預測連續數值（房價、銷售額）
邏輯迴歸
監督 · 分類
名字有「迴歸」但其實是分類演算法。輸出 0–1 機率再判類別
決策樹
監督 · 分類
用一連串是非題畫成樹。可解釋性高，容易過擬合
隨機森林
監督 · 集成
種一片決策樹，多數決定答案。可透過特徵重要度輔助特徵選取
SVM
監督 · 分類
在兩類資料之間畫最公平的分隔線，適合高維資料
KNN
監督 · 分類
物以類聚，看周圍 K 個最近鄰居誰多就分到哪一類
Naive Bayes
監督 · 分類
用機率做分類，假設每個特徵互相獨立。適合垃圾信過濾
K-means
非監督 · 分群
指定要分 K 群，演算法自己把資料分 K 群。原理簡單但限制多
PCA
非監督 · 降維
找出資料中最重要的幾個方向，降維 + 保留資訊（CH2 已介紹）
樣題 S1-Q20（答 B）— 線性迴歸適用場景
線性迴歸模型最適合解決哪種類型的問題？
(A) 圖像分類
(B) 銷售額預測 ✓
(C) 聚類分析
(D) 遊戲策略學習
詳解：(A) 圖像分類 → 分類演算法。(B) 銷售額 → 連續數值預測，正解。(C) 聚類 → 非監督式。(D) 遊戲策略 → 強化學習。
記法：「看到預測一個數字就想線性迴歸；看到預測一個類別就想分類演算法」。
樣題 S1-Q17（答 A）— 決策樹：醫療研究
某醫院研究特定心血管疾病的成因，收集了 50 名病患與 150 名正常人的年齡、血壓、血型等三項屬性變數。此研究適合使用下列哪一種機器學習模型來建立？
(A) 決策樹（Decision Tree） ✓
(B) 線性迴歸
(C) DBSCAN
(D) K-means 聚類
詳解：任務 = 分類（病 / 不病）；屬性 = 數值（年齡、血壓）+ 類別（血型）混合；醫療研究需要可解釋（醫生要看得懂規則）→ 決策樹三項都符合。
記法：「醫療研究 + 可解釋」就想決策樹。
(04)

K-means 的限制（必考陷阱）

⚠️ S1-Q9 必考陷阱：K-means 不能處理類別型資料。這是最常被誤選的考點。
K-means 的限制白話說明
❌ 不適合類別資料因為要算「距離」，類別（如顏色、性別）沒有距離概念
❌ K 要事先指定不知道要分幾群時很麻煩
❌ 對離群值敏感一個離群值會把中心拉歪
❌ 不適合非球形群集環狀、長條狀的群會被切錯
❌ 不同起始點結果可能不同隨機性影響穩定性
樣題 S1-Q9（答 D）— K-means 不正確敘述
關於 K 平均法（K-means），下列敘述何者「不」正確？
(A) 希望找出 k 個互不交集的群集
(B) 不同的起始群集中心，可能會造成不同的分群結果
(C) 容易受雜訊與離群值（Outlier）影響其群集中心
(D) 可以處理類別型資料 ✓ ← K-means 不能處理類別型
詳解：A、B、C 都是 K-means 的真實特性。D 錯誤 — K-means 依賴距離計算，類別資料無法直接算距離。
注意：本題在問「不正確」的選項，看清楚題目要選的方向。
(05)

3.6 過擬合 vs 欠擬合（重點）

過擬合（Overfitting）vs 欠擬合（Underfitting）完整對比
過擬合（Overfitting）
訓練集表現好（甚至 99% 以上）
測試集表現差（大幅落差）
白話比喻學生背題庫考 100 分，遇新題就掛
解法加正則化、Early Stopping、增加資料量、Data Augmentation
欠擬合（Underfitting）
訓練集表現差
測試集表現差
白話比喻學生連考古題都看不懂，根本沒學進去
解法加複雜度、加更多特徵、訓練更久
「訓練好測試差」= 過擬合；「訓練都差」= 欠擬合

防過擬合三大策略：

策略白話比喻
正則化（Regularization）給模型加規矩，不准它變太複雜「考試時限制只能用 3 種解題技巧」
Early Stopping訓練時觀察驗證集，表現開始下降就停「看到學生開始死背就停下來」
Data Augmentation用旋轉、翻轉、加噪音等擴增訓練資料「同一張圖以不同角度給模型看」

正則化兩種主要寫法：

名稱白話副作用（考試考點）
L1 正則化（Lasso）加總絕對值，會把不重要的權重壓成 0自動做特徵選擇（不重要的特徵權重 = 0）
L2 正則化（Ridge）加總平方值，會把所有權重壓小但不為 0所有特徵都保留，只是影響變小
💡
記法：L1 像「斷捨離」— 不要的直接歸零；L2 像「節食」— 大家都瘦一點。
樣題 S1-Q3（答 D）— 防過擬合最直接的方法
深度學習模型中，下列哪一項通常用來降低過擬合（Overfitting）問題？
(A) 增加訓練數據量
(B) 增加模型的複雜度
(C) 增加學習率
(D) 增加正則化項 ✓
詳解：(A) 增加訓練數據量也能防過擬合（Data Augmentation 的精神），但本題在問「通常用來降低過擬合」的標準技術。(B) 增加複雜度 → 過擬合更嚴重，反向操作。(C) 學習率影響收斂速度，跟過擬合不直接相關。(D) 正則化 = 防過擬合最直接的標準方法。
嚴格說 (A) 也對，但 (D) 是教科書最常列出的「直接防過擬合手段」。考試遇兩個都對的時候，選最直接的那個。
(06)

3.7 模型評估：指標四兄弟 + 混淆矩陣

以「預測員工會不會離職」為例，混淆矩陣把模型預測對 / 預測錯的情況列成表格：

實際 \ 預測預測會離職預測不會離職
實際會離職 TP（真陽）
正確預測到了		 FN（偽陰）
漏報 — 縱放壞人
實際不會離職 FP（偽陽）
誤報 — 冤枉好人		 TN（真陰）
正確預測到了
指標公式概念白話適合用在
準確率（Accuracy）(TP+TN) / 全部預測對的比例類別均衡時
精確率（Precision）TP / (TP+FP)「預測說會離職的人，真的離職的比例」不能誤報的場景（醫療核可、信用評分）
召回率（Recall）TP / (TP+FN)「真正會離職的人，被找出來的比例」不能漏報的場景（疾病篩檢、設備停機）
F1 Score精確率與召回率的調和平均兩者平衡的綜合指標兩個都重要時
精確率 vs 召回率：何時用哪個？
精確率（Precision）— 怕誤報
分母「模型說會的所有人」（TP + FP）
精確率高表示模型說「會」的時候通常是對的，誤報少
適用場景醫療核可、垃圾信過濾（不想把正常信丟掉）
召回率（Recall）— 怕漏報
分母「真的會的所有人」（TP + FN）
召回率高表示真正有問題的大多被抓到，漏報少
適用場景疾病篩檢、設備異常停機（不能漏掉任何一個）
怕漏報 → 看 Recall；怕誤報 → 看 Precision
樣題 S1-Q24（答 B）— 必考題，漏報率用哪個指標
一家智慧工廠使用機器學習分類模型預測關鍵設備是否會異常停機。團隊希望評估模型在偵測異常停機時的「漏報率」（即未能正確偵測出異常事件的比例）。請問下列哪項指標最適合用來衡量？
(A) 準確率（Accuracy）
(B) 召回率（Recall） ✓
(C) F1 分數（F1 Score）
(D) 假陽性率（False Positive Rate）
詳解：「漏報率」= 真的會發生但沒抓到 = FN 比例。召回率 = TP / (TP+FN) = 「找出來的 / 真正會的」。召回率高 → 漏報少。題目問「衡量漏報率最合適的指標」→ 直接看 Recall 就能反推漏報。
樣題 S1-Q23（答 C）— 交叉驗證的目的
交叉驗證的主要目的是什麼？
(A) 提高模型的訓練速度
(B) 驗證數據是否線性可分
(C) 減少模型的過擬合風險 ✓
(D) 測試模型的容錯能力
詳解：(A) 交叉驗證反而會變慢（要跑 K 次）。(B) 跟線性可分性無關。(C) 透過多次切分平均，能更穩定地評估泛化能力，降低過擬合風險。(D) 容錯能力是另一回事。
定義：把資料切成 K 份，輪流當測試集，跑 K 次取平均。

3.8 特徵選取 — S1-Q22 必考陷阱

⚠️ S1-Q22 是全章最容易答錯的題目。答案是 C（迴歸分析），不是 D。很多學員誤選 D（隨機森林），但隨機森林透過特徵重要度可以輔助特徵選取，是常見方法，不能選。
名詞定義
特徵選取（Feature Selection）：從一堆欄位中挑出對預測最有用的幾個。跟 PCA 的差別：PCA 是把欄位「轉換」成新欄位；特徵選取是「從原欄位中挑」。
方法類別白話是否為特徵選取
Pearson 相關係數Filter Methods看特徵跟目標的線性相關度，低的丟✓ 是
PCA 主成分分析廣義特徵工程找最大變異方向組合新特徵✓ 是（廣義）
隨機森林特徵重要度Embedded Methods用樹模型在訓練時順便算每個特徵的重要度✓ 是
L1 正則化（Lasso）Embedded Methods訓練時自動把不重要的特徵權重壓 0✓ 是
遞迴特徵消除（RFE）Wrapper Methods反覆訓練模型，每次刪掉最不重要的特徵✓ 是
迴歸分析（Regression Analysis）預測模型用來「預測一個數字」，例如房價、銷售額✗ 不是
樣題 S1-Q22（答 C）— 展開查看詳解（必背）
請問下列何者不是常見的特徵選取技術或方法？
(A) 皮爾森積差相關分析（Pearson Correlation）
(B) 主成分分析（Principal Components Analysis, PCA）
(C) 迴歸分析（Regression Analysis） ✓
(D) 隨機森林（Random Forest）
詳解：(A) Pearson 是 Filter Methods 的經典代表，是特徵選取技術。(B) PCA 是廣義特徵工程，是常見方法。(C) 迴歸分析是預測模型，不是特徵選取技術。這是正解。 (D) 隨機森林透過特徵重要度（Feature Importance）輔助特徵選取，是常見方法。

一句話記法：「迴歸是預測，不是選特徵」。千萬不要記成「隨機森林不是特徵選取技術」— 那是錯的。
(08)

3.9 NLP 與神經網路基礎

名詞定義
NLP（Natural Language Processing，自然語言處理）：讓電腦理解與生成人類語言的技術。典型應用：機器翻譯、情感分析、文本分類、聊天機器人、語音辨識。
樣題 S1-Q19（答 B）+ S1-Q25（答 B）— NLP 與神經網路
S1-Q19：銀行想建立聊天機器人，可透過下列哪一種領域技術來達成？
(A) 資料庫管理技術
(B) 機器學習與自然語言處理 ✓
(C) 網頁開發技術
(D) 網路安全技術
詳解：聊天機器人需要理解使用者輸入、生成回應→ NLP；想要持續變強需要從互動中學習 → ML。正解是 ML + NLP 組合。
S1-Q25：神經網路與傳統機器學習模型的主要區別是什麼？
(A) 神經網路無法處理非線性數據
(B) 神經網路透過多層結構學習複雜特徵 ✓
(C) 神經網路只適用於迴歸問題
(D) 神經網路不需要大量數據支援
詳解：(A) 神經網路正擅長非線性，反向錯誤。(B) 多層結構自動學習特徵 = 深度學習核心特徵。(C) 神經網路可同時做迴歸與分類。(D) 神經網路需要大量資料才能訓練好。
(09)

講師口頭補充——多場整合

這一節是什麼
以下內容整合自 314、315、408、415 等多場實際授課的口頭補充，涵蓋常見考題陷阱與延伸概念。與主線知識點互補，非考試必背範圍，但有助於理解選項的判斷邏輯。各條目標注來源場次供對應查閱。
講師補充 監督式 / 非監督式 / 強化學習三模式記憶法 [314p1]
監督式 = 「有人在看著他」（給 Y 標籤，例：垃圾郵件二分類、房價迴歸）。非監督式 = 沒有明確標籤，做分群（例：RFM 八種客群—消費金額 / 頻率 / 最近時間，2×2×2）。強化學習 = 策略型、有獎勵分數、最後拿最高分（例：圍棋一步累積到最後勝率）。

重要區分：分類（有明確類別 = 資源回收、寶特瓶 / 紙類）vs 分群（看起來像就分一群 = 國際班按長相分歐美 vs 亞洲）——監督式才能做「分類」、非監督式只能做「分群」。AlphaGo 就是強化學習，它在圍棋網站打到頂用了 3 個月（人類要 6–7 年）。
講師補充 召回率的「捉放法」記憶法 [314p2] [415]
高風險疾病早期篩檢題目中，正常人遠多於感染者，漏診（偽陰性）代價可怕——這類題目方向就是看召回率。背的方式：小學自然課的「捉放法」——抓回來幾隻、答對幾個。模型如果只看精確度會變得保守，怕精確度掉就盡量都判沒病；但這樣召回率會變得非常糟。「一個都不能漏」的場景永遠看召回率。

詐騙簡訊偵測也是同理——FP（誤報）→ 使用者更小心、沒太大壞處；FN（漏報）→ 使用者放心點擊、上當，所以詐騙也看召回率。特異度是反過來的召回率（捕捉正常人的能力），考試出現率較低。
講師補充 混淆矩陣的「驗 100 個人」陷阱例 [314p2]
100 個人中只有 1 人有病，全猜「沒病」準確率（accuracy）也有 99%，但精確率（precision）= 0%（你說 Yes 的有多少真的 Yes，你都沒說 Yes 所以沒分母）。這個例子是混淆矩陣最經典的「陷阱」，看到「不平衡資料」+ 「少數類才是重點」（疾病、詐騙、瑕疵、異常）幾乎一定要看精確 / 召回不是準確。

實務應用 = 醫療診斷、金融風暴、製造瑕疵、公共安全。
講師補充 ROC 曲線面積的判讀 [314p2]
ROC 曲線下面積 100% = 完美（TP 跟 TN 完全分得開）；50% = 全靠猜（亂答都對一半）；0% = 完全反過來（其實只要倒過來解讀就是 100%，所以實務出 0% 通常表示模型定義錯了）。

商用 AI 通常以 80% 為基準往上拉，99% 已經很不錯了，「100% 拿出去討論一定會被打槍—世界上沒有絕對的事情，模型出 100% 通常代表你的測試集設計有問題」。
講師補充 訓練集 / 測試集 / 驗證集的「課本-習作-考試」比喻 [314p1]
訓練集 = 課本（70–80%）；測試集 = 老師出的考題（20–30%）；驗證集 = 習作練習，不一定要有。為什麼不能拿全部當訓練集再自己出考題？因為自己出可能會把資料集的規律打亂、變得沒參考性。

過擬合 = 死讀書（記選項排列組合不真理解題目）；欠擬合不是過擬合的反面——欠擬合是訓練集本身就爛，「AI 沒有不讀書還考試考很好的情況，要爛就一起爛」。交叉驗證（K-fold）平均切，每段輪流當測試，缺點是運算成本變 K 倍。
講師補充 泛化能力的車牌辨識五因素 [314p1]
車牌辨識是泛化能力最直觀的例子。影響因素 5 個：(1) 資料量太少 → 模型死記細節；(2) 資料品質差（磨損嚴重的車牌、車牌邊框）；(3) 模型太複雜 → 過擬合；(4) 沒做正則化；(5) 資料多樣性不足（沒涵蓋軍車、重機四碼、舊式 ABC 開頭、現在的 ABC 開頭）。

露天 vs 室內、白天 vs 晚上、起霧、山路水滴——都是泛化能力測試的場景。
講師補充 梯度下降的「下山找坑」比喻 [314p1]
學習率（learning rate）= 你學的速度。從山頂往下走、要記下會造成爬山危險的坑洞。學習率太大 → 跳過整個山谷（彈到很遠、失準）；學習率太小 → 把每個彈珠大小的洞都記下來（學很慢、成本高）。實作技巧：先設適中的學習率，再慢慢往下調。

常見問題：(1) 局部最小值（卡在小坑找不到全局，「以為到山腳了其實還沒」）；(2) 梯度爆炸（權重失控發散，根本找不到任何規律）。
講師補充 CNN 四層的口頭順序 [314p1]
卷積層（找特徵 = 邊緣 / 輪廓 / 材質）→ 池化層（降維 = 留下主要輪廓）→ 全連接層（分類）→ 輸出。卷積層找的「材質」就是電腦學會的紋理：毛、鐵、塑膠等。貓的辨識會留毛、耳朵、腿長度、臉，把腳掌、肚子、尾巴砍掉降維。

增加非線性是激勵函數（activation）的工作——讓神經網路可以處理「圖片這種非線性資料」，否則整個網路退化成單層線性回歸。
講師補充 Dropout = 板凳球員都要練 [314p1]
訓練時刻意關掉一些 feature 的權重，讓模型不依賴特定幾個特徵。比喻：帶隊比賽不能只練先發球員——先發出狀況時板凳上去要能打。下次新資料如果有缺值或錯誤，模型才不會崩。

Dropout 屬於正則化的一種；正則化的核心是「增加懲罰項或弱化獎勵項，限制每個 feature 的權重大小」，AI 模型訓練本質是「學規則不是學數據」，這兩件事差非常多。
講師補充 Focal Loss 的考試決策樹 [415]
題目出現「極端不平衡」、「樣本少類」、「希望模型專注少數類」——直接綁 focal loss。其他三個常見損失函數要這樣記：

均方誤差（MSE）→ 線性迴歸（預測天氣、自然災害、線性股價）；交叉熵 → 標準分類，但極不平衡時容易被多數類帶走；餘弦相似度 → 多元特徵向量方向比對，不會用在二分法。

例如「信用卡盜刷偵測：99.9% 正常、0.1% 盜刷」這種極端不平衡的二分法情境，答案永遠是 focal loss。
講師補充 LIME（局部）vs SHAP（全域）的銀行貸款例子 [314p2] [408] [415]
兩個都「跟模型不可知」（model-agnostic）——可以用在任何模型上。

LIME = 解釋這「一筆」資料為何被判成這樣（例：某客戶只要月收入再加 X、信用紀錄加 Y、負債比降 Z 就會核准）。SHAP = 計算每個「特徵」對整個模型預測的貢獻度，能講出「在這個模型裡，信用紀錄是最重要的、其次是收入、年齡和負債比」。

考試決策樹：「邊際貢獻度」「每個特徵」「整體模型」→ SHAP；「單一筆預測」「局部解釋」→ LIME。模型審查（model audit）的目的就是驗證 model 在挑選參數、學習率調整、決策過程上的合理性與合規性。
講師補充 推論模式四類（線上 / 批次 / 離線 / 分散式） [315p1]
(1) 線上推論 = 即時、低延遲，例：ChatGPT 網頁版、即時影像辨識；(2) 批次推論 = 一次處理大量資料，例：每月財報分析、每月庫存補貨建議；(3) 離線推論 = 不急著輸出，開發測試或非即時場景常用；(4) 分散式推論 = 雲端 AI 服務的主流，OpenAI 就是「線上 + 分散式」結合。

各場景的速度 / 成本 / 算力需求不同，自駕車這種一定要邊緣 + 線上、不能等雲端來回；批次的優勢就是吞吐量最大，但延遲也最大。
講師補充 概念漂移 vs 資料漂移的場域差異 [415]
概念漂移：特徵與目標的「邏輯關係」改變。例：以前流行養狗、現在流行養貓；詐騙手法從現金改成 LINE Pay。
資料漂移：純粹輸入資料的分布改變。例：模型部署到新國家、用戶族群結構變了。

兩者可以同時發生。題目描述「半年後流行趨勢改變」+ 「點擊率下降」就是兩者都有。
講師補充 模式崩潰、災難性遺忘、梯度爆炸的場域對照 [415]
這三個容易混，記得各自的場域：
模式崩潰（mode collapse）— GAN 專屬，生成只剩一種結果、訓練失敗；災難性遺忘（catastrophic forgetting）— 學新忘舊，跟增量學習相反；梯度爆炸（gradient explosion）— 訓練初期參數不收斂、數值極端兩極。

題目場域不同就選對應名詞。
講師補充 增量學習為何成為現代 AI 標配 [415]
模型要「持續學新知識又不能忘舊」。如果不保留這能力，時代一變、更新越多東西模型越亂。所以多數線上 AI 模型都會盡可能保留增量學習機制。
講師補充 GPU 為何取代 CPU 在 AI 場景 [314p1] [415]
CPU 序列執行（一次一件、作業系統管控用）。GPU 平行多工（同時算很多包），AI 把資料轉成數字後要做大量矩陣運算 → GPU 最適合。「機器學習跨到深度學習，最大的轉折就是 GPU 出現」，所以黃仁勳跟 AI 綁定那麼深、輝達現在那麼搶手。

TPU 是 Google 自家為 AI 開發的晶片，類似邏輯但比較少人用。Tensor Core 是 GPU 內部硬體層級加速「混合精度矩陣乘加」的單元，CUDA 負責損失函數 / 激勵函數 / 各種網路運算，Tensor Core 專做矩陣那塊。
講師補充 不平衡資料的「全猜負樣本」陷阱 [314p2]
正樣本 200 筆 / 負樣本 9800 筆（2% vs 98%）→ 全猜負樣本準確率就有 98%，但精確率 0%。要解這種題，重點是「少數類別才是任務的重點」——醫療找疾病、金融抓詐欺、製造業找瑕疵、公共安全找異常。看到這四類場景就要警覺要看精確率 / 召回率而不是準確率。

對策：過採樣（少數類複製 / 合成）+ 欠採樣（多數類隨機抽）+ 改用 focal loss + 改評估指標。
講師補充 公平性指標三層差異 [415]
人口統計對等（demographic parity）：不同族群的核准比例必須一致（保送名額 / 保底名額概念）。均等機會（equal opportunity）：真正會還款的人裡面被核准的機率一致（用結果回推）。精準機會差異跟條件機率有關，記得最關鍵的差異是「全體比例 vs 真正陽性裡的比例」。

題目寫「不論種族為何、批准機率必須完全相等」就綁 demographic parity。
講師補充 QAT vs PTQ 的訓練時機差別 [415]
模型量化部署到邊緣裝置時兩種做法：PTQ（訓練後量化）— 訓練好直接砍精度（四捨五入、無條件捨去），會讓資料間關係亂掉、精度雪崩；QAT（量化感知訓練）— 訓練階段就模擬量化誤差，讓神經網路自己調整權重去適應低精度。

邊緣裝置算力差，所以要在訓練時就考量「我未來只能用這種精度」。「邊緣裝置 + 直接 INT8 + 精度雪崩」這組關鍵字一定走 QAT。
講師補充 履歷篩選 AI 偏見的前處理對策 [415]
性別偏見的修法在「資料前處理」階段最直接的做法是「重新抽樣 + 合成資料平衡」。其他選項的錯誤點：

增加深度 → 反而會過擬合、放大偏見，而且不算「資料前處理」；直接刪除女性樣本 → 邏輯上等於把它當離異值，但題目沒說它是離異值；one-hot encoding → 只改格式，不影響偏差統計分布。
講師補充 特徵庫（Feature Store）的職責邊界 [415]
特徵庫的核心是「統一管理 + 共享重用」。它不是 SQL 替代品、不是自動標註工具（標註屬於資料工程）、不是模型權重存放處（那是 model registry）。

請記住四層分工：訓練流程 → 資料 → 特徵 → 模型，這四層是分開的、各有對應的管理工具。
