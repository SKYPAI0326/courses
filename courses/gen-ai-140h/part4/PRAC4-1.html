報價試算器

從需求分析開始，用 AI 做出一個有條件計算、稅金、折扣功能的可交付報價工具。

任務步驟
5 個步驟
預計時數
3 小時
所屬單元
Part 4
STEPS · 任務步驟

依序完成這五個步驟

1
定義你的報價工具需求
你的行業有哪些收費項目？稅率是多少？有沒有折扣條件？先把這些釐清，用下方工具設計規格。
2
寫工具生成指令
依照 CH4-1 的方式，精確描述欄位、運算邏輯和輸出格式。下方工具會幫你生成完整的指令文字。
3
用 AI 生成第一版
把指令貼給 Claude 或 ChatGPT，生成 HTML 工具。儲存這個版本作為基礎，不要直接開始修改。
4
測試並提出修改要求
測試邊界值（0、負數、非常大的數），提出二次修改。每次修改請求要精確（參考 CH4-1 的二次生成技巧）。
5
完成版本二
加入進階功能：條件折扣、列印功能、複製到剪貼簿。完成後儲存最終版本。

延伸案例

報價試算器的核心是把複雜收費邏輯變成客戶自己點幾下就能得到數字的工具。挑一個貼近你本業的情境案例完整做一次，每個案例 40–60 分鐘。

案例 01 · 室內設計
小型工作室老闆做裝修估算快篩工具
難度 ★★☆ 約 45 分鐘

你經營三人室內設計工作室，每週接到十多通「想請你估價」的電話，但九成最後都說超出預算。希望有個單頁小工具丟在官網，讓客人先自己算出大概範圍再決定要不要進一步聯繫，省雙方時間。

原始資料｜你常用的報價邏輯
基礎裝修：每坪 2.5 萬（含拆除、泥作、木作、油漆） 局部翻新：每坪 1.2 萬（只含油漆、木作局部） 全室設計：每坪 4.5 萬（含設計費、系統櫃、燈具） 加項： - 廚房升級 +8 萬 - 浴室翻新 +12 萬（每間） - 系統衣櫃 +3 萬（每座） 房型屋齡折扣： - 屋齡 < 5 年：基礎 x 0.9（結構好省工） - 屋齡 > 30 年：基礎 x 1.15（管線結構需補強）
你的任務
  1. 用四要素寫 v1 Prompt 讓 AI 產出單頁 HTML（含坪數 input、三種裝修類型 radio、加項 checkbox、屋齡 select、試算按鈕、結果區）
  2. 驗收 AI 產出：計算邏輯對不對、介面是否一眼看懂、手機能不能用
  3. 對照參考 Prompt 找出遺漏限制，再要求 AI 修改一次
評估指標
計算正確
輸入 25 坪基礎裝修 + 浴室翻新 x 2，屋齡 35 年應得 (25×25000×1.15)+24 萬
介面清楚
欄位由上往下順、按鈕位置明顯、結果區看得到金額區間而非精準單一數字
手機可用
手機寬度下欄位不擠壓、輸入框夠大、radio/checkbox 點得中
免責說明
底部有小字標示「估算僅供參考，實際依現場丈量為準」
參考 Prompt（建議先自己寫再看）
點擊展開參考答案
【角色】你是寫純前端 HTML/CSS/JS 單頁工具的工程師，風格簡潔務實，輸出可以直接放到官網。 【任務】根據下方報價邏輯，產出一個「裝修估算快篩工具」的單頁 HTML。 【報價邏輯】 （貼上上方原始資料） 【限制】 - 單一 HTML 檔，所有 CSS/JS 內嵌 - 欄位：坪數（number input）、裝修類型（radio 三選一）、加項（checkbox 複選）、屋齡（select） - 計算結果不是一個精準數字，而是「金額區間」（例如輸出值 ± 10%） - 色系米白底 + 深灰字，不用漸層不用 box-shadow，按鈕只用 border - 手機 375px 寬下不破版 - 底部一行小字：「此估算僅供參考，實際以現場丈量為準」 - 不要加 Google Analytics、不要引用外部 CDN 【輸出格式】 直接輸出完整 HTML 程式碼，不要任何解說。
變體挑戰
A改為「留下 email 才顯示估算結果」版本，拿到名單再聯繫
B加入「預算反推坪數」模式：客人輸入預算 50 萬，工具反算能做幾坪的哪種方案
C產出兩種視覺版本（極簡白底 vs 溫暖木色底），A/B 測哪種轉換率高

案例 02 · 婚禮小物
手作婚禮小物店的「客製報價快算」
難度 ★☆☆ 約 35 分鐘

你在 IG 經營手作婚禮小物，每天私訊一堆「可以算多少？」、「加字會貴多少？」一模一樣的問題。你想做一個 LINE 官網 rich menu 可點開的小工具，客人自己選商品+數量+客製項目就能看到報價，省掉回訊息的時間。

原始資料｜商品與加價邏輯
商品： - 手工皂（單入）NT$45 - 乾燥花瓶（玻璃小瓶）NT$85 - 精油蠟燭（錫盒）NT$120 - 手寫感謝卡 NT$20 數量折扣： - 50 個以上 95 折 - 100 個以上 9 折 - 200 個以上 85 折 客製加價： - 燙金姓名 +NT$8/個 - 客製緞帶顏色（超過 3 色）+NT$5/個 - 包裝袋升級（棉麻袋）+NT$15/個 運費：台北免運（150 件以上），其他每單 NT$120
你的任務
  1. 寫 Prompt 讓 AI 產出單頁工具：商品卡片選購、數量輸入、客製選項、運費判斷、即時顯示總價與折扣明細
  2. 畫面風格要貼近婚禮氣氛（米白、淡粉、手寫感）但不過度花俏
  3. 加入「複製報價單」按鈕，客人可直接貼到訊息給你確認下單
評估指標
折扣邏輯
數量達到級距自動換算，且明細能看到「原價 / 折扣後 / 省多少」
客製加價
勾選客製項目後，數字即時更新不需按計算鈕
視覺一致
配色貼近手作婚禮調性，但仍保持可讀性不花俏
複製友善
「複製報價單」按鈕產生乾淨可貼訊息的純文字格式
參考 Prompt（建議先自己寫再看）
點擊展開參考答案
【角色】你是熟悉電商小型店家需求的前端工程師，特別擅長做讓非技術用戶也能輕鬆操作的工具。 【任務】根據下方商品與規則，產出一個婚禮小物客製報價單頁 HTML。 【商品與規則】 （貼上原始資料） 【限制】 - 單一 HTML 檔內嵌 CSS/JS - 商品用卡片式列出，每張卡片內有數量輸入框與客製 checkbox - 總價區域固定在畫面底部（sticky），使用者拉動頁面仍看得到 - 折扣邏輯明細展開：原價 / 折扣級距 / 客製加價 / 運費 / 總計 - 配色：米白 #faf6f0 底 + 淡粉 #d4a5a0 強調色，文字深灰 #3a3532，字型用 serif（像手寫信件感） - 「複製報價單」按鈕輸出純文字格式，結構清楚可直接貼到 LINE - 手機優先設計，375px 寬完美運作 【輸出格式】 直接輸出完整 HTML，不要任何解說。
變體挑戰
A加入「上傳客製圖樣」欄位（其實只是模擬，上傳後顯示「圖樣已收到，我會 24h 內回覆」）
B做婚禮日期倒數，提醒客人：「距離婚禮還有 X 天，建議最晚 Y 日下訂」
C改成「禮盒組」邏輯：選定價位（1500/2500/3500），自動推薦商品組合

案例 03 · 寵物美容
寵物美容店的品種體型報價試算器
難度 ★★☆ 約 45 分鐘

你開寵物美容店三年，報價隨品種、體重、毛況變動很大，每天電話接到手軟。想做個 LINE 官方帳號裡掛的小工具，飼主選品種、打體重、勾毛況就能看估價，減少來電諮詢，也避免一直報同樣的價。

原始資料｜收費規則
基礎洗澡（含剪指甲、清耳朵）： - 小型犬（< 7kg）NT$600 - 中型犬（7-15kg）NT$900 - 大型犬（15-30kg）NT$1400 - 巨型犬（> 30kg）NT$1900 修剪造型加價： - 小型 +NT$400 - 中型 +NT$600 - 大型 +NT$900 - 巨型 +NT$1300 毛況加價： - 嚴重打結：基礎費 x 1.3 - 掉毛期：基礎費 + NT$200 - 皮膚問題需藥浴：+NT$350 特殊品種說明： - 貴賓、比熊、馬爾濟斯必須修剪造型（價格自動帶入） - 柴犬、哈士奇掉毛期固定加 NT$200
你的任務
  1. 寫 Prompt 產出工具：品種 select、體重 number、造型 checkbox、毛況多選、結果區
  2. 針對「貴賓/比熊/馬爾濟斯」自動勾選造型項並說明「此品種建議修剪」
  3. 加一個「加入 LINE 預約」按鈕，點了會跳出 QR code 或 LINE 連結（用假連結即可）
評估指標
體重分級
輸入 7.1 kg 歸中型、14.9 kg 歸中型、15.1 kg 歸大型，邊界正確
品種邏輯
選貴賓自動勾造型並說明；選柴犬自動帶掉毛期加價
打結加乘
勾嚴重打結後是基礎費 x 1.3，不是 + 固定金額
飼主友善
文案用飼主聽得懂的話，不出現「x 1.3」這種算式而是「打結處理費 +30%」
參考 Prompt（建議先自己寫再看）
點擊展開參考答案
【角色】你是做寵物產業數位工具的前端工程師，了解飼主不是技術背景，介面用詞需要口語化。 【任務】根據下方收費規則，產出一個寵物美容報價試算單頁 HTML。 【收費規則】 （貼上原始資料） 【限制】 - 單一 HTML 檔 - 品種下拉選單：貴賓、比熊、馬爾濟斯、柴犬、哈士奇、米克斯、其他（選項要涵蓋這些） - 特定品種觸發規則：貴賓/比熊/馬爾濟斯自動勾造型 disabled（不能取消）並顯示提示「此品種建議固定修剪造型」 - 體重用 number 輸入框，自動帶入對應體型分級（顯示「小型犬 / 中型犬...」文字給飼主看） - 結果區按明細列出：基礎洗澡 / 造型 / 毛況附加 / 合計，且打結加成顯示為「+30%」不是「x 1.3」 - 底部「加入 LINE 預約」假按鈕，點擊 alert 「將開啟 LINE 官方帳號（demo）」 - 配色淡藍 #a8c5d6 + 米白，給人乾淨安心感 【輸出格式】 直接輸出完整 HTML。
變體挑戰
A加「上傳寵物照片」模擬欄位（不實際上傳，顯示「已收到，會在預約確認時回覆是否需要加價」）
B加「預約時段」選擇（週末 +NT$100 假日加成）
C產出兩種視覺版本（走清新北歐 vs 走可愛日系），觀察哪個版本比較適合你的客群

同儕互評

零代碼工具做出來只是第一步，沒被別人用過永遠不知道自己漏了什麼。找一位同學把作品丟給對方當第一個使用者，用共同評分指標抓出 UX 盲點，比自己盯半小時還快。活動約 25 分鐘。

同儕互評 01 · AI 工具互測
你寫需求，我當第一個白老鼠——互測找出 3 個 UX 問題
兩人一組 約 25 分鐘 需電腦與 AI 生成工具

自己寫的需求 Prompt 自己看永遠覺得「寫得夠清楚了」，但丟給別人用才會發現欄位名稱看不懂、錯誤訊息太抽象、手機版爆版。這個活動讓 A 產出一個單頁工具，B 當完全沒看過需求的真實使用者直接試用，回報 3 個最痛的 UX 問題。

A工具作者
挑一個工作中的小痛點（例：出貨單試算、客戶分類、報價整理），用黃金公式寫一段生成需求 Prompt，丟給 AI 生出單頁 HTML 工具。**不要**向 B 解釋任何使用情境，直接把工具連結或檔案交出去。
B第一個使用者
打開工具當自己是真的要用這個解決問題的人。**不問 A 任何問題**，自己嘗試填入資料、點按鈕、看輸出，列出 3 個最阻礙你用下去的 UX 問題（例：欄位名稱像工程師在說話、手機版破版、錯誤提示不知道在講什麼）。
活動節奏
1A 寫需求 Prompt 並丟 AI 生成單頁工具· 8 分鐘
2交換，B 拿到工具直接試用（不問 A 任何事）· 5 分鐘
3B 用指標逐項打分並寫下 3 個 UX 問題· 5 分鐘
4兩人對談：A 講原本設計意圖、B 講實際撞牆點· 5 分鐘
5A 把 3 個問題改寫成追加需求丟 AI 生 v2· 2 分鐘
互評指標（1-5 分）
看得懂欄位名稱與按鈕文字用人話，不用工程術語
不用猜錯誤提示直接指出哪裡錯、怎麼改
手機可用在手機上開起來沒破版、按鈕按得到
一看就會沒有說明也能在 1 分鐘內用出結果
