STEPS · 任務步驟

依序完成這五個步驟

1
定義你的報價工具需求
你的行業有哪些收費項目？稅率是多少？有沒有折扣條件？先把這些釐清，用下方工具設計規格。
2
寫工具生成指令
依照 CH4-1 的方式，精確描述欄位、運算邏輯和輸出格式。下方工具會幫你生成完整的指令文字。
3
用 AI 生成第一版
把指令貼給 Claude 或 ChatGPT，生成 HTML 工具。儲存這個版本作為基礎，不要直接開始修改。
4
測試並提出修改要求
測試邊界值（0、負數、非常大的數），提出二次修改。每次修改請求要精確（參考 CH4-1 的二次生成技巧）。
5
完成版本二
加入進階功能：條件折扣、列印功能、複製到剪貼簿。完成後儲存最終版本。

案例 01 · 室內設計
小型工作室老闆做裝修估算快篩工具
難度 ★★☆ 約 45 分鐘

你經營三人室內設計工作室，每週接到十多通「想請你估價」的電話，但九成最後都說超出預算。希望有個單頁小工具丟在官網，讓客人先自己算出大概範圍再決定要不要進一步聯繫，省雙方時間。

原始資料｜你常用的報價邏輯
基礎裝修：每坪 2.5 萬（含拆除、泥作、木作、油漆） 局部翻新：每坪 1.2 萬（只含油漆、木作局部） 全室設計：每坪 4.5 萬（含設計費、系統櫃、燈具） 加項： - 廚房升級 +8 萬 - 浴室翻新 +12 萬（每間） - 系統衣櫃 +3 萬（每座） 房型屋齡折扣： - 屋齡 < 5 年：基礎 x 0.9（結構好省工） - 屋齡 > 30 年：基礎 x 1.15（管線結構需補強）
你的任務
  1. 用四要素寫 v1 Prompt 讓 AI 產出單頁 HTML（含坪數 input、三種裝修類型 radio、加項 checkbox、屋齡 select、試算按鈕、結果區）
  2. 驗收 AI 產出：計算邏輯對不對、介面是否一眼看懂、手機能不能用
  3. 對照參考 Prompt 找出遺漏限制，再要求 AI 修改一次
評估指標
計算正確
輸入 25 坪基礎裝修 + 浴室翻新 x 2，屋齡 35 年應得 (25×25000×1.15)+24 萬
介面清楚
欄位由上往下順、按鈕位置明顯、結果區看得到金額區間而非精準單一數字
手機可用
手機寬度下欄位不擠壓、輸入框夠大、radio/checkbox 點得中
免責說明
底部有小字標示「估算僅供參考，實際依現場丈量為準」

案例 03 · 寵物美容
寵物美容店的品種體型報價試算器
難度 ★★☆ 約 45 分鐘

你開寵物美容店三年，報價隨品種、體重、毛況變動很大，每天電話接到手軟。想做個 LINE 官方帳號裡掛的小工具，飼主選品種、打體重、勾毛況就能看估價，減少來電諮詢，也避免一直報同樣的價。

原始資料｜收費規則
基礎洗澡（含剪指甲、清耳朵）： - 小型犬（< 7kg）NT$600 - 中型犬（7-15kg）NT$900 - 大型犬（15-30kg）NT$1400 - 巨型犬（> 30kg）NT$1900 修剪造型加價： - 小型 +NT$400 - 中型 +NT$600 - 大型 +NT$900 - 巨型 +NT$1300 毛況加價： - 嚴重打結：基礎費 x 1.3 - 掉毛期：基礎費 + NT$200 - 皮膚問題需藥浴：+NT$350 特殊品種說明： - 貴賓、比熊、馬爾濟斯必須修剪造型（價格自動帶入） - 柴犬、哈士奇掉毛期固定加 NT$200
你的任務
  1. 寫 Prompt 產出工具：品種 select、體重 number、造型 checkbox、毛況多選、結果區
  2. 針對「貴賓/比熊/馬爾濟斯」自動勾選造型項並說明「此品種建議修剪」
  3. 加一個「加入 LINE 預約」按鈕，點了會跳出 QR code 或 LINE 連結（用假連結即可）
評估指標
體重分級
輸入 7.1 kg 歸中型、14.9 kg 歸中型、15.1 kg 歸大型，邊界正確

打結加乘

參考 Prompt（建議先自己寫再看）

【角色】你是做寵物產業數位工具的前端工程師，了解飼主不是技術背景，介面用詞需要口語化。 【任務】根據下方收費規則，產出一個寵物美容報價試算單頁 HTML。 【收費規則】 （貼上原始資料） 【限制】 - 單一 HTML 檔 - 品種下拉選單：貴賓、比熊、馬爾濟斯、柴犬、哈士奇、米克斯、其他（選項要涵蓋這些） - 特定品種觸發規則：貴賓/比熊/馬爾濟斯自動勾造型 disabled（不能取消）並顯示提示「此品種建議固定修剪造型」 - 體重用 number 輸入框，自動帶入對應體型分級（顯示「小型犬 / 中型犬...」文字給飼主看） - 結果區按明細列出：基礎洗澡 / 造型 / 毛況附加 / 合計，且打結加成顯示為「+30%」不是「x 1.3」 - 底部「加入 LINE 預約」假按鈕，點擊 alert 「將開啟 LINE 官方帳號（demo）」 - 配色淡藍 #a8c5d6 + 米白，給人乾淨安心感 【輸出格式】 直接輸出完整 HTML。

零代碼工具做出來只是第一步，沒被別人用過永遠不知道自己漏了什麼。找一位同學把作品丟給對方當第一個使用者，用共同評分指標抓出 UX 盲點，比自己盯半小時還快。活動約 25 分鐘。

同儕互評 01 · AI 工具互測
你寫需求，我當第一個白老鼠——互測找出 3 個 UX 問題
兩人一組 約 25 分鐘 需電腦與 AI 生成工具

SCENARIOS · 情境演練

把報價試算器套進 3 種業態

真實工作現場不只是「服務類型 × 單價 × 稅率」這麼簡潔。不同業態的報價邏輯藏著加購、分級、倍率、季節係數。挑一個最貼近你本業（或最想拿來當副業的業態）跑一次完整 Prompt，直接把下面的規格丟 ChatGPT / Claude / Gemini，看 AI 產出的工具能不能讓客戶自己就把數字算清楚。每個情境 22–28 分鐘。

情境 A · 婚禮攝影報價 約 25 分鐘｜難度 ★★☆
業務背景與報價邏輯
業務情境：你是接案婚禮攝影師，官網想放一個「婚禮攝影快速估價」工具，讓準新人自己選方案看預算。 必填欄位： - 拍攝時數（基本 4 小時起，NT$2,500 / 小時） - 加購項目（可複選）： - 精修底片全交付 +NT$3,000 - 精修數量加碼（每加 20 張 +NT$1,500） - 實體相本一本（30 面跨頁）+NT$4,800 - 二機攝影師 +NT$6,000 - 旺季加成：10–12 月、農曆過年前兩週、連假，整單 × 1.15 計算公式： 小計 = 時數 × 2500 + 加購金額 旺季加成 = 小計 × 1.15（若勾選旺季） 總計 = 旺季加成後金額（含 5% 營業稅另計顯示）
工具規格 Prompt｜可直接貼 ChatGPT / Claude / Gemini
【角色】你是替接案攝影師做工具的前端工程師，理解準新人需要清楚看到加購項目怎麼堆疊起來。 【任務】產出一份「婚禮攝影快速估價」單頁 HTML 工具。 【輸入】 - 拍攝時數 number input（最小 4、步進 1，預設 6） - 加購項目 checkbox 複選：精修底片全交付 +3000、精修加碼每 20 張 +1500、實體相本 +4800、二機攝影師 +6000 - 旺季加成 switch：10–12 月 / 農曆年前 / 連假整單 × 1.15 【工具規格】 1. 即時計算（不用按鈕），每動一次勾選或改時數，下方總價立刻更新 2. 結果區分 4 行明細：拍攝費、加購總額、旺季加成、小計 → 營業稅 5% → 總計 3. 「複製報價」按鈕：產出純文字報價單格式可直接貼 LINE 4. 「寄給新人」按鈕：假裝會開 mailto:（實際用 alert 示意） 5. 底部免責小字：「本報價含旺季調整，實際以簽約日確認」 【限制】 - 單一 HTML 檔、內嵌 CSS/JS、不依賴外部 library - 米白底 + 霧藍 #6b7fa3 強調色，不用漸層 - 欄位命名全部是新人聽得懂的白話，不要出現「param」「total」英文技術字 - 手機 375px 下不破版，加購 checkbox 要夠大點得中 【輸出格式】直接輸出完整 HTML，不要解說。

延伸挑戰 · 把報價試算器推到真的被用。
  1. 做你自家業務的版本。不要只做練習題裡的三個情境，把報價邏輯換成你真實在接的業務（甚至你家親戚的攤販也算）。AI 幫你把複雜收費邏輯可視化，比「做完留在桌面」有意義多了。
  2. 加報價備存 + email 通知。光複製純文字不夠。在 Prompt 裡加一句「點擊『寄估價』後用 EmailJS 寄給 email 輸入欄位的信箱（同時副本給自己）」，把工具從「估完就丟」升級成「估完有線索」。
  3. 產出 PDF 估價單。用 html2pdf.js 或 window.print() 把結果區轉成可列印 PDF，客戶收到正式文件的感受跟純文字差一個層級。報價單通過率通常會提升。
1一個可用的報價試算器，必備的三個要素是？
展開解析
正解 C。報價試算的本質就是「收到乾淨資料 → 跑對公式 → 丟回可帶走的結果」。欄位驗證防止 0 或負數亂算、計算邏輯是核心、結果可複製讓客戶帶走對話。A/B/D 都是錦上添花，缺了 C 這三項工具根本不成立。
2折扣邏輯該寫在前端還是後端？
展開解析
正解 D。純估價工具（只是讓客戶看個大概）寫在前端無所謂，反正要成交還會再報價一次。但如果這個折扣直接會觸發扣款（例如線上購物車），必須後端複算，否則使用者打開 DevTools 改 JS 就能「超級折扣」。A 太重（估價工具沒必要）、B 有資安風險、C 等著出事。
3發現使用者用 DevTools 改 JS 把折扣改成「99 折變 99 元」怎麼辦？
展開解析
正解 B。前端永遠不可信——任何金額都該在伺服器端以原始規則複算一次，並加上合理上限（例如單張訂單最高折扣 50%）與風控紀錄。A 沒用、C 混淆只是延緩（反編譯幾秒鐘的事）、D 鎖 F12 既擋不住也破壞體驗，還會激怒正常使用者。
4報價試算的「輸出」最好支援哪些形式？
展開解析
正解 D。三種輸出對應三種成交漏斗：純文字讓客戶現場貼 LINE 確認、email 讓你有追蹤線索（誰什麼時候問了什麼）、PDF 讓報價顯得正式會被認真看。A/B/C 單一形式都會漏掉另外兩條路徑。
5AI 第一次生成報價工具最常忘記處理的兩件事？
展開解析
正解 C。AI 預設都順著 happy path 寫——假設使用者乖乖填數字。但真實世界會有人輸入負數、英文字、空白，或什麼都沒填就按計算。這兩個邊界條件通常要 v2 才補齊，交工具給客戶前請務必手動測：欄位空白按計算會怎樣？輸入 -1 會怎樣？輸入 abc 會怎樣？
