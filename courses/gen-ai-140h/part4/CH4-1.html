讓 AI 生成有輸入、有運算、有輸出的工具

3 小時 · Part 4 · 零代碼工具工坊

用 AI 寫程式不是魔法，而是一種需要學習的對話技術。這節課教你如何把「我想要一個能做 X 的工具」精確地告訴 AI，讓它生出能直接跑的 HTML 工具頁面。

學習成果
掌握工具頁面的基本結構：輸入區、運算/AI 呼叫、輸出區三段
能將工作需求轉譯成 AI 能執行的工具生成指令
能對 AI 的第一稿進行「二次生成」，修正欄位和邏輯
RECAP · Part 3 帶走的
Part 3 讓你掌握資料清理、工作流設計與表單自動化——資料層的問題已有一套可重複執行的解法。
PREVIEW · 這節解決什麼
資料處理完，現在讓 AI 直接幫你生工具。這節教你把需求精確告訴 AI，拿到能直接跑的 HTML 工具頁面，不用自己寫程式。
(01) · Tool Anatomy

工具頁的三段結構：輸入、運算、輸出

任何一個有用的 HTML 工具頁，不管看起來多複雜，都可以拆成三個部分。了解這個結構，你就能清楚地告訴 AI 你要什麼。

INPUT
輸入區
  • 文字輸入框（textarea/input）
  • 下拉選單（select）
  • 滑桿（range）
  • 勾選框（checkbox）
  • 每個欄位需要：名稱、預設值、輸入驗證
PROCESS
運算 / 邏輯
  • 計算（加減乘除、條件判斷）
  • 格式轉換（單位換算、日期計算）
  • AI 呼叫（把輸入組成 Prompt，送給 Claude/GPT）
  • 以 JS 函式實現，由按鈕觸發
OUTPUT
輸出區
  • 純文字顯示、格式化卡片、表格
  • 複製按鈕
  • 空狀態（未輸入時顯示什麼）
  • 載入狀態（等待時顯示什麼）
  • 錯誤狀態（出錯時顯示什麼）
◆
告訴 AI 你想要的工具時，不需要說「用 flex 還是 grid」，而是說「我想要左邊輸入、右邊顯示結果」。描述你要的行為，讓 AI 決定怎麼實作。
(02) · From Requirement to Prompt

怎麼把需求轉譯成工具生成指令

工具生成指令和一般對話不同——你需要描述的是「行為」，不是「外觀」。精確的指令讓 AI 可以直接動手，模糊的指令只會讓 AI 猜來猜去。

模糊需求（AI 猜來猜去）
「做一個幫我算薪資的工具」
精確指令（AI 可以直接做）
「做一個 HTML 頁面工具，包含：① 基本薪資輸入框（數字，預設28000）② 加班時數輸入框（數字，預設0）③ 加班費率下拉選單（1.34倍/1.67倍/2倍）④ 按下「計算」按鈕後，顯示：每小時工資（基本薪資÷240）和本月加班費（時薪×時數×費率）⑤ 輸出區要有複製到剪貼簿按鈕」
◎
精確指令的四個要素：
① 每個輸入欄位（類型 + 標籤 + 預設值）
② 運算邏輯（公式或條件，用自然語言寫）
③ 輸出格式（數字/文字/表格？需不需要複製按鈕？）
④ 驗證規則（什麼情況不能送出？）
迷你練習 1 約 10 分鐘
把一個工作痛點，寫成可直接生成的工具指令

挑一個你本週重複做過 3 次以上、會填數字或複製貼上的小事，套用「輸入／運算／輸出」三段結構把它寫成給 AI 的工具生成指令。

1用一句話寫出痛點：「我每次做 ____，都要手動 ____，花我 ____ 分鐘」。
2列出輸入欄位：每欄寫「標籤 · 類型 · 預設值」三件事，至少 2 欄。
3用自然語言寫出運算邏輯（公式、條件、AI 呼叫）和輸出格式（數字？卡片？要不要複製按鈕？）。
範本素材 先跑這份體感一次，再把痛點換成你自己的
原始素材 · n8n 流程節點成本小試算（約 160 字）貼進 Prompt 最下方
痛點：每次設計 n8n 自動化流程前，主管會問「這條 flow 一個月跑幾次、會花多少錢」。目前是打開計算機手算，常把觸發次數乘錯，花 5 分鐘。 目前已知的資訊： - 流程觸發頻率：每日 X 次（由使用者輸入） - 每次流程平均呼叫 OpenAI API：2 次，單次費用 0.02 USD - 每次流程寫入 Google Sheets：1 次，不計費 - 每次流程發送 LINE Notify：0 或 1 次（下拉選） - 匯率：1 USD = 32 TWD（預設值，可改）
範例 Prompt · 工具生成指令可直接貼 ChatGPT / Claude
你是熟悉 HTML 工具製作的前端助理。請把下方「原始素材」做成單一檔案 HTML 工具： 【輸入欄位】 1. 每日觸發次數（數字，預設 30，最小 0） 2. 每次是否發送 LINE Notify（下拉：是 / 否，預設：是） 3. 匯率 USD→TWD（數字，預設 32） 【運算邏輯】 - 每次流程成本（USD）＝ 2 × 0.02 USD；若發 LINE Notify 則不另加費 - 每月成本（USD）＝ 每次成本 × 每日觸發次數 × 30 - 每月成本（TWD）＝ 上述 × 匯率 【輸出區】 - 以卡片顯示：每月觸發總次數、每月成本（USD，小數兩位）、每月成本（TWD，整數四捨五入） - 卡片下方附一顆「複製報價摘要」按鈕，點擊後複製一行文字：每月 N 次，約 TWD X 元 - 欄位空或為負時，輸出區顯示「請先填寫每日觸發次數」，不要彈 alert 請以 Part 4 的霧藍（#6b7fa3）為主色，背景米白，按鈕全寬。 【原始素材】 （貼上上方原始素材）

## 頁面結構
- 單檔 HTML，頂部標題「n8n 流程成本試算器」
- 3 個輸入欄位：每日觸發次數（number，預設 30）、是否發送 LINE Notify（select：是/否）、匯率 USD→TWD（number，預設 32）
- 一顆霧藍主按鈕「試算」，點擊後下方出現結果卡片

## 試算結果（以預設值為例）
- 每月觸發總次數：900 次（30×30）
- 每月成本（USD）：36.00（30×2×0.02×30，小數兩位）
- 每月成本（TWD）：1,152（整數四捨五入）
- 卡片下方一顆「複製報價摘要」按鈕，點擊後剪貼簿出現一行：「每月 900 次，約 TWD 1,152 元」

## 空值處理
清空每日次數再按試算：結果區顯示灰字「請先填寫每日觸發次數」，沒有 alert 彈窗

對照重點
判斷學員版本是否：(1) 每次成本 = 2×0.02 USD，LINE Notify 不另加費（題目說 LINE Notify 本身不計費，選「是」也不應改變金額）；(2) 月成本以「每月 30 天」乘，不是實際當月天數；(3) 空值用灰字提示取代 alert，因為 alert 在手機上會卡一下。
成果產出
一份可以直接貼給 Claude / GPT 的工具生成指令（約 150–250 字），結尾附驗證規則一行。

(03) · Error Design

重設、驗證、錯誤訊息：讓工具不怕被亂用

你只需要用中文說「當 XX 欄位是空的時，不要讓使用者送出，顯示『請先填寫 XX』」，AI 就會幫你加入驗證邏輯。

輸入驗證
「欄位不能是空的」「數字必須大於0」「日期必須是今天或以後」

告訴 AI 規則，它會寫 if 判斷。
重設功能
「加一個清除按鈕，點下去清空所有輸入和輸出」

最簡單的功能，AI 一行就能做到。
錯誤訊息
「輸入不合法時，用紅色文字提示，不要彈出 alert」

指定訊息位置和樣式，讓工具有好的 UX。
ℹ
不要讓工具在輸入空的時候輸出空白或錯誤。設計一個清楚的「必填但未填」提示，比事後解釋為什麼沒有結果更好用。
迷你練習 2 約 8 分鐘
給練習 1 的指令加上「護欄」

回去你剛寫的工具指令，補上驗證規則、重設按鈕與錯誤訊息，讓工具不怕被亂填。

1為每個輸入欄位寫一條驗證規則（空值？負數？超出上限？），用中文描述即可。
2加上「清除所有輸入與輸出」的重設按鈕需求。
3指定錯誤訊息顯示位置與樣式（紅字？欄位正下方？不要 alert）。
範本素材 接續練習 1 的試算器，補護欄
原始素材 · 上一版工具已知的漏洞清單（約 140 字）貼進 Prompt 最下方
目前「n8n 流程成本試算器」測試時發現的狀況： 1. 同事把「每日觸發次數」留空直接按「試算」，結果顯示 NaN TWD 2. 有人把匯率填成 0，導致成本變 0 元，明顯錯誤沒有提醒 3. 有人把每日次數填成 -5，工具照算出負成本 4. 有人快速連按兩次「試算」按鈕，不確定有沒有重複計算 5. 目前錯誤訊息是用瀏覽器 alert 彈窗，手機上會卡一下
範例 Prompt · 補驗證與重設可直接貼 ChatGPT / Claude
這是對上一版工具的追加指令。請只改下列項目，其他保持不動： 【驗證規則】 - 每日觸發次數：必填、數字、不可為負、不可大於 10000。空值時顯示「請先填寫每日觸發次數」 - 匯率 USD→TWD：必填、大於 0、合理範圍 20–50。不在範圍時顯示「匯率不在合理範圍（20–50），請確認」 - 每個欄位的錯誤訊息顯示在該欄位正下方，紅字（#b5703a），不要用 alert 【重設按鈕】 - 在「試算」按鈕右邊加一顆「清除」按鈕（次要樣式，邊框灰色） - 點下後：所有輸入回到預設值，結果卡片清空，所有錯誤訊息消失 【防手快連點】 - 試算按鈕按下後，0.3 秒內不可再次觸發（按鈕暫時 disabled） 請沿用霧藍主色與米白背景。 【原始素材】 （貼上上方漏洞清單）

## 驗證行為
- 清空每日次數按試算：該欄位正下方紅字「請先填寫每日觸發次數」
- 每日次數填 -5：紅字「每日觸發次數不可為負，請輸入 0–10000 的整數」
- 每日次數填 20000：紅字「超過上限 10000，請確認」
- 匯率填 100：欄位下方紅字「匯率不在合理範圍（20–50），請確認」
- 匯率填 0：紅字「匯率必須大於 0」

## 重設與防連點
- 點「清除」：3 個輸入欄位回到預設值（30 / 是 / 32），結果卡片清空，所有紅字消失
- 快按兩次「試算」：按鈕按下後 0.3 秒內 disabled 短暫灰化，第二次點擊不觸發運算
- 錯誤訊息統一用 #b5703a 紅字顯示在欄位正下方，完全沒有 alert

對照重點
判斷學員版本是否：(1) 驗證規則分別擋住空值、負數、超上限、範圍外四種情境，而不是只擋空值；(2) 「清除」按鈕是次要樣式（邊框灰色），不跟「試算」同色搶視覺；(3) 防連點用 disabled + setTimeout 0.3s，不是靠全域鎖（避免按鈕永遠鎖死）；(4) 錯誤訊息位置是「欄位正下方」而非頁首 banner，若學員放頂部表示沒讀到題目的 UX 要求。
成果產出
升級版工具指令：在輸入欄位段落底下新增「驗證規則」與「錯誤顯示方式」兩個小段。

(04) · Second Generation

二次生成：怎麼要求 AI 改到你要的樣子

AI 的第一稿幾乎不可能完全符合你的期望——這不是失敗，而是流程的一部分。二次生成的技巧決定了你最終能不能拿到真正可用的工具。

1
先測功能，再改外觀
確認計算邏輯和 AI 呼叫正確後再調整視覺。在基礎功能跑通前改 CSS 是浪費時間。
2
精確描述問題
不說「這個按鈕看起來不好」，說「確認按鈕太小，請改為全寬（width: 100%），背景色用霧藍（#6b7fa3），文字改成白色」。
3
要求改局部，不要全部重生
「只改輸出卡片的部分，保持其他不動」比「重新生成整個工具」更穩定，也更省時間。
4
版本備份
每次收到可用的版本先存一份，再開始下一輪修改。不要讓 AI 改到最後連可用版本也沒有了。
迷你練習 3 約 15 分鐘
實跑一次二次生成：從第一稿到可用版

把練習 2 的升級版指令真的丟進 Claude / GPT，生成第一稿 HTML。打開瀏覽器測一次，用一句精確的修正指令做一輪二次生成。

1貼指令給 AI 取得第一稿，另存新 HTML 檔，用瀏覽器打開。
2先測功能（數字對不對、錯誤訊息有沒有出現），把第一個發現的問題寫成一句話。
3用「只改 X，其他不動」的句型追加指令，跑第二輪，比對兩版差異。
範本素材 v1 先跑出來，再下第二輪精準指令
原始素材 · v1 實測發現的問題（約 150 字）貼進 Prompt 最下方
把練習 2 的升級指令丟給 AI 後，第一稿 v1.html 打開瀏覽器實測，發現： - 試算邏輯正確，但結果卡片上「每月成本（TWD）」的數字沒有千分位逗號，顯示成 1152 而不是 1,152，不好讀 - 欄位錯誤訊息出現後，修改欄位內容時紅字不會消失，要再按一次試算才清掉 - 手機（寬度 375px）上看，兩顆按鈕（試算／清除）擠在同一行且文字被截斷 - 桌面版 OK，色彩與版面都沒問題，不要動 決定這一輪只修這三項，不擴增新功能。
範例 Prompt · v2 精準修正可直接貼 ChatGPT / Claude
這是針對 v1 的二次生成。請只修下列三處，其他保持完全不動（尤其色彩、版型、既有驗證規則）： 【修正 1 · 數字格式】 結果卡片上「每月成本（TWD）」顯示時加上千分位逗號，例如 1,152。USD 保持小數兩位。 【修正 2 · 錯誤訊息自動清除】 當使用者在有錯誤訊息的欄位輸入新值（input 事件）時，立刻清除該欄位下方的紅字，不必等下次按試算。 【修正 3 · 手機按鈕換行】 螢幕寬度小於 600px 時，「試算」與「清除」兩顆按鈕改為上下排列（各佔一行全寬），桌面版維持原本並排。 請回傳完整 HTML（v2），不要只給 diff。 【原始素材】 （貼上上方實測清單）

## v2 三處修正
- 千分位：試算後 TWD 結果顯示「約 TWD 1,152 元」（有千分位逗號），USD 仍為 36.00 兩位小數
- 錯誤訊息自動清除：欄位報紅字後，使用者開始輸入（input 事件）→ 該欄位紅字立即消失，不必等再按試算
- 手機按鈕換行：瀏覽器拉到 375px 寬，「試算」「清除」兩顆按鈕改為上下排列，各自 width:100%

## 差異對比（v1 vs v2）
- 霧藍主色、米白背景、卡片結構、既有驗證規則：全部保持不動
- 桌面版（寬度 ≥600px）：兩顆按鈕仍並排，跟 v1 視覺一致
- HTML 回傳完整一份，不是只給 diff 片段

對照重點
判斷學員版本是否：(1) 千分位用 toLocaleString() 或 Intl.NumberFormat，不要手刻 regex 因為遇到小數點會出錯；(2) 紅字自動清除綁 input 事件而非 change 事件，change 要失焦才觸發、體感會延遲；(3) RWD 斷點用 600px，跟題目指定一致；(4) 最關鍵：學員若回傳整份重寫且色彩/版面有動，表示沒遵守「只改三處」規則，該退回重做而不是合併結果。
成果產出
v1.html 與 v2.html 兩份檔案，加一張一行筆記：「這次最難讓 AI 改對的是 ____」。

SOLO · 自我演練
用下方 assignment 你做出來的工具 v2（從痛點 → 指令 → 護欄 → 二次生成），開新分頁啟一個沒有上下文的 LLM 對話，請它「扮一位完全沒看過這工具的新使用者」試用，並要求它說出「第一眼最搞不懂的操作是哪一步」與「第二個直覺反應是什麼」，把回饋當成下一輪修工具的依據。
3 分鐘

課堂即時作業 · Assignment ⏱ 核心 25 min + 進階 15 min

從痛點到可用工具：30 分鐘衝刺（含護欄與二次生成）

本作業是「動手做」性質 — 你要在 30 分鐘內，把一個職場痛點變成可用工具。不是寫文件、不是規劃，是真的做出來。

從以下 3 個典型痛點選 1 個（也可用自己的真實痛點替換）：

  1. 業務痛點：每週要把 5 家客戶的進度匯總成「客戶健康度週報」（手動 1.5 小時/週）
  2. 內容痛點：每天要把 3 個產品功能改寫成 4 個社群平台版本（FB / IG / LinkedIn / X，手動 2 小時/天）
  3. 行政痛點：每月要把 200 筆出差單分類報帳（含發票辨識，手動 4 小時/月）

本作業拆解：
• 核心 25 min（必做）：選痛點 + 寫工具指令（10 min · mini-prac 1）+ 加護欄（5 min · mini-prac 2）+ 實跑 + 二次生成（10 min · mini-prac 3）
• 進階 15 min（可選 · 二選一）：路線 A — 把工具用 Claude Artifacts / ChatGPT Canvas 真的跑出 HTML 介面 ｜路線 B — 寫工具的「使用者測試 SOP」（怎麼讓沒看過的人 5 分鐘上手）

3 段工具結構參考 · Materials
本章學的工具基本結構： ``` [輸入區] - 學員要填的欄位（清單、表單、貼資料） - 必要欄位驗證（少填會擋） ↓ [運算/AI 呼叫] - 內含 prompt（角色 + 任務 + 限制 + 輸出格式） - 護欄機制（不允許 X、必含 Y） ↓ [輸出區] - 結構化結果（表格 / 條列 / 段落 / JSON） - 可複製/下載功能 - self-check 提示 ``` 痛點 1 範例輸入結構（業務週報）： - 客戶名稱（5 個） - 本週狀態（每客戶 3 行：本週進度 / 卡點 / 下週計畫） - 預期輸出：5 個 mini-card（含風險標記 + CSM 行動建議） 痛點 2 範例輸入結構（社群改寫）： - 產品功能描述（1 段） - 目標平台（FB/IG/LinkedIn/X 勾選） - 預期輸出：4 個平台版本（每個有字數限制 + hashtag） 痛點 3 範例輸入結構（出差單分類）： - 出差單原始描述 + 金額 - 預期輸出：分類（交通/餐飲/住宿/其他）+ 是否需附發票（依公司規則）+ 報帳金額計算 📋 護欄三類型（本章學的）： 1. 輸入護欄：欄位驗證、必要值、格式檢查 2. 輸出護欄：格式鎖定、值範圍、self-check 3. 使用情境護欄：超出工具範疇時拒絕（如業務週報工具不應幫寫合約）
交付清單 · Tasks
  1. 核心 1：選痛點 + 寫工具指令（10 min · mini-prac 1）— 選 1 個痛點，寫一段「工具生成指令」給 ChatGPT/Claude（描述輸入區、運算邏輯、輸出區三段，不必寫 code，描述邏輯即可）
  2. 核心 2：加 3 類護欄（5 min · mini-prac 2）— 對你的指令補上 (a) 輸入護欄（必要欄位、格式驗證）(b) 輸出護欄（格式鎖定、self-check）(c) 使用情境護欄（拒絕超出範疇的請求）
  3. 核心 3：實跑 + 二次生成（10 min · mini-prac 3）— 把指令貼進 ChatGPT/Claude，看第一稿輸出。第一稿一定有問題 — 找出 2 個問題（欄位漏 / 邏輯錯 / 格式不對），寫第二輪 prompt 修正
  4. 進階路線 A：用 Claude Artifacts / ChatGPT Canvas 把工具真的跑出 HTML 介面（含可填寫的 form + 按鈕 + 結果顯示）
  5. 進階路線 B：寫使用者測試 SOP — 怎麼讓沒看過工具的人 5 分鐘上手（5 個測試步驟 + 預期失敗點）
評分標準 · Rubric（基本 4 條）
  • 三段結構齊全：輸入區、運算、輸出區都明確描述
  • 3 類護欄都有：輸入 / 輸出 / 使用情境護欄各 1 條以上
  • 實跑有證據：第一稿輸出 + 第二輪 prompt + 修正後輸出（要看到差異）
  • 第二稿真的解決問題：不是換個說法，要實際解決你抓到的 2 個問題
📖 展開參考解答（以痛點 1 業務週報為例）
核心 1 · 工具指令第一稿 ``` 請幫我建一個「客戶健康度週報生成器」工具。 【輸入區】 - 客戶數量：5（固定） - 每客戶填寫：客戶名 / 本週進度（80 字內）/ 卡點（80 字內）/ 下週計畫（80 字內） 【運算邏輯】 - 對每個客戶判斷風險等級： • 健康：進度按計畫 + 無卡點 + 下週計畫具體 • 風險：有卡點 OR 進度落後 • 警示：連續 2 週同卡點 OR 客戶未回應 - 對每個客戶產出「CSM 行動建議」（1 句話） 【輸出區】 - 5 個 mini-card（每張含：客戶名 / 風險等級 emoji / 進度摘要 / CSM 行動建議） - 末尾加總體看板：本週健康/風險/警示客戶數 + 推薦本週優先聯絡前 2 名 ``` 核心 2 · 加 3 類護欄 【輸入護欄】 - 客戶名為空 → 「該客戶資料不完整，請補填」 - 進度/卡點/下週計畫任一超過 80 字 → 「請精簡到 80 字內」 - 5 個客戶資料缺少時 → 顯示空白卡片「待填」 【輸出護欄】 - 風險等級必須在 {健康, 風險, 警示} 三類，不得自創 - CSM 行動建議必須是「動詞開頭 + 具體對象」（例：「下週一前致電 ABC 公司確認 demo 時間」） - self-check：「每張卡片是否含 4 個必要元素（客戶名 / 風險等級 / 進度摘要 / 行動建議）？」 【使用情境護欄】 - 若使用者問「順便幫我寫客訴回覆」/「分析戰略」 → 「此工具僅產生週報，其他需求請另開對話」 - 若使用者要求 5 個以上客戶 → 「目前版本固定 5 客戶，6+ 請聯絡管理員擴充」 核心 3 · 實跑 + 二次生成 第一稿問題（實跑後抓到 2 個）： 1. AI 把「客戶健康但有小卡點」直接判「風險」，沒有層級 2. CSM 行動建議寫成「持續關注」這種空話 第二輪 prompt 修正： > 補充規則： > - 風險判定加「強度」：輕微卡點（單次） → 仍標健康但加 ⚠ icon；連續卡點 → 才標風險 > - CSM 行動建議禁止「持續關注」「保持聯絡」這種空話。必須含：對象（誰）+ 動作（具體做什麼）+ 時間（何時）。例如「下週一前致電 ABC 公司確認 demo 時間」 跑第二輪 → 5 張卡片都符合，總體看板出現「優先聯絡：ABC + DEF 公司（連續 2 週同卡點）」具體建議。 常見錯誤 1. 沒實跑：學員只寫指令不丟 AI 跑，看不到問題 2. 三段結構不清：把運算邏輯寫進輸入區 3. 護欄寫成「請小心 X」：要寫具體規則（如「不得超過 80 字」），不是模糊要求 4. 第二稿換句話說：沒解決核心問題，只是改了 prompt 措辭 5. 使用情境護欄沒寫：工具被亂用會輸出垃圾
