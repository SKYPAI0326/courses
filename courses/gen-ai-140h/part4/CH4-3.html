學到這裡，你已經具備所有技能。現在把它們組合起來，用一場 20 分鐘的「個人黑客松」，從一個工作痛點直接產出可以用的工具。

(01)

黑客松思維：完成比完美更重要

黑客松（Hackathon）原本是工程師在 24-48 小時內快速做出產品的競賽模式。我們借用這個思維，但把時間縮短到 20 分鐘，目標只有一個：做出一個可以用的東西。

很多人學完工具後的第一個問題是「我要從哪裡開始？」答案很簡單——從你今天工作中最讓你煩躁的那個重複性動作開始。那個動作，就是你的第一個工具候選。

ℹ
好的工具候選長什麼樣子？你每週至少做一次、步驟固定可預測、大部分時間都在填數字或複製貼上——這就是一個值得自動化的痛點。
迷你練習 1 約 5 分鐘
列出 3 個工具候選，選出本週的黑客松題目

黑客松的成敗一半決定在選題。花 5 分鐘列出 3 個痛點、用三條判準打分，選一個可以今天下班前做完的題目。

1寫出 3 個你本週做過的重複性小事，每個用一句話。
2用三個判準評分（0–2 分）：頻率高／步驟固定／有數字或複製貼上。
3選總分最高的一題當黑客松題目，並寫下「我預期它會幫我省 ___ 分鐘／次」。
範本素材 先跑這份示範選題，再換成你自己的 3 個痛點
原始素材 · 某行政同事本週的 3 個重複性痛點（約 180 字）貼進 Prompt 最下方
候選 A：每週一彙整上週 Google Form 新客戶報名資料，從 Sheets 複製 8 筆資料貼到 CRM 後台，再回信確認，耗時約 25 分／次。 候選 B：每次會議結束後整理會議紀錄寄給 6 位與會者，手動抓出決議與待辦，耗時約 30 分／次，一週發生 2 次。 候選 C：每月月底從 3 個不同平台（廣告、LINE、Email）抓轉換數字，貼進月報 Google Sheet，手動計算佔比並配色，耗時約 45 分／次，一個月 1 次。 判準（每項 0–2 分）：頻率高（一週多次）／步驟固定（每次一樣）／有數字或複製貼上。

## 建議題目
- 勝選：候選 A，總分 5
- 理由：步驟與輸入輸出最固定、每週重複發生，最符合黑客松「20 分鐘可完成、今天下班前用得上」的題型

## 預期效益
「我預期它會幫我省 20 分鐘／次，每週省 20 分鐘」

對照重點
判斷學員版本是否：(1) 三個判準都給出 0/1/2 的具體分數，而不是籠統寫「蠻適合的」；(2) 勝選題目三項評分都 ≥1（沒有任何一項掛 0），否則做出來不值得——頻率 0 會變沒人用、步驟 0 會做不出工具、數字 0 就沒有自動化空間；(3) 預期效益是「每次省 X 分鐘 × 每週 Y 次 = Y·X 分鐘」的乘積而非單點估算；(4) 若學員選了總分較低的題目，應該問「是不是有情緒加分（這題最煩、最想解決）」，情緒加分本身沒錯但要標註清楚避免自欺。

20 分鐘工作流程：照著做就對了

以下是一個可以重複執行的 20 分鐘流程。第一次做可能會花 30 分鐘，第十次你可能 12 分鐘就完成：

0–3m
步驟一
定義痛點（3 分鐘）
用一句話說清楚你要解決的問題。「我每次報價都要手動加三個欄位的稅金和折扣，計算容易出錯」——這樣就夠了，不需要寫需求文件。
3–7m
步驟二
選擇工具類型，套用 Prompt 框架（4 分鐘）
根據痛點決定工具類型（計算 / 視覺化 / 排程…），從 Prompt 庫取出對應的框架，填入你的具體需求。
7–12m
步驟三
提交 Prompt，等待生成（5 分鐘）
把完整 Prompt 貼給 Claude 或 ChatGPT。等待期間可以準備好一個新的 HTML 檔案，等 AI 輸出後直接貼進去存檔。
12–17m
步驟四
測試與快速修正（5 分鐘）
打開瀏覽器測試。找出最明顯的一個問題，用一句話追加指令讓 AI 修正。不要試圖一次修改所有問題——先讓核心功能正確。
17–20m
步驟五
存檔並命名（3 分鐘）
按照 CH4-1 的命名規則存進工具資料夾，加到書籤列。完成。現在你有了一個可以用的工具。
迷你練習 2 約 15 分鐘
真的跑一次 20 分鐘黑客松（計時版）

用練習 1 選出的題目走完 20 分鐘流程。計時器打開，時間到就停——即使沒完成也記錄下來，這比「完美但沒動手」有價值。

1打開手機計時器設 20 分鐘，按下開始。
2照步驟一到五走（痛點 → Prompt → 生成 → 修正 → 存檔），每步卡住超過 1 分鐘就先略過往下。
3時間到記下：實際花幾分鐘、卡在哪一步、下次要改什麼。
範本素材 先照這份跑一次體感 20 分鐘
原始素材 · 候選 A 的完整題目規格（約 180 字）貼進 Prompt 最下方
題目：新客戶報名資料彙整與確認信草稿產生器 痛點：每週一從 Google Form 匯出 5–10 筆新客戶，複製到 CRM 後台後，還要一封一封手寫確認信。 輸入欄位： - 原始資料區：多行文字欄，貼入從 Sheet 複製的 tab 分隔資料（欄位：日期、姓名、公司、email、報名方案、備註） - 主辦人稱呼（預設：小姐 / 先生），影響信件署名 - 活動日期（日期選擇器） 運算： - 解析每一列成一筆客戶 - 產生每位客戶的個人化確認信草稿（繁中、5 句內、提到活動日期與報名方案） 輸出： - 上方：解析後的客戶清單表格（姓名、公司、方案） - 下方：每位客戶一張卡片，內含確認信草稿 + 複製按鈕
範例 Prompt · 20 分鐘黑客松起手式可直接貼 ChatGPT / Claude
你是擅長製作單一 HTML 檔案工具的前端助理。請根據下方「原始素材」的題目規格，直接產出一份可以用的 HTML（單檔、不依賴外部 library、不用串 API，信件草稿用前端 template 字串合成即可，不真的呼叫 AI）。 【必做】 1. 輸入區：多行貼入區（tab 分隔）＋ 稱呼下拉 ＋ 活動日期 2. 按「解析並產生」→ 下方呈現客戶清單表格 + 每位客戶的信件草稿卡片 3. 每張卡片右下角一顆「複製此信」按鈕，點後複製該信到剪貼簿 4. 貼入區為空或格式錯誤時，結果區顯示灰字提示，不要彈 alert 5. 主色霧藍、背景米白，風格簡潔 【信件 template（中繁、5 句以內）】 {姓名} {稱呼}您好， 感謝報名 {活動日期} 的活動，已收到您的「{方案}」報名資料。 如有任何疑問歡迎直接回信。 順祝 安好， 主辦單位 敬上 時間限制：20 分鐘，不追求完美，可用即可。 【原始素材】 （貼上上方題目規格）

## 工具結構
- 上：多行貼入區（tab 分隔，placeholder 提示欄位順序）＋ 稱呼下拉（小姐/先生，預設小姐）＋ 活動日期選擇器
- 中：主按鈕「解析並產生」（霧藍全寬）
- 下：解析後客戶清單表格 + 每位客戶一張信件草稿卡片

## 實際貼入 3 筆後輸出
- 表格三列：王小明｜某公司｜早鳥方案 ／ 李美華｜另一家公司｜一般方案 ／ 陳大同｜第三家｜VIP 方案
- 三張信件卡片，每張 5 句內，自動套入：{姓名} {稱呼}您好 → {活動日期}的活動 → 您的「{方案}」報名資料 → 歡迎回信 → 順祝安好
- 每張卡片右下角「複製此信」按鈕，點擊後該封完整信件進入剪貼簿

## 空值處理
貼入區留空按按鈕：結果區灰字「請先貼入從 Sheet 複製的客戶資料」，不彈 alert

對照重點
判斷學員版本是否：(1) 信件用前端 template 字串合成而非真的呼叫 AI API，因為題目說「20 分鐘內可用就好」不追求動態內容——學員若真的接了 API 就是過度設計，沒遵守黑客松紀律；(2) tab 解析用 split('\t') 而非逗號，因為從 Google Sheets 複製來的就是 tab 分隔；(3) 每封信是**獨立卡片**而非把三封串在一起，這樣學員才能一封一封複製寄給不同人；(4) 時間到停手、即使只完成 70% 也算數——學員若說「我多花了 10 分鐘補完」，代表沒抓住「先完成再完善」的核心心態。
成果產出
一個能打開的 HTML 工具（哪怕只完成 70%）＋ 一張「20 分鐘復盤表」：實際時間／卡點／下次調整。

實戰示範：從痛點到工具

以下是一個完整的黑客松實例，示範如何把一個工作痛點在 20 分鐘內轉換成工具：

實戰示範 · 外勤差旅費報銷計算器 完整流程
①
痛點描述（3 min）
每次出差回來要手算：交通費 + 住宿費 × 天數 + 餐費（每日上限 400 元）× 天數，再扣掉公司預付的備用金，最後換算成報銷金額。用計算機容易按錯，用 Excel 要開檔案。
②
套用框架，寫 Prompt（4 min）
你是擅長製作企業工具的前端工程師。 請幫我做一個「差旅費報銷計算器」HTML 工具： 輸入欄位： - 出差天數 - 交通費（來回合計） - 住宿費（每晚） - 實際餐費（每日，上限 400 元超出自動截斷） - 公司預付備用金 自動計算並顯示：總報銷金額、各項小計、應補/應退金額 單一 HTML，不依賴外部 library，設計簡潔好填
③
生成與測試（5+5 min）
貼給 AI，等待輸出，存成 HTML 打開瀏覽器。發現餐費上限邏輯不對（超出上限後沒有顯示警示文字），追加一句：
若每日餐費超過 400 元，輸入框變橘色邊框並顯示「已截斷至上限 400 元」
④
存檔完成（3 min）
存為「差旅費報銷計算器_2024-03.html」，加到瀏覽器書籤列。下次出差回來，直接打開填數字，30 秒搞定報銷單。
(04)

常見陷阱：新手最容易踩的三個坑

在自己做工具的初期，很多人會卡在以下幾個地方。認識它們，就能快速跳過：

要求「一次做完所有功能」，結果輸出又長又亂，測試時一堆問題
先做核心功能，確認可用後再用追加指令加功能
Prompt 只寫「幫我做個計算機」，AI 猜測你要什麼，輸出不符期待
明確描述：輸入欄位有哪些、輸出顯示什麼、有什麼特殊邏輯
工具有小問題就要求 AI「重寫整個」，浪費時間也丟失已有的好東西
每次只針對一個問題給出精確的修正指令，逐步完善
✦
最重要的心態：第一個工具不需要完美。做出來、存起來、用三次——用過了你才知道哪裡需要改。先完成，再完善。
迷你練習 3 約 8 分鐘
對照三個陷阱，回頭修復你的工具

拿練習 2 跑完的工具（或其 Prompt），對照本節三個陷阱檢查一遍，寫出一條補救指令並實際跑一次。

1對三個陷阱逐項問自己：「我有沒有犯？」誠實打勾。
2針對中招最嚴重的一條，寫出「只改 X，其他不動」格式的精準追加指令。
3跑一輪讓 AI 修正，把修好的版本另存新檔。
範本素材 對照三個陷阱自檢，寫一條精準補救
原始素材 · 黑客松 v1 的三陷阱自檢結果（約 160 字）貼進 Prompt 最下方
工具：新客戶確認信產生器 v1（練習 2 做出來的那版） 陷阱 1｜一次做完所有功能：我原本 Prompt 有要求「同時輸出 Email、LINE、簡訊三種版本」→ 最後三種都不對，放棄了。打勾：中招。 陷阱 2｜Prompt 太模糊：我沒講清楚「貼入的資料欄位順序」，AI 猜錯把「email」解析成「方案」，結果信件稱呼對了但方案全錯。打勾：中招最嚴重。 陷阱 3｜小問題就全部重寫：目前沒有犯。 決定：本輪只處理陷阱 2（解析邏輯），其他完全不動，特別是已經調好的視覺與複製按鈕。
範例 Prompt · 只改 X，其他不動可直接貼 ChatGPT / Claude

## 解析行為（v2）
- 貼入正確 3 筆（TAB 分隔，6 欄）：客戶表格三列正確、信件卡片三張姓名與方案精確對位
- 第 2 列故意少一個 TAB（只有 5 欄）：結果區灰字「第 2 列欄位數不足，請確認是否為『日期 姓名 公司 email 方案 備註』順序」
- 第 3 列 email 不含 @：灰字「第 3 列 email 格式不正確」
- 貼入區完全空白：灰字「請先貼入從 Sheet 複製的客戶資料」
- 所有錯誤訊息顯示在結果區，不彈 alert

## 保持完全不動
- 色彩（霧藍主色、米白背景）、卡片版型、主按鈕、複製按鈕、信件 template 逐字相同
- 表格結構、卡片順序、日期/稱呼下拉欄位全部沿用
- 若 v2 與 v1 視覺做 diff，差異應**只在解析邏輯 JS**那段

對照重點
判斷學員版本是否：(1) 按欄位「順序」精確解析（split('\t') 後固定 index 對應日期/姓名/公司/email/方案/備註），沒有讓 AI 再「猜」欄位；(2) 錯誤訊息明確指出「第 N 列」，讓使用者知道該修哪一列而非整份重貼；(3) 三種錯誤情境（欄位不足、email 無 @、完全空白）都有各自的具體訊息，不是合成一條「格式錯誤」；(4) 學員 v2 若順便改了卡片 CSS 或信件 template，表示沒遵守「只改解析邏輯」——這次練習的重點本來就是「單點修補」的紀律，dragon 過了反而失敗。
成果產出
一張「陷阱自檢表」＋ 一條追加指令 ＋ 修正後的 v2 工具檔。
