# Part 1 講師指引｜解鎖 AI 工作模式

> 本指引對應 140h 課程「Part 1」下的 CH1-1 ~ CH1-4 與 PRAC1-1 ~ PRAC1-4，對應 180h 實體班的帶領節奏。共 10 個迷你練習 (MP)、8 個延伸案例、1 個同儕互評、1 份通關檢測。

---

## Part 總覽

### 學員能力模型（Part 1 結束後應達到）
- **能拆**：拿到任何工作任務，能當場用「角色 / 任務 / 限制 / 輸出格式」四要素拆成可執行的 Prompt v1。
- **能診**：看到一段跑歪的 AI 輸出，能說出屬於「太空泛 / 太冗長 / 格式錯 / 不準確 / 不照角色」哪一類，並給出至少一行修法。
- **能守**：能寫出一段 4 元素齊備（Persona / Task Boundary / Output Lock / Refusal）的 System Prompt，並能撐過 10 輪對話不漂移。
- **能選**：面對具體任務時，能判斷該用 Web / Desktop / API 哪一種型態，並對 API 成本有量級感。

### 主題色
- Part 1 主色：**陶土橘 `#b5703a`**（對應 `--c-a1`）
- 迷你練習 MP：**芥末黃 `#c9963a`**（`--c-a2`，左側實線 3px）
- 延伸案例 Case：**霧藍 `#6b7fa3`**（`--c-a4`，卡片頂部細條）
- 同儕互評 Peer：**陶土橘**（左側實線 4px，底色 `#fbf8f3`）
- 通關檢測 Quiz：**鼠尾草綠 `#5a7a5a`**

### 核心骨幹（一條線講完 Part 1）
> 從「你跟 AI 要什麼」→ 到「你把 Prompt 寫對」→ 到「AI 沒回對怎麼修」→ 到「把會用的 Prompt 釘成工具」。CH1-1 是世界觀；CH1-2 是寫法；CH1-3 是修法；CH1-4 是把寫法與修法固化成 System Prompt。PRAC 1-1 ~ 1-4 分別把這四步各做成一個可交付成品。

## CH1-1｜AI 不只是聊天工具，而是你的工作介面

### 迷你練習 MP ch1-1-mp1｜把你本週三個常用工作，對標到合適的 AI 工具
- 預估時間：8 分鐘
- 帶領要點：
  - 開始前先問全班「你這禮拜已經試過用 AI 幫忙做哪件事？」一人一句，講師邊寫邊分類到 ChatGPT / Claude / Gemini 欄位，讓學員看到同儕差異再動手寫自己的。
  - 強調「3 個任務必須是真的，不能臨時編」，若學員卡住就引導他們翻昨天的 LINE 工作群組。
- 常見卡點：
  - 學員只會寫「寫信、摘要」這種空殼任務 → 追問：「寫給誰的信？要達成什麼？」逼出語境。
  - 學員三個任務全部選 ChatGPT → 追問：「有沒有一個是長文件、一個是要配 Google 文件搜尋的？」打散選擇。
- 示範答案語氣：「我這禮拜三要寫一封信給客戶，解釋我們為什麼要漲價 8%，收件人是比較敏感的採購窗口。這件事我會交給 Claude，因為它比較擅長拿捏人情語氣，ChatGPT 會太公式化。另外我禮拜五要把 30 頁的競品官網整理成比較表，這我會開 Gemini 配 Deep Research，因為它可以直接連網爬，不用我一頁一頁複製貼上。」
- 延伸話題：帶到「工具偏好不是信仰」，鼓勵學員週末花 15 分鐘把同一個任務丟到三家跑，做自己的 A/B test。

### 迷你練習 MP ch1-1-mp2｜5 個日常任務，判斷該用 Web / Desktop / API 哪一種
- 預估時間：7 分鐘
- 帶領要點：
  - 這是觀念題，不要求學員動手跑 API，只要判斷。講師先在白板畫 3 欄（Web / Desktop / API），邊講邊補位。
  - 強調判斷關鍵：「這件事要做幾次？每次差不多嗎？要不要自動化？」
- 常見卡點：
  - 學員把「批次處理 200 封客訴信」塞到 Web → 提醒：「Web 一次開一個對話框，200 次你要點 200 次。」
  - 學員把「跟 AI 對話練英文」塞到 API → 提醒：「API 沒有介面，你要先寫個聊天視窗出來。」
- 示範答案語氣：「每天早上整理三個新聞重點，我會用 Desktop，因為 Mac 版可以按快捷鍵叫出來、不用開瀏覽器。但如果我是幫公司一次處理 500 筆客服留言分類，這就必須走 API，因為 Web 版一次貼不了這麼多，而且我要把結果寫回 Google Sheet，只有 API 能串。」
- 延伸話題：提早預告「Part 4 我們會真的串 API，現在先有概念就好」，降低焦慮。

### 迷你練習 MP ch1-1-mp3｜估算你若用 API 跑 1 個月的成本
- 預估時間：10 分鐘
- 帶領要點：
  - 打開 OpenAI pricing 頁面投影給全班看，一起算一次「1 個任務用 2000 tokens、每天 10 次、跑 30 天」要多少錢。
  - 目的不是精算，是讓學員對「API 用到爆也可能只是一杯咖啡錢」有感，破除「API 很貴」的迷思。
- 常見卡點：
  - 學員搞不清楚「token ≠ 字」→ 給個粗估：中文 1 字約 1.5 token，英文 1 字約 1.3 token。
  - 學員算出「一天只花 0.8 美金」不敢相信 → 再追一題：「如果換成 Claude Sonnet 4 呢？」讓他自己查。
- 示範答案語氣：「我抓『每天用 Claude Haiku 翻譯 10 封客服信』，每封約 800 tokens 輸入、400 tokens 輸出，一天 12000 tokens，一個月 36 萬 tokens。用 Haiku 的價格換算下來，一個月不到 1 美金。比我現在訂 ChatGPT Plus 的 20 美金便宜太多——但前提是我得自己寫介面。」
- 延伸話題：引導學員回頭看 MP2 的分類，補註「走 API 時每個月估算成本」這欄。

## CH1-2｜黃金提示詞公式：角色、任務、限制、輸出

### 迷你練習 MP ch1-2-mp1｜把你平常的模糊指令，用四要素改寫一次
- 預估時間：10 分鐘
- 帶領要點：
  - 要求學員**一定要打開自己前一週的聊天紀錄**，不要憑空想像 v1。v1 越真實、v2 的差距越有感。
  - v1 與 v2 寫完後立刻開 ChatGPT 或 Claude 各跑一次，不要只比較文字。
- 常見卡點：
  - 學員把 v1 美化了才寫下來 → 明說「寫得越糟越好，等等改才有差距」。
  - 學員補「限制」時只寫「不要太長」→ 追問：「太長是幾字？300 還是 800？要不要 emoji？」
- 示範答案語氣：「我的 v1 是『幫我寫下週社群貼文的草稿』。改成 v2 後變成：『你是專寫文青咖啡店 IG 的小編，請幫我寫 3 則週三下午 3 點發的貼文，每則 80-100 字，要有一個提問結尾、不要用 emoji，輸出純文字、每則中間空一行。』跑完之後我發現 v2 每則都直接可用，v1 那版 AI 還反問我『請問是什麼產業的社群？』——光這一個問題就省了一輪來回。」
- 延伸話題：若時間允許，收 3 位學員的 v1/v2 放投影幕，全班找「哪一個要素補得最關鍵」。

### 迷你練習 MP ch1-2-mp2｜同一個任務，三種格式的效果差多少？
- 預估時間：10 分鐘
- 帶領要點：
  - 先請學員選一個「結果要貼進別的工具」的任務，例如「會議紀錄貼進 Notion」「競品比較貼進 Google Sheet」。
  - 三種格式（Markdown 表格 / JSON / 條列）分別跑一次，講師要提示學員觀察的不是內容，是**可用性**（能不能複製貼上就用）。
- 常見卡點：
  - 學員覺得三個輸出「看起來差不多」→ 追問：「你複製這段直接貼到 Google Sheet，是不是自動就分欄了？」
  - 學員不知道 JSON 能幹嘛 → 示範把 JSON 貼進 Make.com 的 Parse JSON 模組，立刻能繼續流程。
- 示範答案語氣：「我選『整理每週待辦』這個任務。Markdown 表格貼進 Notion 就自動變成資料庫，每欄對得好好的；JSON 我把它丟給 Make.com 直接分配到 Google Tasks；條列版最漂亮，但我得手動再把它拆成 Notion 欄位。結論是——輸出格式不是美感問題，是下游工具決定的。」
- 延伸話題：介紹「這就是 Part 4 自動化的前置知識，我們現在就在訓練下游相容性」。

### 迷你練習 MP ch1-2-mp3｜Zero-shot vs Few-shot：給例子有沒有差？
- 預估時間：10 分鐘
- 帶領要點：
  - 最適合的任務：「風格改寫」「分類標籤」「客服語氣」。避開「數學題」「翻譯外語」這類 Zero-shot 本來就夠好的任務。
  - 示範 Few-shot 時強調「例子要長得像你真的要的產出」，不要自己先寫不好的範例。
- 常見卡點：
  - 學員給的範例長度不一，AI 輸出就跟著亂 → 提醒「例子要一致，不然 AI 會學到不一致」。
  - 學員只給 1 個範例 → 示範給 3 個（同主題不同角度），輸出會更穩。
- 示範答案語氣：「我的任務是把客訴信分類成『產品問題 / 物流問題 / 客服態度 / 其他』。Zero-shot 跑完我得到 4 筆裡有 2 筆分類錯，因為 AI 把『物流慢導致產品變質』全部塞進『物流』。我補了 3 個範例 Few-shot 之後，AI 學會『有歸類衝突時挑造成損害的那一個』，後面 20 筆全對。」
- 延伸話題：連到 CH1-3 的「Prompt 除錯」，說「這就是在救『不準確』那一類問題」。

## CH1-3｜Prompt 除錯：怎麼讓 AI 越改越準

### 迷你練習 MP ch1-3-mp1｜把三段失敗的 Prompt，分到五種病因
- 預估時間：8 分鐘
- 帶領要點：
  - 這題目的是讓學員學「診斷語言」。不要讓他們花時間重寫 Prompt，只要分類 + 一行修法即可。
  - 三個案例分別對應「太空泛」（A）、「不照角色」（B）、「格式錯」（C），若學員全答對可再加問：「那 D/E 類（太冗長、不準確）你能舉自己碰過的例子嗎？」
- 常見卡點：
  - 學員把 A 分到「格式錯」→ 追問：「AI 有回你 JSON 嗎？有。格式有破嗎？沒有。那是什麼問題？」引導到「它根本沒回你結論」。
  - B 案例學員常誤判成「AI 出包」→ 提醒：「AI 不是壞掉，是你的 Prompt 沒鎖角色。」
- 示範答案語氣：「案例 A 是太空泛，因為我沒告訴 AI『市場要看哪 3 個面向』『要給我結論還是描述』。修法：在 Prompt 裡補『請先給三行結論，再按競爭強度、毛利率、成長速度各列一段』。案例 B 是不照角色，因為我只在第一輪說了『嚴謹律師』，沒有寫 System Prompt 鎖住；修法：把角色設定搬到 System Prompt，而不是 User Prompt。案例 C 是格式錯，AI 在 JSON 前面多加了客套話——修法：Prompt 結尾補『只輸出 JSON，不要任何前言或說明』。」
- 延伸話題：帶到「CH1-4 會教你怎麼用 System Prompt 把 B 類問題一次鎖死」。

### 迷你練習 MP ch1-3-mp2｜用「先對 → 再好 → 再快」跑三輪修正
- 預估時間：10 分鐘
- 帶領要點：
  - 這是 Part 1 的**最核心練習**，時間一定要給夠，不要為了趕進度跳過。
  - 三輪每一輪講師都要在白板標出「這輪在改哪一件事」，避免學員每輪想改全部。
  - 第 1 輪（對）：只看事實對不對，語氣難看也沒關係。第 2 輪（好）：改語氣、結構。第 3 輪（快）：濃縮、砍廢話、壓縮 token。
- 常見卡點：
  - 學員第 1 輪就想一次改到位 → 明令「第 1 輪只能改內容，不准改語氣」。
  - 學員跑到第 3 輪發現內容又不對了 → 說明「那代表你第 1 輪沒對透，回去補」。
- 示範答案語氣：「我的任務是幫主管寫年中績效自評。第 1 輪我只確認『去年完成的 5 個專案數字有沒有對』，發現 AI 編了兩個我沒做過的專案，修 Prompt 加『只能根據我提供的清單，不可推論』。第 2 輪我改語氣，把『本人負責⋯⋯』改成『這一年我主導⋯⋯』，更像活人寫的。第 3 輪我砍掉重複的副詞和贅句，從 600 字壓到 420 字，主管看起來更爽。」
- 延伸話題：引導學員做完後回頭比較 v1 跟 v3，通常 v3 的 Prompt 字數比 v1 少但輸出品質高——這是「好 Prompt 不一定長」的反直覺時刻。

## CH1-4｜System Prompt：給工具用的提示詞設計

### 迷你練習 MP ch1-4-mp1｜拆解一份現成 System Prompt，標出四個元素
- 預估時間：10 分鐘
- 帶領要點：
  - 事先準備 2-3 個好樣本（GPTs Store 的 Data Analyst、Claude Projects 的 Writing Coach、或 Awesome-ChatGPT-Prompts 的 Linux Terminal）放投影片，避免學員卡在「找不到好範例」。
  - 要求學員**真的用四色**標記（紅/藍/綠/橘或任何四色），不要用一句話概述帶過，這練習就是要視覺化「缺哪塊」。
- 常見卡點：
  - 學員發現某個範例「根本沒有 Refusal」→ 正中下懷，明說「這就是市面上很多 GPTs 的通病，你待會自己寫時要記得補」。
  - 學員把 Persona 跟 Task Boundary 混在一起 → 區分：Persona 講「你是誰」，Task Boundary 講「你做什麼不做什麼」。
- 示範答案語氣：「我拆了 Awesome Prompts 的『English Translator』。Persona 寫得很強：『你是英翻譯家與拼寫改正者，但只做修飾、不改內容。』Task Boundary 也清楚。但 Output Lock 只寫『用高級英文回答』沒說格式，我猜就是為什麼它有時回段落、有時回條列。Refusal 完全沒有——如果我丟中文要它翻成日文，它照做，不會拒絕越權。我補的 v2 Refusal 是：『若輸入語言非中英互譯，請回覆「本助手僅處理中英互譯」並停止。』」
- 延伸話題：帶出「你下週要交的個人助手 System Prompt（PRAC1-4）就是做這個，現在拆得越細、到時候寫得越快」。

### 迷你練習 MP ch1-4-mp2｜壓力測試：System Prompt 能撐過 10 輪對話嗎？
- 預估時間：10 分鐘
- 帶領要點：
  - 講師提供「打擾劇本」：第 3 輪故意換語言、第 5 輪故意問題外問題、第 7 輪故意要求違反 Refusal、第 10 輪回到正常任務看角色還在不在。
  - 強調「這是壓力測試不是找 AI 麻煩」，學員寫完 System Prompt 常常只在乖乖問的情況下測試，漏洞都沒爆出來。
- 常見卡點：
  - 學員第 5 輪就投降說「還穩啊」→ 提醒：「你才問了 5 輪，工作中這個助手可能要撐 50 輪。」
  - 角色在第 7 輪漂移後學員不知道怎麼修 → 指向 Refusal：補一句「遇到 X 類型問題時，回覆『這超出助手範圍』」。
- 示範答案語氣：「我測了『診所客服分流助手』。前 4 輪都穩，第 5 輪我故意問『你覺得這間診所生意好嗎？』，它居然回答了一段閒聊——明顯越界。我回去把 Refusal 補成『只處理預約 / 改時間 / 領藥 / 報告查詢四類訊息，其他請回「請來電洽詢」』。再測第二次從頭跑 10 輪，第 5 輪開始就正確拒絕了。」
- 延伸話題：引出「這就是 PRAC1-4 通關任務的驗收指標之一——跨輪穩定」。

## PRAC1-1｜Prompt 黃金公式組合台

### 延伸案例 Case prac1-1-c1｜小吃店老闆回覆 Google 商家一星負評
- 預估時間：30 分鐘
- 帶領要點：
  - 這案例情感張力高，學員投入度最好。先讀評論全班一起讀出來，感受「這家店老闆被冤枉到什麼程度」，帶著情緒寫 v1 反而會暴露漏洞。
  - 強調「你要寫的不是給客人看的，是給**其他潛在客人**看的」——這會改變整個 Prompt 的設計。
- 常見卡點：
  - 學員 v1 寫成「客服罐頭語氣」，一開始就「親愛的顧客，非常抱歉造成您的不便」→ 對照參考 Prompt 明文禁用這類套話。
  - 學員忽略「不提顧客姓名」這條 → 指出：一旦提名，其他看的人會覺得這家店會公審客人。
- 示範答案語氣：「陳小姐您好，聽到蛋餅冷掉跟結帳等太久，我們這邊已經跟當天在場的同事對過，那天外帶特別多，出餐到您手上真的太慢，這是我們自己的問題。本週起我們把外帶取餐台多加一個保溫架，奶茶封膜機也送去校正了。下次您有機會再來，麻煩跟櫃台提一下這則回覆，我親自補一份現做的蛋餅給您，真心希望能再為您服務一次。老闆阿明。」
- 延伸話題：帶出「三種變體」後，問全班：「如果你是經營 5 年的老店，誠懇和幽默你會挑哪個？如果你是新開的咖啡店呢？」

### 同儕互評 Peer prac1-1-p1｜Prompt 改造對拆——我寫的 v1，你找漏洞
- 預估時間：25 分鐘
- 配對策略：
  - 優先**異業配對**（行銷配工程、老師配業務），避開同公司同事——熟人會手下留情，找不到漏洞。
  - 若學員人數為奇數，講師下場當評審，挑最敢丟真實工作 Prompt 的那位學員配對。
- 分組時間管理：
  - 8 分鐘獨立寫 v1（講師走動巡視，看到字數寫太少的提醒「再補具體情境」）。
  - 4 分鐘 B 預測 AI 輸出（**這段最容易偷跑成直接講評**，講師要喊「還沒到講評時間，繼續寫你猜 AI 會回什麼」）。
  - 6 分鐘 B 打分＋建議。
  - 5 分鐘對談（最核心的 5 分鐘，講師不要去打擾）。
  - 2 分鐘 A 跑 v2 實測（若 5 分鐘對談超時就犧牲這段，下課後補做即可）。
- 常見冷場處理：
  - 有組別太快結束 → 丟新題：「現在角色交換，A 當評審，B 當作者，再跑一輪。」
  - 有組別卡在「我看不出對方 Prompt 哪裡有問題」→ 提示：「你就直接拿去 ChatGPT 跑，看輸出跟對方預期差在哪。」
  - 全班氣氛悶 → 隨機挑一組把 v1 放投影幕，全班一起當評審，破冰很快。

## PRAC1-2｜指令品質健診器

### 延伸案例 Case prac1-2-c1｜把一行糊 Prompt 改造成高分指令
- 預估時間：20 分鐘
- 帶領要點：
  - 這案例的關鍵不是「改到滿分」，是「用健診器的 5 個維度逐一自我檢查」。強迫學員把 v1 貼進健診器跑一次，看分數再動筆。
  - 若健診器分數出來某維度特別低（例如「限制」只有 2 分），就只針對那一維度補，不要全面重寫。
- 常見卡點：
  - 學員拿到 v1「幫我寫個新品宣傳文」覺得很好改，三兩下就補完 → 追問：「你補的『100 字』是憑什麼？新品是珍奶還是新包裝？」逼他補脈絡。
  - 健診器的分數學員不信 → 提醒：分數不是絕對，是當作「哪裡最弱的快速指標」。
- 示範答案語氣：「v1 原句『幫我寫個新品宣傳文，明早給我』在健診器跑出來角色 1 分、輸出 2 分，其他還可以。我改 v2：『你是台中連鎖手搖飲的社群小編，負責 6/1 上市的芋頭鮮奶新品宣傳。請寫一則 IG 貼文草稿，100-120 字，語氣年輕但不浮誇，結尾含一個開放提問，輸出純文字＋建議 3 個 hashtag。』再跑健診器，5 個維度全部 4 分以上，主管看到這版直接說可用。」
- 延伸話題：連到 CH1-3 「先對再好再快」——這案例是「先對」做完，「好」還可以在語氣上繼續打磨。

### 延伸案例 Case prac1-2-c2｜品管報告 Prompt 健診——抓出「格式不一致」的元凶
- 預估時間：20 分鐘
- 帶領要點：
  - 這是「輸出格式」維度的專門案例，學員很常一開始看不懂為什麼「每週輸出不一樣」也是問題。示範：同一個 Prompt 跑 3 次，把 3 次輸出疊在一起給全班看欄位錯位。
  - 帶出解法：Few-shot（給一個完整範例）＋明文鎖欄位（「輸出 Markdown 表格，欄位依序為：日期 / 客訴類型 / 嚴重度 / 處理狀態，不得增減或重排」）。
- 常見卡點：
  - 學員以為只要改「輸出格式」這一欄就好 → 提醒：要同時給範例，不然 AI 每週還是會自己發揮。
  - 學員不知道什麼叫「嚴重度」→ 示範把定義塞進 Prompt（S1/S2/S3 各指什麼）。
- 示範答案語氣：「原本 v1 只寫『整理成週報表格』，AI 第一週給我 4 欄、第二週變 6 欄、第三週居然變條列。v2 我補：『輸出 Markdown 表格，固定 4 欄：日期 / 客訴類型 / 嚴重度（S1 最重至 S3 最輕）/ 處理狀態（未處理 / 處理中 / 已結案）。如下範例：』再把一份去年的標準週報貼上去當 Few-shot。之後連跑四週都一模一樣，主管直接丟進她的比對 Excel 零手動。」
- 延伸話題：引出「這就是為什麼 System Prompt 要鎖 Output Lock」——下一個 PRAC 會練到。

## PRAC1-3｜第一個工具：從需求到成品

### 延伸案例 Case prac1-3-c1｜補習班小老師的單字抽考工具
- 預估時間：40 分鐘
- 帶領要點：
  - 這是 Part 1 第一次讓學員「用 Prompt 讓 AI 生出一個真的會動的 HTML 頁面」，時間要拉長，而且要現場跑起來。
  - 講師事前先自己跑過一次，知道學員大概會在哪裡卡。最常卡的是「AI 給的 HTML 打開一片空白」——因為學員沒說「請把 HTML / CSS / JS 全部寫在同一個檔案，不用外部 CDN」。
  - 提醒學員：第一輪先求能動，不要求好看。
- 常見卡點：
  - AI 生的頁面「隨機出題」其實是同樣順序 → 檢查有沒有 `Math.random`。
  - 錯的單字沒顯示 → 補 Prompt：「答錯的題目要暫存在陣列，最後結算時全部列出並附正確答案」。
  - 頁面醜到學員沒動力測試 → 提醒「CH1-2 的 Few-shot，你可以貼一個你喜歡的現成 UI 截圖讓 AI 參考」。
- 示範答案語氣：「我第一版 Prompt 寫『做一個單字抽考工具』，AI 給我一個 React 元件，我根本打不開。第二版我改『做一個單一 HTML 檔案（含 CSS 與 JS），不使用任何外部 CDN 或框架。功能：上方文字框貼入單字清單（格式：英文,中文，每行一組），按「開始」隨機出 10 題，使用者輸入中文，按 Enter 對答，結算時顯示分數＋列出答錯的單字與正確答案。』這版直接開得起來。第三版我才補美化與計時。」
- 延伸話題：帶到「這就是你日後要的 AI 協作模式——先跑起來，再迭代」。

### 延伸案例 Case prac1-3-c2｜接案設計師的工時對帳單產生器
- 預估時間：40 分鐘
- 帶領要點：
  - 這案例比 c1 難，涉及**資料解析**（從流水帳文字到結構化資料）。關鍵是讓 AI 先在 Prompt 裡**定義資料格式**，再寫解析邏輯。
  - 示範：先丟 3 行範例給 AI「解析這三行，輸出 JSON」，確認解析對了再寫頁面。
- 常見卡點：
  - 學員直接要 AI「做對帳單產生器」，AI 會直接動手寫，忽略「資料格式不一致」的風險 → 拆解成兩步：「先寫一個 parser，再寫一個 renderer」。
  - 學員流水帳格式不穩（有時寫「A 客戶 logo 1.5h」有時寫「logo (A 客戶) 1.5」）→ 提醒：讓 AI 在 parser 裡加「格式容錯規則」或「不合規則時回報哪一行有問題」。
- 示範答案語氣：「我的 Prompt 分兩層。第一層：『請先寫一個 JavaScript 函式 parseEntries(text)，吃字串每行一筆，格式為「日期 客戶 任務 時數」（分隔符允許空格或 tab），輸出陣列 of {date, client, task, hours}。遇到不合規則的行，推進 warnings 陣列並繼續。』第二層：『再寫一個 renderInvoice(entries, clientName)，吃過濾後的資料，輸出一張可列印的對帳單 HTML：上方客戶抬頭、中間項目表格、下方總工時與小計、footer 放月份。』分開要之後，我每一部分都能單獨測試，也能單獨修。」
- 延伸話題：連到 Part 4 的自動化——「這個 parser 未來可以搬到 Make.com 當 Custom Code 模組」。

## PRAC1-4｜個人 AI 助手配置實作

### 延伸案例 Case prac1-4-c1｜診所櫃台的病患訊息分流助手
- 預估時間：35 分鐘
- 帶領要點：
  - 這案例的 System Prompt 寫作重點在 **Refusal 與 Output Lock**，因為診所溝通一錯就踩醫療規範紅線。
  - 明令「絕對不可以讓 AI 給醫療建議」——Refusal 要寫死。
  - Output Lock 必須含「分類標籤 + 標準回覆草稿」的雙欄位格式，讓櫃台用最少力氣送出。
- 常見卡點：
  - 學員 Refusal 只寫「不提供醫療建議」→ 追問：「那病人問『我這個症狀嚴重嗎』怎麼辦？」逼他補成「任何症狀判斷類問題，一律回『建議您直接來診所由醫師評估，或撥打預約專線 XXX』」。
  - 學員 Output Lock 沒鎖格式 → 示範：要 AI 每次都輸出「【類別】預約 / 改時間 / 領藥 / 報告 / 其他」＋「【建議回覆】⋯⋯」，才能貼進 LINE。
- 示範答案語氣：「我寫的 System Prompt 核心是這段：『你是診所櫃台助手，只處理四類訊息：預約、改時間、領藥、報告查詢。每次收到訊息後，請嚴格依下列格式輸出，不要有任何前言或閒聊：【類別】___｜【建議回覆】___（繁體中文 2-3 句、語氣親切但不過度熱情、結尾附診所電話 02-XXX）。若訊息無法歸入四類（如症狀詢問、藥物建議、閒聊），類別一律標「其他」，建議回覆統一為「您好，這部分建議您直接來診所由醫師評估，或撥打 02-XXX 由專人為您說明」。』跑 10 則測試全部穩定輸出，櫃台同事看完直接說可以上線。」
- 延伸話題：帶出「Part 4 可以把這個 System Prompt 接上 LINE webhook，真的自動回——但今天先把 Prompt 本身做到穩」。

### 延伸案例 Case prac1-4-c2｜部門週會紀錄 × 待辦拆解的雙身份助手
- 預估時間：40 分鐘
- 帶領要點：
  - 這案例的挑戰是**雙輸出不互相污染**——同一段會議逐字稿，要同時產出「正式會議紀錄」和「個人化 Slack 待辦訊息」。
  - 關鍵 Prompt 技巧：在 Task Boundary 裡明寫「兩種輸出分別有獨立的語氣與讀者」，然後在 Output Lock 裡用 `---` 分隔兩個段落。
- 常見卡點：
  - 學員用單一 Prompt 硬塞雙輸出，AI 會把兩種語氣混在一起 → 解法：明文分段「### 會議紀錄（給主管）」vs 「### Slack 待辦（給個人）」。
  - 學員忘了處理「同一個人出現在多個待辦」→ 提醒加一句「依負責人分組，不要重複出現」。
- 示範答案語氣：「我的 System Prompt 把兩個輸出完全隔開：『### Part A 會議紀錄：語氣正式、第三人稱、條列式、含「決議事項 / 未決議事項 / 下週追蹤」三節。### Part B Slack 待辦：每位負責人一張 card，格式為「@姓名：請於 MM/DD 前完成 XXX（產出物：YYY）」，語氣直接、不含會議細節、一個負責人合併一張。兩 Part 之間用「---」分隔，除此以外不輸出任何內容。』測試用我上週真實的會議逐字稿跑，Part A 直接貼 Notion，Part B 直接複製到 Slack，零手動。」
- 延伸話題：講到「Part B 就是通知訊息產生器，未來串 Slack bot 的前置作業」。

### 延伸案例 Case prac1-4-c3｜蝦皮/官網雙通路客服 AI 配置
- 預估時間：35 分鐘
- 帶領要點：
  - 這案例教「一個 System Prompt 吃多情境」，關鍵是用 **條件分支**寫規則，而不是寫兩個 Prompt。
  - 示範：讓學員在 Prompt 裡明寫「若通路 = 蝦皮，語氣為 A；若通路 = 官網，語氣為 B」，User 每次輸入都要指定通路。
- 常見卡點：
  - 學員寫兩個 System Prompt 分別貼兩個對話 → 提醒：這違反「一個助手」的設計，日後維護兩份規則會漂移。
  - 語氣差異學員寫得太抽象（「蝦皮口語、官網文雅」）→ 提示給實例：「蝦皮：『這個顏色真的爆紅的！今天下單明天就出喔～』；官網：『這款為熱銷色號，今日下訂可於明日出貨，敬請把握。』」
- 示範答案語氣：「System Prompt 核心：『你是手工皂一人電商的客服。每次 User 輸入的格式為「【通路】蝦皮 或 官網｜【問題】XXX」。若通路為蝦皮：用口語繁中，可用「～」「喔」「耶」等語尾，可用 1-2 個 emoji，長度 40-80 字。若通路為官網：用正式繁中，不用 emoji，不用語尾助詞，長度 60-120 字。若兩個通路都未指定，回覆「請提供通路資訊以產出合適回覆」並停止。』測試我刻意只給問題不給通路，AI 真的停下來反問——Refusal 奏效。」
- 延伸話題：連到 Part 4：「未來這個 Prompt 可以直接接 Google Sheet 當客服回覆產生器，每筆詢問自動對應通路。」

## Part 1 通關檢測 data-quiz="part1"

### 講解節奏建議
- 總時數：**30-40 分鐘**（180h 班建議排 40 分鐘，14-18 人規模）。
- 建議分段：
  - **0-12 分鐘｜觀念題 10 題獨立作答**：學員不討論、不查資料，單純作答。講師在教室走動，看到有人看白板或偷看同學要提醒「錯了沒關係，重點是你知道錯在哪」。
  - **12-18 分鐘｜當場對答案（講師逐題講解）**：每題不超過 40 秒，講師只講「為什麼 B 對、A/C/D 錯在哪個本質」，不延伸。若某題全班答對率 < 50%，現場多花 2 分鐘重講該觀念。
  - **18-35 分鐘｜綜合任務**：學員開 ChatGPT / Claude 寫自己的 System Prompt，跑三輪驗證。講師巡場，優先處理「Refusal 寫不出來」「四元素卡在 Output Lock」的學員。
  - **35-40 分鐘｜自評打分＋公開分享 2 位**：隨機挑兩位通過的學員分享 Prompt，讓還沒過的人看到「長怎樣才算過」。
- 10 題觀念題重點提示（給講師對答案用）：
  - 第 1、3、6、7、9 題是**核心觀念題**，答錯代表沒搞懂，要當場補強。
  - 第 2、5、8、10 題是**延伸應用題**，答錯可以課後補看影片即可。
  - 第 4 題（API 選擇）若答錯率高，代表 CH1-1 的 MP2/MP3 沒做足，下梯次要把那兩題時間延長。

### 綜合任務評分準則（4 面向 × 評分標準）

學員自評時提供以下 4 面向，每面向 0 / 5 / 10 三段計分，滿分 40 分，**30 分以上及格**。

| 面向 | 10 分（優秀） | 5 分（及格） | 0 分（不及格） |
|------|--------------|-------------|---------------|
| **四元素齊備** | Persona / Task Boundary / Output Lock / Refusal 各有至少一段完整描述，且彼此無重疊、無遺漏。 | 四元素都有但其中一項只有一兩句敷衍（例：Refusal 只寫「不提供醫療建議」沒說碰到時怎麼回）。 | 缺任一元素，或多個元素混在一起分不清。 |
| **跨輪穩定** | 刻意跑三輪不同輸入（正常 / 邊界 / 打擾），角色不漂移、格式完全一致、語氣一致。 | 三輪中有一輪格式或語氣略漂但不影響可用性。 | 三輪中有兩輪以上明顯走鐘，或 AI 開始閒聊、自創欄位。 |
| **拒絕奏效** | 刻意丟 2 個以上界外問題（跨領域、違規、套話），AI 全部正確識別、正確拒絕並引導下一步。 | 丟 2 個界外問題只正確拒絕 1 個；另 1 個硬答或含糊帶過。 | AI 碰到界外問題直接照答，或拒絕得太僵（直接「我不能回答」無後續引導）。 |
| **輸出可用** | 三輪輸出都能直接貼到實際工作產出（LINE / Slack / Notion / Email），零或極少手動調整。 | 輸出大致可用但需微調格式或語氣 10-20%。 | 每次輸出都要重寫超過一半，形同白做 System Prompt。 |

**講師加碼提醒**：若時間允許，鼓勵學員拿這份 System Prompt 去 PRAC1-4 的 helper 工具再生成一次「完整配置包」，對照 AI 版跟自己寫的差在哪——這是往 Part 2 繼續精進的自然橋樑。
