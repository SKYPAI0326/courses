開發環境建置指引

Part 5–7 的課程內容需要在你的電腦上執行真實的程式碼。這份指引花 30–60 分鐘，幫你一次完成所有前置設定，讓後面的課程能真正跑起來。

不需要本指引
AI 工作應用路徑

Prompt 設計、文書工作、資料清理、零代碼工具
完全不需要安裝任何程式環境
需要本指引
AI 應用開發路徑

前端實作、API 串接、後端部署、專題開發

Step 0 · Before You Start

終端機：你的指令輸入介面

後面所有安裝步驟都需要在「終端機」（Terminal）裡輸入指令。如果你從來沒用過，先花 2 分鐘認識它。

按 Command（⌘）+ 空白鍵 開啟 Spotlight，輸入「Terminal」按 Enter。或打開「應用程式 → 工具程式 → 終端機」。

💡
建議安裝 iTerm2（免費），功能比內建 Terminal 更好用。從 iterm2.com 下載安裝即可。

按 Win + X，選「Windows Terminal」或「PowerShell」。或搜尋「cmd」開啟命令提示字元。

💡
建議安裝 Windows Terminal（Microsoft Store 免費下載），並在課程中使用 PowerShell 而非 cmd。
🔑
終端機使用心態：輸入指令 → 看輸出 → 沒有錯誤訊息就代表成功。錯誤訊息通常直接說明了問題，貼給 AI 請它幫你解讀是最快的除錯方式。
Step 1 · Runtime

安裝 Node.js

Node.js 讓你的電腦能執行 JavaScript 程式碼，也是後端代理服務的執行環境。安裝後同時會有 npm（套件管理器）可以用。

1
下載 Node.js LTS 版本
前往 nodejs.org，點選左側的 LTS（長期支援版）下載。選擇 LTS 而非 Current，因為 LTS 更穩定。
⚠️
選 LTS（綠色按鈕），不要選 Current（灰色按鈕）。版本號只要是 18.x、20.x 或 22.x 開頭都可以。
2
安裝（一直點下一步即可）
下載完成後執行安裝程式，所有選項保持預設，一直點「Next」或「繼續」直到安裝完成。Mac 上可能會要求輸入系統密碼。
3
驗證安裝成功
開啟終端機，輸入以下兩個指令（分別按 Enter）：
TERMINAL
node --version
npm --version
✅
應該看到版本號，如 v22.x.x 和 10.x.x。如果出現「找不到命令」，重新開啟終端機後再試一次。
Step 2 · Editor

安裝 VS Code

VS Code 是你的程式碼編輯器。即使已經有其他編輯器，課程建議使用 VS Code，因為它有最多的 AI 輔助插件可以配合。

1
下載並安裝
前往 code.visualstudio.com 下載對應作業系統的版本，安裝後開啟。
2
安裝兩個必要插件
在 VS Code 左側工具列點擊「擴充功能」圖示（四個方塊），搜尋並安裝：
🔑
ESLint（wyl 出品）— 幫你抓 JavaScript 語法錯誤
Prettier（Prettier 出品）— 自動格式化程式碼
3
設定預設終端機
在 VS Code 中按 Ctrl+`（或 ⌃+`）可以開啟內建終端機，不用切換視窗。Mac 使用者建議在 VS Code 終端機裡設定使用 zsh。
Step 3 · First Project

建立第一個專案：Hello World

先用一個最簡單的範例確認環境正常。成功執行 Hello World 代表 Node.js 已正確安裝。

1
建立專案資料夾
在終端機執行以下指令（你可以把 my-ai-project 改成你想要的名稱）：
TERMINAL
mkdir my-ai-project
cd my-ai-project
2
建立並執行 hello.js
在 VS Code 裡開啟這個資料夾（File → Open Folder），新增一個檔案 hello.js，輸入以下內容後存檔：
hello.js
console.log('Hello, World!');
console.log('Node.js 版本：', process.version);
在終端機執行：
TERMINAL
node hello.js
✅
應該看到 Hello, World! 和 Node.js 版本： v22.x.x。恭喜，Node.js 執行正常！
Step 4 · API Key

建立 .env 檔案：保存 API 金鑰

.env 是一個儲存「環境變數」的特殊檔案，用來放 API 金鑰等敏感資訊。它的規則很簡單：絕對不能 push 到 GitHub（之後會設定 .gitignore 來防止這件事）。

⚠️
重要：API 金鑰一旦外洩，任何人都可以用你的帳號呼叫 API 並產生費用。請確認 .env 檔案永遠在 .gitignore 裡，且不要將金鑰貼到任何公開的地方（包括 Discord、LINE 群組）。
1
取得你的 Anthropic API 金鑰
前往 console.anthropic.com，登入後點「API Keys」→「Create Key」。複製金鑰（sk-ant-api03-...），它只會顯示一次。
2
在專案目錄建立 .env 檔案
在 VS Code 裡新增一個名為 .env 的檔案（注意：檔名以點開頭，沒有副檔名），輸入以下內容：
.env
ANTHROPIC_API_KEY=sk-ant-api03-你的金鑰
⚠️
等號兩邊不要有空格。把 sk-ant-api03-你的金鑰 替換成你從 console 複製的真實金鑰。
3
建立 .gitignore 防止金鑰外洩
在同一個目錄新增 .gitignore 檔案，輸入：
.gitignore
.env
node_modules/
Step 5 · Backend

跑起第一個後端：Express + Claude API

這是整個課程最關鍵的一步——建立一個能實際呼叫 Claude API 的本地後端服務。成功後，你的電腦就會有一個 API 代理伺服器在跑。

1
安裝必要套件
在終端機（確認你在專案目錄裡）執行：
TERMINAL
npm init -y
npm install express @anthropic-ai/sdk dotenv cors
✅
安裝完成後，專案目錄會多出一個 node_modules/ 資料夾和 package.json 檔案。
2
建立 server.js
新增 server.js 檔案，輸入以下最小可行後端代碼：
server.js · 最小後端代理
require('dotenv').config();
const express = require('express');
const cors = require('cors');
const Anthropic = require('@anthropic-ai/sdk');

const app = express();
app.use(cors());
app.use(express.json());
app.use(express.static('.'));   // 讓 HTML 工具頁也能從這裡開啟

const client = new Anthropic();   // 自動讀取 .env 的 ANTHROPIC_API_KEY

app.post('/api/chat', async (req, res) => {
  try {
    const { model, max_tokens, temperature, system, messages } = req.body;
    const response = await client.messages.create({
      model: model || 'claude-haiku-4-5-20251001',
      max_tokens: max_tokens || 300,
      temperature: temperature ?? 0.7,
      system: system || '',
      messages
    });
    res.json(response);
  } catch (err) {
    res.status(500).json({ error: err.message });
  }
});

app.listen(3000, () => console.log('Server running at http://localhost:3000'));
3
啟動後端，測試第一個真實 API 呼叫
TERMINAL
node server.js
✅
應該看到 Server running at http://localhost:3000。開啟瀏覽器進入 http://localhost:3000，如果看到你的 HTML 工具頁，後端代理就成功了。
💡
停止後端：在終端機按 Ctrl+C（Mac 也是 Ctrl+C，不是 ⌘+C）。每次重新開發前要記得重新執行 node server.js。
