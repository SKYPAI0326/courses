CH 3-3 · 2H

表單、自動化工具與 Webhook 基礎

自動化不是黑魔法，而是把你每次手動做的步驟告訴一個工具，讓它幫你重複執行。這節課拆解 Make 和 n8n 的核心邏輯，以及讓流程自動觸發的關鍵概念：Webhook。

學習成果 · OUTCOMES
理解觸發 → 處理 → 輸出的自動化三段結構
能說明 Webhook 的工作原理，並在實際流程中正確使用
能在 Make / n8n 中設計最小可行的 AI 自動化鏈
(01) · Form Data

Google 表單的資料結構：自動化的起點

Google 表單是自動化流程最常見的觸發起點——免費、熟悉、不需要寫程式。理解表單怎麼存資料，才能知道自動化工具怎麼接它。

結構 01
表單 → 試算表
每次提交在 Google Sheets 新增一列，欄位名稱就是表單的題目。這是自動化工具讀取資料的標準入口。
結構 02
時間戳記
每筆紀錄都有自動生成的提交時間，是觸發自動化的判斷依據——「新資料出現」就從這裡判斷。
結構 03
欄位格式
短答案 → 文字；選擇題 → 固定選項值（注意 AI 清理時的一致性）；評分 → 數字。格式決定後續處理方式。
結構 04
觸發時機
「收到新提交」是最常用的觸發條件；也可以用「試算表有新列」觸發，彈性更高。
◆
選擇自動化入口的原則：用你最常用、最熟悉的格式當輸入。Google Sheets 是絕大多數零代碼自動化的最佳起點。
(02) · Make and n8n

Make / n8n 的流程邏輯：三步驟就能理解

Make 和 n8n 這類工具的核心邏輯非常一致——無論介面怎麼設計，所有流程都可以用同一個框架理解。掌握這個框架，任何自動化工具都能快速上手。

觸發 · TRIGGER
什麼事件讓流程開始？
每條流程都從一個觸發事件開始，沒有觸發就不會執行。
Google 表單新提交 / 定時排程 / Webhook 收到資料 / Email 收到信
處理 · ACTION
要做什麼動作？
觸發後執行的一個或多個動作，可以串接、可以加條件判斷。
呼叫 AI API / 讀取試算表 / 發送 Email / 格式轉換 / 條件判斷
輸出 · OUTPUT
結果放哪？
處理完成後，把結果送到目的地或觸發下一個動作。
寫回試算表 / 發送 Slack / 寄 Email / 觸發另一條流程
Make.com 特色
適合初學者的視覺化工具
視覺化模組連接，直覺易懂
適合無程式基礎的初學者
有豐富的 app 連接器（1000+）
付費版功能較完整
n8n 特色
開源彈性高的自架工具
開源可自架，完全掌控資料
程式碼節點彈性高
可免費自架，無限執行次數
社群模板豐富
✦
初學者建議先用 Make.com，因為視覺化拖拉比較直觀。有 IT 基礎或想自架的人考慮 n8n。兩者的流程邏輯幾乎一樣，學會一個就很容易轉換。
迷你練習 1 約 10 分鐘
畫一條最小可行的 AI 自動化鏈草圖

在還沒打開 Make / n8n 之前，先用紙筆或白板把「觸發 → 處理 → 輸出」三段畫成流程圖，逼自己想清楚每個節點是什麼。

1從 CH3-2 挑出的工作流任務出發，畫三個方框代表觸發、處理、輸出。
2每個方框寫一行：具體工具（例：Google 表單、OpenAI、Google Sheets）＋ 動作（新提交、分類、寫入）。
3在箭頭上標「傳遞什麼資料」（例：原始回饋文字 → 分類結果 JSON）。
成果產出
一張三節點流程草圖（拍照或 Excalidraw 也可），節點清楚標示工具＋動作，箭頭標示資料型態。

(03) · Webhook

Webhook：讓外部事件直接通知你的流程

Webhook 是自動化工具裡最重要、也最容易讓人困惑的概念。先從一個類比開始理解它：

Webhook 就像快遞的「到貨簡訊」。你不需要一直打電話問「快遞到了嗎」——而是告訴快遞公司一個手機號碼（Webhook URL），包裹一到就自動傳訊息給你。Webhook URL 就是你的手機號碼，外部事件發生就是包裹到達。

概念 01
Webhook URL
Make / n8n 自動生成的唯一網址，是「接收資料」的入口。複製這個網址，貼到觸發來源。
概念 02
誰來發送
任何能發送 HTTP 請求的系統——表單、網站、另一個自動化工具、你自己的程式都可以。
概念 03
資料格式
通常是 JSON，包含事件的詳細資訊（觸發時間、資料內容等）。Make / n8n 會自動解析。
概念 04
觸發即執行
Webhook 收到資料的瞬間，流程就啟動，不需要排程。即時性是 Webhook 最大的優點。
ℹ
在 Make / n8n 設定的第一步，就是複製 Webhook URL 貼到觸發來源（如 Google 表單的回覆通知、付款系統等）。之後這條路就會自動通了。
迷你練習 2 約 15 分鐘
建立你的第一個 Webhook 並送出測試資料

實際在 Make 或 n8n 建立一個 Webhook 觸發節點，用瀏覽器或 curl 送一筆測試資料，親眼看到資料進流程的那一刻，就不再怕 Webhook。

1在 Make 建立新 Scenario（或 n8n 新 workflow），第一個節點選 Webhook / Custom Webhook，複製生成的 URL。
2按下「Run once / Listen for test event」讓它等待接收。
3把 Webhook URL 後面加 ?name=test&msg=hello 貼到瀏覽器網址列按 Enter（或用 curl 發 POST）。
4回到 Make / n8n，確認節點顯示收到資料，截圖保存欄位對應結構。
成果產出
一個可用的 Webhook URL + 一張收到測試資料的截圖 + 能說出資料欄位結構（name / msg 在 JSON 的哪個位置）。

(04) · Debugging

流程失敗時：三種常見原因與除錯方式

自動化流程第一次不成功很正常。好消息是失敗的模式通常就那幾種，學會識別就能快速修復。不要一看到失敗就全部重來——先定位問題在哪一段。

1
觸發沒有收到
Webhook URL 貼錯、表單設定連結錯誤 → 在 Make / n8n 用「手動觸發」或「Send Test」確認是否收到資料。先確認資料有進來，再看後面的步驟。
2
AI 節點失敗
API 金鑰錯誤、請求格式不對、回應解析失敗 → 先在 API 測試工具確認呼叫成功，再回來接流程。不要在流程裡 debug API，太費時。
3
輸出寫入失敗
試算表欄位名稱不一致、權限未開放 → 確認試算表已授權工具存取，欄位對應用「動態欄位」而非手動輸入名稱。
✦
除錯最重要的心態：一次只改一個地方，確認後再改下一個。同時改多個地方不知道是哪個修好了，下次遇到同樣問題還是不會。
