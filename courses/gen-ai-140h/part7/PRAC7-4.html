PRAC 7.4 · 1 小時

部署與驗收

把 v1 部署到公開網址，完成 10 筆測試案例，更新作品集。這一小時結束，你有了一個真實存在於網路上的 AI 應用。

任務目標
部署到 Vercel / Netlify / Render，取得公開 URL，執行驗收測試並記錄通過率，更新作品集展示頁。
1
部署（20 分鐘）
靜態頁面：連接 GitHub repo 到 Vercel 或 Netlify，設定環境變數，點部署。
含後端：連接到 Render 或 Railway，設定環境變數，觀察 build log 確認成功。
最常見的部署失敗原因：環境變數名稱打錯（記得是 ANTHROPIC_API_KEY，不是 API_KEY 或其他變形）；或 build command 設定錯誤。
2
驗收測試（25 分鐘）
在部署後的正式網址上執行測試，不是 localhost。用下方 Checklist 完成 10 筆驗收，記錄通過率。
3
更新作品集（15 分鐘）
打開 PRAC5-3 建的作品展示首頁，加入這個專題：名稱 + 一句話說明 + 公開網址 + GitHub 連結。
部署 + 驗收 Checklist
完成進度：0 / 15
部署準備
部署確認

作品集更新

延伸案例

部署與驗收不只是「推上 Vercel 就好」，還要應付真實用戶的誤用、安全攻擊、交付文件。挑一個貼近你處境的情境完整做一次。每個案例 30–45 分鐘。

案例 01 · 部署前驗收
Vercel 上線前的 12 項驗收清單（含限流與 API key 保護）
難度 ★★☆ 約 40 分鐘

你幫一家 20 人的茶飲加盟總部做了「新店選址評分 AI」，明天要上線給加盟顧問內部用。你很擔心：API key 外洩會被盜刷、沒限流有人連按 50 下、URL 外流到 PTT 被亂玩。上線前要一份 12 項驗收清單，老闆會陪你一起跑一次。

原始資料｜專案技術現況
前端：靜態 HTML 單頁（呼叫 Claude API） 部署：Vercel 免費方案 網域：score.xxxxxx-tea.com（已買好） 使用者：8 位加盟顧問（無需登入，用 iPad） 使用頻率：平均每人每天 5–10 次 API 預算：每月 2,500 元 上限（老闆底線） 既有 issue： - API key 目前還寫在前端 JS - 沒有設任何 rate limit - console 會印出 debug log - 沒有 404 頁面
你的任務
  1. 寫 12 項驗收清單，分 4 類：安全（4）、效能（3）、UX（3）、監控（2）
  2. 每項寫「怎麼驗」——不是「確認 X」，是「打開 Y 看 Z 是否 = W」
  3. 決定 1 項可以「上線後 24h 內補」，9 項必須上線前過，2 項可跳過並解釋
評估指標
API key 不外洩
明確寫「打開 DevTools → Sources → 搜 'sk-' 應找不到」
限流可驗
寫「連按 20 次應看到 429 或友善提示」，不是「有設限流」
成本可控
至少 1 項驗「如果有人濫用會被擋下」，關聯 2500 元預算
取捨合理
跳過的 2 項有充分理由，不是「時間不夠」
參考驗收清單（建議先自己列再看）
點擊展開參考答案
【安全 · 必過】 1. [必過] 打開網站 → F12 → Sources → Ctrl+F 搜 "sk-ant"、"CLAUDE_KEY" → 應 0 個結果 2. [必過] 查 Vercel 環境變數：ANTHROPIC_API_KEY 設為 Secret，不對外 expose 3. [必過] API route 檢查 Referer header，非自家網域的請求回 403 4. [24h 內補] CORS 只允許 score.xxxxxx-tea.com（上線先全開，24h 內收緊） 【效能 · 必過】 5. [必過] 首次載入 <2 秒（用 lighthouse 量） 6. [必過] 連按 20 次「評分」按鈕 → 第 11 次起回應「請稍候 10 秒」（前端限流） 7. [跳過] PWA 離線可用——加盟顧問都有網路，先不做 【UX · 必過】 8. [必過] 載入中有 loading 動畫（不白屏） 9. [必過] API 失敗顯示「網路不穩請重試」，不噴 JSON 錯誤 10. [跳過] Dark mode——iPad 內部用不需要 【監控】 11. [必過] Vercel Analytics 開啟，可看每日呼叫量 12. [必過] Anthropic console 設 API 月預算警示 2,000 元（80%）+ 2,500 元 hard cap 【跳過理由】 #7 PWA 離線：iPad 永遠有網路，成本不划算 #10 Dark mode：內部工具、使用時間都是白天 【24h 內補理由】 #4 CORS：先全開方便第一天 debug，確認穩定後再收緊，風險可接受（#3 referer 已擋住大部分盜用）
變體挑戰
A老闆說「我想讓總部外的加盟主也能用」——重寫安全那 4 條，加上登入機制的最小版
B預算從 2500 砍到 800——重排優先級，哪些要 cut
C上線 3 天後發現某天燒掉 1,200 元（正常日 50 元）——寫你的排查 5 步
自我檢核
案例 02 · 客戶交付 SOP
交付給非技術客戶的驗收 SOP + 3 分鐘教學錄影逐字稿
難度 ★★☆ 約 40 分鐘

你是自由接案者，幫一位牙醫診所老闆做了「AI 預約確認簡訊生成器」，費用 2.5 萬元已收訂金 1 萬。這週要交付，但老闆 55 歲不懂 HTML、看到 code 就頭痛。你要一份 SOP 讓老闆驗收、加一支 3 分鐘錄影讓她能教給櫃檯小姐。

原始資料｜交付情境
專案：預約確認簡訊生成器 使用者：牙醫診所櫃檯（1 位主櫃 + 1 位兼職） 使用流程：輸入病患姓名、約診時間、治療項目 → 生成中文簡訊（含醒酸注意、遲到規則） 網址：sms.xxxxxx-dental.com（已設好） 帳號：無需登入 已付：訂金 1 萬，驗收通過付尾款 1.5 萬 老闆的擔心：「我是不是要每年付你維護費？」「會不會被駭？」 老闆會 LINE 你但不會 GitHub 交付需要： - 驗收書（老闆簽字用） - 使用手冊（1 頁給櫃檯看） - 3 分鐘錄影（老闆能轉給新進櫃檯）
你的任務
  1. 寫驗收 SOP（老闆簽字用）：6 項可勾選的驗收條件，每項 20 字內
  2. 寫 3 分鐘教學錄影逐字稿（約 360 字）：開場 → 實操 → 常見問題
  3. 寫給老闆的 FAQ 3 題：維護費、安全、找人改的方法
評估指標
不用技術詞
不出現 API、HTML、部署、deploy、cache 等詞
可勾選驗收
6 項都是「做了 X → 看到 Y」，老闆可自己驗，不用你在旁邊
錄影可朗讀
逐字稿實際念出來 180–200 秒，留 30 秒緩衝
FAQ 誠實
維護費說清楚「哪些免費、哪些另計」，不含糊
參考交付文件（建議先自己寫再看）
點擊展開參考答案
【驗收書 · 6 項】 □ 1. 打開 sms.xxxxxx-dental.com，不到 3 秒看到輸入表單 □ 2. 填一則真實預約（王小美／週五下午 3:00／根管治療）→ 按生成 → 10 秒內看到中文簡訊 □ 3. 生成的簡訊複製到 LINE 貼給自己 → 中文不亂碼、標點正常 □ 4. 連續生成 3 則不同病患資料 → 內容不會互相混淆 □ 5. 用櫃檯電腦 Chrome 實測 → 操作流暢 □ 6. 老闆自己操作 1 次，不用看手冊也能完成 甲方簽章：_______ 乙方簽章：_______ 日期：2026/___/___ 【3 分鐘錄影逐字稿（約 360 字）】 （0:00–0:30 開場） 各位好，我是 [設計師姓名]，接下來花三分鐘帶你用這個工具。打開網址 sms.xxxxxx-dental.com，出現這個頁面就對了。 （0:30–2:00 實操） 假設等下要發簡訊給王小美，週五下午 3 點來根管治療。姓名欄填「王小美」，時間欄選「週五 15:00」，治療項目下拉選「根管治療」。按「生成簡訊」。等 10 秒。出來一段 80 字的中文，已經含了我們診所慣用的提醒語，像是治療前兩小時不要吃東西、遲到超過 15 分鐘要改約。你直接點「複製」，貼到 LINE 或簡訊 App 就能發。 （2:00–3:00 常見問題） 三個可能遇到的狀況。第一，偶爾網路慢要等 15 秒，不是壞掉。第二，簡訊內容你覺得不適合，可以直接在生成出來的欄位改，改完再複製。第三，治療項目下拉沒有的，選「其他」然後在下方備註欄寫上去。有任何狀況 LINE 我，24 小時內回。 【FAQ · 3 題】 Q1：每年要付維護費嗎？ A：不用。系統放在雲端免費方案，每月 AI 成本大約 300–500 元由你直接付給服務商（已教你登入查帳）。我只在你想加新功能時再報價。 Q2：會不會被駭客攻擊？ A：目前無登入設計，任何知道網址的人都能用，但不會看到別人的紀錄（系統不存資料）。網址建議只給櫃檯知道，不要發到 FB。若你想改成要登入才能用，另報價 1.2 萬。 Q3：如果以後想找別人改，我要給他什麼？ A：我會把 GitHub 專案權限轉給你或你指定的工程師。程式碼是你的，不綁我。這封交付信附上操作說明。
變體挑戰
A老闆驗收第 3 條不過（LINE 貼上有亂碼）——寫你的 3 步診斷給老闆看得懂
B改成交付給「有 IT 廠商」的客戶，技術詞可以用，驗收書要加 3 條技術驗證
C寫一封交付 Email（<300 字）附上上述文件清單
