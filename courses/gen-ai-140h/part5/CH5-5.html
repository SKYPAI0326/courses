CH 5-5 · 1.5h 課堂 / 1h 自學

AI Canvas 純拖拉部署 SOP

5 分鐘把任何 AI Canvas 產出的單檔 HTML 變成一個可分享的 https:// 網址。全程不開 Terminal、不裝任何軟體、不背任何 git 指令。

學習成果 · OUTCOMES
能在 5 分鐘內把 AI Canvas 產出的單檔 HTML 部署成 GitHub Pages 連結
能判斷 AI 給的程式碼是否為「可直接部署的單檔」
能獨立排除 6 種最常見的部署卡關（404、中文 repo 名、忘記 commit 等）
能用「除錯回貼三段式」在 30 秒內讓助教或 AI 診斷你的問題
(01)

問題不是你的 HTML 不好，是你沒有網址

本節解決什麼
行政小陳上週跟 Gemini 聊了一小時做出漂亮的個人介紹頁，但朋友打不開。不能用 LINE 傳、不能用 Email 寄、不能截圖（截圖不能點按鈕）。她卡了兩小時，最後放棄。這 1.5 小時要解決的，就是「把 AI 給你的程式碼變成一個能分享的 https:// 網址」。
你不需要懂 GitHub，你只需要會點按鈕。

只要學會這 5 步，你以後做的所有 AI 工具、遊戲、單頁網站，都能丟出去給朋友、客戶、面試官看。全程不打開 Terminal、不裝任何軟體、不背任何 git 指令。

這節課之後，CH5-6、PRAC5-13、PRAC5-14、PRAC5-15 全部都會回頭叫你「請執行 CH5-5 五步」。這 5 步務必記熟。

(02)

開始前先問自己：這是單檔嗎？

開始之前只要回答一個問題：AI 給你的程式碼，是不是「一個 index.html 檔，裡面什麼都包了」？

你在程式碼裡看到的 判定
所有 CSS 寫在 <style>...</style> 內 是單檔
所有 JavaScript 寫在 <script>...</script> 內 是單檔
圖片是 https://... 開頭的網址（公開 CDN） 是單檔
程式碼裡有 <script src="./script.js"> 不是 — 有本地相對路徑
程式碼裡有 <link href="./style.css"> 不是 — 有本地相對路徑
程式碼裡有 <img src="photo.jpg"> 但 AI 沒給你 photo.jpg 不是（或修一下）
→
是單檔 → 直接走本單元的 5 步部署。
不是（有本地相對路徑） → 先去 CH5-6 學「請 AI 幫我整合成單檔」提示詞，再回來走本單元。

灰色地帶：看到 <script src="https://cdn.jsdelivr.net/npm/chart.js"></script> 這種公開 CDN 連結，算單檔，可走本 SOP。CDN 是網路上的公開檔案，瀏覽器自己會去拿。

簡單判斷：src= 或 href= 後面是 https:// 開頭 = OK；是 ./ 或 photo.jpg = 不 OK。

(03)

4 種 AI 工具的複製按鈕在哪

不同 AI 工具產出程式碼的「複製按鈕」位置不同。記不住沒關係，這頁是隨時翻的查表。

AI 工具 找哪個按鈕 注意事項
Claude Artifacts Artifact 視窗右上「Copy」（兩個方塊重疊的圖示） 預設就單檔，最順
ChatGPT Canvas Canvas 右上「下載」（向下箭頭）→ 存成 .html 檔；或全選 Cmd+A → Cmd+C 預設單檔
Gemini Canvas Canvas 右上「Copy code」（兩個方塊圖示） 常常多檔，先跑 CH5-6
Google AI Studio Code 區塊右上的複製圖示 常常多檔，先跑 CH5-6
⚠
Gemini 與 AI Studio 是多檔重災區。Canvas 視窗看起來只顯示一個主檔案，但內部還有 style.css、script.js，甚至 assets/ 資料夾。Phase 1 先學單檔場景（Claude Artifacts、ChatGPT Canvas），熟練後再回 CH5-6 解多檔。
(04)

5 步部署動線：核心示範

這段是本單元的核心。講師現場示範一次、學員現場跟做一次。

前置條件（開課前一週應已完成）
學員已註冊 GitHub 帳號、已完成 2FA 設定（可裝 Google Authenticator 掃 QR Code）、已在課堂確認網路能通 GitHub Pages（部分學校網路會擋）。

準備：從 Claude Artifacts 隨便產一個單檔 HTML（例如「請做一個顯示時鐘的單檔 HTML 頁」），複製整段程式碼放剪貼簿備用。

講師示範 · 5 步從零到公開網址
步驟 1 — 開新 repo（1 分鐘）
動作：開 github.com → 右上「+」→ New repository
填寫表單
Repository name：my-clock（全英文小寫、用連字號連單字、不要中文） Description：可空白 Public：必須勾選（免費版 GitHub Pages 只支援 Public repo） Add a README file：必須勾選（不然 repo 是空的進不去） 其他不用動 → 點綠色按鈕 Create repository
畫面跳到：新 repo 頁面，左上會有「你的帳號/my-clock」。
步驟 2 — 按鍵盤的「.」鍵進 github.dev（30 秒）
動作：在 repo 頁面任何位置按一下鍵盤上的 「.」（句點鍵）
畫面跳到：跟 VS Code 長得一模一樣的編輯器，但在瀏覽器裡。網址從 github.com/... 變成 github.dev/...。

按「.」沒反應？可能還沒從 GitHub 登入。重新整理頁面、確認右上頭像顯示你的帳號名，再試一次。
步驟 3 — 新建 index.html 貼上程式碼（2 分鐘）
動作：左側檔案區點「+」（新檔案）→ 命名 index.html（全小寫）→ Enter
重要：檔名一定是 index.html，不是 Index.html、不是 INDEX.html、不是 首頁.html。檔名錯一個字母 Pages 就 404。
貼入程式碼
在右側編輯區點一下游標進去 → Cmd+V（Mac）/ Ctrl+V（Windows） 貼上從 AI 複製的程式碼
畫面會看到

整段 HTML 出現在編輯器。

左側檔案列表的 index.html 旁邊出現一個藍色 U（代表 Untracked，還沒存進 git）。

如果貼上後什麼都沒出現，確認剪貼簿裡存的是文字（程式碼），不是截圖或圖片檔。
步驟 4 — Commit + Sync（2 分鐘）
動作：左側最左邊的圖示列 → 找「像三個圈圈連著線的圖示」（Source Control）→ 點它
在 Source Control 面板操作
在「Changes」清單找到 index.html → 滑鼠移上去出現「+」→ 點下去（stage） → 上方輸入框打 first deploy → 點藍色按鈕 Commit & Push（或 Mac 上按 Cmd+Enter）
畫面變化：左下角出現小轉圈圈，然後變成「Sync successful」。代表程式碼已傳到 GitHub 主站。
步驟 5 — 啟用 GitHub Pages（1 分鐘 + 等 1~3 分鐘）
動作：回到 GitHub 主站（重新整理 github.com 那個分頁）→ 進 repo 頁面 → 點上方 Settings（齒輪圖示）
Settings 內操作
左側選單捲到 Pages（在 Code and automation 區段下面）→ 點它 右側 Pages 設定畫面： 1. Build and deployment 的 Source 下拉選 Deploy from a branch 2. Branch 下拉選 main → 旁邊資料夾選 / (root) → 點 Save
畫面變化：上方出現「Your site is being built...」或「Your site is live at https://你的帳號.github.io/my-clock/」。

等 1~3 分鐘。第一次部署 GitHub 要建環境。喝口水。
最終驗收

把 https://你的帳號.github.io/my-clock/ 複製貼到新分頁打開，應該看到你的時鐘。

你剛剛沒有開 Terminal、沒打任何 git 指令、沒裝任何軟體、沒理解 git 是什麼。

如果等超過 3 分鐘還是 404，用 Cmd+Shift+R 強制重新整理（繞過快取）再試一次。
這 5 步完成後你會有：一個 GitHub repo + 一個公開網址。朋友、客戶、面試官、隨便誰打開都能看到你的東西。
(05)

卡關地圖：6 個必教的常見錯誤

學員會在這裡卡住，先講過一次省得到時候助教救火。

錯誤 01
檔名打成 index.HTML（大寫）→ Pages 404
症狀：Pages 啟用後打開網址，看到「404 - There isn't a GitHub Pages site here.」
原因：GitHub Pages 只認識小寫 index.html。
修法：進 github.dev → 對檔名按右鍵 → Rename → 改成全小寫 → commit & push → 等 1 分鐘。
錯誤 02
repo 名用中文 → URL 亂碼、Pages 失敗
症狀：想用「我的時鐘」當 repo 名 → GitHub 顯示沒問題，但網址變成 %E6%88%91%E7%9A%84...。
修法：Settings → 拉到最下面 → Delete this repository → 重做一個英文小寫的。
記住：repo 名一律全英文小寫、用連字號（my-clock 不是 MyClock）。
錯誤 03
Pages 啟用後沒等 1~3 分鐘 → 看到 404 以為壞了
症狀：步驟 5 做完馬上打開 URL，看到 404。
原因：GitHub 第一次部署要建環境，沒有錯。
修法：喝口水。3 分鐘後用 Cmd+Shift+R 強制重新整理（繞過快取）。
錯誤 04
改完忘了 commit → 看到舊版
症狀：在 github.dev 改了程式碼，但打開 Pages 網址還是舊內容。
原因：步驟 4 的 commit & push 沒做。改完一定要 commit。
修法：回 github.dev → Source Control → 看「Changes」有沒有東西 → 有就 stage + commit & push。
錯誤 05
直接把 Canvas 截圖傳上去 → 不是 HTML 是圖片
症狀：學員把 Claude / Gemini Canvas 視窗截圖、存成 index.html 想上傳。
原因：截圖是圖片，不是程式碼。GitHub Pages 看到 index.html 裡只有圖片二進位資料，跑不起來。
修法：回 AI 工具找「Copy」或「下載」按鈕，複製的是文字（程式碼），不是截圖。
錯誤 06
AI 產出含 <script src="./local.js"> → 找不到檔案
症狀：部署上去，網頁打開沒互動；開瀏覽器開發者工具看到「找不到 local.js」紅字。
原因：AI 給的不是單檔，是多檔。./local.js 是本地檔案，但你只傳了 index.html 沒傳 local.js。

這是「不是單檔」的訊號 → 回 AI 工具丟這句提示詞：
丟給 AI 的整合提示詞
請把這個專案重新整理成單一 HTML 檔的版本：所有 CSS 寫進 <style> 標籤、所有 JavaScript 寫進 <script> 標籤、圖片用 https://placehold.co/ 連結代替。我要複製整段就能存成 index.html 直接用的版本。

拿到單檔後回步驟 3 重貼。

(06)

錯了就重做：5 分鐘復活術

新手最大的敵人不是錯誤本身，是「卡住超過 5 分鐘還在試」。鐵律：任何卡關超過 5 分鐘 → 你不是 debug，是重做。

怎麼重做（3 步）
1
進 repo 頁面 → Settings → 拉到最下面紅色區塊 Danger Zone
2
點 Delete this repository → 照指示輸入 repo 名確認 → Delete
3
repo 沒了。回步驟 1 重做一遍，5 分鐘搞定。
為什麼這樣最快：GitHub 免費 repo 數量沒上限、做第二次只要 5 分鐘、這個 repo 你只有 1 個檔案 5 分鐘工作量。砍掉重練比修慢炸還快——這是工程智慧，不是放棄。
(07)

除錯回貼三段式：讓 AI 30 秒內診斷你的問題

當你真的不知道怎麼辦——15 分鐘卡住——你會去找助教，或把錯誤丟回 AI 問。問題是：學員常常只貼一句「它壞了」「不能用」「怪怪的」——沒人能診斷。

除錯回貼三段式格式
我做的事
剛剛點了什麼、貼了什麼、做了哪一步
我看到的
螢幕上實際出現的訊息 / 截圖 / 錯誤碼
我期待的
原本以為應該會發生什麼

範例（直接套用）：

貼給助教或 AI 的訊息
【我做的事】 我在 github.dev 把 index.html 貼好程式碼，點 Source Control 的 Commit & Push，等了 5 分鐘。 【我看到的】 Pages 設定頁顯示「Your site is live at https://chen.github.io/my-clock/」，但我點進去看到「404 - There isn't a GitHub Pages site here.」 【我期待的】 看到我的時鐘畫面。 請問我下一步該做什麼？
助教 / AI 大概會回

診斷：應該是快取問題或 Pages 還沒建好。

下一步：1. 用 Cmd+Shift+R 強制重新整理（繞過快取）。2. 如果還是 404，去 repo 的 Actions 分頁看部署狀態——是否有黃色轉圈圈（還在跑）或紅色叉叉（失敗）。3. 如果是紅叉，回步驟 4 確認有 commit & push 成功。

實際回覆每次略有差異，重點是診斷速度——三段式貼過去，助教或 AI 在 30 秒內就能給你下一步。

把這個三段式存成手機備忘錄，卡關時直接套。
