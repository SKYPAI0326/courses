簡單

文字敏感詞過濾器

貼文發出前的最後防線。設定你的敏感詞清單，把文章貼進來，工具自動標出所有問題詞彙並統計次數。

PRACTICE GOALS
練習字串搜尋與 HTML 高亮標記的實作
理解如何用正則表達式做全域取代
產出一個公關或社群管理人員可用的文字審核工具
TOOL · 互動工具

輸入敏感詞清單，貼入文章，一鍵掃描高亮

在左側輸入要偵測的關鍵字（每行一個），右側貼入需要審查的文字內容，按下「開始掃描」後，所有符合的詞彙會以半透明暖黃色高亮標示，並在下方統計每個關鍵字出現的次數。

💡
掃描不區分大小寫。建議先用常見的廣告違規用語測試，例如「保證」「最低價」「免費領取」，確認工具正常運作後，再換成你自己工作場景的關鍵字清單。

1
打開 Gemini，切換到「Canvas」模式
進入 gemini.google.com，點右上角切換到 Canvas 模式，這樣 AI 生成的 HTML 會直接可以預覽。
2
貼入以下指令
把下方指令完整複製，貼入 Gemini 的輸入框，直接送出。
3
等待生成，預覽並存檔
Gemini 會生成一個完整的 HTML 工具。點「預覽」確認功能正常後，點「下載」或「複製」儲存到你的電腦。
→ 複製這段 Prompt 貼入 Gemini
你是一位擅長製作文字工具的前端工程師。 請製作一個「文字敏感詞過濾器」的單頁 HTML 工具，功能如下： 【功能需求】 - 左側：敏感詞清單輸入框（每行一個關鍵字），顯示總關鍵字數 - 右側：待審查文字輸入框 + 「開始掃描」按鈕 - 掃描後在下方顯示原文，所有符合的關鍵字以螢光標記色 highlight - 統計每個關鍵字出現的次數，以標籤形式顯示在結果下方 - 若無符合項目，顯示綠色通過提示 【設計要求】 - 背景 #f5f3ee，強調色 #b5703a，highlight 用半透明暖黃（rgba(201,150,58,.3)） - 字體 Google Fonts（Shippori Mincho + Noto Sans TC） - 左右分欄佈局，圓角 6px，無陰影 請生成完整的單一 HTML 檔案。
CHALLENGE · 進階挑戰

完成基本練習後，試試這些變化

PRACTICE VARIATIONS
01
加入「嚴重程度分級」：讓使用者為每個關鍵字標記「高風險（紅）/ 中等（黃）/ 低風險（綠）」，highlight 顏色對應不同層級。
02
加入「一鍵替換」功能：點擊某個關鍵字標籤後，提示輸入替代詞，自動把文章中的敏感詞換掉。
03
用 Gemini 幫這個工具加上「產業關鍵字包」——讓使用者可以一鍵載入「金融業禁用詞」「醫療廣告法規詞」等預設清單。

延伸案例

過濾器工具本身可以掃敏感詞，但真實情境下最難的是「這個詞要不要過」。帶一份真實社群貼文來練，並寫下判斷理由。

案例 10 · 品牌小編
金融科技品牌的 IG 小編，每篇貼文發出前都要掃合規
難度 ★★☆ 約 35 分鐘

你是某信用卡比較平台的 IG 小編，公司因為是金融科技類，法規嚴格。法遵說過「保證」「穩賺」「零風險」「最高回饋」這類詞絕對不能出現，否則 NCC 檢舉會罰很重。你寫完新一篇貼文後要用敏感詞工具掃一次，但真正難的是「踩線的詞」要怎麼處理。

原始資料｜今天要發的 IG 貼文草稿
📣 今天要發的貼文草稿： 想選一張最適合自己的信用卡嗎？ 我們整理了 2026 年最高回饋的三張卡，幫你穩穩賺回日常消費： 💳 A 卡：海外消費 5% 回饋，絕對不虧 💳 B 卡：加油回饋保證 8%，適合開車族 💳 C 卡：超市消費零風險 3% 現金回饋 點擊 bio 連結查看完整比較表，讓每一筆消費都回本！ #信用卡 #理財 #回饋王 #2026最優選
你的任務
  1. 掃出所有違規詞（法遵紅線清單 + 你自己判斷的灰色地帶）
  2. 逐詞決定：直接砍、改寫、或保留但加小字免責
  3. 改出 v2 乾淨版貼文（仍然要吸引人，不是變成法律文件）
評估指標
紅線不留
「保證」「穩賺」「零風險」「絕對不虧」全砍或改寫
灰色處理
「最高回饋」「讓每一筆消費都回本」這類誘導語，有判斷理由
可讀性
v2 版讀起來還是人話，不是一串免責聲明
可追溯
每個修改都寫一句原因，法遵抽查時說得出來
參考 Prompt（建議先自己寫再看）
點擊展開參考答案
【角色】你是金融科技品牌 IG 小編，既要做有人看的內容，又要通過法遵審核，常在「禁用詞」「灰色誘導」之間拿捏。 【任務】針對下方貼文草稿，產出： 1. 逐詞列表：抓出的違規/灰色詞、分類（紅線 / 灰色 / 保留）、處理方式 2. 每個修改的一句理由（法規面 + 讀起來的影響） 3. v2 改寫版貼文（仍然有情緒、有賣點，但合規） 【限制】 - 紅線詞（保證/穩賺/零風險/絕對不虧）一律砍或改寫 - 「最高回饋」若有佐證可改成「本月回饋率最高的三張」，不保證未來 - emoji 可以留，但「#2026最優選」這類主觀排名標籤要拿掉 - v2 字數可以多，但不要超過原版 1.3 倍 【輸出格式】 分三段：違規詞表、修改理由、v2 貼文。
變體挑戰
A法遵加一條新規：不能用「你」（不是錯字，法規實際如此）。檢查 v2 並改掉
B改成保險商品貼文（「定存領息」「免體檢」），重新設一份 10 個紅線詞清單
C用 JavaScript 寫一個簡化版的過濾函式，能把紅線詞自動替換成 [已移除] 並回報替換次數
自我檢核
RUBRIC · 自評檢核（4 點全勾才算過關）
SOLO · 自我演練
把敏感詞清單打開放在手邊；開新對話請 LLM 扮「想偷渡訊息的不友善使用者」，要它一次給你 ≥10 種繞過寫法（同音字、空格切割、emoji 替代、火星文、圖文混合、英譯回譯⋯⋯）；你逐一測進過濾器，把成功繞過的字加回清單，做出 v2。注意：這只是文字過濾演練，不要拿真實惡意內容生成大量素材。
12 分鐘（演練 8 分 + v2 詞庫 4 分）
進階選配 · 不計入課程時數（給想加練的學員）
把 LLM 對話截圖存到學習資料夾，或貼到工作群組／社群求 review。這些痕跡在轉職投履歷時可作為作品集素材。
