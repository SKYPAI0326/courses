正式練習

一鍵生個人單頁網站演練（⑦正式化）

複製一張提示詞、貼上你的資料、走 CH5-5 五步——60 分鐘後你有一個可以丟給面試官的個人介紹網址。

本單元 vs PRAC5-1 的差異
PRAC5-1 練的是
需求迭代：多輪對話打磨需求、調整版面、精修細節，是慢工出細活。
PRAC5-13 練的是
部署閉環：一次提示詞 → AI 一次產出 → 部署成可分享連結，最短路徑、最快有成品。
你會練到的能力 · OUTCOMES
用⑦提示詞一次讓 AI 產出單檔 HTML 個人介紹頁
照 CH5-5 五步把 HTML 部署到 GitHub Pages 取得公開網址
看懂 4 種常見卡關情境並獨立修復
(01) 破題

行政小陳的 6 分鐘個人網站

行政小陳要轉職找新工作了。下週要面試一家數位行銷公司。

履歷投出去後，她想到一件事：「面試官 Google 我的名字，會看到什麼？」

答案是：LinkedIn 半成品、十年前的部落格、噗浪上的廢文、一張高中畢業照。

她想做一個「個人介紹單頁」放最新的自己——但她不會寫程式碼。

平常做 PowerPoint 沒問題，但 PowerPoint 不能變網頁；用免費 Notion 模板可以，但那不像「自己做的」；用 Wix / Webflow 又要學一套新介面。

她在課堂上學過一個提示詞——⑦ 一鍵生個人單頁網站。

3 分鐘把資料貼給 Gemini，3 分鐘走 CH5-5 五步上 GitHub Pages，6 分鐘她有了 https://chen-hr.github.io/about-me/。

面試官打開來，看到一頁簡潔現代、米白底、手機打開也漂亮的個人介紹。她錄取了。

💡
這 1 小時，我們把⑦這張提示詞卡做成你的第一個「60 分鐘交付成品」。你會帶著一個 https://{你的帳號}.github.io/{repo} 的網址離開教室。
(02) 提示詞卡

⑦ 一鍵生個人單頁網站——原版提示詞

這張卡來自 fun-apps.html ⑦，是課堂彩蛋中學員反應最好的一張。今天我們不只貼提示詞，還要走完整部署流程。

貼給 ChatGPT / Claude / Gemini Canvas ★★★★
⑦ 一鍵生個人單頁網站
請用單一 HTML 檔做一份「個人介紹單頁」，包含：姓名與一句話自我定位、關於我、3 項技能、3 個作品占位區、聯絡方式。風格簡潔現代、米白底、手機可讀。我先貼上我的資料你再產生程式碼，請先回我「請貼上你的資料」。
今天的演練 AI：Gemini Canvas（最常用）、Claude Artifacts（最穩）、ChatGPT Canvas 都可以，三選一即可。

為什麼這張提示詞會贏？

①
限定單檔 HTML：直接走 CH5-5 主路徑，不會卡多檔場景，一個 index.html 就搞定。
②
「請先回我『請貼上你的資料』」：AI 變成填空主導者，你只負責貼資料。AI 不會先亂猜你是誰、幫你編故事。
③
米白底、手機可讀：預設好看 + 預設手機友善，不用學 RWD，不用懂 CSS media query。

(04) 部署

走 CH5-5 五步——把 HTML 變成可分享連結

🔁
這部分回頭走 CH5-5 五步，不重複教，只給你今天的對應動作。沒跑過 CH5-5？先回去看 CH5-5 教案再來。
Step 今天的對應動作
1 開 github.com → New repository → 命名 about-me（或你的英文名如 chen-yj）→ Public + 勾 README → Create
2 在 repo 頁面按 . 鍵進 github.dev（線上編輯器，不用裝軟體）
3 左側新建 index.html → 把 Gemini Canvas 給的整段 HTML 貼進去（全選全貼，不要只貼部分）
4 Source Control → stage → commit message 寫 init → Commit & Push
5 回 repo → Settings → Pages → Branch 選 main → Save → 等 1–3 分鐘

最後拿到 https://{你的帳號}.github.io/about-me/。這個網址在世界任何地方都可以打開。

(05) 卡關地圖

4 個最常見的卡關情境與修法

卡關 01
AI 把照片占位區寫成 <img src="photo.jpg"> 但你沒上傳照片
症狀：部署後打開網頁，照片區是一個破損的「X」圖示。
原因：AI 隨手寫的 photo.jpg 是假檔名，你只傳了 index.html，沒傳 photo.jpg。
兩個修法：
選擇 A · 用免費網路圖片代替
回 Gemini 說：「請把所有本地圖片連結改成 https://placehold.co/400x400」——Gemini 會把所有 <img src="photo.jpg"> 改成 placeholder 圖，重貼 → commit → 推上去。
選擇 B · 把圖片區改成純文字框
回 Gemini 說：「請把 3 個作品占位區的圖片改成純文字框（深灰底、白字），不要用圖片。」
卡關 02
Google Fonts 字型部署後變 Times New Roman
症狀：Canvas 預覽很美，部署後字型變預設。
原因：<link href="https://fonts.googleapis.com/..."> 被 AI 漏寫進 head。
修法：
回 Gemini 說：「請把字型 link 加進 <head> 區、確認所有裝置都能載入。」重貼 → commit → 部署。
卡關 03
手機打開字太大、跑版
症狀：桌機看正常，手機看每個字都頂到邊。
原因：AI 寫的 RWD 沒做透。
修法：
回 Gemini 說：「請優化手機版面：標題不超過 2 行、padding 縮小到 16px、字級在手機降到 14px。」重貼 → commit → 部署。
卡關 04
改完忘了重新整理（卡關 5 分鐘的 #1 元兇）
症狀：在 github.dev 改完、commit 完，打開 Pages 網址還是舊版。
原因：瀏覽器有快取，看到的是舊內容。
修法：
Cmd+Shift+R（Mac）/ Ctrl+Shift+R（Windows）強制重新整理。等 1–3 分鐘讓 GitHub Pages 部署完再刷。

(08) 自學包 · 1 小時

變化版練習——三選一或全做

同一份個人資料、開三個 Gemini Canvas 分頁、各跑一個變化版、各部署一個 repo。

夜間風 ★★★
變化版 A — 深色底版
請把風格改成夜間風：深灰底（#1a1a1a）、淺色文字、霓虹紫色（#a855f7）強調色、字型用襯線 + 等寬字混搭。
部署 repo 名稱建議：about-me-night
印刷風 ★★★
變化版 B — 雜誌封面感
請把風格改成印刷雜誌風：米黃紙質底（#f4e9d0）、襯線標題（Playfair Display）、黑字、加細格線分隔、像 The Economist 雜誌封面排版。
部署 repo 名稱建議：about-me-print
賽博龐克 ★★★
變化版 C — 未來感
請把風格改成賽博龐克風：黑底、霓虹粉紫漸層、發光邊框、字型用未來感無襯線、像 Cyberpunk 2077 主機版選單。
部署 repo 名稱建議：about-me-cyber
自學完成標準
✅
至少完成 1 個變化版部署
✅
三個版本（含主版）並排對比 → 寫一句「我最喜歡哪個版本、為什麼」筆記
✅
把最終版網址分享到課程群組「同學作品」討論串

最後挑一個最喜歡的版本變成「正式版」 → 把網址寫進 LinkedIn / 履歷。
