正式練習

Gemini Canvas / AI Studio 多檔產出整合演練

故意讓 AI 做多檔，再用一段提示詞壓成單檔，走 CH5-5 五步部署上線。整合失敗也有備援——這一頁讓你把完整動線練成肌肉記憶。

(01) 破題

先在練習裡製造麻煩，真實上線就不怕

CH5-6 教了「整合單檔」提示詞，但你可能還沒真的拿一個多檔產出練過。這一頁就是要你故意製造多檔、再親手壓成單檔。

為什麼故意？因為 Gemini Canvas、AI Studio 在真實使用時很常給你多檔——與其上線時才踩雷，不如現在先在演練裡踩一次、把救援流程練熟。

📌
這節口號：先在練習裡製造麻煩，真實上線就不怕。

這一頁是 CH5-5（部署）+ CH5-6（整合）的綜合演練。跑完你就有一條「AI 多檔產出 → 上線」的完整肌肉記憶。

(02) 提示詞卡

兩張提示詞——一張製造多檔，一張壓回單檔

今天的演練順序是：先貼①讓 Gemini 故意做出多檔，確認看到多檔後，再貼②把它整合成單一 HTML。

步驟一 · 故意做多檔（貼給 Gemini Canvas） ★★★★
① 故意做一個多檔網站
請用 Gemini Canvas 幫我做一個 3 頁個人作品集網站： - 首頁（自我介紹）、作品頁、關於我頁 - 三頁共用一份 CSS - 手機可讀、米白色底、簡約風 請開始製作。
Gemini 很可能把它做成 index.html + style.css（+ 各頁）——這正是我們要的多檔。貼完後，仔細看有沒有 <link rel="stylesheet" href="./style.css">，有的話就是多檔，繼續下一步。
步驟二 · 整合單檔（沿用 CH5-6） ★★★★★
② 整合成單一 HTML 檔
請把你剛剛做的這個網站，重新整理成單一 HTML 檔的版本：所有 CSS 寫進 <style>、所有 JS 寫進 <script>、圖片用 https://picsum.photos/ 連結、字型用 Google Fonts CDN、多頁改成單頁錨點切換。我要複製整段就能存成 index.html 直接用的版本。請開始重做。
這張卡來自 CH5-6 課程素材包。整合成功後，確認 HTML 裡沒有 <link rel="stylesheet" href=...>，CSS 全部在 <style> 裡面，才算真的整合乾淨。
(03) 操作示範

觀察多檔 + 整合動線——完整 5 步

講師現場示範一次、學員現場跟做一次。完整動線如下：

1
產出：貼提示詞① → Gemini Canvas 做出多檔作品集
2
觀察：打開看到 <link rel="stylesheet" href="./style.css"> 或分頁檔案 → 確認「這是多檔」
3
整合：貼提示詞② → Gemini 重做 → 給出一個全包的 index.html
4
部署：複製整段 → 走 CH5-5 五步（開 repo → 按 . → 建 index.html 貼上 → commit & push → 啟用 Pages）
5
驗證：打開 https://{你}.github.io/{repo}/ → 三個區塊用錨點切換、CSS 正常、圖片不破
📌
最低完成線（傻瓜路徑）：拿到整合後的單檔即算過；實際部署成功是加分；沒環境者交「多檔 vs 單檔」對照截圖。
(04) 備援演習

「失敗就走備援」——這段刻意練

這段刻意演練「整合不順」時怎麼辦，免得真實上線慌。就算整合成功了，每位學員也要主動走一次備援 A——把救援動作變肌肉記憶。

情境 A · 整合後檔案太大 / 整合反覆失敗
走 CH5-6 備援 A：拖拉 ZIP 上傳
把 Gemini 給的整個資料夾壓 ZIP → repo 頁「Add file → Upload files」→ 拖進去 → Commit。3 步上線，不碰 Terminal。
情境 B · 整合後樣式跑掉
用「描述現況 + 描述期望」讓 AI 自己改
回貼 AI：「目前畫面是 _____（描述你看到的），我預期是 _____（描述你要的），請修。」不要自己動 CSS，讓 AI 修。
📝
演習要求：每位學員至少主動走一次備援 A（就算整合成功了也練一次拖拉 ZIP），截圖為證。整合失敗走備援不是恥辱，是常態。
(05) 卡關地圖

4 個最常見卡關 · 修法速查表

卡關情境 修法
整合後圖片變 404（用了 placehold.com） 回貼「請把圖片連結改用 https://picsum.photos/600/400」。picsum 比較穩，placehold.com 不時掛掉。
多頁變單頁錨點後，切換很生硬沒動畫 回貼「請幫區塊切換加上 CSS transition 讓它平滑滑動」。
Gemini 給的單檔仍含 <link rel="stylesheet"> 回貼「你還留著 style.css 的連結，請把那份 CSS 內容直接內嵌進 <style> 標籤，不要外連」。這是最常見的「整合不乾淨」。
部署後看到舊版（改完忘了 commit） 回 github.dev → Source Control → stage + commit & push → 強制重新整理（Cmd+Shift+R）。
(06) 動手做

現在輪到你——用 Gemini Canvas 跑完整條動線

用 Gemini Canvas（或 AI Studio）做一個你自己的多檔小網站（作品集 / 活動頁 / 商店頁皆可），整合成單檔並部署。

驗收門檻 · 三條線擇一完成即過
✅
最低完成線：拿到整合後的單檔版本（CSS 全在 <style>、無外連 stylesheet）
✅
加分：部署到 GitHub Pages、錨點切換 + 圖片 + CSS 都正常
✅
備援也算：至少示範一次備援 A 拖拉 ZIP（截圖為證）
💡
講師 / 助教 SOP：
助教 A 守「Gemini 整合後仍外連 style.css」→ 教追問內嵌
助教 B 守「部署破圖」→ 教換 picsum.photos
助教 C 守備援 A 拖拉 ZIP 動作
講師處理「Gemini 怎麼弄都給多檔」的個案（直接判走備援 A）

跑完這頁你掌握了

✓
整合不出來 → 拖 ZIP 上線
✓

(08) 自學包 · 1.5 小時

變化版練習——三選一或全做

同一條「多檔 → 整合 → 部署」動線，換不同網站類型再跑一遍。

多頁店家 ★★★
練習 A — 多頁店家網站
首頁 + 菜單 + 聯絡，整合單檔部署
目標：三頁共用 CSS，整合後錨點切換正常
活動報名 ★★★
練習 B — 活動報名頁
主視覺 + 流程 + 報名表單（表單先做樣式即可），整合部署
表單只做樣式，不用真的接後端

在課堂作品上加一個「作品 lightbox 放大」互動，整合時注意 JS 也要內嵌
整合時最容易漏掉 JS，這版是主動挑戰

課前準備與課後收尾
