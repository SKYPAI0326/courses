Markdown 轉專業報表樣式

把 AI 生成的 Markdown 文字，秒變適合列印給長官的精美報告。不需要 Word，不需要排版——貼進來就完成。

PRACTICE GOALS
學會用 JavaScript 解析 Markdown 轉換為 HTML
理解 CSS 列印樣式（@media print）的設定
產出一個可直接列印為 PDF 的報告工具
GUIDED · 15–20 分鐘照著做主線

先跟著做一遍：用 AI 生成一個「Markdown 轉專業報表工具」

這頁的完整任務是用 AI 做出一個能把 Markdown 文字秒變精美報表的工具。但這件事最容易卡住的，不是把樣式做漂亮，而是把要支援的語法與列印行為描述清楚，再驗證 AI 真的把規格做齊了——這類工具的本質是「左側貼 Markdown 文字 → 右側即時渲染成排版好的報表 → 一鍵列印為 PDF」，少支援一種語法、列印時忘了隱藏輸入框，交出去的報表就會出包。這條主線只做一件事：拿一份已經寫好的工具規格，用一套固定 Prompt 叫 AI 生成單檔 HTML 工具，存下來打開，再用一張功能對照檢查表驗它有沒有做齊。全程貼文字進 AI、存成 .html 開來測，不用寫一行程式，15–20 分鐘做完。先把這條主線走完；做完還想多練，再往下走「進階練習池」——那裡有現成的互動工具、自己復刻的 Prompt、進階挑戰、延伸案例與自我演練。主線和練習池是同一主題的不同深度，主線是正式入門路徑，練習池是自選加練。

照著做主線 · STEP 1 / 4 · 場景設定（約 3 分鐘）

怡君的麻煩

怡君，34 歲，一家行銷顧問公司的專案經理。她每週都要交一份專案進度報告給總監。她習慣用 AI 把資料整理成 Markdown，但 Markdown 純文字直接交出去太陽春，總監要的是「排版整齊、可以列印歸檔」的報表。她現在的做法是把 Markdown 一段段貼進 Word 再手動套格式，光排版就花 30 分鐘。她想要一個小工具：左側貼 Markdown，右側立刻變成有標題底線、有清單縮排、有引用色條的精美報表，再一鍵列印成 PDF。可是她不會寫程式。

今天你就是怡君。你要用 AI 把這個 Markdown 轉報表工具生出來——重點不是把它做得多炫，是把要支援的語法與列印行為描述精確，再驗證 AI 真的做齊了。成品工具長這樣：一個左右分割版面，左側是深色背景的 Markdown 輸入框，右側即時渲染預覽；支援標題（#/##/###）、粗體、斜體、清單、引用、行內代碼；右上角有「列印為 PDF」按鈕，列印時只留預覽區、隱藏所有工具 UI。本主線的完整工具規格寫在 STEP 2 的主版 Prompt 裡，直接用即可，不需自己設計。

照著做主線 · STEP 2 / 4 · 主版工具生成 Prompt（約 4 分鐘）

主版工具生成 Prompt（直接複製，不用改）

這套 Prompt 把上面的工具規格寫成 AI 看得懂的生成指令，由 5 個部分組成：角色、版面結構、語法需求、列印行為、輸出規格。你不用自填，直接按右上角「複製」就能用。

→ 複製這段 Prompt 貼入 AI
你是專為文件排版設計的前端工程師，理解使用者要的是「貼進 Markdown 就立刻看到排版好、能列印的報表」。 請產出一份「Markdown 轉專業報表工具」單檔 HTML，內嵌 CSS/JS、不依賴任何外部 library。使用者可用它把 AI 生成的 Markdown 變成可列印給主管的報告。 【版面結構】 - 左右分割版面：左側為 Markdown 文字輸入框（深色背景、等寬字、code 感），右側即時渲染預覽 - 左側輸入時右側即時更新（oninput），不需按鈕 【語法需求】用 JavaScript 手工解析以下常用 Markdown 語法： - 標題 #、##、### - 粗體 **文字**、斜體 *文字* - 無序清單 -、有序清單 1. - 引用 > - 行內代碼 `文字` - 分隔線 --- 【列印行為】 - 頁面右上角有「列印為 PDF」按鈕，呼叫 window.print() - 用 @media print：列印時只顯示右側預覽區內容，隱藏左側輸入框與所有工具 UI 【設計】背景 #f5f3ee、預覽區白色背景、強調色 #b5703a；h1 有強調色底線、h2 有分隔線、清單有縮排、blockquote 有左色條；字體 Shippori Mincho（標題）+ Noto Sans TC（內文）；整體簡潔，預覽區排版要像一份正式報告。 【輸出格式】只輸出完整的單一 HTML 程式碼，所有功能寫在 inline JS 中，前後不要加任何說明文字或驗證報告——這段你會整段存成 .html 檔，多餘的文字會被一起存進去、變成工具裡的雜訊。

為何這樣寫：語法需求逐條列出，是這套 Prompt 的關鍵——Markdown 轉換工具最常見的漏做就是「只支援標題與粗體、漏了引用或行內代碼」，把每種語法拆開寫清楚，AI 才不會自行簡化。列印行為單獨開一段，是因為 @media print 隱藏輸入框最容易被忽略，沒寫死的話列印出來會把深色輸入框也印進去。最後一段明令只輸出 HTML、不附驗證報告：產出是要存檔的程式碼，AI 若在前後加說明或報告，會被一起存進 .html 變成工具裡的雜訊。驗證改用更可靠的方式——STEP 4 打開工具，用功能對照檢查表逐項看它做齊了沒。

照著做主線 · STEP 3 / 4 · 上機（約 6 分鐘）

跟著這 4 步生出工具並實際用一次

1
複製主版 Prompt 貼進 AI 送出
按 STEP 2 主版 Prompt 右上角的「複製指令」鈕。ChatGPT、Gemini、Claude.ai 的免費版都可以，開一個新對話視窗貼上去送出，AI 會回一整段 HTML。
2
存成 .html 檔
把 AI 給的 HTML 整段複製，貼到記事本／文字編輯器，存檔時把檔名取成 markdown-to-report.html（副檔名一定要 .html）。若 AI 一次沒給完整，回「請接續輸出剩下的 HTML」即可。
3
用瀏覽器打開實際操作
雙擊 markdown-to-report.html 用瀏覽器開。在左側貼一段含標題、清單、引用的 Markdown，看右側即時渲染，再按右上角「列印為 PDF」確認列印預覽只剩報表、沒有輸入框。
4
對照功能檢查表
用下方 STEP 4 的功能對照檢查表逐項確認。對不上的，把問題追問給 AI 重生一次。
照著做主線 · STEP 4 / 4 · 工具功能對照檢查表（約 4 分鐘）

離開前對照功能檢查表打 4 個勾

AI 生的工具看起來再漂亮，都要逐項驗。把你打開的 markdown-to-report.html 對照下面 4 項檢查，全部確認過，今天的主線才算完成。對不上的，回 STEP 3 把問題追問給 AI 重生一次。

工具功能對照檢查表

4 個勾全打完，你已經用 AI 做出一個能把 Markdown 秒變專業報表的工具了。想再多練——往下走「進階練習池」：那裡有現成可玩的互動工具、自己復刻的 Prompt、進階挑戰、延伸案例與自我演練，把今天這套「描述規格 → 生成 → 對照驗收」的方法練得更熟。

延伸案例

Markdown 轉樣式工具會自動套一套專業樣式，但真正的問題是「交出去前，內容結構對不對」。帶一份會被甲方看的真實文件來練。

案例 08 · 自由顧問
接了品牌顧問案，工作坊會議記錄要交給甲方行銷總監
難度 ★★☆ 約 40 分鐘

你是自由品牌顧問，帶客戶做完兩小時 Brand Sprint 工作坊，習慣用 Markdown 邊帶邊記。現在要把這份原始筆記整理成一份「交得出手」的會議紀錄交給客戶的行銷總監，她平常看 PowerPoint 比較多，不熟 Markdown。你要用工具轉成網頁報表，再決定怎麼交。

原始資料｜工作坊筆記（Markdown）
# Brand Sprint 會議紀錄 時間：2026-04-15 14:00–16:00 客戶：XX 手作皂品牌 出席：行銷總監 Ada / 品牌經理 Ben / 我 ## 1. 目前品牌痛點（前 30 分鐘） - 官網轉換率 0.8%，業界平均 2.1% - 近半年新客比例從 40% 掉到 22% - 客服常被問「你們跟其他手作皂差在哪」 ## 2. 品牌核心三問 - 我們為誰而做？→ 30–40 歲女性，有皮膚敏感問題 - 我們不是什麼？→ 不是網美款、不是大量量產 - 我們希望客人記住的一句話？→ 「洗一次就知道」 ## 3. Action Items - [ ] Ada：4/22 前提供過去一年客訴文字雲 - [ ] Ben：4/25 前列出 5 家最像的競品與差異 - [ ] 我：5/2 前交第一版品牌定位書 ## 4. 我的觀察（內部） - Ada 在價格敏感度這點跟 Ben 明顯不同調 - Ben 對「網美款」一詞反應很強，值得追
你的任務
  1. 判斷哪幾段要給客戶看、哪些要砍（第 4 段「內部觀察」就是例子）
  2. 調整結構讓行銷總監 10 分鐘能抓到重點（排序？加一段 TL;DR？）
  3. 決定交付格式（網頁連結 / PDF / 附件）與理由，寫一段寄信措辭
評估指標
內部/對外切乾淨
第 4 段內部觀察不交出去，不留任何模糊字眼
排序合理

Action 突出

參考 Prompt（建議先自己寫再看）
點擊展開參考答案
【角色】你是接獨立案的品牌顧問，交給客戶的文件要「正式但不裝、用詞精準不堆疊形容詞」，知道什麼該寫、什麼要刪。 【任務】根據下方 Markdown 原始筆記，產出： 1. 客戶版 Markdown（砍掉第 4 段內部觀察，重排順序：先 TL;DR、再 Action、再痛點、再核心三問） 2. 交付格式選擇與理由：網頁連結（用本課程 Markdown 工具轉）／PDF／附件，三選一 3. 寄信 email 草稿（≤120 字），中文，平實語氣 【限制】 - 不要保留「我的觀察（內部）」任何一個字，連副標題都砍 - TL;DR 三行內寫完：會議目標、關鍵結論、下一步 - 寄信不准用「你好」「非常感謝」，直接用客戶名字 - 三條 Action 用 checkbox 或粗體呈現，不用純 bullet 【輸出格式】 分三段：客戶版 Markdown、交付格式決策、寄信草稿。
