互動小遊戲上 GitHub（三選一）

你不需要懂 JavaScript，只要會在提示詞裡指名「請加最高分功能」——AI 就把程式碼補上。60 分鐘後你會有一個朋友能玩、會記分、能在手機上跑的小遊戲網址。

PRACTICE GOALS
用提示詞讓 AI 產出單檔 HTML 互動小遊戲（Wordle / 記憶配對 / 打地鼠 三選一）
學會「請用 localStorage 存 [某物]」這招，讓任何工具記得最高分、偏好設定
走完 CH5-5 五步部署，拿到可分享的 GitHub Pages 網址
(01) · 破題

6 分鐘讓新人記住 16 個同事名字

下班時間 6 點半，行政小陳跟同事在茶水間聊天。

同事說：「上週你的個人單頁網站做超快，是怎麼弄的？」

她想到一個更好玩的點子：「新人到職這週，要不要我做一個『同事姓名記憶配對遊戲』，讓他用玩遊戲的方式背名字？」

3 分鐘她貼提示詞給 Claude Artifacts、3 分鐘走 CH5-5 五步、6 分鐘新人桌上多了一個網址 —— 點開是 16 張卡片配對遊戲、每張卡片背後是部門同事的名字和頭像、配對成功會發出叮咚聲。

新人邊玩邊笑，一週後她全部 16 個同事姓名都認得。

💡
這 1 小時，我們把「遊戲也只是 HTML」這個觀念內化。你選任一種遊戲，跑完整流程，拿到可分享網址。
(02) · 概念

三選一提示詞卡——先挑遊戲，再跑流程

今天有 3 種遊戲，任挑一個跑完整流程。現場示範用選項 B 記憶配對，自學包三個都跑完。

選項 A
中文 Wordle 猜詞遊戲
課程複習 / 術語學習
適合做「課程複習」「公司術語學習」「行銷詞彙練習」—— AI 先問你要猜的詞，你丟清單它做題庫。
請用單一 HTML 檔做一個中文猜詞遊戲：我會給你 10 個課程關鍵詞，玩家每輪猜一個詞，系統提示「字數」與「猜對幾個字」，6 次內猜中得分。介面用米白底、深灰格線。請先回我「請給我關鍵詞清單」。
選項 B
記憶配對翻牌
現場示範
最實用、最容易客製。主題可換：同事姓名、課程術語、公司產品特色——貼 8 對內容就好，程式碼 AI 全包。
請用單一 HTML 檔做一個 4×4 記憶配對翻牌遊戲。要求： - 8 對配對卡片（共 16 張） - 點擊翻牌、配對成功保持翻開、失敗 1 秒後翻回去 - 配對成功播放音效（用 Web Audio API 純音、不要外部音檔） - 顯示步數計時器，全部配對完顯示「恭喜！X 步、Y 秒」 - 用 localStorage 存最少步數紀錄 - 介面米白底、卡片正面用主題色（米白底 + 深綠 / 棕的襯線數字 #1-#8）、卡片反面顯示內容 - 內容由我提供，請先回我「請給我 8 對配對的內容（每對：標題 + 對應描述）」 主題我會說，你先回我那句確認。
選項 C
打地鼠計分遊戲
5 分鐘紓壓 / 最快完成
提示詞最直接，不需 AI 先問問題、直接給完整 HTML。時間緊或想穩穩過關就選這個。
請用單一 HTML 檔做一個打地鼠遊戲： - 9 格 3×3 地洞網格、米白底、地洞用深灰色 - 地鼠隨機冒出 1 秒（用 emoji 🐹 或純文字「地鼠」） - 滑鼠點到地鼠加 1 分、沒點到不扣分 - 60 秒倒數計時、結束顯示總分 - 最高分用 localStorage 存 - 手機可玩（觸控也算點擊） - 開始按鈕、結束顯示「再玩一次」按鈕 請直接給我完整 HTML。

(04) · 元能力

「遊戲要存最高分」這招要背起來

這是本單元最值得帶走的「元能力」：你不需要懂 localStorage 是什麼，只要會在提示詞裡指名。

背起來這個句型
「請用 localStorage 存 [某個東西]，下次打開還在。」
凡是你做的小遊戲、小工具，只要結果想存下來（最高分、設定、上次填過的內容），在原提示詞末尾加一句就好。
範例：
「請用 localStorage 存玩家最高分，下次打開還在。」 「請用 localStorage 存最後一次填過的姓名和 Email，下次打開自動填回去。」 「請用 localStorage 存深色 / 淺色模式偏好，下次打開記得。」

你不需要知道：localStorage 是什麼（瀏覽器內建的小資料庫）、它怎麼存（key-value 字串）、容量多大（5MB，遊戲分數綽綽有餘）。你只要會「指名」這個功能存在。AI 會自動把對應的 JS 程式碼補進去。

(06) · 卡關地圖

常見的 5 個卡關點與修法

1
AI 用了 emoji 但部分裝置顯示方框
症狀：遊戲在 Mac 看正常，Windows / Android 學員打開看到「□」。原因：部分舊版 Windows / Android 不支援較新的 emoji。
回 AI 說：「請把所有 emoji 圖示（🐹 等）改成純文字符號（用「鼠」「貓」這種中文）或 SVG inline。」
2
翻牌動畫不順、看起來卡頓
症狀：點卡片、3D 翻牌動畫一頓一頓。原因：AI 預設沒加 CSS transition 過渡效果。
回 AI 說：「請在卡片翻牌動畫加 transition: transform 0.4s ease-in-out，讓動作平順。」
3
音效在 iOS Safari 不響
症狀：Mac / Android 有叮咚音效，iPhone Safari 沒聲音。原因：iOS Safari 要求使用者先點擊頁面才能播音效。
回 AI 說：「請加一個「開始遊戲」按鈕，玩家點下去才開始計時。在這個點擊事件裡初始化 AudioContext，這樣 iOS Safari 才能播聲音。」
4
localStorage 紀錄沒存到，重整就消失
症狀：破了 20 步紀錄、重整頁面、最少步數變回 0。原因：AI 可能寫成 sessionStorage（關分頁就消失）而非 localStorage（永久存）。
回 AI 確認：「請確認最少步數是用 localStorage.setItem() / localStorage.getItem()，不是 sessionStorage。」
5
4×4 卡片在手機上排成 1 欄（跑版）
症狀：手機打開，16 張卡片變成上下堆疊 16 行。原因：AI 的 RWD 寫太緊，手機螢幕容不下 4×4。
回 AI 說：「請優化手機版面：在 375px 寬度時、卡片用 grid-template-columns: repeat(4, 1fr) 強制保持 4 欄，卡片字級可以縮小到 24px。」

(09) · 自學包

1 小時自學：把遊戲加 1–2 個功能

把選定的遊戲加 1–2 個功能，重新跑提示詞 + 部署新版本。在原提示詞末尾加一句就夠了。

1
加難度選擇
在開始畫面加一個難度選擇：簡單 / 中等 / 困難。簡單版減少配對數量（記憶遊戲變 3×4 共 12 張）、困難版增加數量（記憶遊戲變 5×4 共 20 張）。
2
加計時器壓力
請加 60 秒倒數計時。時間到還沒完成 → 顯示「時間到！你完成了 X 對」。
3
加音效豐富度
請把音效豐富化：配對成功 = 高音「叮」、配對失敗 = 低音「噗」、遊戲開始 = 上行音階、遊戲結束 = 下行音階。全部用 Web Audio API 合成，不要外部音檔。
4
加首頁選關卡
請加一個首頁畫面：標題、3 個按鈕（簡單 / 中等 / 困難），點按鈕後才開始遊戲。
