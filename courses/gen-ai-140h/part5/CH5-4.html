Part 5 · 前端與部署實戰
CH 5.4 · 2 小時

含後端服務的完整部署：
讓 API 金鑰不外洩

把工具放上網不難——難的是呼叫 AI API 的時候，怎麼讓金鑰不出現在前端程式碼裡。這節課教你加一層後端，把敏感操作藏在伺服器端，然後整個部署到雲端。

學習成果 · OUTCOMES
能判斷哪些場景需要後端，哪些靜態部署就夠了
知道如何用環境變數（.env）保護 API 金鑰，讓金鑰不出現在程式碼或 GitHub 裡
能建立一個最小後端（Node.js 或 Python），提供一個 /api/ask 代理端點
能將含後端的應用部署到 Railway 或 Render，取得可公開訪問的網址
Section 1 · Static vs Backend

靜態 vs 後端：
什麼時候你需要後端？

不是所有網頁都需要後端。搞清楚這條界線，能讓你選對部署方式，省下不必要的複雜度。

靜態部署就夠了（Vercel / Netlify）
純展示頁面、作品集、說明文件
使用者在前端直接呼叫第三方 API（且該 API 允許前端呼叫）
沒有需要保護的金鑰或私有邏輯
不需要儲存使用者資料
需要後端的情況（Railway / Render）
呼叫需要保密的 API 金鑰（如 OpenAI、Anthropic）
需要處理表單送出後發送 Email
需要讀寫資料庫（儲存使用者輸入、紀錄）
需要排程、背景任務、Webhook 接收
這節課的核心情境：你在 Part 4 做了一個呼叫 AI 的工具，金鑰直接寫在 JavaScript 裡。只要別人開 DevTools，兩秒就能看到你的金鑰。後端的第一個用途，就是讓這件事不發生。
部署類型 適用場景 平台選項 難度
靜態 HTML/CSS/JS 只有前端，不需要伺服器執行程式碼 Vercel、Netlify、GitHub Pages 低，連接 GitHub 後自動部署
含後端 有 Node.js / Python 伺服器，需要在後端執行邏輯 Railway、Render、Fly.io 中，需要設定環境變數與啟動指令
ℹ️
Vercel 的 Edge Functions / API Routes：Vercel 也支援後端邏輯（Next.js API routes 或 Vercel Functions），可以保護金鑰。但這節課選擇 Railway/Render 的獨立後端，因為架構更清晰，學起來的概念適用範圍更廣。
Section 2 · Environment Variables

安全第一：
環境變數與 API 金鑰保護

API 金鑰是你的帳單責任。一旦洩漏到公開的 GitHub repo，有人的爬蟲會在幾分鐘內找到它，用你的帳號發出大量請求。環境變數是這個問題的標準解法。

規則一：金鑰絕對不寫死在程式碼裡
不要這樣做：const apiKey = "sk-ant-abc123..."。這樣的字串只要進入任何一個 commit，即使你之後刪掉，git 歷史裡還是找得到。
規則二：用 .env 存放本機的金鑰
在專案根目錄建立 .env 檔，寫入 ANTHROPIC_API_KEY=sk-ant-abc123...。程式碼裡用 process.env.ANTHROPIC_API_KEY（Node.js）或 os.environ["ANTHROPIC_API_KEY"]（Python）讀取。
規則三：.env 必須加入 .gitignore
在 .gitignore 裡加一行 .env，讓 Git 永遠不追蹤這個檔案。雲端平台的金鑰則在部署設定頁面的「環境變數」欄位填入，不是用 .env 檔。
BASH 基本的 .env 設定 
# .env（這個檔案不進 git）
ANTHROPIC_API_KEY=sk-ant-api03-...
PORT=3000

# .gitignore（確保 .env 不被追蹤）
.env
node_modules/
__pycache__/
⚠️
已經不小心推上去了怎麼辦？立刻去 Anthropic Console / OpenAI 後台把那把金鑰 revoke 掉，然後產生一把新的。不要只刪程式碼後重新 commit——git 歷史裡的紀錄永遠都在，那把金鑰已經不能再用了。

一個好的習慣是同時維護一個 .env.example 檔案——列出所有需要的環境變數名稱，但不填入實際值。這個檔案可以進 git，讓其他人或未來的自己知道需要設定哪些金鑰。

BASH .env.example（可以進 git 的範本） 
# 複製這個檔案為 .env，填入你的實際金鑰
ANTHROPIC_API_KEY=your_key_here
PORT=3000
Section 3 · Minimal Backend

後端最小架構：
一個 /api/ask 代理端點

你不需要學完整的後端開發。這節課的目標是一個「最薄的後端」——只做一件事：接收前端傳來的問題，用金鑰呼叫 AI API，把回應傳回去。

瀏覽器
使用者操作的前端頁面
→ POST /api/ask →
你的後端
Railway / Render 上的伺服器
→ API call（含金鑰）→
AI API
Anthropic / OpenAI
金鑰只存在後端伺服器裡，瀏覽器永遠看不到它。前端只知道後端的網址，不知道 API 金鑰是什麼。

以下是 Node.js（Express）版本的最小後端，只有一個路由：

JAVASCRIPT server.js — Node.js 最小後端 
const express = require('express');
const Anthropic = require('@anthropic-ai/sdk');
require('dotenv').config();

const app = express();
app.use(express.json());
app.use(express.static('public')); // 前端 HTML 放在 public/ 目錄

const client = new Anthropic(); // 自動讀取 ANTHROPIC_API_KEY 環境變數

app.post('/api/ask', async (req, res) => {
  const { prompt } = req.body;
  if (!prompt) return res.status(400).json({ error: '需要 prompt' });

  try {
    const message = await client.messages.create({
      model: 'claude-haiku-4-5-20251001',
      max_tokens: 1024,
      messages: [{ role: 'user', content: prompt }]
    });
    res.json({ result: message.content[0].text });
  } catch (err) {
    res.status(500).json({ error: err.message });
  }
});

const PORT = process.env.PORT || 3000;
app.listen(PORT, () => console.log(`伺服器啟動在 port ${PORT}`));

前端只需要用 fetch 呼叫這個路由，不碰任何金鑰：

JAVASCRIPT 前端呼叫後端代理（不含金鑰） 
async function askAI(prompt) {
  const res = await fetch('/api/ask', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ prompt })
  });
  const data = await res.json();
  return data.result;
}
💡
Python 版本（Flask）：如果你更熟悉 Python，用 Flask 做同樣的事只需要約 20 行。核心邏輯完全相同：讀取 os.environ["ANTHROPIC_API_KEY"]，建立一個 POST 路由，呼叫 API 後回傳結果。選 Node.js 還是 Python 不重要，架構概念一樣。

在本機測試的步驟：

01
安裝依賴套件
npm install express @anthropic-ai/sdk dotenv
02
建立 .env 並填入金鑰
ANTHROPIC_API_KEY=sk-ant-...
03
啟動本機伺服器
node server.js，瀏覽器開 http://localhost:3000 測試
04
確認 .gitignore 排除 .env
執行 git status，確認 .env 沒有出現在追蹤清單裡
Section 4 · Deploy

部署到 Railway / Render：
從 GitHub 到公開網址

Railway 和 Render 都能直接連接你的 GitHub repo，偵測到是 Node.js 或 Python 專案後自動安裝套件並啟動。主要差異是 Railway 啟動更快、免費額度按用量計算；Render 免費方案固定，但閒置後第一次請求會有冷啟動延遲。兩個都適合這節課的練習。

1
確認專案結構正確
根目錄有 package.json（Node.js）或 requirements.txt（Python），啟動指令寫在 package.json 的 scripts.start 裡，如 "start": "node server.js"。
2
push 到 GitHub
確認 .env 已在 .gitignore 裡後，執行 git add . → git commit -m "後端初版" → git push。
3
在 Railway / Render 建立新服務
登入平台 → New Project / New Web Service → 選「從 GitHub 部署」→ 授權並選擇你的 repo。平台會自動偵測語言和安裝指令。
4
設定環境變數
在平台的 Variables / Environment 頁面，手動新增 ANTHROPIC_API_KEY，值填入你的金鑰。這個金鑰只存在平台的加密設定裡，不在任何程式碼裡。
5
等待部署完成，取得網址
部署成功後平台會給你一個 https://your-app.railway.app 或 https://your-app.onrender.com 的網址。在瀏覽器開啟測試，功能與本機一致即完成。
JSON package.json 最小範例 
{
  "name": "my-ai-tool",
  "version": "1.0.0",
  "scripts": {
    "start": "node server.js"
  },
  "dependencies": {
    "express": "^4.18.0",
    "@anthropic-ai/sdk": "^0.30.0",
    "dotenv": "^16.0.0"
  }
}
🔑
後續每次更新：本機改好 → git commit → git push。Railway 和 Render 都設有 GitHub webhook，偵測到新 push 後會自動重新部署，你不需要手動操作平台。
