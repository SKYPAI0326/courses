看懂 HTML / CSS / JavaScript的最低必要知識

你不需要從零學會寫網頁，但你需要能夠讀懂 AI 幫你生出來的程式碼。這節課把前端三大語言拆解成「能讀、能判斷、能改」所需要的最小知識集合。

學習成果 · OUTCOMES
能識別 HTML 各類標籤的功能，並在 AI 產出的頁面中快速定位你要修改的結構
能讀懂 CSS 規則，理解字型、顏色、間距、排版的屬性含義，知道改哪裡
能跟著 JavaScript 的邏輯走：變數、函式、事件、DOM 操作各代表什麼意圖
建立「看 AI 程式碼→定位問題→用 AI 修改」的閱讀工作流
RECAP · Part 4 帶走的
Part 4 讓你能用 AI 生成有計算邏輯的 HTML 工具，並優化到真正好用——從需求到可交付工具的完整閉環已建立。
PREVIEW · 這節解決什麼
改 AI 生的程式碼之前要先讀懂它。這節給你「能讀、能判斷、能改」所需要的前端最小知識集合，不需要從零學寫網頁。
(01) · Mindset First

不是要你寫，而是要你能讀

這堂課的定位和多數前端教學不同。我們不打算教你從空白頁面刻出一個網站——那需要數百小時的練習。我們的目標是：當 AI 幫你生出一份 HTML/CSS/JS 程式碼時，你能夠讀懂它在做什麼、知道要改哪裡、能判斷它有沒有問題。

這個能力比「會寫」更接近現在的 AI 協作現實。90% 的情況下，你的角色是審閱者與指揮官，而不是打字員。

傳統前端學習路徑
從 HTML 語法全背起來
手刻 CSS 排版練習幾百次
JS 基礎→框架→生態系
需要 300+ 小時才能做出可用頁面
AI 協作閱讀路徑（本課）
認識各類標籤的意圖與結構
能看懂 CSS 屬性，知道改哪個數字
能跟著 JS 邏輯走，理解流程
3 小時可達「能讀能改」的工作水平
「你不需要會寫樂譜，
但你要能聽出哪個音跑掉了。讀懂程式碼就是這個能力。」
💡
本節核心心態：把 HTML/CSS/JS 想成三種語言，各自負責一件事——結構、外觀、行為。弄清楚哪個問題屬於哪種語言，你就已經比 80% 的人更會跟 AI 溝通前端問題了。
(02) · HTML

HTML：頁面的骨架

HTML（HyperText Markup Language）負責定義頁面的結構與內容——什麼地方有標題、什麼地方有段落、什麼地方是按鈕。它不管顏色、不管大小，只管「這個東西是什麼」。

HTML 基本頁面骨架 
<!-- 整個頁面的容器 -->
<div class="card">

  <!-- 標題 -->
  <h2>我的工具</h2>

  <!-- 說明文字 -->
  <p>這是一個 AI 工具頁面</p>

  <!-- 輸入框 + 標籤 -->
  <label>輸入你的文字</label>
  <textarea id="userInput"></textarea>

  <!-- 按鈕 -->
  <button onclick="generate()">產生</button>

  <!-- 輸出區域 -->
  <div id="output"></div>

</div>

每個 HTML 標籤都有開頭 <tag> 和結尾 </tag>，中間包住內容。認識常見標籤是讀懂 AI 程式碼的第一步：

<h1> ~ <h6>
標題，數字越大越小。h1 是頁面主標題，h2 是章節標題，h3 以下是小節。AI 通常用 h2/h3 來組織工具頁的段落結構。
<p>
段落文字。最常見的文字容器，AI 會用它包說明文字或錯誤提示。
<div>
通用容器，本身沒有外觀，靠 class 和 CSS 賦予樣式。AI 大量使用 div 來分組和布局，是你最常看到的標籤。
<input>
單行輸入框。常見 type 屬性：text（文字）、number（數字）、checkbox（勾選）。注意：input 是自閉合標籤，不需要結尾 </input>。
<textarea>
多行輸入框。適合輸入較長文字，如 Prompt 或文章段落。AI 工具頁常用。
<button>
按鈕。最重要的屬性是 onclick，指向要執行的 JS 函式名稱。
<select> / <option>
下拉選單。select 是容器，option 是每個選項。AI 工具頁常用來切換語氣、語言等設定。
<ul> / <li>
無序清單（ul）和清單項目（li）。ul 是容器，每個 li 是一條。有序清單用 <ol>。
class 屬性
每個標籤可以帶 class 名稱，CSS 用 .class-name 來指定外觀，JS 用 querySelector 來找到它。這是 HTML/CSS/JS 三者溝通的橋樑。
id 屬性
全頁面唯一的識別符。JS 常用 document.getElementById('id名') 來操作特定元素。看到 id 就知道這個元素有特定邏輯在操作它。
📌
閱讀技巧：看到一大段 HTML 不知從哪讀起，先找 id 屬性。有 id 的元素通常是最重要的——它是 JavaScript 要操作的目標，也是功能的核心。

## 問題清單
- 沒用語意標籤：整份檔用 <div> 包，沒有 <header> <main> <form> <footer>。後果：螢幕閱讀器無法跳區、SEO 不認得內容類型、未來 AI 修改頁面時更容易動到不該動的地方
- label 沒綁 input：「年資（年）：」「假別：」「請假天數：」都用 <div> 顯示，沒用 <label for="...">。後果：點 label 不會 focus 到 input、無障礙不及格
- button 寫成 a 標籤：「查詢」「清除」是動作（會跑 JS），應該用 <button type="button">，不是 <a href="#">。後果：tab 鍵 focus 順序怪、Enter 鍵行為不一致、有些瀏覽器會記入歷史
- 重複 id="result"：兩個 div 都用 id="result"，document.getElementById('result') 只會抓到第一個。後果：第二個 div 永遠寫不進去
- input type="text" 用在數字欄：「年資」「請假天數」應該用 type="number"。後果：手機跳出文字鍵盤、可輸入「abc」結果計算時 NaN
- inline onclick 寫死 JS：onclick="check(); return false" 寫在 HTML 屬性裡。後果：邏輯散落、難 debug、無法統一加事件監聽

## 修正指令
請重寫這份 HTML 片段：(1) 用 header/main/form/footer 包裝結構（2) 每個欄位前文字用 <label for="..."> 綁對應 input 的 id（3) 「查詢」「清除」改 <button type="button">，移除 href="#" 與 inline onclick（4) 重複 id="result" 改成 result-main 與 result-detail（5) 年資、請假天數改 <input type="number" min="0">（6) 移除所有 inline onclick，事件交給 JS 區塊用 addEventListener 處理。保留 class 名稱、欄位順序、不加新功能。

對照重點
1. 學員若漏掉「重複 id」這項，提醒 getElementById 只回傳第一個——這是學員未來自己 debug 時最容易卡的雷
2. 修正指令要明確寫「保留」哪些（class 名 / 欄位順序），AI 才不會順手大改
3. 學員若把 button 改成 <input type="submit">，雖然能用但會觸發 form 預設送出行為（可能跳頁），用 type="button" 比較安全

(03) · CSS

CSS：外觀的控制器

CSS（Cascading Style Sheets）負責所有視覺設定——顏色、字型大小、間距、排版、圓角。它的語法是「選誰、設什麼」的規則：

CSS 基本語法結構 
/* 選到 .card 這個 class，設定它的樣式 */
.card {
  background: #ffffff;      /* 背景色：白色 */
  border: 1px solid #d8d4cb; /* 邊框：1像素 實線 米色 */
  border-radius: 6px;      /* 圓角：6像素 */
  padding: 24px 28px;    /* 內距：上下24 左右28 */
  margin-bottom: 16px;  /* 外距（下方）：16像素 */
  font-size: 0.9rem;     /* 字型大小：0.9倍基準值 */
  color: #2c2b28;        /* 文字顏色：深棕黑 */
}

不需要把所有屬性都記住，只需要能認出最常出現的類別，知道改哪個數字會影響什麼：

屬性類別 常見屬性 你改這個時，頁面會變什麼
顏色 color / background / border-color 文字色 / 背景色 / 邊框色。十六進位色碼（如 #b5703a）或 CSS 變數（如 var(--c-a1)）都是顏色值。
字型 font-size / font-weight / font-family / line-height 字大小 / 粗細（400=正常,700=粗）/ 字體 / 行距。font-size 的 rem 單位是相對於頁面基準值（通常 1rem = 16px）。
間距 padding / margin padding 是元素內部空間；margin 是元素外部空間。兩個數字如「24px 28px」代表「上下 左右」；四個數字代表「上 右 下 左」（順時針）。
尺寸 width / height / max-width / min-height 元素的寬高限制。max-width 常用來限制內容區不超過某個寬度（如 780px），防止在寬螢幕上散開。
排版 display / flex / grid / gap display:flex 讓子元素橫排；display:grid 配合 grid-template-columns 做欄位布局；gap 控制元素之間的間距。
邊框與圓角 border / border-radius / border-left border-radius 越大越圓。border-left 單獨設定左邊框，常用於卡片的色彩點綴（如左側 3px 色條）。
定位 position / top / left / z-index sticky 讓元素跟著滾動但停在頂端（常見於 topbar）；fixed 讓元素固定在螢幕位置；z-index 控制疊加層次（數字越大越上層）。
類比理解 · ANALOGY
把 CSS 選擇器想成「收件人地址」，屬性想成「指令」。.card { padding: 24px } 就是「寄給所有 class=card 的元素，告訴它們：內部留 24px 空間」。如果你想改一個特定按鈕，先確認它的 class 名稱，再在 CSS 裡搜尋那個 class，就能找到控制它外觀的所有規則。
🔍
實用技巧：在瀏覽器中對任何元素按右鍵→「檢查」（Inspect），就能在 DevTools 裡直接看到它套用了哪些 CSS 規則，也可以即時修改預覽效果，確認要改什麼再告訴 AI。

## 問題清單
- 顏色硬編碼散落：#aabbcc / #333 / #555 / #4a8 / #5b9 / #abc / #999 / white / #fff 等 9 個顏色直接寫在規則裡，沒用 CSS 變數。後果：改主色要找 N 個地方、容易漏改、設計師看不出系統色票
- !important 濫用：.page-top 的 background、.form-area div 的 color、.btn-go 的 color、:hover 的 background、#result 的 color 都加 !important。後果：以後想再蓋過去只能用更高 specificity 或更多 !important，越疊越糟
- 單位用 px 不用 rem：font-size 24px / 14px / 15px / 16px / 12px 全寫死。後果：使用者調整瀏覽器字級不會生效，無障礙不及格
- 媒體查詢分散：max-width: 600px 與 min-width: 1200px 寫在檔案兩處。後果：未來看不出有哪些斷點、改一個忘了另一個
- #result 用 id selector：style 跟有問題的 id 結構耦合，且 HTML 那邊已知有重複 id。後果：無法重用樣式
- shorthand 與 longhand 混用：.page-top 用 background（shorthand），.btn-go 用 background-color（longhand），:hover 又回 background。後果：reset / unset / 級聯覆蓋行為不一致

## 重構指令
請重構這段 CSS：(1) 把所有顏色提到 :root 用 CSS 變數定義（--c-primary, --c-text, --c-text-soft, --c-bg, --c-result-bg, --c-footer 等），規則內全換 var(--xxx)（2) 移除所有 !important（3) font-size 從 px 改 rem，以 16 為 base（14px=0.875rem、15px=0.9375rem、16px=1rem、12px=0.75rem、24px=1.5rem）（4) 把兩個 @media 規則集中到檔案最底部，先 max-width 再 min-width（5) #result 改 .result（6) 統一用 background-color（longhand）。保留所有規則順序與選擇器名稱（除 #result→.result）。

對照重點
1. CSS 變數最該優先做——這是維護性轉折點，學員看到 :root 就明白「以後改主色只改一個地方」
2. !important 移除不能無腦砍：要先讀懂原作者用 !important 想蓋誰的 specificity，再決定是調整選擇器還是真的需要強制
3. px → rem 換算 16 為 base：學員若把所有 px 直接除以 16 就好

(04) · JavaScript

JavaScript：行為的驅動器

JavaScript（JS）負責頁面的動態行為——使用者點了按鈕發生什麼事、輸入被如何處理、輸出怎麼顯示在畫面上。它是三者中邏輯最強的語言，但你只需要認識四個概念就能讀懂大多數 AI 工具頁的 JS 程式碼。

概念一：變數——儲存資料的容器

JavaScript 變數宣告 
const userName = '小明';         // 固定值，不會改變
let inputText = '';              // 可變值，初始為空字串
let count = 0;                   // 數字，初始為 0
let isLoading = false;           // 布林值：true/false

看到 const 或 let 就是在宣告變數。const 是固定值，let 是可以被重新賦值的。

概念二：函式——可以重複執行的步驟包

JavaScript 函式定義與呼叫 
// 定義一個叫 generate 的函式
function generate() {
  // 取得輸入框的值
  const input = document.getElementById('userInput').value;

  // 如果輸入是空的，顯示警告
  if (input === '') {
    alert('請先輸入內容');
    return;
  }

  // 把結果顯示在 output 元素裡
  document.getElementById('output').textContent = '結果：' + input;
}

// 呼叫這個函式（執行它）
generate();

函式用 function 名稱() { ... } 定義，用 名稱() 執行。按鈕的 onclick="generate()" 就是「按下去時執行 generate 函式」。

概念三：事件——使用者動作的觸發點

JavaScript 事件監聽 
// 方式一：直接在 HTML 標籤上寫 onclick
// <button onclick="doSomething()">點我</button>

// 方式二：用 addEventListener（更靈活）
document.getElementById('myButton')
  .addEventListener('click', function() {
    // 使用者點擊按鈕時執行這裡的程式碼
    console.log('按鈕被點了');
  });

// 常見事件類型
// 'click'   → 點擊
// 'input'   → 輸入框內容改變
// 'change'  → 下拉選單選項改變
// 'scroll'  → 頁面滾動

概念四：DOM 操作——用 JS 修改頁面內容

JavaScript DOM 操作常用模式 
// 取得元素（找到它）
const el = document.getElementById('output');
const btn = document.querySelector('.submit-btn');

// 修改文字內容
el.textContent = '新的文字';

// 修改 HTML 結構（可插入標籤）
el.innerHTML = '<strong>粗體文字</strong>';

// 修改 CSS 樣式
el.style.color = 'red';
el.style.display = 'none';   // 隱藏元素
el.style.display = 'block';  // 顯示元素

// 加上 / 移除 CSS class
el.classList.add('active');
el.classList.remove('hidden');
el.classList.toggle('open');  // 有就移除，沒有就加上

DOM（Document Object Model）就是「JS 看到的頁面結構」。getElementById 是「去找 id 叫這個名字的元素」，querySelector 是「去找第一個符合這個 CSS 選擇器的元素」。找到之後就可以改它的文字、樣式、或套用不同的 class。

📌
讀 JS 的訣竅：從事件開始讀。找到 onclick 或 addEventListener，那個函式就是程式的入口點。進入函式後，先看它「取了什麼輸入」，再看它「對輸入做了什麼處理」，最後看它「把結果放到哪個元素」。這三步走完，你就知道這段程式碼在做什麼了。
(05) · Reading in Practice

實戰：讀懂 AI 產出的程式碼

現在你有了三種語言的基本閱讀能力。但實際工作時，AI 往往一次給出幾百行混合的 HTML/CSS/JS。以下是一套可以重複使用的閱讀與修改工作流：

01
先看 HTML，定位結構。快速掃過 HTML 標籤，找出「輸入區在哪」、「按鈕在哪」、「輸出區在哪」。這三個元素的 id 是你最重要的地圖座標。
02
用瀏覽器 Inspect 確認視覺問題。如果看起來不對（顏色錯、跑版、太擠），右鍵→「檢查」，找到對應元素，直接在 DevTools 看它套用了哪個 CSS class，再回 <style> 區塊找那個 class 修改。
03
找事件入口點，追邏輯。如果功能不對（按下去沒反應、結果錯誤），找按鈕的 onclick 屬性，進入對應函式，逐行讀取「取了什麼值→做了什麼→輸出到哪」。
04
描述問題給 AI，不要貼整段程式碼。告訴 AI：「第 47 行的 generate 函式，它取了 input 的值但沒有做 trim()，導致輸入全空白時也能執行，請修正這個邏輯。」比「幫我修 JS」有效得多。
05
改完立刻在瀏覽器測試。存檔後重新載入頁面（F5），逐一測試正常路徑和邊界情況（空輸入、超長輸入、特殊字元）。有問題繼續描述給 AI，不要一次要求 AI 猜測所有可能問題。

只是順便讓電腦執行。」

本節核心帶走：HTML 管結構、CSS 管外觀、JS 管行為——遇到問題先判斷是哪一層的問題，再精確告訴 AI 哪個元素、哪個屬性、期望是什麼、實際是什麼。這個能力會讓你的 AI 前端協作效率提高 3–5 倍。

## 問題清單
- 用 .innerText 讀 input：input 的值要用 .value，.innerText 對 input 永遠回空字串。後果：years / leave / days 全部是 ''，後續計算都是 NaN 或 undefined
- 字串 vs 數字比較：days > 3 時 days 從 input 來是字串，'10' > '3' 看似對，但 '2' > '10' 字串比較為 true。後果：天數判斷在邊界出錯
- 缺空值檢查：使用者沒填欄位直接按查詢，結果區出現 undefined 或 NaN。後果：UI 顯示破窗
- alert 跳窗：手機體驗差、無法統一樣式、blocking。後果：使用者要按確定才能繼續，且樣式不能客製
- var 全域變數：years / leave / days 寫在 function 外是全域。後果：頁面有多個工具時相互覆蓋、未來 minify 容易撞名
- quota = years * 7 缺 parseInt：'3' * 7 = 21（恰好對），但 'abc' * 7 = NaN。後果：使用者亂填會 NaN
- innerHTML 寫變數：result 雖是固定字串還行，未來加上使用者輸入會 XSS。後果：未來擴充必須先重寫
- 沒處理特休不足：rest 可能是負數，顯示「你還剩 -3 天特休」。後果：使用者看不懂，且邏輯應該是「特休不足，缺 3 天」

## 修正指令
請重寫這段 JS，保留 check() 函式名稱與三個請假規則邏輯。具體修正：(1) 用 .value 讀 input/select，不是 .innerText（2) 用 parseInt(..., 10) 把 years 與 days 轉成數字，並做 isNaN 檢查（3) 加空值檢查：年資、天數任一空白 → 顯示「請填寫完整資訊」到 #result-main（不要 alert）（4) 全部錯誤訊息寫到 #result-main 用 .textContent，不要用 innerHTML、不要 alert（5) years / leave / days 改用 const / let 放 function 內（6) 加負數防呆：天數或年資 < 0 → 顯示「請輸入正數」（7) 特休 rest < 0 時顯示「特休不足，缺 X 天」而非「你還剩 -3 天」。請只輸出修正後的 JS。

對照重點
1. .innerText 對 input 永遠是空字串——這是初學者最常踩的雷，學員若沒抓到要回頭實際在瀏覽器試一次
2. 字串比較行為（'2' > '10' = true）要實際試一次，光講學員不會記得
3. textContent vs innerHTML 差別要強調「除非真的要塞 HTML tag 否則永遠用 textContent」
4. 修正指令要明寫「保留 check() 函式名稱」，避免 AI 順手改名
