站立會議發言抽籤器

晨會從此不無聊。輸入團隊名單，按下抽籤，下一位報告者隨機揭曉——直到所有人都發言完畢。

PRACTICE GOALS
練習陣列隨機抽取與狀態管理
理解 CSS 動畫搭配 JavaScript 觸發的互動效果
產出一個可立刻在晨會上使用的趣味工具
TOOL · 互動工具

輸入成員名單，隨機抽出下一位發言者

在文字框中輸入團隊成員名單（每行一位），按「開始抽籤」初始化，再按「Next」逐一隨機抽出。抽過的成員會以刪除線標示，直到全員發言完畢。

打開 Gemini，切換到「Canvas」模式
進入 gemini.google.com，點右上角切換到 Canvas 模式，這樣 AI 生成的 HTML 會直接可以預覽。
2
貼入以下指令
把下方指令完整複製，貼入 Gemini 的輸入框，直接送出。
3
等待生成，預覽並存檔
Gemini 會生成一個完整的 HTML 工具。點「預覽」確認功能正常後，點「下載」或「複製」儲存到你的電腦。
→ 複製這段 Prompt 貼入 Gemini
你是一位擅長製作互動工具的前端工程師。 請製作一個「站立會議發言抽籤器」的單頁 HTML 工具，功能如下： 【功能需求】 - 一個文字框讓使用者輸入團隊成員名單（每行一位） - 「開始抽籤」按鈕初始化名單 - 頁面中央顯示一個大型「Next」按鈕，每次按下隨機抽出一位成員名字 - 抽出的名字以大字體顯示，並有彈入動畫效果（CSS animation） - 已抽到的成員名字會以刪除線標籤顯示在下方 - 所有人發言完畢後按鈕自動 disabled，顯示完成訊息 【設計要求】 - 背景 #f5f3ee，主按鈕用 #b5703a，字體 Google Fonts（Shippori Mincho + Noto Sans TC） - 名字顯示區塊使用 2rem 以上的大字型，白色卡片背景 - 圓角 6px，無陰影漸層 請生成完整的單一 HTML 檔案。
CHALLENGE · 進階挑戰

完成基本練習後，試試這些變化

PRACTICE VARIATIONS
01
加入「計時模式」：每位成員發言時啟動 2 分鐘倒數，時間到自動以聲音或視覺提醒。
02
讓工具支援「子團隊」模式：可設定兩個以上的小組，抽籤時顯示組別標籤。
03
加入「今日主持人」功能：在名單之外另外抽一位「今日會議主持人」，結果用不同顏色標示。

延伸案例

抽籤器工具搞定之後，帶回真實 Scrum 團隊的情境修一下規則。重點在「讓誰先發言」背後的隱藏議程——是要避免同一人先講、還是讓卡住的人早點被看見。

案例 07 · Scrum Master
每週站立會議 Tim 都第一個發言，其他人變成點頭附議
難度 ★★☆ 約 30 分鐘

你是 8 人工程團隊的 Scrum Master，每天 10:00 站立會議 15 分鐘。問題很明顯：資深工程師 Tim 永遠第一個發言，他講完其他人就順著他的議程補充，新來的兩位 Junior 幾乎沒獨立意見。你想導入抽籤器，但不能讓 Tim 覺得被針對。

原始資料｜團隊清單與近期觀察
【團隊成員】 - Tim（資深，5 年） - Luna（中階，3 年） - Max（中階，2 年） - Ken（中階，2 年） - Ivy（法務支援，半職） - Ava（Junior，進來 2 個月） - Leo（Junior，進來 1 個月） - 你（Scrum Master） 【近兩週觀察】 - 10 場會議 Tim 先發言 9 次 - Ava 主動發言 0 次，都是被問才回 - Leo 發言 3 次，都是跟 Tim 補充細節 - Ivy 半職，每週只來兩次 - 上週 Leo 私訊你：「我想講但不敢打斷 Tim」
你的任務
  1. 設計抽籤規則（隨機？加權？排除某些人？）並寫出設計理由
  2. 寫一段 150 字內的 Slack 公告，告訴團隊要開始用抽籤器，不能讓 Tim 尷尬
  3. 想一個「抽到 Ivy 但她今天沒來」的降級方案，寫進規則
評估指標
規則清楚
隨機 / 加權 / 排除半職，選了哪種都能說明理由
處理 Ivy
半職同事納入規則，不會抽到她缺席還要等十秒尷尬
顧及 Tim
公告不會讓 Tim 覺得被針對，同時明確表達要改變發言節奏
支援 Junior
規則或公告裡有提到「先抽到的人可以說『我今天沒太多進度』，不用勉強」
參考 Prompt（建議先自己寫再看）
點擊展開參考答案
【角色】你是 Scrum Master，懂團隊動力，會看「誰沒說話」比「誰說了什麼」更重要。 【任務】根據下方團隊清單與觀察，產出： 1. 抽籤規則（演算法層面）：誰在池子裡、權重怎麼設、Ivy 怎麼處理 2. Slack 公告（≤150 字）：告訴團隊要開始用抽籤器，不點名 Tim 但明確表達「讓發言順序多樣化」 3. 降級方案：抽到缺席者、抽到連續兩天同一人、抽到新進沒東西說該怎麼辦 【限制】 - 不要在公告裡寫「因為 Tim 每次先發言」這類會讓人不舒服的話 - Ivy 半職應該排除在每日抽籤外（但她到的那天納入） - Junior 剛來兩個月，設計規則時要考慮「抽到也可以說沒進度」 - 不要建議改會議時間或砍掉站立會議（題目就是要保留） 【輸出格式】 三段：規則描述、Slack 公告、降級方案。
變體挑戰
A一週後 Tim 私下說「其實我也不想每次第一個講」，調整規則讓他知道自己沒被邊緣化
B公司遠距團隊，加一條「抽到的人要同步在 Slack thread 寫下三點」，規則要怎麼改
C把抽籤器換成「誰卡最久誰先講」的反向規則，寫出利弊比較
