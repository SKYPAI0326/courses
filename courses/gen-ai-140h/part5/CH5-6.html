Part 5 · 前端與部署實戰
CH 5-6 · 1.5h 課堂 / 1h 自學

強迫多檔變單檔 —— 請 AI 幫我打包

AI 給你四個檔案、版面全垮？不要學 git 多檔 push，改用一段提示詞叫 AI 把多檔打包成單一 HTML，再回頭走 CH5-5 五步。全程不開 Terminal、不學 git。

學習成果 · OUTCOMES
能用「整合單檔」提示詞，把 AI 多檔產出（index.html + style.css + script.js）重做成單一 HTML
能看到 ./ 開頭路徑就判斷「這是多檔」，知道要先整合
整合失敗時能走備援 A（拖拉 ZIP）或備援 B（一檔一檔貼），不會卡死
整合後回 CH5-5 五步完成部署，拿到可分享的 GitHub Pages 網址
PART 5 · 本節定位
這是 CH5-5 的前置救援單元。AI 給你多檔 → 先跑這節整合成單檔 → 再回 CH5-5 五步部署。全程傻瓜路徑：貼一段提示詞、拿回一個檔案。
(01)

版面全垮？不是你的問題，是多檔問題

設計助理小薇的遭遇
設計助理小薇用 Gemini Canvas 做了一個「3 頁個人作品集」，畫面超漂亮。她照 CH5-5 五步部署，結果打開網址——版面全垮、圖片全破、按鈕沒反應。她沒做錯任何一步。問題是 Gemini 把網站做成了四個檔案，她只貼了 index.html。
Gemini 給了什麼 小薇有沒有貼 結果
index.html 有 頁面能打開，但
style.css 沒有 版面全垮
script.js 沒有 按鈕沒反應
assets/me.jpg 沒有 圖片全破
不要學怎麼傳多檔，要學怎麼叫 AI 把多檔變回一檔。

這 1.5 小時教你另一條路：用一段「整合單檔」提示詞，叫 AI 把四個檔案重新打包成一個 index.html，你再回頭走 CH5-5 的五步。改變 AI 產出的形態，比學 git 通用一萬倍——這是元能力。

→
最低完成線（傻瓜路徑，人人能過）：只要把多檔產出貼上整合提示詞、拿到「一段就是全部」的單檔，就算過。真的部署上 GitHub 是加分——沒環境的同學交「整合前 vs 整合後」對照截圖也算完成。
(02)

為什麼 AI 會給你多檔？

先理解「多檔不是 bug，是 AI 在模仿真實工程習慣」。

來源 通常 為什麼
Claude Artifacts 多半單檔 設計成「一個 artifact 一個檔」
ChatGPT Canvas 多半單檔 同上
Gemini Canvas 常多檔 模仿工程師「CSS/JS 分開」的習慣
Google AI Studio 常多檔 同上，還常多一層 assets/
怎麼一眼看出「這是多檔」？

不用看懂程式碼，只要找 ./：

多檔訊號 01
程式碼裡有 <link rel="stylesheet" href="./style.css">
CSS 被分開了，沒有 style.css 版面會垮
多檔訊號 02
程式碼裡有 <script src="./script.js">
JS 被分開了，沒有 script.js 互動全失效
多檔訊號 03
程式碼裡有 <img src="me.jpg"> 或 ./assets/...
圖片用本地路徑，沒有圖片檔就全破圖
💡
看到任何 ./ 或沒有 https:// 開頭的本地檔名 = 「該整合了」。反之，https:// 開頭的（公開 CDN）→ 不算，那是單檔 OK 的。不用看懂程式碼，看到 ./ 就動手。
(03)

核心：整合單檔提示詞拆解 + 現場示範（30 分鐘）

這段是本單元核心。講師現場示範一次，學員現場跟做一次。

整合單檔提示詞範本
課程素材包有「複製」按鈕，按了就進剪貼簿。
整合單檔提示詞範本（直接複製貼給 AI）
請把你剛剛做的這個網站，重新整理成單一 HTML 檔的版本： - 所有 CSS 寫進 <style> 標籤 - 所有 JavaScript 寫進 <script> 標籤 - 圖片用 https://placehold.co/ 或 https://picsum.photos/ 連結代替本地檔案 - 字型用 Google Fonts 的 CDN 連結，不要下載到本地 - 如果有多頁（about / contact 等），改成單頁內用錨點切換 我要可以複製整段就存成 index.html 直接用的版本。請開始重做。

逐句為什麼（給學員一句話理解，不用深究）：

那句要求 解決什麼
CSS 寫進 <style> 不用再傳 style.css
JS 寫進 <script> 不用再傳 script.js
圖片用線上連結 不用再傳 assets 圖片
字型用 CDN 不用下載字型檔
多頁變單頁錨點 不用傳 about.html / contact.html

一句話：把所有散出去的東西全部塞回一個檔。

現場示範動線（講師做給學員看）
步驟 1 — 用 Gemini Canvas 做一個多檔網站
範例指令：「做一個 2 頁的咖啡店網站，共用 CSS」
Gemini 給了什麼

Canvas 裡顯示 index.html，但同時也有 style.css（多檔）。

程式碼裡可以看到 <link href="./style.css">——這是多檔訊號。

步驟 2 — 貼上整合單檔提示詞
動作：在同一個 Gemini 對話框，直接貼上上方的整合單檔提示詞
Gemini 重新給了什麼

一個全包的 index.html，裡面有 <style>...全部 CSS...</style>，沒有 <link href="./style.css">。

程式碼裡沒有任何 ./ 開頭的本地路徑。

如果還是看到 ./style.css，直接再貼一次：「你還留著 style.css 連結，請把那個 CSS 內容也直接內嵌進 <style>」
步驟 3 — 複製整段 → 回 CH5-5 五步部署
動作：複製 Gemini 給的整段程式碼 → 走 CH5-5 Step 1~5 部署到 GitHub Pages
打開網址後：版面正常、圖片用 placehold.co 佔位、按鈕有反應。完成。
走完這一輪：你不需要懂 CSS 分開是什麼意思，你只需要知道「看到 ./ 就貼提示詞」。
(04)

卡關地圖：兩條備援逃生路徑 + 常見整合失敗（15 分鐘）

整合提示詞大多數時候會成功。但偶爾 AI 整合不乾淨，這時不要硬修，走備援。

備援 A —— GitHub 網頁 Upload files（拖拉 ZIP，3 步）
1
把 AI 產出的整個資料夾，在電腦上壓成 ZIP（或直接選全部檔案）
2
進你的 repo 頁面 → 點「Add file」→「Upload files」
3
把檔案/資料夾整個拖進去 → 下面打一行字 → Commit changes
不需要 Terminal、不需要 github.dev、不需要懂 git。拖進去就好。
備援 B —— 請 AI 一檔一檔給我（github.dev 手建）

當你想保留多檔結構，回 AI 丟這句：

貼給 AI 的指令
請列出這個專案的所有檔案，每個檔案前面用 markdown ``` 包住檔名、後面接整段內容。我會一個一個建檔貼上去。
拿到後在 github.dev 按「+」一個一個建檔貼進去（沿用 CH5-5 Step 3 的建檔手法）。
⚠
鐵律（沿用 CH5-5）：任何卡關超過 5 分鐘 → 不要 debug、直接走備援 A 拖拉 ZIP。把東西弄上線比弄漂亮重要。
三個常見整合失敗 + 一句話修法
失敗 01
整合後圖片破圖（placehold.co 404）
症狀：部署後圖片顯示錯誤，placehold.co 連結沒圖。
修法（回貼給 AI）：「請把圖片連結改用 https://picsum.photos/600/400」
失敗 02
整合後還是有 <link href="./style.css">
症狀：AI 整合了，但程式碼裡還是有本地路徑。
修法（回貼給 AI）：「你還留著 style.css 連結，請把那個 CSS 內容也直接內嵌進 <style>」
失敗 03
多頁變單頁後切換很突兀
症狀：錨點切換頁面時直接跳，沒有過渡效果。
修法（回貼給 AI）：「請幫切換加上 CSS transition 讓它平滑」
(05)

動手做：你來跑一遍（30 分鐘）

課堂練習
30 分鐘
用 Gemini Canvas 做多檔、整合成單檔、走 CH5-5 部署
用 Gemini Canvas（或 AI Studio）做一個至少 2 檔的小網站，用整合單檔提示詞壓成 1 檔，走 CH5-5 部署。
1
打開 Gemini Canvas，輸入「做一個 2 頁的[你想要的]網站，CSS 和 JS 分開寫」，讓它故意產多檔
2
確認看到 ./style.css 或 ./script.js（多檔訊號），然後貼上整合單檔提示詞
3
拿到整合後的單檔，確認沒有任何 ./ 開頭的本地路徑
4
（加分）走 CH5-5 五步部署，打開網址確認版面 / 互動正常
5
（備援）若整合反覆失敗 → 改走備援 A 拖拉 ZIP，並寫一句「我為什麼改走備援」

(07)

自學包（1 小時）

回家練兩個任務，加深「看到多檔 → 整合提示詞 → 單檔 → 部署」這個反射動作。

自學任務
任務 A 請 Gemini Canvas 做一個「3 頁餐廳網站」（首頁 / 菜單 / 訂位），用整合單檔提示詞壓成 1 檔並部署。完成標準：拿到一個能打開、三區塊用錨點切換正常的 GitHub Pages 網址。
任務 B 故意叫 AI「用 style.css 分開寫」做一個網站，再用整合提示詞救回單檔，體會前後差別。
💡
自學遇到卡關：套「除錯回貼三段式」（CH5-5 教過）貼回 AI。不要 Google（Google 出來都是給工程師的 git 教學，會更亂）。
(08)

講師端 SOP（學員略過）

開課前準備
1
先用 Gemini Canvas 親手產一個多檔網站、跑一次整合提示詞，錄成示範影片
2
準備一個「故意多檔」的範例（含 style.css + script.js + 一張本地圖）當現場示範素材
3
確認課程素材包的「整合單檔提示詞」複製按鈕可用
助教分工（課中）
A
助教 A：守「整合提示詞貼了但 AI 還是給多檔」→ 教追問「請把 style.css 內容也內嵌」
B
助教 B：守「整合成功但部署破圖」→ 教換 picsum.photos
C
助教 C：守「乾脆走備援 A」→ 示範 Upload files 拖拉
講
講師：處理「ZIP 也傳不上去」這種需要決策的個案
課中觀察重點
1
學員最常卡：整合後 AI 仍留一行 <link href="./style.css">（要教追問內嵌）
2
第二常卡：placehold.co 偶發 404（直接教換 picsum.photos）
3
約 20% 學員整合反覆失敗 → 講師當場示範備援 A 拖拉 ZIP，強調「上線比漂亮重要」
課後：收集學員「整合前多檔 vs 整合後單檔」對照，作下次教學示範素材。
