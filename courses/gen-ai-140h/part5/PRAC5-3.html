個人作品展示首頁

把你在這門課做的所有工具集中到一個首頁。這不只是個人網站的起點，更是你未來求職、接案、分享時最有力的一張名片。

產出目標 · DELIVERABLE
收納至少 3 個工具的個人作品展示首頁（index.html）
每個工具有標題、一句話說明和連結
首頁已部署，有公開網址
前提：你已完成 PRAC5-2 並取得至少一個已部署、有公開網址的工具頁。本頁的任務是把你做過的工具集中到一個作品展示首頁。
GUIDED · 45 分鐘照著做主線

先跟著做一遍：用 AI 生成一個「個人作品展示首頁」

這頁的完整任務是把你做過的工具集中到一個首頁。但這件事最容易卡住的，不是把首頁做漂亮，而是把首頁該有的東西描述清楚，再驗證 AI 真的把每一塊都做出來——作品首頁的本質就是「頭像自介 → 一排作品卡片 → 點得進去的連結」，少一塊、或連結點不開，這張名片就遞不出手。這條主線只做一件事：拿一份已經寫好的首頁規格，用一套固定 Prompt 叫 AI 生成單檔 HTML 首頁，存下來打開，再用「該有的功能逐項對照」檢查它有沒有做齊。全程貼文字進 AI、存成 .html 開來測，不用寫一行程式，45 分鐘做完，你就握住了「用 AI 生作品首頁」最關鍵的那一段。先把這條主線走完；做完還想多練，再往下走「進階練習池」——那裡有把工具資訊累積成 HTML 卡片的互動生成器、三段交付清單、延伸案例與自我演練。主線和練習池是同一主題的不同深度，主線是正式入門路徑，練習池是自選加練。

照著做主線 · STEP 1 / 5 · 場景設定（5 分鐘）

柏宇的那一張名片

柏宇，31 歲，做了三年行政、正在轉職的人。這幾週他用 AI 拼了幾個單檔小工具——一個會議記錄整理器、一個費用試算表、一個請假天數計算器，都能用，也都丟在資料夾裡。問題是面試時對方說「給我看你的東西」，他只能一個一個傳檔案，對方根本看不出他到底會什麼。他想要一個首頁：打開就看到他是誰、做過哪些工具，每個工具點一下就能試用。可是他不會寫程式，也不知道一個「拿得出手的首頁」到底該長什麼樣。

今天你就是柏宇。你要用 AI 把這個個人作品展示首頁生出來——重點不是做得多炫，是把首頁該有的每一塊描述精確，再驗證 AI 真的全做出來、連結真的點得開。本主線的首頁規格就在 STEP 2 下方（一份完整的版面與內容清單，直接用即可，不需自己設計）。

照著做主線 · STEP 2 / 5 · 看成品（10 分鐘）

先看這個首頁做完長什麼樣

動手前先看一次。網頁不像數字結果有「唯一正解」，但有「該具備的功能」。下面是這個個人作品展示首頁做完後該有的成品描述——記住這份清單，STEP 5 你會拿它逐項對照自己生出來的首頁。

這個首頁做完後，用瀏覽器打開你會看到：

【頁首區】一個圓形頭像佔位（純色色塊即可，不必放真圖）、底下一行姓名「林柏宇」、再一行自介「轉職中的行政人 · 用 AI 做小工具解決辦公室的麻煩」。

【作品卡片區】3 張並排的作品卡片（手機上自動疊成單欄），每張卡片由上而下有：作品標題、一句話說明（講用途、不是技術）、一個「開啟工具」連結。三張內容固定為——① 會議記錄整理器／把逐字稿貼進去自動整理成決議與待辦清單／meeting-tool.html；② 費用快速試算表／輸入品項與金額即時算出含稅總額／expense-tool.html；③ 請假天數計算器／選起訖日期自動扣掉週末算出實際請假天數／leave-tool.html。卡片滑鼠移上去（hover）時邊框顏色變深。

【頁尾區】一行小灰字「© 2026 林柏宇 · 用生成式 AI 打造」。

【外觀】單欄置中、最大寬度約 760px、米白底 #f5f3ee、卡片白底圓角、連結強調色霧藍 #6b7fa3。手機開起來不會超出螢幕。

【全部塞在一個 .html 檔】CSS 寫在同一個檔案內，存成單一 .html 雙擊就能開，不依賴任何外部檔案或網路。

看懂這份描述的關鍵：一個「拿得出手」的作品首頁要有頁首、作品卡片、頁尾三段，作品卡片必須是「標題＋一句話用途＋可點連結」三件齊全——少了連結，對方就點不進去試用；說明寫成技術名詞，對方就看不懂你解決了什麼。STEP 5 你會拿這份清單逐張卡片、逐個連結檢查 AI 有沒有做齊。

照著做主線 · STEP 3 / 5 · 主版工具生成 Prompt（5 分鐘）

主版生成 Prompt（直接複製，不用改）

這套 Prompt 把 STEP 2 的首頁規格寫成 AI 看得懂的生成指令，由 4 個骨架組成：角色、頁面規格（頁首／作品卡片／頁尾／外觀）、內容資料、輸出形式。你不用自填，直接按「複製」就能用。先看完整版，再看下方「為何這樣寫」。

主版 Prompt · 生成個人作品展示首頁（骨架：角色＋頁面規格＋內容資料＋輸出形式）
你是熟悉 HTML 頁面製作的前端助理。請把下方規格做成一個「單一檔案 HTML 頁面」，所有 CSS 都寫在同一個 .html 檔內，我存檔後雙擊就能用。 【頁面名稱】個人作品展示首頁 【頁首區】 - 一個圓形頭像佔位，用純色色塊即可，不必放真圖 - 頭像下方一行姓名：林柏宇 - 姓名下方一行自介：轉職中的行政人 · 用 AI 做小工具解決辦公室的麻煩 【作品卡片區】3 張並排的作品卡片（手機畫面自動疊成單欄），每張卡片由上而下有：作品標題、一句話說明、一個「開啟工具」連結。三張卡片內容依序為： 1. 標題：會議記錄整理器｜說明：把逐字稿貼進去，自動整理成決議事項與待辦清單｜連結：meeting-tool.html 2. 標題：費用快速試算表｜說明：輸入品項與金額，即時算出含稅總額｜連結：expense-tool.html 3. 標題：請假天數計算器｜說明：選起訖日期，自動扣掉週末算出實際請假天數｜連結：leave-tool.html 每張卡片滑鼠移上去（hover）時邊框顏色變深一點。 【頁尾區】一行小灰字：© 2026 林柏宇 · 用生成式 AI 打造 【外觀】單欄置中，內容最大寬度約 760px，背景米白 #f5f3ee，卡片白底、圓角，連結強調色霧藍 #6b7fa3。手機開啟時內容不可超出螢幕。 請直接回傳完整的單檔 HTML 程式碼，不要附任何說明文字或檢查報告。
為何這樣寫：
· 角色讓 AI 進入「前端助理」視角，並明確要「單一檔案、雙擊可用」——少了這句，AI 常把 CSS 拆成多檔，你存一個 .html 打開卻是空白。
· 頁面規格拆成四段（頁首／作品卡片／頁尾／外觀）、三張卡片內容逐條寫死是這套 Prompt 的核心：作品首頁錯都錯在漏掉卡片、漏掉連結，把每張卡片的標題、說明、連結列清楚，AI 才不會少做一張或忘了連結。描述「要有什麼」，不講「用 flex 還是 grid」，讓 AI 決定怎麼實作。
· 結尾「只回傳完整單檔 HTML、不要附說明或檢查報告」很關鍵：產出是要存檔的程式碼、不是資料。少了這句，AI 常在程式碼後面附一段說明，你貼進 .html 會把說明文字也存進去、頁面就壞了。驗證改用更可靠的方式——STEP 5 你打開生成的首頁，逐張卡片、逐個連結實際對照。
照著做主線 · STEP 4 / 5 · 換你跑（20 分鐘）

跟著這 5 步生出首頁並實際用一次

1
複製主版 Prompt
按 STEP 3 主版 Prompt 框裡的「複製主版 Prompt」鈕，整段指令就到剪貼簿了，裡面的首頁規格與三張卡片內容都寫好了，不用改。
2
貼進 AI 工具送出
ChatGPT、Gemini、Claude.ai 的免費版都可以，開一個新對話視窗，把 Prompt 貼上去送出。AI 會回一整段 HTML。
3
存成 .html 檔
把 AI 給的 HTML 整段複製，貼到記事本／文字編輯器，存檔時把檔名取成 portfolio.html（副檔名一定要 .html，不是 .txt）。
4
雙擊打開、實際看一次
雙擊 portfolio.html 用瀏覽器開，確認有頭像、姓名自介、3 張作品卡片、頁尾，滑鼠移到卡片上邊框會變深。
5
逐項對照成品描述
用 STEP 5 的工具功能對照檢查表逐項驗。對不上的，回 STEP 4 把缺的功能寫成一句話追問 AI 重生一次，不要自己動手改程式碼。

免費版提醒：這份 Prompt 約 450 字，所有主流免費版 AI 都放得下，全程貼文字、不必上傳檔案。若 AI 一次沒把 HTML 給完整（被截斷），直接回「請接續上面繼續輸出剩下的程式碼」即可。生成的首頁是純單檔 HTML，存下來就能離線開、要分享也只要傳一個檔。

照著做主線 · STEP 5 / 5 · 工具功能對照檢查表（5 分鐘）

離開前對照功能檢查表打 4 個勾

網頁不像數字結果有唯一正解，但有「該具備的功能」。拿 STEP 2 的成品描述，逐項對照你生出來的 portfolio.html。下面 4 項全部確認過，今天的主線才算完成。對不上的，回 STEP 4 把缺的功能寫成一句話追問 AI 重生一次。

工具功能對照檢查表

4 個勾全打完，你已經用 AI 做出一個能拿出手的個人作品展示首頁了。想再多練——往下走「進階練習池」：那裡有把工具資訊累積成 HTML 卡片的互動生成器、三段交付清單、延伸案例與自我演練，把今天這套「描述規格 → 生成 → 逐項對照」的方法套到你自己真正要投的那份作品首頁上。

PRACTICE POOL · 進階練習池

完成主線後的自選加練區

延伸加練 · Part 6 預習（選做）
以下三段交付清單屬於「API fetch 入門」，是 Part 6 的前置練習，和上方「作品首頁」是兩個獨立任務。若課堂時間不足，作品首頁完成即可結束；有興趣繼續的，接著走下方三段。

三段交付清單（每 30 分鐘一段）—— API fetch 入門

第 1 段（30 min）：找一個免費 API + 讀文件
選一個有公開文件的免費 API，讀懂它的基本使用方式：
  1. 推薦選項：Open-Meteo（天氣）/ REST Countries（國家資料）/ JokeAPI（笑話）/ Dog CEO（隨機狗圖）
  2. 找到 endpoint URL 和 response 的 JSON 結構
  3. 用瀏覽器直接貼 URL，確認能看到 JSON 資料
第 2 段（30 min）：寫第一個 fetch + 顯示資料
用 AI 幫你生成包含 fetch 的 HTML，讓資料顯示在頁面上：
  1. 告訴 AI：你的 API endpoint、想顯示的欄位、顯示樣式（卡片/清單/數字）
  2. 生成單檔 HTML（含 fetch + DOM 操作）
  3. 在本機開啟，確認資料有抓到並顯示
第 3 段（30 min）：加錯誤處理 + loading 狀態
讓頁面在網路慢或 API 出錯時不要白屏或靜止：
  1. 加 loading 狀態（fetch 進行中顯示「載入中...」或 spinner）
  2. 加 catch：API 失敗時顯示友善錯誤訊息（不是讓 console 噴 error 就算了）
  3. 模擬錯誤：把 API URL 改成一個不存在的網址，確認錯誤訊息有顯示在頁面上

async function loadProjects() {

loadProjects();
解法參考（先寫自己的版本再展開）
原版的問題： 1. fetch() 本身不會在 HTTP 4xx/5xx 時 throw error——res.ok 是 false 但程式繼續跑，嘗試解析可能是 HTML 的錯誤頁面為 JSON，導致 JSON parse error 2. 沒有 try/catch——網路斷線時 fetch() 本身 throw，但沒有被捕捉，會變成 unhandled promise rejection（console 報錯，使用者看到白畫面） 3. 沒有 loading 狀態和 error 狀態的 UI 回饋——使用者不知道資料在載入中，也不知道失敗了 修正版： async function loadProjects() { const container = document.getElementById('projects-container'); // 顯示 loading 狀態 container.innerHTML = '<p class="loading">載入中…</p>'; try { const res = await fetch('/api/projects'); // 明確檢查 HTTP 狀態碼 if (!res.ok) { throw new Error(`API 回傳錯誤：${res.status} ${res.statusText}`); } const data = await res.json(); renderProjects(data); } catch (err) { // 網路失敗 or HTTP 錯誤 or JSON parse 失敗都會到這裡 console.error('loadProjects failed:', err); container.innerHTML = ` <p class="error">作品集載入失敗，請重新整理頁面。</p> <p class="error-detail">${err.message}</p> `; } } loadProjects(); 關鍵改動：加 if (!res.ok) throw（明確捕捉 HTTP 錯誤）；加 try/catch（捕捉網路錯誤和 JSON parse 錯誤）；loading/error 狀態更新 DOM（使用者有視覺回饋）；console.error 保留技術資訊（方便 debug），但 UI 顯示對使用者友善的訊息。

案例 03 · 自由工作者
轉職前端的第 90 天，作品集首頁要投 3 家公司
難度 ★★☆ 約 50 分鐘

你上班是行銷，晚上自學前端九十天，手上累積四個專案：一個電商商品頁 clone、一個個人記帳工具、一個幫朋友咖啡店做的菜單頁、一個本課程的 Gemini 工具。下週要投三家公司（兩家新創、一家數位代理商），HR 平均只會看作品集首頁十秒。你要決定放哪些、怎麼排。

原始資料｜四個專案簡述
1. 電商商品頁 clone（跟著 YouTube 教學做的，視覺像 Apple 官網，但邏輯都抄） 2. 個人記帳工具（純 localStorage，沒後端，自己每天在用，六個月紀錄） 3. 咖啡店菜單頁（真的上線了，店主有在用，週末平均 80 個訪客，Google Analytics 截圖有） 4. Gemini 匯率即時比對工具（本課程作業，整合真實 API，有 loading / error 處理） 目標職缺： - 新創 A：Junior 前端，要 React - 新創 B：全端，要 Node.js + 一點前端 - 代理商：接案型前端，要能獨立交付整頁
你的任務
  1. 判斷四個專案的取捨：放幾個、順序怎麼排、每個專案主打什麼賣點
  2. 為每個選上的專案寫一句「hook 標題」（不超過 20 字）與三個 bullet 點
  3. 寫首頁 hero 區的一句 tagline（告訴看的人「你是誰、會什麼、想做什麼」）
評估指標
取捨誠實
跟教學 clone 的不放第一，真實上線、自己有在用的往前推
賣點對應
不同職缺可能強調不同專案，對應 JD 關鍵字而不是自我感動
證據具體
用數字說話：80 個訪客、六個月資料、loading/error 都有處理
十秒可懂
hero tagline 一眼看出定位，不是「熱愛前端、持續精進」這種空話
參考 Prompt（建議先自己寫再看）
點擊展開參考答案
【角色】你是幫轉職工程師看作品集的資深前端，看過很多「塞滿 clone 專案的首頁」，會直接指出哪些該砍。 【任務】針對下方四個專案與三個目標職缺，做兩件事： 1. 排序取捨：哪幾個留、哪幾個砍、為什麼 2. 針對留下的每個專案，寫 hook 標題（≤20 字）+ 三個 bullet 點，bullet 用「動詞開頭 + 具體數字或技術」 最後輸出首頁 hero 區 tagline 三個版本：偏新創 A、偏新創 B、偏代理商。 【限制】 - 教學 clone 若沒有自己加東西就砍掉，不要為了湊四個留下來 - bullet 不要寫「使用 HTML/CSS/JS」這種廢話，要寫「解決了什麼問題」 - tagline 不准出現「熱愛」「充滿熱情」「持續學習」 - tagline 要說清楚「想要什麼職位 / 什麼類型的合作」 【輸出格式】 三段：取捨判斷、專案介紹卡片（標題 + 三 bullet）、tagline 三版。
