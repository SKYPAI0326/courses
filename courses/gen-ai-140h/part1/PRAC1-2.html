延伸案例

健診工具只能打分，真正的練習是拿一條自己寫過的爛 Prompt 開刀。挑一個下列案例，把健診五維度當作 checklist，把 v1 改到五項都拉滿。

案例 01 · 行銷
把一行糊 Prompt 改造成高分指令
難度 ★☆☆ 約 30 分鐘

你是手搖飲連鎖店的行銷專員，老闆半夜丟訊息說：「幫我寫個新品宣傳文，明早給我。」你第一版 Prompt 寫得很短，AI 生出的貼文平平無奇。現在用健診器把 v1 拆解，補齊缺的維度。

原始資料｜v1 Prompt（明顯有問題）
幫我寫一篇新品宣傳文，是芒果冰沙，好喝又消暑，請用親切的語氣，產出社群貼文。
你的任務
  1. 把 v1 貼入健診器，記下五個維度各自得幾分、被扣在哪
  2. 針對分數最低的兩個維度補資訊（產品規格、受眾、平台、字數、CTA 等）
  3. 寫出 v2 Prompt，再健診一次，目標讓最低分拉到 3 分以上
評估指標
角色明確
有指定職業、產業、語氣風格，不只是「幫我寫」
任務具體
寫什麼平台、什麼目的、給誰看都說清楚
限制完整
字數、禁用詞、必含元素（CTA/hashtag）都有規範
格式可用
輸出直接貼得上 IG/FB，不需要人工再編排
參考 Prompt（建議先自己寫再看）
點擊展開參考答案
【角色】你是手搖飲連鎖品牌的社群小編，熟悉 20–35 歲上班族女性的溝通語言，語氣輕鬆但不裝年輕。 【任務】為新品「泰式鹹芒果冰沙」撰寫一則 Instagram 貼文，主打夏日午後解膩、泰式風味差異化。上架日 6/15，首週買一送一。 【產品資訊】 - 售價：單杯 75 元，買一送一限午 2–4 點 - 特色：真實芒果果肉＋泰式辣椒鹽，鹹甜辣三層次 - 區隔：市面上芒果冰沙多走甜膩路線，我們主打鹹甜解膩 【限制】 - 貼文正文 120 字內，分 2–3 段 - 第一句要有鉤子，不要用「炎炎夏日」這類陳腔 - 含一句 CTA（導購到店或外送 App） - 最後列 5 個 hashtag，其中 2 個為品牌專屬 tag - 不用 emoji 塞滿整段，最多 2 個 【輸出格式】 直接輸出貼文內容＋hashtag，不需要任何前言或標題。
變體挑戰
A同樣產品改為 Facebook 貼文（受眾年齡上移到 35–50），觀察「角色 + 限制」如何連動改變
B產出三個不同語氣版本（輕鬆 / 文青 / 幽默），讓老闆挑一個
C把 v2 Prompt 改造成可重複套用的 template，下次換新品時只要改變數

案例 02 · 製造業
品管報告 Prompt 健診——抓出「格式不一致」的元凶
難度 ★★☆ 約 40 分鐘

你是電子零件廠的品管工程師，主管要你每週用 AI 整理客訴為標準週報。你用了兩週，發現 AI 每次產出的表格欄位都不一樣，主管沒辦法比對。這是典型「輸出格式」維度失分。

原始資料｜本週要處理的客訴原始紀錄
2026/04/12 A 客戶回報：第 3 批連接器插拔 50 次後外殼龜裂。樣本編號 A1203，嚴重度高。 2026/04/13 B 客戶（東莞廠）：插針偏移 0.3mm，影響組裝良率約 4%。批號 B0408。 2026/04/15 C 客戶反映端子氧化變色，外觀問題，未影響功能。單週 3 件。 2026/04/16 A 客戶第 4 批再次出現類似龜裂，但只有 1 件，懷疑運送問題。
你的任務
  1. 先寫 v1 Prompt（不看參考），把上週的實際 AI 輸出也貼入健診器一起看
  2. 在健診器找出「格式」「限制」維度被扣分的具體原因
  3. 寫 v2 Prompt 明確定義每個欄位的格式，要求連續兩週輸出欄位完全一致
評估指標
欄位固定
每列必填欄位（日期/客戶/批號/嚴重度/建議動作）明確
嚴重度量化
用 P1/P2/P3 或高/中/低，並定義判斷準則
可比對性
下週同格式再跑一次，可直接 diff 出差異
決策導向
最後一欄是具體動作，不是「持續觀察」這類空話
參考 Prompt（建議先自己寫再看）
點擊展開參考答案
【角色】你是連接器製造廠的品管工程師，熟悉 ISO 9001 客訴分類流程，輸出語言為繁體中文、以數據與動作為主、不寫空話。 【任務】把下方本週客訴原始紀錄整理成「標準週報表格」，供主管 5 分鐘內判讀並決策。 【原始紀錄】 （貼入紀錄） 【限制】 - 嚴重度只能用 P1 / P2 / P3 三級，判準如下： P1 = 功能失效或良率衝擊 >3% P2 = 良率衝擊 1–3% 或外觀明顯問題 P3 = 外觀瑕疵且無功能影響 - 同客戶同問題反覆出現要合併為一列，並在備註註明「本週 N 次」 - 建議動作欄禁用「持續觀察」「待確認」，必須是動詞開頭的具體行動（例如：抽驗 B0408 批 30 pcs、停用第 3 批模具等） - 欄位與順序固定為：日期 | 客戶 | 批號 | 問題描述 | 嚴重度 | 建議動作 | 負責人預設 【輸出格式】 直接輸出 Markdown 表格，表頭固定，不需要前言或結論。若本週無客訴則輸出「本週無新客訴」。
變體挑戰
A把輸出改為 JSON，直接能餵進公司內部的客訴 Dashboard
B新增「建議動作」需附預估工時與負責部門兩欄，並保持表格寬度一致
C讓 AI 自己比對上週表格（附檔）與本週表格，產生「變化摘要」段落
