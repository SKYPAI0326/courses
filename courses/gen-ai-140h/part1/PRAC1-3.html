STEP 1 · 確認你的需求

在要求 AI 生成工具之前，先回答三個問題：

1這個工具要解決什麼問題？（例：每次開會都超時，需要有人幫忙盯時間）
2使用者要輸入什麼、看到什麼？（例：輸入議程，看到倒數計時）
3有哪些限制或特殊需求？（例：不需要後端，純 HTML 單一檔案，支援手機瀏覽器）

案例 01 · 教育
補習班小老師的單字抽考工具
難度 ★☆☆ 約 40 分鐘

你是國中英語補習班的兼職助教，每週要抽考學生本週背的 30 個單字。目前用紙本隨機挑 10 題，學生常抱怨題目重複。你想要一個 HTML 頁面，丟入單字清單就能自動出題、計分、顯示錯的單字。

原始資料｜本週單字清單
apologize - 道歉 environment - 環境 decision - 決定 recognize - 認出 vegetable - 蔬菜 necessary - 必要的 experience - 經驗 immediately - 立刻 opportunity - 機會 explanation - 解釋 （省略其餘 20 字，結構相同）
你的任務
  1. 先列出工具需求：出題數、方向（中翻英/英翻中）、計分規則、錯題顯示方式
  2. 寫 Prompt 讓 AI 產出單檔 HTML（含 CSS + JS），可直接雙擊打開用
  3. 實測：貼入單字清單、做完一輪，確認計分與錯題顯示正常
評估指標
資料可替換
單字清單放在一個 array，下週換題只改那一段
隨機不重複
同一輪內題目不重複，且分布均勻
即時回饋
答完顯示分數 + 錯題清單，可直接給學生看
零依賴
單檔離線可跑，不需要網路或套件

【角色】你是熟悉前端教學情境的工程師，擅長用最少程式碼做出補習班老師能直接用的單檔工具。 【任務】產生一個單檔 HTML（含 CSS + JS 內嵌），功能為國中英語單字抽考。 【資料結構】 在檔案頂部宣告一個 words 陣列，每筆 {en, zh}，目前有 30 筆（我會自己替換內容）。 範例：const words = [{en:'apologize', zh:'道歉'}, ...]; 【功能】 1. 開始畫面：輸入題目數（預設 10）、方向選項（中翻英 / 英翻中 / 隨機混合），按「開始」 2. 作答畫面：一次顯示一題，文字輸入答案，Enter 送出或按按鈕。答對綠色 / 答錯紅色並顯示正解，2 秒後自動下一題 3. 結果畫面：顯示 X/10 分、列出所有錯題（題目｜你的答案｜正確答案），提供「再考一次」按鈕 【限制】 - 單一 .html 檔，雙擊可用，不需要網路 - 同一輪內題目不重複 - 英翻中容許答案的同義詞（可簡單以「答案包含正確翻譯」判斷，不用太嚴格） - 視覺風格簡潔、米白底，手機也能用 - 不引入任何外部函式庫 【輸出格式】 直接輸出完整 HTML 程式碼，從 開始，不需要任何說明文字。

案例 02 · 自由工作者

難度 ★★☆ 約 45 分鐘

你是 SOHO 平面設計師，每月月底要給 3–5 個客戶對帳。你每天用手機記事本隨手記「4/3 A 客戶 logo 修 1.5h」，月底要手動換算成 PDF 發票附件。你要一個工具：貼入流水帳文字、按一下就生出分客戶彙總＋可列印的對帳單。

原始資料｜一個月的流水帳

你的任務
  1. 拆需求：輸入格式（每行一筆）、時薪表、輸出內容、列印樣式
  2. 寫 Prompt 產出單檔 HTML，含貼文字區、時薪設定、彙總表、可列印樣式

金額正確

(05) · 延伸挑戰與檢核

把四個工具都做完，再加一點自己的味道

工具三要素跑過一次，剩下就是量的累積。把四個情境都跑一輪，你會發現 Prompt 寫法開始穩定：輸入、運算、輸出三段一貼，AI 就能給你堪用成品。之後的改造與優化，就是把「堪用」推到「你自己會想天天用」。

●
延伸挑戰（約 20 分鐘）
挑戰 1｜把四個工具都做出來。每個情境抓 15–20 分鐘：複製 Prompt、送給 AI、存檔、實測。四個做完你手上就有四個能用的小工具，也會知道哪一類需求 AI 一次就到位、哪一類要再補資訊。
挑戰 2｜改造成你自己的版本。挑一個情境換成你真實工作會用到的需求。例：把情境 B 的「課程報價」換成你家行業的報價公式（接案設計師算工時 × 時薪、補習班算月費 × 月數 × 堂數、物流算重量 × 距離 × 單價）。重點不是 Prompt 完全照抄，而是把「輸入什麼、算什麼、輸出什麼」三段換掉。
挑戰 3｜加一個優化。挑其中一個工具，加上以下任一項並再送一次 Prompt 讓 AI 改：
· 重置按鈕：一鍵把所有欄位與狀態回到初始值
· 深色模式：加一顆切換按鈕，切換 body class 對應深淺色 token
· 錯誤提示：輸入不合理時顯示紅字提示（不是用 alert，要就地顯示在欄位下方）
