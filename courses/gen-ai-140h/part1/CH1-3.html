Prompt 除錯：怎麼讓 AI 越改越準

第一次的輸出幾乎從來不是最好的。這節課教你診斷問題出在哪裡、用什麼節奏修正、以及如何要求 AI 自我重寫——讓每一輪對話都比上一輪更接近你要的結果。

學習成果 · OUTCOMES
能識別 5 種常見的 Prompt 問題類型並知道對應修法
掌握「先對、再好、再快」的多輪修正節奏
能要求 AI 做自我檢查、自我重寫、版本比較
RECAP · 上一節帶走的
四要素公式讓每個 Prompt 都有結構——角色、任務、限制、格式——不再憑感覺拼湊，能穩定拿到想要的品質。
PREVIEW · 這節解決什麼
第一次輸出很少是最好的。這節教你診斷問題出在哪裡、用什麼節奏修正、怎麼要求 AI 自我重寫，讓每輪對話都比上一輪更接近目標。
(01) · Bug Taxonomy

五種最常見的 Prompt 問題，以及怎麼修

當 AI 的輸出不對的時候，問題幾乎都出在提示詞的某一個面向。先學會診斷問題類型，才能精準修正——不是每次都需要重寫整個 Prompt。

太空泛
輸出太籠統，沒有具體資訊
修法：加上「請給出具體的數字、案例或步驟」，或用 few-shot 給範例。根本原因：任務太模糊，AI 只能給通則。
太冗長
輸出過長，摻雜不必要的廢話
修法：在 Prompt 結尾加「不超過 X 字」「只輸出核心內容，不需要前言和總結」。根本原因：沒有設長度限制。
格式錯
輸出格式不是你要的
修法：明確指定格式，如「以 Markdown 表格輸出」「輸出純 JSON，不含說明文字」。根本原因：格式要素缺失。
不準確
內容有錯誤或與事實不符
修法：加上「如果你不確定，請說不知道而不是猜測」「請標注你的資訊來源或信心程度」。根本原因：AI 幻覺，需要驗證節點。
不照角色
AI 沒有維持你設定的角色或語氣
修法：把角色描述移到 Prompt 最前面，並加上「請全程維持這個角色，不要跳出角色說明你的想法」。根本原因：角色約束不夠強。
Mini Practice · 迷你練習 01
約 10 分鐘
把三段失敗的 Prompt，分到五種病因

先能判斷病因，才知道改哪裡。這個練習不需要你動手寫，只要把下面三個案例對應到「太空泛 / 太冗長 / 格式錯 / 不準確 / 不照角色」其中一類。

1
案例 A：你輸入「幫我分析這個市場」，AI 回了一大段「市場通常可以從以下幾個面向分析⋯⋯」但沒給具體數字或結論。
2
案例 B：你設定 AI 扮演嚴謹律師，但問到第三題時 AI 突然開始說「個人覺得⋯⋯這其實蠻有趣的啦！」
3
案例 C：你要 JSON 輸出，AI 給了你 JSON，但前面還附贈一段「好的！這是你要的結果⋯⋯」導致 parse 失敗。
4
各寫一句修法（一句話即可，不必重寫整個 Prompt）。
應產出 · OUTPUT
一張診斷表：案例 A / B / C 的病因分類 + 一行修法建議
自我檢核 · CHECKLIST
REFLECTION · 你的答案
最近一次跟 AI 溝通失敗，你當時犯的是哪一種 Prompt 病因（太空泛 / 太冗長 / 格式錯 / 不準確 / 不照角色）？你後來怎麼修？
SOLO · 自我演練
把你的「診斷表」貼到 LLM，請它扮演「挑剔的技術主管」，從案例 A/B/C 挑一個，判斷你的修法是否真的對到病根；如果方向不對，要它寫出一句更準的改法（看 LLM 回覆是否對齊本節主題）。
5 分鐘
進階選配 · 不計入課程時數（給想加練的學員）
把 LLM 對話截圖存到學習資料夾，或貼到自己的工作群組／社群求 review。這些痕跡在轉職投履歷時可作為作品集素材。
(02) · Iteration Rhythm

多輪修正的節奏：先對，再好，再快

很多人拿到不滿意的輸出，直覺是重寫整個 Prompt。這通常不是最有效率的做法。建議用三階段節奏修正：

1
先對
確認方向、內容、事實正確
→
2
再好
改善語氣、格式、細節品質
→
3
再快
把成功的 Prompt 存起來複用

不要在第一輪就要求「正確且漂亮且精簡」。先讓 AI 把方向對了（第一輪），再要求品質（第二輪），最後把這個 Prompt 存成模板（第三步）。這樣的迭代比每次重來更有效率。

💡
關鍵心態：在 AI 時代，「修改提示詞」本身就是一種工作技能。不要覺得第一次沒成功是失敗——那只是第一輪草稿。
Mini Practice · 迷你練習 02
約 15 分鐘
用「先對 → 再好 → 再快」跑三輪修正

選一個真實工作任務，完整走一遍三階段節奏，親身體驗迭代的力量。這是本節最關鍵的練習。

1
選一個工作任務（如「改寫產品文案」），先寫一個最基本的 Prompt v1（約 1 行），跑一次。
2
先對：檢查方向和事實對不對，針對錯的地方補規則，寫成 v2 再跑一次。
3
再好：v2 跑出結果後，針對語氣 / 格式 / 細節補強，寫成 v3 再跑一次。
4
再快：把 v3 存到你的 Prompt 庫（Notion / Google Doc），下次同類任務直接複用。
範本素材 · Sample
這份範例完整走過三輪。任務：寫一封給客戶的提案開發信。依序把 v1 / v2 / v3 丟到 ChatGPT 跑，你會看到每輪修掉的是不同類型的毛病。
背景情境（跑每個版本前都需要這段）
你任職於某顧問公司企業訓練部門。客戶：某電子零件製造商，約 300 人規模，主要痛點是新進工程師養成太久（3–6 個月）。你已在上週電話初談中，了解對方有興趣但尚未拍板。現在要寫一封 email，目的是約下週的實地需求訪談。
v1 · 最原始版第 0 輪草稿
幫我寫一封給客戶的提案開發信，介紹我們的企業訓練服務。
v1 跑完常見問題：AI 產出一封通用行銷信，沒有客戶資訊、沒有痛點連結、沒有明確下一步。方向對了（是 email），但內容無差異化。
v2 · 先對（補上客戶資訊、痛點、訴求）第 1 輪修
你是某顧問公司企業訓練部的業務顧問。請幫我寫一封開發信給某電子零件製造商的 HR 主管。 客戶背景： - 約 300 人規模，主要痛點為新進工程師養成期長達 3–6 個月 - 上週已電話初談，對方表態有興趣但尚未拍板 - 尚未了解我們完整服務內容 email 目的：約下週實地需求訪談（不是直接推銷方案）。 請在信件中包含： 1. 點出「3–6 個月養成期」的具體成本概念 2. 簡述我們做過的類似案例類型（不具名） 3. 提出兩個訪談時段讓對方選 --- 請直接產出 email 全文。
v2 跑完改善點：方向和內容都對了，信件結構完整、有打中痛點。但可能出現：語氣偏死板、段落冗長、結尾制式（「敬祝商祺」等）、或開頭過於客套。
v3 · 再好（補上語氣、字數、格式限制）第 2 輪修
你是某顧問公司企業訓練部的業務顧問。請幫我寫一封開發信給某電子零件製造商的 HR 主管。 客戶背景： - 約 300 人規模，主要痛點為新進工程師養成期長達 3–6 個月 - 上週已電話初談，對方表態有興趣但尚未拍板 email 目的：約下週實地需求訪談。 內容要求： 1. 點出「3–6 個月養成期」的具體成本概念（用員薪水 × 養成月數換算） 2. 簡述類似案例類型（不具名） 3. 提出兩個訪談時段讓對方選（週三下午、週五上午擇一） 寫作限制： - 總字數 250–300 字（不含署名） - 口語化、每句不超過 25 字、避免文言 - 開頭不用「您好」「敬啟者」，直接帶情境（例如「上週與您通話後…」） - 結尾不用「敬祝商祺 / 順頌時祺」 - 禁用「賦能」「提升效能」「解決方案」這類業務套話 輸出格式： 主旨：一行（不超過 18 字） 內文：分 3 段，每段 3–5 句 署名：先 Wu（你）／顧問公司企業訓練部
v3 跑完差別：同樣內容但語氣自然、字數可控、開頭吸引閱讀（因為不落俗套）、結尾明確（兩個時段讓客戶秒選）。這版就是值得存進 Prompt 庫（Notion / Google Doc / Obsidian）的「再快」版本，下次開發信只要換客戶資訊與痛點即可。
三輪修改對照（迭代重點）
v1 → v2：補方向（角色、背景、痛點、訴求）— 解決「AI 不知道我是誰、客戶是誰」
v2 → v3：補品質（語氣、字數、格式、禁忌語）— 解決「內容對但讀起來很塑膠」
v3 → 存檔：變成模板（以後換客戶即可）— 解決「下次同類任務還要重新想」
應產出 · OUTPUT
一份三輪修改紀錄：v1 / v2 / v3 + 每輪改了什麼 + 最終儲存位置
自我檢核 · CHECKLIST
SOLO · 自我演練
把你的 v1、v2、v3 三版 Prompt 一次貼到 LLM，請它扮演「外部 reviewer」逐版比對，指出「三輪裡哪一輪的改動最關鍵、為什麼」。對照它的判斷與你自己的感受（看 LLM 回覆是否對齊本節主題）。
5 分鐘
進階選配 · 不計入課程時數（給想加練的學員）
把 LLM 對話截圖存到學習資料夾，或貼到自己的工作群組／社群求 review。這些痕跡在轉職投履歷時可作為作品集素材。
(03) · Version Control

建立 Prompt 版本意識

當你對同一個任務做了多輪修正，你需要記錄「改了什麼、為什麼改、結果有沒有變好」。這就是 Prompt 版本意識——它讓你的提示詞可以持續進步，而不是每次重來。

v1 初稿 幫我寫一份工作週報
輸出：一份通用的週報模板，格式不固定，語氣偏正式，沒有包含我的工作內容。
問題：任務太模糊，無角色，無格式，無限制。
v2 改善 你是行政助理，請根據以下工作紀錄，整理成週報格式，包含：本週完成事項（條列）、待確認事項（條列）、下週計畫（條列），語氣簡潔，每項不超過 30 字。
輸出：結構清楚的週報，三個區塊都在，長度剛好，可以直接寄出。
改進：加了角色、加了格式、加了三個區塊要求、加了長度限制。
📝
工具建議：用 Google Doc 或 Notion 建一個「Prompt 庫」，每個模板記錄：任務名稱、v1 → v2 的差異、最終版本。這是你在這門課最重要的學習產出之一。
(04) · Self-Review

讓 AI 自己檢查、自己重寫

你不一定要親自重寫 Prompt——你可以直接要求 AI 批評自己的輸出，然後重寫一個更好的版本。這個技巧特別適合你知道輸出有問題但說不清楚哪裡不對的情況。

1
自我檢查：「請檢查你剛才的輸出，有哪些地方不夠準確、不夠清楚、或不符合我的需求？列出 3 個問題。」
2
自我重寫：「根據你列出的問題，重新寫一個改善版本。」——這讓 AI 帶著批評意識重寫，品質通常比第一版高。
3
版本比較：「請給我版本 A 和版本 B，並說明你認為哪個更好及原因。」——讓 AI 給你多個選擇，再由你做最終決定。
4
一致性測試：把同一個 Prompt 發送 3 次，比較輸出是否穩定。如果每次結果差很多，說明 Prompt 還需要加更多限制條件。
✅
本節帶走產出：Prompt 除錯紀錄單——選一個你最常用的工作任務，做 3 輪 Prompt 優化，記錄每一輪的問題、修改點、輸出差異。這份紀錄就是你自己的 Prompt 除錯指南。
RESULT COMPARE · 三模型對打：壞掉的 prompt 誰修得最好？
【有問題的 prompt v1，直接貼給三個模型跑】 幫我寫一個關於我們產品的文章，要很好，要讓客戶喜歡， 不要太長，要有重點，謝謝。 【觀察問題後，把這個 v2 也分別貼給三個模型跑】 你是 B2B 軟體公司的內容行銷專員。請為我們的流程自動化產品 寫一篇 300 字的 LinkedIn 文章，目標是讓中小企業主點進官網。 文章結構：痛點開場（1 段）→ 解法（2 段）→ CTA（1 句）。
CHECKPOINT · 講師檢核
把你修壞掉 → 修好的 v1 / v2 prompt 對照表（含 3 輪修改說明）貼到協作板。
