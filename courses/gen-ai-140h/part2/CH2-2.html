CH 2-2 · 2.5H

文書工作流：Email、公告、週報、會議紀錄

把職場最耗時的四種文書任務，從「每次從零開始」變成「有 Prompt 模板的可重複工作流」。一次建好，終身受用。

學習成果 · OUTCOMES
能建立四種 Email 類型（確認/拒絕/請示/跟進）的可複用 Prompt 模板
能設計「會議記錄 → 待辦清單」的標準轉換流程
能用 AI 快速生成組織一致語氣的週報和公告
(01)

Email 的四種類型模板化

Email 是知識工作中最耗時、重複性最高的文書任務之一。大多數商業 Email 都可以歸類為四種類型，每種類型都有固定的結構和語氣模式。建立這四種 Prompt 模板，你可以把「回一封 Email 的時間」從 20 分鐘壓縮到 3 分鐘。

TYPE 01 · 確認
確認回覆
開場：感謝 + 確認收到 → 確認內容重點 → 說明後續步驟 → 結尾邀請
TYPE 02 · 拒絕
婉拒回覆
感謝提案 → 表達遺憾 → 說明原因（可省略）→ 保留未來合作可能 → 祝福結尾
TYPE 03 · 請示
請示/詢問回覆
說明背景 → 明確提問（不超過3個問題）→ 說明截止時間 → 說明影響範圍
TYPE 04 · 跟進
跟進提醒
引用前次溝通 → 說明目前狀態 → 具體的下一步請求 → 給出決策截止時間
Email · 通用模板 可複用的 Email Prompt 框架
你是我的商業郵件撰寫助理。請幫我撰寫一封【類型：確認/拒絕/請示/跟進】類型的 Email。 收件人：[姓名或職稱] 關係：[上下屬/客戶/合作夥伴/同事] 情境：[說明這封信的背景] 核心訊息：[你最想傳達的主旨] 語氣：[正式/親切/中性] 格式要求： - 主旨：15字以內，清楚說明信件目的 - 正文：200字以內 - 結尾：明確的下一步行動 - 不要使用「敬啟者」或「謹此敬告」等過時套語
把類型、收件人、情境填入，即可快速生成符合語境的 Email 初稿，只需微調個人化細節。
迷你練習 1 約 10 分鐘
把本週要寫的信套進 Email 模板

從你收件匣中挑一封還沒回的信，用本節的四類型（確認 / 拒絕 / 請示 / 跟進）模板 Prompt 產生初稿，比較自己從頭寫的時間差距。

1選一封你還沒回的信；判斷屬於四種類型中的哪一種。
2把類型、收件人、情境、核心訊息填入通用模板 Prompt，生成初稿。
3人工微調個人化細節（名字、引用前次對話內容），直到可以送出。
範本素材 收件匣沒題材？用這四個情境對應四類型，照著跑一次再改成自己的。
原始素材 · 四種來信情境（擇一使用）貼進 Prompt 最下方
【情境 A · 確認】某客戶問：「上週談的報價單 04/30 可以正式發出嗎？另外想確認是否需要我方先簽 NDA。」 【情境 B · 拒絕】某外部單位邀請主管 05/05（三）下午參加論壇分享，主管當日有既定客戶會議無法出席，需婉拒並表達未來合作可能。 【情境 C · 請示】主管想請財務同意把本月 Google Ads 預算從 8 萬提高到 12 萬，因 04/25 新品上市期流量需求大，預期 ROAS 維持 3.5 以上。 【情境 D · 跟進】04/10 已寄給某供應商的樣品需求單至今未回覆，希望對方於 04/22 前給出明確交期。
範例 Prompt · 通用四類型模板可直接貼 ChatGPT / Claude
你是一位協助我回信的行政助理。請依據以下參數，產出一封可以直接寄出的 Email： Email 類型：[確認 / 拒絕 / 請示 / 跟進] 擇一 收件人角色：[客戶 / 主管 / 同事 / 外部單位] 情境描述：[貼上上方情境片段] 必要元素： - 主旨 15 字內，看到即知目的 - 正文 200 字內，第一句先表態，第二段補理由或行動 - 結尾一句「下一步」（誰、做什麼、何時） 禁用： - 「您好，希望這封信找到您時一切安好」此類開場套話 - 「如有任何問題請不吝告知」此類結尾套話 輸出格式： 主旨：xxx 正文：xxx
預期輸出片段（情境 B 婉拒）
主旨：05/05 論壇邀請婉謝並期待下次合作
正文：感謝貴單位 05/05 論壇邀請，可惜當日已排定與重點客戶的定期會議，無法出席。本次題目與我方近期關注方向契合，若下半年（09 月以後）再辦類似活動，請優先讓我們知道，會盡可能排進行程。
下一步：下半年活動若已有規劃，請於 07/31 前告知時間與主題。
成果產出
一封可以直接寄出的 Email 回覆，並記下「AI 初稿 → 可送出」花了多少分鐘。

(02)

會議記錄的轉換邏輯

會議記錄最大的問題不是「怎麼記」，而是「記了沒有用」——因為記錄沒有轉換成行動。AI 最能幫上忙的地方，正是把雜亂的會議記錄或逐字稿，轉換成有負責人、有截止日期的結構化待辦清單。

1
輸入：原始記錄
逐字稿、手寫筆記截圖、或你自己的速記（不需要整理，直接貼入）
→
AI 處理：萃取結構
識別決策、待辦事項、後續問題、值得存檔的資訊
→
輸出：可用格式
結構化待辦清單（負責人 + 事項 + 截止日）+ 決議摘要 + 下次議題
會議記錄 逐字稿 → 結構化待辦
以下是一場會議的逐字稿（或筆記），請： 1. 列出所有決議事項（每項一行） 2. 列出所有待辦事項，格式：【負責人】任務內容 → 截止日期 3. 列出所有「待確認」或「下次討論」的問題 4. 生成一段 100 字以內的會議摘要，可直接轉寄給未出席者 如果某欄位沒有資訊，寫「本次無」。 逐字稿： [貼上逐字稿或筆記]
關鍵設計：強迫 AI 用你設定的格式輸出，讓結果可以直接貼入任務管理工具或轉寄 Email。
迷你練習 2 約 12 分鐘
把最近一場會議記錄轉成待辦清單

用最近一次會議的逐字稿或速記，跑一次「逐字稿 → 結構化待辦」Prompt，體驗從「記錄」到「可行動」的轉換。

1準備一份逐字稿或會議筆記（300–800 字即可，潦草沒關係）。
2套用本節「逐字稿 → 結構化待辦」Prompt，產出決議 / 待辦 / 待確認 / 摘要。
3把「待辦事項」直接貼到你的任務管理工具（Notion / Reminder / Google Tasks）。
範本素材 還沒開會？先用這份 500 字逐字稿跑一次，感受萃取結構的威力。
原始素材 · 專案週會逐字稿（約 520 字）貼進 Prompt 最下方
【某課程上線專案會議 · 04/17】 A 主管：距離上線 05/06 剩 19 天，今天確認三件事。第一，招生頁文案 B 專員跟我對到第二版，主視覺還在等設計師，預估 04/22 收到。第二，報名表 C 專員已建好，金流測試還沒跑，D 專員這邊 04/23 協助 end-to-end 測一次。第三，課程講義 E 組長進度如何？ E 組長：講義 28 單元已完成 22 單元，剩下 6 個單元需要補演練素材，這週內會完成，04/24 交稿給設計師排版。 D 專員：金流我週三（04/23）可以測，但需要 C 專員先建一筆 1 元測試商品。 C 專員：沒問題，明天建好。另外問一下：04/30 前要不要發預告貼文？ A 主管：要，04/28 前社群同仁給我初稿。確認三個待辦：04/22 主視覺進場、04/23 金流測試、04/24 講義交稿。我週五（04/19）會跟行銷同步預告貼文節奏。下次 04/24 同一時間追進度。 B 專員：我想確認一下——定價有沒有要最後再校一次？上次會議提過早鳥折扣要不要延續。 A 主管：這個我還沒決定，下週一（04/21）前給答覆。
範例 Prompt · 逐字稿 → 結構化待辦可直接貼 ChatGPT / Claude
以下是一場會議的逐字稿，請： 1. 決議事項（每項一行） 2. 待辦事項，格式：【負責人】任務內容 → 截止日期 3. 待確認 / 下次討論的問題 4. 100 字內會議摘要，可直接轉寄給未出席者 規則： - 待辦必須有負責人與明確日期，沒寫到的日期要標「未指定」而不是猜 - 摘要第一句先寫「本次會議主要處理 X 件事」 - 如果某欄位沒有資訊，寫「本次無」 - 保留原始逐字稿的具體數字與日期 逐字稿： [貼上上方原始素材]
預期輸出片段（供對照）
【決議】05/06 上線日期確認不變；預告貼文確定要做
【待辦】
・【B 專員】主視覺進場確認 → 04/22
・【C 專員】建立 1 元測試商品 → 04/18
・【D 專員】金流 end-to-end 測試 → 04/23
・【E 組長】講義 6 單元補完並交稿 → 04/24
・【社群同仁】預告貼文初稿 → 04/28
【待確認】早鳥折扣是否延續（A 主管 04/21 前答覆）
【摘要】本次會議主要處理課程上線前三項里程碑確認……

(03)

週報的快速生成框架

週報的難點不在「寫不出來」，而在「每次都要重新回憶這週做了什麼」。解決方法是建立一個「週報 Prompt 模板」，每週只需填入本週的亮點和數字，讓 AI 負責組裝成可讀的報告格式。

週報 結構化週報生成模板
你是我的週報撰寫助理。請根據以下原始資料，生成本週工作週報。 格式要求： - 標題：[週次] 工作週報 - 本週完成：條列，每項 20 字以內 - 本週亮點：1-2 項有成果感的事，可附數字 - 未完成/延後：列出原因 - 下週計畫：3-5 項 - 需要支援：如有則列出，無則略去 語氣：正式、具體、有自信。避免「持續推進」「積極跟進」這類模糊說法。 本週原始筆記： [貼上你的速記、完成清單或關鍵數字]
建議：把這個 Prompt 存成你的 System Prompt 或 Claude Project 指令，每週五花 5 分鐘填入速記，即可生成週報。
迷你練習 3 約 10 分鐘
建立你自己的「週報生成器」

把本節的週報 Prompt 客製化成符合你職位的版本，儲存成 Claude Project 或 ChatGPT Custom Instruction，讓下一次寫週報只需要貼速記就能完成。

1修改本節 Prompt：加入你職位常用的欄位（如「客戶進展」「專案里程碑」）。
2儲存成 Claude Project、ChatGPT GPT 或 Gemini Gem，命名為「週報生成器」。
3用本週速記跑一次，對照以往自己寫的週報，看哪個可讀性高。
範本素材 沒速記可用？貼這份「行銷企劃週報原料」跑一次，體驗客製化 System Prompt 差異。
原始素材 · 本週速記（約 180 字）貼進 Prompt 最下方
【本週 04/14–04/18 速記 · 某行銷企劃】 完成： - 5 月新品上市提案過會，預算 180 萬，ROAS 目標 3.5 - 4 月社群貼文 8 篇完成，平均互動率 4.2%（比 Q1 的 3.1% 提升） - KOL 合作名單初篩 12 人，已約訪 5 位 延後： - 官網改版提案因設計師請假延至 04/25 - Email 自動化流程待 IT 串接，預計 04/30 亮點：某合作 KOL 單篇帶來 230 張訂單 下週重點：確認 5 月預算分配、完成 KOL 約訪、官網改版提案。 需要支援：希望業務部派一位夥伴協助 KOL 名單評估。
範例 Prompt · 行銷企劃週報生成器（存成 Claude Project）可直接貼 ChatGPT / Claude
你是一位協助行銷企劃撰寫週報的助理，請根據下方速記，生成一份本週工作週報。 輸出格式： - 標題：[W15] 行銷企劃週報 · {日期區間} - 本週完成（4–6 條，每條 25 字內，附具體數字） - 本週亮點（1–2 項，一定要有可量化成果） - 延後 / 風險（列原因與補救時間） - 下週計畫（3–5 項，動詞開頭） - 需要支援（列部門與具體事項，無則寫「本週無」） 語氣規則： - 正式、具體、有自信 - 禁用「持續推進」「積極跟進」「密切關注」「多管齊下」 - 每一項都要能回答「做完了沒？」而不是「有沒有在做？」 - 數字一律具體（寫 4.2% 而不是「有提升」） 本週原始速記： [貼上上方原始素材]
預期輸出片段（供對照）
[W15] 行銷企劃週報 · 04/14–04/18
本週完成：
・5 月新品上市提案過會，預算 180 萬（ROAS 目標 3.5）
・4 月社群 8 篇貼文完成，互動率 4.2%（↑ Q1 的 3.1%）
・KOL 名單初篩 12 人，完成 5 位約訪
本週亮點：合作 KOL 單篇帶來 230 張訂單
延後：官網改版提案（設計師請假）→ 04/25 送出；Email 自動化 → 04/30
下週計畫：敲定 5 月預算分配、完成 KOL 剩餘 7 位約訪、送出官網改版提案
需要支援：請業務部派一位夥伴協助 KOL 名單評估
成果產出
一個可重複呼叫的週報生成器 + 本週用它產出的週報全文。

文風一致化：讓組織說同一種語言

當一個組織有多個人在撰寫對外文件時，常見的問題是每個人的語氣、措辭、稱謂方式都不一樣。AI 可以作為「文風校準器」，把任何人撰寫的初稿，對齊到同一套組織語言標準。

文件類型需要對齊的語言元素AI 任務
對外公告稱謂（貴賓/各位）、措辭正式度、結尾用語改寫為組織標準格式
客戶郵件開場白、語氣（親切/正式）、行動號召語批次校準語氣一致性
部門通知標題格式、標準說明結構、時間日期格式套用模板並填入資訊
社群貼文品牌語氣、用詞偏好、hashtag 風格根據品牌指南改寫
文風校準 把草稿校準到組織語言標準
以下是一份公告初稿，請按照我們組織的語言標準改寫： 語言標準： - 稱謂：統一使用「各位同仁」 - 語氣：正式但不生硬，有溫度 - 段落：每段不超過 3 句 - 結尾：「如有疑問，請聯絡 [部門]，謝謝。」 - 避免：「茲通知」「敬希遵照」等公文語 初稿： [貼上原始草稿] 請同時提供修改說明，列出哪些地方做了調整和原因。
◆
進階應用：把你組織的「語言標準」整理成 3-5 條規則，存入 System Prompt。之後所有對外文件的草稿，直接交給 AI 校準，不需要每次重新說明標準。這是從「省時間」升級到「提升組織文字品質」的關鍵步驟。
