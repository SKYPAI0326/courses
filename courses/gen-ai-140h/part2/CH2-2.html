PART 2 · 建立文字與知識產能
CH 2-2 · 2.5H

文書工作流：
Email、公告、週報、會議紀錄

把職場最耗時的四種文書任務，從「每次從零開始」變成「有 Prompt 模板的可重複工作流」。一次建好，終身受用。

學習成果 · OUTCOMES
能建立四種 Email 類型（確認/拒絕/請示/跟進）的可複用 Prompt 模板
能設計「會議記錄 → 待辦清單」的標準轉換流程
能用 AI 快速生成組織一致語氣的週報和公告
Section 01

Email 的四種類型模板化

Email 是知識工作中最耗時、重複性最高的文書任務之一。大多數商業 Email 都可以歸類為四種類型，每種類型都有固定的結構和語氣模式。建立這四種 Prompt 模板，你可以把「回一封 Email 的時間」從 20 分鐘壓縮到 3 分鐘。

TYPE 01 · 確認
確認回覆
開場：感謝 + 確認收到 → 確認內容重點 → 說明後續步驟 → 結尾邀請
TYPE 02 · 拒絕
婉拒回覆
感謝提案 → 表達遺憾 → 說明原因（可省略）→ 保留未來合作可能 → 祝福結尾
TYPE 03 · 請示
請示/詢問回覆
說明背景 → 明確提問（不超過3個問題）→ 說明截止時間 → 說明影響範圍
TYPE 04 · 跟進
跟進提醒
引用前次溝通 → 說明目前狀態 → 具體的下一步請求 → 給出決策截止時間
Email · 通用模板 可複用的 Email Prompt 框架
你是我的商業郵件撰寫助理。請幫我撰寫一封【類型：確認/拒絕/請示/跟進】類型的 Email。 收件人：[姓名或職稱] 關係：[上下屬/客戶/合作夥伴/同事] 情境：[說明這封信的背景] 核心訊息：[你最想傳達的主旨] 語氣：[正式/親切/中性] 格式要求： - 主旨：15字以內，清楚說明信件目的 - 正文：200字以內 - 結尾：明確的下一步行動 - 不要使用「敬啟者」或「謹此敬告」等過時套語
把類型、收件人、情境填入，即可快速生成符合語境的 Email 初稿，只需微調個人化細節。
Section 02

會議記錄的轉換邏輯

會議記錄最大的問題不是「怎麼記」，而是「記了沒有用」——因為記錄沒有轉換成行動。AI 最能幫上忙的地方，正是把雜亂的會議記錄或逐字稿，轉換成有負責人、有截止日期的結構化待辦清單。

1
輸入：原始記錄
逐字稿、手寫筆記截圖、或你自己的速記（不需要整理，直接貼入）
→
AI 處理：萃取結構
識別決策、待辦事項、後續問題、值得存檔的資訊
→
輸出：可用格式
結構化待辦清單（負責人 + 事項 + 截止日）+ 決議摘要 + 下次議題
會議記錄 逐字稿 → 結構化待辦
以下是一場會議的逐字稿（或筆記），請： 1. 列出所有決議事項（每項一行） 2. 列出所有待辦事項，格式：【負責人】任務內容 → 截止日期 3. 列出所有「待確認」或「下次討論」的問題 4. 生成一段 100 字以內的會議摘要，可直接轉寄給未出席者 如果某欄位沒有資訊，寫「本次無」。 逐字稿： [貼上逐字稿或筆記]
關鍵設計：強迫 AI 用你設定的格式輸出，讓結果可以直接貼入任務管理工具或轉寄 Email。
Section 03

週報的快速生成框架

週報的難點不在「寫不出來」，而在「每次都要重新回憶這週做了什麼」。解決方法是建立一個「週報 Prompt 模板」，每週只需填入本週的亮點和數字，讓 AI 負責組裝成可讀的報告格式。

週報 結構化週報生成模板
你是我的週報撰寫助理。請根據以下原始資料，生成本週工作週報。 格式要求： - 標題：[週次] 工作週報 - 本週完成：條列，每項 20 字以內 - 本週亮點：1-2 項有成果感的事，可附數字 - 未完成/延後：列出原因 - 下週計畫：3-5 項 - 需要支援：如有則列出，無則略去 語氣：正式、具體、有自信。避免「持續推進」「積極跟進」這類模糊說法。 本週原始筆記： [貼上你的速記、完成清單或關鍵數字]
建議：把這個 Prompt 存成你的 System Prompt 或 Claude Project 指令，每週五花 5 分鐘填入速記，即可生成週報。
Section 04

文風一致化：讓組織說同一種語言

當一個組織有多個人在撰寫對外文件時，常見的問題是每個人的語氣、措辭、稱謂方式都不一樣。AI 可以作為「文風校準器」，把任何人撰寫的初稿，對齊到同一套組織語言標準。

文件類型需要對齊的語言元素AI 任務
對外公告稱謂（貴賓/各位）、措辭正式度、結尾用語改寫為組織標準格式
客戶郵件開場白、語氣（親切/正式）、行動號召語批次校準語氣一致性
部門通知標題格式、標準說明結構、時間日期格式套用模板並填入資訊
社群貼文品牌語氣、用詞偏好、hashtag 風格根據品牌指南改寫
文風校準 把草稿校準到組織語言標準
以下是一份公告初稿，請按照我們組織的語言標準改寫： 語言標準： - 稱謂：統一使用「各位同仁」 - 語氣：正式但不生硬，有溫度 - 段落：每段不超過 3 句 - 結尾：「如有疑問，請聯絡 [部門]，謝謝。」 - 避免：「茲通知」「敬希遵照」等公文語 初稿： [貼上原始草稿] 請同時提供修改說明，列出哪些地方做了調整和原因。
◆
進階應用：把你組織的「語言標準」整理成 3-5 條規則，存入 System Prompt。之後所有對外文件的草稿，直接交給 AI 校準，不需要每次重新說明標準。這是從「省時間」升級到「提升組織文字品質」的關鍵步驟。
