解法參考（先寫自己的版本再展開）
原版的問題： 1. 「讓主管看了覺得合理」是主觀標準，AI 不知道你的主管對請假說明的期待深度 2. 「格式要正式」沒有定義正式的具體元素（是否要有日期、收件人、事由、請假天數、代理人？） 3. 生成出來的可能是「完整範文」而不是「可反覆套用的模板」——沒有 [變數佔位符] 參考改法： 你是職場行政文書專家（角色）。 請產出一份「病假申請書」模板，供員工每次請病假時套用（任務，用途明確）。 模板必須包含以下欄位（佔位符用 [方括號] 標記）： - 申請日期：[YYYY/MM/DD] - 申請人：[姓名]、[部門] - 請假類別：病假 - 請假期間：[起始日] 至 [結束日]，共 [N] 天 - 請假事由：[簡述病況，如「因急性腸胃炎就醫」] - 代理人：[代理人姓名]（如無，填「無」） - 附件：[醫療證明單，如有] 限制：語調正式但不冷硬；全文不超過 150 字；不要加多餘的祝福語。 輸出格式：直接給模板正文，每個欄位一行，不加說明。 關鍵改動：把「格式要正式」→ 具體列出 7 個欄位；用 [佔位符] 標記（讓模板可以直接反覆套用）；加字數上限（確保不會有人因為太長而不用）。

難度 ★★☆ 約 40 分鐘

你是 120 人規模台灣科技公司的 HR 主管。昨天董事會決議下季開始調整差旅與加班費核銷規則，你拿到董事長特助整理的逐字稿摘要，需要今天下班前把這份「內部語言」改寫成「全員看得懂的公告」，貼到公司內部 Slack 與 email。

原始資料｜董事會逐字稿摘要
議題三：差旅與加班費政策調整 決議： 1. 因應 Q2 營運成本壓力，差旅日支由 NT$1,200 調整為 NT$900，住宿實報實銷但單夜上限 NT$2,500（原 NT$3,000）。 2. 加班費申請流程改為線上系統預先申請，事後補單原則上不予受理，特殊情況由部門主管簽核後例外處理。 3. 新制自 2026-05-01 起實施，5 月份前已出差或加班者依舊制辦理。 4. 由人資部發布內部公告，同時上傳新版差旅申請表與 FAQ。 5. 董事長補充：「希望傳達節流的意圖但不要讓同仁覺得被砍福利，重點是讓大家知道公司在做什麼、為什麼做。」
你的任務

案例 02 · 零售小店
花店老闆把 Q1 訂單郵件整理成客戶關懷信
難度 ★★☆ 約 35 分鐘

你是台北民生社區的花店老闆，Q1 累積了 80 多筆訂單。四月母親節檔期開跑前，你想挑出「Q1 訂過兩次以上」「單次消費破 NT$2,000」這兩類客戶，寄一封溫度適中的關懷信提前預告新品與早鳥優惠。你手邊有訂單資料文字版。

原始資料｜訂單摘要（節選三筆）
訂單 #1042｜林雅婷｜2026-01-14｜桌花訂閱 4 週｜NT$4,800｜備註：希望週一送達，不要百合 訂單 #1056｜林雅婷｜2026-02-22｜春節祝賀盆花｜NT$3,500｜備註：送給合作廠商 訂單 #1078｜陳冠宇｜2026-03-03｜求婚捧花｜NT$5,200｜備註：要粉紅系，當日取貨 訂單 #1091｜陳冠宇｜2026-03-28｜情人節補送｜NT$1,800｜備註：同一位女友，想修復形象 （實際資料 80+ 筆，這裡只節選作為 Prompt 範例）

語氣像店家

自我演練 · 受眾改寫自我盲測

文字改寫的盲點是自己對自己稿子過度熟悉，看不出「讀者角度」是否真的差異化。靠三件組強制把自己變成陌生讀者：(1) 限時錄影/截圖留 v1、(2) 24 小時隔離後自評（時間是讓記憶冷卻的關鍵）、(3) 叫 LLM 扮 3 種陌生受眾各給回饋。約 25 分鐘 + 隔日 10 分鐘自評。

自我演練 01 · 受眾改寫自我盲測（三件組）
我寫給誰，24 小時後的自己 + 三個 LLM 陌生人看得出來嗎？
單人作業 當日 25 分鐘 + 隔日 10 分鐘 需錄影/截圖工具 + LLM 帳號

三件組強制讓自己抽離：限時錄影/截圖留 v1 當作品痕跡（不可回頭改）；24 小時隔離後自評（記憶冷卻才看得出問題）；LLM 扮 3 種陌生受眾各給回饋（補真人讀者多元視角）。三件齊備才能逼近「真人盲測」的訊號強度。

主流程 · 5 分鐘
把【v1 受眾改寫】貼 LLM，請扮 1 種陌生受眾給回饋。看是否點到本節核心。
進階選配 · 不計入課程時數（給想加練的學員）

完整三件組訓練：限時錄影 + 24h 隔離自評 + LLM 多角色追問。為轉職前 demo 壓測做準備。

1作品 v1 留檔
準備一段 150–200 字原文（例：自家產品介紹、服務條款、活動說明）。挑兩個差異明顯的受眾（例：50+ 傳統小吃店老闆 vs 25 歲自由接案設計師），限時 12 分鐘各寫一版改寫稿，期間用螢幕錄影或每 3 分鐘截圖一張，產出 v1 鎖檔不再修改。
224h 隔離自評
v1 寫完後關掉檔案、不可回看。隔日（最少 24 小時後）以「陌生讀者」心態重新打開兩版改寫稿，不看你當初記下的受眾設定，倒推「這版在寫給誰？」並寫下判斷依據。和原本設定對答案、找落差。
三件組節奏
1【件 1 · 錄影留檔】開螢幕錄影或截圖計時器，限時產出兩版改寫稿 v1，私下記下受眾設定後鎖檔（檔名加 v1 + 日期）· 12 分鐘
2【件 3 · LLM 多角色】把兩版改寫稿（不附受眾設定）貼給 LLM，叫它分別扮 3 種陌生受眾人格（例：50+ 小吃店老闆、25 歲自由接案設計師、第三組任選），各倒推「我覺得這版在對誰說話」＋ 2 個判斷依據，並用下方 4 指標各自打分· 8 分鐘
3對 LLM 三組回饋的答案，記錄哪些指標被一致認出、哪些指標被誤判· 5 分鐘
4【件 2 · 24h 隔離】關檔。隔日重開檔，以「陌生讀者」心態再次盲測兩版，倒推受眾後對答案· 隔日 5 分鐘
5整合三件回饋（錄影回看 + 24h 自評 + LLM 三角色），改 v2 並截圖留存 v1 vs v2 對照· 隔日 5 分鐘
自評指標（1-5 分，三組 LLM + 24h 自評共四份打分）
受眾可辨陌生讀者能在 1 分鐘內倒推出合理受眾輪廓
語氣到位兩版語氣差異明顯，不是只換幾個詞
舉例切題引用的情境／例子貼近該受眾日常
重點重排兩版強調的點不同，不是同順序換字

情境 A · 人資行政 可複製 Prompt + 預期模板
情境 A｜請假單 / 出差申請統一模板
使用情境：你是部門助理，主管要你把「請假單」與「出差申請」合併成一份通用表單樣板，貼到 Notion 與內部 HR 系統都能用。 必填欄位：申請人、部門、假別（事假/病假/特休/出差）、起訖日期時間、天數、代理人、工作交接、附件（診斷證明 / 出差行程），請假理由只要一行摘要。 觸發時機：部門成員每次請假或出差都要填，一年約 100 次以上；模板要能塞進 Google Form 或直接貼 email。
Prompt 起手式（可複製）
你是 HR 行政模板設計師。請幫我產出一份「請假／出差申請」通用表單模板，符合以下規則： 1. 欄位必須包含：申請人姓名、員工編號、部門、申請類型（事假／病假／特休／出差）、起訖日期與時間、請假／出差天數、代理人、工作交接重點（3 行內）、附件清單、理由摘要（一行）、主管簽核欄。 2. 假別用單選，出差需多一欄「目的地 + 預估費用」。 3. 語氣中性、正式精準、用詞不堆疊形容詞，不使用「敬啟」「懇請」等過度敬語。 4. 輸出格式：先給一份可直接貼 Google Form 的欄位清單（每行一個欄位、標示必填／選填），再給一份純文字版本可貼 email。 5. 最後附一段 30 字內的填寫提示，給第一次填的同事看。
預期模板產出
— Google Form 欄位清單：10 個欄位，前 6 必填，代理人與附件選填，出差自動跳出目的地欄位。
— 純文字 email 版：以「申請類型｜日期｜天數｜代理人｜附件」開頭，後接 3 行交接摘要，結尾留主管簽核位。
— 填寫提示：「請於出發／請假前 3 個工作日提交，附件可用 Google Drive 連結。」

情境 B｜系統維護 / 活動通知 / 政策更新三合一公告模板
使用情境：你是產品營運，每個月要發 3–5 則對外公告，包含：系統維護、新活動開跑、條款政策更新。過去每次都重新想開頭，風格參差。 必填欄位：公告類型、標題、生效日、影響範圍、用戶要做什麼、客服聯絡方式。 觸發時機：每次要貼網站跑馬燈、官方社群、email 電子報時使用。
Prompt 起手式（可複製）
你是對外溝通模板設計師。請幫我做一份「對外公告通用模板」，需支援 3 種類型：系統維護、活動通知、政策更新。規格如下： 1. 結構：標題（15 字內）／一句話重點（告訴用戶關鍵影響）／細節說明（分點，3–5 點）／用戶行動指引（條列）／聯絡方式。 2. 語氣：正式但不冷漠，像「已有信任基礎的同事」，避免「敬啟者」「謹此公告」這類距離感太強的字眼，也不要出現「親愛的用戶」。 3. 三種類型各出一個填空版骨架＋一個填好範例，範例要貼近真實情境（例：系統維護用「週三 02:00–04:00 帳務系統升級」）。 4. 每則公告結尾固定一句致歉或致謝（維護致歉、活動致謝、政策告知感謝配合）。 5. 最後列出「這份模板哪些地方要換字、哪些地方不要動」的說明，3 行內。
預期模板產出
— 系統維護版：「標題／預計影響時段／受影響服務／期間替代方案／聯絡信箱」5 段結構，語氣致歉但不卑微。
— 活動通知版：以「新服務一句話介紹」開頭，3 點細節＋行動 CTA，結尾致謝。
— 政策更新版：第一段先講「對你的影響」而非「條款修改」，最後列明新舊版對照連結與生效日。
情境 C · 內部管理 可複製 Prompt + 預期模板
情境 C｜內部週報模板（上週成果 / 本週計劃 / 需協助事項）
使用情境：你是工程團隊的 Team Lead，每週五要繳一份給部門主管看的團隊週報；主管希望能在 2 分鐘掃完抓到三件事：做了什麼、下週要做什麼、我們卡在哪。 必填欄位：團隊名稱、週期、上週完成、本週計畫、需協助、風險預警。 觸發時機：每週五 16:00 前送出，送 email 也貼 Notion 頁面。
Prompt 起手式（可複製）
你是組織管理週報模板設計師。請幫我設計一份「工程團隊週報」模板，規格如下： 1. 固定區塊：團隊名稱、週期（MM/DD–MM/DD）、上週成果、本週計畫、需協助事項、風險預警。 2. 每個區塊的內容規則： - 上週成果：3–5 點，每點必須「動詞 + 具體產出 + 量化指標」（例：上線 v2.3 登入流程、轉換率由 34% 提升至 41%）。 - 本週計畫：3–5 點，每點必須「要做什麼 + 誰負責 + 預計完成日」，禁止只寫「推進 A 專案」這種模糊描述。 - 需協助：最多 3 點，每點寫清楚「需要誰協助什麼、何時前要」。 - 風險預警：0–2 點，寫「風險描述 + 預計影響 + 當前緩解方案」。 3. 語氣中性直接，不用「辛苦大家」「請多指教」這類客套。 4. 最後附一個 Markdown 版（直接貼 Notion）＋純文字版（貼 email）。
預期模板產出
— 上週成果：每點 20–30 字，都帶量化（百分比、數字、完成項目）。
— 本週計畫：每點鎖定「承辦人 + 截止日」，主管掃過就知道找誰對齊。
— 需協助：不超過 3 點，避免變成抱怨清單；風險預警欄空著也沒關係（「本週無重大風險」）。
情境 D · 客戶溝通 可複製 Prompt + 預期模板
情境 D｜客戶道歉信（正式 / 標準 / 誠懇三語氣切換）
使用情境：你是客服主管，處理客戶投訴要寄道歉信，依客戶關係與事件嚴重度在三種語氣間切換：對大客戶用「正式」（有法務色彩）、對一般 B2B 用「標準」、對長期合作的熟客用「誠懇」（帶人味）。 必填欄位：事件描述、影響範圍、補救措施、預計時程、客戶身分分級。 觸發時機：客戶投訴處理後 24 小時內，主動回覆。
Prompt 起手式（可複製）
你是 B2B 客戶溝通模板設計師。請幫我為「客戶道歉信」設計一組三語氣模板，規則如下： 1. 結構固定：主旨／開場致歉／事件描述（客觀陳述，不卸責）／影響評估／補救措施（分點）／預計時程／聯絡窗口／結尾。 2. 三種語氣變體，差別只在「用字與開場」，結構與資訊量一致： - 正式（Formal）：用於大客戶或合約糾紛，語氣有法務感但不冷，主旨「關於 XX 事件之說明與後續處理方案」。 - 標準（Standard）：用於一般 B2B，語氣正式精準、親切直接，主旨「針對 XX 事件的說明與補救」。 - 誠懇（Warm）：用於長期熟客，開場可用「昨天的事，我第一時間想親自跟你說明」，仍維持事實陳述，不過度自責。 3. 禁止：不用「萬分抱歉」「給客戶帶來極大困擾」這類罐頭致歉；不預設對方情緒。 4. 三份範例共用同一個虛擬事件（例：SaaS 系統當機 3 小時影響客戶結帳），方便對照差異。 5. 最後加一段「三語氣選用指引」，告訴使用者看什麼條件挑哪一版，3 行內。
預期模板產出
— 三版同事件：同一次當機，正式版強調服務水準協議（SLA）與補償機制；標準版聚焦「影響 + 修復 + 下一步」；誠懇版加一段「我知道你們昨天忙到晚上 10 點，這件事我們會認真處理」。
— 選用指引：列 3 行判斷條件（合約層級／客戶關係長度／事件法務敏感度）。
— 檢核提醒：三版都不出現「造成困擾」「敬請見諒」這類套話，結尾都留直線聯絡人與手機。
延伸挑戰 · 模板庫升級

把工具頁變成你自己的模板資產

跑完四個情境只是起點。真正的產能來自「把這些 Prompt 收進你部門的知識庫，每次要用就複製、微調、產出」。下面三個延伸挑戰各抓一個角度，讓你把這次學到的技巧變成可重複使用的資產。估時 20–30 分鐘。

延伸挑戰｜選一個開始做
  • 🎯 建立你部門的 5 個最常用模板：盤點你這週寫過的行政文書，挑出出現頻率最高的 5 種（請假、週報、公告、會議邀請、客戶回信…），各用本頁工具生一版 Prompt，存成 Notion 資料庫欄位：「模板名稱 / 使用情境 / Prompt 起手式 / 預期輸出 / 最後更新日」。下週上班直接從資料庫開新頁生內容。
  • 🎯 模板版本控制：挑其中一個高頻模板，一次寫出「正式／半正式／口語」三版，整理成一組放在同一個 Notion 頁面。用「上次寫的是哪個客戶、當時對方怎麼回」當備註，下次選版本時有依據可查。
  • 🎯 自動填入實作：把一份真實數據（例：這個月的業績摘要、上週的客戶名單、下個月的排班表）交給 AI，請它用你寫好的模板「自動填入」欄位，產出一份可直接寄出的正式文件。重點不在 AI 寫得多漂亮，而在你能把「模板 + 數據 + AI」串成一條可重複跑的生產線。
小測驗 · 5 題檢查你學到了什麼
模板化思維 vs 單次生成
1用 AI 做「模板」和「單次生成」最關鍵的差別是什麼？
展開解析
正解 B。模板的價值在「重用性」：結構一旦定下，後續只要替換欄位就能快速產出，節省決策成本。單次生成是針對當下情境的一次性產物，很難直接複用。
2設計「請假單」模板時，下列哪一項「最不應該」寫進固定欄位？
展開解析
正解 C。請假理由屬於個人隱私，模板只需「一行摘要」讓主管有判斷依據即可，要求寫長理由會讓員工抗拒填寫，也暴露不必要資訊。
3對外公告（系統維護／活動通知／政策更新）的語氣應該？
展開解析
正解 C。對外公告的受眾是一般用戶，過度敬語讓人有距離感、完全冷漠像機器人通知、太輕佻又失去正式感。「正式但有溫度」是最通用的基準。
4週報「本週計畫」要寫到什麼程度，主管才能掃過就知道進度可控？
展開解析
正解 C。本週計畫要可執行、可追蹤，三要素缺一不可：具體里程碑（不是大方向）＋承辦人（主管知道找誰）＋時間（下週五可以回來對帳）。
5發現 AI 每次生的模板都有小差異（欄位順序、用字偶爾換），怎麼把輸出「固定」下來？
展開解析
正解 B。固定輸出有兩個有效手段：(1) 把結構與用字規則寫進 system prompt 當背景設定；(2) 附上 1–2 份完整輸出範例當 few-shot 錨點，AI 會跟著範例格式走，變動幅度大幅收斂。
